# <a name="setting-up-the-components-of-a-job"></a>การตั้งค่าส่วนประกอบของงาน

หัวข้อนี้อธิบายถึงองค์ประกอบแนวคิดสามารถรวมงาน และแสดงตัวอย่างของวิธีการที่คุณสามารถใช้องค์ประกอบเหล่านั้นในองค์กรของคุณ 

ก่อนที่คุณสามารถสร้างงาน คุณต้องตั้งค่าข้อมูลบางข้อมูลอ้างอิง คุณสามารถสร้างงานที่มีชื่อเท่านั้น อย่างไรก็ตาม โดยการรวมข้อมูลเพิ่มเติม เช่นตำแหน่งงาน คุณระบุค่าเริ่มต้นสำหรับตำแหน่งที่กำหนดให้แก่งาน นอกจากนี้ ที่คุณป้อนข้อมูลบางอย่างสามารถใช้เพื่อกรองข้อมูลแผนค่าตอบแทนสำหรับงานที่ระบุ ถ้าคุณต้องการตั้งค่าสิทธิ์ที่คุณสามารถใช้เพื่อกรองข้อมูลแผนค่าตอบแทนกับงานเฉพาะ คุณควรตั้งค่าฟังก์ชันงานและชนิดงานก่อนที่คุณเซ็ตอัพงาน โดยมีค่าเริ่มต้นเหล่านี้พร้อมใช้งาน คุณจะช่วยประหยัดเวลาเมื่อคุณเพิ่มตำแหน่งงาน 

บางงานมีรายละเอียดของ เช่นตำแหน่งงาน ชนิด และ ฟังก์ชัน วันมีผลบังคับใช้ ถ้าคุณสร้างงานวันนี้ แต่ไม่ต้องเพิ่มรายละเอียดเหล่านี้จนกว่าจะมีรุ่นที่ใหม่กว่า และคุณแล้วดูงาน ณวันสร้าง รายละเอียดเหล่านี้จะไม่ปรากฏขึ้น ดังนั้น คุณควรสร้างบางส่วนของข้อมูลอ้างอิงนี้ก่อนที่คุณต้องใช้ วิธีนี้ คุณสามารถเพิ่มข้อมูลไปยังงานใหม่เมื่อคุณสร้าง

## <a name="job-titles"></a>ตำแหน่งงาน
ก่อนที่คุณจะสร้างงาน คุณต้องตั้งชื่อสำหรับงานเหล่านั้น  โดยตำแหน่งจะสืบทอดตำแหน่งงานจากงานที่เชื่อมโยงกับตำแหน่งนั้น 

รักษาตำแหน่งงานโดยใช้การ**เกม**หน้า ซึ่งคุณสามารถเปิดได้ โดยใช้ฟังก์ชันการค้นหา บนการ ** เรื่อง ** หน้า ป้อนชื่อเรื่องที่คุณวางแผนที่จะใช้สำหรับงานของคุณ

## <a name="job-types"></a>ชนิดงาน
คุณสามารถใช้ชนิดงานในการจัดกลุ่มงานที่คล้ายกันเป็นประเภท ชนิดงานไม่ถูกต้อง อย่างไรก็ตาม ถ้าคุณวางแผนที่จะใช้ชนิดงานเมื่อคุณตั้งค่ากฎการมีสิทธิ์สำหรับการจัดการค่าตอบแทน คุณควรตั้งค่าชนิดงานก่อนที่คุณตั้งค่างาน  บางตัวอย่างของชนิดงาน full-time และ part-time หรือเงินเดือนและรายชั่วโมงการชำระค่าจ้าง คุณสามารถรักษาชนิดของงาน โดยใช้การ**ชนิดของงาน**หน้า ในการ**ชนิดของงาน**หน้า ป้อนชื่อและคำอธิบายโดยย่อสำหรับชนิดงาน ในการ**ยกเว้นสถานะ**ฟิลด์ เลือกหนึ่งในอ็อพชันต่อไปนี้เพื่อบ่งชี้สถานะยกเว้นยุติธรรมแรงงานมาตรฐาน Act (FLSA) ของงานที่มีชนิดงานนี้:

-   **ยกเว้น**– งานที่ได้รับยกเว้นค่าล่วงเวลาภายใต้ FLSA
-   **ยกเว้นที่ไม่ใช่**– งานที่ไม่ได้รับยกเว้นจากการทำงานล่วงเวลาภายใต้ FLSA
-   **ไม่สามารถใช้**– ความครอบคลุม FLSA ไม่สามารถใช้

## <a name="job-functions"></a>ฟังก์ชันงาน
Junctions งานอธิบายประเภทฟังก์ชันระดับสูง และระดับสูงการแบ่งแยกหน้าที่ที่เกี่ยวข้อง ฟังก์ชันงานไม่ถูกต้อง คุณสามารถใช้ฟังก์ชันงาน ร่วมกับชนิดงาน เพื่อกรองข้อมูลแผนค่าตอบแทนสำหรับงานที่ระบุ คุณเชื่อมโยงฟังก์ชันงานและชนิดงานกับแผนค่าตอบแทน โดยการตั้งค่ากฎการมีสิทธิ์ในการ**กฎการมีคุณสมบัติเหมาะสม**หน้า จากนั้นคุณสามารถแนบชุดของระดับกับแผนค่าตอบแทนที่ใช้กับชุดข้อมูลเฉพาะของชนิดงานและฟังก์ชันงานที่คุณได้กำหนดไว้โดยผ่านกฎการมีคุณสมบัติเหมาะสม (คุณลักษณะเหล่านี้นำไปใช้กับทั้งแผนค่าตอบแทนคงที่และแผนค่าตอบแทนผันแปร) อย่างไรก็ตาม ถ้าคุณวางแผนที่จะใช้งานฟังก์ชันเมื่อคุณตั้งค่ากฎการมีคุณสมบัติเหมาะสมสำหรับการจัดการค่าตอบแทน คุณควรตั้งค่าฟังก์ชันงานก่อนที่คุณเซ็ตอัพงาน ตารางต่อไปนี้แสดงตัวอย่างของฟังก์ชันงาน

| งาน           | ฟังก์ชันงาน      |
|---------------|-------------------|
| ผู้จัดการฝ่ายขาย | ผู้จัดการระดับกลาง |
| นักบัญชี    | ผู้เชี่ยวชาญ     |

คุณรักษาฟังก์ชันงาน โดยใช้การ**งานฟังก์ชัน**หน้า ในการ**งานฟังก์ชัน**หน้า ป้อนรหัสและคำอธิบายโดยย่อสำหรับฟังก์ชันงาน

## <a name="job-tasks"></a>การปฏิบัติงาน

คุณรักษาการปฏิบัติงาน โดยใช้การ**งาน**หน้า ในการ**งาน**หน้า ป้อนชื่อและคำอธิบายสำหรับรายการงาน ในการ**หมายเหตุ**ฟิลด์ คุณสามารถเลือกที่จะป้อนข้อมูลเพิ่มเติม บันทึกย่อถูกปรับปรุงสำหรับงานหนึ่ง ๆ โดยไม่เปลี่ยนแปลงบันทึกย่อที่คุณป้อนที่นี่

## <a name="areas-of-responsibility"></a>ขอบเขตความรับผิดชอบ
คุณสามารถใช้พื้นที่ของความรับผิดชอบเพื่อระบุหน้าที่การทำงาน กระบวนการ และผลิตภัณฑ์ที่ผู้ปฏิบัติงานในตำแหน่งงานสำหรับงานที่รับผิดชอบ ตัวอย่างเช่น สำหรับงานที่มีชื่อว่า "นักบัญชี" หนึ่งพื้นที่ของความรับผิดชอบอาจเป็น "การเงินรายงานสำหรับสำนักกฎหมาย a.ผลิตภัณฑ์" คุณรักษาขอบเขตความรับผิดชอบ โดยใช้การ**พื้นที่ของความรับผิดชอบ**หน้า ซึ่งคุณสามารถค้นหาได้ โดยใช้ฟังก์ชันการค้นหา ในการ**พื้นที่ของความรับผิดชอบ**หน้า ป้อนชื่อและคำอธิบายสำหรับความรับผิดชอบ ในการ**หมายเหตุ**ฟิลด์ คุณสามารถเลือกที่จะป้อนข้อมูลเพิ่มเติม บันทึกย่อถูกปรับปรุงสำหรับงานหนึ่ง ๆ โดยไม่เปลี่ยนแปลงบันทึกย่อที่คุณป้อนที่นี่
