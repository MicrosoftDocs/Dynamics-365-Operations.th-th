# <a name="activity-based-subcontracting"></a>ใช้กิจกรรมที่รับเหมารายย่อย

หัวข้อนี้อธิบาย รายละเอียด วิธีการใช้กิจกรรมที่รับเหมารายย่อยในขั้นตอนการผลิตสำหรับการผลิตแบบ lean ด้วย

ใน Microsoft Dynamics 365 สำหรับการดำเนินงาน มีอยู่ด้วยกันสองวิธีสำหรับการรับเหมารายย่อย: ใบสั่งการผลิตและการผลิตแบบ lean ในวิธีการผลิตแบบ lean งานที่รับเหมารายย่อยแบบจำลองเป็นบริการที่เกี่ยวข้องกับกิจกรรมของขั้นตอนการผลิต ชนิดพิเศษของชนิดกลุ่มต้นทุนที่ชื่อ**การเอาท์ซอร์สโดยตรง**มีการแนะนำ และบริการรับเหมารายย่อยไม่ได้เป็นส่วนหนึ่งของรายการวัสดุและส่วนประกอบ (BOM) การลงบัญชีต้นทุนของงานที่รับเหมารายย่อยทั้งหมดได้รับการรวมเข้าในโซลูชันที่มีการคำนวณต้นทุนสำหรับการผลิตแบบ lean

## <a name="production-flows-that-involve-subcontractors"></a>ขั้นตอนการผลิตที่เกี่ยวข้องกับเหมา
หลักการเบื้องต้นของขั้นตอนการผลิตไม่เปลี่ยนแปลงเมื่อมีการรับเหมารายย่อยกิจกรรม วัสดุยังคงลื่นไหลระหว่างที่ตั้ง กิจกรรมของกระบวนการแปลงวัตถุดิบเป็นผลิตภัณฑ์ และกิจกรรมการโอนย้ายวัสดุหรือผลิตภัณฑ์จากสถานหนึ่งไปยังอีก คุณสามารถจำลองสถานที่เก็บ และเซลล์เป็นผู้จัดจำหน่ายจัดการการทำงาน โดยการกำหนดให้กับบัญชีผู้ขาย ไปยังคลังสินค้า หรือทรัพยากรเป็นของกลุ่มทรัพยากร  

ขึ้นอยู่กับความสามารถเหล่านี้ การผลิตแบบ lean ไม่จำเป็นต้องมีคุณลักษณะเฉพาะเพื่อสนับสนุนการทำงานของวัตถุดิบและผลิตภัณฑ์ สถานการณ์สมมติเป็นไปได้ทั้งหมดที่เกี่ยวข้องกับผู้จัดจำหน่ายเป็นผู้ให้บริการของการบริการขนส่งหรือการผลิตสามารถสร้างแบบจำลอง ที่ขึ้นอยู่กับสถาปัตยกรรมของขั้นตอนการผลิตและกิจกรรม  

ตัวอย่างเช่น ผู้รับเหมาย่อยที่ทำงานออกจากซุปเปอร์มาร์เก็ตที่อยู่ที่ผู้รับเหมาย่อยที่ เมื่อหน่วยจัดการวัสดุเป็นว่างที่ผู้รับเหมาย่อยที่ บัตรคัมบังที่จะถูกส่งกลับไปยังเซลล์แอสเซมบลีร่วมกับการจัดส่งถัดไป ซุปเปอร์มาร์เก็ตที่ที่ผู้รับเหมาย่อยที่แล้วได้มีเติม โอนย้ายไปยัง และ จากผู้รับเหมาย่อยที่สามารถสร้างแบบจำลองเป็นกิจกรรมการโอนย้ายที่ชัดเจนเพื่อสนับสนุนการเบิกสินค้าและกระบวนการจัดส่ง ถ้าการลงทะเบียนอย่างชัดเจนไม่จำเป็นต้องใช้เพื่อสนับสนุนการขนส่งที่มีอยู่จริง กิจกรรมการโอนย้ายสามารถถูกตัดออก  

ผู้รับเหมาย่อยที่สามารถใช้เพื่อสร้างสมดุลกำลังการผลิตโดยรวมของขั้นตอนการผลิต ตัวอย่างเช่น คือจำลองขั้นตอนการผลิต โดยใช้กฎคัมบังที่จัดกำหนดการไว้ ผู้วางแผนการใช้บอร์ดเพื่อจัดกำหนดการ และโหลดระดับการจัดกำหนดการคัมบังทั้งสองเซลล์ทำงานตามความต้องการ ผู้วางแผนการตรวจสอบกำหนดการจัดหาวัสดุรวมสำหรับซุปเปอร์มาร์เก็ตที่นอกจากนี้ในการ**กำหนดการจัดหาวัสดุ**หน้า เหมาหลายสามารถเป็นแบบจำลองในขั้นตอนการผลิตหนึ่ง หรือหลาย และอาจมีหลายกฎคัมบังที่สามารถใช้เพื่อจัดหาผลิตภัณฑ์เดียวกันกับตำแหน่งที่ตั้งเดียวกันผ่านกิจกรรมต่าง ๆ จากนักวางแผนสามารถแปลงคัมบังเป็นกฎคัมบังสำรองเพื่อจัดกำหนดการคัมบังที่ถูกสร้างขึ้นสำหรับการผลิตภายในกับกระบวนการอื่น อยู่ แท้ที่จริง ลักษณะรับเหมารายย่อยของเซลล์ทำงานได้ไม่มีผลกระทบในขั้นตอนการผลิต หลักการทำงานเดียวกันใช้ สำหรับงานภายในขนานสองเซลล์ หรือสองเซลล์ที่รับเหมารายย่อย   

เช่นเดียวกับกิจกรรมใด ๆ ที่อื่น ๆ ในขั้นตอนการผลิต กิจกรรมที่รับเหมารายย่อยสามารถใช้อุปสงค์ และอุปทานที่ไม่ใช่คงคลัง คงคลัง (ทำงานในกระบวนการ\[WIP\]), และวัสดุกึ่งสำเร็จและผลิตภัณฑ์ได้ ในกรณีที่ทั้งหมด กระบวนการสำหรับการวางแผน และดำเนินการกิจกรรมที่รับเหมารายย่อยจะเหมือนกัน นอกจากนี้ เหล่านี้ประมวลผลเหมือนกับกระบวนการสำหรับการทำงานภายใน

## <a name="purchase-process-for-subcontracted-activities-services"></a>กระบวนการซื้อสำหรับกิจกรรมที่รับเหมารายย่อย (บริการ)
กระบวนการซื้อสำหรับกิจกรรมที่รับเหมารายย่อยจะขึ้นอยู่กับขั้นตอนวัสดุทางกายภาพที่มีการลงทะเบียนความคืบหน้าของงานคัมบัง เช่น เริ่มต้น หรือเสร็จสมบูรณ์ เงินไหล ตัวอย่าง ต้นทุนของงานที่รับเหมารายย่อย มีขั้นตอนการสำรองที่ทำตามขั้นตอนมีอยู่จริง ในเวลาเดียวกัน กระบวนการซื้อคือ กระบวนการขึ้นอยู่กับที่ให้สำหรับการปรับปรุงเอกสารการซื้อในทุกขั้นตอนด้วยตนเอง นี่คือกระบวนการซื้อสำหรับกิจกรรมที่รับเหมารายย่อย:

1.  สร้างข้อตกลงการซื้อ ข้อตกลงการซื้อถูกสร้างขึ้นสำหรับการบริการ และเชื่อมโยงกับกิจกรรมของขั้นตอนการผลิต
2.  สร้างใบสั่งซื้อ สามารถสร้างใบสั่งซื้อย่อยสำหรับการบริการ โดยยึดตามงานคัมบังที่จัดกำหนดการไว้ งานสำหรับการบริการเดียวกันสามารถถูกจัดกลุ่มบรรทัดใบสั่งซื้อ ขึ้นวัน สัปดาห์ เดือน คุณสามารถสร้างบรรทัดใบสั่งซื้อหลังจากที่งานคัมบังสร้างขึ้น แม้ว่าคุณสามารถสร้างบรรทัดใบสั่งซื้อหลังจากข้อเท็จจริง โดยปกติแล้วจะมีเลือกตัวเลือกนี้ถ้าผู้รับเหมาย่อยที่ให้บริการโดยไม่ต้องแจ้งให้ทราบเพิ่มเติม ตามบัตรคัมบังหรือคัมบังที่ผู้รับเหมาย่อยที่ได้รับ ในกรณีนี้ ความแตกต่างระหว่างใบสั่งซื้อและใบแจ้งหนี้สามารถถูกย่อให้เล็กสุด
3.  สร้างบัตรคัมบัง วัสดุ และรายการเบิกสินค้าเพื่อส่งไปยังผู้รับเหมาย่อยเพื่อจัดเตรียมสำหรับการประมวลผล ตามโมเดลรายละเอียดของขั้นตอนการผลิต การเตรียมเสร็จสิ้นบนบอร์ดคัมบังสำหรับกิจกรรมของกระบวนการ โดยใช้รายการเบิกสินค้าและฟังก์ชันการจัดเตรียม อีกวิธีหนึ่งคือ การเตรียมเสร็จสิ้นบนบอร์ดคัมบังสำหรับงานการโอนย้าย โดยใช้รายการเบิกสินค้า และเริ่มต้น หรือเสร็จสมบูรณ์ สำหรับวัสดุคงคลัง กระบวนการทั้งสองสามารถได้รับการสนับสนุน โดย WMS-เบิกสินค้าและกระบวนการจัดส่ง คุณสามารถสร้างใบตราส่งตามความต้องการ
4.  สร้างหน่วยจัดการวัสดุคัมบังและบัตรคัมบัง หลังจากประมวลผล บัตรจะถูกส่งกลับจากการรับเหมารายย่อย โดยปกติ บัตรรวมกับบันทึกการจัดส่งที่ระบุวัสดุทางกายภาพที่มีการจัดส่ง การอ้างอิงถึงบริการให้มาไม่จำเป็น มีการลงทะเบียนการมาถึงของวัสดุหรือผลิตภัณฑ์ที่ลูกค้าบนบอร์ดคัมบัง ขึ้นอยู่กับบัตรคัมบัง (บอร์ดคัมบังสำหรับงานการโอนย้ายหรือบอร์ดคัมบังสำหรับกิจกรรมของกระบวนการการผลิต ขึ้นอยู่กับกิจกรรมที่สร้างแบบจำลอง)
5.  สร้างการรับคำแนะนำ การรับคำแนะนำสามารถใช้เพื่อแทนส่งเอกสารบันทึกการจัดสำหรับบริการได้รับได้ คำแนะนำสำหรับการรับสินค้าสามารถสร้างขึ้นโดยยึดตามงานคัมบังที่เสร็จสมบูรณ์สำหรับกิจกรรมที่รับเหมารายย่อยสำหรับรอบระยะเวลาที่เลือก สำหรับแต่ละใบรับงาน คำแนะนำสำหรับสร้างไว้สำหรับบรรทัดใบสั่งซื้อที่เกี่ยวข้อง สามารถพิมพ์ และส่งไปยังผู้รับเหมาย่อยที่เป็นการยืนยันใบรับสินค้าใบรับคำแนะนำ
6.  สร้างใบแจ้งหนี้

กระบวนการสิ้นสุดเมื่อผู้รับเหมาย่อยที่จะออกใบแจ้งหนี้สำหรับรอบระยะเวลา การจับคู่ใบแจ้งหนี้ที่เสร็จสิ้นแล้วโดยเทียบกับคำแนะนำสำหรับการรับสินค้าที่สร้างขึ้น การจับคู่สามทางคือสมัยใหม่ได้เนื่องจากคำแนะนำสำหรับใบรับสินค้าแสดงถึงการรับทางกายภาพที่แน่นอนของวัสดุ

## <a name="configuring-activities-for-subcontracting"></a>การตั้งค่าคอนฟิกกิจกรรมสำหรับการรับเหมารายย่อย
ส่วนต่อไปนี้อธิบายวิธีการตั้งค่าคอนฟิกกิจกรรมสำหรับการรับเหมารายย่อย

### <a name="subcontracted-services"></a>บริการรับเหมารายย่อย

รายการชำระเงินที่ใช้ในการใช้กิจกรรมที่รับเหมารายย่อยต้องเป็นผลิตภัณฑ์ที่มีคุณสมบัติต่อไปนี้:

-   **ชนิดผลิตภัณฑ์:**บริการ
-   **กลุ่มแบบจำลองสินค้าคงคลัง:**ที่ไม่ได้เก็บในคลัง

ข้อกำหนดนี้บังคับใช้แรกใน แบบจำลองสินค้าคงคลังแรก (fifo) **หมายเหตุ:**ต้องคำนวณต้นทุนของผลิตภัณฑ์ว่า มีกำหนดต้นทุนมาตรฐานของการบริการ ข้อตกลงการซื้อกับผู้จัดจำหน่ายที่ถูกต้อง มิฉะนั้น บริการไม่สามารถใช้สำหรับใช้กิจกรรมที่รับเหมารายย่อย

### <a name="subcontracted-process-activities"></a>กิจกรรมของกระบวนการรับเหมารายย่อย

การตั้งค่ากิจกรรมกระบวนการเป็นกิจกรรมที่รับเหมารายย่อย ให้ทำตามขั้นตอนเหล่านี้

1.  กำหนดเซลล์ทำงานที่รับเหมารายย่อย กำหนดเซลล์ทำงานตามที่รับเหมารายย่อย คุณต้องการทรัพยากรของการ**ของผู้จัดจำหน่าย**พิมพ์ และเชื่อมโยงกับเซลล์ทำงาน (กลุ่มทรัพยากร) ประเภทต้นทุนรันไทม์ของการ**การเอาท์ซอร์สโดยตรง**ควรกำหนดชนิดกลุ่มต้นทุนให้กับเซลล์ทำงาน ประเภทต้นทุนสำหรับปริมาณและการตั้งค่าไม่ถูกต้อง
2.  หลังจากกิจกรรมกระบวนการถูกสร้างขึ้น และที่เกี่ยวข้องกับเซลล์ทำงานที่รับเหมารายย่อย คุณต้องกำหนดค่าบริการสำหรับกิจกรรมก่อนที่สามารถเรียกใช้เวอร์ชันขั้นตอนการผลิต คุณทำตามขั้นตอนนี้ในการ**กิจกรรม****รายละเอียด**หน้า สำหรับกิจกรรมที่เกี่ยวข้องกับเซลล์ทำงานที่รับเหมารายย่อย**บริการเงื่อนไข**ด่วนจะแสดงขึ้น บนแท็บด่วนนี้ เพิ่มบริการเริ่มต้นที่มีผลบังคับใช้สำหรับสินค้าทั้งหมดของผลผลิต ถ้าผลลัพธ์เฉพาะสินค้าจำเป็นต้องมีบริการที่แตกต่างกันหรือพารามิเตอร์การคำนวณบริการอื่น (ตัวอย่างเช่น อัตราส่วนบริการที่แตกต่างกัน), คุณสามารถเพิ่มบริการอื่น ๆ ให้กับกิจกรรม

## <a name="subcontracted-transfer-activities"></a>กิจกรรมการโอนย้ายที่รับเหมารายย่อย
กิจกรรมการโอนย้ายการตั้งค่าคอนฟิกเป็นกิจกรรมที่รับเหมารายย่อย ขึ้นอยู่กับการ**ขนส่งโดย**การตั้งค่าของกิจกรรมการโอนย้าย มีตัวเลือกดังต่อไปนี้

-   **ผู้จัดส่ง**– กิจกรรมจะรับเหมารายย่อยถ้าการโอนย้ายจากคลังสินค้าจะถูกจัดการ โดยผู้จัดจำหน่าย (ตามที่กำหนด โดยคุณสมบัติของคลังสินค้า) ข้อตกลงการซื้อที่เลือกทั้งหมดสำหรับบริการต้องมีหมายเลขผู้ขายเดียวกันเป็นคลังสินค้า
-   **ผู้รับ**– กิจกรรมจะรับเหมารายย่อยถ้าการโอนย้ายไปยังคลังสินค้าจะถูกจัดการ โดยผู้จัดจำหน่าย (ตามที่กำหนด โดยคุณสมบัติของคลังสินค้า) ข้อตกลงการซื้อที่เลือกทั้งหมดสำหรับบริการต้องมีหมายเลขผู้ขายเดียวกันเป็นคลังสินค้า
-   **บริษัทขนส่ง**– กิจกรรมคือการรับเหมารายย่อยกับผู้จัดจำหน่ายที่ให้บริการ ถูกต้อง ผู้ขนส่งต้องถูกสร้างขึ้นสำหรับการจัดการคลังสินค้า และต้องมีบัญชีผู้จัดจำหน่ายที่กำหนดให้

สำหรับกิจกรรมของกระบวนการ คุณต้องกำหนดค่าบริการเริ่มต้นสำหรับกิจกรรมการโอนย้ายที่รับเหมารายย่อยในการ**บริการเงื่อนไข**ด่วนของ**กิจกรรม****รายละเอียด**หน้า

## <a name="service-quantity-calculation"></a>การคำนวณปริมาณบริการ
กระบวนการซื้อทั้งหมดเป็นไปตามการอ้างอิงสินค้าสำหรับการบริการ การอ้างอิงสินค้านี้มีวัดในหน่วยวัดของการบริการ บริการมักจะวัด เป็นจำนวนบริการ (หน่วย) หรือ ในเวลา ในการคำนวณปริมาณบริการ ขึ้นอยู่กับความสมบูรณ์ที่ลงทะเบียนของงานคัมบัง คุณสามารถใช้วิธีการต่อไปนี้:

-   **การคำนวณที่จะขึ้นอยู่กับจำนวนงาน**– งานคัมบังหนึ่งเท่ากับ*n*หน่วยของการบริการ โดยไม่คำนึงถึงปริมาณผลิตภัณฑ์ที่ได้รับการสนับสนุน ในการผลิตแบบ lean หนึ่งงานสอดคล้องกับหน่วยจัดการวัสดุหนึ่ง วิธีการคำนวณนี้ใช้กับบริการทั้งหมดที่มีราคาคงที่ต่อหน่วยจัดการวัสดุ ดังนั้น วิธีนี้มักจะใช้กับกิจกรรมการโอนย้าย อย่างไรก็ตาม ก็ยังสามารถใช้กับกิจกรรมของกระบวนการที่จัดการทั้งหน่วยประมวลผล
-   **การคำนวณที่ยึดตามปริมาณผลิตภัณฑ์**– ปริมาณบริการจะสัมพันธ์กับปริมาณผลิตภัณฑ์ที่มีการจัดตารางผลิต/จัดหา เมื่อคำนวณปริมาณผลิตภัณฑ์ที่ให้มา ปริมาณข้อผิดพลาดสามารถถูกรวม หรือแยกออก วิธีการคำนวณนี้ใช้ได้กับทุกกรณีที่ตกลงราคาการบริการสำหรับแต่ละหน่วยของผลิตภัณฑ์ที่ดำเนินการไว้นั้น
-   **การคำนวณที่ยึดตามเวลากิจกรรม**– เวลากิจกรรมตามทฤษฎีคำนวณ ขึ้นอยู่กับเวลาการประมวลผลของกิจกรรม ยอดรวม ปริมาณและอัตราส่วนอัตราความเร็วของการประมวลผลผลิตภัณฑ์การประมวลผล วิธีการคำนวณนี้ใช้กับการบริการที่มีการชำระเงินเป็นรายชั่วโมง และมีผลต่างการเวลาสำหรับแต่ละผลิตภัณฑ์ที่ประมวลผลแล้ว

## <a name="cost-accounting-of-subcontracted-services"></a>การลงบัญชีต้นทุนของบริการที่รับเหมารายย่อย
เมื่อได้รับคำแนะนำหรือผู้จัดจำหน่ายในใบสั่งซื้อที่สร้างขึ้นสำหรับขั้นตอนการผลิต (อีกนัยหนึ่ง ใบสั่งซื้อที่สร้างขึ้นโดยยึดตามงานคัมบังสำหรับกิจกรรมที่รับเหมารายย่อย) บันทึกในการจัดส่งถูกลงรายการบัญชี มูลค่าของการรับสินค้ามีการลงบัญชีในบัญชี WIP ของขั้นตอนการผลิต ความแตกต่างของใบแจ้งหนี้จะยังถูกนำมาพิจารณาถึงขั้นตอนการผลิต ประเภทต้นทุนสำหรับงานที่รับเหมารายย่อยมีการแนะนำ ประเภทต้นทุนนี้เปิดใช้งานการติดตามแบบโปร่งใสของค่าของงานที่รับเหมารายย่อยที่ถูกปันส่วนไปยังงานระหว่างทำ และใช้สำหรับแต่ละรอบระยะเวลา  

ผลต่างที่เกิดขึ้นจริงของผลิตภัณฑ์ที่ผลิตจากขั้นตอนการผลิตในระหว่างรอบระยะเวลาการคิดต้นทุน แบบย้อนกลับการคิดต้นทุนสำหรับการผลิตแบบ lean เมื่อสิ้นสุดรอบระยะเวลาการคิดต้นทุนคำนวณ

## <a name="modeling-transfers-as-subcontracted-activities"></a>โอนย้ายโมเดลเป็นกิจกรรมที่รับเหมารายย่อย
ผู้คนมักจะพิจารณารวบขนส่งใบแจ้งหนี้แล้วและคิดว่า จะเพิ่มค่าไม่มี อย่างไรก็ตาม เมื่อมีการเปรียบเทียบต้นทุนของการรับเหมารายย่อยเป็นต้นทุนของการผลิตภายใน ต้นทุนของกิจกรรมการขนส่งเพิ่มเติมต้องพิจารณา ขั้นตอนการผลิตที่ครอบคลุมหลายแห่ง และต้องการการบริการขนส่งควรแบบจำลองต้นทุนการขนส่งเป็นส่วนหนึ่งของต้นทุนของการจัดหาผลิตภัณฑ์ให้กับลูกค้า 

กิจกรรมตามการรับเหมารายย่อยในการผลิตแบบ lean ให้คุณรวมผู้ขนส่ง และการขนส่งผู้จัดจำหน่ายที่เคลื่อนย้ายวัสดุและผลิตภัณฑ์ระหว่างสถานที่เก็บของขั้นตอนการผลิต คุณสามารถกำหนดให้กับผู้ขนส่งหรือผู้จัดจำหน่าย โดยกิจกรรมการโอนย้าย การสร้างโมเดล กิจกรรม/งานการโอนย้ายเป็นไปตามข้อตกลงการบริการและการซื้อ และคุณสามารถสร้างใบสั่งซื้อและรับคำแนะนำสำหรับ ขึ้นอยู่กับงานการโอนย้ายจริง ฟังก์ชันนี้จะเหมือนกับฟังก์ชันการทำงานสำหรับกิจกรรมของกระบวนการรับเหมารายย่อย  

ดังนั้น Dynamics 365 สำหรับการดำเนินงานในขณะนี้สนับสนุนการคำนวณ BOM ที่มีการบริการขนส่ง การสร้างที่เกี่ยวข้องใบสั่งซื้อ การลงทะเบียนการรับสินค้าแบบครบวงจร และการรวมของการขนส่งบริการต้นทุนเป็นต้นทุนขั้นตอนการผลิต
