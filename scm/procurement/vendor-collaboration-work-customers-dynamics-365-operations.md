# <a name="vendor-collaboration-with-customers"></a>การทำงานร่วมกันกับผู้จัดจำหน่ายกับลูกค้า

[!include[banner](../includes/banner.md)]


หัวข้อนี้อธิบายวิธีการใช้การทำงานร่วมกันกับผู้จัดจำหน่ายใน Dynamics 365 for Operations เพื่อทำงานกับ PO และตรวจสอบสินค้าคงคลังที่มีการส่งมอบ

หัวข้อนี้อธิบายวิธีการใช้การทำงานร่วมกันกับผู้จัดจำหน่ายเพื่อทำงานกับลูกค้าใน Microsoft Dynamics 365 for Operations มีข้อมูลเกี่ยวกับวิธีการตรวจสอบและตอบสนองต่อใบสั่งซื้อ และวิธีการตรวจสอบสินค้าคงคลังที่มีการส่งมอบ ดังนั้นจึงสามารถใช้การทำงานร่วมกันกับผู้จัดจำหน่ายเพื่อทำงานกับใบแจ้งหนี้ได้ ดูข้อมูลเพิ่มเติมที่ [พื้นที่ทำงานการออกใบแจ้งหนี้การทำงานร่วมกันกับผู้จัดจำหน่าย](/dynamics365/operations/financials/accounts-payable/vendor-portal-invoicing-workspace)

## <a name="working-with-purchase-orders"></a>การทำงานกับใบสั่งซื้อ
พื้นที่ทำงาน **การยืนยันใบสั่งซื้อ** ช่วยให้คุณตอบสนอง PO ที่ถูกส่งเพื่อให้คุณตรวจทาน นอกจากนี้ยังช่วยให้คุณสามารถดูข้อมูลเกี่ยวกับ PO ที่กำลังรอการดำเนินการจากลูกค้า และ PO ที่ยืนยันแล้ว แต่ยังคงเปิดอยู่ มีสามรายการในพื้นที่ทำงาน **การยืนยันใบสั่งซื้อ**:

-   **ใบสั่งซื้อสำหรับการตรวจทาน** -รายการนี้แสดง PO ที่ส่งถึงคุณ และกำลังรอการตอบสนองจากคุณ หลังจากที่คุณตอบสนองแล้ว PO จะถูกลบออกจากรายการ ถ้าลูกค้าส่ง PO เวอร์ชันใหม่ให้กับคุณก่อนที่คุณจะตอบสนองเวอร์ชันก่อนหน้า ระบบจะแสดงเฉพาะเวอร์ชันล่าสุดเท่านั้น
-   **กำลังรอการดำเนินการของลูกค้า** - รายการนี้ช่วยให้คุณสามารถมองเห็น PO ที่ตอบสนอง แต่ยังไม่ได้ได้รับการยืนยันจากลูกค้า ถ้าคุณยอมรับ PO คุณสามารถตรวจสอบได้ในรายการนี้จนกว่าสถานะจะเปลี่ยนเป็น **ยืนยันแล้ว** ถ้าคุณปฏิเสธ PO หรือยอมรับการเปลี่ยนแปลง คุณสามารถตรวจสอบ PO ได้ที่นี่จนกว่าลูกค้าจะส่งเวอร์ชันใหม่มา
-   **เปิดใบสั่งซื้อที่ยืนยันแล้ว** - รายการนี้ประกอบด้วย PO ทั้งหมดสำหรับบัญชีของคุณที่มีสถานะเป็น **ยืนยันแล้ว** เมื่อผลิตภัณฑ์หรือบริการได้รับโดยเทียบกับ PO ทั้งหมดแล้ว PO จะถูกลบออกจากรายการ

รายการต่อไปนี้แสดงรายการสี่หน้าที่คุณสามารถใช้เพื่อทำงานกับใบสั่งซื้อ โดยมีสองหน้าที่มีรายละเอียดที่เหมือนกันกับรายการในพื้นที่ทำงาน:

-   **ใบสั่งซื้อสำหรับการตรวจทาน** (ดูข้างบน)
-   **ประวัติการยืนยันใบสั่งซื้อของผู้จัดจำหน่าย** -หน้านี้ประกอบด้วย PO ทั้งหมดและ เวอร์ชันทั้งหมดของ PO ที่ถูกส่งไปยังผู้จัดจำหน่าย และการตอบสนองทั้งหมดที่มีการส่งคืนจากผู้จัดจำหน่าย
-   **เปิดใบสั่งซื้อที่ยืนยันแล้ว** (ดูข้างบน)
-   **ใบสั่งซื้อที่ยืนยันแล้วทั้งหมด** - หน้านี้ประกอบด้วย PO ที่ยืนยันแล้วทั้งหมด ซึ่งรวมถึงผลิตภัณฑ์หรือบริการที่ได้รับ คุณสามารถใช้รายการนี้เพื่อตรวจสอบว่า PO ใดที่คุณสามารถส่งใบแจ้งหนี้ได้

### <a name="responding-to-purchase-orders"></a>การตอบสนองต่อใบสั่งซื้อ

ใบสั่งซื้อที่ลูกค้าส่งเพื่อตรวจทานคุณจะมองเห็นได้ในพื้นที่ทำงาน **การยืนยันใบสั่งซื้อ** และ **ใบสั่งซื้อสำหรับการตรวจทาน** หลังจากที่คุณเปิด PO คุณสามารถเลือกที่จะยอมรับ ปฏิเสธ หรือยอมรับการเปลี่ยนแปลง อาจเป็นสิ่งที่แนบมาในส่วนหัวของ PO หรือในแต่ละบรรทัด นอกจากนี้คุณยังสามารถแนบข้อมูลให้กับการตอบสนองของคุณในส่วนหัวของ PO หรือในแต่ละบรรทัดได้ ตัวอย่างเช่น คุณอาจแนะนำสินค้าทดแทนสำหรับบรรทัดใดบรรทัดหนึ่ง คุณสามารถแสดงตัวอย่างและพิมพ์ PO เป็นไฟล์ PDF ได้โดยใช้ตัวเลือก **แสดงตัวอย่าง/พิมพ์** คุณสามารถซ่อนหรือแสดงคอลัมน์มิติต่อไปนี้โดยใช้การดำเนินการ **แสดงมิติ**: ไซต์ คลังสินค้า สี ขนาด ลักษณะ การตั้งค่าคอนฟิก ถ้าคุณใช้ตัวเลือก **ยอมรับการเปลี่ยนแปลง** คุณสามารถยอมรับหรือปฏิเสธแต่ละบรรทัดได้ และคุณยังสามารถทำการเปลี่ยนแปลงต่อไปนี้ไปยังบรรทัดต่างๆ:

-   เปลี่ยนแปลงวันที่หรือปริมาณ ถ้าคุณต้องการอัพเดตวันที่จัดส่งที่ยืนยันแล้วในรายการทั้งหมด ให้ใช้ตัวเลือก **อัพเดตวันที่จัดส่ง** บนส่วนหัวของ PO
-   แบ่งรายการสำหรับวันที่หรือปริมาณจัดส่งที่แตกต่างกัน
-   ทดแทนสินค้า เมื่อต้องการทำเช่นนี้ ให้ป้อนคำอธิบายสินค้าและหมายเลขสินค้าของคุณในฟิลด์ **ภายนอก** ในส่วน **รายละเอียดรายการ**

คุณไม่สามารถเปลี่ยนข้อมูลหรือค่าธรรมเนียมการกำหนดราคาได้ แต่คุณสามารถเปลี่ยนคำแนะนำสำหรับการเปลี่ยนแปลงให้กับรายการเหล่านี้ได้โดยใช้หมายเหตุ ถ้าลูกค้าของคุณส่ง PO เวอร์ชันใหม่ให้กับคุณ PO เวอร์ชันใหม่นี้จะมีคำเสริมท้ายเวอร์ชันที่บ่งชี้ว่า มีการปรับเปลี่ยน PO ที่มีการอธิบายไว้ก่อนหน้านี้ หน้า **ประวัติการยืนยันใบสั่งซื้อของผู้จัดจำหน่าย** อนุญาตให้คุณติดตามประวัติของแต่ละใบสั่ง

## <a name="monitoring-consignment-inventory"></a>การตรวจสอบสินค้าคงคลังที่มีการส่งมอบ
ถ้าคุณใช้สินค้าคงคลังที่มีการส่งมอบ คุณสามารถใช้อินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายเพื่อดูข้อมูลในหน้าต่อไปนี้:

-   **ใบสั่งซื้อที่ใช้สินค้าคงคลังที่มีการส่งมอบ** -ใบสั่งซื้อสำหรับสินค้าคงคลังที่มีการส่งมอบจะถูกสร้างขึ้นเมื่อลูกค้าเข้าเป็นเจ้าของสินค้าคงคลัง ใบสั่งซื้อที่มีการส่งมอบเหล่านี้จะแสดงขึ้นในหน้า **ใบสั่งซื้อที่ใช้สินค้าคงคลังที่มีการส่งมอบ** เท่านั้น โดยจะไม่รวมอยู่ในหน้า **ใบสั่งซื้อที่ยืนยันแล้วทั้งหมด**
-   **ผลิตภัณฑ์ที่ได้รับจากสินค้าคงคลังที่มีการส่งมอบ** -หน้านี้แสดงรายการธุรกรรมทั้งหมดที่เจ้าของของผลิตภัณฑ์มีการโอนย้ายไปยังบริษัทที่กำลังใช้งานสินค้าคงคลังดังกล่าว คุณสามารถใช้ข้อมูลนี้เพื่อออกใบแจ้งหนี้ให้ลูกค้าได้
-   **สินค้าคงคลังที่มีการส่งมอบคงเหลือ** -หน้านี้แสดงสินค้าคงคลังที่มีการส่งมอบคงเหลือที่เป็นเจ้าของโดยบริษัทของคุณที่มีปริมาณคงเหลือในคลังสินค้าของลูกค้า
