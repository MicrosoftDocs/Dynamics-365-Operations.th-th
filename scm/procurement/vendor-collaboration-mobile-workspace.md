## <a name="overview"></a>ภาพรวม
พื้นที่ทำงานแบบเคลื่อนที่ของการทำงานร่วมกันกับผู้จัดจำหน่ายช่วยให้ผู้จัดจำหน่ายได้ทราบเกี่ยวกับใบสั่งซื้อใหม่เพื่อให้พวกเขาสามารถดู และตอบสนองต่อใบสั่งซื้อในเว็บไคลเอนต์ Dynamics 365 for Operations 

**หมายเหตุ:** ควรใช้พื้นที่ทำงานแบบเคลื่อนที่เป็นส่วนเพิ่มเติมของเว็บไคลเอนต์การทำงานร่วมกันของผู้จัดจำหน่าย แต่ไม่ได้แทนที่ 

ด้วยพื้นที่ทำงานแบบเคลื่อนที่ของการทำงานร่วมกันกับผู้จัดจำหน่าย ผู้จัดจำหน่ายของคุณสามารถดูใบสั่งซื้อใหม่ที่ถูกส่งไปเพื่อขออนุมัติได้ ระบบจะแสดงข้อมูลใบสั่งซื้อ เช่น ผลิตภัณฑ์ ปริมาณ และวันที่จัดส่งที่ร้องขอ มีข้อมูลราคา โดยขึ้นอยู่กับการตั้งค่าคอนฟิกสำหรับผู้จัดจำหน่ายแต่ละราย 

เมื่อผู้ใช้ล็อกอินเป็นผู้จัดจำหน่าย พวกเขาจะเห็นว่าใบสั่งซื้อใดได้รับการตอบรับ หรือใบสั่งซื้อใดยังคงรอการดำเนินการของลูกค้า ผู้จัดจำหน่ายอาจแนะนำวันที่จัดส่งอื่นที่ยังไม่ได้ตกลงกับลูกค้าเพื่อให้ใบสั่งซื้อดังกล่าวรอการดำเนินการของลูกค้า ผู้จัดจำหน่ายจะเห็นรายการของใบสั่งซื้อที่ยืนยันแล้ว แต่ยังไม่ได้จัดส่ง 

เพื่อตอบสนองต่อใบสั่งซื้อ ผู้จัดจำหน่ายจำเป็นต้องใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายที่พร้อมใช้งานในเว็บไคลเอนต์ Dynamics 365 for Operations และยังเป็นที่ที่ผู้จัดจำหน่ายจะได้รับข้อมูลเพิ่มเติมเกี่ยวกับใบสั่ง เช่น สิ่งที่แนบของเอกสาร ที่อยู่จัดส่งในบรรทัด และค่าธรรมเนียมที่เกี่ยวข้องกับผู้จัดจำหน่ายด้วย 

ด้วยบทบาทความปลอดภัยพิเศษ ผู้จัดจำหน่ายสามารถดูว่าผู้ติดต่อใดที่ลงทะเบียนสำหรับบัญชีผู้จัดจำหน่าย ด้วยบทบาทความปลอดภัยเดียวกันนี้ ผู้จัดจำหน่ายสามารถดูสถานะของคำขอของผู้ใช้ใดๆ ที่ได้ถูกส่งไป 

การสร้างผู้ติดต่อใหม่และการส่งคำขอของผู้ใช้ใหม่จะต้องกระทำในอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายที่มีอยู่ในเว็บไคลเอนต์ Dynamics 365 for Operations 

ด้วยพื้นที่ทำงานแบบเคลื่อนที่ ผู้จัดจำหน่ายของคุณสามารถ:

-   ดูใบสั่งซื้อใหม่ที่ถูกส่งไปยังผู้จัดจำหน่าย
-   ดูใบสั่งซื้อที่ผู้จัดจำหน่ายได้ตอบสนองและกำลังรอการดำเนินการของลูกค้าอยู่
-   ดูใบสั่งซื้อที่อยู่ในสถานะที่ยืนยันแล้วแต่ยังไม่ได้รับทั้งหมด
-   ดูข้อมูลผู้ติดต่อที่ลงทะเบียนสำหรับบัญชีผู้จัดจำหน่าย (ต้องมีบทบาทความปลอดภัยเพิ่มเติม)
-   ดูรายละเอียดและติดตามสถานะของคำขอของผู้ใช้ที่ถูกส่งโดยผู้จัดจำหน่าย (ต้องมีบทบาทความปลอดภัยเพิ่มเติม)

## <a name="get-started"></a>เริ่มต้น
เมื่อต้องการเริ่มต้นใช้งานบนอุปกรณ์เคลื่อนที่ของคุณ:

1.  บนร้านค้าแอพบนมือถือของคุณ ดาวน์โหลดและติดตั้งแอพ Microsoft Dynamics 365 for Operations
2.  เริ่มการทำงานแอพบนอุปกรณ์ของคุณ
3.  ป้อน URL ของ Dynamics 365
4.  ป้อนบริษัทเพื่อลงชื่อเข้าใช้ ตัวอย่างเช่น ป้อน **USMF**
5.  ในครั้งแรกที่คุณเข้าสู่ระบบ คุณจะได้รับพร้อมท์สำหรับชื่อผู้ใช้และรหัสผ่านสำหรับบัญชี Microsoft Dynamics 365 for Operations ของคุณ

หลังจากที่คุณเข้าสู่ระบบในแอพลิเคชันแล้ว จะไม่สามารถดูพื้นที่ทำงานใดๆ ได้ เมื่อต้องการดูพื้นที่ทำงานในแอพบนมือถือของคุณ คุณต้องเผยแพร่พื้นที่ทำงานที่ต้องการไปยังแอพ Dynamics 365 for Operations คุณต้องมีสิทธิ์การดูแลระบบเพื่อที่จะเผยแพร่พื้นที่ทำงาน

1.  เริ่มการทำงาน Dynamics 365 for Operations
2.  ไปที่ **การจัดการระบบ** &gt; **การตั้งค่า** &gt; **พารามิเตอร์ระบบ**
3.  เลือก **จัดการแอพบนมือถือ**
4.  เลือกพื้นที่ทำงาน **การทำงานร่วมกันกับผู้จัดจำหน่าย** เพื่อเผยแพร่ไปยังแพลตฟอร์มเคลื่อนที่
5.  เลือก **เผยแพร่พื้นที่ทำงาน**
6.  รีเฟรชอุปกรณ์ของคุณเพื่อดูพื้นที่ทำงานที่เผยแพร่แล้ว
7.  เลือกพื้นที่ทำงาน **การทำงานร่วมกันกับผู้จัดจำหน่าย** คุณจะเห็นหน้าต่อไปนี้

    [![แอพบนมือถือการทำงานร่วมกันกับผู้จัดจำหน่าย](./media/vendor-collaboration-mobile-app.png)](./media/vendor-collaboration-mobile-app.png)

## <a name="contacts"></a>ผู้ติดต่อ
หน้า **ผู้ติดต่อ** ช่วยให้คุณสามารถดูผู้ติดต่อทั้งหมดที่ตั้งค่าไว้สำหรับบัญชีผู้จัดจำหน่าย ระบบจะแสดงชื่อ อีเมลหลัก และนามแฝงผู้ใช้ของผู้ติดต่อถ้ามี และยังแสดงว่าบัญชีผู้ใช้ของผู้ติดต่อยังมีการใช้งานอยู่หรือไม่อีกด้วย เมื่อคุณเลือกผู้ติดต่อ คุณจะเห็นรายละเอียดผู้ติดต่อ เช่น นิติบุคคลใดที่บุคคลเป็นผู้ติดต่อ และข้อมูลติดต่อ เช่น หมายเลขโทรศัพท์หรือที่อยู่อีเมลอื่น

## <a name="user-requests"></a>คำขอของผู้ใช้
หน้า **คำขอของผู้ใช้** ช่วยให้คุณสามารถดูคำขอของผู้ใช้ทั้งหมดที่คุณได้ส่งผ่านเว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายและติดตามสถานะ เมื่อคุณเลือกคำขอของผู้ใช้ คุณสามารถดูสิ่งที่ถูกร้องขอ เพิ่ม หรือปิดการทำงานของผู้ใช้ การเปลี่ยนแปลงการรักษาความปลอดภัย และดูบทบาทความปลอดภัยที่มีการร้องขอสำหรับผู้ใช้ได้

## <a name="purchase-orders-ready-for-review"></a>ใบสั่งซื้อที่พร้อมสำหรับการตรวจทาน
หน้า **ใบสั่งซื้อที่พร้อมสำหรับการตรวจทาน** ช่วยให้คุณสามารถดูใบสั่งซื้อทั้งหมดที่ถูกส่งไปที่ยังลูกค้าและยังไม่ได้รับคำตอบ คุณสามารถดูข้อมูลเกี่ยวกับใบสั่ง เช่น ผลิตภัณฑ์ใดมีการร้องขอและจะต้องจัดส่งเมื่อใด จะมีข้อมูลราคาถ้ามีการตั้งค่าคอนฟิกสำหรับผู้จัดจำหน่ายเท่านั้น คุณสามารถดูว่าใบสั่งซื้อมีหมายเหตุหรือเอกสารแนบหรือไม่ เมื่อต้องการเปิดเอกสารแนบ คุณจำเป็นต้องใช้เว็บไคลเอนต์การทำงานร่วมกันกับผู้จัดจำหน่าย เลือก **บรรทัดใบสั่งซื้อ** เพื่อดูบรรทัดทั้งหมดที่มีรายละเอียด หมายเหตุว่าสำหรับแต่ละบรรทัด ตัวบ่งชี้จะแสดงว่ามีบันทึกย่อหรือเอกสารแนบหรือไม่ หรือว่ามีที่อยู่จัดส่งที่แตกต่างจากสิ่งที่แสดงในหัวข้อหรือไม่ เมื่อต้องการตอบสนองต่อใบสั่งซื้อ คุณต้องใช้เว็บไคลเอนต์การทำงานร่วมกันกับผู้จัดจำหน่าย

## <a name="awaiting-customer-action"></a>กำลังรอการดำเนินการของลูกค้า
หน้า **กำลังรอการดำเนินการของลูกค้า** ช่วยให้คุณสามารถค้นหาใบสั่งซื้อที่คุณ หรือบุคคลอื่นในบริษัทของคุณที่มีการเข้าถึงไปยังการทำงานร่วมกันกับผู้จัดจำหน่ายตอบสนอง ใบสั่งซื้อจะปรากฏในรายการนี้เท่านั้นถ้าลูกค้าต้องการดำเนินการหนึ่งในการดำเนินการต่อไปนี้ในใบสั่งซื้อ

-   ถ้าใบสั่งซื้อถูกปฏิเสธ ลูกค้าจำเป็นต้ออัพเดตใบสั่งที่ส่งแล้วและส่งอีกครั้ง หรือยกเลิกใบสั่งและส่งอีกครั้ง อย่างใดอย่างหนึ่ง เมื่อใบสั่งซื้อถูกส่งไปอีกครั้ง ใบสั่งซื้อดังกล่าวจะหายไปจากหน้า **กำลังรอการดำเนินการของลูกค้า**
-   ถ้ามีการยอมรับใบสั่งซื้อโดยมีการเปลี่ยนแปลง ลูกค้าอาจจำเป็นต้องอัพเดตใบสั่งซื้อต้นฉบับและส่งอีกครั้งเพื่อตรวจทาน หรืออัพเดตตามการเปลี่ยนแปลงและยืนยันโดยทันที ในทั้งสองกรณี ใบสั่งซื้อดังกล่าวจะหายไปจากหน้า **กำลังรอการดำเนินการของลูกค้า**
-   ถ้าใบสั่งซื้อได้รับการยอมรับและปรากฏในหน้า **กำลังรอการดำเนินการของลูกค้า** เป็นเพราะใบสั่งซื้อไม่ได้รับการยืนยันโดยอัตโนมัติเมื่อทำการยอมรับ ซึ่งกำลังรอให้ตัวแทนจัดซื้อเปลี่ยนแปลงใบสั่งเป็น ยืนยันแล้ว โดยทั่วไป ใบสั่งซื้อจะถูกพิจารณาเป็นข้อตกลงระหว่างลูกค้าและผู้จัดจำหน่ายทันทีที่ผู้จัดจำหน่ายยอมรับใบสั่ง การย้ายใบสั่งซื้อไปยังสถานะ ยืนยันแล้ว อาจเป็นแบบแผน

โดยการเลือกใบสั่งซื้อ รายละเอียดเพิ่มเติมจะปรากฏเกี่ยวกับการตอบสนอง คุณสามารถดูรายละเอียดของรายการและตอบสนองสำหรับทุกบรรทัด สถานะของบรรทัดจะแสดงว่ามีการให้การตอบสนองใดต่อไปนี้

-   ที่ยอมรับได้
-   ปฏิเสธแล้ว
-   ยอมรับการเปลี่ยนแปลงแล้ว
-   ใช้ทดแทน/ทดแทน
-   แบ่งเป็นบรรทัดกำหนดการ/กำหนดการ

หมายเหตุว่าตัวบ่งชี้จะแสดง **การจัดส่ง**=ใช่/ไม่ ซึ่งจะถูกใช้เพื่อบ่งชี้ว่าบรรทัดจะไม่ถูกส่ง ซึ่งอาจเนื่องจากบรรทัดถูกปฏิเสธหรือแทนที่บรรทัดต้นฉบับที่ไม่ได้คาดว่าจะมีการจัดส่ง หรือบรรทัดที่ถูกแยกเป็นหลายบรรทัดกำหนดการ และบรรทัดต้นฉบับไม่ได้คาดว่าจะมีการจัดส่งตามที่ร้องขอในใบสั่งที่ได้รับ การเปลี่ยนแปลงใดๆ ที่เกิดขึ้นกับการตอบสนองบรรทัดใบสั่งจะแสดงขึ้น ยกเว้นสำหรับบันทึกย่อและเอกสารแนบที่อัปโหลด ซึ่งคุณสามารถดูได้โดยใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่าย

## <a name="open-confirmed-orders"></a>ใบสั่งซื้อที่ยืนยันแล้วและเปิดอยู่
เมื่อมีการยืนยันใบสั่งซื้อจากลูกค้า ซึ่งหมายความว่าใบสั่งซื้อจะเปลี่ยนเป็นสถานะยืนยันแล้ว จะปรากฏในใบสั่งที่ยืนยันแล้วที่เปิดอยู่ ซึ่งจะยังคงอยู่ในรายการจนกว่าจะมีการลงทะเบียนว่า ได้รับแล้ว จากลูกค้า
