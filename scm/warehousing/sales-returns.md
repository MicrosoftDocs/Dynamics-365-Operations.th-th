# <a name="sales-returns"></a>การส่งคืนสินค้าจากการขาย

หัวข้อนี้แสดงข้อมูลเกี่ยวกับกระบวนการสำหรับใบสั่งส่งคืนสินค้า ซึ่งจะรวมถึงข้อมูลเกี่ยวกับการส่งคืนของลูกค้าและผลที่เกิดขึ้นในการคำนวณต้นทุน และการถือครองสินค้าคงคลังปริมาณ

ลูกค้าสามารถส่งคืนสินค้าสำหรับเหตุผลต่าง ๆ ตัวอย่างเช่น สินค้าอาจเป็นสินค้าที่มีตำหนิ หรือดังกล่าวอาจไม่ตรงกับความคาดหวังของลูกค้า กระบวนการส่งคืนสินค้าเริ่มต้นเมื่อลูกค้าออกการร้องขอการส่งคืนสินค้า หลังจากที่ได้รับการร้องขอของลูกค้า ใบสั่งส่งคืนสินค้าถูกสร้างขึ้นใน 365 Dynamics Microsoft สำหรับการดำเนินงาน

## <a name="return-order-process"></a>กระบวนการใบสั่งส่งคืนสินค้า
ภาพประกอบต่อไปนี้ให้ภาพรวมของกระบวนการใบสั่งส่งคืนสินค้า  

[![salesreturns01](./media/salesreturns01.jpg)](./media/salesreturns01.jpg)  

มีอยู่สองชนิดของกระบวนการใบสั่งส่งคืนสินค้า: ทางกายภาพกลับ และเครดิตเท่านั้น

-   **ผลตอบแทนทางกายภาพ**– ใบสั่งส่งคืนสินค้าที่อนุมัติให้ส่งคืนทางกายภาพของผลิตภัณฑ์
-   **เครดิตเฉพาะ**– ใบสั่งส่งคืนสินค้าที่อนุมัติให้เครดิตลูกค้า แต่ไม่จำเป็นต้องให้ลูกค้ากลับผลิตภัณฑ์ทางกายภาพ

### <a name="physical-return-order-process"></a>กระบวนการใบสั่งส่งคืนสินค้าทางกายภาพ

1.  **สร้างใบสั่งส่งคืนสินค้า** Formally เอกสารการตรวจสอบสำหรับการซื้อผลิตภัณฑ์ใด ๆ ที่ไม่พึงประสงค์ หรือมีข้อบกพร่อง ใบสั่งส่งคืนสินค้าไม่จำเป็นต้องให้เครดิตสำหรับลูกค้า หรือยอมรับผลิตภัณฑ์ที่ส่งคืนบริษัท ถ้าจะส่งคืนการยอมรับ คุณสามารถอนุมัติสินค้าทดแทนที่จะส่งก่อนมีการส่งคืนสินค้ามีตำหนิ
2.  **มาถึงที่คลังสินค้าตรวจสอบ** ดำเนินการตรวจสอบการเริ่มต้นและตรวจสอบกับเอกสารใบสั่งส่งคืนสินค้า ใบสั่งส่งคืนสินค้ายังสนับสนุนการตรวจสอบสินค้าของสินค้าส่งคืนสำหรับการตรวจสอบเพิ่มเติมและการควบคุมคุณภาพ
3.  **ตรวจสอบการโอนการครอบครองหรือไม่** ดำเนินการกระบวนการตรวจสอบขั้นสุดท้าย และตัดสินใจว่า ควรจะทำอะไรกับผลิตภัณฑ์ที่ส่งคืน เป็นส่วนหนึ่งของขั้นตอนนี้ ตัดสินใจว่า คุณจะลดหนี้ลูกค้า ปฏิเสธการส่งคืนผลิตภัณฑ์ หรือยอมรับการส่งคืนผลิตภัณฑ์ ผลิตภัณฑ์ที่เป็นของเสีย แล้ว ส่งผลิตภัณฑ์ทดแทนให้กับลูกค้า
4.  **สร้างบันทึกการจัดส่ง** สร้างบันทึกการจัดส่ง และยืนยันการตัดสินใจการโอนการครอบครองที่คุณทำไว้ในขั้นตอนที่ 3 เสร็จสิ้นกระบวนการลอจิสติกส์
5.  **สร้างใบแจ้งหนี้** ปิดใบสั่งส่งคืนสินค้า

### <a name="credit-only-process"></a>สินเชื่อกระบวนการเท่านั้น

1.  **สร้างใบสั่งส่งคืนสินค้า** Formally เอกสารการตรวจสอบสำหรับลูกค้าที่คุณจะได้รับเครดิต โดยไม่มีการส่งคืนผลิตภัณฑ์มีข้อบกพร่อง หรือที่ไม่พึงประสงค์ การ**เครดิตเฉพาะ**รหัสการโอนการครอบครองที่อนุมัติการตัดสินใจที่ลงเครดิตลูกค้า โดยไม่มีการส่งคืนสินค้าที่มีอยู่จริง
2.  **สร้างใบแจ้งหนี้** สร้างใบลดหนี้ และใบสั่งส่งคืนสินค้าที่ปิดแล้ว

## <a name="return-material-authorization"></a>ตรวจสอบวัสดุที่ส่งคืน
ประมวลผลการตรวจสอบวัสดุ (RMA) ที่ส่งคืนสินค้าที่สร้างขึ้นตามฟังก์ชันการทำงานของใบสั่งขาย RMA ถูกลงทะเบียนเป็นใบสั่งส่งคืน ซึ่งจะสร้างเป็นใบสั่งขาย และใบสั่งขายอื่นที่เกี่ยวข้อง เรียกใบสั่งเปลี่ยนทดแทนอาจ ทั้งใบสั่งขายที่เชื่อมโยงกับหมายเลข RMA เริ่มต้น

-   **ใบสั่งส่งคืน**– ลงทะเบียนผิด RMA คุณสร้างใบสั่งส่งคืน ซึ่งเป็นใบสั่งขายที่มีชนิดเป็นกำหนด**ใบสั่งส่งคืน** การเปลี่ยนแปลงใด ๆ ที่คุณทำกับข้อมูล RMA ถูกอัพเดตโดยอัตโนมัติในใบสั่งขาย จนกว่าใบสั่งส่งคืนสินค้าที่มีสถานะเป็น**เปิด**จะไม่ปรากฏในรายการของใบสั่งขาย คุณใช้ RMA ใน การจัดการเมื่อมาถึงและการรับสินค้าส่งคืน ตลอดจน การอนุญาตการดำเนินการเครดิตเท่านั้นโอนการครอบครอง (ดูส่วน**รหัสการโอนการครอบครองและการดำเนินการโอนการครอบครอง**) จัดการกระบวนการติดตามผลอื่น ๆ ทั้งหมดในใบสั่งขาย
-   **ใบสั่งเปลี่ยนทดแทน**– เมื่อต้องจัดส่งใบสั่งเปลี่ยนทดแทนให้กับลูกค้า RMA สามารถรวมใบสั่งขายที่เกี่ยวข้องที่สองได้ คุณสามารถสร้างใบสั่งเปลี่ยนทดแทนสำหรับ RMA เพื่อสนับสนุนการจัดส่งในทันทีด้วยตนเอง อีกวิธีหนึ่งคือ ใบสั่งเปลี่ยนทดแทนสามารถสร้างโดยอัตโนมัติหลังจากการมาถึง ตรวจสอบ และใบเสร็จสมบูรณ์สำหรับสินค้าในบรรทัด RMA ที่มีรหัสการโอนการครอบครองที่บ่งชี้ถึงการเปลี่ยนทดแทน ใบสั่งเปลี่ยนทดแทนได้เป็นฟังก์ชันเดียวกันที่เกี่ยวข้องกับใบสั่งขาย ตัวอย่างเช่น คุณสามารถใช้ได้เพื่อจัดโครงแบบผลิตภัณฑ์ที่กำหนดเองเป็นสินค้าทดแทน สร้างใบสั่งการผลิตเมื่อต้องการซ่อมแซมสินค้าที่ส่งคืน สร้างใบสั่งซื้อการจัดส่งโดยตรงในการส่งการแทนที่จากผู้จัดจำหน่าย หรือสนับสนุนวัตถุประสงค์อื่น

## <a name="create-a-return-order"></a>การสร้างใบสั่งส่งคืนสินค้า
กระบวนการใบสั่งส่งคืนสินค้าเริ่มต้นเมื่อลูกค้าติดต่อองค์กรของคุณ เพื่อกลับเป็นผลิตภัณฑ์ที่ไม่พึงประสงค์ หรือมีข้อบกพร่อง และ/หรือ ที่จะเครดิต หลังจากที่องค์กรของคุณยอมรับการส่งคืน ส่งคืนจะจัดทำเอกสารตามใบสั่งส่งคืนสินค้า ใบสั่งส่งคืนสินค้านี้กลายเป็น จุดโฟกัสของการประมวลผลภายในของผลิตภัณฑ์ส่งคืน ภาพประกอบต่อไปนี้แสดงขั้นตอนเกี่ยวกับการสร้างใบสั่งส่งคืนสินค้า

### <a name="create-return-order-lines"></a>สร้างบรรทัดใบสั่งส่งคืนสินค้า

หลังจากที่คุณดำเนินการใบสั่งส่งคืนสินค้า คุณสามารถสร้างบรรทัดการส่งคืนสินค้า โดยใช้หนึ่งในวิธีต่อไปนี้:

-   ป้อนรายละเอียดรายการ ปริมาณ และข้อมูลอื่น ๆ สำหรับทุกบรรทัดส่งคืนด้วยตนเอง
-   สร้างบรรทัดการส่งคืนสินค้า โดยใช้การ**ค้นหาใบสั่งขาย**ฟังก์ชัน เราขอแนะนำว่า คุณควรใช้ฟังก์ชันนี้เมื่อคุณสร้างใบสั่งส่งคืนสินค้า การ**ค้นหาใบสั่งขาย**ฟังก์ชันที่สร้างการอ้างอิงจากบรรทัดการส่งคืนสินค้าไปยังบรรทัดใบสั่งขายที่ออกใบแจ้งหนี้ และดึงบรรทัดรายละเอียดเช่นหมายเลขสินค้า ปริมาณ ราคา ส่วนลด และมูลค่าต้นทุนจากบรรทัดการขาย การอ้างอิงช่วยรับประกันว่า เมื่อมีการส่งคืนผลิตภัณฑ์ไปยังบริษัท นั้นได้ตามปกติมีมูลค่าต้นทุนต่อหน่วยเดียวกันนั้นถูกขายไปแล้วที่ นอกจากนี้การอ้างอิงตรวจสอบว่าไม่ได้สร้างใบสั่งสำหรับปริมาณที่เกินกว่าปริมาณที่ถูกขายบนใบแจ้งหนี้ ที่ส่งคืน

**หมายเหตุ:**จัดการบรรทัดการส่งคืนสินค้าที่มีการอ้างอิงถึงใบสั่งขายเป็นการแก้ไข หรือกลับรายการ การขาย สำหรับข้อมูลเพิ่มเติม ให้ดูส่วน "การลงรายการบัญชีในบัญชีแยกประเภท" ในภายหลังในหัวข้อนี้

### <a name="charges"></a>ค่าธรรมเนียม

ค่าธรรมเนียมและค่าธรรมเนียมที่สามารถเพิ่มลงในใบสั่งที่ส่งคืนโดยใช้อย่างน้อยหนึ่งวิธีต่อไปนี้:

-   คุณสามารถเพิ่มค่าธรรมเนียมกับหัวข้อใบสั่งส่งคืนสินค้า บรรทัดใบสั่งส่งคืนสินค้า หรือทั้งสองอย่างด้วยตนเอง
-   ค่าธรรมเนียมสามารถจะเพิ่มลงในส่วนหัวของใบสั่งส่งคืนสินค้าเป็นฟังก์ชันของรหัสเหตุผลการส่งคืน
-   ค่าธรรมเนียมสามารถถูกเพิ่มโดยอัตโนมัติกับบรรทัดใบสั่งส่งคืนสินค้า ขึ้นอยู่กับรหัสการโอนการครอบครองของบรรทัด

ค่าธรรมเนียมจะถูกเพิ่มโดยอัตโนมัติหลังจากรหัสเหตุผลการส่งคืน หรือรหัสการโอนการครอบครองที่ถูกกำหนดให้กับบรรทัด ถ้ามีการเปลี่ยนแปลงรหัสเหตุผลได้ในภายหลัง รายการค่าธรรมเนียมที่มีอยู่จะไม่ถูกลบออก แต่อาจจะเพิ่มรายการค่าธรรมเนียมใหม่ ขึ้นอยู่กับรหัสเหตุผลใหม่ เมื่อคุณเพิ่มค่าธรรมเนียมให้กับบรรทัดใบสั่งส่งคืน ค่าธรรมเนียมที่คำนวณเป็นเปอร์เซ็นต์ของค่าบรรทัดหรือสั่ง กลายเป็นค่าลบเมื่อบรรทัดหนึ่งหรือหลายใบสั่งในบรรทัดเป็นค่าลบ ยกเว้นว่าเปอร์เซ็นต์เป็นตัวเลขค่าลบ ค่าธรรมเนียมที่เป็นค่าลบแทนเครดิตให้กับลูกค้า

### <a name="return-reason-codes"></a>รหัสเหตุผลการส่งคืน

ด้วยการใช้รหัสเหตุผลการส่งคืนสินค้า คุณสามารถช่วยทำให้ง่ายต่อการวิเคราะห์รูปแบบการส่งคืนสินค้า รหัสเหตุผลให้ข้อมูลเกี่ยวกับสาเหตุที่ลูกค้าต้องการส่งคืนสินค้า บางองค์กรมีรหัสเหตุผลได้มาก องค์กรเหล่านี้อาจจัดกลุ่มรหัสเหตุผล เป็นกลุ่มรหัสเหตุผลเพื่อให้ได้ภาพรวมดีขึ้น และ การรายงานการสะสม

### <a name="disposition-codes-and-disposition-actions"></a>รหัสการโอนการครอบครองและการดำเนินการโอนการครอบครอง

ขั้นตอนสำคัญในกระบวนการใบสั่งส่งคืนสินค้ามีการกำหนดรหัสการโอนการครอบครองกับบรรทัดใบสั่งส่งคืนสินค้าเป็นส่วนหนึ่งของการลงทะเบียนการมาถึง รหัสการโอนการครอบครองกำหนดข้อมูลต่อไปนี้:

-   **ผลกระทบทางการเงิน**– ควรลูกค้าจะถูกเครดิตสำหรับสินค้าส่งคืน และควรมีค่าธรรมเนียมเพิ่มบรรทัดใบสั่งส่งคืนสินค้าหรือไม่
-   **การจัดการสินค้าคืน**– ควรมีสินค้าเพิ่มกลับไปยังสินค้าคงคลัง ควรจะเป็นเศษซาก หรือควรจะส่งคืนให้แก่ลูกค้าหรือไม่
-   **ลอจิสติกส์ของสินค้าที่ส่งคืน**– ควรทดแทนสินค้าออกให้แก่ลูกค้าหรือไม่

นอกเหนือจากการกำหนดวิธีการตัดจำหน่ายสินค้าที่ส่งคืนของ รหัสการโอนการครอบครองสามารถทำให้ค่าธรรมเนียมที่จะใช้กับบรรทัดการส่งคืน พวกเขายังสามารถใช้เพื่อจัดกลุ่มสินค้าที่ส่งคืนสำหรับการวิเคราะห์ทางสถิติ มีกำหนดรหัสการโอนการครอบครองเป็นส่วนหนึ่งของการตั้งค่าของใบสั่งส่งคืนสินค้า อย่างไรก็ตาม แต่ละรหัสการโอนการครอบครองต้องอ้างอิงการดำเนินการโอนการครอบครองที่มีอยู่แล้วอย่างใดอย่างหนึ่ง ตารางต่อไปนี้แสดงรายการรหัสการโอนการครอบครองภายในและการกระทำของ **สิ่งสำคัญ:**ถ้าควรไม่สามารถส่งคืนสินค้า แต่ยังคงควรจะเครดิตลูกค้า กำหนด**เครดิตเฉพาะ**รหัสการโอนการครอบครองไปยังบรรทัดการส่งคืนสินค้า

## <a name="arrival-at-the-warehouse-for-inspection"></a>มาถึงที่คลังสินค้าตรวจสอบ
ก่อนที่คุณสามารถจริงได้รับสินค้าที่ส่งคืนลงในสินค้าคงคลัง โดยการลงรายการบัญชีบันทึกการจัดส่ง สินค้าต้องไปผ่านการตรวจสอบหรือไม่ก็ได้และการลงทะเบียนการมาถึง ภาพประกอบต่อไปนี้ให้ภาพรวมของกระบวนการมาถึง ในส่วนต่อจากนั้นอธิบายแต่ละขั้นตอนที่แสดงในภาพประกอบ  

[![กระบวนการมาถึง](./media/salesreturn03.png)](./media/salesreturn03.png)  

กระบวนการมีหลายรูปแบบอื่น ๆ ที่ไม่ครอบคลุมอยู่ในหัวข้อนี้ ต่อไปนี้เป็นบางอย่างของชุดรูปแบบเหล่านี้:

-   อย่าใช้แบบ**ภาพรวมการมาถึง**รายการเพื่อสร้างสมุดรายวันการมาถึง แทน สร้างสมุดรายวันการมาถึงด้วยตนเอง ใบสั่งจะมีการส่งคืน**สั่งขาย**เป็นการอ้างอิง
-   ถ้าคุณกำลังใช้การบริหารคลังสินค้า สร้างการขนส่งแท่นวางสินค้า บรรทัดการส่งคืนจะมีสถานะเป็น**ข้อผิดพลาดแล้ว**ในระหว่างการขนส่งแท่นวางสินค้า
-   ลงทะเบียนการมาถึงของสินค้าที่ส่งคืนโดยตรงจากบรรทัดใบสั่งส่งคืนสินค้า โดยใช้การ**ลงทะเบียน**ฟังก์ชัน

ในระหว่างกระบวนการมาถึง ส่งคืนสินค้ามีการรวมเข้ากับกระบวนการทั่วไปสำหรับสินค้าที่มาถึงคลังสินค้า นอกจากนี้กระบวนการมาถึงยังสนับสนุนการสร้างใบสั่งตรวจสอบสินค้าสำหรับสินค้าที่ส่งคืนที่ต้องผ่านการตรวจสอบความแยกต่างหาก

### <a name="identify-products-in-the-arrival-overview-list"></a>ระบุผลิตภัณฑ์ในตลาดภาพรวมการมาถึง

การ**ภาพรวมการมาถึง**หน้าแสดงรายการทั้งหมดที่วางแผนไว้ขาเข้ามาถึง **หมายเหตุ:**ต้องดำเนินมาถึงจากใบสั่งส่งคืนสินค้าแยกต่างหากจากธุรกรรมมาถึงชนิดอื่น หลังจากที่คุณได้ระบุเป็นแพคเกจขาเข้าบน**ภาพรวมการมาถึง**หน้า (ตัวอย่างเช่น โดยใช้เอกสารมาพร้อมกับ RMA), บนบานหน้าต่างการดำเนินการ คลิ**เริ่มต้นมาถึง**ในการสร้าง และการกำหนดค่าเริ่มต้นสมุดรายวันการมาถึงที่ตรงกับการมาถึง

### <a name="edit-the-arrival-journal"></a>แก้ไขสมุดรายวันการมาถึง

โดยการตั้งค่า**ตรวจสอบจัดการ**ตัวเลือกเพื่อ**ใช่**คุณสามารถสร้างใบสั่งตรวจสอบสินค้าสำหรับบรรทัดการส่งคืนสินค้าได้ ถ้าบรรทัดจะถูกส่งไปตรวจสอบสำหรับการตรวจสอบ คุณไม่สามารถระบุรหัสการโอนการครอบครอง **หมายเหตุ:**ถ้าคุณตั้งค่า**ตรวจสอบจัดการ**ตัวเลือกเพื่อ**ใช่**ในกลุ่มแบบจำลองสินค้าคงคลังของสินค้า**จัดการการตรวจสอบสินค้า**ตัวเลือกบน**บรรทัดสมุดรายวัน**หน้าจะทำเครื่องหมายสำหรับบรรทัดสมุดรายวันการมาถึง และไม่สามารถเปลี่ยนแปลง ถ้ามีส่งบรรทัดไปตรวจสอบ คุณต้องระบุคลังสินค้าตรวจสอบสินค้าที่เหมาะสม หากไม่ได้ส่งบรรทัดมาถึงสำหรับการตรวจสอบ เจ้าหน้าที่มาถึงคลังสินค้าต้องระบุรหัสการโอนการครอบครองโดยตรงบนบรรทัดสมุดรายวันการมาถึง และจากนั้น ลงรายการบัญชีสมุดรายวันการมาถึง ถ้าไม่ควรกำหนดรหัสการโอนการครอบครองเดียวกันให้ปริมาณทั้งหมดของบรรทัดการส่งคืนสินค้า หรือถ้ายังไม่ได้รับปริมาณทั้งหมดของบรรทัด คุณต้องแยกบรรทัด เมื่อคุณแยกบรรทัดสมุดรายวันการมาถึง คุณยังสามารถแบ่งบรรทัดการส่งคืน (**SalesLine**) และสร้างรหัสล็อตใหม่ คุณสามารถแยกบรรทัดได้ โดยการลดปริมาณของบรรทัดสมุดรายวันการมาถึง เมื่อสมุดรายวันถูกลงรายการบัญชี ใบส่งคืนสร้างบรรทัดใหม่ที่มีสถานะเป็น**คาดหมาย**สำหรับปริมาณเหลืออยู่ นอกจากนี้คุณสามารถแบ่งบรรทัด โดยการคลิก**ฟังก์ชัน**&gt;**แยก**ได้

### <a name="process-the-quarantine-order"></a>ประมวลผลใบสั่งตรวจสอบสินค้า

ถ้ามีส่งผลิตภัณฑ์ที่ส่งคืนสำหรับการตรวจสอบที่คลังสินค้าตรวจสอบสินค้า การประมวลผลเสร็จสมบูรณ์แล้วในใบสั่งตรวจสอบสินค้า ใบสั่งตรวจสอบสินค้าหนึ่งใบถูกสร้างขึ้นสำหรับแต่ละบรรทัดมาถึงที่ส่งไปตรวจสอบ รหัสการโอนการครอบครองบ่งชี้ผลลัพธ์ของกระบวนการตรวจสอบ คุณสามารถแบ่งใบสั่งตรวจสอบสินค้า เช่นเดียวกับที่คุณสามารถแบ่งสมุดรายวันการมาถึง ถ้าคุณแบ่งใบสั่งตรวจสอบสินค้า ต้องทำให้เกิดการแบ่งที่สอดคล้องกันของบรรทัดการส่งคืน หลังจากที่มีป้อนรหัสการโอนการครอบครอง ทำใบสั่งตรวจสอบสินค้า โดยใช้การ**สิ้นสุด**ฟังก์ชันหรือ**รายงานเมื่อเสร็จสมบูรณ์**ฟังก์ชัน ถ้าคุณเลือก**รายงานเมื่อเสร็จสมบูรณ์**ที่มาถึงใหม่ถูกสร้างในคลังสินค้าที่กำหนดไว้ คุณสามารถประมวลผลนี้มาถึงแล้ว โดยใช้การ**ภาพรวมการมาถึง**หน้าได้ ถ้าคุณมาถึงที่มีต้นกำเนิดจากใบสั่งตรวจสอบสินค้า คุณไม่สามารถเปลี่ยนแปลงรหัสการโอนการครอบครองที่ถูกกำหนดในระหว่างการตรวจสอบ ถ้าคุณดำเนินการใบสั่งตรวจสอบสินค้า โดยใช้การ**สิ้นสุด**ฟังก์ชัน ล็อตมีการลงทะเบียนโดยอัตโนมัติ บางครั้ง สินค้าอาจถูกส่งกลับจากการตรวจสอบเพื่อการจัดส่งและการรับแผนก ตัวอย่างเช่น ตัวตรวจสอบการตรวจสอบสินค้าอาจไม่ทราบตำแหน่งที่จะเก็บสินค้าในสินค้าคงคลัง ในกรณีนี้ บันทึกการจัดส่งที่เกี่ยวข้องต้องปรับปรุงการลงทะเบียนอย่างถูกต้อง และดำเนินการกับรหัสการโอนการครอบครองที่ระบุได้เนื่องจาก มีการตรวจสอบสินค้า ยอมรับใบรับสามารถส่งไปยังลูกค้าเมื่อมีการลงทะเบียนรายการส่งคืน การ**ตอบรับการส่งคืนสินค้า**รายงานคล้ายคลึงกับเอกสารใบสั่งส่งคืนสินค้า การ**ตอบรับการส่งคืนสินค้า**รายงานไม่ได้จัดทำสมุดราย หรือมิฉะนั้น การลงทะเบียนในระบบ และไม่จำเป็นต้องใช้ขั้นตอนในกระบวนการใบสั่งส่งคืนสินค้า

## <a name="replace-a-product"></a>แทนผลิตภัณฑ์
มีสองวิธีสำหรับการจัดการการเปลี่ยนทดแทนผลิตภัณฑ์:

-   **การเปลี่ยนทดแทนล่วงหน้า**– แทนผลิตภัณฑ์ก่อนได้รับสินค้าส่งคืนจากลูกค้า
-   **แทนที่ ด้วยรหัสการโอนการครอบครอง**โดยอัตโนมัติ– สร้างบรรทัดใบสั่งเปลี่ยนทดแทนใหม่

### <a name="up-front-replacement"></a>การเปลี่ยนทดแทนล่วงหน้า

ในการเปลี่ยนทดแทนล่วงหน้า สินค้าทดแทนสามารถจัดส่งไปยังลูกค้าก่อนจะส่งคืนสินค้า วิธีนี้มีประโยชน์ถ้า ตัวอย่าง สินค้า ส่วนประกอบเครื่องจักรที่ไม่สามารถเอาออกยกเว้นแต่มีการเกิดขึ้นของอะไหล่ หรือ ถ้าคุณเพียงต้องการให้ลูกค้ามีผลิตภัณฑ์ทดแทนโดยเร็วที่สุด ใบสั่งเปลี่ยนทดแทนล่วงหน้าคือ ใบสั่งขายที่เป็นอิสระต่อกัน ข้อมูลหัวข้อถูกเตรียมใช้งานจากลูกค้า และรายละเอียดของบรรทัดถูกเตรียมใช้งานจากใบสั่งส่งคืนสินค้า คุณสามารถแก้ไข ประมวลผล และลบใบสั่งเปลี่ยนทดแทนเป็นอิสระจากใบสั่งส่งคืนสินค้า เมื่อคุณลบใบสั่งเปลี่ยนทดแทน คุณได้รับข้อความว่า มีการสร้างใบสั่งเป็นใบสั่งเปลี่ยนทดแทน ภาพประกอบต่อไปนี้แสดงขั้นตอนสำหรับการเปลี่ยนทดแทนล่วงหน้า  

[![กระบวนการเปลี่ยนทดแทนล่วงหน้า](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn04.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn04.png)  

ใบสั่งส่งคืนสินค้ารวมถึงการอ้างอิงถึงใบสั่งเปลี่ยนทดแทน ถ้ามีสร้างใบสั่งเปลี่ยนทดแทนล่วงหน้าสำหรับใบสั่งส่งคืนสินค้าก่อนมีการส่งคืนสินค้ามีตำหนิ คุณไม่สามารถเลือกรหัสการโอนการครอบครองสำหรับเปลี่ยนทดแทนหลังจากที่มีการส่งคืนสินค้ามีตำหนิ

### <a name="replacement-by-disposition-code"></a>แทนที่ ด้วยรหัสการโอนการครอบครอง

ถ้าคุณจัดส่งสินค้าทดแทนให้กับลูกค้า และคุณใช้**แทนและเศษซาก**หรือ**แทนและเครดิต**ดำเนินการโอนการครอบครองบนใบสั่งส่งคืน ใช้กระบวนการที่จะแสดงในภาพประกอบต่อไปนี้  

[![กระบวนการแทนที่เมื่อมีใช้รหัสการโอนการครอบครอง](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn05.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn05.png)  

สินค้าทดแทนจะถูกจัดส่ง โดยใช้ขึ้นอยู่กับใบสั่งขาย ใบสั่งขายแทน ใบสั่งขายนี้ถูกสร้างขึ้นเมื่อสร้างบันทึกการจัดส่งสำหรับใบสั่งส่งคืนสินค้า หัวข้อใบสั่งใช้ข้อมูลจากลูกค้าที่มีการอ้างอิงบนหัวข้อใบสั่งส่งคืนสินค้า รายละเอียดของบรรทัดจะถูกรวบรวมจากข้อมูลที่ป้อนไว้ในแบบ**สินค้าทดแทน**หน้า การ**สินค้าทดแทน**หน้าต้องกรอกสำหรับบรรทัดที่มีการดำเนินการโอนการครอบครองที่ขึ้นต้น ด้วยคำว่า "แทน" อย่างไรก็ตาม ไม่มีปริมาณหรือข้อมูลเฉพาะตัวของสินค้าทดแทนจะถูกตรวจสอบ หรือจำกัด ลักษณะการทำงานนี้ได้ สำหรับกรณีที่ลูกค้าต้องการสินค้าเดียวกัน แต่ ในการตั้งค่าคอนฟิกที่แตกต่างกัน หรือขนาด และกรณีที่ลูกค้าต้องการสินค้าที่แตกต่างกันโดยสิ้นเชิง โดยค่าเริ่มต้น ป้อนสินค้าเหมือนกันในการ**สินค้าทดแทน**หน้า อย่างไรก็ตาม คุณสามารถเลือกสินค้าแตกต่างกัน โดยที่ฟังก์ชันมีการตั้งค่า **หมายเหตุ:**คุณสามารถแก้ไข และลบใบสั่งขายแทนหลังจากที่มีสร้างขึ้น

## <a name="generate-a-packing-slip"></a>สร้างบันทึกการจัดส่ง
ก่อนที่จะได้รับสินค้าที่ส่งคืนไปยังสินค้าคงคลัง คุณต้องอัพเดตบันทึกการจัดส่งสำหรับใบสั่งที่เป็นของสินค้า เช่นเดียวกับกระบวนการปรับปรุงใบแจ้งหนี้มีการปรับปรุงของธุรกรรมทางการเงิน บันทึกการจัดส่งการปรับปรุงขั้นตอนการอัพเดตจริงของเรกคอร์ดในสินค้าคงคลังที่มีอยู่ อีกนัยหนึ่ง กระบวนการนี้ยอมรับการเปลี่ยนแปลงสินค้าคงคลัง ในกรณีของการส่งคืนสินค้า ขั้นตอนต่าง ๆ ที่กำหนดให้กับการดำเนินการโอนการครอบครองถูกนำไปใช้ในระหว่างการส่งการอัพเดตบันทึกการจัดส่ง เมื่อคุณสร้างบันทึกการจัดส่ง เหตุการณ์ต่อไปนี้เกิดขึ้น:

-   ในคลังสินค้า กระบวนการมาตรฐานถูกใช้เพื่อดำเนินการรับสินค้าจริง ลงรายการบัญชีในบัญชีแยกประเภทจะถูกสร้างขึ้นถ้ากลุ่มของแบบจำลองสินค้าคงคลัง (**ลงรายการบัญชีสต็อกทางกายภาพ**) และพารามิเตอร์ลูกหนี้ (**ลงรายการบัญชีบันทึกการจัดส่งในบัญชีแยกประเภท**) ถูกตั้งค่าอย่างเหมาะสม
-   สินค้าที่ได้ถูกกำหนดให้กับการดำเนินการโอนการครอบครองที่ประกอบด้วยคำว่า "ของเสีย" เป็นของเสีย และขาดทุนจากสินค้าคงคลังลงรายการบัญชีในบัญชีแยกประเภท
-   สินค้าที่ทำเครื่องหมายด้วยการ**กลับไปยังลูกค้า**ดำเนินการโอนการครอบครองจะได้รับ และจัดส่งให้กับลูกค้า รายการเหล่านี้จะไม่มีผลสุทธิในสินค้าคงคลัง
-   แทนที่จะสร้างใบสั่งขาย ใบสั่งขายนี้เป็นไปตามรายละเอียด**สินค้าทดแทน**หน้า

คุณสามารถสร้างบันทึกการจัดส่งเฉพาะบรรทัดที่มีสถานะเป็นส่งคืน**ลงทะเบียน**และเฉพาะ สำหรับปริมาณทั้งหมดบนบรรทัดการส่งคืนสินค้าได้ ถ้ามีบรรทัดหลายบรรทัดบนใบสั่งส่งคืนสินค้า**ลงทะเบียน**สถานะ คุณสามารถสร้างบันทึกการจัดส่งสำหรับชุดย่อยของรายการ โดยการลบบรรทัดจาก**ลงรายการบัญชีบันทึกการจัดส่ง**หน้าได้ ส่งกลับค่าบางส่วนมีกำหนดในรูปแบบของบรรทัดใบสั่งส่งคืนสินค้า ไม่ในแง่ของการจัดส่งใบสั่งส่งคืนสินค้า ดังนั้น ถ้าคุณได้รับปริมาณทั้งหมดที่ระบุไว้ในบรรทัดใบสั่งส่งคืนสินค้าหนึ่ง แต่คุณได้รับสิ่งใดจากบรรทัดในใบสั่งส่งคืน การจัดส่งไม่ได้ส่งเป็นบางส่วน อย่างไรก็ตาม ถ้าบรรทัดใบสั่งส่งคืนสินค้าต้องว่า 10 หน่วยของสินค้าส่งคืน แต่คุณได้รับเฉพาะสี่หน่วย การจัดส่งถูกส่งเป็นบางส่วน ถ้าไม่มีรับสินค้าส่งคืนที่คาดไว้ทั้งหมด คุณสามารถตั้งค่าการจัดส่ง และรอสำหรับเวลาเหลือของปริมาณที่ส่งคืนมาถึง อีกวิธีหนึ่งคือ คุณสามารถลงทะเบียน และลงรายการบัญชีปริมาณบางส่วน เป็นส่วนหนึ่งของกระบวนการสำหรับการลงรายการบัญชีบันทึกการจัดส่ง คุณสามารถเชื่อมโยงหมายเลขอ้างอิงบันทึกการจัดส่งจากเอกสารการจัดส่งของลูกค้ากับบรรทัดใบสั่ง ความสัมพันธ์นี้หรือไม่ก็ได้ และสำหรับการอ้างอิงเท่านั้น จะไม่สร้างการปรับปรุงใด ๆ เกี่ยวกับธุรกรรม โดยทั่วไป คุณสามารถข้ามการบันทึกกระบวนการจัดส่ง และไปยังการออกใบแจ้งหนี้โดยตรงได้ ขั้นตอนที่คุณจะดำเนินการในระหว่างการสร้างบันทึกการจัดส่งสินค้าในกรณีนี้ จะเสร็จสมบูรณ์ในระหว่างการออกใบแจ้งหนี้

## <a name="generate-an-invoice"></a>สร้างใบแจ้งหนี้
ถึงแม้ว่าการ**ใบสั่งส่งคืน**หน้าประกอบด้วยข้อมูลและการดำเนินการที่จำเป็นเพื่อจัดการข้อมูลเชิงลักษณะพิเศษของใบสั่งส่งคืนสินค้า คุณต้องใช้**ใบสั่งขาย**หน้าเพื่อทำกระบวนการออกใบแจ้งหนี้ องค์กรของคุณสามารถแล้วออกอินวอยซ์ใบสั่งส่งคืนสินค้าและใบสั่งขายในเวลาเดียวกัน และบุคคลเดียวสามารถทำให้เสร็จสมบูรณ์กระบวนการออกใบแจ้งหนี้ ตามความจำเป็น เมื่อต้องการดูใบสั่งส่งคืนสินค้าจากการ**สั่งขาย**หน้า คลิกการเชื่อมโยงสำหรับหมายเลขใบสั่งขายเพื่อเปิดใบสั่งขายที่เกี่ยวข้อง คุณยังสามารถค้นหาใบสั่งส่งคืนสินค้าบนการ**ใบสั่งขายทั้งหมด**หน้าได้ ส่งคืนใบสั่งเป็นใบสั่งขายที่มีชนิดของใบสั่งของ**ใบสั่งส่งคืน**

### <a name="credit-correction"></a>การแก้ไขสินเชื่อ

เป็นส่วนหนึ่งของกระบวนการออกใบแจ้งหนี้ ตรวจสอบว่า ค่าธรรมเนียมเบ็ดเตล็ดใด ๆ ไม่ถูกต้องหรือไม่ เมื่อต้องการทำให้การลงรายการบัญชีแยกประเภทเป็น การแก้ไข (บันทึก), ลองใช้**เครดิตการแก้ไข**ตัวเลือกบน**อื่น ๆ**แท็บของ**การลงรายการบัญชีใบแจ้งหนี้**page เมื่อคุณลงรายการบัญชีใบแจ้งหนี้/ใบลดหนี้ **หมายเหตุ:**โดยค่าเริ่มต้น**เครดิตการแก้ไข**ตัวเลือกระบบจะเรียก**ใบลดหนี้เป็นการแก้ไข**ตัวเลือกบน**พารามิเตอร์ลูกหนี้**หน้าได้ถูกเปิดใช้งาน อย่างไรก็ตาม เราขอแนะนำว่า คุณไม่ลงรายการบัญชีส่งกลับ ด้วยการบันทึก

## <a name="create-intercompany-return-orders"></a>สร้างใบสั่งส่งคืนสินค้าระหว่างบริษัท
สามารถทำให้เสร็จสมบูรณ์ใบสั่งส่งคืนสินค้าระหว่างบริษัทสองแห่งภายในองค์กรของคุณ สถานการณ์สมมติต่อไปนี้ได้รับการสนับสนุน:

-   ส่งกลับค่าอย่างง่ายระหว่างบริษัทระหว่างบริษัทสองแห่งที่มีส่วนร่วมในความสัมพันธ์ระหว่างบริษัท
-   กลุ่มที่กำหนดไว้เมื่อสร้างใบสั่งส่งคืนสินค้าของลูกค้าในบริษัทผู้ขาย ระหว่างบริษัท
-   มีเครือข่ายระหว่างบริษัทที่สร้างขึ้นเมื่อสร้างใบสั่งส่งคืนสินค้าของผู้จัดจำหน่ายในบริษัทจัดซื้อ
-   ส่งกลับการจัดส่งการจัดส่งโดยตรงระหว่างลูกค้าภายนอกและบริษัทสองแห่งที่มีส่วนร่วมในความสัมพันธ์ระหว่างบริษัท

### <a name="setup"></a>การตั้งค่า

ภาพประกอบต่อไปนี้การตั้งค่าขั้นต่ำที่จำเป็นสำหรับบริษัทสองแห่งจะมีส่วนร่วมในความสัมพันธ์ระหว่างบริษัท และใช้ประโยชน์ของการค้าระหว่างบริษัท  

[![Minimum setup](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn06.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn06.png)  

ในสถานการณ์สมมติต่อไปนี้ CompBuy บริษัทผู้ซื้อ และ CompSell เป็นบริษัทผู้จัดจำหน่าย โดยปกติ บริษัทผู้จัดจำหน่ายเป็นผู้จัดส่งสินค้า ไปยังบริษัทผู้ซื้อ หรือ ใน สถานการณ์ จัดส่งสินค้าที่จัดส่งโดยตรงโดยตรงไปยังลูกค้าปลายทาง ใน CompBuy ผู้จัดจำหน่าย IC\_CompSell ถูกกำหนดเป็นปลายทางระหว่างบริษัทที่เกี่ยวข้องกับบริษัท CompSell ในเวลาเดียวกัน ใน CompSell ลูกค้า IC\_CompBuy ถูกกำหนดเป็นปลายทางระหว่างบริษัทที่เกี่ยวข้องกับบริษัท CompBuy ต้องกำหนดรายละเอียดของนโยบายการดำเนินการที่เหมาะสมและการแม็ปค่าในทั้งสองบริษัท ในสถานการณ์สมมติการจัดส่งสินค้าจัดส่งโดยตรง ผิดระหว่างบริษัทใบสั่งส่งคืน ซึ่งเป็นใบสั่งขายระหว่างบริษัท ถูกสร้างขึ้นในบริษัทผู้จัดจำหน่าย หมายเลข RMA ของใบสั่งส่งคืนสินค้าระหว่างบริษัทสามารถเบิกจากลำดับหมายเลข RMA ใน CompSell หรือสามารถถูกคัดลอกจากหมายเลข RMA ที่ถูกกำหนดให้กับใบสั่งส่งคืนสินค้าดั้งเดิมใน CompBuy ตั้งค่าหมายเลข RMA บนใบ**PurchaseRequisition**นโยบายการดำเนินการใน CompBuy ตรวจสอบการกระทำเหล่านี้ ถ้าหมายเลข RMA จะซิงโครไนส์ คุณควรวางแผนเพื่อลดความเสี่ยงในหมายเลข clashes ถ้าบริษัทสองแห่งใช้ลำดับหมายเลขเดียวกัน

### <a name="simple-intercompany-returns"></a>การส่งคืนสินค้าระหว่างบริษัทอย่างง่าย

สถานการณ์สมมตินี้เกี่ยวข้องกับบริษัทสองแห่งในองค์กรเดียวกัน ดังที่แสดงในภาพประกอบต่อไปนี้  

[![การส่งคืนระหว่างบริษัทอย่างง่าย](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn07.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn07.png)  

สามารถสร้างห่วงโซ่ใบสั่งเมื่อมีสร้างใบสั่งส่งคืนสินค้าของผู้จัดจำหน่ายในบริษัทผู้ซื้อ หรือใบสั่งส่งคืนสินค้าของลูกค้าถูกสร้างขึ้นในบริษัทผู้จัดจำหน่าย Dynamics 365 สำหรับการดำเนินการสร้างใบสั่งที่สอดคล้องกันในบริษัทอื่น และช่วยให้แน่ใจว่า ส่วนหัวและบรรทัดรายละเอียดของผู้จัดจำหน่ายส่งคืนใบสั่งแสดงใบสั่งการส่งคืนการตั้งค่าจากลูกค้า ใบสั่งส่งคืนสินค้าที่สร้างขึ้นสามารถรวม หรือไม่รวมการอ้างอิง (**ค้นหาใบสั่งขาย**) กับใบแจ้งหนี้ของลูกค้าที่มีอยู่ได้ บันทึกการจัดส่งและใบแจ้งหนี้ของใบสั่งสองจะสามารถประมวลผลแต่ละรายการ ตัวอย่างเช่น คุณไม่จำเป็นต้องสร้างบันทึกการจัดส่งสำหรับใบสั่งส่งคืนสินค้าของผู้จัดจำหน่ายก่อนที่คุณสร้างบันทึกการจัดส่งสำหรับใบสั่งส่งคืนสินค้าของลูกค้า

### <a name="direct-delivery-shipment-returns-among-three-parties"></a>ส่งกลับการจัดส่งการจัดส่งโดยตรงระหว่างฝ่ายที่สาม

สามารถสร้างสถานการณ์สมมตินี้ถ้าการขายก่อนหน้านี้ของใบ**จัดส่งโดยตรง**ชนิดเสร็จเรียบร้อย และถ้าใบแจ้งหนี้กับ ลูกค้าที่มีอยู่ในบริษัทที่โต้ตอบกับลูกค้าได้ ในภาพประกอบต่อไปนี้ บริษัท CompBuy มีก่อนหน้านี้ที่ขาย และผลิตภัณฑ์ให้กับลูกค้า Extern ที่ออกใบแจ้งหนี้ ผลิตภัณฑ์ถูกจัดส่งจากบริษัท CompSell โดยตรงไปยังลูกค้าผ่านทางห่วงโซ่ใบสั่งระหว่างบริษัท  

[![ส่งกลับการจัดส่งการจัดส่งโดยตรงระหว่างฝ่ายที่สาม](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn08.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn08.png)  

ถ้าลูกค้า Extern ต้องการการส่งคืนผลิตภัณฑ์ มีสร้างใบสั่งส่งคืนสินค้า (RMA02) สำหรับลูกค้าที่อยู่ในบริษัท CompBuy การสร้างเครือข่ายในระหว่างบริษัท ใบสั่งส่งคืนสินค้าต้องทำเครื่องหมายสำหรับการจัดส่งโดยตรง เมื่อคุณใช้แบบ**ค้นหาใบสั่งขาย**ฟังก์ชันเพื่อรับใบแจ้งหนี้ของลูกค้าเพื่อกลับ สำเร็จห่วงโซ่ใบสั่งระหว่างบริษัทที่ประกอบด้วยเอกสารต่อไปนี้:

-   **ใบสั่งส่งคืนสินค้าดั้งเดิม:** RMA02 (บริษัท CompBuy)
-   **ใบสั่งซื้อ:** PO02 (บริษัท CompBuy)
-   **ใบสั่งส่งคืนสินค้าระหว่างบริษัท:** RMA\_00032 (บริษัท CompSell)

หลังจากเครือข่ายระหว่างบริษัทจัดส่งสินค้าโดยตรงถูกสร้างขึ้น การจัดการทางกายภาพและการประมวลผลของการส่งคืนทั้งหมดต้องเกิดในบริบทของการส่งคืนใบสั่งระหว่างบริษัท RMA\_00032 ในบริษัท CompSell ผลิตภัณฑ์ไม่ได้รับในบริษัท CompBuy เมื่อมีกำหนดรหัสการโอนการครอบครอง ใบสั่งส่งคืนสินค้าระหว่างบริษัทจะถูกซิงโครไนส์กับต้นฉบับใบสั่งส่งคืนสินค้าเมื่อต้องการอนุญาตให้มีการออกใบแจ้งหนี้ของใบสั่งต้นฉบับเหมาะสม

## <a name="post-to-the-ledger"></a>ลงรายการบัญชีในบัญชีแยกประเภท
การลงรายการบัญชีแยกประเภทที่สร้างขึ้นเมื่อออกอินวอยซ์ใบสั่งส่งคืนสินค้าที่จะมีผลต่อ โดยการตั้งค่าที่สำคัญและพารามิเตอร์บาง:

-   **ราคาต้นทุนการส่งคืนสินค้า**– สำหรับรูปแบบสินค้าคงคลังอื่นที่ไม่ใช่**ต้นทุนมาตรฐาน**,**ราคาต้นทุนการส่งคืนสินค้า**พารามิเตอร์เป็นตัวกำหนดต้นทุนของสินค้าเมื่อมีการยอมรับกลับเข้าไปในสินค้าคงคลัง หรือกำจัดเป็นเศษซาก เมื่อต้องการคำนวณการคิดมูลค่าสินค้าคงคลังถูกต้อง เป็นสิ่งสำคัญที่คุณตั้งค่าการ**กลับราคาต้นทุน**พารามิเตอร์อย่างถูกต้อง ถ้าคุณใช้การ**ค้นหาใบสั่งขาย**ฟังก์ชันเพื่อสร้างบรรทัดใบสั่งส่งคืนสินค้าที่มีการอ้างอิงถึงใบแจ้งหนี้ของลูกค้า การ**กลับราคาต้นทุน**ค่าจะเท่ากับราคาต้นทุนของสินค้าที่ขาย มิฉะนั้น ค่าราคาต้นทุนมาจากการตั้งค่าสินค้า หรือคุณสามารถป้อนด้วยตนเอง
-   **การแก้ไข/บันทึกสินเชื่อ**–**เครดิตการแก้ไข**พารามิเตอร์บน**การลงรายการบัญชีใบแจ้งหนี้**หน้าพิจารณาว่า บัญชีควร บันทึกไว้ เป็นรายการ (DR/CR) เป็นบวก หรือ เป็นการแก้ไข ลบรายการ

ในตัวอย่างที่ทำตาม ส่งคืนสินค้าราคาต้นทุนจะแสดงเป็น**ราคาต้นทุนสินค้าคงคลัง**

### <a name="example-1-the-return-order-doesnt-reference-a-customer-invoice"></a>ตัวอย่างที่ 1: ใบสั่งส่งคืนสินค้าไม่ได้อ้างอิงใบแจ้งหนี้ของลูกค้า

ใบสั่งส่งคืนสินค้าไม่อ้างอิงใบแจ้งหนี้ของลูกค้า สินค้าที่ส่งคืนถูกเครดิต การ**เครดิตการแก้ไข**ไม่ได้เลือกพารามิเตอร์ใบแจ้งหนี้ใบสั่งส่งคืนสินค้า หรือบันทึกสินเชื่อ ถูกสร้าง  

[![ใบสั่งส่งคืนสินค้าไม่ได้อ้างอิง invoic เป็นลูกค้า](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn09.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn09.png)  

**หมายเหตุ:**ราคาสินค้าหลักที่จะใช้เป็นค่าเริ่มต้นสำหรับการ**ราคาต้นทุนการส่งคืนสินค้า**พารามิเตอร์ ราคาเริ่มต้นที่แตกต่างจากราคาต้นทุนในเวลาของการตัดสินค้าจากคลังของสินค้าคงคลัง ดังนั้น ทั้งที่อยู่ว่า เป็นการแพ้ 3 ได้เกิดขึ้น นอกจากนี้ ใบสั่งส่งคืนสินค้าไม่ได้รวมส่วนลดที่ให้กับลูกค้าในใบสั่งขาย ดังนั้น เครดิตมากเกินไปเกิดขึ้น

### <a name="example-2-credit-correction-is-selected-for-the-return-order"></a>ตัวอย่างที่ 2: เลือกการแก้ไขสินเชื่อสำหรับใบสั่งส่งคืนสินค้า

ตัวอย่างที่ 2 จะเหมือนกับตัวอย่าง 1 แต่**เครดิตการแก้ไข**พารามิเตอร์ถูกเลือกเมื่อมีสร้างใบแจ้งหนี้ใบสั่งส่งคืนสินค้า  

[![ใบสั่งส่งคืนสินค้าที่มีเลือกการแก้ไขสินเชื่อ](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn10.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn10.png)  

**หมายเหตุ:**จะมีป้อนการลงรายการบัญชีในบัญชีแยกประเภทเป็นการแก้ไขค่าลบ

### <a name="example-3-the-return-order-line-is-created-by-using-the-find-sales-order-function"></a>ตัวอย่างที่ 3: สร้าง โดยใช้ฟังก์ชันการค้นหาใบสั่งขายบรรทัดใบสั่งส่งคืนสินค้า

ในตัวอย่างนี้ บรรทัดใบสั่งส่งคืนสินค้าถูกสร้างขึ้น โดยใช้การ**ค้นหาใบสั่งขาย**ฟังก์ชัน การ**เครดิตการแก้ไข**พารามิเตอร์ที่ไม่ถูกเลือกเมื่อมีสร้างใบแจ้งหนี้  

[![บรรทัดใบสั่งที่สร้างขึ้น โดยใช้การค้นหาใบสั่งการขายส่งคืน](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn11.png)](https://msdynamics.blob.core.windows.net/media/2017/02/SalesReturn11.png)  

**หมายเหตุ:****ส่วนลด**และ**ราคาต้นทุนการส่งคืนสินค้า**ถูกตั้งค่าอย่างถูกต้อง ดังนั้น ย้อนกลับที่แน่นอนของใบแจ้งหนี้ของลูกค้าเกิดขึ้น
