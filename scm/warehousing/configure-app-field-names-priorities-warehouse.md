# <a name="configure-app-field-names-in-warehousing-app"></a>การตั้งค่าคอนฟิกชื่อเขตข้อมูลของโปรแกรมประยุกต์ในแอพลิเคชัน Warehousing

หัวข้อนี้อธิบายวิธีการกำหนด และการตั้งค่าคอนฟิกชื่อเขตข้อมูลโปรแกรมประยุกต์ของคลังสินค้าและระดับความสำคัญใน 365 Dynamics สำหรับการดำเนินงาน 

**หมายเหตุ:**หัวข้อนี้ใช้กับลักษณะการทำงานในการจัดการคลังสินค้า ไม่สามารถใช้กับลักษณะการทำงานในการบริหารสินค้าคงคลัง Dynamics 365 สำหรับการดำเนินงาน - คลังสินค้าเป็นโปรแกรมประยุกต์ที่คุณสามารถใช้เพื่อทำงานในคลังสินค้า คุณสามารถกำหนด และตั้งค่าคอนฟิกของชื่อเขตข้อมูลที่ใช้ในโปรแกรมประยุกต์ ตลอดจนกำหนดระดับความสำคัญที่ควรจะกำหนดชื่อเขตข้อมูล หัวข้อนี้อธิบายวิธีการกำหนด และการกำหนดค่าคลังสินค้าชื่อเขตข้อมูลของโปรแกรมประยุกต์เหล่านี้และระดับความสำคัญ และวิธีการใช้ใน Dynamics 365 สำหรับการดำเนินงาน - คลังสินค้า สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการกำหนดค่าการเชื่อมต่อกับ 365 Dynamics สำหรับการดำเนินงาน - คลังสินค้า การอ้างอิงถึงบทช่วยสอน[ติดตั้ง และตั้งค่าคอนฟิก 365 Dynamics สำหรับการดำเนินงาน - คลังสินค้า](install-configure-warehousing-app.md)

<a name="configure-warehouse-app-field-names"></a>ชื่อเขตข้อมูลโปรแกรมประยุกต์ของคลังสินค้าที่ตั้งค่าคอนฟิก
===================================

เมื่อคุณใช้ Dynamics 365 สำหรับการดำเนินงาน - คลังสินค้าบนอุปกรณ์มือถือของคุณ คุณสามารถกำหนดค่าวิธีควรแสดงข้อมูลเมตาของอุปกรณ์ของคุณบน**คลังสินค้าชื่อเขตข้อมูลโปรแกรมประยุกต์**หน้าได้ ในบริษัทใหม่ใน 365 Dynamics สำหรับการดำเนินการ เลือก**สร้างการตั้งค่าเริ่มต้น**ในการสร้างชื่อเขตข้อมูลทั้งหมดที่จะถูกใช้ในเวิร์กโฟลว์ของอุปกรณ์เคลื่อนคลังสินค้า และกำหนดโหมดการป้อนข้อมูลที่ต้องการและชนิดการป้อนข้อมูลเหล่านั้น หลังจากที่คุณได้สร้างชื่อเขตข้อมูลทั้งหมด คุณสามารถเลือกตัวเลือกสำหรับการป้อนค่าต่อไปนี้

<a name="configure-warehouse-app-field-priority"></a>กำหนดค่าระดับความสำคัญฟิลด์แอพลิเคชันของคลังสินค้า
======================================

ในการ**คลังสินค้าระดับความสำคัญของฟิลด์แอพลิเคชัน**หน้า คุณสามารถใส่ชื่อของเขตข้อมูลเป็นกลุ่มของระดับความสำคัญที่แตกต่างกันได้ ซึ่งจะทำการตัดสินใจว่า ควรจะแสดงข้อมูลบนหน้าภารกิจหลักเมื่อคนงานคลังสินค้าทำงานโดยใช้โปรแกรมประยุกต์ ถ้าคุณคลิ**สร้างการตั้งค่าเริ่มต้น**ชุดค่าเริ่มต้นของระดับความสำคัญของกลุ่มจะถูกสร้างขึ้น คุณสามารถสร้างกลุ่มการระดับความสำคัญได้มากเท่าที่จำเป็นได้ แต่จะแสดงเพียงสามระดับความสำคัญของกลุ่มบนหน้างาน เมื่อ 365 Dynamics สำหรับการดำเนินงานส่งข้อมูลเมตาไปโปรแกรมประยุกต์ ดังกล่าวจะกำหนดแต่ละเขตข้อมูลลำดับความสำคัญแบบสัมพัทธ์โดยขึ้นอยู่กับกลุ่มของระดับความสำคัญ และโปรแกรมประยุกต์จะแสดงระดับความสำคัญสามกลุ่มแรกอยู่ในข้อมูลเมตาที่หน้างาน ส่วนเหลือของข้อมูลเมตาล้นจะแสดงบนเพจรายละเอียดรอง ตารางต่อไปนี้แสดงตัวอย่างของลำดับความสำคัญ 5 กลุ่ม

ตัวอย่างเช่น เมื่อผู้ปฏิบัติงานที่คลังสินค้ากำลังทำงานบนอุปกรณ์เคลื่อนที่ ถ้าข้อมูลเมตาที่จะแสดงในโปรแกรมประยุกต์ประกอบด้วยเขตข้อมูลต่อไปนี้:

-   สินค้า
-   ปริมาณ
-   หน่วยวัด
-   คำอธิบายสินค้า
-   ขนาดและตำแหน่งที่ตั้ง

ตามคลังสินค้า app ฟิลด์ระดับความสำคัญที่ตั้งค่าไว้ในตารางข้างต้น 3 แถวต่อไปนี้ของข้อมูลจะแสดงที่หน้างาน:

-   แถวที่ 1: หน่วยวัดสินค้า ปริมาณ
-   แถวที่ 2: คำอธิบายรายการ
-   แถว 3: ขนาด

ข้อมูลเมตาที่เหลือ เช่น ที่ตั้ง จะไม่แสดงที่หน้างาน แต่จะแสดงบนหน้ารายละเอียด เมื่อต้องการเรียนรู้เพิ่มเติม และดูตัวอย่างของส่วนติดต่อผู้ใช้ การอ้างอิงถึงการติดประกาศบล็อก[Announcing 365 Dynamics สำหรับการดำเนินงาน - คลังสินค้า](https://blogs.msdn.microsoft.com/dynamicsaxscm/2017/01/20/announcing-dynamics-365-for-operations-warehousing/)
