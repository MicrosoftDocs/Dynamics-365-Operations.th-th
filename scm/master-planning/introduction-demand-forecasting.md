# <a name="demand-forecasting-overview"></a>ภาพรวมการคาดการณ์ความต้องการ

การคาดการณ์ความต้องการถูกใช้ในการคาดการณ์ความต้องการอิสระจากใบสั่งขายและความต้องการแบบอิสระที่จุดที่มีการแยกใดๆสำหรับใบสั่งของลูกค้า ความต้องการรับการปรับปรุงการคาดการณ์การลดกฎให้เป็นโซลูชันที่เหมาะสมที่สุดสำหรับการกำหนดเองโดยรวม

ในการสร้างการคาดการณ์พื้นฐาน สรุปธุรกรรมในอดีตจะถูกส่งไปยังบริการ Microsoft Azure Machine Learning ที่โฮสต์ใน Azure เนื่องจากบริการนี้ไม่ได้ใช้ร่วมกันระหว่างผู้ใช้ จึงสามารถกำหนดค่าเองได้โดยง่ายเพื่อให้ตรงกับความต้องการอุตสาหกรรมเฉพาะ คุณสามารถใช้ Dynamics 365 สำหรับการดำเนินงานเพื่อให้เห็นภาพการคาดการณ์ ปรับการคาดการณ์ และดูตัวบ่งชี้ประสิทธิภาพหลัก (Kpi) เกี่ยวกับความถูกต้องของการคาดการณ์

## <a name="key-features-of-demand-forecasting"></a>ลักษณะการทำงานหลักของการคาดการณ์ความต้องการ
นี่คือคุณลักษณะหลักบางประการของการคาดการณ์ความต้องการ:

-   สร้างการคาดการณ์พื้นฐานทางสถิติซึ่งเป็นไปตามข้อมูลประวัติ
-   ใช้ชุดไดนามิกของมิติการคาดการณ์
-   แสดงให้เห็นแนวโน้มความต้องการ ช่วงเวลาของความเชื่อมั่น และการปรับปรุงของการคาดการณ์
-   อนุมัติการคาดการณ์ที่มีการปรับปรุงซึ่งจะใช้ในกระบวนการวางแผน
-   ลบจุดนอกขอบเขต
-   สร้างการวัดความถูกต้องของการคาดการณ์

## <a name="major-themes-in-demand-forecasting"></a>ชุดรูปแบบหลักในการคาดการณ์ความต้องการ
มีการใช้ชุดรูปแบบหลักสามชุดในการคาดการณ์ความต้องการ:

-   **Modularity** – การคาดการณ์ความต้องการเป็นส่วนย่อยๆและง่ายต่อการตั้งค่าคอนฟิก คุณสามารถเปิดใช้งานฟังก์ชันการทำงาน และปิด โดยการเปลี่ยนคีย์การตั้งค่าคอนฟิกที่**การค้า**&gt;**การคาดการณ์สินค้าคงคลัง**&gt;**คาดการณ์อุปสงค์**ได้
-   **นำมาใช้ใหม่ของกองซ้อน Microsoft** – Microsoft เปิดใช้แพลตฟอร์มเรียนรู้เครื่องจักรในเดือน 2015 กุมภาพันธ์ เรียนรู้เครื่องจักร ซึ่งขณะนี้เป็นส่วนหนึ่งของชุด Microsoft Cortana Analytics ช่วยให้คุณได้อย่างรวดเร็ว และง่ายดายสร้างทดลองวิเคราะห์ทำนาย เช่นทดลองประเมินความต้องการ โดยใช้อัลกอริทึม R หรือภาษาการเขียนโปรแกรม Python และอินเทอร์เฟซแบบลาก และวางอย่างง่าย
    -   คุณสามารถดาวน์โหลด 365 Dynamics ที่สำหรับความต้องการการดำเนินงานที่คาดการณ์ทดลอง เปลี่ยนแปลงให้ตรงกับความต้องการทางธุรกิจของคุณ เผยแพร่เป็นบริการบนเว็บบน Azure และใช้ในการสร้างการคาดการณ์ความต้องการ การทดลองจะพร้อมใช้งานสำหรับการดาวน์โหลดถ้าคุณซื้อไว้เป็น 365 Dynamics สำหรับการดำเนินการบอกรับเป็นสมาชิกสำหรับผู้วางแผนการผลิตเป็นผู้ใช้ระดับองค์กร
    -   คุณสามารถดาวน์โหลดข้อมูลใดๆของการทดลองการคาดการณ์ความต้องการที่พร้อมใช้งานในปัจจุบันจาก [Cortana Analytics Gallery](https://gallery.cortanaanalytics.com/)ได้ ขณะ 365 Dynamics สำหรับความต้องการการดำเนินงานที่คาดการณ์ทดลองได้รวมเข้ากับ 365 Dynamics สำหรับการดำเนินงานโดยอัตโนมัติ ลูกค้าและคู่ค้าต้องจัดการกับการรวมของการทดลองที่จะดาวน์โหลดมาจาก[แกลเลอรี Analytics Cortana](https://gallery.cortanaanalytics.com/) ดังนั้น ทดลองจาก[แกลเลอรี Analytics Cortana](https://gallery.cortanaanalytics.com/)ไม่ได้เป็นตรงไปตรงมาเพื่อใช้เป็น 365 Dynamics สำหรับความต้องการการดำเนินงานที่คาดการณ์ทดลอง คุณต้องแก้ไขรหัสของการทดลองเพื่อให้พวกเขาใช้การ 365 Dynamics สำหรับการดำเนินการแอพลิเคชันเขียนโปรแกรมอินเทอร์เฟซ (API)
    -   คุณสามารถสร้างการทดลองของคุณเองใน Microsoft Azure Machine Learning Studio เผยแพร่เป็นบริการใน Azure และใช้ในการสร้างการคาดการณ์ความต้องการ
    -   ถ้าคุณไม่ต้องการประสิทธิภาพสูง หรือถ้าคุณไม่ต้องการให้มีการประมวลผลข้อมูลจำนวนมาก คุณสามารถใช้ระดับอิสระของ Machine Learning เราขอแนะนำให้คุณเริ่มต้นจากระดับนี้เสมอ โดยเฉพาะอย่างยิ่งในระหว่างการใช้งานและระยะของการทดสอบ หากคุณต้องการประสิทธิภาพที่สูงขึ้นและที่จัดเก็บข้อมูลเพิ่มเติม คุณสามารถใช้ระดับมาตรฐานของ Machine Learning ได้ ระดับนี้จำเป็นต้องมีการบอกรับเป็นสมาชิกของ Azure และเกี่ยวข้องกับต้นทุนเพิ่มเติม สำหรับรายละเอียดเกี่ยวกับการกำหนดราคาของ Machine Learning ให้ดู <http://aka.ms/machine-learning-price-info>
-   **การคาดการณ์การลดที่จุดใด ๆ decoupling** – อุปสงค์ที่คาดการณ์ใน Dynamics 365 สำหรับรุ่นของการดำเนินงานในการทำงานนี้ ซึ่งช่วยให้คุณคาดการณ์ขึ้นอยู่ และขึ้นอยู่กับความต้องการ ณจุดใดก็ decoupling

## <a name="basic-flow-in-demand-forecasting"></a>ขั้นตอนพื้นฐานในการคาดการณ์ความต้องการ
แผนภาพต่อไปนี้แสดงขั้นตอนพื้นฐานสำหรับการคาดการณ์ความต้องการ 

[![ไดอะแกรมบทนำการคาดการณ์ความต้องการ](./media/demand-forecasting-introduction.png)](./media/demand-forecasting-introduction.png)

ความต้องการการคาดการณ์เริ่มต้นสร้างใน 365 Dynamics สำหรับการดำเนินงาน ข้อมูลเกี่ยวกับธุรกรรมในอดีตจาก 365 Dynamics สำหรับการดำเนินงานฐานข้อมูลที่เกี่ยวกับธุรกรรมการรวบรวม และเติมข้อมูลตารางกำลังเตรียม ภายหลังมีติดตามตารางการจัดเตรียมไปยังบริการการเรียนรู้ของเครื่องจักร โดยดำเนินการกำหนดเองที่น้อยที่สุด คุณสามารถเชื่อมต่อแหล่งข้อมูลต่าง ๆ ลงในตารางกำลังเตรียม แหล่งข้อมูลอาจรวมถึง Microsoft Excel แฟ้ม แฟ้มที่ใช้จุลภาคคั่นระหว่างค่า (CSV) และข้อมูลจาก Microsoft Dynamics AX 2009 และ Microsoft Dynamics AX 2012 ดังนั้น คุณสามารถสร้างการคาดการณ์ความต้องการที่พิจารณาข้อมูลในอดีตที่จะแพร่กระจายจากหลายระบบ อย่างไรก็ตาม ข้อมูลหลักเช่น ชื่อสินค้าและหน่วยของการวัด ต้องเหมือนกันระหว่างแหล่งข้อมูลต่างๆ

ถ้าคุณใช้ 365 Dynamics สำหรับความต้องการการดำเนินงานที่คาดการณ์ทดลองเรียนรู้เครื่องจักร พวกเขาหาเหมาะสมระหว่าง 5 เวลาชุดคาดการณ์วิธีคำนวณพื้นฐานการคาดการณ์ พารามิเตอร์สำหรับวิธีการคาดการณ์เหล่านี้จะถูกจัดการใน 365 Dynamics สำหรับการดำเนินงาน 

คาดการณ์ ข้อมูลประวัติ และการเปลี่ยนแปลงใด ๆ ที่ทำไว้ในการคาดการณ์ความต้องการในการคำนวณซ้ำก่อนหน้า นั้นสามารถใช้งานได้ใน 365 Dynamics สำหรับการดำเนินงาน 

คุณสามารถใช้ Dynamics 365 สำหรับการดำเนินงานเพื่อให้เห็นภาพ และปรับเปลี่ยนการคาดการณ์พื้นฐาน การปรับปรุงด้วยตนเองต้องได้รับการอนุญาต ก่อนที่จะสามารถใช้การคาดการณ์สำหรับการวางแผน

## <a name="limitations"></a>การจำกัด
การคาดการณ์ความต้องการใน 365 Dynamics สำหรับการดำเนินงานเป็นเครื่องมือที่ช่วยให้ลูกค้าในอุตสาหกรรมการผลิตที่สร้างกระบวนการคาดการณ์ มีความสามารถในการทำงานหลักของโซลูชันการคาดการณ์ความต้องการ และถูกออกแบบมาเพื่อให้คุณสามารถได้อย่างง่ายดายขยาย ความต้องการการคาดการณ์ที่อาจไม่เป็นดีที่สุดให้พอดีสำหรับลูกค้าในอุตสาหกรรมเช่นขายปลีก ส่ง คลังสินค้า ขนส่ง หรือบริการอื่น ๆ ระดับมืออาชีพ
