# <a name="retail-peripherals-overview"></a>ภาพรวมของอุปกรณ์ต่อพ่วงขายปลีก

หัวข้อนี้อธิบายแนวคิดที่เกี่ยวข้องกับอุปกรณ์ต่อพ่วงขายปลีก จะอธิบายวิธีการต่าง ๆ ที่สามารถเชื่อมต่ออุปกรณ์ต่อพ่วงเพื่อจุดประสงค์ของการขาย (POS) และคอมโพเนนต์ที่รับผิดชอบในการจัดการการเชื่อมต่อกับ POS

<a name="concepts"></a>แนวคิด
--------

### <a name="pos-registers"></a>เครื่องบันทึกเงินสด POS

นำทาง: คลิก**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**การลงทะเบียน** จุดประสงค์ของเครื่องบันทึกเงินสดการขาย (POS) เป็นเอนทิตีที่ใช้ในการกำหนดลักษณะของอินสแตนซ์ที่ระบุของ POS ลักษณะเหล่านี้รวมถึงส่วนกำหนดค่าของฮาร์ดแวร์หรือการตั้งค่าสำหรับอุปกรณ์ต่อพ่วงขายปลีกที่จะใช้ในการลงทะเบียน และร้านค้าที่ลงทะเบียนถูกแมปกับ การแสดงผลทางภาพระดับสำหรับผู้ใช้ที่ลงทะเบียนในการลงทะเบียนนั้น

### <a name="devices"></a>อุปกรณ์

นำทาง: คลิก**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**อุปกรณ์** อุปกรณ์คือเอนทิตีที่แสดงถึงอินสแตนซ์ที่มีอยู่จริงของอุปกรณ์ที่ถูกแม็ปไปยังเครื่องบันทึกเงินสด POS เมื่อมีสร้างอุปกรณ์ ถูกแมปไปยัง POS เครื่องบันทึกเงินสด เอนทิตีอุปกรณ์จะติดตามข้อมูลเกี่ยวกับเวลาที่เครื่องบันทึกเงินสด POS ถูกเรียกใช้ ชนิดของไคลเอนต์ที่ถูกใช้ และแพคเกจของแอพลิเคชันที่มีการปรับใช้กับอุปกรณ์เฉพาะ อุปกรณ์สามารถแม็ปไปยังชนิดของแอพลิเคชันต่อไปนี้: Retail POS สมัยใหม่ เมฆ Retail POS, Retail POS สมัยใหม่ – Windows Phone, Retail POS สมัยใหม่ – Android และสมัยใหม่ Retail POS – iOS ได้

### <a name="retail-modern-pos"></a>Retail Modern POS

POS สมัยใหม่คือ โปรแกรม POS สำหรับ Microsoft Windows สามารถปรับใช้ได้บนระบบปฏิบัติการ Windows 10 (OSs)

### <a name="cloud-pos"></a>เมฆ POS

เมฆ POS จะใช้เบราว์เซอร์รุ่นของโปรแกรม POS สมัยใหม่ที่สามารถเข้าถึงได้ในเว็บเบราว์เซอร์

### <a name="modern-pos-for-ios"></a>POS สมัยใหม่สำหรับ iOS

POS สมัยใหม่สำหรับ iOS เป็นโปรแกรม POS สมัยใหม่ที่สามารถปรับใช้บนอุปกรณ์ iOS รุ่นที่ใช้ iOS

### <a name="modern-pos-for-android"></a>POS สมัยใหม่สำหรับ Android

POS สมัยใหม่สำหรับ Android เป็นโปรแกรม POS สมัยใหม่ที่สามารถปรับใช้ได้บนอุปกรณ์ Android รุ่นที่ใช้ Android

### <a name="pos-peripherals"></a>อุปกรณ์ต่อพ่วง POS

อุปกรณ์ต่อพ่วง POS เป็นอุปกรณ์ที่สนับสนุนอย่างชัดเจนสำหรับฟังก์ชัน POS อุปกรณ์ต่อพ่วงเหล่านี้โดยทั่วไปจะแบ่งออกเป็นระดับชั้นเฉพาะ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับระดับชั้นเหล่านี้ ดูส่วน "คลาสของอุปกรณ์" ของหัวข้อนี้

### <a name="hardware-station"></a>สถานีฮาร์ดแวร์

นำทาง: คลิก**ขายปลีกและพาณิชย์**&gt;**สถานี**&gt;**ร้านค้าปลีก**&gt;**ทั้งหมดขายปลีกร้านค้า** เลือกร้านค้า และคลิก FastTab **สถานีฮาร์ดแวร์** การ**สถานีฮาร์ดแวร์**ตั้งค่าเป็นการตั้งค่าระดับช่องสัญญาณที่จะใช้ในการกำหนดอินสแตนซ์ที่จะปรับใช้ตรรกะการต่อพ่วงขายปลีก การตั้งค่านี้ในระดับสถานีจะใช้เพื่อระบุลักษณะของฮาร์ดแวร์สถานีดังกล่าว นอกจากนี้ยังใช้กับสถานีฮาร์ดแวร์รายการที่พร้อมใช้งานสำหรับอินสแตนซ์ POS สมัยใหม่ในร้านค้าที่กำหนด อุปกรณ์ฮาร์ดแวร์ที่อยู่ในโปรแกรม POS สมัยใหม่สำหรับ Windows อุปกรณ์ฮาร์ดแวร์สามารถปรับเป็นอิสระต่อกันเป็นโปรแกรมแบบสแตนด์อโลน Microsoft Internet Information Services (IIS) ในกรณีนี้ จะสามารถเข้าถึงผ่านเครือข่าย

### <a name="hardware-profile"></a>โพรไฟล์ฮาร์ดแวร์

นำทาง: คลิก**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โพรไฟล์ POS**&gt;**โพรไฟล์ฮาร์ดแวร์** โพรไฟล์ฮาร์ดแวร์คือ รายการของอุปกรณ์ที่ถูกกำหนดค่าสำหรับเครื่องบันทึกเงินสด POS หรือสถานีฮาร์ดแวร์ โพรไฟล์ฮาร์ดแวร์สามารถแม็ปกับเครื่องบันทึกเงินสด POS หรือสถานีฮาร์ดแวร์โดยตรง

## <a name="devices-classes"></a>คลาสของอุปกรณ์
อุปกรณ์ต่อพ่วง POS โดยทั่วไปจะแบ่งออกเป็นระดับชั้น ส่วนนี้อธิบาย และให้ภาพรวมของอุปกรณ์ที่สนับสนุน POS สมัยใหม่

### <a name="printer"></a>เครื่องพิมพ์

เครื่องพิมพ์รวมถึงเครื่องพิมพ์ใบเสร็จ POS ดั้งเดิมและเครื่องพิมพ์เต็มหน้า เครื่องพิมพ์ที่ได้รับการสนับสนุนผ่านวัตถุเชื่อมโยง และการฝังตัวสำหรับอินเทอร์เฟซการโปรแกรมควบคุม (OPOS Retail POS) และ Microsoft Windows ได้ถึงสองเครื่องพิมพ์สามารถใช้พร้อมกัน ความสามารถนี้สนับสนุนสถานการณ์สมมติที่รับสินค้าของลูกค้า cash-and-carry ถูกพิมพ์บนเครื่องพิมพ์ใบเสร็จ ในขณะที่การสั่งซื้อของลูกค้า ซึ่งมีข้อมูลเพิ่มเติม พิมพ์บนเครื่องพิมพ์เต็มหน้า เครื่องพิมพ์ใบเสร็จสามารถเชื่อมต่อโดยตรงกับคอมพิวเตอร์ผ่าน USB การเชื่อมต่อกับเครือข่ายโดยใช้อีเทอร์เน็ต หรือเชื่อมต่อผ่าน Bluetooth

### <a name="scanner"></a>สแกนเนอร์

ได้ถึงสองบาร์โค้ด สแกนเนอร์สามารถใช้พร้อมกัน ความสามารถนี้สนับสนุนสถานการณ์สมมติที่สแกนเนอร์ที่มือถือเพิ่มเติมจำเป็นเพื่อสแกนสินค้าจำนวนมาก หรือขนาดใหญ่ ในขณะที่สแกนเนอร์ฝังตัวคงที่ใช้สำหรับรายการส่วนใหญ่มีขนาดมาตรฐาน เพื่อเร่งความเร็วเวลาเช็คเอาท์ สแกนเนอร์สามารถได้รับการสนับสนุนผ่าน OPOS แพลตฟอร์ม Windows สากล (UWP), หรืออินเทอร์เฟซของเครื่องอ่านบัตรเครดิตแป้นพิมพ์ สามารถใช้ USB หรือบลูทูธเพื่อเชื่อมต่อสแกนเนอร์กับคอมพิวเตอร์

### <a name="msr"></a>MSR

อ่านแถบแม่เหล็ก USB หนึ่ง (MSR) สามารถตั้งค่าได้ โดยใช้โปรแกรมควบคุม OPOS ถ้าคุณต้องการใช้ MSR แบบสแตนด์อโลนสำหรับธุรกรรมการชำระเงินทางอิเล็กทรอนิกส์ (EFT) การโอนย้าย MSR ต้องได้รับการจัดการ โดยตัวเชื่อมต่อการชำระเงิน สามารถใช้ MSRs แบบสแตนด์อโลนสำหรับลูกค้าสมาชิกรายการ พนักงาน ในการเข้าระบบ และ รายการบัตรของขวัญ อย่างเป็นอิสระจากตัวเชื่อมต่อการชำระเงิน

### <a name="cash-drawer"></a>ลิ้นชักเงินสด

ลิ้นชักเงินสดสองสามารถได้รับการสนับสนุนสำหรับแต่ละโพรไฟล์ฮาร์ดแวร์ ความสามารถนี้เปิดใช้งานสองกะที่ใช้งานอยู่สำหรับแต่ละการลงทะเบียนจะพร้อมใช้งานในเวลาเดียวกัน ในกรณีของกะที่ใช้ร่วมกัน หรือแบบลิ้นชักเงินสดที่ถูกใช้ โดยอุปกรณ์ POS เคลื่อนหลายชนิดในเวลาเดียวกัน ลิ้นชักเงินสดเดียวเท่านั้นได้รับอนุญาตสำหรับแต่ละโพรไฟล์ฮาร์ดแวร์ ลิ้นชักเงินสดสามารถเชื่อมต่อโดยตรงกับคอมพิวเตอร์ผ่าน USB เชื่อมต่อไปยังเครือข่าย หรือเชื่อมต่อกับเครื่องพิมพ์ใบเสร็จผ่านอินเทอร์เฟซ RJ12 ในบางกรณี ลิ้นชักเงินสดนอกจากนี้ยังสามารถเชื่อมต่อผ่าน Bluetooth

### <a name="line-display"></a>จอแสดงผลรายการ

แสดงบรรทัดจะใช้เพื่อแสดงผลิตภัณฑ์ ยอดดุลของธุรกรรม และข้อมูลอื่น ๆ ที่เป็นประโยชน์ให้กับลูกค้าในระหว่างทรานแซคชัน จอแสดงผลรายการหนึ่งสามารถเชื่อมต่อไปยังคอมพิวเตอร์โดยใช้ USB โดยใช้โปรแกรมควบคุม OPOS

### <a name="signature-capture"></a>การเก็บบันทึกลายเซ็น

อุปกรณ์เก็บบันทึกลายเซ็นสามารถเชื่อมต่อกับคอมพิวเตอร์ผ่าน USB ได้โดยตรง โดยใช้โปรแกรมควบคุม OPOS เมื่อมีการกำหนดค่าการเก็บบันทึกลายเซ็น ลูกค้าถูกหยุดให้สมัครสมาชิกบนอุปกรณ์ หลังจากที่มีให้ลายเซ็น นั้นจะแสดงขึ้นให้พนักงานเก็บเงินสำหรับการยอมรับ

### <a name="scale"></a>สเกล

สเกลจะสามารถเชื่อมต่อไปยังคอมพิวเตอร์ผ่าน USP โดยใช้โปรแกรมควบคุม OPOS ผลิตภัณฑ์ที่ถูกทำเครื่องหมายเป็นผลิตภัณฑ์ "Weighed" ถูกเพิ่มลงในธุรกรรม POS อ่านน้ำหนักจากเครื่องชั่ง เพิ่มผลิตภัณฑ์ลงในธุรกรรม และใช้ปริมาณที่ระบุมาตราส่วน

### <a name="pin-pad"></a>แป้นกดรหัส

แผ่น (PIN) หมายเลขรหัสประจำตัวได้รับการสนับสนุนผ่าน OPOS แต่พวกเขาต้องถูกจัดการผ่านทางตัวเชื่อมต่อการชำระเงิน

### <a name="secondary-display"></a>จอแสดงผลรอง

เมื่อมีการกำหนดค่าจอแสดงผลรอง จอแสดงผล Windows หมายเลข 2 ใช้แสดงรายละเอียดพื้นฐาน วัตถุประสงค์ของจอแสดงผลรองจะสนับสนุนส่วนขยายของซอฟต์แวร์อิสระ (ISV) ของผู้จัดจำหน่าย เนื่องจากออกจากกล่อง จอแสดงผลรองไม่สามารถกำหนดค่า และแสดงเนื้อหาที่จำกัด

### <a name="payment-device"></a>อุปกรณ์การชำระเงิน

สนับสนุนอุปกรณ์การชำระเงินถูกนำมาใช้ผ่านตัวเชื่อมต่อการชำระเงิน อุปกรณ์การชำระเงินสามารถปฏิบัติหนึ่งหรือหลายฟังก์ชันที่ให้อุปกรณ์คลาสอื่น ๆ ตัวอย่างเช่น อุปกรณ์การชำระเงินสามารถทำหน้าที่เป็นตัวอ่าน การ์ด/MSR จอแสดงผลรายการ อุปกรณ์เก็บบันทึกลายเซ็น หรือแป้นกดรหัส สนับสนุนสำหรับอุปกรณ์การชำระเงินถูกนำมาใช้อย่างเป็นอิสระจากการสนับสนุนอุปกรณ์แบบสแตนด์อโลนที่ให้ไว้สำหรับอุปกรณ์อื่น ๆ ที่รวมอยู่ในโพรไฟล์ฮาร์ดแวร์

## <a name="supported-interfaces"></a>อินเทอร์เฟซที่ได้รับการสนับสนุน
### <a name="opos"></a>OPOS

เมื่อต้องการช่วยรับประกันว่า จะสามารถใช้กับ Microsoft Dynamics 365 ช่วงมากที่สุดของอุปกรณ์สำหรับการดำเนินงาน - ขายปลีก OLE สำหรับมาตรฐานอุตสาหกรรม POS อยู่แพลตฟอร์มอุปกรณ์ต่อพ่วงขายปลีกหลักที่ได้รับการสนับสนุนใน 365 Dynamics Microsoft สำหรับการดำเนินงาน - ขายปลีก OLE สำหรับ POS มาตรฐานถูกผลิตโดยชาติขายปลีกสหพันธรัฐ (NRF), ซึ่งสร้างโพรโทคอลการติดต่อสื่อสารมาตรฐานอุตสาหกรรมสำหรับอุปกรณ์ต่อพ่วงขายปลีก OPOS ถูกดำเนินการอย่างกว้างขวาง adopted ของ OLE สำหรับ POS มาตรฐาน ได้รับการพัฒนาในแบบ mid-1990s และได้รับการปรับปรุงนับแต่นั้นหลายครั้ง OPOS แสดงสถาปัตยกรรมของโปรแกรมควบคุมอุปกรณ์ที่ช่วยให้นักพัฒนาสามารถรวมของฮาร์ดแวร์ POS กับระบบ POS ที่ใช้ Windows OPOS ควบคุมการจัดการการติดต่อสื่อสารระหว่างฮาร์ดแวร์ที่เข้ากันได้กับซอฟต์แวร์ POS ตัวควบคุม OPOS ประกอบด้วยสองส่วน:

-   **วัตถุตัวควบคุม**– วัตถุตัวควบคุมสำหรับคลาอุปกรณ์ (เช่นแสดงบรรทัด) ให้อินเทอร์เฟซสำหรับโปรแกรมซอฟต์แวร์ บริการปรึกษา Monroe ([www.monroecs.com](http://www.monroecs.com/)) แสดงชุดของออบเจ็กต์ควบคุม OPOS ที่ทราบว่าเป็นวัตถุตัวควบคุมทั่วไป (CCOs) มาตรฐาน CCOs จะใช้เมื่อต้องการทดสอบคอมโพเนนต์ POS ของ 365 Dynamics Microsoft สำหรับการดำเนินงาน - ขายปลีก ดังนั้น การทดสอบช่วยรับประกัน ซึ่งถ้า 365 Dynamics Microsoft สำหรับการดำเนินงาน - สนับสนุนการขายปลีกที่มีคลาสอุปกรณ์ผ่าน OPOS อุปกรณ์หลายชนิดสามารถได้รับการสนับสนุน ให้ว่า ผู้ผลิตมีวัตถุที่ให้บริการที่มีอยู่สำหรับ OPOS คุณไม่จำเป็นต้องทดสอบชนิดอุปกรณ์แต่ละอย่างชัดเจน
-   **วัตถุที่ให้บริการ**– วัตถุที่ให้บริการให้สามารถสื่อสารระหว่างตัวควบคุมวัตถุ (CCO) และอุปกรณ์ โดยทั่วไป วัตถุที่ให้บริการสำหรับอุปกรณ์ที่ได้รับจากผู้ผลิตอุปกรณ์ อย่างไรก็ตาม ในบางกรณี คุณอาจจำเป็นต้องดาวน์โหลดวัตถุที่ให้บริการจากเว็บไซต์ของผู้ผลิต ตัวอย่างเช่น วัตถุที่ให้บริการใหม่กว่าอาจจะพร้อมใช้งาน เมื่อต้องการค้นหาอยู่ของเว็บไซต์ของผู้ผลิต ดูเอกสารประกอบฮาร์ดแวร์ของคุณ

[![ควบคุมวัตถุและวัตถุที่ให้บริการ](./media/retail_peripherals_overview01.png)](./media/retail_peripherals_overview01.png)สนับสนุนสำหรับการใช้งาน OLE OPOS POS ช่วยรับประกันว่า หากผู้ผลิตอุปกรณ์และผู้เผยแพร่ที่ POS ใช้มาตรฐานอย่างถูกต้อง ระบบ POS และอุปกรณ์สามารถทำงานร่วมกัน แม้ว่าพวกเขาไม่ได้ทดสอบก่อนหน้านี้เข้าด้วยกัน **หมายเหตุ:** OPOS สนับสนุนไม่สามารถรับประกันการสนับสนุนสำหรับอุปกรณ์ทั้งหมดที่มีไดรเวอร์ OPOS Microsoft Dynamics 365 สำหรับการดำเนินงาน - ขายปลีกต้องแรกสนับสนุนนั้นชนิดอุปกรณ์ หรือคลาส ผ่าน OPOS นอกจากนี้ วัตถุที่ให้บริการอาจเสมอไม่ทันสมัยอยู่เสมอ ด้วยการ CCOs รุ่นล่าสุด นอกจากนี้คุณยังควรตระหนักว่า ทั่วไป คุณภาพของวัตถุที่ให้บริการแตกต่างกันไป

### <a name="windows"></a>Windows

การพิมพ์ใบเสร็จที่ POS จะเหมาะ OPOS OPOS tends จะเร็วกว่าการพิมพ์ผ่านทาง Windows ดังนั้น จึงเป็นความคิดที่ดีที่จะใช้ OPOS โดยเฉพาะในสภาพแวดล้อมการขายปลีกที่มีพิมพ์ใบเสร็จรับเงิน 40 คอลัมน์ และเวลาธุรกรรมต้องเป็นอย่างรวดเร็ว สำหรับอุปกรณ์ส่วนใหญ่ คุณจะใช้ตัวควบคุม OPOS อย่างไรก็ตาม เครื่องพิมพ์ใบเสร็จ OPOS บางรุ่นยังสนับสนุนโปรแกรมควบคุมของ Windows โดยการใช้ไดรเวอร์ของ Windows คุณสามารถเข้าถึงแบบอักษรต่าง ๆ ล่าสุดและเครื่องพิมพ์บนเครือข่ายหนึ่งสำหรับเครื่องบันทึกเงินสดหลาย อย่างไรก็ตาม มีข้อเสียของการใช้โปรแกรมควบคุมของ Windows ต่อไปนี้เป็นบางตัวอย่างของข้อเสียเหล่านี้:

-   เมื่อใช้โปรแกรมควบคุมของ Windows รูปภาพจะแสดงก่อนที่พิมพ์เกิดขึ้น ดังนั้น พิมพ์ tends จะช้ากว่าเครื่องพิมพ์ที่ใช้ควบคุม OPOS
-   อุปกรณ์ที่เชื่อมต่อผ่านเครื่องพิมพ์ ("เดซี่-chained") อาจไม่ทำงานอย่างถูกต้องเมื่อมีใช้ไดรเวอร์ของ Windows ตัวอย่างเช่น อาจไม่มีเปิดลิ้นชักเงินสด หรือเครื่องพิมพ์บันทึกการจัดอาจมีคำคุณคาดหวัง
-   OPOS ยังสนับสนุนชุดของตัวแปรที่เกี่ยวข้องกับเครื่องพิมพ์ใบรับสินค้าขายปลีก เช่นตัดกระดาษหรือจัดพิมพ์เฉพาะ มากขึ้น

ถ้าตัวควบคุม OPOS พร้อมใช้งานสำหรับเครื่องพิมพ์ของ Windows ที่คุณกำลังใช้ เครื่องพิมพ์ควรยังคงทำงานได้อย่างถูกต้องกับ 365 Dynamics Microsoft สำหรับการดำเนินงาน - ขายปลีก

### <a name="universal-windows-platform"></a>แพลตฟอร์ม Windows สากล

UWP ในกรณีของอุปกรณ์ต่อพ่วงขายปลีก เกี่ยวข้องกับการสนับสนุนของ Windows สำหรับอุปกรณ์ Plug and Play เมื่ออุปกรณ์แบบ Plug and Play ที่เชื่อมต่อกับระบบปฏิบัติการ Windows รุ่นที่สนับสนุนชนิดของอุปกรณ์ ไม่มีโปรแกรมควบคุมที่ถูกต้องสำหรับอุปกรณ์ที่จะใช้เป็นจุดมุ่งหมาย ตัวอย่างเช่น ถ้า Windows ตรวจพบอุปกรณ์ Bluetooth ลำโพง ระบบปฏิบัติการทราบว่า มีอุปกรณ์**ลำโพง**คลาสชนิด ดังนั้น และก็ถือว่าอุปกรณ์นั้นเป็นผู้บรรยาย จำเป็นต้องมีการตั้งค่าเพิ่มเติม ในกรณีของอุปกรณ์ POS สามารถต่ออุปกรณ์ USB หลาย และ Windows จะจดจำเหล่านั้นเป็นอุปกรณ์ติดต่อบุคคล (HIDs) อย่างไรก็ตาม ดังกล่าวอาจไม่สามารถตรวจสอบความสามารถในการจัดเตรียมอุปกรณ์ เนื่องจากอุปกรณ์ไม่ได้ระบุคลาส หรือชนิด อุปกรณ์ ใน Windows 10 คลาสของอุปกรณ์สำหรับบาร์โค้ดสแกนเนอร์และ MSRs มีการเพิ่ม ดังนั้น ถ้าอุปกรณ์ประกาศเอง Windows 10 เป็นอุปกรณ์ของคลาสที่เหล่านี้อย่างใดอย่างหนึ่ง Windows จะฟังเหตุการณ์จากอุปกรณ์ตามช่วงเวลาที่เหมาะสม POS สมัยใหม่สนับสนุน UWP MSRs และสแกนเนอร์ ดังนั้น เมื่อพร้อมสำหรับการป้อนข้อมูลจากอุปกรณ์เหล่านี้อย่างใดอย่างหนึ่ง และเชื่อมต่ออุปกรณ์ที่เป็นของคลาสที่เหล่านี้อย่างใดอย่างหนึ่ง อุปกรณ์ที่สามารถใช้ ตัวอย่างเช่น ถ้าเป็น UWP บาร์โค้ดสแกนเนอร์เสียบเข้ากับเครื่องคอมพิวเตอร์ Windows 10 และบาร์โค้ดในการเข้าระบบถูกกำหนดค่าสำหรับ POS สมัยใหม่ บาร์โค้ดสแกนเนอร์จะเปิดใช้งานบนหน้าจอการเข้าสู่ระบบ จำเป็นต้องมีการตั้งค่าเพิ่มเติม ประเภทเพิ่มเติมของจุดของอุปกรณ์ UWP บริการจะมีการเพิ่มลงใน Windows คลาสที่เหล่านี้รวมถึงระดับชั้นสำหรับลิ้นชักเงินสดและเครื่องพิมพ์ใบเสร็จ การสนับสนุนสำหรับคลาสของอุปกรณ์เหล่านี้ใหม่ใน POS สมัยใหม่ค้างอยู่

### <a name="keyboard-wedge"></a>เครื่องอ่านบัตรเครดิตของแป้นพิมพ์

อุปกรณ์เครื่องอ่านบัตรเครดิตแป้นพิมพ์ส่งข้อมูลไปยังคอมพิวเตอร์ราวกับว่าข้อมูลนั้นถูกพิมพ์บนแป้นพิมพ์ ดังนั้น โดยค่าเริ่มต้น ฟิลด์ที่ใช้งานอยู่ที่ POS จะได้รับข้อมูลที่ถูกสแกน หรือรูด ในบางกรณี การทำงานนี้อาจทำให้ชนิดไม่ถูกต้องของข้อมูลจะถูกสแกนลงในเขตข้อมูลไม่ถูกต้อง ตัวอย่างเช่น อาจสแกนบาร์โค้ดลงในเขตข้อมูลสำหรับป้อนข้อมูลบัตรเครดิต ในหลายกรณี ไม่มีตรรกะที่ POS ที่กำหนดว่า ข้อมูลที่ถูกสแกน หรือรูดเป็นบาร์โค้ดหรือรูดบัตร ดังนั้น ข้อมูลถูกจัดการอย่างถูกต้อง อย่างไรก็ตาม เมื่ออุปกรณ์ถูกตั้งค่าเป็น OPOS แทนอุปกรณ์เครื่องอ่านบัตรเครดิตแป้นพิมพ์ มีอยู่ควบคุมได้มากขึ้นว่าข้อมูลจากอุปกรณ์เหล่านั้นสามารถใช้ เนื่องจากเพิ่มเติม "ทราบ" เกี่ยวกับอุปกรณ์ที่ข้อมูลมีต้นกำเนิดมาจาก ตัวอย่างเช่น ถูกจดจำข้อมูลจากสแกนเนอร์บาร์โค้ดเป็นบาร์โค้ดโดยอัตโนมัติ และพบเรกคอร์ดที่เชื่อมโยงในฐานข้อมูลได้อย่างง่ายดาย และรวดเร็วกว่าถ้าการค้นหาสายอักขระทั่วไปถูกใช้ ดังเช่นในกรณีของอุปกรณ์เครื่องอ่านบัตรเครดิตของแป้นพิมพ์

### <a name="native-printer"></a>เครื่องพิมพ์ท้องถิ่น

ดั้งเดิม (หรือ "อุปกรณ์" เป็นชนิดถูกตั้งชื่อในโพรไฟล์ฮาร์ดแวร์) คุณสามารถกำหนดค่าเครื่องพิมพ์เพื่อพร้อมท์ให้ผู้ใช้สามารถเลือกเครื่องพิมพ์ที่ถูกกำหนดค่าสำหรับคอมพิวเตอร์นี้ เมื่อเครื่องพิมพ์ของการ**อุปกรณ์**มีการกำหนดค่าชนิด POS สมัยใหม่พบคำสั่งพิมพ์ มีพร้อมต์ผู้ใช้ต้องเลือกเครื่องพิมพ์ในรายการ ลักษณะการทำงานนี้แตกต่างจากลักษณะการทำงานสำหรับโปรแกรมควบคุมของ Windows เนื่องจากการ**Windows**เครื่องพิมพ์ชนิดโพรไฟล์ฮาร์ดแวร์ไม่แสดงรายการของเครื่องพิมพ์ แทน ต้องว่า เครื่องพิมพ์มีชื่อมีอยู่ในการ**ชื่ออุปกรณ์**เขตข้อมูล

### <a name="windows"></a>Windows

การ**Windows**ใช้สำหรับเครื่องพิมพ์เฉพาะชนิดอุปกรณ์ เมื่อมีการกำหนดค่าเครื่องพิมพ์ของ Windows ในโพรไฟล์ฮาร์ดแวร์ ชื่อเครื่องพิมพ์ที่ระบุต้องมีอยู่ เมื่อ POS สมัยใหม่พบเหตุการณ์ที่พิมพ์ ถ้ามีการกำหนดค่าเครื่องพิมพ์ของ Windows เหตุการณ์จะถูกส่งผ่านไปยังเครื่องพิมพ์ Windows ที่ระบุ ผู้ใช้จะไม่ได้รับพร้อมท์ให้เลือกเครื่องพิมพ์

### <a name="network"></a>เครือข่าย

ลิ้นชักเงินสดเครือข่ายได้ เครื่องพิมพ์ใบเสร็จ และเทอร์มินัลการชำระเงินสามารถใช้ผ่านทางเครือข่าย ผ่านสถานีฮาร์ดแวร์สื่อสาร Interprocess (IPC) ที่มีอยู่ในแอพลิเคชันสมัยใหม่ POS สำหรับ Windows โดยตรง หรือ ผ่านอุปกรณ์ฮาร์ดแวร์ของ IIS สำหรับไคลเอนต์ POS สมัยใหม่อื่น ๆ

## <a name="hardware-station-deployment-options"></a>ตัวเลือกการปรับใช้อุปกรณ์ฮาร์ดแวร์
### <a name="ipc-built-in"></a>IPC (มีอยู่ภายใน)

อุปกรณ์ฮาร์ดแวร์สื่อสาร Interprocess (IPC) มีอยู่ในแอพลิเคชันสมัยใหม่ POS สำหรับ Windows การใช้อุปกรณ์ฮาร์ดแวร์ IPC กำหนดโพรไฟล์ฮาร์ดแวร์กับเครื่องบันทึกเงินสดที่จะใช้โปรแกรมประยุกต์สมัยใหม่ POS สำหรับ Windows จากนั้น สร้างสถานีฮาร์ดแวร์ของการ**Dedicated**ชนิดสำหรับร้านค้าที่จะใช้ลงทะเบียน เมื่อคุณเริ่มใช้งาน POS สมัยใหม่ IPC ฮาร์ดแวร์สถานีจะมีผลบังคับ และอุปกรณ์ต่อพ่วงของ POS ที่ได้รับการกำหนดค่าจะพร้อมใช้งานแล้ว ถ้าคุณเป็นการชั่วคราวไม่จำเป็นต้องมีฮาร์ดแวร์เฉพาะเหตุผลบางประการ ใช้**จัดการฮาร์ดแวร์สถานี**การดำเนินงานเมื่อต้องการปิดความสามารถของอุปกรณ์ฮาร์ดแวร์ POS สมัยใหม่สามารถใช้อุปกรณ์ฮาร์ดแวร์ IPC ที่สามารถสื่อสารกับอุปกรณ์ต่อพ่วงของเครือข่ายโดยตรง

### <a name="iis"></a>IIS

คุณสามารถใช้ IIS หรือรุ่นสแตนด์อะโลนของอุปกรณ์ฮาร์ดแวร์ในสองวิธี ตัวบอกเกี่ยวกับ "IIS" บ่งชี้ว่า แอพลิเคชันของ POS ที่เชื่อมต่อกับอุปกรณ์ฮาร์ดแวร์ผ่านทาง Microsoft Internet Information Services แอพลิเคชันของ POS ที่เชื่อมต่อกับ IIS ฮาร์ดแวร์สถานีผ่านทางเว็บเซอร์วิสที่เรียกใช้บนคอมพิวเตอร์ที่เชื่อมต่ออุปกรณ์ เมื่อมีใช้ IIS สามารถใช้อุปกรณ์ต่อพ่วงที่ขายปลีกที่เชื่อมต่อไปยังสถานีฮาร์ดแวร์ โดยเครื่องบันทึกเงินสด POS ที่อยู่บนเครือข่ายเดียวกันเป็นอุปกรณ์ฮาร์ดแวร์ IIS ได้ เนื่องจากเฉพาะสมัยใหม่ POS สำหรับ Windows มีอยู่แล้วภายในในการสนับสนุนสำหรับอุปกรณ์ต่อพ่วงขายปลีก โปรแกรม POS สมัยใหม่อื่น ๆ ทั้งหมดต้องใช้อุปกรณ์ฮาร์ดแวร์ IIS สามารถสื่อสารกับอุปกรณ์ต่อพ่วง POS ที่ถูกกำหนดค่าในโพรไฟล์ฮาร์ดแวร์ ดังนั้น แต่ละอินสแตนซ์ของอุปกรณ์ฮาร์ดแวร์ IIS ต้องการคอมพิวเตอร์ที่เรียกใช้บริการบนเว็บและโปรแกรมประยุกต์ที่สื่อสารกับอุปกรณ์ อุปกรณ์ฮาร์ดแวร์ IIS ที่ถูกต้องสำหรับโปรแกรมประยุกต์ที่ไม่ใช่ - Windows สมัยใหม่ POS ทั้งหมด

#### <a name="dedicated"></a>เฉพาะเจาะจง

POS สมัยใหม่ใช้ฮาร์ดแวร์สถานีของการ**Dedicated**ชนิดเพื่อตรวจสอบว่า อุปกรณ์ต่อพ่วงเชื่อมต่อโดยตรงไปยังคอมพิวเตอร์ที่กำลังใช้โปรแกรมประยุกต์ที่ อย่างไรก็ตาม**Dedicated**ยังสามารถใช้ชนิดสำหรับสถานีฮาร์ดแวร์ IIS ได้ ในแบบดั้งเดิม ถาวร POS สถานการณ์สมมติที่ใช้ Cloud POS เป็นโปรแกรมประยุกต์ POS, **Dedicated**ใช้ชนิดอุปกรณ์ฮาร์ดแวร์สำหรับ IIS ฮาร์ดแวร์สถานีที่มีการติดตั้งบนคอมพิวเตอร์ที่กำลังเรียกใช้ Cloud POS เดียวกัน จากมุมมองอุปกรณ์ต่อพ่วงขายปลีก ฮาร์ดแวร์สถานีเฉพาะของ IIS ได้ดีกว่าการขายปลีกสนับสนุนอุปกรณ์ต่อพ่วงสำหรับสถานการณ์ POS ดั้งเดิม ถาวร ฮาร์ดแวร์เฉพาะสถานีสนับสนุนอุปกรณ์ต่อพ่วงทั้งหมดที่ได้รับการสนับสนุนในโพรไฟล์ฮาร์ดแวร์

#### <a name="shared"></a>ที่มีส่วนแบ่ง

ฮาร์ดแวร์ที่สถานีมีจุดมุ่งหมายที่จะใช้ โดยอุปกรณ์ POS หลายผ่านหลักสูตรของวันที่ใช้ร่วมกัน ฮาร์ดแวร์ที่สถานีที่ปรับให้เหมาะสมกับลิ้นชักเงินสดเท่านั้นสนับสนุน เครื่องพิมพ์ใบเสร็จ และเทอร์มินัลการชำระเงินที่ใช้ร่วมกัน คุณไม่สามารถเชื่อมต่อโดยตรงแบบสแตนด์อโลนบาร์โค้ดสแกนเนอร์ MSRs แสดงบรรทัด ปรับขนาด หรืออุปกรณ์อื่น ๆ มิฉะนั้น ความขัดแย้งจะเกิดขึ้นเมื่ออุปกรณ์ POS หลายพยายามอ้างสิทธิอุปกรณ์ต่อพ่วงเหล่านั้นในเวลาเดียวกัน นี่เป็นวิธีจัดการความขัดแย้งของอุปกรณ์ที่ได้รับการสนับสนุน:

-   **ลิ้นชักเงินสด**– เปิดลิ้นชักเงินสดผ่านเหตุการณ์ที่จะถูกส่งไปยังอุปกรณ์ เฉพาะปัญหาที่อาจเกิดขึ้นเมื่อเรียกลิ้นชักเงินสดที่เกิดขึ้นถ้ายังได้เปิดลิ้นชักเงินสด ในกรณีของสถานีที่ใช้ร่วมกันของฮาร์ดแวร์ ลิ้นชักเงินสดควรถูกตั้งค่าเป็น**ที่ใช้ร่วมกัน**ในโพรไฟล์ฮาร์ดแวร์ การตั้งค่านี้ป้องกันไม่ให้ POS ว่า ลิ้นชักเงินสดถูกเปิดอยู่เมื่อส่งคำสั่งเปิดการตรวจสอบ
-   **เครื่องพิมพ์ใบเสร็จ**– ถ้ารับสองคำสั่งพิมพ์จะส่งไปสถานีฮาร์ดแวร์พร้อม ๆ กัน หนึ่งคำสั่งอาจสูญหาย ขึ้นอยู่กับอุปกรณ์ อุปกรณ์บางอย่างมีหน่วยความจำภายใน หรือร่วมกันที่สามารถป้องกันไม่ให้ปัญหานี้ ถ้าคำสั่งพิมพ์ไม่เสร็จเรียบร้อยแล้ว พนักงานเก็บเงินได้รับข้อผิดพลาด และสามารถลองคำสั่งพิมพ์จาก POS
-   **การชำระเงินที่เทอร์มินัล**– ถ้าเป็นพนักงานเก็บเงินเพจการแม็ปเพื่อชำระเงินธุรกรรมในการชำระเงินที่เทอร์มินัลที่ถูกใช้แล้ว ข้อความแจ้งให้ทราบว่าพนักงานเก็บเงินใช้เทอร์มินัล และขอให้พนักงานเก็บเงินเพื่อลองอีกครั้งในภายหลัง โดยปกติ พนักงานเก็บเงินสามารถดูเทอร์มินัลถูกใช้แล้ว และจะรอจนกว่าธุรกรรมอื่น ๆ เสร็จสมบูรณ์ก่อนที่จะพยายามอีกครั้งการชำระเงิน

มีการวางแผนการตรวจสอบสำหรับรุ่นต่อไป เพื่อตรวจสอบว่า อุปกรณ์ที่ไม่สนับสนุนจะถูกตั้งค่าสำหรับโพรไฟล์ฮาร์ดแวร์ที่ถูกแมปไปยังสถานีฮาร์ดแวร์ที่ใช้ร่วมกัน ถ้าตรวจพบอุปกรณ์ใด ๆ ที่ไม่สนับสนุน ผู้ใช้จะได้รับข้อความแจ้งว่า อุปกรณ์ไม่ได้รับการสนับสนุนสำหรับสถานีฮาร์ดแวร์ที่ใช้ร่วมกัน ในกรณีของสถานีที่ใช้ร่วมกันของฮาร์ดแวร์ การ**เลือกเมื่อ tendering**ตัวเลือกถูกตั้งค่าเป็น**ใช่**ในระดับเครื่องบันทึกเงินสด จากนั้นมีพร้อมต์ผู้ใช้ POS เพื่อเลือกสถานีฮาร์ดแวร์เมื่อมีเลือกการชำระเงินสำหรับธุรกรรมที่ POS เมื่อมีเลือกสถานีฮาร์ดแวร์เท่าในเวลาของการชำระเงิน การเลือกอุปกรณ์ฮาร์ดแวร์ถูกเพิ่มโดยตรงกับเวิร์กโฟลว์ POS สำหรับสถานการณ์เคลื่อน เป็นสวัสดิการเพิ่มเติม การแสดงรายการบนเทอร์มินัลการชำระเงินไม่ได้ใช้สำหรับสถานการณ์สมมติที่ใช้ร่วมกัน ถ้าใช้เทอร์มินัลการชำระเงินเป็นจอแสดงผลแบบรายการ ผู้ใช้รายอื่นอาจถูกบล็อกจากการใช้เทอร์มินัลนั้นจนกว่าธุรกรรมเสร็จสมบูรณ์ ในสถานการณ์โทรศัพท์มือถือ อาจเพิ่มบรรทัดลงในธุรกรรมรอบระยะเวลายาวขึ้น ดังนั้น**เลือกเมื่อ tendering**ตัวเลือกจำเป็นต้องใช้เพื่อรับประกันถึงการมีอยู่ของอุปกรณ์สูงสุด

### <a name="network-peripherals"></a>อุปกรณ์ต่อพ่วงของเครือข่าย

การแต่งตั้งเครือข่ายสำหรับอุปกรณ์ในโพรไฟล์ฮาร์ดแวร์ช่วยให้ลิ้นชักเงินสด เครื่องพิมพ์ใบเสร็จ และชำระเงินเทอร์มินัลเพื่อเชื่อมต่อผ่านการเชื่อมต่อเครือข่าย

#### <a name="modern-pos-for-windows"></a>POS สมัยใหม่สำหรับ Windows

คุณสามารถระบุที่อยู่ IP สำหรับอุปกรณ์ต่อพ่วงของเครือข่ายในสถานที่สอง ถ้าไคลเอนต์ POS สมัยใหม่ Windows จะใช้ชุดเดียวของอุปกรณ์ต่อพ่วงเครือข่าย คุณควรตั้งค่า IP แอดเดรสสำหรับอุปกรณ์เหล่านั้น โดยใช้การ**การกำหนดค่า IP**ตัวเลือกในบานหน้าต่างการดำเนินการสำหรับการลงทะเบียนตัวเอง ในกรณีของอุปกรณ์เครือข่ายที่จะใช้ร่วมกันระหว่างเครื่องบันทึกเงินสด POS โพรไฟล์ฮาร์ดแวร์ที่มีอุปกรณ์เครือข่ายที่กำหนดไว้สามารถแม็ปได้โดยตรงไปยังสถานีที่ฮาร์ดแวร์ที่ใช้ร่วมกันได้ เมื่อต้องการกำหนดที่อยู่ IP เลือกสถานีที่ฮาร์ดแวร์บน**ร้านค้าปลีก**หน้า และจากนั้น ใช้**การกำหนดค่า IP**ตัวเลือกในการ**สถานีฮาร์ดแวร์**ส่วนการระบุอุปกรณ์เครือข่ายที่ถูกกำหนดให้กับอุปกรณ์ฮาร์ดแวร์ที่ สำหรับฮาร์ดแวร์สถานีที่มีเฉพาะอุปกรณ์เครือข่าย คุณไม่จำเป็นต้องปรับใช้อุปกรณ์ฮาร์ดแวร์ตัวเอง ในกรณีนี้ อุปกรณ์ฮาร์ดแวร์จำเป็นต้องใช้เพียงเพื่อจัดกลุ่มอุปกรณ์เครือข่ายได้ตามตำแหน่งที่ตั้งในร้านค้าปลีกทางแนวคิด

#### <a name="cloud-pos-modern-pos-for-ios-and-modern-pos-for-android"></a>เมฆ POS, POS สมัยใหม่สำหรับ iOS และ POS สมัยใหม่สำหรับ Android

ประกอบด้วยตรรกะที่ไดรฟ์เครือข่ายได้ และเชื่อมต่อทางกายภาพอุปกรณ์ต่อพ่วงอุปกรณ์ฮาร์ดแวร์ ดังนั้น POS ไคลเอนต์ทั้งหมดยกเว้นสมัยใหม่ POS สำหรับ Windows อุปกรณ์ฮาร์ดแวร์ IIS ต้องถูกปรับใช้ และใช้งานเพื่อเปิดใช้งาน POS สามารถสื่อสารกับอุปกรณ์ต่อพ่วง โดยไม่คำนึงถึงว่าอุปกรณ์ต่อพ่วงเหล่านั้นเป็นจริงเชื่อมต่อไปยังสถานีที่ฮาร์ดแวร์ หรือส่งผ่านเครือข่าย

### <a name="opos-device-setup-and-configuration"></a>การตั้งค่าอุปกรณ์ OPOS และตั้งค่าคอนฟิก

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับคอมโพเนนต์ OPOS ให้ดูส่วน "สนับสนุนอินเทอร์เฟซ" ของเอกสารนี้ โดยทั่วไปแล้ว ไดรเวอร์ OPOS ไว้ให้ โดยผู้ผลิตอุปกรณ์ เมื่อมีการติดตั้งโปรแกรมควบคุมอุปกรณ์ OPOS จะเพิ่มคีย์รีจิสทรีของ Windows ในตำแหน่งต่อไปนี้หนึ่ง:

-   **ระบบ 32 บิต:** HKEY\_ท้องถิ่น\_MACHINESOFTWAREOLEforRetailServiceOPOS
-   **ระบบ 64 บิต:** HKEY\_ท้องถิ่น\_MACHINESOFTWAREWOW6432NodeOLEforRetailServiceOPOS

## <a name="configuration-for-supported-scenarios"></a>การตั้งค่าคอนฟิกสำหรับสถานการณ์สมมติที่ได้รับการสนับสนุน
สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการสร้างโพรไฟล์ฮาร์ดแวร์ ดู[กำหนด และรักษาไคลเอนต์สถานี รวมทั้งเครื่องบันทึกเงินสดและฮาร์ดแวร์สถานี](define-maintain-channel-clients-registers-hw-stations.md) **หมายเหตุ:**สำหรับ Microsoft Dynamics 365 สำหรับการดำเนินงานรุ่น 1611 โพรไฟล์อุปกรณ์ฮาร์ดแวร์ไม่ใช้งาน แอตทริบิวต์ที่คุณเคยตั้งค่าในโพรไฟล์ฮาร์ดแวร์การสถานี ในขณะนี้เป็นส่วนหนึ่งของอุปกรณ์ฮาร์ดแวร์ตัวเอง

### <a name="modern-pos-for-windows-with-an-ipc-built-in-hardware-station"></a>สมัยใหม่ POS สำหรับ Windows กับอุปกรณ์ฮาร์ดแวร์ IPC (มีอยู่ภายใน)

การตั้งค่าคอนฟิกนี้จะตั้งค่าคอนฟิกทั่วไปมากที่สุดสำหรับทะเบียน POS ดั้งเดิม ถาวร สำหรับสถานการณ์สมมตินี้ ข้อมูลส่วนกำหนดค่าฮาร์ดแวร์ที่ถูกแมปโดยตรงเพื่อลงทะเบียนตัวเอง หมายเลขเทอร์มินัล EFT ควรถูกกำหนดในการลงทะเบียนตัวเอง เมื่อต้องการตั้งค่าการตั้งค่าคอนฟิกนี้ ให้ทำตามขั้นตอนเหล่านี้

1.  สร้างโพรไฟล์ฮาร์ดแวร์ที่มีการกำหนดค่าอุปกรณ์ต่อพ่วงที่จำเป็นทั้งหมด
2.  แผนผังโพรไฟล์ฮาร์ดแวร์เพื่อเครื่องบันทึกเงินสด POS
3.  สร้างสถานีของฮาร์ดแวร์**Dedicated**ชนิดสำหรับร้านค้าปลีกที่จะใช้ที่เครื่องบันทึกเงินสด POS คำอธิบายหรือไม่ก็ **หมายเหตุ:**คุณไม่จำเป็นต้องตั้งค่าคุณสมบัติอื่น ๆ บนสถานีฮาร์ดแวร์ ข้อมูลทั้งหมดอื่น ๆ ที่จำเป็น เช่นโพรไฟล์ฮาร์ดแวร์ จะมาจากการลงทะเบียนตัวเอง
4.  คลิ**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**
5.  เลือก**1090**กำหนดการกระจายการซิงค์โพรไฟล์ฮาร์ดแวร์ใหม่ไปยังที่เก็บ คลิ**เรียกใช้เดี๋ยวนี้**การซิงค์การเปลี่ยนแปลงกับ POS
6.  เลือก**1040 ติด**กำหนดการกระจายการซิงค์สถานีฮาร์ดแวร์ใหม่ไปยังที่เก็บ คลิ**เรียกใช้เดี๋ยวนี้**การซิงค์การเปลี่ยนแปลงกับ POS
7.  ติดตั้ง และเปิดใช้งานแบบสมัยใหม่ POS สำหรับ Windows
8.  เริ่มการทำงานแบบสมัยใหม่ POS สำหรับ Windows และเริ่มต้นการใช้อุปกรณ์ต่อพ่วงเชื่อมต่ออยู่

### <a name="all-modern-pos-clients-that-have-a-dedicated-iis-hardware-station"></a>ไคลเอนต์ POS สมัยใหม่ทั้งหมดที่มีฮาร์ดแวร์สถานี IIS โดยเฉพาะ

การกำหนดค่านี้สามารถใช้สำหรับไคลเอนต์ POS สมัยใหม่ทั้งหมดที่มีฮาร์ดแวร์ที่ใช้แบบเอกสิทธิ์เฉพาะบุคคล โดย POS หนึ่งสถานี ลงทะเบียนได้ เมื่อต้องการตั้งค่าการตั้งค่าคอนฟิกนี้ ให้ทำตามขั้นตอนเหล่านี้

1.  สร้างโพรไฟล์ฮาร์ดแวร์ที่มีการกำหนดค่าอุปกรณ์ต่อพ่วงที่จำเป็นทั้งหมด
2.  สร้างสถานีของฮาร์ดแวร์**Dedicated**ชนิดสำหรับร้านค้าปลีกที่จะใช้ที่เครื่องบันทึกเงินสด POS
3.  บนอุปกรณ์ฮาร์ดแวร์เฉพาะ ตั้งค่าคุณสมบัติต่อไปนี้:
    -   **ชื่อโฮสต์**– ชื่อของโฮสต์คอมพิวเตอร์ที่อุปกรณ์ฮาร์ดแวร์ที่จะทำงาน **หมายเหตุ:**เมฆ POS สามารถแก้ไข**โฮสต์ท้องถิ่น**เพื่อตรวจสอบเครื่องคอมพิวเตอร์ที่ทำงาน Cloud POS ได้ อย่างไรก็ตาม ใบรับรองที่จำเป็นเพื่อให้คู่เมฆ POS อุปกรณ์ฮาร์ดแวร์ต้องมี "โฮสต์ท้องถิ่น" เป็นชื่อคอมพิวเตอร์ เมื่อต้องการหลีกเลี่ยงปัญหา เราขอแนะนำให้ คุณทำรายการอินสแตนซ์ของแต่ละอุปกรณ์ฮาร์ดแวร์ที่จัดสรรไว้สำหรับร้านค้า ตามความจำเป็น สำหรับแต่ละอุปกรณ์ฮาร์ดแวร์ ชื่อโฮสต์ควรเป็นชื่อคอมพิวเตอร์ที่เฉพาะเจาะจงที่สถานีฮาร์ดแวร์จะสามารถปรับใช้
    -   **พอร์ต**– พอร์ตที่ใช้สำหรับอุปกรณ์ฮาร์ดแวร์ที่สามารถสื่อสารกับไคลเอนต์ POS สมัยใหม่
    -   **โพรไฟล์ฮาร์ดแวร์**– ถ้าโพรไฟล์ฮาร์ดแวร์ที่ไม่ได้มีให้บนอุปกรณ์ฮาร์ดแวร์เอง จะใช้โพรไฟล์ฮาร์ดแวร์ที่กำหนดให้กับการลงทะเบียน
    -   **หมายเลข EFT POS** – รหัสเทอร์มินัล EFT ที่จะใช้เมื่อตรวจสอบ EFT จะถูกส่งไป รหัสนี้จะได้มาจากตัวประมวลผลบัตรเครดิต
    -   **ชื่อแพคเกจ**– แพคเกจอุปกรณ์ฮาร์ดแวร์ที่จะใช้เมื่อมีการใช้งานอุปกรณ์ฮาร์ดแวร์

4.  คลิ**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**
5.  เลือก**1090**กำหนดการกระจายการซิงค์โพรไฟล์ฮาร์ดแวร์ใหม่ไปยังที่เก็บ คลิ**เรียกใช้เดี๋ยวนี้**การซิงค์การเปลี่ยนแปลงกับ POS
6.  เลือก**1040 ติด**กำหนดการกระจายการซิงค์สถานีฮาร์ดแวร์ใหม่ไปยังที่เก็บ คลิ**เรียกใช้เดี๋ยวนี้**การซิงค์การเปลี่ยนแปลงกับ POS
7.  ติดตั้งอุปกรณ์ฮาร์ดแวร์นี้ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการติดตั้งอุปกรณ์ฮาร์ดแวร์ ดู[การตั้งค่าคอนฟิกอุปกรณ์ฮาร์ดแวร์และการติดตั้งการขายปลีก](retail-hardware-station-configuration-installation.md)
8.  ติดตั้ง และเรียกใช้ POS สมัยใหม่ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการติดตั้ง POS สมัยใหม่ ดู[การตั้งค่าคอนฟิก Retail POS สมัยใหม่และการติดตั้ง](retail-modern-pos-device-activation.md)
9.  ล็อกอินเพื่อ POS สมัยใหม่ และเลือก**ดำเนินการไม่มีลิ้นชัก**
10. เริ่มการทำงานแบบ**จัดการฮาร์ดแวร์สถานี**ดำเนินการ
11. คลิ**จัดการ**
12. บนเพจการจัดการอุปกรณ์ฮาร์ดแวร์ ตั้งค่าตัวเลือกในการเปิดสถานีฮาร์ดแวร์
13. เลือกอุปกรณ์ฮาร์ดแวร์ให้ใช้ และจากนั้น คลิ**คู่**
14. หลังจากที่อุปกรณ์ฮาร์ดแวร์จะจับคู่ คลิ**ปิด**
15. บนเพจการเลือกสถานีฮาร์ดแวร์ คลิกอุปกรณ์ฮาร์ดแวร์ที่เพิ่งเลือกเพื่อเปิดใช้งาน

### <a name="all-modern-pos-clients-that-have-a-shared-iis-hardware-station"></a>ไคลเอนต์ POS สมัยใหม่ทั้งหมดที่มีฮาร์ดแวร์สถานี IIS ที่ใช้ร่วมกัน

การกำหนดค่านี้สามารถใช้สำหรับไคลเอนต์ POS สมัยใหม่ทั้งหมดที่ใช้ฮาร์ดแวร์สถานีร่วมกับอุปกรณ์อื่น เมื่อต้องการตั้งค่าการตั้งค่าคอนฟิกนี้ ให้ทำตามขั้นตอนเหล่านี้

1.  สร้างโพรไฟล์ฮาร์ดแวร์ที่มีการกำหนดค่าอุปกรณ์ต่อพ่วงที่จำเป็น
2.  สร้างสถานีของฮาร์ดแวร์**ใช้ร่วมกัน**ชนิดสำหรับร้านค้าปลีกที่จะใช้ที่เครื่องบันทึกเงินสด POS
3.  บนอุปกรณ์ฮาร์ดแวร์ที่ใช้ร่วมกัน ตั้งค่าคุณสมบัติต่อไปนี้:
    -   **ชื่อโฮสต์**– ชื่อของโฮสต์คอมพิวเตอร์ที่อุปกรณ์ฮาร์ดแวร์ที่จะทำงาน
    -   **คำอธิบาย**– ข้อความที่จะช่วยระบุสถานีฮาร์ดแวร์ เช่น**ส่งกลับค่า**หรือ**หน้าของร้านค้า**
    -   **พอร์ต**– พอร์ตที่ใช้สำหรับอุปกรณ์ฮาร์ดแวร์ที่สามารถสื่อสารกับไคลเอนต์ POS สมัยใหม่
    -   **โพรไฟล์ฮาร์ดแวร์**– สำหรับสถานีฮาร์ดแวร์ที่ใช้ร่วมกัน แต่ละสถานีฮาร์ดแวร์ควรมีโพรไฟล์ฮาร์ดแวร์ โพรไฟล์ฮาร์ดแวร์สามารถถูกแบ่งกันระหว่างสถานีฮาร์ดแวร์ แต่พวกเขาต้องถูกแมปแต่ละอุปกรณ์ฮาร์ดแวร์ นอกจากนั้น เราขอแนะนำให้ คุณใช้กะที่ใช้ร่วมกันเมื่ออุปกรณ์หลายสถานีฮาร์ดแวร์ที่ใช้ร่วมกันแบบเดียวกับที่ใช้ เมื่อต้องการเซ็ตอัพกะที่ใช้ร่วมกัน คลิ**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โพรไฟล์ POS**&gt;**โพรไฟล์ฮาร์ดแวร์** สำหรับแต่ละโพรไฟล์ของฮาร์ดแวร์ที่ใช้ร่วมกัน เลือกลิ้นชักเงินสด และการตั้งค่า**ลิ้นชักกะร่วม**ตัวเลือกเพื่อ**ใช่**
    -   **หมายเลข EFT POS** – รหัสเทอร์มินัล EFT ที่จะใช้เมื่อตรวจสอบ EFT จะถูกส่งไป รหัสนี้จะได้มาจากตัวประมวลผลบัตรเครดิต
    -   **ชื่อแพคเกจ**– แพคเกจอุปกรณ์ฮาร์ดแวร์ที่จะใช้เมื่อมีการใช้งานอุปกรณ์ฮาร์ดแวร์

5.  คลิ**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**

15. ทำซ้ำขั้นตอนที่ 14 สำหรับแต่ละอุปกรณ์ฮาร์ดแวร์ที่จะใช้ POS สมัยใหม่

## <a name="security"></a>ความปลอดภัย
ตามมาตรฐานความปลอดภัยปัจจุบัน การตั้งค่าต่อไปนี้ควรใช้ในสภาพแวดล้อมการผลิต:**หมายเหตุ:**ตัวติดตั้งอุปกรณ์ฮาร์ดแวร์จะดำเนินการแก้ไขรีจิสทรีเหล่านี้เป็นส่วนหนึ่งของการติดตั้งผ่านระบบบริการตนเอง

-   การรักษาความปลอดภัย Secure Sockets Layer (SSL) ควรถูกปิดใช้งาน
-   เฉพาะขนส่งเลเยอร์ความปลอดภัย (TLS) เวอร์ชัน 1.2 (หรือรุ่นสูงสุด) จะมีการเปิดใช้งาน และใช้ **หมายเหตุ:**โดยค่าเริ่มต้น SSL และ TLS ยกเว้น TLS 1.2 รุ่นทั้งหมดถูกปิดใช้งาน เมื่อต้องการแก้ไข หรือเปิดใช้งานค่าเหล่านี้ ให้ทำตามขั้นตอนเหล่านี้:
    1.  กดโลโก้ Windows คีย์ + R เพื่อเปิดตัว**รัน**หน้าต่าง
    2.  ในการ**เปิด**ฟิลด์ พิมพ์**Regedit**แล้ว คลิก**ตกลง**
    3.  ถ้าเป็น**การควบคุมบัญชีผู้ใช้**กล่องข้อความปรากฏขึ้น คลิก**ใช่**
    4.  ในการ**ตัวแก้ไขรีจิสทรี**หน้าต่าง นำทางไปยัง**HKEY\_ท้องถิ่น\_MACHINESystemCurrentControlSetSecurityProvidersSCHANNELProtocols** คีย์ต่อไปนี้ได้ถูกป้อนโดยอัตโนมัติที่อนุญาตสำหรับ TLS 1.2 เท่านั้น:
        -   TLS 1.2Server: เปิดใช้งาน = 1
        -   TLS 1.2Server:DisabledByDefault = 0
        -   TLS 1.2Client: เปิดใช้งาน = 1
        -   TLS 1.2Client:DisabledByDefault = 0
        -   TLS 1.1Server: เปิดใช้งาน = 0
        -   TLS 1.1Client: เปิดใช้งาน = 0
        -   TLS 1.0Server: เปิดใช้งาน = 0
        -   TLS 1.0Client: เปิดใช้งาน = 0
        -   SSL 3.0Server: เปิดใช้งาน = 0
        -   SSL 3.0Client: เปิดใช้งาน = 0
        -   SSL 2.0Server: เปิดใช้งาน = 0
        -   SSL 2.0Client: เปิดใช้งาน = 0
-   ไม่มีพอร์ตเครือข่ายเพิ่มเติมควรเปิด เว้นแต่ว่าจำเป็นเพื่อทราบ ระบุเหตุผล
-   ข้ามจุดเริ่มต้นของทรัพยากรที่ใช้ร่วมกันถูกปิดใช้งาน และต้องระบุจุดเริ่มต้นของการอนุญาตที่ได้รับการยอมรับ
-   หน่วยงานจัดเก็บใบรับรองที่เชื่อถือได้เท่านั้นที่ควรจะใช้เพื่อขอรับใบรับรองที่จะใช้บนคอมพิวเตอร์ที่เรียกใช้อุปกรณ์ฮาร์ดแวร์

**หมายเหตุ:**จำเป็นอย่างยิ่งให้คุณตรวจสอบคำแนะนำด้านความปลอดภัยสำหรับ IIS และความต้องการการชำระเงินบัตรอุตสาหกรรม (PCI)

## <a name="troubleshooting"></a>การแก้ไขปัญหา
### <a name="modern-pos-can-detect-the-hardware-station-in-its-list-for-selection-but-it-cant-complete-the-pairing"></a>POS สมัยใหม่สามารถตรวจพบฮาร์ดแวร์สถานีในรายการสำหรับเลือก แต่จะไม่สามารถทำการจับคู่

**โซลูชัน:**ตรวจสอบจุดล้มเหลวอาจมีรายการต่อไปนี้:

-   คอมพิวเตอร์ที่กำลังเรียกใช้ POS สมัยใหม่เชื่อถือใบรับรองที่ใช้บนคอมพิวเตอร์ที่รันในอุปกรณ์ฮาร์ดแวร์
    -   เมื่อต้องการตรวจสอบเซ็ตอัพนี้ เว็บเบราว์เซอร์ ไปที่ URL ต่อไปนี้: https://&lt;คอมพิวเตอร์ชื่อ&gt;:&lt;หมายเลขพอร์ต&gt;/HardwareStation/ping
    -   URL นี้ใช้การ ping เพื่อตรวจสอบว่า สามารถเข้าถึงคอมพิวเตอร์ และเบราว์เซอร์ที่บ่งชี้ว่า ใบรับรองเชื่อถือได้ (ตัวอย่างเช่น ใน Internet Explorer ไอคอนล็อกปรากฏขึ้นในแถบอยู่ เมื่อคุณคลิกไอคอนนี้ Internet Explorer ตรวจสอบว่า ใบรับรองเชื่อถือได้ในขณะนี้ คุณสามารถติดตั้งใบรับรองบนคอมพิวเตอร์เครื่องนี้ โดยการดูรายละเอียดของใบรับรองที่แสดงอยู่)
-   บนคอมพิวเตอร์ที่รันสถานีฮาร์ดแวร์ พอร์ตที่จะถูกใช้ โดยอุปกรณ์ฮาร์ดแวร์ที่เปิดอยู่ในไฟร์วอลล์
-   อุปกรณ์ฮาร์ดแวร์ติดตั้งข้อมูลบัญชีร้านค้ารับบัตรผ่านเครื่องมือติดตั้งข้อมูลผู้ขายที่เรียกใช้เมื่อสิ้นสุดโปรแกรมติดตั้งอุปกรณ์ฮาร์ดแวร์ ได้ถูกต้อง

### <a name="modern-pos-cant-detect-the-hardware-station-in-its-list-for-selection"></a>POS สมัยใหม่ไม่สามารถตรวจพบฮาร์ดแวร์สถานีในรายการสำหรับเลือก

**โซลูชัน:**อย่างใดอย่างหนึ่งตัวแปรต่อไปนี้อาจทำให้เกิดปัญหานี้:

-   อุปกรณ์ฮาร์ดแวร์ไม่ได้ถูกตั้งค่าอย่างถูกต้องใน headquarters ใช้ขั้นตอนก่อนหน้านี้ในหัวข้อนี้เพื่อตรวจสอบว่า โพรไฟล์สถานีฮาร์ดแวร์และอุปกรณ์ฮาร์ดแวร์ไม่ถูกต้องได้ถูกป้อน
-   งานยังไม่ได้ถูกเรียกใช้เพื่อปรับปรุงการตั้งค่าคอนฟิกช่องสัญญาณ ในกรณีนี้ เรียกใช้งาน 1070 สำหรับการตั้งค่าคอนฟิกช่องสัญญาณ

### <a name="modern-pos-doesnt-reflect-new-cash-drawer-settings"></a>POS สมัยใหม่ไม่สะท้อนถึงการตั้งค่าลิ้นชักเงินสดใหม่

**โซลูชัน:**ปิดชุดงานปัจจุบัน ไม่ได้เปลี่ยนแปลงลิ้นชักเงินสดเพื่อ POS สมัยใหม่จนกว่าจะปิดชุดงานปัจจุบัน

### <a name="modern-pos-is-reporting-an-issue-with-a-retail-peripheral"></a>POS สมัยใหม่การรายงานปัญหาเกี่ยวกับการจำหน่ายปลีกอุปกรณ์ต่อพ่วง

**โซลูชัน:**ต่อไปนี้คือสาเหตุทั่วไปของปัญหานี้:

-   ตรวจสอบให้แน่ใจว่า ได้ปิดโปรแกรมอรรถประโยชน์การกำหนดค่าโปรแกรมควบคุมอุปกรณ์อื่น ๆ ถ้าโปรแกรมอรรถประโยชน์เหล่านี้ถูกเปิด พวกเขาอาจป้องกันไม่ให้ POS สมัยใหม่หรืออุปกรณ์ฮาร์ดแวร์อ้างว่า อุปกรณ์ดังกล่าว
-   ถ้าอุปกรณ์พ่วงขายปลีกใช้ร่วมกันกับอุปกรณ์ POS หลาย ตรวจสอบให้แน่ใจว่า ถูกต้องเป็นของหนึ่งในประเภทต่อไปนี้:
    -   ลิ้นชักเงินสด
    -   เครื่องพิมพ์ใบเสร็จ
    -   เทอร์มินัลการชำระเงิน 

    อุปกรณ์พ่วงไม่อยู่ในประเภทเหล่านี้อย่างใดอย่างหนึ่ง อุปกรณ์ฮาร์ดแวร์ไม่ได้ถูกออกแบบมาเพื่อเปิดใช้งานอุปกรณ์พ่วงสามารถใช้ร่วมกันระหว่างอุปกรณ์ POS ได้หลายชนิด
-   บางครั้ง โปรแกรมควบคุมอุปกรณ์สามารถทำให้วัตถุตัวควบคุมทั่วไป (CCOs) เมื่อต้องการหยุดการทำงานได้อย่างถูกต้อง หากอุปกรณ์ถูกติด ตั้งเมื่อเร็ว ๆ นี้ แต่ไม่ได้ทำงานอย่างถูกต้อง หรือหากสังเกตเห็นปัญหาอื่น ๆ คุณมักจะสามารถแก้ไขปัญหานี้ โดยการติดตั้งแบบ CCOs เมื่อต้องการดาวน์โหลด CCOs, <http://monroecs.com/oposccos_current.htm>
-   ถ้าคุณทำการเปลี่ยนแปลงต่อพ่วงที่ใช้บ่อยในระหว่างการทดสอบ หรือการแก้ไขปัญหาเบื้องต้น คุณอาจต้องรีเซ็ต IIS แทนที่จะรอสำหรับแคชของการฟื้นฟูตัวเอง การตั้งค่า IIS ให้ทำตามขั้นตอนเหล่านี้:
    1.  จาก**เริ่ม**เมนู ชนิด**CMD**
    2.  คลิกขวาในผลลัพธ์การค้นหา**พรอมต์คำสั่ง**แล้ว คลิก**เรียกใช้ในฐานะผู้ดูแล**
    3.  ในการ**พรอมต์คำสั่ง**หน้าต่าง ชนิด**iisreset /Restart**แล้ว กด Enter
    4.  หลังจาก IIS ได้เริ่มใหม่ รีสตาร์ท POS สมัยใหม่
-   ในขณะที่คุณกำลังทำการเปลี่ยนแปลงบ่อยครั้งกับอุปกรณ์ต่อพ่วง ถ้าคุณมักเริ่มต้น และจบการทำงานของไคลเอนต์ POS กระบวนการ dllhost จากเซสชันก่อนหน้าของ POS อาจรบกวนเซสชันปัจจุบัน ในกรณีนี้ อุปกรณ์อาจไม่สามารถใช้งานได้จนกว่าคุณปิดโฮสต์ไลบรารีการเชื่อมโยงแบบไดนามิก (DLL) ที่มีการจัดการรอบเวลาก่อนหน้านี้ เมื่อต้องการปิดการโฮสต์ DLL ให้ทำตามขั้นตอนเหล่านี้:
    1.  จาก**เริ่ม**เมนู ชนิด**ตัวจัดการงาน**
    2.  ในผลลัพธ์การค้นหา คลิ**ตัวจัดการงาน**
    3.  ในตัวจัดการงาน บน**รายละเอียด**แท็บ คลิกที่หัวคอลัมน์จะมีป้ายชื่อว่า**ชื่อ**ในการเรียงลำดับตารางตามลำดับอักษรตามชื่อ
    4.  เลื่อนลงมาจนกว่าคุณพบ dllhost.exe
    5.  เลือกแต่ละโฮสต์ DLL และจากนั้น คลิ**จบการทำงาน**
    6.  หลังจากได้ถูกปิดโฮสต์ DLL รีสตาร์ท POS สมัยใหม่
