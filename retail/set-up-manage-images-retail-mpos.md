# <a name="set-up-and-manage-images-for-retail-modern-pos"></a>ตั้งค่า และจัดการรูปสำหรับ Retail POS สมัยใหม่

บทความนี้อธิบายถึงขั้นตอนที่เกี่ยวข้องในการตั้งค่า และการจัดการรูปภาพสำหรับหน่วยงานต่าง ๆ ที่ปรากฏในขายปลีกสมัยใหม่ POS (MPOS)

<a name="setting-up-the-media-base-url-and-defining-media-templates-to-configure-the-format-for-image-urls"></a>การตั้งค่าฐานสื่อ URL และการกำหนดเท็มเพลตสื่อเพื่อตั้งค่าคอนฟิกรูปแบบสำหรับ URL ของรูปภาพ
-------------------------------------------------------------------------------------------------

รูปแบบที่ปรากฏในขายปลีกสมัยใหม่ POS (MPOS) ต้องถูกโฮสต์ภายนอก อยู่นอก 365 Dynamics Microsoft สำหรับการดำเนินงาน - ขายปลีก โดยทั่วไป จะถูกโฮสต์ในระบบการจัดการเนื้อหา เครือข่ายส่งเนื้อหา (CDN) หรือเซิร์ฟเวอร์สื่อ จากนั้น MPOS นำมาใช้และแสดงรูปภาพสำหรับเอนทิตี้ที่เหมาะสม เช่น ผลิตภัณฑ์และแค็ตตาล็อก โดนการเข้าถึง URL เป้าหมาย เพื่อที่จะนำรูปภาพที่ถูกโฮสต์จากภายนอกเหล่านี้มาใช้ MPOS จำเป็นต้องมีรูปแบบ URL ที่ถูกต้องสำหรับรูปภาพ คุณสามารถตั้งค่าคอนฟิกรูปแบบ URL ที่จำเป็นสำหรับรูปภาพโดยการตั้งค่าค่า **URL ฐานสื่อ **ในโพรไฟล์ช่องและใช้ฟังก์ชัน **กำหนดเท็มเพลตสื่อ **สำหรับแต่ละเอนทิตี้ได้ คุณสามารถบันทึกทับรูปแบบ URL มาตรฐานสำหรับชุดย่อยของเอนทิตี โดยการใช้ฟังก์ชั่น **แก้ไขใน Excel** ได้ **หมายเหตุสำคัญ:**ในรุ่นปัจจุบันของ Dynamics 365 สำหรับการดำเนินงาน คุณสามารถไม่ตั้งค่ารูปแบบ URL โดยใช้ตัว**รูป**ของแอตทริบิวต์ XML สำหรับ MPOS ในการ**เริ่มต้น**กลุ่มแอททริบิวต์สำหรับเอนทิตีได้ ถ้าคุณคุ้นเคยกับ R3 2012 ของ AX Dynamics Microsoft และขณะนี้มีการใช้รุ่น Dynamics 365 ปัจจุบันสำหรับการดำเนินงาน โปรดตรวจสอบให้แน่ใจว่า คุณใช้เสมอใหม่**แม่สื่อกำหนด**หน้าที่การใช้งานการตั้งค่ารูปภาพ อย่าใช้หรือปรับเปลี่ยนแอททริบิวต์ **รูปภาพ** ในกลุ่มแอททิรบิวต์ **เริ่มต้น** สำหรับเอนทิตีใด ๆ รวมถึงผลิตภัณฑ์ การเปลี่ยนแปลงที่คุณทำได้โดยตรงในกลุ่มแอททริบิวต์ **เริ่มต้น** สำหรับรูปภาพจะไม่สะท้อนให้เห็น ตัวเลือกนี้จะไม่ทำงานในรุ่นต่อไป ในกระบวนงานต่อไปนี้ รูปภาพจะถูกตั้งค่าสำหรับเอนทิตี้แค็ตตาล็อกเป็นตัวอย่าง กระบวนงานเหล่านี้จะช่วยรับประกันว่า เส้นทางปลายทางของรูปภาพที่ถูกต้องถูกตั้งค่าสำหรับรูปภาพแค็ตตาล็อกทั้งหมดที่ใช้เส้นทางทั่วไปโดยนัย ตัวอย่างเช่น ถ้าคุณได้ตั้งค่าเซิร์ฟเวอร์สื่อหรือ CDN จากภายนอก และต้องการให้รูปภาพปรากฏใน MPOS สำหรับร้านค้าที่กำหนด ฟังก์ชัน **กำหนดเท็มเพลตสื่อ** จะช่วยคุณตั้งเส้นทางที่ MPOS สามารถค้นหาและดึงข้อมูลรูปภาพได้ **หมายเหตุ:** สำหรับตัวอย่างข้อมูลสาธิตนี้ เซิร์ฟเวอร์สื่อถูกจัดวางบนเซิร์ฟเวอร์ขายปลีก อย่างไรก็ตาม คุณไม่ที่ใดก็ได้ภายนอก 365 Dynamics สำหรับการดำเนินงาน

### <a name="set-up-the-media-base-url-for-a-channel"></a>ตั้งค่า URL ฐานสื่อสำหรับช่องทาง

1.  เปิด 365 Dynamics สำหรับเว็บไซต์ HQ การดำเนินงาน
2.  คลิ**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**สถานีโพรไฟล์** [![profile1 ช่องทาง](./media/channel-profile1.png)](./media/channel-profile1.png)
3.  ในโพรไฟล์ช่องทางที่ร้านค้าของคุณใช้สำหรับ MPOS อัพเดตฟิลด์ **URL ฐานสื่อ** ด้วย URL ฐานสื่อของเซิร์ฟเวอร์สื่อหรือ CDN ของคุณ ฐาน URL เป็นส่วนแรกของ URL ที่ใช้ร่วมกัน ด้วยโฟลเดอร์รูปภาพทั้งหมดของเอนทิตีอื่น [![profile2 ช่องทาง](./media/channel-profile2.png)](./media/channel-profile2.png)

### <a name="define-the-media-template-for-an-entity"></a>กำหนดเท็มเพลตสื่อสำหรับเอนทิตี้

1.  คลิ**ขายปลีกและพาณิชย์**&gt;**แค็ตตาล็อกการจัดการ**&gt;**รูปของแค็ตตาล็อก**
2.  ในหน้า **รูปภาพแค็ตตาล็อก** บนบานหน้าต่างการดำเนินการ คลิก **กำหนดเท็มเพลตสื่อ** ในกล่องโต้ตอบ **กำหนดเท็มเพลตสื่อ** ในฟิลด์ **เอนทิตี้** **แค็ตตาล็อก** ควรถูกเลือกตามค่าเริ่มต้น
3.  ในแท็บด่วน **เส้นทางสื่อ** ป้อนเส้นที่เหลืออยู่ของสถานที่เก็บรูปภาพ เส้นทางสื่อสนับสนุน **LanguageID** เป็นตัวแปร ตัวอย่างเช่น สำหรับข้อมูลสาธิต คุณสามารถสร้างโฟลเดอร์ **แค็ตตาล็อก** สำหรับรูปภาพแค็ตตาล็อกทั้งหมดภายใต้ฐานสื่อ URL สำหรับเซิร์ฟเวอร์สื่อของคุณ (https://testax3ret.cloud.test.dynamics.com/RetailServer/MediaServer) จากนั้นคุณสามารถมีโฟลเดอร์สำหรับแต่ละภาษา เช่น อังกฤษ-สหรัฐอเมริกา หรือ ฝรั่งเศส-ฝรั่งเศส และคัดลอกรูปภาพที่เหมาะสมภายใต้แต่ละโฟลเดอร์ ถ้าคุณไม่มีรูปภาพที่แตกต่างกันสำหรับภาษาต่าง ๆ คุณสามารถข้ามตัวแปร **LanguageID** จากโครงสร้างโฟลเดอร์ของคุณและมุ่งโดยตรงไปยังโฟลเดอร์แค็ตตาล็อกที่ประกอบด้วยรูปภาพแค็ตตาล็อกได้ **หมายเหตุ:** รุ่นปัจจุบันของ Dynamics AX สนับสนุนโทเคน **{LanguageId}** สำหรับเอนทิตี้แค็ตตาล็อก ผลิตภัณฑ์ และประเภท (โทเคน **{LanguageID}** ไม่ได้รับการสนับสนุนสำหรับเอนทิตีลูกค้าและผู้ปฏิบัติงาน ตามมาตรฐานที่มีอยู่ที่มีการมีผลบังคับใช้นับตั้งแต่ Microsoft Dynamics AX 6.x)
4.  สำหรับรูปภาพ รูปแบบชื่อไฟล์ตายตัวกับชื่อแค็ตตาล็อกและไม่สามารถถูกเปลี่ยนแปลงได้ ดังนั้น เปลี่ยนชื่อรูปภาพของคุณเพื่อให้มีชื่อแค็ตตาล็อกที่เหมาะสม เพื่อที่จะช่วยรับประกันว่า MPOS จัดการได้อย่างถูกต้อง
5.  ในฟิลด์ **นามสกุลไฟล์** เลือกนามสกุลไฟล์ที่คาดไว้ โดยขึ้นอยู่กับชนิดของรูปภาพที่คุณมี ตัวอย่างเช่น สำหรับข้อมูลสาธิต รูปภาพแค็ตตาล็อกจะถูกกำหนดเป็นนามสกุล .jpg (ไฟล์รูปภาพจะถูกเปลี่ยนชื่อเพื่อให้มีชื่อแค็ตตาล็อกด้วยเช่นกัน)
6.  Click **OK**.
7.  เพื่อตรวจสอบว่าเท็มเพลตสื่อสำหรับรูปภาพถูกบันทึกอย่างถูกต้อง ในหน้า **รูปภาพแค็ตตาล็อก** คลิก **กำหนดเท็มเพลตสื่อ** อีกครั้ง เพื่อตรวจสอบเท็มเพลตโดยไม่มีการปิดกล่องโต้ตอบ **กำหนดเท็มเพลตสื่อ** คุณสามารถใช้แท็บด่วน **สร้าง URL ของรูปภาพสำหรับ Excel** ได้ เช็คลักษณะของ URL ของรูปภาพ และตรวจสอบว่า URL นั้นสอดคล้องกับมาตรฐานเท็มเพลตที่ถูกกล่าวถึงก่อนหน้านี้ กล่องโต้ตอบ **กำหนดเท็มเพลตสื่อ** ขณะนี้ได้ตั้งเส้นทางโดยนัยสำหรับรูปภาพแค็ตตาล็อกทั้งหมดที่ใช้เส้นทาง URL ทั่วไปนี้ เส้นทาง URL นี้ใช้กับรูปภาพแค็ตตาล็อกทั้งหมดยกเว้นว่าถูกบันทึกทับ ส่วนแรกของเส้นทางรูปภาพจะถูกนำมาจาก URL ฐานสื่อที่คุณกำหนดไว้ในโพรไฟล์ช่องทาง ส่วนที่เหลือของเส้นทางจะถูกนำมาจากเส้นทางที่คุณกำหนดไว้ในเท็มเพลตสื่อ ทั้งสองส่วนจะถูกเรียงต่อกันเพื่อจัดเตรียม URL แบบเต็มของสถานที่เก็บรูปภาพ ตัวอย่างเช่น แค็ตตาล็อกในข้อมูลสาธิตที่ถูกตั้งชื่อแค็ตตาล็อกฐานของ Fabrikam ดังนั้น ชื่อรูปภาพต้องมี Fabrikam Base Catalog.jpg เพื่อให้ใช้ชื่อแค็ตตาล็อกและนามสกุลไฟล์ .jpg ที่ถูกตั้งค่าคอนฟิกในเท็มเพลต ในกรณีนี้ หลังจากการเรียงต่อกัน URL จะเป็น https://testax3ret.cloud.test.dynamics.com/RetailServer/MediaServer/Catalogs/en-US/Fabrikam Base Catalog.jpg
8.  รันงานการซิงโครไนส์เพื่อที่จะกำหนดเท็มเพลตใหม่ไปยังฐานข้อมูลช่องทาง เพื่อให้ MPOS สามารถใช้เท็มเพลตนั้นในการเข้าถึงรูปภาพได้
9.  เมื่อต้องการปรับปรุงแม่แบบสื่อสำหรับแค็ตตาล็อกรูปทางด้านสถานี ให้แน่ใจว่าคุณได้เรียกใช้**1150 งานแค็ตตาล็อก**จาก**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**[![catalog1](./media/catalog1.png)](./media/catalog1.png)

## <a name="previewing-an-image-from-the-entity-level"></a>แสดงตัวอย่างรูปภาพจากระดับเอนทิตี้
1.  จากหน้าสำหรับสินค้าเอนทิตี้แบบ HQ คุณสามารถแสดงตัวอย่าง URL รูปภาพที่ถูกส่งมาจากเท็มเพลตสื่อ สำหรับตัวอย่างนี้ ไปยังแค็ตตาล็อกที่เหมาะสม จากนั้น ในบานหน้าต่างการดำเนินการ คลิ**สื่อ**&gt;**รูป** ใช้รายการแบบหล่นลงเพื่อเลือกร้านค้าต่าง ๆ ที่อาจมีโพรไฟล์ช่องทางที่แตกต่างกัน
2.  เพื่อแก้ไขหรือลบเท็มเพลตสื่อโดยนัย คุณต้องส่งคืนไปยังกล่องโต้ตอบ **กำหนดเท็มเพลตสื่อ** สำหรับหน้า **รูปภาพแค็ตตาล็อก**
3.  คุณสามารถใช้ปุ่ม **เพิ่ม** และ **ลบ** เพื่อที่จะเปลี่ยนเส้นทางที่ยึดตามเท็มเพลตโดยนัย และใช้สำหรับรูปภาพเฉพาะ ด้วยตนเองได้ สำหรับข้อมูลเพิ่มเติม ดูส่วน "บันทึกทับสื่อเท็มเพลตสำหรับเอนทิตี้สินค้า" ภายหลังในบทความนี้
4.  หลังจากที่คุณได้เสร็จสิ้นการแสดงตัวอย่างรูปภาพ และทำการเปลี่ยนแปลงใด ๆ ที่คุณต้องการ เริ่มต้นอินสแตนซ์ MPOS สำหรับร้านค้าที่เหมาะสม และดูว่า มีแสดงรูปภาพแค็ตตาล็อก [![catalog4](./media/catalog4.png)](./media/catalog4.png)

**หมายเหตุ:** คุณสามารถใช้กระบวนงานเดียวกันสำหรับทั้งหมดห้าเอนทิตี้ที่ถูกสนับสนุน: ผู้ปฏิบัติงาน ลูกค้า แค็ตตาล็อก ประเภท และผลิตภัณฑ์ "แค็ตตาล็อกผลิตภัณฑ์" (ผลิตภัณฑ์ที่ถูกตั้งค่าไว้ในระดับแค็ตตาล็อก) และ "ผลิตภัณฑ์ตามช่องทาง" (ผลิตภัณฑ์ที่ถูกตั้งค่าในระดับช่องทาง) ใช้เท็มเพลตสื่อที่ถูกตั้งค่าสำหรับเอนทิตี้ผลิตภัณฑ์ สำหรับเท็มเพลตสื่อผลิตภัณฑ์ คุณสามารถเลือกหมายเลขของรูปภาพผลิตภัณฑ์เพื่อแสดงแต่ละผลิตภัณฑ์ คุณยังสามารถกำหนดรูปภาพเริ่มต้นสำหรับผลิตภัณฑ์ที่ระบุได้ด้วยเช่นกัน ด้วยวิธีนี้ คุณสามารถป้องกันไม่ให้มีรูปภาพที่ว่างเปล่าใน MPOS และช่วยควบคุมว่ารูปภาพใดจะถูกใช้เป็นรูปภาพเริ่มต้นสำหรับผลิตภัณฑ์หนึ่ง ๆ ในตัวอย่างต่อไปนี้ แต่ละผลิตภัณฑ์มีรูปภาพห้ารูป และรูปภาพแรกถูกตั้งเป็นรูปภาพเริ่มต้น ผลิตภัณฑ์ย่อยจะถูกปฏิบัติในลักษณะเดียวกันกับผลิตภัณฑ์หลัก ชื่อไฟล์ของไฟล์รูปภาพควรจะขึ้นอยู่กับหมายเลขผลิตภัณฑ์ อักขระบางตัวจะหายไปขณะที่มีสร้างชื่อไฟล์ด้วยเช่นกัน ดังนั้น จึงเป็นการดีที่จะตรวจสอบชื่อไฟล์ โดยการใช้ส่วน **สร้าง URL ของรูปภาพสำหรับ Excel** [![prods](./media/prods.png)](./media/prods.png)  

## <a name="synchronization-jobs-to-send-a-media-template-to-the-channel-side"></a>งานการซิงโครไนส์เพื่อส่งเท็มเพลตสื่อไปยังด้านช่องทาง
ทั้งหมดห้าสนับสนุนเอนทิตี (ผู้ปฏิบัติงาน ลูกค้า แค็ตตาล็อก ประเภท และผลิตภัณฑ์), เมื่อใดก็ตามคุณปรับปรุงการ**แม่สื่อกำหนด**กล่องโต้ตอบเพื่อตั้งค่ารูปภาพ ตรวจสอบให้แน่ใจว่า คุณเรียกใช้งานแค็ตตาล็อก (1150) จาก**ไอทีขายปลีก**&gt;**กำหนดการกระจาย** งานนี้จะเปิดใช้งานเท็มเพลตสื่อที่อัพเดตแล้วเพื่อที่จะซิงค์ไปยังช่องทาง และถูกใช้โดย MPOS รันงานแค็ตตาล็อก (1150) หลังจากที่คุณทำการเปลี่ยนแปลงใด ๆ ต่อไปนี้:

-   คุณปรับปรุงแม่แบบแค็ตตาล็อกภาพสื่อจาก**แค็ตตาล็อกรูป**&gt;**แม่สื่อกำหนด**
-   คุณปรับปรุงแม่แบบสื่อรูปภาพพนักงานจาก**รูปพนักงาน**&gt;**แม่สื่อกำหนด**
-   คุณปรับปรุงแม่แบบลูกค้ารูปสื่อจาก**รูปลูกค้า**&gt;**แม่สื่อกำหนด**
-   คุณปรับปรุงแม่แบบสื่อรูปผลิตภัณฑ์จาก**รูปผลิตภัณฑ์**&gt;**แม่สื่อกำหนด**
-   คุณปรับปรุงแม่แบบประเภทรูปสื่อจาก**รูปประเภท**&gt;**แม่สื่อกำหนด** คุณยังต้องเผยแพร่ช่องทางด้วยเช่นกัน

## <a name="overwriting-the-media-template-for-entity-items"></a>การบันทึกทับเท็มเพลตสื่อสำหรับเอนทิตี้
ตามที่คุณได้เรียนรู้ในส่วนก่อนหน้านี้ เท็มเพลตสื่อสำหรับเอนทิตีที่กำหนดสนับสนุนเพียงหนึ่งเส้นทางทั่วไป เส้นทางนี้จะขึ้นอยู่กับ URL ฐานสื่อที่ถูกตั้งค่าคอนฟิกและเส้นทางสื่อที่ถูกกำหนดไว้ อย่างไรก็ตาม ในหลายกรณี ผู้ค้าปลีกต้องการให้สามารถใช้รูปภาพจากแหล่งข้อมูลที่แตกต่างกันสำหรับชุดย่อยของสินค้าในเอนทิตี้ได้ ตัวอย่างเช่น ร้านค้าใช้เซิร์ฟเวอร์สื่อที่ตนเองเป็นโฮสต์สำหรับชุดของรูปภาพแค็ตตาล็อก แต่ใช้ URL ของ CDN สำหรับชุดอื่น เพื่อที่จะบันทึกทับ URL ของรูปภาพที่ขึ้นอยู่กับเท็มเพลตสื่อสำหรับรูปภาพเอนทิตี้ที่ระดับเอนทิตี้ คุณสามารถใช้การแก้ไขใน Excel และฟังก์ชันแก้ไขด้วยตนเอง จากหน้า **แสดงตัวอย่าง** ได้

### <a name="overwrite-by-using-edit-in-excel"></a>เขียนทับโดยการแก้ไขใน Excel

1.  คลิ**ขายปลีกและพาณิชย์**&gt;**แค็ตตาล็อกการจัดการ**&gt;**รูปของแค็ตตาล็อก**
2.  ในหน้า **รูปภาพแค็ตตาล็อก** คลิก **กำหนดเท็มเพลตสื่อ** ในกล่องโต้ตอบ **กำหนดเท็มเพลตสื่อ** ในฟิลด์ **เอนทิตี้****แค็ตตาล็อก** ควรถูกเลือก
3.  ในแท็บด่วน **เส้นทางสื่อ** สังเกตสถานที่เก็บรูปภาพ
4.  ในแท็บด่วน **สร้าง URL ของรูปภาพสำหรับ Excel** คลิก **สร้าง** **สิ่งสำคัญ:** ทุกครั้งที่เท็มเพลตสื่อถูกเปลี่ยนแปลง คุณต้องคลิก **สร้าง** ก่อนที่คุณจะสามารถใช้ฟังก์ชันแก้ไขใน Excel [![excel1](./media/excel1.jpg)](./media/excel1.jpg)ตอนนี้ทำให้คุณดูตัวอย่างของรูป Url ที่สร้างขึ้นโดยยึดตามแม่แบบสื่อที่บันทึกไว้ล่าสุด [![excel2](./media/excel2.png)](./media/excel2.png)**หมายเหตุ:** Url แบบที่สร้างขึ้นสำหรับ Excel ใช้เส้นทางและแบบแผนของต้นแบบสื่อที่กำหนดไว้ แบบแผนเหล่านี้รวมถึงข้อตกลงสำหรับชื่อไฟล์ สิ่งที่คาดหวังคือคุณได้ตั้งค่ารูปภาพทางกายภาพที่มีอยู่จริงนอก Dynamics AX และรูปภาพสามารถถูกดึงข้อมูลจาก URL ที่ถูกส่งมาจากเท็มเพลตสื่อที่คุณได้กำหนดไว้ก่อนหน้านี้ คุณสามารถบันทึกทับ URL ที่ได้รับโดยการใช้ฟังก์ชันแก้ไขใน Excel
5.  คลิก **แก้ไขใน Excel**.
6.  หลังจากที่มีเปิดแผ่นงาน Microsoft Excel **การเปิดใช้งานการแก้ไข** เมื่อคุณได้รับพร้อมท์
7.  เมื่อคุณได้รับพร้อม คลิก **เชื่อถือ add-in นี้** ในบานหน้าต่างด้านขวา และรอให้ add-in นี้ทำการติดตั้งให้เสร็จสมบูรณ์ [![เชื่อถือ add-in นี้](./media/excel4.jpg)](./media/excel4.jpg)
8.  ถ้าคุณพร้อมที่จะลงชื่อเข้าใช้ ป้อนข้อมูลประจำตัวที่คุณใช้ในการลงชื่อเข้า HQ [![พร้อมท์การเข้าสู่ระบบ](./media/excel5.png)](./media/excel5.png)
9.  หลังจากที่คุณลงชื่อเข้าใช้ คุณควรจะสามารถดูรายการของ URL ของรูปภาพสำหรับรายการต่าง ๆ ในแค็ตตาล็อกได้
10. คุณแก้ไข เพิ่ม และลบ URL ของรูปภาพสำหรับสินค้าต่าง ๆ ของเอนทิตี้
11. สำหรับเอนทิตี้ทั้งหมดยกเว้นผลิตภัณฑ์ คุณสามารถบันทึกทับ URL ของรูปภาพได้ แก้ไข URL รูปภาพที่มีอยู่ เพื่อให้ใช้ URL ปลายทางใหม่ของรูปภาพ และอัพเดตชื่อไฟล์ด้วยชื่อไฟล์ใหม่สำหรับไฟล์รูปภาพ ชื่อไฟล์ต้องไม่ซ้ำกันเพื่อช่วยในการรับประกันว่าเรกคอร์ดไม่ซ้ำกัน [![เขียนทับรูป Url ใน Excel](./media/excel6.jpg)](./media/excel6.jpg)**หมายเหตุ:**เมื่อคุณเขียนทับรูป Url สำหรับเอนทิตีของผลิตภัณฑ์ โดยใช้การแก้ไขในการทำงาน Excel หรือหน้ารายการของเอนทิตี MPOS เสมอแสดงสื่อที่ Url รูปแม่พร้อมกับ Url ของรูปที่เขียนทับ
12. หลังจากที่คุณเสร็จสิ้นการทำการเปลี่ยนแปลงของคุณ คลิก **เผยแพร่ใน Excel** เพื่อสร้างการเชื่อมโยงที่ชัดเจนรายการใหม่
13. กลับไปยัง HQ และคลิก **ตกลง**
14. รันงานการซิงโครไนส์ที่เหมาะสมสำหรับเอนทิตี้ และเช็คการแสดงตัวอย่างบนหน้าเอนทิตี้หรือใน MPOS

#### <a name="creating-new-records"></a>การสร้างเรกคอร์ดใหม่

คุณสามารถสร้างเรกคอร์ดใหม่ได้ใน Excel อย่างไรก็ตาม ตรวจสอบให้แน่ใจว่าคุณจัดเตรียมข้อมูลที่ถูกต้องไว้ ตัวอย่างเช่น เพื่อสร้างรายการใหม่สำหรับแค็ตตาล็อก ตรวจสอบให้แน่ใจว่ารหัสแค็ตตาล็อกและชื่อแค็ตตาล็อกถูกต้อง และจัดเตรียมให้ชื่อไฟล์ไม่ซ้ำกัน ชื่อไฟล์ที่ไม่ซ้ำกันมีความสำคัญมาก เนื่องจากการไม่ซ้ำกันของเรกคอร์ดใน Excel จะผ่านการตรวจสอบในระหว่างการเผยแพร่ อันดับแรก คัดลอกรายละเอียดจากแค็ตตาล็อกที่คุณต้องการสร้างเรกคอร์ดใหม่ และคัดลอกเรกคอร์ด คุณเพียงต้องอัพเดตชื่อไฟล์และ URL เนื่องจากข้อมูลที่เหลือจะเป็นแบบเดียวกัน เพื่อสร้างเรกคอร์ดใหม่สำหรับสินค้าเอนทิตี้ผลิตภัณฑ์ คุณต้องใช้กระบวนงานพื้นฐานเดียวกัน จากแผ่นงาน Excel คัดลอกเรกคอร์ดที่มีอยู่แล้วสำหรับผลิตภัณฑ์ที่คุณจะสร้างเรกคอร์ดใหม่ และจากนั้นแทนที่ URL ของรูปภาพและชื่อไฟล์ ตรวจสอบให้แน่ใจว่าชื่อแฟ้มไม่ซ้ำกัน

#### <a name="deleting-an-existing-record"></a>การลบเรกคอร์ดที่มีอยู่

เฉพาะเรกคอร์ด URL ของรูปภาพที่ถูกบันทึกทับเท่านั้นที่ถูกลบได้ หลังจากที่รูปภาพถูกลบ และการซิงโครไนส์เสร็จสมบูรณ์ รูปภาพจะไม่ปรากฏในหน้า **แสดงตัวอย่าง** หรือใน MPOS เรกคอร์ด URL ของรูปภาพที่ถูกส่งมาจากเท็มเพลสื่อไม่สามารถถูกลบได้ เนื่องจากเรกคอร์ดเหล่านี้ถูกส่งมาจากเท็มเพลตสื่อเสมอทุกครั้ง

### <a name="overwrite-from-the-entity-level-preview-page"></a>บันทึกทับจากหน้าแสดงตัวอย่างระดับเอนทิตี้

สำหรับเอนทิตีทั้งหมดยกเว้นผลิตภัณฑ์ คุณสามารถบันทึกทับ URL ของรูปภาพสำหรับเอนทิตี้ที่ระบุสินค้าที่ระดับสินค้าเอนทิตี้จากหน้า **แสดงตัวอย่าง** ได้ สำหรับผลิตภัณฑ์ คุณสามารถใช้หน้าเอนทิตี้ "ผลิตภัณฑ์แค็ตตาล็อก" ตัวอย่างนี้แสดงวิธีการบันทึกทับรูปภาพแค็ตตาล็อก

1.  คลิ**แค็ตตาล็อก**&gt;**สื่อ**&gt;**รูป**และเลือกรูปแค็ตตาล็อกการปรับปรุง
2.  คลิก **เพิ่ม** และป้อน URL ของรูปภาพเพื่อบันทึกทับ URL เท็มเพลตสื่อ
3.  ถ้าคุณต้องการให้รูปภาพนี้ถูกแสดงใน MPOS สำหรับแค็ตตาล็อก คุณสามารถตั้งเป็นรูปภาพเริ่มต้น
4.  คลิก **ตกลง** ระบบจะนำเข้าข้อมูลการชำระเงิน URL ของรูปภาพมีถูกอัพเดตสำหรับรูปภาพแค็ตตาล็อกนี้ และการแสดงตัวอย่างจะถูกแสดงขึ้น [![preview3](./media/preview3.png)](./media/preview3.png)
5.  คุณยังสามารถดูตัวอย่างรูปภาพสำหรับ URL รูปภาพที่ถูกบันทึกทับทั้งหมดในหน้าแกลเลอรี **รูปภาพแค็ตตาล็อก**

**[![4 แสดงตัวอย่าง](./media/preview-4.png)](./media/preview-4.png)หมายเหตุ:**อยู่ในขณะนี้ แกลเลอรีไม่แสดงรูปตัวอย่างสำหรับสื่อ Url ของรูปต้นแบบ สำหรับเอนทิตี้แค็ตตาล็อก ผู้ปฏิบัติงาน ลูกค้า และประเภท ถ้าผู้ใช้จัดเตรียม URL อย่างชัดเจนผ่านหน้านี้ เราขอแนะนำให้คุณระบุรูปภาพที่เป็นรูปภาพเริ่มต้น เนื่องจากลูกค้าเซิร์ฟเวอร์ขายปลีกแสดงรูปภาพเดียวเท่านั้นต่อแค็ตตาล็อก ลูกค้า ผู้ปฏิบัติงาน และประเภท ถ้าผู้ใช้ไม่ได้ระบุรูปภาพเริ่มต้น ระบบจะกำหนดรูปภาพเริ่มต้นและส่งไปยังผู้เรียกบริการขายปลีก (MPOS หรือ Ecommerce)

### <a name="overwrite-the-image-url-for-catalog-product-images-from-the-preview-page"></a>บันทึกทับ URL ของรูปภาพสำหรับรูปภาพผลิตภัณฑ์แค็ตตาล็อกจากหน้าแสดงตัวอย่าง

เพื่อที่จะบันทึกทับ URL ของรูปภาพสำหรับรูปภาพผลิตภัณฑ์แค็ตตาล็อก คุณต้องใช้หน้า **แสดงตัวอย่าง** คุณไม่สามารถใช้ฟังก์ชันแก้ไขใน Excel ได้

1.  เพื่อที่จะบันทึกทับรูปภาพผลิตภัณฑ์ที่ระดับแค็ตตาล็อก เลือกแค็ตตาล็อก และจากนั้นเลือกผลิตภัณฑ์เพื่อบันทึกทับรูปภาพ
2.  คลิก **แอททริบิวต์**
3.  ในหน้าถัดไป เลือก **รูปภาพ** และจากนั้น คลิก **แก้ไข**. หน้า **แสดงตัวอย่าง** เปิดเป็นกล่องโต้ตอบแบบเลื่อน
4.  คลิก **เพิ่ม** และบันทึกทับ URL ของรูปภาพด้วย URL ใหม่
5.  คลิก **ตกลง** ระบบจะนำเข้าข้อมูลการชำระเงิน ขณะนี้คุณจะเห็นการแสดงตัวอย่างรูปภาพใหม่ และสามารถตั้งค่าเป็นรูปภาพเริ่มต้น

**[![cat3](./media/cat3.png)](./media/cat3.png)หมายเหตุ:**หลังจากการเชื่อมโยงของรูปภาพประเภท คุณต้องเผยแพร่ช่องทาง และรันงานสถานีจะช่วยรับประกันว่า การเปลี่ยนแปลงที่เผยแพร่ไปยังช่องฐานข้อมูล

## <a name="setting-up-images-to-appear-in-offline-mode-for-mpos"></a>การตั้งค่ารูปภาพผลิตภัณฑ์เพื่อที่จะแสดงอยู่ในโหมดออฟไลน์สำหรับ MPOS
MPOS สามารถรันในโหมดออนไลน์ (เมื่อ MPOS เชื่อมต่อกับเซิร์ฟเวอร์ขายปลีก) หรือโหมดออฟไลน์ (เมื่อไม่มีการเชื่อมต่อเซิร์ฟเวอร์หรือเครือข่ายขายปลีก และธุรกรรมจะถูกเก็บในฐานข้อมูลท้องถิ่นแบบออฟไลน์) เมื่อ MPOS รันในโหมดออฟไลน์ จะไม่สามารถเรียกรูปภาพจากเซิร์ฟเวอร์รูปภาพภายนอกเพื่อแสดงจากเซิร์ฟเวอร์ขายปลีกได้ เนื่องจากการเชื่อมต่อกับเซิร์ฟเวอร์ขายปลีกสูญหาย อย่างไรก็ตาม คุณยังคงสามารถตั้งค่ารูปภาพเพื่อที่จะถูกแสดงเมื่อ MPOS รันในโหมดออฟไลน์ได้

### <a name="set-up-product-images-to-appear-in-offline-mode-for-mpos"></a>ตั้งค่ารูปภาพผลิตภัณฑ์เพื่อที่จะปรากฏขึ้นในโหมดออฟไลน์สำหรับ MPOS

รูปภาพผลิตภัณฑ์ที่ต้องถูกใช้ในโหมดออฟไลน์สามารถตั้งค่าได้โดยการอัพโหลดรูปภาพทางกายภาพที่จำเป็นไปยังรูปภาพผลิตภัณฑ์พื้นฐาน

1.  คลิ**การจัดการข้อมูลผลิตภัณฑ์**&gt;**ผลิตภัณฑ์**&gt;**ผลิตภัณฑ์**
2.  เลือกผลิตภัณฑ์เพื่อตั้งค่ารูปภาพสำหรับข้อมูลออฟไลน์
3.  คลิก **แก้ไข** และจากนั้นคลิกลูกศรในมุมด้านขวาเพื่อแสดงบานหน้าต่างด้านขวา
4.  ในแท็บด่วน **รูปภาพผลิตภัณฑ์** คลิก **เปลี่ยนรูป** และอัพโหลดรูปภาพทางกายภาพเพื่อที่จะใช้สำหรับผลิตภัณฑ์ที่ถูกเลือกอยู่ในโหมดออฟไลน์
5.  บันทึกและปิดหน้า
6.  ขณะที่ MPOS อยู่ในโหมดออนไลน์ รันงานแค็ตตาล็อกแบบ HQ เพื่อให้แน่ใจว่าข้อมูลจะถูกส่งอย่างน้อยหนึ่งครั้งไปยังฐานข้อมูลแบบออฟไลน์
7.  ตั้ง MPOS เป็นโหมดออฟไลน์ คุณควรเห็นรูปภาพที่คุณได้อัพโหลดสำหรับผลิตภัณฑ์เฉพาะแบบ HQ [![offline1](./media/offline1.png)](./media/offline1.png)

 

### <a name="set-up-catalog-category-employee-and-customer-images-to-appear-in-offline-mode-for-mpos"></a>ตั้งค่าแค็ตตาล็อก ประเภท และ พนักงานลูกค้ารูปเพื่อที่จะปรากฏขึ้นในโหมดออฟไลน์สำหรับ MPOS

แค็ตตาล็อก ประเภท พนักงาน และรูปภาพของลูกค้าที่ต้องถูกใช้ในโหมดออฟไลน์สามารถตั้งค่าโดยการเพิ่มลิงค์ปลายทางของรูปภาพจำเป็นไปยังแกลเลอรี และตั้งค่ารูปภาพเป็นรูปภาพเริ่มต้นสำหรับเอนทิตี้ที่ถูกเลือก

1.  ไปยังแค็ตตาล็อก จากนั้น ในบานหน้าต่างการดำเนินการ คลิ**สื่อ**&gt;**รูป**
2.  ทำตามขั้นตอนในส่วน "**บันทึกทับจากหน้าแสดงตัวอย่างระดับเอนทิตี้**" เพื่อที่จะเพิ่ม URL ของรูปภาพภายนอก
3.  ทำเครื่องหมายรูปภาพนี้เป็นรูปภาพเริ่มต้นสำหรับแค็ตตาล็อกโดยการเลือกกล่องกาเครื่องหมายสำหรับรูปภาพที่ถูกแสดงรายการอยู่ในกริด
4.  รันงานแค็ตตาล็อก ขณะนี้รูปภาพนี้จะถูกใช้เป็นรูปภาพแบบออฟไลน์สำหรับแค็ตตาล็อกนั้นใน MPOS
5.  ทำตามกระบวนการที่คล้ายกันสำหรับเอนทิตีอื่น เช่น ประเภท พนักงาน และลูกค้า
