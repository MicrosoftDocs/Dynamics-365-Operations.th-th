# <a name="customer-orders-overview"></a>ภาพรวมใบสั่งของลูกค้า

หัวข้อนี้แสดงข้อมูลเกี่ยวกับใบสั่งของลูกค้าในขายปลีกสมัยใหม่ POS (MPOS) ใบสั่งของลูกค้าหรือที่เรียกอีกอย่างหนึ่งว่าเป็นใบสั่งพิเศษ หัวข้อมีคำอธิบายเกี่ยวกับพารามิเตอร์ที่เกี่ยวข้องและขั้นตอนของธุรกรรม

ในโลกขายปลีกสถานี omni ผู้ค้าปลีกหลายแสดงตัวเลือกของลูกค้า ใบสั่งหรือใบสั่งพิเศษ เพื่อให้ตรงกับความต้องการผลิตภัณฑ์และการตอบสนองต่าง ๆ ต่อไปนี้เป็นบางสถานการณ์ทั่วไป:

-   ลูกค้าต้องการผลิตภัณฑ์ที่จะจัดส่งไปยังที่อยู่ที่ระบุในวันเฉพาะ
-   ลูกค้าต้องการรับผลิตภัณฑ์จากร้านค้าหรือสถานที่แตกต่างจากร้านค้าหรือสถานที่ที่ลูกค้าสั่งซื้อผลิตภัณฑ์ดังกล่าว
-   ลูกค้าต้องการผู้อื่นเพื่อได้รับผลิตภัณฑ์ที่สั่งซื้อของลูกค้า

นอกจากนี้ผู้ค้าปลีกใช้ใบสั่งของลูกค้าเพื่อลดการสูญเสียขายหุ้นขัดข้องอาจมิฉะนั้นสาเหตุ เนื่องจากสินค้าสามารถจัดส่งสินค้า หรือเบิกสินค้าที่แตกต่างของเวลาหรือสถานที่

## <a name="set-up-customer-orders"></a>ตั้งค่าใบสั่งของลูกค้า
นี่เป็นบางส่วนของพารามิเตอร์ที่สามารถกำหนดได้ในการ**พารามิเตอร์การขายปลีก**หน้าเพื่อกำหนดวิธีการปฏิบัติตามใบสั่งของลูกค้า:

-   **เปอร์เซ็นต์เงินฝากเริ่มต้น**– ระบุยอดเงินที่ลูกค้าต้องชำระเป็นเงินฝากก่อนที่จะได้รับการยืนยันใบสั่ง ยอดเงินฝากเงินเริ่มต้นถูกคำนวณเป็นเปอร์เซ็นต์ของมูลค่าของใบสั่ง ขึ้นอยู่กับสิทธิ์การใช้งาน พนักงานขายประจำร้านอาจจะแทนยอดเงิน โดยใช้**ใบนำฝากธนาคารแทน**ได้
-   **เปอร์เซ็นต์ของค่าธรรมเนียมการยกเลิก**– ถ้าจะใช้ค่าธรรมเนียมเมื่อมีการยกเลิกใบสั่งลูกค้า ยอดเงินของค่าธรรมเนียมที่ระบุ
-   **รหัสค่าธรรมเนียมการยกเลิก**– ถ้าค่าธรรมเนียมจะใช้เมื่อมีการยกเลิกใบสั่งลูกค้า ว่า ค่าธรรมเนียมจะปรากฏภายใต้รหัสของค่าธรรมเนียมในใบสั่งขายใน Microsoft Dynamics AX ใช้พารามิเตอร์นี้เมื่อต้องการกำหนดรหัสค่าธรรมเนียมยกเลิก
-   **รหัสค่าธรรมเนียมการจัดส่งสินค้า**– ผู้ค้าปลีกสามารถเรียกเก็บค่าธรรมเนียมเพิ่มเติมสำหรับการจัดส่งสินค้าให้แก่ลูกค้าได้ ยอดเงินของค่าธรรมเนียมการจัดส่งนั้นจะมีผลภายใต้รหัสของค่าธรรมเนียมในใบสั่งขายใน Dynamics AX ใช้พารามิเตอร์นี้เมื่อต้องการแมปรหัสค่าธรรมเนียมการจัดส่งการจัดส่งค่าธรรมเนียมในใบสั่งของลูกค้า
-   **การเรียกคืนเงินค่าธรรมเนียมการจัดส่ง**– ระบุว่า ค่าธรรมเนียมการจัดส่งที่เกี่ยวข้องกับใบสั่งของลูกค้าถูกเรียกคืนได้
-   **ยอดเงินสูงสุด โดยไม่มีการอนุ**– ถ้ามีการเรียกคืนได้ ค่าธรรมเนียมการจัดส่งระบุยอดเงินสูงสุดของการขอคืนเงินค่าธรรมเนียมการจัดส่งสินค้าระหว่างใบสั่งส่งคืนสินค้า ถ้ามีเกินยอดเงินนี้ แทนตัวจัดการต้องใช้เพื่อดำเนินต่อ ด้วยการคืนเงิน เพื่อให้เหมาะสมกับสถานการณ์ต่อไปนี้ การคืนเงินของค่าธรรมเนียมการจัดส่งสามารถเกินยอดเงินที่จ่ายดั้งเดิม:
    -   ค่าธรรมเนียมค่าใช้จ่ายในระดับ ของหัวข้อใบสั่งขาย และบางบรรทัดผลิตภัณฑ์ถูกส่ง คืนปริมาณ การคืนเงินสูงสุดของค่าธรรมเนียมการจัดส่งสินค้าที่ได้รับอนุญาตสำหรับผลิตภัณฑ์ และปริมาณไม่สามารถกำหนดในลักษณะที่เหมาะสมสำหรับลูกค้าขายปลีกทั้งหมด
    -   ค่าธรรมเนียมการจัดส่งสินค้าจะเกิดขึ้นสำหรับทุก ๆ อินสแตนซ์ของการจัดส่ง ถ้าลูกค้าส่งคืนผลิตภัณฑ์หลายครั้ง และนโยบายของผู้ค้าปลีกระบุว่า ผู้ค้าปลีกที่จะเก็บแต้มต้นทุนของค่าธรรมเนียมการจัดส่งคืนสินค้า ค่าธรรมเนียมการจัดส่งคืนสินค้าจะมากกว่าค่าธรรมเนียมการจัดส่งจริง

## <a name="transaction-flow-for-customer-orders"></a>ขั้นตอนของธุรกรรมสำหรับใบสั่งของลูกค้า
### <a name="create-a-customer-order-in-retail-modern-pos"></a>สร้างใบสั่งของลูกค้าใน Retail POS สมัยใหม่

1.  เพิ่มลูกค้าให้กับธุรกรรม
2.  เพิ่มผลิตภัณฑ์ลงในรถเข็นของ
3.  คลิ**สร้างใบสั่งของลูกค้า**และจากนั้น เลือกชนิดใบสั่ง ชนิดของใบสั่งอาจเป็น**ใบสั่งลูกค้า**หรือ**อ้างอิง**ได้
4.  คลิ**การจัดส่งที่เลือก**หรือ**จัดส่งทั้งหมด**เพื่อจัดส่งผลิตภัณฑ์ไปยังที่อยู่ในรหัสลูกค้า วันจัดส่งที่ร้องขอระบุ และระบุค่าธรรมเนียมการจัดส่ง
5.  คลิ**เลือกรับค่า**หรือ**เบิกสินค้าทั้งหมด**เพื่อเลือกผลิตภัณฑ์ที่จะถูกรับค่าจากร้านค้าปัจจุบันหรือร้านค้าแตกต่างกันในวันระบุ
6.  รวบรวมยอดเงินฝาก ถ้าจำเป็นต้องการเงินมัดจำ

### <a name="edit-an-existing-customer-order"></a>แก้ไขใบสั่งของลูกค้าที่มีอยู่

1.  บนโฮมเพจ คลิ**ค้นหาใบสั่ง**
2.  ค้นหา และเลือกใบสั่งที่จะแก้ไข ที่ด้านล่างของหน้า คลิก**แก้ไข**

### <a name="pick-up-an-order"></a>ใบสั่งการรับของ

1.  บนโฮมเพจ คลิ**ค้นหาใบสั่ง**
2.  เลือกใบสั่งที่จะเบิกสินค้า ที่ด้านล่างของหน้า คลิ**การเบิกสินค้า และบันทึกการส่ง**
3.  คลิ**รับ**

### <a name="cancel-an-order"></a>ยกเลิกใบสั่ง

1.  บนโฮมเพจ คลิ**ค้นหาใบสั่ง**
2.  เลือกใบสั่งที่จะยกเลิก ที่ด้านล่างของหน้า คลิ**ยก**

#### <a name="create-a-return-order"></a>การสร้างใบสั่งส่งคืนสินค้า

1.  บนโฮมเพจ คลิ**ค้นหาใบสั่ง**
2.  เลือกใบสั่งส่งคืนสินค้า ใบแจ้งหนี้สำหรับใบสั่งที่เลือก จากนั้นบรรทัดผลิตภัณฑ์สำหรับสินค้าเมื่อต้องการกลับ
3.  ที่ด้านล่างของหน้า คลิก**ใบสั่งส่งคืน**

## <a name="asynchronous-transaction-flow-for-customer-orders"></a>ขั้นตอนการแบบอะซิงโครนัสธุรกรรมสำหรับใบสั่งของลูกค้า
สามารถสร้างใบสั่งของลูกค้าจากจุดของไคลเอ็นต์การขาย (POS) ในโหมดแบบซิงโครนัสหรือโหมดแบบอะซิงโครนัส

### <a name="enable-customer-orders-to-be-created-in-asynchronous-mode"></a>เปิดใช้งานการสั่งซื้อของลูกค้าที่จะสร้างในโหมดแบบอะซิงโครนัส

1.  ใน Dynamics AX คลิ**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โปรไฟล์ POS**&gt;**โพรไฟล์ฟังก์ชัน**
2.  บน**ทั่วไป**ด่วน การตั้งค่า**สร้างใบสั่งของลูกค้าในโหมดต่างเวลา**ตัวเลือกเพื่อ**ใช่**

เมื่อการ**สร้างใบสั่งของลูกค้าในโหมดต่างเวลา**ตัวเลือกถูกตั้งค่าเป็น**ใช่**ใบสั่งของลูกค้ามักถูกสร้างในโหมดแบบอะซิงโครนัส แม้ขายปลีก Transaction Service (RTS) จะพร้อมใช้งาน ถ้าคุณกำหนดตัวเลือกนี้เป็น**ไม่ใช่**ใบสั่งของลูกค้ามักถูกสร้างในโหมดแบบซิงโครนัส โดยใช้ RTS เมื่อใบสั่งของลูกค้าจะถูกสร้างขึ้นในโหมดแบบอะซิงโครนัส พวกเขาจะถูกดึงมา และแทรกลงใน Dynamics AX โดยงานดึง (P) มีสร้างใบสั่งขายที่สอดคล้องกันใน Dynamics AX เมื่อ**การซิงโครไนส์ใบสั่ง**เรียกใช้ด้วยตนเอง หรือโดย ใช้กระบวนการชุดงาน
