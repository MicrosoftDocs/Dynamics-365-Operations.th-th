# <a name="order-holds"></a>การระงับใบสั่ง

หัวข้อนี้อธิบายการระงับใบสั่งโดยใช้การขายปลีกและการค้าใน Dynamics AX

ใบสั่งสามารถถูกระงับไว้เพื่อเหตุผลต่าง ๆ ตัวอย่างเช่น คุณอาจระงับใบสั่งจนกระทั่งที่อยู่ของลูกค้าหรือวิธีการชำระเงินสามารถถูกตรวจสอบได้ หรือผู้จัดการสามารถตรวจทานวงเงินสินเชื่อของลูกค้า ในระหว่างกระบวนการขาย มีหลายครั้งที่ใบสั่งขายต้องถูกระงับ ตัวอย่างเช่น ใบสั่งขายอาจถูกระงับเนื่องจากปัญหาเกี่ยวกับการชำระเงินของลูกค้า เนื่องจากการฉ้อโกงที่ต้องสงสัย หรือเนื่อง จากผู้จัดการต้องตรวจทานใบสั่ง เมื่อใบสั่งขายถูกระงับ รหัสการระงับใบสั่งถูกกำหนดให้กับใบสั่งขายเพื่อระบุเหตุผลสำหรับการระงับ รหัสระงับใบสั่งขายที่มีการกำหนดค่า**ขาย และการตลาด**&gt;**ตั้งค่า**&gt;**ใบสั่งขาย**&gt;**สั่งเก็บรหัส** ใบสั่งขายสามารถถูกระงับด้วยตนเองในเวลาของการสร้างใบสั่ง หรือในภายหลัง นอกจากนี้ ใบสั่งสามารถถูกระงับโดยอัตโนมัติ ตามกฎการฉ้อโกง ขณะที่ใบสั่งขายถูกระงับ คุณอาจต้องอัพเดตข้อมูลเพิ่มเติม โดยมีอีกทางหนึ่ง คุณอาจต้องการเช็คใบสั่งขายในขณะที่คุณทำงานนั้นต่อ คุณสามารถตรวจสอบหาใบสั่งขาย ตรวจสอบกลับ และแม้แต่แทนเช็คเอาท์ของผู้ใช้อื่นโดยใช้ใบสั่งค้างไว้บนเก้าอี้ทำงาน (**ขายปลีกและพาณิชย์**&gt;**ลูกค้า**&gt;**สั่งหยุด**) เมื่อใบสั่งพร้อมดำเนินการให้เสร็จสมบูรณ์ คุณต้องลบการระงับก่อนจึงจะสามารถดำเนินกระบวนการใบสั่ง
