# <a name="organizations-and-organizational-hierarchies-commerce-essentials"></a>องค์กรและลำดับชั้นขององค์กร (สิ่งจำเป็นสำหรับการค้า)

[!include[banner](includes/banner.md)]


สิ่งจำเป็นสำหรับการค้ามีองค์กรภายในสามชนิดที่คุณสามารถกำหนดเพื่อช่วยให้องค์กรดำเนินกระบวนการทางธุรกิจหรือบรรลุเป้าหมาย 

องค์กรคือ กลุ่มของบุคคลที่ทำงานร่วมกันเพื่อดำเนินกระบวนการทางธุรกิจหรือดำเนินงานให้บรรลุเป้าหมาย ลำดับชั้นขององค์กรแสดงความสัมพันธ์ระหว่างองค์กรที่สร้างธุรกิจ

## <a name="organizations"></a>องค์กร
ในองค์ประกอบความสัมพันธ์ คุณสามารถกำหนดได้ 3 ชนิดขององค์กรภายในต่อไปนี้: นิติบุคคล หน่วยปฏิบัติงาน และทีมงาน ใน Microsoft Dynamics AX องค์กรภายในทั้งหมดคือ รูปแบบของเอนทิตี้ฝ่าย ดังนั้น องค์กรเหล่านี้ใช้สมุดที่อยู่เพื่อจัดเก็บที่อยู่และข้อมูลติดต่ออื่นๆ ฝ่ายสามารถเป็นบุคคลหรือองค์กร และจึงอาจอยู่ในสมุดที่อยู่หนึ่งรายการ ขึ้นไป
### <a name="legal-entities"></a>นิติบุคคล

นิติบุคคลมีโครงสร้างที่มีโครงสร้างทางกฎหมายจดทะเบียน หรือ legislated นิติบุคคลที่สามารถป้อนลงในสัญญาทางกฎหมาย และจำเป็นต้องใช้เพื่อจัดเตรียมรายงานที่รายงานเกี่ยวกับประสิทธิภาพการทำงานของตน บริษัทถือเป็นของนิติบุคคลในรูปแบบ Microsoft Dynamics AX และนิติบุคคลทั้งหมดจะเชื่อมโยงกับ id บริษัท ความสัมพันธ์นี้มีได้เนื่องจากเป็นรหัสบริษัท หรือ DataAreaId ถูกใช้ในรูปแบบข้อมูลบางอย่างที่จะใช้เป็นเส้นขอบเขตบริษัทสำหรับความปลอดภัยของข้อมูล ผู้ใช้สามารถเข้าถึงข้อมูลสำหรับบริษัทที่พวกเขาจะเข้าสู่ระบบในปัจจุบันเท่านั้น

### <a name="operating-units"></a>หน่วยปฏิบัติงาน

หน่วยปฏิบัติงานเป็นองค์กรที่ใช้ในการแบ่งการควบคุมทรัพยากรทางเศรษฐกิจและกระบวนการในการดำเนินงานในธุรกิจหนึ่ง ๆ บุคลากรในหน่วยปฏิบัติงานมีภาษีการขยายการใช้ทรัพยากร scarce กระบวนการปรับปรุง และบัญชีสำหรับประสิทธิภาพการทำงานของตน ใน Commerce essentials ชนิดของหน่วยปฏิบัติงานรวมศูนย์ต้นทุน หน่วยธุรกิจ สายธารคุณค่า แผนก และช่องทางการขายปลีก ตารางต่อไปนี้มีข้อมูลเพิ่มเติมเกี่ยวกับหน่วยปฏิบัติงานแต่ละชนิด
|                         |                                                                                         |                                                                                                                                             |
|-------------------------|-----------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------|
| **ชนิดของหน่วยปฏิบัติงาน** | **คำอธิบาย**                                                                         | **วัตถุประสงค์**                                                                                                                                 |
| หน่วยธุรกิจ           | หน่วยปฏิบัติงานกึ่งอิสระที่สร้างขึ้นเพื่อให้ตรงตามวัตถุประสงค์เชิงกลยุทธ์ทางธุรกิจ | ใช้หน่วยทางธุรกิจสำหรับการรายงานทางการเงินที่เป็นไปตามอุตสาหกรรมหรือรายการผลิตภัณฑ์ที่องค์กรทำหน้าที่อย่างเป็นอิสระจากนิติบุคคล |
| ช่องทางการขายปลีก          | หน่วยปฏิบัติงานที่แสดงถึงร้านค้าเดินจ่าย                             | ใช้เพื่อจัดการ และควบคุมอย่างน้อยหนึ่งร้านค้าภายใน หรือ ระหว่างนิติบุคคล                                                               |

## <a name="organizational-hierarchies"></a>ลำดับชั้นขององค์กร
ใน Commerce essentials แต่ละลำดับชั้นได้ถูกกำหนดวัตถุประสงค์ วัตถุประสงค์ของลำดับชั้นกำหนดชนิดขององค์กรที่สามารถรวมอยู่ในลำดับชั้น วัตถุประสงค์ยังกำหนดสถานการณ์การใช้ที่สามารถใช้ลำดับชั้นอีกด้วย ตัวอย่างเช่น สามารถใช้ลำดับชั้นการขายปลีกเพื่อซื้อและขายสินค้าในร้านค้าปลีก องค์กรในลำดับชั้นสามารถใช้ร่วมกันพารามิเตอร์ นโยบาย และธุรกรรม องค์กรที่สามารถสืบทอด หรือพารามิเตอร์ขององค์กรหลักของการแทน อย่างไรก็ตาม ข้อมูลหลักที่ใช้ร่วมกัน เช่น ผลิตภัณฑ์ และสมุดที่อยู่ นำไปใช้กับทั้งองค์กรและไม่สามารถถูกแทนที่สำหรับแต่ละองค์กรได้
### <a name="best-practices-for-setting-up-an-organization-in-a-hierarchy"></a>แนวทางปฏิบัติสำหรับการตั้งค่าองค์กรในลำดับชั้น

พิจารณาแนวทางปฏิบัติที่ดีที่สุดต่อไปนี้เมื่อคุณใช้ลำดับชั้นขององค์กร:
-   สร้างแผนกเพื่อระบุจุดตัดระหว่างนิติบุคคลและหน่วยธุรกิจ คุณสามารถแล้วสะสมข้อมูล จากแผนกนิติบุคคลสำหรับการรายงานตามกฎหมาย และจากแผนกไปยังหน่วยธุรกิจสำหรับรายงานภายใน
-   ในนิติบุคคลเดียว อย่าจำลองหลายลำดับชั้นสำหรับวัตถุประสงค์ประเภทเดียวกัน
-   อย่าสร้างลำดับชั้นสำหรับทุกวัตถุประสงค์ โดยปกติ คุณสามารถใช้ลำดับชั้นหนึ่งสำหรับหลายๆวัตถุประสงค์ ตัวอย่างเช่น หนึ่งลำดับขั้นสามารถกำหนดนโยบายทั้งหมดที่เกี่ยวข้องกับวัตถุประสงค์
-   สร้างลำดับชั้นให้สมดุล ในลำดับชั้น โหนดทั้งหมดที่มีระยะห่างจากโหนดรากเดียวกันถูกกำหนดเป็นระดับ ลำดับชั้นแบบสมดุล เพียงหนึ่งชนิดของหน่วยปฏิบัติงานสามารถเกิดขึ้นในแต่ละระดับ และระยะห่างจากโหนดรากไปแต่ละระดับจะสอดคล้องกัน ถ้ามีระดับกลางระหว่างแผนก และนิติบุคคล หรือหน่วยธุรกิจ ตัวยึดองค์กรอาจจำเป็นต้องสร้างลำดับชั้นแบบสมดุล
-   อย่าติดตั้งแยกต่างหากลำดับชั้นของหน่วยปฏิบัติงานถ้าโครงสร้างสำหรับนิติบุคคลเป็นโครงสร้างของการดำเนินงานเช่นกัน ลำดับชั้นการผสมของนิติบุคคลและหน่วยปฏิบัติงานอาจใช้เพื่อทั้งสองวัตถุประสงค์ทั้งสอง
-   ก่อนที่คุณตั้งค่าสถานการณ์ปรับโครงสร้างหลัก ใช้วันที่ที่มีผลบังคับใช้ของลำดับชั้นเพื่อดำเนินการวิเคราะห์ผลกระทบและทดสอบการยืนยัน
-   บันทึกลำดับชั้นเป็นแบบร่างถ้าคุณอาจเปลี่ยนลำดับชั้นก่อนที่คุณเผยแพร่
-   จำกัดจำนวนผู้ที่มีสิทธิ์ในการเพิ่มหรือลบองค์กรออกจากลำดับชั้นในสภาพแวดล้อมการผลิต จำนวนต่ำกว่าช่วยลดความเสี่ยงที่อาจเกิดความผิดพลาดเสียค่าใช้จ่าย และต้องทำการแก้ไข
