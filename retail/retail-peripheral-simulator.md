# <a name="retail-peripheral-simulator"></a>โปรแกรมจำลองการต่อพ่วงขายปลีก

หัวข้อนี้อธิบายเกี่ยวกับเครื่องมือโปรแกรมจำลองต่อพ่วงที่มาพร้อมกับ 365 Dynamics Microsoft สำหรับการดำเนินงาน - ขายปลีก

<a name="overview"></a>ภาพรวม
--------

365 Dynamics ของ Microsoft สำหรับการดำเนินงาน - โปรแกรมจำลองต่อพ่วงขายปลีกเป็นเครื่องมือที่ช่วยคุณในการติดตั้ง ทดสอบ และการแก้ไขปัญหาอุปกรณ์ต่อพ่วงที่ใช้ในสภาพแวดล้อมของการขายปลีก คุณสามารถใช้โปรแกรมจำลองการต่อพ่วงราบการทดสอบของอุปกรณ์ต่อพ่วงขายปลีก และแยกปัญหาที่เกิดจากการตั้งค่าไม่ถูกต้องหรือไดรเวอร์อุปกรณ์ที่ชำรุด โปรแกรมจำลองการต่อพ่วงรวมถึงโปรแกรมเดสก์ท็อปที่มีคุณลักษณะของอุปกรณ์เสมือนรุ่น 365 Dynamics นั้นสำหรับการดำเนินงาน - สนับสนุนการขายปลีก ส่วนสำหรับแต่ละอุปกรณ์เสมือนแสดงการโต้ตอบระหว่างอุปกรณ์และจุดขายปลีกการขาย (POS) นอกจากนี้คุณสามารถใช้เพื่อให้ป้อนข้อมูลที่ถูกต้องสำหรับสถานการณ์ต่าง ๆ ของ POS โปรแกรมจำลองการต่อพ่วงที่สนับสนุนการโต้ตอบระหว่าง POS และอุปกรณ์เสมือนดังต่อไปนี้:

-   **เครื่องพิมพ์**– โปรแกรมจำลองการต่อพ่วงสามารถแสดงใบรับสินค้าที่จัดโครงแบบสำหรับเครื่องพิมพ์ POS ได้
-   **บรรทัดแสดง**– คุณสามารถกำหนดค่าจอแสดงผลเสมือนบรรทัดเพื่อแสดงกิจกรรมบนจอแสดงผลเป็นรายการที่มีอยู่จริงได้
-   **เครื่องอ่านแถบแม่เหล็ก (MSR)** – คุณสามารถส่งเหตุการณ์แถบแม่เหล็กเลียนแบบ POS จากโปรแกรมจำลองที่ต่อพ่วงได้
-   **ลิ้นชัก**– คุณสามารถจำลองแบบลิ้นชักเงินสดที่มีอยู่จริงได้
-   **ลิ้นชัก 2** – โดยการตั้งค่าแบบลิ้นชักเงินสดที่สองในโปรแกรมจำลองการต่อพ่วง คุณสามารถจำลองสถานการณ์ที่เกี่ยวข้องกับการลงทะเบียน POS เดียวที่มีสองกะที่ใช้งานอยู่ ได้
-   **สแกนเนอร์**– เสมือนบาร์โค้ดสแกนเนอร์ที่สนับสนุนโปรแกรมจำลองการต่อพ่วงสามารถออกเหตุการณ์สแกนบาร์โค้ดได้
-   **มาตราส่วน**– มาตราส่วนแบบเสมือนช่วยให้คุณสามารถจำลองการโต้ตอบของสินค้าที่ชั่งน้ำหนักให้กับ POS
-   **แผ่น (PIN) หมายเลขประจำตัวประชาชน**– คุณสามารถจำลองการดำเนินงานแผ่น PIN ได้ **หมายเหตุ:**คุณต้องใช้การสนับสนุนสำหรับแป้นกดรหัสมีอยู่จริงผ่านตัวเชื่อมต่อการชำระเงิน
-   **เก็บบันทึกลายเซ็น**– โปรแกรมจำลองการต่อพ่วงอุปกรณ์จับภาพเสมือนลายเซ็นที่คุณสามารถตั้งค่าการแสดงพร้อมท์สำหรับลายเซ็นที่จำเป็นสำหรับบางวิธีการชำระเงิน เช่นการชำระเงินในบัญชีลูกค้า ที่มีอยู่

นอกจากนี้คุณสามารถใช้โปรแกรมจำลองการต่อพ่วงเพื่อจำลองเหตุการณ์เครื่องอ่านบัตรเครดิตของแป้นพิมพ์ที่มีต้นกำเนิดมาจากบาร์โค้ดสแกนเนอร์และ MSR โปรแกรมจำลองการต่อพ่วงเสมือนโดยเฉพาะอย่างยิ่งสนับสนุนวัตถุเชื่อมโยงและ Embedding สำหรับอุปกรณ์ (OPOS Retail POS)

## <a name="key-scenarios"></a>สถานการณ์สมมติที่สำคัญ
### <a name="troubleshooting"></a>การแก้ไขปัญหา

คุณสามารถใช้โปรแกรมจำลองการต่อพ่วงเพื่อการแก้ไขปัญหาการติดตั้งอุปกรณ์ ถ้าคุณไม่มีโปรแกรมจำลองที่ต่อพ่วงหรืออุปกรณ์สองของประเภทเดียวกัน อาจเป็นเรื่องยากที่จะกำหนดที่มาจากปัญหา อย่างไรก็ตาม เมื่อคุณมีโปรแกรมจำลองที่ต่อพ่วง คุณสามารถตั้งค่าอุปกรณ์เสมือน และรันตรรกะทางธุรกิจที่ใช้งานสำหรับอุปกรณ์ที่มีอยู่จริงและเส้นทางรหัสเดียวกัน จากมุมมองของโปรแกรมจำลองการต่อพ่วง ความแตกต่างหลักระหว่างอุปกรณ์เสมือนและอุปกรณ์ที่มีอยู่จริงคือ วัตถุที่ให้บริการ หรือโปรแกรมควบคุมอุปกรณ์ สำหรับอุปกรณ์ที่มีอยู่จริง วัตถุที่ให้บริการมีให้ โดยผู้ผลิตอุปกรณ์ ในทางกลับกัน สำหรับโปรแกรมจำลองต่อพ่วง วัตถุที่ให้บริการไว้ให้เป็นส่วนหนึ่งของโปรแกรมจำลองการต่อพ่วง เมื่อโปรแกรมจำลองที่ต่อพ่วงทำงานได้อย่างถูกต้อง ถ้าอุปกรณ์ไม่ทำงานอย่างถูกต้องหลังจากที่มีเปลี่ยนชื่ออุปกรณ์ในโพรไฟล์ฮาร์ดแวร์ให้เป็นชื่อของอุปกรณ์จริง คุณสามารถสมมติว่า ไม่มีปัญหากับวัตถุที่ให้บริการที่ผู้ผลิตให้มา

### <a name="training"></a>การฝึกอบรม

คุณสามารถใช้โปรแกรมจำลองการต่อพ่วงเพื่อเพิ่มชั้นที่เป็นจริงเพื่อฝึกอบรมการตั้งค่าฮาร์ดแวร์จริงไม่พร้อมใช้งานเมื่อพนักงานเก็บเงิน เมื่อโปรแกรมจำลองที่ต่อพ่วงจะรวมอยู่ในสถานการณ์การฝึกอบรม พนักงานเก็บเงินสามารถได้อย่างมีประสิทธิภาพมากขึ้นโต้ตอบกับ POS โดยให้ป้อนข้อมูลเช่นผลิตภัณฑ์บาร์โค้ดสแกนและของขวัญบัตร swipes และปฏิบัติตามนโยบายการรับที่จะพิมพ์สำหรับธุรกรรมที่เฉพาะเจาะจง

### <a name="testing"></a>การทดสอบ

คุณสามารถใช้โปรแกรมจำลองการต่อพ่วงเพื่อทดสอบผลิตภัณฑ์บาร์โค รูปแบบใบเสร็จ และอื่น ๆ โดยไม่ต้องปรับใช้ฮาร์ดแวร์ในสภาพแวดล้อมเสมือนที่มีอยู่จริง เนื่องจากฮาร์ดแวร์จริงไม่จำเป็น และคุณไม่จำเป็นต้องปรับใช้ไคลเอนต์ POS บนสถานีฮาร์ดแวร์หรือคอมพิวเตอร์ที่มีอยู่จริง คุณสามารถทดสอบการเปลี่ยนแปลงที่ทำในสำนักงานย้อนกลับได้รวดเร็วขึ้น

## <a name="set-up-the-peripheral-simulator"></a>การติดตั้งโปรแกรมจำลองที่ต่อพ่วง
### <a name="set-up-a-hardware-profile"></a>ตั้งค่าโพรไฟล์ฮาร์ดแวร์

1.  เมื่อต้องการติดตั้งโปรแกรมจำลองที่ต่อพ่วง ไป**ขายปลีกและพาณิชย์**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โพรไฟล์ POS**&gt;**โพรไฟล์ฮาร์ดแวร์**
2.  เมื่อต้องการสร้างโพรไฟล์ใหม่ คลิ**สร้าง**
3.  ป้อนค่าในการ**หมายเลขของโพรไฟล์**และ**คำอธิบาย**เขตข้อมูล
4.  ใช้ตารางต่อไปนี้เพื่อติดตั้งอุปกรณ์เสมือนที่ต้องมีทดสอบ นี่คือคำอธิบายของคอลัมน์ในตาราง:
    -   **อุปกรณ์**– คอลัมน์นี้แสดงชื่อของแท็บด่วนที่คุณขยายการติดตั้งอุปกรณ์
    -   **ชนิดอุปกรณ์**– คอลัมน์นี้แสดงค่าที่คุณเลือกในฟิลด์ซึ่งจะติดป้ายชื่อของอุปกรณ์
    -   **ชื่ออุปกรณ์**– คอลัมน์นี้แสดงค่าที่แน่นอนที่คุณป้อนสำหรับชื่ออุปกรณ์ **สิ่งสำคัญ:**ชื่ออุปกรณ์ที่จะถูกกำหนดที่นี่จำเป็น เนื่องจากสถานีฮาร์ดแวร์ใช้ชื่อเหล่านี้เฉพาะที่อยู่ในอุปกรณ์ ถ้าคุณไม่ได้ใช้ชื่อเหล่านี้เฉพาะ อุปกรณ์จะไม่สามารถใช้งานได้

    | อุปกรณ์            | ชนิดอุปกรณ์ | ชื่ออุปกรณ์              |
    |-------------------|-------------|--------------------------|
    | เครื่องพิมพ์           | OPOS        | MockOPOSPrinter          |
    | จอแสดงผลรายการ      | OPOS        | MockOPOSLineDisplay      |
    | MSR               | OPOS        | MockOPOSMSR              |
    | ผู้ออกตั๋วเงิน            | OPOS        | MockOPOSDrawer1          |
    | Drawer2           | OPOS        | MockOPOSDrawers          |
    | สแกนเนอร์           | OPOS        | MockOPOSScanner          |
    | สเกล             | OPOS        | MockOPOSScale            |
    | แป้นกดรหัส           | OPOS        | MockOPOSPinPad           |
    | การเก็บบันทึกลายเซ็น | OPOS        | MockOPOSSignatureCapture |

**หมายเหตุ:**ไม่มีการใช้การตั้งค่าเฉพาะในโพรไฟล์ฮาร์ดแวร์ต้องจำลองเหตุการณ์เครื่องอ่านบัตรเครดิตแป้นพิมพ์จากสแกนเนอร์บาร์โค้ดและ MSR

### <a name="assign-the-hardware-profile-to-a-register"></a>กำหนดโพรไฟล์ฮาร์ดแวร์ให้กับรีจิสเตอร์

1.  หลังจากที่มีสร้างโพรไฟล์ฮาร์ดแวร์ ไป**พาณิชย์และผู้ค้าปลีก**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**การลงทะเบียน**
2.  ในการ**เครื่องบันทึกเงินสด POS**รายการ คลิกการเชื่อมโยงในการ**หมายเลขลงทะเบียน**สำหรับเครื่องบันทึกเงินสดที่ควรใช้โปรแกรมจำลองที่ต่อพ่วง
3.  คลิก **แก้ไข**
4.  ในการ**โพรไฟล์**จากนั้น ใน**โพรไฟล์ฮาร์ดแวร์**ฟิลด์ เลือกโพรไฟล์ฮาร์ดแวร์ที่คุณสร้างขึ้นสำหรับอุปกรณ์ต่อพ่วงเสมือน
5.  Click **Save**.

### <a name="synchronize-changes-to-the-channel-database"></a>ซิงโครไนส์การเปลี่ยนแปลงไปยังฐานข้อมูลสถานี

1.  ไปที่**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**
2.  เลือก**1090**กำหนดการกระจาย
3.  คลิ**เรียกใช้เดี๋ยวนี้**การซิงโครไนส์การเปลี่ยนแปลงกับ POS

หลังจากที่ข้อมูลจะซิงโครไนส์ โพรไฟล์ฮาร์ดแวร์ใหม่และเปลี่ยนแปลงบนเครื่องบันทึกเงินสดมีอยู่ในฐานข้อมูลสถานี

## <a name="install-the-peripheral-simulator"></a>ติดตั้งโปรแกรมจำลองที่ต่อพ่วง
1.  ไปที่**พาณิชย์และผู้ค้าปลีก**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โพรไฟล์ POS**&gt;**โพรไฟล์ฮาร์ดแวร์**
2.  คลิ**ดาวน์โหลด**แล้ว คลิก**PeripheralSimulator** **หมายเหตุ:**คุณต้องปิดใช้งานตัวบล็อกป๊อปอัพก่อนที่คุณสามารถดาวน์โหลดโปรแกรมจำลองที่ต่อพ่วง
3.  หลังจากการดาวน์โหลดเสร็จสมบูรณ์ เปิด**ดาวน์โหลด**โฟลเดอร์และคลิกสองครั้ง**VirtualPeripherals.msi**เพื่อเริ่มโปรแกรมติดตั้ง
4.  ติดตั้งโปรแกรมจำลองที่ต่อพ่วง โดยใช้การตั้งค่าเริ่มต้น

นอกเหนือจากโปรแกรมจำลองต่อพ่วง คุณต้องติดตั้งตัววัตถุตัวควบคุมทั่วไปจากบริการปรึกษา Monroe มิฉะนั้น โปรแกรมจำลองการต่อพ่วงจะไม่ทำงานอย่างถูกต้อง เมื่อต้องการดาวน์โหลดตัววัตถุตัวควบคุมทั่วไป ไป<http://monroecs.com/oposccos_current.htm>

## <a name="using-the-peripheral-simulator"></a>ใช้โปรแกรมจำลองการต่อพ่วง
เมื่อต้องการเริ่มโปรแกรมจำลองที่ต่อพ่วง คลิ**เริ่ม**บนคอมพิวเตอร์ของคุณ พิมพ์**โปรแกรมจำลองต่อพ่วงขายปลีก**แล้ว เลือกโปรแกรมประยุกต์เมื่อปรากฏในผลลัพธ์การค้นหา หลังจากที่คุณเริ่มโปรแกรมจำลองที่ต่อพ่วง คลิกชื่ออุปกรณ์เพื่อดูอุปกรณ์ได้รับการสนับสนุน อุปกรณ์เหล่านี้จะแสดงรายการเป็นแท็บทางด้านซ้ายของหน้าต่าง เมื่อต้องการดูอุปกรณ์เฉพาะ คลิกแท็บสำหรับอุปกรณ์นั้น

### <a name="line-display-capabilities"></a>ความสามารถในการแสดงบรรทัด

การแสดงรายการเป็นอุปกรณ์แรกที่แสดงอยู่ในโปรแกรมจำลองที่ต่อพ่วง เมื่อมีการกำหนดค่าจอแสดงผลรายการเสมือน โปรแกรมจะแสดงรายการบรรทัด ตามที่พวกเขาจะสแกนในธุรกรรม POS นอกเหนือจากรายการ จอแสดงผลแสดงผลรวมที่ครบกำหนดเมื่อมีเลือกการชำระเงินที่ POS โปรแกรมจะแสดงดุลที่ครบกำหนดนอกจากนี้ถ้ามีป้อนการชำระเงิน แต่ดุลยังคงเป็นครบกำหนดชำระสำหรับธุรกรรม เมื่อไม่ได้กำลังใช้ POS คุณสามารถแสดงข้อความเพื่อบ่งชี้ว่า มีปิดลิ้นชักเก็บเงิน คุณต้องกำหนดค่าข้อความในแบบ**บรรทัดแสดง**ด่วนในโพรไฟล์ฮาร์ดแวร์

### <a name="cash-drawer-capabilities"></a>ความสามารถในลิ้นชักเงินสด

ลิ้นชักเงินสดเป็นอุปกรณ์ที่สองที่แสดงอยู่ในโปรแกรมจำลองที่ต่อพ่วง เมื่อมีการกำหนดค่าโพรไฟล์ฮาร์ดแวร์จะใช้ลิ้นชักเงินสดเสมือน POS เปิดลิ้นชักเงินสดสำหรับกะที่ใช้งานอยู่ในการตอบสนองต่อการดำเนินงานในลิ้นชักเช่นการนับ และเพื่อ ให้พนักงานเก็บเงินสามารถทำการเปลี่ยนแปลง หรือฝากเงินสดระหว่างธุรกรรม cash-and-carry มาตรฐาน ลิ้นชักเงินสดเสมือนที่มีป้ายชื่อ**ลิ้นชักหลัก**และ**ลิ้นชักรอง** ป้ายชื่อเหล่านี้แสดงถึงลิ้นชักและลิ้นชัก 2 ในโพรไฟล์ฮาร์ดแวร์ ตามลำดับนั้น เมื่อปิดลิ้นชักแบบ แสดงรูปแบบลิ้นชักเงินสดปิด และปุ่มในลิ้นชักเงินสดปิดจะติดป้าย**เปิดลิ้นชัก** ถ้าคุณคลิกปุ่มนี้ รูปภาพจะถูกแทนที่ ด้วยรูปภาพของตัวลิ้นชักเงินสดที่เปิดเพื่อบ่งชี้ว่า จะเปิดลิ้นชัก ปุ่มในลิ้นชักเงินสดที่เปิดจะมีป้ายชื่อว่า**ปิดลิ้นชัก** หลาย ๆ การดำเนินงานที่ POS สามารถทำให้เกิดการเปิดลิ้นชักเงินสด การดำเนินการส่วนใหญ่ไม่สามารถดำเนินการขณะเปิดลิ้นชักเงินสด ข้อยกเว้นมีบางขั้นตอนสิ้นสุดวัน ถ้าผู้ใช้ POS ได้รับข้อความแสดงข้อผิดพลาดที่แจ้งว่า ไม่สามารถทำการดำเนินการขณะเปิดลิ้นชักเงินสด ผู้ใช้ต้องปิดลิ้นชักเงินสดที่มีอยู่จริง หรือเสมือนเพื่อดำเนินต่อ ถ้าลิ้นชักเงินสดที่ถูกทำเครื่องหมายเป็น**ที่ใช้ร่วมกัน**ในโพรไฟล์ฮาร์ดแวร์ ระบบจะไม่ตรวจสอบว่า ลิ้นชักถูกปิดก่อนที่จะดำเนินการ การดำเนินงานที่ดำเนินการตามปกติ ถ้าเปิดลิ้นชักเงินสด ลักษณะการทำงานนี้สนับสนุนสถานการณ์ใช้ร่วมกันระหว่างสมาคมขายลิ้นชักเงินสด และหนึ่งกรณีเชื่อมโยงใช้เป็นลิ้นชักเงินสดในขณะที่การเชื่อมโยงอื่นทำงานต่าง ๆ ที่ไม่เกี่ยวข้องกับอุปกรณ์ POS ของ ตนเอง การเปลี่ยนแปลงที่เกิดขึ้นกับลิ้นชักเงินสดไม่ได้ evident จนกว่าปิดกะปัจจุบัน และมีเปิดกะใหม่

### <a name="msr-capabilities"></a>ความสามารถของ MSR

โปรแกรมจำลองที่ต่อพ่วงให้แข็งแกร่งสำหรับการดำเนินการ MSR เสมือน ด้วยการทำงานในโหมด OPOS หรือโหมดเครื่องอ่านบัตรเครดิตของแป้นพิมพ์ โหมด OPOS ต้องว่า MSR ที่ถูกกำหนดค่าในโพรไฟล์ฮาร์ดแวร์จะทำงานเป็นอุปกรณ์ OPOS แป้นพิมพ์เครื่องอ่านบัตรเครดิตโหมดส่งเหตุการณ์ข้อมูลเครื่องอ่านบัตรเครดิตแป้นพิมพ์ Microsoft Windows เป็นเพียง นอกเหนือจากความแตกต่างในการตั้งค่า โหมดเครื่องอ่านบัตรเครดิต OPOS และแป้นพิมพ์แตกต่างกันไปตามวิธีการต่อไปนี้:

-   ไคลเอนต์ POS ช่วยให้อุปกรณ์ OPOS MSR สำหรับสถานการณ์ เช่นสถานการณ์ที่อนุญาตให้ข้อมูลแถบแม่เหล็กสำหรับสมาชิกหรือรายการบัตรของขวัญ
-   ในโหมดของเครื่องอ่านบัตรเครดิตแป้นพิมพ์ โปรแกรมจำลองที่ต่อพ่วงส่งข้อมูลของเครื่องอ่านบัตรเครดิตแป้นพิมพ์ไปยังเขตข้อมูลที่ใช้งานอยู่เมื่อมีส่งข้อมูล ลักษณะการทำงานนี้คล้ายคลึงกับลักษณะการทำงานที่เกิดขึ้นเมื่อมีป้อนข้อมูล ด้วยการใช้แป้นพิมพ์ การใช้ msr ด้วยเป็นเครื่องอ่านบัตรเครดิตแป้นพิมพ์ ผู้ใช้ต้องสลับการขายปลีกสมัยใหม่ POS (MPOS) เพื่อให้แน่ใจว่า ข้อมูลจะได้รับในเขตข้อมูลถูกต้อง ดังนั้น คุณสามารถกำหนดค่าการหน่วงเวลา เพื่อให้ผู้ใช้มีเวลาเพื่อให้แน่ใจว่า ข้อมูลจะถูกส่งไปยังเขตข้อมูลที่ถูกต้อง

#### <a name="testing-gift-and-payment-card-swipes"></a>การทดสอบ swipes บัตรของขวัญและการชำระเงิน

Msr ด้วยเสมือนซึ่งมีโปรแกรมจำลองที่ต่อพ่วงยังช่วยให้คุณตั้งค่าคอนฟิกข้อมูล MSR ที่เฉพาะเจาะจงเพื่อทดสอบสถานการณ์สมมติของการ swipes บัตรของขวัญและการชำระเงิน เมื่อต้องการสร้างบัตร คลิกเครื่องหมายบวก (**+**) ปุ่ม และเลือกชนิดของบัตร จากนั้นระบุหมายเลขบัตร หรือติดตามข้อมูลที่ควรถูกส่งไปยัง POS พร้อมด้วยวันหมดอายุเดือนและปีสำหรับบัตรที่คุณกำลังกำหนด ค่าที่คุณเลือกในแบบ**ชนิดของบัตร**ฟิลด์จะเพียงแค่ป้ายชื่อที่สามารถแม็ปกับบัตร ป้ายชื่อนี้ทำให้ง่ายต่อการระบุบัตรเมื่อพวกเขากำลังรูดผ่านโปรแกรมจำลองที่ต่อพ่วง คุณสามารถเลือกบัตรที่ได้รับการกำหนดค่าในโปรแกรมจำลองการต่อพ่วง โดยใช้ลูกศรซ้าย (**&lt;**) และลูกศรขวา (**&gt;**) ปุ่มด้านบนรูปของบัตรได้ คุณสามารถแก้ไข และลบบัตรโดย**แก้ไข**และ**ลบ**ปุ่มที่อยู่ติดกับเครื่องหมายบวก (**+**) ปุ่มได้

### <a name="pin-pad"></a>แป้นกดรหัส

คุณสามารถกำหนดค่าโปรแกรมจำลองการแผ่น PIN เพื่อจำลองแผ่นผิด OPOS PIN เมื่อธุรกรรมการโอนย้าย (EFT) เงินทางอิเล็กทรอนิกส์มีดำเนินการที่ POS และจำเป็นต้องมีการป้อน PIN อุปกรณ์ฮาร์ดแวร์ที่เรียกอุปกรณ์ PIN จะแสดงพร้อมท์สำหรับการป้อน PIN การทำงาน แป้นกดรหัสในโปรแกรมจำลองการต่อพ่วงต้องสนับสนุนตัวเชื่อมต่อการชำระเงิน EFT

### <a name="printer"></a>เครื่องพิมพ์

เครื่องพิมพ์ที่ต่อพ่วงเสมือนแสดงใบเสร็จรับเงินเพียงแค่พิมพ์จาก POS ถ้าการดำเนินงานพิมพ์ที่ก่อให้เกิดการรับสินค้าหลาย คุณสามารถเลื่อนการรับสินค้า

#### <a name="configure-receipt-printing"></a>การตั้งค่าคอนฟิกการพิมพ์ใบเสร็จ

1.  ไปที่**พาณิชย์และผู้ค้าปลีก**&gt;**ตั้งค่าช่องสัญญาณ**&gt;**ตั้งค่า POS**&gt;**โพรไฟล์ POS**&gt;**โพรไฟล์ฮาร์ดแวร์**
2.  เลือกโพรไฟล์ฮาร์ดแวร์ที่คุณสร้างขึ้นสำหรับอุปกรณ์ต่อพ่วงเสมือน
3.  ในการ**เครื่องพิมพ์**ด่วน คลิ**แก้ไข**
4.  ในการ**รหัสโพรไฟล์ใบเสร็จ**ฟิลด์ เลือกโพรไฟล์ใบเสร็จ
5.  Click **Save**.

### <a name="scale"></a>สเกล

เมื่อมีเพิ่มผลิตภัณฑ์มาตราส่วนกับธุรกรรม POS และมีการกำหนดค่าเครื่องชั่ง POS ดึงน้ำหนักจากเครื่องชั่ง สำหรับทั้งทางกายภาพ และเสมือนเครื่องชั่ง ผลิตภัณฑ์หรือน้ำหนักควรระบุก่อนผลิตภัณฑ์จะเพิ่มลงในธุรกรรม ก่อนที่คุณเพิ่มผลิตภัณฑ์ที่ชั่งน้ำหนักลงในธุรกรรม ไปมาตราส่วนในโปรแกรมจำลองการต่อพ่วง และใช้เครื่องหมายบวก (**+**) และเครื่องหมายลบ (**–**) ปุ่มปรับน้ำหนักที่เครื่องชั่งควรรายงาน คุณยังสามารถป้อนน้ำหนักที่ต้องการได้โดยตรงในการ**ค่าปัจจุบัน**เขตข้อมูลได้ คุณสามารถปรับเปลี่ยนหน่วยของน้ำหนักสำหรับเครื่องชั่ง โดยใช้เครื่องหมายบวก (**+**),**แก้ไข**และ**ลบ**ปุ่มได้ ด้วยวิธีนี้ หน่วยสามารถระบุโดยยึดตามผลิตภัณฑ์ที่จะให้น้ำหนักหรือตำแหน่งที่ตั้งที่มีใช้มาตราส่วน

#### <a name="configure-a-scale-product"></a>การตั้งค่าคอนฟิกผลิตภัณฑ์มาตราส่วน

1.  ไปที่**ขายปลีก และ****พาณิชย์**&gt;**ผลิตภัณฑ์และประเภท**&gt;**ผลิตภัณฑ์ที่นำออกใช้ตามประเภท**
2.  เปิดเรกคอร์ดผลิตภัณฑ์
3.  เลือกผลิตภัณฑ์ชั่งน้ำหนัก
4.  บน**ขายปลีก**ด่วน การตั้งค่า**ผลิตภัณฑ์ที่ชั่งน้ำหนัก**ตัวเลือกจาก**ไม่ใช่**ไป**ใช่**

#### <a name="synchronize-changes-to-the-channel-database"></a>ซิงโครไนส์การเปลี่ยนแปลงไปยังฐานข้อมูลสถานี

1.  ไปที่**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**
2.  เลือก**1040 ติด**กำหนดการกระจาย
3.  คลิ**เรียกใช้เดี๋ยวนี้**การซิงโครไนส์การเปลี่ยนแปลงกับ POS

หลังจากที่ข้อมูลจะซิงโครไนส์ เมื่อมีเพิ่มมาตราส่วนผลิตภัณฑ์กับธุรกรรม POS, POS ตรวจสอบเครื่องชั่งน้ำหนัก

### <a name="signature-capture"></a>การเก็บบันทึกลายเซ็น

อุปกรณ์เก็บบันทึกลายเซ็นเสมือนพร้อมท์ให้ผู้ใช้เพื่อให้ลายเซ็นบนแผ่นการเก็บบันทึกลายเซ็นเสมือนเมื่อวิธีการชำระเงินที่ใช้ต้องมีลายเซ็น ผู้ใช้สามารถยอมรับลายเซ็นเพื่อแสดงหน้าต่างที่ POS พนักงานเก็บเงินสามารถยอมรับการลายเซ็นแล้ว ลายเซ็นจะถูกบันทึกไว้แล้วพร้อมกับวิธีการชำระเงิน และจะซิงโครไนส์กับ office ค้างส่งพร้อมกับข้อมูลธุรกรรมอื่น ๆ

#### <a name="set-up-a-tender-to-require-a-signature"></a>ตั้งค่าวิธีการชำระเงินที่ต้องใช้ลายเซ็น

1.  ไปที่**พาณิชย์และผู้ค้าปลีก**&gt;**สถานี**&gt;**ร้านค้าปลีก**&gt;**ทั้งหมดขายปลีกร้านค้า**
2.  เลือกร้านค้าปลีก
3.  คลิก **แก้ไข**
4.  คลิ**ตั้งค่า**จากนั้น ใน**ตั้งค่า**ส่วน คลิ**วิธีการชำระเงิน**
5.  คลิก **แก้ไข**
6.  เลือกวิธีการชำระเงินที่ต้องใช้ลายเซ็น
7.  ในการ**ทั่วไป**จากนั้น ภายใต้**จับภาพลายเซ็น**ตั้งค่าการ**อุปกรณ์เก็บบันทึกลายเซ็นใช้**ตัวเลือกเพื่อ**ใช่**
8.  ในการ**ยอดเงินต่ำสุดของการเก็บบันทึกลายเซ็น**ฟิลด์ ป้อนยอดเงินต่ำสุดที่ควรทริกเกอร์การเก็บบันทึกลายเซ็น

#### <a name="synchronize-changes-to-the-channel-database"></a>ซิงโครไนส์การเปลี่ยนแปลงไปยังฐานข้อมูลสถานี

1.  ไปที่**ขายปลีกและพาณิชย์**&gt;**ไอทีขายปลีก**&gt;**กำหนดการกระจาย**
2.  เลือก**1070**กำหนดการกระจาย
3.  คลิ**เรียกใช้เดี๋ยวนี้**การซิงโครไนส์การเปลี่ยนแปลงกับ POS

หลังจากที่ข้อมูลจะซิงโครไนส์ เมื่อใช้วิธีการชำระเงินที่ต้องใช้ลายเซ็น และยอดเงินสอดคล้องกับขีดจำกัดลายเซ็น POS แสดงพร้อมท์สำหรับลายเซ็นบนอุปกรณ์เก็บบันทึกลายเซ็นเสมือน

## <a name="additional-configuration"></a>การตั้งค่าคอนฟิกเพิ่มเติม
คุณสามารถแก้ไขแฟ้มการกำหนดค่าของโปรแกรมจำลองการต่อพ่วงเพื่อแก้ไขสถานการณ์ที่คุณกำลังทดสอบอย่างเหมาะสมมากขึ้น คุณสามารถค้นหาแฟ้มการกำหนดค่าที่ c:\\แฟ้มโปรแกรม (x86)\\Microsoft Dynamics 365\\70\\VirtualPeripherals\\Microsoft.Dynamics.Commerce.VirtualPeripherals.Client.exe.config ได้ แฟ้มการกำหนดค่ากำหนดหน่วยที่พร้อมใช้งานสำหรับการทดสอบบนเครื่องชั่ง ชนิดของบัตรที่พร้อมใช้งานสำหรับการทดสอบ และชนิดของบาร์โค้ด ตัวอย่างเช่น ด้วยการปรับเปลี่ยนค่าในไฟล์การตั้งค่าคอนฟิกข้อความ คุณสามารถเพิ่มชนิดของบัตรใหม่หรือหน่วยวัดที่สามารถเลือกได้ในขณะใช้งาน ค่าใหม่จะปรากฏขึ้นหลังจากที่เริ่มโปรแกรมประยุกต์นี้

## <a name="troubleshooting"></a>การแก้ไขปัญหา
กิจกรรมสำหรับโปรแกรมจำลองที่ต่อพ่วงเข้าสู่ระบบภายในโปรแกรมจำลองที่ต่อพ่วง คุณสามารถค้นหาแฟ้มบันทึกที่ c:\\แฟ้มโปรแกรม (x86)\\Microsoft Dynamics 365\\70\\VirtualPeripherals\\Microsoft.Dynamics.Commerce.VirtualPeripherals.Client.exe.config ได้ โปรแกรมจำลองที่ต่อพ่วงยังแจ้งปัญหาการล็อกเหตุการณ์ของ Windows ซึ่งคุณสามารถเข้าถึงใน**แอพลิเคชันบันทึกและบริการ**&gt;**Microsoft**&gt;**DynamicsAX** ถ้าการเปลี่ยนแปลงที่คุณทำกับโพรไฟล์ฮาร์ดแวร์หรือส่วนอื่นๆ ไม่ evident เมื่อคุณใช้ MPOS หรือโปรแกรมจำลองการต่อพ่วง ตรวจสอบการกระจายตัวจัดกำหนดการงานที่คุณใช้ในการซิงโครไนส์ข้อมูลไปยังฐานข้อมูลสถานี ถ้ามีการเปลี่ยนแปลงถูกทำให้ตรงกัน แต่ยังคง ไม่ใช่ evident ที่ POS รีสตาร์ทไคลเอนต์ POS ลิ้นชักเงินสดที่จัดโครงแบบการเปลี่ยนแปลงไม่มีผลจนกว่ากะใหม่ถูกสร้างขึ้น ดังนั้น ถ้าคุณทำการเปลี่ยนแปลงลิ้นชักเงินสด ตรวจสอบให้แน่ใจว่า คุณปิดกะที่มีอยู่เมื่อต้องการทดสอบการตั้งค่าใหม่ของลิ้นชักเงินสดเสมอ บางครั้ง ถ้ามีการติดตั้งโปรแกรมควบคุมของบริษัทผู้ผลิตหลังจากวัตถุตัวควบคุมทั่วไปจากบริการปรึกษา Monroe โปรแกรมควบคุมอาจทำให้วัตถุตัวควบคุมทั่วไปสามารถทำงานได้อย่างถูกต้อง ในกรณีนี้ คุณควรติดตั้งตัววัตถุตัวควบคุมทั่วไป
