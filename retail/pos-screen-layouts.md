# <a name="configure-screen-layouts-for-pos"></a>กำหนดค่าโครงร่างหน้าจอสำหรับ POS

หัวข้อนี้ให้ข้อมูลเกี่ยวกับโครงร่างหน้าจอสำหรับ 365 Dynamics ของ Microsoft สำหรับการดำเนินงาน - จุดขายปลีกของประสบการณ์การขาย (POS)

365 Dynamics ของ Microsoft สำหรับการดำเนินงาน - จุดขายปลีกของอินเทอร์เฟซการขาย (POS) ผู้ใช้สามารถกำหนดค่าโดยใช้การรวมกันของโพรไฟล์ภาพและเค้าโครงหน้าจอ การกำหนดให้กับร้านค้า ทะเบียน และ/หรือผู้ใช้

## <a name="visual-profile"></a>โพรไฟล์ภาพ
โพรไฟล์ภาพจะถูกกำหนดให้กับเครื่องบันทึกเงินสด และใช้ในการระบุองค์ประกอบภาพที่เครื่องบันทึกเงินสดเฉพาะ และที่ใช้ร่วมกันระหว่างผู้ใช้ ผู้ใช้ที่ล็อกไว้ในการลงทะเบียนจะได้แบบเดียวกับชุดรูปแบบ สี และรูป **หมายเลขโพรไฟล์**-หมายเลขโพรไฟล์เป็นรหัสเฉพาะสำหรับโพรไฟล์ภาพ **คำอธิบาย**-คำอธิบายช่วยให้คุณสามารถระบุชื่อที่มีความหมายซึ่งจะช่วยให้คุณระบุโพรไฟล์ที่ถูกต้องสำหรับสถานการณ์ของคุณ **ชุดรูปแบบ**-ผู้ใช้สามารถเลือกระหว่างชุดรูปแบบแอพลิเคชันอ่อนหรือเข้มขึ้น การตั้งค่าเหล่านี้มีผลกระทบต่อสีของแบบอักษรและพื้นหลังทั่วทั้งแอพลิเคชัน **เน้นสี**-สีส่วนที่ถูกเน้นที่ใช้ตลอดทั้ง POS เพื่อแยกความแตกต่าง หรือเน้นบางองค์ประกอบด้านภาพเช่นไพ่ ปุ่มคำสั่ง หรือการเชื่อมโยงหลายมิติ องค์ประกอบเหล่านี้จะสามารถดำเนินการแบบปกติ **สีส่วนหัว**-สีของหัวข้อช่วยให้ผู้ใช้สามารถกำหนดสีของส่วนหัวของหน้าเพื่อให้ตรงกับความต้องการตราสินค้าของผู้ค้าปลีกที่ คุณลักษณะนี้จะพร้อมใช้งานใน Dynamics 365 สำหรับการดำเนินงานรุ่น 1611 เท่านั้น **พื้นหลังการเข้าสู่ระบบ**-ผู้ใช้สามารถระบุรูปพื้นหลังสำหรับหน้าจอการเข้าสู่ระบบได้ ขนาดแฟ้มของรูปภาพพื้นหลังควรเก็บไว้ขนาดเล็กสุด ตามที่จัดเก็บ และโหลดแฟ้มที่มีขนาดใหญ่สามารถมีผลกระทบต่อลักษณะการทำงานของแอพลิเคชันและประสิทธิภาพการทำงาน **พื้นหลังโปรแกรมประยุกต์**-POS ยังสามารถใช้รูปภาพเป็นพื้นหลังทั่วทั้งแอพลิเคชันแทนที่ชุดรูปแบบสีทึบสีได้ เช่นเดียวกับพื้นหลังการเข้าสู่ระบบ นั้นจะแนะนำการเก็บขนาดแฟ้มที่น้อยที่สุดที่เป็นไปได้

## <a name="screen-layouts"></a>โครงร่างหน้าจอ
การตั้งค่าคอนฟิกโครงร่างหน้าจอกำหนดว่าการดำเนินการ เนื้อหา และตำแหน่งของตัวควบคุม UI ในหน้าจอต้อนรับ POS และหน้าจอของธุรกรรม ** หน้าจอต้อนรับ ** -ในกรณีส่วนใหญ่ หน้าจอต้อนรับคือ เพจที่ผู้ใช้จะเห็นเมื่อพวกเขาก่อนเข้าสู่ระบบ POS หน้าจอต้อนรับอาจประกอบด้วยรูปแบบตราสินค้าและกริดปุ่มที่ช่วยให้เข้าถึงการดำเนินงานของ POS โดยปกติ การดำเนินการที่ไม่เฉพาะเจาะจงกับธุรกรรมปัจจุบันจะถูกวางไว้ที่นี่ **หน้าจอธุรกรรม**-หน้าจอของธุรกรรมที่เป็นหน้าจอหลักใน POS สำหรับธุรกรรมการขายและใบสั่งประมวลผล หน้าจอธุรกรรมคุณสามารถกำหนดค่าโดยใช้โปรแกรมออกแบบโครงร่างหน้าจอ **หน้าจอเริ่มต้น**-ผู้ค้าปลีกบางอย่างต้องการให้พนักงานเก็บเงินไปยังหน้าจอธุรกรรมโดยตรงหลังจากการเข้าสู่ระบบใน การตั้งค่าหน้าจอเริ่มต้นค่าเริ่มต้นช่วยให้ผู้ใช้สามารถตั้งค่านี้สำหรับแต่ละโครงร่างหน้าจอ

### <a name="assignment"></a>การกำหนด

สามารถกำหนดโครงร่างหน้าจอในระดับร้านค้า ทะเบียน หรือผู้ใช้ การมอบหมายผู้ใช้แทนการลงทะเบียน และจัดเก็บกำหนด และการแทนการกำหนดค่าเครื่องบันทึกเงินสดการกำหนดร้านค้า ในสถานการณ์อย่างง่ายที่ใช้เค้าโครงเดียวกันโดยไม่คำนึงถึงเครื่องบันทึกเงินสดหรือบทบาทของผู้ใช้ทั้งหมด สามารถตั้งโครงร่างหน้าจอที่ร้านค้าเท่านั้น ในกรณีที่เครื่องบันทึกเงินสดหรือผู้ใช้บางอย่างจำเป็นต้องมีเค้าโครงที่มีความเชี่ยวชาญ สามารถกำหนดอย่างเหมาะสม

### <a name="layout-sizes"></a>ขนาดโครงร่าง

คุณลักษณะนี้ใช้เฉพาะกับ Dynamics 365 สำหรับการดำเนินงานรุ่น 1611 เนื่องจากในหลายกรณี โครงร่างหน้าจอสามารถใช้งานข้ามหลายขนาดหน้าจอและวิธีแก้ปัญหา ผู้ใช้สามารถกำหนดเค้าโครงและเนื้อหาของพวกเขาสำหรับแต่ละ แอพลิเคชันของ POS โดยอัตโนมัติจะเลือกขนาดเค้าโครงใกล้เคียงที่สุดสำหรับอุปกรณ์ในขณะเริ่มต้น เค้าโครงหน้าจอยังอาจประกอบด้วยการตั้งค่าคอนฟิกสำหรับอุปกรณ์แบบเต็ม และแบบย่อ การตั้งค่าคอนฟิกนี้ช่วยให้ผู้ใช้สามารถถูกกำหนดให้กับเค้าโครงหน้าจอเดียวที่จะทำงานข้ามขนาดและสัดส่วนของแบบฟอร์มภายในร้านค้าต่าง ๆ **เต็ม POS สมัยใหม่ -** -เต็มรูปแบบเค้าโครงจะโดยทั่วไปจะใช้ได้ดีสำหรับแสดงมีขนาดใหญ่เช่นจอภาพของพีซีหรือ tablets ผู้ใช้สามารถเลือกว่าองค์ประกอบ UI การรวม กำหนดขนาดและตำแหน่งของพวกเขา และคุณสมบัติของรายละเอียดการตั้งค่าคอนฟิก เค้าโครงทั้งหมดสนับสนุนการตั้งค่าคอนฟิกทั้งแนวตั้งและแนวนอน **กระชับ POS สมัยใหม่ -** -เค้าโครงกระชับจะโดยทั่วไปจะใช้ได้ดีสำหรับโทรศัพท์หรือ tablets ขนาดเล็ก ออกแบบคือจำกัดสำหรับอุปกรณ์ที่กระชับข้อมูล ผู้ใช้สามารถกำหนดค่าคอลัมน์และเขตข้อมูลสำหรับบานหน้าต่างการรับสินค้าและผลรวม

### <a name="screen-layout-designer"></a>ตัวออกแบบโครงร่างหน้าจอ

แต่ละขนาดโครงร่างภายในเค้าโครงหน้าจอต้องถูกกำหนดค่าโดยใช้โปรแกรมออกแบบโครงร่างหน้าจอ ผู้ออกแบบช่วยให้ผู้ใช้สามารถระบุ และตั้งค่าคอนฟิกองค์ประกอบ UI ของหน้าจอของธุรกรรม โปรแกรมออกแบบโครงร่างหน้าจอใช้ ClickOnce เพื่อดาวน์โหลด ติดตั้ง และเปิดใช้งานแต่ละครั้งที่ผู้ใช้เข้าถึงโปรแกรมประยุกต์รุ่นล่าสุด ให้แน่ใจว่าคุณได้ตรวจสอบข้อกำหนดของเบราว์เซอร์สำหรับใช้ ClickOnce — เบราว์เซอร์บางชนิด เช่นโครเมียม ต้องมีส่วนขยาย **แผ่นหมายเลข**-แผ่นตัวเลขคือ การป้อนข้อมูลของผู้ใช้หลักในหน้าจอธุรกรรม POS สามารถกำหนดค่านั้นจะแสดงบนหน้าจอทั้งแบบแผ่น ซึ่งก็คือเหมาะสำหรับหน้าจอสัมผัส หรือเฉพาะฟิลด์ป้อนข้อมูล ซึ่งสามารถใช้กับแป้นพิมพ์ของจริง การตั้งค่าแผ่นหมายเลขไม่มีอยู่ในโครงร่างแบบเต็มเท่านั้น ใน 365 Dynamics สำหรับการดำเนินงานรุ่น 1611 เค้าโครงกระชับมีแผ่นตัวเลขทั้งหมดที่พร้อมใช้งานจากหน้าจอธุรกรรมเสมอ **ผลรวมแผง**- แผงผลรวมคุณสามารถกำหนดค่าในใดตัวหนึ่ง หรือสองคอลัมน์เพื่อแสดงเขตข้อมูลเช่นบรรทัดการตรวจนับ ยอดเงินส่วนลด ค่าธรรมเนียม ผลรวมย่อย และภาษี ใน 365 Dynamics สำหรับการดำเนินงานรุ่น 1611 เค้าโครงกระชับข้อมูลสนับสนุนคอลัมน์ผลรวมที่เดียวเท่านั้น **รับ**- ** ** แผงรับประกอบด้วยบรรทัดการขาย บรรทัดการชำระเงิน และรายละเอียดการจัดส่งสำหรับผลิตภัณฑ์และบริการที่มีการประมวลผลใน POS ผู้ใช้สามารถระบุคอลัมน์ ความกว้าง และการจัดวาง กระชับข้อมูลเค้าโครงใน Dynamics 365 สำหรับการดำเนินงานรุ่น 1611 คุณสามารถกำหนดค่าข้อมูลเพิ่มเติมซึ่งจะปรากฏในแถวใต้รายการหลัก **บัตรลูกค้า**- ข้อมูลแสดงบัตรลูกค้าที่เกี่ยวข้องกับลูกค้าที่สัมพันธ์กับธุรกรรม สามารถตั้งค่าคอนฟิกบัตรลูกค้าเพื่อซ่อน หรือแสดงรายละเอียดเพิ่มเติม **ตัวควบคุมแท็บ**- ตัวควบคุมแท็บสามารถวางลงบนเค้าโครงหน้าจอ และตัวควบคุมอื่น ๆ เช่นแผ่นตัวเลข บัตรลูกค้า หรือกริดปุ่มสามารถถูกวางไว้ภายในแท็บนั้น ตัวควบคุมแท็บเป็นคอนเทนเนอร์ที่ช่วยให้ผู้ใช้ที่เหมาะสมกับเนื้อหาเพิ่มเติมในหน้าจอ ตัวควบคุมแท็บจะพร้อมใช้งานสำหรับโครงร่างแบบเต็มเท่านั้น ** รูป ** -สามารถใช้ตัวควบคุมรูปภาพจะแสดงร้านค้าโลโก้หรือรูปตราสินค้าอื่นบนหน้าจอของธุรกรรมได้ ตัวควบคุมรูปภาพจะพร้อมใช้งานสำหรับโครงร่างแบบเต็มเท่านั้น **แนะนำผลิตภัณฑ์**- ถ้ามีการตั้งค่าคอนฟิกสำหรับสภาพแวดล้อม ผลิตภัณฑ์แนะนำตัวควบคุมจะแสดงคำแนะนำผลิตภัณฑ์ตามการเรียนรู้ของเครื่องจักร ผลิตภัณฑ์ที่แนะนำตัวควบคุมจะพร้อมใช้งานสำหรับโครงร่างเต็มใน Dynamics 365 สำหรับการดำเนินงานรุ่น 1611 เท่านั้น ** กำหนดเองควบคุม ** -ตัวควบคุมแบบกำหนดเองทำหน้าที่เป็นตัวยึดภายในโครงร่างหน้าจอผู้ใช้สามารถจองพื้นที่ว่างสำหรับเนื้อหาแบบกำหนดเอง ตัวควบคุมแบบกำหนดเองจะพร้อมใช้งานสำหรับโครงร่างแบบเต็มเท่านั้น
