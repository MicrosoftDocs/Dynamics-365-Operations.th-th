# <a name="personalized-product-recommendations-overview"></a>ภาพรวมของคำแนะนำผลิตภัณฑ์แบบส่วนบุคคล

ใน 365 Dynamics สำหรับการดำเนินงาน สามารถแสดงคำแนะนำผลิตภัณฑ์ on the point of อุปกรณ์การขาย (POS) คำแนะนำเป็นสินค้าที่ลูกค้าอาจสนใจโดยยึด บนของประวัติการซื้อ สินค้าในรายการของพวกเขาพร และสินค้าที่ลูกค้ารายอื่นซื้อแบบออนไลน์ และ ในร้านค้าลายอิฐ และปูน สำหรับผู้ค้าปลีกกับแค็ตตาล็อกที่มีขนาดใหญ่ คำแนะนำช่วยลูกค้า ด้วยการค้นหาผลิตภัณฑ์ โดย showcasing ผลิตภัณฑ์ที่มุ่งเป้ามายังให้ความสนใจและพฤติกรรมการซื้อของลูกค้า คำแนะนำผลิตภัณฑ์จะช่วยให้ผู้ค้าปลีกขายขึ้นและข้ามขาย และสามารถเพิ่มเงินวางประกันของลูกค้า ใน 365 Dynamics สำหรับการดำเนินงาน คำแนะนำผลิตภัณฑ์จะถูกเปิด โดยบริการการรู้คิดและเรียนรู้เครื่องจักร Microsoft Azure

<a name="scenarios"></a>สถานการณ์จำลอง
---------

คำแนะนำผลิตภัณฑ์ถูกเปิดใช้งานสำหรับ POS สถานการณ์ต่อไปนี้ จะพร้อมใช้งานใน Cloud POS หรือ POS สมัยใหม่ (MPOS)

1.  ในการ**รายละเอียดผลิตภัณฑ์**หน้า:

-   ถ้ามีการเยี่ยมชมที่เชื่อมโยงกับร้านค้า**รายละเอียดผลิตภัณฑ์**หน้าเมื่อดูที่ธุรกรรมก่อนหน้านี้ผ่านช่องทางต่าง ๆ โปรแกรมแนะนำแนะนำเพิ่มเติมสินค้าที่มีแนวโน้มที่จะซื้อเข้าด้วยกัน
-   ถ้าร้านค้าที่เชื่อมโยง เพิ่มลูกค้าลงในธุรกรรม และการเยี่ยมชมแล้ว เป็น**รายละเอียดผลิตภัณฑ์**หน้า โปรแกรมคำแนะนำให้คำแนะนำแบบส่วนบุคคลโดยใช้ประวัติธุรกรรมของลูกค้า

[![proddetails](./media/proddetails.png)](./media/proddetails.png)

1.  ในการ**ธุรกรรม**หน้า:

-   โปรแกรมแนะนำแนะนำสินค้าตามรายการทั้งหมดของสินค้าในตะกร้า
-   ถ้าร้านค้าที่เชื่อมโยง เพิ่มลูกค้าลงในธุรกรรม โปรแกรมแนะนำให้คำแนะนำส่วนบุคคลโดยใช้ประวัติธุรกรรมของลูกค้าและรายการสินค้าในตะกร้า

**หมายเหตุ**เพื่อแสดงคำแนะนำในการ**ธุรกรรม**หน้า ผู้ค้าปลีกที่ต้องปรับปรุงโครงร่างหน้าจอใน 365 Dynamics สำหรับการดำเนินงาน การ**คำแนะนำ**ควบคุมที่ต้องทิ้งบนไป**ธุรกรรม**หน้า [![transactionscreenmultipleproductslargemessengersbag-5](./media/transactionscreenmultipleproductslargemessengersbag-5.jpg)](./media/transactionscreenmultipleproductslargemessengersbag-5.jpg)

1.  ในการ**รายละเอียดลูกค้า**หน้า:
    -   โปรแกรมแนะนำแนะนำรายการโดยยึดตาม ID ผู้ใช้และรายการในชุดรายการพรของลูกค้า

[![customerdetailsrecommendations](./media/customerdetailsrecommendations.png)](./media/customerdetailsrecommendations.png)

## <a name="configure-dynamics-365-for-operations-to-enable-pos-recommendations"></a>การตั้งค่าคอนฟิก 365 Dynamics สำหรับการดำเนินงานเพื่อเปิดใช้งานคำแนะนำของ POS
เมื่อต้องการตั้งค่าคำแนะนำผลิตภัณฑ์ คุณจำเป็นต้องทำสิ่งต่อไปนี้

1.  ตรวจสอบให้แน่ใจว่า คุณได้เลือกที่ถูกต้อง**นิติบุคคล**
2.  นำทางไปยัง**เก็บเอนทิตี**เลือก**ขายปลีก**แล้ว คลิก**รีเฟรช**** ** นี้จะใช้ข้อมูลสาธิต (หรือข้อมูลของคุณ) จากฐานข้อมูลของคุณในการดำเนินงาน และย้ายไปเก็บเอนทิตี
3.  ตัวเลือก: เมื่อต้องการแสดงคำแนะนำบนหน้าจอธุรกรรม ไป ** โครงร่างหน้าจอ**เลือกโครงร่างหน้าจอของคุณ เปิดใช้**โปรแกรมออกแบบโครงร่างหน้าจอ**,** ** แล้ว วาง ** ควบคุมคำแนะนำ ** จำเป็น
4.  ไปที่**พารามิเตอร์การขายปลีก**เลือก**เครื่องจักรเลิร์**เลือก ** ใช่ ** ภายใต้**คำแนะนำเกี่ยวกับการเปิดใช้งาน POS**
5.  เมื่อต้องการดูคำแนะนำบน POS เรียกใช้งานการตั้งค่าคอนฟิกส่วนกลาง**1110** การเปลี่ยนแปลงที่เกิดขึ้นกับโปรแกรมออกแบบโครงร่างหน้าจอ POS เรียกใช้งานการตั้งค่าคอนฟิกช่องสัญญาณ**1070**

## <a name="how-does-it-work"></a>[]()วิธีทำงานหรือไม่
เมื่อคุณรีเฟรชแบบ**เก็บเอนทิตี**เอนทิตี การดำเนินการต่อไปนี้เกิดขึ้น

-   ข้อมูลในรูปแบบที่กำหนด โดยการบริการที่มีการรู้คิดจะแยกจาก 365 Dynamics สำหรับฐานข้อมูลในการดำเนินงานของการดำเนินงาน และส่งไปยังที่เก็บเอนทิตี
-   ข้อมูลถูกใช้โดยแฟกทอรี Azure ข้อมูล (ADF) เพื่อลบข้อมูลโดยใช้สคริปต์ในการโหลดไฮฟ์เป็นส่วนหนึ่งของกิจกรรม ADF Cleansed ข้อมูลถูกเก็บไว้ในที่เก็บ blob
-   ข้อมูลจากที่เก็บ blob ถูกใช้ โดยบริการการรู้คิด API เพื่อฝึกให้คำแนะนำเกี่ยวกับรูปแบบ

เมื่อคุณเปิดใช้**คำแนะนำเกี่ยวกับการเปิดใช้งาน**และเรียกใช้งานการตั้งค่าคอนฟิก การดำเนินการต่อไปนี้เกิดขึ้น

-   ข้อมูลประจำตัวแบบจำลองและ ID จะยกหูจาก API และเก็บไว้ ใน 365 Dynamics สำหรับฐานข้อมูลในการดำเนินงานของการดำเนินการ ใน web.config สำหรับ AOS และนอกจากนี้ ในเซิร์ฟเวอร์การขายปลีก
-   ข้อมูลประจำตัวแบบจำลองและ ID จะทำพร้อมใช้งานสำหรับจอภาพ CRT เพื่อให้มีการเรียกคำแนะนำผลิตภัณฑ์จาก Cloud POS และ MPOS ในโหมดออนไลน์สามารถจะจ่ายเงิน
