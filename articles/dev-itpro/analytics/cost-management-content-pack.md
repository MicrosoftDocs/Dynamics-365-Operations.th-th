# <a name="cost-management-power-bi-content"></a>เนื้อหาการจัดการต้นทุนใน Power BI

[!include [banner](../includes/banner.md)]

## <a name="overview"></a>ภาพรวม

เนื้อหา Microsoft Power BI ของ **การจัดการต้นทุน** มีไว้สำหรับผู้จัดทำบัญชีสินค้าคงคลังหรือบุคคลในองค์กรที่รับผิดชอบ หรือสนใจสถานะของสินค้าคงคลังหรืองานระหว่างทำ (WIP) หรือผู้ที่รับผิดชอบ หรือสนใจในการวิเคราะห์ผลต่างต้นทุนมาตรฐาน

> [!Note]
> เนื้อหา Power BI ของ **การจัดการต้นทุน** ที่อธิบายไว้ในหัวข้อนี้ จะนำไปใช้กับ Dynamics 365 for Finance and Operations 8.0
> 
> ชุดเนื้อหา Power BI ของ **การจัดการต้นทุน** ที่เผยแพร่ไปยังไซต์ PowerBI.com ไม่ได้รับการสนับสนุน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับค่าเสื่อมราคา ดู [ชุดเนื้อหา Power BI ที่เผยแพร่ไปยัง PowerBI.com](../migration-upgrade/deprecated-features.md#power-bi-content-packs-published-to-powerbicom)


เนื้อหา Power BI นี้ให้รูปแบบที่จัดประเภทที่ช่วยคุณในการตรวจสอบประสิทธิภาพการทำงานของสินค้าคงคลัง และแสดงให้เห็นวิธีการที่ต้นทุนหมุนเวียนผ่านรายการเหล่านั้น คุณสามารถได้รับข้อมูลเชิงลึกเชิงการจัดการ เช่น อัตราหมุนเวียน จำนวนวันที่สินค้าคงคลังคงเหลือ ความถูกต้อง และ "การจัดประเภท ABC" ที่ระดับแบบรวมที่ต้องการของคุณ (บริษัท สินค้า กลุ่มสินค้า หรือไซต์) นอกจากนี้ ข้อมูลที่ถูกทำให้พร้อมใช้งานยังสามารถใช้เป็นภาคผนวกอย่างละเอียดสำหรับงบการเงินได้ด้วย

เนื้อหา Power BI ขึ้นอยู่กับการวัดแบบรวม **CostObjectStatementCacheMonthly** ซึ่งมีตาราง **CostObjectStatementCache** เป็นแหล่งข้อมูลหลัก ตารางนี้จะถูกจัดการโดยกรอบงานแคชชุดข้อมูล โดยค่าเริ่มต้น ตารางจะได้รับการอัพเดตทุกๆ 24 ชั่วโมง แต่คุณสามารถเปลี่ยนความถี่ของอัพเดตหรือเปิดใช้งานการอัพเดตด้วยตนเองได้ในการตั้งค่าคอนฟิกของแคชชุดข้อมูล สามารถรันการปรับปรุงด้วยตนเองในพื้นที่ทำงาน **การจัดการต้นทุน** หรือพื้นที่ทำงาน **การวิเคราะห์ต้นทุน** ได้

หลังจากการอัพเดตทุกครั้งของตาราง **CostObjectStatementCache** การวัดแบบรวม **CostObjectStatementCacheMonthly** ต้องได้รับการอัพเดต ก่อนที่จะมีการอัพเดตข้อมูลในภาพประกอบ Power BI

## <a name="accessing-the-power-bi-content"></a>การเข้าถึงเนื้อหา Power BI

เนื้อหา Power BI **การจัดการต้นทุน** จะปรากฏในพื้นที่ทำงาน **การจัดการต้นทุน** และ **การวิเคราะห์ต้นทุน**

พื้นที่ทำงาน **การจัดการต้นทุน** ประกอบด้วยแท็บต่อไปนี้:

- **ภาพรวม** – แท็บนี้แสดงข้อมูลแอพลิเคชัน
- **สถานะการบัญชีสินค้าคงคลัง** – แท็บนี้แสดงเนื้อหา Power BI
- **สถานะการบัญชีการผลิต** – แท็บนี้แสดงเนื้อหา Power BI

พื้นที่ทำงาน **การวิเคราะห์ต้นทุน** ประกอบด้วยแท็บต่อไปนี้:

- **ภาพรวม** – แท็บนี้แสดงข้อมูลแอพลิเคชัน
- **การวิเคราะห์การบัญชีสินค้าคงคลัง** – แท็บนี้แสดงเนื้อหา Power BI
- **การวิเคราะห์การบัญชีการผลิต** – แท็บนี้แสดงเนื้อหา Power BI
- **การวิเคราะห์ผลต่างต้นทุนมาตรฐาน** – แท็บนี้แสดงเนื้อหา Power BI

### <a name="understanding-the-data-model-and-entities"></a>การทำความเข้าใจเกี่ยวกับแบบจำลองข้อมูลและเอนทิตี้

ข้อมูลจาก Microsoft Dynamics 365 for Finance and Operations ถูกใช้ในการกรอกข้อมูลหน้ารายงานในเนื้อหา Power BI ของ **การจัดการต้นทุน** ข้อมูลนี้จะถูกแสดงเป็นการวัดแบบรวมที่มีการแบ่งระยะในร้านค้าเอนทิตี ซึ่งเป็นฐานข้อมูล Microsoft SQL Server ที่ปรับให้เหมาะสมสำหรับการวิเคราะห์ สำหรับข้อมูลเพิ่มเติม ดู [การรวม Power BI กับร้านค้าเอนทิตี](power-bi-integration-entity-store.md)

การวัดแบบรวมหลักของออบเจ็กต์ต่อไปนี้ ถูกใช้เป็นข้อมูลพื้นฐานของเนื้อหา Power BI

| ออบเจ็กต์                          | การวัดแบบรวมหลัก | แหล่งข้อมูลสำหรับ Finance and Operations | ฟิลด์               |
|---------------------------------|----------------------------|----------------------------------------|---------------------|
| CostObjectStatementCacheMonthly | จำนวน                     | CostObjectStatementCache               | จำนวน              |
| CostObjectStatementCacheMonthly | ปริมาณ                   | CostObjectStatementCache               | ปริมาณ                 |
| CostInventoryAccountingKPIGoal  | AnnualInventoryTurn        | CostInventoryAccountingKPIGoal         | AnnualInventoryTurn |
| CostInventoryAccountingKPIGoal  | InventoryAccuracy          | CostInventoryAccountingKPIGoal         | InventoryAccuracy   |

ตารางต่อไปนี้แสดงการวัดที่คำนวณได้หลักในเนื้อหา Power BI

| การประเมิน                            | การคำนวณ |
|------------------------------------|-------------|
| ยอดดุลต้นงวด                  | ยอดดุลต้นงวด = [ยอดดุลสิ้นงวด] - [การเปลี่ยนแปลงสุทธิ] |
| ปริมาณยอดดุลต้นงวด             | ปริมาณยอดดุลต้นงวด = [ปริมาณยอดดุลสิ้นงวด]-[ปริมาณการเปลี่ยนแปลงสุทธิ] |
| ยอดดุลสิ้นงวด                     | ยอดดุลสิ้นงวด = (CALCULATE(SUM([Amount]), FILTER(ALL(FiscalCalendar), FiscalCalendar [MONTHSTARTDATE] \<= MAX(FiscalCalendar[MONTHSTARTDATE])))) |
| ปริมาณยอดดุลสิ้นงวด                | ปริมาณยอดดุลสิ้นงวด = CALCULATE(SUM([QTY]), FILTER(ALL(FiscalCalendar),FiscalCalendar[MONTHSTARTDATE] \<= MAX(FiscalCalendar[MONTHSTARTDATE]))) |
| การเปลี่ยนแปลงสุทธิ                         | การเปลี่ยนแปลงสุทธิ = SUM([AMOUNT]) |
| ปริมาณการเปลี่ยนแปลงสุทธิ                    | ปริมาณการเปลี่ยนแปลงสุทธิ = SUM([QTY]) |
| อัตราการหมุนเวียนของสินค้าคงคลังตามจำนวน | อัตราการหมุนเวียนของสินค้าคงคลังตามจำนวน = ifif(OR([ยอดดุลเฉลี่ยของสินค้าคงคลัง] \<= 0, [ปัญหาของสินค้าคงคลังที่ขายได้หรือที่ใช้] \>= 0), 0, ABS([Iปัญหาของสินค้าคงคลังที่ขายได้หรือที่ใช้])/[ยอดดุลเฉลี่ยของสินค้าคงคลัง]) |
| ยอดดุลเฉลี่ยของสินค้าคงคลัง          | ยอดดุลค่าเฉลี่ยของสินค้าคงคลัง = (([ยอดดุลสิ้นงวด] + [ยอดดุลต้นงวด]) / 2) |
| จำนวนวันที่มีสินค้าคงคลังคงเหลือ             | จำนวนวันที่มีสินค้าคงคลังคงเหลือ = 365 / CostObjectStatementEntries[อัตราการหมุนเวียนของสินค้าคงคลังตามจำนวน] |
| ความถูกต้องของสินค้าคงคลัง                 | ความถูกต้องของสินค้าคงคลังตามจำนวน = IF([ยอดดุลสิ้นงวด] \<= 0, IF(OR([ยอดเงินที่ตรวจนับของสินค้าคงคลัง] \<\> 0, [ยอดดุลสิ้นงวด] \< 0), 0, 1), MAX(0, ([ยอดดุลสิ้นงวด] - ABS([ยอดเงินที่ตรวจนับของสินค้าคงคลัง))/[ยอดดุลสิ้นงวดของสินค้าคงคลัง])) |

มิติหลักต่อไปนี้ถูกใช้เป็นตัวกรองเพื่อแบ่งส่วนการวัดแบบรวม เพื่อให้คุณสามารถได้รับส่วนประกอบที่มากขึ้นและข้อมูลเชิงลึกเชิงวิเคราะห์ที่ละเอียดมากขึ้น
