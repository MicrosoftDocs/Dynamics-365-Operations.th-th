# <a name="project-service-automation-integration-parameters"></a>พารามิเตอร์การรวม Project Service Automation

ในหน้า **พารามิเตอร์การรวม Project Service Automation** คุณสามารถตั้งค่าคอนฟิกว่าข้อมูลควรมีค่าเริ่มต้นอย่างไร เมื่อมีการรวม Project Service Automation กับ Dynamics 365 for Finance and Operations รายการต่อไปนี้ต้องถูกตั้งค่าสำหรับโครงการที่จะถูกซิงโครไนส์โดยสำเร็จจาก Project Service Automation ใน Finance and Operations

> [!NOTE]
> การรวมงานโครงการ ประเภทธุรกรรมค่าใช้จ่าย การประเมินชั่วโมง การประเมินค่าใช้จ่าย และการล็อคฟังก์ชัน พร้อมใช้งานใน Dynamics 365 for Finance and Operations รุ่น 8.0




| **แท็บ**                      | **ฟิลด์**                          | **คำอธิบาย**                    |
|------------------------------|------------------------------------|--------------------------------|
| **ทั่วไป**                  | **ชนิดโครงการเริ่มต้น**               | เลือกค่าเริ่มต้นสำหรับ **ชนิดโครงการ** ซึ่งคือ เมื่อโครงการถูกซิงโครไนส์จาก Project Service Automation ถ้าคุณไม่ได้ระบุค่าเริ่มต้นไว้ในเท็มเพลตการรวม ในระหว่างการซิงโครไนส์ โครงการใหม่จะตั้งค่า **ชนิดโครงการ** เป็นค่านี้ และอาจถูกปรับปรุง เมื่อมีการซิงโครไนส์รายการสัญญาของโครงการจาก Project Service Automation               |
| **ทั่วไป**                  | **ประเภทเวลา**                      | เลือกค่าเริ่มต้นสำหรับ **ประเภทเวลา** ซึ่งคือ เมื่อการประเมินชั่วโมงถูกซิงโครไนส์จาก Project Service Automation ในระหว่างการซิงโครไนส์ การคาดการณ์ชั่วโมงโครงการใหม่ใน Finance and Operations จะตั้งค่า **ประเภท** เป็นค่านี้ เมื่อมีการซิงโครไนส์การประเมินชั่วโมงและชั่วโมงจริงจาก Project Service Automation   |
| **ทั่วไป**                  | **ประเภทค่าธรรมเนียม**                       | เลือกค่าเริ่มต้นสำหรับ **ประเภทค่าธรรมเนียม** ซึ่งคือ เมื่อค่าธรรมเนียมจริงถูกซิงโครไนส์จาก Project Service Automation ในระหว่างการซิงโครไนส์ ธุรกรรมค่าธรรมเนียมใหม่ใน Finance and Operations จะตั้งค่า **ประเภท** เป็นค่านี้ เมื่อมีการซิงโครไนส์ค่าธรรมเนียมจริงจาก Project Service Automation          |
| **ค่าเริ่มต้นกลุ่มโครงการ**   | **ชนิดโครงการ** | คลิก **สร้าง** เพื่อเพิ่มแถวที่คุณสามารถเลือกชนิดโครงการที่คุณต้องการตั้งค่ากลุ่มโครงการเริ่มต้นให้ สามารถเลือกชนิดโครงการที่เฉพาะเจาะจงได้เพียงครั้งเดียวในการตั้งค่าคอนฟิก              
|                              | **กลุ่มโครงการ**          | เลือกกลุ่มโครงการเริ่มต้นสำหรับชนิดโครงการที่ระบุ เมื่อมีการซิงค์โครงการใหม่จาก Project Service Automation **กลุ่มโครงการ** จะเป็นค่าเริ่มต้นโดยยึดตามชนิดโครงการ ถ้าคุณไม่ได้ระบุค่าเริ่มต้นไว้ในเท็มเพลตการรวม  |
| **ค่าเริ่มต้นชนิดการเรียกเก็บเงิน**    | **ชนิดการเรียกเก็บเงิน** | คลิก **สร้าง** เพื่อเพิ่มแถวที่คุณสามารถเลือกชนิดการเรียกเก็บเงินที่คุณต้องการตั้งค่าลักษณะของรายการเริ่มต้นให้ สามารถเลือกชนิดการเรียกเก็บเงินที่เฉพาะเจาะจงได้เพียงครั้งเดียวในการตั้งค่าคอนฟิก          |
|                              | **ลักษณะของรายการ**| เลือกลักษณะของรายการเริ่มต้นสำหรับชนิดการเรียกเก็บเงินที่ระบุ เมื่อการประเมินชั่วโมงใหม่ การประเมินค่าใช้จ่ายใหม่ หรือยอดจริง ถูกซิงค์จาก Project Service Automation **ลักษณะของรายการ** จะเป็นค่าเริ่มต้นโดยขึ้นอยู่กับชนิดการเรียกเก็บเงิน          |
| **การล็อกฟังก์ชันการทำงาน**    |                   | เลือกฟังก์ชันเพื่อปิดใช้งานใน Finance and Operations สำหรับโครงการและสัญญาที่สร้างขึ้นจาก Project Service Automation ตัวอย่างเช่น คุณสามารถเลือกที่จะปิดใช้งานความสามารถในการแก้ไขสัญญาและโครงการ สร้างโครงสร้างการแบ่งงาน และป้อนแผ่นเวลาใน Finance and Operations ฟิลด์ที่เกี่ยวข้องกับการลงบัญชีจะยังคงเปิดใช้งานต่อไป แม้ว่าถูกทำให้ไม่พร้อมใช้งานโดยการตั้งค่าพารามิเตอร์ โดยค่าเริ่มต้น ฟังก์ชันทั้งหมดจะถูกเปิดใช้งาน           |
