# <a name="work-breakdown-structures"></a><span data-ttu-id="47235-104">โครงสร้างการแบ่งงาน</span><span class="sxs-lookup"><span data-stu-id="47235-104">Work breakdown structures</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="47235-105">โครงสร้างการแบ่งโครงสร้างการแบ่งงาน (WBS) คือ คำอธิบายของงานที่สามารถทำได้สำหรับโครงการได้</span><span class="sxs-lookup"><span data-stu-id="47235-105">A work breakdown structure (WBS) is a description of the work that will be done for a project.</span></span> <span data-ttu-id="47235-106">ลำดับชั้นของงานที่แสดงถึงความเข้าใจของทีมโครงการของส่วนประกอบ ของงาน และขนาด ต้นทุน และระยะเวลาของแต่ละส่วนประกอบหรืองาน ได้</span><span class="sxs-lookup"><span data-stu-id="47235-106">It’s a hierarchy of tasks that represents the project team’s understanding of the composition of work, and of the size, cost, and duration of each component or task.</span></span> <span data-ttu-id="47235-107">WBS มีวัตถุประสงค์หลักสามข้อ:</span><span class="sxs-lookup"><span data-stu-id="47235-107">A WBS has three major purposes:</span></span>

-   <span data-ttu-id="47235-108">อธิบายถึงการแบ่งงานหรือส่วนประกอบของงานในงาน</span><span class="sxs-lookup"><span data-stu-id="47235-108">Describe the breakdown or composition of work in tasks.</span></span>

-   <span data-ttu-id="47235-110">ประเมินต้นทุนของแต่ละงาน</span><span class="sxs-lookup"><span data-stu-id="47235-110">Estimate the cost of each task.</span></span>

<span data-ttu-id="47235-111">ระดับของรายละเอียดใน WBS ขึ้นอยู่กับระดับของความถูกต้องที่จำเป็นในการประเมิน และระดับของการติดตามที่ต้องใช้กับการประเมินดังกล่าว</span><span class="sxs-lookup"><span data-stu-id="47235-111">The degree of detail in a WBS depends on the level of accuracy that is required in estimates and the level of tracking that is required against those estimates.</span></span> <span data-ttu-id="47235-112">โครงการที่มีการยอมรับต่ำมากสำหรับ slippages ในกำหนดการหรือต้นทุน มักจะขอ WBS ที่ละเอียดมากขึ้น และต้องติดตามควบคุมงานระหว่างทำและต้นทุนเทียบกับ WBS</span><span class="sxs-lookup"><span data-stu-id="47235-112">Projects that have very low tolerance for slippages in schedule or cost usually require a more detailed WBS, and diligent tracking of work progress and cost against the WBS.</span></span> <span data-ttu-id="47235-113">โครงการชนิดนี้จะใช้กันทั่วไปในการก่อสร้างและอุตสาหกรรมวิศวกรรม</span><span class="sxs-lookup"><span data-stu-id="47235-113">This kind of project is common in the construction and engineering industries.</span></span> 

<span data-ttu-id="47235-114">ในทางกลับกัน โครงการในอุตสาหกรรมเช่นสื่อโฆษณา ซอฟต์แวร์ และโครงสร้างพื้นฐาน IT มีแนวโน้มจะเป็นที่หนึ่ง และประสิทธิผลจะสัมพันธ์กับประสบการณ์และความสามารถของบุคคลที่ทำงาน</span><span class="sxs-lookup"><span data-stu-id="47235-114">By contrast, projects in industries such as media and advertising, software, and IT infrastructure tend to be one of a kind, and productivity is relative to the experience and competency of the individual who is performing the task.</span></span> <span data-ttu-id="47235-115">ดังนั้น อุตสาหกรรมเหล่านี้ใช้ WBS เพื่อรับการประมาณขนาดของโครงการ ไม่ใช่เพื่อติดตามความคืบหน้าของโครงการนั้นโดยละเอียด</span><span class="sxs-lookup"><span data-stu-id="47235-115">Therefore, these industries use a WBS to get an approximation of the size of a project, not to track the progress of that project in detail.</span></span> 

<span data-ttu-id="47235-116">การสร้าง WBS เป็นกระบวนการที่ดำเนินการมากซึ่งโดยปกติแล้วใช้ระยะเวลานาน และจำเป็นต้องทำงานร่วมกันและขอข้อมูลจากบุคคลที่หลากหลาย</span><span class="sxs-lookup"><span data-stu-id="47235-116">Building a WBS is an intensive process that is usually done over a long period, and that requires collaboration and information from a wide variety of people.</span></span> <span data-ttu-id="47235-117">หัวข้อนี้อธิบายวิธีการใช้ว่าคุณสามารถใช้การปรับปรุง WBS ใน Dynamics 365 for Finance and Operations ได้ตรงกับความต้องการสำหรับการประเมินและการติดตามได้อย่างไร</span><span class="sxs-lookup"><span data-stu-id="47235-117">This topic describes how you can use WBS enhancements in Microsoft Dynamics 365 for Finance and Operations to meet your requirements for estimates and tracking.</span></span>

### <a name="how-to-use-the-concepts-of-planned-value-earned-value-and-actual-cost"></a><span data-ttu-id="47235-325">วิธีใช้แนวคิดของมูลค่าที่วางแผน มูลค่าที่ได้มา และต้นทุนจริง</span><span class="sxs-lookup"><span data-stu-id="47235-325">How to use the concepts of planned value, earned value, and actual cost</span></span>

<span data-ttu-id="47235-326">**ผลต่างกำหนดการ** ระหว่างการวางแผน คุณสามารถสร้างการคาดการณ์การทำงานตามเส้นเวลา</span><span class="sxs-lookup"><span data-stu-id="47235-326">**Schedule variance** During planning, you create a forecast for work on a timeline.</span></span> <span data-ttu-id="47235-327">ดังนั้น มูลค่าที่วางแผนคือระยะที่ผู้วางแผนโครงการเชื่อว่างานโครงการจะเสร็จสมบูรณ์</span><span class="sxs-lookup"><span data-stu-id="47235-327">Therefore, planned value is the rate at which project planners thought work would be completed on the project.</span></span> <span data-ttu-id="47235-328">หลังจากโครงการอยู่ในระยะดำเนินการ งานก็จะเสร็จสมบูรณ์ แล้วโครงการก็จะมีมูลค่า</span><span class="sxs-lookup"><span data-stu-id="47235-328">After a project is in progress, work is completed, and the project earns value.</span></span> <span data-ttu-id="47235-329">เมื่อเปรียบเทียบระหว่างมูลค่าที่วางแผนและมูลค่าที่ได้มา คุณสามารถดูระยะการดำเนินการของงาน</span><span class="sxs-lookup"><span data-stu-id="47235-329">By comparing planned value to earned value, you can view how work on a project is progressing.</span></span> <span data-ttu-id="47235-330">ผลของการเปรียบเทียบนี้คือผลต่างกำหนดการ</span><span class="sxs-lookup"><span data-stu-id="47235-330">The result of this comparison is called schedule variance.</span></span>

<span data-ttu-id="47235-337">**ผลต่างต้นทุน** เนื่องมูลค่ที่ได้มาใช้ราคาต้นทุนเป็นตัวคูณที่ มูลค่าที่ได้มาบ่งชี้ว่าต้นทุนของงานที่ทำเสร็จแล้วในโครงการ</span><span class="sxs-lookup"><span data-stu-id="47235-337">**Cost variance** Because earned value uses the cost price as the multiplier, earned value indicates the cost of the work that is done on a project.</span></span> <span data-ttu-id="47235-338">ขณะโครงการกำลังดำเนินการอ ล็อกธุรกรรมมีเรกคอร์ดของเงินที่ถูกใช้จริงเกี่ยวกับโครงการนั้น</span><span class="sxs-lookup"><span data-stu-id="47235-338">As a project progresses, the transaction log provides a record of money that is actually spent on that project.</span></span> <span data-ttu-id="47235-339">โดยเปรียบเทียบมูลค่าที่ได้มากับต้นทุนจริง คุณสามารถดูจำนวนเงินที่กำลังใช้เทียบกับค่าที่จะได้รับ</span><span class="sxs-lookup"><span data-stu-id="47235-339">By comparing earned value to actual cost, you can view the amount of money that is being spent versus the value that is earned.</span></span> <span data-ttu-id="47235-340">ผลลัพธ์ของการเปรียบเทียบนี้เรียกว่าผลต่างต้นทุน</span><span class="sxs-lookup"><span data-stu-id="47235-340">The result of this comparison is called cost variance.</span></span> 

<span data-ttu-id="47235-341">ถ้าต้นทุนจริงที่ใช้สำหรับรอบระยะเวลามีค่ามากกว่าค่าได้รับ เงินก็จะถูกใช้มากกว่าที่จะได้รับ</span><span class="sxs-lookup"><span data-stu-id="47235-341">If the actual cost that is spent for a period is more than the earned value, more money was spent than earned.</span></span> <span data-ttu-id="47235-342">ดังนั้น โครงการจะเกินงบประมาณ</span><span class="sxs-lookup"><span data-stu-id="47235-342">Therefore, the project is over budget.</span></span>
