[!include[banner](../includes/banner.md)]


<span data-ttu-id="80dbb-105">วิธีและเวลาที่มีการชำระธุรกรรมอาจเป็นเรื่องซับซ้อน ดังนั้นคุณจำเป็นต้องเข้าใจ และกำหนดพารามิเตอร์เพื่อตอบสนองความต้องการทางธุรกิจของคุณอย่างถูกต้อง</span><span class="sxs-lookup"><span data-stu-id="80dbb-105">How and when transactions are settled can be complex subjects, so it's essential that you understand and correctly define the parameters to meet your business requirements.</span></span> <span data-ttu-id="80dbb-106">บทความนี้อธิบายพารามิเตอร์ที่ใช้สำหรับการชำระบัญชีสำหรับทั้งบัญชีเจ้าหนี้และบัญชีลูกหนี้</span><span class="sxs-lookup"><span data-stu-id="80dbb-106">This article describes the parameters that are used for settlement for both Accounts payable and Accounts receivable.</span></span>

-   <span data-ttu-id="80dbb-110">**การชำระเงินอัตโนมัติ**– ตั้งค่าตัวเลือกนี้ไปยัง **ใช่** ธุรกรรมหนึ่งควรจะจับคู่โดยอัตโนมัติกับธุรกรรมเปิดอื่นๆ เมื่อมีการลงรายการบัญชี</span><span class="sxs-lookup"><span data-stu-id="80dbb-110">**Automatic settlement** – Set this option to **Yes** if a transaction should be settled automatically against other open transactions when it is posted.</span></span> <span data-ttu-id="80dbb-111">ถ้าตัวเลือกนี้ถูกกำหนดเป็น **ไม่ใช่**ผู้ใช้สามารถชำระธุรกรรมด้วยตนเอง เมื่อพวกเขาป้อนการชำระเงิน หรือภายหลัง โดยการใช้หน้า **ธุรกรรมการชำระเงิน** ได้</span><span class="sxs-lookup"><span data-stu-id="80dbb-111">If this option is set to **No**, users can manually settle transactions when they enter payments, or later, by using the **Settle transactions** page.</span></span>
-   <span data-ttu-id="80dbb-112">**การจัดการส่วนลดเงินสด** – ระบุวิธีการ [จัดการส่วนลดเงินสดเมื่อมีการชำระใบแจ้งหนี้เกิน](cash-discount-handling-overpayments.md).</span><span class="sxs-lookup"><span data-stu-id="80dbb-112">**Cash discount administration** – Specify how a [cash discount is handled when an invoice is overpaid](cash-discount-handling-overpayments.md).</span></span> <span data-ttu-id="80dbb-113">สำหรับการชำระมากเกิน ก็สามารถหักส่วนลดเงินสด สามารถถือเป็นผลต่าง หรือคงอยู่ในบัญชีสำหรับผู้จัดจำหน่ายหรือลูกค้า</span><span class="sxs-lookup"><span data-stu-id="80dbb-113">For an overpayment, the cash discount can be reduced, it can be treated as a difference, or it can remain on account for the vendor or customer.</span></span>
