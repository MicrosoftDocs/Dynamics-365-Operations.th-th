## <a name="determine-which-invoice-matching-validation-to-use"></a><span data-ttu-id="bb856-110">กำหนดการตรวจสอบความถูกต้องของการจับคู่ใบแจ้งหนี้ที่จะใช้</span><span class="sxs-lookup"><span data-stu-id="bb856-110">Determine which invoice matching validation to use</span></span>
<span data-ttu-id="bb856-111">มีชนิดการตรวจสอบความถูกต้องของการจับคู่ที่แตกต่างกันสี่ชนิด</span><span class="sxs-lookup"><span data-stu-id="bb856-111">Four different types of matching validation are available.</span></span> 

- <span data-ttu-id="bb856-112">**การจับคู่ระดับรายการ** – ชนิดของการจับคู่ที่พบมากที่สุดคือ การจับคู่รายการ</span><span class="sxs-lookup"><span data-stu-id="bb856-112">**Line level matching** – The most common type of matching is line matching.</span></span> <span data-ttu-id="bb856-113">การจับคู่ระดับรายการสามารถเป็นการจับคู่สองทาง หรือสามทาง</span><span class="sxs-lookup"><span data-stu-id="bb856-113">Line level matching can be two-way or three-way matching.</span></span> <span data-ttu-id="bb856-114">สามารถระบุการจับคู่ระดับรายการเริ่มต้นสำหรับนิติบุคคลบนหน้า **พารามิเตอร์บัญชีเจ้าหนี้**</span><span class="sxs-lookup"><span data-stu-id="bb856-114">The default line level matching can be specified for a legal entity on the **Accounts payable parameter** page.</span></span> <span data-ttu-id="bb856-115">การจับคู่สองทางเปรียบเทียบราคาต่อหน่วยของใบแจ้งหนี้กับราคาต่อหน่วยของใบสั่งซื้อ</span><span class="sxs-lookup"><span data-stu-id="bb856-115">Two-way matching compares the unit price of the invoice to the unit price of the purchase order.</span></span> <span data-ttu-id="bb856-116">การจับคู่สามทางยังเปรียบเทียบปริมาณในใบแจ้งหนี้กับปริมาณการรับผลิตภัณฑ์ที่จับคู่แล้วอีกด้วย</span><span class="sxs-lookup"><span data-stu-id="bb856-116">Three-way matching additionally compares the invoice quantity to the matched product receipt quantity.</span></span>
- <span data-ttu-id="bb856-117">**การจับคู่ผลรวมในใบแจ้งหนี้** – จับคู่ยอดเงินรวมในใบแจ้งหนี้กับยอดรวมในใบสั่งซื้อ</span><span class="sxs-lookup"><span data-stu-id="bb856-117">**Invoice totals matching** – Match the total amounts on the invoice to the total amounts on the purchase order.</span></span> <span data-ttu-id="bb856-118">การจับคู่ใบแจ้งหนี้ชนิดนี้รวมถึงรายละเอียดจำนวนที่น้อยที่สุด ดังนั้นคุณสามารถใช้ตัวเลือกนี้เพื่อตั้งค่าตัวควบคุมที่ลดเวลาพนักงานที่จำเป็นเพื่อตรวจทานข้อมูลการจับคู่ใบแจ้งหนี้</span><span class="sxs-lookup"><span data-stu-id="bb856-118">This type of invoice matching includes the least amount of detail, so you can use this option to set up controls that minimize the staff time that is required to review invoice matching information.</span></span> <span data-ttu-id="bb856-119">มีการเปรียบเทียบยอดรวมหกรายการ ซึ่งรวมถึงผลรวมย่อย ส่วนลดรวม ค่าธรรมเนียม ภาษีขาย การปัดเศษ และยอดเงินในใบแจ้งหนี้</span><span class="sxs-lookup"><span data-stu-id="bb856-119">Six totals are compared, including Subtotal, Total discount, Charges, Sales taxes, Round off, and Invoice amount.</span></span> <span data-ttu-id="bb856-120">ระบบจะตรวจสอบความถูกต้องของค่าใดๆ เหล่านี้ในใบแจ้งหนี้แตกต่างจากยอดเงินที่คาดไว้มากกว่าผลต่างที่ยอมรับได้</span><span class="sxs-lookup"><span data-stu-id="bb856-120">The system will validate if any of these values on the invoice deviate from expected amounts by more than an acceptable variance.</span></span>
- <span data-ttu-id="bb856-121">**การจับคู่ค่าธรรมเนียม** – จับคู่ข้อมูลค่าธรรมเนียม (ยอดเงิน) ในใบแจ้งหนี้กับข้อมูลค่าธรรมเนียม (ยอดเงิน) ในใบสั่งซื้อ</span><span class="sxs-lookup"><span data-stu-id="bb856-121">**Charges matching** – Match the charges information (amounts) on the invoice to the charges information (amounts) on the purchase order</span></span>
- <span data-ttu-id="bb856-122">**ผลรวมราคาสำหรับการจับคู่สินค้าในรายการ** – ชนิดของการจับคู่นี้มีประโยชน์สำหรับบริษัทที่โดยทั่วไปจะได้รับใบแจ้งหนี้หลายใบสำหรับรายการใบสั่งซื้อเดียว</span><span class="sxs-lookup"><span data-stu-id="bb856-122">**Price totals for line item matching** – This type of matching is useful for companies that typically receive multiple invoices for a single purchase order line.</span></span> <span data-ttu-id="bb856-123">โดยทั่วไป ถ้าคุณได้รับใบแจ้งหนี้เดียวสำหรับแต่ละรายการใบสั่งซื้อ จะไม่จำเป็นต้องมีการจับคู่ชนิดนี้</span><span class="sxs-lookup"><span data-stu-id="bb856-123">If you typically receive only one invoice per purchase order line, this type of matching is not necessary.</span></span> <span data-ttu-id="bb856-124">การจับคู่นี้จำเป็นต้องมีการเปิดใช้งานการจับคู่สองทางหรือสามทาง และทำหน้าที่เป็นการตรวจสอบความถูกต้องของยอดเงินสุทธิที่ไม่เกิน ตามเปอร์เซ็นต์การยอมรับและจำนวนเงิน</span><span class="sxs-lookup"><span data-stu-id="bb856-124">This matching requires that two-way or three-way matching is enabled and serves as a not-to-exceed net amount validation, based on a tolerance percentages and amounts.</span></span>  <span data-ttu-id="bb856-125">ชนิดของการจับคู่นี้เปรียบเทียบข้อมูลราคาสำหรับยอดเงินสุทธิของแต่ละรายการในใบแจ้งหนี้ และรายการใบแจ้งหนี้ที่ค้างอยู่ และที่ลงรายการบัญชีไว้ก่อนหน้านี้ทั้งหมด ด้วยยอดเงินสุทธิของรายการใบสั่งซื้อที่สอดคล้องกัน</span><span class="sxs-lookup"><span data-stu-id="bb856-125">This type of matching compares price information for the net amount of each line on the invoice, and all pending and previously posted invoice lines, with the net amount of the corresponding purchase order line.</span></span> <span data-ttu-id="bb856-126">ยอดเงินสุทธิถูกกำหนดโดยสูตรต่อไปนี้: (ราคาต่อหน่วย \* ปริมาณในรายการ) + ค่าธรรมเนียมรายการ - ส่วนลดต่อรายการสินค้า</span><span class="sxs-lookup"><span data-stu-id="bb856-126">The net amount is determined by the following formula: (Unit price \* Line quantity) + Line charges - Line discounts.</span></span> <span data-ttu-id="bb856-127">เมื่อราคารวมที่ตรงกันตามเปอร์เซนต์ ระบบจะเปรียบเทียบค่าโดยใช้สกุลเงินธุรกรรม</span><span class="sxs-lookup"><span data-stu-id="bb856-127">When matching price totals by percentage, the system compares values using the transaction currency.</span></span> <span data-ttu-id="bb856-128">เมื่อราคารวมที่ตรงกันตามปริมาณ ระบบจะเปรียบเทียบค่าโดยใช้สกุลเงินทางบัญชี</span><span class="sxs-lookup"><span data-stu-id="bb856-128">When matching price totals by amount, the system compares the values using the accounting currency.</span></span>

3. <span data-ttu-id="bb856-132">เลือกหรือยกเลิกการเลือกกล่องกาเครื่องหมาย **เปิดใช้งานการตรวจสอบการจับคู่ใบแจ้งหนี้**</span><span class="sxs-lookup"><span data-stu-id="bb856-132">Select or clear the **Enable invoice matching validation** check box.</span></span>
