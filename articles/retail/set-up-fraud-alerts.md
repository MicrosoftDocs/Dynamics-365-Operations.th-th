# <a name="set-up-fraud-alerts"></a><span data-ttu-id="676f9-104">ตั้งค่าการแจ้งเตือนการตรวจสอบการฉ้อโกง</span><span class="sxs-lookup"><span data-stu-id="676f9-104">Set up fraud alerts</span></span>

[!include[banner](includes/banner.md)]

<span data-ttu-id="676f9-105">หัวข้อนี้อธิบายวิธีการตั้งค่าเงื่อนไขและกฎ เพื่อระงับใบสั่งขายการฉ้อโกงสำหรับการตรวจทานเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="676f9-105">This topic explains how to set up criteria and rules to place potentially fraudulent sales orders on hold for further review.</span></span> <span data-ttu-id="676f9-106">ฟังก์ชันการตรวจทานการฉ้อโกงจะถูกใช้เพื่อกำหนดความถูกต้องของข้อมูลในใบสั่งขาย</span><span class="sxs-lookup"><span data-stu-id="676f9-106">Fraud review functionality is used to determine the validity of information in a sales order.</span></span> <span data-ttu-id="676f9-107">ถ้าข้อมูลในใบสั่งขายปรากฏว่าน่าสงสัย ตามเกณฑ์การฉ้อโกงและกฎขององค์กร ใบสั่งอาจถูกระงับสำหรับการตรวจทานเพิ่มเติมโดยผู้ดูแลระบบ</span><span class="sxs-lookup"><span data-stu-id="676f9-107">If the information in the sales order appears to be questionable based on an organization’s fraud criteria and rules, the order may be put on hold for further review by an administrator.</span></span>

> [!NOTE]
> <span data-ttu-id="676f9-108">คุณลักษณะนี้สามารถถูกใช้ได้ด้วยกระบวนการใบสั่งขายช่องทาง Retail Call Center</span><span class="sxs-lookup"><span data-stu-id="676f9-108">This feature can only be used with Retail Call Center channel sales order processing.</span></span> 

<span data-ttu-id="676f9-109">เมื่อมีการกำหนดช่องทาง Call Center **เปิดใช้งานการดำเนินการใบสั่งให้เสร็จสมบูรณ์** ต้องถูกกำหนดเป็น **ใช่**</span><span class="sxs-lookup"><span data-stu-id="676f9-109">When the Call Center channel is defined, **Enable Order Completion** must be set to **Yes**.</span></span> <span data-ttu-id="676f9-110">เมื่อเปิดใช้งานการดำเนินการใบสั่งให้เสร็จสมบูรณ์ ผู้ใช้สามารถดูสรุปใบสั่ง และคลิก **ส่ง** เพื่อสรุปใบสั่งได้</span><span class="sxs-lookup"><span data-stu-id="676f9-110">When order completion is enabled, users can view the order recap and click **Submit** to finalize the order.</span></span> <span data-ttu-id="676f9-111">ผู้ใช้ยังจะได้รับตัวเลือกในการระงับใบสั่งขายการฉ้อโกงด้วยตนเอง</span><span class="sxs-lookup"><span data-stu-id="676f9-111">Users will also be provided options to manually place the sales order on fraud review hold.</span></span> <span data-ttu-id="676f9-112">ใบสั่งขายที่ถูกป้อนโดยผู้ใช้ Call Center ได้ถูกประมวลผลโดยใช้กฎการตรวจสอบการฉ้อโกงและเกณฑ์ในระหว่างกระบวนการส่ง</span><span class="sxs-lookup"><span data-stu-id="676f9-112">Sales orders keyed in by a Call Center user are processed through fraud check rules and criteria during the submit process.</span></span>

<span data-ttu-id="676f9-113">มีอยู่เกณฑ์การฉ้อโกงสองชนิดที่ระบบจะอ้างอิงเพื่อตรวจสอบว่า ใบสั่งควรถูกระงับสำหรับการตรวจทานการฉ้อโกงหรือไม่:</span><span class="sxs-lookup"><span data-stu-id="676f9-113">There are two types of fraud criteria that the system will reference to check if the order should be held for fraud review:</span></span>

-   <span data-ttu-id="676f9-114">**กฎแบบคงที่** ใช้ค่าเฉพาะ เช่น หมายเลขโทรศัพท์ที่ได้ถูงลงรายชื่อบัญชีดำหรือที่อยู่อีเมลที่ได้ถูกแฟล็ก โดยเป็นที่รู้กันว่าถูกใช้สำหรับธุรกรรมการฉ้อโกงในอดีต</span><span class="sxs-lookup"><span data-stu-id="676f9-114">**Static rules** use a specific value, such as a phone number that has been blacklisted or an email address that has been flagged as being known as used for past fraudulent transactions.</span></span> <span data-ttu-id="676f9-115">ในหน้า **ข้อมูลการฉ้อโกงแบบคงที่** ข้อมูลการฉ้อโกงสามารถถูกเพิ่มได้ด้วยตนเอง หรือโดยใช้การนำเข้าข้อมูล ด้วยคะแนนที่แนบอยู่กับข้อมูลการฉ้อโกง</span><span class="sxs-lookup"><span data-stu-id="676f9-115">On the **Static Fraud Data** page, fraud information can be added manually or through data import, with scores attached to the fraudulent information.</span></span> <span data-ttu-id="676f9-116">ถ้ามีการเปิดการตรวจสอบการฉ้อโกง ใบสั่งขายทุกรายการที่ถูกป้อนจะถูกเปรียบเทียบกับข้อมูลแบบคงที่</span><span class="sxs-lookup"><span data-stu-id="676f9-116">If fraud checking is turned on, every sales order entered will be compared to the static data.</span></span> <span data-ttu-id="676f9-117">ถ้าข้อมูลถูกพบอยู่ในการเรียกเก็บเงินของลูกค้าหรือที่อยู่จัดส่ง หรือถ้าข้อมูลถูกพบในที่อยู่จัดส่งในรายการการขาย จะมีรวมคะแนนการจับคู่ทั้งหมดที่ไม่ซ้ำกัน</span><span class="sxs-lookup"><span data-stu-id="676f9-117">If the data is found in either the customer’s billing or delivery address, or if the data is found in the delivery address on the sales line, the scores of all unique matches will be summed.</span></span>  
-   <span data-ttu-id="676f9-118">**กฎแบบไดนามิก** ประกอบด้วย ตัวแปรและเงื่อนไขที่กำหนดไว้สำหรับตัวแปรดังกล่าว</span><span class="sxs-lookup"><span data-stu-id="676f9-118">**Dynamic rules** are composed from variables and conditions defined for those variables.</span></span> <span data-ttu-id="676f9-119">ด้วยกฎแบบไดนามิก เกณฑ์อื่นที่ไม่มีการกำหนดในกฎแบบคงที่สามารถถูกตรวจสอบได้</span><span class="sxs-lookup"><span data-stu-id="676f9-119">With dynamic rules, other criteria not defined in the static rules can be checked.</span></span> <span data-ttu-id="676f9-120">คำสั่ง “AND/OR” data-ttu-id="676f9-121">ตัวอย่างเช่น ถ้าผู้ใช้ต้องการระงับการตรวจทานการฉ้อโกงใบสั่งทั้งหมดสำหรับลูกค้าที่ผูกกับค่ากลุ่มลูกค้าเฉพาะ และที่สั่งซื้อผลิตภัณฑ์เฉพาะ <span data-ttu-id="676f9-123">ผู้ใช้ศูนย์บริการอาจวางใบสั่งลงในคิวระงับการฉ้อโกงด้วยตนเองได้ด้วย</span><span class="sxs-lookup"><span data-stu-id="676f9-123">A call center user may also manually place an order into the fraud hold queue.</span></span> <span data-ttu-id="676f9-124">ถ้าผู้ใช้ที่ป้อนใบสั่งเชื่อว่า ลูกค้าที่วางใบสั่งอาจน่าสงสัยและต้องการให้บุคคลอื่นตรวจทานความถูกต้องของใบสั่งก่อนที่จะดำเนินต่อไป ผู้ใช้ที่ป้อนใบสั่งสามารถเลือกตัวเลือก **คู่มือการระงับการฉ้อโกง** จากเมนูแบบหล่นลง **ระงับ** ในหน้า **สรุปใบสั่งขาย** (ซึ่งปรากฏหลังจากมีการเรียกใช้ฟังก์ชัน **ทำให้เสร็จสมบูรณ์** ในใบสั่ง)</span><span class="sxs-lookup"><span data-stu-id="676f9-124">If the user entering the order believes the customer placing the order may be suspicious and wants someone else to further review the validity of the order before it is processed, the user entering the order can choose the **Manual Fraud Hold** option from the **Holds** dropdown menu on the **Sales Order Summary** page (this appears after the **Complete** order function is called).</span></span> <span data-ttu-id="676f9-125">ผู้ใช้จะได้รับพร้อมต์ให้ป้อนบันทึกย่อเพื่ออธิบายว่า เพราะเหตุใดพวกเขาจึงรู้สึกว่าใบสั่งอาจปลอม เพื่อให้ผู้ตรวจทานมีบริบทเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="676f9-125">The user will be prompted to enter a note to further explain why they feel the order may be fraudulent so that the reviewer has more context.</span></span>

<span data-ttu-id="676f9-126">ใบสั่งที่ถูกระงับทั้งหมดโดยใช้การระงับการฉ้อโกงด้วยตนเอง หรือโดยการคำนวณทางระบบของคะแนนการฉ้อโกง จะปรากฏในหน้า **ระงับใบสั่ง** ที่ซึ่งใบสั่งสามารถถูกตรวจทาน แล้วถูกยกเลิกหรือนำออกใช้ไปยังการประมวลผล</span><span class="sxs-lookup"><span data-stu-id="676f9-126">All orders held through manual fraud hold or by systematic calculation of fraud scores will appear on the **Order Holds** page, where the order can be reviewed and then either canceled or released to processing.</span></span>
