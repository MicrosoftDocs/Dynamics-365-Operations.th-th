# <a name="cart-module"></a>โมดูลรถเข็น

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

หัวข้อนี้ครอบคลุมถึงโมดูลรถเข็น และอธิบายวิธีการเพิ่มลงในเพจของไซต์ใน Microsoft Dynamics 365 Commerce

## <a name="overview"></a>ภาพรวม

โมดูลรถเข็นเป็นคอนเทนเนอร์ที่ใช้สำหรับโฮสต์โมดูลทั้งหมดที่จำเป็นต้องใช้ในการแสดงรายการที่อยู่ในรถเข็น ตัวอย่างเช่น จะรวมสินค้าทั้งหมดที่มีการเพิ่มลงในรถเข็น สรุปใบสั่ง และรหัสโปรโมชัน

โมดูลรถเข็นแสดงข้อมูลตามรหัสรถเข็น รหัสรถเข็นเป็นคุกกี้ของเบราว์เซอร์ที่มีให้ใช้งานทั่วทั้งไซต์

## <a name="cart-module-properties-and-slots"></a>คุณสมบัติของโมดูลรถเข็นและช่อง

ในฐานะที่เป็นคอนเทนเนอร์ โมดูลรถเข็นจะควคุณสมบัติพื้นฐานบางอย่าง เช่น หัวข้อง และ ความกว้าง โดยทั่วไปหัวข้อจะเป็นข้อความ เช่น **ถุงช้อปปิ้ง** **รถเข็นของคุณ** หรือ **สินค้าในรถเข็นของคุณ** การตั้งค่าความกว้างจะกำหนดว่าโมดูลภายในคอนเทนเนอร์ต้องพอดีกับคอนเทนเนอร์ หรือเต็มหน้าจอ

หน้ารถเข็นสินค้าจะถูกแบ่งออกเป็นหลายภาค: สินค้าในรายการรถเข็น สรุปใบสั่ง และการชำระเงิน และฟังก์ชัน "ค้นหาในร้านค้า"  แต่ละภาคจะถูกแม็ปกับช่องในโมดูลรถเข็น และช่องทุกช่องยอมรับชุดของโมดูลที่กำหนดไว้ล่วงหน้า

## <a name="modules-that-can-be-used-in-a-cart-module"></a>โมดูลที่สามารถใช้ในโมดูลรถเข็น

- **สินค้าในรถเข็น** –โมดูลนี้แสดงสรุปสินค้าทุกรายการที่มีการเพิ่มลงในรถเข็น ข้อมูลรวมถึงชื่อผลิตภัณฑ์ มิติที่เลือก ราคา ปริมาณ และส่วนลด นอกจากนี้โมดูลยังสนับสนุนการดำเนินการ เช่น "ลบออกจากรถเข็น" และ "เพิ่มไปยังรายการที่ต้องการ" สำหรับสินค้าแต่ละรายการ จะมีตัวเลือกในการจัดส่งสินค้า หรือรับสินค้าจากร้านค้า คุณต้องตั้งค่าคอนฟิกตัวเลือกนี้แยกต่างหาก ในโมดูลการรับสินค้าในร้านค้า
- **สรุปใบสั่ง** –โมดูลนี้แสดงสรุปใบสั่ง ข้อมูลรวมถึง ยอดรวม การจัดส่ง ภาษี และการประหยัด สามารถตั้งค่าคอนฟิกหัวข้อสำหรับโมดูลนี้
- **รหัสโปรโมชัน** –โมดูลนี้จะช่วยให้ลูกค้าสามารถแลกรหัสโปรโมชันได้ คุณสามารถใช้หรือลบรหัสโปรโมชันได้
- **การชำระเงิน** –โมดูลนี้จะเริ่มต้นการดำเนินการชำระเงิน และเปลี่ยนเส้นทางผู้ใช้ไปที่หน้าเช็คเอาท์ ต้องตั้งค่าคอนฟิกลิงค์สามลิงค์สำหรับโมดูลนี้:

    - **การเงินชำระเงิน** –ลิงก์นี้บังคับให้ลูกค้าล็อกอินถ้ายังไม่ได้ล็อกอิน
    - **การเช็คเอาท์สำหรับแขก** –ลิงค์นี้ช่วยให้ลูกค้าที่ไม่ระบุชื่อสั่งซื้อได้ แสดงเฉพาะเมื่อลูกค้าไม่ได้ล็อกอินเท่านั้น
    - **กลับไปที่การช้อปปิ้ง** –ลิงค์นี้ให้ลูกค้าไปที่โฮมเพจหรือหน้าอื่นๆ เพื่อให้สามารถดำเนินการเลือกซื้อต่อไป

- **บล็อคเนื้อหา** –โมดูลนี้สนับสนุนการส่งข้อความแบบกำหนดเองในโมดูลรถเข็น ข้อความจะถูกขับเคลื่อนโดยระบบการจัดการเนื้อหา (CMS) คุณสามารถเพิ่มข้อความใดก็ได้ เช่น "สำหรับปัญหาในการสั่งซื้อ โปรดติดต่อ 1-800-Fabrikam"
- **เบิกสินค้าในร้านค้า** –โมดูลนี้แสดงรายการของร้านค้าใกล้เคียง ที่มีสินค้าพร้อมสำหรับการเบิกสินค้า โดยค่าเริ่มต้น โมดูลนี้จะแสดงร้านค้าที่อยู่ภายในรัศมี 50 ไมล์จากสถานที่ของลูกค้า อย่างไรก็ตาม คุณสามารถกำหนดรัศมีในการค้นหาได้ในโมดูล สำหรับร้านค้าแต่ละร้าน ทำการตรวจสอบสินค้าคงคลัง ถ้ามีการเปิดใช้งานฟังก์ชันการตรวจสอบสินค้าคงคลัง และมีการแสดงข้อความ มีสินค้าในสต๊อก หรือ ไม่มีสินค้าในสต๊อก ที่เหมาะสม
- **ค้นหาร้านค้า โดย Bing Maps** - โมดูลนี้สามารถใช้ในโมดูลเบิกสินค้าจากร้านค้า ช่วยให้ลูกค้าค้นหาร้านค้าด้วยการป้อนตำแหน่งที่ตั้ง อินเทอร์เฟสเขียนโปรแกรมกำหนดพิกัดทางภูมิศาสตร์ของ Bing Maps (API) ถูกใช้เพื่อแปลงสถานที่ที่ลูกค้าป้อนให้เป็นละติจูดและลองจิจูด ถ้าไม่มีการใช้โมดูลนี้ เฉพาะร้านค้าที่อยู่ใกล้กับสถานที่ปัจจุบันของลูกค้าจะถูกแสดงขึ้น และลูกค้าไม่สามารถค้นหาสถานที่อื่นได้

## <a name="cart-module-settings"></a>การตั้งค่าโมดูลรถเข็น

โมดูลรถเข็นมีสามการตั้งค่า ที่สามารถตั้งค่าคอนฟิกได้:

- **ปริมาณสูงสุด** - จำนวนสูงสุดของสินค้าแต่ละรายการที่สามารถเพิ่มเข้าในรถเข็น ตัวอย่างเช่น ผู้ค้าปลีกอาจตั้งให้สามารถขายผลิตภัณฑ์แต่ละอย่างได้เพียง 10 ชุด ในหนึ่งธุรกรรมเท่านั้น
- **การตรวจสอบสินค้าคงคลัง** – เมื่อมีการกำหนดค่า เป็น **จริง** สินค้าจะถูกเพืเมลงในรถเข็น เฉพาะหลังจากที่โมดูลกล่องการซื้อได้ตรวจสอบให้แน่ใจว่ามีสินค้าอยู่ในสต๊อก การตรวจสอบสินค้าคงคลังนี้ดำเนินการทั้งสองอย่างสำหรับสถานการณ์ที่สินค้าจะถูกจัดส่ง และสำหรับสถานการณ์ที่จะมีการเบิกสินค้าในร้านค้า ถ้าค่าถูกกำหนดเป็น **เท็จ** จะไม่มีการตรวจสอบสินค้าคงคลัง ก่อนที่จะมีการเพิ่มสินค้าลงในรถเข็นและใบสั่งถูกสั่งซื้อ
- **บัฟเฟอร์สินค้าคงคลัง** –สินค้าคงคลังจะมีการเก็บรักษาในเวลาจริง และเมื่อลูกค้าหลายรายที่สั่งซื้อ อาจเป็นเรื่องยากที่จะรักษาการตรวจนับสินค้าคงคลังที่ถูกต้อง ดังนั้น จึงสามารถกำหนดบัฟเฟอร์ให้กับสินค้าคงคลังได้ เมื่อมีการตรวจสอบสินค้าคงคลังแล้ว ถ้าสินค้าคงคลังน้อยกว่าจำนวนบัฟเฟอร์ ผลิตภัณฑ์จะถือว่าไม่มีในสต็อก ดังนั้น เมื่อการขายเกิดขึ้นอย่างรวดเร็วผ่านหลายช่องทาง เพื่อให้การตรวจนับสินค้าคงคลังไม่ได้รับการซิงค์อย่างครบถ้วน มีความเสี่ยงน้อยกว่าที่สินค้าที่ไม่มีในสต๊อกจะถูกขาย

## <a name="retail-server-interaction"></a>การโต้ตอบเซิร์ฟเวอร์การขายปลีก

โมดูลรถเข็นจะดึงข้อมูลผลิตภัณฑ์โดยใช้ Retail Server APIs รหัสรถเข็นจากคุกกี้ของเบราว์เซอร์ใช้เพื่อดึงข้อมูลผลิตภัณฑ์ทั้งหมดจาก Retail Server

## <a name="add-a-cart-module-to-a-page"></a>เพิ่มโมดูลรถเข็นไปยังหน้า

การเพิ่มโมดูลรถเข็นไปยังหน้าใหม่ และตั้งค่าคุณสมบัติที่จำเป็น ให้ทำตามขั้นตอนต่อไปนี้

1. สร้างส่วนย่อยที่มีชื่อว่า **ส่วนย่อยของรถเข็น** และเพิ่มโมดูลรถเข็น
1. ในช่อง **สินค้าของรายการรถเข็น** ของโมดูลรถเข็นให้เพิ่มโมดูลสินค้าของรายการรถเข็น
1. ในช่อง **สรุปใบสั่งซื้อ** ให้เพิ่มโมดูลสรุปใบสั่ง
1. ในช่อง **รหัสโปรโมชัน** เพิ่มโมดูลรหัสโปรโมชัน
1. ในช่อง **การชำระเงิน** ให้เพิ่มโมดูลการชำระเงิน
1. ในช่อง **ค้นหาในร้าค้า** ให้เพิ่มโมดูลการเบิกสินค้าในร้านค้า
1. ในโมดูิเบิกสินค้าในร้าน เลือกช่อง **การค้นหาร้านค้า** และเพิ่มการค้นหาร้านค้าโดยใช้โมดูล Bing Maps
1. เช็คอินส่วน และเผยแพร่
1. สร้างเท็มเพลตที่มีชื่อ **เท็มเพลตรถเข็น** และเพิ่มส่วนของรถเข็นที่คุณเพิ่งสร้างขึ้น
1. บันทึกเท็มเพลต เช็คอิน และเผยแพร่
1. สร้างหน้าที่ใช้เท็มเพลตใหม่
1. บันทึกและแสดงตัวอย่างหน้า
1. เช็คอินหน้า และเผยแพร่
