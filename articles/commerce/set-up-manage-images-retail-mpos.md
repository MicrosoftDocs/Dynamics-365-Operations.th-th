# <a name="set-up-and-manage-images-for-modern-pos-mpos"></a>ตั้งค่าและจัดการรูปภาพสำหรับ Modern POS (MPOS)

[!include [banner](includes/banner.md)]

บทความนี้อธิบายขั้นตอนที่เกี่ยวข้องในการตั้งค่าและการจัดการรูปภาพสำหรับเอนทิตีต่างๆ ที่ปรากฏขึ้นใน Modern POS (MPOS)

## <a name="setting-up-the-media-base-url-and-defining-media-templates-to-configure-the-format-for-image-urls"></a>การตั้งค่าฐานสื่อ URL และการกำหนดเทมเพลตสื่อเพื่อตั้งค่าคอนฟิกรูปแบบสำหรับ URL ของรูปภาพ

รูปภาพที่ปรากฏใน Modern POS (MPOS) ต้องถูกโฮสภายนอก ด้านนอก Commerce โดยทั่วไป จะถูกโฮสต์ในระบบการจัดการเนื้อหา เครือข่ายส่งเนื้อหา (CDN) หรือเซิร์ฟเวอร์สื่อ จากนั้น MPOS นำมาใช้และแสดงรูปภาพสำหรับเอนทิตี้ที่เหมาะสม เช่น ผลิตภัณฑ์และแค็ตตาล็อก โดนการเข้าถึง URL เป้าหมาย เพื่อที่จะนำรูปภาพที่ถูกโฮสต์จากภายนอกเหล่านี้มาใช้ MPOS จำเป็นต้องมีรูปแบบ URL ที่ถูกต้องสำหรับรูปภาพ คุณสามารถตั้งค่าคอนฟิกรูปแบบ URL ที่จำเป็นสำหรับรูปภาพโดยการตั้งค่าค่า **URL ฐานสื่อ** ในโพรไฟล์ช่องและใช้ฟังก์ชัน **กำหนดเทมเพลตสื่อ** สำหรับแต่ละเอนทิตี้ได้ คุณสามารถบันทึกทับรูปแบบ URL มาตรฐานสำหรับชุดย่อยของเอนทิตี โดยการใช้ฟังก์ชั่น **แก้ไขใน Excel** ได้

> [!IMPORTANT]
> ในรุ่นปัจจุบันของ Commerce คุณไม่สามารถตั้งค่ารูปแบบ URL โดยการใช้ XML แอตทริบิวต์ **รูปภาพ** สำหรับ MPOS ได้อีกต่อไปในกลุ่มแอตทริบิวต์ **ค่าเริ่มต้น** สำหรับเอนทิตี ถ้าคุณคุ้นเคยกับ Microsoft Dynamics AX 2012 R3 และขณะนี้กำลังใช้รุ่นปัจจุบันของ Commerce กรุณาตรวจสอบให้แน่ใจว่า คุณใช้ฟังก์ชัน **กำหนดเทมเพลตสื่อ** ใหม่เสมอ เพื่อตั้งค่ารูปภาพ อย่าใช้หรือปรับเปลี่ยนแอตทริบิวต์ **รูปภาพ** ในกลุ่มแอททิรบิวต์ **เริ่มต้น** สำหรับเอนทิตีใด ๆ รวมถึงผลิตภัณฑ์ การเปลี่ยนแปลงที่คุณทำได้โดยตรงในกลุ่มแอตทริบิวต์ **เริ่มต้น** สำหรับรูปภาพจะไม่สะท้อนให้เห็น ตัวเลือกนี้จะไม่ทำงานในรุ่นต่อไป

ในกระบวนงานต่อไปนี้ รูปภาพจะถูกตั้งค่าสำหรับเอนทิตี้แค็ตตาล็อกเป็นตัวอย่าง กระบวนงานเหล่านี้จะช่วยรับประกันว่า เส้นทางปลายทางของรูปภาพที่ถูกต้องถูกตั้งค่าสำหรับรูปภาพแค็ตตาล็อกทั้งหมดที่ใช้เส้นทางทั่วไปโดยนัย ตัวอย่างเช่น ถ้าคุณได้ตั้งค่าเซิร์ฟเวอร์สื่อหรือ CDN จากภายนอก และต้องการให้รูปภาพปรากฏใน MPOS สำหรับร้านค้าที่กำหนด ฟังก์ชัน **กำหนดเทมเพลตสื่อ** จะช่วยคุณตั้งเส้นทางที่ MPOS สามารถค้นหาและดึงข้อมูลรูปภาพได้

> [!NOTE]
> สำหรับตัวอย่างข้อมูลสาธิตนี้ เซิร์ฟเวอร์สื่อถูกปรับใช้ใน Commerce Scale Unit อย่างไรก็ตาม คุณสามารถใช้ที่ใดก็ได้ภายนอก Commerce

### <a name="set-up-the-media-base-url-for-a-channel"></a>ตั้งค่า URL ฐานสื่อสำหรับช่องทาง

1. เปิดพอร์ทัล Commerce HQ
2. คลิก **Retail และ Commerce** &gt; **การตั้งค่าช่องทาง** &gt; **โพรไฟล์ช่องทาง**

    [![การนำทาง](./media/channel-profile1.png)](./media/channel-profile1.png)

3. ในโพรไฟล์ช่องทางที่ร้านค้าของคุณใช้สำหรับ MPOS อัปเดตฟิลด์ **URL ฐานสื่อ** ด้วย URL ฐานสื่อของเซิร์ฟเวอร์สื่อหรือ CDN ของคุณ ฐาน URL คือ ส่วนแรกของ URL ที่ถูกใช้ร่วมกันโดยโฟลเดอร์รูปภาพทั้งหมดของเอนทิตี้ที่แตกต่างกัน

    [![หน้าโพรไฟล์ช่องทาง](./media/channel-profile2.png)](./media/channel-profile2.png)

### <a name="define-the-media-template-for-an-entity"></a>กำหนดเทมเพลตสื่อสำหรับเอนทิตี้

1. คลิก **Retail และ Commerce** &gt; **การจัดการแค็ตตาล็อก** &gt; **รูปภาพแค็ตตาล็อก**
2. ในหน้า **รูปภาพแค็ตตาล็อก** บนบานหน้าต่างการดำเนินการ คลิก **กำหนดเทมเพลตสื่อ** ในกล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** ในฟิลด์ **เอนทิตี้** **แค็ตตาล็อก** ควรถูกเลือกตามค่าเริ่มต้น
3. ในแท็บด่วน **เส้นทางสื่อ** ป้อนเส้นที่เหลืออยู่ของสถานที่เก็บรูปภาพ เส้นทางสื่อสนับสนุน **LanguageID** เป็นตัวแปร ตัวอย่างเช่น สำหรับข้อมูลสาธิต คุณสามารถสร้างโฟลเดอร์ **แค็ตตาล็อก** สำหรับรูปภาพแค็ตตาล็อกทั้งหมด ภายใต้ URL ฐานสื่อสำหรับเซิร์ฟเวอร์สื่อของคุณ (`https://testax3ret.cloud.test.dynamics.com/RetailServer/MediaServer`) จากนั้นคุณสามารถมีโฟลเดอร์สำหรับแต่ละภาษา เช่น อังกฤษ-สหรัฐอเมริกา หรือ ฝรั่งเศส-ฝรั่งเศส และคัดลอกรูปภาพที่เหมาะสมภายใต้แต่ละโฟลเดอร์ ถ้าคุณไม่มีรูปภาพที่แตกต่างกันสำหรับภาษาต่าง ๆ คุณสามารถข้ามตัวแปร **LanguageID** จากโครงสร้างโฟลเดอร์ของคุณและมุ่งโดยตรงไปยังโฟลเดอร์แค็ตตาล็อกที่ประกอบด้วยรูปภาพแค็ตตาล็อกได้

    > [!NOTE]
    > รุ่นปัจจุบันของ Commerce สนับสนุนโทเคน **{LanguageId}** สำหรับเอนทิตีแค็ตตาล็อก ผลิตภัณฑ์ และประเภท (โทเคน **{LanguageID}** ไม่ได้รับการสนับสนุนสำหรับเอนทิตีลูกค้าและผู้ปฏิบัติงาน ตามมาตรฐานที่มีอยู่ที่มีการมีผลบังคับใช้นับตั้งแต่ Microsoft Dynamics AX 6.x.)

4. สำหรับรูปภาพ รูปแบบชื่อไฟล์ตายตัวกับชื่อแค็ตตาล็อกและไม่สามารถถูกเปลี่ยนแปลงได้ ดังนั้น เปลี่ยนชื่อรูปภาพของคุณเพื่อให้มีชื่อแค็ตตาล็อกที่เหมาะสม เพื่อที่จะช่วยรับประกันว่า MPOS จัดการได้อย่างถูกต้อง
5. ในฟิลด์ **นามสกุลไฟล์** เลือกนามสกุลไฟล์ที่คาดไว้ โดยขึ้นอยู่กับชนิดของรูปภาพที่คุณมี ตัวอย่างเช่น สำหรับข้อมูลสาธิต รูปภาพแค็ตตาล็อกจะถูกกำหนดเป็นนามสกุล .jpg (ไฟล์รูปภาพจะถูกเปลี่ยนชื่อเพื่อให้มีชื่อแค็ตตาล็อกด้วยเช่นกัน)
6. คลิก **ตกลง**
7. เพื่อตรวจสอบว่าเทมเพลตสื่อสำหรับรูปภาพถูกบันทึกอย่างถูกต้อง ในหน้า **รูปภาพแค็ตตาล็อก** คลิก **กำหนดเทมเพลตสื่อ** อีกครั้ง เพื่อตรวจสอบเทมเพลตโดยไม่มีการปิดกล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** คุณสามารถใช้แท็บด่วน **สร้าง URL ของรูปภาพสำหรับ Excel** ได้ เช็คลักษณะของ URL ของรูปภาพ และตรวจสอบว่า URL นั้นสอดคล้องกับมาตรฐานเทมเพลตที่ถูกกล่าวถึงก่อนหน้านี้ กล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** ขณะนี้ได้ตั้งเส้นทางโดยนัยสำหรับรูปภาพแค็ตตาล็อกทั้งหมดที่ใช้เส้นทาง URL ทั่วไปนี้ เส้นทาง URL นี้ใช้กับรูปภาพแค็ตตาล็อกทั้งหมดยกเว้นว่าถูกบันทึกทับ ส่วนแรกของเส้นทางรูปภาพจะถูกนำมาจาก URL ฐานสื่อที่คุณกำหนดไว้ในโพรไฟล์ช่องทาง ส่วนที่เหลือของเส้นทางจะถูกนำมาจากเส้นทางที่คุณกำหนดไว้ในเทมเพลตสื่อ ทั้งสองส่วนจะถูกเรียงต่อกันเพื่อจัดเตรียม URL แบบเต็มของสถานที่เก็บรูปภาพ ตัวอย่างเช่น แค็ตตาล็อกในข้อมูลสาธิตที่ถูกตั้งชื่อแค็ตตาล็อกฐานของ Fabrikam ดังนั้น ชื่อรูปภาพต้องมี Fabrikam Base Catalog.jpg เพื่อให้ใช้ชื่อแค็ตตาล็อกและนามสกุลไฟล์ .jpg ที่ถูกตั้งค่าคอนฟิกในเทมเพลต ในกรณีนี้ หลังจากการเรียงต่อกัน URL จะเป็น `https://testax3ret.cloud.test.dynamics.com/RetailServer/MediaServer/Catalogs/en-US/Fabrikam Base Catalog.jpg`
8. รันงานการซิงโครไนส์เพื่อที่จะกำหนดเทมเพลตใหม่ไปยังฐานข้อมูลช่องทาง เพื่อให้ MPOS สามารถใช้เทมเพลตนั้นในการเข้าถึงรูปภาพได้
9. เพื่ออัปเดตเทมเพลตสื่อสำหรับรูปภาพแค็ตตาล็อกในฝั่งช่องทาง ให้แน่ใจว่าได้รัน **งานแค็ตตาล็อก 1150** จาก **Retail และ Commerce IT** &gt; **กำหนดการกระจาย**

    [![กล่องโต้ตอบกำหนดเทมเพลตของสื่อ](./media/catalog1.png)](./media/catalog1.png)

## <a name="previewing-an-image-from-the-entity-level"></a>แสดงตัวอย่างรูปภาพจากระดับเอนทิตี้

1. จากหน้าสำหรับสินค้าเอนทิตี้แบบ HQ คุณสามารถแสดงตัวอย่าง URL รูปภาพที่ถูกส่งมาจากเทมเพลตสื่อ สำหรับตัวอย่างนี้ ไปยังแค็ตตาล็อกที่เหมาะสม และจากนั้น บนบานหน้าต่างการดำเนินการ คลิก **สื่อ** &gt; **> รูปภาพ** ใช้รายการแบบหล่นลงเพื่อเลือกร้านค้าต่าง ๆ ที่อาจมีโพรไฟล์ช่องทางที่แตกต่างกัน
2. เพื่อแก้ไขหรือลบเทมเพลตสื่อโดยนัย คุณต้องส่งคืนไปยังกล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** สำหรับหน้า **รูปภาพแค็ตตาล็อก**
3. คุณสามารถใช้ปุ่ม **เพิ่ม** และ **ลบ** เพื่อที่จะเปลี่ยนเส้นทางที่ยึดตามเทมเพลตโดยนัย และใช้สำหรับรูปภาพเฉพาะ ด้วยตนเองได้ สำหรับข้อมูลเพิ่มเติม ดูส่วน [บันทึกทับสื่อเทมเพลตสำหรับรายการเอนทิตี](#overwriting-the-media-template-for-entity-items) ภายหลังในบทความนี้
4. หลังจากที่คุณได้เสร็จสิ้นการแสดงตัวอย่างรูปภาพ และทำการเปลี่ยนแปลงใด ๆ ที่คุณต้องการ เริ่มต้นอินสแตนซ์ MPOS สำหรับร้านค้าที่เหมาะสม และดูว่ารูปภาพแค็ตตาล็อกถูกแสดงหรือไม่

    [![กล่องโต้ตอบภาพ](./media/catalog4.png)](./media/catalog4.png)

> [!NOTE]
> คุณสามารถใช้กระบวนงานเดียวกันสำหรับเอนทิตีทั้งหมดห้ารายการที่ได้รับการสนับสนุน: ผู้ปฏิบัติงาน ลูกค้า แค็ตตาล็อก ประเภท และผลิตภัณฑ์ "แค็ตตาล็อกผลิตภัณฑ์" (ผลิตภัณฑ์ที่ถูกตั้งค่าไว้ในระดับแค็ตตาล็อก) และ "ผลิตภัณฑ์ตามช่องทาง" (ผลิตภัณฑ์ที่ถูกตั้งค่าในระดับช่องทาง) ใช้เทมเพลตสื่อที่ถูกตั้งค่าสำหรับเอนทิตี้ผลิตภัณฑ์ สำหรับเทมเพลตสื่อผลิตภัณฑ์ คุณสามารถเลือกหมายเลขของรูปภาพผลิตภัณฑ์เพื่อแสดงแต่ละผลิตภัณฑ์ คุณยังสามารถกำหนดรูปภาพเริ่มต้นสำหรับผลิตภัณฑ์ที่ระบุได้ด้วยเช่นกัน ด้วยวิธีนี้ คุณสามารถป้องกันไม่ให้มีรูปภาพที่ว่างเปล่าใน MPOS และช่วยควบคุมว่ารูปภาพใดจะถูกใช้เป็นรูปภาพเริ่มต้นสำหรับผลิตภัณฑ์หนึ่ง ๆ ในตัวอย่างต่อไปนี้ แต่ละผลิตภัณฑ์มีรูปภาพห้ารูป และรูปภาพแรกถูกตั้งเป็นรูปภาพเริ่มต้น ผลิตภัณฑ์ย่อยจะถูกปฏิบัติในลักษณะเดียวกันกับผลิตภัณฑ์หลัก ชื่อไฟล์ของไฟล์รูปภาพควรจะขึ้นอยู่กับหมายเลขผลิตภัณฑ์ อักขระบางตัวจะหายไปขณะที่มีสร้างชื่อไฟล์ด้วยเช่นกัน ดังนั้น จึงเป็นการดีที่จะตรวจสอบชื่อไฟล์ โดยการใช้ส่วน **สร้าง URL ของรูปภาพสำหรับ Excel** ดูส่วน [เขียนทับโดยใช้การแก้ไขใน Excel](#overwrite-by-using-edit-in-excel) ในภายหลังในบทความนี้

## <a name="synchronization-jobs-to-send-a-media-template-to-the-channel-side"></a>งานการซิงโครไนส์เพื่อส่งเทมเพลตสื่อไปยังด้านช่องทาง

สำหรับเอนทิตีที่ได้รับการสนับสนุนทั้งห้ารายการ (ผู้ปฏิบัติงาน ลูกค้า แค็ตตาล็อก ประเภท และผลิตภัณฑ์) เมื่อใดก็ตามที่คุณอัปเดตกล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** เพื่อตั้งค่ารูปภาพ ตรวจสอบให้แน่ใจว่าคุณรันงานแค็ตตาล็อก (1150) จาก **Retail และ Commerce IT** &gt; **กำหนดการกระจาย** งานนี้จะเปิดใช้งานเทมเพลตสื่อที่อัปเดตแล้วเพื่อที่จะซิงค์ไปยังช่องทาง และถูกใช้โดย MPOS รันงานแค็ตตาล็อก (1150) หลังจากที่คุณทำการเปลี่ยนแปลงใด ๆ ต่อไปนี้:

- คุณอัปเดตเทมเพลตสื่อรูปภาพแค็ตตาล็อกจาก **รูปภาพแค็ตตาล็อก** &gt; **กำหนดเทมเพลตสื่อ**
- คุณอัปเดตเทมเพลตสื่อรูปภาพพนักงานจาก **รูปภาพพนักงาน** &gt; **กำหนดเทมเพลตสื่อ**
- คุณอัปเดตเทมเพลตสื่อรูปภาพลูกค้าจาก **รูปภาพลูกค้า** &gt; **กำหนดเทมเพลตสื่อ**
- คุณอัปเดตเทมเพลตสื่อรูปภาพผลิตภัณฑ์จาก **รูปภาพผลิตภัณฑ์** &gt; **กำหนดเทมเพลตสื่อ**
- คุณอัปเดตเทมเพลตสื่อรูปภาพของประเภทจาก **รูปภาพของประเภท** &gt; **กำหนดเทมเพลตสื่อ** คุณยังต้องเผยแพร่ช่องทางด้วยเช่นกัน

## <a name="overwriting-the-media-template-for-entity-items"></a>การบันทึกทับเทมเพลตสื่อสำหรับเอนทิตี้

ตามที่คุณได้เรียนรู้ในส่วนก่อนหน้านี้ เทมเพลตสื่อสำหรับเอนทิตีที่กำหนดสนับสนุนเพียงหนึ่งเส้นทางทั่วไป เส้นทางนี้จะขึ้นอยู่กับ URL ฐานสื่อที่ถูกตั้งค่าคอนฟิกและเส้นทางสื่อที่ถูกกำหนดไว้ อย่างไรก็ตาม ในหลายกรณี ผู้ค้าปลีกต้องการให้สามารถใช้รูปภาพจากแหล่งข้อมูลที่แตกต่างกันสำหรับชุดย่อยของสินค้าในเอนทิตี้ได้ ตัวอย่างเช่น ร้านค้าใช้เซิร์ฟเวอร์สื่อที่ตนเองเป็นโฮสต์สำหรับชุดของรูปภาพแค็ตตาล็อก แต่ใช้ URL ของ CDN สำหรับชุดอื่น เพื่อที่จะบันทึกทับ URL ของรูปภาพที่ขึ้นอยู่กับเทมเพลตสื่อสำหรับรูปภาพเอนทิตี้ที่ระดับเอนทิตี้ คุณสามารถใช้การแก้ไขใน Excel และฟังก์ชันแก้ไขด้วยตนเอง จากหน้า **แสดงตัวอย่าง** ได้

### <a name="overwrite-by-using-edit-in-excel"></a>เขียนทับโดยการแก้ไขใน Excel

1. คลิก **Retail และ Commerce** &gt; **การจัดการแค็ตตาล็อก** &gt; **รูปภาพแค็ตตาล็อก**
2. ในหน้า **รูปภาพแค็ตตาล็อก** คลิก **กำหนดเทมเพลตสื่อ** ในกล่องโต้ตอบ **กำหนดเทมเพลตสื่อ** ในฟิลด์ **เอนทิตี้** **แค็ตตาล็อก** ควรถูกเลือก
3. ในแท็บด่วน **เส้นทางสื่อ** สังเกตสถานที่เก็บรูปภาพ
4. ในแท็บด่วน **สร้าง URL ของรูปภาพสำหรับ Excel** คลิก **สร้าง**

    > [!IMPORTANT]
    > ทุกครั้งที่เทมเพลตสื่อถูกเปลี่ยนแปลง คุณต้องคลิก **สร้าง** ก่อนที่คุณจะสามารถใช้ฟังก์ชันแก้ไขใน Excel ได้

    ขณะนี้คุณจะเห็นการแสดงตัวอย่างของรูปภาพ URL ที่ถูกสร้างขึ้นตามเทมเพลตสื่อที่ถูกบันทึกไว้ล่าสุด

    [![FastTab สร้าง URL รูปภาพสำหรับ Excel หลังจากที่มีการเลือก สร้าง](./media/excel2.png)](./media/excel2.png)

    > [!NOTE]
    > URL ที่ถูกสร้างขึ้นสำหรับ Excel ใช้เส้นทางและแบบแผนของเทมเพลตสื่อที่ถูกกำหนดไว้ แบบแผนเหล่านี้รวมถึงข้อตกลงสำหรับชื่อไฟล์ การคาดหวังคือคุณได้ทำการตั้งค่ารูปภาพจริงไว้ภายนอก Commerce และรูปภาพสามารถถูกดึงข้อมูลจาก URL ที่ถูกส่งมาจากเทมเพลตสื่อที่คุณได้กำหนดไว้ก่อนหน้านี้ คุณสามารถบันทึกทับ URL ที่ได้รับโดยการใช้ฟังก์ชันแก้ไขใน Excel

5. คลิก **แก้ไขใน Excel**.
6. หลังจากที่มีการเปิดแผ่นงาน Microsoft Excel คลิก **เปิดใช้งานการแก้ไข** เมื่อคุณได้รับพร้อมท์
7. เมื่อคุณได้รับพร้อม คลิก **เชื่อถือ add-in นี้** ในบานหน้าต่างด้านขวา และรอให้ add-in นี้ทำการติดตั้งให้เสร็จสมบูรณ์

    [![เชื่อถือ Add-in นี้](./media/excel4.jpg)](./media/excel4.jpg)

8. ถ้าคุณพร้อมที่จะลงชื่อเข้าใช้ ป้อนข้อมูลประจำตัวที่คุณใช้ในการลงชื่อเข้า HQ

    [![พร้อมต์ให้ลงชื่อเข้าใช้](./media/excel5.png)](./media/excel5.png)

9. หลังจากที่คุณลงชื่อเข้าใช้ คุณควรจะสามารถดูรายการของ URL ของรูปภาพสำหรับรายการต่าง ๆ ในแค็ตตาล็อกได้
10. คุณแก้ไข เพิ่ม และลบ URL ของรูปภาพสำหรับสินค้าต่าง ๆ ของเอนทิตี้
11. สำหรับเอนทิตี้ทั้งหมดยกเว้นผลิตภัณฑ์ คุณสามารถบันทึกทับ URL ของรูปภาพได้ แก้ไข URL รูปภาพที่มีอยู่ เพื่อให้ใช้ URL ปลายทางใหม่ของรูปภาพ และอัปเดตชื่อไฟล์ด้วยชื่อไฟล์ใหม่สำหรับไฟล์รูปภาพ ชื่อไฟล์ต้องไม่ซ้ำกันเพื่อช่วยในการรับประกันว่าเรกคอร์ดไม่ซ้ำกัน

    [![เขียนทับ URL ของรูปภาพใน Excel](./media/excel6.jpg)](./media/excel6.jpg)

    > [!NOTE]
    > เมื่อคุณบันทึกทับ URL ของรูปภาพสำหรับเอนทิตีผลิตภัณฑ์ โดยการใช้ฟังก์ชันแก้ไขใน Excel หรือหน้าสินค้าเอนทิตี MPOS จะแสดง URL ของรูปภาพเทมเพลตสื่อทั้งหมดร่วมกับ URL ของรูปภาพที่ถูกบันทึกทับ

12. หลังจากที่คุณเสร็จสิ้นการทำการเปลี่ยนแปลงของคุณ คลิก **เผยแพร่ใน Excel** เพื่อสร้างการเชื่อมโยงที่ชัดเจนรายการใหม่
13. กลับไปยัง HQ และคลิก **ตกลง**
14. รันงานการซิงโครไนส์ที่เหมาะสมสำหรับเอนทิตี้ และเช็คการแสดงตัวอย่างบนหน้าเอนทิตี้หรือใน MPOS

#### <a name="creating-new-records"></a>การสร้างเรกคอร์ดใหม่

คุณสามารถสร้างเรกคอร์ดใหม่ได้ใน Excel อย่างไรก็ตาม ตรวจสอบให้แน่ใจว่าคุณจัดเตรียมข้อมูลที่ถูกต้องไว้ ตัวอย่างเช่น เพื่อสร้างรายการใหม่สำหรับแค็ตตาล็อก ตรวจสอบให้แน่ใจว่ารหัสแค็ตตาล็อกและชื่อแค็ตตาล็อกถูกต้อง และจัดเตรียมให้ชื่อไฟล์ไม่ซ้ำกัน ชื่อไฟล์ที่ไม่ซ้ำกันมีความสำคัญมาก เนื่องจากการไม่ซ้ำกันของเรกคอร์ดใน Excel จะผ่านการตรวจสอบในระหว่างการเผยแพร่ อันดับแรก คัดลอกรายละเอียดจากแค็ตตาล็อกที่คุณต้องการสร้างเรกคอร์ดใหม่ และคัดลอกเรกคอร์ด คุณเพียงต้องอัปเดตชื่อไฟล์และ URL เนื่องจากข้อมูลที่เหลือจะเป็นแบบเดียวกัน เพื่อสร้างเรกคอร์ดใหม่สำหรับสินค้าเอนทิตี้ผลิตภัณฑ์ คุณต้องใช้กระบวนงานพื้นฐานเดียวกัน จากแผ่นงาน Excel คัดลอกเรกคอร์ดที่มีอยู่แล้วสำหรับผลิตภัณฑ์ที่คุณจะสร้างเรกคอร์ดใหม่ และจากนั้นแทนที่ URL ของรูปภาพและชื่อไฟล์ ตรวจสอบให้แน่ใจว่าชื่อแฟ้มไม่ซ้ำกัน

#### <a name="deleting-an-existing-record"></a>การลบเรกคอร์ดที่มีอยู่

เฉพาะเรกคอร์ด URL ของรูปภาพที่ถูกบันทึกทับเท่านั้นที่ถูกลบได้ หลังจากที่รูปภาพถูกลบ และการซิงโครไนส์เสร็จสมบูรณ์ รูปภาพจะไม่ปรากฏในหน้า **แสดงตัวอย่าง** หรือใน MPOS เรกคอร์ด URL ของรูปภาพที่ถูกส่งมาจากเท็มเพลสื่อไม่สามารถถูกลบได้ เนื่องจากเรกคอร์ดเหล่านี้ถูกส่งมาจากเทมเพลตสื่อเสมอทุกครั้ง

### <a name="overwrite-from-the-entity-level-preview-page"></a>บันทึกทับจากหน้าแสดงตัวอย่างระดับเอนทิตี้

สำหรับเอนทิตีทั้งหมดยกเว้นผลิตภัณฑ์ คุณสามารถบันทึกทับ URL ของรูปภาพสำหรับเอนทิตี้ที่ระบุสินค้าที่ระดับสินค้าเอนทิตี้จากหน้า **แสดงตัวอย่าง** ได้ สำหรับผลิตภัณฑ์ คุณสามารถใช้หน้าเอนทิตี้ "ผลิตภัณฑ์แค็ตตาล็อก" ตัวอย่างนี้แสดงวิธีการบันทึกทับรูปภาพแค็ตตาล็อก

1. คลิก **แค็ตตาล็อก** &gt; **สื่อ** &gt; **รูปภาพ** และเลือกรูปภาพแค็ตตาล็อกเพื่ออัปเดต
2. คลิก **เพิ่ม** และป้อน URL ของรูปภาพเพื่อบันทึกทับ URL เทมเพลตสื่อ
3. ถ้าคุณต้องการให้รูปภาพนี้ถูกแสดงใน MPOS สำหรับแค็ตตาล็อก คุณสามารถตั้งเป็นรูปภาพเริ่มต้น
4. คลิก **ตกลง** ระบบจะนำเข้าข้อมูลการชำระเงิน URL ของรูปภาพมีถูกอัปเดตสำหรับรูปภาพแค็ตตาล็อกนี้ และการแสดงตัวอย่างจะถูกแสดงขึ้น

    [![URL ที่ปรับปรุงในกล่องโต้ตอบรูปภาพใหม่](./media/preview3.png)](./media/preview3.png)

5. คุณยังสามารถดูตัวอย่างรูปภาพสำหรับ URL รูปภาพที่ถูกบันทึกทับทั้งหมดในหน้าแกลเลอรี **รูปภาพแค็ตตาล็อก**

    [![หน้าแกลเลอรีรูปภาพในแค็ตตาล็อก](./media/preview-4.png)](./media/preview-4.png)

> [!NOTE]
> เฉพาะรูปภาพที่เผยแพร่ต่อสาธารณะและไม่ระบุชื่อเท่านั้นที่จะแสดงบน POS POS สนับสนุนการแสดงภาพที่โฮสต์ภายนอก และกําหนดว่าภาพจะถูกส่งคืนเป็น octet-stream แบบอินไลน์ไปยังการร้องขอ GET โดยไม่มีส่วนหัว ด้วยนโยบายการเข้าถึงแบบไม่ระบุชื่อ เฉพาะสำหรับภาพที่โฮสต์บน SharePoint ซึ่งต้องการให้ส่วนหัวมีทั้งโฮสต์และส่วนหัวของตัวแทนผู้ใช้ จะส่งคืนการตอบสนองเป็น "ไม่ได้รับอนุญาต" ดังนั้น การจัดการรูปภาพโดยใช้ SharePoint เป็นโฮสต์จึงไม่รับการสนับสนุนทันทีในขณะนี้ หน้าแกลเลอรี **รูปภาพในแค็ตตาล็อก** ไม่แสดงตัวอย่างรูปภาพสำหรับ URL ของรูปภาพเทมเพลตสื่อ เนื่องจากไคลเอ็นต์ Commerce Scale Unit (CSU) แสดงเพียงภาพเดียวต่อเอนทิตีแค็ตตาล็อก ผู้ปฏิบัติงาน ลูกค้า และประเภท ถ้าผู้ใช้จัดเตรียม URL อย่างชัดเจนผ่านหน้านี้สำหรับเอนทิตีแค็ตตาล็อก ผู้ปฏิบัติงาน ลูกค้า และประเภทเราขอแนะนำให้คุณระบุรูปภาพที่เป็นรูปภาพเริ่มต้น ถ้าคุณไม่ได้ระบุรูปภาพเริ่มต้น ระบบจะกำหนดรูปภาพเริ่มต้นและส่งไปยังผู้เรียกบริการ Commerce (MPOS หรืออีคอมเมิร์ซ)

### <a name="overwrite-the-image-url-for-catalog-product-images-from-the-preview-page"></a>บันทึกทับ URL ของรูปภาพสำหรับรูปภาพผลิตภัณฑ์แค็ตตาล็อกจากหน้าแสดงตัวอย่าง

เพื่อที่จะบันทึกทับ URL ของรูปภาพสำหรับรูปภาพผลิตภัณฑ์แค็ตตาล็อก คุณต้องใช้หน้า **แสดงตัวอย่าง** คุณไม่สามารถใช้ฟังก์ชันแก้ไขใน Excel ได้

1. เพื่อที่จะบันทึกทับรูปภาพผลิตภัณฑ์ที่ระดับแค็ตตาล็อก เลือกแค็ตตาล็อก และจากนั้นเลือกผลิตภัณฑ์เพื่อบันทึกทับรูปภาพ
2. คลิก **แอตทริบิวต์**
3. ในหน้าถัดไป เลือก **รูปภาพ** และจากนั้น คลิก **แก้ไข**. หน้า **แสดงตัวอย่าง** เปิดเป็นกล่องโต้ตอบแบบเลื่อน
4. คลิก **เพิ่ม** และบันทึกทับ URL ของรูปภาพด้วย URL ใหม่
5. คลิก **ตกลง** ระบบจะนำเข้าข้อมูลการชำระเงิน ขณะนี้คุณจะเห็นการแสดงตัวอย่างรูปภาพใหม่ และสามารถตั้งค่าเป็นรูปภาพเริ่มต้น

    [![พรีวิวรูปภาพในกล่องโต้ตอบรูปภาพใหม่](./media/cat3.png)](./media/cat3.png)

> [!NOTE]
> หลังจากการเชื่อมโยงรูปภาพประเภท คุณต้องเผยแพร่ช่องทางและช่องทางที่จะช่วยรับประกันว่า การเปลี่ยนแปลงถูกเผยแพร่ไปยังฐานข้อมูลช่องทาง

## <a name="setting-up-images-to-appear-in-offline-mode-for-mpos"></a>การตั้งค่ารูปภาพผลิตภัณฑ์เพื่อที่จะแสดงอยู่ในโหมดออฟไลน์สำหรับ MPOS

MPOS สามารถรันในโหมดออนไลน์ (เมื่อ MPOS เชื่อมต่อกับ Commerce Scale Unit) หรือโหมดออฟไลน์ (เมื่อไม่มี Commerce Scale Unit หรือการเชื่อมต่อเครือข่าย และธุรกรรมจะถูกเก็บในฐานข้อมูลท้องถิ่นแบบออฟไลน์) เมื่อ MPOS รันในโหมดออฟไลน์ จะไม่สามารถเรียกรูปภาพจากเซิร์ฟเวอร์รูปภาพภายนอกเพื่อแสดงจาก Commerce Scale Unit ได้ เนื่องจากการเชื่อมต่อสูญหาย อย่างไรก็ตาม คุณยังคงสามารถตั้งค่ารูปภาพเพื่อที่จะถูกแสดงเมื่อ MPOS รันในโหมดออฟไลน์ได้

### <a name="set-up-product-images-to-appear-in-offline-mode-for-mpos"></a>ตั้งค่ารูปภาพผลิตภัณฑ์เพื่อที่จะปรากฏขึ้นในโหมดออฟไลน์สำหรับ MPOS

รูปภาพผลิตภัณฑ์ที่ต้องถูกใช้ในโหมดออฟไลน์สามารถตั้งค่าได้โดยการอัพโหลดรูปภาพทางกายภาพที่จำเป็นไปยังรูปภาพผลิตภัณฑ์พื้นฐาน

1. คลิก **การจัดการข้อมูลผลิตภัณฑ์** &gt; **ผลิตภัณฑ์** &gt; **ผลิตภัณฑ์**
2. เลือกผลิตภัณฑ์เพื่อตั้งค่ารูปภาพสำหรับข้อมูลออฟไลน์
3. คลิก **แก้ไข** และจากนั้นคลิกลูกศรในมุมด้านขวาเพื่อแสดงบานหน้าต่างด้านขวา
4. ในแท็บด่วน **รูปภาพผลิตภัณฑ์** คลิก **เปลี่ยนรูป** และอัพโหลดรูปภาพทางกายภาพเพื่อที่จะใช้สำหรับผลิตภัณฑ์ที่ถูกเลือกอยู่ในโหมดออฟไลน์
5. บันทึกและปิดหน้า
6. ขณะที่ MPOS อยู่ในโหมดออนไลน์ รันงานแค็ตตาล็อกแบบ HQ เพื่อให้แน่ใจว่าข้อมูลจะถูกส่งอย่างน้อยหนึ่งครั้งไปยังฐานข้อมูลแบบออฟไลน์
7. ตั้ง MPOS เป็นโหมดออฟไลน์ คุณควรเห็นรูปภาพที่คุณได้อัพโหลดสำหรับผลิตภัณฑ์เฉพาะแบบ HQ

    [![รูปภาพของผลิตภัณฑ์ในโหมดออฟไลน์](./media/offline1.png)](./media/offline1.png)

### <a name="set-up-catalog-category-employee-and-customer-images-to-appear-in-offline-mode-for-mpos"></a>ตั้งค่าแค็ตตาล็อก ประเภท และ พนักงานลูกค้ารูปเพื่อที่จะปรากฏขึ้นในโหมดออฟไลน์สำหรับ MPOS

แค็ตตาล็อก ประเภท พนักงาน และรูปภาพของลูกค้าที่ต้องถูกใช้ในโหมดออฟไลน์สามารถตั้งค่าโดยการเพิ่มลิงค์ปลายทางของรูปภาพจำเป็นไปยังแกลเลอรี และตั้งค่ารูปภาพเป็นรูปภาพเริ่มต้นสำหรับเอนทิตี้ที่ถูกเลือก

1. ไปยังแค็ตตาล็อก และจากนั้น บนบานหน้าต่างการดำเนินการ คลิก **สื่อ** &gt; **รูปภาพ**
2. ทำตามขั้นตอนในส่วน [บันทึกทับจากหน้าการแสดงตัวอย่างระดับเอนทิตี](#overwrite-from-the-entity-level-preview-page) เพื่อที่จะเพิ่ม URL ของรูปภาพภายนอก
3. ทำเครื่องหมายรูปภาพนี้เป็นรูปภาพเริ่มต้นสำหรับแค็ตตาล็อกโดยการเลือกกล่องกาเครื่องหมายสำหรับรูปภาพที่ถูกแสดงรายการอยู่ในกริด
4. รันงานแค็ตตาล็อก ขณะนี้รูปภาพนี้จะถูกใช้เป็นรูปภาพแบบออฟไลน์สำหรับแค็ตตาล็อกนั้นใน MPOS
5. ทำตามกระบวนการที่คล้ายกันสำหรับเอนทิตีอื่น เช่น ประเภท พนักงาน และลูกค้า

    [![รูปภาพแบบออฟไลน์](./media/offline2.png)](./media/offline2.png)
