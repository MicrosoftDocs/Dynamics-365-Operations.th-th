### <a name="customer-creation-in-pos-offline-mode"></a><span data-ttu-id="e7c21-154">การสร้างลูกค้าในโหมดออฟไลน์ของ POS</span><span class="sxs-lookup"><span data-stu-id="e7c21-154">Customer creation in POS offline mode</span></span>

<span data-ttu-id="e7c21-155">ถ้า POS ออฟไลน์ในขณะที่เปิดใช้งานโหมดการสร้างลูกค้า Async เรกคอร์ดของลูกค้าใหม่จะถูกสร้างขึ้นแบบอะซิงโครนัส</span><span class="sxs-lookup"><span data-stu-id="e7c21-155">If the POS goes offline while the Async customer creation mode is enabled, new customer records are created asynchronously.</span></span> <span data-ttu-id="e7c21-156">ถ้า POS ออฟไลน์ในขณะที่ปิดใช้งานโหมดการสร้างลูกค้า Async ระบบจะสลับไปยังโหมดการสร้างลูกค้า Async โดยอัตโนมัติ</span><span class="sxs-lookup"><span data-stu-id="e7c21-156">If the POS goes offline while the Async customer creation mode is disabled, the system automatically switches to the Async customer creation mode.</span></span> <span data-ttu-id="e7c21-157">หรือกล่าวได้ว่า การสร้างเรกคอร์ดของลูกค้าอาจเกิดขึ้นแบบอะซิงโครนัสแม้ว่าโหมดการสร้างลูกค้า Async จะถูกปิด</span><span class="sxs-lookup"><span data-stu-id="e7c21-157">In other words, customer records might be created asynchronously even if the Async customer creation mode is disabled.</span></span> <span data-ttu-id="e7c21-158">ดังนั้น ผู้ดูแลระบบศูนย์ควบคุม Commerce จึงต้องสร้างและจัดตารางชุดงานประจำให้กับงาน P งาน **ซิงโครไนส์ลูกค้าและคู่ค้าธุรกิจจากงานโหมด async** และงาน **1010** เพื่อให้ลูกค้า Async ทุกรายถูกแปลงเป็นลูกค้า Sync ในศูนย์ควบคุม Commerce</span><span class="sxs-lookup"><span data-stu-id="e7c21-158">Therefore, Commerce headquarters administrators must create and schedule a recurring batch job for the P-job, the **Synchronize customers and business partners from async mode** job, and the **1010** job, so that any Async customers are converted to Sync customers in Commerce headquarters.</span></span>
