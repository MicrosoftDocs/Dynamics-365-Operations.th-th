# <a name="configure-service-to-service-authentication"></a>ตั้งค่าคอนฟิกการรับรองความถูกต้องแบบบริการกับบริการ

[!include [banner](includes/banner.md)]

ในบทความนี้จะอธิบายวิธีตั้งค่าคอนฟิกการพิสูจน์ตัวจริงแบบบริการ (S2S) ใน Microsoft Dynamics 365 Commerce เพื่อเรียก Application Programming Interface (API) การบริการเพื่อการจัดอันดับและบทวิจารณ์อย่างปลอดภัย

Dynamics 365 Commerce เสนอ [การให้คะแนนและบทวิจารณ์](ratings-reviews-overview.md) เป็นโซลูชันช่องทาง Omni โซลูชันนี้ช่วยให้สามารถเข้าถึง API บริการจากภายนอก Commerce เพื่อที่จะสามารถปฏิบัติงานต่างๆ ได้ งานเหล่านี้รวมถึงการนําเข้าการให้คะแนนและบทวิจารณ์จากระบบภายนอกของคุณไปยัง Commerce และการส่งออกการให้คะแนนและบทวิจารณ์จาก Commerce เพื่อให้ Commerce สามารถเรียกการให้คะแนนและบทวิจารณ์ของ API บริการได้ปลอดภัย คุณต้องตั้งค่าคอนฟิกการรับรองความถูกต้อง S2S ก่อน โดยปฏิบัติตามขั้นตอนต่างๆ ในบทความนี้ให้เสร็จสมบูรณ์

## <a name="add-a-new-app-registration"></a>เพิ่มการลงทะเบียนแอปใหม่

ก่อนที่คุณจะเพิ่มการลงทะเบียนแอปใหม่ คุณต้องสร้างแอปพลิเคชันโดยใช้ [พอร์ทัล Azure](https://portal.azure.com) หากต้องการลงทะเบียนแอปใน Azure Active Directory (Azure AD) และเปิดใช้งานการรับรองความถูกต้อง ให้ปฏิบัติตามขั้นตอนต่างๆ ใน [ใช้ Azure Active Directory กับตัวเชื่อมต่อที่กำหนดเองใน Power Automate](/connectors/custom-connectors/azure-active-directory-authentication)

รวบรวมรหัสต่อไปนี้จากพอร์ทัล Azure คุณจะต้องใช้รหัสเหล่านี้ในขั้นตอนที่ตามมา

- รหัสแอปพลิเคชันไคลเอนต์
- รหัสไดเรกทอรีไคลเอ็นต์ (ผู้เช่า)

เมื่อต้องการเพิ่มการลงทะเบียนแอปใหม่ในโปรแกรมสร้างไซต์ Commerce ให้ทำตามขั้นตอนต่อไปนี้

1. ไปที่ **หน้าแรก \> ความคิดเห็น \> การตั้งค่า**
1. ภายใต้ **การรับรองความถูกต้องแบบบริการกับบริการ (S2S)** ให้เลือก **จัดการ**

    ![ปุ่ม จัดการ ในส่วนการรับรองความถูกต้องแบบบริการกับบริการ (S2S) ในโปรแกรมสร้างไซต์ Commerce](media/Ratings-reviews-settings-service-to-service-authentication.png)

1. ในบานหน้าต่าง **รายการแอป S2S** ที่จะปรากฏขึ้นทางด้านขวา ให้เลือก **เพิ่มการลงทะเบียนแอป S2S ใหม่**
1. ในกล่องโต้ตอบ **เพิ่มรายการแอป S2S** ให้ป้อนข้อมูลที่จำเป็นดังต่อไปนี้ ใช้ค่าจากการลงทะเบียนแอปพลิเคชัน Azure ของคุณ

    - **ชื่อ** –ป้อนชื่อแอปพลิเคชันของคุณ (เช่น **แอป Fabrikam**)
    - **รหัสแอปไคลเอ็นต์** – ป้อนรหัสแอปพลิเคชัน (เช่น **00000000-0000-0000-0000-000000000000**)
    - **รหัสไดเรกทอรี (ผู้เช่า)** – ป้อนรหัสไดเรกทอรี (เช่น **00000000-0000-0000-0000-000000000000**)

    ![กล่องโต้ตอบเพิ่มรายการแอป S2S ในโปรแกรมสร้างไซต์ Commerce](media/Ratings-reviews-settings-S2S-APP-entry.png)

1. เลือก **ส่ง** ชื่อแอปพลิเคชันของคุณควรปรากฏในรายการในบานหน้าต่าง **รายการแอป S2S**
1. ปิดบานหน้าต่าง **รายการแอป S2S**
1. เลือก **บันทึก**

## <a name="edit-an-existing-app-registration"></a>แก้ไขการลงทะเบียนแอปที่มีอยู่

เมื่อต้องการแก้ไขการลงทะเบียนแอปที่มีอยู่ในโปรแกรมสร้างไซต์ Commerce ให้ทำตามขั้นตอนต่อไปนี้

1. ไปที่ **หน้าแรก \> ความคิดเห็น \> การตั้งค่า**
1. ภายใต้ **การรับรองความถูกต้องแบบบริการกับบริการ (S2S)** ให้เลือก **จัดการ**
1. ในบานหน้าต่าง **รายการแอป S2S** ให้เลือกสัญลักษณ์ดินสอที่อยู่ถัดจากรายการที่คุณต้องการแก้ไข
1. อัปเดตค่าในฟิลด์ **ชื่อ**, **รหัสแอปไคลเอ็นต์** และ **รหัสไดเรกทอรี (ผู้เช่า)** ตามที่ต้องการ
1. เลือก **ส่ง**
1. ปิดบานหน้าต่าง **รายการแอป S2S**
1. เลือก **บันทึก**

## <a name="remove-an-existing-app-registration"></a>เอาการลงทะเบียนแอปที่มีอยู่ออก

หากเอาการลงทะเบียนแอปที่มีอยู่ออกในโปรแกรมสร้างไซต์ Commerce ให้ทำตามขั้นตอนเหล่านี้

1. ไปที่ **หน้าแรก \> ความคิดเห็น \> การตั้งค่า**
1. ภายใต้ **การรับรองความถูกต้องแบบบริการกับบริการ (S2S)** ให้เลือก **จัดการ**
1. ในบานหน้าต่าง **รายการแอป S2S** ให้เลือกสัญลักษณ์ถังขยะที่อยู่ถัดจากรายการที่คุณต้องการเอาออก รายการดังกล่าวจะถูกเอาออกจากรายการ
1. ปิดบานหน้าต่าง **รายการแอป S2S**
1. เลือก **บันทึก**
