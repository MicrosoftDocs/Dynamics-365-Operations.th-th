# <a name="tender-based-discount"></a>ส่วนลดตามการชำระเงิน

[!include [banner](includes/banner.md)]

บทความนี้อธิบายฟังก์ชันส่วนลดตามวิธีการชำระเงิน ที่ให้ผู้ค้าปลีกสามารถตั้งค่าคอนฟิกส่วนลดสำหรับชนิดการชำระเงินเฉพาะใน Microsoft Dynamics 365 Commerce

เป็นการปฏิบัติโดยทั่วไปของผู้ค้าปลีกที่จะนำออกใช้บัตรเครดิตที่มีตราสินค้าส่วนตัว ผู้ค้าปลีกได้รับผลประโยชน์ เนื่องจากพวกเขาได้รับอัตราที่ต้องการจากธนาคาร นอกจากนี้ เนื่องจากบัตรเครดิตเหล่านี้จะสนับสนุนให้ลูกค้าแวะไปที่ร้านค้าบ่อยครั้ง ซึ่งจะช่วยปรับปรุงกำไรสุทธิของผู้จัดจำหน่ายได้ ดังนั้น ผู้ค้าปลีกมีแรงจูงใจที่จะเพิ่มการใช้งานของบัตรเครดิตที่มีตราสินค้าของตนของลูกค้า เพื่อให้บรรลุเป้าหมายนี้ บ่อยครั้งที่พวกเขาให้ส่วนลดเพิ่มเติมแก่ลูกค้าที่ใช้บัตรเครดิตเหล่านี้

อีกทางหนึ่งคือ ผู้ค้าปลีกที่ไม่มีบัตรเครดิตที่มีตราสินค้าอาจต้องการส่งเสริมให้ลูกค้าชำระเงินโดยใช้ชนิดการชำระเงินอื่นๆ เช่น เงินสด บัตรของขวัญ หรือคะแนนสะสมสำหรับสมาชิก ด้วยวิธีนี้ คุณสามารถช่วยลดค่าใช้จ่ายของค่าธรรมเนียมการประมวลผลบัตรเครดิตได้ ดังนั้น ผู้ค้าปลีกอาจให้ส่วนลดแก่ลูกค้าที่ใช้ชนิดการชำระเงินอื่นเหล่านี้

## <a name="tender-based-discount"></a>ส่วนลดตามการชำระเงิน

Commerce สนับสนุน *ส่วนลดตามการชำระเงิน* ที่ผู้ค้าปลีกตั้งค่าคอนฟิกเปอร์เซ็นต์ส่วนลดที่ใช้กับธุรกรรม ถ้าผลรวมธุรกรรมเกินกว่าจํานวนที่ระบุ และลูกค้าจ่ายโดยใช้ชนิดการชำระเงินที่ต้องการ ส่วนลดตามการชำระเงินเป็นส่วนลดตามระดับ เพื่อที่เปอร์เซ็นต์ของส่วนลดที่แตกต่างกันสามารถเชื่อมโยงกับขีดจำกัดยอดเงินที่แตกต่างกัน ลูกค้าสามารถตัดสินใจได้ว่าจะทำการชำระเงินบางส่วนหรือชำระเงินเต็มจำนวน และกลไกการกำหนดราคาจะกำหนดยอดส่วนลดที่เหมาะสม ส่วนลดตามการชำระเงินจะถูกระบุในยอดก่อนภาษีของบรรทัดการขายเสมอ

ส่วนลดตามการชำระเงินสามารถใช้เฉพาะกับบรรทัดขายที่ราคาไม่ได้ถูกล็อคไว้ และสัดส่วนของส่วนลดจะกระจายไปยังแต่ละรายการ ถ้ามีการเพิ่มรายการขายใหม่ในใบสั่ง ส่วนลดตามการชำระเงินจะใช้กับรายการขายใหม่เท่านั้น ในระหว่างการชำระเงิน เมื่อใบสั่งของลูกค้าสำหรับการเบิกสินค้าหรือการจัดส่งสินค้าถูกวาง ส่วนลดตามการชำระเงินจะใช้กับยอดเงินฝากเท่านั้น หลังจากที่วางใบสั่งแล้ว ในระหว่างการเติมสินค้า ราคาของรายการขายจะถูกล็อค ไม่มีการใช้ส่วนลดตามการชำระเงินกับยอดดุลใดๆ ที่ถูกชำระในระหว่างการเบิกสินค้า หรือได้รับอนุญาตในระหว่างการจัดส่ง ส่วนลดที่ใช้ในการชำระเงินสามารถใช้กับยอดเงินทั้งหมดของใบสั่งของลูกค้าได้ เฉพาะเมื่อผู้ค้าปลีกเก็บยอดเงินทั้งหมดเป็นเงินฝาก ในขณะที่มีการวางใบสั่ง

ส่วนลดตามการชำระเงินไม่แข่งขันกับส่วนลดตามสินค้า เช่น ส่วนลดแบบง่าย ส่วนลดปริมาณ ส่วนลดขีดควบคุม ส่วนลดการซื้อคละกัน และส่วนลดที่ป้อนเอง ส่วนลดตามการชำระเงินจะถูกรวมโดยผ่านส่วนลดตามสินค้าเสมอ ดังนั้น แม้ว่าจะมีการใช้ส่วนลดพิเศษกับสินค้า ส่วนลดตามการชำระเงินสามารถใช้เพิ่มเติมจากส่วนลดพิเศษได้ ในทำนองเดียวกัน ถ้ามีการใช้ส่วนลดขีดจำกัดกับธุรกรรม และส่วนลดที่ใช้ในการชำระเงินจะลดยอดรวมต่ำกว่าขีดจำกัด ส่วนลดขีดจำกัดจะยังคงถูกนำไปใช้กับธุรกรรม

ถึงแม้ว่าส่วนลดตามการชำระเงินจะลดผลรวมย่อยของธุรกรรม ค่าธรรมเนียมอัตโนมัติที่ใช้กับธุรกรรมจะไม่ได้รับผลกระทบ ตัวอย่างเช่น ถ้ามีการคำนวณค่าธรรมเนียมการจัดส่งเป็น $5 เนื่องจากมีผลรวมย่อยเกินกว่า $100 และส่วนลดที่ใช้ในการชำระเงินจะลดยอดเงินดังกล่าวเพื่อให้น้อยกว่า $100 ค่าธรรมเนียมการจัดส่งจะยังคงเป็น $5 สำหรับใบสั่ง

ส่วนลดตามการชำระเงินในขณะนี้ถูกจำกัดอยู่ที่การชำระเงินสองชนิด: บัตรเครดิต และเงินสด ถ้ามีการตั้งค่าคอนฟิกส่วนลดตามการชำระเงินสำหรับชนิดการชำระเงิน จะมีการใช้ส่วนลดตามการชำระเงินที่ดีที่สุดเท่านั้น

## <a name="pos-user-experience"></a>ประสบการณ์ผู้ใช้ POS

ถ้ามีการตั้งค่าส่วนลดตามการชำระเงินสำหรับเงินสด และพนักงานเก็บเงินที่การขายหน้าร้าน (POS) เลือกปุ่ม **ชำระเงินสด** เพื่อดำเนินการธุรกรรมเงินสดและการขนส่ง ส่วนลดตามการชำระเงินจะถูกนำไปใช้กับธุรกรรมนี้โดยอัตโนมัติ จากนั้น ยอดเงินที่ลดลงจะแสดงเป็นยอดดุล อย่างไรก็ตาม ถ้าพนักงานเก็บเงินเลือกปุ่ม **ย้อนกลับ** บนหน้าจอการชำระเงิน ส่วนลดจะถูกลบออก และจะมีการแสดงยอดเงินดั้งเดิมบนหน้าจอธุรกรรม ส่วนลดที่ใช้ในการชำระเงินจะถูกลบออก ถ้ารายการชำระเงินถูกยกเลิก

สำหรับการชำระเงินด้วยบัตรเครดิต ผู้ค้าปลีกสามารถตั้งค่าส่วนลดที่ใช้ในการชำระเงินในบัตรเครดิตหนึ่งชนิดขึ้นไป อย่างไรก็ตาม ระบบไม่สามารถตรวจสอบความถูกต้องของชนิดของบัตรเครดิตที่ใช้งานได้ เว้นแต่จะมีการอนุมัติบัตร ถ้ามีการใช้ส่วนลดหลังจากการตรวจสอบ การตรวจสอบการชำระเงินจะมีไว้สำหรับจำนวนเงินที่ใหญ่กว่า แต่การบันทึกการชำระเงินมีไว้สำหรับจำนวนเงินที่น้อยกว่า เพื่อช่วยป้องกันสถานการณ์ดังกล่าว ถ้าลูกค้าชำระเงินด้วยบัตรเครดิต พนักงานเก็บเงินจะถูกแจ้งเตือนโดยกล่องโต้ตอบที่แสดงรายการบัตรเครดิตที่ต้องการใดๆ ที่จะให้ส่วนลดเพิ่มเติมแก่ลูกค้า จากนั้น พนักงานเก็บเงินสามารถถามได้ว่าลูกค้าต้องการใช้บัตรที่ต้องการอย่างใดอย่างหนึ่งเพื่อให้ได้รับส่วนลดเพิ่มเติมหรือไม่ ถ้าพนักงานเก็บเงินเลือกบัตรที่ต้องการ จะมีการใช้ส่วนลดที่ใช้ในการชำระเงินกับธุรกรรม และยอดเงินที่ลดลงจะแสดงอยู่บนหน้าจอการชำระเงิน การตรวจสอบจะมีไว้สำหรับยอดเงินที่ลดลง ถ้าลูกค้าเสียบบัตรซึ่งแตกต่างจากบัตรที่พนักงานเก็บเงินเลือก ข้อความแสดงข้อผิดพลาดจะแสดงขึ้น และการตรวจสอบจะถูกยกเลิก

## <a name="call-center-user-experience"></a>ประสบการณ์การใช้งานของผู้ใช้ศูนย์บริการ

ถ้าผู้ใช้ศูนย์บริการเลือก **ดำเนินการให้เสร็จสมบูรณ์** ในระหว่างการสั่งซื้อของศูนย์บริการ หน้าจอ **ยอดรวม** จะแสดงขึ้น ในตอนแรก ผลรวมบนหน้าจอนี้จะไม่รวมส่วนลดที่ใช้ในการชำระเงิน เนื่องจากยังไม่ได้เลือกวิธีการชำระเงิน บนหน้าจอ **เพิ่มการชำระเงิน** ถ้าผู้ใช้เลือกวิธีการชำระเงินที่มีการตั้งค่าคอนฟิกส่วนลดที่ใช้ในการชำระเงิน จะมีการปรับปรุงยอดการชำระเงินโดยอัตโนมัติเพื่อให้สะท้อนถึงยอดส่วนลด เช่นเดียวกับลูกค้าที่ POS ลูกค้าของศูนย์บริการสามารถเลือกได้ว่าต้องการชำระเงินเต็มจำนวนหรือชำระเงินบางส่วน ตามยอดเงินที่จะมีการชำระ จะมีการใช้ส่วนลดที่ใช้ในการชำระเงินกับใบสั่งขาย

> [!NOTE]
> ไม่ได้ทำการตรวจสอบความถูกต้องของบัตรสำหรับใบสั่งของศูนย์บริการ ตัวอย่างเช่น ถ้าผู้ใช้ศูนย์บริการเลือก Visa เป็นบัตรเครดิต แต่ลูกค้าใช้ Mastercard ระบบก็ยังคงใช้ส่วนลด
