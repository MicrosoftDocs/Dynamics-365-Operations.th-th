# <a name="provision-a-dynamics-365-commerce-sandbox-environment"></a>เตรียมใช้งานสภาพแวดล้อม Sandbox ของ Dynamics 365 Commerce

[!include [banner](includes/banner.md)]

บทความนี้อธิบายวิธีการเตรียมใช้งานสภาพแวดล้อม Sandbox ของ Microsoft Dynamics 365 Commerce สำหรับการใช้งานในการสาธิตที่มีข้อมูลสาธิตในตัว กระบวนการในการตั้งค่าสภาพแวดล้อมการทำงานจริงจะคล้ายคลึงกันแต่จะอธิบายอย่างละเอียดมากขึ้นเนื่องจากข้อกำหนดเบื้องต้นหลายอย่างของสภาพแวดล้อม Sandbox มีให้อยู่แล้วในข้อมูลสาธิต

ก่อนที่คุณจะเริ่มต้น เราขอแนะนำให้คุณอ่านผ่านบทความนี้เพื่อดูว่ากระบวนการต้องใช้อะไร

หากต้องการเตรียมใช้งานสภาพแวดล้อม Sandbox ของ Commerce ให้สำเร็จ คุณต้องระบุพารามิเตอร์บางอย่างให้กับสภาพแวดล้อมและ Commerce Scale Unit (CSU) ที่จะใช้เมื่อคุณเตรียมใช้งาน Commerce ในภายหลัง คำแนะนำในบทความนี้อธิบายขั้นตอนที่จำเป็นทั้งหมดที่คุณต้องทำการเตรียมใช้งานให้เสร็จสมบูรณ์ และพารามิเตอร์ที่คุณต้องใช้

หลังจากที่คุณเตรียมใช้งานสภาพแวดล้อม Commerce ของคุณเสร็จสิ้น คุณต้องดำเนินการขั้นตอนหลังการเตรียมใช้งานสองสามขั้นตอนเพื่อจัดเตรียมสำหรับการใช้งาน ขั้นตอนบางอย่างอาจไม่จำเป็น ขึ้นอยู่กับแง่มุมของระบบที่คุณต้องการใช้ คุณสามารถดำเนินการตามขั้นตอนที่ไม่จำเป็นในภายหลังได้เสมอ

สำหรับข้อมูลเกี่ยวกับวิธีการตั้งค่าคอนฟิกสภาพแวดล้อม Commerce ของคุณ หลังจากที่คุณเตรียมใช้งาน ให้ดูที่ [ตั้งค่าคอนฟิกสภาพแวดล้อม Sandbox ของ Commerce](cpe-post-provisioning.md) สำหรับข้อมูลเกี่ยวกับวิธีการตั้งค่าคอนฟิกสภาพแวดล้อม Commerce ของคุณ หลังจากที่คุณเตรียมใช้งาน ให้ดูที่ [ตั้งค่าคอนฟิกคุณลักษณะเพิ่มเติมสำหรับสภาพแวดล้อม Sandbox ของ Commerce](cpe-optional-features.md)

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ต้องมีข้อกำหนดเบื้องต้นต่อไปนี้ก่อนที่คุณจะสามารถเตรียมใช้งานสภาพแวดล้อม Commerce ของคุณได้:

- คุณมีสิทธิ์เข้าถึงพอร์ทัล Microsoft Dynamics Lifecycle Services (LCS)
- คุณเป็นคู่ค้าหรือลูกค้า Microsoft Dynamics 365 เดิมและมีการสร้างโครงการสำหรับการใช้งานไว้อยู่แล้วและพร้อมใช้งานใน LCS  
- คุณได้สร้างกลุ่มความปลอดภัย Azure Active Directory (Azure AD) ที่สามารถใช้เป็นกลุ่มผู้ดูแลระบบ Commerce และคุณมีรหัสที่พร้อมใช้งาน
- คุณได้สร้างกลุ่มความปลอดภัย Azure AD ที่สามารถใช้เป็นกลุ่มผู้ดูแลการให้คะแนนและบทวิจารณ์ และคุณมีรหัสที่พร้อมใช้งาน (กลุ่มความปลอดภัยนี้อาจเหมือนกับกลุ่มผู้ดูแลระบบ Commerce)
- คุณได้ปรับใช้อินสแตนซ์ศูนย์ควบคุมภายใน LCS สำหรับข้อมูลเพิ่มเติม โปรดดู [ปรับใช้สภาพแวดล้อมใหม่](/dynamics365/fin-ops-core/dev-itpro/deployment/deployenvironment-newinfrastructure)

โปรดทราบว่ารายการนี้ยังไม่ครอบคลุมทั้งหมด หากคุณพบปัญหาใดๆ ให้ติดต่อคู่ค้า Microsoft ของคุณสำหรับความช่วยเหลือ

## <a name="provision-your-commerce-environment"></a>เตรียมใช้งานสภาพแวดล้อม Commerce ของคุณ

กระบวนงานต่อไปนี้จะอธิบายวิธีการเตรียมใช้งานสภาพแวดล้อม Commerce หลังจากที่คุณดำเนินการตามขั้นตอนเสร็จสมบูรณ์แล้ว สภาพแวดล้อม Commerce ของคุณจะพร้อมสำหรับการตั้งค่าคอนฟิก ขั้นตอนทั้งหมดที่อธิบายไว้ที่นี่จะเกิดขึ้นในพอร์ทัล LCS

### <a name="initialize-the-commerce-scale-unit-cloud"></a>เริ่มต้น Commerce Scale Unit (ระบบคลาวด์)

เมื่อต้องการเริ่มต้น CSU ให้ทำตามขั้นตอนเหล่านี้

1. ใน LCS ให้เลือกสภาพแวดล้อมของคุณจากรายการ
1. ในมุมมอง **สภาพแวดล้อม** ทางด้านขวา ให้เลือก **รายละเอียดแบบเต็ม** มุมมองรายละเอียดของสภาพแวดล้อมจะปรากฏขึ้น
1. ในส่วน **จัดการสภาพแวดล้อม** ภายใต้ **คุณลักษณะของสภาพแวดล้อม** ให้เลือก **จัดการ**
1. บนแท็บ **Commerce Scale Unit** ให้เลือก **เริ่มต้น** มุมมอง **เพิ่มสเกลยูนิต** จะปรากฏขึ้น
1. ในฟิลด์ **ภูมิภาค** ให้เลือกภูมิภาคที่เหมือนกันหรือใกล้เคียงกับภูมิภาคที่คุณปรับใช้สภาพแวดล้อม
1. ในรายการแบบหล่นลง **รุ่น** ให้เลือกรุ่นล่าสุดที่มีอยู่
1. เลือก **เริ่มต้น**
1. ในกล่องโต้ตอบคําเตือนที่จะขอให้คุณยืนยันการเริ่มต้น Commerce Scale Unit ให้เลือก **ใช่** ตอนนี้ CSU ของคุณได้รับการจัดคิวสำหรับการเตรียมใช้งาน
1. ก่อนที่คุณจะดำเนินการต่อ ตรวจสอบให้แน่ใจว่าสถานะของ CSU ของคุณคือ **สำเร็จ** การเริ่มต้นใช้งานจะใช้เวลาประมาณสองถึงห้าชั่วโมง

ถ้าคุณไม่พบลิงก์ **จัดการ** ในมุมมองรายละเอียดของสภาพแวดล้อม โปรดติดต่อผู้ติดต่อ Microsoft ของคุณสำหรับความช่วยเหลือ

### <a name="initialize-e-commerce"></a>ตั้งค่าเพื่อเริ่มต้นใช้งานอีคอมเมิร์ซ

เมื่อต้องการเริ่มต้น Commerce ให้ทำตามขั้นตอนเหล่านี้

1. บนแท็บ **อีคอมเมิร์ซ** ให้เลือก **ตั้งค่า**
1. ในฟิลด์ **ชื่อสภาพแวดล้อมอีคอมเมิร์ซ** ป้อนชื่อ โปรดทราบว่าชื่อนี้จะปรากฏในบาง URL ที่ชี้ไปยังอินสแตนซ์อีคอมเมิร์ซของคุณ
1. ในฟิลด์ **ชื่อ Commerce Scale Unit** เลือก CSU ของคุณในรายการ (รายการควรมีเพียงตัวเลือกเดียว)

    ฟิลด์ **ภูมิศาสตร์ของอีคอมเมิร์ซ** ถูกตั้งค่าโดยอัตโนมัติ

1. เลือก **ถัดไป** เพื่อดำเนินการต่อ
1. ในฟิลด์ **ชื่อโฮสต์ที่ได้รับการสนับสนุน** ให้ป้อนโดเมนที่ถูกต้องใดๆ เช่น `www.fabrikam.com`
1. ในฟิลด์ **กลุ่มรักษาความปลอดภัย AAD สำหรับผู้ดูแลระบบ** ให้ป้อนตัวอักษรสองสามตัวแรกของชื่อกลุ่มรักษาความปลอดภัยที่คุณต้องการใช้ แล้วจากนั้น เลือกสัญลักษณ์แว่นขยายเพื่อดูผลการค้นหา เลือกกลุ่มความปลอดภัยที่ถูกต้องในรายการ
1.  ในฟิลด์ **กลุ่มรักษาความปลอดภัย AAD สำหรับการให้คะแนนและรีวิวผู้ตรวจสอบ** ให้ป้อนตัวอักษรสองสามตัวแรกของชื่อกลุ่มรักษาความปลอดภัยที่คุณต้องการใช้ แล้วจากนั้น เลือกสัญลักษณ์แว่นขยายเพื่อดูผลการค้นหา เลือกกลุ่มความปลอดภัยที่ถูกต้องในรายการ
1. ปล่อยให้ตัวเลือก **เปิดใช้งานการให้คะแนนและรีวิวบริการ** ถูกตั้งค่าเป็น **ใช่**
1. เลือก **เริ่มต้นใช้งาน** มุมมอง **การจัดการ Commerce** จะแสดงอีกครั้งโดยที่แท็บ **อีคอมเมิร์ซ** จะถูกเลือก การเริ่มใช้งานอีคอมเมิร์ซได้เริ่มต้นขึ้นแล้ว
1. ก่อนที่คุณจะดำเนินการต่อไป ให้รอจนกว่าสถานะของการเริ่มต้น Commerce เป็น **การเริ่มต้นใช้งานสำเร็จ**
1. ภายใต้ **ลิงก์** ที่ด้านล่างขวา ให้จดบันทึก URL สำหรับลิงก์ต่อไปนี้:

    * **ไซต์อีคอมเมิร์ซ** – การเชื่อมโยงไปยังรากของไซต์อีคอมเมิร์ซของคุณ
    * **ตัวสร้างไซต์ Commerce** – การเชื่อมโยงไปยังเครื่องมือการจัดการไซต์
