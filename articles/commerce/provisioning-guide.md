# <a name="provision-a-commerce-preview-environment"></a>เตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายวิธีการเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Microsoft Dynamics 365 Commerce

ก่อนที่คุณจะเริ่ม อย่างน้อยที่สุดเราขอแนะนำให้คุณอ่านหัวข้อทั้งหมดนี้อย่างผ่านๆ เพื่อให้เข้าใจแนวคิดเกี่ยวกับกระบวนการที่เกี่ยวข้องและหัวข้อนี้มีอะไรบ้าง

> [!NOTE]
> ถ้าคุณยังไม่ได้รับอนุญาตให้เข้าถึงการแสดงตัวอย่างของ Dynamics 365 Commerce คุณสามารถร้องขอการเข้าถึงการแสดงตัวอย่างได้จาก [เว็บไซต์ Commerce](https://aka.ms/Dynamics365CommerceWebsite)

## <a name="overview"></a>ภาพรวม

เพื่อเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณให้สำเร็จ คุณต้องสร้างโครงการที่มีชื่อและชนิดของผลิตภัณฑ์ที่เฉพาะเจาะจง สภาพแวดล้อมและ Retail Cloud Scale Unit (RCSU) ก็มีพารามิเตอร์เฉพาะบางอย่างที่คุณต้องใช้ เมื่อคุณเตรียมใช้งานอีคอมเมิร์ซในภายหลัง คำแนะนำในหัวข้อนี้อธิบายขั้นตอนที่จำเป็นทั้งหมดที่คุณต้องทำให้เสร็จสมบูรณ์และพารามิเตอร์ที่คุณต้องใช้

หลังจากที่คุณเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณเสร็จสิ้น คุณต้องดำเนินการขั้นตอนหลังการเตรียมใช้งานสองสามขั้นตอนเพื่อจัดเตรียม ขั้นตอนบางอย่างอาจไม่จำเป็น ขึ้นอยู่กับแง่มุมของระบบที่คุณต้องการประเมิน คุณสามารถดำเนินการตามขั้นตอนที่ไม่จำเป็นในภายหลังได้เสมอ

สำหรับข้อมูลเกี่ยวกับวิธีการตั้งค่าคอนฟิกสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณ หลังจากที่คุณเตรียมใช้งาน ให้ดูที่ [ตั้งค่าคอนฟิกสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce](cpe-post-provisioning.md) สำหรับข้อมูลเกี่ยวกับวิธีการตั้งค่าคอนฟิกสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณ หลังจากที่คุณเตรียมใช้งาน ให้ดูที่ [ตั้งค่าคอนฟิกคุณลักษณะเพิ่มเติมสำหรับสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce](cpe-optional-features.md)

หากคุณมีคำถามใดๆ เกี่ยวกับขั้นตอนการเตรียมใช้งาน หรือหากคุณพบปัญหาใดๆ โปรดแจ้งให้ Microsoft ทราบใน [กลุ่ม Microsoft Dynamics 365 Commerce Preview Yammer](https://aka.ms/Dynamics365CommercePreviewYammer)

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ต้องมีข้อกำหนดเบื้องต้นต่อไปนี้ก่อนที่คุณจะสามารถเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณได้:

- คุณมีสิทธิ์เข้าถึงพอร์ทัล Microsoft Dynamics Lifecycle Services (LCS)
- คุณได้รับการยอมรับในโปรแกรม Dynamics 365 Commerce Preview
- คุณมีสิทธิ์ที่จำเป็นในการสร้างโครงการสำหรับ **การเปิดจำหน่ายล่วงหน้าสำหรับผู้ที่มีแนวโน้มจะเป็นลูกค้า** หรือ **โยกย้าย สร้างวิธีแก้ไข และเรียนรู้**
- คุณเป็นส่วนหนึ่งของบทบาท **ผู้จัดการสภาพแวดล้อม** หรือ **เจ้าของโครงการ** ในโครงการที่คุณจะเตรียมใช้งานสภาพแวดล้อม
- คุณมีสิทธิ์เข้าถึงในฐานะผู้ดูแลระบบสำหรับการสมัครใช้งาน Microsoft Azure ของคุณ หรือคุณมีการติดต่อกับผู้ดูแลระบบการสมัครใช้งานที่สามารถดำเนินการขั้นตอนสองขั้นตอนที่ต้องการสิทธิ์ของผู้ดูแลระบบในนามของคุณให้เสร็จสมบูรณ์ได้
- คุณมีรหัสผู้เช่า Azure Active Directory (Azure AD) ของคุณที่พร้อมใช้งาน
- คุณได้สร้างกลุ่มรักษาความปลอดภัย Azure AD ที่สามารถใช้เป็นกลุ่มผู้ดูแลระบบอีคอมเมิร์ซ และคุณมีรหัสที่พร้อมใช้งาน
- คุณได้สร้างกลุ่มรักษาความปลอดภัย Azure AD ที่สามารถใช้เป็นกลุ่มผู้ดูแลการให้คะแนนและบทวิจารณ์ และคุณมีรหัสที่พร้อมใช้งาน (กลุ่มความปลอดภัยนี้อาจเหมือนกับกลุ่มผู้ดูแลระบบพาณิชย์อิเล็กทรอนิกส์)

### <a name="find-your-azure-ad-tenant-id"></a>ค้นหารหัสผู้เช่า Azure AD ของคุณ

รหัสผู้เช่า Azure AD ของคุณเป็น Global Unique Identifier (GUID) ที่คล้ายกับตัวอย่างนี้: **72f988bf-86f1-41af-91ab-2d7cd011db47**

#### <a name="find-your-azure-ad-tenant-id-by-using-the-azure-portal"></a>ค้นหารหัสผู้เช่า Azure AD ของคุณโดยใช้พอร์ทัล Azure

1. ลงชื่อเข้าใช้ [พอร์ทัล Azure](https://portal.azure.com/)
1. ตรวจสอบให้แน่ใจว่ามีการเลือกไดเรกทอรีที่ถูกต้อง
1. บนเมนูด้านซ้าย ให้เลือก **Azure Active Directory**
1. ภายใต้ **จัดการ** เลือก **คุณสมบัติ** รหัสผู้เช่า Azure AD ของคุณปรากฏภายใต้ **รหัสไดเรกทอรี**

#### <a name="find-your-azure-ad-tenant-id-by-using-openid-connect-metadata"></a>ค้นหารหัสผู้เช่า Azure AD ของคุณโดยการใช้ข้อมูลเมตา OpenID Connect

สร้าง URL ของ OpenID โดยการแทนที่ **\{YOUR\_DOMAIN\}** ด้วยโดเมนของคุณ เช่น `microsoft.com` ตัวอย่างเช่น `https://login.microsoftonline.com/{YOUR_DOMAIN}/.well-known/openid-configuration` จะกลายเป็น `https://login.microsoftonline.com/microsoft.com/.well-known/openid-configuration`

1. ไปที่ URL ของ OpenID ที่มีโดเมนของคุณ

    คุณสามารถค้นหารหัสผู้เช่า Azure AD ของคุณได้ในค่าคุณสมบัติหลายค่า

1. ค้นหา **authorization\_endpoint** และแยก GUID ที่ปรากฏทันทีหลังจาก `login.microsoftonline.com/`

### <a name="find-your-azure-ad-security-group-id"></a>ค้นหารหัสกลุ่มรักษาความปลอดภัย Azure AD ของคุณ

รหัสของกลุ่มความปลอดภัย Azure AD ของคุณคือ GUID ซึ่งคล้ายกับตัวอย่างนี้: **436ea7f5-ee6c-40c1-9f08-825c5811066a**

กระบวนงานนี้สันนิษฐานว่าคุณเป็นสมาชิกของกลุ่มที่คุณกำลังพยายามค้นหารหัสให้

1. เปิด [ตัวสำรวจกราฟ](https://developer.microsoft.com/graph/graph-explorer#)
1. เลือก **ลงชื่อเข้าใช้ด้วย Microsoft** และลงชื่อเข้าใช้โดยใช้ข้อมูลประจำตัวของคุณ
1. ที่ด้านซ้าย ให้เลือก**แสดงตัวอย่างเพิ่มเติม**
1. ในบานหน้าต่างด้านขวา เปิดใช้งาน **กลุ่ม**
1. ปิดบานหน้าต่างด้านขวา
1. เลือก **ทุกกลุ่มที่ฉันเป็นสมาชิกอยู่**
1. ในฟิลด์ **การแสดงตัวอย่างการตอบ** ให้ค้นหากลุ่มของคุณ รหัสกลุ่มความปลอดภัยจะปรากฏภายใต้คุณสมบัติ **รหัส**

## <a name="provision-your-commerce-preview-environment"></a>เตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ของคุณ

กระบวนงานเหล่านี้จะอธิบายวิธีการเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce หลังจากที่คุณดำเนินการเสร็จสมบูรณ์แล้ว สภาพแวดล้อมของการแสดงตัวอย่างของ Commerce จะพร้อมสำหรับการตั้งค่าคอนฟิก กิจกรรมทั้งหมดที่อธิบายไว้ที่นี่จะเกิดขึ้นในพอร์ทัล LCS

> [!IMPORTANT]
> การเข้าถึงการแสดงตัวอย่างจะเชื่อมโยงกับบัญชี LCS และองค์กรที่คุณระบุไว้ในแอพลิเคชันการแสดงตัวอย่างของคุณ คุณต้องใช้บัญชีเดียวกันเพื่อเตรียมใช้งานสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce ถ้าคุณต้องใช้บัญชี LCS หรือผู้เช่าที่แตกต่างกันสำหรับสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce คุณต้องให้รายละเอียดเหล่านั้นไปยัง Microsoft สำหรับข้อมูลผู้ติดต่อ โปรดดูส่วน [การสนับสนุนสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce](#commerce-preview-environment-support) ภายหลังในหัวข้อนี้

### <a name="grant-access-to-e-commerce-applications"></a>ให้สิทธิการเข้าถึงแอปพลิเคชันอีคอมเมิร์ซ

> [!IMPORTANT]
> บุคคลที่ลงชื่อเข้าใช้ต้องเป็นผู้ดูแลระบบผู้เช่า Azure AD ที่มีรหัสผู้เช่า Azure AD หากขั้นตอนนี้ไม่เสร็จสมบูรณ์ ขั้นตอนการเตรียมใช้งานที่เหลือจะล้มเหลว

ในการอนุมัติให้แอพลิเคชันอีคอมเมิร์ซเข้าถึงการสมัครใช้งาน Azure ของคุณ ให้ทำตามขั้นตอนเหล่านี้

1. ประกอบ URL ในรูปแบบต่อไปนี้:

    `https://login.windows.net/{AAD_TENANT_ID}/oauth2/authorize?client_id=fbcbf727-cd18-4422-a723-f8274075331a&response_type=code&redirect_uri=https://sb.manage.commerce.dynamics.com/_commerce/Consent&response_mode=query&prompt=admin_consent&state=12345`

1. คัดลอกและวาง URL ลงในเบราว์เซอร์หรือตัวแก้ไขข้อความของคุณ และแทนที่ **\{AAD\_TENANT\_ID\}** ด้วยรหัสผู้เช่า Azure AD ของคุณ จากนั้น เปิด URL
1. ในกล่องโต้ตอบการลงชื่อเข้าใช้ Azure AD ให้ลงชื่อเข้าใช้ และยืนยันว่าคุณต้องการให้สิทธิ์การเข้าถึง **Dynamics 365 Commerce (การแสดงตัวอย่าง)** ไปยังการสมัครใช้งานของคุณ คุณจะถูกส่งไปยังหน้าที่บ่งชี้ว่าการดำเนินงานนี้ประสบความสำเร็จหรือไม่

### <a name="confirm-that-preview-features-are-available-and-turned-on-in-lcs"></a>ยืนยันว่าคุณลักษณะการแสดงตัวอย่างพร้อมใช้งานและถูกเปิดใช้งานใน LCS

เพื่อยืนยันว่าคุณลักษณะการแสดงตัวอย่างพร้อมใช้งานและถูกเปิดใช้งานใน LCS ให้ทำตามขั้นตอนเหล่านี้

1. ลงชื่อเข้าใช้ [พอร์ทัล LCS](https://lcs.dynamics.com) โดยใช้บัญชี LCS เดียวกันกับที่คุณใช้ในการร้องขอการเข้าถึงไปยังการแสดงตัวอย่าง
1. ในโฮมเพจของ LCS ให้เลื่อนไปทางขวาสุด และเลือกไทล์ **การจัดการคุณลักษณะการแสดงตัวอย่าง**

    ![ดูตัวอย่างไทล์การจัดการ](./media/preview1.png)

1. เลื่อนลงไปที่ **คุณลักษณะการแสดงตัวอย่างส่วนตัว** และยืนยันว่าคุณลักษณะต่อไปนี้พร้อมใช้งานและถูกเปิดใช้งาน:

    - การประเมินอีคอมเมิร์ซ
    - สภาพแวดล้อมของโปรแกรมตัวอย่างของ Commerce

    ![คุณลักษณะแสดงตัวอย่างแบบส่วนตัว](./media/preview2.png)

1. ถ้าคุณลักษณะเหล่านั้นไม่ปรากฏในรายการ โปรดติดต่อ Microsoft และระบุที่อยู่อีเมลที่ทำงานของคุณ บัญชี LCS และรายละเอียดผู้เช่า สำหรับข้อมูลผู้ติดต่อ โปรดดูส่วน [การสนับสนุนสภาพแวดล้อมของการแสดงตัวอย่างของ Commerce](#commerce-preview-environment-support)

### <a name="create-a-new-project"></a>สร้างโครงการใหม่

เมื่อต้องการสร้างโครงการใหม่ใน LCS ให้ทำตามขั้นตอนเหล่านี้

1. บนโฮมเพจ LCS ให้เลือกเครื่องหมายบวก (**+**) เพื่อสร้างโครงการ
1. ในบานหน้าต่างด้านขวา ให้ทำตามหนึ่งในขั้นตอนเหล่านี้:

    - ถ้าคุณเป็นคู่ค้า ให้เลือก **โยกย้าย สร้างวิธีแก้ไข และเรียนรู้**
    - ถ้าคุณเป็นลูกค้า ให้เลือก **การเปิดจำหน่ายล่วงหน้าสำหรับผู้ที่มีแนวโน้มจะเป็นลูกค้า**

1. ป้อนชื่อ คำอธิบาย และอุตสาหกรรม
1. ในฟิลด์ **ชื่อผลิตภัณฑ์** ให้เลือก **Dynamics 365 Retail**
1. ในฟิลด์ **รุ่นผลิตภัณฑ์** ให้เลือก **Dynamics 365 Retail**

### <a name="initialize-rcsu"></a>ตั้งค่าเพื่อเริ่มใช้งาน RCSU

2.  ในฟิลด์ **กลุ่มความปลอดภัย AAD สำหรับการให้คะแนนและตรวจสอบผู้ตรวจสอบ** ให้ป้อนอักษรสองสามตัวแรกของชื่อของกลุ่มความปลอดภัยที่คุณต้องการใช้ เลือกไอคอนคลาสการขยายเพื่อแสดงผลลัพธ์การค้นหา เลือกกลุ่มความปลอดภัยจากรายการ
1. ปล่อยให้ตัวเลือก **เปิดใช้งานบริการการให้คะแนนและบทวิจารณ์** เปิด
1. ถ้าคุณได้ดำเนินการขั้นตอนความยินยอมของ Microsoft Azure Active Directory (Azure AD) เสร็จสิ้น ตามที่อธิบายไว้ในส่วน "ให้สิทธิ์การเข้าถึงไปยังแอพลิเคชันอีคอมเมิร์ซ" ให้เลือกกล่องกาเครื่องหมายเพื่อยืนยันความยินยอมของคุณ หากคุณยังไม่ได้ทำขั้นตอนนี้ให้เสร็จสมบูรณ์ คุณจำเป็นต้องดำเนินการดังกล่าวก่อนที่จะดำเนินการกับการเริ่มใช้งาน เลือกการเชื่อมโยงภายในข้อความที่อยู่ถัดจากกล่องกาเครื่องหมาย เพื่อเปิดกล่องโต้ตอบความยินยอมและทำขั้นตอนให้เสร็จสมบูรณ์
1. เลือก **เริ่มต้น** คุณจะถูกส่งกลับไปยังมุมมอง **การจัดการการขายปลีก** ที่ซึ่งมีการเลือกแท็บ **อีคอมเมิร์ซ (การแสดงตัวอย่าง)** การเริ่มใช้งานอีคอมเมิร์ซได้เริ่มต้นขึ้นแล้ว
1. ก่อนที่คุณจะดำเนินการต่อไป ให้รอจนกว่าสถานะของการเริ่มต้นใช้งานอีคอมเมิร์ซจะเป็น **การเริ่มต้นใช้งานสำเร็จ**
1. ภายใต้ **ลิงค์** ที่ด้านล่างขวา ให้จดบันทึก URL สำหรับลิงค์ต่อไปนี้:

    * **ไซต์อีคอมเมิร์ซ** – การเชื่อมโยงไปยังรากของไซต์อีคอมเมิร์ซของคุณ
    * **เครื่องมือการจัดการไซต์อีคอมเมิร์ซ** – การเชื่อมโยงไปยังเครื่องมือการจัดการไซต์

## <a name="commerce-preview-environment-support"></a>การสนับสนุนสภาพแวดล้อมการแสดงตัวอย่างของ Commerce

ถ้าคุณประสบปัญหาขณะที่คุณกำลังดำเนินการขั้นตอนการจัดเตรียมให้เสร็จสมบูรณ์ โปรดเยี่ยมชม [กลุ่ม Microsoft Dynamics 365 Commerce Preview Yammer](https://aka.ms/Dynamics365CommercePreviewYammer) สำหรับความช่วยเหลือ
