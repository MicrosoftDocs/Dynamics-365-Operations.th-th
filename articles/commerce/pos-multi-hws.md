# <a name="dedicated-payment-terminals-and-prompts-for-a-printer-and-cash-drawer"></a>เทอร์มินัลการชำระเงินเฉพาะและพร้อมต์สำหรับเครื่องพิมพ์และลิ้นชักเงินสด

[!include [banner](includes/banner.md)]

หัวข้อนี้จะให้ข้อมูลเกี่ยวกับความสามารถในการมีเทอร์มินัลการชำระเงินที่เฉพาะเจาะจงและพร้อมท์ให้ผู้ใช้เลือกลิ้นชักเงินสดและเครื่องพิมพ์ใบเสร็จ

## <a name="overview"></a>ภาพรวม

ร้านค้าปลีกสมัยใหม่กำลังค้นหาวิธีต่างๆ ในการปรับปรุงประสบการณ์การชำระเงินในร้านค้า แนวโน้มล่าสุดสำหรับการเช็คเอาท์แบบไร้กระดาษผ่านการชำระเงินทางอิเล็กทรอนิกส์ไม่เพียงแต่ช่วยในการทำให้ประสบการณ์การจัดซื้อราบรื่นขึ้น แต่ยังลดความจำเป็นในการใช้อุปกรณ์ต่อพ่วงที่ใช้งานสำหรับทุกการเชื่อมโยงของร้านค้า

Microsoft Dynamics 365 Commerce สนับสนุนแนวโน้มเหล่านี้ โดยการเปิดใช้งานสถานการณ์ที่อุปกรณ์การขายหน้าร้าน (POS) มีเทอร์มินัลการชำระเงินเฉพาะที่กำหนดให้เสมอ แต่ไม่มีเครื่องพิมพ์ใบเสร็จหรือลิ้นชักเงินสดของตนเอง เมื่อร้านค้าต้องพิมพ์ใบเสร็จหรือใช้การชำระเงินด้วยเงินสด ระบบจะขอให้คุณเลือกสถานีฮาร์ดแวร์ที่มีการตั้งค่าคอนฟิกอุปกรณ์เหล่านั้น

## <a name="key-terms"></a>เงื่อนไขสำคัญ

| เงื่อนไข | คำอธิบาย |
|---|---|
| การลงทะเบียน | เอนทิตีที่ใช้ในการตั้งค่าคอนฟิกอินสแตนซ์ของเครื่องบันทึกเงินสด POS |
| อุปกรณ์ | การแสดงตัวอย่างของอินสแตนซ์ที่มีอยู่จริงของเครื่องบันทึกเงินสด POS และแอปพลิเคชัน Modern POS ที่กำหนดให้ |
| สถานีฮาร์ดแวร์เฉพาะ | ตรรกะทางธุรกิจของสถานีฮาร์ดแวร์ถูกสร้างขึ้นใน Modern POS สำหรับ Windows และแอปพลิเคชัน Modern POS สำหรับ Android |
| พอร์ตดำเนินการของลิ้นชัก (d/k) | วิธีการแบบดั้งเดิมสำหรับการเชื่อมต่อลิ้นชักเงินสดเข้ากับเครื่องพิมพ์ใบเสร็จ |
| อุปกรณ์ต่อพ่วงของเครือข่าย | การสนับสนุนที่มีอยู่แล้วสำหรับเทอร์มินัลการชำระเงินที่เปิดใช้งานเครือข่าย เครื่องพิมพ์ใบเสร็จ และลิ้นชักเงินสด |

## <a name="supported-pos-clients-and-devices"></a>ไคลเอนต์และอุปกรณ์ POS ที่สนับสนุน

ฟังก์ชันที่อธิบายไว้ในหัวข้อนี้ได้รับการสนับสนุนโดย Modern POS สำหรับ Windows และ Modern POS สำหรับไคลเอนต์ Android POS

ฟังก์ชันนี้จะสนับสนุนเทอร์มินัลการชำระเงินที่เปิดใช้งานบนเครือข่ายและเครื่องพิมพ์ใบเสร็จ คุณสามารถให้การสนับสนุนลิ้นชักเงินสดได้ โดยการเชื่อมต่อลิ้นชักเงินสดเข้ากับเครื่องพิมพ์ใบเสร็จที่เปิดใช้งานบนเครือข่ายผ่านทางพอร์ต d/k

การสนับสนุนแบบสำเร็จรูปสำหรับฟังก์ชันนี้ได้รับการสนับสนุนโดย [ตัวเชื่อมต่อการชำระเงิน Dynamics 365 สำหรับ Adyen](https://docs.microsoft.com/dynamics365/commerce/dev-itpro/adyen-connector?tabs=8-1-3) อย่างไรก็ตาม การเชื่อมต่อการชำระเงินอื่นๆอาจได้รับการสนับสนุนผ่านชุดพัฒนาซอฟต์แวร์ Commerce (SDK) สำหรับการชำระเงิน เครื่องพิมพ์ใบเสร็จที่ได้รับการสนับสนุนรวมเครื่องพิมพ์ใบรเสร็จที่เปิดใช้งานบนเครือข่ายจาก Star Micronics และ Epson

หากต้องการตั้งค่าเครื่องพิมพ์ใบเสร็จของ Star Micronics ให้ใช้ยูทิลิตี้เครื่องพิมพ์ Star Micronics เพื่อตั้งค่าคอนฟิกอุปกรณ์ เพื่อให้สามารถใช้งานผ่านเครือข่ายได้ ยูทิลิตี้นี้ยังจะให้ที่อยู่ IP ของอุปกรณ์

เมื่อต้องการตั้งค่าเครื่องพิมพ์ใบเสร็จของ Epson ให้ใช้อยูทิลิตีของ Epson ePOS-Print เพื่อตั้งค่าอุปกรณ์เพื่อใช้โพรโทคอลเครือข่าย

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการตั้งค่าอุปกรณ์ต่อพ่วงของเครือข่าย ดูที่ [ภาพรวมของการสนับสนุนอุปกรณ์ต่อพ่วงของเครือข่าย](https://go.microsoft.com/fwlink/?linkid=2129965)

## <a name="set-up-a-dedicated-payment-terminal-and-a-prompt-for-a-printer-and-cash-drawer"></a>ตั้งค่าเทอร์มินัลการชำระเงินเฉพาะและพร้อมต์สำหรับเครื่องพิมพ์และลิ้นชักเงินสด

### <a name="set-up-hardware-profiles"></a>การตั้งค่าโพรไฟล์ฮาร์ดแวร์

คุณต้องมีโปรไฟล์ฮาร์ดแวร์สองชนิด ชนิดแรกกำหนดให้กับเครื่องบันทึกเงินสด ชนิดที่สองกำหนดให้กับสถานีฮาร์ดแวร์ในระดับร้านค้า และใช้ในการจัดกลุ่มเครื่องพิมพ์ใบเสร็จสำหรับเครือข่าย และลิ้นชักเงินสดในทางตรรกะ

#### <a name="set-up-a-hardware-profile-for-the-register"></a>ตั้งค่าโปรไฟล์ฮาร์ดแวร์สำหรับเครื่องบันทึกเงินสด

เมื่อต้องการตั้งค่าโปรไฟล์ฮาร์ดแวร์ที่กำหนดให้กับเครื่องบันทึกเงินสด ให้ทำตามขั้นตอนต่อไปนี้

1. ใน Dynamics 365 Commerce ให้ค้นหา **โปรไฟล์ฮาร์ดแวร์**
2. เลือก **ใหม่**
3. กำหนดหมายเลขโปรไฟล์ฮาร์ดแวร์ แล้วป้อนคำอธิบาย โปรไฟล์ฮาร์ดแวร์นี้จะถูกกำหนดให้กับเครื่องบันทึกเงินสดเอง ดังนั้น คำอธิบาย เช่น **เฉพาะกับการย้อนกลับ** จะพอเพียง
4. บนแท็บด่วนสำหรับชนิดอุปกรณ์ต่างๆ ให้ตั้งค่าชนิดอุปกรณ์ต่อไปนี้

    | อุปกรณ์ | ชนิดข้อมูล | ชื่ออุปกรณ์ | รายละเอียดเพิ่มเติม |
    |---|---|---|---|
    | เครื่องพิมพ์ | ย้อนกลับ | *ใดๆ* | ชื่ออุปกรณ์ต้องตรงตัวพิมพ์ **รหัสโปรไฟล์ใบเสร็จ** ควรเป็นรหัสเดียวกันกับ **รหัสโปรไฟล์ใบเสร็จ** ที่ถูกแม็ปกับเครื่องพิมพ์บนเครือข่ายที่ตั้งค่าไว้ในโปรไฟล์ฮาร์ดแวร์ ที่มีการกำหนดให้กับสถานีฮาร์ดแวร์ที่ระดับช่องทาง |
    | ลิ้นชักเงินสด | ย้อนกลับ | *ใดๆ* | ชื่ออุปกรณ์ต้องตรงตัวพิมพ์ ตั้งค่าตัวเลือก **ใช้กะที่ใช้ร่วมกัน** เป็น **ใช่** |
    | บริการ EFT | Adyen | ใช้ไม่ได้ | สำหรับข้อมูลเกี่ยวกับวิธีการตั้งค่าตัวเชื่อมต่อ Adyen สำเร็จรูป ดูที่ [ตัวเชื่อมต่อการชำระเงิน Dynamics 365 สำหรับ Adyen](https://docs.microsoft.com/dynamics365/commerce/dev-itpro/adyen-connector?tabs=8-1-3) การเชื่อมต่อการชำระเงินอื่นๆ สามารถได้รับการสนับสนุนผ่าน [ชุดพัฒนาซอฟต์แวร์ Commerce (SDK) สำหรับการชำระเงิน](https://docs.microsoft.com/dynamics365/commerce/dev-itpro/end-to-end-payment-extension) |
    | แป้นกดรหัส | เครือข่าย | **MicrosoftAdyenDeviceV001** | ไม่มี |

5. ใน Dynamics 365 Commerce ให้ค้นหา **เครื่องบันทึกเงินสด**
6. เลือกเครื่องบันทึกเงินสดโดยการเลือกหมายเลขเครื่องบันทึกเงินสด แล้วเลือก **แก้ไข**
7. กำหนดโปรไฟล์ฮาร์ดแวร์ที่คุณเพิ่งสร้างขึ้นเพื่อเครื่องบันทึกเงินสด ซึ่งควรใช้เทอร์มินัลการชำระเงินเฉพาะ อุปกรณ์ที่ถูกแม็ปกับเครื่องบันทึกเงินสดนี้ต้องใช้ Modern POS สำหรับแอปพลิเคชัน Windows หรือ Modern POS สำหรับแอปพลิเคชัน Android
8. เลือก **บันทึก**
9. บนบานหน้าต่างการดำเนินการ บนแท็บ **เครื่องบันทึกเงินสด** เลือก **กำหนดที่อยู่ IP**
10. บนแท็บด่วน **แป้นกดรหัส** ให้ป้อนที่อยู่ IP ของเทอร์มินัลการชำระเงิน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเรียกใช้ที่อยู่ IP ของเทอร์มินัลการชำระเงินโดยใช้ตัวเชื่อมต่อ Adyen ดูที่ [ตัวเชื่อมต่อการชำระเงิน Dynamics 365 สำหรับ Adyen](https://docs.microsoft.com/dynamics365/commerce/dev-itpro/adyen-connector?tabs=8-1-3)
11. เลือก **บันทึก**

#### <a name="set-up-a-hardware-profile-for-the-receipt-printer-and-cash-drawer"></a>ตั้งค่าโปรไฟล์ฮาร์ดแวร์สำหรับเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด

เมื่อต้องการตั้งค่าโปรไฟล์ฮาร์ดแวร์ที่ใช้ในการจัดกลุ่มเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด ให้ทำตามขั้นตอนต่อไปนี้

1. ใน Dynamics 365 Commerce ให้ค้นหา **โปรไฟล์ฮาร์ดแวร์**
2. เลือก **ใหม่**
3. กำหนดหมายเลขโปรไฟล์ฮาร์ดแวร์ แล้วป้อนคำอธิบาย โปรไฟล์ฮาร์ดแวร์นี้จะใช้ในการจัดกลุ่มเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด ดังนั้น คำอธิบาย เช่น **เครื่องพิมพ์บนเครือข่ายและลิ้นชักเงินสด** จะพอเพียง
4. บนแท็บด่วนสำหรับชนิดอุปกรณ์ต่างๆ ให้ตั้งค่าชนิดอุปกรณ์ต่อไปนี้

    | อุปกรณ์ | ชนิดข้อมูล | คำอธิบาย | รายละเอียดเพิ่มเติม |
    |---|---|---|---|
    | เครื่องพิมพ์ | เครือข่าย | **Epson** หรือ **Star** | ชื่ออุปกรณ์ต้องตรงตัวพิมพ์ **รหัสโปรไฟล์ใบเสร็จ** ควรเป็นรหัสเดียวกันกับ **รหัสโปรไฟล์ใบเสร็จ** ที่ถูกแม็ปกับเครื่องพิมพ์ที่ตั้งค่าไว้ในโปรไฟล์ฮาร์ดแวร์ ที่มีการกำหนดให้กับเครื่องบันทึกเงินสด |
    | ลิ้นชักเงินสด | เครือข่าย | **Epson** หรือ **Star** | ชื่ออุปกรณ์ต้องตรงตัวพิมพ์ ตั้งค่าตัวเลือก **ใช้กะที่ใช้ร่วมกัน** เป็น **ใช่** |

5. เลือก **บันทึก**

### <a name="set-up-hardware-stations"></a>ตั้งค่าสถานีฮาร์ดแวร์

คุณต้องมีสถานีฮาร์ดแวร์สองสถานี สถานีแรกจะถูกแม็ปกับเครื่องบันทึกเงินสด สถานีที่สองจะเลือกเมื่อจำเป็น เมื่อใบเสร็จต้องพิมพ์ หรือต้องเปิดลิ้นชักเงินสด

#### <a name="register-a-hardware-station"></a>ลงทะเบียนสถานีฮาร์ดแวร์

1. ใน Dynamics 365 Commerce ให้ค้นหา **ร้านค้าทั้งหมด**
2. เลือกร้านค้าโดยเลือกค่า **รหัสช่องทางการขายปลีก** แล้วเลือก **แก้ไข**
3. บน FastTab **สถานีฮาร์ดแวร์** ให้เลือก **เพิ่ม**
4. ตั้งค่าฟิลด์ **ชนิดของสถานีฮาร์ดแวร์** เป็น **เฉพาะ**
5. ป้อนคำอธิบาย แต่ไม่ต้องตั้งค่าใดๆ สำหรับสถานีฮาร์ดแวร์นี้ สถานีฮาร์ดแวร์นี้จะมีการใช้สำหรับเครื่องบันทึกเงินสดตลอดเวลา 

#### <a name="set-up-a-hardware-station-for-the-receipt-printer-and-cash-drawer"></a>ตั้งค่าสถานีฮาร์ดแวร์สำหรับเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด

1. ใน Dynamics 365 Commerce ให้ค้นหา **ร้านค้าทั้งหมด**
2. เลือกร้านค้าโดยเลือกค่า **รหัสช่องทางการขายปลีก** แล้วเลือก **แก้ไข**
3. บน FastTab **สถานีฮาร์ดแวร์** ให้เลือก **เพิ่ม**
4. ตั้งค่าฟิลด์ **ชนิดของสถานีฮาร์ดแวร์** เป็น **เฉพาะ**
5. ป้อนคำอธิบาย สถานีฮาร์ดแวร์นี้จะใช้สำหรับเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด
6. ในฟิลด์ **โปรไฟล์ฮาร์ดแวร์** ให้เลือกโปรไฟล์ฮาร์ดแวร์ที่คุณสร้างไว้ก่อนหน้านี้สำหรับเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด
7. เลือก **บันทึก**
8. ในขณะที่สถานีฮาร์ดแวร์สำหรับเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสดยังคงถูกเลือกอยู่ ให้เลือก **ตั้งค่าคอนฟิกที่อยู่ IP**
9. รับที่อยู่ IP สำหรับเครื่องพิมพ์ และป้อนที่อยู่ IP สำหรับทั้งเครื่องพิมพ์ใบเสร็จและลิ้นชักเงินสด
10. เลือก **บันทึก**
11. ค้นหา **กำหนดการกระจาย**
12. เลือกกำหนดการกระจาย **1090** แล้วเลือก **เรียกใช้เดี๋ยวนี้**
13. เลือกกำหนดการกระจาย **1070** แล้วเลือก **เรียกใช้เดี๋ยวนี้**

### <a name="set-up-the-system-to-prompt-for-receipt-printer-and-cash-drawer-selection-as-its-required"></a>ตั้งค่าระบบเพื่อแสดงพร้อมท์สำหรับเครื่องพิมพ์ใบเสร็จและการเลือกลิ้นชักเงินสดตามต้องการ

1. ในไคลเอนต์ POS ที่ได้รับการสนับสนุน ให้ปิดกะปัจจุบัน ถ้ากะเปิดอยู่
2. ลงชื่อเข้าใช้ แล้วเลือก **การดำเนินงานลิ้นชักโดยไม่มีลิ้นชัก**
3. ใช้การดำเนินงาน **จัดการสถานีฮาร์ดแวร์** เพื่อเปิดสถานีฮาร์ดแวร์
4. เลือกสถานีฮาร์ดแวร์ที่คุณสร้างไว้สำหรับเครื่องบันทึกเงินสดเพื่อให้ใช้งานได้
5. ลงชื่อออกจาก POS เข้าสู่ระบบกลับเข้าไปใหม่ และเปิดกะ

เทอร์มินัลการชำระเงินที่กำหนดให้กับโปรไฟล์ฮาร์ดแวร์จะเปิดใช้งานอยู่เสมอ และคุณจะได้รับพร้อมท์ หากต้องใช้เครื่องพิมพ์ใบเสร็จหรือลิ้นชักเงินสด

ผู้ค้าจำนวนมากที่ร้องขอคุณสมบัตินี้มีความสนใจในการลดปริมาณขยะโดยการให้ใบเสร็จทางอีเมล และการส่งเสริมการชำระเงินทางอิเล็กทรอนิกส์ ทั้งนี้ขึ้นอยู่กับการตั้งค่าคอนฟิกของ POS สมาคมร้านค้าได้รับพร้อมท์ให้เลือกเครื่องพิมพ์ใบเสร็จหรือลิ้นชักเงินสดเฉพาะเมื่อลูกค้าต้องการใบเสร็จจริง หรือต้องการชำระเงินด้วยเงินสด

สมาคมร้านค้าได้รับพร้อมท์ให้เลือกสถานีฮาร์ดแวร์เพียงครั้งเดียวต่อธุรกรรม เว้นแต่จะต้องมีการพิมพ์ใบเสร็จและมีการใช้เงินสดสำหรับการชำระเงิน แต่โปรไฟล์ฮาร์ดแวร์ที่เลือกไว้ในครั้งแรกจะไม่มีทั้งสองอุปกรณ์ ในกรณีดังกล่าว สมาคมร้านค้าจะมีการพร้อมต์อีกครั้ง เพื่อเลือกสถานีฮาร์ดแวร์ที่สามารถใช้ในการทำธุรกรรมให้เสร็จสมบูรณ์ได้
