# <a name="enable-adls-in-a-dynamics-365-commerce-environment"></a>เปืดใช้งาน ADLS ในสภาพแวดล้อม Dynamics 365 Commerce

[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายวิธีเปิดใช้งานและทดสอบ Azure Data Lake Storage (ADLS) สำหรับสภาพแวดล้อม Dynamics 365 Commerce ซึ่งเป็นข้อกำหนดเบื้องต้นสำหรับการเปิดใช้งานคำแนะนำผลิตภัณฑ์

## <a name="overview"></a>ภาพรวม

ในโซลูชัน Dynamics 365 Commerce ข้อมูลผลิตภัณฑ์และธุรกรรมทั้งหมดจะถูกติดตามในที่เก็บเอนทิตีของสภาพแวดล้อม เพื่อให้ข้อมูลนี้สามารถเข้าถึงได้จากบริการอื่น ๆ ของ Dynamics 365 เช่น การวิเคราะห์ข้อมูล ข่าวกรองทางธุรกิจ และคำแนะนำส่วนตัว โดยจำเป็นต้องเชื่อมต่อสภาพแวดล้อมกับโซลูชัน Azure Data Lake Storage Gen 2 (ADLS) ของลูกค้าเอง

เนื่องจาก ADLS ได้รับการกำหนดค่าในสภาพแวดล้อม ข้อมูลที่จำเป็นทั้งหมดจะถูกสะท้อนจากที่เก็บเอนทิตีในขณะที่ยังคงได้รับการป้องกันไว้อยู่และอยู่ภายใต้การควบคุมของลูกค้า

หากเปิดใช้งานคำแนะนำผลิตภัณฑ์หรือคำแนะนำส่วนตัวในสภาพแวดล้อมด้วยแล้ว สแตกคำแนะนำผลิตภัณฑ์จะได้รับสิทธิ์การเข้าถึงโฟลเดอร์เฉพาะใน ADLS เพื่อดึงข้อมูลลูกค้าและคำนวณคำแนะนำตามข้อมูลนั้น

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ลูกค้าจำเป็นต้องมีการกำหนดค่า ADLS ในการสมัครใช้งาน Azure ที่พวกเขาเป็นเจ้าของ หัวข้อนี้ไม่ครอบคลุมการซื้อการสมัครใช้งาน Azure หรือการตั้งค่าบัญชีที่เก็บข้อมูลที่เปิดใช้งาน ADLS

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับ ADLS ดู [เอกสารอย่างเป็นทางการของ ADLS](https://azure.microsoft.com/pricing/details/storage/data-lake)
  
## <a name="configuration-steps"></a>ขั้นตอนการกำหนดค่า 

ส่วนนี้ครอบคลุมขั้นตอนการตั้งค่าคอนฟิกที่จำเป็นสำหรับการเปิดใช้งานการ ADLS ในสภาพแวดล้อมตามที่เกี่ยวข้องกับคำแนะนำผลิตภัณฑ์
สำหรับภาพรวมเชิงลึกเพิ่มเติมเกี่ยวกับขั้นตอนต่างๆ ที่จำเป็นในการเปิดใช้งาน ADLS ให้ดูที่ [ทำให้ที่จัดเก็บเอนทิตี้มีไว้เป็น Data Lake](../fin-ops-core/dev-itpro/data-entities/entity-store-data-lake.md)

### <a name="enable-adls-in-the-environment"></a>เปืดใช้งาน ADLS ในสภาพแวดล้อม 

1. ล็อกอินเข้าสู่พอร์ทัลสนับสนุนของสภาพแวดล้อม
1. ค้นหา **พารามิเตอร์ระบบ** และนำทางไปที่แท็บ **การเชื่อมโยงข้อมูล** 
1. ตั้งค่า **เปิดใช้งานการรวม Data Lake** เป็น **ใช่**
1. ตั้งค่า **เปิดใช้งานการปรับปรุงทริกเกิล Data Lake** เป็น **ใช่**
1. จากนั้นป้อนข้อมูลที่จำเป็นต่อไปนี้:
    1. **รหัสแอปพลิเคชัน** // **ข้อมูลลับของแอปพลิเคชัน** // **ชื่อ DNS** - จำเป็นต้องเชื่อมต่อกับ KeyVault ที่ซึ่งความลับ ADLS ถูกจัดเก็บไว้
    1. **ชื่อลับ** - ชื่อลับที่เก็บไว้ใน KeyVault และใช้เพื่อรับรองความถูกต้องกับ ADLS
1. บันทึกการเปลี่ยนแปลงของคุณในหน้าที่อยู่มุมบนซ้าย

ภาพต่อไปนี้แสดงตัวอย่างของการกำหนดค่า ADLS

![ตัวอย่างการกำหนดค่า ADLS](./media/exampleADLSConfig1.png)

### <a name="test-the-adls-connection"></a>ทดสอบการเชื่อมต่อกับ ADLS 

1. ทดสอบการเชื่อมต่อกับ KeyVault โดยใช้ลิงก์ **ทดสอบ Azure Key Vault**
1. ทดสอบการเชื่อมต่อกับ ADLS โดยใช้ลิงก์ **ทดสอบที่เก็บข้อมูล Azure**

> [!NOTE]
> หากการทดสอบไม่ประสบผลสำเร็จ ตรวจสอบอีกครั้งว่าข้อมูล KeyVault ทั้งหมดที่เพิ่มข้างต้นถูกต้อง จากนั้นลองอีกครั้ง

เมื่อการทดสอบการเชื่อมต่อสำเร็จ คุณต้องเปิดใช้งานการรีเฟรชอัตโนมัติสำหรับที่เก็บเอนทิตี

เพื่อเปิดใช้งานที่เก็บเอนทิตีสำหรับการรีเฟรชอัตโนมัติ ให้ทำตามขั้นตอนเหล่านี้

1. ค้นหา **ที่เก็บเอนทิตี**
1. ในรายการด้านซ้าย นำทางไปยังรายการ **RetailSales** และเลือก **แก้ไข**
1. ตรวจสอบให้แน่ใจว่า **ได้เปิดใช้งานการรีเฟรชอัตโนมัติ** ซึ่งถูกตั้งค่าเป็น **ใช่** เลือก **รีเฟรช** แล้วจึงเลือก **บันทึก**

เพื่อเปิดใช้งานที่เก็บเอนทิตีสำหรับการรีเฟรชอัตโนมัติ ให้ทำตามขั้นตอนเหล่านี้ รูปภาพต่อไปนี้แสดงตัวอย่างของที่เก็บเอนทิตีพร้อมกับการเปิดใช้งานการรีเฟรชอัตโนมัติ

![ตัวอย่างของที่เก็บเอนทิตีที่เปิดใช้งานการรีเฟรชอัตโนมัติ](./media/exampleADLSConfig2.png)

ขณะนี้ ADLS ได้รับการกำหนดค่าสำหรับสภาพแวดล้อม 

หากยังไม่เสร็จสิ้น ให้ทำตามขั้นตอนสำหรับ [การเปิดใช้งานคำแนะนำผลิตภัณฑ์และการตั้งค่าส่วนบุคคล](enable-product-recommendations.md) สำหรับสภาพแวดล้อม
