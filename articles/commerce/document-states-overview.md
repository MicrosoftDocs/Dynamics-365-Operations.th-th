# <a name="document-states-and-lifecycle"></a>สถานะและระยะเวลาใช้งานของเอกสาร

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

หัวข้อนี้จะครอบคลุมถึงสถานะเอกสารต่างๆขององค์ประกอบหน้าใน Microsoft Dynamics 365 Commerce

## <a name="document-state-descriptions"></a>คำอธิบายสถานะเอกสาร

หัวข้อ [องค์ประกอบของหน้า](page-elements-overview.md) จะแสดงรายการชนิดเอกสารต่างๆในระบบการจัดการเนื้อหา (CMS) ชนิดเอกสารเหล่านี้อาจมีหลายสถานะในเครื่องมือการสร้าง สถานะเอกสารช่วยป้องกันความขัดแย้งของข้อมูล และบังคับใช้การควบคุมเวอร์ชัน ซึ่งจะกำหนดว่าใครบ้างที่สามารถเปลี่ยนแปลงเอกสารได้ เมื่อใดที่สามารถเปลี่ยนเอกสารได้ และเมื่อใดที่สามารถดูการเปลี่ยนแปลงของบุคคลอื่นได้

ตารางต่อไปนี้แสดงสถานะเอกสารที่เป็นไปได้ขององค์ประกอบใน Commerce

| สถานะเอกสาร | คำอธิบาย |
|---|---|
| เช็คเอาท์แล้ว | เมื่อมีการเช็คเอาท์ CMS ใด ๆ ให้กับคุณ จะไม่สามารถแก้ไขได้โดยผู้ใช้ระบบที่มีการพิสูจน์ตัวจริงอื่น ๆ ทุกความเปลี่ยนแปลงที่คุณสร้างบนรายการ จะมองเห็นได้เฉพาะคุณเท่านั้น |
| เช็คอินแล้ว | เมื่อมีการเช็คอิน CMS การเปลี่ยนแปลงทั้งหมดจะสามารถมองเห็นได้สำหรับผู้ใช้ที่มีการพิสูจน์ตัวจริงอื่น ๆ และผู้ใช้เหล่านั้นสามารถตรวจสอบและแก้ไขได้ การเช็คอินแต่ละครั้งจะสร้างเรกคอร์ดเวอร์ชันเอกสารในประวัติของรายการ |
| เผยแพร่แล้ว | เมื่อมีการเผยแพร่รายการ CMS จะมีการผลักดันให้กับไซต์จริงของคุณ และจะสามารถมองเห็นได้บนอินเทอร์เน็ตโดยผู้ใช้ภายนอกที่ไม่ได้ยืนยันตัวตน สามารถเผยแพร่รายการได้เฉพาะเมื่อเช็คอินแล้วเท่านั้น |
| บันทึกแล้ว | คุณสามารถบันทึกการเปลี่ยนแปลงที่เกิดขึ้นกับรายการที่มีการเช็คเอาท์ CMS ไปยัง CMS ก่อนที่จะเช็คอินหรือเผยแพร่รายการ ผู้ใช้ที่ได้รับการยืนยันตัวตนที่บันทึกไว้ จะไม่สามารถมองเห็นได้จนกว่าจะมีการเช็คอินรายการ ผู้ใช้ภายนอกไม่สามารถมองเห็นได้จนกว่าจะมีการเผยแพร่รายการ |
| ละทิ้งเช็คเอาท์ | เมื่อมีการยกเลิกรายการ CMS ที่มีการเช็คเอาท์ การเปลี่ยนแปลงที่บันทึกไว้ทั้งหมดจะถูกลบออกและรายการนั้นจะเปลี่ยนไปเป็นเวอร์ชันที่เช็คอินล่าสุด |
