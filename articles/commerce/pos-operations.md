# <a name="online-and-offline-point-of-sale-pos-operations"></a>การดำเนินงานการขายหน้าร้าน (POS) แบบออนไลน์และออฟไลน์

[!include [banner](includes/banner.md)]
[!include [banner](includes/preview-banner.md)]

การดำเนินการส่วนใหญ่ที่ผู้ใช้ทำในการขายหน้าร้าน (POS) เป็นการดำเนินงานที่พิจารณาแล้ว การดำเนินงานถูกตั้งค่าคอนฟิก และจัดการในฝ่ายสนับสนุน Dynamics 365 Commerce การดำเนินการจำนวนมากสามารถถูกเพิ่มไปยังปุ่มต่างๆ ในกริดปุ่ม POS จากนั้น ผู้ใช้สามารถเลือกปุ่มเพื่อเรียกการดำเนินงาน และดำเนินการทำงานนั้นได้ เมื่อต้องการทำให้การดำเนินการพร้อมใช้งานเพื่อเพิ่มลงในกริดปุ่มในตัวออกแบบกริดปุ่ม อันดับแรก ต้องทำเครื่องหมายเป็นการดำเนินการของผู้ใช้ในมุมมองการดำเนินงานของ POS ใน Commerce headquarters (**การขายปลีกและการค้า \> การตั้งค่าช่องทาง \> การตั้งค่า POS \> POS \> การดําเนินงาน POS**)

การดำเนินการอื่นเป็นส่วนหนึ่งของแอปพลิเคชัน POS หลัก และถูกเรียกจากปุ่มบนหน้าจอ หรือเป็นส่วนหนึ่งของเวิร์กโฟลว์หรือกระบวนการอื่นๆ

การดำเนินการบางอย่างไม่สามารถใช้งานได้ใน Modern POS หรือ Cloud POS การดำเนินงานเหล่านี้บางอย่างเป็นการดำเนินเฉพาะตำแหน่งที่ตั้งที่ต้องการส่วนขยายเพิ่มเติมและการตั้งค่าคอนฟิก อื่นๆ เป็นลักษณะการทำงานจาก Microsoft Dynamics AX 2012 ที่ไม่ได้รับการสนับสนุนในปัจจุบัน

ตารางด้านล่างให้รายละเอียดเกี่ยวกับการดำเนินการที่มีอยู่ใน Modern POS และ Cloud POS ตารางยังระบุตำแหน่งที่การดำเนินงานสามารถถูกเรียกได้ในแอปพลิเคชัน และพร้อมใช้งานหรือไม่ เมื่อ POS อยู่ในโหมดออฟไลน์

คอลัมน์ต่อไปนี้ระบุตำแหน่งที่สามารถเรียกการดำเนินงานได้:

- **การดำเนินการของผู้ใช้** – การดำเนินงานสามารถถูกมอบหมายให้ปุ่มในกริดปุ่ม POS ซึ่งเป็นส่วนหนึ่งของโครงร่างหน้าจอ POS ได้
- **หน้าจอธุรกรรม** – การดำเนินงานสามารถถูกเรียกจากกริดปุ่ม POS ที่ถูกตั้งค่าคอนฟิกบนหน้าจอธุรกรรม POS ได้
- **หน้าจอต้อนรับ** – การดำเนินงานสามารถถูกเรียกจากกริดปุ่ม POS ที่ถูกตั้งค่าคอนฟิกบนหน้าจอต้อนรับ POS ได้
