# <a name="asynchronous-customer-creation-mode"></a>โหมดการสร้างลูกค้าแบบอะซิงโครนัส

[!include [banner](includes/banner.md)]

บทความนี้จะอธิบายถึงโหมดการสร้างลูกค้าแบบอะซิงโครนัสใน Microsoft Dynamics 365 Commerce

ใน Commerce มีการสร้างลูกค้าอยู่สองโหมด ได้แก่ ซิงโครนัส (หรือซิงค์) และอะซิงโครนัส (หรือเอซิงค์) ตามค่าเริ่มต้น ลูกค้าจะถูกสร้างแบบซิงโครนัส กล่าวอีกอย่างคือ ถูกสร้างขึ้นใน Commerce Headquarters ตามเวลาจริง โหมดการสร้างลูกค้าแบบซิงค์เป็นประโยชน์เนื่องจากลูกค้าใหม่สามารถค้นหาได้ทันทีจากช่องทางต่างๆ อย่างไรก็ตาม ก็มีข้อเสียเปรียบด้วยเช่นกัน เนื่องจากระบบจะสร้างการเรียก [Commerce Data Exchange: บริการตามเวลาจริง](dev-itpro/define-retail-channel-communications-cdx.md#realtime-service) ไปยัง Commerce Headquarters จึงอาจกระทบกับประสิทธิภาพถ้าการสร้างลูกค้าเกิดขึ้นพร้อมกันหลายราย

ถ้าตัวเลือก **สร้างลูกค้าในโหมดเอซิงค์** ถูกตั้งค่าเป็น **ใช่** ในโพรไฟล์การทำงานของร้านค้า (**Retail และ Commerce \> การตั้งค่าช่องทาง \> การตั้งค่าร้านค้าออนไลน์ \> โพรไฟล์การทำงาน**) การเรียกใช้บริการตามเวลาจะไม่ถูกนำไปใช้เพื่อสร้างเรกคอร์ดลูกค้าในฐานข้อมูลช่องทางการขาย โหมดการสร้างลูกค้าแบบเอซิงค์จะไม่ส่งผลต่อประสิทธิภาพของ Commerce Headquarters รหัสเฉพาะสากล (GUID) แบบชั่วคราวจะถูกกำหนดให้กับเรกคอร์ดลูกค้าแบบเอซิงค์ใหม่ทุกเรกคอร์ดและใช้เป็นรหัสบัญชีลูกค้า GUID นี้ไม่แสดงต่อผู้ใช้การขายหน้าร้าน (POS) ผู้ใช้เหล่านั้นจะเห็น **กำลังรอการซิงค์** เป็นรหัสบัญชีลูกค้าแทน

> [!IMPORTANT]
> เมื่อ POS ออฟไลน์ ระบบจะสร้างลูกค้าแบบอะซิงโครนัสโดยอัตโนมัติ แม้ในขณะที่ปิดใช้งานโหมดการสร้างลูกค้าแบบเอซิงค์ ดังนั้น ไม่ว่าคุณจะเลือกระหว่างการสร้างลูกค้า async หรือ sync ผู้ดูแลระบบ Commerce headquarters จึงต้องสร้างและจัดตารางชุดงานประจำให้กับ **งาน P** งาน **ซิงโครไนส์ลูกค้าและคู่ค้าธุรกิจจากงานโหมด async** และงาน **1010** เพื่อให้ลูกค้า Async ทุกรายถูกแปลงเป็นลูกค้า Sync ใน Commerce headquarters

## <a name="async-customer-limitations"></a>ข้อจํากัดของลูกค้าแบบเอซิงค์

ฟังก์ชันการทำงานของลูกค้าแบบเอซิงค์มีข้อจํากัดดังต่อไปนี้

- ไม่สามารถออกบัตรสะสมคะแนนให้กับลูกค้าแบบเอซิงค์ได้ เว้นแต่จะซิงค์รหัสบัญชีลูกค้าใหม่กลับไปยังช่องทางการขาย

## <a name="async-customer-enhancements"></a>การเพิ่มประสิทธิภาพของลูกค้าแบบเอเซิงค์

เพื่อช่วยให้องค์กรใช้โหมดการสร้างลูกค้า Async เพื่อจัดการลูกค้า และเพื่อลดการสื่อสารแบบเรียลไทม์กับ Commerce headquarters การปรับปรุงต่อไปนี้ได้เริ่มต้นขึ้นเพื่อพัฒนาความต่อเนื่องระหว่างโหมดการซิงค์และ Async ในช่องทางต่างๆ 

| การปรับปรุงคุณลักษณะ | รุ่น Commerce | รายละเอียดคุณสมบัติ |
|---|---|---|
| การปรับปรุงประสิทธิภาพเมื่อมีการดึงข้อมูลลูกค้าจากฐานข้อมูลช่องทาง | 10.0.20 และใหม่กว่า | เพื่อช่วยพัฒนาประสิทธิภาพ เอนทิตีลูกค้าจะแบ่งออกเป็นเอนทิตีที่เล็กลง จากนั้น ระบบดึงข้อมูลที่ต้องใช้จากฐานข้อมูลช่องทางเท่านั้น |
| ความสามารถในการสร้างที่อยู่แบบอะซิงโครนัสระหว่างการเช็คเอาท์ | 10.0.22 และใหม่กว่า | <p>สวิตช์คุณลักษณะ: **เปิดใช้งานการสร้างแบบอะซิงโครนัสสำหรับที่อยู่ลูกค้า**</p><p>รายละเอียดคุณสมบัติ:</p><ul><li>ความสามารถในการเพิ่มที่อยู่โดยไม่เรียกบริการแบบเรียลไทม์ไปยัง Commerce headquarters</li><li>ความสามารถในการระบุที่อยู่ในฐานข้อมูลช่องทางโดยไม่ใช้รหัสเรกคอร์ด (ค่า **RecId**) โดยไม่ซ้ำกัน</li><li>การประทับเวลาการติดตามการสร้างที่อยู่</li><li>การซิงโครไนส์ที่อยู่ใน Commerce headquarters</li></ul><p>คุณลักษณะนี้จะมีผลกระทบต่อทั้งลูกค้าการซิงค์และ async หากต้องการแก้ไขที่อยู่แบบอะซิงโครนัส นอกเหนือจากการสร้างที่อยู่แบบอะซิงโครนัส คุณต้องเปิดใช้งาน **การแก้ไขลูกค้าในโหมดอะซิงโครนัส**</p> |
| เปิดใช้งานพาริตี้ระหว่างการสร้างลูกค้าแบบซิงโครนัสและแบบอะซิงโครนัส | 10.0.24 และใหม่กว่า | <p>สวิตช์คุณลักษณะ: **เปิดใช้งานการสร้างลูกค้าแบบอะซิงโครนัสที่ได้รับการปรับปรุง**</p><p>รายละเอียดคุณลักษณะ: ความสามารถในการรวบรวมข้อมูลเพิ่มเติม เช่น ชื่อ สังกัดจากลูกค้าเริ่มต้น และข้อมูลการติดต่อรอง (หมายเลขโทรศัพท์และที่อยู่อีเมล) ขณะที่คุณสร้างอะซิงโครนัสของลูกค้า</p> |
| ข้อความแสดงข้อผิดพลาดที่ที่เป็นมิตร | 10.0.28 และใหม่กว่า | การปรับปรุงเหล่านี้จะจะช่วยปรับปรุงข้อความแสดงข้อผิดพลาดที่ช่วยให้ผู้ใช้สามารถแก้ไขข้อมูลในทันทีขณะที่การซิงโครไนส์อยู่ในระหว่างกระบวนการ คุณเปิดใช้งานการปรับปรุงเหล่านี้โดยการใช้การตั้งค่า **อนุญาตองค์ประกอบ UI บางตัวที่จะไม่สามารถแก้ไขได้โดยลูกค้า async** ที่ **การตั้งค่าไซต์ \> ส่วนขยาย** ของโปรแกรมสร้างไซต์ Commerce |
| ความสามารถในการแก้ไขข้อมูลลูกค้าแบบอะซิงโครนัส | 10.0.29 และใหม่กว่า | <p>สวิตช์คุณลักษณะ: **เปิดใช้งานการแก้ไขลูกค้าในโหมดอะซิงโครนัส**</p><p>รายละเอียดคุณลักษณะ: ความสามารถในการแก้ไขข้อมูลลูกค้าแบบอะซิงโครนัส</p><p>หากต้องการคําตอบของคําถามทั่วไปเกี่ยวกับปัญหาที่เกี่ยวข้องกับการแก้ไขข้อมูลลูกค้าแบบอะซิงโครนัส โปรดดูที่ [FAQ โหมดการสร้างลูกค้าแบบอะซิงโครนัส](async-customer-mode-faq.md)</p> |
| ความสามารถในการซิงโครไนส์การตรวจสอบการดําเนินงานจัดการลูกค้า | 10.0.31 และใหม่กว่า | การปรับปรุงนี้จะช่วยให้ผู้ใช้สามารถตรวจสอบการซิงโครไนส์การดําเนินงานจัดการลูกค้าใน Commerce Headquarters ได้ และยังอนุญาตให้ผู้ใช้เปลี่ยนแปลงได้ถ้าต้องการและซิงโครไนส์ข้อมูล |

### <a name="feature-switch-hierarchy"></a>ลำดับชั้นสวิตช์คุณลักษณะ

เนื่องจากลำดับชั้นของสวิตช์คุณลักษณะ ก่อนจะเปิดใช้งานคุณลักษณะ **เปิดใช้งานการแก้ไขลูกค้าในโหมดอะซิงโครนัส** คุณต้องเปิดใช้งานคุณลักษณะต่อไปนี้ 

- **การปรับปรุงประสิทธิภาพสำหรับใบสั่งและธุรกรรมลูกค้า** – คุณสมบัติบังคับนับตั้งแต่รุ่น 10.0.28 ของ Commerce 
- **เปิดใช้งานการสร้างลูกค้าแบบอะซิงโครนัสที่ได้รับการปรับปรุง**
- **เปิดใช้งานการสร้างแบบอะซิงโครนัสสำหรับที่อยู่ลูกค้า**

สำหรับคําตอบของคําถามทั่วไปเกี่ยวกับการแก้ไขปัญหาเบื้องต้น ดูที่ [FAQ โหมดการสร้างลูกค้าแบบอะซิงโครนัส](async-customer-mode-faq.md) 

หลังจากคุณเปิดใช้งานคุณลักษณะที่กล่าวถึงก่อนหน้า คุณต้องจัดกำหนดการชุดงานประจำให้กับ **งาน P** งาน **ซิงโครไนส์ลูกค้าและคู่ค้าธุรกิจจากงานโหมดเอซิงค์** และงาน **1010** เพื่อให้ลูกค้าแบบเอซิงค์ทุกรายถูกแปลงเป็นลูกค้าแบบซิงค์ใน Commerce Headquarters

### <a name="customer-creation-in-pos-offline-mode"></a>การสร้างลูกค้าในโหมดออฟไลน์ของ POS

ตามที่กล่าวถึงไว้ก่อนหน้า ทุกครั้งที่ POS ออฟไลน์ ระบบจะสร้างลูกค้าพร้อมกันเสมอโดยอัตโนมัติ แม้ในขณะที่ปิดใช้งานโหมดการสร้างลูกค้าแบบเอซิงค์ ดังนั้น ผู้ดูแลระบบ Commerce Headquarters จึงต้องสร้างและจัดตารางชุดงานประจำให้กับ **งาน P** งาน **ซิงโครไนส์ลูกค้าและคู่ค้าธุรกิจจากงานโหมดเอซิงค์** และงาน **1010** เพื่อให้ลูกค้าแบบเอซิงค์ทุกรายถูกแปลงเป็นลูกค้าแบบซิงค์ใน Commerce Headquarters

> [!NOTE]
> ถ้าตัวเลือก **กรองตารางข้อมูลลูกค้าที่แชร์** ได้รับการตั้งค่าเป็น Commerce **ใช่** บนหน้า **ผังช่องทางการขาย** (**Retail และ Commerce \> การตั้งค่าศูนย์ควบคุม \> ตัวกำหนดตารางเวลาการขาย \> กลุ่มฐานข้อมูลช่องทางการขาย**) เรกคอร์ดลูกค้าจะไม่ถูกสร้างขึ้นในโหมดออฟไลน์ของ POS สำหรับข้อมูลเพิ่มเติม ให้ดู [การยกเว้นข้อมูลออฟไลน์](dev-itpro/implementation-considerations-cdx.md#offline-data-exclusion)
