# <a name="opt-out-of-personalized-recommendations"></a>ยกเลิกคำแนะนำส่วนบุคคล

[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายวิธีที่คุณสามารถให้ลูกค้าเลือกที่จะไม่รับคำแนะนำแบบส่วนตัวใน Microsoft Dynamics 365 Commerce

## <a name="overview"></a>ภาพรวม

ในระหว่างการสร้างลูกค้าองค์กร ลูกค้าใหม่จะได้รับการตั้งค่าโดยอัตโนมัติเพื่อรับคำแนะนำแบบส่วนตัว อย่างไรก็ตาม Dynamics 365 Commerce มีวิธีการต่างๆสำหรับผู้ค้าปลีกเพื่อให้ผู้ใช้เลือกที่จะไม่รับคำแนะนำเหล่านี้ และจำกัด การประมวลผลข้อมูลส่วนบุคคลของพวกเขา ผู้ใช้ที่ได้รับการรับรองความถูกต้องซึ่งไม่เข้าร่วมรับคำแนะนำแบบส่วนตัวจะหยุดการดูรายการแบบส่วนตัวในทันที นอกจากนี้ ข้อมูลส่วนบุคคลทั้งหมดที่เก็บรวบรวมสำหรับการตั้งค่าส่วนบุคคลจะถูกลบออกจากโมเดลคำแนะนำแบบส่วนตัว

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับคำแนะนำผลิตภัณฑ์แบบส่วนตัว โปรดดู [เปิดใช้งานคำแนะนำแบบส่วนตัว](personalized-recommendations.md)

## <a name="ways-for-retailers-to-implement-an-opt-out-experience"></a>วิธีที่ผู้ค้าปลีกจะใช้ประสบการณ์การปฏิเสธเข้าร่วม

ผู้ค้าปลีกมีสามวิธีที่จะใช้ประสบการณ์การปฏิเสธเข้าร่วม

### <a name="opting-out-on-behalf-of-users"></a>ปฏิเสธเข้าร่วมที่กระทำในนามของผู้ใช้อื่น

ในการจัดการลูกค้าองค์กรในการสนับสนุนการค้า ผู้ค้าปลีกสามารถปฏิเสธเข้าร่วมในนามของผู้ใช้อื่น

1. จากหน้าแรกของการสนับสนุน ให้ค้นหา **ลูกค้าทั้งหมด**
1. ค้นหาและเลือกลูกค้า จากนั้นเลือก FastTab **การขายปลีก**

    ![FastTab การขายปลีก](./media/Disablepersonalizationpart1.png)

1. ภายใต้ **ความเป็นส่วนตัว** ตั้งค่าตัวเลือก **ปิดใช้งานการตั้งค่าส่วนบุคคล** เป็น **ใช่**

    ![การตั้งค่าความเป็นส่วนตัว](./media/Disablepersonalizationpart2.png)

1. เลือก **บันทึก** และปิดหน้า

### <a name="module-based-opt-out-experience"></a>ประสบการณ์การเลือกไม่ใช้การยึดตามโมดูล

ผู้ค้าปลีกสามารถอนุญาตให้ผู้ใช้ที่ผ่านการรับรองความถูกต้องยกเลิกคำแนะนำแบบส่วนตัวได้ด้วยตนเอง เพื่อมอบประสบการณ์การไม่เข้าร่วมนี้ ให้เพิ่มโมดูลการยกเลิกผู้ใช้ในหน้าโปรไฟล์บัญชีลูกค้า

### <a name="custom-extensions"></a>ส่วนขยายที่กำหนดเอง

ผู้ค้าปลีกสามารถสร้างส่วนขยายของตนเองเพื่อจัดการประสบการณ์การเลือกไม่ใช้สำหรับผู้ใช้ สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [เรียกเซิร์ฟเวอร์การขายปลีก API](e-commerce-extensibility/call-retail-server-apis.md) และ [การเพิ่มช่องทางออนไลน์](e-commerce-extensibility/overview.md)

## <a name="obtain-a-digital-copy-of-personalized-recommendations-data-on-behalf-of-an-authenticated-user"></a>รับสำเนาของข้อมูลคำแนะนำแบบส่วนตัวในรูปแบบดิจิทัล ในนามของผู้ใช้ที่ได้รับการรับรองความถูกต้อง

ลูกค้าอาจต้องการได้รับสำเนาดิจิตอลของข้อมูลส่วนบุคคลและดูมุมมองที่ส่งออกของผลลัพธ์คำแนะนำนั้น หากลูกค้าร้องขอข้อมูลนี้ ผู้ค้าปลีกจะต้องสร้างส่วนขยายที่กำหนดเองซึ่งเรียกใช้อินเทอร์เฟซการเขียนโปรแกรมประยุกต์เซิร์ฟเวอร์การค้าปลีก (API) และแบบสอบถามสำหรับผลลัพธ์แบบเต็มจากรายการ **เลือกให้คุณ** อิงตามรหัสลูกค้าของลูกค้า ผลลัพธ์สามารถส่งออกในรูปแบบค่าที่คั่นด้วยเครื่องหมายจุลภาค (CSV) และแบ่งปันไปยังลูกค้า

ตัวอย่างต่อไปนี้แสดงวิธีที่ผู้ค้าปลีกสามารถทำงานนี้ให้สำเร็จ

1. ผู้ค้าปลีกสร้างส่วนขยายที่กำหนดเองเพื่อดึงข้อมูลคำแนะนำแบบส่วนตัวในนามของผู้ใช้ สำหรับข้อมูลเกี่ยวกับวิธีสร้างโมดูล ลอกแบบโมดูลที่มีอยู่ เรียกเซิร์ฟเวอร์การขายปลีก API และเรียกการดำเนินการข้อมูล โปรดดู [การขยายช่องทางออนไลน์](e-commerce-extensibility/overview.md)
2. ส่วนขยายที่กำหนดเองทำให้การเรียกการดำเนินการข้อมูลหลัก **รับคำแนะนำ** และส่งผ่านข้อมูลที่จำเป็นไปโดยขึ้นอยู่กับความต้องการในรายการ ในกรณีของรายการ **เลือกให้คุณ** ส่วนขยายจะต้องส่งชื่อรายการที่ถูกต้องและรหัสลูกค้าไปยังการดำเนินการข้อมูล

    วิธีหนึ่งในการสร้างส่วนขยายที่กำหนดเองคือการลอกแบบโมดูลชุดผลิตภัณฑ์ที่มีอยู่ที่ใช้เพื่อส่งคืนผลลัพธ์คำแนะนำ ด้วยการลอกแบบโมดูลที่มีอยู่นี้ ผู้ค้าปลีกสามารถแก้ไขรหัสที่มีอยู่และเพิ่มปุ่มใหม่ที่ส่งออกผลลัพธ์คำแนะนำไปยังไฟล์ CSV สำหรับข้อมูลเพิ่มเติม โปรดดู [ลอกแบบโมดูลชุดสินค้าเริ่มต้น](e-commerce-extensibility/clone-starter-module.md) และ [โมดูลชุดผลิตภัณฑ์](product-collection-module-overview.md)

    สำหรับมุมมองแบบเต็มของไลบรารีเซิร์ฟเวอร์การขายปลีก API ให้ดู [เซิร์ฟเวอร์การขายปลีกลูกค้าและผู้บริโภค API](dev-itpro/retail-server-customer-consumer-api.md)

3. หลังจากสร้างส่วนขยายที่กำหนดเองแล้ว ผู้ค้าปลีกสามารถส่งออกไฟล์ CSV ของผลลัพธ์คำแนะนำทั้งหมด โดยยึดตามรหัสลูกค้าที่ไม่ซ้ำกันของผู้ใช้ที่ได้รับการรับรองความถูกต้อง
4. ผู้ค้าปลีกสามารถแชร์ไฟล์ CSV ที่ส่งออกซึ่งมีรายการส่วนบุคคลแบบเต็มของผลิตภัณฑ์ที่แนะนำให้กับผู้ใช้ที่ได้รับการรับรองความถูกต้อง
