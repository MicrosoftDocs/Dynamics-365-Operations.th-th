<span data-ttu-id="9d82a-104">หัวข้อนี้ครอบคลุมโมดูลกล่องการฃื้อ และอธิบายวิธีการเพิ่มไปยังหน้าไซต์ใน Microsoft Dynamics 365 Commerce</span><span class="sxs-lookup"><span data-stu-id="9d82a-104">This topic covers buy box modules and describes how to add them to site pages in Microsoft Dynamics 365 Commerce.</span></span>

## <a name="overview"></a><span data-ttu-id="9d82a-105">ภาพรวม</span><span class="sxs-lookup"><span data-stu-id="9d82a-105">Overview</span></span>

<span data-ttu-id="9d82a-106">โดยทั่วไปแล้วข้อกำหนด *กล่องการซื้อ* อ้างถึงพื้นที่ของเพจรายละเอียดผลิตภัณฑ์ที่อยู่ "เหนือโฟลด์" และที่โฮสต์ข้อมูลที่สำคัญทั้งหมดที่จำเป็นต่อการทำการซื้อผลิตภัณฑ์</span><span class="sxs-lookup"><span data-stu-id="9d82a-106">The term *buy box* typically refers to the area of a product details page that is "above the fold," and that hosts all the most important information that is required to make a product purchase.</span></span> <span data-ttu-id="9d82a-107">(พื้นที่ที่อยู่ "เหนือโฟลด์" จะปรากฏขึ้นเมื่อมีการโหลดหน้าแรก ดังนั้นผู้ใช้จึงไม่ต้องเลื่อนลงเพื่อดูรายการนั้น)</span><span class="sxs-lookup"><span data-stu-id="9d82a-107">(An area that is "above the fold" is visible when the page is first loaded, so that users don't have to scroll down to see it.)</span></span>

<span data-ttu-id="9d82a-108">โมดูลกล่องการซื้อคือตู้คอนเทนเนอร์พิเศษที่ใช้เพื่อโฮสต์โมดูลทั้งหมดที่แสดงอยู่ในพื้นที่กล่องการซื้อของหน้ารายละเอียดผลิตภัณฑ์</span><span class="sxs-lookup"><span data-stu-id="9d82a-108">A buy box module is special container that is used to host all the modules that are shown in the buy box area of a product details page.</span></span>

<span data-ttu-id="9d82a-109">URL ของหน้ารายละเอียดผลิตภัณฑ์รวมถึงรหัสผลิตภัณฑ์</span><span class="sxs-lookup"><span data-stu-id="9d82a-109">The URL of a product details page includes the product ID.</span></span> <span data-ttu-id="9d82a-110">ข้อมูลทั้งหมดที่จำเป็นต้องใช้ในการแสดงโมดูลกล่องการซื้อ จะได้รับมาจากรหัสผลิตภัณฑ์นี้</span><span class="sxs-lookup"><span data-stu-id="9d82a-110">All the information that is required to render a buy box module is derived from this product ID.</span></span> <span data-ttu-id="9d82a-111">ถ้าไม่ได้ระบุหมายเลขผลิตภัณฑ์ จะไม่มีการแสดงผลโมดูลกล่องการซื้ออย่างถูกต้องบนเพจ</span><span class="sxs-lookup"><span data-stu-id="9d82a-111">If a product ID isn't provided, the buy box module won't be rendered correctly on a page.</span></span> <span data-ttu-id="9d82a-112">ดังนั้น จึงไม่สามารถใช้โมดูลกล่องการซื้อบนหน้าใดๆ ที่ไม่มีบริบทของผลิตภัณฑ์ (ตัวอย่าง เช่น โฮมเพจหรือเพจการตลาด)</span><span class="sxs-lookup"><span data-stu-id="9d82a-112">Therefore, a buy box module can't be used on any page that doesn't have product context (for example, a home page or a marketing page).</span></span>

<span data-ttu-id="9d82a-114">ในฐานะที่เป็นคอนเทนเนอร์ กล่องการซื้อจะควบคุมคุณสมบัติพื้นฐานบางอย่าง เช่น ความกว้าง</span><span class="sxs-lookup"><span data-stu-id="9d82a-114">As a container, a buy box module controls some basic properties, such as the width.</span></span> <span data-ttu-id="9d82a-115">การตั้งค่าความกว้างจะกำหนดว่าโมดูลภายในคอนเทนเนอร์ต้องพอดีกับคอนเทนเนอร์ หรือเต็มหน้าจอ</span><span class="sxs-lookup"><span data-stu-id="9d82a-115">The width setting determines whether the modules inside the container must fit within that container, or whether they can fill the screen.</span></span>

<span data-ttu-id="9d82a-120">ตามค่าเริ่มต้น อัตราส่วนของความกว้างคอลัมน์สำหรับภาคสื่อและภาคเนื้อหาคือ 2:1</span><span class="sxs-lookup"><span data-stu-id="9d82a-120">By default, the ratio of column widths for the media region and the content region is 2:1.</span></span> <span data-ttu-id="9d82a-121">อย่างไรก็ตาม ความกว้างของคอลัมน์สามารถแทนที่ด้วยชุดรูปแบบนี้ได้</span><span class="sxs-lookup"><span data-stu-id="9d82a-121">However, the column widths can be overridden by the theme.</span></span> <span data-ttu-id="9d82a-122">นอกจากนี้ บนอุปกรณ์เคลื่อนที่ ทั้งสองภาคจะเป็นแบบซ้อนกัน เพื่อให้พื้นที่หนึ่งปรากฏอยู่ด้านล่างของภาคอื่นๆ</span><span class="sxs-lookup"><span data-stu-id="9d82a-122">In addition, on mobile devices, the two regions are stacked, so that one region appears below the other region.</span></span>
