# <a name="clienteling-overview"></a>ภาพรวมของความสัมพันธ์อันดีกับลูกค้า

[!include [banner](includes/banner.md)]


ร้านค้าปลีกจำนวนมาก โดยเฉพาะร้านค้าปลีกพิเศษระดับสูงโดยเฉพาะอย่างยิ่ง ต้องการความสัมพันธ์ด้านการขายของตนเพื่อสร้างความสัมพันธ์ระยะยาวกับลูกค้าหลักของพวกเขา ผู้เชื่อมโยงคาดว่าจะทราบเกี่ยวกับ การถูกใจและการไม่ชอบ ประวัติการซื้อ ความต้องการผลิตภัณฑ์ และวันที่สำคัญของลูกค้า เช่น วันครบรอบปี และวันเกิด ผู้เชื่อมโยงต้องมีสถานที่ที่สามารถเก็บข้อมูลนี้ และค้นหาข้อมูลนี้ได้ง่ายเมื่อต้องการ ถ้าข้อมูลนี้พร้อมใช้งานในมุมมองเดียว ผู้เชื่อมโยงสามารถกำหนดกลุ่มลูกค้าเป้าหมายที่ตรงตามเกณฑ์ที่ระบุได้อย่างง่ายดาย ตัวอย่างเช่น ผู้ใช้สามารถค้นหาลูกค้าทั้งหมดที่ต้องการซื้อกระเป๋าถือ หรือลูกค้าที่มีเหตุการณ์สำคัญอันใกล้ เช่น วันเกิด หรือวันครบรอบ

## <a name="client-book"></a>สมุดบัญชีลูกค้า

ใน Microsoft Dynamics 365 Commerce ผู้ค้าปลีกสามารถใช้ฟังก์ชันของสมุดบัญชีลูกค้า เพื่อช่วยจัดเก็บความสัมพันธ์แบบระยะยาวกับลูกค้าหลัก

สมุดบัญชีลูกค้ามีบัตรของลูกค้า ซึ่งแสดงข้อมูลการติดต่อสำหรับลูกค้าแต่ละราย พร้อมด้วยคุณสมบัติเพิ่มเติมสามรายการซึ่งกำหนดโดยผู้จัดจำหน่าย และตั้งค่าคอนฟิกในศูนย์ควบคุม ร้านค้าปลีกสามารถเลือกสิ่งที่สำคัญที่สุดสามประการ ที่ผู้ขายควรรู้เกี่ยวกับลูกค้า ตัวอย่างเช่น ผู้ค้าปลีกเครื่องประดับอาจต้องการรวมวันที่สำคัญ เช่น วันครบรอบ ปีหรือวันเกิด เนื่องจากวันเหล่านี้มีโอกาสที่ผู้คนอาจซื้อเครื่องประดับเพิ่มเติม ในทำนองเดียวกัน ร้านค้าปลีกแฟชั่นอาจต้องการรวม ความสนใจการซื้อ และแบรนด์ที่ลูกค้าต้องการ

นอกจากนี้สมุดลูกค้ายังให้ผู้ขายสามารถกรองข้อมูลรายการที่แสดงเฉพาะลูกค้าที่ตรงตามเกณฑ์ที่ระบุ ตัวอย่างเช่น คอลเลกชันใหม่ของรองเท้ามีจำหน่ายในร้านค้าแล้ว และผู้เชื่อมโยงต้องการแจ้งให้ลูกค้าที่ต้องการซื้อรองเท้าทราบ ในกรณีนี้ ผู้เชื่อมโยงสามารถกรองข้อมูลสมุดบัญชีลูกค้าเพื่อหาลูกค้าที่เกี่ยวข้อง แล้วดำเนินการต่อไป

ถ้าลูกค้าใดๆ ไม่ได้รับการพิจารณาเป็นลูกค้าที่สำคัญอีกต่อไปด้วยเหตุผลบางประการ ดังนั้นจึงไม่ควรมีการจัดการอย่างใกล้ชิด ผู้ขายสามารถลบลูกค้าเหล่านี้ออกจากสมุดบัญชีลูกค้าได้

ร้านค้าปลีกบางรายไม่ต้องการจัดการลูกค้าที่ระดับผู้ขาย แต่ต้องการจัดการรายชื่อลูกค้าหลักในระดับร้านค้า ผู้ค้าปลีกเหล่านี้สามารถดูลูกค้าจากสมุดบัญชีลูกค้าของร้านค้า โดยใช้ตัวเลือกนี้ ร้านค้าปลีกสามารถดูลูกค้าจากสมุดบัญชีลูกค้าของผู้ขายทั้งหมดที่มีสมุดที่อยู่ตรงกับสมุดที่อยู่ของร้านค้าปัจจุบัน ด้วยวิธีนี้ ถ้าผู้ขายทำงานในหลายร้านค้าของนิติบุคคล สมุดบัญชีลูกค้าจะแสดงลูกค้าจากร้านค้าเหล่านั้นทั้งหมด ฟังก์ชันนี้สนับสนุนความสามารถเพิ่มเติม ตัวอย่างเช่น คุณสามารถกำหนดลูกค้าใหม่จากผู้ขายหนึ่งไปยังผู้ขายอื่นได้ ความสามารถนี้มีประโยชน์เมื่อผู้ขายถูกโอนย้ายหรือออกจากบริษัท

ผู้ขายแต่ละคนอาจมีสมุดบัญชีลูกค้าหนึ่งสมุดสำหรับแต่ละนิติบุคคล และผู้ขายสามารถเพิ่มลูกค้าหนึ่งรายขึ้นไปในสมุดบัญชีลูกค้าของตน ใน Commerce ในขณะนี้ลูกค้าแต่ละรายสามารถถูกเพิ่มไปยังสมุดบัญชีลูกค้าเดียวเท่านั้น อย่างไรก็ตาม Microsoft วางแผนจะเพิ่มฟังก์ชันที่ช่วยให้ลูกค้ารายเดียวเพิ่มเข้าไปในสมุดบัญชีลูกค้าหลายสมุดได้

> [!NOTE]
> เนื่องจากการค้นหาลูกค้า สมุดบัญชีลูกค้าจะไม่กรองเรกคอร์ดลูกค้าโดยใช้สมุดที่อยู่ของร้านค้า

## <a name="activities-and-notes"></a>กิจกรรมและบันทึกย่อ

ช่องทางออนไลน์ช่วยให้ผู้ค้าปลีกสามารถเรียนรู้เกี่ยวกับ การตั้งค่าลูกค้าได้โดยไม่ต้องให้ลูกค้าระบุข้อมูลนี้อย่างชัดแจ้ง ด้วยความคมชัด เมื่อลูกค้าโต้ตอบกับผู้ขายในร้านค้า พวกเขาได้แบ่งปันข้อมูลเกี่ยวกับการกำหนดลักษณะของตนเองอย่างชัดแจ้ง แต่ข้อมูลนี้อาจสูญหายไปหลังจากการขายสิ้นสุดแล้ว อย่างไรก็ตาม ถ้ามีการบันทึกข้อมูลนี้ จะช่วยให้ร้านค้าปลีกเข้าใจลูกค้าได้ดียิ่งขึ้น ดังนั้นจึงช่วยให้ลูกค้าสามารถแนะนำให้ดีขึ้น และประสบการณ์ในการซื้อสินค้าโดยรวมได้ดียิ่งขึ้น

การรวบรวมข้อมูลที่สำคัญที่ลูกค้าแบ่งปัน ผู้ขายไม่เพียงแต่สามารถใช้แอททริบิวต์ของสมุดบัญชีลูกค้า แต่ยังสามารถใช้ฟังก์ชันกิจกรรมและบันทึกย่อได้อีกด้วย ผู้ค้าปลีกสามารถตั้งค่าคอนฟิกชนิดกิจกรรม เช่น ข้อมูลเกี่ยวกับการเยี่ยมชมร้านค้า ที่อยู่อีเมล หมายเลขโทรศัพท์ และการนัดหมาย กิจกรรมที่ผู้ขายสร้าง สามารถดูได้ในรูปแบบของไทม์ไลน์ในการขายหน้าร้าน (POS) นอกจากนี้ยังสามารถดูบนแท็บ **กิจกรรม** บนหน้า **ลูกค้าทั้งหมด \> ทั่วไป** ในศูนย์ควบคุม

นอกจากนี้ผู้ขายยังสามารถใช้บันทึกย่อเพื่อรวบรวมข้อมูลลูกค้าทั่วไป ที่สามารถอ้างอิงได้ก่อนการโต้ตอบทั้งหมด บันทึกย่อเหล่านี้จะถูกบันทึกในศูนย์ควบคุม และสามารถดูได้ในโพรไฟล์ลูกค้า หรือบนหน้ารายละเอียดลูกค้าในศูนย์บริการ

> [!NOTE]
> ปัจจุบัน บันทึกและกิจกรรมทั้งหมดสามารถดูได้โดยผู้ขายที่ทำงานให้นิติบุคคล และสามารถดูหน้ารายละเอียดลูกค้าได้ บันทึกย่อและกิจกรรมไม่จำกัดเฉพาะผู้ขายที่เพิ่มลูกค้าลงในสมุดรายชื่อลูกค้า

## <a name="integration-with-dynamics-365-customer-insights"></a>การรวมกับ Dynamics 365 Customer Insights

โดยใช้โปรแกรมประยุกต์ Dynamics 365 Customer Insights ร้านค้าปลีกสามารถรวมข้อมูลจากระบบต่างๆ ที่ลูกค้าใช้เพื่อโต้ตอบกับแบรนด์ของผู้จัดจำหน่าย หลังจากนั้นสามารถใช้ข้อมูลนี้เพื่อสร้างมุมมองเดียวของลูกค้าและดึงข้อมูลเชิงลึก การรวม Customer Insights กับ Commerce ทำให้ผู้ค้าปลีกสามารถเลือกมาตรการหนึ่งอย่างขึ้นไปที่ควรแสดงอยู่บนบัตรลูกค้าในสมุดบัญชีลูกค้า ตัวอย่างเช่น ผู้ค้าปลีกสามารถใช้ข้อมูลใน Customer Insights เพื่อคำนวณ "ความน่าจะเป็นของลูกค้า" สำหรับลูกค้า และกำหนด "การดำเนินการที่ดีที่สุดถัดไป" ถ้าค่าเหล่านี้ถูกกำหนดเป็นมาตรการ จะแสดงขึ้นบนบัตรลูกค้าและสามารถให้ข้อมูลที่สำคัญแก่ผู้ขาย สำหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อมูลเชิงลึกของลูกค้า ดูเอกสาร [Dynamics 365 Customer Insights](https://docs.microsoft.com/dynamics365/ai/customer-insights/overview) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับมาตรการ ดู [มาตรการ](https://docs.microsoft.com/dynamics365/ai/customer-insights/pm-measures)

## <a name="set-up-clienteling"></a>ตั้งค่าความสัมพันธ์อันดีกับลูกค้า

การเปิดใช้งานฟังก์ชัน clienteling ในสภาพแวดล้อมของคุณ ให้ทำตามขั้นตอนต่อไปนี้

1. ในพื้นที่ทำงาน **การจัดการคุณลักษณะ** กรองคุณลักษณะ โดยโมดูล **การขายปลีกและการค้า**

    ![ความสัมพันธ์อันดีกับลูกค้า ในรายการขอคุณลักษณะสำหรับโมดูล Commerce](./media/Enable_clienteling.png "ความสัมพันธ์อันดีกับลูกค้า ในรายการขอคุณลักษณะสำหรับโมดูลการขายปลีกและการค้า")

2. เปิดใช้งานคุณลักษณะ **ความสัมพันธ์อันดีกับลูกค้า** โดยการเลือก **เปิดใช้งานเดี๋ยวนี้**
3. บนหน้า **พารามิเตอร์ Commerce** บนแท็บ **ลำดับหมายเลข** เลือกแถว **ตัวระบุสมุดบัญชีของลูกค้า** จากนั้น ในฟิลด์ **รหัสลำดับหมายเลข** ให้เลือกลำดับหมายเลข ระบบจะใช้ลำดับหมายเลขนี้เพื่อกำหนดรหัสให้กับสมุดบัญชีลูกค้า
4. เลือก **บันทึก**
5. สร้างกลุ่มแอททริบิวต์ใหม่ที่มีแอททริบิวต์ที่คุณต้องการรวม สำหรับลูกค้าที่มีการจัดการในสมุดบัญชีลูกค้า สำหรับวิธีการ ดู [Attributes and attribute groups](https://docs.microsoft.com/dynamics365/retail/attribute-attributegroups-lifecycle).

    - กำหนดแอททริบิวต์ที่ต้องการ เป็น **สามารถจำกัดได้** ผู้ขายสามารถใช้แอททริบิวต์เหล่านี้ ในการกรองข้อมูลสมุดบัญชีของลูกค้า
    - ตั้งค่าลำดับการแสดงผลสำหรับแอททริบิวต์เหล่านี้ คำสั่งการแสดงผลนี้กำหนดว่าควรแสดงแอททริบิวต์ใดบนบัตรลูกค้าในสมุดบัญชีลูกค้า ลำดับการแสดงผลที่1ถือว่าสูงกว่าลำดับการแสดงผลที่ 2 ดังนั้น แอททริบิวต์ที่มีลำดับการแสดงผล 1 จะแสดงก่อนที่แอททริบิวต์ที่มีลำดับการแสดงผล 2

    > [!NOTE]
    > คุณสามารถทำให้ Customer Insights พร้อมใช้งานได้จากหน้าเดียวกัน อย่างไรก็ตาม รหัสโปรแกรมประยุกต์ของ Azure และความลับต้องถูกสร้างขึ้น เพื่อวัตถุประสงค์ในการรับรองความถูกต้อง (สำหรับข้อมูลเกี่ยวกับความต้องการ ให้ดูส่วน [เปิดใช้งานการรวม Customer Insights กับ Commerce](#turn-on-the-integration-of-customer-insights-with-commerce) หลังหัวข้อนี้) ถ้ามีการเปิดใช้ Customer Insights และคุณเลือกมาตรการหนึ่งอย่างขึ้นไปซึ่งควรแสดงอยู่บนบัตรลูกค้า มาตรการดังกล่าวจะแสดงขึ้นก่อน ถัดไปก ลุ่มแอททริบิวต์ของสมุดบัญชีลูกค้าจะแสดง ขึ้นอยู่กับลำดับการแสดงผล ตัวอย่างเช่น ถ้าคุณเลือกมาตรการสองอย่างจาก Customer Insights มาตรการสองอย่างนี้และแอตทริบิวต์สมุดรายชื่อลูกค้า จะแสดงขึ้นบนบัตรลูกค้า (แอททริบิวต์ของสมุดรายชื่อลูกค้าที่แสดง จะเป็นแอททริบิวต์ที่มีลำดับการแสดงผลสูงสุด)

6. บนหน้า **พารามิเตอร์ Commerce** บนแท็บ **ความสัมพันธ์อันดีกับลูกค้า** ในฟิลด์ **กลุ่มแอททริบิวต์ของสมุดบัญชีลูกค้า** ให้เลือกกลุ่มแอททริบิวต์ที่คุณเพิ่งสร้างขึ้น

    ![กลุ่มแอททริบิวต์ของสมุดบัญชีลูกค้าที่เลือก](./media/Client%20book%20attributes.png "กลุ่มแอททริบิวต์ของสมุดบัญชีลูกค้าที่เลือก")

7. เมื่อต้องการรวมกิจกรรมที่เกิดขึ้นที่ POS ให้กำหนดชนิดของกิจกรรมบนหน้า **ชนิดกิจกรรม** (**Retail และ Commerce \> ลูกค้า \> ชนิดกิจกรรม**)

    > [!NOTE]
    > ชนิดของกิจกรรมจะถูกดึงโดย Commerce Scale Unit เมื่อทำการเรียกใช้งานแบบเรียลไทม์ในครั้งแรก หลังจากที่มีการดึงกิจกรรมเหล่านั้น จะถูกเก็บไว้ชั่วคราวสองสามชั่วโมง ถ้าคุณเปลี่ยนชนิดของกิจกรรม ให้รอจนกว่าแคชไม่มีผลบังคับใช้อีกต่อไป อีกทางหนึ่งคือ สำหรับสภาพแวดล้อมที่ไม่ใช้งานจริง ให้รีสตาร์ทบริการ Commerce Scale Unit

8. เพิ่มปุ่มสองปุ่มให้กับโครงร่างหน้าจอ POS ที่เหมาะสม เพื่อให้ผู้ขายสามารถดูสมุดบัญชีลูกค้าของพวกขเ และสมุดบัญชีลูกค้าของร้านค้า (สมุดบัญชีลูกค้าของร้านค้ารวมลูกค้าจากสมุดบัญชีลูกค้าทั้งหมดของผู้ขายทั้งหมดที่แบ่งปันสมุดที่อยู่ร่วมกับร้านค้า) การดำเนินงานที่สอดคล้องกัน เรียกว่า **ดูลูกค้าในสมุดบัญชีลูกค้า** และ **ดูลูกค้าจากสมุดบัญชีลูกค้าของร้านค้า** ตามลำดับ มีการดำเนินงานเพิ่มเติมสามอย่างที่เกี่ยวข้องกับสมุดบัญชีลูกค้าที่สามารถใช้งานได้ การดำเนินงานเหล่านี้จะกำหนดว่าผู้ขายด้สามารถ เพิ่ม ลบ และกำหนดลูกค้า จากสมุดบัญชีลูกค้าได้ เรียกว่า **เพิ่มลูกค้าให้กับสมุดบัญชีลูกค้า** **ลบลูกค้าออกจากสมุดบัญชีลูกค้า** และ **กำหนดลูกค้าให้กับสมุดบัญชีลูกค้า** ตามลำดับ
9. ดำเนินการงานกำหนดการกระจายต่อไปนี้: 1040 1150 1110 และ 1090

หลังจากที่คุณดำเนินการขั้นตอนนี้เสร็จสมบูรณ์แล้ วผู้ขายสามารถเปิดหน้ารายละเอียดลูกค้าที่ POS และเพิ่มลูกค้าในสมุดบัญชีลูกค้า ดูและบันทึกกิจกรรม และบันทึกย่อสำหรับลูกค้า และกำหนดลูกค้าเป้าหมายโดยใช้แอททริบิวต์ของสมุดบัญชีลูกค้าและลูกค้า เพื่อกรองข้อมูลสมุดบัญชีลูกค้า ในแผนภาพต่อไปนี้แสดงตัวอย่างของสมุดบัญชีลูกค้า

![ตัวอย่างของสมุดบัญชีลูกค้า](./media/client_book.png "ตัวอย่างของสมุดบัญชีลูกค้า")

## <a name="turn-on-the-integration-of-customer-insights-with-commerce"></a>เปิดการรวม Customer Insights กับ Commerce

การเปิดการรวม Customer Insights กับ Commerce คุณต้องตรวจสอบให้แน่ใจว่าคุณมีอินสแตนซ์ที่ใช้งานอยู่ของ Customer Insights ในผู้เช่าที่มีการเตรียมใช้งาน Commerce นอกจากนี้คุณต้องมีบัญชีผู้ใช้ Azure Active Directory (Azure AD) ที่มีการบอกรับเป็นสมาชิก Azure

ดำเนินการตามขั้นตอนต่อไปนี้เพื่อตั้งค่าการรวม

1. ในพอร์ทัล Azure ลงทะเบียนโปรแกรมประยุกต์ โปรแกรมประยุกต์นี้จะใช้สำหรับการตรวจสอบความถูกต้องของ Customer Insights สำหรับคำแนะนำ ดู [การเริ่มต้นแบบด่วน: ลงทะเบียนโปรแกรมประยุกต์ด้วย Microsoft identity platform](https://docs.microsoft.com/azure/active-directory/develop/quickstart-register-app)
2. สร้างความลับสำหรับโปรแกรมประยุกต์ จดบันทึกความลับ และเก็บรักษาไว้ในที่ที่ปลอดภัย เพราะคุณจะต้องใช้ในภายหลัง เลือกระยะเวลาการหมดอายุสำหรับความลับ

    > [!IMPORTANT]
    > ดำเนินการตามขั้นตอนเพื่อให้คุณจำไว้ว่าจะต้องเปลี่ยนความลับก่อนที่จะหมดอายุ มิฉะนั้น การรวมจะหยุดทำงานโดยไม่คาดคิด

3. สร้าง Azure Key Vault และบันทึกความลับของโปรแกรมประยุกต์ สำหรับคำแนะนำ ดู [การเริ่มต้นแบบด่วน: ตั้งค่าและดึงข้อมูลความลับจาก Azure Key Vault โดยใช้พอร์ทัล Azure ](https://docs.microsoft.com/azure/key-vault/quick-create-portal)
4. เปิดใช้งานการเข้าถึง Azure Key Vault จาก Commerce เมื่อต้องการทำขั้นตอนนี้ให้เสร็จสมบูรณ์ คุณต้องมีรหัสโปรแกรมประยุกต์ และความลับ โปรแกรมประยุกต์อาจเป็นโปรแกรมประยุกต์เดียวกันกับที่คุณสร้างขึ้น ในขั้นตอนที่ 1 หรืออาจเป็นโปรแกรมประยุกต์ใหม่ก็ได้ (กล่าวคือ คุณสามารถใช้โปรแกรมประยุกต์ที่คุณสร้างขึ้นในขั้นตอนที่ 1 สำหรับทั้งการเข้าถึง Key Vault และ Customer Insights หรือคุณสามารถสร้างโปรแกรมประยุกเฉพาะ สำหรับการเข้าถึงแต่ละชนิดได้) สำหรับคำแนะนำ ดู [จัดเก็บข้อมูลประจำตัวหลัก ใน Azure Stack Key Vault](https://docs.microsoft.com/azure-stack/user/azure-stack-key-vault-store-credentials?view=azs-1908#create-a-service-principal)
5. ในศูนย์ควบคุม ให้ไปที่ **การจัดการระบบ \> ตั้งค่า \> พารามิเตอร์ Key Vault** และป้อนข้อมูลที่จำเป็นสำหรับ key vault จากนั้นใน ฟิลด์ **ไคลเอนต์ของ Key Vault** ให้ป้อนรหัสของโปรแกมประยุกต์ที่คุณใช้ในขั้นตอนที่ 4 เพื่อให้ Commerce สามารถเข้าถึงความลับใน Key Vault
6. เมื่อต้องการเพิ่มแอปพลิเคชันที่คุณสร้างขึ้นในขั้นตอนที่ 1 ไปยังรายการของแอปพลิเคชันที่ปลอดภัย (บางครั้งเรียกว่า รายการที่ปลอดภัย) ให้ไปที่ Customer Insights และให้การเข้าถึง **มุมมอง** แก่แอปพลิเคชัน สำหรับคำแนะนำ ให้ดูที่ [สิทธิ์](https://docs.microsoft.com/dynamics365/ai/customer-insights/pm-permissions)
7. ใน Commerce บนหน้า **พารามิเตอร์ Commerce** ในแท็บ **ความสัมพันธ์อันดีกับลูกค้า** บน FastTab **Dynamics 365 Customer Insights** ให้ทำตามขั้นตอนต่อไปนี้

    1. ในฟิลด์ **รหัสแอปพลิเคชัน** ให้ป้อนรหัสของแอปพลิเคชันที่คุณใช้ในขั้นตอนที่ 1
    2. ในฟิลด์ **ชื่อลับ** ให้ป้อนชื่อ Key Vault ที่เป็นความลับที่คุณสร้างขึ้นในขั้นตอนที่ 5
    3. ตั้งค่าตัวเลือก **เปิดใช้งาน Customer Insights** เป็น **ใช่** ถ้าการตั้งค่าไม่สำเร็จเนื่องจากสาเหตุใดก็ตาม คุณจะได้รับข้อความแสดงข้อผิดพลาดและตัวเลือกนี้จะถูกตั้งค่าเป็น **ไม่มี**
    4. คุณสามารถมีสภาพแวดล้อมหลายอย่างใน Customer Insights เช่น สภาพแวดล้อมการทดสอบและการผลิต ในฟิลด์ **รหัสอินสแตนซ์ของสภาพแวดล้อม** ให้ป้อนสภาพแวดล้อมที่เหมาะสม
    5. ในฟิลด์ **รหัสลูกค้าอื่น** ให้ป้อนคุณสมบัติใน Customer Insights ที่ถูกแม็ปกับหมายเลขบัญชีลูกค้า (ใน Commerce หมายเลขบัญชีลูกค้าคือรหัสลูกค้า)
    6. คุณสมบัติที่เหลือสามอย่างคือการวัดที่จะแสดงบนบัตรลูกค้าในสมุดบัญชีลูกค้า คุณสามารถเลือกได้สูงสุดสามมาตรการที่จะให้แสดงบนบัตรลูกค้า (อย่างไรก็ตาม คุณไม่จำเป็นต้องเลือกการวัดใดๆ) ดังที่กล่าวถึงก่อนหน้านี้ ระบบจะแสดงค่าเหล่านี้ก่อนแล้วจึงแสดงค่าของกลุ่มแอตทริบิวต์ของสมุดบัญชีลูกค้า
