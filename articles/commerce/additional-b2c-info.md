# <a name="additional-b2c-information"></a>ข้อมูล B2C เพิ่มเติม

[!include [banner](includes/banner.md)]

บทความนี้ให้ข้อมูลเพิ่มเติมเกี่ยวกับวิธีการตั้งค่าผู้เช่าแบบธุรกิจ-ผู้บริโภค (B2C) Microsoft Dynamics 365 Commerce ของคุณ

### <a name="customer-migration"></a>การโยกย้ายลูกค้า

ถ้าคุณกำลังพิจารณาการย้ายเรกคอร์ดลูกค้าจากแพลตฟอร์มของผู้ให้บริการข้อมูลประจำตัวก่อนหน้า โปรดทำงานกับทีม Dynamics 365 Commerce เพื่อตรวจสอบความต้องการการโยกย้ายลูกค้าของคุณ

สำหรับเอกสาร Azure AD B2C เพิ่มเติมสำหรับการโยกย้ายลูกค้า โปรดดู [โยกย้ายผู้ใช้ไปยัง Azure Active Directory B2C](/azure/active-directory-b2c/active-directory-b2c-user-migration)การ B2C

### <a name="custom-policies"></a>นโยบายที่กำหนดเอง

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการปรับแต่งการโต้ตอบ Azure AD B2C และขั้นตอนนโยบายนอกเหนือจากที่มีนำเสนอโดยนโยบายมาตรฐานของ B2C โปรดดู [นโยบายที่กำหนดเองใน Azure Active Directory B2C](/azure/active-directory-b2c/active-directory-b2c-overview-custom) 

### <a name="secondary-admin"></a>ผู้ดูแลระบบรอง

คุณสามารถเพิ่มบัญชีผู้ดูแลระบบรองหรือไม่ก็ได้ในส่วน **ผู้ใช้** ของผู้เช่า B2C ของคุณ นี่อาจเป็นบัญชีโดยตรงหรือบัญชีทั่วไป ถ้าคุณต้องการแบ่งปันบัญชีระหว่างทรัพยากรทีมร่วมกัน ยังสามารถสร้างบัญชีทั่วไปได้ด้วย เนื่องจากความสำคัญของข้อมูลที่จัดเก็บไว้ใน Azure AD B2C ควรมีการตรวจสอบความถูกต้องของบัญชีทั่วไปอย่างใกล้ชิดตามแนวทางปฏิบัติด้านความปลอดภัยของบริษัทของคุณ

### <a name="set-up-a-custom-sign-in-domain"></a>ตั้งค่าโดเมนการลงชื่อเข้าใช้ที่กำหนดเอง

Azure AD B2C อนุญาตให้คุณตั้งค่าโดเมนการลงชื่อเข้าใช้ที่กำหนดเองของผู้เช่า Azure AD B2C ได้ สหรับคำแนะนำ ดูที่ [เปิดใช้งานโดเมนที่กำหนดเองสำหรับ Azure Active Directory B2C](/azure/active-directory-b2c/custom-domain) 

ถ้าคุณใช้โดเมนการลงชื่อเข้าใช้ที่กำหนดเอง ต้องป้อนโดเมนลงในโปรแกรมสร้างไซต์ Commerce

เมื่อต้องการป้อนโดเมนการลงชื่อเข้าใช้ที่กำหนดเองในโปรแกรมสร้างไซต์ ให้ทำตามขั้นตอนเหล่านี้

1. ที่มุมบนขวาของโปรแกรมสร้างไซต์ ให้เลือกตัวสลับไซต์ จากนั้นเลือก **จัดการไซต์**
1. ในบานหน้าต่างนำทางด้านซ้าย ให้เลือก **การตั้งค่าผู้เช่า \> การตั้งค่าการรับรองความถูกต้องของไซต์**
1. ในส่วน **โพรไฟล์การรับรองความถูกต้องของไซต์** ให้เลือก **จัดการ**
1. ในเมนูลอยทางด้านขวา ให้เลือกปุ่ม **แก้ไข** (เครื่องหมายรูปดินสอ) ถัดจากโพรไฟล์การรับรองความถูกต้องของไซต์ที่คุณต้องการป้อนโดเมนที่กำหนดเอง
1. ในกล่องโต้ตอบ **แก้ไขโพรไฟล์การรับรองความถูกต้องของไซต์** ภายใต้ **โดเมนที่กำหนดเองสำหรับการเข้าสู่ระบบ** ป้อนโดเมนการลลงชื่อเข้าใช้ที่กำหนดเอง (ตัวอย่างเช่น 'login.fabrikam.com')

> [!WARNING]
> เมื่อคุณอัปเดตโดเมนที่กำหนดเองของผู้เช่า Azure AD B2C การเปลี่ยนแปลงจะส่งผลต่อรายละเอียดผู้อนุมัติของผู้เช่าในโทเค็นที่สร้างขึ้น รายละเอียดผู้อนุมัติจะรวมโดเมนที่กำหนดเองแทนโดเมนเริ่มต้นที่ระบุโดย Azure AD B2C การตั้งค่าคอนฟิก **ผู้อนุมัติ** ที่แตกต่างกันใน Commerce headquarters (**การขายปลีกและการค้า \> การตั้งค่าศูนย์ควบคุม \> พารามิเตอร์ \> พารามิเตอร์ที่ใช้ร่วมกันของ Commerce \> ผู้ให้บริการข้อมูลประจำตัว**) จะเปลี่ยนการโต้ตอบของระบบกับผู้ใช้ไซต์ ซึ่งอาจทำให้มีการสร้างเรกคอร์ดลูกค้าใหม่ถ้าผู้ใช้รับรองความถูกต้องกับผู้อนุมัติใหม่ การเปลี่ยนแปลงโดเมนที่กำหนดเองควรได้รับการทดสอบอย่างถี่ถ้วนก่อนที่จะสลับเป็นโดเมนที่กำหนดเองในสภาพแวดล้อม Azure AD B2C ที่ใช้งานจริง
