# <a name="configure-sales-tax-for-online-orders"></a>ตั้งค่าคอนฟิกภาษีขายสำหรับใบสั่งออนไลน์

[!include [banner](../includes/banner.md)]
[!include [banner](includes/preview-banner.md)]

หัวข้อนี้แสดงภาพรวมของการเลือกกลุ่มภาษีขายสำหรับชนิดใบสั่งออนไลน์ต่าง ๆ 

ช่องทางอิเล็กทรอนิกส์ของคุณอาจต้องการสนับสนุนตัวเลือก เช่น การจัดส่งหรือการเบิกสินค้าสำหรับใบสั่งซื้อออนไลน์ ความเกี่ยวข้องของภาษีขายจะขึ้นอยู่กับตัวเลือกที่ผู้ใช้ออนไลน์เลือกไว้ เมื่อลูกค้าเลือกที่จะซื้อสินค้าออนไลน์และได้รับการจัดส่งไปยังที่อยู่ จะมีการกำหนดภาษีขายตามการตั้งค่ากลุ่มภาษีที่อยู่ในการจัดส่งของลูกค้า เมื่อลูกค้าต้องการรับสินค้าที่ซื้อที่ร้านค้า จะมีการกำหนดภาษีขายตามการตั้งค่ากลุ่มภาษีของร้านค้า 

## <a name="orders-shipped-to-a-customer-address"></a>จัดส่งคำสั่งซื้อไปยังที่อยู่ลูกค้าแล้ว 

โดยทั่วไป ภาษีสำหรับใบสั่งซื้อออนไลน์ที่ที่อยู่ของลูกค้าจะถูกกำหนดโดยปลายทาง กลุ่มภาษีขายแต่ละกลุ่มจะมีการตั้งค่าคอนฟิกการจัดทำภาษีตามปลายทางการขายปลีก ซึ่งธุรกิจของคุณสามารถกำหนดรายละเอียดปลายทาง เช่น เขต/ภูมิภาค รัฐ เขต และเมืองในฟอร์มลำดับชั้น เมื่อมีการวางใบสั่งออนไลน์ ระบบภาษี Commerce จะใช้ที่อยู่ที่จัดส่งของสินค้าในบรรทัดทั้งหมดในใบสั่ง และค้นหากลุ่มภาษีขายที่มีเกณฑ์ภาษีตามปลายทางที่ตรงกัน ตัวอย่างเช่น สำหรับใบสั่งออนไลน์ที่มีที่อยู่ที่จัดส่งสินค้าในบรรทัดไปยังซานฟรานซิสโก รัฐแคลิฟอร์เนีย กลไกจัดการภาษีจะค้นหากลุ่มภาษีขายและรหัสภาษีขายสำหรับรัฐแคลิฟอร์เนียแล้วคำนวณภาษีสำหรับสินค้าแต่ละรายการตามต้องการ  

## <a name="customer-based-tax-groups"></a>กลุ่มภาษีตามลูกค้า

ในศูนย์ควบคุม Commerce มีสองสถานที่ที่มีการตั้งค่าคอนฟิกกลุ่มภาษีของลูกค้า:

- **โพรไฟล์ของลูกค้า**
- **ที่อยู่จัดส่งของลูกค้า**

### <a name="if-a-customers-profile-has-a-tax-group-configured"></a>ถ้าโพรไฟล์ของลูกค้ามีกลุ่มภาษีที่มีการตั้งค่าคอนฟิก

เรกคอร์ดของโพรไฟล์ของลูกค้าใน Retail Headquarters อาจมีการตั้งค่าคอนฟิกกลุ่มภาษีขายไว้อย่างไรก็ตามสำหรับใบสั่งซื้อที่มีการตั้งค่าคอนฟิกกลุ่มภาษีขายในโพรไฟล์ของลูกค้าจะไม่ถูกใช้โดยกลไกจัดการภาษี 

### <a name="if-a-customers-shipping-address-has-a-tax-group-configured"></a>ถ้าที่อยู่จัดส่งของลูกค้ามีกลุ่มภาษีที่มีการตั้งค่าคอนฟิก

ถ้าเรกคอร์ดที่อยู่ที่จัดส่งของลูกค้ามีกลุ่มภาษีถูกตั้งค่าคอนฟิกและมีการจัดส่งสินค้าตามใบสั่งซื้อทางออนไลน์ (หรือสินค้าในรายการ) ให้กับที่อยู่ที่จัดส่งของลูกค้า กลุ่มภาษีที่ตั้งค่าคอนฟิกไว้ในเรกคอร์ดที่อยู่ของลูกค้าจะถูกใช้โดยกลไกจัดการภาษีสำหรับการคำนวณภาษี

#### <a name="configure-a-tax-group-for-a-customers-shipping-address-record"></a>ตั้งค่าคอนฟิกกลุ่มภาษีสำหรับเรกคอร์ดที่อยู่จัดส่งของลูกค้า

เมื่อต้องการตั้งค่าคอนฟิกกลุ่มภาษีสำหรับเรกคอร์ดที่อยู่ที่จัดส่งของลูกค้าในศูนย์ควบคุม Commerce ให้ทำตามขั้นตอนต่อไปนี้

1. ไปที่ **ลูกค้าทั้งหมด** แล้วเลือกลูกค้าที่ต้องการ 
1. บนแท็บด่วน **ที่อยู่** ให้เลือกที่อยู่ที่ต้องการ แล้วเลือก **ตัวเลือกเพิ่มเติม \> ขั้นสูง** 
1. ภายใต้แท็บ **ทั่วไป** บนหน้า **จัดการที่อยู่** ให้กำหนดมูลค่าภาษีขายตามต้องการ

> [!NOTE]
> มีการกำหนดกลุ่มภาษีโดยใช้ที่อยู่ที่จัดส่งของบรรทัดใบสั่งและภาษีตามปลายทางจะได้รับการตั้งค่าคอนฟิกที่กลุ่มภาษีเอง สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [ตั้งค่าภาษีสำหรับร้านค้าออนไลน์ตามปลายทาง](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/set-up-taxes-for-online-stores-based-on-destination)

## <a name="order-pickup-in-store"></a>การเบิกสินค้าตามใบสั่งในร้านค้า

สำหรับรายการใบสั่งที่มีการเบิกสินค้าในร้านค้าหรือเท้าที่ระบุ กลุ่มภาษีจากร้านค้าเบิกสินค้าที่เลือกจะถูกนำไปใช้ สำหรับรายละเอียดเกี่ยวกับวิธีการตั้งค่าคอนฟิกกลุ่มภาษีสำหรับร้านค้าที่กำหนด ให้ดูที่ [ตั้งค่าตัวเลือกภาษีอื่นๆสำหรับร้านค้า](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/set-other-tax-options-for-stores)

> [!NOTE]
> เมื่อมีการเบิกสินค้าตามบรรทัดใบสั่งที่ร้านค้า การตั้งค่าภาษีที่อยู่ของลูกค้า (ถ้าตั้งค่า) จะถูกละเว้นโดยกลไกจัดการภาษี และการตั้งค่าคอนฟิกภาษีของร้านค้าเบิกสินค้าจะถูกนำไปใช้
