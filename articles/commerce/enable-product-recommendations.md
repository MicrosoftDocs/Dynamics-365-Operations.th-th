# <a name="enable-product-recommendations"></a>เปิดใช้งานคำแนะนำผลิตภัณฑ์

[!include [banner](includes/banner.md)]

บทความนี้จะอธิบายถึงวิธีการให้คำแนะนำเกี่ยวกับผลิตภัณฑ์ที่ยึดตามการเรียนรู้ของเครื่องมือปัญญาประดิษฐ์ (AL-ML) ที่มีให้ใช้งานสำหรับลูกค้า Microsoft Dynamics 365 Commerce สำหรับข้อมูลเพิ่มเติมเกี่ยวกับคำแนะนำผลิตภัณฑ์ โปรดอ่าน [ภาพรวมคำแนะนำผลิตภัณฑ์](product-recommendations.md)

## <a name="recommendations-pre-check"></a>การตรวจสอบคำแนะนำ

1. ตรวจสอบให้แน่ใจว่าคุณมีลิขสิทธิ์คำแนะนำ Dynamics 365 Commerce ที่ถูกต้อง
1. ตรวจสอบให้แน่ใจว่าร้านค้าเอนทิตีเชื่อมต่อกับบัญชี Azure Data Lake Storage Gen2 ของลูกค้า สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [ตรวจสอบให้แน่ใจว่ามีการซื้อ Azure Data Lake Storage และตรวจสอบความถูกต้องในสภาพแวดล้อมเรียบร้อยแล้ว](enable-ADLS-environment.md)
1. ยืนยันว่าการตั้งค่าคอนฟิกข้อมูลเฉพาะตัว Azure AD มีรายการสำหรับคำแนะนำ ข้อมูลเพิ่มเติมเกี่ยวกับวิธีการดำเนินการนี้อยู่ด้านล่าง
1. ตรวจสอบให้แน่ใจว่าการรีเฟรชรายวันของการจัดเก็บเอนทิตีไปยัง Azure Data Lake Storage Gen2 ได้รับการจัดกำหนดการแล้ว สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [ตรวจสอบให้แน่ใจว่ามีการรีเฟรชที่จัดเก็บเอนทิตี้โดยอัตโนมัติ](../fin-ops-core/dev-itpro/data-entities/entity-store-data-lake.md)
1. เปิดใช้งานการวัด RetailSale ของร้านค้าเอนทิตี สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการตั้งค่ากระบวนการนี้ ให้ดูที่ [การทำงานกับการวัด](/dynamics365/ai/customer-insights/pm-measures)
1. ตรวจสอบให้แน่ใจว่าสภาพแวดล้อมของคุณได้ตั้งค่าคอนฟิกภูมิภาคที่ใช้อยู่และภูมิภาคหนึ่งๆ ในภูมิภาคที่สนับสนุนในปัจจุบัน ดังนี้

    - **ภูมิภาคคุกกี้ที่สนับสนุน:** EU/US/CA/AU
    - **ภูมิภาคการบริการที่สนับสนุน:** US/CA/AU ถ้าภูมิภาคที่บริการนั้นไม่ตรงกับภูมิภาคใดภูมิภาคหนึ่งที่สนับสนุนที่มีอยู่ บริการที่แนะนำใช้จะเลือกภูมิภาคที่สนับสนุนที่ใกล้ที่สุด

หลังจากขั้นตอนข้างต้นเสร็จสมบูรณ์แล้ว คุณจะต้องพร้อมเปิดใช้งานคำแนะนำ

> [!NOTE]
> มีปัญหาที่ทราบซึ่งการแนะนำไม่ปรากฏขึ้นหลังจากที่ปฏิบัติตามขั้นตอนต่อไปนี้เสร็จสมบูรณ์แล้ว ปัญหานี้อาจเกิดจากปัญหาโฟลว์ของข้อมูลในสภาพแวดล้อม ถ้าสภาพแวดล้อมของคุณไม่แสดงผลลัพธ์การแนะนำ ให้ตั้งค่าคอนฟิกข้อมูลทางเลือกของบริการข้อแนะนำด้วยการปฏิบัติตามขั้นตอนต่างๆ ใน [ตั้งค่าโฟลว์ข้อมูลทางเลือกสำหรับคำแนะนำ](set-up-alternate-data-flow.md) คุณต้องมีสิทธิ์ระดับผู้ดูแลระบบ Azure เพื่อปฏิบัติตามขั้นตอนต่อไปนี้ให้เสร็จสมบูรณ์ หากคุณต้องการความช่วยเหลือ โปรดติดต่อตัวแทน FastTrack ของคุณ

## <a name="azure-ad-identity-configuration"></a>การตั้งค่าคอนฟิกข้อมูลประจำตัวของ Azure AD

ขั้นตอนนี้จำเป็นสำหรับลูกค้าที่ใช้โครงสร้างแบบอินฟราเรดเป็นการตั้งค่าคอนฟิกการบริการ (IaaS) เท่านั้น การตั้งค่าคอนฟิกข้อมูลประจำตัวของ Azure AD เป็นแบบอัตโนมัติสำหรับให้ลูกค้าเรียกใช้บน Azure Service Fabric แต่ควรตรวจสอบด้วยว่าการตั้งค่าถูกตั้งค่าคอนฟิกตามที่คาดไว้หรือไม่

### <a name="setup"></a>ตั้งค่า

1. ในศูนย์ควบคุม Commerce ให้ค้นหาหน้า **โปรแกรมประยุกต์ Azure Active Directory**
1. ตรวจสอบว่ามีรายการสำหรับ **RecommendationSystemApplication-1** หรือไม่ ถ้าไม่มีรายการอยู่ ให้สร้างรายการโดยใช้ข้อมูลต่อไปนี้:

    - **รหัสไคลเอนต์**: d37b07e8-dd1c-4514-835d-8b918e6f9727
    - **ชื่อ**: RecommendationSystemApplication-1
    - **รหัสผู้ใช้**: RetailServiceAccount

1. บันทึกและปิดหน้า 

## <a name="turn-on-recommendations"></a>เปิดคำแนะนำ

ถ้าต้องการเปิดคำแนะนำผลิตภัณฑ์ ให้ปฏิบัติตามขั้นตอนเหล่านี้

1. ในศูนย์ควบคุม Commerce ให้ค้นหา **การจัดการคุณลักษณะ**
1. เลือก **ทั้งหมด** เพื่อดูรายการของคุณลักษณะที่มีอยู่ 
1. ในกล่องค้นหา ให้ป้อน **คำแนะนำ**
1. เลือกคุณลักษณะ **คำแนะนำผลิตภัณฑ์**
1. ในบานหน้าต่างคุณสมบัติ **คำแนะนำผลิตภัณฑ์** ให้เลือก **เปิดใช้งานเดี๋ยวนี้**

![กำลังเปิดคำแนะนำ](./media/FeatureManagement_Recommendations.PNG)

> [!NOTE]
> - ขั้นตอนข้างต้นจะเริ่มต้นกระบวนการสร้างรายการคำแนะนำผลิตภัณฑ์ อาจจำเป็นต้องใช้หลายชั่วโมงก่อนที่รายการจะพร้อมใช้งาน และสามารถดูได้จากการขายหน้าร้าน (POS) หรือใน Dynamics 365 Commerce
> - การตั้งค่าคอนฟิกนี้ไม่ได้เปิดใช้งานคุณลักษณะของคำแนะนำทั้งหมด คุณลักษณะขั้นสูง เช่น คำแนะนำแบบส่วนบุคคล "เลือกซื้อสินค้าที่คล้ายกัน" และ "คำอธิบายเลือกซื้อสินค้าที่คล้ายกัน" จะควบคุมโดยรายการการจัดการคุณลักษณะการทำงานเฉพาะ สำหรับข้อมูลเกี่ยวกับการเปิดใช้งานลักษณะการใช้งานเหล่านี้ในศูนย์ควบคุม Commerce โปรดดูที่ [เปิดใช้งานคำแนะนำที่เป็นแบบส่วนบุคคล](personalized-recommendations.md) [เปิดใช้งานคำแนะนำ "เลือกซื้อสินค้าที่คล้ายกัน"](shop-similar-looks.md) และ [เปิดใช้งานคำแนะนำ "คำอธิบายเลือกซื้อสินค้าที่คล้ายกัน"](shop-similar-description.md)

## <a name="configure-recommendation-list-parameters"></a>ตั้งค่าพารามิเตอร์รายการคำแนะนำ

โดยค่าเริ่มต้นรายการคำแนะนำผลิตภัณฑ์ที่ใช้ AI-ML จะแสดงค่าที่แนะนำ คุณสามารถเปลี่ยนค่าเริ่มต้นที่แนะนำเพื่อให้เหมาะสมกับกระบวนการทางธุรกิจของคุณ เมื่อต้องการเรียนรู้เพิ่มเติมเกี่ยวกับวิธีการเปลี่ยนพารามิเตอร์เริ่มต้นให้ไปที่ [การจัดการผลลัพธ์การแนะนำของผลิตภัณฑ์ที่ใช้ AI-ML](modify-product-recommendation-results.md)

## <a name="include-recommendations-in-e-commerce-experiences"></a>รวมคำแนะนำในประสบการณ์อีคอมเมิร์ซ

หลังจากเปิดใช้งานคำแนะนำในศูนย์ควบคุม Commerce แล้ว โมดูล Commerce ใช้เพื่อแสดงผลลัพธ์ของคำแนะนำเกี่ยวกับประสบการณ์อีคอมเมิร์ซที่พร้อมจะตั้งค่าคอนฟิก สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [โมดูลคอลเลกชันผลิตภัณฑ์](product-collection-module-overview.md)

## <a name="show-recommendations-on-pos-devices"></a>แสดงคำแนะนำบนอุปกรณ์ POS

หลังจากเปิดใช้งานคำแนะนำในศูนย์ควบคุม Commerce จะต้องเพิ่มแผงคำแนะนำลงในหน้าจอการควบคุม POS โดยใช้เครื่องมือเค้าโครง หากต้องการเรียนรู้เกี่ยวกับกระบวนการนี้ ให้ดู [เพิ่มการควบคุมคำแนะนำให้กับหน้าจอธุรกรรมบนอุปกรณ์ POS](add-recommendations-control-pos-screen.md) 

## <a name="enable-personalized-recommendations"></a>เปิดใช้งานคำแนะนำที่เป็นแบบส่วนบุคคล

ใน Dynamics 365 Commerce ผู้ค้าปลีกสามารถทำให้คำแนะนำผลิตภัณฑ์แบบส่วนตัวพร้อมใช้งาน (หรืออีกชื่อหนึ่งว่า การตั้งค่าส่วนบุคคล) ด้วยวิธีนี้ คำแนะนำแบบส่วนตัวสามารถรวมเข้ากับประสบการณ์ของลูกค้าออนไลน์และในระบบขายหน้าร้าน เมื่อเปิดใช้งานฟังก์ชั่นการตั้งค่าส่วนบุคคล ระบบจะสามารถเชื่อมโยงข้อมูลการซื้อและผลิตภัณฑ์ของผู้ใช้เพื่อสร้างคำแนะนำผลิตภัณฑ์เฉพาะบุคคล

หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับคำแนะนำแบบส่วนตัว โปรดดูที่ [เปิดใช้งานคำแนะนำแบบส่วนตัว](personalized-recommendations.md)
