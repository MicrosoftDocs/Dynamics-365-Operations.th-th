# <a name="define-and-maintain-retail-channels"></a>กำหนดและรักษาช่องทางการขายปลีก

[!include [banner](includes/banner.md)]

หัวข้อนี้ให้ภาพรวมของกระบวนการสำหรับการตั้งร้านค้าแบบดั้งเดิม ซึ่งเรียกว่าร้านค้าใน Dynamics 365 Commerce รวมไปถึงข้อมูลเกี่ยวกับงานที่คุณต้องทำให้สมบูรณ์ก่อนและหลังการตั้งร้านค้า

การรองรับการค้าหลายช่องทาง เช่น ร้านค้าออนไลน์ ศูนย์บริการ และร้านค้าแบบดั้งเดิม เรียกร้านค้าแบบดั้งเดิมว่าร้านค้า ร้านค้าแต่ละร้านสามารถมีวิธีการชำระเงินของตัวเอง กลุ่มราคา การลงทะเบียนรวมระบบขายหน้าร้าน (POS) บัญชีรายได้และบัญชีรายจ่าย และพนักงาน คุณต้องตั้งค่าองค์ประกอบเหล่านี้สำหรับร้านค้าทั้งหมดก่อนคุณจึงจะสามารถสร้างได้ หลังจากที่คุณสร้างร้านค้าแล้ว คุณมอบหมายผลิตภัณฑ์ที่คุณต้องการให้มีอยู่ คุณยังต้องกำหนดพนักงาน เครื่องบันทึกเงินสด และลูกค้าไปยังร้าน ในตอนท้าย คุณจะเพิ่มร้านค้าใหม่ไปยังลำดับชั้นขององค์กร

## <a name="setting-up-stores"></a>ตั้งค่าร้านค้า

ก่อนที่คุณจะสามารถตั้งค่าร้านค้าในคอมเมิร์ซ คุณต้องทำงานเบื้องต้นที่ต้องทำให้เสร็จสมบูรณ์ จากนั้นคุณสามารถสร้างร้านค้าและเพิ่มรายละเอียดได้

### <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

คุณต้องทำสิ่งเหล่านี้ให้เสร็จสมบูรณ์ก่อนตั้งค่าร้านค้า:

1. ตั้งค่าคอนฟิกโครงสร้างองค์กรของคุณ และตั้งค่าลำดับชั้นขององค์กรสำหรับการจัดประเภทการขายปลีก การเติมสินค้า และการรายงาน
2. ตั้งค่าคลังสินค้าที่แสดงถึงร้านค้า
3. ตั้งค่าหมายเลขลำดับสำหรับร้านค้า คำชี้แจงร้านค้า และคำชี้แจงใบสำคัญ
4. กำหนดค่าพารามิเตอร์เพื่อการพาณิชย์
5. ตั้งค่าวิธีการชำระเงินที่ร้านค้ายอมรับ
6. เมื่อต้องการประมวลผลธุรกรรมบัตรเครดิตที่เครื่องบันทึกเงินสด POS คุณยังสามารถตั้งค่าการบริการชำระเงินได้ด้วย
7. ตั้งค่ากลุ่มภาษีขาย
8. ตั้งค่าผลิตภัณฑ์ ในฐานะที่เป็นส่วนหนึ่งของงานนี้ คุณยังตั้งค่าลำดับชั้นของผลิตภัณฑ์ ตัวแปรผลิตภัณฑ์ และการจัดประเภทผลิตภัณฑ์
9. ตั้งค่ากลุ่มราคาผลิตภัณฑ์
10. ตั้งค่าราคาผลิตภัณฑ์ เนื่องจากเป็นส่วนหนึ่งของงานนี้ คุณยังตั้งค่าการปรับปรุงราคา ส่วนลด และรอบระยะเวลาส่วนลดได้
11. ตั้งค่าพนักงาน

    > [!NOTE]
    > คุณต้องมอบหมายสิทธิ์ที่เหมาะสมให้กับพนักงาน เพื่อให้พวกเขาสามารถลงชื่อเข้าใช้และทำงานโดยใช้ระบบ POS ได้

12. กำหนดค่าโปรไฟล์ POS เพื่อมอบหมายให้กับร้านค้า งานนี้รวมหลายงานอื่น เช่น การตั้งค่าเครื่องบันทึกเงินสด การตั้งค่าโพรไฟล์ออฟไลน์ และการตั้งค่ารูปแบบใบเสร็จและโพรไฟล์

ตรวจสอบงานทั้งหมดที่รวมอยู่ในข้อกำหนดเบื้องต้น และทำงานที่เกี่ยวข้องเฉพาะกับคุณให้เสร็จสิ้น

### <a name="set-up-a-store"></a>ตั้งค่าร้านค้า

หลังจากที่คุณทำข้อกำหนดเบื้องต้นเสร็จสิ้นแล้ว คุณต้องทำงานเหล่านี้ให้เสร็จสมบูรณ์เพื่อตั้งค่ารายละเอียดสำหรับร้านค้า:

1. สร้างร้านค้า
2. กำหนดกลุ่มภาษีขายให้กับร้านค้า
3. กำหนดวิธีการชำระเงินที่ยอมรับให้กับร้านค้า
4. เพิ่มรายละเอียดลงในคำอธิบายผลิตภัณฑ์สำหรับผลิตภัณฑ์ที่คุณเสนอในร้านค้าของคุณ ตัวอย่างเช่น คุณสามารถเพิ่มรูปแบบ rich text และรูปภาพ รายละเอียดผลิตภัณฑ์เหล่านี้จะปรากฏขึ้นในบริบทต่าง ๆ เช่น บนเครื่องบันทึกเงินสด POS หรือป้ายชื่อที่พิมพ์
5. เพิ่มร้านค้าไปยังลำดับชั้นขององค์กรค่าเริ่มต้น ที่ถูกมอบหมายไปยังวัตถุประสงค์ของ **การจัดประเภทการขายปลีก** **การเติมสินค้าขายปลีก** หรือ **รายงานการขายปลีก**

### <a name="after-you-set-up-a-store"></a>หลังจากที่คุณตั้งค่าร้านค้า

หลังจากคุณป้อนรายละเอียดสำหรับร้านค้า ให้ทำงานเหล่านี้ให้เสร็จสมบูรณ์เพื่อส่งข้อมูลร้านค้าใหม่ไปยัง POS:

1. ตั้งค่าคอนฟิกเครื่องบันทึกเงินสดการขายหน้าร้าน (POS) สำหรับร้านค้า
2. กำหนดการจัดประเภทผลิตภัณฑ์ให้กับร้านค้า
3. การจัดประเภทกระบวนการเพื่อสร้างรายการผลิตภัณฑ์ที่รวมอยู่ในการจัดประเภทและเพื่อให้ผลิตภัณฑ์มีอยู่ในร้านค้า
4. ส่งข้อมูล เช่น ลำดับหมายเลข โปรไฟล์ฮาร์ดแวร์ และเค้าโครงหน้าจอ POS ไปยังการลงทะเบียน POS
5. เผยแพร่ร้านค้าเพื่อส่งข้อมูลร้านค้าไปยัง POS
6. รันงานเพื่อส่งข้อมูลร้านค้าไปยัง POS

## <a name="organization-hierarchies"></a>ลำดับชั้นขององค์กร

การค้าใช้ลำดับชั้นขององค์กรเพื่อจัดโครงสร้างช่องทาง ลำดับชั้นขององค์กรแสดงความสัมพันธ์ระหว่างองค์กรที่สร้างธุรกิจของคุณ เมื่อคุณตั้งค่าร้านค้า คุณสามารถเพิ่มร้านเหล่านั้นเข้าไปในลำดับชั้นขององค์กร ร้านค้าจะแบ่งปันข้อมูลที่ใช้สำหรับการจัดประเภท การเพิ่มเติมสินค้า และการรายงาน

> [!NOTE]
> หากต้องการใช้ฟังก์ชันการขาย Commerce ต้องเปิดใช้งานคีย์การตั้งค่าคอนฟิกสำหรับ **ที่อยู่ที่จัดส่งหลายแห่ง** คีย์การตั้งค่าคอนฟิกนี้สามารถพบได้ในคีย์ **การตั้งค่าคอนฟิกการค้า** ใต้ **การตั้งค่าการดูแลระบบ**\> **ตั้งค่า**\> **การตั้งค่าคอนฟิกลิขสิทธิ์** นี่มีความจำเป็น เนื่องจากการตรวจสอบความถูกต้องต่างๆ ตามที่อยู่ที่จัดส่งที่ตั้งค่าคอนฟิกไว้ที่ระดับรายการใบสั่งขาย
