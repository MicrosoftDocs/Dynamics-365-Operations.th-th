- <span data-ttu-id="6ea04-150">ในช่องจัดเก็บ ถ้าใบสั่งของลูกค้าสำหรับการเบิกสินค้าจะถูกสร้างขึ้นโดยใช้แอพลิเคชันขายหน้าร้าน (POS) สมาคมการขายได้รับการพร้อมท์ให้เลือกจากวิธีการจัดส่งของการจัดส่งที่พร้อมใช้งาน ถ้ามีการตั้งค่าคอนฟิกมีการตั้งค่าคอนฟิกไว้</span><span class="sxs-lookup"><span data-stu-id="6ea04-150">In store channels, if a customer order for pickup is created through the point of sale (POS) application, the sales associate is prompted to choose among the available pickup delivery modes, if any have been configured.</span></span> <span data-ttu-id="6ea04-151">ถ้ามีวิธีการจัดส่งสินค้าที่ใช้ได้เพียงหนึ่งวิธีเท่านั้นที่สามารถใช้ได้สำหรับช่องทางและสินค้า สมาคมการขายจะไม่มีการแสดงพร้อมท์ให้เลือก</span><span class="sxs-lookup"><span data-stu-id="6ea04-151">If only one valid pickup delivery mode is available for the channel and item, the sales associate isn't prompted to select it.</span></span> <span data-ttu-id="6ea04-152">วิธีการจัดส่งสินค้าที่มีการเบิกสินค้าที่มีอยู่จะถูกนำไปใช้กับบรรทัดใบสั่งโดยอัตโนมัติ</span><span class="sxs-lookup"><span data-stu-id="6ea04-152">Instead, the available pickup delivery mode is automatically applied to the order lines.</span></span>

- <span data-ttu-id="6ea04-154">ในช่องทางของศูนย์บริการ เมื่อผู้ใช้สร้างใบสั่งการเบิกสินค้า ผู้ใช้จะสามารถเลือกวิธีการจัดส่งที่กำหนดไว้ใด ๆ ที่มีการเชื่อมโยงกับช่องทางของศูนย์บริการได้ด้วยตนเอง</span><span class="sxs-lookup"><span data-stu-id="6ea04-154">In call center channels, when users create pickup orders, they can manually select any defined pickup delivery mode that is linked to the call center channel.</span></span> <span data-ttu-id="6ea04-155">ระบบจะตรวจสอบว่าสามารถใช้วิธีการจัดส่งสินค้าที่เบิกสินค้าที่เลือกเมื่อมีการสั่งสินค้าที่เชื่อมโยงกับสินค้านี้</span><span class="sxs-lookup"><span data-stu-id="6ea04-155">The system then validates that the selected pickup delivery mode can be used when the item that is being linked to it is ordered.</span></span> <span data-ttu-id="6ea04-156">เมื่อมีการเลือกวิธีการจัดส่งสินค้าที่เบิกสินค้าในช่องทางของศูนย์บริการ บรรทัดใบสั่งขายต้องเชื่อมโยงกับคลังสินค้าของร้านค้าที่ถูกต้อง</span><span class="sxs-lookup"><span data-stu-id="6ea04-156">When a pickup delivery mode is selected in call center channels, the sales order lines must be linked to a valid store warehouse.</span></span> <span data-ttu-id="6ea04-157">ถ้ามีการกำหนดคลังสินค้าที่ไม่ใช่ร้านค้าบนบรรทัดการขายของศูนย์บริการ วิธีการจัดส่งที่เบิกสินค้าไม่สามารถตั้งค่าในรายการขายนั้นได้</span><span class="sxs-lookup"><span data-stu-id="6ea04-157">If a non-store warehouse is defined on a call center sales line, a pickup delivery mode can't be set on that sales line.</span></span>

## <a name="considerations-for-distributed-order-management"></a><span data-ttu-id="6ea04-163">การตั้งค่าคอนฟิกสำหรับการจัดการใบสั่งแบบกระจาย</span><span class="sxs-lookup"><span data-stu-id="6ea04-163">Considerations for distributed order management</span></span>

<span data-ttu-id="6ea04-164">ลักษณะการทำงาน [การจัดการใบสั่งแบบกระจาย (DOM)](./dom.md) ใน Commerce ละเว้นบรรทัดการขายใด ๆ ที่ทำเครื่องหมายไว้สำหรับการเบิกสินค้าของร้านค้า</span><span class="sxs-lookup"><span data-stu-id="6ea04-164">The [distributed order management (DOM)](./dom.md) features in Commerce ignore any sales lines that are marked for store pickup.</span></span> <span data-ttu-id="6ea04-165">ลักษณะการทำงานเหล่านี้ได้รับการอัพเดตแล้วเพื่อให้แน่ใจว่าบรรทัดการขายที่เชื่อมโยงกับวิธีการจัดส่งของการจัดส่งที่ตั้งค่าคอนฟิกข้ามตรรกะ DOM และจะไม่ถูกจัดสรรไปยังคลังสินค้าการเติมสินค้าใหม่</span><span class="sxs-lookup"><span data-stu-id="6ea04-165">These features have been updated to ensure that sales lines that are linked to configured pickup delivery modes bypass the DOM logic and won't be reallocated to a new fulfillment warehouse.</span></span>
