# <a name="security-and-role-management-in-attract"></a>การจัดการความปลอดภัยและบทบาทใน Attract

[!include[banner](../includes/banner.md)]

Microsoft Dynamics 365 for Talent: Attract ใช้ความปลอดภัยตามบทบาท กล่าวอีกอย่างหนึ่งคือ ไม่ได้อนุญาตการเข้าถึงให้แก่ผู้ใช้แต่ละคน แต่ให้กับบทบาทความปลอดภัยที่กำหนดให้กับผู้ใช้ ผู้ใช้ที่ถูกกำหนดให้กับบทบาทความปลอดภัย มีสิทธิ์เข้าถึงชุดของสิทธิ์ที่สัมพันธ์กับบทบาทนั้น

Attract แสดงบทบาทของผู้ใช้พื้นฐานห้ารายการ:

- ผู้ดูแลระบบ
- ผู้จัดการที่ว่าจ้าง
- ผู้สรรหาบุคลากร
- ผู้สัมภาษณ์
- อ่านอย่างเดียว

บทบาทผู้ดูแลระบบเป็นเพียงบทบาทที่มีสิทธิ์ในการเพิ่มผู้ใช้รายอื่น และเปลี่ยนสิทธิ์ของพวกเขา

- **เพิ่ม** – ในศูนย์การจัดการบนแท็บ **สิทธิ์ผู้ใช้** เลือก **กำหนดบทบาท** ค้นหาผู้ใช้เพื่อเพิ่ม และจากนั้นกำหนดสิทธิ์ให้แก่ผู้ใช้นั้น
- **แก้ไข** – ค้นหาผู้ใช้ หรือค้นหาผู้ใช้ในรายการ แล้วจากนั้นเลือก **แก้ไข** เพื่อเปลี่ยนแปลงสิทธิ์ของเขาหรือเธอ
- **ลบ** – ถ้าคุณลบสิทธิ์ของผู้ใช้ คุณไม่ลบผู้ใช้ออกจากระบบ อย่างไรก็ตาม คุณจำกัดการเข้าถึงของผู้ใช้และสิทธิ์ใน Attract ตัวอย่างเช่น Hilda มีการกำหนดให้กับบทบาทผู้จัดการว่าจ้าง และเธอจะถูกเพิ่มเข้าในงานในฐานะผู้จัดการว่าจ้าง ถ้า Hilda จะถูกลบออกจากบทบาทผู้จัดการว่าจ้างในภายหลัง เธอยังคงเป็นผู้จัดการว่าจ้างในงาน และยังคงมีสิทธิ์เข้าถึงงานนั้น อย่างไรก็ตาม เธอไม่สามารถสร้างงานอื่นๆ ได้

ส่วนต่อไปนี้ให้คำอธิบายระดับสูงของแต่ละบทบาท ตารางต่อไปในหัวข้อให้ข้อมูลโดยละเอียดเพิ่มเติม

> [!NOTE]
> คุณลักษณะบางรายการพร้อมใช้งานเฉพาะกับ add-on การว่าจ้างที่ครอบคลุมสำหรับ Attract

## <a name="administrator"></a>ผู้ดูแลระบบ

ผู้ใช้ที่ถูกกำหนดให้กับบทบาทผู้ดูแลระบบ สามารถเข้าถึงและเปลี่ยนข้อมูลทั้งหมดใน Attract ผู้ดูแลระบบสามารถสร้าง อ่าน อัพเดต และลบข้อมูล นอกจากนี้ พวกเขาจะยังสามารถเข้าถึงศูนย์ผู้ดูแลระบบ ซึ่งพวกเขาสามารถตั้งค่าคอนฟิกแอพลิเคชัน Attract และตั้งค่าข้อมูลผู้ใช้ เราขอแนะนำให้บุคคลอย่างน้อยหนึ่งรายถูกกำหนดให้กับบทบาทผู้ดูแลระบบ โดยค่าเริ่มต้น ผู้ดูแลระบบสภาพแวดล้อมใน Microsoft PowerApps ถูกตั้งค่าเป็นผู้ดูแลระบบใน Attract ถ้าคุณลงทะเบียนไว้สำหรับรุ่นทดลองของ Attract บทบาทผู้ดูแลระบบจะถูกกำหนดให้กับคุณโดยอัตโนมัติ ในปัจจุบัน เพื่อสร้างงาน ผู้ใช้ที่มีบทบาทผู้ดูแลระบบยังต้องมีบทบาทผู้สรรหาหรือบทบาทผู้จัดการว่าจ้าง อย่างใดอย่างหนึ่งด้วย

## <a name="hiring-manager"></a>ผู้จัดการที่ว่าจ้าง

ผู้ใช้ที่ถูกกำหนดให้กับบทบาทผู้จัดการว่าจ้าง สามารถสร้างงาน และอัพเดตงานที่พวกเขาสร้างขึ้นก่อนหน้านี้ ผู้จัดการว่าจ้างสามารถดำเนินการได้เฉพาะชุดที่จำกัดของการดำเนินการงาน และในใบสมัครที่เชื่อมโยงกับงานนั้น สามารถเพิ่มเฉพาะผู้ใช้ที่ถูกกำหนดให้กับบทบาทผู้จัดการว่าจ้างให้กับทีมงานที่ว่าจ้างเป็นผู้จัดการว่าจ้าง

## <a name="recruiter-role"></a>บทบาทผู้สรรหา

ผู้ใช้ที่ถูกกำหนดให้กับบทบาทผู้สรรหามีสิทธิ์ในการอ่าน สร้าง อัพเดต และลบทั้งหมดสำหรับงานที่พวกเขาสร้างขึ้น นอกจากนี้ พวกเขายังมีสิทธิ์ในการสร้าง อ่าน อัพเดต และลบทั้งหมดสำหรับใบสมัครที่เชื่อมโยงกับงานที่พวกเขาเป็นเจ้าของ สามารถเพิ่มเฉพาะผู้ใช้ที่ถูกกำหนดให้กับบทบาทผู้สรรหาให้กับทีมงานที่ว่าจ้างเป็นผู้สรรหา

## <a name="interviewer"></a>ผู้สัมภาษณ์

สามารถเพิ่มผู้ใช้ใดๆ ที่มีบัญชี Microsoft Azure Active Directory (Azure AD) ในองค์กร ให้กับทีมงานที่ว่าจ้างเป็นผู้สัมภาษณ์ ผู้ใช้ที่กำหนดให้กับบทบาทผู้สัมภาษณ์สามารถดูรายละเอียดของงานและข้อมูลผู้สมัคร สำหรับงานที่พวกเขาต้องอยู่ในทีมงานสำหรับการว่าจ้าง สำหรับงานเหล่านั้น ผู้สัมภาษณ์ยังสามารถสร้างคำแนะนำการว่าจ้าง และให้ผลป้อนกลับเกี่ยวกับผู้สมัครได้ด้วย อย่างไรก็ตาม พวกเขาจะไม่สามารถปรับปรุงรายละเอียดงานหรือข้อมูลผู้สมัครได้

## <a name="read-only"></a>อ่านอย่างเดียว

ผู้ใช้ที่ถูกกำหนดให้กับบทบาทแบบอ่านอย่างเดียว มีการเข้าถึงแบบอ่านอย่างเดียวไปยังข้อมูลทั้งหมดในสภาพแวดล้อม Attract อย่างไรก็ตาม พวกเขาจะไม่สามารถสร้าง หรือแก้ไขข้อมูลใดๆ ได้

## <a name="delegated-roles"></a>บทบาทที่ได้รับมอบหมาย

สำหรับงานแต่ละงานที่พวกเขาต้องอยู่ในทีมงานว่าจ้าง ผู้สรรหาและผู้จัดการว่าจ้างสามารถกำหนดผู้รับมอบสิทธิ์ได้อย่างน้อยหนึ่งรายสำหรับพวกเขาเอง อย่างไรก็ตาม พวกเขาไม่สามารถกำหนดผู้รับมอบสิทธิ์สำหรับบุคคลอื่นในทีมงานว่าจ้างได้

ผู้รับมอบสิทธิ์มีสิทธิ์เดียวกันกับบุคคลที่กำหนดให้พวกเขา ตัวอย่างเช่น ถ้ผู้จัดการว่าจ้างกำหนดผู้รับมอบสิทธิ์สำหรับตัวเธอเองสำหรับงาน ผู้รับมอบสิทธิ์จะมีสิทธิ์เดียวกันเป็นผู้จัดการว่าจ้างสำหรับงานนั้น ผู้รับมอบสิทธิ์ไม่สามารถลบผู้รับมอบสิทธิ์อื่นๆ จากทีมงานว่าจ้างได้ นอกจากนี้ พวกเขาจะไม่สามารถลบบุคคลที่กำหนดให้พวกเขาเป็นผู้รับมอบสิทธิ์ได้

\*\*ตัวเลือกการตั้งค่าคอนฟิกใน [การตั้งค่ากิจกรรมผลป้อนกลับ](./activities-attract.md#feedback-activity) ควบคุมว่าผู้สัมภาษณ์สามารถดูผลป้อนกลับของผู้อื่นได้หรือไม่

## <a name="process-templates"></a>เท็มเพลตกระบวนการ

สิทธิ์ต่อไปนี้นำไปใช้กับเท็มเพลตกระบวนการจ้างงาน มีการตั้งค่าคอนฟิกความสามารถของสมาชิกในทีมในการสร้างและแก้ไขเท็มเพลตใน **การจัดการเท็มเพลต** ในศูนย์การจัดการ ถ้าเปิดการจัดการเท็มเพลตในผู้สรรหาและผู้จัดการว่าจ้าง สามารถสร้างและแก้ไขเท็มเพลตกระบวนการของตนเองได้ ถ้าการจัดการเท็มเพลตถูกปิด เฉพาะผู้ดูแลระบบเท่านั้นที่สามารถสร้าง และแก้ไขเท็มเพลตกระบวนการได้ ตารางต่อไปนี้ถือว่า การจัดการเท็มเพลตถูกเปิดอยู่

| ข้อมูล              | ผู้สรรหาบุคลากร                                           | ผู้จัดการที่ว่าจ้าง                                      | ผู้สัมภาษณ์ |
|-------------------|-----------------------------------------------------|-----------------------------------------------------|-------------|
| เท็มเพลตกระบวนการ | สิทธิ์ทั้งหมดสำหรับเท็มเพลตที่ผู้ใช้สร้าง | สิทธิ์ทั้งหมดสำหรับเท็มเพลตที่ผู้ใช้สร้าง | ไม่มีการเข้าถึง   |

## <a name="email-and-email-templates"></a>อีเมลและเท็มเพลตอีเมล

สิทธิ์ต่อไปนี้นำไปใช้กับเท็มเพลตอีเมลและการดำเนินการที่สามารถดำเนินการได้ในอีเมล เฉพาะผู้ดูแลระบบเท่านั้นที่สามารถสร้าง และแก้ไขเท็มเพลตอีเมลได้

| ข้อมูลหรือการดำเนินการ     | ผู้สรรหาบุคลากร          | ผู้จัดการที่ว่าจ้าง     | ผู้สัมภาษณ์ |
|--------------------|--------------------|--------------------|-------------|
| เท็มเพลตอีเมล    | การเข้าถึงแบบอ่านอย่างเดียว   | การเข้าถึงแบบอ่านอย่างเดียว   | ไม่มีการเข้าถึง   |
| ส่งอีเมล         | ส่งต่อกิจกรรม  | ส่งต่อกิจกรรม  | ไม่มีการเข้าถึง   |
| แก้ไขเนื้อหาอีเมล | แก้ไขเนื้อหาอีเมล | แก้ไขเนื้อหาอีเมล | ไม่มีการเข้าถึง   |

## <a name="talent-pools"></a>กลุ่มผู้มีความสามารถพิเศษ

สิทธิ์ต่อไปนี้นำไปใช้กับกลุ่ม talent กลุ่ม talent จะมองเห็นได้เฉพาะสำหรับบุคคลที่สร้าง ยกเว้นว่าบุคคลนั้นเลือกที่จะแบ่งปัน สามารถใช้การค้นหาผู้สมัครเพื่อค้นหาผู้สมัครที่ไม่ได้ถูกเพิ่มไปยังกลุ่มที่มีชื่อ

| ข้อมูลหรือการดำเนินการ   | ผู้สรรหาบุคลากร                                       | ผู้จัดการที่ว่าจ้าง                                  | ผู้สัมภาษณ์ |

| กลุ่มที่มีชื่อ       | สิทธิ์ทั้งหมดสำหรับกลุ่มที่ผู้ใช้สร้าง | สิทธิ์ทั้งหมดสำหรับกลุ่มที่ผู้ใช้สร้าง | ไม่มีการเข้าถึง   |
| ใช้กลุ่มร่วมกัน       | เฉพาะกลุ่มที่ผู้ใช้สร้าง                | เฉพาะกลุ่มที่ผู้ใช้สร้าง                | ไม่มีการเข้าถึง   |
| ค้นหาผู้สมัคร | ความสามารถในการค้นหาทั้งหมด                        | ความสามารถในการค้นหาทั้งหมด                        | ไม่มีการเข้าถึง   |
