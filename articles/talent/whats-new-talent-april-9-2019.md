# <a name="whats-new-or-changed-in-dynamics-365-talent-april-9-2019"></a>มีอะไรใหม่หรือการเปลี่ยนแปลงอะไรใน Dynamics 365 Talent (9 เมษายน 2019)

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่หรือที่มีการเปลี่ยนแปลง อย่างใดอย่างหนึ่ง ใน Dynamics 365 Talent

## <a name="changes-in-attract"></a>การเปลี่ยนแปลงใน Attract

### <a name="lifecycle-services-lcs-integration-with-report-a-problem"></a>การรวม Lifecycle Services (LCS) กับ 'รายงานปัญหา'
ใน Attract และ Onboard ปัญหาที่บันทึกไว้โดยผู้ใช้โดยใช้คุณลักษณะรายงานปัญหาจะสร้างปัญหาการสนับสนุนในโครงการ LCS ของลูกค้าโดยอัตโนมัติในขณะนี้ จากนั้นผู้ดูแลระบบสามารถคัดแยกปัญหา และส่งไปยัง Microsoft เมื่อจำเป็น ซึ่งสอดคล้องกับวิธีจัดการปัญหาการสนับสนุนของผู้ใช้ปลายทางของ Core HR

### <a name="relevance-search"></a>การค้นหาความเกี่ยวข้อง
ในกลุ่มผู้มีความสามารถพิเศษ คุณสามารถค้นหาฐานข้อมูลของผู้สมัครทั้งหมดสำหรับทักษะ ชื่อ หรือประวัติการศึกษาเฉพาะได้แล้ว คุณไม่จำเป็นต้องระบุว่าคุณต้องการค้นหาในส่วนใดของโพรไฟล์ของผู้สมัคร Attract จะค้นหาโพรไฟล์ทั้งหมดและเน้นการจับคู่ทั้งหมดที่พบ นอกจากนี้ Attract จะค้นหาเอกสารทั้งหมดที่พร้อมใช้งานสำหรับผู้สมัครและจัดลำดับผลการค้นหาอย่างชาญฉลาดอีกด้วย นอกจากนี้ คุณสามารถกรองข้อมูลผลลัพธ์ตามแหล่งที่มาหรือตามว่าเป็นผู้ได้รับเหรียญเงินหรือไม่ สำหรับข้อมูลเพิ่มเติม โปรดดู [ค้นหาและดูโพรไฟล์ผู้สมัคร](https://docs.microsoft.com/dynamics365/unified-operations/talent/attract-talent-pools#search-and-view-candidate-profiles)

### <a name="prospect-recommendations"></a>คำแนะนำสำหรับผู้สมัครงาน
Attract สามารถช่วยในการเริ่มหางานทันทีที่คุณเรียกใช้โดยการสร้างคำแนะนำสำหรับผู้สมัครที่ชาญฉลาดจากฐานข้อมูลผู้สมัครขององค์กรของคุณ คำแนะนำประกอบด้วยทักษะและข้อมูลการศึกษาที่ระบุขณะที่ค้นหาผู้ที่มีแนวโน้มจะเป็นลูกค้าที่เกี่ยวข้อง คำแนะนำเหล่านี้ปรากฏในแท็บ **ผู้ที่มีแนวโน้มจะเป็นลูกค้า** ภายใต้งาน ถ้าคุณเปิดใช้งานระหว่างกระบวนการว่าจ้างงานของงาน สำหรับข้อมูลเพิ่มเติม ดู [คำแนะนำสำหรับผู้ที่มีแนวโน้มจะเป็นลูกค้า](https://docs.microsoft.com/dynamics365/unified-operations/talent/intelligent-recommendations#prospect-recommendations)

### <a name="interviewer-availability-statuses"></a>สถานะความพร้อมของผู้สัมภาษณ์
ในอีกไม่ช้าตัวกำหนดตารางทำงานการสัมภาษณ์จะสามารถดูสถานะ **ไม่อยู่ที่สำนักงาน ทำงานที่อื่น** สำหรับผู้สัมภาษณ์ เพื่อช่วยในจัดกำหนดเวลาที่อาจทำงานได้ดีขึ้นสำหรับผู้สัมภาษณ์

### <a name="candidate-interview-experience-save-calendar-invites"></a>ประสบการณ์การสัมภาษณ์ผู้สมัคร: บันทึกการเชิญปฏิทิน
ในตอนนี้ผู้สมัครสามารถดาวน์โหลดและบันทึกเหตุการณ์การสัมภาษณ์ของพวกเขาลงในปฏิทินส่วนบุคคลโดยใช้ไฟล์ .ics ที่แนบกับอีเมลสรุปการสัมภาษณ์ที่ส่งไปยังผู้สมัคร

## <a name="changes-in-onboard"></a>การเปลี่ยนแปลงในการเตรียมความพร้อม

### <a name="lifecycle-services-lcs-integration-with-report-a-problem"></a>การรวม Lifecycle Services (LCS) กับรายงานปัญหา
ใน Attract และ Onboard ปัญหาที่บันทึกไว้โดยผู้ใช้โดยใช้คุณลักษณะรายงานปัญหาจะสร้างปัญหาการสนับสนุนในโครงการ LCS ของลูกค้าโดยอัตโนมัติในขณะนี้ จากนั้นผู้ดูแลระบบสามารถคัดแยกปัญหา และส่งไปยัง Microsoft เมื่อจำเป็น ซึ่งสอดคล้องกับวิธีจัดการปัญหาการสนับสนุนของผู้ใช้ปลายทางของ Core HR

## <a name="changes-in-core-hr"></a>การเปลี่ยนแปลงใน Core HR
การเปลี่ยนแปลงที่อธิบายไว้ในส่วนนี้นำไปใช้กับการสร้างหมายเลข 8.1.2225

### <a name="percent-of-basis-variable-plans-load-incorrect-amount"></a>เปอร์เซ็นต์ของจำนวนที่ไม่ถูกต้องของจำนวนงานในศูนย์การผลิตแผนผันแปรพื้นฐาน
การนำออกใช้สัปดาห์นี้ช่วยแก้ไขปัญหาเมื่อโหลดรางวัลค่าตอบแทนผันแปรโดยใช้เปอร์เซ็นต์ของแผนพื้นฐาน
 
### <a name="date-picker-on-last-day-worked-doesnt-work-correctly"></a>ตัวเลือกวันที่ในวันสุดท้ายที่ทำงานทำงานไม่ถูกต้อง
การเปลี่ยนแปลงนี้ช่วยแก้ไขปัญหา: เมื่อผู้ใช้แก้ไข **วันที่สิ้นสุดการจ้างงาน** และเลือกวันที่โดยใช้ตัวควบคุมปฏิทิน วันที่จะถูกเพิ่มลงในสิ่งที่เลือก

###  <a name="limit-personnel-action-types-by-the-action-taken"></a>จำกัดชนิดการดำเนินการด้านบุคลากรตามการดำเนินการ
การเปลี่ยนแปลงนี้จำกัดชนิดการดำเนินการที่ปรากฏขึ้นเมื่อทำการดำเนินการเฉพาะ ตัวอย่างเช่น เมื่อมีการร้องขอตำแหน่งใหม่ เฉพาะการดำเนินการตำแหน่งใหม่ที่จะปรากฏในรายการของการดำเนินการเพื่อเลือก ก่อนหน้านี้ ทั้งการดำเนินการใหม่และที่แก้ไขจะปรากฏขึ้น 

### <a name="transferring-an-employee-with-compensation-in-a-second-legal-entity"></a>การโอนย้ายพนักงานที่มีค่าตอบแทนในนิติบุคคลที่สอง
การนำออกใช้นี้อนุญาตให้ค่าตอบแทนเป็นที่สิ้นสุดในนิติบุคคลที่สองถ้ามีการโอนย้ายสำหรับการโอนย้ายระหว่างบริษัท

### <a name="unable-to-select-compensation-for-a-future-employment-during-a-transfer"></a>ไม่สามารถเลือกค่าตอบแทนสำหรับการจ้างงานในอนาคตในระหว่างการโอนย้ายได้
เมื่อโอนย้ายพนักงานไปยังนิติบุคคลใหม่ คุณสามารถตั้งค่าตอบแทนสำหรับนิติบุคคลใหม่ในระหว่างกระบวนการโอนย้ายได้แล้ว

### <a name="user-isnt-able-to-assign-compensation-during-position-assignment"></a>ผู้ใช้ไม่สามารถกำหนดค่าตอบแทนระหว่างการมอบหมายตำแหน่งงาน
ในตอนนี้การเพิ่มการมอบหมายตำแหน่งงานใหม่ช่วยให้สามารถตั้งค่าเรกคอร์ดค่าตอบแทนคงที่ 

## <a name="in-preview"></a>ในการแสดงตัวอย่าง

### <a name="allow-reason-codes-to-be-specified-on-leave-types"></a>อนุญาตให้ระบุรหัสเหตุผลในชนิดการลางาน
องค์กรอาจต้องการข้อมูลเพิ่มเติมเกี่ยวกับคำขอเวลาหยุดพัก ขณะนี้คุณสามารถระบุรหัสเหตุผลสำหรับชนิดการลางาน และเปิดใช้งานให้พนักงานสามารถเลือกรหัสเหตุผลในคำขอเวลาหยุดพักของพวกเขาได้

### <a name="require-reason-codes-for-certain-leave-types-on-time-off-requests"></a>ต้องมีรหัสเหตุผลสำหรับชนิดการลางานบางชนิดในคำขอเวลาหยุดพัก
องค์กรอาจต้องมีรหัสเหตุผลในชนิดการลางานเฉพาะ เมื่อพนักงานส่งเวลาหยุดพัก ซึ่งอาจเกิดจากนโยบายบริษัทหรือข้อกำหนดที่เป็นระเบียบข้อบังคับ ขณะนี้คุณสามารถระบุชนิดการลางานที่ต้องใช้รหัสเหตุผล และคุณสามารถกำหนดให้พนักงานระบุรหัสเหตุผลสำหรับชนิดการลางานในคำขอเวลาหยุดพักพวกเขา

### <a name="provide-leave-and-absence-transaction-list-for-hr"></a>แสดงรายการธุรกรรมการขาดงานและการลางานสำหรับฝ่ายทรัพยากรบุคคล
การติดตามเวลาหยุดพักของพนักงาน และการทำความเข้าใจวิธีคำนวณเวลาหยุดพักไม่เพียงช่วยฝ่ายทรัพยากรบุคคลในการตอบคำถามพนักงานเท่านั้น แต่ยังช่วยให้มั่นใจถึงรางวัลเวลาหยุดพักที่ถูกต้องสำหรับพนักงานอีกด้วย ขณะนี้ฝ่ายทรัพยากรบุคคลมีมุมมองใหม่เกี่ยวกับธุรกรรม (เงินช่วยเหลือ การค้างรับค้างจ่าย การปรับปรุง และคำขอ) เพื่อดูเหตุผลหลังยอดดุล 

## <a name="coming-soon"></a>เร็วๆ นี้

### <a name="improvements-to-the-user-interface-for-duplicate-employee-check"></a>การปรับปรุงถึงอินเทอร์เฟสผู้ใช้สำหรับ ทำซ้ำการตรวจสอบพนักงาน
ด้วยการเปลี่ยนแปลงนี้ ข้อมูลซ้ำจะถูกตรวจพบขณะที่คุณป้อนในฟิลด์ชื่อ และสถานะจะแสดงจำนวนของข้อมูลซ้ำที่พบ คุณสามารถเลือกการเชื่อมโยงที่ให้มาเพื่อเปิดหน้าใหม่เพื่อประเมินว่าจะใช้การจับคู่ที่ตรวจพบหรือไม่ หากต้องการหลีกเลี่ยงการรบกวนการป้อนข้อมูล ฟอร์มที่ซ้ำกันจะไม่เปิดขึ้นโดยอัตโนมัติ

###  <a name="email-support-for-alerts"></a>การสนับสนุนอีเมลสำหรับข้อความแจ้งเตือน
ด้วย Platform update 25 สำหรับ Finance and Operations ผู้ใช้สามารถสร้างกฎการแจ้งเตือนที่จัดส่งการแจ้งเตือนทางอีเมลโดยอัตโนมัติไปยังผู้ติดต่อที่ถูกทริกเกอร์โดยเหตุการณ์
