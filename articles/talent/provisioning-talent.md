# <a name="provision-microsoft-dynamics-365-for-talent"></a>เตรียมใช้งาน Microsoft Dynamics 365 for Talent
หัวข้อนี้นำคุณไปสู่กระบวนการของการเตรียมใช้งานสภาพแวดล้อมใหม่สำหรับ Microsoft Dynamics 365 for Talent หัวข้อนี้สันนิษฐานว่า คุณได้ซื้อ Talent ผ่านผู้ให้บริการโซลูชัน Cloud (CSP) หรือข้อตกลงสถาปัตยกรรมองค์กร (EA) ถ้าคุณมีลิขสิทธิ์ Microsoft Dynamics 365 ที่มีอยู่ ซึ่งรวมแผนบริการ Talent และคุณไม่สามารถทำตามขั้นตอนในหัวข้อนี้ให้เสร็จสิ้นได้ ให้ติดต่อฝ่ายสนับสนุน

เพื่อเริ่มต้น ผู้ดูแลระบบส่วนกลางควรลงชื่อเข้าใช้ [Microsoft Dynamics Lifecycle Services](http://lcs.dynamics.com) (LCS) และสร้างโครงการ Talent ใหม่ ถ้ามีปัญหาการให้ลิขสิทธิ์ไม่ป้องกันคุณจากการเตรียมใช้งาน Talent ความช่วยเหลือจากเจ้าหน้าที่ฝ่ายสนับสนุนหรือตัวแทน Dynamics Service Engineering ก็ไม่จำเป็น

## <a name="create-an-lcs-project"></a>สร้างโครงการ LCS
เพื่อใช้ LCS ในการจัดการสภาพแวดล้อม Talent ของคุณ ลำดับแรกคุณต้องสร้างโครงการ LCS ก่อน

1. ลงชื่อเข้าใช้ [LCS](https://lcs.dynamics.com/Logon/Index) โดยใช้บัญชีที่คุณใช้ในการสมัครสมาชิก Talent
2. เลือกเครื่องหมายบวก (**+**) เพื่อสร้างโครงการ
3. เลือก **Microsoft Dynamics 365 for Talent** ให้เป็นชื่อผลิตภัณฑ์และรุ่นผลิตภัณฑ์
4. เลือกระเบียบวิธี **Dynamics 365 for Talent**
5. เลือก **สร้าง**

สำหรับข้อมูลเกี่ยวกับวิธีการเริ่มต้นใช้งาน Talent ดูระเบียบวิธี **Talent** ที่คุณสร้างไว้ในโครงการใหม่ของคุณ หลังจากที่คุณเสร็จสิ้นการสร้างโครงการ ทำกระบวนงานต่อไปนี้ให้เสร็จสมบูรณ์ เพื่อเตรียมใช้งานสภาพแวดล้อม Talent ของคุณ

## <a name="provision-a-talent-project"></a>เตรียมใช้งานโครงการ Talent
หลังจากที่คุณได้สร้างโครงการ LCS คุณสามารถเตรียมใช้งาน Talent ลงในสภาพแวดล้อมได้

1. ในโครงการ LCS ของคุณ เลือกไทล์ **การจัดการแอป Talent**
2. Talent ถูกเตรียมใช้งานลงในสภาพแวดล้อม Microsoft PowerApps เสมอ เพื่อเปิดใช้งานการรวม PowerApps และความสามารถในการขยาย ถ้าคุณไม่มีสภาพแวดล้อม PowerApps ให้ทำตามขั้นตอนในส่วน "สร้างสภาพแวดล้อม PowerApps ใหม่ (ถ้าจำเป็น)" ของหัวข้อนี้ ก่อนที่คุณดำเนินการต่อ

    > [!NOTE]
    > เมื่อต้องการดูสภาพแวดล้อมที่มีอยู่ หรือสร้างสภาพแวดล้อมใหม่ ผู้ดูแลระบบผู้เช่าที่เตรียมใช้งาน Talent ต้องถูกกำหนดให้กับสิทธิ์การใช้งาน PowerApps P2 ถ้าองค์กรของคุณไม่มีสิทธิ์การใช้งาน PowerApps P2 คุณสามารถได้รับจาก CSP ของคุณ หรือจาก [หน้าการกำหนดราคา PowerApps](https://powerapps.microsoft.com/en-us/pricing/) ได้

3. เลือก **เพิ่ม** แล้วจากนั้น เลือกสภาพแวดล้อมที่จะเตรียมใช้งาน Talent ลงไป
4. เลือก **ใช่** เพื่อยอมรับเงื่อนไข และเริ่มต้นการปรับใช้

    สภาพแวดล้อมใหม่ของคุณ ปรากฏในรายการของสภาพแวดล้อมในบานหน้าต่างนำทางด้านซ้าย อย่างไรก็ตาม คุณไม่สามารถเริ่มต้นใช้สภาพแวดล้อมได้ จนกว่าสถานะการปรับใช้จะถูกอัพเดตเป็น **ถูกปรับใช้** กระบวนการนี้โดยปกติใช้เวลาเพียงไม่กี่นาที ถ้าการเตรียมใช้งานล้มเหลว คุณต้องติดต่อฝ่ายสนับสนุน

6. เลือก **ล็อกออนเข้าสู่ Talent** เพื่อใช้สภาพแวดล้อมใหม่ของคุณ

> [!NOTE]
> ถ้าคุณยังไม่ได้ออกจากระบบในความต้องการขั้นสุดท้าย คุณสามารถปรับใช้อินสแตนซ์ของ Talent ในโครงการได้ จากนั้น คุณสามารถใช้อินสแตนซ์นี้เพื่อทดสอบโซลูชันของคุณได้จนกว่าคุณจะออกจากระบบ ถ้าคุณใช้สภาพแวดล้อมใหม่สำหรับการทดสอบ คุณต้องทำซ้ำกระบวนงานนี้ เพื่อสร้างสภาพแวดล้อมการผลิต

## <a name="create-a-new-powerapps-environment-if-required"></a>สร้างสภาพแวดล้อม PowerApps ใหม่ (ถ้าจำเป็น)

วิสัยทัศน์เบื้องหลังการรวม Talent กับสภาพแวดล้อม PowerApps คือ การเปิดใช้งานขั้นตอนการรวมข้อมูลและการขยายผ่านการใช้เครื่องมือ PowerApps ที่ด้านบนของข้อมูล Talent ดังนั้น จำเป็นต้องเข้าใจวัตถุประสงค์ของสภาพแวดล้อม PowerApps เมื่อเลือกสภาพแวดล้อมที่จะใช้สำหรับ Talent สำหรับข้อมูลเพิ่มเติมเกี่ยวกับสภาพแวดล้อม PowerApps รวมถึงขอบเขตของสภาพแวดล้อม การเข้าถึงสภาพแวดล้อม และการสร้างและการเลือกสภาพแวดล้อม ดู [การประกาศสภาพแวดล้อม PowerApps](https://powerapps.microsoft.com/en-us/blog/powerapps-environments/)  ขณะที่ผู้เช่าแต่ละรายถูกเตรียมใช้งานในสภาพแวดล้อม PowerApps เริ่มต้นโดยอัตโนมัติ นี้อาจไม่ใช่สภาพแวดล้อมที่ดีที่สุดที่จะใช้สำหรับการปรับใช้ Talent ของคุณ การรวมข้อมูลและการทดสอบกลยุทธ์ควรได้รับการพิจารณาในระหว่างขั้นตอนนี้ ดังนั้นเราจึงขอแนะนำว่า คุณควรพิจารณาถึงผลกระทบต่างๆ สำหรับการปรับใช้ของคุณ เนื่องจากจะเปลี่ยนในภายหลังได้ยาก

1. เลือก **จัดการสภาพแวดล้อม** ใน LCS คุณกำลังถูกนำไปยัง [ศูนย์การจัดการ PowerApps](https://preview.admin.powerapps.com/environments) ที่ซึ่งคุณสามารถดูสภาพแวดล้อมที่มีอยู่ และสร้างสภาพแวดล้อมใหม่ได้
2. เลือกปุ่ม (**+**) **สภาพแวดล้อมใหม่**
3. ป้อนชื่อเฉพาะสำหรับสภาพแวดล้อม และเลือกที่ตั้งในการปรับใช้

    > [!NOTE]
    > Talent ไม่พร้อมใช้งานในทุกภูมิภาค ดังนั้น ต้องแน่ใจว่าตรวจสอบความพร้อมใช้งาน ก่อนที่คุณเลือกที่ตั้งสำหรับสภาพแวดล้อมของคุณ

4. เมื่อคุณถูกถามว่าคุณต้องการสร้างฐานข้อมูลหรือไม่ เลือก **สร้างฐานข้อมูล** เพื่อสร้างฐานข้อมูล Common Data Service (CDS) ที่ต้องโฮสต์ส่วนของข้อมูล Talent โดยการสร้างฐานข้อมูล คุณยังสามารถรวมแอพลิเคชัน PowerApps เข้ากับ Talent ได้ด้วย
5. คุณจะถูกถามเกี่ยวกับระดับการเข้าถึงที่คุณต้องการใช้สำหรับฐานข้อมูล เราขอแนะนำให้ คุณเลือก **จำกัดการเข้าถึง** เนื่องจากตัวเลือกนี้ป้องกันผู้ใช้ Talent จากการเข้าถึงข้อมูลที่อ่อนไหวโดยตรง โดยใช้แอพลิเคชัน PowerApps
6. ฐานข้อมูลซีดีที่สร้างขึ้น ประกอบด้วยข้อมูลสาธิต ข้อมูลสาธิตนี้มีประโยชน์ เนื่องจากคุณสามารถใช้บริษัทข้อมูลสาธิตสำหรับการทดสอบ หรือเพื่อสร้างการบันทึกงานหรือคู่มืองาน อย่างไรก็ตาม ข้อมูลสาธิตเพิ่มพนักงานที่ไม่ได้ใช้งานและที่อยู่สมมติ ระหว่างข้อมูลอื่น ไปยังสภาพแวดล้อมการผลิตของคุณ เพื่อลบข้อมูลสาธิต ทำตามขั้นตอนเหล่านี้ หลังจากที่คุณเสร็จสิ้นการสร้างฐานข้อมูลซีดี:

    > [!IMPORTANT]
    > ถ้าคุณสร้างฐานข้อมูลซีดีไว้ก่อนหน้านี้ และป้อนข้อมูลการผลิตใดๆ ของบริษัทของคุณ โปรดระวังว่า ขั้นตอนเหล่านี้จะลบข้อมูล **ทั้งหมด** ในฐานข้อมูลที่เลือก แม้แต่ข้อมูลการผลิตของบริษัทของคุณ

    1. ลงชื่อเข้าใช้ [PowerApps](https://preview.web.powerapps.com/home) และเลือกสภาพแวดล้อมที่คุณสร้างไว้ในขั้นตอนที่ 2 จากรายการแบบหล่นลงบนด้านขวาของหน้า
    2. ขยาย **บริการข้อมูลทั่วไป** บนบานหน้าต่างนำทางด้านซ้าย และเลือก **เอนทิตี้**
    3. ทางด้านขวาของหน้า เลือกปุ่มจุดไข่ปลา (**...**) และจากนั้น เลือก **ล้างข้อมูลทั้งหมด**
    4. เลือก **ลบข้อมูล** เพื่อยืนยันว่าคุณต้องการลบข้อมูล การดำเนินการนี้ลบข้อมูลสาธิตทั้งหมดที่ถูกรวมอยู่ในซีดีโดยค่าเริ่มต้น นอกจากนี้ ยังลบข้อมูลใดๆ ที่ป้อนไว้ในฐานข้อมูลที่เลือกอีกด้วย
    
ขณะนี้คุณสามารถใช้สภาพแวดล้อมใหม่ของคุณได้

## <a name="granting-access-to-the-environment"></a>การให้สิทธิ์การเข้าถึงสภาพแวดล้อม
ผู้ดูแลระบบส่วนกลางที่สร้างสภาพแวดล้อมจะมีการเข้าถึงโดยค่าเริ่มต้น แต่ผู้ใช้แอพลิเคชันเพิ่มเติมต้องได้รับสิทธิ์การเข้าถึงอย่างชัดแจ้ง ซึ่งสามารถทำได้โดย [การเพิ่มผู้ใช้](../dev-itpro/sysadmin/tasks/create-new-users.md) และ [การมอบหมายบทบาทที่เหมาะสม](../dev-itpro/sysadmin/tasks/assign-users-security-roles.md) ภายในสภาพแวดล้อม HR หลัก นอกจากนี้ ยังจำเป็นต้องเพิ่มผู้ใช้เหล่านั้นไปยังสภาพแวดล้อม PowerApps เพื่อให้พวกเขาสามารถเข้าถึงแอพลิเคชัน Attract และ Onboard ได้  ประกาศบล็อก [การแนะนำศูนย์การจัดการ PowerApps](https://powerapps.microsoft.com/en-us/blog/introducing-admin-center-for-powerapps/) อาจช่วยคุณในการดำเนินขั้นตอนเหล่านั้นให้เสร็จสมบูรณ์ ซึ่งมีการระบุไว้ที่นี่:

> 1.    ผู้ดูแลระบบส่วนกลางที่ปรับใช้สภาพแวดล้อม Talent ควรนำทางไปยัง [ศูนย์การจัดการ PowerApps](https://preview.admin.powerapps.com/environments)   
> 2.    เลือกสภาพแวดล้อมที่สงสัย
> 3.    ภายใต้แท็บความปลอดภัย เพิ่มผู้ใช้ที่จำเป็นให้กับบทบาท "ผู้สร้างสภาพแวดล้อม"

โปรดทราบว่า ขั้นตอนสุดท้ายนี้ของการเพิ่มผู้ใช้ไปยังสภาพแวดล้อม PowerApps เป็นแบบชั่วคราว ในที่สุด เราจะเพิ่มฟังก์ชันการทำงานเพื่อเปิดใช้งานโดยอัตโนมัติ เมื่อมีการเพิ่มผู้ใช้ภายใน HR หลัก
