# <a name="create-a-job-in-attract"></a>สร้างตำแหน่งงานใน Attract

[!include [banner](includes/banner.md)]

หัวข้อนี้จะอธิบายองค์ประกอบของงานใน Microsoft Dynamics 365 Talent: Attract: Attract และยังอธิบายถึงวิธีการสร้างงาน

## <a name="job-creation"></a>การสร้างงาน

ผู้ดูแล ผู้สรรหา และผู้จัดการว่าจ้าง สามารถสร้างงานได้ เมื่อคุณสร้างงาน คุณจะได้รับพร้อมท์ให้เลือกบทบาทของคุณในกระบวนการ: ผู้จัดการว่าจ้าง หรือผู้สรรหา หลังจากที่คุณเลือกบทบาท คุณจะได้รับพร้อมท์เพื่อเลือกเท็มเพลตกระบวนการ ถ้าคุณเลือก **ข้าม** จะมีการใช้เท็มเพลตเริ่มต้น สำหรับข้อมูลเพิ่มเติมเกี่ยวกับเท็มเพลตกระบวนการ ดู [สร้างเท็มเพลตกระบวนการใน Attract](./process-templates-attract.md)

งานใน Attract มีรายละเอียดงาน ทีมว่าจ้าง กระบวนการว่าจ้าง การลงรายการบัญชีงาน และการวิเคราะห์

## <a name="job-details"></a>รายละเอียดงาน

แท็บ **รายละเอียดงาน** ประกอบด้วยรายละเอียดเกี่ยวกับความรับผิดชอบและแอททริบิวต์ของงาน ต้องมีฟิลด์สำหรับตำแหน่งงาน คำอธิบายงาน และตำแหน่งงาน ฟิลด์อื่นเป็นฟิลด์ที่ไม่จำเป็นต้องระบุ

โดยค่าเริ่มต้น ฟิลด์ **จำนวนตำแหน่งที่เปิดรับ** ถูกกำหนดเป็น **1** อย่างไรก็ตาม คุณสามารถเปลี่ยนค่าได้ เมื่อมีการจัดเตรียมข้อเสนอสำหรับงาน มูลค่าของฟิลด์ **จำนวนตำแหน่งที่เปิดรับที่พร้อมใช้** จะถูกลดลง

ถ้าการจัดการตำแหน่งถูกเปิดอยู่ในศูนย์การจัดการ การค้นหา **อัพเดตตำแหน่ง** จะพร้อมใช้งาน การค้นหานี้อ่านเอนทิตี JobPosition ใน Common Data Service และส่งคืนรายการของตำแหน่งที่สามารถถูกเลือกได้สำหรับงาน ถ้าจำนวนของตำแหน่งที่คุณเลือกเกินจำนวนตำแหน่งที่เปิด คุณจะได้รับคำเตือน นอกจากนี้ คุณได้รับคำเตือน ถ้ามีการใช้งานในงานหลายรายการ

> [!NOTE]
> การจัดการตำแหน่งจะพร้อมใช้งานโดยมี add-on การว่าจ้างที่ครอบคลุม

โดยขึ้นอยู่กับการตั้งค่าในกิจกรรมข้อเสนอของกระบวนการว่าจ้าง สามารถใช้หมายเลขตำแหน่งได้สองครั้งในหนึ่งข้อเสนอ สำหรับข้อมูลเพิ่มเติม ดู [กิจกรรมในกระบวนการจ้างงาน](./activities-attract.md)

Attract มีชุดค่าเริ่มต้นของ **ทักษะ** ทักษะเหล่านี้ปรากฏเป็นคำแนะนำขณะที่คุณพิมพ์ คุณสามารถเพิ่มทักษะเพิ่มเติมได้โดยป้อนเนื้อหาทักษะใหม่ในฟิลด์ แล้วกดป้อน

Attract รวมชุดค่าเริ่มต้นของ **ฟังก์ชันงาน** คุณสามารถเพิ่มฟังก์ชันงานเพิ่มเติมได้ถึงสามรายการ โดยป้อนฟังก์ชันงานใหม่ในฟิลด์ แล้วกดป้อน

Attract รวมชุดค่าเริ่มต้นของ **อุตสาหกรรมของบริษัท** คุณสามารถเพิ่มอุตสาหกรรมบริษัทเพิ่มเติมได้ถึงสามรายการ โดยป้อนฟังก์ชันอุตสาหกรรมบริษัทใหม่ในฟิลด์ แล้วกดป้อน

## <a name="hiring-team"></a>ทีมงานการจ้างงาน

แท็บ **ทีมการว่าจ้าง** ประกอบด้วยรายชื่อของบุคคลที่จะเกี่ยวข้องกับงาน เมื่อมีการเพิ่มผู้ใช้ไปยังทีมงานที่ว่าจ้าง พวกเขาต้องถูกกำหนดบทบาทในทีมงานว่าจ้าง บทบาทกำหนดข้อมูลที่ผู้ใช้มีสิทธิ์ในการเข้าถึงและการแจ้งเตือนที่พวกเขาได้รับ บทบาทที่สามารถถูกเลือกคือ **ผู้สรรหา** **ผู้จัดการที่ว่าจ้าง** **ผู้รับมอบสิทธิ์** และ **ผู้สัมภาษณ์** สำหรับข้อมูลเพิ่มเติมเกี่ยวกับสิทธิพิเศษของบทบาท ดูเอกสาร "การจัดการบทบาท" ผู้สรรหาและผู้จัดการการว่าจ้างสามารถแต่งตั้งผู้รับมอบสิทธิ์อย่างน้อยหนึ่งราย เพื่อทำงานในนามของพวกเขา สำหรับข้อมูลเพิ่มเติมเกี่ยวกับผู้รับมอบสิทธิ์ ดู [การจัดการความปลอดภัยและบทบาทใน Attract](./security-attract.md)

สามารถอัพเดตทีมว่าจ้างได้ หลังจากที่มีการเรียกใช้งาน

## <a name="process"></a>กระบวนการ

ข้อมูลเริ่มต้นเกี่ยวกับกระบวนการว่าจ้างจะขึ้นอยู่กับเท็มเพลตกระบวนการที่ถูกเลือก เมื่อมีการสร้างงาน ถ้าเท็มเพลตที่ระบุไม่ถูกเลือกในขณะนั้น จะมีการใช้เท็มเพลตเริ่มต้น เมื่อคุณกำหนดกระบวนการว่าจ้าง คุณสามารถเพิ่ม หรือลบขั้นตอนต่างๆ ได้ ยกเว้นขั้นผู้ที่มีแนวโน้มจะเป็นลูกค้า แอพลิเคชัน และข้อเสนอ แม้ว่าจะไม่สามารถลบขั้นผู้ที่มีแนวโน้มจะเป็นลูกค้าได้ แต่คุณสามารถปิดได้ ภายในแต่ละขั้นตอน คุณสามารถเพิ่ม หรือลบกิจกรรมที่กำหนดไว้ล่วงหน้าหนึ่งรายการขึ้นไป

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับกิจกรรมที่สามารถถูกเพิ่มลงในกระบวนการว่าจ้าง ดู [กิจกรรมในกระบวนการว่าจ้าง](./activities-attract.md)

> [!NOTE]
> ไม่สามารถอัพเดตกระบวนการว่าจ้างได้ หลังจากที่มีการเรียกใช้งาน

## <a name="postings"></a>การลงรายการบัญชี

หลังจากที่มีการเรียกใช้งาน จึงสามารถลงรายการบัญชีได้ เฉพาะผู้สรรหาและผู้ดูแลระบบสามารถลงรายการบัญชีงานได้ สามารถลงรายการบัญชีงานไปยัง Talent Careers (ไซต์อาชีพ Microsoft Dynamics 365 Talent ) หรือ LinkedIn อย่างใดอย่างหนึ่ง ทีม Attract ทำงานกับคู่ค้ากับตัวรวมบอร์ดงานอย่างต่อเนื่อง รายการนี้จะขยายตลอดช่วงเวลา เมื่องานมีการลงรายการบัญชีเป็นภายในเท่านั้น ผู้สมัครต้องมีบัญชี AAD เพื่อดูและสมัครงาน ถ้างานถูกแสดงรายการเป็นสาธารณะ ผู้สมัครสามารถดูและสมัครงานโดยใช้ตัวเลือกการรับรองความถูกต้องทั้งหมด 

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับตำแหน่งงานที่โพสต์ ให้ดูที่ [ตั้งค่าไซต์อาชีพ ใน Microsoft Dynamics 365 Talent - Attract ของคุณ](career-site.md)

> [!NOTE]
> ฟังก์ชันการลงรายการบัญชีงานพร้อมใช้งานเฉพาะกับ Add-on การว่าจ้างที่ครอบคลุมสำหรับ Attract

## <a name="activate"></a>เปิดใช้งาน

หลังจากที่มีการเรียกใช้งาน สามารถลงรายการบัญชี และสามารถเพิ่มผู้ที่มีแนวโน้มจะเป็นลูกค้าและผู้สมัครในนั้นได้ มีการตั้งค่าตัวเลือกเพื่อเพิ่มผู้ที่มีแนวโน้มจะเป็นลูกค้า ไปยังงานในกิจกรรมผู้ที่มีแนวโน้มจะเป็นลูกค้าในกระบวนการว่าจ้าง

> [!NOTE]
> ไม่สามารถอัพเดตกระบวนการว่าจ้างได้ หลังจากที่มีการเรียกใช้งาน

## <a name="prospects-and-applicants"></a>ผู้ที่มีแนวโน้มจะเป็นลูกค้าและผู้สมัคร

ตัวเลือกเพื่อเพิ่มผู้ที่มีแนวโน้มจะเป็นลูกค้าไปยังงานถูกตั้งค่าใน [กิจกรรมในกระบวนการว่าจ้าง](./activities-attract.md#prospect-activity) ในกระบวนการว่าจ้าง ควรมีการตั้งค่าตัวเลือกนี้ ก่อนที่คุณจะเรียกใช้งาน หลังจากที่มีการเรียกใช้งาน สามารถเพิ่มผู้ที่มีแนวโน้มจะเป็นลูกค้าและผู้สมัครในนั้นได้

## <a name="approvals"></a>การอนุมัติ

สามารถส่งงาน Attract สำหรับการอนุมัติได้ ไม่จำเป็นต้องมีการอนุมัติงานทั้งหมด ความต้องการถูกตั้งค่าที่ระดับเท็มเพลต โดยค่าเริ่มต้น การอนุมัติถูกปิดบนเท็มเพลต ในการตั้งค่าการอนุมัติ ไปที่เท็มเพลตกระบวนการ และตั้งค่าฟิลด์ **การอนุมัติ** เป็นค่าเริ่มต้น จากนั้น เลือกเท็มเพลตนั้นเมื่อคุณสร้างงาน

หลังจากที่มีการบันทึกงาน จะสามารถส่งเพื่อขออนุมัติได้ ตารางต่อไปนี้แสดงรายการสถานะของเอกสารที่ใช้การอนุมัติ

| สถานะ   | สถานะ                                                               |
|----------|---------------------------------------------------------------------|
| ร่าง    | มีการบันทึกงานแล้ว แต่ยังไม่ได้ถูกส่งไปยังลำดับงาน |
| ค้างอยู่  | งานได้ถูกส่งไปยังผู้อนุมัติแล้ว                            |
| อนุมัติแล้ว | งานได้รับอนุมัติแล้ว แต่ยังไม่ได้ถูกเรียกใช้            |
| ปฏิเสธแล้ว | มีการปฏิเสธงาน และไม่สามารถเรียกใช้งานได้               |
| ใช้งาน   | งานได้รับการอนุมัติ และถูกเรียกใช้                            |

ในรายการงาน คุณสามารถกรองข้อมูลในสถานะงานนั้นได้

สามารถส่งการอนุมัติไปยังผู้ใช้ Microsoft Azure Active Directory (Azure AD) ใดๆ ในบริษัทได้ การอนุมัติถูกส่งพร้อมกันสำหรับทุกคนที่มีชื่อเป็นผู้อนุมัติ ผู้อนุมัติทั้งหมดต้องอนุมัติงานก่อนที่จะสามารถดำเนินต่อได้ หากผู้อนุมัติคนเดียวปฏิเสธงาน งานจะแสดงสถานะ **ถูกปฏิเสธ** หลังจากที่มีการอนุมัติงาน จะสามารถเรียกใช้งานได้

หากผู้ใช้แก้ไขงานหลังจากที่ได้รับอนุมัติ แต่ยังไม่ได้เรียกใช้ สถานะงานจะถูกรีเซ็ตเป็น **ร่าง** และงานต้องถูกส่งเพื่อขออนุมัติอีกครั้ง หลังจากที่มีการเรียกใช้งานที่ได้รับการอนุมัติแล้ว คุณไม่สามารถแก้ไขได้

บุคคลที่มีรายชื่อเป็นผู้อนุมัติ จะได้รับการแจ้งเตือนใน Attract และในอีเมลเพื่อแจ้งพวกเขาว่ามีรายการที่จะอนุมัติ  ในอีเมล ผู้อนุมัติสามารถคลิกลิงค์เพื่อเปิดงาน ตรวจสอบรายละเอียด และอนุมัติหรือปฏิเสธอย่างใดอย่างหนึ่ง หลังจากที่สถานะของงานถูกตั้งค่าเป็น **อนุมัติแล้ว** หรือ **ถูกปฏิเสธ** ผู้ส่งจะได้รับการแจ้งเตือนใน Attract และพวกเขาจะได้รับอีเมล นอกจากนี้ ผู้อนุมัติจะได้รับอีเมลแจ้งเตือนถ้าพวกเขาไม่ตอบสนองคำขออนุมัติภายใน 24 ชั่วโมง

> [!NOTE]
> คุณสามารถสร้างเท็มเพลตอีเมลที่กำหนดเองสำหรับอีเมลอนุมัติ สำหรับข้อมูลเพิ่มเติม ดู [การสร้างและการจัดการเท็มเพลตอีเมล](https://docs.microsoft.com/dynamics365/unified-operations/talent/email-templates)

## <a name="create-a-job"></a>การสร้างงาน

ทำตามขั้นตอนเหล่านี้เพื่อสร้างงาน

1. ไปที่ **งาน**
2. เลือก **ใหม่**
3. ในฟิลด์ **ตำแหน่งงาน** ป้อนตำแหน่งงาน ในฟิลด์ **บทบาท** ป้อนบทบาทของคุณ
4. ในฟิลด์ **เท็มเพลต** ให้เลือกเท็มเพลต หรือเลือก **ข้าม** ถ้าคุณเลือก **ข้าม** จะมีการใช้เท็มเพลตที่ถูกทำเครื่องหมายเป็นเท็มเพลตเริ่มต้น

    ถ้าเอกสารควรผ่านกระบวนการการอนุมัติ เลือกเท็มเพลตที่ซึ่งฟิลด์ **กระบวนการอนุมัติ** ถูกกำหนดเป็น **เริ่มต้น**

5. บนแท็บ **รายละเอียด** ป้อนรายละเอียดของงาน ต้องมีฟิลด์ **ชื่อตำแหน่ง** **คำอธิบายงาน** และ **สถานที่**
6. เลือก **บันทึก**
7. บนแท็บ **ทีมการว่าจ้าง** เพิ่มผู้จัดการว่าจ้าง ผู้สรรหา หรือผู้สัมภาษณ์
8. เลือก **บันทึก**
9. บนแท็บ **กระบวนการ** เพิ่ม หรือลบขั้น ตามที่คุณต้องการ:

    - ในการเพิ่มขั้น เลือก **+ ขั้นใหม่**
    - ในการลบขั้น โฮเวอร์ตัวชี้เหนือขั้นเพื่อลบ แล้วเลือกปุ่มถังขยะที่ปรากฏขึ้น

        > [!NOTE]
        > ไม่สามารถลบขั้นผู้ที่มีแนวโน้มจะเป็นลูกค้า แอพลิเคชัน และข้อเสนอได้

10. เพิ่ม หรือลบกิจกรรม ตามที่คุณต้องการ:

    - ในการเพิ่มกิจกรรม ลากจากรายชื่อทางขวาไปยังขั้นที่เหมาะสม อีกทางหนึ่งคือ ดับเบิลคลิกที่กิจกรรม และจากนั้น เลือกระยะที่จะเพิ่มเข้าไป
    - ในการลบกิจกรรม ขยายกิจกรรม และจากนั้นเลือกปุ่มถังขยะบนหัวข้อกิจกรรมนั้น

11. เลือก **บันทึก**
12. ถ้าคุณเลือกที่จะใช้กระบวนการอนุมัติ ให้ทำตามขั้นตอนเหล่านี้:

    1. เลือก **+ เพิ่มผู้อนุมัติ** และจากนั้นป้อนผู้ใช้ที่มีบัญชี Azure AD คุณสามารถเพิ่มผู้อนุมัติหลายคนได้
    2. เลือก **ส่งไปยังผู้อนุมัติ**

    ฟิลด์ **สถานะงาน** ของงาน ถูกกำหนดเป็น **ค้างอยู่** หลังจากค่าของฟิลด์ **สถานะงาน** เปลี่ยนเป็น **อนุมัติ** สามารถเรียกใช้งานได้

13. เมื่อต้องการเรียกใช้งาน ให้เลือก **เปิดใช้งาน**
14. เมื่อต้องการลงรายการบัญชีงาน ไปที่ **การลงรายการบัญชี** แล้วเลือก **ลงรายการบัญชีตอนนี้** ภายใต้ไซต์ Talent Careers หรือ LinkedIn
