<span data-ttu-id="74cc7-110">**ชนิดค่าตอบแทนผันแปรชนิดต่างๆ**คือส่วนประกอบที่จำเป็น</span><span class="sxs-lookup"><span data-stu-id="74cc7-110">**Variable compensation types** are a required component.</span></span> <span data-ttu-id="74cc7-111">ชนิดค่าตอบแทนผันแปรช่วยให้คุณสามารถอธิบายชนิดของค่าตอบแทนผันแปรซึ่งเป็นรางวัลขององค์กรคุณ</span><span class="sxs-lookup"><span data-stu-id="74cc7-111">Variable compensation types let you describe the kinds of variable compensation that your organization awards.</span></span> <span data-ttu-id="74cc7-112">พวกเขายังช่วยให้คุณสามารถระบุว่า ค่าตอบแทนจะเป็นเงินสด หรือ ในแบบ ฟอร์มที่ไม่เกี่ยวกับเงินตรา เช่น ปริมาณสินค้าคงคลัง</span><span class="sxs-lookup"><span data-stu-id="74cc7-112">They also let you specify whether the compensation will be in cash or in a non-monetary form, such as stock.</span></span>

## <a name="variable-compensation-plans"></a><span data-ttu-id="74cc7-118">แผนค่าตอบแทนผันแปร</span><span class="sxs-lookup"><span data-stu-id="74cc7-118">Variable compensation plans</span></span>
<span data-ttu-id="74cc7-119">**แผนค่าตอบแทนผันแปร** ประกอบด้วยกฎ วิธีการคำนวณ และค่าเริ่มต้นสำหรับการคำนวณค่าตอบแทนผันแปรของพนักงานที่ลงทะเบียน</span><span class="sxs-lookup"><span data-stu-id="74cc7-119">The **variable compensation plan** contains the rules, calculation methods, and default values for the calculation of variable compensation for enrolled employees.</span></span> <span data-ttu-id="74cc7-120">เมื่อคุณสร้างแผนค่าตอบแทนผันแปร คุณต้องตั้งค่าชนิดค่าตอบแทนผันแปร</span><span class="sxs-lookup"><span data-stu-id="74cc7-120">When you create a variable compensation plan, you must set the variable compensation type.</span></span> <span data-ttu-id="74cc7-121">ชนิดค่าตอบแทนผันแปรกำหนดว่า ระบบจะคำนวณเป็นยอดสกุลเงินหรือจำนวนหน่วยเป็นเงินรางวัล</span><span class="sxs-lookup"><span data-stu-id="74cc7-121">The variable compensation type determines whether the system calculates a currency amount or a number of units as the award.</span></span> <span data-ttu-id="74cc7-122">ทั้งนี้คุณต้องกำหนดวิธีการคำนวณด้วย:</span><span class="sxs-lookup"><span data-stu-id="74cc7-122">You must also set the calculation method:</span></span>

-   <span data-ttu-id="74cc7-123">**ณ เวลานี้** – การคำนวณเงินรางวัลผันแปรจะขึ้นอยู่กับค่าตอบแทนคงที่พนักงานจะได้รับวันที่ที่ระบุไว้</span><span class="sxs-lookup"><span data-stu-id="74cc7-123">**Point in time** – The calculation of the variable award is based on the fixed compensation that the employee had on a specific date.</span></span> <span data-ttu-id="74cc7-124">มีระบุวันที่ในเหตุการณ์กระบวนการเมื่อยอดเงินค่าตอบแทนใหม่ถูกดำเนินการ</span><span class="sxs-lookup"><span data-stu-id="74cc7-124">That date is specified on the process event when new compensation amounts are processed.</span></span>
-   <span data-ttu-id="74cc7-125">**โดยรวมแล้ว**– ยอดเงินรางวัลที่ได้คำนวณสำหรับอัตราค่าจ้างแต่ละค่าตอบแทนคงที่เฉพาะที่พนักงานมีระหว่างวันเริ่มต้นรอบและวันสิ้นสุดของวงจรในเหตุการณ์ในกระบวนการ</span><span class="sxs-lookup"><span data-stu-id="74cc7-125">**Composite** – An award amount is calculated for each unique fixed compensation pay rate that the employee had between the cycle start date and the cycle end date on the process event.</span></span> <span data-ttu-id="74cc7-126">อัตราต่างๆ จะถูกรวมเข้าด้วยกันเพื่อกำหนดเงินรางวัลขั้นสุดท้าย</span><span class="sxs-lookup"><span data-stu-id="74cc7-126">The rates are then added together to determine the final award.</span></span> <span data-ttu-id="74cc7-127">ตัวอย่างเช่น ในระหว่างวงจร พนักงานถูกโอนย้ายไปยังตำแหน่งอื่นที่มีอัตราค่าจ้างที่แตกต่างกัน</span><span class="sxs-lookup"><span data-stu-id="74cc7-127">For example, during the cycle, an employee transferred to a different position that had a different pay rate.</span></span> <span data-ttu-id="74cc7-128">ในกรณีนี้ เงินรางวัลผันแปรถูกปรับปรุงสำหรับระยะเวลาที่พนักงานได้อัตราค่าจ้างของแต่ละอัตรา</span><span class="sxs-lookup"><span data-stu-id="74cc7-128">In this case, the variable award is adjusted for the length of time that the employee had each pay rate.</span></span>

<span data-ttu-id="74cc7-129">ยอดเงินของเงินรางวัลผันแปรสามารถขึ้นกับเปอร์เซ็นต์ของรายได้พื้นฐานปกติของพนักงานหรือจำนวนชุดของหน่วย</span><span class="sxs-lookup"><span data-stu-id="74cc7-129">The amount of the variable award can be based on either a percentage of the employee's regular base earnings or a set number of units.</span></span>

-   <span data-ttu-id="74cc7-130">เลือก **เปอร์เซ็นต์ของตัวเลือกพื้นฐาน** เพื่อป้อนเปอร์เซ็นต์เริ่มต้น และระบุว่า ข้อมูลพื้นฐานของพนักงานควรเป็นอัตราค่าจ้างคงที่หรือจุดควบคุมตามระดับค่าตอบแทนของพนักงาน</span><span class="sxs-lookup"><span data-stu-id="74cc7-130">Select the **Percent of basis** option to enter a default percentage, and specify whether the basis should be the employee's fixed pay rate or the control point for the employee's compensation level.</span></span> <span data-ttu-id="74cc7-131">ระดับค่าตอบแทนถูกกำหนดไว้ในงานของพนักงาน</span><span class="sxs-lookup"><span data-stu-id="74cc7-131">The compensation level is set on the employee's job.</span></span> <span data-ttu-id="74cc7-132">จุดอ้างอิงจากโครงสร้างค่าตอบแทนหนึ่งสามารถตั้งค่าเป็นจุดควบคุมในแผนค่าตอบแทนคงที่</span><span class="sxs-lookup"><span data-stu-id="74cc7-132">One of the reference points from the compensation structure can be set as the control point on the fixed compensation plan.</span></span> <span data-ttu-id="74cc7-133">ระบบจะใช้ระดับค่าตอบแทนจากงานของพนักงานและอ้างอิงไขว้ไว้กับระดับของงานด้วยจุดควบคุมที่ระบุในแผนค่าตอบแทนคงที่ของพนักงานเพื่อหาจุดควบคุมของยอดเงินของระดับค่าตอบแทนพนักงาน</span><span class="sxs-lookup"><span data-stu-id="74cc7-133">The system will use the compensation level from the employee's job and cross-reference it with the control point that is listed on the employee's fixed compensation plan to find the control point amount for the employee's compensation level.</span></span> <span data-ttu-id="74cc7-134">โดยจุดควบคุมของยอดเงินนี้จะนำมาใช้แทนค่าจ้างคงที่ของพนักงานเพื่อเป็นข้อมูลพื้นฐานสำหรับรางวัล</span><span class="sxs-lookup"><span data-stu-id="74cc7-134">The control point amount will then be used instead of the employee's fixed pay rate as the basis for the award.</span></span>
-   <span data-ttu-id="74cc7-135">เลือก**ตัวเลือกจำนวนหน่วย** เพื่อป้อนจำนวนเริ่มต้นของหน่วยต่างๆ มูลค่าของแต่ละหน่วยและสกุลเงินของมูลค่าต่อหน่วย ถ้าแผนค่าตอบแทนเป็นรางวัลที่ไม่ใช่เงินสด(ตัวอย่างเช่น 200 หน่วยของสินค้าคงคลังที่มีมูลค่า USD 40), หรือเป็นเพียงจำนวนของหน่วย ถ้าแผนค่าตอบแทนนั้นมีรางวัลเป็นเงินสด</span><span class="sxs-lookup"><span data-stu-id="74cc7-135">Select the **Number of units** option to enter a default number of units, the value of each unit and the currency of the unit value if the compensation plan is for a non-cash award (for example, 200 units of stock that are valued at 40 USD), or just the number of units if the compensation plan is for a cash award.</span></span> <span data-ttu-id="74cc7-136">สำหรับรางวัลเงินสด พนักงานจะได้รับจำนวนหน่วยของสกุลเงินที่ใช้แผนค่าตอบแทนถาวรสำหรับของเขา หรือเธอ(ตัวอย่างเช่น 500 หน่วย ต่อ 1 USD)</span><span class="sxs-lookup"><span data-stu-id="74cc7-136">For a cash award, the employee will receive the specified number of units of the currency that is used for his or her fixed compensation plan (for example, 500 units of 1 USD).</span></span> <span data-ttu-id="74cc7-137">ตัวควบคุมความสัมพันธ์แบบหนึ่งต่อหนึ่งสามารถใช้เพื่อบ่งชี้ว่า มีการแม็ปหนึ่งต่อหนึ่งโดยตรงระหว่างจำนวนหน่วยและมูลค่าต่อหน่วย</span><span class="sxs-lookup"><span data-stu-id="74cc7-137">The one-to-one relationship control can be used to indicate whether there is a direct one-to-one mapping between the number of units and the unit value.</span></span> <span data-ttu-id="74cc7-138">เมื่อคุณสร้างแผนค่าตอบแทนผันแปรสำหรับแผนแบบเงินสด โดยใช้จำนวนหน่วย ตัวเลือกนี้ถูกล็อคโดยอัตโนมัติไปยัง **ใช่** และมูลค่าต่อหน่วยคือ **1.0000**</span><span class="sxs-lookup"><span data-stu-id="74cc7-138">When you create a variable compensation plan for a cash-based plan by using the number of units, this option is automatically locked to **Yes**, and the unit value is **1.0000**.</span></span>

<span data-ttu-id="74cc7-139">การตั้งค่า **กฎการจ้างงาน** อนุญาตให้คุณระบุว่า พนักงานทั้งหมดควรได้รับการเพิ่มค่าตอบแทนเช่นเดียวกัน โดยไม่คำนึงถึงวันที่มีการจ้างงาน (**กฎการจ้างงาน** = **ไม่มี**) หรือว่าพนักงานควรได้รับรางวัลตามเปอร์เซ็นต์ ตามระยะเวลาที่พวกเขาได้ถูกว่าจ้างในระหว่างวงจร (**กฎการจ้างงาน** = **เปอร์เซ็นต์**)</span><span class="sxs-lookup"><span data-stu-id="74cc7-139">The **Hire rule** setting lets you specify whether all employees should receive the same increase, regardless of the date they were hired (**Hire rule** = **None**), or whether employees should receive a percentage of the award that is based on the length of their employment during the cycle (**Hire rule** = **Percent**).</span></span>
