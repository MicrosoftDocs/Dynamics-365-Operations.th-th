# <a name="whats-new-or-changed-in-dynamics-365-for-talent-september-10-2019"></a>มีอะไรใหม่หรือเปลี่ยนไปอย่างไรบ้างใน Dynamics 365 for Talent (10 กันยายน 2019)

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่หรือที่มีการเปลี่ยนแปลง อย่างใดอย่างหนึ่ง ใน Dynamics 365 for Talent

## <a name="changes-in-attract"></a>การเปลี่ยนแปลงใน Attract

### <a name="candidate-e-mail-login"></a>ล็อกอินอีเมล์ของผู้สมัคร

ขณะนี้ผู้สมัครสามารถใช้ที่อยู่อีเมลใดๆเพื่อสร้างบัญชีและล็อกอินเข้าสู่ไซต์ Talent career เพื่อสมัครงานและค้นหาสถานะใบสมัครของตน นอกจากนี้คุณยังสามารถล็อกออนเข้าสู่ไซต์ Talent career โดยใช้บัญชีทางสังคมหรือข้อมูลส่วนบุคคลขององค์กรของตนเองได้

### <a name="job-activation-and-posting"></a>การเรียกใช้งานและการลงรายการบัญชี

เราได้ทำการเปลี่ยนแปลงเล็กน้อยกับการเรียกใช้งานงานและการลงรายการบัญชี คุณจำเป็นต้องเรียกใช้งานก่อนที่จะลงรายการบัญชีเหล่านั้นไปยังไซต์ Talent career หรือกับบอร์ดงานภายนอกใดๆรวมถึง LinkedIn หรือ Broadbean

## <a name="changes-in-onboard"></a>การเปลี่ยนแปลงในการเตรียมความพร้อม

รุ่นนี้ประกอบด้วยการแก้ไขบักรองสำหรับ Dynamics 365 Talent: Onboard

## <a name="changes-in-core-hr"></a>การเปลี่ยนแปลงใน Core HR

การเปลี่ยนแปลงที่อธิบายไว้ในส่วนนี้นำไปใช้กับการสร้างหมายเลข 8.1.2482 ตัวเลขในวงเล็บในส่วนหัวบางส่วนอ้างอิงถึงหมายเลขที่สนับสนุนใน Microsoft Dynamics Lifecycle Services (LCS)

### <a name="you-can-enable-preview-features-in-sandbox-and-trial-environments"></a>คุณสามารถเปิดใช้งานคุณลักษณะตัวอย่างในสภาพแวดล้อมของ Sandbox และ Trail

เมื่อคุณจัดเตรียมอินสแตนซ์ใหม่ของ Talent คุณสามารถระบุได้ว่าชนิดของอินสแตนซ์เป็น การผลิต หรือ Sandbox ชนิดของอินสแตนซ์ Sandbox อนุญาตให้มีการทดสอบของคุณลักษณะใหม่ก่อนเวลา ถ้าคุณต้องการให้หนึ่งในอินสแตนซ์ที่มีอยู่ของคุณได้รับการปรับปรุงเป็นชนิดของอินสแตนซ์ Sandbox ติดต่อ ฝ่ายสนับสนุน เพื่อเริ่มต้นการร้องขอการเปลี่ยนแปลง

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเผยแพร่การเปลี่ยนแปลง โปรดดู [เตรียมใช้งาน Talent](./provisioning-talent.md)

### <a name="platform-update-29"></a>แพลตฟอร์ม update 29

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับการปรับปรุงแพลตฟอร์ม 29 ดูที่ [คุณลักษณะการแสดงตัวอย่างในการปรับปรุงแพลตฟอร์ม Dynamics 365 for Finance and Operations 29 (ตุลาคม 2019)](https://docs.microsoft.com/dynamics365/unified-operations/fin-and-ops/get-started/whats-new-platform-update-29)

### <a name="new-job-base-entity-available-in-data-management-framework-347202"></a>เอนทิตี้งานพื้นฐานใหม่พร้อมใช้งานในกรอบการจัดการข้อมูล (347202)

ด้วยรุ่นนี้ เอนทิตี้งานพื้นฐานใหม่พร้อมใช้งานสำหรับการนำเข้า/ส่งออกข้อมูล 

### <a name="worker-advanced-security-policy-incorrectly-displays-positions-in-position-hierarchy-354868"></a>นโยบายความปลอดภัยขั้นสูงของผู้ปฏิบัติงานแสดงตำแหน่งในลำดับชั้นของตำแหน่งงานที่ไม่ถูกต้อง (354868)

ด้วยรุ่นนี้ ตำแหน่งจะไม่แสดงว่าเปิดกับผู้ปฏิบัติงาน "ว่างเปล่า" อีกต่อไป เมื่อผู้ใช้มีการจำกัดการเข้าถึง

### <a name="job-form-close-box-wont-close-form-in-certain-situations-342467"></a>งานจากกล่องปิดจะไม่ได้ปิดแบบฟอร์มในบางสถานการณ์ (342467)

รุ่นนี้รวมถึงการเปลี่ยนแปลงที่อนุญาตให้ปิดแบบฟอร์มงานในทุกสถานการณ์

### <a name="new-case-on-employee-record-is-locked-for-human-resources-manager-role-337111"></a>กรณีใหม่บนเรกคอร์ดพนักงานถูกล็อคสำหรับบทบาทผู้จัดการฝ่ายทรัพยากรบุคคล (337111)

ด้วยการเปลี่ยนแปลงนี้ แบบฟอร์มการจัดการกรณีไม่ถูกล็อคไว้อีกต่อไปเมื่อเข้าถึงจากแบบฟอร์มพนักงาน

### <a name="employment-end-date-always-defaults-to-235959-351492"></a>วันที่สิ้นสุดการจ้างงานจะเป็นค่าเริ่มต้นเสมอเป็น 23:59:59 (351492)

ด้วยการเปลี่ยนแปลงนี้ คุณสามารถเปลี่ยนวันที่จ้างงานเป็นเวลาอื่นนอกเหนือจาก23:59:59 เมื่อสิ้นสุดการจ้างงานด้วยตนเอง

### <a name="unable-to-set-up-expiration-date-on-an-earning-code-through-data-management-336604"></a>ไม่สามารถตั้งค่าวันหมดอายุบนรหัสรายได้โดยใช้ Data management (336604)

ในรุ่นนี้ คุณสามารถตั้งค่ารหัสรายได้ที่หมดอายุผ่านทางเอนทิตี้ **PayrollWorkerPositionEarningCodeEntity**

### <a name="employee-development-analytic-report-doesnt-display-data-348737"></a>รายงานวิเคราะห์การพัฒนาพนักงานไม่แสดงข้อมูล (348737)

ในการนำออกใช้ของสัปดาห์นี้ การวิเคราะห์สำหรับทักษะของพนักงานจะถูกอัพเดตเพื่อให้การรายงานที่ถูกต้อง

### <a name="terms-of-employment-datetime-dont-default-to-beginning-of-day-349101"></a>เงื่อนไขการจ้างงาน/เวลาที่ไม่ได้ถูกกำหนดให้เป็นค่าเริ่มต้นสำหรับจุดเริ่มต้นของวัน (349101)

ด้วยการเปลี่ยนแปลงนี้ ขณะนี้วันที่/เวลาเริ่มต้นจะเริ่มต้นที่จุดเริ่มต้นของวันและสิ้นสุดที่วัน/เวลาที่สิ้นสุดของวัน

### <a name="changing-the-employment-end-date-on-worker-action-form-displays-an-error-354092"></a>การเปลี่ยนวันที่สิ้นสุดการจ้างงานในแบบฟอร์มการดำเนินการของผู้ปฏิบัติงานแสดงข้อผิดพลาด (354092) 

การเปลี่ยนแปลงนี้แก้ไขปัญหาเมื่อปรับเปลี่ยนวันที่สิ้นสุดการจ้างงานโดยเป็นส่วนหนึ่งของการดำเนินการของผู้ปฏิบัติงาน

### <a name="applying-onboarding-checklists-across-companies-338433"></a>ใช้รายการตรวจสอบการเตรียมความพร้อมระหว่างบริษัท (338433)

ปัจจุบันรุ่นนี้มีความสามารถในการใช้รายการตรวจสอบสำหรับพนักงานที่มีการจ้างงานในหน่วยงานตามกฎหมายอื่นที่ไม่ใช่นิติบุคคลที่ลงชื่อเข้าใช้

## <a name="in-preview"></a>ในการแสดงตัวอย่าง

### <a name="streamlined-employee-entry-and-navigation"></a>การป้อนข้อมูลและการนำทางของพนักงานที่มีประสิทธิภาพ

ขณะนี้ฟังก์ชันนี้พร้อมใช้งานในสภาพแวดล้อม sandbox เมื่อต้องการเปิดใช้งานคุณลักษณะนี้ นำทางไปที่ **การจัดการระบบ > ลิงค์ > การตั้งค่า > พารามิเตอร์ระบบ > คุณลักษณะการแสดงตัวอย่าง** เลือก **ฟอร์มผู้ปฏิบัติงานที่มีการปรับปรุงและการนำทาง** การทำเช่นนี้จะเปิดใช้งานการเปลี่ยนแปลงเหล่านี้สำหรับผู้ใช้ทั้งหมด คุณสามารถปิดตัวเลือกนี้ได้ตลอดเวลา

สำหรับข้อมูลเพิ่มเติม ให้ดู [การป้อนข้อมูลและการนำทางของพนักงานที่มีประสิทธิภาพ](./streamlined-employee-entry.md)
