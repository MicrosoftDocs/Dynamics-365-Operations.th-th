# <a name="whats-new-or-changed-in-dynamics-365-talent-october-01-2019"></a>มีอะไรใหม่หรือเปลี่ยนไปอย่างไรบ้างใน Dynamics 365 Talent (01 ตุลาคม 2019)

[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่หรือที่มีการเปลี่ยนแปลง อย่างใดอย่างหนึ่ง ใน Dynamics 365 Talent

## <a name="changes-in-attract"></a>การเปลี่ยนแปลงใน Attract

รุ่นนี้ประกอบด้วยการแก้ไขบักรองสำหรับ Dynamics 365 Talent: Attract

## <a name="changes-in-onboard"></a>การเปลี่ยนแปลงในการเตรียมความพร้อม

รุ่นนี้ประกอบด้วยการแก้ไขบักรองสำหรับ Dynamics 365 Talent: Onboard

## <a name="changes-in-core-hr"></a>การเปลี่ยนแปลงใน Core HR

การเปลี่ยนแปลงที่อธิบายไว้ในส่วนนี้นำไปใช้กับการสร้างหมายเลข 8.1.2509 ตัวเลขในวงเล็บในส่วนหัวบางส่วนอ้างอิงถึงหมายเลขที่สนับสนุนใน Microsoft Dynamics Lifecycle Services (LCS)

### <a name="streamlined-employee-entry-and-navigation"></a>การป้อนข้อมูลและการนำทางของพนักงานที่มีประสิทธิภาพ

ขณะนี้ฟังก์ชันนี้พร้อมใช้งานในสภาพแวดล้อมทั้งหมด เมื่อต้องการเปิดใช้งานคุณลักษณะนี้ นำทางไปที่ **การจัดการระบบ > ลิงค์ > การตั้งค่า > พารามิเตอร์ระบบ > คุณลักษณะการแสดงตัวอย่าง** เลือก **ฟอร์มผู้ปฏิบัติงานที่มีการปรับปรุงและการนำทาง** การทำเช่นนี้เปิดใช้งานการเปลี่ยนแปลงเหล่านี้สำหรับผู้ใช้ทั้งหมด คุณสามารถปิดตัวเลือกนี้ได้ตลอดเวลา

สำหรับข้อมูลเพิ่มเติม ให้ดู [การป้อนข้อมูลและการนำทางของพนักงานที่มีประสิทธิภาพ](./streamlined-employee-entry.md) เพื่อดูวิดีโอของการเปลี่ยนแปลงเหล่านี้ ให้ดูที่ [Dynamics 365 for Talent 2019 ปล่อยภาพรวมของเวฟ 2](https://aka.ms/ROGT19RW2ROV)

### <a name="you-can-enable-preview-features-in-sandbox-and-trial-environments"></a>คุณสามารถเปิดใช้งานคุณลักษณะตัวอย่างในสภาพแวดล้อมของ Sandbox และ Trail

เมื่อคุณจัดเตรียมอินสแตนซ์ใหม่ของ Talent คุณสามารถระบุได้ว่าชนิดของอินสแตนซ์เป็น การผลิต หรือ Sandbox อินสแตนซ์ของชนิด Sandbox อนุญาตให้มีการทดสอบของคุณลักษณะใหม่ได้ล่วงหน้า ถ้าคุณต้องการให้หนึ่งในอินสแตนซ์ที่มีอยู่ของคุณได้รับการปรับปรุงเป็นชนิดของอินสแตนซ์ Sandbox ติดต่อ ฝ่ายสนับสนุน เพื่อเริ่มต้นการร้องขอการเปลี่ยนแปลง

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเผยแพร่การเปลี่ยนแปลง โปรดดู [เตรียมใช้งาน Talent](./provisioning-talent.md)

### <a name="compensation-date-defaults-to-a-different-date-than-the-position-assignment-337694"></a>วันที่จ่ายค่าตอบแทนถูกกำหนดให้แตกต่างจากวันกำหนดตำแหน่งงานไว้เป็นค่าเริ่มต้น (337694)

ด้วยการเปลี่ยนแปลงนี้ วันที่เริ่มต้นของตอบแทนจะถูกกำหนดให้เป็นวันที่เริ่มต้นของการกำหนดตำแหน่งไว้เป็นค่าเริ่มต้น

### <a name="not-able-to-end-compensation-along-with-its-position-assignment-328993"></a>ไม่สามารถสิ้นสุดค่าตอบแทนพร้อมกับการกำหนดตำแหน่งงาน (328993)

ด้วยการเปลี่ยนแปลงนี้ คุณสามารถยุติค่าตอบแทนเมื่อคุณสิ้นสุดการมอบหมายตำแหน่งโดยใช้ **สิ้นสุดการมอบหมาย** ในหน้า **การมอบหมายตำแหน่งงานของผู้ปฏิบัติงาน** ด้วยการเปิดใช้งานการดำเนินการด้านบุคลากร

### <a name="bank-account-validation-requires-routing-and-account-numbers-in-all-locations-340403"></a>การตรวจสอบความถูกต้องของบัญชีธนาคารต้องใช้หมายเลขเส้นทางและหมายเลขบัญชีในทุกสถานที่ (340403)

ด้วยการเปลี่ยนแปลงของอาทิตย์นี้ มีการเพิ่มตัวเลือกการตั้งค่าคอนฟิกใหม่เพื่อปิดการใช้งาน **หมายเลขกำหนดเส้นทาง** และ **หมายเลขบัญชี** ที่จำเป็นต้องมีการตรวจสอบความถูกต้อง 

### <a name="attachments-are-not-enabled-in-mss-performance-journals-for-performance-feedback-341794"></a>เอกสารแนบไม่ได้เปิดใช้งานในสมุดรายวันประสิทธิภาพของ MSS สำหรับข้อติชมด้านประสิทธิภาพ (341794)

ด้วยการนำออกใช้ของสัปดาห์นี้ เอกสารแนบจะเปิดใช้งานสำหรับสินค้าที่มีข้อติชมในหน้าสมุดรายวันประสิทธิภาพ

### <a name="leave-request-details-dont-sync-from-common-data-service-to-talent-369608"></a>รายละเอียดคำขอลางานไม่สามารถซิงค์จาก Common Data Service ไปยัง Talent (369608)

ด้วยการเปลี่ยนแปลงเหล่านี้ รายละเอียดของคำขอลางานที่ถูกอัพเดตใน Common Data Service จะซิงค์กลับไปยัง Talent

### <a name="job-description-doesnt-display-for-the-job-in-the-skill-gap-analysis-page-369398"></a>คำอธิบายงานไม่ได้แสดงสำหรับงานที่อยุ่ในหน้าการวิเคราะห์ช่องว่างของทักษะ (369398)

ในการสร้างสัปดาห์นี้ คำอธิบายจะแสดงขึ้นเมื่อเลือกงานในหน้า **การวิเคราะห์ช่องว่างของทักษะ**
