# <a name="leave-and-absence-concepts"></a>แนวคิดการลางานและการหยุดงาน

แนวคิดและเงื่อนไขที่กล่าวถึงในหัวข้อนี้จะช่วยให้คุณกำหนดวิธีที่พนักงานได้รับเวลาหยุดพัก และวิธีการที่ยอดดุลของเวลาของพนักงานมีการคำนวณ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการจัดการการขาดงานและการลางาน ดู [การจัดการลางานและการขาดงาน](https://docs.microsoft.com/dynamics365/unified-operations/talent/leave-absence-overview)

## <a name="leave-plan-details"></a>รายละเอียดแผนการลางาน

### <a name="start-date"></a>วันที่เริ่มต้น

วันที่เริ่มต้นคือ วันที่ที่เริ่มต้นการประมวลผลการรับรู้ นอกจากนี้ วันที่เริ่มต้นจะทำหน้าที่เป็นวันที่ครบรอบปีที่จะใช้ในการคำนวณยอดเงินยกไปด้วย

## <a name="accruals"></a>การรับรู้รายได้รายจ่าย

การรับรู้กำหนดว่าเมื่อใด และบ่อยเพียงใดที่พนักงานจะได้รับเวลาหยุดพัก มีการกำหนดนโยบายเกี่ยวกับเวลาที่ควรให้การรับรู้และนโยบายเกี่ยวกับการแบ่งตามสัดส่วนในส่วน **การรับรู้** เมื่อตั้งค่าแผนการลางานและการขาดงาน

### <a name="accrual-frequency"></a>ความถี่ในการรับรู้

ความถี่ในการรับรู้จะกำหนดความถี่ของการลาที่คงค้างหรือได้รับ โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

- ประจำวัน
- รายสัปดาห์
- รายสองสัปดาห์
- รายครึ่งเดือน
- รายเดือน
- รายไตรมาส
- ปีละสองครั้ง
- รายปี
- ไม่มี

### <a name="accrual-period-basis"></a>เกณฑ์ระยะเวลาการยกยอดวันลา

เกณฑ์ระยะเวลาการยกยอดวันลากำหนดวันที่ที่ใช้ในการคำนวณระยะเวลาการยกยอดวันลา โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

- **วันที่เริ่มต้นแผน**
- **วันที่เฉพาะของพนักงาน** – เมื่อเลือกตัวเลือกนี้ ค่าจะกำหนดแหล่งที่มาของค่าวันที่เริ่มต้นที่จะใช้ในการคำนวณระยะเวลาการยกยอดวันลา โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

    - กำหนดเอง
    - วันที่ครบรอบปี
    - วันที่จ้างงานเดิม
    - วันที่ของอายุงาน
    - วันที่เริ่มต้นที่ปรับปรุงของผู้ปฏิบัติงาน
    - วันที่เริ่มต้นของผู้ปฏิบัติงาน

### <a name="accrual-award-date"></a>วันที่มอบวันลาที่ยกยอด

วันที่มอบวันลาที่ยกยอดกำหนดว่าเมื่อใดที่พนักงานจะได้รับเวลาหยุดพัก โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

- **วันที่สิ้นสุดรอบระยะเวลาการยกยอดวันลา** – พนักงานจะได้รับเวลาหยุดพักในวันสุดท้ายของระยะเวลาการยกยอดวันลา

    ในการรับรู้จำนวนที่ถูกต้อง กระบวนการการยกยอดวันลาต้องรวมรอบระยะเวลาทั้งหมด ตัวอย่างเช่น รอบระยะเวลาการยกยอดวันลาคือ วันที่ 1 มกราคม 2018 ถึง 31 มกราคม 2018 ในกรณีนี้ สำหรับเดือนมกราคมที่จะถูกรวม การรับรู้ต้องถูกรันสำหรับวันที่ 1 กุมภาพันธ์ 2018

- **วันที่เริ่มต้นรอบระยะเวลาการยกยอดวันลา** – พนักงานจะได้รับเวลาหยุดพักในวันแรกของระยะเวลาการยกยอดวันลา

### <a name="accrual-policy-on-enrollment"></a>นโยบายการรับรู้ในการลงทะเบียน

นโยบายการรับรู้ในการลงทะเบียนจะกำหนดวิธีการคำนวณการรับรู้ เมื่อมีการลงทะเบียนพนักงานในระหว่างรอบระยะเวลาการรับรู้ โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

- **ตามสัดส่วน** – ช่วงวันที่ระหว่างวันที่เริ่มต้นและวันที่ลงทะเบียน จะถูกใช้เพื่อกำหนดผลต่างในวันต่างๆ ผลต่างที่นั้นจะถูกใช้เมื่อมีการประมวลผลการรับรู้
- **การรับรู้แบบเต็ม** – จำนวนการรับรู้ทั้งหมด ตามระดับ มอบให้ระหว่างการประมวลผลการรับรู้แรก
- **ไม่มีการรับรู้** – ไม่มีการรับรู้ที่มอบให้ จนถึงรอบระยะเวลาการรับรู้ถัดไป

### <a name="accrual-policy-on-unenrollment"></a>นโยบายการรับรู้ในการยกเลิกการลงทะเบียน

นโยบายการรับรู้ในการยกเลิกการลงทะเบียนจะกำหนดวิธีการคำนวณการรับรู้ เมื่อมีการยกเลิกการลงทะเบียนพนักงานในระหว่างรอบระยะเวลาการรับรู้ โดยตัวเลือกที่คุณสามารถเลือกได้มีดังต่อไปนี้

- **ตามสัดส่วน** – ช่วงวันที่ระหว่างวันที่เริ่มต้นและวันที่ลงทะเบียน จะถูกใช้เพื่อกำหนดผลต่างในวันต่างๆ ผลต่างที่นั้นจะถูกใช้เมื่อมีการประมวลผลการรับรู้
- **การรับรู้แบบเต็ม** – จำนวนการรับรู้ทั้งหมด ตามระดับ มอบให้ระหว่างการประมวลผลการรับรู้แรก
- **ไม่มีการรับรู้** – ไม่มีการรับรู้ที่มอบให้ จนถึงรอบระยะเวลาการรับรู้ถัดไป

## <a name="accrual-schedule"></a>กำหนดการรับรู้

กำหนดการรับรู้กำหนดวิธีที่พนักงานจะรับรู้เวลาหยุดพัก และจำนวนที่พนักงานที่จะได้รับรู้ และยกยอดไป คุณสามารถสร้างระดับเพื่อให้มีการให้เวลาหยุดพักตามระดับต่างๆ

### <a name="months-of-service"></a>จำนวนเดือนของการทำงาน

ค่า **จำนวนเดือนของการทำงาน** กำหนดจำนวนต่ำสุดของเดือนที่พนักงานต้องทำงานเพื่อให้มีสิทธิในการรับรู้ ถ้าไม่มีค่าต่ำสุดสำหรับพนักงาน ตั้งค่าเป็น **0**

### <a name="hours-worked"></a>ชั่วโมงที่ทำงาน

ค่า **ชั่วโมงที่ทำงาน** กำหนดจำนวนต่ำสุดของชั่วโมงที่พนักงานต้องทำงานต่อรอบระยะเวลาการรับรู้ เพื่อให้มีสิทธิในการรับรู้ ถ้าไม่มีค่าต่ำสุดสำหรับพนักงาน ตั้งค่าเป็น **0**

### <a name="accrual-amount"></a>จำนวนวันลาคงค้าง

จำนวนในการรับรู้เป็นจำนวนชั่วโมงหรือจำนวนวันที่พนักงานจะรับรู้สำหรับแต่ละรอบระยะเวลา รอบระยะเวลาจะขึ้นอยู่กับความถี่ในการรับรู้

### <a name="minimum-balance"></a>ยอดดุลขั้นต่ำ

สามารถใช้ค่าลบสำหรับยอดดุลต่ำสุด ถ้าพนักงานสามารถร้องขอการลามากกว่าที่พวกเขามี

### <a name="maximum-carry-forward"></a>การยกยอดไปสูงสุด

กระบวนการรับรู้จะปรับปรุงยอดดุลการลางานที่เกินยอดดุลแบบยอดยกไปในวันครบรอบปีของวันที่เริ่มต้น

### <a name="grant-amount"></a>จำนวนที่จัดสรร

จำนวนที่จัดสรรเป็นจำนวนชั่วโมงหรือจำนวนวันเริ่มต้นที่พนักงานได้รับ เมื่อพวกเขาลงทะเบียนในแผนการลางานครั้งแรก ยอดเงินไม่รับรู้สำหรับแต่ละรอบระยะเวลาการรับรู้

## <a name="enrollments-and-balances"></a>การลงทะเบียนและยอดดุล

### <a name="enrollment-date"></a>วันที่ลงทะเบียน

วันที่ลงทะเบียนกำหนดเวลาที่พนักงานสามารถเริ่มต้นการรับรู้เวลาหยุดพัก ตัวอย่างเช่น ถ้ามีการลงทะเบียนพนักงานในแผนวันหยุดพักผ่อนในวันที่ 15 มิถุนายน 2018 เธอไม่สามารถรับรู้เวลาหยุดพักได้ก่อนวันที่ 15 มิถุนายน 2018

### <a name="current-balance"></a>ยอดดุลปัจจุบัน

ยอดดุลปัจจุบันจะเป็นจำนวนของการลางานที่พร้อมใช้งานสำหรับคำขอเวลาหยุดพัก จำนวนนี้รวมถึงการรับรู้ คำขอที่อนุมัติแล้ว และการปรับปรุงถึงวันที่ในปัจจุบัน
