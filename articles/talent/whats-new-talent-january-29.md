# <a name="whats-new-or-changed-in-dynamics-365-for-talent-january-31-2019"></a>มีอะไรใหม่หรือการเปลี่ยนแปลงอะไรใน Dynamics 365 for Talent (31 มกราคม 2019)

[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่หรือที่มีการเปลี่ยนแปลง อย่างใดอย่างหนึ่งใน Dynamics 365 for Talent

**สร้าง 8.1.2128**

## <a name="core-hr-changes"></a>การเปลี่ยนแปลงของ Core HR

### <a name="time-off-taken-on-leave-people-card-doesnt-consider-leave-plan-dates"></a>บัตรบุคคลของเวลาหยุดพักที่ใช้ขณะลางานไม่ถือว่าเป็นวันที่ของแผนการลางาน
สำหรับชนิดของการลางานที่มีแผนซึ่งไม่ได้รันในปีปฏิทิน การ์ด **ใช้แล้ว** จะแสดงเวลาหยุดพักที่จะนำมานับในปีที่กำหนดแผนการลางานไว้ ตัวอย่างเช่น ถ้าปีหยุดงานขององค์กรเป็นวันที่ 1 มิถุนายนถึงวันที่ 30 พฤษภาคม และพนักงานใช้วันลาหยุดไป 3 วันแล้วในเดือนธันวาคม การ์ด **ใช้แล้ว** ในวันที่ 15 มกราคม จะแสดงเป็น 3 วัน 

### <a name="accrual-amounts-not-matching-tier-date-basis"></a>ยอดเงินตามเกณฑ์คงค้างไม่ตรงกับพื้นฐานวันที่ของระดับ
มีการเพิ่มตัวเลือกใหม่ในการลางานและการขาดงาน (พารามิเตอร์ **ทรัพยากรบุคคล**) เพื่อให้ลูกค้าสามารถกำหนดว่าเมื่อใดที่มีผลบังคับใช้ของเดือนของพนักงานของวันที่ให้บริการ สำหรับบางองค์กร วันที่เป็นวันสิ้นสุดของเดือน แต่บางองค์กรอาจเป็นวันเริ่มต้นของเดือนถัดไป ตัวอย่างเช่น องค์กรหนึ่งอาจให้เวลาหยุดงานในวันที่ 31 ธันวาคม ในขณะที่อีกองค์กรหนึ่งอาจให้เวลาหยุดงานเป็นวันที่ 1 มกราคม ตัวเลือกนี้จะช่วยให้คุณสามารถเลือกได้ว่าควรมีการให้เวลาหยุดงานเมื่อใด 

### <a name="worker-hire-actions-are-stuck-in-workflow-complete-state"></a>การดำเนินการจ้างงานของผู้ปฏิบัติงานติดอยู่ในสถานะ "ลำดับงานเสร็จสมบูรณ์"
มีการเปลี่ยนแปลงเพื่อแก้ไขปัญหาที่มีลำดับงานจำนวนน้อยที่เสร็จสิ้นโดยมีสถานะเป็น "ลำดับงานเสร็จสมบูรณ์" ในตอนนี้ลำดับงานใหม่ควรย้ายไปเป็นสถานะ "เสร็จสมบูรณ์" เมื่อลำดับงานเสร็จสิ้น ลำดับงานใด ๆ ในสถานะลำดับงานที่เสร็จสมบูรณ์จะถูกเปลี่ยนเป็นสถานะข้อผิดพลาดเพื่ออนุญาตให้มีการอัพเดตหรือการลบถ้าจำเป็น
