# <a name="whats-new-or-changed-in-dynamics-365-talent-may-6-2019"></a>มีอะไรใหม่หรือการเปลี่ยนแปลงอะไรใน Dynamics 365 Talent (6 พฤษภาคม 2019)

[!include [banner](includes/banner.md)]

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่หรือที่มีการเปลี่ยนแปลง อย่างใดอย่างหนึ่ง ใน Dynamics 365 Talent

## <a name="changes-in-attract"></a>การเปลี่ยนแปลงใน Attract

### <a name="select-optional-documents-upon-offer-creation"></a>เลือกเอกสารเพิ่มเติม เมื่อสร้างข้อเสนอ

เมื่อคุณเลือกเท็มเพลตของแพคเกจข้อเสนอ ขณะนี้ Attract จะพร้อมต์ให้คุณเลือกเอกสารเพิ่มเติมที่เกี่ยวข้องจากเท็มเพลตของแพคเกจนั้น คุณสามารถเพิ่มเอกสารเพิ่มเติมอื่นๆ ได้ ในขณะที่จัดเตรียมข้อเสนอ

## <a name="changes-in-onboard"></a>การเปลี่ยนแปลงในการเตรียมความพร้อม

รุ่นนี้ประกอบด้วยการแก้ไขบักรองสำหรับ Dynamics 365 Talent: Onboard

## <a name="changes-in-core-hr"></a>การเปลี่ยนแปลงใน Core HR

การเปลี่ยนแปลงที่อธิบายไว้ในส่วนนี้นำไปใช้กับการสร้างหมายเลข 8.1.2282 ตัวเลขในวงเล็บในส่วนหัวบางส่วนอ้างอิงถึงหมายเลขที่สนับสนุนใน Microsoft Dynamics Lifecycle Services (LCS)

### <a name="platform-update-26-for-finance-and-operations"></a>การอัพเดตแพลตฟอร์ม 26 สำหรับ Finance and Operations

สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับการอัพเดตแพลตฟอร์ม 26 สำหรับ Finance and Operations ดูที่ [คุณลักษณะตัวอย่างใน Dynamics 365 Finance and Operations การอัพเดตแพลตฟอร์ม 26 (พฤษภาคม 2019)](https://docs.microsoft.com/dynamics365/unified-operations/fin-and-ops/get-started/whats-new-platform-update-26) 

### <a name="common-data-service-entity-support-for-custom-fields"></a>การสนับสนุนเอนทิตี Common Data Service สำหรับฟิลด์ที่กำหนดเอง

ในการนำออกใช้ในสัปดาห์นี้ ขณะนี้เอนทิตีต่อไปนี้สนับสนุนฟิลด์ที่กำหนดเอง: ความถี่ในการคำนวณสวัสดิการ อัตราการคำนวณสวัสดิการ ชนิดของสวัสดิการ ปฏิทินงาน วันหยุดในปฏิทินงาน รอบการชำระค่าจ้าง และชนิดรหัสของผู้ปฏิบัติงาน

### <a name="leave-mass-enrollment-changing-the-tier-basis-to-seniority-date-doesnt-refresh-the-initial-accrual-rate-318526"></a>การยกเลิกการลงทะเบียนโดยรวม ซึ่งเปลี่ยนแปลงเกณฑ์ระดับเป็น "วันที่ของอายุงาน" ไม่ได้รีเฟรชอัตราคงค้างเริ่มต้น (318526)

เมื่อคุณลงทะเบียนพนักงานโดยรวม และเปลี่ยนเกณฑ์ระดับ ขณะนี้เกณฑ์คงค้างเริ่มต้นจะสะท้อนถึงเกณฑ์ระดับที่คุณเลือกไว้

### <a name="workflow-showing-duplicate-place-holders-313636"></a>ลำดับงานที่แสดงตัวยึดที่ซ้ำกัน (313636)

การเปลี่ยนแปลงในรุ่นนี้ตัดการซ้ำกันของตัวยึด เมื่อมีการส่งการแจ้งเตือนลำดับงาน

### <a name="dimension-fields-arent-updated-when-using-open-in-excel-176261"></a>ฟิลด์มิติจะไม่ได้รับการปรับปรุง เมื่อใช้ "เปิดใน Excel" (176261)

ด้วยการนำออกใช้นี้ ขณะนี้คุณสามารถปรับปรุงมิติทางการเงินได้โดยใช้ **เปิดใน Excel** จากหน้า **ผู้ปฏิบัติงาน** 

### <a name="worker-address-created-in-common-data-service-isnt-synced-to-talent-317555"></a>ที่อยู่ของผู้ปฏิบัติงานที่สร้างใน Common Data Service ไม่ได้ซิงค์กับ Talent (317555)

ด้วยการเปลี่ยนแปลงนี้ ที่อยู่ที่สร้างขึ้นใน Common Data Service ถูกอัพเดตใน Talent: Core HR


## <a name="in-preview"></a>ในการแสดงตัวอย่าง

### <a name="new-page-to-validate-position-hierarchy-data"></a>สร้างหน้าใหม่เพื่อตรวจสอบความถูกต้องของข้อมูลลำดับชั้นของตำแหน่ง

ทรัพยากรบุคคล (HR) และผู้ดูแลระบบ สามารถตรวจสอบความถูกต้องของลำดับชั้นการจัดการสำหรับการอ้างอิงแบบหมุนเวียนใดๆ ที่อาจมีการนำเข้าโดยไม่ได้ตั้งใจ คุณสามารถเข้าถึงหน้าใหม่นี้ได้จาก **การจัดการองค์กร > ลิงค์ > ตำแหน่ง > การตรวจสอบความถูกต้องของลำดับชั้นของตำแหน่ง**

### <a name="specify-reason-codes-on-leave-types"></a>ระบุรหัสเหตุผลในชนิดของการลางาน

องค์กรอาจต้องการข้อมูลเพิ่มเติมเกี่ยวกับคำขอเวลาหยุดพัก ขณะนี้คุณสามารถระบุรหัสเหตุผลสำหรับชนิดการลางาน และอนุญาตให้พนักงานเลือกรหัสเหตุผลในคำขอเวลาหยุดพักของพวกเขาได้

### <a name="require-reason-codes-for-specific-leave-types-on-time-off-requests"></a>ต้องมีรหัสเหตุผลสำหรับชนิดการลางานเฉพาะในคำขอเวลาหยุดพัก

องค์กรอาจต้องมีรหัสเหตุผลในชนิดการลางานเฉพาะ เมื่อพนักงานส่งคำขอเวลาหยุดพัก ข้อกำหนดนี้อาจมีอยู่ เนื่องจากนโยบายของบริษัทหรือข้อกำหนดในการปฏิบัติตามกฎระเบียบ ขณะนี้คุณสามารถระบุชนิดการลางานที่ต้องใช้รหัสเหตุผล และคุณสามารถกำหนดให้พนักงานระบุรหัสเหตุผลสำหรับชนิดการลางานในคำขอเวลาหยุดพักพวกเขา

### <a name="provide-a-leave-and-absence-transaction-list-for-hr"></a>แสดงรายการธุรกรรมการขาดงานและการลางานสำหรับฝ่ายทรัพยากรบุคคล

ความสามารถในการติดตามเวลาหยุดพักของพนักงาน และทำความเข้าใจวิธีคำนวณเวลาหยุดพัก ไม่เพียงช่วยฝ่ายทรัพยากรบุคคลในการตอบคำถามพนักงานเท่านั้น แต่ยังช่วยให้มั่นใจถึงรางวัลเวลาหยุดพักที่ถูกต้องสำหรับพนักงานอีกด้วย ขณะนี้ฝ่ายทรัพยากรบุคคลมีมุมมองใหม่เกี่ยวกับธุรกรรม (เงินช่วยเหลือ การค้างรับค้างจ่าย การปรับปรุง และคำขอ) เพื่อให้เจ้าหน้าที่ฝ่ายทรัพยากรบุคคลสามารถดูเหตุผลหลังยอดดุลได้

## <a name="coming-soon"></a>เร็วๆ นี้

### <a name="indicate-instance-type-when-provisioning-talent"></a>ระบุชนิดของอินสแตนซ์ เมื่อเตรียมใช้งาน Talent

เมื่อเตรียมใช้งานอินสแตนซ์ใหม่ของ Talent คุณจะสามารถบ่งชี้ได้ว่าชนิดของอินสแตนซ์เป็น **การผลิต** หรือ **Sandbox** ซึ่งอนุญาตให้สามารถทดสอบคุณลักษณะใหม่ๆ ได้ก่อนเวลา อินสแตนซ์ Talent ที่มีอยู่ทั้งหมดจะได้รับการปรับปรุงเป็นชนิดอินสแตนซ์การผลิต ถ้าคุณต้องการให้หนึ่งในอินสแตนซ์ที่มีอยู่ของคุณได้รับการปรับปรุงเป็นชนิดของอินสแตนซ์ Sandbox โปรดติดต่อฝ่ายสนับสนุนเพื่อเริ่มต้นการร้องขอการเปลี่ยนแปลง
