# <a name="whats-new-or-changed-in-dynamics-ax-70-february-2016"></a>มีอะไรใหม่หรือเปลี่ยนแปลงใน Dynamics AX 7.0 (กุมภาพันธ์ 2016)

[!include [banner](../includes/banner.md)]

บทความนี้อธิบายคุณลักษณะที่ใหม่หรือเปลี่ยนแปลงในแอพลิเคชัน Microsoft Dynamics AX รุ่น 7.0 รุ่นนี้ประกอบด้วยคุณลักษณะทั้งแพลตฟอร์มและแอพพลิเคชัน และถูกนำออกใช้ในเดือนกุมภาพันธ์ 2016

| รวมเข้ากับ Microsoft Office | ไม่พร้อมใช้งาน | ความสามารถใหม่รวมแอพลิเคชัน Excel Data Connector หน้า **โปรแกรมออกแบบสมุดงาน** ส่งออก API และการจัดการเอกสาร | คุณสามารถสร้างการแก้ไขปัญหาในการทำงานสำหรับผู้ใช้ของคุณ |
| สร้าง ทดสอบ และปรับใช้โดยอัตโนมัติ | พร้อมใช้งานบางส่วน | ปรับใช้โทโพโลยีนักพัฒนา โดยการใช้นักพัฒนาและสร้าง VM ตั้งค่าคอนฟิกอัตโนมัติ สร้าง VM ที่จะพบ สร้างโมดูลจาก Visual Studio Online (VSO) และรันการทดสอบ คอมไพล์และข้อมูลอ้างอิงโมดูล C\# และ X ++ ได้รับการสนับสนุน | จะเพิ่มประสิทธิภาพการทำงานของนักพัฒนา โดยการลดต้นทุนและความพยายามที่ใช้สำหรับการทดสอบและการตรวจสอบ |
| กำหนดการจัดทำชั้นที่มากเกินไปและส่วนขยายด้วยตนเอง | ส่วนขยายไม่พร้อมใช้งาน | รุ่นปัจจุบันของ Dynamics AX มีแบบจำลองการแก้ไข/ปรับปรุงตามคำสั่งใหม่ | คุณสามารถกำหนดรหัสต้นทางและข้อมูลเมตาขององค์ประกอบแบบจำลองที่มีการจัดส่ง โดย Microsoft หรือ บุคคลที่สามที่เป็นคู่ค้าของ Microsoft |
| สร้างตัวควบคุมใหม่และองค์ประกอบ UI โดยใช้ X ++ และกรอบงานของเว็บสมัยใหม่ | ตัวควบคุมแบบกำหนดเองที่ขึ้นอยู่กับกรอบงานภายนอก เช่น Microsoft ActiveX และ Windows Presentation Foundation (WPF) | การสร้างตัวควบคุมในเวอร์ชันปัจจุบันทำได้ง่ายขึ้น สามารถใช้กรอบงาน X ++ สำหรับการลักษณะการทำงานของแอพลิเคชันและตรรกะทางธุรกิจ และไคลเอนต์ HTML/JavaScript-based ใช้งานได้สำหรับการแสดงภาพสมัยใหม่ | การควบคุมของคุณสามารถได้รับการออกแบบเพื่อดูและทำงานเหมือนการควบคุมสำเร็จรูป (OOB) ของ Dynamics AX |
| ประเมินและปรับแต่งประสิทธิภาพการทำงาน โดยการใช้เครื่องมือใหม่ | PerfSDK ชุดเครื่องมือการขยายข้อมูล แอพลิเคชันเว็บ Trace Parser และ PerfTimer ไม่พร้อมใช้งาน | PerfSDK ชุดเครื่องมือการขยายข้อมูล แอพลิเคชันเว็บ Trace Parser Web และ PerfTimer ใหม่ | ชุดพัฒนาซอฟต์แวร์ (SDK) ช่วยให้คุณสามารถทดสอบ และตรวจสอบกระบวนการทางธุรกิจที่สำคัญทั้งหมดสำหรับผลการปฏิบัติงานที่มีผู้ใช้คนเดียว และ ถ้ามีการรันทดสอบผู้ใช้หลายคน ชุดเครื่องมือการขยายข้อมูลช่วยให้คุณขยายการทดสอบประสิทธิภาพการทำงานทั้งหมดที่ต้องมีข้อมูลหลักและขยายข้อมูลธุรกรรมอย่างถูกต้อง Trace Parser ช่วยให้คุณตรวจสอบการทดสอบประสิทธิภาพการทำงานของผู้ใช้คนเดียวหรือแบบหลายผู้ใช้ที่รัน PerfTimer ช่วยให้คุณสามารถดูว่า แบบสอบถามใด ๆ หรือการเรียกวิธีเฉพาะใด ๆ จะก่อให้เกิดปัญหาเกี่ยวกับประสิทธิภาพ ดังนั้น คุณจึงไม่ต้องทำการติดตาม และวิเคราะห์ข้อมูลโดยละเอียดทั้งหมด |
| แสดงมุมมองที่ปรับปรุงได้ โดยการใช้ OData | ไม่พร้อมใช้งาน | รุ่นปัจจุบันของ Dynamics AX แนะนำจุดสิ้นสุดบริการ OData สาธารณะที่เปิดใช้งานไปยังข้อมูล Dynamics AX ในแบบที่สม่ำเสมอทั่วทั้งช่วงที่กว้างของไคลเอ็นต์ | โซลูชันของคุณสามารถโต้ตอบกับบริการ RESTful ใช้ข้อมูลร่วมกันในลักษณะสามารถมองเห็นได้ และเปิดใช้งานการรวมแบบกว้าง โดยการใช้โพรโทคอลสแต็ค HTTP |
| ใช้ประโยชน์จาก business connector เพื่อสร้างตรรกะทางธุรกิจ และสถานการณ์จำลองการสนับสนุนการรวม | Business connector จะพร้อมใช้งานในการเรียกในรหัส X ++ จากรหัสที่มีการจัดการ เราขอแนะนำให้คุณใช้ business connector เฉพาะผู้สร้างตรรกะทางธุรกิจใน C\# ไม่ใช่สำหรับสถานการณ์จำลองการรวม | Business connector ไม่ได้รับการสนับสนุน ความต้องการเขียนจะได้รับมาจากข้อเท็จจริงที่ว่า X ++ ถูกคอมไพล์ให้เป็นรหัสที่มีการจัดการ ดังนั้น interop จะง่ายขึ้น สถานการณ์จำลองการรวมเป็นไปตามโดยการใช้ OData | คุณไม่สามารถใช้ business connector ไปข้างหน้า |
| เลือกสเกล (นั่นคือ จำนวนตำแหน่งทศนิยม) บนฟิลด์ฐานข้อมูลจริงและชนิดข้อมูลที่ถูกขยาย (EDTs) | สเกล 16 เป็นระดับเริ่มต้น และไม่สามารถเปลี่ยนโดยนักพัฒนา | ขณะนี้ EDTs และฟิลด์มีคุณสมบัติระดับที่สามารถใช้ได้กับแต่ละฟิลด์และ EDT ค่าเริ่มต้นจะถูกกำหนดไว้ที่ 6 ไม่ใช่ 16 | ประสิทธิภาพการทำงานกับตาราง NCCI (ในหน่วยความจำสนับสนุน SQL) จะเร็วขึ้น โดยใบสั่งของขนาดเมื่อใช้มาตราส่วนเล็กลง เปลี่ยนแปลงมาตราส่วนสำหรับการใช้งานของแต่ละฟิลด์ที่จำเป็นต้องมี |
