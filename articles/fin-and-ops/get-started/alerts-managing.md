### <a name="frequency-for-change-based-events"></a><span data-ttu-id="224db-116">ความถี่สำหรับเหตุการณ์ตามการเปลี่ยนแปลง</span><span class="sxs-lookup"><span data-stu-id="224db-116">Frequency for change-based events</span></span>
<span data-ttu-id="224db-117">สำหรับเหตุการณ์ตามการเปลี่ยนแปลง คุณสามารถตั้งค่างานในชุดงานที่ทริกเกอร์การประมวลผลเหตุการณ์ได้ทันทีหลังจากที่เหตุการณ์ถูกบันทึกโดยระบบ </span><span class="sxs-lookup"><span data-stu-id="224db-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the event is logged by the system.</span></span> <span data-ttu-id="224db-118">ถ้าคุณตั้งค่าชุดงานให้เกิดซ้ำบ่อยมากขึ้น ผู้ใช้ได้รับข้อความแจ้งเตือนของพวกเขาเร็วมากขึ้น หลังจากเกิดการเปลี่ยนแปลง</span><span class="sxs-lookup"><span data-stu-id="224db-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="224db-119">อย่างไรก็ตาม ความถี่สูงสำหรับการประมวลผลชุดงานอาจส่งผลกระทบประสิทธิภาพของระบบในทางลบ</span><span class="sxs-lookup"><span data-stu-id="224db-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

<span data-ttu-id="224db-120">ในทางตรงกันข้าม ชุดงานที่เกิดซ้ำถี่น้อยลง และที่ถูกจัดกำหนดการไว้สำหรับเวลาที่โหลดของระบบต่ำ อาจช่วยปรับปรุงประสิทธิภาพของระบบ</span><span class="sxs-lookup"><span data-stu-id="224db-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="224db-121">อย่างไรก็ตาม ความถี่ต่ำสำหรับการประมวลผลชุดงานอาจไม่ตรงกับความต้องการของผู้ใช้สำหรับข้อความแจ้งเตือนตามเวลาที่กำหนด</span><span class="sxs-lookup"><span data-stu-id="224db-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="224db-122">ดังนั้น เมื่อคุณกำหนดความถี่ของการประมวลผลชุดงานสำหรับเหตุการณ์ตามการเปลี่ยนแปลง ควรพิจารณาถึงความสมดุลระหว่างกำหนดเวลาของข้อความแจ้งเตือนและประสิทธิภาพของทั้งระบบ</span><span class="sxs-lookup"><span data-stu-id="224db-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="224db-123">ข้อควรพิจารณาเหล่านี้ยิ่งมีความเกี่ยวข้องมากเท่าขึ้น เมื่อจำนวนผู้ใช้ที่สร้างกฎการแจ้งเตือนมีมากขึ้น</span><span class="sxs-lookup"><span data-stu-id="224db-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="224db-124">ความถี่ไม่มีผลต่อจำนวนของเหตุการณ์ที่ต้องถูกประมวลผล</span><span class="sxs-lookup"><span data-stu-id="224db-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="224db-125">อย่างไรก็ตาม ถ้าผู้ใช้ที่มากยิ่งขึ้นสร้างกฎ ก็ยิ่งต้องมีการเช็คเพิ่มเติมมากขึ้น</span><span class="sxs-lookup"><span data-stu-id="224db-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="224db-126">การแลกเปลี่ยนข้อมูลชนิดนี้อาจส่งผลกระทบต่อประสิทธิภาพของระบบ</span><span class="sxs-lookup"><span data-stu-id="224db-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="224db-127">ความเสี่ยงของความถี่ของชุดงานที่ต่ำ</span><span class="sxs-lookup"><span data-stu-id="224db-127">The risks of low batch frequency</span></span>
<span data-ttu-id="224db-128">ถ้าคุณตั้งค่าความถี่ต่ำสำหรับการประมวลผลชุดงานสำหรับเหตุการณ์ตามการเปลี่ยนแปลง ข้อมูลที่เกี่ยวข้องกับเงื่อนไขในกฎการแจ้งเตือนอาจมีการเปลี่ยนแปลง ก่อนที่ชุดงานจะถูกประมวลผล</span><span class="sxs-lookup"><span data-stu-id="224db-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="224db-129">ดังนั้น คุณอาจสูญเสียข้อความแจ้งเตือน</span><span class="sxs-lookup"><span data-stu-id="224db-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="224db-130">ตัวอย่างเช่น กฎการแจ้งเตือนถูกตั้งค่าให้ทริกเกอร์ข้อความแจ้งเตือน เมื่อเหตุการณ์เป็น **ผู้ติดต่อของลูกค้าเปลี่ยนแปลง** และเงื่อนไขคือ **ลูกค้า = BB**</span><span class="sxs-lookup"><span data-stu-id="224db-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="224db-131">กล่าวได้อีกอย่างหนึ่งว่า เมื่อผู้ติดต่อลูกค้าสำหรับลูกค้า BB มีการเปลี่ยนแปลง จะมีการบันทึกเหตุการณ์</span><span class="sxs-lookup"><span data-stu-id="224db-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="224db-132">อย่างไรก็ตาม ระบบการประมวลผลชุดงานถูกตั้งค่าเพื่อให้การประมวลผลชุดงานเกิดขึ้นถี่น้อยกว่าการป้อนข้อมูล</span><span class="sxs-lookup"><span data-stu-id="224db-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="224db-133">ถ้ามีการเปลี่ยนชื่อลูกค้าจาก **BB** เป็น **AA** ก่อนการประมวลผลเหตุการณ์ ข้อมูลในฐานข้อมูลจะไม่สอดคล้องกับเงื่อนไขในกฎ **ลูกค้า = BB** อีกต่อไป</span><span class="sxs-lookup"><span data-stu-id="224db-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="224db-134">ดังนั้น เมื่อมีการประมวลผลเหตุการณ์ จะไม่มีสร้างการแจ้งเตือน</span><span class="sxs-lookup"><span data-stu-id="224db-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
