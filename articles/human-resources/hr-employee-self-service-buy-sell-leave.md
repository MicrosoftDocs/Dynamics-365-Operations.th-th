# <a name="buy-and-sell-leave"></a><span data-ttu-id="053de-103">ซื้อและขายวันลางาน</span><span class="sxs-lookup"><span data-stu-id="053de-103">Buy and sell leave</span></span>

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

<span data-ttu-id="053de-104">ใน Dynamics 365 Human Resources คุณสามารถส่งคำขอเพื่อซื้อและขายวันลาตามนโยบายการซื้อและการขายที่ตั้งค่าโดยบริษัทของคุณ</span><span class="sxs-lookup"><span data-stu-id="053de-104">In Dynamics 365 Human Resources, you can submit requests to buy and sell leave based on the buy and sell leave policies set up by your company.</span></span>  

## <a name="request-to-buy-leave"></a><span data-ttu-id="053de-105">คำขอเพื่อซื้อการลางาน</span><span class="sxs-lookup"><span data-stu-id="053de-105">Request to buy leave</span></span>

1. <span data-ttu-id="053de-106">ในพื้นที่ทำงาน **การบริการตนเองของพนักงาน** ให้เลือก **คำขอเพื่อซื้อการลางาน** ในไทล์ **ยอดดุลเวลาหยุดพัก**</span><span class="sxs-lookup"><span data-stu-id="053de-106">In the **Employee self service** workspace, select **Buy leave request** in the **Time Off Balances** tile.</span></span> 

2. <span data-ttu-id="053de-107">เพิ่ม **ชนิดการลางาน** และป้อน **จำนวน** สำหรับจำนวนของการลางานที่คุณต้องการซื้อ</span><span class="sxs-lookup"><span data-stu-id="053de-107">Add a **Leave type** and enter an **Amount** for the amount of leave you'd like to buy.</span></span> 

3. <span data-ttu-id="053de-108">เลือก **ส่ง** เมื่อคุณพร้อมที่จะส่งคำขอของคุณ</span><span class="sxs-lookup"><span data-stu-id="053de-108">Select **Submit** when you're ready to submit your request.</span></span> 

<span data-ttu-id="053de-109">ยอดคงเหลือของคุณจะอัพเดตโดยอัตโนมัติหรือดำเนินการผ่านกระบวนการอนุมัติก่อนที่จะอัพเดต</span><span class="sxs-lookup"><span data-stu-id="053de-109">Your balances will either automatically update or go through an approval process before updating.</span></span> <span data-ttu-id="053de-110">การดำเนินการนี้จะขึ้นอยู่กับวิธีการตั้งค่าคอนฟิกนโยบายการซื้อ</span><span class="sxs-lookup"><span data-stu-id="053de-110">This depends on how the buy policy has been configured.</span></span>

## <a name="request-to-sell-leave"></a><span data-ttu-id="053de-111">คำขอในการขายวันลา</span><span class="sxs-lookup"><span data-stu-id="053de-111">Request to sell leave</span></span>

1. <span data-ttu-id="053de-112">ในพื้นที่ทำงาน **การบริการตนเองของพนักงาน** ให้เลือก **คำขอขายวันลา** ในไทล์ **ยอดการลาหยุดคงเหลือ**</span><span class="sxs-lookup"><span data-stu-id="053de-112">In the **Employee self service** workspace, select **Sell leave request** in the **Time Off Balances** tile.</span></span> 

2. <span data-ttu-id="053de-113">เพิ่ม **ชนิดการลางาน** และป้อน **จำนวน** สำหรับจำนวนของวันลาที่คุณต้องการขาย</span><span class="sxs-lookup"><span data-stu-id="053de-113">Add a **Leave type** and enter an **Amount** for the amount of leave you'd like to sell.</span></span> 

3. <span data-ttu-id="053de-114">เลือก **ส่ง** เมื่อคุณพร้อมที่จะส่งคำขอของคุณ</span><span class="sxs-lookup"><span data-stu-id="053de-114">Select **Submit** when you're ready to submit your request.</span></span>

<span data-ttu-id="053de-115">ยอดคงเหลือของคุณจะอัพเดตโดยอัตโนมัติหรือดำเนินการผ่านกระบวนการอนุมัติก่อนที่จะอัพเดต</span><span class="sxs-lookup"><span data-stu-id="053de-115">Your balances will either automatically update or go through an approval process before updating.</span></span> <span data-ttu-id="053de-116">การดำเนินการนี้จะขึ้นอยู่กับวิธีการตั้งค่าคอนฟิกนโยบายการซื้อ</span><span class="sxs-lookup"><span data-stu-id="053de-116">This depends on how the buy policy has been configured.</span></span>


## <a name="troubleshooting"></a><span data-ttu-id="053de-117">การแก้ไขปัญหา</span><span class="sxs-lookup"><span data-stu-id="053de-117">Troubleshooting</span></span> 

<span data-ttu-id="053de-118">ถ้าลำดับงานคำขอการซื้อและขายวันลางานล้มเหลว ผู้ใช้ที่มีสิทธิ์ **EssLeaveBuySellRequestApprover** สามารถตรวจทานบันทึกข้อความของคำขอการซื้อและขายทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="053de-118">If a buy or sell leave request workflow fails, users with the **EssLeaveBuySellRequestApprover** privilege can review the message log for all leave buy and sell requests.</span></span> <span data-ttu-id="053de-119">ถ้าต้องการดำเนินการนี้ ให้ไปที่ **การลางานและการหยุดงาน >ลิงค์ > คำขอการซื้อและขายวันลางาน > ล็อกข้อความ** (บนซ้าย)</span><span class="sxs-lookup"><span data-stu-id="053de-119">To do this, go to **Leave and absence > Link > Buy and sell leave requests > Message log** (on the upper left).</span></span> <span data-ttu-id="053de-120">**ล็อกข้อความ** จะแสดงผู้ใช้เกี่ยวกับวิธีการประมวลผลธุรกรรมและประวัติลำดับงานที่เกี่ยวข้อง</span><span class="sxs-lookup"><span data-stu-id="053de-120">The **Message log** shows users how the transactions were processed and the associated workflow history.</span></span>
