# <a name="use-workflows-to-manage-employee-information"></a>ใช้ลำดับงานเพื่อจัดการข้อมูลพนักงาน

[!INCLUDE [PEAP](../includes/peap-1.md)]

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

บทความนี้อธิบายถึงวิธีการที่คุณสามารถใช้ความสามารถของลำดับงานสำหรับฝ่ายทรัพยากรบุคคลเพื่อจัดการข้อมูลพนักงาน ตัวอย่างเช่น คุณสามารถเชื่อมโยงลำดับงานกับตำแหน่งงาน และตั้งค่าคอนฟิกลำดับงานการอนุมัติที่เริ่มต้นเมื่อพนักงานเปลี่ยนบันทึกของพวกเขา

ความสามารถของลำดับงานสำหรับฝ่ายทรัพยากรบุคคลให้ลำดับงานจำนวนมากสำหรับการจัดการกิจกรรมของฝ่ายทรัพยากรบุคคล นอกจากนี้ ยังมีตัวเลือกจำนวนมากที่พร้อมใช้งานเพื่อให้คุณสามารถปรับเปลี่ยนลำดับงานที่เฉพาะเจาะจง และเชื่อมโยงกับลำดับชั้นการรายงาน ลำดับงานจะพร้อมช่วยจัดการการเปลี่ยนแปลงข้อมูลพนักงานหลากหลายประเภท คุณสามารถเชื่อมโยงลำดับงานกับตำแหน่ง จากนั้น ถ้าพนักงานเปลี่ยนบันทึกพนักงานของตน ลำดับงานจะเริ่มต้นซึ่งต้องมีการอนุมัติก่อนที่จะบันทึกข้อมูลใหม่ ลำดับงานจะถูกกำหนดล่วงหน้าให้กับชนิดของข้อมูลต่อไปนี้เพื่อช่วยคุณให้จัดการได้อย่างมีประสิทธิภาพ และรักษาความถูกต้องของข้อมูลพนักงานของคุณ

-   หมายเลขรหัส
-   หลักสูตร
-   การศึกษา
-   รูปภาพ
-   รายการที่ยืม
-   ประสบการณ์ทำงาน
-   ประสบการณ์โครงการ
-   ทักษะ
-   ตำแหน่งตัวแทนของบริษัท
-   การดำเนินการของฝ่ายทรัพยากรบุคคล
-   การลงทะเบียนหลักสูตร

เมื่อพนักงานได้รับการจ้างงาน โอนย้าย สิ้นสุดการจ้างงาน ลำดับงานอาจรวมถึงกระบวนการตรวจทาน ด้วยวิธีนี้ คุณสามารถแก้ไขเอกสารหรือกำหนดเงื่อนไขของการดำเนินการให้เป็นส่วนหนึ่งของลำดับงาน เมื่อกระบวนการตรวจทานเสร็จสมบูรณ์ เอกสารหรือการดำเนินการจะเสร็จสมบูรณ์ และลำดับงานจะย้ายไปยังขั้นตอนการอนุมัติขั้นสุดท้าย

## <a name="associate-a-workflow-with-a-position-hierarchy"></a>เชื่อมโยงลำดับงานกับลำดับชั้นของตำแหน่ง

คุณสามารถเชื่อมโยงลำดับงานกับตำแหน่งใดๆ ที่คุณตั้งค่าคอนฟิก การกำหนดเส้นทางลำดับงานมีการใช้สองลำดับชั้น: **การจัดการ** และ **ตั้งค่าคอนฟิกได้**

- ลำดับชั้น **การจัดการ** แสดงถึงโครงสร้างการรายงานของบริษัทหรือองค์กร สำหรับข้อมูลเพิ่มเติมเกี่ยวกับชนิดลำดับชั้นนี้ โปรดดูที่ [ตำแหน่งที่รายงานต่อ](hr-personnel-positions.md#reports-to-position)
- ลำดับชั้น **ตั้งค่าคอนฟิกได้** แสดงลำดับชั้นเมทริกซ์หรือที่กำหนดเอง สำหรับข้อมูลเพิ่มเติมเกี่ยวกับชนิดลำดับชั้นนี้ โปรดดูที่ [ความสัมพันธ์](hr-personnel-positions.md#relationships)

### <a name="managerial-hierarchy-example"></a>ตัวอย่างลำดับชั้นการจัดการ

คุณอาจตั้งค่าคอนฟิกลำดับงานเพื่อให้เมื่อพนักงานส่งคำขอซื้อคอมพิวเตอร์ใหม่ คำขอถูกกำหนดเส้นทางไปยังผู้จัดการของพนักงานและผู้จัดการข้ามระดับ เมื่อคุณตั้งค่าขั้นตอนของลำดับงาน ให้ตั้งค่าฟิลด์ **ชนิดการกำหนด** เป็น **ลำดับชั้น** แท็บ **ชนิดลำดับชั้น** จะพร้อมใช้งาน สำหรับตัวอย่างนี้ ให้เลือกลำดับชั้น **การจัดการ**

### <a name="configurable-hierarchy-example"></a>ตัวอย่างลำดับชั้นที่ตั้งค่าคอนฟิกได้

ถ้าตำแหน่งถูกเชื่อมโยงกับลำดับชั้นการรายงานเมทริกซ์ คุณอาจตั้งค่าคอนฟิกลำดับงานที่ใช้ในค่าใช้จ่ายสำหรับโครงการที่เฉพาะเจาะจงไปยังผู้นำโครงการแทนผู้จัดการของพนักงาน ในกรณีนี้ ให้ตั้งค่าฟิลด์ **ชนิดการกำหนด** เป็น **ลำดับชั้น** จากนั้นบนแท็บ **ชนิดลำดับชั้น** ให้เลือกลำดับชั้น **ตั้งค่าคอนฟิกได้** หลังจากตั้งค่าลำดับงาน ให้เลือก **เชื่อมโยง** ลำดับชั้นในหน้า **การตั้งค่าลำดับงาน** เพื่อเลือกลำดับชั้นที่ควรใช้สำหรับการกำหนดเส้นทางลำดับงาน

> [!IMPORTANT]
> เมื่อส่งเอกสาร ธุรกรรม หรือเรกคอร์ดหลักสหรับการอนุมัติลำดับงาน ตําแหน่งหลักของผู้ส่งจะถูกใช้เพื่อระบุว่าควรส่งเอกสารนั้นไปที่ใครเป็นลำดับถัดไป

### <a name="hierarchy-setting-in-workflow-parameters"></a>การตั้งค่าลำดับชั้นในพารามิเตอร์ลำดับงาน

1. บนหน้า **พารามิเตอร์ลำดับงาน** ไปที่ **การกำหนดเส้นทางลำดับชั้น**
2. ตามค่าเริ่มต้น ตัวเลือก **ใช้ลำดับชั้นของตำแหน่ง** ตั้งค่าเป็น **ไม่** ในกรณีนี้ ลำดับงานจะใช้ตําแหน่งหลักของผู้ปฏิบัติงานเมื่อนําทางลำดับชั้น ตั้งค่าตัวเลือกเป็น **ใช่** เพื่อสร้างลำดับงานที่จะใช้ข้อมูลหลักของตําแหน่งเมื่อนําทางลำดับชั้น

### <a name="additional-example"></a>ตัวอย่างเพิ่มเติม 

พนักงาน Grace Sturman มีสองตําแหน่งคือ ที่ปรึกษาและผู้ฝึกสอน ตําแหน่งหลักของ Graces คือผู้ฝึกสอน เมื่อเธอไม่ได้ฝึกอบรมพนักงานใหม่ เธอพร้อมสำหรับงานการให้คำปรึกษา จากตําแหน่งหลักของเธอ Grace ขึ้นตรงกับ Claire ซึ่งเป็นผู้อํานวยการฝ่ายทรัพยากรบุคคล Claire ขึ้นตรงกับ Charlie สำหรับตําแหน่งที่ปรึกษาของเธอ Grace มีความสัมพันธ์แบบเส้นประหลายเส้น ทั้งนี้ขึ้นอยู่กับโครงการ

บริษัทของ Grace สร้างกฎการกำหนดเส้นทางลำดับงานตามลำดับชั้นที่ **ตั้งค่าคอนฟิกได้** (ลำดับชั้นแบบเมทริกซ์/ตามโครงการ) ลำดับชั้นนี้ใช้ตำแหน่งที่ปรึกษาของ Grace ถ้าตัวเลือก **ใช้ลำดับชั้นตำแหน่ง** ตั้งค่าเป็น **ไม่** เมื่อเอกสารถูกกำหนดเส้นทางไปยัง Grace สำหรับการอนุมัติ ลำดับงานจะดูที่ตําแหน่งหลัก (ผู้ฝึกสอน) ของเธอเพื่อระบุว่าเอกสารควรถูกจัดเส้นทางไปที่ตำแหน่งใดถัดไป ในกรณีนี้ จะมีการกำหนดเส้นทางไปยัง Claire ก่อน แล้วจึงไปยัง Charlie หากตัวเลือกตั้งค่าเป็น **ใช่** และลำดับงานใช้ลำดับชั้นที่ **ตั้งค่าคอนฟิกได้** ลำดับงานจะดูที่ตำแหน่งที่ปรึกษาของ Grace และความสัมพันธ์ในการรายงานเพื่อกําหนดว่าเอกสารควรถูกกำหนดเส้นทางไปที่ใด

### <a name="configure-a-human-resources-workflow"></a>ตั้งค่าคอนฟิกลำดับงานของฝ่ายทรัพยากรบุคคล
เมื่อต้องการตั้งค่าคอนฟิกลำดับงานพื้นฐานที่เริ่มต้นเมื่อพนักงานร้องขอการเปลี่ยนแปลงรหัสประจำตัวของพวกเขา ให้ทำตามขั้นตอนเหล่านี้

1.  ในหน้า **ลำดับงานของฝ่ายทรัพยากรบุคคล** เลือก **สร้าง**
2.  ในรายการของลำดับงานที่พร้อมใช้งาน เลือก **หมายเลขประจำตัว**
3.  เลือก **เรียกใช้** เพื่อเปิดโปรแกรมออกแบบลำดับงาน และจากนั้น ป้อนชื่อผู้ใช้และรหัสผ่านของคุณ
4.  ย้ายองค์ประกอบ **อนุมัติหมายเลขรหัส** จากรายการขององค์ประกอบลำดับงานไปยังพื้นที่ทำงานของตัวออกแบบ
5.  เชื่อมต่อองค์ประกอบการอนุมัติเพื่อ **เริ่มต้น** และ **สิ้นสุด**
6.  ดับเบิลแท็บ (หรือดับเบิลคลิก) **อนุมัติองค์ประกอบ** เลือกและกดแช่ (หรือคลิกขวา) แล้วเลือก **คุณสมบัติ**
7.  ทำตามขั้นตอนเหล่านี้เพื่อเพิ่มคำแนะนำเกี่ยวกับรายการงาน:

    1.  เลือก **การกำหนด** และจากนั้นเลือก **ลำดับชั้น** ภายใต้ชนิดการกำหนด
    2.  ภายใต้การเลือก **ลำดับชั้น** เลือก **ลำดับชั้นที่ตั้งค่าคอนฟิกได้**
    3.  เพิ่มเงื่อนไขการหยุด และปิดหน้า

8.  ทำตามคำแนะนำเพิ่มเติม
9.  เลือก **บันทึกและปิด** เรียกใช้ลำดับงานใหม่เมื่อกล่องโต้ตอบเปิดขึ้น และเลือก **เปิดใช้งาน**
10. ไปที่ **ทรัพยากรบุคคล** &gt; **ตำแหน่ง** &gt; **ชนิดลำดับชั้นของตำแหน่งงาน**
11. เลือก **เมทริกซ์**
12. เพิ่มลำดับงาน **หมายเลขรหัสผู้ปฏิบัติงาน** ลงในรายการ
