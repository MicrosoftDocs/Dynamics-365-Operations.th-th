# <a name="process-enrollment-eligibility"></a>ประมวลผลการมีสิทธิ์ในการลงทะเบียน

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

บทความนี้อธิบายถึงวิธีการรันกระบวนการมีสิทธิ์ในการลงทะเบียน

1. ในพื้นที่ทำงาน **การจัดการสวัสดิการ** ภายใต้ **กำลังประมวลผล** ให้เลือก **กำลังประมวลผลการมีสิทธิ์ในการลงทะเบียน**

2. ในกล่องโต้ตอบ **ดำเนินการประมวลผลสวัสดิการการลงทะเบียน** ให้ระบุค่าสำหรับฟิลด์ต่อไปนี้:

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **รอบระยะเวลาการลงทะเบียน** | รอบระยะเวลาการลงทะเบียนที่จะประมวลผลการมีสิทธิ์ในการลงทะเบียน |
   | **นิติบุคคล** | นิติบุคคลที่จะประมวลผลการมีสิทธิ์ในการลงทะเบียน |
   | **ผู้ปฏิบัติงาน** | ผู้ปฏิบัติงานที่จะประมวลผลการมีสิทธิ์ในการลงทะเบียน ถ้าคุณปล่อยฟิลด์นี้ว่างไว้ การมีสิทธิ์ในการลงทะเบียนจะมีการดำเนินการสำหรับผู้ปฏิบัติงานทั้งหมด |
   | **แผนสวัสดิการ** | แผนสวัสดิการที่จะประมวลผลการมีสิทธิ์ในการลงทะเบียน

3. ถ้าคุณต้องการดำเนินการกระบวนการในพื้นหลัง ให้เลือก **ดำเนินการในพื้นหลัง** และงานต่อไปนี้:

   1. ป้อนข้อมูลสำหรับกระบวนการ

   2. หากต้องการตั้งค่าให้มีการรันงานนั้นซ้ำ เลือก **การเกิดซ้ำ** ป้อนข้อมูลการเกิดซ้ำ และเลือก **ตกลง**

   3. เมื่อต้องการตั้งค่าการแจ้งเตือนงาน เลือก **การแจ้งเตือน** เลือกการแจ้งเตือนที่จะรับ และจากนั้นเลือก **ตกลง**

   4. เลือก **ตกลง** กระบวนการจะดำเนินการกับพารามิเตอร์ที่คุณตั้งไว้

4. เลือก **ตกลง**

## <a name="view-process-results"></a>ดูผลลัพธ์ของกระบวนการ

บทความนี้อธิบายถึงวิธีการดูผลลัพธ์กระบวนการการมีสิทธิ์

1.  ในพื้นที่ทำงาน **การจัดการสวัสดิการ** ภายใต้ **กำลังประมวลผล** ให้เลือก **ผลลัพธ์ของกระบวนการ**

2.  ในแบบฟอร์ม **ผลลัพธ์ของกระบวนการ** จะมีการระบุฟิลด์ต่อไปนี้ดังนี้:

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **รหัสกระบวนการ** | รหัสเฉพาะสำหรับชุดของผู้ปฏิบัติงาน นิติบุคคล และการรันกระบวนการ |
   | **ชนิดของกระบวนการ** | ซึ่งระบุกระบวนการที่มีการรัน ตัวอย่างเช่น: การลงทะเบียน  |
   | **การประทับเวลา** | เวลาที่มีการรันกระบวนการของการมีสิทธิ์ |
   | **นิติบุคคล** | นิติบุคคลที่ระบุในระหว่างกระบวนการลงทะเบียน |
   | **ผู้ปฏิบัติงาน** | ผู้ปฏิบัติงานที่ได้รับการประมวลผล |
   | **แผน | แผนสวัสดิการที่มีความพยายามในการลงทะเบียน |
   | **กฎการมีสิทธิ์** | กฎการมีสิทธิ์ที่ได้รับการประมวลผล ถ้าพบข้อผิดพลาดก่อนที่จะมีการรันสิทธิ์ นี่จะว่างเปล่า ตัวอย่างเช่น ถ้ายังไม่ได้กำหนดค่าตอบแทนสำหรับผู้ปฏิบัติงาน กระบวนการการมีสิทธิ์จะไม่รันและฟิลด์นี้จะถูกปล่อยว่างไว้ |
   | **สถานะของผลลัพธ์** | การทำเช่นนี้จะเป็นมีสิทธิ์หรือไม่มีสิทธิ์ สถานะผลลัพธ์จะไม่สามารถใช้ได้ ถ้าผู้ปฏิบัติงานไม่ตรงกับเกณฑ์ของกฎการมีสิทธิ์ ถ้าผู้ปฏิบัติงานไม่มีข้อมูลที่จำเป็น เช่น ความถี่ในการชำระค่าจ้างหรือค่าตอบแทนคงที่ หรือหากมีข้อมูลที่ขาดหายไปในแผนสวัสดิการที่ทำให้ผู้ปฏิบัติงานไม่ได้ลงทะเบียน |
   | **ข้อความผลลัพธ์** | บ่งชี้ว่าเหตุใดผู้ปฏิบัติงานจึงไม่มีสิทธิ์สำหรับแผนสวัสดิการ หรือมีการส่งผ่านกฎการมีสิทธิ์ที่เหมาะสมหรือไม่ |
