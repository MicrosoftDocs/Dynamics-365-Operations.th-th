# <a name="configure-eligibility-rules-and-options"></a>ตั้งค่าคอนฟิกกฎการใช้สิทธิ์และตัวเลือก

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

หลังจากที่คุณได้ตั้งค่าคอนฟิกพารามิเตอร์ที่จำเป็นสำหรับการจัดการสิทธิประโยชน์ใน Microsoft Dynamics 365 Human Resources คุณจะสามารถสร้างกฎการใช้สิทธิ์ การรวมกลุ่ม รอบระยะเวลา และโปรแกรมที่คุณจะเชื่อมโยงกับแผนสวัสดิการสิทธิประโยชน์ของคุณได้

## <a name="create-an-eligibility-rule"></a>สร้างกฎการใช้สิทธิ์

กฎการใช้สิทธิ์จะกำหนดพนักงานที่สามารถลงทะเบียนในแผนสิทธิประโยชน์แต่ละรายการได้ หลังจากที่คุณกำหนดกฎการใช้สิทธิ์มแล้ว คุณจะสามารถกำหนดกฎนั้นลงในแผนสวัสดิการสิทธิประโยชน์ จากนั้นคุณจะสามารถประมวลผลสิทธิ์ในการลงทะเบียนเพื่อดูว่าพนักงานคนใดมีสิทธิ์สำหรับแต่ละแผน 

ในระหว่างการลงทะเบียน พนักงานสามารถเลือกแผนสิทธิประโยชน์ได้ ถ้าพนักงานไม่มีสิทธิ์ในแผนสิทธิประโยชน์ตามกฎการใช้สิทธิ์หลังจากที่มีการลงทะเบียนแล้ว จะไม่สามารถยกเลิกการลงทะเบียนโดยอัตโนมัติได้ โดยทั่วไป เมื่อเหตุการณ์ในชีวิตที่เกิดขึ้นมีผลกระทบต่อสิทธิ์ตามแผน รอบระยะเวลาการลงทะเบียนจะเริ่มขึ้นสำหรับพนักงานเพื่อเลือกแผนที่พนักงานมีสิทธิ์ได้รับ 

1. ในพื้นที่ทำงาน **การจัดการสิทธิประโยชน์** ภายใต้ **การตั้งค่า** ให้เลือก **กฎการใช้สิทธิ์และตัวเลือก**

2. บนแท็บ **กฎการใช้สิทธิ์** ให้เลือก **ใหม่** เพื่อสร้างกฎการใช้สิทธิ์ เมื่อต้องการดูแผนที่สัมพันธ์กับกฎการใช้สิทธิ์ ให้เลือก **แผนที่แนบมาด้วย**

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **กฎการมีสิทธิ์** | รหัสเฉพาะของกฎการใช้สิทธิ์ |
   | **คำอธิบาย** | คำอธิบายของกฎการใช้สิทธิ์ |
   | **วันที่และเวลามีผลใช้ตั้งแต่** | วันที่เริ่มต้นของกฎใช้สิทธิ์ | 
   | **วันที่และเวลามีผลใช้ถึง** | วันที่สิ้นสุดของกฎการใช้สิทธิ์ |
   | **ประเภทพนักงานผู้ใช้สิทธิ์** | ระบุว่าจะใช้ชนิดพนักงานของพนักงานในกฎการใช้สิทธิ์ในสิทธิประโยชน์หรือไม่ |
   | **ชนิดผู้ปฏิบัติงาน** | ชนิดของผู้ปฏิบัติงาน ถ้า **การใช้ชนิดพนักงาน** ถูกตั้งค่าเป็น **ใช่** |
   | **ใช้สถานะพนักงาน** | ระบุว่าจะใช้สถานะพนักงานของพนักงานในกฎการใช้สิทธิ์ในสิทธิประโยชน์หรือไม่ |
   | **สถานะ** | สถานะของพนักงาน ถ้า **การใช้สถานะของพนักงาน** ถูกตั้งค่าเป็น **ใช่** ถ้า **การใช้สถานะของพนักงาน** ถูกตั้งค่าเป็น **ไม่ใช่** ส่วนนี้จะไม่ถูกใช้งาน |
   | **ใช้ประเภทการจ้างงาน** | ระบุว่าจะใช้ค่า **ประเภทการจ้างงาน** ของพนักงานเป็นส่วนหนึ่งในกฎการใช้สิทธิ์ในสิทธิประโยชน์หรือไม่ | 
   | **ประเภทการจ้างงาน** | ประเภทการจ้างงานของพนักงานถ้า **การใช้ประเภทการจ้างงาน** ถูกตั้งค่าเป็น **ใช่** |
   | **ใช้กฎพนักงานใหม่** | ระบุว่าจะใช้ค่าของรอบระยะเวลาการจ้างงานใหม่ของการจ้างงานใหม่เป็นส่วนหนึ่งของกฎการใช้สิทธิ์ในสิทธิประโยชน์หรือไม่ |
   | **รอบระยะเวลาการลงทะเบียน** | รอบระยะเวลาเมื่ออนุญาตให้มีการลงทะเบียนการจ้างงานใหม่ ถ้าคุณตั้งค่านี้ในพารามิเตอร์ การตั้งค่าพารามิเตอร์จะมีความสำคัญเหนือกว่ารายการนี้ |
   | **ใช้สถานะการจ้างงานในอดีต** | ระบุว่าจะใช้สถานะการจ้างงานก่อนหน้าของพนักงานเป็นส่วนหนึ่งในกฎการมีสิทธิ์ในสิทธิประโยชน์หรือไม่ ตัวอย่างเช่น คุณสามารถระบุกฎการมีสิทธิ์ที่ละเว้นรอบระยะเวลาการรอคอยความครอบคลุมสำหรับพนักงานทั้งหมดที่แปลงจากสถานะ **เลิกจ้าง** เป็น **ถูกจ้าง** ภายใน 90 วันของการจ้างงานก่อนหน้านี้ |

4. ภายใต้ **เงื่อนไขเพิ่มเติม** ให้เลือกตัวเลือกต่อไปนี้และเพิ่มข้อมูลตามความจำเป็น

   | ตัวเลือก | คำอธิบาย |
   | --- | --- |
   | **อายุที่มีสิทธิ์** | ระบุช่วงอายุหรือช่วงที่จำเป็นสำหรับการตอบสนองกฎการใช้สิทธิ์ |
   | **แผนกที่มีสิทธิ์** | ระบุแผนกหรือแผนกที่พนักงานต้องประจำอยู่ในการตอบสนองกฎการใช้สิทธิ์ |
   | **ชนิดการจ้างงานที่มีสิทธิ์** | ระบุประเภทการจ้างงานหรือประเภทที่พนักงานต้องถูกจัดประเภทในการตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น แบบเต็มเวลาหรือนอกเวลา |
   | **งานที่มีสิทธิ์** | ระบุตัวงานหรืองานที่ตอบสนองกฎการใช้สิทธิ์ ตัวงานเกี่ยวข้องกับตำแหน่งงาน และตำแหน่งที่ถูกกรอกข้อมูลโดยพนักงาน |
   | **ฟังก์ชันงานที่มีสิทธิ์** | ระบุหน้าที่งานหรือหน้าที่ที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น พนักงานขายหรือช่างเทคนิค |
   | **ชนิดงานที่มีสิทธิ์** | ระบุชนิดของงานหรือชนิดที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น งานธุรการหรือผู้บริหาร |
   | **นิติบุคคลที่มีสิทธิ์** | ระบุกลุ่มตามกฎหมายหรือนิติบุคคลที่มีผลบังคับใช้สำหรับกฎการใช้สิทธิ์ ตัวอย่างเช่น Contoso Entertainment System USA |
   | **ภูมิภาคของค่าตอบแทนที่มีสิทธิ์** | ระบุที่ตั้งของพนักงานที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น central US |
   | **ตำแหน่งที่มีสิทธิ์** | ระบุตำแหน่งงานหรือตำแหน่งที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น ผู้ช่วยฝ่ายทรัพยากรบุคคลหรือผู้จัดการฝ่ายทรัพยากรบุคคล |
   | **ชนิดตำแหน่งที่มีสิทธิ์** | ระบุชนิดของตำแหน่งหรือชนิดที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น เต็มเวลา |
   | **รัฐที่มีสิทธิ์** | ระบุรัฐหรือจังหวัดที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น นอร์ทดาโกตา สหรัฐอเมริกา หรือบริติชโคลัมเบีย แคนาดา |
   | **เงื่อนไขการจ้างงานที่มีสิทธิ์** | ระบุเงื่อนไขการจ้างงานที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น แบบสมัครใจหรือสัญญาจ้างแบบกลุ่ม |
   | **สหภาพที่มีสิทธิ์** | ระบุสมาชิกภาพของสหภาพแรงงานที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น คนขับรถยกของอเมริกา </br></br>เมื่อใช้กฎการใช้สิทธิ์ตามสหภาพ บันทึกของสหภาพของผู้ปฏิบัติงานต้องมีวันที่สิ้นสุดสมาชิกภาพ คุณไม่สามารถปล่อยให้ว่างได้ |
   | **รหัสไปรษณีย์ที่มีสิทธิ์** | ระบุรหัสไปรษณีย์ที่ตอบสนองกฎการใช้สิทธิ์ ตัวอย่างเช่น 58104 |

5. ภายใต้ **รายละเอียดเพิ่มเติม** คุณสามารถดูรายละเอียดเพิ่มเติมได้ดังต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **ฟิลด์ผู้ใช้ที่มีสิทธิ์** | ระบุกฎการมีใช้สิทธิ์เพิ่มเติมตามฟิลด์ที่ลูกค้ากำหนดเอง |
   | **ชนิดการมีสิทธิ์** | ระบุประเภทเกณฑ์ที่คุณเลือกไว้ภายใต้ **เงื่อนไขเพิ่มเติม** |
   | **การอ้างอิงการมีสิทธิ์** | ระบุค่าที่คุณเลือกไว้ภายใต้ **เงื่อนไขเพิ่มเติม** |
   | **คำอธิบาย** | คำอธิบายที่คุณเลือกไว้ภายใต้ **เงื่อนไขเพิ่มเติม** |

6. เลือก **บันทึก**

## <a name="using-custom-fields-in-eligibility-rules"></a>การใช้ฟิลด์ที่กำหนดเองในกฎการมีคุณสมบัติเหมาะสม

[ฟิลด์ที่กำหนดเอง](hr-developer-custom-fields.md) สามารถสร้างภายในทรัพยากรบุคคลเพื่อติดตามข้อมูลเพิ่มเติม ฟิลด์เหล่านี้สามารถเพิ่มลงในส่วนติดต่อผู้ใช้โดยตรงได้ และเพิ่มคอลัมน์ลงในตารางพื้นฐานแบบไดนามิก  

ฟิลด์ที่กำหนดเองสามารถใช้ในกระบวนการการมีคุณสมบัติเหมาะสม กฎการมีคุณสมบัติเหมาะสมสามารถใช้ค่าฟิลด์ที่กำหนดเองหนึ่งค่าหรือมากกว่าเพื่อตัดสินการมีคุณสมบัติเหมาะสมของพนักงาน  เมื่อต้องการเพิ่มฟิลด์ที่กำหนดเองให้กับกฎที่มีอยู่หรือเพื่อสร้างกฎใหม่ ให้ไปที่ **การจัดการสวัสดิการ > ลิงค์ >การตั้งค่า > กฎการมีคุณสมบัติเหมาะสม>สิทธิ์ของฟิลด์ที่กำหนดเอง** ภายในหน้านี้ คุณสามารถสร้างกฎที่ใช้ฟิลด์ที่กําหนดเองหนึ่งฟิลด์หรือหลายฟิลด์ และคุณสามารถกําหนดค่าหลายค่าให้กับแต่ละฟิลด์ที่กําหนดเองเพื่อระบุสิทธิ์ได้

ตารางต่อไปนี้สนับสนุนฟิลด์ที่กำหนดเองซึ่งสามารถใช้ในการประมวลผลการมีคุณสมบัติ

- ผู้ปฏิบัติงาน (HcmWorker)  
- งาน (HcmJob)  
- ตําแหน่งงาน (HcmPosition)  
- รายละเอียดตําแหน่งงาน (HcmPositionDetail)  
- การกำหนดตำแหน่งของผู้ปฏิบัติงาน  
- การจ้างงาน (HcmEmployment)  
- EmploymentDetails (HcmEmploymentDetails)  
- รายละเอียดงาน (HcmJobDetails)  

ชนิดฟิลด์ที่กำหนดเองต่อไปนี้สนับสนุนในการประมวลผลการมีคุณสมบัติ

- ข้อความ  
- รายการเบิกสินค้า  
- หมายเลข  
- ทศนิยม  
- กล่องกาเครื่องหมาย  

ตารางต่อไปนี้แสดงข้อมูลฟิลด์ฟอร์มสิทธิ์ของฟิลด์ที่กำหนดเอง

| ฟิลด์  | คำอธิบาย |
|--------|-------------|
| ชื่อ | ชื่อของเงื่อนไขที่จะสร้าง |
| ชื่อตาราง | ชื่อตารางที่ประกอบด้วยฟิลด์ที่กำหนดเองซึ่งจะถูกใช้กับกฎการมีคุณสมบัติเหมาะสม |
| ชื่อฟิลด์ | ฟิลด์ที่จะใช้สำหรับกฎการมีคุณสมบัติเหมาะสม |
| ชนิดตัวดำเนินการ | แสดงตัวดำเนินการที่ใช้ในการกำหนดค่าการมีคุณสมบัติเหมาะสมของฟิลด์ที่กําหนดเอง |
| มูลค่า | แสดงค่าที่ใช้ในการกำหนดค่าการมีคุณสมบัติเหมาะสมของฟิลด์ที่กําหนดเอง |

## <a name="eligibility-logic"></a>ตรรกะการมีคุณสมบัติเหมาะสม

ส่วนต่อไปนี้จะอธิบายถึงวิธีการประมวลผลสิทธิ์ในสิทธิประโยชน์

### <a name="rules-assigned-to-a-plan"></a>กฎที่กำหนดให้กับแผน 
เมื่อมีการมอบหมายกฎการมีคุณสมบัติเหมาะสมหลายกฎให้กับแผนสวัสดิการ พนักงานต้องเป็นไปตามอย่างน้อยหนึ่งกฎเพื่อให้มีสิทธิ์ลงทะเบียนในแผนสวัสดิการ  ในตัวอย่างต่อไปนี้ พนักงานต้องเป็นไปตามความต้องการของกฎ **ชนิดงาน** หรือกฎ **พนักงานที่ทำงานอยู่**

![พนักงานต้องเป็นไปตามความต้องการของกฎ ชนิดงาน หรือกฎ พนักงานที่ทำงานอยู่](media/RulesAssignedToAPlan.png)
 
### <a name="criteria-within-an-eligibility-rule"></a>เงื่อนไขภายในกฎการมีคุณสมบัติเหมาะสม 
ภายในกฎหนึ่งๆ ที่คุณกําหนดเงื่อนไขที่เป็นไปตามกฎนี้ ในตัวอย่างข้างต้น เกณฑ์ของกฎ **ชนิดงาน** คือชนิดงาน = คณะกรรมการ ดังนั้น พนักงานต้องเป็นผู้อํานวยการจึงจะมีสิทธิได้รับ นี่คือกฎที่มีเกณฑ์เพียงข้อเดียวภายในกฎ

คุณสามารถกําหนดกฎที่มีหลายเงื่อนไขได้ เมื่อคุณกําหนดหลายเงื่อนไขภายในกฎการมีคุณสมบัติเหมาะสม พนักงานต้องเป็นไปตามเงื่อนไขทุกข้อภายในกฎเพื่อให้มีสิทธิ์รับแผนสวัสดิการ 

ตัวอย่างเช่น กฎ **พนักงานที่ทำงานอยู่** ด้านบนจะเป็นไปตามเงื่อนไขต่อไปนี้ เพื่อให้พนักงานมีสิทธิตามกฎ **พนักงานที่ทำงานอยู่** พนักงานต้องได้รับการว่าจ้างในนิติบุคคล USMF *และ* มีชนิดตําแหน่งเต็มเวลา  

![เงื่อนไขภายในกฎการมีคุณสมบัติเหมาะสม](media/CriteriaWithinAnEligibilityRule.png) 
 
### <a name="multiple-conditions-within-criteria"></a>หลายเงื่อนไขภายในเกณฑ์

กฎสามารถขยายเพิ่มเติมเพื่อใช้หลายเงื่อนไขภายในเงื่อนไขหนึ่งๆ พนักงานต้องเป็นไปตามเงื่อนไขอย่างน้อยหนึ่งเงื่อนไขเพื่อให้มีสิทธิ์ เมื่อต้องการสร้างตามตัวอย่างข้างต้น **พนักงานที่ทำงานอยู่** สามารถขยายกฎของพนักงานที่ใช้งานอยู่เพื่อรวมพนักงานที่เป็นพนักงานชั่วคราวด้วยได้ ดังนั้น ในขณะนี้ พนักงานต้องเป็นพนักงานใน USMF *และ* พนักงานเต็มเวลาหรือพนักงานชั่วคราว

### <a name="eligibility-conditions-within-a-custom-field-criterion"></a>เงื่อนไขการมีคุณสมบัติเหมาะสมภายในเกณฑ์ฟิลด์ที่กำหนดเอง 
คุณสามารถใช้ฟิลด์ที่กำหนดเองคล้ายกันเมื่อสร้างกฎการมีคุณสมบัติเหมาะสมและงานในลักษณะเดียวกัน ตัวอย่างเช่น คุณอาจต้องการเสนอการเบิกจ่ายเงินผ่านอินเทอร์เน็ตให้แก่พนักงาน Fargo และโคเปนเฮเกนที่ที่ทำงานที่บาน เนื่องจากต้นทุนอินเทอร์เน็ตสูงกว่าในสถานที่ตั้งดังกล่าว เมื่อต้องการทำเช่นนี้ ให้สร้างฟิลด์สองฟิลด์ที่กำหนดเอง: **ที่ตั้งสำนักงาน** (รายการเบิกสินค้า) และ **การทำงานจากที่บ้าน** (กล่องกาเครื่องหมาย) จากนั้นสร้างกฎที่มีชื่อว่า **พนักงาน WFH** เงื่อนไขของกฎคือที่ **ที่ตั้งสํานักงาน = Fargo** หรือ **โคเปนเฮเกน** *และ*  ที่ **ทำงานจากที่บ้าน = ใช่**

กฎการมีคุณสมบัติเหมาะสมที่กำหนดเองของอิเล็กทรอนิกส์ต้องมีการตั้งค่าตามที่ระบุในภาพต่อไปนี้ 

![เงื่อนไขการมีคุณสมบัติเหมาะสมภายในเกณฑ์ฟิลด์ที่กำหนดเอง](media/EligibilityConditionsWithinACustomFieldCriterion.png) 
 
## <a name="configure-bundles"></a>ตั้งค่าคอนฟิกการขายรวม

การขายรวมเป็นชุดของแผนสิทธิประโยชน์ที่มีความเกี่ยวข้องกัน คุณสามารถใช้กลุ่มสิทธิประโยชน์รวมถึงแผนสิทธิประโยชน์ของกลุ่มที่พนักงานต้องเลือกเพื่อลงทะเบียนในแผนสิทธิประโยชน์บางอย่างที่อาจขึ้นอยู่กับการลงทะเบียนแผนสิทธิประโยชน์อื่น ตัวอย่างที่จะบอกว่าเมื่อไหร่ที่คุณอาจต้องการใช้การขายรวม

- กลุ่มแผนสุขภาพที่รวมการประกันสุขภาพหักลดหย่อนขั้นสูงด้วยบัญชีเงินฝากออมทรัพย์ที่เกี่ยวกับสุขภาพ (HSA)

- แผนประกันชีวิตกับข้อบังคับแผนประกันชีวิตของพนักงานจะใช้ร่วมกับแผนประกันชีวิตของผู้อยู่ในอุปการะด้วย การทำเช่นนี้เพื่อให้มั่นใจว่าพนักงานไม่สามารถเลือกความคุ้มครองของกระกันชีวิตของผู้อยู่ในอุปการะได้โดยที่ไม่ได้ลงทะเบียนความคุ้มครองของพนักงาน

1. ในพื้นที่ทำงาน **การจัดการสิทธิประโยชน์** ภายใต้ **การตั้งค่า** ให้เลือก **กฎการใช้สิทธิ์และตัวเลือก**

2. ในแท็บ **การขายรวม** ให้เลือก **สร้าง** เพื่อสร้างกลุ่ม เมื่อต้องการดูแผนที่สัมพันธ์กับการขายรวม ให้เลือก **แผนที่แนบมาด้วย**

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **การขายรวม** | ตัวระบุเฉพาะสำหรับการขายรวม |
   | **คำอธิบาย** | คำอธิบายเกี่ยวกับการขายรวม |
   | **หลัก** | ระบุว่าหนึ่งในแผนในการขายรวมต้องทำเครื่องหมายเป็นแผนหลักหรือไม่ แผนหลักจะต้องถูกเลือกในระหว่างที่เปิดให้ลงทะเบียนซึ่งเป็นส่วนหนึ่งของการขายรวมก่อนที่ผู้ดูแลระบบสิทธิประโยชน์จะสามารถยืนยันการเลือกสิทธิประโยชน์ของพนักงาน |
   | **วันที่และเวลามีผลใช้ตั้งแต่** | วันและเวลาของการขายรวมมีผลบังคับใช้ |
   | **ใช้ได้ถึง** | วันที่การขายรวมสิ้นสุดการบังคับใช้ ค่าเริ่มต้น 12/31/2154 แสดงว่าไม่เคยเกิดขึ้นเลย |

4. เลือก **บันทึก**

## <a name="configure-periods"></a>ตั้งค่าคอนฟิกรอบระยะเวลา

รอบระยะเวลาจะกำหนดเมื่อสิทธิประโยชน์มีผลบังคับใช้และเมื่อพนักงานได้รับอนุญาตให้ลงทะเบียนได้ คุณสามารถสร้างรอบระยะเวลาได้มากเท่าที่คุณต้องการเพื่อยืนยันสิทธิประโยชน์จากการลงทะเบียนแบบเปิดและรอบระยะเวลาความคุ้มครองสิทธิประโยชน์ ปีของแผนสิทธิประโยชน์อาจเป็นไปตามปีปฏิทินหรือไม่ก็ได้ 

1. ในพื้นที่ทำงาน **การจัดการสิทธิประโยชน์** ภายใต้ **การตั้งค่า** ให้เลือก **กฎการใช้สิทธิ์และตัวเลือก**

2. ในแท็บ **รอบระยะเวลา** ให้เลือก **สร้าง** เพื่อสร้างรอบระยะเวลา เพื่อดำเนินการตามกระบวนการที่เกี่ยวข้องกับแผนสิทธิประโยชน์ที่มีผลบังคับใช้อยู่กับรอบระยะเวลาสิทธิประโยชน์ ให้เลือก **แผนที่แนบมาด้วย** เมื่อต้องการดูแผนที่สัมพันธ์กับการขายรวม ให้เลือก **แผนที่แนบมาด้วย** 

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **รอบระยะเวลา** | ป้อนรหัสเฉพาะสำหรับรอบระยะเวลา |
   | **วันที่และเวลามีผลใช้ตั้งแต่** | วันและเวลาเริ่มต้นเมื่อมีการบังคับใช้รอบระยะเวลาสิทธิประโยชน์ |
   | **วันที่และเวลามีผลใช้ถึง** | วันและเวลาเริ่มต้นเมื่อไม่มีการบังคับใช้รอบระยะเวลาสิทธิประโยชน์ |
   | **การเริ่มต้นลงทะเบียน** | วันที่เริ่มต้นสำหรับการลงทะเบียนแบบเปิด การลงทะเบียนแบบเปิดคือการที่พนักงานสามารถเลือกสิทธิประโยชน์ในสิทธิประโยชน์ด้วยตนเองได้ |
   | **การสิ้นสุดการลงทะเบียน** | วันที่สิ้นสุดการลงทะเบียนแบบเปิด |
   | **เปิด** | ระบุว่ารอบระยะเวลาที่เปิดเป็นไปตามวันที่ของระบบและถูกต้องตามวันและเวลาที่มีผลบังคับใช้หรือไม่ | 
   | **รอบระยะเวลาก่อนหน้า** | ระบุรอบระยะเวลาของสิทธิประโยชน์ที่เกิดขึ้นก่อนรอบระยะเวลาของสิทธิประโยชน์ที่ถูกเลือก ข้อมูลนี้จะถูกนำมาใช้ในระหว่างการลงทะเบียนสิทธิ์ของสิทธิประโยชน์เพื่อกำหนดแผน ตัวเลือกความคุ้มครอง และ ผู้กำหนดแผนจากปีก่อนหน้านี้ |
 
4. เลือก **บันทึก**

## <a name="use-a-flex-credit-program"></a>ใช้โปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลา

คุณสามารถใช้โปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลาเพื่อลงทะเบียนพนักงานในสิทธิประโยชน์ตามจำนวนเครดิตแบบยืดหยุ่นที่กำหนดไว้ล่วงหน้า พนักงานสามารถเลือกวิธีการปันส่วนเครดิตการทำงานแบบยืดหยุ่นเวลาได้ ตัวอย่างเช่น ถ้าพนักงานครอบคลุมภายใต้แผนประกันภัยสุขภาพของคู่สมรสของตนเอง พนักงานอาจต้องการใช้เครดิตที่พวกเขาจะใช้ในความคุ้มครองด้านสุขภาพกับสวัสดิการอื่น ๆ

1. ในพื้นที่ทำงาน **การจัดการสิทธิประโยชน์** ภายใต้ **การตั้งค่า** ให้เลือก **กฎการใช้สิทธิ์และตัวเลือก**

2. ในแท็บ **รอบระยะเวลา** เลือก **โปรแกรมเครดิตแบบยืดหยุ่น**

3. เลือกโปรแกรมเครดิตแบบยืดหยุ่นที่จะใช้ ในฟิลด์จะมีข้อมูลต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | รหัสเครดิตสวัสดิการ | ตัวระบุเฉพาะของโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลา |
   | คำอธิบาย | คำอธิบายของโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลา | 
   | วันที่เริ่มต้น | วันที่โปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลามีผลบังคับใช้ |
   | วันที่สิ้นสุด | วันสิ้นสุดของโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลา คุณสามารถปล่อยให้ค่าเริ่มต้น (12/31/2154) บ่งชี้ว่าโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลาไม่มีการหมดอายุที่กำหนดเวลาไว้ |
   | มูลค่าเครดิตรวม | จำนวนเครดิตที่พนักงานแต่ละคนจะต้องใช้เพื่อผลประโยชน์ของตน |
   | กฎการแบ่งตามส่วน | กฎที่จะใช้สำหรับแบ่งส่วนเครดิตการทำงานแบบยืดหยุ่นเวลาเมื่อพนักงานถูกจ้างงานในระหว่างรอบระยะเวลาการทำงานแบบยืดหยุ่นเวลา </br></br><ul><li>**ไม่มี** – พนักงานไม่ได้รับเครดิตการทำงานแบบยืดหยุ่นเวลา ถ้ามีการจ้างงานหลังจากรอบระยะเวลาของโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลาเริ่มต้นแล้ว</li><li>**เครดิตแบบเต็ม** – พนักงานจะได้รับเครดิตแบบยืดหยุ่นเวลาเต็มจำนวน โดยไม่คำนึงถึงเวลาที่ได้รับการจ้างงาน</li><li>**การแบ่งตามส่วน** – พนักงานจะได้รับเครดิตแบบยืดหยุ่นตามสัดส่วนตามวันที่เริ่มต้น</li></ul> |
   | สูตรการแบ่งตามส่วนของเครดิตแบบยืดหยุ่น | กฎที่จะใช้สำหรับแบ่งส่วนเครดิตการทำงานแบบยืดหยุ่นเวลาเมื่อพนักงานถูกจ้างงานในระหว่างรอบระยะเวลาสวัสดิการสำหรับโปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลา การแบ่งตามสัดส่วนจะขึ้นอยู่กับวันที่เริ่มต้นการจ้างงาน ฟิลด์นี้สามารถใช้ได้ก็ต่อเมื่อคุณเลือก **การแบ่งตามส่วน** ไว้ในฟิลด์ **กฏการแบ่งตามส่วน** เท่านั้น </br></br><ul><li>**รายวัน** – แบ่งจำนวนของเครดิตการทำงานแบบยืดหยุ่นเวลาตามส่วนที่พนักงานได้รับไปยังระดับวัน จำนวนรวมของเครดิตการทำงานแบบยืดหยุ่นเวลาถูกหารด้วยจำนวนวันในรอบระยะเวลา ตัวอย่างเช่น ถ้ารอบระยะเวลาสวัสดิการของคุณเป็น 400 วัน ระบบจะแบ่งจำนวนรวมของเครดิตแบบยืดหยุ่นโดย 400 เพื่อคำนวณจำนวนของพนักงานที่มีการทำงานแบบยืดหยุ่นเวลาที่ได้รับต่อวัน</li><li>**เดือนปัจจุบัน** – แบ่งจำนวนของเครดิตการทำงานแบบยืดหยุ่นเวลาตามส่วนที่พนักงานจะได้รับในระดับเดือน ซึ่งปัดเศษเป็นเดือนปัจจุบัน จำนวนรวมของเครดิตการทำงานแบบยืดหยุ่นเวลาถูกหารด้วยจำนวนเดือนในรอบระยะเวลา ตัวอย่างเช่น ถ้ารอบระยะเวลาสวัสดิการของคุณเป็น 15 เดือน ระบบจะแบ่งจำนวนรวมของเครดิตแบบยืดหยุ่นโดย 15 เพื่อคำนวณจำนวนของพนักงานที่มีการทำงานแบบยืดหยุ่นเวลาที่ได้รับต่อเดือน</li><li>**เดือนถัดไป** – แบ่งจำนวนของเครดิตการทำงานแบบยืดหยุ่นเวลาตามส่วนที่พนักงานจะได้รับในระดับเดือน ซึ่งปัดเศษเป็นเดือนถัดไป จำนวนรวมของเครดิตการทำงานแบบยืดหยุ่นเวลาถูกหารด้วยจำนวนเดือนในรอบระยะเวลา ตัวอย่างเช่น ถ้ารอบระยะเวลาสวัสดิการของคุณเป็น 15 เดือน ระบบแบ่งจำนวนรวมของเครดิตแบบยืดหยุ่นโดย 15 เพื่อคำนวณจำนวนของพนักงานที่มีการทำงานแบบยืดหยุ่นเวลาที่ได้รับต่อเดือน</li></ul> |
   
   ตรวจสอบให้แน่ใจว่าแผนสิทธิประโยชน์แต่ละรายการมีการลงทะเบียนเพียงครั้งเดียวในโปรแกรมเครดิตแบบยืดหยุ่นสำหรับแต่ละรอบระยะเวลาของสิทธิประโยชน์เท่านั้น มิฉะนั้น ระบบจะไม่ทราบโปรแกรมแบบยืดหยุ่นที่จะใช้ในการให้เครดิตแบบยืดหยุ่นและคุณจะพบปัญหา 

## <a name="configure-programs"></a>ตั้งค่าคอนฟิกโปรแกรม

โปรแกรมเป็นชุดของแผนสิทธิประโยชน์ที่ใช้ร่วมกับชุดของกฎการใช้สิทธิ์ คุณสามารถกำหนดกฎการใช้สิทธิ์สำหรับโปรแกรมทั้งหมดแทนที่จะใช้สำหรับแผนรายบุคคล ตัวอย่างเช่น โปรแกรม Contoso Canada FTE หรือ โปรแกรม Contoso Europe executive-level 

1. ในพื้นที่ทำงาน **การจัดการสิทธิประโยชน์** ภายใต้ **การตั้งค่า** ให้เลือก **กฎการใช้สิทธิ์และตัวเลือก**

2. ในแท็บ **โปรแกรม** ให้เลือก **สร้าง** เพื่อสร้างโปรแกรม เมื่อต้องการสร้างข้อยกเว้นสำหรับพนักงานที่คุณสมบัติไม่ตรงตามข้อกำหนดของกฎการใช้สิทธิ์ ให้เลือก **กฎการแทนที่การใช้สิทธิ์** เมื่อต้องการดูแผนที่สัมพันธ์กับโปรแกรม ให้เลือก **แผนที่แนบมาด้วย**

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **โปรแกรม** | ตัวระบุเฉพาะสำหรับโปรแกรม |
   | **คำอธิบาย** | คำอธิบายเกี่ยวกับโปรแกรม | 
   | **วันที่และเวลามีผลใช้ตั้งแต่** | วันและเวลาของโปรแกรมที่มีผลบังคับใช้ |
   | **วันที่และเวลามีผลใช้ถึง** | วันและเวลาของโปรแกรมเป็นที่สิ้นสุด ค่าเริ่มต้น 12/31/2154 แสดงว่าไม่เคยเกิดขึ้นเลย |
   | **ช่วงเวลารอความคุ้มครอง** | รอบระยะเวลาที่พนักงานต้องรอก่อนที่จะเริ่มต้นความคุ้มครองสำหรับโปรแกรมสิทธิประโยชน์ |
   | **ช่วงเวลารอการหัก** | รอบระยะเวลาที่พนักงานต้องรอก่อนที่จะเริ่มการหักเงินสำหรับโปรแกรมสิทธิประโยชน์ |
   | **กฎการมีคุณสมบัติเหมาะสม** | เลือกกฎการมีคุณสมบัติเหมาะสมที่จะใช้กับโปรแกรมสิทธิประโยชน์ คุณกำหนดกฎการใช้สิทธิ์บนแท็บ **กฎการใช้สิทธิ์** บนหน้านี้ |
