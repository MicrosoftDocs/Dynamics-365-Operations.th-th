# <a name="provision-human-resources-in-the-finance-and-operations-infrastructure"></a>เตรียมใช้งานทรัพยากรบุคคลในโครงสร้างพื้นฐานของการเงินและการดําเนินงาน

_**ใช้กับ:** ทรัพยากรบุคคลบนโครงสร้างพื้นฐานของแอปการเงินและการดําเนินงาน_ 

> [!NOTE]
> เริ่มใช้กรกฎาคม 2022 สภาพแวดล้อมทรัพยากรบุคคลใหม่ จะไม่สามารถเตรียมใช้งานทรัพยากรบุคคลบนโครงสร้างพื้นฐานแบบสแตนด์อโลนได้ และโครงการ Microsoft Dynamics Lifecycle Services (LCS) ใหม่ จะไม่สามารถสร้างบนสภาพแวดล้อมได้ ลูกค้าสามารถปรับใช้สภาพแวดล้อมของทรัพยากรบุคคลบนโครงสร้างพื้นฐานของการเงินและการดําเนินงาน

หัวข้อนี้อธิบายกระบวนการเตรียมใช้งานสภาพแวดล้อมการทำงานจริงใหม่ของ Microsoft Dynamics 365 Human Resources ในโครงสร้างพื้นฐานของการเงินและการดําเนินงาน

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ก่อนที่คุณจะเตรียมใช้งานสภาพแวดล้อมใหม่ ต้องมีข้อกำหนดเบื้องต้นต่อไปนี้:

- คุณได้ซื้อทรัพยากรบุคคลผ่านผู้ให้บริการโซลูชัน Cloud (CSP) หรือข้อตกลงสถาปัตยกรรมองค์กร (EA) ถ้าคุณมีสิทธิ์การใช้งาน Dynamics 365 ซึ่งรวมแผนบริการทรัพยากรบุคคล และคุณไม่สามารถทำตามขั้นตอนในบทความนี้ให้เสร็จสิ้นได้ ให้ติดต่อฝ่ายสนับสนุน
- ผู้ดูแลระบบส่วนกลางควรลงชื่อเข้าใช้ [Microsoft Dynamics Lifecycle Services (LCS)](https://lcs.dynamics.com) และสร้างโครงการการเงินและการดำเนินงานใหม่

## <a name="provision-a-human-resources-trial-environment"></a>เตรียมใช้งานสภาพแวดล้อมการทดลองทรัพยากรบุคคล

> [!NOTE]
> ตั้งแต่เดือนเมษายน 2022 สภาพแวดล้อมการทดลองใช้ทรัพยากรบุคคล จะไม่สามารถใช้งานในแอปพลิเคชันแบบสแตนด์อโลน ผู้มีโอกาสเป็นลูกค้าที่สนใจในการประเมินความสามารถของทรัพยากรบุคคลในแอปการเงินและการดำเนินงาน สามารถทดลองใช้ฟรี 30 วัน พร้อมกับข้อมูลสาธิต Dynamics 365 Finance จะรวมความสามารถของทรัพยากรบุคคล ที่จะนำในโครงสร้างพื้นฐานทางการเงินผ่านการผสานแอปพลิเคชันแบบสแตนด์อโลน ดูข้อมูลเพิ่มเติมที่ [ข้อเสนอการรวมทรัพยากรบุคคล จะรวบรวมความสามารถต่างๆ เข้าด้วยกันกับลูกค้า](https://cloudblogs.microsoft.com/dynamics365/it/2021/09/15/merging-of-hr-offerings-brings-capabilities-together-for-customers) หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับการทดลองใช้ Dynamics 365 Finance โปรดดูที่ [คู่มือ](../fin-ops-core/fin-ops/get-started/before-you-buy.md) แบบทีละขั้นตอน

## <a name="plan-human-resources-environments"></a>วางแผนสภาพแวดล้อมของทรัพยากรบุคคล

ก่อนที่คุณจะสร้างสภาพแวดล้อมทรัพยากรบุคคลแรกของคุณ คุณควรวางแผนความต้องการสภาพแวดล้อมในโครงการของคุณอย่างรอบคอบ การสมัครใช้งานพื้นฐานในทรัพยากรบุคคลประกอบด้วยสภาพแวดล้อมสองประเภท ได้แก่ สภาพแวดล้อมการทำงานจริง และสภาพแวดล้อมการทดสอบการยอมรับระดับมาตรฐาน (Sandbox) คุณมีทางเลือกที่จะซื้อสภาพแวดล้อมเพิ่มเติมเพื่อสนับสนุนกิจกรรมโครงการ ทั้งนี้ขึ้นอยู่กับความซับซ้อนของโครงการ

นี่คือข้อควรพิจารณาบางประการเกี่ยวกับสภาพแวดล้อมทางเลือกเพิ่มเติม:

- **การย้ายข้อมูล** – กิจกรรมการย้ายข้อมูลเพื่อให้สามารถใช้สภาพแวดล้อม Sandbox ของคุณเพื่อวัตถุประสงค์ในการทดสอบทั่วทั้งโครงการได้ เมื่อคุณมีสภาพแวดล้อมเพิ่มเติม กิจกรรมการย้ายข้อมูลสามารถดำเนินต่อไปในระหว่างการทดสอบและกิจกรรมการตั้งค่าคอนฟิกจะเกิดขึ้นพร้อมกันในสภาพแวดล้อมที่แตกต่างกัน
- **การรวม** – การรวมการตั้งค่าคอนฟิกและการทดสอบ ซึ่งอาจรวมถึงการรวมดั้งเดิมหรือการรวมแบบกำหนดเอง เช่น ระบบค่าจ้าง ระบบติดตามผู้สมัคร หรือระบบด้านสวัสดิการและผู้ให้บริการ
- **การฝึกอบรม** – คุณอาจต้องมีสภาพแวดล้อมแยกต่างหาก ซึ่งตั้งค่าคอนฟิกไว้ด้วยชุดข้อมูลการฝึกอบรม เพื่อให้คุณสามารถฝึกอบรมพนักงานในการใช้ระบบใหม่ 
- **โครงการหลายระยะ** – คุณอาจต้องการสภาพแวดล้อมเพิ่มเติมเพื่อสนับสนุนการตั้งค่าคอนฟิก การย้ายข้อมูล การทดสอบ หรือกิจกรรมอื่นๆ ในระยะโครงการที่วางแผนไว้หลังจากการเริ่มใช้งานของโครงการ
- **การพัฒนา** – ในโครงสร้างพื้นฐานของการเงินและการดําเนินงาน ขณะนี้คุณสามารถขยายโซลูชัน และพัฒนาการเลือกตามความต้องการของคุณเอง แต่ละนักพัฒนาต้องใช้สภาพแวดล้อมการพัฒนาของตนเอง สำหรับข้อมูลเพิ่มเติม ดูที่ [ปรับใช้งานและเข้าถึงสภาพแวดล้อมการพัฒนา](/fin-ops-core/dev-itpro/dev-tools/access-instances)
- **GOLD** – สำหรับการปรับใช้งานใหม่ แนวทางทั่วไปคือการใช้สภาพแวดล้อม GOLD ที่แยกต่างหาก ซึ่งเก็บรักษาไว้ใช้ดั้งเดิมในการตั้งค่าคอนฟิกและการย้ายข้อมูล สภาพแวดล้อมนี้สามารถใช้ได้ตลอดการใช้งานเพื่อรีเฟรชสภาพแวดล้อมอื่นๆ จะใช้เพื่อสร้างสภาพแวดล้อมการทำงานจริงใหม่ที่มีการตั้งค่าคอนฟิกพื้นฐานและการย้ายข้อมูล คุณไม่สามารถปรับใช้สภาพแวดล้อมการทำงานจริงบนโครงสร้างพื้นฐานของการเงินและการดําเนินงานได้ จนกว่าคุณจะดําเนินกระบวนการความพร้อมในระหว่างใช้งาน สำหรับข้อมูลเพิ่มเติม โปรดดู [จัดเตรียมสำหรับการใช้งานจริง](/fin-ops-core/fin-ops/imp-lifecycle/prepare-go-live)

> [!IMPORTANT]
> เมื่อคุณพิจารณาสภาพแวดล้อมของคุณ เราขอแนะนำวิธีต่อไปนี้:
>
> - ใช้สภาพแวดล้อมการทดสอบการยอมรับระดับมาตรฐานเริ่มต้นของคุณ (ซึ่งเดิมคือ Sandbox) หรือสภาพแวดล้อมอื่น ในการจำลองการตัดก่อนการใช้งานจริงของคุณ
> - เก็บรายการตรวจสอบการตัดโดยละเอียด ซึ่งรวมถึงแพคเกจข้อมูลแต่ละชุดที่ต้องใช้ในการย้ายข้อมูลขั้นสุดท้ายในสภาพแวดล้อมการใช้งานจริงระหว่างการตัดการใช้งานจริง คำแนะนำนี้สําคัญอย่างมาก ถ้าคุณไม่ได้มีสภาพแวดล้อม GOLD แยกต่างหากเพื่อจัดเก็บการตั้งค่าคอนฟิกของคุณ
> - ใช้สภาพแวดล้อมการทดสอบการยอมรับระดับมาตรฐานเริ่มต้นของคุณ (ซึ่งเดิมคือ Sandbox) หรือสภาพแวดล้อมระดับ 2 หรือสูงกว่า เป็นสภาพแวดล้อม TEST ของคุณทั่วทั้งโครงการ ถ้าคุณต้องการสภาพแวดล้อมเพิ่มเติม องค์กรของคุณสามารถซื้อสภาพแวดล้อมนั้นที่ต้นทุนเพิ่มเติมได้

## <a name="create-an-lcs-project"></a>สร้างโครงการ LCS

หากต้องการใช้ LCS ในการจัดการสภาพแวดล้อมทรัพยากรบุคคลของคุณ คุณต้องสร้างโครงการ LCS ก่อน หากคุณย้ายสภาพแวดล้อมทรัพยากรบุคคลของคุณไปยังโครงสร้างพื้นฐานของการเงินและการดําเนินงาน คุณต้องสร้างโครงการ LCS ใหม่ให้กับแอปการเงินและการดําเนินงาน สำหรับข้อมูลเพิ่มเติม ดูที่ [การย้ายสภาพแวดล้อมทรัพยากรบุคคลของคุณ](hr-admin-migrate-overview) หากคุณมีโครงการ LCS สำหรับแอปการเงินและการดําเนินงานอื่นๆ อยู่แล้ว คุณสามารถเปิดใช้งานคุณลักษณะทรัพยากรบุคคลในพื้นที่ทำงาน **การจัดการคุณลักษณะ** สำหรับข้อมูลเพิ่มเติม ดูที่ [ภาพรวมการจัดการคุณลักษณะ](/fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview)

เมื่อลูกค้าใหม่ลงชื่อสมัครใช้ทรัพยากรบุคคล การสมัครใช้งานจะรวมถึงพื้นที่ทํางานของโครงการการใช้งาน หลังจากที่ลูกค้าเรียกใช้บริการแล้ว ผู้ดูแลระบบผู้เช่าต้องลงชื่อเข้าใช้ใน <https://lcs.dynamics.com> โดยใช้บัญชีผู้เช่า พื้นที่ทำงานของโครงการจะถูกสร้างขึ้นโดยอัตโนมัติสำหรับองค์กร สำหรับข้อมูลเพิ่มเติม โปรดดู [Lifecycle Services (LCS) สำหรับลูกค้าที่ใช้แอปการเงินและการดำเนินงาน](/fin-ops-core/dev-itpro/lifecycle-services/lcs-works-lcs)

> [!NOTE]
> เพื่อให้แน่ใจในการเตรียมใช้งานที่เสร็จเรียบร้อย บัญชีที่คุณใช้เพื่อเตรียมใช้งานสภาพแวดล้อมทรัพยากรบุคคลจะต้องถูกมอบหมายให้กับบทบาท **ผู้ดูแลระบบ** หรือบทบาท **ผู้กำหนดค่าระบบ** ในสภาพแวดล้อม Power Apps ที่สัมพันธ์กับสภาพแวดล้อมทรัพยากรบุคคล สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีกําหนด Security role ให้กับผู้ใช้ใน Microsoft Power Platform ให้ดูที่ [ตั้งค่าคอนฟิกความปลอดภัยผู้ใช้ให้กับทรัพยากร](/power-platform/admin/database-security)

คุณต้องดำเนินการเตรียมความพร้อมโครงการ LCS ให้เสร็จสมบูรณ์ก่อนจึงจะสามารถเริ่มปรับใช้สภาพแวดล้อม สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [การเตรียมความพร้อมโครงการ](/fin-ops-core/dev-itpro/lifecycle-services/project-onboarding) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการใช้ LCS โปรดดู [คู่มือผู้ใช้ Lifecycle Services (LCS)](/fin-ops-core/dev-itpro/lifecycle-services/lcs-user-guide)

## <a name="deploy-human-resources-environments"></a>ปรับใช้สภาพแวดล้อมของทรัพยากรบุคคล

การปรับใช้งานแอปการเงินและการดําเนินงานต่างๆ รวมถึงทรัพยากรบุคคลใน Cloud มีเงื่อนไขว่าคุณควรเข้าใจสภาพแวดล้อมและการสมัครใช้งานที่คุณใช้งานอยู่ ผู้ที่สามารถดําเนินการงานต่างๆ และข้อมูลและการเลือกกำหนดที่คุณต้องจัดการ ขอแนะนาให้คุณใช้บัญชีบริการแทนผู้ใช้ที่ระบุชื่อ เมื่อคุณปรับใช้สภาพแวดล้อมใหม่ ข้อมูลเพิ่มเติมเกี่ยวกับวิธีปรับใช้สภาพแวดล้อมบนโครงสร้างพื้นฐานของการเงินและการดําเนินงาน โปรดดูที่ [ภาพรวมการปรับใช้งาน Cloud](/fin-ops-core/dev-itpro/deployment/cloud-deployment-overview)

การปรับใช้งานสภาพแวดล้อมการทำงานจริงสำหรับทรัพยากรบุคคลบนโครงสร้างพื้นฐานของการเงินและการดําเนินงาน คุณต้องดําเนินการความพร้อมในการเริ่มใช้งานจริงให้เสร็จสิ้นก่อน สำหรับข้อมูลเพิ่มเติม โปรดดู [จัดเตรียมสำหรับการใช้งานจริง](/fin-ops-core/fin-ops/imp-lifecycle/prepare-go-live) กระบวนการนี้รวมถึงตัวประมาณการการสมัครใช้งานใน LCS สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [ตัวประมาณการการสมัครใช้งาน](/fin-ops-core/dev-itpro/lifecycle-services/subscription-estimator)

## <a name="integrate-microsoft-power-platform-with-human-resources"></a>รวม Microsoft Power Platform กับทรัพยากรบุคคล

Microsoft Power Platform จะจัดเตรียมชุดความสามารถต่างๆ ของแอปลิเคชัน Dynamics 365 ผ่านศูนย์การจัดการ Power Platform คุณสามารถรวมและขยายการใช้ข้อมูลทรัพยากรบุคคลโดยใช้ Microsoft Power Platform สำหรับข้อมูลเกี่ยวกับวิธีรวมทรัพยากรบุคคลกับMicrosoft Power Platform โปรดดูที่ [การรวม Microsoft Power Platform กับแอปการเงินและการดำเนินงาน](/fin-ops-core/dev-itpro/power-platform/overview)

## <a name="supported-geographies"></a>ภูมิศาสตร์ที่ได้รับการสนับสนุน

ข้อมูลเกี่ยวกับภาษาและภูมิศาสตร์ที่สนับสนุนทรัพยากรบุคคล โปรดดูที่ [สากลตามการออกแบบ: เรียนรู้เกี่ยวกับประเทศและภาษาที่รองรับ](https://dynamics.microsoft.com/availability-reports/)

## <a name="grant-access-to-the-environment"></a>ได้รับสิทธิ์ในการเข้าถึงสภาพแวดล้อม

ตามค่าเริ่มต้น ผู้ดูแลระบบส่วนกลางที่เว็บไซต์จะมีสิทธิ์ในการเข้าถึง คุณต้องมอบสิทธิ์ในการเข้าถึงให้กับผู้ใช้แอปพลิเคชันเพิ่มเติมอย่างชัดแจ้ง คุณต้องเพิ่มผู้ใช้ และกำหนดบทบาทที่เหมาะสมให้กับพวกเขาในสภาพแวดล้อมทรัพยากรบุคคล ดูข้อมูลเพิ่มเติมได้ที่ [สร้างผู้ใช้ใหม่](/dynamics365/unified-operations/dev-itpro/sysadmin/tasks/create-new-users) และ [กำหนดผู้ใช้ให้กับบทบาทความปลอดภัย](/dynamics365/unified-operations/dev-itpro/sysadmin/tasks/assign-users-security-roles)
