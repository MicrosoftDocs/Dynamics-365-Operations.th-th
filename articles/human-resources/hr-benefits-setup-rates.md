# <a name="configure-rates"></a>ตั้งค่าคอนฟิกอัตรา

อัตราใน Microsoft Dynamics 365 Human Resources กำหนดว่านายจ้างและพนักงานมีส่วนร่วมสำหรับสวัสดิการมากน้อยเพียงใด ค่าอาจเป็นจำนวนหรือเครดิตแบบยืดหยุ่นเวลา ขึ้นอยู่กับการตั้งค่าคอนฟิกของคุณ

ใช้อัตราเพื่อกำหนดว่าพนักงานและนายจ้างที่ชำระค่าจ้างสำหรับแต่ละสวัสดิการเท่าไหร่ ขึ้นอยู่กับปัจจัยหลายอย่าง อัตราครอบคลุมจะมีผลในวันที่บังคับใช้ ดังนั้นคุณจึงสามารถจัดเก็บบันทึกประวัติของอัตราได้ 

## <a name="set-up-rates"></a>ตั้งค่าอัตรา

1. ในพื้นที่ทำงาน **การจัดการสวัสดิการ** ภายใต้ **การตั้งค่า** ให้เลือก **อัตรา**

2. เลือก **ใหม่**

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- |
   | **อัตรา** | ชื่อเฉพาะที่ระบุอัตราสวัสดิการ |
   | **คำอธิบาย** | คำอธิบายย่อเกี่ยวกับอัตราของสวัสดิการ |
   | **มีผลบังคับใช้** | วันที่อัตรามีผลบังคับใช้ วันที่ปัจจุบันของระบบคือค่าเริ่มต้น 
   | **การหมดอายุ** | วันที่สิ้นสุดการของอัตรา 12/31/2154 (ซึ่งแสดงว่าไม่มี) เป็นค่าเริ่มต้น |
   | **ใช้ชั้น** | ระดับที่จะใช้สำหรับการคำนวณอัตราสวัสดิการ ระดับเดี่ยวสำหรับหนึ่งระดับอัตราสวัสดิการหรือระดับคู่สำหรับสองระดับอัตราสวัสดิการ ตัวอย่างของระดับคู่ขึ้นอยู่กับเพศและอายุ |
   | **ความถี่ในการชำระเงิน** | ความถี่ในการชำระเงินที่กำหนดความถี่ในการชำระค่าสวัสดิการพิเศษสำหรับผู้ให้บริการสวัสดิการ ตัวอย่างเช่น ถ้าความถี่ในการชำระเงินเป็นรายเดือน แล้วอัตราสวัสดิการจะแสดงถึงยอดการชำระเงินรายเดือน |
   | **การปัดเศษอัตราความถี่ในการชำระค่าจ้าง** | วิธีการสำหรับการปัดเศษอัตรา: มาตรฐานหรือตัดให้สั้นลง |
   | **จำนวนของพนักงานที่ไม่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของนายจ้างที่ไม่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของพนักงานที่ผู้สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของนายจ้างที่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนเงินสำหรับการดูแล** | ยอดเงินในการดูแลที่เรียกเก็บโดยผู้ดูแลระบบของบุคคลฝ่ายที่สาม นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ดูแลระบบของบุคคลฝ่ายที่สาม และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **อัตราเครดิตแบบยืดหยุ่น** | จำนวนของเครดิตแบบยืดหยุ่นต้นทุนสวัสดิการ การทำเช่นนี้สามารถใช้ได้กับอัตราที่มีสำหรับแผนสวัสดิการที่เกี่ยวข้องกับโปรแกรมเครดิตแบบยืดหยุ่นเท่านั้น ถ้าคุณใช้อัตราระดับ อัตราระดับเครดิตแบบยืดหยุ่นจะถูกกำหนดไว้ในการดำเนินการ > อัตราระดับ |
   | **วันที่มีผลของการเปลี่ยนแปลง** | วันที่ที่อัตราสวัสดิการเปลี่ยนแปลงจะมีผล ระบบจะเปลี่ยนอัตราสวัสดิการและอัพเดตแผนสวัสดิการทั้งหมดโดยอัตโนมัติที่เกี่ยวข้องกับอัตรานี้ ตราบใดที่คุณดำเนินการเปลี่ยนแปลงการประมวลผลการอัพเดต อย่ากำหนดวันที่นี้ ยกเว้นว่าคุณต้องการให้ระบบอัพเดตแผนสวัสดิการของพนักงานโดยอัตโนมัติตามอัตรานี้ ซึ่งโดยปกติแล้วจะถูกสงวนไว้เพื่อการประมวลผลการเปลี่ยนแปลงอัตราในอนาคตโดยอัตโนมัติ วันที่มีผลบังคับใช้ของการเปลี่ยนแปลงต้องอยู่ภายในวันที่อัตราสวัสดิการมีผลบังคับใช้และวันหมดอายุ |
   | **การเปลี่ยนแปลงอัตราเสร็จสมบูรณ์แล้ว** | กล่องกาเครื่องหมาย **การเปลี่ยนแปลงอัตราเสร็จสมบูรณ์** จะถูกเลือกโดยอัตโนมัติเมื่อการเปลี่ยนแปลงอัตราสวัสดิการเสร็จสมบูรณ์แล้วโดยการประมวลผลการอัพเดตอัตรา |

4. การติดตามและรักษาการเปลี่ยนแปลงการตั้งค่าอัตราสิทธิประโยชน์ ให้เลือก **ดำเนินการ** จากนั้นเลือก **รักษาเวอร์ชัน**

5. เลือก **บันทึก** 

## <a name="set-up-tier-rates"></a>ตั้งค่าอัตราระดับ

คุณสามารถใช้อัตราระดับในการตั้งค่าอัตราของคุณได้ถ้าอัตราแตกต่างกันไปขึ้นอยู่กับปัจจัยต่างๆ ตัวอย่างเช่น คุณสามารถตั้งค่าคอนฟิกอัตราระดับที่จะบอกว่า ถ้าอายุของคุณสูงสุดถึง 34.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 2  ถ้าอายุของคุณสูงถึง 39.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 3

นอกจากนี้คุณยังสามารถใช้ระดับคู่ได้ ถ้าคุณเลือก **ระดับคู่** สำหรับค่า **ระดับที่ใช้** ในฟอร์ม **ตั้งค่าอัตรา** คุณสามารถกำหนดอัตราโดยใช้สองมิติ ตัวอย่างเช่น คุณสามารถตั้งค่าคอนฟิกระบบระดับที่คู่จะบอกว่า ถ้าคุณเป็นเพศชายและอายุของคุณสูงสุดถึง 34.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 2  ถ้าคุณคือเพศชาย และอายุของคุณสูงถึง 39.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 3 ถ้าคุณคือเพศหญิง และอายุของคุณสูงถึง 34.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 1.8 ถ้าคุณคือเพศหญิง และอายุของคุณสูงถึง 39.99 แล้วจำนวนที่ไม่สูบบุหรี่คือ 2.8

1. ในพื้นที่ทำงาน **การจัดการสวัสดิการ** ภายใต้ **การตั้งค่า** ให้เลือก **อัตรา**

2. เลือกอัตราหนึ่งรายการขึ้นไปจากรายการ เลือก **การดำเนินการ** แล้วเลือก **ระดับอัตรา**

3. เลือก **ใหม่**

3. ระบุค่าสำหรับฟิลด์ต่อไปนี้

   | ฟิลด์ | คำอธิบาย |
   | --- | --- | 
   | **คำอธิบาย** | ค่าฟิลด์ **คำอธิบาย** ใช้จากคำอธิบายในเรกคอร์ดการตั้งค่าอัตรา การทำเช่นนี้จะช่วยให้คุณระบุว่าการตั้งค่าอัตราที่จะเชื่อมโยงกับอัตราระดับ |
   | **รหัสชั้น** | เลือกรหัสระดับ รหัวระดับถูกกำหนดใฟอร์มรหัสระดับ  ระบบจะแสดงคำอธิบายของรหัสระดับในกริดทางด้านซ้ายโดยอัตโนมัติ |
   | **ชนิดชั้น** | ระบุฟิลด์ที่ควรใช้เป็นเงื่อนไขการเลือกสำหรับกระบวนการคำนวณอัตราระดับ ตัวอย่างเช่น:</br></br><ul><li>ถ้ามีการใช้ **อายุ** ระบบจะใช้วันเกิดของพนักงานในกระบวนการคำนวณสวัสดิการ</li><li>ถ้ามีการใช้ **เงินเดือน** ระบบจะใช้เงินสวัสดิการรายปีของพนักงานในกระบวนการคำนวณสวัสดิการ</li><li>ถ้ามีการใช้ **ชนิดงาน** ระบบจะใช้เรกคอร์ดข้อมูลตำแหน่งงานปัจจุบันของพนักงานเพื่อกำหนดชนิดงานโดยเรกคอร์ดงานที่เชื่อมโยงกับตำแหน่ง</li></ul></br></br>ชนิดของเทียร์ได้แก่ **อายุ**, **เงินเดือน**, **กายภาพ**, **เพศ**, **เทียบเท่าเต็มเวลา**, **ชนิดงาน**, **ภูมิภาคค่าตอบแทน**, และ **ระดับ** | 
   | **ระดับ** | ค่าที่จะใช้กับชนิดของระดับในระหว่างกระบวนการคำนวณอัตราสวัสดิการ ตัวอย่างเช่น:</br></br><ul><li>ถ้าชนิดของระดับเป็น **อายุ** นี่อาจจะเป็นค่าอายุ</li><li>ถ้าชนิดของระดับเป็น **เงินเดือน** นี่อาจจะเป็นจำนวนเงินเดือน</li><li> ถ้าชนิดของระดับเป็น **ประเภทงาน** นี่อาจจะเป็นประเภทงาน</li></ul></br></br>ด้วยชนิดระดับของ **อายุ** หรือ **เงินเดือน** ค่าในฟิลด์ **ระดับ** จะแสดงถึงขอบเขตบนของระดับ เมื่อประเภทของระดับคือ **ประเภทงาน** ระบบจะใช้วิธีการจับคู่ที่ตรงกันในระหว่างการเลือกอัตราระดับ |
   | **ชนิดการคำนวณ** | ระบุว่าจะใช้ยอดเงินในฟิลด์ยอดเงินที่คำนวณอย่างไร และการคำนวณทางคณิตศาสตร์ใดที่จะดำเนินการ ถ้าจำเป็น ถ้าชนิดการคำนวณเป็นยอดเงินคงที่ ระบบจะใช้ฟิลด์ยอดเงินตามที่เป็นอยู่ ถ้าชนิดการคำนวณเป็นต่อจำนวนหรือครอบคลุมของเงินเดือน ระบบจะใช้ยอดการคำนวณและทิศทางการคำนวณในการคำนวณทางคณิตศาสตร์</br></br>ถ้าชนิดการคำนวณเป็นต่อจำนวนของเงินเดือน ระบบจะใช้สมการคณิตศาสตร์ต่อไปนี้:</br></br>เงินเดือนสวัสดิการประจำปีหารด้วยยอดเงินการคำนวณ (การปัดเศษขึ้นหรือลง) คูณด้วยจำนวนคนสูบบุหรี่หรือไม่สูบบุหรี่ของพนักงานหรือนายจ้าง</br></br>ถ้าชนิดการคำนวณเป็นต่อจำนวนของการครอบคลุม ระบบจะใช้สมการคณิตศาสตร์ต่อไปนี้:</br></br>เงินครอบคลุมหารด้วยยอดเงินการคำนวณ (การปัดเศษขึ้นหรือลง) คูณด้วยจำนวนคนสูบบุหรี่หรือไม่สูบบุหรี่ของพนักงานหรือนายจ้าง</br></br>ในการคำนวณทั้งสอง ทิศทางการคำนวณจะใช้เพื่อกำหนดว่าจะปัดเศษยอดเงินเดือนหรือยอดเงินความครอบคลุมของสวัสดิการรายปีหารด้วยยอดเงินการคำนวณขึ้นหรือลง |
   | **จำนวนเงินในการคำนวณ** | จำนวนที่จะใช้ระหว่างกระบวนการคำนวณอัตราสวัสดิการ ยอดเงินนี้จะเป็นตัวหารระหว่างการคำนวณทางคณิตศาสตร์ของอัตราระดับ |
   | **ทิศทางการคำนวณ** | ทิศทางที่จำนวนผลลัพธ์ได้ถูกคำนวณควรถูกปัดเศษ ระบบสนับสนุนทิศทางการคำนวณสามประการดังนี้: ว่างเปล่า (วิธีการที่แน่นอน) **เพิ่ม** และ **ลด**</br></br><ul><li>ถ้าว่างเปล่า ระบบจะใช้การคำนวณที่แน่นอนของจำนวนเงินเดือน/ความครอบคลุมหารด้วยยอดเงินการคำนวณ ถ้าค่านี้มีเศษส่วน ระบบจะใช้ค่านี้ในการคำนวณ</li><li>ถ้า **เพิ่มขึ้น** ระบบจะเพิ่มการคำนวณทางคณิตศาสตร์ของจำนวนเงินเดือน/ความครอบคลุมหารด้วยยอดเงินการคำนวณเป็นจำนวนเต็มถัดไป ซึ่งหมายความว่า 12.25 จะเพิ่มขึ้นเป็น 13</li><li>ถ้า **ลดลง** ระบบจะลดการคำนวณทางคณิตศาสตร์ของจำนวนเงินเดือน/ความครอบคลุมหารด้วยยอดเงินการคำนวณเป็นจำนวนเต็มในขณะนี้ ซึ่งหมายความว่า 12.25 จะลดลงเป็น 12</li></ul> |
   | **จำนวนของพนักงานที่ไม่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของนายจ้างที่ไม่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของพนักงานที่ผู้สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนของนายจ้างที่สูบบุหรี่** | จำนวนเงินค่าธรรมเนียมของผู้ให้บริการสวัสดิการสำหรับพนักงานที่ไม่สูบบุหรี่ นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ให้สวัสดิการ และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **จำนวนเงินสำหรับการดูแล** | ยอดเงินในการดูแลที่เรียกเก็บโดยผู้ดูแลระบบของบุคคลฝ่ายที่สาม นี่คือจำนวนเงินที่นายจ้างชำระให้กับผู้ดูแลระบบของบุคคลฝ่ายที่สาม และควรยึดตามความถี่ในการชำระเงินสำหรับการตั้งค่าอัตรา |
   | **อัตราผู้ไม่สูบบุหรี่สำหรับเครดิตแบบยืดหยุ่น** | จำนวนของต้นทุนสวัสดิการของเครดิตแบบยืดหยุ่น ตามการคำนวณที่กำหนดสำหรับระดับชั้นสำหรับผู้ไม่สูบบุหรี่ ตัวอย่างเช่น ถ้าชนิดการคำนวณเป็น **ต่อยอดเงินของความครอบคลุม** ยอดเงินที่คำนวณได้คือ 10,000 และอัตราเครดิตแบบยืดหยุ่นคือ 6 ซึ่งหมายความว่าพนักงานที่ไม่สูบบุหรี่เลือกความครอบคลุม $10,000 จะมีต้นทุนคือเครดิตแบบยืดหยุ่น 6 เครดิต ถ้ามีการเลือก $20,000 ของความครอบคลุม จะมีต้นทุนเครดตแบบยืดหยุ่น 12 เครดิต |
