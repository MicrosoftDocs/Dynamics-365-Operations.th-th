# <a name="choose-a-data-integration-technology"></a>เลือกเทคโนโลยีการรวมข้อมูล

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

หัวข้อนี้ให้ข้อมูลสำหรับการรวมกับข้อมูลที่จัดการโดย Dynamics 365 Human Resources ซึ่งอธิบายถึงเทคโนโลยีการรวมที่แตกต่างกัน ที่ช่วยให้คุณตัดสินใจเลือกเทคโนโลยีที่เหมาะกับความต้องการของคุณที่สุด

## <a name="data-integration-background"></a>พื้นหลังการรวมข้อมูล

ข้อมูลธุรกิจเป็นสินทรัพย์สำคัญที่ทำให้บริษัทของคุณมีลักษณะเฉพาะ ข้อมูลธุรกิจของคุณมีมูลค่าสูง คุณสามารถใช้ความสัมพันธ์ระหว่างข้อมูลที่รวบรวมได้ทั่วทั้งธุรกิจของคุณเพื่อปรับปรุงกระบวนการทางธุรกิจและข่าวกรองธุรกิจขององค์กรของคุณ เรามุ่งมั่นที่จะช่วยให้คุณสามารถเข้าถึงข้อมูลธุรกิจของคุณได้อย่างง่ายดาย ปลอดภัย และมั่นคง ไม่ว่าระบบจะมาจากไหนก็ตาม

ในอดีตการรวมข้อมูลเข้ากับหลายระบบเป็นได้ยาก Microsoft มีขั้นตอนในการทำให้การรวมข้อมูลง่ายขึ้นและเป็นขั้นตอนที่ชัดเจนที่เป้าหมายถูกรับรู้ได้ผ่าน [Dataverse](/powerapps/maker/common-data-service/data-platform-intro)

ทรัพยากรบุคคลจะทำให้ Dataverse เป็นอินเทอร์เฟสสาธารณะที่ต้องการสำหรับข้อมูลทรัพยากรบุคคล เมื่อเวลาผ่านไปเราคาดว่าข้อมูลที่สำคัญที่สุดซึ่งจัดการโดยทรัพยากรบุคคลจะมีการเปิดเผยใน Dataverse เราขอแนะนำ Dataverse ให้เป็นเทคโนโลยีของตัวเลือกสำหรับแอพลิเคชันที่มีการรวมมากที่สุด

เราตระหนักดีว่า Dataverse อาจยังไม่มีข้อมูลทั้งหมดที่แอปพลิเคชันของคุณต้องการ นอกจากนี้เรายังตระหนักว่าระยะเวลาโครงการของคุณอาจจำเป็นต้องใช้เทคโนโลยีอื่น ตรวจสอบให้แน่ใจว่าได้แจ้งให้เราทราบเมื่อ Dataverse ไม่ตรงกับความต้องการของการรวมของคุณ

## <a name="integration-technologies"></a>เทคโนโลยีการรวม

ส่วนต่อไปนี้จะอธิบายถึงเทคโนโลยีการรวมข้อมูลที่แตกต่างกันที่มีอยู่สำหรับใช้กับทรัพยากรบุคคล

### <a name="dataverse-tables"></a>ตาราง Dataverse

Dataverse เป็นอินเทอร์เฟสข้อมูลสาธารณะที่ต้องการสำหรับทรัพยากรบุคคล ซึ่งขยายออกมาจากแพลตฟอร์ม Dynamics 365 XRM ซึ่งใช้โดยโซลูชัน [Dynamics 365 Customer Engagement](/dynamics365/?panel=customer-engagement#pivot=business-apps)

Dataverse จัดเตรียมแพลตฟอร์มและ API สำหรับตารางข้อมูล เมื่อคุณปรับใช้ทรัพยากรบุคคล ระบบจะเชื่อมต่อกับอินสแตนซ์ Dataverse เอนทิตี้สำหรับข้อมูลทรัพยากรบุคคลปรับใช้เข้ากับอินสแตนซ์ Dataverse นั้น ตารางและข้อมูลของตารางจะพร้อมใช้งานกับแอปพลิเคชันที่สามารถเชื่อมต่อกับอินสแตนซ์ Dataverse ได้ ทรัพยากรบุคคลซิงโครไนส์ข้อมูลไปถึงและกลับจากตาราง Dataverse

> [!NOTE]
> เอนทิตี Human Resources จะสอดคล้องกับตาราง Dataverse หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับการอัปเดต Dataverse (ซึ่งก่อนหน้านี้เรียกว่า Common Data Service) และการอัปเดตศัพท์ โปรดดูที่ [Microsoft Dataverse คืออะไร](/powerapps/maker/data-platform/data-platform-intro)

เมื่อตารางข้อมูลที่จำเป็นสำหรับแอปพลิเคชันที่รวมอยู่ใน Dataverse คุณสามารถใช้ [Dataverse และ APIs ที่สนับสนุน](/powerapps/?panel=developer#pivot=home) ได้อย่างสมบูรณ์ ในบรรดา APIs ที่สนับสนุนคือ[Dynamics 365 Web API](/dynamics365/customer-engagement/developer/use-microsoft-dynamics-365-web-api) ซึ่งมีการดำเนินการ OData สำหรับการเข้าถึงข้อมูล Dataverse

ตาราง Dataverse และ APIs ที่เกี่ยวข้อง เป็นตัวเลือกที่ดีที่สุดสำหรับการเข้าถึงข้อมูลทรัพยากรบุคคลจากเว็บแอปพลิเคชัน บริการเว็บ/APIs และจากแอปพลิเคชันอื่นที่เชื่อมต่อกับตัวดึงข้อมูล OData

> [!NOTE]
> ด้วยการตัดสินใจที่จะทำให้ Dataverse เป็นอินเทอร์เฟสข้อมูลที่ต้องการสำหรับทรัพยากรบุคคลเป็นข้อมูลที่ค่อนข้างล่าสุด คุณอาจพบว่าเอนทิตี้ข้อมูลทรัพยากรบุคคลที่คุณต้องการสำหรับการรวมของคุณยังไม่พร้อมใช้งานใน Dataverse
> </br>
> สำหรับรายการเอนทิตี้ทรัพยากรบุคคลที่พร้อมใช้งานใน Dataverse โปรดดู [Human Resources และ Dataverse](/dynamics365/unified-operations/talent/corehrentities)
> </br>
> ถ้าเอนทิตี้ทรัพยากรบุคคลที่จำเป็นสำหรับการรวมของคุณยังไม่พร้อมใช้งาน คุณจะต้องรอให้เอนทิตี้ข้อมูลพร้อมใช้งานหรือคุณจะต้องใช้หนึ่งในเทคโนโลยีการรวมอื่นๆ ที่อธิบายไว้ด้านล่างนี้
> </br>
> โดยค่าเริ่มต้น การรวม Dataverse จะถูกปิดใช้งานในสภาพแวดล้อมใหม่ที่ไม่มีข้อมูลสาธิตที่ระบุไว้ให้ การรวม Common Data Service จะมีการเปิดใช้งานในสภาพแวดล้อมใหม่ซึ่งรวมถึงข้อมูลสาธิต และการซิงค์ข้อมูลจะเริ่มต้นขึ้นเมื่อมีการจัดเตรียมสภาพแวดล้อม หลังจากที่สภาพแวดล้อมของคุณพร้อมที่จะซิงค์ข้อมูลแล้ว คุณสามารถเปิดใช้งานการรวมได้

### <a name="dmfdixf-entities"></a>เอนทิตี้ DMF/DIXF

ทรัพยากรบุคคลซึ่งส่วนใหญ่สร้างขึ้นบนแพลตฟอร์มเดียวกันเป็นแอปพลิเคชัน Finance and Operations มี [กรอบงานการจัดการข้อมูล (DMF)](/dynamics365/unified-operations/dev-itpro/data-entities/data-entities-data-packages?toc=%2ffin-and-ops%2ftoc.json) DMF เรียกอีกอย่างว่ากรอบงานการนำเข้าส่งออกข้อมูล (DIXF) ทรัพยากรบุคคลจะให้ชุดของเอนทิตี้ข้อมูลที่คุณสามารถใช้ในการนำเข้าและส่งออกข้อมูลทรัพยากรบุคคลได้ ในขณะที่ตาราง Dataverse เป็นอินเทอร์เฟสการรวมข้อมูลที่ต้องการสำหรับทรัพยากรบุคคล เอนทิตี้ DMF จะยังคงมีประโยชน์ในบางกรณี เช่น:

- ตาราง Dataverse ยังไม่พร้อมใช้งาน

- การรวมจำเป็นต้องมีความสามารถในการนำเข้า/ส่งออกข้อมูลจำนวนมากที่มีประสิทธิภาพสูง

> [!NOTE]
> เอนทิตี Human Resources จะสอดคล้องกับตาราง Dataverse หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับการอัปเดต Dataverse (ซึ่งก่อนหน้านี้เรียกว่า Common Data Service) และการอัปเดตศัพท์ โปรดดูที่ [Microsoft Dataverse คืออะไร](/powerapps/maker/data-platform/data-platform-intro)

เอนทิตี DMF ขณะนี้ให้ความครอบคลุมข้อมูลที่สมบูรณ์มากที่สุดสำหรับข้อมูลทรัพยากรบุคคล

DMF ไม่เหมาะสมสำหรับการรวมแบบ real-time เช่น เมื่อคุณต้องการความคิดเห็นของผู้ใช้ทันทีในอินเทอร์เฟสผู้ใช้ การดำเนินงานของแพคเกจมีการจัดตารางงานชุดงานและมักจะมีเวลาแฝงอย่างน้อย1-2 นาที ก่อนที่บริการชุดงานจะรับงานสำหรับการดำเนินการ รวมถึงเวลาที่จำเป็นในการดำเนินการนำเข้า/ส่งออกให้เสร็จสมบูรณ์

DMF อาจเป็นตัวเลือกที่ดีที่สุดเมื่อจำเป็นต้องมีปริมาณที่สามารถประมวลผลได้สูง (เช่นการนำเข้า/ส่งออกของเรกคอร์ดมากมายในเวลากลางคืน)

> [!NOTE]
> DMF ไม่สามารถใช้ได้สำหรับ Attract และ Onboard

### <a name="dmf-package-rest-api"></a>แพคเกจ DMF REST API

DMF ให้ [REST API](/dynamics365/unified-operations/dev-itpro/data-entities/data-management-api) สำหรับการจัดการแพคเกจข้อมูล API นี้สามารถใช้เพื่อโต้ตอบกับ DMF โดยทางโปรแกรมเพื่อให้การดำเนินการเช่น:

- การนำเข้าแพคเกจข้อมูล

- การส่งออกแพคเกจข้อมูล

- การตรวจสอบสถานะของการดำเนินงานนำเข้า/ส่งออก

แพคเกจ DMF REST API ถูกสนับสนุนอย่างเต็มที่ในทรัพยากรบุคคล

### <a name="azure-sql-db-byod"></a>Azure SQL DB (BYOD)

DMF ยังมีคุณลักษณะที่มีประสิทธิภาพ (เรียกว่า [Bring Your Own Database](/dynamics365/unified-operations/dev-itpro/analytics/export-entities-to-your-own-database) หรือ BYOD) ซึ่งช่วยให้ฝ่ายทรัพยากรบุคคลสามารถส่งออกข้อมูลไปยังฐานข้อมูล Microsoft Azure SQL ของคุณเอง ความสามารถนี้มีความยืดหยุ่นอย่างมาก เมื่อข้อมูลมีอยู่ในฐานข้อมูล SQL ของคุณเอง คุณสามารถใช้แอปพลิเคชันหรือมิดเดิลแวร์ใดๆ ที่สามารถเชื่อมต่อกับที่เก็บข้อมูล SQL

BYOD เป็นโซลูชันแบบอ่านอย่างเดียว ขณะที่คุณสามารถจัดการและจัดเก็บข้อมูลที่คุณต้องการในฐานข้อมูล Azure SQL (เช่น สำหรับข้อมูล mashups) ข้อมูลที่จัดเก็บไว้ในฐานข้อมูล Azure SQL จะไม่มีการซิงโครไนส์ไปยังทรัพยากรบุคคล

BYOD มีความเหมาะสมสำหรับการรายงานโซลูชัน การรวมข้อมูล ข้อมูล mashups ว่าเป็นแหล่งข้อมูลสำหรับขั้นตอน [Azure Data Factory](/azure/data-factory/)

> [!NOTE]
> BYOD ไม่สามารถใช้ได้กับ Attract และ Onboard

### <a name="odata-enabled-entities"></a>เอนทิตี้ OData ที่เปิดใช้งาน

เอนทิตี้ DMF ส่วนใหญ่ยังเปิดใช้งานสำหรับการเข้าถึงผ่านทางบริการข้อมูลของฝ่ายทรัพยากรบุคคล (OData) เอกสารที่ระบุสำหรับ [บริการ Finance and Operations OData](/dynamics365/unified-operations/dev-itpro/data-entities/odata) จะใช้กับทรัพยากรบุคคล ยกเว้นสำหรับการสร้างเอนทิตี้ที่มีการเปิดเผย OData ของคุณเอง

ในขณะที่ Dataverse และการดำเนินงาน OData ที่ระบุโดย Dataverse (โดยผ่าน [Dynamics 365 Web API](/previous-versions/dynamicscrm-2016/developers-guide/mt593051(v=crm.8))) เป็นที่ต้องการผ่านทางบริการข้อมูลของฝ่ายทรัพยากรบุคคล บริการข้อมูลของฝ่ายทรัพยากรบุคคลในปัจจุบันมีความครอบคลุมเอนทิตี้ที่สมบูรณ์สำหรับข้อมูลทรัพยากรบุคคล

### <a name="excel-add-in"></a>Add-in ของ Excel

[Add-in ของ Excel](/dynamics365/unified-operations/dev-itpro/office-integration/use-excel-add-in?toc=%2fdynamics365%2funified-operations%2ftalent%2ftoc.json) ทำให้มีการใช้เอนทิตี้ที่เปิดใช้งาน OData ภายใต้พื้นที่ โดยมีวิธีที่สะดวกสำหรับผู้ใช้ปลายทางในการดึงข้อมูลและปรับเปลี่ยนข้อมูลทรัพยากรบุคคลผ่าน Excel UI ที่คุ้นเคย

Add-in ของ Excel มีความเหมาะสมสำหรับการนำเข้า/ส่งออกข้อมูลเฉพาะกิจโดยผู้เชี่ยวชาญของโดเมนธุรกิจ สำหรับการรวมข้อมูลที่เกิดซ้ำซึ่งจำเป็นต้องมีใช้โปรแกรมอัตโนมัติ เทคโนโลยีการรวมอีกเทคโนโลยีหนึ่งจะมีความเหมาะสมมากกว่า

### <a name="data-integrator"></a>ตัวรวมข้อมูล

คุณสามารถใช้ [บริการตัวรวมข้อมูล](/powerapps/administrator/data-integrator) เพื่อรวมข้อมูลไปยังและกลับจาก Dataverse ตัวรวมข้อมูลช่วยให้คุณกำหนดโครงการการรวม บ่อยครั้งตามเท็มเพลตที่กำหนดไว้ล่วงหน้า ซึ่งนักพัฒนาแอปพลิเคชันได้ปรับแต่งสำหรับการรวมเฉพาะ คุณสามารถจัดกำหนดการให้รันโดยอัตโนมัติบนกำหนดการที่เกิดซ้ำหรือรันด้วยตนเอง

โครงการตัวรวมข้อมูลเหมาะสำหรับการรวมชุดงาน Dataverse ตัวเลือกนี้เป็นตัวเลือกที่ดีสำหรับการรวมกันระหว่างครอบครัว Dynamics 365 ของแอปพลิเคชัน ตัวอย่างเช่น Microsoft มีเท็มเพลตตัวรวมข้อมูลสำหรับการรวมข้อมูลจากทรัพยากรบุคคลไปยัง Dynamics 365 Finance คุณสามารถเรียนรู้เพิ่มเติมเกี่ยวกับเท็มเพลตใน [การรวมจาก Dynamics 365 Human Resources ไปยัง Dynamics 365 Finance](hr-admin-integration-finance.md)

### <a name="power-query"></a>Power Query

ตัวรวมข้อมูลสนับสนุน [Power Query](/power-query/power-query-what-is-power-query) ผ่าน [ลักษณะการทำงานของการสอบถามขั้นสูง](/powerapps/administrator/data-integrator#advanced-data-transformation-and-filtering) Power Query การกรองข้อมูลและการแปลงข้อมูลที่มีประสิทธิภาพและความยืดหยุ่น รวมถึงภาษาสูตร M ที่สมบูรณ์ Power Query มีแนวโน้มที่จะคุ้นเคยถ้าคุณได้พัฒนารายงาน Power BI

## <a name="deciding-on-an-integration-technology"></a>การตัดสินใจเกี่ยวกับเทคโนโลยีการรวม

ด้วยเทคโนโลยีการรวมที่แตกต่างกันมากมายที่มีอยู่ การตัดสินใจว่าจะใช้วิธีการรวมกันในบางครั้งอาจเป็นเรื่องยากได้ เนื่องจากความครอบคลุมของข้อมูลในการเติบโตของ Dataverse การตัดสินใจจะกลายเป็นเรื่องง่ายขึ้นเมื่อ Dataverse เป็นอินเทอร์เฟสข้อมูลที่ต้องการในกรณีส่วนใหญ่ แต่จนถึงตอนนั้น คุณอาจพบว่า Dataverse ยังไม่ตรงกับความต้องการของคุณ ตารางต่อไปนี้จะสรุปลักษณะสำคัญบางประการของตัวเลือกเทคโนโลยีการรวม
