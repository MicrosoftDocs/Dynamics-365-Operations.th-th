# <a name="choose-a-data-integration-technology"></a>เลือกเทคโนโลยีการรวมข้อมูล

Dynamics 365 Human Resources จัดการข้อมูลธุรกิจที่มีประโยชน์ในกระบวนการทางธุรกิจที่หลากหลาย บทความนี้จะให้คำแนะนำเกี่ยวกับตัวเลือกต่างๆสำหรับการรวมกับข้อมูลที่ได้รับการจัดการโดยทรัพยากรบุคคล การอธิบายลักษณะของเทคโนโลยีการรวมที่แตกต่างกันเพื่อให้ผู้รวบรวมสามารถตัดสินใจได้อย่างรอบคอบว่าจะให้เทคโนโลยีใดที่เหมาะสมที่สุดกับ ความต้องการของตน

## <a name="data-integration-vision"></a>การมองเห็นการรวมข้อมูล

Microsoft จะเห็นข้อมูลธุรกิจเป็นสินทรัพย์สำคัญที่ทำให้บริษัทของคุณมีลักษณะเฉพาะ ข้อมูลธุรกิจของคุณมีมูลค่าสูง ข้อมูลที่รวบรวมและรักษาโดยส่วนหนึ่งของธุรกิจเกี่ยวข้องกับข้อมูลที่รวบรวมโดยส่วนหนึ่งของธุรกิจและความสัมพันธ์เหล่านี้สามารถใช้เพื่อปรับปรุงกระบวนการทางธุรกิจและข่าวกรองธุรกิจขององค์กรของคุณ การเข้าถึงข้อมูลธุรกิจของคุณได้อย่างง่ายดาย ปลอดภัย และมั่นคงโดยไม่คำนึงว่าระบบใด "เป็นเจ้าของ" ข้อมูลคือหลักสำคัญของการมองเห็นที่เรามีสำหรับการรวมข้อมูลกับทรัพยากรบุคคล

ในอดีตการรวมข้อมูลเข้ากับหลายระบบเป็นได้ยาก
Microsoft มีขั้นตอนในการทำให้การรวมข้อมูลง่ายขึ้นและเป็นขั้นตอนที่ชัดเจนที่เป้าหมายถูกรับรู้ได้ผ่าน [Common Data Service](https://docs.microsoft.com/powerapps/maker/common-data-service/data-platform-intro)

การไปข้างหน้าทรัพยากรบุคคลจะทำให้ Common Data Service เป็นอินเทอร์เฟสสาธารณะที่ต้องการสำหรับข้อมูลทรัพยากรบุคคล เมื่อเวลาผ่านไปเราคาดว่าข้อมูลที่สำคัญที่สุดซึ่งจัดการโดยทรัพยากรบุคคลจะมีการเปิดเผยใน Common Data Service เราขอแนะนำ Common Data Service ให้เป็นเทคโนโลยีของตัวเลือกสำหรับแอพลิเคชันที่มีการรวมมากที่สุด ในขณะที่เราตระหนักว่าข้อมูลบางอย่างที่ใบสมัครของคุณอาจจำเป็นต้องมีอยู่ใน Common Data Service ขณะนี้และระยะเวลาของโครงการของคุณอาจจำเป็นต้องใช้เทคโนโลยีอื่น โปรดแจ้งให้เราทราบเมื่อ Common Data Service ไม่ตรงกับความต้องการในการรวมของคุณ

## <a name="integration-technologies"></a>เทคโนโลยีการรวม

ส่วนต่อไปนี้จะอธิบายถึงเทคโนโลยีการรวมข้อมูลที่แตกต่างกันที่มีอยู่สำหรับใช้กับทรัพยากรบุคคล

### <a name="common-data-service-entities"></a>เอนทิตี้ Common Data Service

Common Data Service เป็นอินเทอร์เฟสข้อมูลสาธารณะที่ต้องการสำหรับทรัพยากรบุคคล Common Data Service ถูกสร้างบนแพลตฟอร์มสำหรับผู้ใหญ่โดยมีการเติบโตจากแพลตฟอร์ม Dynamics 365 "XRM" ซึ่งเป็น [Dynamics 365 Customer Engagement](https://docs.microsoft.com/dynamics365/#pivot=business-apps&panel=customer-engagement) การแก้ไขปัญหาที่สร้างขึ้นมา

Common Data Service จัดเตรียมแพลตฟอร์มสำหรับเอนทิตี้ข้อมูลและ API สำหรับการเข้าถึงเอนทิตี้เหล่านั้น เมื่อทรัพยากรบุคคลถูกปรับใช้ในองค์กรของคุณ ทรัพยากรบุคคลจะเชื่อมต่อ Common Data Service กับอินสแตนซ์และเอนทิตี้ที่มีรักษาข้อมูลทรัพยากรบุคคลจะถูกปรับใช้ในอินสแตนซ์ Common Data Serviceดังกล่าวจะทำให้เอนทิตี้และข้อมูลของตนมีอยู่ในใบสมัครใดก็ตามที่สามารถเชื่อมต่อกับอินสแตนซ์ Common Data Service ได้ โดยทั้งนี้ขึ้นอยู่กับแอพลิเคชันของฝ่ายทรัพยากรบุคคลที่คุณกำลังใช้ ทรัพยากรบุคคลจะทำการดำเนินการกับข้อมูลโดยตรงต่อเอนทิตี้ Common Data Service (ซึ่งเป็นกรณีของ Attract และ Onboard) หรือซิงโครไนส์ข้อมูลไปยัง/จากเอนทิตี้ Common Data Service

หลังจากที่เอนทิตี้ข้อมูลที่มีการเปิดเผยข้อมูลของการรวมแอพลิเคชันที่ต้องการใน Common Data Service คุณสามารถใช้ [Common Data Service และ APIs ที่สนับสนุนได้ทั้งหมด](https://docs.microsoft.com/powerapps/#pivot=home&panel=developer)  ในบรรดา APIs ที่สนับสนุนคือ[Dynamics 365 Web API](https://docs.microsoft.com/dynamics365/customer-engagement/developer/use-microsoft-dynamics-365-web-api) ซึ่งมีการดำเนินการ OData สำหรับการเข้าถึงข้อมูล Common Data Service

เอนทิตี้ Common Data Service และ APIs ที่เชื่อมโยงเป็นตัวเลือกที่ดีที่สุดสำหรับการเข้าถึงข้อมูลทรัพยากรบุคคลจากเว็บแอพลิเคชัน เว็บเซอร์วิส/APIs และจากแอพลิเคชันอื่นที่เชื่อมต่อกับตัวดึงข้อมูล OData

> [!NOTE]
> ด้วยการตัดสินใจที่จะทำให้ Common Data Service ซึ่งเป็นอินเทอร์เฟสข้อมูลที่ต้องการสำหรับทรัพยากรบุคคลเป็นขอมูลที่ค่อนข้างล่าสุด คุณอาจพบว่าเอนทิตี้ข้อมูลทรัพยากรบุคคลที่คุณต้องการสำหรับการรวมของคุณยังไม่พร้อมใช้งานใน Common Data Service<sup>1</sup> ถ้าไม่มีเอนทิตี้ทรัพยากรบุคคลที่จำเป็นสำหรับการรวมของคุณยังไม่พร้อมใช้งาน คุณจะต้องรอให้เอนทิตี้ข้อมูลพร้อมใช้งานหรือคุณจะต้องใช้หนึ่งในเทคโนโลยีการรวมอื่นๆที่อธิบายไว้ด้านล่างนี้
> 
> โดยค่าเริ่มต้น การรวม Common Data Service จะถูกปิดใช้งานในสภาพแวดล้อมใหม่ที่ไม่มีข้อมูลสาธิตที่ระบุไว้ให้ การรวม Common Data Service จะมีการเปิดใช้งานในสภาพแวดล้อมใหม่ซึ่งรวมถึงข้อมูลสาธิต และการซิงค์ข้อมูลจะเริ่มต้นขึ้นเมื่อมีการจัดเตรียมสภาพแวดล้อม หลังจากที่สภาพแวดล้อมของคุณพร้อมที่จะซิงค์ข้อมูลแล้ว คุณสามารถเปิดใช้งานการรวมได้

<sup>1</sup> สำหรับรายการเอนทิตี้ทรัพยากรบุคคลที่ใน Common Data Service โปรดดู [Human Resources and Common Data Service](https://docs.microsoft.com/dynamics365/unified-operations/talent/corehrentities) สำหรับ Attract และ Onboard เอนทิตี้ทั้งหมดจะพร้อมใช้งานใน Common Data Service

### <a name="dmfdixf-entities"></a>เอนทิตี้ DMF/DIXF

ทรัพยากรบุคคลซึ่งสร้างขึ้นบนแพลตฟอร์มเดียวกันเป็นหลักเช่นเดียวกับแอพลิเคชัน Finance and Operations ให้ [Data Management Framework(DMF)](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/data-entities/data-entities-data-packages?toc=/fin-and-ops/toc.json) บางครั้งเรียกอีกอย่างหนึ่งว่า Data Import Export Framework or DIXF และชุดของเอนทิตี้ข้อมูลที่สามารถใช้สำหรับการนำเข้า/ส่งออกข้อมูลไปยัง/จากทรัพยากรบุคคล ในขณะที่เอนทิตี้ Common Data Service เป็นอินเทอร์เฟสการรวมข้อมูลที่ต้องการสำหรับทรัพยากรบุคคลเอนทิตี้ DMF จะยังคงมีประโยชน์ในบางกรณีเช่น:

- เอนทิตี้ Common Data Service ยังไม่พร้อมใช้งาน

- การรวมจำเป็นต้องมีความสามารถในการนำเข้า/ส่งออกข้อมูลจำนวนมากที่มีประสิทธิภาพสูง

เอนทิตี้ DMF ขณะนี้ให้ความครอบคลุมข้อมูลที่สมบูรณ์มากที่สุดสำหรับข้อมูลทรัพยากรบุคคล

DMF ไม่เหมาะสมสำหรับการรวมแบบ real-time (เช่นเมื่อจำเป็นต้องใช้ผลป้อนกลับของผู้ใช้ในอินเทอร์เฟสผู้ใช้ในทันที) เนื่องจากการดำเนินงานของแพคเกจมีการจัดตารางงานชุดงานและมักจะมีเวลาแฝงที่ 1-2 นาทีก่อนที่การบริการชุดงานจะรับงานสำหรับการดำเนินการรวมถึงเวลาที่จำเป็นต้องดำเนินการนำเข้า/ส่งออกให้เสร็จสมบูรณ์

DMF อาจเป็นตัวเลือกที่ดีที่สุดเมื่อจำเป็นต้องมีปริมาณที่สามารถประมวลผลได้สูง (เช่นการนำเข้า/ส่งออกของเรกคอร์ดมากมายในเวลากลางคืน)

> [!NOTE]
> DMF ไม่สามารถใช้ได้สำหรับ Attract และ Onboard

### <a name="dmf-package-rest-api"></a>แพคเกจ DMF REST API

DMF ให้ [REST API](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/data-entities/data-management-api) สำหรับการจัดการแพคเกจข้อมูล API นี้สามารถใช้เพื่อโต้ตอบกับ DMF โดยทางโปรแกรมเพื่อให้การดำเนินการเช่น:

- การนำเข้าแพคเกจข้อมูล

- การส่งออกแพคเกจข้อมูล

- การตรวจสอบสถานะของการดำเนินงานนำเข้า/ส่งออก

แพคเกจ DMF REST API ถูกสนับสนุนอย่างเต็มที่ในทรัพยากรบุคคล: Core HR

### <a name="azure-sql-db-byod"></a>Azure SQL DB (BYOD)

DMF นอกจากนี้ยังมีลักษณะการทำงานที่มีประสิทธิภาพ (โดยทั่วไปเรียกว่า [Bring Your Own Database](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/analytics/export-entities-to-your-own-database) หรือ BYOD) ซึ่งช่วยให้ Human Resources สามารถส่งออกข้อมูลไปยังฐานข้อมูล Microsoft Azure SQL ของคุณเอง การทำเช่นนี้จะให้ความยืดหยุ่นอย่างมากเนื่องจากเมื่อข้อมูลมีอยู่ในฐานข้อมูล SQL ของคุณเอง คุณสามารถใช้แอพลิเคชันหรือมิดเดิลแวร์ใดๆที่สามารถเชื่อมต่อกับ SQL datastore

โดยทั่วไป BYOD ควรได้รับการพิจารณาเป็นโซลูชันแบบอ่านอย่างเดียว ขณะที่คุณสามารถจัดการและจัดเก็บข้อมูลที่คุณต้องการในฐานข้อมูล Azure SQL (เช่นสำหรับข้อมูล mashups) ข้อมูลที่จัดเก็บไว้ในฐานข้อมูล Azure SQL จะไม่มีการซิงโครไนส์กลับไปยังทรัพยากรบุคคล

BYOD มีความเหมาะสมสำหรับการรายงานโซลูชัน การรวมข้อมูล ข้อมูล mashups ว่าเป็นแหล่งข้อมูลสำหรับขั้นตอน [Azure Data Factory](https://docs.microsoft.com/azure/data-factory/)

> [!NOTE]
> BYOD ไม่สามารถใช้ได้กับ Attract และ Onboard

### <a name="odata-enabled-entities"></a>เอนทิตี้ OData ที่เปิดใช้งาน

เอนทิตี้ DMF ส่วนใหญ่ยังเปิดใช้งานสำหรับการเข้าถึงผ่านทางบริการข้อมูลของฝ่ายทรัพยากรบุคคล (OData) เอกสารที่ระบุสำหรับ [Finance and Operations บริการ OData](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/data-entities/odata) โดยทั่วไปจะใช้กับทรัพยากรบุคคลถึงแม้ว่าการจัดทำเอกสารจะมีการสร้างเอนทิตี้ที่มีการเปิดเผย OData ของคุณเองจะไม่มีผลบังคับใช้

ในขณะที่ Common Data Service และการดำเนินงาน OData ที่ระบุโดย Common Data Service (โดยผ่าน [Dynamics 365 Web API](https://docs.microsoft.com/previous-versions/dynamicscrm-2016/developers-guide/mt593051(v=crm.8))) เป็นที่ต้องการผ่านทางบริการข้อมูลของฝ่ายทรัพยากรบุคคล บริการข้อมูลของฝ่ายทรัพยากรบุคคลในปัจจุบันมีความครอบคลุมเอนทิตี้ที่สมบูรณ์สำหรับข้อมูลทรัพยากรบุคคล

### <a name="excel-add-in"></a>Add-in ของ Excel

[Add-in ของ Excel](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/office-integration/use-excel-add-in?toc=/dynamics365/unified-operations/talent/toc.json) ทำให้มีการใช้เอนทิตี้ที่เปิดใช้งาน OData ภายใต้พื้นที่ โดยมีวิธีที่สะดวกสำหรับผู้ใช้ปลายทางในการดึงข้อมูลและปรับเปลี่ยนข้อมูลทรัพยากรบุคคลผ่าน Excel UI ที่คุ้นเคย

Add-in ของ Excel มีความเหมาะสมสำหรับการนำเข้า/ส่งออกข้อมูลเฉพาะกิจโดยผู้เชี่ยวชาญของโดเมนธุรกิจ สำหรับการรวมข้อมูลที่เกิดซ้ำซึ่งจำเป็นต้องมีใช้โปรแกรมอัตโนมัติ เทคโนโลยีการรวมอีกเทคโนโลยีหนึ่งจะมีความเหมาะสมมากกว่า

### <a name="data-integrator"></a>ตัวรวมข้อมูล

ประสบการณ์ผู้ดูแลระบบ Common Data Service ให้ [บริการตัวรวมข้อมูล](https://docs.microsoft.com/powerapps/administrator/data-integrator) ที่สามารถใช้สำหรับการรวมข้อมูลไปยัง/จาก Common Data Service ตัวรวมข้อมูลสามารถใช้เพื่อกำหนดโครงการการรวม (บ่อยครั้งตามเท็มเพลตที่กำหนดไว้ล่วงหน้าซึ่งนักพัฒนาแอพลิเคชันได้ปรับแต่งสำหรับการรวมเฉพาะ) โครงการการรวมสามารถจัดกำหนดการให้รันโดยอัตโนมัติบนกำหนดการที่เกิดซ้ำหรือรันด้วยตนเอง

โครงการตัวรวมข้อมูลมีความเหมาะสมสำหรับการรวมชุดงาน Common Data Service และสร้างตัวเลือกที่ดีสำหรับการรวมระหว่างตระกูลผลิตภัณฑ์ของ Dynamics 365 ตัวอย่างเช่น Microsoft ให้เท็มเพลตตัวแปลข้อมูลนอกกรอบที่สามารถใช้สำหรับการรวมข้อมูลจากทรัพยากรบุคคลไปยัง Dynamics 365 Finance ได้ สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [การรวมจาก Dynamics 365 Human Resources ไปยัง Dynamics 365 Finance](hr-admin-integration-finance.md)

### <a name="power-query"></a>Power Query

ตัวรวมข้อมูลยังสนับสนุน [Power Query](https://docs.microsoft.com/power-query/power-query-what-is-power-query) (โดยผ่าน [ลักษณะการทำงานของการสอบถามขั้นสูง](https://docs.microsoft.com/powerapps/administrator/data-integrator#advanced-data-transformation-and-filtering))
การสอบถามพลังงานมีการกรองข้อมูลและการแปลงข้อมูลที่ที่มีประสิทธิภาพและยืดหยุ่นซึ่งรวมถึงภาษาสูตร M ที่สมบูรณ์และน่าจะคุ้นเคยกับผู้ที่มีประสบการณ์ในการพัฒนารายงาน Power BI

## <a name="deciding-on-an-integration-technology"></a>การตัดสินใจเกี่ยวกับเทคโนโลยีการรวม

## <a name="summary"></a>สรุป

ข้อมูลธุรกิจของคุณเป็นสินทรัพย์ที่มีค่าแต่มูลค่าของบริษัทอาจลดลงอย่างมากถ้าเป็นเรื่องยากที่จะใช้ข้อมูลสำหรับวัตถุประสงค์เฉพาะ (เช่นการรายงาน mashups ข้อมูลหรือแอปพลิเคชันที่กำหนดเอง) Dynamics 365 Human Resources ให้เทคโนโลยีต่างๆสำหรับการทำงานกับข้อมูลของคุณภายนอกอินเทอร์เฟสผู้ใช้ของแอพลิเคชันทรัพยากรบุคคล (UI) ซึ่งอนุญาตให้มีเข้าถึงการรวมแอพลิเคชันกับข้อมูล หัวข้อนี้อธิบายเทคโนโลยีการรวมที่มีอยู่และลักษณะสำคัญบางประการ ข้อมูลนี้จะช่วยให้คุณตัดสินใจได้ดียิ่งขึ้นเกี่ยวกับวิธีการใช้ประโยชน์สำหรับโครงการการรวมของคุณ
