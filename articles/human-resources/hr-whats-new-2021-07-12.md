# <a name="whats-new-or-changed-in-dynamics-365-human-resources-july-12-2021"></a>มีอะไรใหม่หรือมีการเปลี่ยนแปลงอะไรใน Dynamics 365 Human Resources 12 กรกฎาคม 2021

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

หัวข้อนี้อธิบายถึงคุณลักษณะที่ใหม่ ที่มีการเปลี่ยนแปลง หรือกำลังจะมาถึงในไม่ช้าใน Dynamics 365 Human Resources

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับกระบวนการอัปเดตและกำหนดการของพวกเรา ให้ดูที่ [กระบวนการอัปเดต](hr-admin-setup-update-process.md)

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับลักษณะการทำงานใหม่ และวันที่ที่พร้อมใช้งานทั่วไปที่คาดไว้ ให้ดูที่ [ภาพรวมของ Dynamics 365 Human Resources รุ่น 2021 เวฟ 1](/dynamics365-release-plan/2021wave1/human-resources/dynamics365-human-resources/)

## <a name="in-this-release"></a>ในการเผยแพร่นี้

รุ่นนี้ประกอบด้วยลักษณะการทำงานและการแก้ไขปัญหาบักใหม่ๆ ดังต่อไปนี้ การเปลี่ยนแปลงที่ใช้เพื่อสร้างหมายเลข 8.1.4353

### <a name="new-features"></a>ลักษณะการทำงานใหม่ๆ

โดยทั่วไปลักษณะการทำงานต่อไปนี้จะพร้อมใช้งานกับรุ่นนี้

| ลักษณะการทำงาน | แผนการรีลีส | คู่มือ |
| --- | --- | --- |
|  ปีของการสลับการแสดงการบริการ | |คุณลักษณะนี้ให้ตัวเลือกในการใช้วันที่ต่างๆ เพื่อคํานวณปีของบริการที่แสดงในหน้า **รายการพนักงานที่มีประสิทธิภาพ** และในหน้า **บุคคล**  ซึ่งจะพร้อมใช้งานใน [พารามิเตอร์ Human Resources](/dynamics365/human-resources/hr-setup-parameters) |


### <a name="bug-fixes"></a>การแก้ไขปัญหา

การแก้ไขปัญหาต่อไปนี้จะรวมอยู่ในการนำออกใช้นี้

> [!NOTE]
> เป้าหมายของเราคือการส่งข้อมูลนี้ให้คุณโดยเร็วที่สุดเท่าที่จะเป็นไปได้ พวกเราอาจมีการอัปเดตในหัวข้อนี้เพื่อรวมการแก้ไขข้อผิดพลาดที่ทำให้เกิดข้อผิดพลาดในการสร้างหลังจากที่มีการเผยแพร่หัวข้อนี้ในครั้งแรก

| หมายเลขปัญหา | ปัญหา |  คำอธิบาย |
| --- | --- | --- |
| 595871 | บานหน้าต่างเกี่ยวกับใน Human Resources มีศัพท์ Dataverse ที่ไม่ถูกต้อง | การสร้างแบรนด์ Common Data Service ใหม่ไปยัง Dataverse คำศัพท์มีการอัปเดตในบานหน้าต่างข้อมูล Microsoft Dynamics 365 Human Resources (**วิธีใช้และการสนับสนุน > เกี่ยวกับ**) |
| 598676 | รายการของพนักงานที่มีประสิทธิภาพแทนที่งานสามารถสร้างข้อผิดพลาดเมื่อใช้กับมุมมองที่บันทึกไว้| ในหน้า **ผู้ปฏิบัติงาน** ถ้าเปิดคุณลักษณะ 'รายการพนักงานที่มีประสิทธิภาพ' โปรแกรมประยุกต์อาจล้มเหลวหาก **เปิดเสมอเพื่อการแก้ไข** ถูกตั้งค่าบนมุมมองที่บันทึกไว้ |
| 592344 |ส่วนพนักงานในตําแหน่งควรเป็นแบบอ่านอย่างเดียวเมื่อเปิดใช้งานการจัดการสวัสดิการ | ข้อมูลผู้ปฏิบัติงานถูกสร้างโดยใช้ฟังก์ชันสวัสดิการดั้งเดิม  เมื่อการจัดการสวัสดิการเปิดใช้งาน ฟิลด์จะเป็นแบบอ่านอย่างเดียว เมื่อเปิดการจัดการสวัสดิการและ **ซ่อนฟอร์มสวัสดิการดั้งเดิม** ถูกตั้งค่าเป็น **ใช่** ในพารามิเตอร์ที่ใช้ร่วมกันของ HR แท็บ **ค่าตอบแทนของผู้ปฏิบัติงาน** จะถูกซ่อนไว้ |
| 598617 | แท็บการเรียกใช้ฟอร์ม **HcmDiscussion** ทำให้เกิดการวนรอบที่ไม่สิ้นสุดเมื่อมีการใช้การตั้งค่าส่วนบุคคล | เมื่อเปิดทั้งมุมมองกริดและรายละเอียดเป็น **เปิดเสมอเพื่อการแก้ไข** รหัสที่เรียกใช้แท็บในวิธีการของงานที่แทนที่จะขัดแย้งกับการตั้งค่าส่วนบุคคลว่าการควบคุมควรจะโฟกัสอะไรและสร้างการวนรอบที่ไม่สิ้นสุด |
| 593553 | ฟิลด์วันที่ในสมุดรายวันประสิทธิภาพแสดงใน UTC | ฟิลด์ **วันที่ในสมุดรายวัน** สำหรับสมุดรายวันประสิทธิภาพแสดงอยู่ในโซนเวลา UTC แทนที่จะแสดงโซนเวลาที่กําหนดไว้ในการตั้งค่าวันที่ของระบบ ทําให้วันที่ไม่ถูกต้องในบางโซนเวลา |
| 586930 | การเปิดกิจกรรมของเป้าหมายผลการปฏิบัติงานเปิดเรกคอร์ดอื่นอย่างสมบูรณ์ | เมื่อคุณลักษณะ 'มุมมองรายงานเพิ่มเติมของสมุดรายวันประสิทธิภาพ' เปิดใช้งานในการจัดการคุณลักษณะ การเลือกเป้าหมายประสิทธิภาพของรายงานแบบขยายบนแท็บ **ทีมงานของฉัน** ใน **ระบบพนักงานบริการตนเอง** จะเปิดเป้าหมายของพนักงานแทนที่จะเป็นพนักงานที่เลือก |
| 569959 |  เอนทิตี HcmPositionWorkerAssignmentV2 ไม่ได้กําหนดผู้ปฏิบัติงานให้กับตําแหน่ง | ผู้ใช้ได้รับข้อผิดพลาดเมื่อเพิ่มเรกคอร์ดการมอบหมายตําแหน่งผ่านเอนทิตีและการเผยแพร่ล้มเหลว |
| 582259 | รายงาน VETS 4212 ใช้ฟอร์ม 2017 แทนฟอร์ม 2020 | อัปเดตเป็นรูปแบบ 2020  หากต้องการโหลดรูปแบบใหม่ ให้ไปที่ **การตั้งค่าคอนฟิกรายงาน** และลบรายงาน VETS-4212 ในคอลัมน์ด้านซ้าย ไปที่ **การรายงานทางอิเล็กทรอนิกส์ - ที่เก็บ- ทรัพยากร HR** และเลือก **เปิด**  เลือก **เอกสารที่พิมพ์ VETS-4212 PDF** แล้วเลือก **นําเข้า**|
