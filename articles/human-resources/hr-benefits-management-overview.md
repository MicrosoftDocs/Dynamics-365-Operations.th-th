# <a name="benefits-management-overview"></a>ภาพรวมการจัดการสวัสดิการ

[!include [banner](includes/preview-feature.md)]

ถ้าต้องการการแข่งขันยังคงอยู่ คุณต้องเสนอชุดของสวัสดิการที่สมบูรณ์ เพื่อดึงดูดและรักษาพนักงานที่ดีที่สุดของคุณ นอกเหนือจากสวัสดิการมาตรฐาน เช่น ความครอบคลุมทางการแพทย์และทันตกรรมแล้ว คุณอาจต้องให้บริการแบบขยาย เช่น ความช่วยเหลือในการได้รับการยอมรับ โปรแกรมนันทนาการ และเงินเบี้ยเลี้ยงเสื้อผ้า คุณลักษณะการแสดงตัวอย่างของการจัดการสวัสดิการใน Microsoft Dynamics 365 Human Resources ให้คุณมีโซลูชันที่ยืดหยุ่น ซึ่งสนับสนุนตัวเลือกสวัสดิการที่หลากหลาย ทรัพยากรบุคคลรวมถึงประสบการณ์ของพนักงานที่ใช้งานง่าย ซึ่งแสดงการนำเสนอของคุณ

- แผนสวัสดิการขั้นสูงช่วยให้คุณสามารถสร้าง และจัดการแผนสวัสดิการที่ไม่ซ้ำกัน และสนับสนุนตารางอัตราสวัสดิการที่ซับซ้อนและระดับที่ซ้อนกัน คุณสามารถสร้างโปรแกรมสวัสดิ กลุ่มงาน และกฎการลงทะเบียนอัตโนมัติสำหรับประสบการณ์ของพนักงานที่ง่ายขึ้นได้อย่างง่ายดาย

- โปรแกรมเครดิตการทำงานแบบยืดหยุ่นเวลาช่วยให้คุณสามารถแบ่งเพื่อสนับสนุนการเกษียณอายุ และเหตุการณ์ของชีวิตอื่น ๆ ได้ตามส่วน

- กฎการมีคุณสมบัติที่ครอบคลุมทำให้คุณแน่ใจได้ว่าคุณได้ให้สวัสดิการที่เหมาะสมกับพนักงานที่เหมาะสม

- การลงทะเบียนสวัสดิการออนไลน์ช่วยให้พนักงานของคุณได้รับประสบการณ์การใช้งานอย่างง่าย

- การประมวลผลเหตุการณ์ของชีวิจที่มีคุณสมบัติพร้อมรวมกับการบริการตนเองของพนักงาน และยังสนับสนุนเหตุการณ์ของชีวิตในอนาคต

ถ้าคุณต้องการเข้าถึงข้อมูลสาธิต คุณจะต้องจัดวางสภาพแวดล้อม Sandbox ของคุณใหม่

คุณสามารถให้ความคิดเห็นโดยตรง หรือรายงานปัญหาไปที่: D365BenefitsPreview@microsoft.com

## <a name="benefits-management-known-issues"></a>ปัญหาที่พบเกี่ยวกับการจัดการสวัสดิการ

### <a name="eligibility-processing"></a>การประมวลผลการมีสิทธิ์

เมื่อรันสิทธิ์สำหรับสวัสดิการที่ใช้เงินเดือน 1-5X เปอร์เซ็นต์ของเงินเดือน และยอดเงินคงเหลือที่ครอบคลุม คุณต้องตั้งค่าวันที่ของ **รายละเอียดสวัสดิการ** ให้กับ **วันที่เริ่มต้นการเป็นพนักงาน** ใน **ประวัติการจ้างงาน** นอกจากนี้คุณต้องรวม **ชั่วโมงทำงาน** **ความถี่ในการชำระเงิน** และ **ยอดเงินของเงินเดือนสวัสดิการรายปี** ถ้าผู้ปฏิบัติงานมีค่าตอบแทนคงที่ ป้อน **ชั่วโมงที่ทำงาน** และ **ความถี่ในการชำระเงิน** ยอดเงินเดือนรายปีจะคำนวณ ถ้าพนักงานรับเงินเดือน คุณไม่จำเป็นต้องป้อน **ชั่วโมงที่ทำงาน** เราขอแนะนำว่าเมื่อสร้างผู้ปฏิบัติงานใหม่ ให้ป้อนค่าตอบแทนคงที่ก่อน เมื่อต้องการอัปเดตเรกคอร์ดรายละเอียดสวัสดิการ ให้นำทางไปยัง **ผู้ปฏิบัติงาน > ประวัติปฏิบัติงาน > รายละเอียดการจ้างงาน** ปรับปรุงวันที่เริ่มต้นของผู้ปฏิบัติงาน

### <a name="employee-self-service"></a>ระบบบริการตนเองของพนักงาน

ยอดเงินของพนักงานไม่ได้รับการคำนวณเมื่ออัปเดตยอดเงินความครอบคลุมสำหรับการประกันชีวิต ตัวอย่างเช่น เมื่อพนักงานได้รับการเสนอแผนประกันชีวิต พวกเขาสามารถเลือกได้ว่าจะเลือกความครอบคลุมไม่เกิน $50,000 ที่ต้นทุน $.36 ต่อ $1,000 ของความครอบคลุม  เมื่อพนักงานอัปเดตยอดเงินความครอบคลุม ต้นทุนที่เกี่ยวข้องของพนักงานจะยังคงอยู่ที่ศูนย์

สำหรับแผนสวัสดิการที่อนุญาตให้มีการเลือกชนิดของแผนนั้นเพียงครั้งเดียว ผู้ใช้จะได้รับข้อผิดพลาดถ้ามีการยกเว้นแผนหลังจากเลือกแผน ตัวอย่างเช่น ผู้ใช้จะเลือกแผนการรักษาพยาบาลและใส่ในรถเข็นของตน ผู้ใช้จึงเลือก**ยกเว้น** สำหรับแผนการแพทย์อีกแผนหนึ่ง ผู้ใช้จะได้รับข้อผิดพลาด

## <a name="enable-benefits-management"></a>เปิดใช้งานการจัดการสวัสดิการ

การจัดการสวัสดิการเป็นคุณลักษณะการทำงานของการแสดงตัวอย่าง และมีอยู่ในสภาพแวดล้อม **Sandbox** เท่านั้น บทความเหล่านี้อธิบายวิธีการเปิดใช้งานคุณลักษณะการทำงานการแสดงตัวอย่างในทรัพยากรบุคคล นอกจากนี้ยังระบุว่าคุณลักษณะการทำงานใดที่มีอยู่ในทรัพยากรบุคคลมีการจัดการสวัสดิการ หรือปิดใช้งานเมื่อคุณเปิดใช้งานการจัดการสวัสดิการ
