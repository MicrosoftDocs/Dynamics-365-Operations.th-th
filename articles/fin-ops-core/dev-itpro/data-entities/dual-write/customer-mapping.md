# <a name="integrated-customer-master"></a>ข้อมูลหลักของลูกค้าแบบรวม

[!include [banner](../../includes/banner.md)]



ข้อมูลลูกค้าสามารถทำเป็นต้นฉบับได้ในแอปพลิเคชัน Dynamics 365 มากกว่าหนึ่งรายการ ตัวอย่างเช่น แถวลูกค้าสามารถเริ่มต้นได้ผ่านทางกิจกรรมการขายใน Dynamics 365 Sales (แอปการมีส่วนร่วมกับลูกค้า) หรือแถวสามารถเริ่มต้นได้ผ่านทางกิจกรรมการขายปลีกใน Dynamics 365 Commerce (แอปการเงินและการดำเนินงาน) ไม่ว่าข้อมูลลูกค้าที่มีต้นกำเนิดมาจากที่ใดก็ตาม จะมีการรวมอยู่เบื้องหลัง ข้อมูลหลักของลูกค้าแบบรวมช่วยให้คุณมีความยืดหยุ่นในการเข้าถึงข้อมูลลูกค้าในแอปพลิเคชัน Dynamics 365 และให้มุมมองที่ครอบคลุมของลูกค้าทั่วทั้งชุดแอปพลิเคชัน Dynamics 365

## <a name="customer-data-flow"></a>โฟลว์ข้อมูลของลูกค้า

*ลูกค้า* เป็นแนวคิดที่กำหนดไว้อย่างดีในแอปพลิเคชัน ดังนั้น การรวมของข้อมูลลูกค้าก็เกี่ยวข้องกับความสอดคล้องแนวคิดของลูกค้าระหว่างสองแอปพลิเคชัน ภาพประกอบต่อไปนี้แสดงโฟลว์ข้อมูลของลูกค้า

![โฟลว์ข้อมูลของลูกค้า](media/dual-write-customer-data-flow.png)

ลูกค้าสามารถแบ่งออกย่างกว้างๆ ได้เป็นสองประเภท: ลูกค้าเชิงพาณิชย์/เชิงองค์กร และผู้บริโภค/ผู้ใช้ ลูกค้าสองประเภทนี้จะถูกจัดเก็บและได้รับจัดการโดยแตกต่างกันในการเงินและการดำเนินงานและ Dataverse

ในการเงินและการดำเนินงาน ทั้งลูกค้าเชิงพาณิชย์/เชิงองค์กร และผู้บริโภค/ผู้ใช้ จะมีความเชี่ยวชาญในตารางเดียวที่ชื่อว่า **CustTable** (CustCustomerV3Entity) และจะถูกจำแนกประเภทตามแอตทริบิวต์ **ชนิด** (ถ้า **ชนิด** ถูกตั้งค่าเป็น **องค์กร** ลูกค้าเป็นลูกค้าเชิงพาณิชย์/เชิงองค์กร และถ้ามีการตั้งค่า **ชนิด** เป็น **บุคคล** ลูกค้าเป็นผู้บริโภค/ผู้ใช้) ข้อมูลผู้ติดต่อหลักจะได้รับการจัดการผ่านตาราง SMMContactPersonEntity

ใน Dataverse ลูกค้าเชิงพาณิชย์/เชิงองค์กรมีความเชี่ยวชาญในตารางบัญชี และถูกระบุเป็นลูกค้า เมื่อมีการตั้งค่าแอตทริบิวต์ **RelationshipType** เป็น **ลูกค้า** ทั้งผู้บริโภค/ผู้ใช้และผู้ติดต่อจะถูกแสดงโดยตารางผู้ติดต่อ เพื่อให้การแยกที่ชัดเจนระหว่างผู้บริโภค/ผู้ใช้ และผู้ติดต่อ ตาราง **ผู้ติดต่อ** มีแฟล็กบูลีนที่มีชื่อว่า **Sellable** เมื่อ **Sellable** เป็น **จริง** ผู้ติดต่อเป็นผู้บริโภค/ผู้ใช้ และสามารถสร้างใบเสนอราคาและใบสั่งสำหรับผู้ติดต่อนั้นได้ เมื่อ **Sellable** เป็น **เท็จ** ผู้ติดต่อเป็นเพียงผู้ติดต่อหลักของลูกค้า

เมื่อผู้ติดต่อที่ไม่ใช่ Sellable มีส่วนร่วมในกระบวนการใบเสนอราคาหรือใบสั่ง **Sellable** ถูกตั้งค่าเป็น **จริง** เพื่อแฟล็กผู้ติดต่อเป็นผู้ติดต่อ Sellable ผู้ติดต่อที่กลายเป็นผู้ติดต่อที่สามารถขายได้ยังคงเป็นผู้ติดต่อ Sellable

## <a name="templates"></a>เทมเพลต

ข้อมูลลูกค้ารวมถึงข้อมูลทั้งหมดเกี่ยวกับลูกค้า เช่น กลุ่มลูกค้า ที่อยู่ ข้อมูลผู้ติดต่อ โพรไฟล์การชำระเงิน และสถานะบัตรสมาชิก ชุดของแผนผังตารางทำงานร่วมกันในระหว่างการโต้ตอบข้อมูลลูกค้า ดังที่แสดงในตารางต่อไปนี้
