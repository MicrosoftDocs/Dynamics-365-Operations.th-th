# <a name="general-troubleshooting"></a>การแก้ไขปัญหาเบื้องต้นทั่วไป

[!include [banner](../../includes/banner.md)]



หัวข้อนี้แสดงข้อมูลการแก้ไขปัญหาทั่วไปสำหรับการรวมข้อมูลด้วยการรวมแบบสองทิศทางระหว่างแอปการเงินและการดำเนินงานกับ Dataverse

> [!IMPORTANT]
> ปัญหาบางอย่างที่ที่อยู่ของหัวข้อนี้อาจจำเป็นต้องใช้บทบาทผู้ดูแลระบบ หรือข้อมูลประจำตัวผู้ดูแลระบบของผู้เช่า Microsoft Azure Active Directory (Azure AD) ส่วนสำหรับปัญหาแต่ละอย่างอธิบายว่าจำเป็นต้องมีบทบาทเฉพาะหรือข้อมูลประจำตัวหรือไม่

## <a name="enable-and-view-the-plug-in-trace-log-in-dataverse-to-view-error-details"></a><a id="enable-view-trace"></a>เปิดใช้งานและดูล็อกการติดตามปลั๊กอินใน Dataverse เพื่อดูรายละเอียดข้อผิดพลาด

บันทึกการติดตามมีประโยชน์เมื่อแก้ไขปัญหาการซิงค์ทันทีแบบการรวมแบบสองทิศทางระหว่างการเงินและการดําเนินงานกับ Dataverse บันทึกสามารถระบุรายละเอียดเฉพาะของทีมที่ให้การสนับสนุนทางเทคนิคและวิศวกรรมของ Dynamics 365 บทความนี้ครอบคลุมวิธีการเปิดใช้งานบันทึกการติดตามและวิธีการดูบันทึกการติดตาม บันทึกการติดตามมีการจัดการในหน้าการตั้งค่า Dynamics 365 และต้องใช้สิทธิ์ระดับผู้ดูแลระบบเพื่อเปลี่ยนและดู 

**บทบาทที่จำเป็นในการเปิดล็อกการติดตามและดูข้อผิดพลาด:** ผู้ดูแลระบบ

### <a name="turn-on-the-trace-log"></a>เปิดบันทึกการติดตาม
หากต้องการเปิดล็อกการติดตาม ให้ทำตามขั้นตอนเหล่านี้

1.  เข้าสู่ระบบ Dynamics 365 และเลือก **การตั้งค่า** บนแถบนำทางบนสุด บนหน้าระบบ คลิก **การดูแลระบบ**
2.  บนหน้าการดูแลระบบ คลิก **การตั้งค่าระบบ**
3.  เลือกแท็บ **การปรับแต่ง** และปั๊กอิน จากนั้นในส่วนการติดตามกิจกรรมเวิร์กโฟลว์แบบกำหนดเอง ให้เปลี่ยนดรอปดาวน์เป็น **ทั้งหมด** การดำเนินการนี้จะติดตามกิจกรรมทั้งหมดและจัดเตรียมชุดข้อมูลที่ครอบคลุมสำหรับทีมที่ต้องตรวจสอบปัญหาที่อาจเกิดขึ้น

> [!NOTE]
> การตั้งค่าดรอปดาวน์เป็น **ข้อยกเว้น** จะให้ข้อมูลการติดตามเมื่อเกิดข้อยกเว้น (ข้อผิดพลาด) เท่านั้น

เมื่อเปิดใช้งานแล้ว บันทึกการติดตามปลั๊กอินจะยังคงถูกรวบรวมต่อไปจนกว่าคุณจะปิดด้วยตนเองโดยกลับไปที่ตำแหน่งนี้และเลือก **ปิด**

### <a name="view-the-trace-log"></a>ดูบันทึกการติดตาม
หากต้องการดูการติดตาม ให้ทำตามขั้นตอนเหล่านี้

1. บนหน้าการตั้งค่า Dynamics 365 ให้เลือก **การตั้งค่า** บนแถบนำทางบนสุด 
2. เลือก **บันทึกการติดตามปลั๊กอิน** ในส่วน **การปรับแต่ง** ของหน้า
3. คุณสามารถค้นหารายการของบันทึกการติดตามอิงตามชื่อชนิดและ/หรือชื่อข้อความ
4. เปิดรายการที่ต้องการเพื่อดูบันทึกทั้งหมด บล็อกข้อความในส่วนการปฏิบัติการจะให้ข้อมูลที่สามารถใช้งานสำหรับปลั๊กอิน ถ้ามี ให้ระบุรายละเอียดข้อยกเว้นด้วย 

คุณสามารถคัดลอกเนื้อหาของบันทึกการติดตามและวางลงในแอปพลิเคชันอื่น เช่น Notepad หรือเครื่องมืออื่นๆ เพื่อดูบันทึกหรือไฟล์ข้อความเพื่อให้ดูเนื้อหาทั้งหมดได้ง่ายขึ้น 

## <a name="enable-debug-mode-to-troubleshoot-live-synchronization-issues-in-finance-and-operations-apps"></a>เปิดใช้งานโหมดดีบักเมื่อต้องการแก้ไขปัญหาการซิงโครไนส์ที่เริ่มใช้งานจริงในแอปการเงินและการดำเนินงาน

**บทบาทที่จำเป็นในการดูข้อผิดพลาด:** ผู้ดูแลระบบ

ข้อผิดพลาดในการรวมแบบสองทิศทางที่เกิดขึ้นใน Dataverse สามารถปรากฏขึ้นได้ในแอปการเงินและการดำเนินงาน หากต้องการเปิดใช้งานการบันทึกสำหรับข้อผิดพลาด ให้ปฏิบัติตามขั้นตอนเหล่านี้:

1. สำหรับการตั้งค่าคอนฟิกโครงการทั้งหมดในแอปการเงินและการดำเนินงาน มีแฟล็ก **IsDebugMode** ในตาราง **DualWriteProjectConfiguration**
2. เปิด **DualWriteProjectConfiguration** โดยใช้ add-in ของ Excel เมื่อต้องการใช้ Add in ให้เปิดใช้งานโหมดการออกแบบใน Add in ของ Excel ของการเงินและการดำเนินงาน และเพิ่ม **DualWriteProjectConfiguration** ลงในแผ่นงาน สำหรับข้อมูลเพิ่มเติม ให้ดู [ดูและอัปเดตข้อมูลเอนทิตีด้วย Excel](../../office-integration/use-excel-add-in.md)
3. ตั้งค่า **IsDebugMode** เป็น **ใช่** ในโครงการ
4. เรียกใช้สถานการณ์จำลองที่กำลังสร้างข้อผิดพลาด
5. ล็อก verbose จัดเก็บในตาราง **DualWriteErrorLog**
6. เมื่อต้องการค้นหาข้อมูลบนเบราเซอร์ตาราง ให้ใช้ลิงค์ต่อไปนี้: `https://999aos.cloudax.dynamics.com/?mi=SysTableBrowser&tableName=DualWriteErrorLog` ให้แทนที่ `999` ตามต้องการ
7. อัปเดตอีกครั้งหลังจาก [4595434 KB ](https://fix.lcs.dynamics.com/Issue/Details?kb=4595434&bugId=527820&dbType=3&qc=98e5dc124ac125c57ad633d885ac612aea3ddb8f4abf9d71ab3aa354f2e06cbe) ซึ่งสามารถอัปเดตแพลตฟอร์ม 37 และใหม่กว่า หากคุณได้ติดตั้งการแก้ไขนี้ โหมดดีบักจะเก็บล็อกเพิ่มเติม  

## <a name="check-synchronization-errors-on-the-virtual-machine-for-the-finance-and-operations-app"></a>ตรวจสอบข้อผิดพลาดของการซิงโครไนส์บนเครื่องเสมือนสำหรับแอปการเงินและการดำเนินงาน

**บทบาทที่จำเป็นในการดูข้อผิดพลาด:** ผู้ดูแลระบบ

1. ลงชื่อเข้าใช้ Microsoft Dynamics Lifecycle Services (LCS)
2. เปิดโครงการ LCS ที่คุณเลือกเพื่อทำการทดสอบการรวมแบบสองทิศทาง
3. เลือกไทล์ **สภาพแวดล้อมที่โฮสต์ระบบคลาวด์**
4. ใช้ Remote Desktop เพื่อลงชื่อเข้าใช้ในเครื่องเสมือน (VM) สำหรับแอปการเงินและการดำเนินงาน ใช้บัญชีเฉพาะที่ที่แสดงอยู่ใน LCS
5. เปิดตัวแสดงเหตุการณ์
6. เลือก **ล็อกของแอปพลิเคชันและบริการ \> Microsoft \> Dynamics \> AX-DualWriteSync \> การดำเนินงาน**
7. ตรวจทานรายการของข้อผิดพลาดล่าสุด

### <a name="mitigation"></a>การลดความเสี่ยง
คุณต้องอนุญาตคุกกี้ของบุคคลที่สามในการตั้งค่าเบราว์เซอร์

### <a name="google-chrome-browser"></a>เบราว์เซอร์ Google Chrome
ตัวเลือกที่ 1:
1.  ไปที่การตั้งค่าโดยป้อน chrome://settings/ ในแถบที่อยู่ จากนั้นไปที่ ความเป็นส่วนตัวและความปลอดภัย -> คุกกี้และข้อมูลเว็บไซต์อื่นๆ
2.  เลือก 'อนุญาตคุกกี้ทั้งหมด' หากคุณไม่ต้องการเลือกตัวเลือกนี้ ให้เลือกตัวเลือกที่สอง

ตัวเลือกที่ 2:
1.  ไปที่การตั้งค่าโดยป้อน chrome://settings/ ในแถบที่อยู่ จากนั้นไปที่ ความเป็นส่วนตัวและความปลอดภัย -> คุกกี้และข้อมูลเว็บไซต์อื่นๆ
2.  หากเลือก 'บล็อกคุกกี้ของบุคคลที่สามในโหมดไม่ระบุตัวตน' หรือ 'บล็อกคุกกี้บุคคลที่สาม' ให้ไปที่ 'ไซต์ที่สามารถใช้คุกกี้ได้ตลอดเวลา' แล้วคลิก **เพิ่ม** 
3.  เพิ่มชื่อไซต์แอปการเงินและการดําเนินงานของคุณ - https://<your_FinOp_instance>.cloudax.dynamics.com ตรวจสอบให้แน่ใจว่าคุณเลือกกล่องกาเครื่องหมาย "คุกกี้ทั้งหมดบนไซต์นี้เท่านั้น" 

### <a name="microsoft-edge-browser"></a>เบราว์เซอร์ Microsoft Edge
1.  นําทางไปยังการตั้งค่า ->สิทธิ์ของไซต์ ->คุกกี้และข้อมูลไซต์
2.  ปิด 'บล็อกคุกกี้ของบุคคลที่สาม'  

## <a name="unlink-and-link-another-dataverse-environment-from-a-finance-and-operations-app"></a>ยกเลิกการเชื่อมโยงและเชื่อมโยงสภาพแวดล้อม Dataverse อื่นจากแอปการเงินและการดำเนินงาน

**บทบาทที่จำเป็นในการยกเลิกการเชื่อมโยงสภาพแวดล้อม:** ผู้ดูแลระบบสำหรับแอปการเงินและการดำเนินงานหรือ Dataverse

1. ลงชื่อเข้าใช้แอปการเงินและการดำเนินงาน
2. ไปที่ **พื้นที่ทำงาน \> การจัดการข้อมูล** และเลือกไทล์ **การรวมแบบสองทิศทาง**
3. เลือกการแมปที่กำลังเรียกใช้ทั้งหมด และจากนั้น เลือก **หยุด**
4. เลือก **ยกเลิกการเชื่อมโยงสภาพแวดล้อม**
5. เลือก **ใช่** เพื่อยืนยันการดำเนินงาน

ขณะนี้คุณสามารถเชื่อมโยงสภาพแวดล้อมใหม่

## <a name="unable-to-view-the-sales-order-line-information-form"></a>ไม่สามารถดูฟอร์มข้อมูลรายการใบสั่งขายได้ 

เมื่อคุณสร้างใบสั่งขายใน Dynamics 365 Sales การคลิก **+เพิ่มผลิตภัณฑ์** อาจเปลี่ยนเส้นทางคุณไปยังฟอร์มรายการใบสั่งการดำเนินงานของโครงการ Dynamics 365 Project Operations ไม่มีวิธีการจากฟอร์มนั้นในการดูฟอร์ม **ข้อมูล** รายการใบสั่งขาย ตัวเลือกสำหรับ **ข้อมูล** ไม่ปรากฏในรายการแบบหล่นลงด้านล่าง **รายการใบสั่งใหม่** นี่เกิดขึ้นเนื่องจากมีการติดตั้งการดำเนินงานโครงการในสภาพแวดล้อมของคุณ

เมื่อต้องการเปิดใช้งานตัวเลือกฟอร์ม **ข้อมูล** อีกครั้ง ให้ทำตามขั้นตอนต่อไปนี้:

1. นำทางไปยังตาราง **รายการใบสั่ง**
2. ค้นหาฟอร์ม **ข้อมูล** ภายใต้โหนดฟอร์ม
3. เลือกฟอร์ม **ข้อมูล** และคลิก **เปิดใช้งานบทบาทความปลอดภัย**
4. เปลี่ยนการตั้งค่าความปลอดภัยเป็น **แสดงต่อทุกคน**

## <a name="how-to-ensure-data-integration-is-using-the-most-current-finance-and-operations-schema"></a>วิธีการตรวจสอบให้แน่ใจว่าการรวมข้อมูลใช้ Schema ของการเงินและการดําเนินงานปัจจุบัน

คุณอาจพบปัญหาเกี่ยวกับข้อมูลในการรวมข้อมูลของคุณ ถ้าไม่ได้ใช้ Schema ล่าสุด ขั้นตอนต่อไปนี้จะช่วยคุณรีเฟรชรายการเอนทิตี้ในแอปการเงินและการดําเนินงานและเอนทิตี้ในตัวรวมข้อมูล

### <a name="refresh-entity-list-in-finance-and-operations-environment"></a>รีเฟรชรายการเอนทิตี้ในสภาพแวดล้อมการเงินและการดำเนินงาน
1.  เข้าสู่ระบบสภาพแวดล้อมการเงินและการดำเนินงานของคุณ
2.  เลือก **การจัดการข้อมูล**
3.  ในการจัดการข้อมูล ให้เลือก **พารามิเตอร์ของกรอบงาน**
4.  บนหน้า **พารามิเตอร์กรอบงานการนําเข้า/ส่งออกข้อมูล** ให้เลือกแท็บ **การตั้งค่าเอนทิตี** แล้วจากนั้น เลือก **รีเฟรชรายการเอนทิตี** ซึ่งอาจใช้เวลามากกว่า 30 นาทีในการรีเฟรช ทั้งนี้ขึ้นอยู่กับจํานวนเอนทิตี้ที่เกี่ยวข้อง
5.  ไปยัง **การจัดการข้อมูล** และเลือก **เอนทิตี้ข้อมูล** เพื่อตรวจสอบว่าเอนทิตี้ที่คาดไว้แสดงอยู่ ถ้าเอนทิตี้ที่คาดไว้ไม่อยู่ในรายการ ให้ตรวจสอบว่าเอนทิตี้ปรากฏในสภาพแวดล้อมการเงินและการดําเนินการของคุณ และคืนค่าเอนทิตี้ที่ขาดหายไปตามต้องการ

#### <a name="if-the-refresh-fails-to-resolve-the-issue-delete-and-re-add-the-entities"></a>ถ้าการรีเฟรชไม่สามารถแก้ปัญหาได้ ให้ลบและเพิ่มเอนทิตี้ใหม่

> [!NOTE]
> คุณอาจต้องหยุดกลุ่มการประมวลผลใดๆ ที่ใช้งานอยู่โดยใช้เอนทิตี้ก่อนที่จะลบ

1.  เลือก **การจัดการข้อมูล** ในสภาพแวดล้อมการเงินและการดําเนินงานของคุณ และเลือก **เอนทิตี้ข้อมูล**
2.  ค้นหาเอนทิตี้ที่มีปัญหา จดบันทึกเอนทิตี้เป้าหมาย ตารางการแบ่งระยะ ชื่อเอนทิตี้ และการตั้งค่าอื่นๆ ลบเอนทิตี้จากรายการ
3.  เลือก **ใหม่** และเพิ่มเอนทิตี้ใหม่โดยใช้ข้อมูลจากขั้นตอนที่ 2 

#### <a name="refresh-entities-in-data-integrator"></a>รีเฟรชเอนทิตี้ในตัวรวมข้อมูล
เข้าสู่ระบบศูนย์จัดการ Power Platform และเลือก **การรวมข้อมูล** เปิดโครงการที่เกิดปัญหา และเลือก **รีเฟรชเอนทิตี้**

## <a name="how-to-enable-and-save-network-trace-so-that-traces-can-be-attached-to-support-tickets"></a>วิธีการเปิดใช้งานและบันทึกการติดตามเครือข่าย เพื่อให้สามารถแนบการติดตามกับตั๋วการสนับสนุน

ทีมงานสนับสนุนอาจต้องการตรวจทานการติดตามเครือข่ายเพื่อแก้ไขปัญหาบางอย่าง เมื่อต้องการสร้างการติดตามเครือข่าย ให้ทำตามขั้นตอนเหล่านี้:

### <a name="google-chrome-browser"></a>เบราว์เซอร์ Google Chrome

1. ในแท็บที่เปิด กด **F12** หรือเลือก **เครื่องมือของนักพัฒนา** เพื่อเปิดเครื่องมือของนักพัฒนา
2. เปิดแท็บ **เครือข่าย** และชนิด **integ** ในกล่องข้อความตัวกรองข้อมูล
3. เรียกใช้สถานการณ์ของคุณและพิจารณาการล็อกคำขอ
4. คลิกขวารายการและเลือก **บันทึกทั้งหมดเป็น HAR ที่มีเนื้อหา**

### <a name="microsoft-edge-browser"></a>เบราว์เซอร์ Microsoft Edge

1. ในแท็บที่เปิด กด **F12** หรือเลือก **เครื่องมือของนักพัฒนา** เพื่อเปิดเครื่องมือของนักพัฒนา
2. เปิดแท็บ **เครือข่าย**
3. เรียกใช้สถานการณ์ของคุณ
4. เลือก **บันทึก** เพื่อส่งออกผลลัพธ์เป็น HAR
