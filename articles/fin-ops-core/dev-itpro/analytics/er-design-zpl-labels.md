# <a name="design-a-new-er-solution-to-print-zpl-labels"></a>ออกแบบโซลูชันการรายงานทางอิเล็กทรอนิกส์ใหม่เพื่อพิมพ์ป้ายชื่อ ZPL

[!include [banner](../includes/banner.md)]


บทความนี้อธิบายวิธีการที่ผู้ใช้ในบทบาทผู้ดูแลระบบ ผู้พัฒนาการรายงานทางอิเล็กทรอนิกส์ หรือผู้ที่ปรึกษาด้านการรายงานทางอิเล็กทรอนิกส์ สามารถตั้งค่าคอนฟิกพารามิเตอร์ของกรอบงาน [การรายงานทางอิเล็กทรอนิกส์ (ER)](general-electronic-reporting.md) ออกแบบ [การตั้งค่าคอนฟิก](general-electronic-reporting.md#Configuration) ER ที่จำเป็นของโซลูชัน ER ใหม่เพื่อเข้าถึงข้อมูลของระบบการจัดการคลังสินค้า และสร้างป้ายชื่อสถานที่ตั้งค่าคลังสินค้าที่กำหนดเองในรูปแบบ Zebra Programming Language (ZPL) II คุณสามารถดำเนินการขั้นตอนเหล่านี้ได้ในบริษัท **USRT**

## <a name="business-scenario"></a>สถานการณ์จำลองทางธุรกิจ

คุณเป็นตัวแทนของบริษัทที่ดำเนินการจัดการคลังสินค้าใน Microsoft Dynamics 365 Finance สถานที่ตั้งคลังสินค้าทุกแห่งต้องมีป้ายชื่อแบบติดกาวในตัวที่มีบาร์โค้ด ผู้ปฏิบัติงานคลังสินค้าจะใช้เครื่องอ่านบาร์โค้ดของอุปกรณ์มือถือเพื่อสแกนบาร์โค้ด

สถานที่ตั้งคลังสินค้าทั้งหมดมีป้ายชื่ออยู่ในขอบเขตของกิจกรรมก่อนการใช้งานจริง อย่างไรก็ตาม คุณต้องสามารถพิมพ์ป้ายชื่อสถานที่ตั้งคลังสินค้าตามความต้องการหากป้ายชื่อที่มีอยู่เสียหายหรือการตั้งค่าคอนฟิกการจัดชิ้นคลังสินค้าใหม่ ด้วยการใช้ฟังก์ชัน ER ที่ปล่อยเมื่อเร็วๆ นี้ คุณสามารถตั้งค่าคอนฟิกโซลูชัน ER ใหม่ที่ช่วยให้หัวหน้างานคลังสินค้าสามารถพิมพ์ป้ายชื่อโดยตรงที่เครื่องพิมพ์ป้ายชื่อแบบความร้อน

## <a name="configure-the-er-framework"></a>ตั้งค่าคอนฟิกกรอบงาน ER

ปฏิบัติตามขั้นตอนใน [ตั้งค่าคอนฟิกกรอบงาน ER](er-quick-start2-customize-report.md#ConfigureFramework) เพื่อตั้งค่าชุดที่น้อยที่สุดของพารามิเตอร์ ER คุณต้องเสร็จสิ้นการตั้งค่านี้ก่อนที่คุณจะเริ่มใช้กรอบงาน ER เพื่อออกแบบโซลูชัน ER ใหม่

## <a name="design-a-domain-specific-data-model"></a>ออกแบบรูปแบบข้อมูลเฉพาะโดเมน

สร้างการกำหนดค่า ER ใหม่ที่มีส่วนประกอบ [แบบจำลองข้อมูล](er-overview-components.md#data-model-component) สำหรับโดเมนการจัดการคลังสินค้า รูปแบบข้อมูลนี้จะใช้เป็นแหล่งข้อมูลในภายหลังเมื่อคุณออกแบบรูปแบบ ER เพื่อสร้างป้ายชื่อสถานที่ตั้งคลังสินค้า

### <a name="import-a-data-model-configuration"></a>นำเข้าการตั้งค่าคอนฟิกแบบจำลองข้อมูล

ปฏิบัติตามขั้นตอนเหล่านี้เพื่อนําเข้าแบบจำลองข้อมูลที่ต้องใช้จากไฟล์ XML ซึ่งจัดเตรียมโดย Microsoft หรือคุณสามารถสร้างแบบจำลองข้อมูลของคุณเองตามที่อธิบายไว้ในส่วนถัดไป

1. ดาวน์โหลดไฟล์ [Warehouse model.version.1.xml](https://download.microsoft.com/download/9/f/1/9f136e9b-bf5f-403a-b089-a2b2ed1da2ba/Warehouse-model.version.1.xml) และบันทึกไปยังคอมพิวเตอร์เฉพาะที่ของคุณ
2. ไปที่ **การจัดการองค์กร** \> **พื้นที่ทำงาน** \> **การรายงานทางอิเล็กทรอนิกส์**
3. ในพื้นที่ทำงาน **การรายงานทางอิเล็กทรอนิกส์** เลือก **การตั้งค่าคอนฟิกการรายงาน**
4. บนหน้า **การกำหนดค่าคอนฟิก** บนบานหน้าต่างการดำเนินการ เลือก **การแลกเปลี่ยน** \> **โหลดจากไฟล์ XML**
5. เลือก **เรียกดู** แล้วค้นหาและเลือกไฟล์ **Warehouse model.version.1.xml**
6. เลือก **ตกลง** เพื่อนำเข้าการตั้งค่าคอนฟิก

![การตั้งค่าคอนฟิกแบบจำลองข้อมูล ER ที่นำเข้าบนหน้าการตั้งค่าคอนฟิก](./media/er-design-zpl-labels-imported-model.png)

แทนที่จะนําเข้าไฟล์แบบจำลองข้อมูลของ Microsoft คุณสามารถสร้างแบบจำลองข้อมูลตั้งแต่ต้นได้ สำหรับตัวอย่างที่แสดงวิธีการทำงานนี้ ดูที่ [สร้างการตั้งค่าคอนฟิกแบบจำลองข้อมูลใหม่](er-quick-start1-new-solution.md#DesignDataModel)

## <a name="design-a-model-mapping-for-the-configured-data-model"></a>ออกแบบการแมปรูปแบบสำหรับรูปแบบข้อมูลที่ตั้งค่าคอนฟิก

ในฐานะผู้ใช้ในบทบาทนักพัฒนาการรายงานทางอิเล็กทรอนิกส์ คุณต้องสร้างการตั้งค่าคอนฟิก ER ใหม่ที่มีส่วนประกอบของ [การแมปแบบจำลอง](er-overview-components.md#model-mapping-component) สำหรับแบบจำลองข้อมูลคลังสินค้า ส่วนประกอบนี้ใช้แบบจำลองข้อมูลที่ตั้งค่าคอนฟิกสำหรับ Dynamics 365 Finance และใช้เฉพาะกับแอปนั้น คุณต้องตั้งค่าคอนฟิกส่วนประกอบเพื่อระบุออบเจ็กต์ของแอปพลิเคชันที่จะใช้ในการกรอกข้อมูลแบบจำลองข้อมูลที่ตั้งค่าคอนฟิกด้วยข้อมูลแอปพลิเคชันขณะรันไทม์ เพื่อให้งานนี้เสร็จสมบูรณ์ คุณต้องเข้าใจวิธีการนำโครงสร้างข้อมูลของโดเมนธุรกิจการจัดการคลังสินค้าไปใช้ใน Finance

### <a name="review-the-format"></a>ตรวจทานรูปแบบ

คุณสามารถดูรุ่นที่แก้ไขได้ของรูปแบบที่ตั้งค่าคอนฟิกได้ในหน้า **ตัวออกแบบรูปแบบ**

![โครงสร้างของรูปแบบ ER ในหน้าตัวออกแบบรูปแบบ](./media/er-design-zpl-labels-format.png)

แหล่งข้อมูล `model.Location.Label` ของรูปแบบนี้ได้รับการตั้งค่าคอนฟิกเพื่อสร้างป้ายชื่อที่มีข้อมูลต่อไปนี้

- ชื่อคลังสินค้าเป็นข้อความ
- ชื่อคลังสินค้าเป็นบาร์โค้ด
- ชื่อสถานที่ตั้ง
- ตัวเลขการตรวจสอบ

ในหน้า **ตัวออกแบบสูตร** ของแหล่งข้อมูล สูตร ER ที่ใช้ในการสร้างป้ายชื่อมีฟังก์ชัน `CONCATENATE` ที่รวมข้อมูลในเค้าโครงที่ต้องการ

![สูตรสำหรับแหล่งข้อมูลบนหน้าตัวออกแบบสูตร](./media/er-design-zpl-labels-review-formula.png)

> [!TIP]
> เค้าโครงป้ายชื่อได้รับการออกแบบเพื่อให้ชื่อสถานที่ตั้งและตัวเลขการตรวจสอบมีความสอดคล้องที่กึ่งกลางของป้ายชื่อ อย่างไรก็ตาม ZPL II ไม่รองรับการปรับแนวตรงกลางสำหรับบาร์โค้ด ดังนั้น สูตรของแหล่งข้อมูล `model.Location.Warehouse.Alignment` จึงถูกใช้เพื่อปรับบาร์โค้ดให้ชิดขอบตรงกลางของป้ายชื่อ สูตรนี้จะคํานวณออฟเซ็ตด้านซ้ายของบาร์โค้ดตามจํานวนอักขระในชื่อคลังสินค้า

### <a name="install-and-configure-a-printer-emulator-application"></a>การติดตั้งและตั้งค่าคอนฟิกแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์

เพิ่มแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์ของกลไกจัดการการแสดงภาพ ZPL ลงในเว็บเบราว์เซอร์ Chrome ของคุณ ตัวอย่างนี้ใช้ตัวเลียนแบบ [เครื่องพิมพ์ ZPL](https://chrome.google.com/webstore/detail/zpl-printer/phoidlklenidapnijkabnfdgmadlcmjo) ที่ยึดตาม [บริการเว็บ Labelary ZPL](http://labelary.com/service.html) แอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์จะส่งผ่านป้ายชื่อที่สร้างขึ้นในรูปแบบ ZPL จากเครื่องพิมพ์เฉพาะที่ไปยังบริการเว็บ แล้วส่งคืนป้ายชื่อเป็นไฟล์ PDF หรือ PNG เพื่อแสดงตัวอย่าง

1. ในร้านค้าบนเว็บ Chrome ให้ค้นหาและเลือกแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์ที่คุณต้องการใช้ จากนั้นเลือก **เพิ่มลงใน Chrome** เพื่อเพิ่มลงในเว็บเบราว์เซอร์ Chrome ของคุณ

    ![การเพิ่มแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์ลงในเว็บเบราว์เซอร์ Chrome จากร้านค้าบนเว็บ Chrome](./media/er-design-zpl-labels-add-app.png)

2. เลือก **เปิดใช้แอป** เพื่อเรียกใช้แอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์จากเว็บเบราว์เซอร์ Chrome

    ![การเรียกใช้แอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์จากเว็บเบราว์เซอร์ Chrome](./media/er-design-zpl-labels-run-app.png)

3. ตั้งค่าคอนฟิกแอปพลิเคชันที่เรียกใช้:

    1. ปิดแอปพลิเคชัน
    2. ในการตั้งค่าเครื่องพิมพ์ ให้ตั้งค่าโฮสต์เป็น **127.0.0.1**
    3. ตั้งค่าพอร์ตเป็น **9100**

        ![การตั้งค่าคอนฟิกแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์](./media/er-design-zpl-labels-configure-app.png)

    4. เปิดแอปพลิเคชันอีกครั้ง คุณควรได้รับข้อความที่ระบุว่าเครื่องพิมพ์เริ่มต้นแล้วบนโฮสต์และพอร์ตที่ระบุ

        ![เปิดแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์อีกครั้ง](./media/er-design-zpl-labels-turn-on-app.png)

### <a name="add-and-configure-a-local-printer"></a>เพิ่มและตั้งค่าคอนฟิกเครื่องพิมพ์เฉพาะที่

[เพิ่มเครื่องพิมพ์เฉพาะที่ใหม่](https://support.microsoft.com/windows/install-a-printer-in-windows-10-cc0724cf-793e-3542-d1ff-727e4978638b) ที่อุปกรณ์ปัจจุบันสามารถใช้ส่งผ่านป้ายชื่อที่สร้างจาก DRA ไปยังงแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์

1. ใน Windows ให้เลือก **เริ่มต้น** \> **การตั้งค่า** \> **อุปกรณ์** \> **เครื่องพิมพ์ \& สแกนเนอร์**
2. เลือก **การตั้งค่าเครื่องพิมพ์ \& สแกนเนอร์**
3. สำหรับ **เพิ่มเครื่องพิมพ์หรือสแกนเนอร์** ให้เลือก **เพิ่มอุปกรณ์**
4. สำหรับ **เครื่องพิมพ์ที่ฉันต้องการไม่มีในรายการ** เลือก **เพิ่มด้วยตนเอง**
5. ในฟิลด์ **ค้นหาเครื่องพิมพ์ตามตัวเลือกอื่นๆ** ให้เลือก **เพิ่มเครื่องพิมพ์เฉพาะที่หรือเครื่องพิมพ์เครือข่ายที่มีการตั้งค่าด้วยตนเอง**
6. ในฟิลด์ **เลือกพอร์ตเครื่องพิมพ์** ให้เลือก **สร้างพอร์ตใหม่** แล้วปฏิบัติตามขั้นตอนเหล่านี้

    1. ในฟิลด์ **ชนิดของพอร์ต** เลือก **พอร์ต TCP/IP มาตรฐาน**
    2. ในฟิลด์ **ชื่อโฮสต์หรือที่อยู่ IP** ให้ป้อน **127.0.0.1**
    3. ในฟิลด์ **ชื่อพอร์ต** ให้ป้อน **ZPL**
    4. รอจนกว่าการดำเนินงาน **กำลังตรวจหาพอร์ต TCP/IP** จะเสร็จสมบูรณ์
    5. ในฟิลด์ **ชนิดอุปกรณ์** ให้เลือก **กำหนดเอง** แล้วเลือก **การตั้งค่า**
    6. ตรวจสอบให้แน่ใจว่าได้ระบุการตั้งค่าพอร์ตต่อไปนี้

        - **ชื่อพอร์ต:** ZPL
        - **ชื่อเครื่องพิมพ์หรือที่อยู่ IP:** 127.0.0.1
        - **โพรโทคอล:** แบบ Raw
        - **หมายเลขพอร์ต:** 9100

7. ในฟิลด์ **ติดตั้งโปรแกรมควบคุมเครื่องพิมพ์** ให้เลือก **ทั่วไป / ข้อความเท่านั้น**
8. ในฟิลด์ **ชื่อเครื่องพิมพ์** ให้ป้อน **ZebraPrinter**

![การเพิ่มเครื่องพิมพ์เฉพาะที่ให้กับอุปกรณ์ปัจจุบัน](./media/er-design-zpl-labels-configure-printer.png)

### <a name="configure-the-er-destination"></a>ตั้งค่าคอนฟิกปลายทาง ER

จัดเตรียมปลายทาง ER ที่จะส่งผ่านป้ายชื่อที่สร้างจาก Finance ไปยัง DRA

1. ไปที่ **การจัดการองค์กร** \> **การรายงานทางอิเล็กทรอนิกส์** \> **ปลายทางการรายงานทางอิเล็กทรอนิกส์**
2. บนหน้า **ปลายทางการรายงานทางอิเล็กทรอนิกส์** ในบานหน้าต่างการดำเนินการ ให้เลือก **สร้าง**
3. ในฟิลด์ **การอ้างอิง** ให้เลือก **ป้ายชื่อสถานที่ตั้งคลังสินค้า**
4. บน FastTab **ปลายทางไฟล์** ให้เลือก **ใหม่**
5. ในฟิลด์ **ชื่อ** ให้ป้อน **ป้ายชื่อ**
6. ในฟิลด์ **ชื่อส่วนประกอบของไฟล์** ให้เลือก **รายงาน**
7. เลือก **การตั้งค่า**
8. ในกล่องโต้ตอบ **การตั้งค่าปลายทาง** บนแท็บ **เครื่องพิมพ์** ให้ตั้งค่าตัวเลือก **เปิดใช้งาน** เป็น **ใช่**
9. ในฟิลด์ **ชื่อเครื่องพิมพ์** ให้เลือก **ZebraPrinter**
10. ในฟิลด์ **ชนิดการกำหนดเส้นทางเอกสาร** ให้เลือก **ZPL**
11. เลือก **ตกลง**

![การตั้งค่าคอนฟิกปลายทาง ER สำหรับรูปแบบป้ายชื่อสถานที่ตั้งคลังสินค้าบนหน้าปลายทางการรายงานทางอิเล็กทรอนิกส์](./media/er-design-zpl-labels-configure-destination.png)

2. ในหน้า **สถานที่ตั้ง** กรองเพื่อดูสถานที่ตั้งที่มีค่าในฟิลด์ **ตัวเลขตรวจสอบ**

## <a name="print-warehouse-location-labels"></a>พิมพ์ป้ายชื่อสถานที่ตั้งคลังสินค้า

1. ไปที่ **การจัดการองค์กร** \> **การรายงานทางอิเล็กทรอนิกส์** \> **การตั้งค่าคอนฟิก**
2. ในหน้า **การตั้งค่าคอนฟิก** ในแผนภูมิการตั้งค่าคอนฟิก ขยาย **แบบจำลองคลังสินค้า** และเลือก **ป้ายชื่อสถานที่ตั้งคลังสินค้า**
3. บนบานหน้าต่างการดำเนินการ เลือก **รัน**
4. ในกล่องโต้ตอบ **พารามิเตอร์ของรายงานอิเล็กทรอนิกส์** บนแท็บ **เรกคอร์ดที่จะรวม** เลือก **ตัวกรอง**
5. บนแท็บ **ช่วง** ให้ค้นหาแถวที่มีฟิลด์ **ตาราง** ตั้งค่าเป็น **สถานที่ตั้ง** และฟิลด์ **ฟิลด์** ตั้งค่าเป็น **สถานที่ตั้ง** ในฟิลด์ **เกณฑ์** ให้ป้อน **LPEnabled**
6. เลือก **ตกลง**
7. เลือก **ตกลง** ป้ายชื่อจะถูกสร้างและแสดงบนหน้าแสดงตัวอย่างในแอพลิเคชันแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์

## <a name="modify-the-layout-of-a-label"></a>แก้ไขเค้าโครงของป้ายชื่อ

คุณสามารถเปลี่ยนเค้าโครงปัจจุบันของป้ายชื่อสถานที่ตั้งคลังสินค้าของคุณได้ ตัวอย่างต่อไปนี้แสดงวิธีการเปลี่ยนเค้าโครงเพื่อให้ป้ายชื่อที่สร้างขึ้นมีรหัสโพรไฟล์สถานที่ตั้ง

1. ไปที่ **การจัดการองค์กร** \> **การรายงานทางอิเล็กทรอนิกส์** \> **การตั้งค่าคอนฟิก**
2. ตั้งค่า **ใช้ปลายทางสำหรับสถานะแบบร่าง** [พารามิเตอร์ผู้ใช้ ER](electronic-reporting-destinations.md#applicability) เป็น **ใช่**
3. ในหน้า **การตั้งค่าคอนฟิก** ในแผนภูมิการตั้งค่าคอนฟิก ขยาย **แบบจำลองคลังสินค้า** และเลือก **ป้ายชื่อสถานที่ตั้งคลังสินค้า**
4. เลือก **ตัวออกแบบ**
5. บนหน้า **ตัวออกแบบรูปแบบ** บนแท็บ **การแมป** เลือกแหล่งข้อมูล `model.Location.Label`
6. ในกล่องโต้ตอบ **คุณสมบัติแหล่งข้อมูล** เลือก **แก้ไข** \> **แก้ไขสูตร**
7. บนหน้า **ตัวออกแบบสูตร** ในฟิล **สูตร** ให้ตรวจสอบสูตร ER ที่ใช้ในการสร้างป้ายชื่อ

    ```vb
    CONCATENATE(
    "^XA",CrLf,
    "^CF0,30,30^FO0,30^FB800,1,0,C,0^FD",Warehouse,"\&^FS",CrLf,
    "^BY2,2,50^FT",@.Warehouse.Alignment,",126^BCN,,N,N,N,A^FD",Warehouse,"\&^FS",CrLf,
    "^FO0,150^FB800,1,0,C,0^FD",@.Name,"\&^FS",CrLf,
    "^CF0,20,20^FO0,200^FB800,1,0,C,0^FD",@.CheckDigits,"\&^FS",CrLf,
    "^XZ")
    ```

8. อัปเดตสูตรเพื่อเพิ่มรหัสโพรไฟล์สถานที่ตั้งในป้ายชื่อที่สร้างขึ้น

    ```vb
    CONCATENATE(
    "^XA",CrLf,
    "^CF0,30,30^FO0,30^FB800,1,0,C,0^FD",Warehouse,"\&^FS",CrLf,
    "^BY2,2,50^FT",@.Warehouse.Alignment,",126^BCN,,N,N,N,A^FD",Warehouse,"\&^FS",CrLf,
    "^FO0,150^FB800,1,0,C,0^FD",@.Name,"\&^FS",CrLf,
    "^CF0,20,20^FO0,200^FB800,1,0,C,0^FD",@.CheckDigits,"\&^FS",CrLf,
    "^CF0,40,40^FO0,240^FB800,1,0,C,0^FD",@.ProfileID,"\&^FS",CrLf,
    "^XZ")
    ```

9. เลือก **บันทึก**
10. เลือก **ตกลง**
11. บนบานหน้าต่างการดำเนินการ เลือก **รัน**
12. ในกล่องโต้ตอบ **พารามิเตอร์ของรายงานอิเล็กทรอนิกส์** บนแท็บ **เรกคอร์ดที่จะรวม** เลือก **ตัวกรอง**
13. บนแท็บ **ช่วง** ให้ค้นหาแถวที่มีฟิลด์ **ตาราง** ตั้งค่าเป็น **สถานที่ตั้ง** และฟิลด์ **ฟิลด์** ตั้งค่าเป็น **สถานที่ตั้ง** ในฟิลด์ **เงื่อนไข** , ให้ป้อน **Bay**
14. เลือก **ตกลง**
15. เลือก **ตกลง** ป้ายชื่อจะถูกสร้างและแสดงบนหน้าแสดงตัวอย่างในแอพลิเคชันแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์

![การตรวจสอบป้ายชื่อที่สร้างขึ้นที่มีรหัสโพรไฟล์สถานที่ตั้งในหน้าแสดงตัวอย่างของแอปพลิเคชันตัวเลียนแบบเครื่องพิมพ์ Zpl](./media/er-design-zpl-labels-preview-label2.png)

## <a name="encoding"></a>การเข้ารหัส

> [!NOTE]
> คุณต้องซิงโครไนซ์การตั้งค่าการเข้ารหัสของส่วนประกอบ **Common\\File** ของรูปแบบ ER ที่แก้ไขได้และการตั้งค่าที่เหมาะสมของป้ายชื่อที่ออกแบบ ค่าของฟิลด์ **[การเข้ารหัส](er-suppress-bom-characters.md)** ของส่วนประกอบ **Common\\File** ไม่ควรขัดแย้งกับคำสั่ง ZPL ที่ใช้ควบคุมการเข้ารหัสของป้ายชื่อ (ตัวอย่างเช่น คำสั่ง `^CI`) ER ไม่ได้ตรวจสอบว่าการตั้งค่าเหล่านี้มีการซิงโครไนซ์หรือไม่
