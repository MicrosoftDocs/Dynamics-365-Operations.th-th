# <a name="parse-incoming-documents-in-json-format"></a>แยกวิเคราะห์เอกสารที่กำลังจะมาถึงในรูปแบบ JSON

[!include[banner](../includes/banner.md)]

คุณสามารถออกแบบรูปแบบการรายงานทางอิเล็กทรอนิกส์ (ER) เพื่อแยกวิเคราะห์เอกสารอิเล็กทรอนิกส์ขาเข้า ซึ่งแสดงข้อมูลในรูปแบบข้อความที่เป็นไปตาม JavaScript (กล่าวได้อีกอย่างหนึ่งว่า ไฟล์ที่อยู่ในรูปแบบ JavaScript Object Notation \[JSON\])

เมื่อต้องการเรียนรู้เพิ่มเติมเกี่ยวกับคุณลักษณะ ให้ดาวน์โหลดไฟล์ในตารางต่อไปนี้ แล้วเล่น ER สร้างการตั้งค่าคอนฟิกรูปแบบเพื่อนำเข้าข้อมูลจากคู่มืองานไฟล์ JSON ภายนอก คู่มืองานนี้แสดงวิธีการที่สามารถใช้รูปแบบ ER เพื่อแยกวิเคราะห์ไฟล์ JSON ขาเข้าเพื่อปรับปรุงข้อมูลแอพลิเคชัน

## <a name="requirements"></a>ความต้องการ

ก่อนที่คุณจะดำเนินการ ER สร้างการตั้งค่าคอนฟิกรูปแบบ เพื่อนำเข้าข้อมูลจากคู่มืองานไฟล์ JSON ภายนอก ต้องเป็นไปตามข้อกำหนดต่อไปนี้:

- องค์ประกอบหลักในไฟล์ JSON อาจเป็นองค์ประกอบออบเจ็กต์เท่านั้น
- องค์ประกอบที่ซ้อนกันสำหรับออบเจ็กต์ควรเป็นองค์ประกอบของคุณสมบัติ ดังนั้นจึงมีการสร้างโครงสร้าง JSON ที่ถูกต้อง
- อาร์เรย์ JSON สามารถเป็นได้เฉพาะองค์ประกอบที่ซ้อนกันขององค์ประกอบคุณสมบัติของออบเจ็กต์เท่านั้น
- อาร์เรย์ JSON สามารถประกอบด้วยออบเจ็กต์ JSON เท่านั้น ไม่สามารถมีค่าสตริงตัวอักษรและตัวเลขโดยตรงและอาร์เรย์ที่ซ้อนกันได้ องค์ประกอบในอาร์เรย์เหล่านี้จะได้รับการแยกวิเคราะห์ตามลำดับ เมื่อมีการระบุอยู่ในรูปแบบ การตั้งค่าความหลากหลายของออบเจ็กต์แต่ละ JSON รายการจะถูกพิจารณา

นอกจากนี้ คุณต้องดำเนินการคู่มืองาน [ER สร้างการตั้งค่าคอนฟิกที่จำเป็นเพื่อนำเข้าข้อมูลจากไฟล์ภายนอก](tasks/er-required-configurations-import-data.md) ให้เสร็จสมบูรณ์ ถ้าคุณยังไม่ได้ดำเนินการให้เสร็จสมบูรณ์ ดาวน์โหลดไฟล์ต่อไปนี้เพื่อดำเนินการคู่มืองานให้เสร็จสมบูรณ์
