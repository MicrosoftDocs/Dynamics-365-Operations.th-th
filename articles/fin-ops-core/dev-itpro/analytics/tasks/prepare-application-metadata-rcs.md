# <a name="prepare-application-metadata-to-be-used-in-rcs"></a>จัดเตรียมข้อมูลเมตาของแอพลิเคชันที่จะใช้ใน RCS
[!include [banner](../../includes/banner.md)]

ขั้นตอนต่อไปนี้จะอธิบายวิธีการที่ผู้ใช้ในบทบาทผู้ดูแลระบบหรือนักพัฒนาการรายงานทางอิเล็กทรอนิกส์ สามารถสร้างการตั้งค่าคอนฟิกการรายงานทางอิเล็กทรอนิกส์ (ER) ใหม่ ซึ่งประกอบด้วยข้อมูลเมตาของแอพลิเคชัน สำหรับการออกแบบการตั้งค่าคอนฟิกการแม็ปแบบจำลอง ER ใน Regulatory configuration service (RCS) การตั้งค่าคอนฟิกนี้จะถูกใช้สำหรับการออกแบบการตั้งค่าคอนฟิกการแม็ปแบบจำลอง ER เพื่อเข้าถึงธุรกรรมการค้าต่างประเทศ ในตัวอย่างนี้ คุณจะสร้างการตั้งค่าคอนฟิกสำหรับบริษัทตัวอย่าง Litware, Inc. ขั้นตอนเหล่านี้สามารถถูกดำเนินการได้ในบริษัทใดๆ เพื่อทำขั้นตอนเหล่านี้ให้เสร็จสมบูรณ์ อันดับแรกคุณต้องทำขั้นตอนต่างๆ ในหัวข้อให้เสร็จสมบูรณ์ [สร้างผู้ให้บริการการตั้งค่าคอนฟิก และทำเครื่องหมายเป็น ใช้งานอยู่](er-configuration-provider-mark-it-active-2016-11.md)

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น
1.    ไปที่ **การจัดการองค์กร** > **พื้นที่ทำงาน** > **การรายงานทางอิเล็กทรอนิกส์** 
2.    ตรวจสอบให้แน่ใจว่าผู้ให้บริการการตั้งค่าคอนฟิกสำหรับบริษัทตัวอย่าง Litware, Inc. พร้อมใช้งาน และถูกทำเครื่องหมายเป็น **ใช้งานอยู่** ถ้าคุณไม่เห็นผู้ให้บริการการตั้งค่าคอนฟิกนี้ จะต้องดำเนินขั้นตอนต่างๆ ในกระบวนงาน [สร้างผู้ให้บริการการตั้งค่าคอนฟิก และทำเครื่องหมายเป็น ใช้งานอยู่](er-configuration-provider-mark-it-active-2016-11.md) ให้เสร็จสมบูรณ์ 
3.    คลิก **การตั้งค่าคอนฟิกข้อมูลเมตา** 
4.    สมมติว่า RCS จะถูกใช้เพื่อออกแบบโซลูชัน ER สำหรับแอพลิเคชัน Finance and Operation ที่จะสร้างเอกสารอิเล็กทรอนิกส์ที่มีข้อมูลจากโดเมนธุรกิจการค้าต่างประเทศ เมื่อต้องการระบุการแม็ประหว่างแบบจำลองข้อมูล ER และแหล่งที่มาของข้อมูลที่จำเป็น ใน RCS เราต้องมีสิทธิเข้าถึงข้อมูลเมตาของแอพลิเคชัน Finance and Operation ดังนั้น ในฐานะที่เป็นส่วนหนึ่งของการของการออกแบบโซลูชัน ER เราตั้งค่าคอนฟิกการตั้งค่าคอนฟิกข้อมูลเมตาของ ER ใหม่ที่มีข้อมูลเมตาทั้งหมดที่จำเป็นในปัจจุบัน สำหรับการสร้างรายงาน ER สำหรับโดเมนธุรกิจที่เลือก 

## <a name="add-metadata-configuration"></a>การตั้งค่าคอนฟิกเพิ่มข้อมูลเมตา 
1.    คลิก **สร้างการตั้งค่าคอนฟิก** เพื่อเปิดกล่องโต้ตอบการวาง 
2.    ในฟิลด์ **ชื่อ** ให้พิมพ์ 'ข้อมูลเมตาการค้าต่างประเทศ' 
3.    คลิก **สร้างการตั้งค่าคอนฟิก** 
4.    คลิก **ตัวออกแบบ** 
5.    คลิก **เพิ่ม** 
  
> [!NOTE]
> คุณสามารถเลือกข้อมูลเมตาทั้งหมดสำหรับแอพลิเคชันทั้งหมด หรือแบบจำลองที่เลือกหรือโมดูลที่เลือก โปรดทราบว่าในกรณีนี้ ข้อมูลเมตาต่อไปนี้จะถูกเพิ่มโดยอัตโนมัติ: ตารางของเรกคอร์ด การแจงนับ และชนิดข้อมูลแบบขยาย เมื่อจำเป็นต้องใช้ข้อมูลเมตาชนิดอื่นๆ ต้องมีการเพิ่มด้วยตนเอง 
 
เรามีข้อมูลเมตาบางอย่างที่เกี่ยวข้องกับธุรกรรมการค้าต่างประเทศ โดยการเลือกรายการข้อมูลเมตาด้วยตนเอง 
  
6.    คลิก **เพิ่มแหล่งข้อมูล** 
7.    คลิก **เรกคอร์ดของตาราง** 
8.    ใช้ตัวกรองด่วนเพื่อกรองข้อมูลในฟิลด์ **ชื่อ** ด้วยค่าของ 'อินทราสแทต' 
9.    เลือกเรกคอร์ดตาราง **อินทราสแทต** 
10.    คลิก **ตกลง**
  
คุณได้เพิ่มข้อมูลของข้อมูลเมตาเกี่ยวกับตารางอินทราสแทตของเรกคอร์ด 
  
11.    ในแผนภูมิ ขยาย **อินทราสแทตของเรกคอร์ดตาราง**\>**ความสัมพันธ์** 
12.    ในแผนภูมิ ให้เลือก **อินทราสแทตของเรกคอร์ดตาราง**\>**ความสัมพันธ์\IntrastatCommodity (เรกคอร์ดตาราง EcoResCategory)**     
13.    คลิก **เพิ่มข้อมูลเมตา** 
  
> [!NOTE]
> ต้องมีการเพิ่มข้อมูลเมตาด้วยตนเองเกี่ยวกับความสัมพันธ์ที่จำเป็นสำหรับตารางของเรกคอร์ดที่เลือก 
  
16.    คลิก **เพิ่มแหล่งข้อมูล** 
17.    คลิก **การแจงนับ** 
18.    ใช้ตัวกรองด่วนเพื่อกรองข้อมูลในฟิลด์ **ชื่อ** ด้วยค่าของ 'IntrastatDirection' 
19.    เลือกเรกคอร์ด **การแจงนับ IntrastatDirection** 
20.    คลิก **ตกลง** 
21.    คลิก **บันทึก**  
22.    ปิดหน้า 
  
## <a name="complete-the-draft-version-of-metadata-configuration"></a>ดำเนินการตั้งค่าคอนฟิกข้อมูลเมตารุ่นแบบร่างให้เสร็จสมบูรณ์
1.    คลิก **เปลี่ยนแปลงสถานะ** 
2.    คลิก **เสร็จสมบูรณ์** 
3.    คลิก **ตกลง** 
4.    เลือกรุ่นที่เสร็จสมบูรณ์ **1** 
  
## <a name="export-the-completed-version-of-metadata-configuration-from-application-as-xml-file"></a>ส่งออกการตั้งค่าคอนฟิกข้อมูลเมตารุ่นที่เสร็จสมบูรณ์จากแอพลิเคชันเป็นไฟล์ XML
1.    คลิก **แลกเปลี่ยน** 
2.    คลิก **ส่งออกเป็นไฟล์ XML** 
3.    คลิก **ตกลง** 
    
การตั้งค่าคอนฟิกข้อมูลเมตาของ ER ที่สร้างเป็นไฟล์ XML ที่สามารถนำเข้าไปยัง RCS และใช้เป็นแหล่งที่มาของข้อมูลเกี่ยวกับข้อมูลเมตาสำหรับโดเมนธุรกิจการค้าต่างประเทศ ตามข้อมูลนี้ เราสามารถระบุการแม็ประหว่างข้อมูลเมตาของแอพลิเคชันและแบบจำลองข้อมูล ER ได้
