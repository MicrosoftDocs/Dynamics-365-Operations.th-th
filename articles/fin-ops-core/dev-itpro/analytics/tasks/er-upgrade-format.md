# <a name="er-upgrade-your-format-by-adopting-a-new-base-version-of-that-format"></a>ER อัพเกรดรูปแบบของคุณโดยการใช้เวอร์ชันฐานใหม่ของรูปแบบนั้น

[!include [banner](../../includes/banner.md)]

ขั้นตอนต่อไปนี้อธิบายวิธีที่ผู้ใช้ในบทบาทผู้ดูแลระบบหรือนักพัฒนาการรายงานทางอิเล็กทรอนิกส์ สามารถรักษาการตั้งค่าคอนฟิกการรายงานทางอิเล็กทรอนิกส์ (ER) กระบวนงานนี้อธิบายวิธีการสร้างเวอร์ชันที่กำหนดเองของรูปแบบ ซึ่งเป็นไปตามรูปแบบที่ได้รับจากผู้ให้บริการการตั้งค่าคอนฟิก (CP) นอกจากนี้ ยังอธิบายวิธีการนำเวอร์ชันฐานใหม่ของรูปแบบนั้นไปใช้อีกด้วย

เมื่อต้องการดำเนินขั้นตอนเหล่านี้ให้เสร็จสมบูรณ์ ลำดับแรกคุณต้องดำเนินงานขั้นตอนในกระบวนงาน "สร้างผู้ให้บริการการตั้งค่าคอนฟิก และทำเครื่องหมายเป็น ใช้งานอยู่" และกระบวนงาน "ใช้รูปแบบที่สร้างไว้เพื่อสร้างเอกสารอิเล็กทรอนิกส์สำหรับการชำระเงิน" ให้เสร็จสมบูรณ์ ขั้นตอนเหล่านี้สามารถถูกดำเนินการได้ในบริษัท GBSI

## <a name="select-format-configuration-for-customization"></a>เลือกการตั้งค่าคอนฟิกรูปแบบสำหรับการเลือกกำหนด
1. ไปที่การจัดการองค์กร > พื้นที่ทำงาน > การรายงานทางอิเล็กทรอนิกส์

    ในตัวอย่างนี้ บริษัทตัวอย่าง Litware, Inc. (https://www.litware.com) จะทำหน้าที่เป็นผู้ให้บริการการตั้งค่าคอนฟิก ซึ่งสนับสนุนการตั้งค่าคอนฟิกรูปแบบสำหรับการชำระเงินทางอิเล็กทรอนิกส์สำหรับประเทศเฉพาะ    บริษัทตัวอย่าง Proseware, Inc. (http://www.proseware.com) จะทำหน้าที่เป็นลูกค้าของการตั้งค่าคอนฟิกรูปแบบที่ Litware, Inc. จัดให้ Proseware, Inc. ใช้รูปแบบในภูมิภาคบางแห่งของประเทศนั้น  
2. คลิก การตั้งค่าคอนฟิกการรายงาน
3. คลิกแสดงตัวกรอง
4. ใช้ตัวกรองต่อไปนี้: ป้อนค่าตัวกรอง "BACS (แบบสมมติของ UK)" ในฟิลด์ "ชื่อ" โดยใช้ตัวดำเนินการตัวกรอง "เริ่มต้นด้วย"
  
    BACS (การกำหนดเองแบบสมมติของ UK) ของการตั้งค่าคอนฟิกรูปแบบที่เลือก เป็นเจ้าของโดยผู้ให้บริการ Litware, Inc.  

5. คลิกแสดงตัวกรอง
6. ในรายการนี้ ให้ค้นหาและเลือกเรกคอร์ดที่ต้องการ

    เวอร์ชันของรูปแบบที่มีสถานะเป็น 'เสร็จสมบูรณ์แล้ว' จะถูกใช้โดย Proseware, Inc. สำหรับการกำหนดเอง  

## <a name="create-a-new-configuration-for-your-custom-format-of-electronic-document"></a>สร้างการตั้งค่าคอนฟิกใหม่ สำหรับรูปแบบการชำระเงินที่กำหนดเองของเอกสารอิเล็กทรอนิกส์ของคุณ
Proseware, Inc. ได้รับการตั้งค่าคอนฟิก BACS (แบบสมมติของ UK) เวอร์ชัน 1.1 ที่ประกอบด้วยรูปแบบเริ่มต้น ในการสร้างเอกสารการชำระเงินทางอิเล็กทรอนิกส์จาก Litware, Inc. ที่สอดคล้องกับการสมัครใช้งานบริการ Proseware, Inc. ต้องการเริ่มต้นการใช้ให้เป็นพื้นฐานสำหรับประเทศของพวกเขา แต่การกำหนดค่าเองบางมีความจำเป็นในการสนับสนุนข้อกำหนดเฉพาะส่วนภูมิภาค Proseware, Inc. ยังต้องการเก็บความสามารถในการอัพเกรดรูปแบบที่กำหนดเอง ทันทีที่เวอร์ชันใหม่ออกมา (ซึ่งมีการเปลี่ยนแปลงเพื่อสนับสนุนข้อกำหนดเฉพาะประเทศ)จาก Litware, Inc. และพวกเขาต้องการที่จะดำเนินการอัพเกรดนี้ด้วยต้นทุนที่ต่ำที่สุด  

เพื่อดำเนินการสิ่งนี้ Proseware, Inc. ต้องสร้างการตั้งค่าคอนฟิกโดยใช้ BACS (แบบสมมติของ UK) ของการตั้งค่าคอนฟิก Litware, Inc. เป็นฐาน  

1. ปิดหน้า
2. เลือก Proseware, Inc. เพื่อทำให้เป็นผู้ให้บริการที่ใช้งานอยู่
3. คลิก กำหนดเป็นใช้งานอยู่
4. คลิก การตั้งค่าคอนฟิกการรายงาน
5. ในแผนภูมิ ขยาย 'การชำระเงิน (แบบจำลองอย่างง่าย)'
6. ในแผนภูมิ เลือก 'การชำระเงิน (แบบจำลองอย่างง่าย)\BACS (แบบสมมติ UK)'

    เลือกการตั้งค่าคอนฟิก BACS (แบบสมมติของ UK) จาก Litware, Inc. Proseware, Inc. จะใช้รุ่น 1.1 เป็นฐานสำหรับรุ่นที่กำหนดเอง  

7. คลิก สร้างการตั้งค่าคอนฟิก เพื่อเปิดกล่องโต้ตอบการวาง

    การดำเนินการนี้ช่วยให้คุณสามารถสร้างการตั้งค่าคอนฟิกใหม่สำหรับรูปแบบการชำระเงินที่กำหนดเอง  

8. ในฟิลด์ใหม่ ป้อน ' สืบทอดมาจากชื่อ: BACS (แบบสมมติ UK) Litware, Inc.'

    เลือกตัวเลือก ได้รับมา เพื่อยืนยันการใช้งานของ BACS (แบบสมมติ UK) เป็นฐานสำหรับการสร้างเวอร์ชันที่กำหนดเอง  

9. ในฟิลด์ชือ ให้พิมพ์ 'BACS (การกำหนดเองแบบสมมติของ UK)'
10. ในฟิลด์คำอธิบาย พิมพ์ 'รูปแบบการชำระเงินของผู้จัดจำหน่าย BACS (การกำหนดเองแบบสมมติของ UK)'

    ผู้ให้บริการการตั้งค่าคอนฟิกที่ใช้งานอยู่ (Proseware, Inc.) จะถูกป้อนไว้ที่นี่โดยอัตโนมัติ ผู้ให้บริการนี้จะสามารถรักษาการตั้งค่าคอนฟิกนี้ ผู้ให้บริการอื่นๆสามารถใช้การตั้งค่าคอนฟิกนี้ แต่จะไม่สามารถรักษาค่าคอนฟิกได้  

11. คลิก สร้างการตั้งค่าคอนฟิก

## <a name="validate-the-customized-format"></a>ตรวจสอบความถูกต้องของรูปแบบที่กำหนดเอง
1. คลิก ตรวจสอบความถูกต้อง

    ตรวจสอบโครงร่างรูปแบบที่กำหนดเอง และการเปลี้ยนแปลงการแม็ปของข้อมูล เพื่อให้แน่ใจว่าการผูกข้อมูลทั้งหมดถูกต้อง  

2. ปิดหน้า

## <a name="change-the-status-of-the-current-version-of-the-custom-format-configuration"></a>เปลี่ยนสถานะของเวอร์ชันปัจจุบันของการตั้งค่าคอนฟิกรูปแบบที่กำหนดเอง
เปลี่ยนสถานะของการตั้งค่าคอนฟิกรูปแบบที่ออกแบบแล้วจากแบบร่างเป็นแบบที่เสร็จสมบูรณ์ เพื่อทำให้พร้อมใช้งานสำหรับการตั้งค่าคอนฟิกเอกสารการชำระเงิน  

1. คลิก เปลี่ยนแปลงสถานะ

    หมายเหตุว่าเวอร์ชันปัจจุบันของการตั้งค่าคอนฟิกที่เลือกอยู่ในสถานะแบบร่าง  

2. คลิกเสร็จสมบูรณ์
3. ในฟิลด์ คำอธิบาย ให้พิมพ์ค่า
4. คลิก ตกลง
5. ในรายการนี้ ให้ค้นหาและเลือกเรกคอร์ดที่ต้องการ

    หมายเหตุว่า การตั้งค่าคอนฟิกที่สร้างไว้ถูกบันทึกเป็นเวอร์ชัน 1.1.1 ที่สมบูรณ์ ซึ่งหมายความว่าเป็นเวอร์ชัน 1 ของรูปแบบ BACS (การกำหนดเองแบบสมมติของ UK) ที่กำหนดเอง ซึ่งเป็นไปตามเวอร์ชัน 1 ของรูปแบบ BACS (แบบสมมติ UK) ซึ่งเป็นไปตามเวอร์ชัน 1 ของแบบจำลองข้อมูลการชำระเงิน (แบบจำลองอย่างง่าย)  

## <a name="test-the-customized-format-to-generate-payment-files"></a>ทดสอบรูปแบบที่กำหนดเองเพื่อสร้างไฟล์การชำระเงิน
ดำเนินการขั้นตอนให้เสร็จสมบูรณ์ในกระบวนงาน "ใช้รูปแบบที่สร้างไว้เพื่อสร้างเอกสารอิเล็กทรอนิกส์สำหรับการชำระเงิน" ในรอบเวลา Finance and Operations แบบขนาน เลือกรูปแบบ BACS (การกำหนดเองแบบสมมติของ UK) ในพารามิเตอร์วิธีการชำระเงินทางอิเล็กทรอนิกส์ ตรวจสอบให้แน่ใจว่าไฟล์การชำระเงินที่สร้างประกอบด้วย โหนด XML ที่นำมาใช้ล่าสุด ซึ่งแสดงรหัส IBAN ที่สอดคล้องกับความต้องการทางภูมิภาค  

## <a name="update-the-existing-country-specific-configuration"></a>อัพเดตการตั้งค่าคอนฟิกเฉพาะประเทศที่มีอยู่
Litware, Inc. ต้องอัพเดตการตั้งค่าคอนฟิก BACS (แบบสมมติ UK) และนำข้อกำหนดใหม่ของประเทศมาใช้สำหรับการจัดการรูปแบบของเอกสารทางอิเล็กทรอนิกส์ ในภายหลัง นี่จะถูกแนบในเวอร์ชันใหม่ของการตั้งค่าคอนฟิกซึ่งจะถูกเสนอสำหรับผู้ติดตามบริการ ซึ่งรวมกึง Proseware, Inc.  

ในการเตรียมใช้งานบริการจริงที่เกี่ยวข้องกับกระบวนการ รุ่นใหม่ของ BACS (แบบสมมติของ UK) แต่ละรุ่นสามารถถูกนำเข้าได้โดย Proseware, Inc. จากที่เก็บ LCS ของการตั้งค่าคอนฟิกของ Litware, Inc. ในกระบวนงานนี้ เราจะจำลองโดยการอัพเดต BACS (แบบสมมติ UK) ในนามของผู้ให้บริการ  

1. ปิดหน้า
2. เลือก Litware, Inc. ผู้ให้บริการ
3. คลิก กำหนดเป็นใช้งานอยู่
4. คลิก การตั้งค่าคอนฟิกการรายงาน
5. ในแผนภูมิ ขยาย 'การชำระเงิน (แบบจำลองอย่างง่าย)'
6. ในแผนภูมิ เลือก 'การชำระเงิน (แบบจำลองอย่างง่าย)\BACS (แบบสมมติ UK)'

    เวอร์ชันแบบร่างที่ครอบครองโดย BACS (แบบสมมติ UK) ของผู้ให้บริการ Litware, Inc. จะถูกเลือกเพื่อนำการเปลี่ยนแปลงไปสนับสนุนข้อกำหนดเฉพาะประเทศใหม่

## <a name="change-the-status-of-the-current-version-of-the-base-format-configuration"></a>เปลี่ยนสถานะของเวอร์ชันปัจจุบันของการตั้งค่าคอนฟิกรูปแบบพื้นฐาน
เปลี่ยนสถานะของการตั้งค่าคอนฟิกรูปแบบฐานที่อัพเดต จากแบบร่างเป็นแบบที่เสร็จสมบูรณ์ เพื่อทำให้พร้อมใช้งานสำหรับการสร้างเอกสารการชำระเงิน และอัพเดตการตั้งค่าคอนฟิกรูปแบบที่ได้มา  

1. คลิก เปลี่ยนแปลงสถานะ

    หมายเหตุว่าเวอร์ชันปัจจุบันของการตั้งค่าคอนฟิกที่เลือกอยู่ในสถานะแบบร่าง  

2. คลิกเสร็จสมบูรณ์
3. ในฟิลด์ คำอธิบาย ให้พิมพ์ค่า
4. คลิก ตกลง
5. ในรายการนี้ ให้ค้นหาและเลือกเรกคอร์ดที่ต้องการ

## <a name="change-the-base-version-for-the-custom-format-configuration"></a>เปลี่ยนเวอร์ชันฐานสำหรับการตั้งค่าคอนฟิกรูปแบบที่กำหนดเอง

 Proseware, Inc. ได้รับรายงานว่า เวอร์ชันใหม่ 1.2 ของการตั้งค่าคอนฟิก BACS (แบบสมมติ UK) พร้อมใช้งานในการสร้างเอกสารการชำระเงินอิเล็กทรอนิกส์ ที่สอดคล้องกับข้อกำหนดเฉพาะประเทศที่เพิ่งได้ประกาศมาไม่นานนี้  Proseware, Inc. ต้องการเริ่มต้นการใช้ให้เป็นพื้นฐานสำหรับประเทศ  

เพื่อดำเนินการนี้ Proseware, Inc. ต้องการเปลี่ยนแปลงเวอร์ชันการตั้งค่าคอนฟิกพื้นฐานสำหรับ BACS (การกำหนดเองแบบสมมติของ UK) ของการตั้งค่าคอนฟิกที่กำหนดเอง แทนที่เวอร์ชัน 1.1 ของ BACS (แบบสมมติ UK) ใช้เวอร์ชันใหม่ 1.2  

1. ไปที่การจัดการองค์กร > พื้นที่ทำงาน > การรายงานทางอิเล็กทรอนิกส์
2. เลือก ผู้ให้บริการ Proseware, Inc. เพื่อทำเครื่องหมายเป็น ใช้งานอยู่
3. คลิก กำหนดเป็นใช้งานอยู่
4. คลิก การตั้งค่าคอนฟิกการรายงาน
5. ในแผนภูมิ ขยาย 'การชำระเงิน (แบบจำลองอย่างง่าย)'
6. ในแผนภูมิ ขยาย 'การชำระเงิน (แบบจำลองอย่างง่าย)\BACS (แบบสมมติ UK)'
7. ในแผนภูมิ เลือก 'การชำระเงิน (แบบจำลองอย่างง่าย)\BACS (แบบสมมติ UK)\BACS (การกำหนดเองแบบสมมติของ UK)'

    เลือกการตั้งค่าคอนฟิก BACS (การกำหนดเองแบบสมมติของ UK) ซึ่งเป็นเจ้าของโดย Proseware, Inc.  

    ใช้เวอร์ชันแบบร่างของการตั้งค่าคอนฟิกที่เลือก เพื่อนำการเปลี่ยนแปลงที่จำเป็นไปใช้  

8. คลิกปรับใช้ซ้ำ

    เลือกเวอร์ชันใหม่ 1.2 เป็นการตั้งค่าคอนฟิกพื้นฐาน เพื่อนำไปใช้เป็นฐานใหม่สำหรับการอัพเดตการตั้งค่าคอนฟิก  

9. คลิก ตกลง

    หมายเหตุว่า ความขัดแย้งบางอย่างได้ถูกพบระหว่างการผสานรุ่นที่กำหนดเองและรุ่นฐานใหม่ ซึ่งแสดงการเปลี่ยนแปลงรูปแบบบางอย่างที่ไม่สามารถผสานได้โดยอัตโนมัติ  

## <a name="resolve-rebase-conflicts"></a>แก้ไขความขัดแย้งของการปรับใช้ซ้ำ
1. คลิก ตัวออกแบบ
    
    หมายเหตุว่า การเปลี่ยนแปลงไปยังขีดจำกัดความยาวข้อความของชื่อผู้จัดจำหน่ายไม่สามารถถูกแก้ไขได้โดยอัตโนมัติ ดังนั้น จะถูกแสดงในรายการความขัดแย้ง สำหรับความขัดแย้งของชนิดการอัพเดตแต่ละชนิด ตัวเลือกต่อไปนี้จะพร้อมใช้งาน: - ใช้ค่าฐานก่อนหน้า (ปุ่มบนสุดของกริด) เพื่อนำค่าเวอร์ชันฐานก่อนหน้ามาใช้ (0 ในกรณีนี้)  - ใช้ค่าฐานก่อนหน้า (ปุ่มบนสุดของกริด) เพื่อนำค่าเวอร์ชันฐานใหม่มา (100 ในกรณีนี้)  - คงค่า (ที่กำหนดเอง) ของคุณ (60 ในกรณีนี้)  คลิก ใช้งานค่าฐาน เพื่อใช้งานขีดจำกัดเฉพาะประเทศที่เป็น 100 อักขระ สำหรับความยาวข้อความของชื่อผู้จัดจำหน่าย  

    โปรดทราบว่า Proseware, Inc. และ Litware, Inc. มีเวอร์ชันที่กำหนดเองเฉพาะของรูปแบบนี้ โดยใช้รหัส IBAN และ SWIFT ที่มีส่วนประกอบที่สัมพันธ์กัน ซึ่งถูกผสานโดยอัตโนมัติในรูปแบบการจัดการ  

2. คลิกใช้ค่าฐาน

    คลิกใช้ค่าฐาน เพื่อใช้งานขีดจำกัดเฉพาะประเทศเป็น 100 อักขระสำหรับชื่อผู้จัดจำหน่าย  

3. คลิก บันทึก

    การบันทึกรูปแบบจะลบความขัดแย้งที่แก้ไขแล้วจากรายการความขัดแย้ง  

4. ปิดหน้า

## <a name="change-the-status-of-the-new-version-of-the-custom-format-configuration"></a>เปลี่ยนสถานะของเวอร์ชันใหม่ของการตั้งค่าคอนฟิกรูปแบบที่กำหนดเอง
1. คลิก เปลี่ยนแปลงสถานะ

    เปลี่ยนสถานะจากการตั้งค่าคอนฟิกรูปแบบด้วยตนเองที่อัพเดตแล้วจากแบบร่างเป็นเสร็จสมบูรณ์แล้ว การดำเนินการนี้จะทำให้การตั้งค่าคอนฟิกรูปแบบพร้อมใช้งานสำหรับการสร้างเอกสารการชำระเงิน หมายเหตุว่าเวอร์ชันปัจจุบันของการตั้งค่าคอนฟิกที่เลือกอยู่ในสถานะแบบร่าง  

2. คลิกเสร็จสมบูรณ์
3. ในฟิลด์ คำอธิบาย ให้พิมพ์ค่า
4. คลิก ตกลง

    หมายเหตุว่า การตั้งค่าคอนฟิกที่สร้างไว้ถูกบันทึกเป็นเวอร์ชัน 1.2.2 ที่สมบูรณ์: เวอร์ชัน 2 ของรูปแบบ BACS (การกำหนดเองแบบสมมติของ UK) พื้นฐาน ซึ่งเป็นไปตามเวอร์ชัน 2 ของรูปแบบ BACS (แบบสมมติ UK) พื้นฐาน ซึ่งเป็นไปตามเวอร์ชัน 1 ของแบบจำลองข้อมูลการชำระเงิน (แบบจำลองอย่างง่าย)  

## <a name="test-the-customized-format-for-payment-files-generation"></a>ทดสอบรูปแบบที่กำหนดเองสำหรับการสร้างไฟล์การชำระเงิน
ดำเนินการขั้นตอนให้เสร็จสมบูรณ์ในกระบวนงาน "ใช้รูปแบบที่สร้างไว้เพื่อสร้างเอกสารอิเล็กทรอนิกส์สำหรับการชำระเงิน" ในรอบเวลา Finance and Operations แบบขนาน เลือกรูปแบบ 'BACS (การกำหนดเองแบบสมมติของ UK)' ในพารามิเตอร์วิธีการชำระเงินทางอิเล็กทรอนิกส์ ตรวจสอบให้แน่ใจว่าไฟล์การชำระเงินที่สร้างประกอบด้วยข้อมูลที่นำมาใช้ล่าสุดโดยโหนด XML ของ Proseware, Inc. ซึ่งแสดงรหัส IBAN ที่สอดคล้องกับความต้องการทางภูมิภาค ไฟล์นี้ยังควรประกอบด้วยข้อมูลที่นำมาใช้ล่าสุดโดยโหนด XML ของ Litware, Inc. ซึ่งแสดงรหัส SWIFT ที่สอดคล้องกับความต้องการของประเทศ
