# <a name="automate-testing-with-electronic-reporting"></a>ดำเนินการทดสอบอัตโนมัติกับการรายงานทางอิเล็กทรอนิกส์

[!include[banner](../includes/banner.md)]

หัวข้อนี้จะอธิบายถึงวิธีการที่คุณสามารถใช้กรอบงานการรายงานทางอิเล็กทรอนิกส์ (ER) เพื่อทำให้การทดสอบของบางฟังก์ชันเป็นแบบอัตโนมัติ ตัวอย่างในหัวข้อนี้จะแสดงวิธีการทำให้การทดสอบของการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่ายเป็นแบบอัตโนมัติ

แอพลิเคชันใช้กรอบงาน ER เพื่อสร้างไฟล์การชำระเงินและเอกสารที่เกี่ยวข้องในระหว่างการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย กรอบงาน ER ประกอบด้วย แบบจำลองข้อมูล การแม็ปแบบจำลอง และส่วนประกอบรูปแบบที่สนับสนุนการประมวลผลการชำระเงินสำหรับชนิดการชำระเงินที่แตกต่างกัน และการสร้างเอกสารในรูปแบบที่แตกต่างกัน สามารถดาวน์โหลดส่วนประกอบเหล่านี้ได้จาก Microsoft Dynamics Lifecycle Services (LCS) และสามารถนำเข้าไปยังอินสแตนซ์ได้

นอกจากนี้ คุณยังสามารถปรับแต่งแต่ละองค์ประกอบของ Microsoft และใช้เป็นพื้นฐานของส่วนประกอบที่กำหนดเองของคุณได้ด้วย โดยการสร้างรุ่นที่กำหนดเอง คุณสามารถทำการเปลี่ยนแปลงที่สนับสนุนความต้องการที่เฉพาะเจาะจงได้ ตัวอย่างเช่น คุณสามารถปรับปรุงแบบจำลองข้อมูล ER และการแม็ปแบบจำลอง ER เพื่อเข้าถึงข้อมูลแอพลิเคชันเฉพาะลูกค้า หรือคุณสามารถเปลี่ยนรูปแบบ ER เพื่อแก้ไขโครงร่างของเอกสารที่สร้างขึ้น

คุณสามารถใช้รูปแบบ ER แบบกำหนดเองในอินสแตนซ์ เพื่อประมวลผลไฟล์การชำระเงินที่สร้างการชำระเงินให้แก่ผู้จัดจำหน่าย และเพื่อประมวลผลรายงานการควบคุมด้วย การกำหนดรุ่นได้รับการสนับสนุนในส่วนประกอบ ER ดังนั้น Microsoft จึงสามารถให้โซลูชัน ER รุ่นที่ปรับปรุงสำหรับการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย และคุณสามารถผสานรุ่นที่ปรับปรุงโดยอัตโนมัติกับส่วนประกอบที่กำหนดเองของคุณได้โดยการปรับใช้ซ้ำ อย่างไรก็ตาม คุณต้องทดสอบรุ่นที่ปรับใช้ซ้ำเพื่อตรวจสอบให้แน่ใจว่าจะทำงานได้ตามที่คุณคาดไว้

แบบจำลองข้อมูล ER และการแม็ปการแบบจำลอง ER ใช้ร่วมกันสำหรับรูปแบบ ER หลายแบบที่ใช้ในการประมวลผลการชำระเงินชนิดต่างๆ และเพื่อสร้างเอกสารการชำระเงินเฉพาะประเทศ/ภูมิภาค ดังนั้น จึงน่าพึงพอใจอย่างมากที่จะทำให้การทดสอบการยอมรับและการรวมของผู้ใช้เป็นแบบอัตโนมัติ เพื่อให้สามารถดำเนินการได้โดยอัตโนมัติในบริษัทหลากหลายแห่ง แต่คำนึงถึงบริบทประเทศ/ภูมิภาคของบริษัทปลายทางแต่ละแห่ง ใช้ชุดข้อมูลที่แตกต่างกัน และอื่นๆ

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการสร้างรุ่นที่กำหนดเองของรูปแบบที่ขึ้นอยู่กับรูปแบบที่คุณได้รับจากผู้ให้บริการการตั้งค่าคอนฟิก ดู [ER ปรับรุ่นรูปแบบของคุณโดยปรับใช้รุ่นพื้นฐานใหม่ของรูปแบบนั้น](./tasks/er-upgrade-format.md)

## <a name="key-concepts"></a>แนวคิดหลัก

ผู้ใช้ระดับสูงด้านการใช้งานสามารถสร้างการทดสอบการยอมรับและการรวมของผู้ใช้ได้ โดยไม่ต้องเขียนรหัสต้นทาง

- ใช้คุณลักษณะพื้นฐาน ER เพื่อเปรียบเทียบเอกสารที่สร้างขึ้นกับสำเนาหลัก สำหรับข้อมูลเพิ่มเติม ดู [ติดตามผลรายงานที่สร้างขึ้น และเปรียบเทียบกับค่าพื้นฐาน](er-trace-reports-compare-baseline.md)
- ใช้ตัวบันทึกงานเพื่อบันทึกกรณีทดสอบ และรวมการประเมินพื้นฐาน สำหรับข้อมูลเพิ่มเติม ดูที่ [ทรัพยากรตัวบันทึกงาน](../user-interface/task-recorder.md).
- กรณีการทดสอบกลุ่มสำหรับสถานการณ์จำลองการทดสอบที่จำเป็น สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [สร้างและดำเนินการทดสอบการยอมรับของผู้ใช้อัตโนมัติ](../lifecycle-services/using-task-guides-and-bpm-to-create-user-acceptance-tests.md)

    - ใช้ตัวทำแบบจำลองกระบวนการทางธุรกิจ (BPM) ใน LCS เพื่อสร้างไลบรารีสำหรับการทดสอบการยอมรับของผู้ใช้
    - ใช้ไลบรารีสำหรับการทดสอบ BPM เพื่อสร้างแผนการทดสอบ และทดสอบชุดใน Microsoft Azure DevOps Services (Azure DevOps)

ผู้ใช้ระดับสูงด้านการใช้งานสามารถรันการทดสอบการยอมรับและการรวมของผู้ใช้

- ใช้ Regression Suite Automation Tool (RSAT) เพื่อรันกรณีทดสอบของชุดทดสอบที่ต้องการ
- รายงานผลลัพธ์ของการทดสอบไปยัง Azure DevOps และใช้บริการนี้เพื่อตรวจสอบผลลัพธ์ดังกล่าว

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ก่อนที่คุณจะสามารถทำงานให้เสร็จสมบูรณ์ในหัวข้อนี้ คุณต้องดำเนินการข้อกำหนดเบื้องต้นต่อไปนี้ให้เสร็จสมบูรณ์:

- ปรับใช้โทโพโลยีที่สนับสนุนการทดสอบแบบระบบอัตโนมัติ คุณต้องมีสิทธิ์เข้าถึงอินสแตนซ์ของโทโพโลยีนี้สำหรับบทบาท **ผู้ดูแลระบบ** โทโพโลยีนี้ต้องมีข้อมูลสาธิตซึ่งจะใช้ในตัวอย่างนี้ สำหรับข้อมูลเพิ่มเติม ดูที่ [ปรับใช้และใช้สภาพแวดล้อมที่สนับสนุนการสร้างอย่างต่อเนื่องและระบบอัตโนมัติของการทดสอบ](../perf-test/continuous-build-test-automation.md)
- เมื่อต้องการรันการทดสอบการยอมรับและการรวมของผู้ใช้โดยอัตโนมัติ คุณต้องติดตั้ง RSAT ในโทโพโลยีที่คุณกำลังใช้และตั้งค่าคอนฟิกในลักษณะที่เหมาะสม สำหรับข้อมูลเกี่ยวกับวิธีการติดตั้งและตั้งค่า RSAT และตั้งค่าคอนฟิกให้ทำงานกับแอป Finance and Operations และ Azure DevOps ดูที่ [Regression Suite Automation Tool](https://www.microsoft.com/download/details.aspx?id=57357) ให้ความสำคัญกับข้อกำหนดเบื้องต้นสำหรับการใช้เครื่องมือ ภาพประกอบต่อไปนี้แสดงตัวอย่างของการตั้งค่า RSAT สี่เหลี่ยมสีน้ำเงินล้อมรอบพารามิเตอร์ที่ระบุสิทธิ์การเข้าถึง Azure DevOps สี่เหลี่ยมสีเขียวล้อมรอบพารามิเตอร์ที่ระบุการเข้าถึงอินสแตนซ์

    ![การตั้งค่า RSAT](media/GER-Configure.png "ภาพหน้าจอของกล่องโต้ตอบการตั้งค่า RSAT")

- เมื่อต้องการจัดระเบียบกรณีทดสอบในชุดเพื่อช่วยรับประกันลำดับการดำเนินการที่ถูกต้อง เพื่อให้คุณสามารถรวบรวมล็อกของการดำเนินการทดสอบสำหรับการรายงานและการตรวจสอบเพิ่มเติม คุณต้องมีสิทธิเข้าถึง Azure DevOps จากโทโพโลยีที่ปรับใช้
- เมื่อต้องการดำเนินการตัวอย่างในหัวข้อนี้ให้เสร็จสมบูรณ์ เราขอแนะนำให้คุณดาวน์โหลด [การใช้ ER สำหรับการทดสอบ RSAT](https://go.microsoft.com/fwlink/?linkid=874684) ไฟล์ zip นี้มีคำแนะนำเกี่ยวกับงานต่อไปนี้:

    | ปริมาณความจุ                                           | ชื่อไฟล์และสถานที่ |
    |---------------------------------------------------|------------------------|
    | การบันทึกงานตัวอย่างเพื่อจัดเตรียมข้อมูลสำหรับการทดสอบ | จัดเตรียม\\Recording.xml |
    | การบันทึกงานตัวอย่างเพื่อประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย   | ประมวลผล\\Recording.xml |

## <a name="prepare-the-accounts-payable-module-to-process-vendor-payments"></a>จัดเตรียมโมดูลบัญชีเจ้าหนี้เพื่อประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย

1. ลงชื่อเข้าใช้อินสแตนซ์ของคุณ
2. ดาวน์โหลดการตั้งค่าคอนฟิก ER ต่อไปนี้จาก LCS สำหรับคำแนะนำ ดู [ER นำเข้าการตั้งค่าคอนฟิกจาก Lifecycle Services](./tasks/er-import-configuration-lifecycle-services.md)

    - การตั้งค่าคอนฟิกแบบจำลอง ER ของ **แบบจำลองการชำระเงิน**
    - การตั้งค่าคอนฟิกการแม็ปแบบจำลอง ER ของ **การแม็ปแบบจำลองการชำระเงิน 1611**
    - **BACS (UK)** การตั้งค่าคอนฟิกรูปแบบ ER

    ![การตั้งค่าคอนฟิกการรายงานทางอิเล็กทรอนิกส์](media/GER-Configurations.png "ภาพหน้าจอของหน้าการตั้งค่าคอนฟิกในการรายงานทางอิเล็กทรอนิกส์")

3. เลือกบริษัทข้อมูลสาธิต **GBSI** ซึ่งมีบริบทประเทศ/ภูมิภาคในสหราชอาณาจักร
4. ตั้งค่าคอนฟิกพารามิเตอร์บัญชีเจ้าหนี้:

    1. ไปที่ **บัญชีเจ้าหนี้ \> การตั้งค่าการชำระเงิน \> วิธีการชำระเงิน**
    2. เลือกวิธีการชำระเงิน **อิเล็กทรอนิกส์**
    3. ตั้งค่าคอนฟิกวิธีการชำระเงินที่เลือกเพื่อให้ใช้รูปแบบ ER ของ **BACS (UK)** ที่คุณดาวน์โหลดก่อนหน้านี้ สำหรับการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย:

        1. บน FastTab **รูปแบบไฟล์** ให้ตั้งค่าตัวเลือก **รูปแบบการส่งออกทางอิเล็กทรอนิกส์ทั่วไป** เป็น **ใช่**
        2. ในฟิลด์ **ส่งออกการตั้งค่าคอนฟิกรูปแบบ** ให้เลือก **BACS (UK)**

    ![หน้าวิธีการชำระเงิน](media/GER-APParameters.png "ภาพหน้าจอของหน้าวิธีการชำระเงิน")

    > [!NOTE]
    > ถ้าคุณมีรุ่นที่ได้รับของรูปแบบ ER นี้ที่ถูกสร้างขึ้นเพื่อสนับสนุนการเลือกกำหนด คุณสามารถเลือกการตั้งค่าคอนฟิกนี้ได้ในวิธีการชำระเงิน **อิเล็กทรอนิกส์**

5. สร้างตัวอย่างการชำระเงินให้แก่ผู้จัดจำหน่าย:

    1. ไปที่ **บัญชีเจ้าหนี้ \> การชำระเงิน \> สมุดรายวันการชำระเงิน**
    2. ตรวจสอบให้แน่ใจว่าคุณยังไม่ได้ลงรายการบัญชีสมุดรายวันการชำระเงิน

        ![หน้าสมุดรายวันการชำระเงิน](media/GER-APJournal.png "ภาพหน้าจอของหน้าสมุดรายวันการชำระเงิน")

    3. เลือก **รายการ** และป้อนรายการที่มีข้อมูลต่อไปนี้

        | ฟิลด์               | มูลค่าตัวอย่าง   |
        |---------------------|-----------------|
        | ชื่อผู้จัดจำหน่าย         | GB\_SI\_000001  |
        | เดบิต               | 1,000.00        |
        | สกุลเงิน            | GBP             |
        | ชนิดของบัญชีตรงข้าม | ธนาคาร            |
        | บัญชีตรงข้าม      | GBSI OPER       |
        | วิธีการชำระเงิน   | อิเล็กทรอนิกส์      |

    ![หน้าการชำระเงินให้แก่ผู้จัดจำหน่าย](media/GER-APJournalLines.png "ภาพหน้าจอของหน้าการชำระเงินให้แก่ผู้จัดจำหน่าย")

## <a name="prepare-the-er-framework-to-test-vendor-payment-processing"></a>จัดเตรียมกรอบงาน ER เพื่อทดสอบการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย

### <a name="configure-er-parameters"></a>ตั้งค่าคอนฟิกพารามิเตอร์ ER

1. ไปที่ **การจัดการองค์กร \> การรายงานทางอิเล็กทรอนิกส์ \> พารามิเตอร์การรายงานทางอิเล็กทรอนิกส์**
2. บนแท็บ **เอกสารแนบ** ในฟิลด์ **พื้นฐาน** เลือก **ไฟล์** เป็นชนิดเอกสารที่กรอบงานการจัดการเอกสาร (DM) ใช้เพื่อเก็บเอกสารที่เกี่ยวข้องกับคุณลักษณะพื้นฐานเป็นเอกสารแนบ DM

    ![หน้าพารามิเตอร์การรายงานทางอิเล็กทรอนิกส์](media/GER-ERParameters.png "ภาพหน้าจอของหน้าพารามิเตอร์ของการรายงานทางอิเล็กทรอนิกส์")

### <a name="generate-baseline-copies-of-vendor-paymentrelated-documents"></a>สร้างสำเนาพื้นฐานของเอกสารที่เกี่ยวข้องกับการชำระเงินให้แก่ผู้จัดจำหน่าย

1. ไปที่ **บัญชีเจ้าหนี้ \> การชำระเงิน \> สมุดรายวันการชำระเงิน**
2. เลือก **รายการ**
3. เลือก **สร้างการชำระเงิน**
4. เลือกวิธีการชำระเงิน **อิเล็กทรอนิกส์**
5. เลือกบัญชีธนาคาร **GBSI OPER**
6. ตั้งค่าตัวเลือก **พิมพ์รายงานควบคุม** เป็น **ใช่**
7. ดาวน์โหลดเอาท์พุทที่สร้างเป็นไฟล์ zip
8. เปิดไฟล์ที่ดาวน์โหลด
9. แยกไฟล์ต่อไปนี้จากไฟล์ที่ดาวน์โหลด

    - ไฟล์การชำระเงินของ **ไฟล์** ในรูปแบบข้อความ
    - ไฟล์รายงานการควบคุม **ERVendOutPaymControlReport** ในรูปแบบ XLSX

    ![ไฟล์ที่แยก](media/GER-APJournalProcessed.png "ภาพหน้าจอของชื่อไฟล์ที่แยกไว้ใน Windows explorer")

### <a name="turn-on-the-er-baseline-feature"></a>เปิดใช้งานคุณลักษณะพื้นฐาน ER

1. ไปที่ **การจัดการองค์กร \> การรายงานทางอิเล็กทรอนิกส์ \> การตั้งค่าคอนฟิก**
2. บนบานหน้าต่างการดำเนินการ บนแท็บ **การตั้งค่าคอนฟิก** เลือก **พารามิเตอร์ผู้ใช้**
3. ตั้งค่าตัวเลือก **รันในโหมดดีบัก** เป็น **ใช่**

โดยการเปิดพารามิเตอร์ **รันในโหมดดีบัก** คุณบังคับให้กรอบงาน ER ทำการดำเนินการต่อไปนี้ หลังจากการดำเนินการของรูปแบบ ER ใดๆ ที่สร้างเอกสารขาออก:

1. กำหนดว่ามีการตั้งค่าคอนฟิกข้อมูลพื้นฐานสำหรับส่วนประกอบใดๆ ของรูปแบบ ER ที่ดำเนินการหรือไม่
2. กำหนดว่าข้อมูลพื้นฐานที่ตั้งค่าคอนฟิกแต่ละรายการสามารถใช้ได้ในเงื่อนไขปัจจุบัน (รหัสบริษัทของบริษัทที่ลงชื่อเข้าใช้ ชื่อไฟล์ และนามสกุลของชื่อไฟล์ของเอาท์พุทที่สร้างขึ้น และอื่นๆ)
3. สำหรับข้อมูลพื้นฐานที่ใช้ได้แต่ละรายการ ให้ดำเนินการต่อไปนี้

    1. เปรียบเทียบเอาท์พุทที่สร้างขึ้นในระหว่างการดำเนินการของรูปแบบ ER กับข้อมูลพื้นฐานที่สอดคล้องกัน
    2. จัดเก็บผลลัพธ์ของการเปรียบเทียบในล็อกดีบักการตั้งค่าคอนฟิก ER

### <a name="configure-er-baselines-for-vendor-payment-processing"></a>ตั้งค่าคอนฟิกข้อมูลพื้นฐาน ER สำหรับการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย

1. ไปที่ **การจัดการองค์กร \> การรายงานทางอิเล็กทรอนิกส์ \> การตั้งค่าคอนฟิก**
2. เลือก **พื้นฐาน**
3. เลือก **ใหม่**
4. ใน **การอ้างอิง** ให้เลือกรูปแบบ **BACS (UK)**
5. เลือก **เอกสารแนบ**
6. เพิ่มข้อมูลพื้นฐานใหม่สำหรับไฟล์การชำระเงินของผู้จัดจำหน่าย:

    1. เลือก **ใหม่**
    2. ในฟิลด์ **ชนิด** เลือกชนิดเอกสาร DM ของ **ไฟล์** ที่คุณตั้งค่าคอนฟิกไว้ในพารามิเตอร์ ER เพื่อจัดเก็บสิ่งประดิษฐ์พื้นฐาน
    3. เรียกดูเพื่อเลือกไฟล์การชำระเงินของ **ไฟล์** ที่บันทึกไว้เฉพาะที่ในรูปแบบข้อความ
    4. ในฟิลด์ **คำอธิบาย** ป้อน **ไฟล์ TXT ของการชำระเงิน**

7. เพิ่มข้อมูลพื้นฐานใหม่สำหรับรายงานการควบคุมสำหรับการชำระเงินให้แก่ผู้จัดจำหน่าย:

    1. เลือก **ใหม่**
    2. ในฟิลด์ **ชนิด** เลือกชนิดเอกสาร DM ของ **ไฟล์** ที่คุณตั้งค่าคอนฟิกไว้ในพารามิเตอร์ ER เพื่อจัดเก็บสิ่งประดิษฐ์พื้นฐาน
    3. เรียกดูเพื่อเลือกไฟล์รายงานการควบคุม **ERVendOutPaymControlReport** ที่บันทึกไว้เฉพาะที่ ในรูปแบบ XLSX
    4. ในฟิลด์ **คำอธิบาย** ป้อน **รายงานการควบคุม XLSX ของการชำระเงิน**

    ![ข้อมูลพื้นฐานสำหรับไฟล์และรายงานการควบคุมสำหรับการชำระเงินให้แก่ผู้จัดจำหน่าย](media/GER-BaselineAttachments.png "ภาพหน้าจอของหน้าการตั้งค่าคอนฟิกที่มีรายงานการควบคุมการชำระเงิน XLSX ที่เลือก")

8. ปิดหน้า
9. บน FastTab **พื้นฐาน** ให้เลือก **สร้าง** เพื่อตั้งค่าคอนฟิกพื้นฐานสำหรับไฟล์การชำระเงิน:

    1. ตั้งชื่อรายการ **การตั้งค่าพื้นฐานสำหรับไฟล์การชำระเงิน**
    2. ในฟิลด์ **ชื่อส่วนประกอบไฟล์** ให้เลือก **ไฟล์** ที่จะใช้ข้อมูลพื้นฐานนี้กับเอาท์พุทของรูปแบบ ER ที่สร้างไฟล์การชำระเงินในรูปแบบข้อความ BACS (UK)
    3. ในฟิลด์ **บริษัท** ให้เลือก **GBSI** เพื่อใช้ข้อมูลพื้นฐานนี้ เมื่อมีการรันรูปแบบ ER ของ **BACS (UK)** ในบริษัท GBSI
    4. ในฟิลด์ **ตัวพรางชื่อไฟล์** ป้อน **\*.TXT** เพื่อใช้ข้อมูลพื้นฐานนี้เฉพาะกับเอาท์พุทของชื่อไฟล์ขององค์ประกอบรูปแบบชื่อไฟล์ **ไฟล์** ที่มีส่วนขยายนามสกุลของ **. txt**
    5. ในฟิลด์ **ข้อมูลพื้นฐาน** เลือก **ไฟล์ของ TXT การชำระเงิน** เพื่อให้พื้นฐานนี้ใช้สำหรับการเปรียบเทียบกับผลผลิตที่สร้าง

10. เลือก **สร้าง** เพื่อตั้งค่าคอนฟิกพื้นฐานสำหรับรายงานการควบคุม:

    1. ตั้งชื่อรายการ **การตั้งค่าพื้นฐานสำหรับรายงานการควบคุม**
    2. ในฟิลด์ **ชื่อส่วนประกอบไฟล์** ให้เลือก **ERVendOutPaymControlReport** ที่จะใช้ข้อมูลพื้นฐานนี้กับเอาท์พุทของรูปแบบ ER ที่สร้างไฟล์รายงานการควบคุม
    3. ในฟิลด์ **บริษัท** ให้เลือก **GBSI** เพื่อใช้ข้อมูลพื้นฐานนี้ เมื่อมีการรันรูปแบบ ER ของ **BACS (UK)** ในบริษัท GBSI
    4. ในฟิลด์ **ตัวพรางชื่อไฟล์** ป้อน **\*.XLSX** เพื่อใช้ข้อมูลพื้นฐานนี้เฉพาะกับเอาท์พุทขององค์ประกอบรูปแบบ **ERVendOutPaymControlReport** ที่มีส่วนขยายนามสกุลของ **.xslx**
    5. ในฟิลด์ **ข้อมูลพื้นฐาน** เลือก **รายงานการควบคุมใน XLSX ของการชำระเงิน** เพื่อให้พื้นฐานนี้ถูกนี้ใช้สำหรับการเปรียบเทียบกับผลผลิตที่สร้าง

    ![ข้อมูลพื้นฐานแท็บด่วนบนหน้าการตั้งค่าคอนฟิก](media/GER-BaselineRules.png "ภาพหน้าจอของแท็บด่วนข้อมูลพื้นฐานบนหน้าการตั้งค่าคอนฟิก")

## <a name="record-tests-to-validate-vendor-payment-processing"></a>บันทึกการทดสอบเพื่อตรวจสอบการประมวลผลการชำระเงินของผู้จัดจำหน่าย

ในฐานะผู้ใช้ระดับสูงด้านการใช้งาน คุณสามารถบันทึกขั้นตอนของคุณเองเพื่อทดสอบการประมวลผลการชำระเงินของผู้จัดจำหน่าย เราขอแนะนำให้คุณเล่น (และแก้ไข ตามต้องการ) การบันทึกงาน **เตรียมการ\\Recording.xml** ที่คุณดาวน์โหลดมาก่อนหน้านี้ การบันทึกนี้ใช้ในการตั้งค่าข้อมูลการทดสอบทั้งหมดไปยังสถานะที่ถูกต้อง ต้องใช้ขั้นตอนนั้นเนื่องจากการทดสอบสามารถทำได้หลายครั้ง และการทดสอบทุกครั้งต้องใช้ข้อมูลที่อยู่ในสถานะเดียวกัน

### <a name="reset-user-settings"></a>รีเซ็ตการตั้งค่าผู้ใช้

1. เปิดแดชบอร์ดเริ่มต้น
2. เลือกปุ่ม **การตั้งค่า** (สัญลักษณ์รูปเฟือง)
3. เลือก **ตัวเลือกผู้ใช้**
4. เลือก **ข้อมูลการใช้งาน**
5. เลือก **รีเซ็ต**
6. เลือก **ใช่** เพื่อยืนยันว่าคุณต้องการรีเซ็ตข้อมูลการใช้
7. ปิดหน้า

### <a name="record-the-steps-to-prepare-data-for-testing"></a>บันทึกขั้นตอนเพื่อจัดเตรียมข้อมูลสำหรับการทดสอบ

1. เลือกปุ่ม **การตั้งค่า** (สัญลักษณ์รูปเฟือง)
2. เลือก **ตัวบันทึกงาน**
3. เลือก **เล่นการบันทึก**
4. เลือก **เปิดจากพีซีนี้**
5. เลือก **เรียกดู** และเลือกบันทึกไฟล์ **จัดเตรียม\\Recording.xml** ที่ไว้ภายในเครื่อง
6. เลือก **เริ่มต้น**
7. เลือก **เล่นขั้นตอนถัดไปที่ค้างอยู่** จนกว่าจะมีการเล่นขั้นตอนทั้งหมดในการจัดทำเรกคอร์ด

การบันทึกงานนี้ทำการดำเนินการต่อไปนี้:

1. ตั้งค่าสถานะของรายการการชำระเงินที่ประมวลผลเป็น **None**

    ![ขั้นตอนการบันทึกงานที่ 3 ถึง 4](media/GER-Recording1Review1.png "ภาพหน้าจอของขั้นตอนการบันทึกงานที่ 3 ถึง 4")

2. เปิดใช้งานพารามิเตอร์ผู้ใช้ ER **รันในโหมดดีบัก**

    ![ขั้นตอนการบันทึกงานที่ 9 ถึง 10](media/GER-Recording1Review2.png "ภาพหน้าจอของขั้นตอนการบันทึกงานที่ 9 ถึง 10")

3. ล้างล็อกของดีบัก ER ที่ประกอบด้วย ผลลัพธ์ของการเปรียบเทียบไฟล์เอาท์พุทกับข้อมูลพื้นฐานที่สร้าง

    ![ขั้นตอนการบันทึกงานที่ 13 ถึง 15](media/GER-Recording1Review3.png "ภาพหน้าจอของขั้นตอนการบันทึกงานที่ 13 ถึง 15")

### <a name="record-the-steps-to-test-vendor-payment-processing"></a>บันทึกขั้นตอนเพื่อทดสอบการประมวลผลการชำระเงินของผู้จัดจำหน่าย

เราขอแนะนำให้คุณเล่น (และแก้ไข ตามต้องการ) การบันทึกงาน **ประมวลผล\\Recording.xml** ที่คุณดาวน์โหลดมาก่อนหน้านี้ การบันทึกนี้ใช้ในการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย และตรวจสอบความถูกต้องของผลลัพธ์ของการเปรียบเทียบเอกสารที่สร้างขึ้นกับข้อมูลพื้นฐานที่สอดคล้องกัน

1. เลือกปุ่ม **การตั้งค่า** (สัญลักษณ์รูปเฟือง)
2. เลือก **ตัวบันทึกงาน**
3. เลือก **เล่นการบันทึก**
4. เลือก **เปิดจากพีซีนี้**
5. เลือก **เรียกดู** และเลือกไฟล์ **ประมวลผล\\Recording.xml** ที่บันทึกไว้ภายในเครื่อง
6. เลือก **เริ่มต้น**
7. เลือก **เล่นขั้นตอนถัดไปที่ค้างอยู่** จนกว่าจะมีการเล่นขั้นตอนทั้งหมดในการจัดทำเรกคอร์ด

การบันทึกงานนี้ทำการดำเนินการต่อไปนี้:

1. เริ่มต้นการประมวลผลการชำระเงินให้แก่ผู้จัดจำหน่าย
2. เลือกพารามิเตอร์รันไทม์ที่ถูกต้อง และเปิดการสร้างของรายงานการควบคุม

    ![ขั้นตอนการบันทึกงานที่ 3 ถึง 8](media/GER-Recording2Review1.png "ภาพหน้าจอของขั้นตอนการบันทึกงานที่ 3 ถึง 8")

3. เข้าถึงล็อกของดีบัก ER เพื่อบันทึกผลลัพธ์ของการเปรียบเทียบเอาท์พุทกับข้อมูลพื้นฐานที่สอดคล้องกัน

    ในล็อกดีบัก ER ผลลัพธ์ของการเปรียบเทียบจะปรากฏในฟิลด์ **ข้อความที่สร้างขึ้น** ฟิลด์ **องค์ประกอบของรูปแบบ** และ **พาธรูปแบบที่ทำให้เกิดรายการล็อก** อ้างอิงถึงส่วนประกอบของไฟล์ที่มีการเปรียบเทียบเอาท์พุทกับข้อมูลพื้นฐาน

    ![รายการในหน้าบันทึกการรันการรายงานทางอิเล็กทรอนิกส์](media/GER-ERDebugLog.png "ภาพหน้าจอของรายการในหน้าบันทึกการรันการรายงานทางอิเล็กทรอนิกส์")

4. การเปรียบเทียบเอาท์พุทปัจจุบันกับข้อมูลพื้นฐานจะถูกบันทึกโดยการใช้ตัวเลือกตัวบันทึกงานของ **ตรวจสอบความถูกต้อง** และการเลือก  **มูลค่าปัจจุบัน**

    ![การใช้ตัวเลือกตรวจสอบความถูกต้องสำหรับการเปรียบเทียบกับมูลค่าปัจจุบัน](media/GER-TRRecordValidation.png "ภาพหน้าจอของการใช้ตัวเลือกตรวจสอบความถูกต้องสำหรับการเปรียบเทียบกับมูลค่าปัจจุบัน")

    ภาพประกอบต่อไปนี้แสดงให้เห็นว่าขั้นตอนการตรวจสอบความถูกต้องที่บันทึกไว้มีลักษณะอย่างไรในการบันทึกงาน

    ![ขั้นตอนการบันทึกงานที่ 13 และ 15](media/GER-Recording2Review2.png "ภาพหน้าจอของขั้นตอนการบันทึกงานที่ 13 และ 15")

## <a name="add-the-recorded-tests-to-azure-devops"></a>เพิ่มการทดสอบที่บันทึกไปยัง Azure DevOps

1. เปิดสภาพแวดล้อม Azure DevOps
2. เลือกโครงการที่คุณกำหนดในพารามิเตอร์ RSAT เมื่อคุณ [ตั้งค่าคอนฟิกเครื่องมือ](#prerequisites)
3. เลือกแผนทดสอบที่คุณกำหนดในพารามิเตอร์ RSAT เมื่อคุณ [ตั้งค่าคอนฟิกเครื่องมือ](#prerequisites)
4. สร้างกรณีทดสอบใหม่สำหรับแผนทดสอบที่เลือก:

    1. ตั้งชื่อแผนทดสอบ **จัดเตรียมข้อมูลเพื่อทดสอบการประมวลผลของการชำระเงินทางอิเล็กทรอนิกส์ของผู้จัดจำหน่าย**
    2. แนบไฟล์ **Recording.xml** จากโฟลเดอร์ **จัดเตรียม** ที่คุณดาวน์โหลดมาก่อนหน้านี้

5. สร้างกรณีทดสอบใหม่สำหรับแผนทดสอบที่เลือก:

    1. ตั้งชื่อแผนทดสอบ **การประมวลผลการทดสอบของการชำระเงินให้แก่ผู้จัดจำหน่ายโดยใช้ BACS (UK) รูปแบบ ER**
    2. แนบไฟล์ **Recording.xml** จากโฟลเดอร์ **ประมวลผล** ที่คุณดาวน์โหลดมาก่อนหน้านี้

    ![กรณีทดสอบใหม่สำหรับแผนทดสอบที่เลือก](media/GER-RSAT-DevOps-Tests-Passed.png "ภาพหน้าจอของกรณีทดสอบใหม่สำหรับแผนทดสอบที่เลือก")

> [!NOTE]
> ให้ความสำคัญกับลำดับการดำเนินการที่ถูกต้องของการทดสอบที่ถูกเพิ่ม

## <a name="prepare-rsat-to-run-the-recorded-tests"></a>จัดเตรียม RSAT เพื่อรันการทดสอบที่บันทึกไว้

### <a name="load-the-tests-from-azure-devops-to-rsat"></a>โหลดการทดสอบจาก Azure DevOps ไปยัง RSAT

1. เปิดแอพลิเคชันของ RSAT เฉพาะที่ในโทโพโลยีปัจจุบัน
2. เลือก **โหลด** เพื่อโหลดการทดสอบที่ขณะนี้อยู่ใน Azure DevOps ลงใน RSAT

    ![การทดสอบที่โหลดไปยัง RSAT](media/GER-RSAT-RSAT-Tests-Loaded.png "ภาพหน้าจอของการทดสอบที่โหลดลงใน RSAT")

### <a name="create-automation-and-parameters-files"></a>สร้างไฟล์ระบบอัตโนมัติและพารามิเตอร์

1. ใน RSAT เลือกการทดสอบที่คุณโหลดจาก Azure DevOps
2. เลือก **สร้าง** เพื่อสร้างไฟล์ระบบอัตโนมัติและพารามิเตอร์ของ RSAT

    ![ไฟล์ระบบอัตโนมัติและพารามิเตอร์ RSAT ที่สร้างใน RSAT](media/GER-RSAT-RSAT-Tests-Initiated.png "ภาพหน้าจอของไฟล์ระบบอัตโนมัติและพารามิเตอร์ RSAT ที่สร้างใน RSAT")

### <a name="modify-the-parameters-files"></a>ปรับเปลี่ยนไฟล์พารามิเตอร์

1. ใน RSAT เลือกกรณีทดสอบ **จัดเตรียมข้อมูลเพื่อทดสอบการประมวลผลของการชำระเงินทางอิเล็กทรอนิกส์ของผู้จัดจำหน่าย**
2. เลือก **แก้ไข**
3. ในสมุดงาน Microsoft Excel ที่เปิดอยู่ บนแผ่นงาน **ทั่วไป** ให้เปลี่ยนรหัสบริษัทเป็น **GBSI** เนื่องจากบริษัทนี้จะใช้สำหรับการดำเนินการทดสอบ
4. ใน RSAT เลือกกรณีทดสอบของ **การประมวลผลการทดสอบของการชำระเงินให้แก่ผู้จัดจำหน่ายโดยใช้ BACS (UK) รูปแบบ ER**
5. เลือก **แก้ไข**
6. ในสมุดงาน Excel ที่เปิดอยู่ บนแผ่นงาน **ทั่วไป** ให้เปลี่ยนรหัสบริษัทเป็น **GBSI**
7. บนแผ่นงาน **ERFormatMappingRunLogTable** ให้สังเกตว่าเซลล์ A:3 และ C:3 มีข้อความของฟิลด์ในตารางล็อกดีบัก ER ที่ใช้เพื่อตรวจสอบความถูกต้องของผลลัพธ์ของการเปรียบเทียบเอาท์พุทกับข้อมูลพื้นฐาน ข้อความเหล่านี้จะใช้ในการประเมินเรกคอร์ดของล็อกดีบัก ER ซึ่งสร้างขึ้นในระหว่างการดำเนินการทดสอบ

    ![แผ่นงาน ERFormatMappingRunLogTable](media/GER-RSAT-RSAT-ExcelParameters.png "ภาพหน้าจอของแผ่นงาน ERFormatMappingRunLogTable")

### <a name="simulate-a-situation-where-tests-fail"></a>จำลองสถานการณ์ที่ซึ่งการทดสอบล้มเหลว

ชุดการทดสอบนี้ต้องล้มเหลว เมื่อเอาท์พุทที่สร้างขึ้นอย่างน้อยหนึ่งรายการไม่ตรงกับพื้นฐานที่สอดคล้องกัน เมื่อต้องการให้บรรลุสถานการณ์นี้ คุณสามารถใช้รุ่นที่ได้รับของรูปแบบ **BACS (UK)** ของคุณ ซึ่งจะสร้างไฟล์การชำระเงินที่มีเนื้อหาที่แตกต่างจากข้อมูลพื้นฐานที่สอดคล้องกัน เมื่อต้องการจำลองสถานการณ์นี้ คุณสามารถใช้รูปแบบ **BACS (UK)** เดียวกัน แต่เปลี่ยนยอดการชำระเงินในรายการการชำระเงินที่ประมวลผลแล้ว

1. เปิดแอพลิเคชั่น แล้วไปยัง **บัญชีเจ้าหนี้ \> การชำระเงิน \> สมุดรายวันการชำระเงิน**
2. เลือก **รายการ**
3. เลือกรายการการชำระเงิน แล้วเลือก **สถานะการชำระเงิน \> ไม่มี**
4. ในฟิลด์ **เดบิต** เปลี่ยนค่าจาก **1,000.00** เป็น **2,000.00**
5. เลือก **บันทึก** เพื่อบันทึกการเปลี่ยนแปลงของคุณ

2. เลือก **รัน**

โปรดสังเกตว่ากรณีทดสอบจะรันโดยอัตโนมัติในแอพลิเคชั่น โดยใช้เว็บเบราเซอร์

### <a name="analyze-the-results-of-test-execution"></a>วิเคราะห์ผลลัพธ์ของการดำเนินการทดสอบ

ผลลัพธ์ของการดำเนินการทดสอบจะถูกจัดเก็บไว้ใน RSAT โปรดสังเกตว่าการทดสอบที่สองล้มเหลวในระหว่างการดำเนินการที่สอง

![ผลการทดสอบที่ล้มเหลวใน RSAT](media/GER-RSAT-RSAT-Tests-Failed.png "ภาพหน้าจอของผลการทดสอบที่ล้มเหลวใน RSAT")

![ผลการทดสอบที่ล้มเหลวใน Azure DevOps](media/GER-RSAT-DevOps-Tests-Failed.png "ภาพหน้าจอของผลการทดสอบที่ล้มเหลวใน Azure DevOps")

คุณสามารถเข้าถึงสถานะของการทดสอบแต่ละครั้งได้ นอกจากนี้ คุณยังสามารถเข้าถึงล็อกการดำเนินการเพื่อให้คุณวิเคราะห์เหตุผลสำหรับความล้มเหลวใดๆ ในภาพประกอบต่อไปนี้ ล็อกการดำเนินการแสดงว่าความล้มเหลวเกิดขึ้น เนื่องจากความแตกต่างในเนื้อหาระหว่างไฟล์การชำระเงินที่สร้างขึ้นและข้อมูลพื้นฐาน
