# <a name="when-to-reset-a-data-mart"></a>เมื่อใดที่จะรีเซ็ต Data Mart

การรีเซ็ต Data Mart อาจใช้เวลานาน ทั้งนี้ขึ้นอยู่กับสถานการณ์ การดำเนินการนี้อาจรจะไม่ใช่โซลูชันที่ต้องการ หัวข้อนี้แสดงรายการสถานการณ์ที่อาจมีการปรับปรุงด้วยการรีเซ็ต Data Mart และสถานการณ์ที่การรีเซ็ต Data Mart เป็นไปได้ยาก  

## <a name="when-do-i-need-to-do-a-data-mart-reset"></a>ฉันต้องรีเซ็ต Data Mart เมื่อใด
พิจารณาคำถามต่อไปนี้ก่อนการรีเซ็ต Data Mart การตอบว่าใช่กับคําถามหนึ่งคําถามหรือมากกว่าอาจบ่งชี้ว่าองค์กรของคุณสามารถได้รับประโยชน์จากการรีเซ็ต Data Mart

- ฐานข้อมูลแอปพลิเคชันคืนค่าหรือไม่
- หากคุณได้เปิดเหตุการณ์สนับสนุนและวิศวกรฝ่ายสนับสนุนแนะนำให้คุณสามารถรีเซ็ต Data Mart โดยเป็นส่วนหนึ่งของขั้นตอนการแก้ไขปัญหาเบื้องต้น
 
## <a name="when-is-it-not-appropriate-to-reset-a-data-mart"></a>เมื่อใดที่การรีเซ็ต Data Mart ไม่เหมาะสม
มีบางสถานการณ์ที่เราไม่แนะนำให้รีเซ็ต Data Mart ได้แก่สิ่งต่อไปนี้ 

- คุณประสบปัญหาด้านประสิทธิภาพที่เกี่ยวข้องกับการซิงค์ข้อมูล 
- ถ้าคุณมีรูปแบบการรีเซ็ตที่เกิดเหตุการณ์ที่เกิดขึ้นเป็นชุดเนื่องจากเหตุผลใด ๆ ต่อไปนี้: 
  - **ข้อมูลขาดหาย** 
  - **สถานะการรวมที่ติดขัด** 
  - **เรกคอร์ดเก่า** - เรกคอร์ดเก่าอย่างเดียวไม่อาจพิสูจน์ได้ว่าควรรีเซ็ต Data Mart หรือไม่ ถ้าคุณมีชุดข้อมูลขนาดใหญ่ กระบวนการรีเซ็ตจะใช้เวลาในกรเรียกใช้ แต่ไม่อาจส่งผลต่อการปรับปรุง
 
## <a name="what-is-a-data-mart-reset"></a>การรีเซ็ต Data Mart คืออะไร
- การรีเซ็ตจะเริ่มต้นเฉพาะเมื่องานที่มีอยู่เสร็จสมบูรณ์แล้วเท่านั้น ซึ่งช่วยให้มั่นใจว่าจะไม่แทรกข้อมูลเก่าเข้ามา ณ จุดนี้ คุณอาจเห็นข้อความ เช่น "ไม่สามารถประมวลผลการรีเซ็ต Data Mart เนื่องจากงานที่ใช้งานอยู่ โปรดลองอีกครั้งในภายหลัง"
- การรีเซ็ตจะปิดใช้งานงานการรวมและลบข้อมูล Data Mart ทั้งหมด การรวมจะเปิดใช้งานอีกครั้ง

## <a name="if-i-reset-the-data-mart-will-i-lose-reports-that-ive-already-designed"></a>ถ้าฉันรีเซ็ต Data Mart ฉันจะสูญเสียรายงานที่ออกแบบไว้แล้วหรือไม่ 
ไม่ รายงานของคุณถูกจัดเก็บไว้ในตาราง SQL ที่ไม่ได้รับผลกระทบจากการรีเซ็ต Data Mart ถ้าคุณกังวลเกี่ยวกับการสูญเสียรายงานใด ๆ ที่คุณออกแบบ คุณสามารถสำรองข้อมูลการออกแบบที่คุณไม่ต้องการสูญเสียได้ เมื่อต้องการสำรองข้อมูล ให้เปิดตัวออกแบบรายงาน และไปที่ **บริษัท > บริษัท > ส่วนประกอบ > ส่งออก**
 
## <a name="is-it-necessary-for-all-users-to-exit-the-system-to-reset-the-data-mart"></a>ผู้ใช้ทั้งหมดต้องออกจากระบบเพื่อรีเซ็ต Data Mart หรือไม่
ไม่ ผู้ใช้สามารถทำงานในระบบต่อไปได้ในระหว่างการรีเซ็ต Data Mart อย่างไรก็ตาม ผู้ใช้จะไม่สามารถเข้าถึงรายงานใด ๆ ที่สร้างขึ้นด้วยโปรแกรมรายงานทางการเงินได้จนกว่าการรีเซ็ตจะเสร็จสิ้น
