## <a name="expense-management"></a>การจัดการค่าใช้จ่าย

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| กำหนดธุรกรรมบัตรเครดิตของพนักงานที่สิ้นสุดการจ้างงานใหม่ | บางครั้ง เมื่อพนักงานสิ้นสุดการจ้างงาน บัญชี Active Directory Domain Services (AD DS) ของบุคคลนั้นจะถูกปิดใช้งานเมื่อมีการนำเข้าธุรกรรมบัตรเครดิตที่ใช้งานอยู่ที่จะต้องถูกชำระนั้นจะถูกนำเข้า ก่อนหน้านี้ คุณไม่สามารถกำหนดผู้รับมอบสิทธิ์สำหรับรายการค่าใช้จ่ายหรือแนบธุรกรรมบัตรเครดิตให้กับรายงานค่าใช้จ่าย คุณสามารถใช้หน้า **ธุรกรรมบัตรเครดิต** ในการกำหนดพนักงานสำหรับธุรกรรมบัตรเครดิตโดยพนักงานที่เกี่ยวข้องนั้นถูกเลิกจ้างพนักงานแล้วได้ หลังจากที่คุณกำหนดธุรกรรมบัตรเครดิต คุณสามารถเลือกธุรกรรมสำหรับรายงานค่าใช้จ่าย และทำการชำระผ่านกระบวนการปกติสำหรับการชำระเงินคืนตามรายงานค่าใช้จ่าย |
| เปลี่ยนข้อมูลบัตรเครดิตค่าใช้จ่ายสำหรับพนักงานที่ค้างอยู่และพนักงานในอดีต | คุณสามารถเปลี่ยนข้อมูลบัตรเครดิตการจัดการค่าใช้จ่ายสำหรับผู้ปฏิบัติงานที่ค้างอยู่และผู้ปฏิบัติงานในอดีต ก่อนหน้านี้ คุณสามารถเปลี่ยนข้อมูลบัตรเครดิตค่าใช้จ่ายเฉพาะถ้าผู้ปฏิบัติงานเป็นพนักงานที่กำลังทำงานอยู่เท่านั้น |
| คัดลอกรายงานค่าใช้จ่าย | ขณะนี้ คุณสามารถคัดลอกค่าใช้จ่ายที่มีอยู่ได้เมื่อคุณสร้างค่าใช้จ่ายใหม่ในรายงานค่าใช้จ่ายเดียวกัน คุณสามารถคัดลอกรายงานค่าใช้จ่ายและค่าใช้จ่ายที่ไม่ใช่บัตรเครดิตที่เกี่ยวข้องไปยังรายงานค่าใช้จ่ายใหม่ได้ นอกจากนี้คุณยังคงต้องดำเนินการแสดงรายการที่จำเป็นใดๆ และแนบใบรับสินค้าที่จำเป็น ตามนโยบายค่าใช้จ่ายและการจัดประเภท คุณสามารถเพิ่มธุรกรรมบัตรเครดิตไปยังรายงานค่าใช้จ่ายโดยการเลือกที่จะเพิ่มค่าใช้จ่ายที่ไม่ได้กระทบยอด |
| แก้ไขค่าใช้จ่ายจำนวนมาก | ธุรกรรมบัตรเครดิตบางรายการอาจไม่ได้แม็ปกับประเภทค่าใช้จ่าย หรือการแม็ปอาจไม่ถูกต้องขณะที่ทำการนำเข้า ในกรณีนี้ พนักงานต้องเปลี่ยนประเภทค่าใช้จ่ายที่เกี่ยวข้องด้วยตนเอง เมื่อคุณจัดการค่าใช้จ่ายที่ยังไม่ได้กระทบยอด คุณสามารถแก้ไขประเภทค่าใช้จ่ายจำนวนมากสำหรับค่าใช้จ่ายที่เลือกไว้ได้ นอกจากนี้ เมื่อคุณจัดการค่าใช้จ่ายที่เลือกสำหรับรายงานค่าใช้จ่ายหนึ่งๆ คุณสามารถแก้ไขโครงการและข้อมูลเพิ่มเติมเป็นจำนวนมากได้ |
| เปลี่ยนอัตราแลกเปลี่ยนสำหรับธุรกรรมบัตรเครดิต | อัตราแลกเปลี่ยนจริงที่ใช้สำหรับธุรกรรมบัตรเครดิตอาจแตกต่างจากอัตราแลกเปลี่ยนที่ตั้งค่าไว้ในระบบ ก่อนหน้านี้ พนักงานไม่สามารถเปลี่ยนอัตราแลกเปลี่ยนสำหรับธุรกรรมบัตรเครดิตที่ถูกนำเข้าลงในการจัดการค่าใช้จ่าย ขณะนี้คุณสามารถตั้งค่าพารามิเตอร์ในหน้า **พารามิเตอร์การจัดการค่าใช้จ่าย** เพื่ออนุญาตให้มีการเปลี่ยนอัตราแลกเปลี่ยนสำหรับธุรกรรมบัตรเครดิตได้ |
| ตั้งค่าการรับรองการป้องกันการทุจริตสำหรับค่าใช้จ่าย | พนักงานบางคนขององค์กรอาจมีธุรกิจกับเจ้าหน้าที่รัฐบาล และอาจได้รับค่าใช้จ่ายที่เกิดขึ้นโดยเป็นส่วนหนึ่งของธุรกิจนั้นบางรายการโดยเป็นการติดสินบน ในการจัดการค่าใช้จ่าย ขณะนี้คุณสามารถตั้งค่าการรับรองการป้องกันการทุจริตที่แสดงสำหรับประเภทค่าใช้จ่ายที่เลือก เช่น ค่าอาหารและของขวัญ พนักงานต้องเลือกว่าค่าใช้จ่ายที่มีการตั้งค่าสำหรับการรับรองการป้องกันการทุจริตตรงตามเกณฑ์ที่กำหนดโดยนโยบายขององค์กรของคุณหรือไม่ |
| ป้องกันการป้อนข้อมูลด้วยตนเองของค่าใช้จ่ายสำหรับประเภทค่าใช้จ่ายหนึ่งๆ | สามารถตั้งค่าประเภทค่าใช้จ่ายเพื่อแสดงถึงธุรกรรมบัตรเครดิตที่ไม่สามารถเปลี่ยนประเภทได้ ตัวอย่างคือค่าธรรมเนียมสำหรับการชำระเงินล่าช้าของบัตรเครดิต ขณะนี้คุณสามารถตั้งค่าประเภทค่าใช้จ่ายที่อนุญาตให้สร้างค่าใช้จ่ายโดยใช้การนำเข้าเท่านั้น คุณลักษณะนี้ป้องกันไม่ให้พนักงานสร้างค่าใช้จ่ายสำหรับประเภทดังกล่าวด้วยตนเอง นอกจากนี้ คุณไม่สามารถอนุญาตให้มีการเปลี่ยนประเภทสำหรับธุรกรรมที่มีการนำเข้าและที่ใช้ประเภทนี้ได้ |
| แนบใบเสร็จกับค่าใช้จ่ายหลายรายการ | ใบรับสินค้าที่อัพโหลดเป็นยังการจัดการค่าใช้จ่ายอาจเกี่ยวข้องกับค่าใช้จ่ายหลายรายการ ก่อนหน้านี้ ใบรับสินค้าที่เกี่ยวข้องกับค่าใช้จ่ายหลายรายการถูกอัพโหลดหนึ่งครั้งสำหรับแต่ละค่าใช้จ่าย ขณะนี้คุณสามารถแนบใบรับสินค้าให้กับค่าใช้จ่ายที่ถูกอัพโหลดแล้วและแนบกับค่าใช้จ่ายอื่นในรายงานค่าใช้จ่ายเดียวกัน นอกจากนี้ ถ้าคุณอัพโหลดใบรับสินค้าไปยังส่วนหัวของรายงานค่าใช้จ่าย คุณสามารถเลือกหนึ่งรายการขึ้นไปเพื่อแนบกับใบรับสินค้า |
| สร้างค่าใช้จ่ายที่มีวันที่ในอนาคต | เมื่อคุณคัดลอกรายงานค่าใช้จ่าย ตัวอย่างเช่น วันที่ธุรกรรมสำหรับค่าใช้จ่ายอาจมีวันที่ในอนาคต รุ่นก่อนหน้านี้ไม่อนุญาตสำหรับค่าใช้จ่ายที่มีวันที่ในอนาคต ขณะนี้คุณสามารถสร้างและบันทึกค่าใช้จ่ายที่มีวันที่ในอนาคต อย่างไรก็ตาม คุณไม่สามารถส่งค่าใช้จ่ายที่มีวันที่ในอนาคตได้ |
| ตั้งค่าคอนฟิกภาษีค่าใช้จ่ายสำหรับรัฐ/จังหวัด | ในบางประเทศ/ภูมิภาค ค่าใช้จ่ายที่เกิดขึ้นในรัฐ/จังหวัดที่แตกต่างกันอาจต้องเสียภาษีในอัตราภาษีขายที่แตกต่างกัน ขณะนี้คุณสามารถตั้งค่าการตั้งค่าคอนฟิกภาษีค่าใช้จ่ายในระดับรัฐ/จังหวัด ไม่ใช่แค่ที่ระดับประเทศ/ภูมิภาค ถ้าคุณเว้นฟิลด์ **รัฐ/จังหวัด** ว่างไว้ในหน้า **การตั้งค่าคอนฟิกภาษี** กลุ่มภาษีขายจะใช้กับรัฐ/จังหวัดทั้งหมดสำหรับประเทศ/ภูมิภาคที่ระบุ |
| ตั้งค่าชนิดบัตรเครดิตค่าใช้จ่าย | ก่อนหน้านี้ ไม่มีหน้าที่คุณจะสามารถสร้างชนิดบัตรเครดิตใหม่ได้ เช่น วีซ่า มาสเตอร์การ์ด หรือ AMEX ที่สามารถใช้กับการจัดการค่าใช้จ่าย ขณะนี้คุณสามารถสร้างชนิดบัตรเครดิตเพื่อใช้กับการจัดการค่าใช้จ่ายได้ หน้าดังกล่าวจะอยู่ในพื้นที่ **การตั้งค่า** ในส่วน **การคำนวณและรหัส** |
| กำหนดลำดับงานการอนุมัติหลายระดับสำหรับรายงานค่าใช้จ่าย | ลำดับงานใหม่สำหรับรายงานค่าใช้จ่ายสนับสนุนกระบวนการอนุมัติหลายระดับ พนักงานป้อนผู้อนุมัติชั่วคราวและผู้อนุมัติขั้นสุดท้ายเมื่อสร้างรายงานค่าใช้จ่าย จากนั้นลำดับงานจะกำหนดเส้นทางรายงานค่าใช้จ่ายไปยังผู้อนุมัติชั้นคราวก่อน หลังจากที่รายงานค่าใช้จ่ายได้รับการอนุมัติที่ระดับดังกล่าว ลำดับงานจะกำหนดเส้นทางไปยังผู้อนุมัติขั้นสุดท้ายสำหรับการอนุมัติขั้นสุดท้าย |
| สร้างและรักษาค่าใช้จ่ายที่ไม่ได้ถูกแนบกับรายงานค่าใช้จ่าย | ขณะนี้ผู้ใช้สามารถสร้างค่าใช้จ่ายและรักษา (ตัวอย่างเช่น โดยการแนบหรือการแสดงรายการใบรับสินค้า หรือการกำหนดผู้เยี่ยมชม) โดยไม่จำเป็นต้องสร้างรายงานค่าใช้จ่ายก่อน สามารถเลือกและเพิ่มค่าใช้จ่ายอย่างน้อยหนึ่งรายการให้กับรายการค่าใช้จ่ายใหม่ เพื่อให้สามารถส่งเพื่อขออนุมัติ หน้าใหม่สำหรับการรักษาค่าใช้จ่ายจะพร้อมใช้งานที่ **การจัดการค่าใช้จ่าย** &gt; **ค่าใช้จ่ายของฉัน** &gt; **ค่าใช้จ่าย** |

| ตั้งค่าคอนฟิกรายงาน ER เพื่อแทรกข้อมูล ในหลายรูปแบบ จากที่เก็บข้อมูลการจัดการเอกสารลงในข้อความอิเล็กทรอนิกส์ที่สร้างขึ้น | รายงาน ER สามารถแทรกข้อมูลลงในข้อความอิเล็กทรอนิกส์ที่สร้างขึ้นจากที่เก็บข้อมูลการจัดการเอกสาร ดังนั้น จึงสามารถเพิ่มเอกสารแนบของเอกสารทางธุรกิจไปยังข้อความอิเล็กทรอนิกส์ที่สร้างขึ้นสำหรับเอกสารดังกล่าวตามข้อกำหนดทางกฎหมาย ในปัจจุบัน สามารถเพิ่มเอกสารแนบเหล่านี้ในรูปแบบ MIME ไปยังข้อความ XML ที่สร้างขึ้น อีกทางหนึ่งคือสามารถเพิ่มเอกสารแนบในรูปแบบ Base64 ไปยังข้อความไบนารีที่สร้างขึ้น |
| ตั้งค่าคอนฟิกรายงาน ER เพื่อสร้างเอกสารทางอิเล็กทรอนิกส์ใน Excel Microsoft Word หรือรูปแบบ PDF | การตั้งค่าคอนฟิกหนึ่งๆ จะเปิดใช้งานรายงาน ER เพื่อสร้างเอกสารทางอิเล็กทรอนิกส์ในรูปแบบที่แตกต่างกันสามแบบ: แผ่นงาน OpenXML (Excel), Word และ XML Forms Data Format (XFDF) (PDF) ผู้ใช้สามารถเลือกรูปแบบโดยการเพิ่มเท็มเพลตรูปแบบไปยังรายงาน ER เป็นเอกสาร Excel, Word หรือ PDF |
| ตั้งค่าคอนฟิกรายงาน ER เพื่อแทรกข้อมูลลงในหัวกระดาษและท้ายกระดาษของเอกสารทางอิเล็กทรอนิกส์ที่สร้างขึ้นในรูปแบบของแผ่นงาน OpenXML และเพื่อควบคุมตัวแบ่งหน้า | รายงาน ER สามารถป้อนข้อมูลธุรกิจลงในหัวกระดาษและท้ายกระดาษ และควบคุมตำแหน่งที่จะให้มีตัวแบ่งหน้า ดังนั้น รายงานสามารถสนับสนุนส่วนด้านบนและด้านล่างแบบคงที่สำหรับหน้าของเอกสารทางอิเล็กทรอนิกส์ที่สร้างขึ้น และยังสามารถสนับสนุนการแบ่งหน้าเฉพาะของเอกสารเหล่านั้นเพื่อให้เป็นไปตามข้อกำหนดทางกฎหมาย |
| ตั้งค่าคอนฟิกปลายทางของรายงาน ER เพื่อให้ผลลัพธ์ถูกส่งเป็นอีเมล และเพื่อให้สามารถใช้ข้อมูลธุรกิจและตรรกะ ER (นิพจน์) เพื่อระบุที่อยู่อีเมลที่จะใช้ขณะรันไทม์ | ก่อนหน้านี้ เมื่อคุณตั้งค่าคอนฟิกปลายทาง ER ที่อยู่อีเมลของผู้รับผลลัพธ์อาจถูกกำหนดในขณะออกแบบ ขณะนี้ คุณสามารถตั้งค่าคอนฟิกนิพจน์ได้ในรูปแบบ ER จากนั้น คุณสามารถเลือกนิพจน์ในปลายทางเป็นแหล่งที่มาที่อยู่อีเมลสำหรับการตั้งค่าคอนฟิกรูปแบบแต่ละรายการ และองค์ประกอบผลลัพธ์แต่ละรายการ (โฟลเดอร์หรือไฟล์) โดยแยกจากกันได้ ดังนั้น เมื่อรายงาน ER ทำงาน แต่ละไฟล์ที่ถูกสร้างขึ้นอาจถูกส่งให้กับผู้รับอื่น และสามารถกำหนดที่อยู่เมลตามตรรกะ ER และข้อมูลธุรกิจ |
| ตั้งค่าคอนฟิกปลายทางของรายงาน ER เพื่อส่งเอาต์พุตไปยังโฟลเดอร์ Microsoft SharePoint เป็นไฟล์ที่มีชื่อใหม่หรือรุ่นใหม่ของไฟล์ที่มีอยู่ และเพื่อให้ข้อมูลทางธุรกิจสามารถใช้ได้ในกรอบงาน Microsoft Power BI เป็นชุดของข้อมูลหรือรายงานอย่างใดอย่างหนึ่ง | เมื่อคุณตั้งค่าคอนฟิกรายงาน ER ขณะนี้คุณสามารถ (โดยไม่ต้องเขียนโค้ด) เตรียมการข้อมูลทางธุรกิจที่ร้องขอได้อย่างง่ายดาย เพื่อให้สามารถใช้ได้โดยกรอบงาน Power BI เมื่อคุณรันรายงาน ER เหล่านั้น คุณให้กรอบงาน Power BI ที่มีข้อมูลทางธุรกิจที่เหมาะสมและ/หรือรายงาน Excel ที่พร้อมใช้งานอยู่แล้ว ถ้าคุณจัดกำหนดการรายงานให้รันในโหมดที่เกิดซ้ำ คุณสามารถสร้างการส่งที่จัดกำหนดการไว้ของข้อมูลทางธุรกิจจาก Dynamics 365 for Operations ไปยัง Power BI เพื่อสนับสนุนกำหนดการปรับปรุงของ Power BI–ที่ยึดตามรายงาน |
| ตั้งค่าคอนฟิกรายงาน ER เพื่อใช้บางส่วนของเอกสารอิเล็กทรอนิกส์ที่ถูกสร้างขึ้นโดยเป็นแหล่งข้อมูลเพื่อสร้างเอกสารส่วนที่เหลือ | คุณสามารถตั้งค่าคอนฟิกรายงาน ER ที่สร้างผลลัพธ์ในรูปแบบข้อความเพื่อทำการตรวจนับรายการในเอกสาร จากนั้นจึงสามารถใช้ข้อมูลนี้ในส่วนอื่นของเอกสารเพื่อสร้างรายการที่มีรายละเอียดสรุป ข้อมูลสรุป (ผลรวมและตัวเลข) สามารถได้รับการคำนวณและพิมพ์ไปยังเอกสารทางอิเล็กทรอนิกส์ที่ถูกสร้างขึ้น โดยไม่จำเป็นต้องมีการแปลงข้อมูลเพิ่มเติม ดังนั้น คุณลักษณะนี้จึงช่วยปรับปรุงประสิทธิภาพของการดำเนินการรายงาน และช่วยทำให้การบำรุงรักษาในอนาคตของรูปแบบ ER ที่มีการตั้งค่าคอนฟิกง่ายขึ้น |
| ตั้งค่าคอนฟิกรายงาน ER เพื่อระบุนามสกุลของชื่อไฟล์สำหรับเอกสารอิเล็กทรอนิกส์ที่สร้างขึ้นในรูปแบบข้อความ | คุณสามารถตั้งค่าคอนฟิกรายงาน ER เพื่อสร้างผลลัพธ์ในรูปแบบข้อความ เพื่อให้สามารถบันทึกเป็นไฟล์ที่มีนามสกุลที่ระบุ นอกเหนือจากนามสกุล .txt ที่เป็นค่าเริ่มต้น คุณสามารถตั้งค่าคอนฟิกนามสกุลต่าๆ เช่น .csv และ. prn ที่สอดคล้องกับข้อมูลจำเพาะรูปแบบ |
| สร้างรายงาน ER ใหม่ที่ยึดตามแบบจำลอง ER รุ่นที่เฉพาะเจาะจง | ก่อนหน้านี้ เมื่อคุณสร้างรูปแบบ ER ใหม่ จะสามารถใช้ได้เฉพาะเวอร์ชันล่าสุดของแบบจำลอง ER ที่เลือกโดยเป็นที่ตั้งของแหล่งข้อมูลของรูปแบบ ขณะนี้คุณสามารถเลือกเวอร์ชันใดๆ ที่พร้อมใช้งานของรูปแบบ ER ที่เลือก คุณลักษณะนี้ช่วยให้คุณรักษารายงาน ER สำหรับปีปัจจุบันและออกแบบแบบจำลอง ER เวอร์ชันใหม่สำหรับปีถัดไปพร้อมกัน |
| ค้นหาแหล่งข้อมูลและรูปแบบ/แบบจำลองตามข้อความหรือวัตถุที่เลือกในหนึ่งคลิก แก้ไขความขัดแย้งการปรับใช้ซ้ำในเชิงรุกและแก้ไขความขัดแย้งระหว่างการตั้งค่าคอนฟิก ตั้งค่าคอนฟิกรายงาน ER เพื่อสร้างข้อความการตรวจสอบความถูกต้องของข้อผิดพลาดหลายรายการที่พบจนกระทั่งหยุดการดำเนินการรายงาน | การปรับปรุงประสบการณ์การใช้งานของผู้ใช้ (UX) หลายรายในวิธีใช้กรอบงาน ER ช่วยให้ผู้ใช้สามารถทำงานกับ ER ได้อย่างมีประสิทธิภาพและง่ายดายมากขึ้น |
| แสดงพื้นที่ทำงาน **ER** ในรายงาน Power BI | ผู้ใช้สามารถตั้งค่าคอนฟิกหน้า **พื้นที่ทำงาน ER** เพื่อให้รวมรายการเมนูใหม่และไทล์ปัจจุบ้นที่รัน Power BI–ที่ยึดตามรายงาน |

## <a name="retail-and-commerce"></a>การขายปลีกและการค้า

### <a name="advanced-warehouse-management-in-a-retail-store"></a>การจัดการคลังสินค้าขั้นสูงในร้านค้าปลีก

ผู้ค้าปลีกสามารถใช้โซลูชันการจัดการคลังสินค้าขั้นสูงในร้านค้าของตน และทำการปรับปรุงที่จำเป็นสำหรับคุณลักษณะของการขายหน้าร้าน (POS) ปัจจุบันเพื่อสนับสนุน กระบวนการของโซลูชันอุปกรณ์มือถือควรทำงานในร้านค้าเช่นเดียวกับที่ทำงานในคลังสินค้าอื่น กระบวนการสินค้าคงคลังของร้านค้าปลีกปัจจุบันทั้งหมด และกระบวนการ POS ใดๆ ที่สร้างหรือปรับปรุงธุรกรรมสินค้าคงคลังได้รับการปรับปรุงเพื่อให้ใช้ข้อกำหนดเฉพาะสำหรับใช้คลังสินค้าที่เปิดใช้งาน WHS

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| ใช้ POS เพื่อค้นหาสินค้าคงคลังที่พร้อมใช้งานในร้านค้าที่เปิดใช้งาน WHS | เมื่อคุณค้นหาสินค้าคงคลัง กระบวนการควรทำงานเช่นเดียวกับที่ทำงานก่อนหน้านี้ การค้นหาควรแสดงปริมาณที่พร้อมใช้งานในระดับคลังสินค้า และไม่ควรพิจารณาที่ตั้ง สถานะสินค้าคงคลัง หรือมิติป้ายทะเบียน |
| ใช้ POS เพื่อประมวลผลใบรับสินค้าสำหรับใบสั่งโอนย้ายในร้านค้าที่เปิดใช้งาน WHS | คุณสามารถได้รับใบสั่งโอนย้ายที่ POS สำหรับร้านค้าและสินค้าที่เปิดใช้งาน WHS ผู้ใช้ควรจะสามารถเลือกที่ตั้งเพื่อรับ และควรจะสามารถป้อนรหัสทะเบียนเพื่อเติมรายการการรับโดยอัตโนมัติ |
| ใช้ POS เพื่อประมวลผลใบรับสินค้าสำหรับใบสั่งซื้อในร้านค้าที่เปิดใช้งาน WHS | คุณสามารถได้รับใบสั่งซื้อที่ POS สำหรับร้านค้าและสินค้าที่เปิดใช้งาน WHS ผู้ใช้ควรจะสามารถเลือกที่ตั้งเพื่อรับ และควรจะสามารถป้อนรหัสทะเบียนเพื่อเติมรายการการรับโดยอัตโนมัติ |
| ใช้ POS เพื่อประมวลผลการจัดส่งผลิตภัณฑ์จากร้านค้าที่เปิดใช้งาน WHS | คุณสามารถลงทะเบียนการโอนย้ายจาก POS สำหรับร้านค้าและสินค้าที่เปิดใช้งาน WHS ผู้ใช้ควรจะสามารถเลือกที่ตั้งที่จะจัดส่งจาก สถานะสินค้าคงคลังควรถูกสร้างขึ้นโดยอัตโนมัติ และป้ายทะเบียนควรถูกละเว้น |

### <a name="enable-seamless-omni-channel-commerce"></a>เปิดใช้งานการค้าช่องทาง Omni ที่ไม่มีขอบเขต

การค้าช่องทาง Omni ที่ไม่มีขอบเขตอ้างอิงถึงการจัดการและการประมวลผลใบสั่งระหว่างร้านค้าจริง Storefront ออนไลน์ ศูนย์บริการ และประสบการณ์การค้าบนอุปกรณ์เคลื่อนที่ สำหรับรุ่นนี้ มีการสร้างการลงทุนต่อไปนี้ในพื้นที่นี้:

- การปรับปรุงการประกาศสำหรับ Storefront พาณิชย์อิเล็กทรอนิกส์
- แอพบนมือถือที่เชื่อมต่อกับผู้ใช้ใหม่ที่ช่วยให้สามารถค้นหาผลิตภัณฑ์ จัดการบัญชี และซื้อของทางออนไลน์

### <a name="store-system-providing-a-seamless-on-premises-store-experience"></a>ระบบร้านค้า: ให้ประสบการณ์การจัดเก็บในสถานที่อย่างราบรื่น

ระบบร้านค้าเป็นตัวเลือกการปรับใช้สำหรับผู้ค้าปลีกที่ช่วยกระตุ้นชุดของการดำเนินการจัดเก็บ ไม่ว่าจะอยู่ในร้านค้า on-premises ระบบคลาวด์สาธารณะของ Microsoft หรือระบบคลาวด์แบบส่วนตัวของลูกค้าเอง สำหรับการนำออกใช้นี้ ขอบเขตจะเป็นแบบในร้านค้าเท่านั้น เมื่อต้องการสนับสนุนสภาพแวดล้อมที่มีการเชื่อมต่อเครือข่ายที่ช้าและไม่น่าเชื่อถือให้ดียิ่งขึ้น เราต้องระบุตัวเลือกสำหรับผู้ค้าปลีกเพื่อปรับใช้ฐานข้อมูลช่องทางและเซิร์ฟเวอร์ระบบการขายปลีกในร้านค้า จากนั้นพวกเขาสามารถดำเนินสถานการณ์ทางธุรกิจของพวกเขาต่อ แม้ไม่มีการเชื่อมต่อกับสำนักงานใหญ่ (HQ) ทั้งนี้ขึ้นอยู่กับจุดข้อมูลต่างๆ ซึ่งรวมการสนทนาของทีมวิศวกรรม ผลการสำรวจความคิดเห็นของลูกค้าและการวิเคราะห์คู่แข่ง เราได้ระบุขอบเขตของโซลูชันต่อไปนี้โดยเป็นสิ่งที่เหมาะสมที่สุดสำหรับลูกค้าเป้าหมายของเรา:

- แพคเกจบริการตนเองจะพร้อมใช้งานสำหรับระบบร้านค้า
- การติดตั้งค่าเริ่มต้นคือการปรับใช้หนึ่งกล่อง แต่อนุญาตให้มีการปรับใช้ที่กำหนดเอง
- เปิดใช้งานการปรับใช้/การบริการตนเองอย่างง่าย
- เซิร์ฟเวอร์ระบบการขายปลีกและฐานข้อมูลร้านค้าอยู่ในร้านค้าพร้อมกับบริการ Async Client
- เซิร์ฟเวอร์ระบบการขายปลีกในร้านค้าสื่อสารโดยตรงกับ Application Object Server (AOS) ใน HQ สำหรับระบบร้านค้า
- สนับสนุนสถานการณ์จำลองระหว่างเทอร์มินัลข้ามที่ไม่มีการเชื่อมต่อ HQ
- Retail Modern POS และ Cloud POS สื่อสารกับ Retail Server ในร้านค้าเสมอ
- สนับสนุน Retail Modern POS และ Cloud POS ที่ซึ่งไม่มีการเชื่อมต่อ HQ
- สนับสนุน Retail Modern POS–ฐานข้อมูลแบบออฟไลน์เฉพาะ (แยกไปยังอินสแตนซ์ Retail Modern POS แต่ละรายการ) ที่ซึ่งไม่มีการเชื่อมต่อ HQ
- การรับรองความถูกต้องเป็นไปตามบริการต่อบริการสำหรับระบบร้านค้าเท่านั้น
- ไม่สนับสนุนการเรียกบริการแบบเรียลไทม์ ถ้าไม่มีการเชื่อมต่ออินเทอร์เน็ต
- การเชื่อมต่อฐานข้อมูลโดยตรงของ Retail Modern POS ไปยังฐานข้อมูลช่องทาง ไม่ได้รับการสนับสนุน
- เปิดใช้งานการตรวจวัดระยะไกลและการตรวจสอบ

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| ผู้ขายปลีกดาวน์โหลดตัวติดตั้งแบบบริการตนเองของระบบร้านค้าจากหน้าฐานข้อมูลช่องทางใน Dynamics AX HQ และดาวน์โหลดไฟล์การตั้งค่าคอนฟิก | ผู้ค้าปลีกสามารถดาวน์โหลดแพคเกจบริการตนเองได้อย่างราบรื่น |
| ผู้ค้าปลีกติดตั้งระบบร้านค้าโดยใช้โปรแกรมติดตั้งบริการตนเอง | ผู้ค้าปลีกสามารถติดตั้งระบบร้านค้าโดยใช้แพคเกจบริการตนเอง |

| ผู้ขายปลีกดำเนินการ Retail Modern POS ในร้านค้า on-premises และสามารถทำการดำเนินการแบบเรียลไทม์ได้ เช่น ออกบัตรของขวัญ เมื่อมีการเชื่อมต่อ | ผู้ค้าปลีกสามารถดำเนินการแบบเรียลไทม์จากระบบร้านค้าเมื่อมีการเชื่อมต่อ |
| ผู้ค้าปลีกสามารถซิงค์ข้อมูลจากระบบร้านค้าในสถานที่ไปยัง HQ ทุกครั้งที่มีการเชื่อมต่อ | ผู้ค้าปลีกสามารถซิงค์ข้อมูลไปยังและจากระบบร้านค้าเมื่อมีการเชื่อมต่อ |
| ผู้ค้าปลีกสามารถมีการสื่อสารที่ปลอดภัยระหว่างระบบร้านค้าในสถานและ HQ | ผู้ค้าปลีกสามารถติดต่อสื่อสารจากระบบร้านค้าได้อย่างปลอดภัยเมื่อมีการเชื่อมต่อ |
| ผู้จัดการฝ่ายไอทีและการดำเนินงานของ Microsoft สามารถตรวจสอบและรายงานระบบร้านค้าในสถานที่ (การเปลี่ยนแปลงการวินิจฉัยและการรายงาน) | ผู้จัดการฝ่ายไอทีและการดำเนินงานของ Microsoft สามารถตรวจสอบระบบร้านค้าได้อย่างปลอดภัย และแก้ไขปัญหาอย่างมีประสิทธิภาพ |

### <a name="universal-windows-platform-app-for-retail-modern-pos"></a>แอป Universal Windows Platform สำหรับ Retail Modern POS

ในขณะนี้ Retail Modern POS พร้อมใช้งานเฉพาะเป็นแอพลิเคชัน Windows 8.1 สำหรับคอมพิวเตอร์เดสก์ท็อปและแท็บเล็ต และเป็น Cloud POS สำหรับเบราเซอร์เดสก์ท็อปและแท็บเล็ต ในรุ่นนี้ Retail Moderns POS ถูกแปลงเป็นแอพแพลตฟอร์ม Universal Windows (UWP) การเปลี่ยนแปลงนี้จะเปิดใช้งาน Retail Modern POS ให้รันบนอุปกรณ์ Windows 10 ใดๆ (เดสก์ท็อป แท็บเล็ต หรือโทรศัพท์) และแม้แต่การสลับระหว่างมุมมองสำหรับอุปกรณ์ที่เปิดใช้งาน Continuum

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| เปิดใช้งานฟังก์ชัน POS ที่สำคัญสำหรับอุปกรณ์ปัจจัยแบบฟอร์มขนาดเล็ก | ฟังก์ชัน Retail POS จะพร้อมใช้งานสำหรับโทรศัพท์และอุปกรณ์ปัจจัยแบบฟอร์มขนาดเล็กอื่นๆ ที่รัน Windows 10 |

| ในฐานะผู้จัดจำหน่าย คุณสามารถจัดเก็บวัตถุดิบที่คลังสินค้าของลูกค้าได้ ในฐานะลูกค้า คุณสามารถเลื่อนการซื้อจริงจนกระทั่งจำเป็นต้องใช้วัตถุดิบสำหรับการผลิต | การรักษาสินค้าคงคลังของวัตถุดิบอาจเกี่ยวข้องกับต้นทุนที่ร้ายแรง โดยใช้สินค้าคงคลังที่มีการส่งมอบ ผู้จัดจำหน่ายสามารถจัดเก็บวัตถุดิบที่คลังสินค้าของลูกค้าได้ จากนั้นลูกค้าจึงสามารถเลื่อนการซื้อจริงจนกระทั่งจำเป็นต้องใช้วัตถุดิบสำหรับการผลิต วัตถุดิบจะถูกสั่งและได้รับโดยใช้ใบสั่งเพิ่มเติมสินค้าในการส่งมอบ ใบสั่งการเพิ่มเติมสินค้านี้บันทึกธุรกรรมที่เกิดขึ้นจริง แต่ไม่มีผลต่อบัญชีแยกประเภททั่วไป เพื่อแยกความแตกต่างระหว่างสินค้าคงคลังที่เป็นของลูกค้าและสินค้าคงคลังที่เป็นของผู้จัดจำหน่าย คุณสามารถใช้มิติสินค้าคงคลังของเจ้าของ การเปลี่ยนแปลงความเป็นเจ้าของสินค้าคงคลังถูกจัดการโดยกระบวนการของสมุดรายวันที่จัดการกับการโอนย้ายความเป็นเจ้าของของวัตถุดิบจากสินค้าคงคลังที่เป็นของผู้จัดจำหน่ายไปที่สินค้าคงคลังที่เป็นของลูกค้า กระบวนการนี้จะทริกเกอร์การสร้างใบสั่งซื้อและใบรับสินค้า<blockquote>[!NOTE] คุณลักษณะนี้จะจำกัดอยู่ที่การส่งมอบขาเข้าของวัตถุดิบสำหรับการผลิต ซึ่งจะไม่รวมเข้ากับการจัดการคลังสินค้า (WHS) และการจัดการการขนส่ง (TMS)</blockquote> |
| ในฐานะผู้จัดจำหน่าย ให้เรียกดูข้อมูลเกี่ยวกับจำนวนของสินค้าคงคลังที่ส่งที่ถูกโอนย้ายไปยังลูกค้า | เมื่อต้องการเรียกเก็บเงินจากลูกค้า ผู้จัดจำหน่ายจำเป็นต้องมีข้อมูลเกี่ยวกับวัตถุดิบที่ซื้อจากสินค้าคงคลังที่มีการส่งมอบและวันที่ของการซื้อ ผู้จัดจำหน่ายยังสามารถตรวจสอบปริมาณคงคลังคงเหลือที่ไซต์ของลูกค้าโดยใช้อินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่าย |
| ย้ายสินค้าคงคลังที่เป็นของผู้จัดจำหน่ายโดยใช้สมุดรายวันการโอนย้าย | เพื่อติดตามตำแหน่งจริงของสินค้าคงคลังที่เป็นของผู้จัดจำหน่าย คุณต้องสามารถบันทึกตำแหน่งในระบบได้ โดยใช้สมุดรายวันการโอนย้าย คุณสามารถบันทึกความเคลื่อนไหวจริงของสินค้าคงคลัง เช่น การเคลื่อนย้ายจากสถานที่หนึ่งในคลังสินค้าไปยังอีกสถานที่หนึ่งในคลังสินค้านั้น |
| ปรับปรุงสินค้าคงคลังที่เป็นของผู้จัดจำหน่ายโดยใช้สมุดรายวันการตรวจนับ | คุณจะต้องให้ปริมาณคงคลังคงเหลือของระบบซิงค์กับสินค้าคงคลังทางกายภาพจริงอยู่เสมอ สามารถปรับปรุงสินค้าคงคลังที่เป็นของผู้จัดจำหน่ายโดยใช้กระบวนการตรวจนับ เช่น กระบวนการการปรับปรุงปริมาณ และสมุดรายวันการตรวจนับ |
| ค้นหาข้อมูลเพิ่มเติมเกี่ยวกับการสนับสนุนการฝากขายใน Dynamics 365 for Operations | ดูข้อมูลเพิ่มเติมเกี่ยวกับการสนับสนุนสำหรับกระบวนการส่งมอบที่ [การส่งมอบ](../../../supply-chain/inventory/consignment.md) [การตั้งค่าการส่งมอบ](../../../supply-chain/inventory/set-up-consignment.md) [สร้างใบสั่งการเพิ่มเติมสินค้าที่มีการส่งมอบ (คู่มืองาน)](../../../supply-chain/inventory/tasks/create-consignment-replenishment-order.md)และ [เปลี่ยนความเป็นเจ้าของสินค้าคงคลังที่มีการส่งมอบตามความต้องการของการผลิต (คู่มืองาน)](../../../supply-chain/inventory/tasks/change-ownership-consignment.md) |

### <a name="vendor-collaboration-previously-known-as-the-vendor-portal"></a>การทำงานร่วมกันกับผู้จัดจำหน่าย (เดิมเรียกว่าพอร์ทัลผู้จัดจำหน่าย)

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| อนุญาตให้ผู้จัดจำหน่ายสามารถตอบสนองกับรายการใบสั่งซื้อแต่ละรายการและแนะนำการเปลี่ยนแปลง | ในบางกรณี ผู้จัดจำหน่ายต้องการยอมรับรายการใบสั่งซื้อบางรายการ แต่ปฏิเสธรายการอื่นๆ ขณะนี้ผู้จัดจำหน่ายสามารถจัดการรายการใบสั่งซื้อทีละรายการได้ แต่ละรายการสามารถถูกปฏิเสธ ยอมรับ หรือยอมรับโดยมีการเปลี่ยนแปลง ตัวอย่างเช่น ผู้จัดจำหน่ายสามารถเปลี่ยนวันที่จัดส่ง แบ่งการจัดส่งและปริมาณ หรือแนะนำสินค้าสำรอง |
| อนุญาตให้ผู้จัดจำหน่ายสามารถจัดการข้อมูลผู้ติดต่อ | ผู้จัดจำหน่ายสามารถเก็บรักษาข้อมูลติดต่อสำหรับบริษัทของพวกเขา ข้อมูลนี้รวมถึงชื่อ ที่อยู่อีเมล และหมายเลขโทรศัพท์ การเข้าถึงคุณลักษณะนี้ได้รับอนุญาตผ่านบทบาทความปลอดภัยที่จัดสรรไว้ |
| เอกสารที่ใช้ร่วมกันที่เกี่ยวข้องกับใบสั่งซื้อกับผู้จัดจำหน่าย | เมื่อคุณต้องใช้เอกสารร่วมกับผู้จัดจำหน่าย เช่น เอกสารเกี่ยวกับข้อกำหนด มีความสะดวกสบายที่จะเชื่อมโยงเอกสารกับใบสั่งซื้อที่เกี่ยวข้อง จากนั้นผู้จัดจำหน่ายสามารถใช้บันทึกย่อและเอกสารแนบร่วมกันกับลูกค้าโดยการเชื่อมโยงเอกสารกับการตอบสนองของบุคคลดังกล่าวกับใบสั่งซื้อ การจัดการเอกสารเป็นกรอบงานที่สนับสนุนที่เน้น และเฉพาะบันทึกย่อและเอกสารแนบที่มีการจัดประเภทเป็น "ภายนอก" เท่านั้นที่สามารถใช้ร่วมกับผู้จัดจำหน่ายได้ |
| เตรียมใช้งานผู้ใช้ที่เป็นผู้จัดจำหน่ายใหม่ | ถ้าผู้จัดจำหน่ายของคุณใช้อินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่าย พวกเขาจะมีวิธีการที่ไร้ที่ติในการร้องขอบัญชีผู้ใช้ใหม่เมื่อผู้ติดต่อใหม่ต้องการการเข้าถึงการทำงานร่วมกันกับผู้จัดจำหน่าย ผู้เชี่ยวชาญด้านการจัดซื้อสามารถส่งคำขอสำหรับบัญชีผู้ใช้สำหรับผู้ติดต่อในองค์กรของผู้จัดจำหน่าย ผู้ติดต่อของผู้จัดจำหน่ายที่มีผู้ใช้การทำงานร่วมกันกับผู้จัดจำหน่ายอยู่แล้วสามารถส่งคำขอชนิดนี้ได้ ในที่สุดคำขอนี้สร้างผู้ใช้ใหม่ใน Dynamics 365 for Operations ที่มี Security role เฉพาะผู้จัดจำหน่าย นอกจากนี้ ยังช่วยอำนวยความสะดวกให้คำขอไปยังพอร์ทัล Microsoft Azure B2B ในการเตรียมใช้งานผู้ใช้ที่มีบัญชีผู้ใช้ Azure Active Directory (Azure AD) ใหม่ ผู้จัดจำหน่ายยังสามารถส่งคำขอให้ปิดใช้งานบัญชีผู้ใช้ที่เป็นผู้จัดจำหน่ายเฉพาะ หรือให้มีการปรับเปลี่ยนบทบาทความปลอดภัย |
| ค้นหาข้อมูลเพิ่มเติมเกี่ยวกับการสนับสนุนสำหรับการทำงานร่วมกันของผู้จัดจำหน่ายใน Dynamics 365 for Operations | ดูข้อมูลเพิ่มเติมเกี่ยวกับการทำงานร่วมกันกับผู้จัดจำหน่ายที่ [การทำงานร่วมกันกับผู้จัดจำหน่ายกับผู้จัดจำหน่ายภายนอก](../../../supply-chain/procurement/vendor-collaboration-work-external-vendors.md) [การทำงานร่วมกันกับผู้จัดจำหน่ายกับลูกค้า](../../../supply-chain/procurement/vendor-collaboration-work-customers-dynamics-365-operations.md) [จัดการผู้ใช้การทำงานร่วมกันกับผู้จัดจำหน่าย](../../../supply-chain/procurement/manage-vendor-collaboration-users.md) [ตั้งค่าและรักษาการทำงานร่วมกันกับผู้จัดจำหน่าย](../../../supply-chain/procurement/set-up-maintain-vendor-collaboration.md) และ [พื้นที่ทำงานการออกใบแจ้งหนี้ของการทำงานร่วมกันกับผู้จัดจำหน่าย](../../../finance/accounts-payable/vendor-portal-invoicing-workspace.md) |

### <a name="warehouse-enhancements-for-high-volume-distribution-centers"></a>การปรับปรุงคลังสินค้าสำหรับศูนย์การกระจายสินค้าปริมาณสูง

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| สร้างงานหลังจากที่มีการบรรจุสินค้าที่สถานีบรรจุ | คุณลักษณะนี้มีประโยชน์เมื่อมีขั้นตอนเพิ่มเติมหลังจากงานการบรรจุสินค้า (เช่น การวางบนแท่นวางสินค้า ตรวจสอบคุณภาพ การรวมการจัดส่ง หรือการเปลี่ยนแปลงท่าบรรทุก) ด้วยตนเอง ชนิดงาน **การเบิกสินค้าของตู้บรรจุสินค้าที่บรรจุแล้ว** ใหม่ช่วยให้คุณสามารถทำแบบจำลองงานเหล่านี้โดยใช้รายการงานที่แยกต่างหาก ขณะนี้คุณสามารถทำแบบจำลองสถานีบรรจุสินค้าเพื่อสร้างงานหลังจากการบรรจุสินค้า สามารถใช้งานนี้ในการจัดการความเคลื่อนไหวของสินค้าคงคลังที่บรรจุแล้ว |
| จัดกลุ่มตู้บรรจุสินค้าที่สถานีบรรจุสินค้า | คุณลักษณะนี้ทำให้สามารถจัดกลุ่มแพคเกจหลายรายการในคอนเทนเนอร์หรือป้ายทะเบียนเดียวที่สถานีบรรจุสินค้า ตัวอย่างเช่น ตัวดำเนินการพาณิชย์อิเล็กทรอนิกส์/การขายส่งสามารถจัดกลุ่มแต่ละแพคเก็ตที่บรรจุแล้ว 100 รายการเป็นคอนเทนเนอร์เดียว (เช่น แท่นวางสินค้าหรือกรง) จากนั้นสามารถย้าย แบ่งระยะ หรือโหลดคอนเทนเนอร์นี้โดยใช้คำสั่งงานเดียวที่ผู้ปฏิบัติงานสร้างโดยการสแกนบาร์โค้ด (ป้ายทะเบียน) เดียวสำหรับคอนเทนเนอร์ที่มีการจัดกลุ่ม คุณลักษณะนี้ช่วยลดขั้นตอนของธุรกรรมของงานสำหรับการย้ายคอนเทนเนอร์ที่บรรจุแล้วที่มีขนาดเล็กกว่าจำนวนมาก ดูข้อมูลเพิ่มเติมที่ [สร้างรายการเมนูอุปกรณ์เคลื่อนที่สำหรับการรวมป้ายทะเบียน (คู่มืองาน)](../../../supply-chain/warehousing/tasks/create-mobile-device-license-plate-consolidation.md) |
| สร้างนโยบายการนำออกใช้สำหรับตู้บรรจุสินค้าที่บรรจุแล้ว | งานที่ถูกทริกเกอร์โดยการนำคอนเทนเนอร์ออกใช้อาจถูกสร้างขึ้นโดยอัตโนมัติหรือด้วยตนเอง เมื่อทำงานโดยอัตโนมัติ งานจะถูกสร้างในคอนเทนเนอร์แบบปิดโดยใช้คำสั่งที่ตั้งและกรอบงานเท็มเพลตของงาน เมื่อการนำออกใช้ทำงานด้วยตนเอง ผู้บรรจุสินค้าสามารถตัดสินใจว่าควรสร้างงานสำหรับคอนเทนเนอร์เดียวหรือสำหรับกลุ่มคอนเทนเนอร์ คุณลักษณะนี้ช่วยลดความเสี่ยงที่คอนเทนเนอร์แบบปิดที่ถูกจะเบิกสินค้า แล้วย้ายไปก่อนที่จะพร้อมที่จะย้ายจากสถานีบรรจุ นอกจากนี้ยังใช้ได้กับการนำออกใช้ที่มีการจัดกลุ่มที่ระบบไม่ได้กำหนด |
| การปันส่วนการเบิกสินค้าที่ขาด | มีการเพิ่มการสนับสนุนสำหรับกระบวนการเบิกสินค้าที่ขาด เพื่อช่วยในการเชื่อมโยงที่ไม่สามารถเบิกสินค้า และต้องการเติมเต็มลำดับเมื่อสินค้าจำเป็นต้องพร้อมใช้งานในที่ตั้งอื่น คุณสามารถใช้กระบวนการปันส่วนโดยอัตโนมัติที่ใช้คำสั่งที่ตั้งเดียวกันในการดึงข้อมูลสินค้าถ้าพร้อมใช้งานในที่ตั้งอื่น อีกทางหนึ่งคือ เมื่อมีใช้การปันส่วนด้วยตนเอง อุปกรณ์เคลื่อนที่จะแสดงรายชื่อของสถานที่พร้อมกับปริมาณที่พร้อมใช้งาน จากนั้นผู้ปฏิบัติงานคลังสินค้าสามารถเลือกสถานที่ที่จะใช้สินค้าคงคลัง มีสองวิธีที่สามารถตั้งค่าทีละรายการหรือแบบรวมเป็นคำสั่งเดียวในเมนูของผู้ปฏิบัติงานคลังสินค้า เมื่อมีการใช้การปันส่วนใหม่ด้วยตนเอง ผู้ปฏิบัติงานคลังสินค้าสามารถเบิกสินค้าในปริมาณของสินค้าที่เบิกขาดจากหลายสถานที่ ดูข้อมูลเพิ่มเติมที่ [ตั้งค่าการปันส่วนสินค้าใหม่สำหรับการเบิกสินค้าที่ขาด (คู่มืองาน)](../../../supply-chain/warehousing/tasks/set-up-short-picking-item-reallocation.md) |
| ย้ายสินค้าคงคลังที่มีงานที่เชื่อมโยง | ขณะนี้คุณสามารถย้ายสินค้าคงคลังที่มีงานที่เชื่อมโยง คุณลักษณะนี้อาจเป็นประโยชน์ในกรณี เช่น ผู้ปฏิบัติงานต้องการล้างข้อมูลเนื้อที่ท่าสินค้าขาเข้า และย้ายแท่นวางสินค้าที่ลงทะเบียนที่กำลังรอที่จะย้ายไปยังที่ตั้งขาเข้าอื่น ท่าสินค้าจะถูกล้างข้อมูล และสามารถรับจำนวนงานในศูนย์การผลิตของสินค้าเพิ่มเติม และสินค้าคงคลังที่มีการย้ายจะได้รับการรับปรุงด้วยข้อมูลสำรองใหม่ สามารถใช้คุณลักษณะนี้เพื่อเพิ่มพื้นที่ว่างที่สถานที่เบิกสินค้าโดยการย้ายสินค้าคงคลังที่มีงานที่เกี่ยวข้องและการสร้างพื้นที่ว่างสำหรับการเติมสินค้า นอกจากนี้คุณยังสามารถใช้เพื่อย้ายโหลดจากสถานที่การแบ่งระยะไปยังอีกสถานที่หนึ่งได้โดยไม่สูญเสียงานการโหลดที่ถูกสร้างขึ้น ด้วยวิธีนี้ คุณลักษณะนี้สามารถช่วยปรับเวลาที่ต้องใช้ในการโหลดบรรทุกเมื่อมาถึงให้เหมาะสม คุณสามารถย้ายสินค้าคงคลังที่จองไว้สำหรับใบสั่งขาย การออกใบสั่งโอนย้าย ใบรับสินค้าใบสั่งโอนย้าย และใบสั่งซื้อ การโอนย้ายได้รับการป้องกันถ้าจะทำให้รายการถูกแบ่ง ตัวอย่างเช่น ถ้าคุณมีรายการงานสำหรับสินค้า 100 รายการ คุณจะไม่สามารถย้ายสินค้าเพียง 30 รายการไปยังสถานที่อื่นได้ |
| ผสานป้ายทะเบียนที่มีงานที่เกี่ยวข้อง | คุณสามารถผสานป้ายทะเบียนที่มีงานขาออกที่เกี่ยวข้อง ตัวอย่างเช่น เมื่อการเบิกสินค้ามีการแบ่งออกเป็นงานหลายรายการ สิ่งนี้อาจเป็นประโยชน์ในการอนุญาตให้ผสานป้ายทะเบียนหลังจากที่มีการจัดส่งที่สถานที่การแบ่งระยะ สามารถรวมป้ายทะเบียนเฉพาะเมื่ออยู่ที่ตำแหน่งเดียวกัน มาจากโหลดเดียวกัน และมีข้อมูลการจัดส่งเดียวกัน |
| หยุดงานการเบิกสินค้าที่เกี่ยวข้องกับการเพิ่มเติมสินค้าในระดับรายการ | ขณะนี้คุณสามารถหยุดงานในระดับรายการแทนที่ระดับหัวข้อ เพื่อให้ผู้ปฏิบัติงานสามารถเติมเต็มรายการเบิกสินค้าที่ไม่ได้ถูกหยุด ตามการเพิ่มเติมสินค้าตามความต้องการ ดังนั้น ผู้ขายส่งที่มีใบสั่งขายขนาดใหญ่ หรือผู้ค้าปลีกที่มีใบสั่งขายหรือใบสั่งโอนย้ายการเติมสินค้าขนาดใหญ่ สามารถอนุญาตให้เริ่มต้นงานการเบิกสินค้าในรายการทั้งหมดที่ไม่ได้ขึ้นอยู่กับงานการเติมสินค้า งานการเบิกสินค้าและการเติมสินค้าสามารถกระทำได้พร้อมกัน สามารถตั้งค่าคอนฟิกคุณลักษณะนี้เพื่อให้คุณสามารถเลือกว่าจะหยุดในระดับส่วนหัวหรือระดับรายการ นอกจากนี้คุณยังสามารถใช้ทั้งสองวิธีและสร้างเท็มเพลตงานสำหรับแต่ละวิธี การตั้งค่าคอนฟิกหัวข้อ/รายการถูกตั้งค่าในเท็มเพลตงาน แต่คุณสามารถเปลี่ยนได้โดยตรงในงานที่ถูกสร้างขึ้น |
| ยกเลิกงานโดยใช้อุปกรณ์เคลื่อนที่ | ขณะนี้คุณสามารถยกเลิกงานได้โดยใช้อุปกรณ์เคลื่อนที่ โดยมีหรือไม่มีการเพิ่มเติมสินค้าตามความต้องการก็ได้ ก่อนหน้านี้ คุณจะต้องใช้ไคลเอนต์ที่สมบูรณ์ เพื่อเปิดใช้งานฟังก์ชันนี้ ให้สร้างรายการเมนูอุปกรณ์เคลื่อนที่มีการตั้งค่าโหมดเป็น **ทางอ้อม** และรหัสกิจกรรมตั้งค่าเป็น **ยกเลิกงาน** |

### <a name="warehouse-operation-enhancements"></a>การปรับปรุงสำหรับการดำเนินงานคลังสินค้า

| สิ่งที่คุณสามารถทำได้ | ทำไมนี่เป็นสิ่งสำคัญ |
|-----------------|-----------------------|
| ทำแบบจำลองชนิดคอนเทนเนอร์ที่แตกต่างกัน | คุณอาจใช้ชนิดคอนเทนเนอร์ต่างๆ ในคลังสินค้าเพื่อเพิ่มประสิทธิภาพที่เก็บข้อมูล และเพื่อเหตุผลอื่นๆ เอนทิตี้ชนิดคอนเทนเนอร์ใหม่มีลักษณะทางกายภาพของชนิดคอนเทนเนอร์ ขณะนี้คุณสามารถเชื่อมโยงป้ายทะเบียนกับชนิดคอนเทนเนอร์เฉพาะ และใช้ขีดจำกัดการเก็บสต็อกในสถานที่ ตัวอย่างเช่น คุณสามารถใช้คุณลักษณะนี้เพื่อควบคุมจำนวนแท่นวางสินค้า (หรือคอนเทนเนอร์ชนิดอื่นๆ) ที่คุณอนุญาตให้มีในที่สถานที่ที่ระบุได้ ชนิดคอนเทนเนอร์ยังถูกเพิ่มไปยังกลุ่มลำดับหน่วยเพื่อเพิ่มชนิดคอนเทนเนอร์เริ่มต้นสำหรับกระบวนการรับสินค้า สามารถใช้ชนิดคอนเทนเนอร์กับคำสั่งสถานที่ขาเข้าและขาออก และยังสามารถใช้ในมุมมองปริมาณคงคลังคงเหลือเพื่อช่วยให้คุณกำหนดจำนวนของชนิดคอนเทนเนอร์ที่ถูกจัดเก็บอยู่ในขณะนี้ ดูข้อมูลเพิ่มเติมที่ประกาศบล็อก [การใช้ป้ายทะเบียนที่เกี่ยวข้องกับชนิดคอนเทนเนอร์เพื่อขับเคลื่อนกระบวนการจัดการคลังสินค้า](https://blogs.msdn.microsoft.com/dynamicsaxscm/2016/06/20/use-of-license-plates-associated-with-a-container-type-to-drive-warehouse-management-processes/) ถึงแม้ว่าโพสต์บล็อกนี้อธิบายการปรับปรุงไปยัง Microsoft Dynamics AX 2012 ขณะนี้ การสนับสนุนเดียวกันได้ถูกเพิ่มไปยัง Dynamics 365 for Operations |
| การกลับรายการการจัดส่ง | ในคลังสินค้า คุณต้องสามารถจัดการกับการเปลี่ยนแปลงที่ล่าช้า ตัวอย่างเช่น สินค้าบางรายการอาจเสียหาย ดังนั้นคุณจึงไม่สามารถจัดส่งได้ อีกทางหนึ่งคือ ลูกค้าอาจทำการร้องขอที่ล่าช้าเพื่อยกเลิกหรือเปลี่ยนแปลงใบสั่ง Dynamics 365 for Operations ขณะนี้อนุญาตให้คุณกลับรายการการจัดส่งได้ ดังนั้น คุณสามารถยกเลิกบันทึกการจัดส่งเพื่อให้คุณสามารถปรับปรุงปริมาณที่ถูกต้องในภายหลัง ในทำนองเดียวกัน ในลำดับขาเข้า คุณสามารถยกเลิกใบรับสินค้าเพื่อให้สามารถสร้างรุ่นที่ปรับปรุงแล้ว |
| ใช้แท่นวางสินค้าที่มีสินค้าผสม | ขณะนี้คุณสามารถรับและลงทะเบียนแท่นวางสินค้า "ผสม" ได้ แท่นวางสินค้าผสมประกอบด้วยสินค้าที่แตกต่างกันที่ประกอบเป็นแท่นวางสินค้าหนึ่งรายการสำหรับใบสั่งซื้อหรือรายการอย่างน้อยหนึ่งรายการ สามารถสรุปการลงทะเบียนทั้งหมดได้ในป้ายทะเบียนเป้าหมายเดียว กระบวนการนี้จะเปิดใช้งานผ่านอุปกรณ์เคลื่อนที่ของคลังสินค้า ตัวอย่างเช่น เมื่อแท่นวางสินค้าของสินค้ามาถึงที่คลังสินค้า เจ้าหน้าที่รับสินค้าจะระบุสินค้าและปริมาณบนแท่นวางสินค้าก่อนที่จะย้ายแท่นวางสินค้าไปยังสถานที่เก็บเฉพาะ มีการระบุสถานที่เก็บตามเท็มเพลตงานและคำสั่งสถานที่ ถ้าสถานที่เก็บครอบคลุมหลายรายการที่มีสถานที่คงที่ คุณลักษณะนี้จะสร้างรายการจัดวางสินค้ามากเท่าที่มีสินค้าต่างๆ บนแท่นวางสินค้าแบบผสม คุณลักษณะนี้ทำให้การลงทะเบียนและการสำรองแท่นวางสินค้าแบบผสมที่ได้รับเร็วและยืดหยุ่นขึ้น ดูข้อมูลเพิ่มเติมที่ประกาศบล็อก [รับและลงทะเบียนแท่นวางสินค้าที่มีรายการเอกสารแหล่งที่มาแบบผสมโดยใช้ป้ายทะเบียนเดียว](https://blogs.msdn.microsoft.com/dynamicsaxscm/2016/05/13/receive-and-register-a-pallet-with-mixed-source-document-lines-using-1-license-plate-purchase-order-matching/) และข้อมูลเกี่ยวกับการสนับสนุนแท่นวางสินค้าแบบผสม ใน [ประกาศการอัพเดตแบบสะสมของเรา](https://blogs.msdn.microsoft.com/dynamicsaxscm/2016/06/30/whats-new-in-cu11-for-wms-and-tms/) ถึงแม้ว่าโพสต์บล็อกนี้อธิบายการปรับปรุงไปยัง AX 2012 ขณะนี้ การสนับสนุนเดียวกันได้ถูกเพิ่มไปยัง Dynamics 365 for Operations |
