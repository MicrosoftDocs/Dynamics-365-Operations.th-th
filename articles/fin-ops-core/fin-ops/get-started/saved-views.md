-    <span data-ttu-id="921c9-117">ซึ่งแตกต่างจากการตั้งค่าส่วนบุคคลแบบดั้งเดิม มุมมองจะไม่ถูกบันทึกโดยอัตโนมัติ เมื่อผู้ใช้ดำเนินการการตั้งค่าส่วนบุคคลที่ชัดเจนหรือกรองรายการ</span><span class="sxs-lookup"><span data-stu-id="921c9-117">Unlike traditional personalization, views are not automatically saved when a user performs explicit personalizations or filters a list.</span></span> <span data-ttu-id="921c9-118">ต้องมีการบันทึกที่ชัดเจนเพื่อให้ความยืดหยุ่นในการสร้างมุมมองก่อน หรือหลังจากที่มีการทำการเปลี่ยนแปลงที่เกี่ยวข้องกับมุมมองนั้น และเพื่อตรวจสอบให้แน่ใจว่ามีข้อกำหนดมุมมองไม่ถูกสลับโดยไม่ได้ตั้งใจโดยตัวกรอง หรือการตั้งค่าส่วนบุคคลที่ไม่ได้มีไว้สำหรับการใช้งานในระยะยาว</span><span class="sxs-lookup"><span data-stu-id="921c9-118">Explicit saves are required to provide flexibility in creating a view before or after the changes associated with that view have been made and to ensure that view definitions are not unintentionally altered by filters or personalizations that are not intended for long-term use.</span></span>

<span data-ttu-id="921c9-122">หลังจากที่เปิดใช้งานมุมมองสำหรับสภาพแวดล้อมแล้ว หน้าใดก็ตามที่สนับสนุนมุมมองจะรวมตัวควบคุมมุมมองที่ยุบแล้วไว้ที่ด้านบนของแบบฟอร์มซึ่งแสดงชื่อของมุมมองปัจจุบัน</span><span class="sxs-lookup"><span data-stu-id="921c9-122">After views have been enabled for an environment, any page that supports views will include a collapsed view selector control at the top of the form that shows the name of the current view.</span></span>

<span data-ttu-id="921c9-308">สำหรับหน้าที่มีตัวค้นหามุมมองเล็กๆ (เฉพาะบุคคลเท่านั้นที่สามารถบันทึกไปยังมุมมอง) คุณจะเห็นข้อมูลเดียวกันกับที่คุณมีอยู่เสมอเมื่อคุณไปที่หน้า</span><span class="sxs-lookup"><span data-stu-id="921c9-308">For pages with small view selectors (only personalizations can be saved to the view), you will see the same data as you always have when you visit the page.</span></span> 

<span data-ttu-id="921c9-309">สำหรับหน้าที่มีตัวแบ่งมุมมองที่มีขนาดใหญ่ (บุคคลและการสอบถามสามารถบันทึกไปยังมุมมอง) คุณจะเห็นข้อมูลที่เชื่อมโยงกับการสอบถามที่เชื่อมโยงกับมุมมองเริ่มต้นของคุณเป็นหลัก</span><span class="sxs-lookup"><span data-stu-id="921c9-309">For pages with large view selectors (personalizations and queries can be saved to the view), you will primarily see the data linked to the query associated with your default view.</span></span> <span data-ttu-id="921c9-310">มี 2 ข้อยกเว้นหลักๆ: - ถ้าคุณนำทางไปยังหน้าจากไทล์ การสอบถามของไทล์จะดำเนินการโดยไม่คำนึงถึงการสอบถามที่เชื่อมโยงกับมุมมองเริ่มต้น</span><span class="sxs-lookup"><span data-stu-id="921c9-310">There are two main exceptions to this: - If you navigate to a page from a tile, the tile query will execute regardless of the query associated with the default view.</span></span> <span data-ttu-id="921c9-311">ถ้าคุณสร้างไทล์ดังกล่าวหลังจากที่เปิดใช้งานมุมมองแล้ว การเลือกไทล์จะเปิดหน้าที่มีมุมมองที่เกี่ยวข้องกับไทล์นั้น</span><span class="sxs-lookup"><span data-stu-id="921c9-311">If you created that tile after views have been enabled, selecting a tile will open the page with the view associated with that tile.</span></span>   
     <span data-ttu-id="921c9-312">- ถ้าคุณนำทางไปยังหน้าและจุดเข้าใช้งานนั้นมีการสอบถาม การสอบถามดั้งเดิมจะดำเนินการแทนการสอบถามของมุมมองเริ่มต้นในครั้งแรก</span><span class="sxs-lookup"><span data-stu-id="921c9-312">- If you navigate to a page and that entry point includes a query, the original query will execute originally in place of the default view's query.</span></span> <span data-ttu-id="921c9-313">คุณควรจะได้รับการแจ้งเตือนเมื่อเหตุการณ์เกิดขึ้นโดยข้อความที่ให้ข้อมูล เมื่อกำลังโหลดมุมมอง</span><span class="sxs-lookup"><span data-stu-id="921c9-313">You should be alerted when this occurs via an informational message when the view is loading.</span></span> <span data-ttu-id="921c9-314">นอกจากนี้ คุณยังสามารถยืนยันได้โดยสลับไปที่มุมมองนี้ หลังจากที่โหลดหน้า เนื่องจากรายการนั้นควรอนุญาตให้การสอบถามมุมมองสามารถดำเนินการโดยไม่คำนึง</span><span class="sxs-lookup"><span data-stu-id="921c9-314">You can also confirm by switching to this view after the page loads, as that should allow the view query to execute regardless.</span></span>
