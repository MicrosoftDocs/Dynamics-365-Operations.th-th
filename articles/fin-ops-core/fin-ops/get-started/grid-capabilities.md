# <a name="grid-capabilities"></a>ความสามารถของกริด

[!include [banner](../includes/banner.md)]

ตัวควบคุมกริดใหม่มีความสามารถในการใช้งานที่มีประโยชน์และมีประสิทธิภาพ ซึ่งคุณสามารถใช้ในการปรับปรุงประสิทธิภาพการทำงานของผู้ใช้ สร้างมุมมองที่น่าสนใจต่างๆ ของข้อมูลของคุณ และรับข้อมูลเชิงลึกสำคัญในข้อมูลของคุณได้ บทความนี้จะครอบคลุมความสามารถต่อไปนี้: 

- การคำนวณยอดรวม
- การพิมพ์ล่วงหน้าของระบบ
- การประเมินนิพจน์ทางคณิตศาสตร์ 
- การจัดกลุ่มข้อมูลแบบตาราง (เปิดใช้งานแยกต่างหากโดยใช้คุณลักษณะ **การจัดกลุ่มในตาราง**)
- การตรึงคอลัมน์ (เปิดใช้งานแยกต่างหากโดยใช้คุณลักษณะ **การตรึงคอลัมน์ในกริด**)
- ปรับความกว้างของคอลัมน์ให้พอดีอัตโนมัติ
- คอลัมน์ที่ยืดได้

## <a name="calculating-totals"></a>การคำนวณยอดรวม
ในแอปการเงินและการดำเนินงาน ผู้ใช้มีความสามารถในการดูผลรวมที่ด้านล่างของคอลัมน์ตัวเลขในกริด ส่วนท้ายที่ด้านล่างของกริด แสดงผลรวมเหล่านี้ 

### <a name="showing-the-grid-footer"></a>การแสดงส่วนท้ายของกริด
มีพื้นที่ท้ายกระดาษที่ด้านล่างของกริดตารางทั้งหมดในแอปการเงินและการดำเนินงาน ส่วนท้ายสามารถแสดงข้อมูลที่เป็นประโยชน์ซึ่งเกี่ยวข้องกับข้อมูลที่ปรากฏในกริด ต่อไปนี้เป็นตัวอย่างของข้อมูลนี้:

- จำนวนแถวที่เลือกในตาราง (เมื่อคุณเรกคอร์ดมากกว่าหนึ่งเรกคอร์ด)
- ผลรวมสุดท้ายที่ด้านล่างของคอลัมน์ตัวเลขที่ตั้งค่าคอนฟิก
- จำนวนแถวในชุดข้อมูล 

ส่วนท้ายนี้ถูกซ่อนไว้โดยค่าเริ่มต้น แต่คุณสามารถเปิดใช้งานได้ เมื่อต้องการแสดงส่วนท้ายของกริด ให้เลือกปุ่ม **ตัวเลือกกริด** ในส่วนหัวของกริด และเลือกตัวเลือก **แสดงส่วนท้าย** หลังจากที่คุณเปิดส่วนท้ายของกริดหนึ่งๆ การตั้งค่านั้นจะถูกจำจนกว่าผู้ใช้จะเลือกที่จะซ่อนส่วนท้าย เมื่อต้องการซ่อนส่วนท้าย ให้เลือก **ซ่อนส่วนท้าย** บนเมนู **ตัวเลือกกริด**

### <a name="specifying-columns-with-totals"></a>การระบุคอลัมน์ที่มียอดรวม
ปัจจุบัน ไม่มีคอลัมน์แสดงผลรวมโดยค่าเริ่มต้น การทำเช่นนี้ถือเป็นกิจกรรมการตั้งค่าแบบครั้งเดียวแทน คล้ายกับการปรับปรุงความกว้างของคอลัมน์ในกริด เมื่อคุณระบุว่าคุณต้องการดูยอดรวมสำหรับคอลัมน์ การตั้งค่านั้นจะจดจำในครั้งต่อไปที่คุณเข้าชมหน้า

มีสองวิธีในการตั้งค่าคอนฟิกคอลัมน์เพื่อแสดงยอดรวมดังนี้: 

- คลิกขวาในคอลัมน์ที่คุณต้องการดูยอดรวม และจากนั้น เลือก **ยอดรวมคอลัมน์นี้** การดำเนินการนี้ทำให้เกิดเหตุการณ์สามประการดังนี้:

    1. ส่วนท้ายจะปรากฏขึ้น 
    2. การกำหนดลักษณะของคุณสำหรับการดูยอดรวมสำหรับคอลัมน์นี้ถูกบันทึก 
    3. การคำนวณยอดรวมเริ่มต้นขึ้นสำหรับคอลัมน์นี้ และคอลัมน์อื่นๆ ที่คุณได้ตั้งค่าคอนฟิกไว้ก่อนหน้านี้เพื่อดูยอดรวม เวลาที่ต้องใช้ในการแสดงยอดรวมจะขึ้นอยู่กับขนาดของชุดข้อมูลที่คุณกำลังรวมยอด

- หลังจากมองเห็นส่วนท้ายแล้ว ให้เลือก **แสดงผลรวม** ในพื้นที่ท้ายกระดาษที่ด้านล่างของคอลัมน์ที่คุณต้องการดูผลรวม ถ้าไม่มีคอลัมน์ที่มีการตั้งค่าคอนฟิกไว้ ปุ่ม **แสดงยอดรวม** จะพร้อมใช้งานสำหรับคอลัมน์ตัวเลขทั้งหมด 

    หลังจากที่มีการตั้งค่าคอนฟิกอย่างน้อยหนึ่งคอลัมน์สำหรับยอดรวม ปุ่ม **แสดงยอดรวม** จะพร้อมใช้งานบนโฮเวอร์หรือโฟกัสเท่านั้น การดำเนินการเลือก **แสดงยอดรวม** จะบันทึกเฉพาะการกำหนดลักษณะของคุณสำหรับการดูผลรวมในคอลัมน์นี้ เพื่อให้การกำหนดลักษณะถูกนำไปใช้ในระหว่างการเยี่ยมชมในอนาคตไปยังหน้า ในส่วนท้าย สถานะนี้จะบ่งชี้โดยเส้นประที่ปรากฏในคอลัมน์ (หรือถ้าชุดข้อมูลมีขนาดเล็กพอ จะมีการแสดงผลรวมทันที)

ถ้าคุณทำผิดพลาดและไม่ต้องการดูยอดรวมทั้งหมดในคอลัมน์หนึ่งๆแล้วให้คลิกขวาบนคอลัมน์และเลือก **ซ่อนยอดรวม** หรือเลือกปุ่ม **ซ่อนยอดรวม** ในส่วนท้ายของคอลัมน์นั้น การกำหนดลักษณะนี้จะถูกบันทึกไว้สำหรับการเยี่ยมชมในอนาคตไปยังหน้า 

### <a name="calculating-totals"></a>การคำนวณยอดรวม
เมื่อคุณมายังหน้าที่มีส่วนท้ายที่มองเห็นและคอลัมน์มีการตั้งค่าคอนฟิกสำหรับยอดรวม ยอดรวมอาจหรือไม่อาจแสดงอยู่ในส่วนท้าย ลักษณะการทำงานจะขึ้นอยู่กับขนาดของชุดข้อมูลบนหน้า ถ้าชุดข้อมูลมีขนาดเล็กเพียงพอ ยอดรวมจะแสดงขึ้นโดยอัตโนมัติพร้อมด้วยจำนวนแถวในชุดข้อมูล ถ้ามีเส้นประในส่วนท้ายของคอลัมน์ที่คุณตั้งค่าคอนฟิกไว้สำหรับยอดรวมแล้ว แสดงว่าชุดข้อมูลมีขนาดใหญ่เกินไปสำหรับระบบเพื่อแสดงยอดรวมทันทีและต้องมีการดำเนินการที่ชัดเจนเพื่อคำนวณยอดรวม เมื่อต้องการทำเช่นนี้ให้คลิกปุ่ม **คำนวณ** ในท้ายกระดาษหรือคลิกขวาบนคอลัมน์ที่คุณต้องการให้ยอดรวมและเลือก **ยอดรวมคอลัมน์นี้**

ถ้าการคำนวณใช้เวลานานกว่าจะเสร็จสมบูรณ์ คุณสามารถยกเลิกการดำเนินงานได้โดยเลือกปุ่ม **ยกเลิก** ในบางครั้งชุดข้อมูลจะมีขนาดใหญ่เกินกว่าที่จะคำนวณยอดรวม (ข้อจำกัดที่กำหนดโดยองค์กรของคุณ) และคุณจะได้รับการแจ้งเตือนให้กรองข้อมูลของคุณเพิ่มเติมแทน 

> [!NOTE]
> การจัดการระบบสามารถปรับเปลี่ยนขีดจํากัดของจํานวนเรกคอร์ดที่มีอยู่เพื่อคํานวณยอดรวมโดยการปรับปรุงพารามิเตอร์ **จำนวนสูงสุดของเรกคอร์ดภายในสหรับแต่ละกริด** ในหน้า **ตัวเลือกประสิทธิภาพของไคลเอ็นต์** ค่าเริ่มต้นคือ 25,000 เรกคอร์ด ผู้ดูแลระบบควรระวังเมื่อปรับปรุงค่านี้ เนื่องจากค่าที่มีขนาดใหญ่เกินไปอาจทำให้สูญเสียหน่วยความจําที่มีอยู่บนเครื่องของผู้ใช้ ขอแนะนำว่าไม่ควรเกิน 50,000 เรกคอร์ด   

ยอดรวมจะอัปเดตโดยอัตโนมัติเมื่อคุณอัปเดต ลบ หรือสร้างแถวในชุดข้อมูล

## <a name="typing-ahead-of-the-system"></a>การพิมพ์ล่วงหน้าของระบบ
ในสถานการณ์ธุรกิจจำนวนมาก ความสามารถในการป้อนข้อมูลลงในระบบได้อย่างรวดเร็วมีความสำคัญมาก ก่อนที่จะใช้ตัวควบคุมกริดใหม่ ผู้ใช้สามารถเปลี่ยนแปลงข้อมูลได้เฉพาะในแถวปัจจุบันเท่านั้น ก่อนที่จะสามารถสร้างแถวใหม่หรือสลับไปยังแถวอื่นได้ มีการบังคับให้รอให้ระบบตรวจสอบความถูกต้องของการเปลี่ยนแปลงใดๆ ให้เสร็จสมบูรณ์ ในความพยายามที่จะลดจำนวนเวลาที่ผู้ใช้รอการตรวจสอบความถูกต้องเหล่านี้ให้เสร็จสมบูรณ์ และเพื่อปรับปรุงประสิทธิภาพการทำงานของผู้ใช้ กริดใหม่จะปรับปรุงการตรวจสอบความถูกต้องเหล่านี้เพื่อให้เป็นแบบอะซิงโครนัส ดังนั้น ผู้ใช้สามารถย้ายไปยังแถวอื่นเพื่อทำการเปลี่ยนแปลง ในขณะที่การตรวจสอบความถูกต้องของแถวก่อนหน้านี้ค้างอยู่ 

เมื่อต้องการสนับสนุนลักษณะการทำงานใหม่นี้ คอลัมน์ใหม่สำหรับสถานะแถวถูกเพิ่มเข้าไปด้านขวาของคอลัมน์การเลือกแถว เมื่อกริดอยู่ในโหมดแก้ไข คอลัมน์นี้จะบ่งชี้หนึ่งในสถานะต่อไปนี้:

- **ว่างเปล่า** – ไม่มีรูปภาพสถานะบ่งชี้ว่าแถวถูกบันทึกโดยระบบเสร็จเรียบร้อยแล้ว
- **การประมวลผลที่ค้างอยู่** – สถานะนี้จะบ่งชี้ว่าการเปลี่ยนแปลงในแถวยังไม่ได้รับการบันทึกโดยเซิร์ฟเวอร์ แต่อยู่ในคิวของการเปลี่ยนแปลงที่ต้องมีการประมวลผล ก่อนที่คุณจะดำเนินการภายนอกกริด คุณต้องรอให้มีการประมวลผลการเปลี่ยนแปลงที่ค้างอยู่ทั้งหมด นอกจากนี้ ข้อความในแถวเหล่านี้จะถูกทำเป็นตัวเอียงเพื่อระบุสถานะที่ไม่ได้บันทึกไว้ของแถว 
- **สถานะไม่ถูกต้อง** – สถานะนี้บ่งชี้ว่ามีการทริกเกอร์คำเตือนหรือข้อความบางอย่างในระหว่างการประมวลผลแถว และอาจทำให้ระบบไม่สามารถบันทึกการเปลี่ยนแปลงในแถวนั้นได้ ในกริดเก่า หากการดำเนินการบันทึกไม่สำเร็จ คุณถูกบังคับให้กลับเข้าไปยังแถวเพื่อแก้ไขปัญหานี้โดยทันที อย่างไรก็ตาม ในกริดใหม่ คุณจะได้รับการแจ้งเตือนว่าเกิดปัญหาการตรวจสอบความถูกต้อง แต่คุณสามารถเลือกได้ว่าคุณต้องการแก้ไขปัญหาใดๆ ในแถวเมื่อใด เมื่อคุณพร้อมที่จะแก้ไขปัญหา คุณสามารถย้ายโฟกัสกลับไปยังแถวด้วยตนเองได้ อีกทางหนึ่งคือ คุณสามารถเลือกการดำเนินการ **แก้ไขปัญหานี้** ได้ การดำเนินการนี้จะย้ายโฟกัสกลับไปยังแถวที่มีปัญหาโดยทันที และช่วยให้คุณสามารถแก้ไขภายในหรือภายนอกกริดได้ โปรดทราบว่าการประมวลผลแถวที่ค้างอยู่ในลำดับต่อมาอจะหยุดดำเนินการ จนกว่าคำเตือนการตรวจสอบความถูกต้องนี้จะได้รับการแก้ไข 
- **หยุดชั่วคราว** – สถานะนี้บ่งชี้ว่าการประมวลผลโดยเซิร์ฟเวอร์ถูกหยุดชั่วคราว เนื่องจากการตรวจสอบความถูกต้องของแถวทริกเกอร์กล่องโต้ตอบที่ต้องมีการป้อนข้อมูลของผู้ใช้ เนื่องจากผู้ใช้อาจกำลังป้อนข้อมูลในแถวอื่นๆ กล่องโต้ตอบป็อปอัปจะไม่ได้รับการนำเสนอให้กับผู้ใช้รายนั้นในทันที แต่จะแสดงขึ้นเมื่อผู้ใช้เลือกที่จะดำเนินการประมวลผลต่อไป สถานะนี้จะมาพร้อมกับการแจ้งเตือนซึ่งจะแจ้งให้ผู้ใช้ทราบเกี่ยวกับสถานการณ์ การแจ้งเตือนรวมถึงการดำเนินการ **ดำเนินการประมวลผลต่อ** ซึ่งจะทริกเกอร์กล่องโต้ตอบป็อปอัป

เมื่อผู้ใช้กำลังป้อนข้อมูลก่อนสถานที่ที่เซิร์ฟเวอร์ประมวลผล ผู้ใช้จะสามารถคาดหวังการลดประสิทธิภาพบางอย่างในประสบการณ์การป้อนข้อมูล เช่น การขาดการค้นหา การตรวจสอบความถูกต้องของระดับการควบคุม และการป้อนค่าเริ่มต้น ผู้ใช้ที่จำเป็นต้องใช้รายการแบบหล่นลงในการค้นหาค่า ได้รับการแนะนำให้รอให้เซิร์ฟเวอร์เข้าถึงแถวปัจจุบัน นอกจากนี้ การตรวจสอบความถูกต้องระดับการควบคุมและรายการของค่าเริ่มต้น จะเกิดขึ้นเมื่อเซิร์ฟเวอร์ประมวลผลแถวนั้น

### <a name="pasting-from-excel"></a>การวางจาก Excel
ผู้ใช้มีความสามารถในการส่งออกข้อมูลจากกริดในแอปการเงินและการดำเนินงานไปยัง Microsoft Excel เสมอโดยใช้กลไก **ส่งออกไปยัง Excel** อย่างไรก็ตาม ความสามารถในการป้อนข้อมูลก่อนระบบจะเปิดใช้งานกริดใหม่ เพื่อสนับสนุนการคัดลอกตารางจาก Excel และวางตารางเหล่านั้นโดยตรงลงในกริดในแอปการเงินและการดำเนินงาน เซลล์กริดที่การดำเนินการวางเริ่มต้นมา จะกำหนดตำแหน่งที่ซึ่งตารางที่คัดลอกเริ่มต้นที่จะถูกวาง เนื้อหาของกริดถูกเขียนทับโดยเนื้อหาของตารางที่คัดลอก ยกเว้นในสองกรณี:

- ถ้าจำนวนของคอลัมน์ในตารางที่มีการคัดลอกเกินกว่าจำนวนของคอลัมน์ที่คงเหลือในกริด โดยเริ่มต้นจากสถานที่วาง ผู้ใช้จะได้รับการแจ้งเตือนว่าคอลัมน์พิเศษถูกละเว้น 
- ถ้าจำนวนแถวในตารางที่ถูกคัดลอกมากเกินกว่าจำนวนแถวในกริด โดยเริ่มต้นจากสถานที่วาง เซลล์ที่มีอยู่จะถูกเขียนทับโดยเนื้อหาที่วาง และแถวพิเศษใดๆ จากตารางที่คัดลอกจะถูกแทรกเป็นแถวใหม่ที่ด้านล่างของกริด 

## <a name="evaluating-math-expressions"></a>การประเมินนิพจน์ทางคณิตศาสตร์
ในฐานะที่เป็นผู้สนับสนุนประสิทธิภาพ ผู้ใช้สามารถป้อนสูตรทางคณิตศาสตร์ในเซลล์ตัวเลขในกริด ไม่จำเป็นต้องทำการคำนวณในแอปภายนอกระบบ ตัวอย่างเช่น ถ้าคุณป้อน **=15\*4** แล้วกดคีย์ **แท็บ** เพื่อย้ายออกจากฟิลด์ ระบบจะประเมินนิพจน์และบันทึกค่าเป็น **60** สำหรับฟิลด์

เมื่อต้องการให้ระบบจดจำค่าเป็นนิพจน์ ให้เริ่มต้นค่าโดยใช้เครื่องหมายเท่ากับ (**=**) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับผู้ปฏิบัติงานและไวยากรณ์ที่ได้รับการสนับสนุน โปรดดูที่ [สัญลักษณ์คณิตศาสตร์ที่สนับสนุน](http://bugwheels94.github.io/math-expression-evaluator/#supported-maths-symbols)

## <a name="grouping-tabular-data"></a>การจัดกลุ่มข้อมูลแบบตาราง
ผู้ใช้ทางธุรกิจมักต้องทำการวิเคราะห์เฉพาะกิจของข้อมูล ในขณะที่การดำเนินการนี้สามารถทำได้โดยการส่งออกข้อมูลไปยัง Microsoft Excel และใช้ตาราง Pivot คุณลักษณะ **การจัดกลุ่มในเส้นตาราง** ซึ่งขึ้นอยู่กับคุณลักษณะการควบคุมตารางใหม่ซึ่งช่วยให้ผู้ใช้สามารถจัดระเบียบข้อมูลแบบตารางได้ในวิธีที่น่าสนใจภายในแอปการเงินและการดำเนินงาน เพราะคุณลักษณะนี้ขยายคุณลักษณะ **ยอดรวม** **การจัดกลุ่ม** จะช่วยให้คุณสามารถรับข้อมูลเชิงลึกที่มีความสำคัญไปยังข้อมูลโดยการจัดหายอดรวมย่อยที่ระดับกลุ่ม

เมื่อต้องการใช้ลักษณะการทำงานนี้ให้คลิกขวาที่คอลัมน์ที่คุณต้องการจัดกลุ่มตามแล้วเลือก **จัดกลุ่มตามคอลัมน์นี้** การดำเนินการนี้จะเรียงลำดับข้อมูลตามคอลัมน์ที่เลือก เพิ่มคอลัมน์ **กลุ่มตาม** ใหม่ไปที่จุดเริ่มต้นไปยังกริด และแทรก "แถวส่วนหัว" ที่จุดเริ่มต้นของกลุ่มแต่ละกลุ่ม แถวส่วนหัวเหล่านี้จะให้ข้อมูลเกี่ยวกับแต่ละกลุ่มต่อไปนี้:

- ค่าข้อมูลสำหรับกลุ่ม 
- ชื่อคอลัมน์ (ข้อมูลนี้จะมีประโยชน์โดยเฉพาะอย่างยิ่งเมื่อคุณมีการจัดกลุ่มหลายระดับ)
- จำนวนของแถวข้อมูลในกลุ่มนี้
- ยอดรวมย่อยสำหรับคอลัมน์ใดๆที่ตั้งค่าคอนฟิกให้แสดงยอดรวม

เมื่อเปิดใช้งาน [มุมมองที่บันทึกไว้](saved-views.md) คุณจะสามารถบันทึกการจัดกลุ่มเป็นส่วนหนึ่งของมุมมองบนหน้าที่อนุญาตให้บันทึกการสอบถามไปยังมุมมองต่างๆ ได้ ตัวอย่างเช่น ผู้ที่เลือกตัวเลือกมุมมองขนาดใหญ่ ดูส่วน [การสลัระหว่างบมุมมอง](saved-views.md#switching-between-views) สำหรับรายละเอียดเพิ่มเติม 

### <a name="multiple-levels-of-grouping"></a>หลายระดับของการจัดกลุ่ม
หลังจากที่คุณได้จัดกลุ่มข้อมูลตามคอลัมน์เดียวแล้ว คุณสามารถจัดกลุ่มข้อมูลตามคอลัมน์อื่นได้โดยการเลือก **กลุ่มตามคอลัมน์นี้** บนคอลัมน์ที่ต้องการ กระบวนการนี้สามารถทำซ้ำได้จนกว่าคุณจะมีระดับการจัดกลุ่ม 5 ระดับซ้อนกันซึ่งเป็นความลึกที่สนับสนุนสูงสุด ณ จุดนี้ คุณจะไม่สามารถจัดกลุ่มตามคอลัมน์เพิ่มเติมได้อีกต่อไป

เมื่อใดก็ตาม คุณสามารถลบการจัดกลุ่มบนคอลัมน์ใดก็ได้โดยการคลิกขวาที่คอลัมน์นั้นแล้วเลือก **ยกเลิกการจัดกลุ่ม** นอกจากนี้คุณยังสามารถลบการจัดกลุ่มจากคอลัมน์ทั้งหมดด้วยการเลือก **ตัวเลือกกริด** แล้ว **ยกเลิกการจัดกลุ่มทั้งหมด**

### <a name="sorting-grouped-data"></a>การเรียงลำดับข้อมูลที่จัดกลุ่ม
หลังจากที่คุณจัดกลุ่มข้อมูลตามคอลัมน์หนึ่งคอลัมน์หรือมากกว่า คุณสามารถเปลี่ยนทิศทางของคอลัมน์ที่จัดกลุ่มผ่านส่วนหัวคอลัมน์ที่สอดคล้องกัน 

ลักษณะการทำงานเมื่อคุณเรียงลำดับคอลัมน์ที่ไม่มีการจัดกลุ่มจะขึ้นอยู่กับรุ่นผลิตภัณฑ์ของคุณดังนี้

- ในรุ่น 10.0.24 และก่อนหน้า ถ้าคุณเรียงลำดับคอลัมน์ที่ไม่มีการจัดกลุ่ม การจัดกลุ่มจะถูกลบออกจากคอลัมน์ทั้งหมด และข้อมูลจะถูกจัดเรียงตามคอลัมน์ที่เลือก 
- ในรุ่น 10.0.25 และรุ่นใหม่ ถ้าคุณเรียงลำดับคอลัมน์ที่ไม่มีการจัดกลุ่ม การจัดกลุ่มจะยังคงเดิม และข้อมูลจะถูกจัดเรียงภายในแต่ละกลุ่มตามคอลัมน์ที่เลือก

### <a name="expanding-and-collapsing-groups"></a>การขยายและการยุบกลุ่ม
การจัดกลุ่มเริ่มต้นของข้อมูลจะมีการขยายกลุ่มทั้งหมด คุณสามารถสร้างมุมมองสรุปของข้อมูลได้โดยการยุบกลุ่มแต่ละกลุ่ม หรือคุณสามารถใช้กลุ่มที่ขยายและยุบเพื่อช่วยในการนำทางไปยังข้อมูล เมื่อต้องการขยายหรือยุบกลุ่ม ให้เลือกปุ่มบั้ง (>) ในแถวส่วนหัวของกลุ่มที่สอดคล้องกัน โปรดสังเกตว่าสถานะขยาย/ยุบของกลุ่มแต่ละกลุ่มจะ **ไม่** ถูกบันทึกไว้ในการตั้งค่าส่วนบุคคล

### <a name="selecting-and-unselecting-rows-at-the-group-level"></a>การเลือกและการยกเลิกการเลือกแถวในระดับกลุ่ม
ในลักษณะเดียวกันกับที่คุณสามารถเลือก (หรือยกเลิกการเลือก) แถวทั้งหมดในตารางโดยการเลือกกล่องกาเครื่องหมายที่ด้านบนของคอลัมน์แรกในตาราง คุณยังสามารถเลือก (หรือยกเลิกการเลือก) แถวทั้งหมดในกลุ่มได้อย่างรวดเร็วได้โดยการเลือกกล่องกาเครื่องหมายในแถวส่วนหัวของกลุ่มที่สอดคล้องกัน กล่องกาเครื่องหมายในแถวส่วนหัวของกลุ่มจะสะท้อนถึงสถานะการเลือกปัจจุบันของแถวในกลุ่มเสมอ โดยไม่คำนึงว่ามีการเลือกแถวทั้งหมด ไม่มีการเลือกแถว หรือจะมีการเลือกแถวใดแถวหนึ่งเท่านั้น

### <a name="hiding-column-names"></a>การซ่อนชื่อคอลัมน์
เมื่อจัดกลุ่มข้อมูล ลักษณะการทำงานเริ่มต้นจะแสดงชื่อคอลัมน์ในแถวส่วนหัวของกลุ่ม คุณสามารถเลือกที่จะระงับชื่อคอลัมน์ในแถวส่วนหัวของกลุ่มได้โดยการเลือก **ตัวเลือกตาราง** > **ซ่อนชื่อคอลัมน์ของกลุ่ม**

### <a name="grouping-on-date-and-time-columns"></a>การจัดกลุ่มคอลัมน์วันที่และเวลา
ในรุ่น 10.0.24 หรือฟิลด์วันที่หรือวันที่เวลามีการเพิ่มตัวเลือกจัดกลุ่มตามปี เดือน หรือวัน "ค่า" กลุ่มในแถวส่วนหัวที่เกี่ยวข้องจะตรงกับรูปแบบจากฟิลด์นั้น นอกจากนี้ สำหรับฟิลด์วันที่เวลาและวันที่ คุณยังสามารถจัดกลุ่มตามชั่วโมง นาที หรือวินาทีได้ 

## <a name="freezing-columns"></a>การตรึงคอลัมน์
บางคอลัมน์ในกริดอาจสําคัญมากพอสำหรับบริบทที่คุณไม่ต้องการให้เลื่อนมุมมองออกไป แต่ คุณอาจต้องการให้ค่าในคอลัมน์เหล่านั้นมองเห็นได้เสมอ คุณลักษณะ **การตรึงคอลัมน์ในกริด** จะมีความยืดหยุ่นนี้ให้ผู้ใช้ 

เมื่อต้องการตรึงคอลัมน์ ให้คลิกขวาในส่วนหัวของคอลัมน์ แล้วเลือก **ตรึงคอลัมน์** ครั้งแรกที่คุณดำเนินการขั้นตอนนี้จนเสร็จสมบูรณ์ คอลัมน์ที่เลือกจะกลายเป็นคอลัมน์แรกและจะไม่เลื่อนออกจากมุมมองอีกต่อไป คอลัมน์ต่อมาใดๆ ที่คุณตรึง จะถูกเพิ่มที่ด้านขวาของคอลัมน์ที่ตรึงไว้ล่าสุด คุณสามารถใช้ฟังก์ชัน ย้าย มาตรฐาน เพื่อจัดลำดับคอลัมน์ที่ตรึงไว้ใหม่ได้ตามต้องการ อย่างไรก็ตาม ไม่สามารถย้ายคอลัมน์ที่ตรึงไว้เพื่อให้คอลัมน์เหล่านั้นปรากฏระหว่างชุดของคอลัมน์ที่ไม่ได้ตรึงได้ ในทำนองเดียวกัน ไม่สามารถย้ายคอลัมน์ที่ไม่ได้ตรึงไว้เพื่อให้คอลัมน์เหล่านั้นปรากฏระหว่างชุดของคอลัมน์ที่ตรึงได้

เมื่อต้องการยกเลิกตรึงคอลัมน์ ให้คลิกขวาในส่วนหัวของคอลัมน์ที่ตรึงไว้ แล้วเลือก **ยกเลิกตรึงคอลัมน์** 

โปรดทราบว่าคอลัมน์การเลือกแถวและสถานะแถวในกริดใหม่จะถูกตรึงไว้เป็นสองคอลัมน์แรกเสมอ ดังนั้น เมื่อคอลัมน์เหล่านี้รวมอยู่ในกริด ผู้ใช้จะมองเห็นคอลัมน์เหล่านั้นเสมอ ไม่ว่าจะเป็นตำแหน่งเลื่อนแนวนอนในกริดใด ไม่สามารถจัดลำดับ 2 คอลัมน์นี้ใหม่ได้

## <a name="autofit-column-width"></a>ปรับความกว้างของคอลัมน์ให้พอดีอัตโนมัติ
คล้ายกับ Excel ผู้ใช้สามารถบังคับให้คอลัมน์ปรับขนาดตามเนื้อหาที่แสดงอยู่ในคอลัมน์นั้นโดยอัตโนมัติ ถ้าต้องการทำเช่นนี้ คลิกสองครั้งที่การจัดการการปรับขนาดในคอลัมน์ หรือโดยการใส่โฟกัสไว้ในหัวข้อคอลัมน์และการกด **A** (เพื่อปรับให้พอดีโดยอัตโนมัติ) ความสามารถนี้จะพร้อมใช้งานเริ่มต้นในรุ่น 10.0.23

## <a name="frequently-asked-questions"></a>คำถามที่ถามบ่อย
### <a name="how-do-i-enable-the-new-grid-control-in-my-environment"></a>ฉันจะเปิดใช้งานการควบคุมกริดใหม่ในสภาพแวดล้อมของฉันได้อย่างไร 

คุณลักษณะ **การควบคุมกริดใหม่** จะพร้อมใช้งานในการจัดการคุณลักษณะในสภาพแวดล้อมต่างๆ โดยตรง หลังจากเปิดใช้งานคุณลักษณะในการจัดการคุณลักษณะ เซสชันผู้ใช้ที่ตามมาทั้งหมดจะใช้การควบคุมกริดใหม่ 

คุณลักษณะนี้เปิดใช้งานโดยค่าเริ่มต้นจะเริ่มต้นในรุ่น 10.0.21 และตั้งเป้าหมายเป็นแบบบังคับในเดือนตุลาคม 2022  

## <a name="developer-opting-out-individual-pages-from-using-the-new-grid"></a>[นักพัฒนา] การเลือกไม่ใช้กริดใหม่ในแต่ละหน้า 
ถ้าองค์กรของคุณค้นพบหน้าที่มีปัญหาในการใช้กริดใหม่ จะมี API ที่จะพร้อมใช้งาน เพื่ออนุญาตให้แบบฟอร์มแต่ละแบบใช้ตัวควบคุมกริดดั้งเดิม ในขณะที่ยังคงอนุญาตให้ระบบส่วนที่เหลือใช้ตัวควบคุมกริดใหม่ เมื่อไม่ต้องการให้บางหน้าใช้กริดใหม่ ให้เพิ่มการลงรายการบัญชีต่อไปนี้ `super()` ในวิธีการ `run()` สำหรับแบบฟอร์ม

```this.forceLegacyGrid();```

API นี้จะมีการใช้จนกว่าการควบคุมกริดใหม่กลายเป็นข้อบังคับ ในขณะนี้ การเปลี่ยนแปลงมีเป้าหมายอยู่ที่เดือนตุลาคม 2022 ถ้ามีปัญหาใดๆ ต้องการให้มีการใช้ API นี้ ให้รายงานไปยัง Microsoft

### <a name="forcing-a-page-to-use-the-new-grid-after-previously-opting-out-the-grid"></a>การบังคับให้หน้าใช้กริดใหม่หลังจากเลือกไม่เข้าร่วมกริดก่อนหน้านี้
ถ้าคุณเลือกไม่เข้าร่วมแต่ละหน้าจากการใช้กริดใหม่ คุณอาจต้องการเปิดใช้งานกริดใหม่ในภายหลัง หลังจากที่แก้ไขปัญหาที่อยู่ภายใต้แล้ว การทำดังนี้ คุณเพียงแต่ต้องลบการเรียกไปยัง `forceLegacyGrid()` การเปลี่ยนแปลงจะไม่มีผลจนกว่าการเปลี่ยนแปลงอย่างใดอย่างหนึ่งต่อไปนี้จะเกิดขึ้น

- **การปรับใช้สภาพแวดล้อมใหม่**: เมื่ออัปเดตและปรับใช้สภาพแวดล้อมใหม่ ตารางที่จัดเก็บหน้าที่เลือกออกจากกริดใหม่ (FormControlReactGridStats) จะถูกล้างโดยอัตโนมัติ
- **การล้างข้อมูลของตารางด้วยตนเอง**: สำหรับสถานการณ์การพัฒนา คุณต้องใช้ SQL เพื่อล้างตาราง FormControlReactGridStatre แล้วเริ่มต้น AOS ใหม่ ชุดของการดำเนินการนี้จะรีเซ็ตการแคชของหน้าที่เลือกจากกริดใหม่

## <a name="developer-opting-individual-grids-out-of-the-typing-ahead-of-the-system-capability"></a>[นักพัฒนา] การเลือกกริดแต่ละกริดจากความสามารถการพิมพ์ล่วงหน้าของระบบ
มีบางสถานการณ์ที่ไม่เอื้อให้ทำงานได้ดีกับความสามารถของ *การพิมพ์ล่วงหน้าของระบบ* ของกริด (ตัวอย่างเช่น บางรหัสที่ทริกเกอร์เมื่อมีการตรวจสอบความถูกต้องของแถวจะส่งผลให้การวิจัยแหล่งข้อมูลถูกทริกเกอร์ จากนั้นการวิจัยสามารถมีการแก้ไขที่ยังไม่ได้ยืนยันความเสียหายในแถวที่มีอยู่) หากองค์กรของคุณพบสถานการณ์ดังกล่าว API จะสามารถใช้งานเพื่อช่วยให้นักพัฒนาสามารถเลือกแต่ละกริดที่ไม่ผ่านการตรวจสอบความถูกต้องของแถวแบบอสมการ และแปลงกลับเป็นลักษณะการทำงานดั้งเดิมได้

เมื่อปิดใช้งานการตรวจสอบความถูกต้องของแถวแบบอะซิงโครนัสในกริด ผู้ใช้จะไม่สามารถสร้างแถวใหม่หรือย้ายไปที่แถวที่มีอยู่อื่นในกริดได้ ขณะที่มีปัญหาการตรวจสอบความถูกต้องในแถวปัจจุบัน เนื่องจากผลกระทบของการดําเนินการนี้ จึงไม่สามารถวางตารางจาก Excel ลงในกริดของการเงินและการดําเนินงานได้

การเลือกกริดแต่ละรายการออกจากการตรวจสอบความถูกต้องของแถวแบบอะซิงโครนัส ให้เพิ่มการเรียกต่อไปนี้หลัง `super()` ในวิธีการ `run()` ของแบบฟอร์ม

```<gridControl>.allowPreemptiveClient(false);```

> [!NOTE]
> - การเรียกนี้ควรเรียกใช้เฉพาะในกรณีที่เป็นกรณีพิเศษเท่านั้น และไม่ควรเป็นลักษณะปกติของกริดทั้งหมด
> - เราไม่แนะนำให้คุณสลับเปิดปิด API นี้ขณะรันไทม์หลังจากโหลดแบบฟอร์ม

## <a name="developer-size-to-available-width-columns"></a>[นักพัฒนา] ความกว้างของคอลัมน์ที่มี
ถ้านักพัฒนาตั้งค่าคุณสมบัติ **WidthMode** ให้ **SizeToAvailable** สำหรับคอลัมน์ภายในกริดใหม่ คอลัมน์เหล่านี้จะมีความกว้างเดียวกันถ้ามีการตั้งค่าคุณสมบัติให้กับ **SizeToContent** อย่างไรก็ตามสามรถยืดได้เพื่อใช้ความกว้างพิเศษที่มีอยู่ภายในกริด ถ้าคุณสมบัติถูกตั้งค่าเป็น **SizeToAvailable** สำหรับคอลัมน์หลายคอลัมน์ คอลัมน์ทั้งหมดจะมีความกว้างพิเศษที่มีอยู่ภายในกริดเหมือนกัน อย่างไรก็ตามถ้าผู้ใช้ปรับขนาดคอลัมน์อย่างใดอย่างหนึ่งต่อไปนี้ด้วยตนเอง คอลัมน์จะกลายเป็นแบบคงที่ จะยังคงอยู่ที่ความกว้างดังกล่าวและจะไม่ยืดให้มีความกว้างของกริดที่พร้อมใช้งานเพิ่มเติม

## <a name="known-issues"></a>ปัญหาที่ทราบ
ส่วนนี้มีรายการของปัญหาที่ทราบเกี่ยวกับตัวควบคุมกริดใหม่

### <a name="open-issues"></a>ปัญหาที่เปิด
- หลังจากเปิดใช้งานลักษณะการทำงานการ **ควบคุมกริดใหม่** แล้ว บางหน้าจะยังคงใช้ตัวควบคุมกริดเดิม ซึ่งจะเกิดขึ้นในสถานการณ์ต่อไปนี้
 
    - มีรายการบัตรอยู่บนหน้าเว็บที่แสดงอยู่ในหลายคอลัมน์
    - รายการบัตรที่มีการจัดกลุ่มมีอยู่บนหน้า
    - คอลัมน์กริดที่มีตัวควบคุมแบบขยายได้ที่ไม่ตอบสนอง

    เมื่อผู้ใช้พบสถานการณ์อย่างใดอย่างหนึ่งต่อไปนี้ข้อความจะแสดงขึ้นเกี่ยวกับการรีเฟรชหน้า หลังจากที่ข้อความนี้ปรากฏขึ้น หน้าจะยังคงใช้กริดที่มีอยู่สำหรับผู้ใช้ทั้งหมดจนกว่าจะอัปเดตรุ่นผลิตภัณฑ์ถัดไป การจัดการสถานการณ์เหล่านี้ที่ดีขึ้นเพื่อให้สามารถใช้กริดใหม่ได้ จะมีการพิจารณาสำหรับการอัปเดตในอนาคต

- [KB 4582758] เรกคอร์ดมีความพร่ามัวเมื่อคุณเปลี่ยนการย่อ/ขยายจาก 100 เป็นเปอร์เซ็นต์อื่นๆ
- [KB 4592012] ข้อผิดพลาดของไคลเอนต์ที่ไม่คาดคิดใน IE11 เมื่อวางหลายบรรทัดจาก Excel
