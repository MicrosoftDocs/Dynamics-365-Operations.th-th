# <a name="grid-capabilities"></a>ความสามารถของกริด

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

ตัวควบคุมกริดใหม่มีความสามารถในการใช้งานที่มีประโยชน์และมีประสิทธิภาพซึ่งสามารถใช้ในการปรับปรุงประสิทธิภาพการทำงานของผู้ใช้ สร้างมุมมองที่น่าสนใจต่างๆของข้อมูลของคุณ และรับข้อมูลเชิงลึกสำคัญในข้อมูลของคุณได้ บทความนี้จะครอบคลุมความสามารถต่อไปนี้: 

-  การคำนวณยอดรวม
-  การจัดกลุ่มข้อมูล
-  การพิมพ์ล่วงหน้าของระบบ
-  การประเมินนิพจน์ทางคณิตศาสตร์ 

## <a name="calculating-totals"></a>การคำนวณยอดรวม
ในแอป Finance and Operations ผู้ใช้มีความสามารถในการดูผลรวมที่ด้านล่างของคอลัมน์ตัวเลขในกริด ผลรวมเหล่านี้จะแสดงอยู่ในส่วนท้ายกระดาษที่ด้านล่างของกริด 

### <a name="showing-the-grid-footer"></a>การแสดงส่วนท้ายของกริด
มีพื้นที่ท้ายกระดาษที่ด้านล่างของกริดตารางทั้งหมดในแอป Finance and Operations ส่วนท้ายสามารถแสดงข้อมูลที่เป็นประโยชน์ซึ่งเกี่ยวข้องกับข้อมูลที่ปรากฏในกริด ต่อไปนี้เป็นตัวอย่างของข้อมูลนี้:

- จำนวนแถวที่เลือกในตาราง (เมื่อมีเรกคอร์ดมากกว่าหนึ่งเรกคอร์ดได้ถูกเลือก)
- ผลรวมสุดท้ายที่ด้านล่างของคอลัมน์ตัวเลขที่ตั้งค่าคอนฟิก
- จำนวนแถวในชุดข้อมูล 

ท้ายกระดาษนี้ถูกซ่อนไว้โดยค่าเริ่มต้นแต่สามารถเปิดใช้งานได้ง่าย เมื่อต้องการแสดงส่วนท้ายของกริดให้คลิกขวาบนส่วนหัวของคอลัมน์ในกริดและเลือกตัวเลือก **แสดงส่วนท้าย** เมื่อส่วนท้ายมีการเปิดใช้งานสำหรับกริดเฉพาะแล้วการตั้งค่านั้นจะจดจำจนกว่าผู้ใช้จะเลือกให้ซ่อนส่วนท้ายซึ่งสามารถทำได้โดยการคลิกขวาที่ส่วนหัวของคอลัมน์และเลือก **ซ่อนส่วนท้าย**  บันทึกการจัดวางของการดำเนินการ **แสดงส่วนท้าย/ซ่อนส่วนท้าย** ที่คาดว่าจะย้ายตำแหน่งอีกครั้งในการปรับปรุงในอนาคต 

### <a name="specifying-columns-with-totals"></a>การระบุคอลัมน์ที่มียอดรวม
ในปัจจุบันจะไม่มีคอลัมน์ไหนที่จะมีการตั้งค่าคอนฟิกเพื่อให้แสดงยอดรวมตามค่าเริ่มต้น การทำเช่นนี้ถือเป็นกิจกรรมการตั้งค่าแบบครั้งเดียวแทน คล้ายกับการปรับปรุงความกว้างของคอลัมน์ในกริด เมื่อคุณระบุว่าคุณต้องการดูยอดรวมสำหรับคอลัมน์ การตั้งค่านั้นจะจดจำในครั้งต่อไปที่คุณเข้าชมหน้า  

มีสองวิธีในการตั้งค่าคอนฟิกคอลัมน์เพื่อแสดงยอดรวมดังนี้: 

- คลิกขวาในคอลัมน์ที่คุณต้องการดูยอดรวม และจากนั้น เลือก **ยอดรวมคอลัมน์นี้** การดำเนินการนี้ทำให้เกิดเหตุการณ์สามประการดังนี้:

    1. ส่วนท้ายจะปรากฏขึ้น 
    2. การกำหนดลักษณะของคุณสำหรับการดูยอดรวมสำหรับคอลัมน์นี้ถูกบันทึก 
    3. การคำนวณยอดรวมเริ่มต้นขึ้นสำหรับคอลัมน์นี้ และคอลัมน์อื่นๆ ที่คุณได้ตั้งค่าคอนฟิกไว้ก่อนหน้านี้เพื่อดูยอดรวม เวลาที่ต้องใช้ในการแสดงยอดรวมจะขึ้นอยู่กับขนาดของชุดข้อมูลที่คุณกำลังรวมยอด

- หลังจากมองเห็นส่วนท้ายแล้ว ให้เลือก **แสดงผลรวม** ในพื้นที่ท้ายกระดาษที่ด้านล่างของคอลัมน์ที่คุณต้องการดูผลรวม ถ้าไม่มีคอลัมน์ที่มีการตั้งค่าคอนฟิกไว้ ปุ่ม **แสดงยอดรวม** จะพร้อมใช้งานสำหรับคอลัมน์ตัวเลขทั้งหมด 

    หลังจากที่มีการตั้งค่าคอนฟิกอย่างน้อยหนึ่งคอลัมน์สำหรับยอดรวม ปุ่ม **แสดงยอดรวม** จะพร้อมใช้งานบนโฮเวอร์หรือโฟกัสเท่านั้น การดำเนินการเลือก **แสดงยอดรวม** จะบันทึกเฉพาะการกำหนดลักษณะของคุณสำหรับการดูผลรวมในคอลัมน์นี้ เพื่อให้การกำหนดลักษณะถูกนำไปใช้ในระหว่างการเยี่ยมชมในอนาคตไปยังหน้า ในส่วนท้าย สถานะนี้จะบ่งชี้โดยเส้นประที่ปรากฏในคอลัมน์ (หรือถ้าชุดข้อมูลมีขนาดเล็กพอ จะมีการแสดงผลรวมทันที)

ถ้าคุณทำผิดพลาดและไม่ต้องการดูยอดรวมทั้งหมดในคอลัมน์หนึ่งๆแล้วให้คลิกขวาบนคอลัมน์และเลือก **ซ่อนยอดรวม** หรือเลือกปุ่ม **ซ่อนยอดรวม** ในส่วนท้ายของคอลัมน์นั้น การกำหนดลักษณะนี้จะถูกบันทึกไว้สำหรับการเยี่ยมชมในอนาคตไปยังหน้า 

### <a name="calculating-totals"></a>การคำนวณยอดรวม
เมื่อคุณมายังหน้าที่มีส่วนท้ายที่มองเห็นและคอลัมน์มีการตั้งค่าคอนฟิกสำหรับยอดรวม ยอดรวมอาจหรือไม่อาจแสดงอยู่ในส่วนท้าย ลักษณะการทำงานจะขึ้นอยู่กับขนาดของชุดข้อมูลบนหน้า ถ้าชุดข้อมูลมีขนาดเล็กเพียงพอ ยอดรวมจะแสดงขึ้นโดยอัตโนมัติพร้อมด้วยจำนวนแถวในชุดข้อมูล ถ้ามีเส้นประในส่วนท้ายของคอลัมน์ที่คุณตั้งค่าคอนฟิกไว้สำหรับยอดรวมแล้ว แสดงว่าชุดข้อมูลมีขนาดใหญ่เกินไปสำหรับระบบเพื่อแสดงยอดรวมทันทีและต้องมีการดำเนินการที่ชัดเจนเพื่อคำนวณยอดรวม เมื่อต้องการทำเช่นนี้ให้คลิกปุ่ม **คำนวณ** ในท้ายกระดาษหรือคลิกขวาบนคอลัมน์ที่คุณต้องการให้ยอดรวมและเลือก **ยอดรวมคอลัมน์นี้**  

ถ้าการคำนวณใช้เวลานานเกินไปคุณสามารถยกเลิกการดำเนินงานได้โดยเลือกปุ่ม **ยกเลิก** อย่างไรก็ตามในบางครั้งชุดข้อมูลจะมีขนาดใหญ่เกินกว่าที่จะคำนวณยอดรวม (ข้อจำกัดที่กำหนดโดยองค์กรของคุณ) และคุณจะได้รับการแจ้งเตือนให้กรองข้อมูลของคุณเพิ่มเติมแทน

ยอดรวมจะอัพเดตโดยอัตโนมัติเมื่อคุณอัพเดต ลบ หรือสร้างแถวในชุดข้อมูล  

## <a name="grouping-data"></a>การจัดกลุ่มข้อมูล
ผู้ใช้ทางธุรกิจมักต้องทำการวิเคราะห์เฉพาะกิจของข้อมูล ในขณะที่การดำเนินการนี้สามารถทำได้โดยการส่งออกข้อมูลไปยัง Microsoft Excel และใช้ตารางสาระสำคัญ ความสามารถใน **การจัดกลุ่ม** ในกริดตารางช่วยให้ผู้ใช้สามารถจัดระเบียบข้อมูลของตนได้ในวิธีที่น่าสนใจภายในแอป Finance and Operations ในฐานะที่ลักษณะนี้ขยายลักษณะการทำงานของ **ยอดรวม** **การจัดกลุ่ม** จะช่วยให้คุณสามารถรับข้อมูลเชิงลึกที่มีความสำคัญไปยังข้อมูลโดยการจัดหายอดรวมย่อยที่ระดับกลุ่ม

เมื่อต้องการใช้ลักษณะการทำงานนี้ให้คลิกขวาที่คอลัมน์ที่คุณต้องการจัดกลุ่มตามแล้วเลือก **จัดกลุ่มตามคอลัมน์นี้** การดำเนินการนี้จะเรียงลำดับข้อมูลตามคอลัมน์ที่เลือก เพิ่มกลุ่มใหม่ตามคอลัมน์ไปที่จุดเริ่มต้นไปยังกริด และแทรก "แถวส่วนหัว" ที่จุดเริ่มต้นของกลุ่มแต่ละกลุ่ม แถวส่วนหัวเหล่านี้จะให้ข้อมูลเกี่ยวกับแต่ละกลุ่มต่อไปนี้: 
-  ค่าข้อมูลสำหรับกลุ่ม 
-  ป้ายชื่อคอลัมน์ (ข้อมูลนี้จะมีประโยชน์โดยเฉพาะอย่างยิ่งหลังจากที่ได้รับการสนับสนุนการจัดกลุ่มหลายระดับ)
-  จำนวนของแถวข้อมูลในกลุ่มนี้
-  ยอดรวมย่อยสำหรับคอลัมน์ใดๆที่ตั้งค่าคอนฟิกให้แสดงยอดรวม

เมื่อเปิดใช้ [มุมมองที่บันทึก](saved-views.md) การจัดกลุ่มนี้จะสามารถบันทึกโดยการกำหนดเป็นแบบส่วนบุคคลโดยเป็นส่วนหนึ่งของมุมมองเพื่อให้สามารถเข้าถึงหน้าได้อย่างรวดเร็วในครั้งต่อไปที่คุณเข้าชมหน้า  

ถ้าคุณเลือก **จัดกลุ่มตามคอลัมน์นี้** สำหรับคอลัมน์ที่แตกต่างกัน การจัดกลุ่มต้นฉบับถูกแทนที่เนื่องจากการจัดกลุ่มหนึ่งระดับเท่านั้นที่ได้รับการสนับสนุนในรุ่น 10.0.9 ที่มีการปรับปรุงแพลตฟอร์ม 33

เมื่อต้องการย้อนกลับการจัดกลุ่มในกริดให้คลิกขวาบนคอลัมน์การจัดกลุ่มแล้วเลือก **ยกเลิกการจัดกลุ่ม**  


## <a name="evaluating-math-expressions"></a>การประเมินนิพจน์ทางคณิตศาสตร์
ในฐานะที่เป็นผู้สนับสนุนประสิทธิภาพ ผู้ใช้สามารถป้อนสูตรทางคณิตศาสตร์ในเซลล์ตัวเลขในกริด ไม่จำเป็นต้องทำการคำนวณในแอปภายนอกระบบ ตัวอย่างเช่น ถ้าคุณป้อน **=15\*4** แล้วกดคีย์ **แท็บ** เพื่อย้ายออกจากฟิลด์ ระบบจะประเมินนิพจน์และบันทึกค่าเป็น **60** สำหรับฟิลด์

เมื่อต้องการให้ระบบจดจำค่าเป็นนิพจน์ ให้เริ่มต้นค่าโดยใช้เครื่องหมายเท่ากับ (**=**) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับผู้ปฏิบัติงานและไวยากรณ์ที่ได้รับการสนับสนุน โปรดดูที่ [สัญลักษณ์คณิตศาสตร์ที่สนับสนุน](http://bugwheels94.github.io/math-expression-evaluator/#supported-maths-symbols)

## <a name="frequently-asked-questions"></a>คำถามที่ถามบ่อย
### <a name="how-do-i-enable-the-new-grid-control-in-my-environment"></a>ฉันจะเปิดใช้งานการควบคุมกริดใหม่ในสภาพแวดล้อมของฉันได้อย่างไร 

**10.0.9 / แพลตฟอร์มที่อัพเดต 33 หรือรุ่นหลัง** คุณลักษณะ **การควบคุมกริดใหม่** จะพร้อมใช้งานในการจัดการคุณลักษณะในสภาพแวดล้อมต่างๆ โดยตรง เช่นเดียวกับคุณลักษณะการแสดงตัวอย่างสาธารณะอื่นๆ การเปิดใช้งานลักษณะการทำงานนี้ในการผลิตจะอยู่ภายใต้ [ข้อกำหนดเพิ่มเติมของข้อตกลงการใช้งาน](https://go.microsoft.com/fwlink/?linkid=2105274)  

**10.0.8 / แพลตฟอร์มอัพเดต 32 และ10.0.7 / แพลตฟอร์มอัพเดต 31** คุณลักษณะ **การควบคุมกริดใหม่** สามารถถูกใช้งานในสภาพแวดล้อมระดับ 1 (การพัฒนา/การทดสอบ) และแบบระดับ 2 (Sandbox) เพื่อให้การทดสอบเพิ่มเติมและการออกแบบโดยปฏิบัติตามขั้นตอนต่อไปนี้

1.  **เปิดใช้งานเที่ยวบิน**: ดำเนินการคำสั่ง SQL ต่อไปนี้: 

    `INSERT INTO SYSFLIGHTING (FLIGHTNAME, enabled, FLIGHTSERVICEID, PARTITION) VALUES('CLIReactGridEnableFeature', 1, 0, 5637144576);`

2. **รีเซ็ต IIS** เพื่อล้างแคชการสลับแบบคงที่ 

3.  **ค้นหาคุณลักษณะ**: ไปยังพื้นที่ทำงาน **การจัดการคุณลักษณะ** ถ้า **การควบคุมกริดใหม่** ไม่ปรากฏในรายการของคุณลักษณะการทำงานทั้งหมด ให้เลือก **ตรวจหาการอัพเดต**   

4.  **เปิดใช้งานคุณลักษณะ**: ค้นหาคุณลักษณะ **การควบคุมกริดใหม่** ในรายการของคุณลักษณะ และเลือก **เปิดใช้งานเดี๋ยวนี้** บนบานหน้าต่างรายละเอียด โปรดทราบว่าจำเป็นต้องมีการรีเฟรชเบราว์เซอร์ 

เซสชันผู้ใช้ที่ตามมาทั้งหมดจะเริ่มต้นด้วยการควบคุมกริดใหม่ที่เปิดใช้งาน
