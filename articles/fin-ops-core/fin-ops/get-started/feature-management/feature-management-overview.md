# <a name="feature-management-overview"></a>ภาพรวมของการจัดการคุณลักษณะ

[!include [banner](../../includes/banner.md)]

จะมีการเพิ่มเติมและปรับปรุงคุณลักษณะทุกครั้งที่นำไปใช้ ประสบการณ์การจัดการคุณลักษณะให้พื้นที่ทำงานที่คุณสามารถดูรายการของคุณลักษณะที่ได้ถูกจัดส่งในการนำออกใช้แต่ละครั้ง จากนั้นคุณสามารถใช้พื้นที่ทำงานเพื่อดูคู่มือคุณลักษณะ และเพื่อเปิดใช้งานหรือปิดใช้งานคุณลักษณะ

## <a name="the-feature-management-workspace"></a>พื้นที่ทำงานการจัดการคุณลักษณะ

คุณสามารถเปิดพื้นที่ทำงาน **การจัดการคุณลักษณะ** ด้วยการเลือกไทล์ที่เหมาะสมบนแดชบอร์ด คุณจะเห็นหน้าที่แสดงรายการของคุณลักษณะสำหรับการนำออกใช้ทั้งหมดที่ได้รับการสนับสนุนโดยประสบการณ์การจัดการคุณลักษณะ 

รายการคุณลักษณะประกอบด้วยข้อมูลต่อไปนี้:

- **ชื่อคุณลักษณะ** – คำอธิบายของคุณลักษณะที่ถูกเพิ่ม
- **สถานะ** – สัญลักษณ์บ่งชี้ว่าคุณลักษณะถูกเปิด (เครื่องหมายถูก) หรือถูกเปิด (ว่างเปล่า) ได้รับการจัดกำหนดการให้เปิด (นาฬิกา) หรือโดยบังคับ (ล็อค) ต้องการความสนใจก่อนที่คุณจะเปิด (สัญลักษณ์คำเตือน) หรือไม่สามารถเปิดใช้งานได้ (X) มีการใช้การตั้งค่าที่แสดงไว้สำหรับนิติบุคคลทั้งหมด โปรดทราบว่า แม้ว่าเมื่อคุณลักษณะถูกเปิดใช้งาน จะยังคงถูกควบคุมโดยความปลอดภัย ดังนั้น คุณลักษณะนี้จะพร้อมใช้งานเฉพาะสำหรับผู้ใช้ที่มีสิทธิ์เข้าถึง ตามบทบาทความปลอดภัยของพวกเขา นอกจากนี้ ยังพร้อมใช้งานเฉพาะในนิติบุคคลที่ผู้ใช้มีสิทธิ์เข้าถึงเท่านั้น
- **วันที่เปิดใช้งาน** – วันที่ที่มีการเปิดคุณลักษณะ หรือมีการจัดกำหนดการให้ถูกเปิด
- **เพิ่มคุณลักษณะแล้ว** – วันที่ที่มีการเพิ่มคุณลักษณะไปยังสภาพแวดล้อมของคุณ วันที่นี้จะถูกป้อนโดยอัตโนมัติ เมื่อคุณปรับปรุงสภาพแวดล้อมของคุณในระหว่างรอบการนำออกใช้รายเดือน
- **สถานะของคุณลักษณะ** – สถานะของวงจรการใช้ปัจจุบันของคุณลักษณะ: **พรีวิว** **รีลีส** (แสดงเป็นว่างเปล่า) **ตามค่าเริ่มต้น** และ **บังคับ** สถานะครอบคลุมอยู่ในรายละเอียดเพิ่มเติม ในตอนท้ายของหัวข้อนี้ 
- **โมดูล** – โมดูลที่ได้รับผลกระทบจากคุณลักษณะใหม่

> [!NOTE]
> คอลัมน์ **สถานะของคุณลักษณะ** ถูกรวมไว้รุ่น 10.0.21

เมื่อคุณเลือกคุณลักษณะ ข้อมูลเพิ่มเติมจะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียดทางด้านขวาของรายการคุณลักษณะ ที่ด้านบนของบานหน้าต่าง คุณจะเห็นชื่อคุณลักษณะ วันที่ที่มีการเพิ่มคุณลักษณะ โมดูลที่ได้รับผลกระทบจากคุณลักษณะ และลิงก์ **เรียนรู้เพิ่มเติม** เลือกลิงก์นี้เพื่อดูเอกสารประกอบสำหรับคุณลักษณะ ถ้าเอกสารไม่พร้อมใช้งาน คุณจะถูกพาไปยังหน้าชั่วคราว บานหน้าต่างรายละเอียดจะรวมฟิลด์ **ข้อคิดเห็น** ที่ซึ่งคุณสามารถเพิ่มข้อคิดเห็นของคุณเองเกี่ยวกับคุณลักษณะได้

พื้นที่ทำงาน **การจัดการคุณลักษณะ** ยังมีแท็บหลายแท็บ ซึ่งแต่ละแท็บแสดงรายการของคุณลักษณะ

- **ใหม่** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ถูกเพิ่มไว้ นับตั้งแต่การปรับปรุงรายเดือนครั้งสุดท้าย ถ้าคุณได้ข้ามการปรับปรุงรายเดือนใดๆ แท็บจะแสดงคุณลักษณะใหม่ทั้งหมดที่ถูกเพิ่มตั้งแต่ครั้งล่าสุดที่คุณปรับปรุง คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ จำนวนรวมของคุณลักษณะใหม่จะแสดงอยู่ในไทล์ที่ด้านบนของหน้าด้วย
- **ไม่ได้เปิดใช้งาน** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ยังไม่ได้ถูกเปิด คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ ไทล์ที่ด้านบนของหน้าจะแสดงจํานวนรวมของคุณลักษณะใหม่ที่ขณะนี้ปิดอยู่
- **จัดกำหนดการ** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ได้ถูกจัดกำหนดการให้เปิดในอนาคต คุณลักษณะที่มีวันที่ที่จัดกำหนดการที่เร็วที่สุด จะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ ไทล์ที่ด้านบนของหน้าจะแสดงจํานวนรวมของคุณลักษณะที่จัดกำหนดการไว้
- **ทั้งหมด** – แท็บนี้แสดงคุณลักษณะทั้งหมด คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ

## <a name="feature-states"></a>สถานะคุณลักษณะ
คุณลักษณะสามารถเปลี่ยนไปมาระหว่างหลายสถานะ ตั้งแต่ถูกนำมาใช้ในการจัดการคุณลักษณะจนกลายมาเป็นข้อบังคับในผลิตภัณฑ์ในที่สุด ส่วนนี้อธิบายสถานะคุณลักษณะที่มีผลบังคับใช้

### <a name="preview-features-optional"></a>พรีวิวคุณลักษณะ (ไม่บังคับ)

ทีมผลิตภัณฑ์สามารถตัดสินใจที่จะเริ่มต้นคุณลักษณะใหม่เป็นคุณลักษณะพรีวิว คุณลักษณะพรีวิวจะไม่เปิดใช้งานโดยค่าเริ่มต้น และเลือกกำหนดได้ ทีมผลิตภัณฑ์ที่เป็นเจ้าของจะอัปเดตคุณลักษณะเพื่อนำออกใช้ หลังจากเสร็จสิ้นรอบระยะเวลาพรีวิวแล้ว

> [!NOTE]
> ลักษณะพรีวิวจะขึ้นอยู่กับ [ข้อตกลงและเงื่อนไข](https://go.microsoft.com/fwlink/?linkid=2105274) ของพรีวิวเฉพาะ 

### <a name="released-features-optional"></a>คุณลักษณะที่นำออกใช้ (ไม่บังคับ)

คอลัมน์ **สถานะของคุณลักษณะ** ของคุณลักษณะเหล่านี้ว่างเปล่า คุณลักษณะที่เพิ่มในครั้งแรกเมื่อ่นำออกใช้ ไม่ได้เปิดตามค่าเริ่มต้น และการเปิดใช้งานคุณลักษณะเหล่านั้นสามารถเลือกกำหนดได้ คุณลักษณะที่มีการอัปเดตจากพรีวิว จะรักษาสถานะการเปิดใช้งานไว้

### <a name="on-by-default-features-optional"></a>คุณลักษณะที่เปิดใช้งานตามค่าเริ่มต้น (ไม่บังคับ)

คุณลักษณะที่มีการอัปเดตเป็น **เปิดใช้งานตามค่าเริ่มต้น** จะถูกเปิดใช้งานตามค่าเริ่มต้น แต่คุณสามารถปิดใช้งานได้ หลังจากคุณลักษณะที่สามารถปิดใช้งานได้อยู่ในสถานะ **นำออกใช้** เป็นเวลาอย่างน้อยหกเดือน คาดว่าคุณลักษณะเหล่านี้จะย้ายไปที่สถานะนี้ ในรุ่นหลักถัดไป คุณลักษณะที่แปลงเป็น **เปิด ตามค่าเริ่มต้น** คาดว่าจะสามารถสื่อสารได้ในหัวข้อ [มีอะไรใหม่](../whats-new-changed.md) ในรุ่น การอัปเดตจะเริ่มต้นโดยทีมงานผลิตภัณฑ์ที่เป็นเจ้าของ

> [!NOTE]
> เนื่องจากคุณลักษณะเหล่านี้จะเปิดใช้งานโดยอัตโนมัติ คุณจึงควรระบุว่าองค์กรของคุณพร้อมแล้วที่จะใช้งานคุณลักษณะเหล่านี้ หรือต้องการเวลาเพิ่มเติมหรือไม่ ถ้าต้องการเวลาเพิ่มเติม คุณอาจจําเป็นต้องปิดใช้งานคุณลักษณะเหล่านี้ชั่วคราว โปรดทราบว่าการเปลี่ยนคุณลักษณะเป็น **เปิด ตามค่าเริ่มต้น** จะเสร็จสิ้นในรุ่นหลัก ก่อนที่คุณลักษณะจะถูกกำหนดเป้าหมายเป็น **บังคับ** ในจุดนั้น คุณจะไม่มีตัวเลือกในการปิดใช้งานคุณลักษณะ 

### <a name="mandatory"></a>จำเป็น

**บังคับ** เป็นสถานะสุดท้ายที่คาดไว้ของคุณลักษณะ ซึ่งบ่งชี้ว่าคุณลักษณะดังกล่าวเปิดอยู่ และคุณไม่สามารถปิดใช้งานได้โดยไม่ติดต่อ Microsoft คุณลักษณะที่เลือกกำหนดได้คาดว่าจะเปลี่ยนเป็นคุณลักษณะบังคับ หลังจากรุ่นหลักสองรุ่น คุณลักษณะที่วิกฤติ อาจมีข้อยกเว้น เป็นข้อบังคับ

## <a name="example-of-expected-feature-lifecycles"></a>ตัวอย่างของวงจรการใช้งานของคุณลักษณะที่คาดไว้

คุณลักษณะที่สามารถปิดใช้งานได้ และที่เพิ่มเป็นนำออกใช้และเป็นตัวเลือกก่อนหรือเป็นส่วนหนึ่งของรุ่นเดือนเมษายน คาดว่าเปลี่ยนเป็น **เปิด ตามค่าเริ่มต้น** ในรุ่นเดือนตุลาคมที่จะถึงนี้ จากนั้นคาดว่าจะเปลี่ยนเป็น **บังคับ** ในเดือนเมษายนของปีถัดไป

คุณลักษณะที่ไม่สามารถปิดใช้งานได้ และที่เพิ่มเป็นนำออกใช้และเป็นตัวเลือกก่อนหรือเป็นส่วนหนึ่งของรุ่นเดือนเมษายน คาดว่าเปลี่ยนเป็น **บังคับ** ในเดือนเมษายนของปีถัดไป

## <a name="enable-a-feature"></a>เปิดใช้งานคุณลักษณะ

ถ้าไม่ได้เปิดคุณลักษณะ ปุ่ม **เปิดใช้งานตอนนี้** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อเปิดใช้งานคุณลักษณะ

ไม่สามารถปิดใช้งานคุณลักษณะบางอย่างได้ หลังจากที่คุณเปิดใช้งาน ถ้าคุณลักษณะที่คุณกำลังพยายามเปิด ไม่สามารถเปิดใช้งานได้ คุณจะได้รับคำเตือน ณ จุดนั้น คุณสามารถเลือก **ยกเลิก** เพื่อยกเลิกการดำเนินงาน และปล่อยให้คุณลักษณะถูกปิดใช้งาน อย่างไรก็ตาม ถ้าคุณเลือก **เปิดใช้งาน** เพื่อเปิดใช้งานคุณลักษณะ คุณจะไม่สามารถปิดใช้งานได้ในภายหลัง

คุณลักษณะบางอย่างจะแสดงข้อความที่ให้ข้อมูลเพิ่มเติม ก่อนที่คุณจะเปิดใช้งานคุณลักษณะเหล่านั้น คุณสมบัติเหล่านี้จะถูกระบุด้วยสัญลักษณ์คำเตือนสีเหลือง คุณควรอ่านข้อมูลเพิ่มเติมอย่างระมัดระวัง เพื่อให้มั่นใจว่าคุณเข้าใจสิ่งที่จะเกิดขึ้น เมื่อมีการเปิดใช้งานคุณลักษณะนี้ อย่างไรก็ตาม คุณยังคงสามารถเลือก **เปิดใช้งาน** เพื่อเปิดใช้งานคุณลักษณะ

คุณลักษณะบางอย่างจะแสดงข้อความว่า คุณลักษณะนี้ไม่สามารถเปิดใช้งานได้จนกว่าจะมีการดำเนินการ คุณสมบัติเหล่านี้จะถูกระบุด้วยสัญลักษณ์ X สีแดง คุณต้องดำเนินการตามที่อธิบายไว้ในคำอธิบาย ก่อนที่จะเปิดใช้งานคุณลักษณะนี้ ตัวอย่างเช่น ถ้าคุณไม่สามารถใช้คุณลักษณะได้จนกว่าคีย์การตั้งค่าคอนฟิกถูกปิดใช้งาน จากนั้น คุณต้องปิดใช้งานคีย์การตั้งค่าคอนฟิกก่อน แล้วจากนั้น จึงกลับไปที่การจัดการคุณลักษณะเพื่อเปิดใช้งานคุณลักษณะนี้

หลังจากเปิดใช้งานคุณลักษณะ ข้อความจะปรากฏขึ้นด้านล่างลิงก์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะถูกเปิดใช้งาน หรือบ่งชี้วันที่ในอนาคตที่คุณลักษณะจะถูกจัดกำหนดการให้เปิดใช้งาน อย่างใดอย่างหนึ่ง ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ

คุณลักษณะที่ถูกกำหนดให้เปิดใช้งานในอนาคตจะปรากฏบนแท็บ **จัดกำหนดการไว้** กระบวนการชุดงานจะเปิดใช้งานในเวลาเที่ยงคืนของวันที่ที่ระบุ ตามโซนเวลาที่แสดงอยู่ในวันที่ของระบบ

## <a name="reschedule-a-feature"></a>จัดกำหนดการคุณลักษณะใหม่

ถ้ามีการจัดกำหนดการคุณลักษณะให้เปิดใช้งานในอนาคต ปุ่ม **จัดกำหนดการไว้** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อเปลี่ยนค่า **วันที่เปิดใช้งาน** เป็นวันที่อื่นได้

1. เลือกคุณลักษณะที่จัดกำหนดการที่จะจัดกำหนดการใหม่ และจากนั้น ในบานหน้าต่างแสดงรายละเอียด เลือก **จัดกำหนดการ**
2. ในกล่องโต้ตอบที่ปรากฏขึ้น ในฟิลด์ **วันที่เปิดใช้งาน** ให้ระบุวันที่ใหม่ที่คุณลักษณะควรถูกเปิดใช้งาน
3. เลือก **เปิดใช้งาน** เพื่อจัดกำหนดการคุณลักษณะ หรือ **ปิดใช้งาน** เพื่อยกเลิกกำหนดการ

## <a name="disable-a-feature"></a>ปิดใช้งานคุณลักษณะ

ถ้ามีการเปิดใช้งานคุณลักษณะอยู่แล้ว ปุ่ม **ปิดใช้งาน** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อปิดใช้งานคุณลักษณะ ปุ่ม **ปิดใช้งาน** จะไม่พร้อมใช้งาน ถ้าไม่สามารถปิดใช้งานคุณลักษณะได้ 

หลังจากปิดใช้งานคุณลักษณะ ข้อความจะปรากฏขึ้นด้านล่างลิงก์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะยังไม่ได้ถูกเปิดใช้งาน ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ คุณลักษณะที่ยังไม่ได้ถูกเปิดใช้งาน จะปรากฏขึ้นบนแท็บ **ไม่ได้เปิดใช้งาน**

## <a name="features-that-must-be-enabled"></a>คุณลักษณะที่ต้องเปิดใช้งาน

บางครั้ง คุณลักษณะที่สำคัญที่จะถูกจัดส่ง ต้องถูกเปิดใช้งานโดยอัตโนมัติ เมื่อคุณทำการอัปเดต คุณลักษณะเหล่านี้จะถูกเปิดใช้งานโดยอัตโนมัติ ในวันที่ที่ระบุไว้ในฟิลด์ **วันที่เปิดใช้งาน** สำหรับคุณลักษณะเหล่านี้ ข้อความจะปรากฏขึ้นด้านล่างลิงก์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะจะถูกเปิดใช้งาน หรือบ่งชี้วันที่ในอนาคตที่จะเปิดใช้งานคุณลักษณะ อย่างใดอย่างหนึ่ง ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ

## <a name="enable-all-features"></a>เปิดใช้คุณลักษณะทั้งหมด

คุณสามารถเปิดใช้งานคุณลักษณะทั้งหมดได้โดยการเลือกปุ่ม **เปิดใช้งานทั้งหมด** 

เมื่อคุณเลือก **เปิดใช้งานทั้งหมด** ตัวเลือกจะปรากฏในตำแหน่งที่คุณต้องให้ข้อมูลต่อไปนี้:

- รายการของคุณลักษณะทั้งหมดที่ต้องการการยืนยัน ก่อนที่จะสามารถเปิดใช้งานได้ ถ้าคุณต้องการเปิดใช้งานคุณลักษณะในรายการ ให้เลือก **ใช่** สำหรับปุ่ม **เปิดใช้งานคุณลักษณะที่ต้องการการยืนยัน**
- รายการของคุณลักษณะทั้งหมดที่ไม่สามารถเปิดใช้งานได้จะถูกแสดง คุณลักษณะเหล่านั้นจะไม่ถูกเปิดใช้งาน

คุณลักษณะทั้งหมดที่สามารถเปิดใช้งานได้จะถูกเปิดใช้งาน ถ้าคุณลักษณะถูกจัดกำหนดการให้เปิดใช้งานเรียบร้อยแล้วในอนาคต กำหนดการจะไม่เปลี่ยนแปลง 

## <a name="enable-all-features-automatically"></a>เปิดใช้งานคุณลักษณะทั้งหมดโดยอัตโนมัติ

ถ้าคุณต้องการเปิดใช้งานคุณลักษณะใหม่ทั้งหมดโดยอัตโนมัติ คุณสามารถใช้รายการแบบหล่นลงภายใต้ชื่อพื้นที่ทำงาน เพื่อเปลี่ยนแปลงสิ่งที่เกิดขึ้นเมื่อมีการเพิ่มคุณลักษณะใหม่

- เลือก **เปิดใช้งานคุณลักษณะใหม่โดยอัตโนมัติ** เพื่อเปิดใช้งานคุณลักษณะใหม่ทั้งหมดโดยอัตโนมัติ เมื่อมีการเพิ่มไปยังสภาพแวดล้อมของคุณ
- เลือก **ไม่ต้องเปิดใช้งานคุณลักษณะใหม่โดยอัตโนมัติ** ถ้าคุณลักษณะใหม่ที่ใช้งานได้ทั้งหมดควรจะปิดไว้ตามค่าเริ่มต้น เมื่อมีการเพิ่มไปยังสภาพแวดล้อมของคุณ

เมื่อคุณเปิดใช้งานคุณลักษณะทั้งหมดโดยอัตโนมัติ จะเปิดใช้งานคุณลักษณะทั้งหมดที่จะถูกเปิดใช้งานเมื่อคุณคลิกปุ่ม **เปิดใช้งานทั้งหมด** จะไม่เปิดใช้งานคุณลักษณะที่ต้องการการยืนยัน หรือคุณลักษณะที่ไม่สามารถเปิดใช้งานได้จนกว่าจะมีการดำเนินการ

## <a name="check-for-updates"></a>ตรวจหาการอัปเดต

คุณลักษณะจะถูกเพิ่มลงในสภาพแวดล้อมของคุณ หลังจากการปรับปรุงแต่ละครั้ง อย่างไรก็ตาม คุณสามารถตรวจสอบการปรับปรุงด้วยตนเองได้โดยการคลิกที่ปุ่ม **ตรวจสอบการปรับปรุง** คุณลักษณะใดๆ ที่ถูกเพิ่มลงในระบบ หลังจากที่การปรับปรุงจะถูกเพิ่มลงในรายการของคุณลักษณะ ตัวอย่างเช่น ถ้าคุณลักษณะที่รวดเร็วถูกเปิดใช้งานหลังจากการนำออกใช้งาน จากนั้น คุณจะสามารถตรวจสอบการปรับปรุงได้ และคุณลักษณะจะถูกเพิ่มลงในรายการของคุณ

## <a name="assigning-roles"></a>การกำหนดบทบาท

พื้นที่ทำงาน **การจัดการคุณลักษณะ** สามารถเปิดได้โดยผู้ดูแลระบบ และโดยผู้ใช้ที่ได้รับการกำหนดให้กับบทบาทตัวจัดการคุณลักษณะ หรือบทบาทตัวแสดงคุณลักษณะด้วย บทบาทสองบทบาทนี้ถูกสร้างขึ้นเพื่อสนับสนุนประสบการณ์การจัดการคุณลักษณะ ผู้ใช้ในบทบาทผู้จัดการคุณลักษณะสามารถเปิดหรือปิดคุณลักษณะใดๆ ได้ นอกจากนี้ ยังสามารถปรับปรุงฟิลด์ **ความคิดเห็น** สำหรับคุณลักษณะได้ด้วย ผู้ใช้ในบทบาทของตัวแสดงคุณลักษณะสามารถดูได้เฉพาะพื้นที่ทำงาน **การจัดการคุณลักษณะ** ผู้ใช้ไม่สามารถเปิดหรือปิดคุณลักษณะได้

บทบาทของตัวจัดการคุณลักษณะ และบทบาทของตัวแสดงคุณลักษณะ ไม่ได้แทนที่ความปลอดภัยที่มีอยู่ซึ่งผู้ใช้มี เพียงควบคุมว่าผู้ใช้สามารถเปิดและปิดคุณลักษณะการทำงานได้หรือไม่ จะไม่ให้การเข้าถึงไปยังคุณลักษณะเอง

## <a name="features-that-use-configuration-keys"></a>คุณลักษณะที่ใช้คีย์การตั้งค่าคอนฟิก

ถ้าคุณลักษณะใช้คีย์การตั้งค่าคอนฟิก แต่ไม่มีการเปิดคีย์การตั้งค่าคอนฟิก พื้นที่ทำงาน **การจัดการคุณลักษณะ** จะไม่แสดงคุณลักษณะในรายการของคุณลักษณะที่พร้อมใช้งาน หลังจากที่คุณเปิดคีย์การตั้งค่าคอนฟิก คุณต้องปรับปรุงรายการคุณลักษณะโดยใช้รายการเมนู **ตรวจสอบสำหรับการปรับปรุง** จากนั้น คุณลักษณะจะปรากฏในรายการคุณลักษณะ

ถ้าคุณปิดคีย์การตั้งค่าคอนฟิก คุณลักษณะจะไม่ถูกลบออกจากรายการคุณลักษณะ

## <a name="data-entities"></a>เอนทิตี้ข้อมูล

เอนทิตีข้อมูลที่มีชื่อว่า **การจัดการคุณลักษณะ** ช่วยให้คุณสามารถส่งออกการตั้งค่าการจัดการคุณลักษณะจากสภาพแวดล้อมหนึ่ง แล้วนำเข้าไปยังสภาพแวดล้อมอื่น เอนทิตีนี้ปรับปรุงคุณลักษณะที่มีอยู่เท่านั้น ตรรกะทางธุรกิจในเอนทิตียังช่วยรับประกันว่า กฎเดียวกันที่ใช้ในพื้นที่ทำงาน **การจัดการคุณลักษณะ** จะถูกนำไปใช้ เมื่อการนำเข้าเสร็จสิ้น ตัวอย่างเช่น คุณไม่สามารถแทนที่การตั้งค่าคุณลักษณะแบบบังคับได้โดยการลบวันที่ในระหว่างการนำเข้า

ตัวอย่างต่อไปนี้จะอธิบายสิ่งที่เกิดขึ้น เมื่อคุณใช้เอนทิตี **การจัดการคุณลักษณะ** ในการนำเข้าข้อมูล

- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** คุณลักษณะนี้จะถูกเปิดใช้งาน และฟิลด์ **วันที่เปิดใช้งาน** จะถูกกำหนดเป็นวันที่ปัจจุบัน
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** หรือปล่อยให้ฟิลด์ **EnableDate** ว่างเปล่า คุณลักษณะนี้จะถูกปิดใช้งาน และฟิลด์ **วันที่เปิดใช้งาน** จะถูกล้างข้อมูล คุณไม่สามารถปิดใช้งานคุณลักษณะแบบบังคับ หรือคุณลักษณะที่ไม่สามารถปิดใช้งานได้หลังจากที่เปิดใช้งานแล้ว
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะจะมีการจัดกำหนดการสำหรับวันที่นั้น
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** และเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะนี้จะถูกจัดกำหนดการสำหรับวันที่นั้น 
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ไม่** แต่นอกจากนี้ คุณยังเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะนี้จะถูกจัดกำหนดการสำหรับวันที่นั้น
- ถ้ามีการเปิดใช้งานคุณลักษณะ และคุณเพิ่มฟิลด์ **EnableDate** ที่ถูกตั้งค่าเป็นวันที่ในอนาคต คุณลักษณะจะยังคงเปิดใช้งานอยู่ เมื่อต้องการจัดกำหนดการคุณลักษณะใหม่ คุณต้องเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ไม่**

## <a name="feature-management-and-flighting"></a>การจัดการคุณลักษณะและการสลับ

การจัดการคุณลักษณะช่วยให้คุณควบคุมคุณลักษณะที่ถูกจัดส่งในการนำออกใช้แต่ละครั้ง การสลับช่วยให้ Microsoft teams สามารถนำออกใช้คุณลักษณะไปยังลูกค้าจำนวนจำกัด เพื่อให้คุณลักษณะเหล่านั้นสามารถได้รับการทดสอบและได้รับการตรวจสอบความถูกต้องโดยไม่มีผลกระทบต่อลูกค้าทั้งหมด การจัดการคุณลักษณะไม่ได้ควบคุมการสลับของคุณลักษณะใดๆ

## <a name="using-feature-management-to-turn-on-isv-features-or-custom-features"></a>การใช้การจัดการคุณลักษณะเพื่อเปิดคุณลักษณะ ISV หรือคุณลักษณะที่กำหนดเอง

ปัจจุบันการจัดการคุณลักษณะไม่พร้อมใช้งานสำหรับคุณลักษณะจากผู้จำหน่ายซอฟต์แวร์อิสระ (ISVs) และคุณลักษณะที่กำหนดเอง อย่างไรก็ตาม Microsoft กำลังเพิ่มฟังก์ชันเพิ่มเติมเพื่อปรับปรุงการจัดการคุณลักษณะ หลังจากการปรับปรุงเหล่านั้นเสร็จสมบูรณ์ Microsoft จะทำให้การจัดการคุณลักษณะพร้อมใช้งานสำหรับคุณลักษณะทั้งหมด และให้คำแนะนำเกี่ยวกับการปรับปรุงคุณลักษณะของคุณในการใช้งาน

## <a name="frequently-asked-questions-faq"></a>คำถามที่ถามบ่อย (FAQ)

### <a name="when-are-features-added-removed-or-changed"></a>เมื่อใดที่คุณลักษณะถูกเพิ่ม ลบ หรือเปลี่ยนแปลง 
คุณลักษณะได้ถูกเพิ่ม ลบ และเปลี่ยนแปลง ผ่านการเปลี่ยนแปลงรหัส โดยทีมงานผลิตภัณฑ์ที่เป็นเจ้าของ สภาพแวดล้อมต้องได้รับการอัปเดตเพื่อให้ได้รับการเปลี่ยนแปลงดังกล่าว

### <a name="does-a-feature-become-mandatory-automatically"></a>คุณลักษณะกลายเป็นข้อบังคับโดยอัตโนมัติหรือไม่ 
ไม่ คุณลักษณะไม่ได้กลายเป็นข้อบังคับโดยอัตโนมัติหรือไม่ ทีมงานผลิตภัณฑ์ที่เป็นเจ้าของ ต้องทำการเปลี่ยนแปลงรหัส

### <a name="why-isnt-there-a-specific-mandatory-enabled-date"></a>เหตุใดจึงไม่มี 'วันที่เปิดใช้งานแบบบังคับ' เฉพาะ 
การปรับปรุงช่วงเวลาที่นำออกใช้เป็นตัวแปร ช่วงเวลาการปรับปรุงสภาพแวดล้อมเป็นตัวแปร และลูกค้าสามารถเลือกที่จะข้ามการปรับปรุงบางอย่างได้ ด้วยเหตุนี้ วันที่ที่เฉพาะเจาะจงจึงยากที่จะกำหนด 

### <a name="wheres-the-documentation-for-features-that-are-mandatory"></a>เอกสารประกอบสำหรับคุณลักษณะที่เป็นแบบบังคับอยู่ที่ใด 
เอกสารประกอบนี้มาจากแต่ละทีมแอปพลิเคชัน Dynamics 365 คุณลักษณะเหล่านี้มักจะกล่าวถึงใน [การอัปเดตสถานะคุณลักษณะของไคลเอนต์](/dynamics365-release-plan/2021wave1/finance-operations/finance-operations-crossapp-capabilities/updates-client-feature-states) หรือ [คุณลักษณะที่ลบออกหรือคุณลักษณะที่เลิกใช้](../../../dev-itpro/migration-upgrade/deprecated-features.md) 

### <a name="is-there-an-in-product-notification-or-signal-that-a-feature-is-going-to-be-mandatory-enabled"></a>มีการแจ้งเตือนหรือสัญญาณแบบในผลิตภัณฑ์ที่จะมีการเปิดใช้งานคุณลักษณะให้เป็นแบบบังคับหรือไม่ 
ไม่มีกลไกการแจ้งเตือนที่เกี่ยวข้องกับการทำให้คุณลักษณะเป็นแบบบังคับอยู่ในปัจจุบัน

### <a name="do-features-ever-get-enabled-without-the-customer-knowing-about-it"></a>เคยมีการเปิดใช้งานคุณลักษณะโดยที่ลูกค้าไม่ทราบหรือไม่ 
ได้ คุณสามารถเปิดใช้งานคุณลักษณะ โดยไม่มีความรู้ของลูกค้าในสถานการณ์ต่อไปนี้
- คุณลักษณะถูกย้ายไปยัง **เปิด ตามค่าเริ่มต้น** ในสถานะนี้ คุณลักษณะนี้ยังคงสามารถปิดใช้งานได้ 
- คุณลักษณะถูกอัปเดตเป็น **บังคับ** การเปลี่ยนแปลงนี้จะเกิดขึ้นร่วมกับรุ่นหลักเท่านั้น อาจมีข้อยกเว้น คุณลักษณะที่สำคัญถูกย้ายไปยัง **บังคับ** ที่การอัปเดตใดๆ

### <a name="what-is-feature-flighting-and-how-does-it-relate-to-feature-management"></a>การสลับคุณลักษณะคืออะไร และเกี่ยวข้องกับการจัดการคุณลักษณะอย่างไร 
การสลับคุณลักษณะคือสวิตช์เปิด/ปิดแบบเรียลไทม์ที่ Microsoft ควบคุม ซึ่งแยกต่างหากจากการควบคุมของลูกค้าที่ให้ไว้โดยการจัดการคุณลักษณะ 
- คุณลักษณะตัวอย่างส่วนตัวจะไม่ถูกแสดงไว้ในการจัดการคุณลักษณะ จนกว่าจะมีการสลับเป็นเปิด ในการผลิต ลูกค้าต้องตกลงที่จะเป็นส่วนหนึ่งของโปรแกรมพิเศษเพื่อให้เกิดขึ้น
- คุณลักษณะตัวอย่างสำหรับสาธารณะและที่นำออกใช้ (พร้อมใช้งานโดยทั่วไป) จะแสดงรายการในการจัดการคุณลักษณะ ยกเว้นว่าจะมีการสลับเป็นปิด การสลับคุณลักษณะเป็นปิดจะถือเป็นตัวเลือกการใช้ล่าสุดสำหรับทีมผลิตภัณฑ์ หากพบปัญหาที่สำคัญและมักจะเป็นการดำเนินงานสำหรับลูกค้าแต่ละราย

### <a name="do-features-ever-get-flighted-off-without-the-customer-knowing-about-it"></a>เคยมีการสลับคุณลักษณะเป็นปิดโดยที่ลูกค้าไม่ทราบหรือไม่ 
ใช่ ถ้าคุณลักษณะมีผลกระทบต่อการทำงานของสภาพแวดล้อมที่ไม่มีผลกระทบต่อการทำงาน จะสามารถเปิดใช้งานตามค่าเริ่มต้นได้

### <a name="how-can-feature-enablement-be-checked-in-code"></a>การเปิดใช้งานลักษณะการทำงานสามารถตรวจสอบรหัสได้อย่างไร
ใช้วิธีการ **isFeatureEnabled** บนคลาส **FeatureStateProvider** ผ่านอินสแตนซ์ของคลาสของลักษณะการทำงาน ให้ใส่คำหรือวลีที่คุณกำลังค้นหาไว้ในเครื่องหมายคำพูด

```xpp
if (FeatureStateProvider::isFeatureEnabled(BatchContentionPreventionFeature::instance()))
```

### <a name="how-can-feature-enablement-be-checked-in-metadata"></a>การเปิดใช้งานลักษณะการทำงานสามารถตรวจสอบข้อมูลเมตาได้อย่างไร
คุณสมบัติ **FeatureClass** สามารถใช้เพื่อบ่งชี้ว่าข้อมูลเมตาบางอย่างเชื่อมโยงกับลักษณะการทำงาน ชื่อคลาสที่ใช้สำหรับลักษณะการทำงานควรใช้ เช่น **BatchContentionPreventionFeature** ข้อมูลเมตานี้จะมองเห็นได้เฉพาะในลักษณะการทำงานนั้นเท่านั้น คุณสมบัติ **FeatureClass** มีอยู่บนเมนู รายการเมนู ค่า enum และฟิลด์ตาราง/มุมมอง

### <a name="what-is-a-feature-class"></a>คลาสลักษณะการทำงานคืออะไร
ลักษณะการทำงานในการจัดการลักษณะการทำงานกำหนดเป็น *คลาสลักษณะการทำงาน* คลาสลักษณะการทำงาน **ดำเนินการ IFeatureMetadata** และใช้แอททริบิวต์คลาสของลักษณะการทำงาน เพื่อระบุตัวเองให้กับพื้นที่ทำงานการจัดการลักษณะการทำงาน มีตัวอย่างจำนวนมากของคลาสลักษณะการทำงานที่สามารถตรวจสอบสำหรับการเปิดใช้งานในรหัสโดยใช้ API **FeatureStateProvider** และในเมตาดาต้าโดยใช้คุณสมบัติ **FeatureClass** ให้ใส่คำหรือวลีที่คุณกำลังค้นหาไว้ในเครื่องหมายคำพูด

```xpp
[ExportAttribute(identifierStr(Microsoft.Dynamics.ApplicationPlatform.FeatureExposure.IFeatureMetadata))]
internal final class BankCurrencyRevalGlobalEnableFeature implements IFeatureMetadata
```

### <a name="what-is-the-ifeaturelifecycle-implemented-by-some-feature-classes"></a>IFeatureLifecycle ที่ใช้โดยคลาสที่มีลักษณะการทำงานคืออะไร
IFeatureLifecycle เป็นกลไกภายในของ Microsoft สำหรับการบ่งชี้ระยะของววงจรการใช้งานของลักษณะการทำงาน ลักษณะการทำงานอาจเป็น:
- `PrivatePreview` - ต้องการเที่ยวบินเพื่อให้มองเห็นได้
- `PublicPreview` - แสดงตามค่าเริ่มต้นแต่มีคำเตือนว่าลักษณะการทำงานอยู่ในการแสดงตัวอย่าง
- `Released` - ออกใช้ทั้งหมด
