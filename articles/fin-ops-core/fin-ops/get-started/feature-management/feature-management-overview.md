# <a name="feature-management-overview"></a>ภาพรวมของการจัดการคุณลักษณะ

[!include [banner](../../includes/banner.md)]

จะมีการเพิ่มเติมและปรับปรุงคุณลักษณะทุกครั้งที่นำไปใช้ ประสบการณ์การจัดการคุณลักษณะให้พื้นที่ทำงานที่คุณสามารถดูรายการของคุณลักษณะที่ได้ถูกจัดส่งในการนำออกใช้แต่ละครั้ง ตามค่าเริ่มต้น คุณลักษณะใหม่จะถูกปิด คุณสามารถใช้พื้นที่ทำงานเพื่อเปิดและดูเอกสารประกอบได้

## <a name="the-feature-management-workspace"></a>พื้นที่ทำงานการจัดการคุณลักษณะ

คุณสามารถเปิดพื้นที่ทำงาน **การจัดการคุณลักษณะ** ด้วยการเลือกไทล์ที่เหมาะสมบนแดชบอร์ด คุณจะเห็นหน้าที่แสดงรายการของคุณลักษณะสำหรับการนำออกใช้ทั้งหมดที่ได้รับการสนับสนุนโดยประสบการณ์การจัดการคุณลักษณะ เมื่อเวลาผ่านไป Microsoft จะปรับปรุงประสบการณ์การจัดการคุณลักษณะ เพื่อให้มีฟังก์ชันเพิ่มเติมเพื่อช่วยคุณในการจัดการคุณลักษณะ

รายการคุณลักษณะประกอบด้วยข้อมูลต่อไปนี้:

- **ชื่อคุณลักษณะ** – คำอธิบายของคุณลักษณะที่ถูกเพิ่ม
- **สถานะที่เปิดใช้งาน** – สัญลักษณ์บ่งชี้ว่าคุณลักษณะถูกเปิด (เครื่องหมายถูก) หรือไม่ถูกเปิด (ว่างเปล่า) ได้รับการจัดกำหนดการให้เปิด (นาฬิกา) หรือถูกเปิดโดยบังคับ (ล็อค) ต้องการความสนใจก่อนที่คุณจะเปิด (คำเตือน) หรือไม่สามารถเปิดใช้งานได้ (X) มีการใช้การตั้งค่าที่แสดงไว้สำหรับนิติบุคคลทั้งหมด โปรดทราบว่า แม้ว่าเมื่อคุณลักษณะถูกเปิดใช้งาน จะยังคงถูกควบคุมโดยความปลอดภัย ดังนั้น คุณลักษณะนี้จะพร้อมใช้งานเฉพาะสำหรับผู้ใช้ที่มีสิทธิ์เข้าถึง ตามบทบาทความปลอดภัยของพวกเขา นอกจากนี้ ยังพร้อมใช้งานเฉพาะในนิติบุคคลที่ผู้ใช้มีสิทธิ์เข้าถึงเท่านั้น
- **วันที่เปิดใช้งาน** – วันที่ที่มีการเปิดคุณลักษณะ หรือมีการจัดกำหนดการให้ถูกเปิด
- **เพิ่มคุณลักษณะแล้ว** – วันที่ที่มีการเพิ่มคุณลักษณะไปยังสภาพแวดล้อมของคุณ วันที่นี้จะถูกป้อนโดยอัตโนมัติ เมื่อคุณปรับปรุงสภาพแวดล้อมของคุณในระหว่างรอบการนำออกใช้รายเดือน
- **โมดูล** – โมดูลที่ได้รับผลกระทบจากคุณลักษณะใหม่

เมื่อคุณเลือกคุณลักษณะ ข้อมูลเพิ่มเติมจะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียดทางด้านขวาของรายการคุณลักษณะ ที่ด้านบนของบานหน้าต่าง คุณจะเห็นชื่อคุณลักษณะ วันที่ที่มีการเพิ่มคุณลักษณะ โมดูลที่ได้รับผลกระทบจากคุณลักษณะ และลิงค์ **เรียนรู้เพิ่มเติม** เลือกลิงค์นี้เพื่อดูเอกสารประกอบสำหรับคุณลักษณะ ถ้าเอกสารไม่พร้อมใช้งาน คุณจะถูกพาไปยังหน้าชั่วคราว บานหน้าต่างรายละเอียดจะรวมฟิลด์ **ข้อคิดเห็น** ที่ซึ่งคุณสามารถเพิ่มข้อคิดเห็นของคุณเองเกี่ยวกับคุณลักษณะได้

พื้นที่ทำงาน **การจัดการคุณลักษณะ** ยังมีแท็บหลายแท็บ ซึ่งแต่ละแท็บแสดงรายการของคุณลักษณะ

- **ใหม่** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ถูกเพิ่มไว้ นับตั้งแต่การปรับปรุงรายเดือนครั้งสุดท้าย ถ้าคุณได้ข้ามการปรับปรุงรายเดือนใดๆ แท็บจะแสดงคุณลักษณะใหม่ทั้งหมดที่ถูกเพิ่มตั้งแต่ครั้งล่าสุดที่คุณปรับปรุง คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ จำนวนรวมของคุณลักษณะใหม่จะแสดงอยู่ในไทล์ที่ด้านบนของหน้าด้วย
- **ไม่ได้เปิดใช้งาน** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ยังไม่ได้ถูกเปิด คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ จำนวนรวมของคุณลักษณะใหม่ที่ไม่ได้ถูกเปิดจะแสดงอยู่ในไทล์ที่ด้านบนของหน้าด้วย
- **จัดกำหนดการ** – แท็บนี้แสดงคุณลักษณะทั้งหมดที่ได้ถูกจัดกำหนดการให้เปิดในอนาคต คุณลักษณะที่มีวันที่ที่จัดกำหนดการที่เร็วที่สุด จะปรากฏขึ้นที่ด้านบนของรายการ นอกจากนี้ จำนวนรวมของคุณลักษณะใหม่ที่จัดกำหนดการจะแสดงอยู่ในไทล์ที่ด้านบนของหน้าด้วย
- **ทั้งหมด** – แท็บนี้แสดงคุณลักษณะทั้งหมด คุณลักษณะใหม่ล่าสุดจะปรากฏขึ้นที่ด้านบนของรายการ

## <a name="turn-on-a-feature"></a>เปิดคุณลักษณะ

ถ้าไม่ได้เปิดคุณลักษณะ ปุ่ม **เปิดใช้งานตอนนี้** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อเปิดคุณลักษณะ

- เลือกคุณลักษณะที่จะเปิด และจากนั้น ในบานหน้าต่างแสดงรายละเอียด เลือก **เปิดใช้งานตอนนี้** คุณลักษณะถูกเปิด

คุณลักษณะบางอย่างไม่สามารถปิดได้ หลังจากที่คุณเปิด ถ้าคุณลักษณะที่คุณกำลังพยายามเปิด ไม่สามารถปิดได้ คุณจะได้รับคำเตือน ณ จุดนั้น คุณสามารถเลือก **ยกเลิก** เพื่อยกเลิกการดำเนินงาน และปล่อยให้คุณลักษณะถูกปิด อย่างไรก็ตาม ถ้าคุณเลือก **เปิดใช้งาน** เพื่อเปิดคุณลักษณะ คุณจะไม่สามารถปิดได้ในภายหลัง

คุณลักษณะบางอย่างจะแสดงข้อความที่ให้ข้อมูลเพิ่มเติม ก่อนที่คุณจะเปิดใช้งาน คุณสมบัติเหล่านี้จะถูกระบุด้วยสัญลักษณ์คำเตือนสีเหลือง คุณควรอ่านข้อมูลเพิ่มเติมอย่างระมัดระวังเพื่อให้เข้าใจสิ่งที่จะเกิดขึ้นได้ดียิ่งขึ้น เมื่อมีการเปิดใช้งานคุณลักษณะนี้ อย่างไรก็ตาม คุณยังคงสามารถเลือก **เปิดใช้งาน** เพื่อเปิดคุณลักษณะ

คุณลักษณะบางอย่างจะแสดงข้อความว่า คุณลักษณะนี้ไม่สามารถเปิดใช้งานได้จนกว่าจะมีการดำเนินการ คุณสมบัติเหล่านี้จะถูกระบุด้วยสัญลักษณ์ X สีแดง คุณต้องดำเนินการตามที่อธิบายไว้ในคำอธิบาย ก่อนที่จะเปิดใช้งานคุณลักษณะนี้ ตัวอย่างเช่น ถ้าคุณไม่สามารถใช้คุณลักษณะได้จนกว่าคีย์การตั้งค่าคอนฟิกถูกปิดใช้งาน จากนั้น คุณต้องปิดใช้งานคีย์การตั้งค่าคอนฟิกก่อน แล้วจากนั้น จึงกลับไปที่การจัดการคุณลักษณะเพื่อเปิดใช้งานคุณลักษณะนี้

หลังจากการปิดคุณลักษณะ ข้อความจะปรากฏขึ้นด้านล่างลิงค์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะถูกเปิด หรือบ่งชี้วันที่ในอนาคตที่คุณลักษณะจะถูกจัดกำหนดการให้เปิด อย่างใดอย่างหนึ่ง ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ

คุณลักษณะที่ถูกกำหนดให้เปิดอยู่ในอนาคตจะปรากฏบน**แท็บ**จัดตารางการผลิต กระบวนการชุดงานจะเปิดขึ้นในเวลาเที่ยงคืนของวันที่ที่ระบุตามโซนเวลาที่แสดงอยู่ในวันที่ของระบบ

## <a name="reschedule-a-feature"></a>จัดกำหนดการคุณลักษณะใหม่

ถ้ามีการจัดกำหนดการคุณลักษณะให้เปิดในอนาคต ปุ่ม **จัดกำหนดการ** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อเปลี่ยนค่า **วันที่เปิดใช้งาน** เป็นวันที่อื่นได้

1. เลือกคุณลักษณะที่จัดกำหนดการที่จะจัดกำหนดการใหม่ และจากนั้น ในบานหน้าต่างแสดงรายละเอียด เลือก **จัดกำหนดการ**
2. ในกล่องโต้ตอบที่ปรากฏขึ้น ในฟิลด์ **วันที่เปิดใช้งาน** ให้ระบุวันที่ใหม่ที่คุณลักษณะควรถูกเปิด
3. เลือก **เปิดใช้งาน** เพื่อจัดกำหนดการคุณลักษณะ หรือ **ปิดใช้งาน** เพื่อยกเลิกกำหนดการ

## <a name="turn-off-a-feature"></a>ปิดคุณลักษณะ

ถ้ามีการเปิดคุณลักษณะอยู่แล้ว ปุ่ม **ปิดใช้งาน** จะปรากฏขึ้นในบานหน้าต่างแสดงรายละเอียด คุณสามารถใช้ปุ่มนี้เพื่อปิดคุณลักษณะ ปุ่ม **ปิดใช้งาน** จะไม่พร้อมใช้งาน ถ้าไม่สามารถปิดคุณลักษณะได้ หลังจากที่เปิด

- เลือกคุณลักษณะที่จะปิด และจากนั้น ในบานหน้าต่างแสดงรายละเอียด เลือก **ปิดใช้งาน** คุณลักษณะถูกปิด และฟิลด์ **วันที่เปิดใช้งาน** จะถูกล้างข้อมูล

หลังจากการปิดคุณลักษณะ ข้อความจะปรากฏขึ้นด้านล่างลิงค์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะยังไม่ได้ถูกเปิด ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ คุณลักษณะที่ยังไม่ได้ถูกเปิดจะปรากฏขึ้นบนแท็บ **ไม่ได้เปิดใช้งาน**

## <a name="features-that-must-be-turned-on"></a>คุณลักษณะที่ต้องถูกเปิด

บางครั้ง คุณลักษณะที่สำคัญจะถูกจัดส่งที่ต้องถูกเปิดโดยอัตโนมัติ เมื่อคุณทำการปรับปรุง คุณลักษณะเหล่านี้จะถูกเปิดโดยอัตโนมัติในวันที่ที่ระบุไว้ในฟิลด์ **วันที่เปิดใช้งาน** สำหรับคุณลักษณะเหล่านี้ ข้อความจะปรากฏขึ้นด้านล่างลิงค์ **เรียนรู้เพิ่มเติม** ในบานหน้าต่างแสดงรายละเอียด ข้อความนี้ระบุว่าคุณลักษณะถูกเปิด หรือบ่งชี้วันที่ในอนาคตที่คุณลักษณะจะถูกเปิด อย่างใดอย่างหนึ่ง ซึ่งจะปรากฏขึ้นทุกครั้งที่คุณเลือกคุณลักษณะในรายการคุณลักษณะ

## <a name="enable-all-features"></a>เปิดใช้งานคุณลักษณะทั้งหมด

โดยค่าเริ่มต้น คุณลักษณะที่ถูกเพิ่มเข้าในสภาพแวดล้อมของคุณจะถูกปิด คุณสามารถเปิดใช้งานคุณลักษณะทั้งหมดได้โดยการเลือกปุ่ม **เปิดใช้งานทั้งหมด** 

เมื่อคุณเลือก **เปิดใช้งานทั้งหมด** ตัวเลือกจะปรากฏในตำแหน่งที่คุณต้องการให้ข้อมูลต่อไปนี้:
- รายการของคุณลักษณะทั้งหมดที่ต้องการการยืนยัน ก่อนที่จะสามารถเปิดใช้งานได้ ถ้าคุณต้องการเปิดใช้งานคุณลักษณะในรายการ ให้เลือก **ใช่** สำหรับปุ่ม **เปิดใช้งานคุณลักษณะที่ต้องการการยืนยัน**
- รายการของคุณลักษณะทั้งหมดที่ไม่สามารถเปิดใช้งานได้จะถูกแสดง คุณลักษณะเหล่านั้นจะไม่ถูกเปิดใช้งาน

คุณลักษณะทั้งหมดที่สามารถเปิดใช้งานได้จะถูกเปิดใช้งาน ถ้าคุณลักษณะถูกจัดกำหนดการให้เปิดใช้งานเรียบร้อยแล้วในอนาคต กำหนดการจะไม่เปลี่ยนแปลง 

## <a name="turn-on-all-features-automatically"></a>เปิดคุณลักษณะทั้งหมดโดยอัตโนมัติ

โดยค่าเริ่มต้น คุณลักษณะที่ถูกเพิ่มเข้าในสภาพแวดล้อมของคุณจะถูกปิด เว้นแต่จะเป็นคุณลักษณะแบบบังคับ อย่างไรก็ตาม ถ้าคุณต้องการเปิดคุณลักษณะใหม่ทั้งหมดโดยอัตโนมัติ คุณสามารถใช้รายการแบบหล่นลงภายใต้ชื่อพื้นที่ทำงาน เพื่อเปลี่ยนแปลงสิ่งที่เกิดขึ้นเมื่อมีการเพิ่มคุณลักษณะใหม่

- เลือก **เปิดใช้งานคุณลักษณะใหม่โดยอัตโนมัติ** เพื่อเปิดคุณลักษณะใหม่ทั้งหมดโดยอัตโนมัติ เมื่อมีการเพิ่มไปยังสภาพแวดล้อมของคุณ
- เลือก **ห้ามเปิดใช้งานคุณลักษณะใหม่โดยอัตโนมัติ** เพื่อตั้งค่าเริ่มต้นคุณลักษณะใหม่ทั้งหมดเป็นปิด เมื่อมีการเพิ่มไปยังสภาพแวดล้อมของคุณ


เมื่อคุณเปิดใช้งานคุณลักษณะทั้งหมดโดยอัตโนมัติ จะเปิดใช้งานคุณลักษณะทั้งหมดที่จะถูกเปิดใช้งานเมื่อคุณคลิกปุ่ม **เปิดใช้งานทั้งหมด** จะไม่เปิดใช้งานคุณลักษณะที่ต้องการการยืนยัน หรือคุณลักษณะที่ไม่สามารถเปิดใช้งานได้จนกว่าจะมีการดำเนินการ

## <a name="check-for-updates"></a>ตรวจหาการอัปเดต

คุณลักษณะจะถูกเพิ่มลงในสภาพแวดล้อมของคุณ หลังจากการปรับปรุงแต่ละครั้ง อย่างไรก็ตาม คุณสามารถตรวจสอบการปรับปรุงด้วยตนเองได้โดยการคลิกที่ปุ่ม **ตรวจสอบการปรับปรุง** คุณลักษณะใดๆ ที่ถูกเพิ่มลงในระบบ หลังจากที่การปรับปรุงจะถูกเพิ่มลงในรายการของคุณลักษณะ ตัวอย่างเช่น ถ้าคุณลักษณะที่รวดเร็วถูกเปิดใช้งานหลังจากการนำออกใช้งาน จากนั้น คุณจะสามารถตรวจสอบการปรับปรุงได้ และคุณลักษณะจะถูกเพิ่มลงในรายการของคุณ

## <a name="assigning-roles"></a>การกำหนดบทบาท

พื้นที่ทำงาน **การจัดการคุณลักษณะ** สามารถเปิดได้โดยผู้ดูแลระบบ และโดยผู้ใช้ที่ได้รับการกำหนดให้กับบทบาทตัวจัดการคุณลักษณะ หรือบทบาทตัวแสดงคุณลักษณะด้วย บทบาทสองบทบาทนี้ถูกสร้างขึ้นเพื่อสนับสนุนประสบการณ์การจัดการคุณลักษณะ ผู้ใช้ในบทบาทผู้จัดการคุณลักษณะสามารถเปิดหรือปิดคุณลักษณะใดๆ ได้ นอกจากนี้ ยังสามารถปรับปรุงฟิลด์ **ความคิดเห็น** สำหรับคุณลักษณะได้ด้วย ผู้ใช้ในบทบาทของตัวแสดงคุณลักษณะสามารถดูได้เฉพาะพื้นที่ทำงาน **การจัดการคุณลักษณะ** ผู้ใช้ไม่สามารถเปิดหรือปิดคุณลักษณะได้

บทบาทของตัวจัดการคุณลักษณะ และบทบาทของตัวแสดงคุณลักษณะ ไม่ได้แทนที่ความปลอดภัยที่มีอยู่ซึ่งผู้ใช้มี เพียงควบคุมว่าผู้ใช้สามารถเปิดและปิดคุณลักษณะการทำงานได้หรือไม่ จะไม่ให้การเข้าถึงไปยังคุณลักษณะเอง

## <a name="features-that-use-configuration-keys"></a>คุณลักษณะที่ใช้คีย์การตั้งค่าคอนฟิก

ถ้าคุณลักษณะใช้คีย์การตั้งค่าคอนฟิก แต่ไม่มีการเปิดคีย์การตั้งค่าคอนฟิก พื้นที่ทำงาน **การจัดการคุณลักษณะ** จะไม่แสดงคุณลักษณะในรายการของคุณลักษณะที่พร้อมใช้งาน หลังจากที่คุณเปิดคีย์การตั้งค่าคอนฟิก คุณต้องปรับปรุงรายการคุณลักษณะโดยใช้รายการเมนู **ตรวจสอบสำหรับการปรับปรุง** จากนั้น คุณลักษณะจะปรากฏในรายการคุณลักษณะ

ถ้าคุณปิดคีย์การตั้งค่าคอนฟิก คุณลักษณะจะไม่ถูกลบออกจากรายการคุณลักษณะ

## <a name="data-entities"></a>เอนทิตี้ข้อมูล

เอนทิตีข้อมูลที่มีชื่อว่า **การจัดการคุณลักษณะ** ช่วยให้คุณสามารถส่งออกการตั้งค่าการจัดการคุณลักษณะจากสภาพแวดล้อมหนึ่ง แล้วนำเข้าไปยังสภาพแวดล้อมอื่น เอนทิตีนี้ปรับปรุงคุณลักษณะที่มีอยู่เท่านั้น ตรรกะทางธุรกิจในเอนทิตียังช่วยรับประกันว่า กฎเดียวกันที่ใช้ในพื้นที่ทำงาน **การจัดการคุณลักษณะ** จะถูกนำไปใช้ เมื่อการนำเข้าเสร็จสิ้น ตัวอย่างเช่น คุณไม่สามารถแทนที่การตั้งค่าคุณลักษณะแบบบังคับได้โดยการลบวันที่ในระหว่างการนำเข้า

ตัวอย่างต่อไปนี้จะอธิบายสิ่งที่เกิดขึ้น เมื่อคุณใช้เอนทิตี **การจัดการคุณลักษณะ** ในการนำเข้าข้อมูล

- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** คุณลักษณะนี้จะถูกเปิด และฟิลด์ **วันที่เปิดใช้งาน** จะถูกกำหนดเป็นวันที่ปัจจุบัน
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** หรือปล่อยให้ฟิลด์ **EnableDate** ว่างเปล่า คุณลักษณะนี้จะถูกปิด และฟิลด์ **วันที่เปิดใช้งาน** จะถูกล้างข้อมูล คุณไม่สามารถปิดคุณลักษณะแบบบังคับ หรือคุณลักษณะที่ไม่สามารถปิดได้หลังจากที่เปิดแล้ว
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะจะมีการจัดกำหนดการสำหรับวันที่นั้น
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ใช่** และเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะนี้จะถูกจัดกำหนดการสำหรับวันที่นั้น 
- ถ้าคุณเปลี่ยนค่าของฟิลด์ **เปิดใช้งาน** เป็น **ไม่** แต่นอกจากนี้ คุณยังเปลี่ยนค่าของฟิลด์ **EnableDate** เป็นวันที่ในอนาคต คุณลักษณะนี้จะถูกจัดกำหนดการสำหรับวันที่นั้น
- ถ้ามีการเปิดคุณลักษณะ และคุณเพิ่มฟิลด์ **EnableDate** ที่ถูกตั้งค่าเป็นวันที่ในอนาคต คุณลักษณะจะยังคงเปิดอยู่ เมื่อต้องการจัดกำหนดการคุณลักษณะใหม่ คุณต้องเปลี่ยนฟิลด์ **เปิดใช้งาน** เป็น **ไม่**

## <a name="feature-management-and-flighting"></a>การจัดการคุณลักษณะและการสลับ

การจัดการคุณลักษณะช่วยให้คุณสามารถควบคุมคุณลักษณะที่ถูกจัดส่งในการนำออกใช้แต่ละครั้ง การสลับช่วยให้ Microsoft teams สามารถนำออกใช้คุณลักษณะไปยังลูกค้าจำนวนจำกัด เพื่อให้คุณลักษณะเหล่านั้นสามารถได้รับการทดสอบและได้รับการตรวจสอบความถูกต้องโดยไม่มีผลกระทบต่อลูกค้าทั้งหมด การจัดการคุณลักษณะไม่ได้ควบคุมการสลับของคุณลักษณะใดๆ

## <a name="using-feature-management-to-turn-on-isv-features-or-custom-features"></a>การใช้การจัดการคุณลักษณะเพื่อเปิดคุณลักษณะ ISV หรือคุณลักษณะที่กำหนดเอง

ปัจจุบันการจัดการคุณลักษณะไม่พร้อมใช้งานสำหรับคุณลักษณะจากผู้จำหน่ายซอฟต์แวร์อิสระ (ISVs) และคุณลักษณะที่กำหนดเอง อย่างไรก็ตาม Microsoft กำลังเพิ่มฟังก์ชันเพิ่มเติมเพื่อปรับปรุงการจัดการคุณลักษณะ หลังจากการปรับปรุงเหล่านั้นเสร็จสมบูรณ์ Microsoft จะทำให้การจัดการคุณลักษณะพร้อมใช้งานสำหรับคุณลักษณะทั้งหมด และให้คำแนะนำเกี่ยวกับการปรับปรุงคุณลักษณะของคุณในการใช้งาน

## <a name="frequently-asked-questions-faq"></a>คำถามที่ถามบ่อย (FAQ)

### <a name="when-are-features-added-removed-or-changed"></a>เมื่อใดที่คุณลักษณะถูกเพิ่ม ลบ หรือเปลี่ยนแปลง 
มีการเพิ่ม ลบ และเปลี่ยนแปลงคุณลักษณะ ผ่านการเปลี่ยนแปลงรหัส จำเป็นต้องปรับปรุงสภาพแวดล้อมเพื่อให้ได้รับการเปลี่ยนแปลงดังกล่าว

### <a name="does-a-feature-become-mandatory-automatically"></a>คุณลักษณะกลายเป็นข้อบังคับโดยอัตโนมัติหรือไม่ 
ไม่ คุณลักษณะที่กลายเป็นข้อบังคับไม่ใช่การดำเนินการอัตโนมัติ ทีมผลิตภัณฑ์จำเป็นต้องทำการเปลี่ยนแปลงรหัส

### <a name="when-do-features-become-mandatory"></a>คุณลักษณะกลายเป็นข้อมูลบังคับเมื่อใด 
นโยบายคือคุณลักษณะใหม่ทั้งหมดจะเข้าร่วมสำหรับรอบระยะเวลา12 เดือน และจะไม่ต้องมีการจัดการการเปลี่ยนแปลงใดๆ จนกว่าคุณจะเปิดใช้งานคุณลักษณะนี้ ทีมผลิตภัณฑ์สามารถเลือกว่าจะทำให้คุณลักษณะเป็นข้อบังคับหรือไม่ หลังจากรอบระยะเวลานั้นสิ้นสุดแล้ว 

### <a name="why-isnt-there-a-specific-mandatory-enabled-date"></a>เหตุใดจึงไม่มี 'วันที่เปิดใช้งานแบบบังคับ' เฉพาะ 
การปรับปรุงช่วงเวลาที่นำออกใช้เป็นตัวแปร ช่วงเวลาการปรับปรุงสภาพแวดล้อมเป็นตัวแปร และลูกค้าสามารถเลือกที่จะข้ามการปรับปรุงบางอย่างได้ ด้วยเหตุนี้ วันที่ที่เฉพาะเจาะจงจึงยากที่จะกำหนด 

### <a name="wheres-the-documentation-for-features-that-are-being-made-mandatory"></a>เอกสารประกอบสำหรับคุณลักษณะที่ถูกทำให้เป็นแบบบังคับอยู่ที่ใด 
เอกสารประกอบนี้มาจากทึมแอพลิเคชัน บ่อยครั้งที่รายการเหล่านี้จะถูกกล่าวถึงใน [คุณลักษณะที่ถูกลบหรือที่ถูกเลิกใช้](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/migration-upgrade/deprecated-features) 

### <a name="is-there-an-in-product-notification-or-signal-that-a-feature-is-going-to-be-mandatory-enabled"></a>มีการแจ้งเตือนหรือสัญญาณแบบในผลิตภัณฑ์ที่จะมีการเปิดใช้งานคุณลักษณะให้เป็นแบบบังคับหรือไม่ 
ไม่มีกลไกการแจ้งเตือนที่เกี่ยวข้องกับการทำให้คุณลักษณะเป็นแบบบังคับอยู่ในปัจจุบัน

### <a name="do-features-ever-get-enabled-without-the-customer-knowing-about-it"></a>เคยมีการเปิดใช้งานคุณลักษณะโดยที่ลูกค้าไม่ทราบหรือไม่ 
ใช่ ถ้าคุณลักษณะไม่มีผลกระทบต่อการทำงาน จะสามารถเปิดใช้งานได้ตามค่าเริ่มต้น

### <a name="what-is-feature-flighting-and-how-does-it-relate-to-feature-management"></a>การสลับคุณลักษณะคืออะไร และเกี่ยวข้องกับการจัดการคุณลักษณะอย่างไร 
การสลับคุณลักษณะคือสวิตช์เปิด/ปิดแบบเรียลไทม์ที่ Microsoft ควบคุม ซึ่งแยกต่างหากจากการควบคุมของลูกค้าที่ให้ไว้โดยการจัดการคุณลักษณะ 
- คุณลักษณะตัวอย่างส่วนตัวจะไม่ถูกแสดงไว้ในการจัดการคุณลักษณะ จนกว่าจะมีการสลับเป็นเปิด ในการผลิต ลูกค้าต้องตกลงที่จะเป็นส่วนหนึ่งของโปรแกรมพิเศษเพื่อให้เกิดขึ้น
- คุณลักษณะตัวอย่างสำหรับสาธารณะและที่นำออกใช้ (พร้อมใช้งานโดยทั่วไป) จะแสดงรายการในการจัดการคุณลักษณะ ยกเว้นว่าจะมีการสลับเป็นปิด การสลับคุณลักษณะเป็นปิดจะถือเป็นตัวเลือกการใช้ล่าสุดสำหรับทีมผลิตภัณฑ์ หากพบปัญหาที่สำคัญและมักจะเป็นการดำเนินงานสำหรับลูกค้าแต่ละราย

### <a name="do-features-ever-get-flighted-off-without-the-customer-knowing-about-it"></a>เคยมีการสลับคุณลักษณะเป็นปิดโดยที่ลูกค้าไม่ทราบหรือไม่ 
ใช่ ถ้าคุณลักษณะมีผลกระทบต่อการทำงานของสภาพแวดล้อมที่ไม่มีผลกระทบต่อการทำงาน จะสามารถเปิดใช้งานตามค่าเริ่มต้นได้
