## <a name="overview"></a><span data-ttu-id="69a59-110">ภาพรวม</span><span class="sxs-lookup"><span data-stu-id="69a59-110">Overview</span></span>
-   <span data-ttu-id="69a59-111">**กรอบงาน SysTest** – กรอบงาน SysTest มีความน่าเชื่อถือสำหรับการเขียน unit tests</span><span class="sxs-lookup"><span data-stu-id="69a59-111">**SysTest framework** – The SysTest framework is reliable for writing unit tests.</span></span> <span data-ttu-id="69a59-112">เนื่องจาก unit tests โดยทั่วไปจะมีการทดสอบวิธีการหรือฟังก์ชัน การทดสอบหน่วยควรเป็นการวินิจฉัยข้อมูลอยู่เสมอและขึ้นอยู่กับข้อมูลที่ป้อนไว้เท่านั้นที่ให้ไว้เป็นส่วนหนึ่งของการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="69a59-112">Because unit tests are generally testing a method or function, they should always be data agnostic and dependent only on the input data that is provided as part of the test.</span></span>
-   <span data-ttu-id="69a59-113">**กรอบงาน ATL** – Microsoft มีกรอบงาน ATL ซึ่งเป็นการเชื่อมต่อบนกรอบงาน SysTest และทำให้การทดสอบการเขียนทำงานง่ายและน่าเชื่อถือมากขึ้น</span><span class="sxs-lookup"><span data-stu-id="69a59-113">**ATL framework** – Microsoft has an ATL framework that is an abstraction on the SysTest framework and makes functional test writing much more simple and reliable.</span></span> <span data-ttu-id="69a59-114">กรอบงานนี้ควรใช้สำหรับการเขียน component tests หรือ integration tests อย่างง่าย</span><span class="sxs-lookup"><span data-stu-id="69a59-114">This framework should be used for writing component tests or simple integration tests.</span></span>
-   <span data-ttu-id="69a59-115">**RSAT** – The RSAT ถูกใช้สำหรับ integration tests และการทดสอบวงจรธุรกิจ</span><span class="sxs-lookup"><span data-stu-id="69a59-115">**RSAT** – The RSAT is used for integration tests and business cycle tests.</span></span> <span data-ttu-id="69a59-116">การทดสอบวงจรธุรกิจเรียกอีกอย่างหนึ่งว่าการทดสอบการตรวจสอบความถูกต้องของการถดถอยที่จะขึ้นอยู่กับข้อมูลที่มีอยู่</span><span class="sxs-lookup"><span data-stu-id="69a59-116">The business cycle tests, also called the regression validation tests, are dependent on existing data.</span></span> <span data-ttu-id="69a59-117">อย่างไรก็ตามการทดสอบเหล่านี้สามารถกลายมาเป็นการวินิจฉัยขอมูลถ้าคุณพิจารณาถึงปัจจัยเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="69a59-117">However, these tests can become data agnostic if you consider additional factors.</span></span> 

    - <span data-ttu-id="69a59-118">ที่ใด unit tests และ component tests อยู่ในระดับต่ำและสามารถเป็นการวินิจฉัยข้อมูลได้โดยสมบูรณ์ (ไม่ขึ้นอยู่กับชุดข้อมูลที่มีอยู่), วงจรธุรกิจหรือการทดสอบการตรวจสอบความถูกต้องของการถดถอยจะขึ้นอยู่กับข้อมูลบางอย่างที่มีอยู่</span><span class="sxs-lookup"><span data-stu-id="69a59-118">o Where unit tests and component tests are low level and can fully be data agnostic (not dependent on existing dataset), the business cycle or regression validation tests are dependent on some existing data.</span></span> <span data-ttu-id="69a59-119">ข้อมูลนี้รวมถึงการตั้งค่า การตั้งค่าคอนฟิก (พารามิเตอร์) และข้อมูลหลัก (ลูกค้า ผู้จัดจำหน่าย สินค้า และอื่นๆ) แต่ไม่มีข้อมูลธุรกรรม</span><span class="sxs-lookup"><span data-stu-id="69a59-119">This data includes setup, configuration settings (parameters), and master data (customer, vendors, items, etc.), but never transaction data.</span></span> <span data-ttu-id="69a59-120">ตรวจสอบให้แน่ใจว่ามีการเปลี่ยนแปลงใดๆเกิดขึ้นในระหว่างการทดสอบถ้ามี การเปลี่ยนแปลงข้อมูลเหล่านี้จะถูกแปลงกลับเป็นส่วนหนึ่งของการทดสอบครั้งสุดท้าย</span><span class="sxs-lookup"><span data-stu-id="69a59-120">Make sure that during the test, if any of these are being changed, that they are reverted back as part of the final test.</span></span>
    - <span data-ttu-id="69a59-121">เลือกข้อมูลหลักตามเงื่อนไขบางอย่างแทนที่จะเลือกเรกคอร์ดเฉพาะ</span><span class="sxs-lookup"><span data-stu-id="69a59-121">Select master data based on certain criteria instead of selecting a particular record.</span></span> <span data-ttu-id="69a59-122">ตัวอย่างเช่น ถ้าคุณต้องการเลือกสินค้าตามค่ามิติของสินค้าและความพร้อมใช้งานของสินค้าคงคลัง ให้กรองข้อมูลรายการผลิตภัณฑ์ที่มีค่าดังกล่าวโดยให้เลือกสินค้าแรกและคัดลอกหมายเลขที่จะใช้สำหรับการทดสอบในอนาคต</span><span class="sxs-lookup"><span data-stu-id="69a59-122">For example, if you want to select an item based on its dimension values and stock availability, filter the product list with those values, select the first item, and copy the number to be used for future tests.</span></span> <span data-ttu-id="69a59-123">ถ้าเป็นบรรทัดข้อมูลหลักอย่างง่ายเช่นลูกค้าผู้จัดจำหน่ายหรือสินค้าจะสามารถสร้างขึ้นโดยเป็นส่วนหนึ่งของการทำงานอัตโนมัติและใช้ในการทดสอบในอนาคตผ่านการผูกมัด</span><span class="sxs-lookup"><span data-stu-id="69a59-123">If it’s a simple master data line such as customer, vendor, or item, it can be created as part of the automation and used in future tests through chaining.</span></span> 
    - <span data-ttu-id="69a59-124">ป้อนตัวระบุเฉพาะเช่นหมายเลขใบแจ้งหนี้ โดยใช้ลำดับหมายเลขหรือโดยการใช้ฟังก์ชัน Microsoft Excel เช่น =TEXT(NOW(),"yyyymmddhhmm")</span><span class="sxs-lookup"><span data-stu-id="69a59-124">o Enter the unique identifiers, such as invoice numbers, through the number sequence or by using Microsoft Excel functions such as =TEXT(NOW(),"yyyymmddhhmm").</span></span> <span data-ttu-id="69a59-125">ฟังก์ชันนี้จะให้หมายเลขที่ไม่ซ้ำกันทุกนาทีซึ่งช่วยให้คุณสามารถติดตามเมื่อมีการดำเนินการเกิดขึ้น</span><span class="sxs-lookup"><span data-stu-id="69a59-125">This function will provide a unique number every minute, which allows you to track when the action happened.</span></span> <span data-ttu-id="69a59-126">นี้สามารถใช้สำหรับตัวแปรเช่นหมายเลขใบรับสินค้าและหมายเลขใบแจ้งหนี้ของผู้จัดจำหน่าย</span><span class="sxs-lookup"><span data-stu-id="69a59-126">This can be used for variables such as product receipt numbers and vendor invoice numbers.</span></span> <span data-ttu-id="69a59-127">การทดสอบเหล่านี้ยังคงทำงานอยู่ในฐานข้อมูลเดียวกันซ้ำแล้วซ้ำอีกโดยไม่ต้องมีการคืนค่าใดๆ</span><span class="sxs-lookup"><span data-stu-id="69a59-127">These tests continue to work on the same database again and again, without requiring any restoration.</span></span>
    - <span data-ttu-id="69a59-128">ตั้งค่า **โหมดการแก้ไข** ของสภาพแวดล้อมเพื่อให้ **อ่าน** หรือ **แก้ไข** เป็นกรณีการทดสอบแรกเสมอเนื่องจากตัวเลือกเริ่มต้นเป็นแบบ **อัตโนมัติ**  ตัวเลือก **อัตโนมัติ** จะใช้ในการตั้งค่าก่อนหน้านี้เสมอและสามารถทำให้เกิดการทดสอบที่ไม่น่าเชื่อถือ</span><span class="sxs-lookup"><span data-stu-id="69a59-128">Always set the **Edit mode** of the environment to **Read** or **Edit** as the first test case because the default option is **Auto**.     - <span data-ttu-id="69a59-130">ตรวจสอบความถูกต้องหลังจากที่คุณกรองข้อมูลธุรกรรมเฉพาะแทนการตรวจสอบทั่วไปเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="69a59-130">Only validate after you filter on a particular transaction instead of generic validation.</span></span> <span data-ttu-id="69a59-131">ตัวอย่างเช่นสำหรับหมายเลขของเรกคอร์ดให้กรองข้อมูลสำหรับหมายเลขธุรกรรมหรือวันที่ทำธุรกรรมเพื่อให้การตรวจสอบความถูกต้องไม่รวมธุรกรรมอื่นๆทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="69a59-131">For example, for the number of records, filter for the transaction number or the transaction date so that the validation excludes all other transactions.</span></span>
