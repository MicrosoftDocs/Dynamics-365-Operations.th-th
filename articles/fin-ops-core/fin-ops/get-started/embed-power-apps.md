# <a name="embed-canvas-apps-from-power-apps"></a>ฝังแอปพื้นที่ทำงาน จาก Power Apps

[!include [banner](../includes/banner.md)]


[!INCLUDE [PEAP](../../../includes/peap-1.md)]

Microsoft Power Apps เป็นบริการที่อนุญาตให้ทั้งนักพัฒนาและผู้ใช้ที่ไม่ใช่ทางเทคนิคสามารถสร้างแอปทางธุรกิจที่กำหนดเองได้สำหรับอุปกรณ์เคลื่อนที่ แท็บเล็ต และเว็บ โดยไม่ต้องมีการเขียนรหัส การรวมการสนับสนุนแอปการเงินและการดำเนินงานกับ Power Apps แอปพื้นที่ทำงาน หรือระบบแวดล้อมที่กว้างขึ้น ที่คุณ องค์กรของคุณพัฒนา จะสามารถฝังตัวในแอปการเงินและการดำเนินงานเพื่อเพิ่มฟังก์ชันการทำงานของผลิตภัณฑ์ ตัวอย่างเช่น คุณอาจสร้างแอปพื้นที่ทำงานจาก Power Apps เพื่อเสริมแอปการเงินและการดำเนินงานด้วยข้อมูลที่ดึงมาจากระบบอื่น

เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับการฝังแอปพื้นที่ทำงาน ดูวิดีโอ [วิธีการฝังแอปพื้นที่ทำงาน](https://www.youtube.com/watch?v=x3qyA1bH-NY) สั้นๆ

## <a name="adding-an-embedded-canvas-app-from-power-apps-to-a-page"></a>การเพิ่มแอปพื้นที่ทำงานที่ฝังจาก Power Apps ไปยังหน้าหนึ่ง

ก่อนที่จะฝังแอปพื้นที่ทำงาน จาก Power Apps ลงในไคลเอนต์ คุณต้องค้นหาหรือสร้างแอปที่มีภาพหรือฟังก์ชันการทำงานที่ต้องการ หัวข้อนี้จะไม่มีคำอธิบายโดยละเอียดเกี่ยวกับกระบวนการสำหรับการสร้างแอป ถ้าคุณเป็นมือใหม่สำหรับ Power Apps ดูที่ [คู่มือ Power Apps](/powerapps/)

มีสามวิธีในการฝังแอปพื้นที่ทำงานลงในแอปการเงินและการดำเนินงาน คุณสามารถใช้วิธีการที่เหมาะสมกับสถานการณ์ของคุณที่สุดได้ 

- ฝังแอปพื้นที่ทำงานลงในปุ่ม **Power Apps** บนบานหน้าต่างการดำเนินการมาตรฐานของหน้า แอปที่คุณเพิ่มในลักษณะนี้จะปรากฏเป็นรายการบนปุ่มเมนู **Power Apps** และแอปเปิดอยู่ในบานหน้าต่างด้านซ้าย 
- ฝังแอปพื้นที่ทำงานโดยตรงในหน้าที่มีอยู่เป็นหน้าแท็บใหม่ (แท็บ Pivot, FastTab, เบลด หรือส่วนพื้นที่ทำงาน)
- สร้างประสบการณ์การใช้งานแอปพื้นที่ทำงานแบบเต็มหน้าใหม่จากแดชบอร์ด

เมื่อคุณตั้งค่าคอนฟิกแอปพื้นที่ทำงานที่ฝังไว้ คุณสามารถเลือกฟิลด์เดียวที่คุณต้องการส่งเป็นบริบทไปยังแอป ขั้นตอนนี้เปิดการใช้งานให้แอปสามารถตอบสนอง ตามข้อมูลที่คุณกำลังดูอยู่

> [!NOTE]
> คุณไม่สามารถใช้กลไกนี้ในการฝังแอปแบบจำลองได้

### <a name="embedding-a-canvas-app-on-an-existing-page"></a>การฝังแอปพื้นที่ทำงานในหน้าที่มีอยู่

กระบวนงานต่อไปนี้แสดงวิธีการฝังแอปพื้นที่ทำงานในหน้าที่มีอยู่จาก Power Apps

1. ไปที่หน้าที่คุณต้องการฝังแอปพื้นที่ทำงาน หน้านี้จะมีข้อมูลใดๆ ที่จำเป็นต้องส่งผ่านไปยังแอปเหมือนกับข้อมูลป้อนเข้า
2. เปิดบานหน้าต่าง **เพิ่มแอปจาก Power Apps** :

    - ถ้าแอปจะถูกฝังอยู่ในหน้าโดยตรง ให้เลือก **ตัวเลือก** \> **ปรับหน้านี้ให้เป็นส่วนตัว** \> **เพิ่มเติม** แล้วปฏิบัติตามขั้นตอนใดขั้นตอนหนึ่งต่อไปนี้:

        - หากเปิดคุณลักษณะ **แอปแบบเต็มหน้า** ให้เลือก **เพิ่มหน้า** แล้วเลือกภูมิภาคที่คุณต้องการเพิ่มแอป หากต้องการฝังแอปลงในปุ่มเมนู **Power Apps** ให้เลือกบานหน้าต่างการดำเนินการ หากต้องการฝังแอปลงบนหน้าโดยตรงให้เลือกแท็บ แท็บด่วน แผ่นหรือส่วนที่เหมาะสม (ถ้าคุณอยู่ในพื้นที่ทำงาน) จากนั้นในบานหน้าต่าง **เพิ่มแอป** ให้เลือก **Power Apps**
        - หากปิดคุณลักษณะ **แอปแบบเต็มหน้า** ให้เลือก **เพิ่มหน้าจาก Power Apps** แล้วเลือกภูมิภาคที่คุณต้องการเพิ่มแอป หากต้องการฝังแอปลงในปุ่มเมนู **Power Apps** ให้เลือกบานหน้าต่างการดำเนินการ หากต้องการฝังแอปลงบนหน้าโดยตรงให้เลือกแท็บ แท็บด่วน แผ่นหรือส่วนที่เหมาะสม (ถ้าคุณอยู่ในพื้นที่ทำงาน)

    - หากแอปเข้าถึงได้โดยใช้ปุ่มเมนู **Power Apps** คุณสามารถคลิกปุ่มเมนู **Power Apps** ในบานหน้าต่างการดำเนินการมาตรฐาน แล้วเลือก **เพิ่มแอป**

3. กำหนดค่าแอปที่ฝังไว้ สำหรับข้อมูลเพิ่มเติม โปรดดูที่ส่วน [การตั้งค่าคอนฟิกแอปพื้นที่ทำงาน](#configuring-a-canvas-app) ในภายหลังในหัวข้อนี้
4. หลังจากที่คุณยืนยันว่าการตั้งค่าคอนฟิกถูกต้องแล้ว ให้เลือก **แทรก**

    - หากปิดคุณลักษณะ **มุมมองที่บันทึกไว้** ระบบจะขอให้คุณรีเฟรชเบราเซอร์เพื่อดูแอปที่ฝังอยู่
    - ถ้าคุณลักษณะ **มุมมองที่บันทึกไว้** เปิดอยู่ คุณต้องบันทึกมุมมองเพื่อให้การเปลี่ยนแปลงยังคงอยู่

### <a name="embedding-a-canvas-app-as-a-full-page-experience-from-the-dashboard"></a>การฝังแอปพื้นที่ทำงานเป็นประสบการณ์แบบเต็มหน้าจากแดชบอร์ด

คุณอาจต้องการแอปพื้นที่ทำงานจากแดชบอร์ดหากแอปไม่เกี่ยวข้องกับหน้าที่มีอยู่ หรือถ้าคุณเพียงต้องการประสบการณ์แอปแบบเต็มหน้าสำหรับแอปภายในแอปการเงินและการดำเนินงาน

> [!NOTE]
> เพื่อให้ความสามารถนี้พร้อมใช้งาน คุณต้องเปิดคุณลักษณะ **แอปแบบเต็มหน้า** ในการจัดการคุณลักษณะ 

1. เปิดแดชบอร์ด
2. เลือกหน้าค้างไว้ (หรือคลิกขวา) เลือก **กำหนดเป็นแบบส่วนบุคคล** แล้วจากนั้น เลือก **เพิ่มหน้า**
3. ในบานหน้าต่าง **เพิ่มหน้า** ให้เลือก **Power Apps**
4. กำหนดค่าแอปที่ฝังไว้ สำหรับข้อมูลเพิ่มเติม โปรดดูที่ส่วน [การตั้งค่าคอนฟิกแอปพื้นที่ทำงาน](#configuring-a-canvas-app) ในภายหลังในหัวข้อนี้
5. เลือก **บันทึก** เพื่อเพิ่มแอปลงในแดชบอร์ดเป็นไทล์ใหม่
6. เลือกไทล์ใหม่บนแดชบอร์ด และยืนยันว่าแอปพื้นที่ทำงานปรากฏตามที่คาดไว้

### <a name="configuring-a-canvas-app"></a>การตั้งค่าคอนฟิกแอปพื้นที่ทำงาน

เมื่อคุณฝังแอปพื้นที่ทำงาน คุณต้องตั้งค่าพารามิเตอร์ต่อไปนี้:

- **ชื่อ** – ป้อนข้อความที่ควรแสดงไว้สำหรับปุ่มหรือแท็บที่จะมีแอปที่ฝังอยู่ บ่อยครั้ง คุณอาจต้องการทำชื่อแอปในฟิลด์นี้ซ้ำ
- **รหัสของแอป** – ระบุรหัสเฉพาะสากล (GUID) สำหรับแอปพื้นที่ทำงานที่คุณต้องการฝัง ในการดึงค่านี้ ให้หาแอปใน [make.powerapps.com](https://make.powerapps.com) จากนั้นหาในฟิลด์ **รหัสแอป** ภายใต้ **รายละเอียด**
- **บริบทการป้อนข้อมูลสำหรับแอป** – คุณสามารถเลือกฟิลด์ที่มีข้อมูลที่คุณต้องการส่งผ่านไปยังแอปเป็นข้อมูลป้อนเข้า สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีที่แอปสามารถเข้าถึงข้อมูลที่ส่งจากแอปการเงินและการดำเนินงาน ให้ดูที่ส่วน [การสร้างแอปที่ใช้ประโยชน์จากข้อมูลที่ส่งจากแอปการเงินและการดำเนินงาน](#building-a-canvas-app-that-uses-data-that-is-sent-from-finance-and-operations-apps) ที่อยู่ต่อไปในหัวข้อนี้

    ณ รุ่น 10.0.19 นิติบุคคลปัจจุบันจะถูกส่งผ่านเป็นบริบทไปยังแอปพื้นที่ทำงานผ่านทางพารามิเตอร์ URL ของ **cmp** ด้วย ลักษณะการทำงานนี้จะไม่มีผลต่อแอปพื้นที่ทำงานเป้าหมายจนกว่าแอปนั้นจะใช้ข้อมูลนั้น

- **แอปพลิเคชัน** – เลือกชนิดของแอปที่คุณฝังไว้อยู่ เลือก **Thin** สำหรับแอปที่สร้างขึ้นสำหรับอุปกรณ์มือถือหรือ **Wide** สำหรับแอปที่สร้างขึ้นสำหรับแท็บเล็ต พารามิเตอร์นี้ทำให้แน่ใจว่ามีการจัดสรรพื้นที่เพียงพอสำหรับแอปที่ฝังไว้
- **นิติบุคคล** – คุณสามารถเลือกนิติบุคคลที่แอปนี้จะพร้อมใช้งาน ตามค่าเริ่มต้น แอปจะพร้อมใช้งานเฉพาะนิติบุคคลทั้งหมด ตัวเลือกนี้จะพร้อมใช้งานเฉพาะเมื่อคุณฝังโดยตรงในหน้าที่มีอยู่และคุณลักษณะ **[มุมมองที่บันทึกไว้](saved-views.md)** ปิดอยู่เท่านั้น

## <a name="sharing-an-embedded-app"></a>การใช้แอปที่ฝังไว้ร่วมกัน

หลังจากคุณฝังแอปพื้นที่ทำงานลงในหน้าและยืนยันว่าแอปทำงานอย่างถูกต้อง คุณอาจต้องการแบ่งปันแอปกับผู้ใช้รายอื่นในระบบ เมื่อต้องการใช้แอปพื้นที่ทำงานที่ฝัง ให้ทำตามขั้นตอนต่อไปนี้

1. [แบ่งปันแอปพื้นที่ทำงานใน Power Apps](/powerapps/maker/canvas-apps/share-app) กับผู้ใช้ที่เหมาะสม เพื่อให้สามารถเข้าถึงแอปโดยตรงใน Power Apps
2. แบ่งปันการตั้งค่าส่วนบุคคลที่เชื่อมโยงกับแอปที่ฝังอยู่กับผู้ใช้ที่ต้องการ คุณสามารถใช้อย่างใดอย่างหนึ่งต่อไปนี้:

    - **เผยแพร่มุมมอง (แนะนำ):** ถ้าเปิดคุณลักษณะ **[มุมมองที่บันทึกไว้](saved-views.md)** วิธีการที่แนะนำและที่ต้องการคือการสร้างมุมมองที่รวมแอปพื้นที่ทำงานที่ฝังอยู่ แล้วเผยแพร่มุมมองนั้นไปยังผู้ใช้ที่ต้องการ วิธีการนี้ช่วยให้มั่นใจว่าผู้ใช้ทั้งหมดที่มีบทบาทความปลอดภัยที่เป็นเป้าหมายโดยมุมมองที่เผยแพร่ จะเห็นแอปพื้นที่ทำงานบนหน้า

        นอกจากนี้ คุณยังสามารถเผยแพร่แอปพื้นที่ทำงานที่ถูกฝังอยู่เป็นประสบการณ์แบบเต็มหน้าจากแดชบอร์ดได้ บนแดชบอร์ด ให้เลือกไทล์ที่เชื่อมโยงกับแอปค้างไว้ (หรือคลิกขวา) เลือก **กำหนดเป็นแบบส่วนบุคคล** แล้วจากนั้น เลือก **เผยแพร่หน้า** ประสบการณ์ที่จะคล้ายกับประสบการณ์ *มุมมองการเผยแพร่* จะแสดงขึ้น และคุณสามารถเลือกบทบาทความปลอดภัยที่จะเผยแพร่ไป ในการอัปเดต 10.0.21 หรือใหม่กว่า หากเปิดคุณลักษณะ **นิติบุคคลที่ปรับปรุงแล้วของมุมมองที่บันทึกไว้** คุณยังสามารถเผยแพร่แอปไปยังนิติบุคคลที่ต้องการได้

    - ถ้าปิดคุณลักษณะ **มุมมองที่บันทึกไว้** ผู้ดูแลระบบสามารถให้การส่วนบุคคลที่รวมถึงแอปพื้นที่ทำงานไปยังชุดผู้ใช้ที่เหมาะสมผ่านหน้า **การตั้งค่าส่วนบุคคล** อีกทางหนึ่งคือ คุณสามารถส่งออกการตั้งค่าเป็นแบบส่วนบุคคลของหน้าของคุณ และส่งไปยังผู้ใช้อย่างน้อยหนึ่งราย แต่ละผู้ใช้เหล่านั้นสามารถนำเข้าการตั้งค่าเป็นแบบส่วนบุคคลได้ แถบเครื่องมือการตั้งค่าส่วนบุคคลมีปุ่มที่ทำให้คุณสามารถส่งออกและนำเข้าการตั้งค่าส่วนบุคคล

> [!NOTE]
> ถ้าแอปพื้นที่ทำงานถูกแชร์กับผู้ใช้ภายนอก ผู้ใช้เหล่านั้นไม่สามารถใช้แอปที่ฝังไว้ภายในแอปการเงินและการดำเนินงานได้ อย่างไรก็ตาม ผู้ใช้สามารถเข้าถึงแอปได้โดยตรงภายใน Power Apps ผู้ใช้ภายนอกรวมถึงผู้ใช้และผู้ใช้ที่ไม่ได้เป็นสมาชิกของ Microsoft 365 Azure Directory ที่มีการปรับใช้แอปการเงินและการดำเนินงาน

ดู [กำหนดประสบการณ์ผู้ใช้เป็นแบบส่วนบุคคล](personalize-user-experience.md) สำหรับรายละเอียดเพิ่มเติมเกี่ยวกับความสามารถในการกำหนดประสบการณ์ผู้ใช้เป็นแบบส่วนบุคคลในผลิตภัณฑ์และวิธีการใช้

## <a name="building-a-canvas-app-that-uses-data-that-is-sent-from-finance-and-operations-apps"></a>การสร้างแอปพื้นที่ทำงานที่ใช้ข้อมูลที่ส่งจากแอปการเงินและการดำเนินงาน

เมื่อคุณสร้างแอปพื้นที่ทำงานที่จะฝังในแอปการเงินและการดำเนินงาน ส่วนหนึ่งที่สำคัญของกระบวนการจะใช้ข้อมูลป้อนเข้าจากแอปการเงินและการดำเนินงาน จากประสบการณ์การพัฒนา Power Apps ข้อมูลที่ป้อนเข้าที่ส่งผ่านจากแอปการเงินและการดำเนินงานสามารถเข้าถึงได้โดยใช้ตัวแปร **Param("EntityId")** นอกจากนี้ โดยเริ่มต้นในรุ่น 10.0.19 นิติบุคคลปัจจุบันจะถูกส่งผ่านไปยังแอปพื้นที่ทำงานผ่านทางตัวแปร **Param("cmp")** ด้วย 

ตัวอย่างเช่น ในฟังก์ชัน OnStart ของแอป คุณสามารถตั้งค่าข้อมูลป้อนเข้าจากแอปการเงินและการดำเนินงานเป็นตัวแปรดังนี้:

``` Power Apps
If(!IsBlank(Param("EntityId")), Set(FinOpsInput, Param("EntityId")), Set(FinOpsInput, ""));

If(!IsBlank(Param("cmp")), Set(FinOpsLegalEntity, Param("cmp")), Set(FinOpsLegalEntity, ""));
```

## <a name="viewing-a-canvas-app"></a>การดูแอปพื้นที่ทำงาน

ในการดูแอปพื้นที่ทำงานที่ฝังในหน้าในแอปการเงินและการดำเนินงาน เพียงแค่ไปที่หน้าที่มีแอปที่ฝังไว้ โปรดจำไว้ว่าแอปสามารถเข้าถึงได้โดยใช้ปุ่ม **Power Apps** ในบานหน้าต่างการดำเนินการมาตรฐาน อีกทางเลือกหนึ่ง แอปสามารถปรากฎบนหน้าโดยตรงเป็นแท็บใหม่ หรือแท็บด่วน หรือเบลด หรือเป็นส่วนใหม่ในพื้นที่ทำงาน เมื่อผู้ใช้ลองโหลดแอปบนหน้าครั้งแรก จะมีการแจ้งเตือนให้ล็อกอินเข้าสู่ระบบ ขั้นตอนนี้จะช่วยให้มั่นใจว่าผู้ใช้มีสิทธิการได้รับอนุญาตที่เหมาะสมที่จะใช้แอป

## <a name="editing-an-embedded-app"></a>การแก้ไขแอปที่ฝังไว้

หลังจากที่ฝังแอปลงบนหน้า คุณอาจต้องทำการเปลี่ยนแปลงบางอย่างในการกำหนดค่าของแอป ตัวอย่างเช่น คุณอาจต้องการแก้ไขป้ายชื่อที่เชื่อมโยงกับแอปที่ฝังอยู่หรือสร้างแอปรุ่นใหม่ และคุณจำเป็นต้องปรับปรุงรหัสแอปให้บ่งชี้ไปที่การปรับปรุงล่าสุด

ทำตามขั้นตอนเหล่านี้เพื่อแก้ไขการกำหนดค่าของแอปที่ฝังอยู่:

1. ไปที่บานหน้าต่าง **การแก้ไขแอป**

    - หากเข้าถึงแอปที่ฝังไว้ผ่านปุ่มเมนู Power Apps เลือกและกดค้าง (หรือคลิกขวา) ที่ปุ่มเมนู Power Apps และเลือก **ปรับให้เป็นแบบส่วนตัว** เลือกแอปที่คุณต้องการกำหนดค่าจากเมนูแบบหล่นลง **เลือกแอป** 
    - หากแอปที่ฝังไว้ปรากฏบนเพจโดยตรง ให้เลือก **ตัวเลือก** แล้วจึงเลือก **ปรับหน้านี้ให้เป็นแบบส่วนตัว** ใช้เครื่องมือ **เลือก** คลิกแอปที่ฝังไว้
    - ถ้าแอปที่ฝังอยู่ถูกเพิ่มจากแดชบอร์ด ให้เปิดแดชบอร์ด เลือกและระงับ (หรือคลิกขวา) ไทล์ที่สัมพันธ์กับแอปพื้นที่ทำงาน เลือก **ปรับรูปแบบให้เป็นส่วนตัว** แล้วเลือก **แก้ไขหน้า**

2. ทำการปรับเปลี่ยนที่จำเป็นสำหรับการกำหนดค่าแอป แล้วคลิก **บันทึก** 

## <a name="removing-an-app"></a>ลบแอป

หลังจากที่ฝังแอปลงบนหน้า มีไม่กี่วิธีในการลบ หากจำเป็น:

- ไปที่บานหน้าต่าง **แก้ไขแอป** โดยใช้คำแนะนำก่อนหน้าจากส่วน [การแก้ไขแอปที่ฝังไว้](#editing-an-embedded-app) ในหัวข้อนี้ ยืนยันว่าบานหน้าต่างได้แสดงข้อมูลสำหรับแอปที่ฝังไว้ที่คุณต้องการลบ แล้วคลิกปุ่ม **ลบ** 
- ถ้าแอปที่ฝังอยู่ถูกเพิ่มจากแดชบอร์ด ให้เปิดแดชบอร์ด เลือกและระงับ (หรือคลิกขวา) ไทล์ที่สัมพันธ์กับแอปพื้นที่ทำงาน เลือก **ปรับรูปแบบให้เป็นส่วนตัว** แล้วเลือก **เอาหน้าออก** 
- เนื่องจากแอปที่ฝังอยู่นั้นถูกบันทึกเป็นข้อมูลส่วนบุคคล การล้างข้อมูลส่วนบุคคลในหน้าของคุณจะลบแอปที่ฝังไว้ในหน้านั้นด้วย โปรดทราบว่า การล้างการตั้งค่าส่วนบุคคลของเพจเป็นแบบถาวร และไม่สามารถยกเลิกได้ หากต้องการลบการตั้งค่าส่วนบุคคลของคุณในหน้า ให้เลือก **ตัวเลือก** แล้วจึง **ปรับหน้านี้ให้เป็นแบบส่วนตัว** และสุดท้ายคลิกปุ่ม **ล้างข้อมูล** หลังจากการรีเฟรชเบราเซอร์ของคุณ การกำหนดเป็นแบบส่วนบุคคลก่อนหน้านี้ทั้งหมดสำหรับหน้านี้จะถูกเอาออก ดู [กำหนดประสบการณ์ผู้ใช้นี้เป็นแบบส่วนบุคคล](personalize-user-experience.md) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการปรับปรุงหน้าโดยใช้การกำหนดเป็นแบบส่วนบุคคล

## <a name="appendix"></a>ภาคผนวก

### <a name="developer-modeling-a-canvas-app-on-a-form"></a>[นักพัฒนา] การสร้างโมเดล แอปพื้นที่ทำงานบนฟอร์ม

ถึงแม้ว่าหัวข้อนี้จะเน้นที่แอปพื้นที่ทำงานที่ฝังอยู่ผ่านทางการกำหนดเป็นแบบส่วนบุคคล แต่นักพัฒนายังมีตัวเลือกในการเพิ่มแอปพื้นที่ทำงานในฟอร์มโดยใช้ประสบการณ์การพัฒนา Visual Studio ด้วย ถ้าต้องการดำเนินการนี้ เพียงเพิ่ม PowerAppsHostControl ในฟอร์ม คุณสมบัติข้อมูลเมตาที่พร้อมใช้งานบนตัวควบคุม มีความสามารถเหมือนกับประสบการณ์การกำหนดเป็นแบบส่วนบุคคล

### <a name="developer-specifying-where-an-app-can-be-embedded"></a>[นักพัฒนา] กำหนดที่ๆ สามารถฝังแอปไว้ได้

ตามค่าเริ่มต้น ผู้ใช้สามารถฝังแอปในหน้าใดก็ได้ภายใต้ปุ่มเมนู Power Apps หรือฝังโดยตรงบนหน้าในแท็บใหม่ FastTab แผ่นหรือเป็นส่วนใหม่ในพื้นที่ทำงาน อย่างไรก็ตามหากจำเป็น นักพัฒนาสามารถกำหนดค่าคุณลักษณะนี้เพื่ออนุญาตให้ฝังแอปเฉพาะบางหน้าได้โดยใช้วิธีการดังต่อไปนี้:

- **isPowerAppPersonalizationEnabled** – หากวิธีนี้คืนค่าเท็จสำหรับหน้าเฉพาะ ปุ่มเมนู Power Apps จะไม่ปรากฏขึ้น และผู้ใช้จะไม่สามารถฝังแอปที่ใดก็ได้ในหน้านี้รวมถึงที่เป็นแท็บ
- **isPowerAppTabPersonalizationEnabled** – หากวิธีนี้คืนค่าเท็จสำหรับหน้าเฉพาะ ผู้ใช้จะไม่สามารถฝังแอปโดยตรงบนหน้าที่เป็นแท็บ FastTab หรือส่วนรูปแบบพาโนรามา ผู้ใช้จะยังคงสามารถฝังแอปผ่านปุ่มเมนู Power Apps หากการฝังได้รับอนุญาตบนหน้านั้น

ตัวอย่างต่อไปนี้แสดงคลาสใหม่ที่มีสองวิธีที่จำเป็นในการกำหนดค่าที่ที่สามารถฝังแอปได้

```powerapps
[ExtensionOf(classStr(FormRunConfigurationPowerAppsConfiguration))]

public final class ClassTest_Extension
{
    public static boolean isPowerAppPersonalizationEnabled(str pageName)
    {
        var result = next isPowerAppPersonalizationEnabled(pageName);
        return result;
    }
    
    public static boolean isPowerAppTabPersonalizationEnabled(str pageName)
    {
        var result = next isPowerAppTabPersonalizationEnabled(pageName);
        return result;
    }
}
```
