# <a name="no-matching-result-could-be-found"></a>ไม่พบผลลัพธ์ที่ตรงกัน

[!include [banner](../includes/banner.md)]

หัวข้อนี้อธิบายขั้นตอนการแก้ไขปัญหาที่คุณสามารถทำได้หากคุณได้รับข้อผิดพลาด "ไม่พบผลลัพธ์ที่ตรงกัน" ของบริการคํานวณภาษี

## <a name="symptom"></a>อาการ

คุณได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "ส่วนหัว/บรรทัด - 1 กลุ่มภาษี ไม่พบผลลัพธ์ที่ตรงกัน"

```json
======================Tax service calculation result JSON:===========================
{
    "taxDocument": {
        "Header": [
            {
                "Lines": [
                    {
                        ...
                        "Errors": [
                            {
                                "Code": "TaxSetup20000",
                                "Message": "Header/Lines - 1, Tax group applicability, no matching result could be found."
                            }
                        ],
                        "Adjustment": null
                    }
                ],
                "Measures": {
                    ...
                },
                ...
            }
        ]
    },
    ...
}
```

## <a name="cause"></a>สาเหตุ

ปัญหาเกิดขึ้นเมื่อการตั้งค่าคุณลักษณะใน Regulatory Configuration Service (RCS) ไม่ถูกต้อง

## <a name="troubleshoot"></a>แก้ไขปัญหา

1. ดาวน์โหลดไฟล์การแก้ไขปัญหา สำหรับข้อมูลเพิ่มเติม โปรดดู [เปิดใช้งานโหมดดีบักสำหรับการแก้ไขปัญหา](tcs-troubleshooting-enable-debug-mode.md)
2. เปรียบเทียบข้อมูลป้อนเข้าของการคํานวณบริการภาษีกับการตั้งค่าคุณลักษณะเพื่อแก้ปัญหาการตั้งค่า

    ตัวอย่างต่อไปนี้แสดงข้อมูลป้อนเข้าของการคํานวณบริการภาษี

    ```json
    ===============================Tax service calculation input JSON:=====================================
    {
        "TaxableDocument": {
            "Header": [
                {
                    "Lines": [
                        {
                            ...
                        }
                    ],
                    "Business Process": "Sales",
                    "Ship From Zip Code": "30159",
                }
            ]
        },
        "Parameter": {
            ...
        },
        "Adjustment": {
            "Lines": {}
        }
    }
    ```

    ตารางต่อไปนี้แสดงรายการการใช้งานกลุ่มภาษีใน RCS

    | ส่วนหัว.กระบวนการธุรกิจ | บรรทัด.หน่วยธุรกิจ | ส่วนหัว.จัดส่งจากรหัสไปรษณีย์ | กลุ่มภาษี |
    |-------------------------|---------------------|---------------------------|-----------|
    | สมุดรายวัน                 |                     |                           | กลุ่ม A   |
    | การขาย                   |                     | 30160                     | กลุ่ม B   |

    ตามข้อมูลการคำนวณของบริการภาษี ค่า **กระบวนการธุรกิจ** ในส่วนหัวเป็น **การขาย** และค่า **จัดส่งจากรหัสไปรษณีย์** ในส่วนหัวเป็น **30159** ข้อมูลนี้จะขึ้นอยู่กับการตั้งค่ากฎการใช้งานใน RCS เนื่องจากไม่มีรายการที่ตรงกัน จึงเกิดข้อผิดพลาดขึ้น

    > [!NOTE]
    > ถ้าค่าในกฎการใช้งานว่างเปล่า กฎจะสามารถใช้ได้กับทุกค่า

## <a name="mitigation"></a>การลดความเสี่ยง

ทำตามขั้นตอนเหล่านี้เพื่อลดข้อผิดพลาด

1. ใน RCS ให้ไปที่ **คุณลักษณะโลกาภิวัตน์** \> **การคํานวณภาษี**
2. สร้างคุณลักษณะรุ่นใหม่
3. เพิ่มรายการสำหรับข้อมูลที่สอดคล้องกัน

    | ส่วนหัว.กระบวนการธุรกิจ | บรรทัด.หน่วยธุรกิจ | ส่วนหัว.จัดส่งจากรหัสไปรษณีย์| กลุ่มภาษี |
    |-------------------------|---------------------|--------------------------|-----------|
    | สมุดรายวัน                 |                     |                          | กลุ่ม A   |
    | การขาย                   |                     | 30160                    | กลุ่ม B   |
    | การขาย                   |                     | 30159                    | กลุ่ม B   |

4. เผยแพร่รุ่นของการตั้งค่าคุณลักษณะ
5. ใน Microsoft Dynamics 365 Finance ไปที่ **ภาษี** \> **ตั้งค่า** \> **การตั้งค่าคอนฟิกภาษี** \> **พารามิเตอร์การคํานวณภาษี** และเลือกรุ่นใหม่
