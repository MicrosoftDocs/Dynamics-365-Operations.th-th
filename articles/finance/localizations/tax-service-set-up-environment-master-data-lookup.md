# <a name="enable-master-data-lookup-for-tax-calculation-configuration"></a>เปิดใช้งานการค้นหาข้อมูลหลักเกี่ยวกับการตั้งค่าคอนฟิกการคํานวณภาษี 

[!include [banner](../includes/banner.md)]

บทความนี้อธิบายวิธีการตั้งค่าและเปิดใช้งานฟังก์ชันการค้นหาข้อมูลหลักของการคํานวณภาษี รายการแบบหล่นลงจะพร้อมใช้งานเพื่อเลือกค่าในการตั้งค่าคอนฟิกการคํานวณภาษี สำหรับฟิลด์ เช่น **นิติบุคคล**, **บัญชีผู้จัดจำหน่าย**, **รหัสสินค้า** และ **เงื่อนไขการจัดส่ง** ค่าเหล่านี้มาจากสภาพแวดล้อมของ Microsoft Dynamics 365 Finance ที่เชื่อมต่อโดยใช้แหล่งข้อมูล Microsoft Dataverse

> [!NOTE] 
> ฟังก์ชันการค้นหาข้อมูลหลักการคํานวณภาษีเป็นฟังก์ชันตัวเลือก คุณสามารถข้ามขั้นตอนต่อไปนี้ได้ถ้าคุณปิดใช้งานคุณลักษณะ **การสนับสนุนแหล่งข้อมูล Tax Service Dataverse** ใน Regulatory Configuration Service (RCS) อย่างไรก็ตาม รายการแบบหล่นลงจะไม่สามารถใช้งานในการตั้งค่าคอนฟิกการคํานวณภาษีในกรณีนั้น

หากต้องการเปิดใช้งานรายการแบบดรอปดาวน์ในการตั้งค่าคอนฟิกรุ่นของลักษณะการคํานวณภาษี ให้ปฏิบัติตามขั้นตอนต่อไปนี้

1. [เปิดใช้งาน Microsoft Power Platform การรวมและเปิด Dataverse สภาพแวดล้อม](#enable)
2. [ติดตั้งเอนทิตีเสมือนของการเงินและการดำเนินงาน](#install)
3. [ลงทะเบียนแอปพลิเคชัน Azure Active Directory (Azure AD)](#register)
4. [ให้สิทธิ์แอปในแอปการเงินและการดําเนินงาน](#grant)
5. [ตั้งค่าคอนฟิกแหล่งข้อมูลสำหรับเอนทิตีเสมือน](#configure)
6. [เปิดใช้งานเอนทิตีเสมือน Dataverse](#virtual)
7. [ตั้งค่าแอปลิเคชันที่เชื่อมต่อเพื่อคํานวณภาษี](#set-up)
8. [นำเข้าไฟล์และกำหนดค่าการแม็ปแบบจำลอง Dataverse](#import)

## <a name="enable-microsoft-power-platform-integration-and-open-the-dataverse-environment"></a><a name='enable'></a>เปิดใช้งานการรวม Microsoft Power Platform และเปิดสภาพแวดล้อม Dataverse

คุณสามารถเปิดใช้งานการรวมแอปการเงินและการดําเนินงานด้วย Microsoft Power Platform ได้ เมื่อคุณสร้างสภาพแวดล้อมแอปการเงินและการดําเนินงานใหม่ใน Microsoft Dynamics Lifecycle Services (LCS) สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [การรวม Microsoft Power Platform - ภาพรวม Add-in](../../fin-ops-core/dev-itpro/power-platform/add-ins-overview.md) หลังจากการดำเนินการขั้นตอนนี้เสร็จสมบูรณ์แล้ว ชื่อของสภาพแวดล้อม Microsoft Power Platform จะปรากฏในส่วน **การรวม Power Platform**

1. ใน LCS ในสภาพแวดล้อมด้านการเงินและการดําเนินงานของคุณ ในส่วน **Power Platform ในส่วน** ให้ค้นหาและจดบันทึกค่า **ชื่อสภาพแวดล้อม** จากสภาพแวดล้อมที่เชื่อมโยง

    [![ค่าชื่อสภาพแวดล้อม](./media/tcs-dataverse-master-data-lookup-1.png)](./media/tcs-dataverse-master-data-lookup-1.png)

2. ใน [ศูนย์ผู้ดูแลระบบ Power Platform](https://admin.powerplatform.microsoft.com/environments) บนแท็บ **สภาพแวดล้อม** ให้เลือกสภาพแวดล้อมที่ตรงกับค่า **ชื่อสภาพแวดล้อม** ที่คุณสร้างขึ้น
3. ในหน้า **รายละเอียด** ให้ค้นหาค่า **URL สภาพแวดล้อม** ของสภาพแวดล้อม Dataverse บันทึกค่านี้เนื่องจากคุณจะต้องใช้ค่าใน [ขั้นตอนที่ 7 ตั้งค่าแอปลิเคชันที่เชื่อมต่อเพื่อคํานวณภาษี](#set-up)
4. ตรวจสอบให้แน่ใจว่าคุณสามารถเปิดสภาพแวดล้อม Dataverse ในเบราเซอร์ของคุณโดยเลือกค่า **URL สภาพแวดล้อม**

    [![หน้าสภาพแวดล้อม Dataverse](./media/tcs-dataverse-master-data-lookup-2.png)](./media/tcs-dataverse-master-data-lookup-2.png)

    > [!NOTE]
    > เปิดสภาพแวดล้อม Keep the Dataverse ไว้ในเบราว์เซอร์ของคุณ คุณจะต้องใช้อีกใน [ขั้นตอนที่ 5 ตั้งค่าคอนฟิกแหล่งข้อมูลของเอนทิตีเสมือนจริง](#configure)

หากต้องการข้อมูลเพิ่มเติม โปรดดูที่ [การเปิดใช้งานการรวม Microsoft Power Platform](../../fin-ops-core/dev-itpro/power-platform/enable-power-platform-integration.md)

## <a name="install-finance-and-operations-virtual-entities"></a><a name='install'></a>ติดตั้งเอนทิตีเสมือนจริงของการเงินและการดำเนินงาน

ต้องติดตั้งโซลูชัน Dataverse ของเอนทิตีเสมือนจริงของการเงินและการดําเนินงานจากโซลูชันเอนทิตีเสมือนจริงของ Microsoft AppSource

1. ใน AppSource ให้ค้นหา [เอนทิตีเสมือนจริงของการเงินและการดำเนินงาน](https://appsource.microsoft.com/product/dynamics-365/mscrm.finance_and_operations_virtual_entity)
2. ให้เลือก **เริ่มต้นใช้งาน**
3. ในฟิลด์ **เลือกสภาพแวดล้อม** ให้ป้อนค่า **ชื่อสภาพแวดล้อม** ที่คุณสร้างขึ้นในบันทึกก่อนหน้านี้
4. เลือกกล่องกาเครื่องหมาย แล้วเลือก **ติดตั้ง**

เมื่อการติดตั้งเสร็จสิ้น คุณสามารถค้นหา **แอปเอนทิตีเสมือนจริงของการเงินและการดําเนินงาน** ใน [ศูนย์ผู้ดูแบระบบ Power Platform](https://admin.powerplatform.microsoft.com/) ภายใต้ **ทรัพยากร**\>**แอป Dynamics 365**

หากต้องการข้อมูลเพิ่มเติม ให้ดู [เปิดใช้งานโซลูชันเอนทิตีเสมือนจริง](../../fin-ops-core/dev-itpro/power-platform/admin-reference.md#get-virtual-entity-solution)

## <a name="register-an-azure-ad-application"></a><a name='register'></a>ลงทะเบียนแอปพลิเคชัน Azure AD

คุณต้องลงทะเบียนแอปลิเคชัน Azure AD ในผู้เช่าเดียวกับแอปการเงินและการดําเนินงาน เพื่อให้ Dataverse สามารถเรียกใช้ได้

1. [พอร์ทัล Azure](https://portal.azure.com) ให้ไปที่ **Azure Active Directory \> การลงทะเบียนแอป**
2. ให้เลือก **ลงทะเบียนใหม่** และป้อนข้อมูลต่อไปนี้:

    - คลิกที่ **ชื่อ** ให้ป้อนชื่อเฉพาะ
    - คลิกที่ **ชนิดบัญชี** – ป้อน **ไดเรกทอรี Azure AD ใดๆ** (ผู้เช่าคนเดียวหรือผู้เช่าหลายราย)
    - **เปลี่ยนเส้นทาง URI** ให้ปล่อยฟิลด์นี้ให้ว่างไว้

3. เลือก **การลงทะเบียน**
4. จัดทำบันทึกของค่า **รหัสแอปพลิเคชัน (ไคลเอนต์)** เนื่องจากคุณจะต้องใช้ในภายหลัง

    [![Azure AD ค่ารหัสแอปพลิเคชัน (ไคลเอนต์)](./media/tcs-dataverse-master-data-lookup-3.png)](./media/tcs-dataverse-master-data-lookup-3.png)

5. สร้างคีย์สมมาตรสำหรับแอปลิเคชัน
6. ในแอปลิเคชันใหม่ ให้เลือก **ประกาศนียบัตรและความลับ**
7. เลือก **ข้อมูลลับของไคลเอนต์ใหม่**
8. ป้อนข้อความอธิบาย เลือกวันหมดอายุ แล้วเลือก **บันทึก** จะมีการสร้างและแสดงคีย์ คัดลอกค่าที่จะใช้ในภายหลัง

หากต้องการข้อมูลเพิ่มเติม โปรดดู [ลงทะเบียนแอปพลิเคชัน Azure AD](../../fin-ops-core/dev-itpro/power-platform/admin-reference.md#register-the-app-in-the-azure-portal)

## <a name="grant-app-permissions-in-finance-and-operations-apps"></a><a name='grant'></a>ให้สิทธิ์แอปในแอปการเงินและการดําเนินงาน

Dataverse จะใช้แอปลิเคชัน Azure AD ที่คุณสร้างขึ้นเพื่อเรียกใช้แอปการเงินและการดําเนินงาน ดังนั้นแอปลิเคชันต้องได้รับความเชื่อถือจากแอปการเงินและการดําเนินงานและเชื่อมโยงกับบัญชีผู้ใช้ที่มีสิทธิ์ที่เหมาะสม ในแอปการเงินและการดําเนินงาน คุณต้องสร้างผู้ใช้บริการพิเศษที่มีสิทธิ์เฉพาะกับฟังก์ชันเอนทิตีเสมือนจริง **เท่านั้น** ผู้ใช้การบริการนี้จะต้องไม่มีสิทธิ์อื่น เมื่อคุณดําเนินการขั้นตอนนี้เสร็จสิ้นแล้ว แอปลิเคชันใดๆ ที่มีความลับของแอปพลิเคชัน Azure AD ที่คุณสร้างขึ้นจะสามารถเรียกสภาพแวดล้อมของแอปการเงินและการดําเนินงาน และเข้าถึงฟังก์ชันเอนทิตีเสมือนจริงได้

1. ในสภาพแวดล้อมของคุณ ให้ไปที่ **การจัดการระบบ** \> **ผู้ใช้** \> **ผู้ใช้**
2. ให้เลือก **สร้าง** เพื่อเพิ่มผู้ใช้ใหม่ และป้อนข้อมูลต่อไปนี้:

    - **รหัสผู้ใช้** – ให้ป้อน **dataverseintegration** หรือค่าอื่น
    - **ชื่อผู้ใช้** – ให้ป้อน **dataverse integration** หรือค่าอื่น
    - **ผู้ให้บริการ** – ตั้งค่าฟิลด์นี้เป็น **NonAAD**
    - **อีเมล** – ให้ป้อน **dataverseintegration** หรือค่าอื่น (ค่านี้ไม่ต้องเป็นบัญชีอีเมลที่ถูกต้อง)

3. กำหนดบทบาทความปลอดภัย **แอปการรวมเอนทิตี้เสมือนของ Dataverse** ให้ผู้ใช้
4. ลบบทบาทอื่นทั้งหมด รวมถึง **ผู้ใช้ของระบบ**
5. ให้ไปที่ **การจัดการระบบ** \> **การตั้งค่า** \> **แอปพลิเคชัน Azure Active Directory** เพื่อลงทะเบียนใช้งาน Dataverse 
6. เพิ่มแถว แล้วในฟิลด์ **รหัสไคลเอนต์** ให้ป้อนค่า **รหัสแอปลิเคชัน (ไคลเอนต์)** ที่คุณบันทึกไว้ก่อนหน้านี้
7. ในฟิลด์ **ชื่อ** ป้อนชื่อ ตัวอย่างเช่น ป้อน **Dataverse Integration**
8. ในฟิลด์ **รหัสผู้ใช้** ให้เลือกรหัสผู้ใช้ที่คุณสร้างขึ้นก่อนหน้า

หากต้องการข้อมูลเพิ่มเติม โปรดดูที่ [การให้สิทธิ์แอปพลิเคชันในแอปการเงินและการดำเนินงาน](../../fin-ops-core/dev-itpro/power-platform/admin-reference.md#grant-app-permissions-in-finance-and-operations-apps)

## <a name="configure-the-virtual-entity-data-source"></a><a name='configure'></a>ตั้งค่าคอนฟิกแหล่งข้อมูลสำหรับเอนทิตีเสมือน

คุณต้องระบุอินสแตนซ์แอปการเงินและการดําเนินงานให้ Dataverse เพื่อเชื่อมต่อกัน

1. สภาพแวดล้อม Dataverse ของคุณจะยังคงเปิดอยู่ในเบราเซอร์ของคุณจาก [ขั้นตอนที่ 1 เปิดใช้งานการรวม Microsoft Power Platform และเปิดสภาพแวดล้อม Dataverse](#enable) เลือกปุ่ม การตั้งค่า (สัญลักษณ์รูปเฟือง) ที่มุมขวาบน จากนั้น เลือก **การตั้งค่าขั้นสูง**

    [![การเปิดการตั้งค่าขั้นสูงในสภาพแวดล้อม Dataverse](./media/tcs-dataverse-master-data-lookup-4.png)](./media/tcs-dataverse-master-data-lookup-4.png)

2. บนเมนูแบบดรอปดาวน์ **Settings** ให้เลือก **การจัดการ**

    [![การจัดการ](./media/tcs-dataverse-master-data-lookup-5.png)](./media/tcs-dataverse-master-data-lookup-5.png)

3. เลือก **แหล่งข้อมูลของเอนทิตีเสมือนจริง**

    [![แหล่งข้อมูลของเอนทิตีเสมือนจริง](./media/tcs-dataverse-master-data-lookup-6.png)](./media/tcs-dataverse-master-data-lookup-6.png)

4. ให้เลือกแหล่งข้อมูลที่มีชื่อว่า **การเงินและการดำเนินงาน**

    [![แหล่งข้อมูลสำหรับ การเงินและการดำเนินงาน](./media/tcs-dataverse-master-data-lookup-7.png)](./media/tcs-dataverse-master-data-lookup-7.png)

5. ป้อนข้อมูลต่อไปนี้จากขั้นตอนก่อนหน้า

    - **URL เป้าหมาย** – ป้อน URL ที่คุณสามารถเข้าถึงแอปการเงินและการดําเนินงานได้
    - **OAuth URL** – ป้อน `https://login.windows.net/`
    - **รหัสผู้เช่า** – ระบุรหัสผู้เช่าของคุณ ค่านี้สามารถเป็นชื่อโดเมนของอีเมลบริษัทของคุณ (เช่น contoso.com)
    - **AAD Application ID** – ป้อนค่า **รหัสของแอแพลิเคชัน (ไคลเอนต์)** ที่คุณสร้างไว้ก่อนหน้า
    - **AAD Application Secret** – ป้อนข้อมูลความลับที่สร้างไว้ก่อนหน้า
    - **ทรัพยากร AAD** – ป้อน **00000015-0000-0000-c000-000000000000** ค่านี้เป็นแอปลิเคชัน Azure AD ที่แสดงถึงแอปการเงินและการดําเนินงาน ซึ่งควรจะเป็นค่าเดียวกันนี้เสมอ

6. บันทึกการเปลี่ยนแปลง
7. ปิดหน้าเพื่อกลับไปที่หน้า **การจัดการ** ที่คุณจะเริ่มต้น [ขั้นตอนที่ 6 เปิดใช้งานเอนทิตีเสมือนจริงของ Dataverse](#virtual)

    [![การปิดเอนทิตีเพื่อแก้ไข](./media/tcs-dataverse-master-data-lookup-8.png)](./media/tcs-dataverse-master-data-lookup-8.png)

หากต้องการข้อมูลเพิ่มเติม ให้ดู [การตั้งค่าคอนฟืกแหล่งข้อมูลของเอนทิตีเสมือนจริง](../../fin-ops-core/dev-itpro/power-platform/admin-reference.md#configure-the-virtual-entity-data-source)

## <a name="enable-dataverse-virtual-entities"></a><a name='virtual'></a> เปิดใช้งานเอนทิตีเสมือนจริง Dataverse

การมองเห็นได้ของเอนทิตีเสมือนจริงจากแอปการเงินและการดําเนินงานต้องมีการตั้งค่าเป็น **ใช่** ก่อนจึงจะสามารถใช้เอนทิตีโดยการตั้งค่าคอนฟิกการคํานวณภาษีได้

> [!NOTE]
> คุณสามารถข้ามขั้นตอนนี้ได้โดยเปิดใช้งานเอนทิตีเสมือนจริงที่เกี่ยวข้องกับการคํานวณภาษีในคลิกเดียวใน [ขั้นตอนที่ 8 ตั้งค่าแอปลิเคชันที่เชื่อมต่อเพื่อคํานวณภาษี](#import) อย่างไรก็ตาม ขอแนะนำว่าคุณควรเปิดใช้งานเอนทิตีเสมือนจริงอย่างน้อยหนึ่งเอนทิตีด้วยตนเอง เพื่อบ่งชี้ว่าการเชื่อมต่อระหว่างแอปการเงินและการดําเนินงานกับ Dataverse ได้รับการยอมรับ

1. ในสภาพแวดล้อม Dataverse ของคุณ บนหน้า **การจัดการ** ให้เลือกปุ่มตัวกรองข้อมูล (สัญลักษณ์ช่วงระยะ) ในมุมขวาบน

    [![ปุ่มตัวกรองข้อมูล](./media/tcs-dataverse-master-data-lookup-9.png)](./media/tcs-dataverse-master-data-lookup-9.png)

2. ในหน้าต่าง **การค้นหาขั้นสูง** ในฟิลด์ **ค้นหา** ให้เลือก **เอนทิตีการเงินและการดําเนินงานที่มีอยู่**
3. ให้เพิ่มกฎต่อไปนี้: **ชื่อ** – **ประกอบด้วย** – **CompanyInfoEntity** จากนั้นเลือก **ผลลัพธ์**

    [![หน้าต่างการค้นหาขั้นสูง](./media/tcs-dataverse-master-data-lookup-10.png)](./media/tcs-dataverse-master-data-lookup-10.png)

4. ให้เลือก **CompanyInfoEntity** ในผลการค้นหา เลือกกล่องกาเครื่องหมาย **ที่มองเห็นได้** แล้วเลือก **บันทึก**

    [![การตั้งค่าความสามารถในการมองเห็นเอนทิตี](./media/tcs-dataverse-master-data-lookup-11.png)](./media/tcs-dataverse-master-data-lookup-11.png)

5. ทําซ้ําขั้นตอนก่อนหน้านี้ให้กับเอนทิตีต่อไปนี้ที่อ้างถึงในการตั้งค่าคอนฟิกการคํานวณภาษี:

    - CompanyInfoEntity
    - CurrencyEntity
    - CustCustomerV3Entity
    - DeliveryTermsEntity
    - EcoResProductCategoryEntity
    - EcoResReleasedProductV2Entity
    - LogisticsAddressCountryRegionTranslationEntity
    - LogisticsAddressStateEntity
    - PurchProcurementChargeCDSEntity
    - SalesChargeCDSEntity
    - TaxGroupEntity
    - TaxItemGroupHeadingEntity
    - VendVendorV2Entity
    - InventOperationalSiteV2Entity
    - TaxExemptCodeEntity
    - InventWarehouseEntity

    > [!NOTE]
    > เฉพาะเรกคอร์ด 5,000 เรกคอร์ดแรกของเอนทิตีเท่านั้นที่สามารถดึงข้อมูลโดย Dataverse และกําหนดให้พร้อมใช้งานในรายการแบบดรอปดาวน์ของการตั้งค่าคอนฟิกการคํานวณภาษี

สำหรับข้อมูลเพิ่มเติม ให้ดู [เปิดใช้งานเอนทิตีเสมือน Microsoft Dataverse](../../fin-ops-core/dev-itpro/power-platform/enable-virtual-entities.md)

## <a name="set-up-the-connected-application-for-tax-calculation"></a><a name='set-up'></a>ตั้งค่าแอปลิเคชันที่เชื่อมต่อเพื่อคํานวณภาษี

1. ให้ไปที่ **การรายงานทางอิเล็กทรอนิกส์** จากนั้น ในส่วน **ลิงก์ที่เกี่ยวข้อง** ให้เลือก **แอปพลิเคชันที่เชื่อมต่อ**

    [![แอปพลิเคชันที่เชื่อมต่อ](./media/tcs-dataverse-master-data-lookup-12.png)](./media/tcs-dataverse-master-data-lookup-12.png)

2. ให้เลือก **สร้าง** เพื่อเพิ่มเรกคอร์ด และป้อนข้อมูลฟิลด์ต่อไปนี้

    - **ชื่อ** – ให้ป้อนชื่อ
    - คลิก **ประเภท** – เลือก **Dataverse**
    - ที่ **แอปลิเคชัน** – ป้อนค่า Dataverse **URL สภาพแวดล้อม** ของสภาพแวดล้อมของคุณ ที่ได้บันทึกไว้ใน [ขั้นตอนที่ 1 เปิดใช้งานการรวม Microsoft Power Platform และเปิดสภาพแวดล้อม Dataverse ของคุณ](#enable)
    - **ผู้เช่า** – ป้อนชื่อผู้เช่าของคุณ
    - ที่ **URL แบบกำหนดเอง** – ให้ป้อน URL ของ Dataverse และการผนวก **/api/data/v9.1** เข้าไปด้วย

3. ให้เลือก **ตรวจสอบการเชื่อมต่อ** จากนั้น ในกล่องโต้ตอบ ให้เลือก **คลิกที่นี่เพื่อเชื่อมต่อกับแอปลิเคชันระยะไกลที่เลือก**

    [![การตรวจสอบการเชื่อมต่อ](./media/tcs-dataverse-master-data-lookup-13.png)](./media/tcs-dataverse-master-data-lookup-13.png)
4. ตรวจสอบให้แน่ใจว่าคุณได้รับข้อความ "สาเร็จ" เป็นข้อความที่บ่งชี้ว่าการเชื่อมต่อสำเร็จสมบูรณ์แล้ว

    [![ข้อความบอกความสำเร็จ](./media/tcs-dataverse-master-data-lookup-14.png)](./media/tcs-dataverse-master-data-lookup-14.png)
    
5. ใน RCS เปิดพื้นที่ทำงาน **การจัดการคุณลักษณะ** และเปิดใช้งานคุณลักษณะต่อไปนี้

    - คุณลักษณะที่ใช้ทั่วโลก
    - การสนับสนุนแหล่งข้อมูล Dataverse การรายงานทางอิเล็กทรอนิกส์
    - การสนับสนุนแหล่งข้อมูล Dataverse สำหรับบริการด้านภาษี

## <a name="import-and-set-up-the-dataverse-model-mapping-configuration"></a><a name='import'></a>นำเข้าไฟล์และกำหนดค่าคอนฟิกการแม็ปแบบจำลอง Dataverse

Microsoft มีการตั้งค่าคอนฟิกการแม็ปแบบจำลองเริ่มต้นให้กับเอนทิตีจากแอปการเงินและการดําเนินงานไปยังการคํานวณภาษี

1. ใน RCS ให้ไปที่ **การรายงานทางอิเล็กทรอนิกส์**
2. ในส่วน **ผู้ให้บริการการกำหนดค่าคอนฟิก** บนไทล์สำหรับผู้ให้บริการ **Microsoft** ให้เลือก **ที่เก็บ**

    [![ที่เก็บ](./media/tcs-dataverse-master-data-lookup-15.png)](./media/tcs-dataverse-master-data-lookup-15.png)

3. ให้เลือกเรกคอร์ด **เก็บการตั้งค่าคอนฟิกส่วนกลาง** จากนั้นเลือก **เปิด**
4. ภายใต้ **แบบจำลองข้อมูลภาษี**\>**แบบจำลองการคํานวณภาษี** ให้เลือกการตั้งค่าคอนฟิก **การแม็ปแบบโมเดล Dataverse**
5. บนแท็บด่วน **รุ่น** ให้เลือกรุ่นที่ตรงกับเวอร์ชันแอปการเงินและการดําเนินงานของคุณ แล้วเลือก **นําเข้า**

    [![นำเข้าการกำหนดค่าคอนฟิกการแม็ปแบบจำลอง Dataverse](./media/tcs-dataverse-master-data-lookup-16.png)](./media/tcs-dataverse-master-data-lookup-16.png)

    > [!IMPORTANT]
    > การตั้งค่าคอนฟิก **การแม็ปแบบจำลอง Dataverse** จะมีผลบังคับใช้เฉพาะบนรุ่นที่นําเข้าสูงสุดเท่านั้น ดังนั้น คุณไม่ควรนําเข้ารุ่นของการตั้งค่าคอนฟิก **การแม็ปแบบจำลอง Dataverse** ที่สูงกว่ารุ่นของการตั้งค่าคอนฟิกการคํานวณภาษีที่คุณวางแผนจะใช้ ตัวอย่างเช่น หากคุณวางแผนที่จะใช้รุ่น 40.50.225 ของการกำหนดค่าการคำนวณภาษี คุณควรนำเข้าเฉพาะรุ่น 40.50.13 ของการกำหนดค่าคอนฟิก **การแม็บแบบจำลอง Dataverse** คุณไม่ควรนําเข้ารุ่น 40.54.14 มิฉะนั้นจะมีแบบจำลองข้อมูลที่ไม่ตรงกันในการตั้งค่าคอนฟิก

6. ให้กลับไปที่ **การรายงานทางอิเล็กทรอนิกส์** และเลือกไทล์ **การตั้งค่าคอนฟิกภาษี**
7. ให้เลือกนำเข้าการตั้งค่า **การแม็บแบบจำลอง Dataverse** แล้วเลือก **แก้ไข**
8. ตั้งค่าตัวเลือก **ค่าเริ่มต้นสำหรับการแมปแบบจำลอง** เป็น **ใช่**
9. ในฟิลด์ **แอปลิเคชันที่เชื่อมต่อ** ให้เลือกแอปลิเคชันที่เชื่อมต่อซึ่งคุณตั้งค่าไว้ใน [ขั้นตอนที่ 7 ตั้งค่าแอปลิเคชันที่เชื่อมต่อเพื่อการคํานวณภาษี](#set-up)
10. ให้ตั้งค่าตัวเลือก **ตั้งค่าการมองเห็นได้ของตารางเสมือนจริง** เป็น **ใช่** เพื่อตั้งค่าเอนทิตีเสมือนจริงที่เกี่ยวข้องกับการคํานวณภาษีทั้งหมดเพื่อให้มองเห็นได้

ขณะนี้คุณได้ตั้งค่าฟังก์ชันการค้นหาข้อมูลหลักเสร็จสมบูรณ์แล้ว รายการแบบดรอปดาวน์ของฟิลด์ต่างๆ เช่น **นิติบุคคล** **รายชื่อผู้จัดจำหน่าย** **รหัสสินค้า** และ **เงื่อนไขการจัดส่ง** จาก Dynamics 365 Finance จะเปิดใช้งานในการตั้งค่า **รุ่นคุณลักษณะการคํานวณภาษี**

[![รายการแบบดรอปดาวน์ของนิติบุคคล](./media/tcs-dataverse-master-data-lookup-17.png)](./media/tcs-dataverse-master-data-lookup-17.png)
