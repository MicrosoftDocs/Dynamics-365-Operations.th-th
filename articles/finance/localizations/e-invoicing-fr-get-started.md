# <a name="get-started-with-the-electronic-invoicing-add-on-for-france"></a>เริ่มต้นใช้งาน Add-On การออกใบแจ้งหนี้อิเล็กทรอนิกส์สำหรับฝรั่งเศส

[!include [banner](../includes/banner.md)]

[!include [banner](../includes/preview-banner.md)]

บทความนี้ให้ข้อมูลซึ่งจะช่วยคุณในการเริ่มต้นใช้งานการออกใบแจ้งหนี้อิเล็กทรอนิกส์สำหรับฝรั่งเศส โดยจะแนะนำให้คุณทราบถึงขั้นตอนการตั้งค่าคอนฟิกที่ขึ้นอยู่กับประเทศใน Regulatory Configuration Service (RCS) ขั้นตอนเหล่านี้เพิ่มเติมเกี่ยวกับขั้นตอนที่อธิบายไว้ใน [เริ่มต้นใช้งานด้วย add-on การออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์](e-invoicing-get-started.md)

## <a name="country-specific-configuration-for-french-chorus-pro-submission-fr-electronic-invoicing-feature"></a>การกำหนดค่าคอนฟิกเฉพาะประเทศสำหรับคุณลักษณะการออกใบแจ้งหนี้อิเล็กทรอนิกส์ของการส่ง Chorus Pro ของฝรั่งเศส (FR)

บางขั้นตอนต้องระบุในการตั้งค่าคอนฟิกคุณลักษณะการออกใบแจ้งหนี้อิเล็กทรอนิกส์ **การส่ง Chorus Pro ของฝรั่งเศส (FR)** บางพารามิเตอร์จากการตั้งค่าคอนฟิกจะเผยแพร่ด้วยค่าเริ่มต้น ต้องตรวจทานและอัปเดตค่าเหล่านี้เพื่อให้สะท้อนถึงการดําเนินงานทางธุรกิจของคุณได้ดียิ่งขึ้น

### <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ก่อนที่คุณจะเริ่มกระบวนงานต่างๆ ในบทความนี้ ต้องทำตามข้อกำหนดเบื้องต้นต่อไปนี้ให้เสร็จสมบูรณ์ก่อน:

- สร้างความคุ้นเคยกับการออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์ สำหรับข้อมูลเพิ่มเติม โปรดดู [ภาพรวมการออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์](e-invoicing-service-overview.md)
- ลงทะเบียนสำหรับ RCS และตั้งค่าการออกใบแจ้งหนี้อิเล็กทรอนิกส์ สำหรับข้อมูลเพิ่มเติม ให้ดูบทความต่อไปนี้:

    - [ลงทะเบียนและติดตั้งบริการออกใบแจ้งหนี้อิเล็กทรอนิกส์](e-invoicing-sign-up-install.md)
    - [ตั้งค่าทรัพยากร Azure สำหรับการออกใบแจ้งหนี้อิเล็กทรอนิกส์](e-invoicing-set-up-azure-resources.md)
    - [ติดตั้ง add-in สำหรับไมโครเซอร์วิสใน Lifecycle Services](e-invoicing-install-add-in-microservices-lcs.md)
    - [เปิดใช้งานและตั้งค่าการรวมกับการออกใบแจ้งหนี้อิเล็กทรอนิกส์](e-invoicing-activate-setup-integration.md) – ใช้ข้อมูลในบทความนี้เพื่อเปิดใช้งาน การรวมกับ Microsoft Dynamics 365 Finance หรือแอป Dynamics 365 Supply Chain Management ของคุณและบริการออกใบแจ้งหนี้อิเล็กทรอนิกส์
    - [รหัส NAF และหมายเลข siret](emea-fra-naf-codes-siret-numbers.md) และ [ตั้งค่ารหัส NAF และหมายเลข Siret](tasks/fr-00003-naf-codes-siret-numbers.md) – ใช้ข้อมูลในบทความเหล่านี้เพื่อตั้งค่ารหัส NAF และหมายเลข Siret ในนิติบุคคลของคุณ 

- คุณต้องลงทะเบียนองค์กรของคุณเพื่อดําเนินงานกับ Chos Pro Microsoft ให้การรวมกับ Javaus pro ในโหมด OAuth2 ผ่านแอปพลิเคชันโปรแกรมเมอร์อินเทอร์เฟส (API) หากต้องการทราบข้อมูลอย่างละเอียดเกี่ยวกับการลงทะเบียนและการเรียกใช้แอปพลิเคชัน Chorus Pro นั้น โปรดดูที่ [เอกสารที่เป็นทางการ](https://communaute.chorus-pro.gouv.fr/documentation/help-for-api-developers-in-oauth2-mode/)

    ให้ทำตามขั้นตอนเหล่านี้เพื่อลงทะเบียนกับ Chorus Pro

    1. ไปที่ [พอร์ทัล PISTE](https://piste.gouv.fr/en/component/apiportal/registration) เพื่อเริ่มต้นการลงทะเบียนของคุณ 
    2. ลงทะเบียนแอปพลิเคชัน และเรียกใช้ข้อมูลข้อมูลส่วนบุคคล Open Authorization (OAuth)
    3. คัดลอกและบันทึกรหัสไคลเอ็นต์ของข้อมูลส่วนบุคคล OAuth และคีย์ลับ คุณจะใช้ข้อมูลนี้ในภายหลัง
    4. ไปที่ [พอร์ทัล Chorus Pro](https://portail.chorus-pro.gouv.fr/aife_csm/?id=aife_enrollment) เพื่อลงทะเบียน 
    5. สร้างบัญชีทางเทคนิคเพื่อเข้าถึง API สำหรับข้อมูลเพิ่มเติม ดูที่ [การสร้างบัญชีทางเทคนิคของการเข้าถึง API ในการผลิต](https://communaute.chorus-pro.gouv.fr/documentation/creation-of-a-technical-account-for-an-api-access-in-production/)
    6. คัดลอกรหัสผู้ใช้ของบัญชีทางเทคนิคและรหัสผ่าน คุณจะใช้ข้อมูลนี้ในภายหลัง

## <a name="country-specific-configuration-of-the-application-setup-for-the-french-chorus-pro-submission-fr-electronic-invoicing-feature"></a>การกำหนดค่าการตั้งค่าแอพพลิเคชันเฉพาะประเทศสำหรับคุณลักษณะการออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์ของการส่ง Chorus Pro ของฝรั่งเศส (FR)

พารามิเตอร์บางรายการจากคุณลักษณะการออกใบแจ้งหนี้อิเล็กทรอนิกส์ของ **การส่ง Chorus Pro ของฝรั่งเศส (FR)** มีการเผยแพร่ด้วยค่าเริ่มต้น ก่อนที่คุณจะปรับใช้คุณลักษณะการออกใบแจ้งหนี้อิเล็กทรอนิกส์กับสภาพแวดล้อมการบริการ ให้ตรวจทานค่าเริ่มต้น และอัปเดตตามความจำเป็นเพื่อให้สะท้อนถึงการดำเนินงานทางธุรกิจของคุณได้ดียิ่งขึ้น

1. ใน Azure Key Vault สร้างข้อมูลลับและการอ้างอิงที่เกี่ยวข้อง สำหรับข้อมูลเพิ่มเติม ดูที่ [ใบรับรองและข้อมูลลับของลูกค้า](e-invoicing-customer-certificates-secrets.md)
2. นำเข้ารุ่นล่าสุดของคุณลักษณะมาตรฐานโลกของ **การส่ง Chorus Pro ของฝรั่งเศส (FR)** ตามที่อธิบายไว้ใน [นำเข้าคุณลักษณะจากที่เก็บส่วนกลาง](e-invoicing-import-feature-global-repository.md)
3. สร้างสําเนาของคุณลักษณะสากลที่นําเข้า และเลือกตัวให้บริการการตั้งค่าคอนฟิกของคุณ สำหรับข้อมูลเพิ่มเติม ดูที่ [สร้างคุณลักษณะมาตรฐานโลก](e-invoicing-create-new-globalization-feature.md)
4. บนแท็บ **รุ่น** ตรวจสอบว่ารุ่น **ดราฟท์** ได้ถูกเลือก
5. บนแท็บ **การตั้งค่า** เลือกการตั้งค่าคุณลักษณะ **ใบแจ้งหนี้การขาย UBL ที่ได้รับมา** ในกริด
6. เลือก **แก้ไข** แล้วบนแท็บ **ไปป์ไลน์การประมวลผล** ในส่วน **ไปป์ไลน์การประมวลผล** เลือก **(แสดงตัวอย่าง) รวมกับ Chorus Pro ของฝรั่งเศส** พร้อมชื่อการดำเนินการ **ส่ง Chorus Pro ของฝรั่งเศส**
7. ในส่วน **พารามิเตอร์** ในฟิลด์ **ชื่อข้อมูลลับของรหัสไคลเอ็นต์** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อรหัสไคลเอ็นต์ใน key vault นี้
8. ในฟิลด์ **ชื่อข้อมูลลับของข้อมูลลับไคลเอ็นต์** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อข้อมูลลับไคลเอ็นต์ใน key vault นี้
9. ในฟิลด์ **ชื่อข้อมูลลับในการเข้าสู่ระบบทางเทคนิค** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อการเข้าสู่ระบบทางเทคนิคใน key vault นี้
10. ในฟิลด์ **ชื่อข้อมูลลับในรหัสผ่านระบบทางเทคนิค** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อรหัสผ่านระบบทางเทคนิคใน key vault นี้
11. บนแท็บ **ไปป์ไลน์การประมวลผล** ในส่วน **ไปป์ไลน์การประมวลผล** เลือก **รวมกับ Chorus Pro ของฝรั่งเศส** พร้อมชื่อการดำเนินการ **สถานะคำขอ Chorus Pro ของฝรั่งเศส**
12. ในส่วน **พารามิเตอร์** ในฟิลด์ **ชื่อข้อมูลลับของรหัสไคลเอ็นต์** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อรหัสไคลเอ็นต์ใน key vault นี้
13. ในฟิลด์ **ชื่อข้อมูลลับของข้อมูลลับไคลเอ็นต์** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อข้อมูลลับไคลเอ็นต์ใน key vault นี้
14. ในฟิลด์ **ชื่อข้อมูลลับในการเข้าสู่ระบบทางเทคนิค** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อการเข้าสู่ระบบทางเทคนิคใน key vault นี้
15. ในฟิลด์ **ชื่อข้อมูลลับในรหัสผ่านระบบทางเทคนิค** ให้เลือกชื่อข้อมูลลับที่คุณสร้างขึ้นเพื่อรหัสผ่านระบบทางเทคนิคใน key vault นี้
16. เลือก **ตกลง** และจากนั้นปิดหน้า
17. ทําซ้ําขั้นตอน 6 ถึง 16 ของการตั้งค่าคุณลักษณะ **ใบแจ้งหนี้โครงการ UBL ที่ได้รับมา** **UBL ใบลดหนี้การขายที่ได้รับมา** และ **UBL ใบลดหนี้โครงการที่ได้รับมา**
