# <a name="get-started-with-the-electronic-invoicing-add-on"></a>เริ่มต้นใช้งาน add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์

[!include [banner](../includes/banner.md)]

หัวข้อนี้ให้ข้อมูลซึ่งจะช่วยให้คุณเริ่มต้นใช้งาน add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ ก่อนอื่น จะแนะนำให้คุณทราบถึงขั้นตอนการตั้งค่าคอนฟิกใน Microsoft Dynamics Lifecycle Services (LCS), Regulatory Configuration Services (RCS) และ Dynamics 365 Finance ถัดไป จะอธิบายถึงกระบวนการสำหรับการส่งเอกสารผ่านทางบริการโดยใช้ Dynamics 365 Finance หรือ Dynamics 365 Supply Chain Management นอกจากนี้คุณจะได้เรียนรู้วิธีการแปลล็อกการส่ง

## <a name="availability"></a>ความพร้อมใช้งาน

Add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์เริ่มต้นพร้อมใช้งานสำหรับหลายประเทศ Add-in สนับสนุนการสร้างใบแจ้งหนี้อิเล็กทรอนิกส์ และส่งเอกสารทางธุรกิจต่อไปนี้:

|-----------------|--------------------------------------------|
| ออสเตรีย         | ใบแจ้งหนี้การขายและโครงการ                 |
| เบลเยียม         | ใบแจ้งหนี้การขายและโครงการ                 |
| บราซิล          | เอกสารทางการเงินแบบอิเล็กทรอนิกส์ แบบจำลอง 55 (NF-e) |
| เดนมาร์ก         | ใบแจ้งหนี้การขายและโครงการ                 |
| เอสโตเนีย         | ใบแจ้งหนี้การขายและโครงการ                 |

## <a name="licensing"></a>การให้ลิขสิทธิ์

คุณสามารถใช้ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์กับใบอนุญาตปัจจุบันของคุณได้ ไม่จำเป็นต้องมีลิขสิทธิ์เพิ่มเติมสำหรับใช้บริการ

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

ก่อนที่คุณจะทำขั้นตอนต่างๆ ในหัวข้อนี้ให้เสร็จสมบูรณ์ คุณต้องมีข้อกำหนดเบื้องต้นต่อไปนี้:

- เข้าถึงบัญชี LCS ของคุณ
- โครงการที่มีการปรับใช้ LCS ซึ่งรวมถึง Finance หรือ Supply Chain Management เวอร์ชัน 10.0.13 หรือใหม่กว่า
- เข้าถึงบัญชี RCS ของคุณ
- เปิดใช้งานคุณลักษณะโลกาภิวัตน์สำหรับบัญชี RCS ของคุณ ผ่านโมดูล **การจัดการคุณลักษณะ** สำหรับข้อมูลเพิ่มเติม ดู [Regulatory Configuration Services (RCS) - คุณลักษณะแบบโลกาภิวัตน์](rcs-globalization-feature.md)
- สร้างทรัพยากรเพดานคีย์และบัญชีจัดเก็บใน Azure สำหรับข้อมูลเพิ่มเติม ดู [Create Azure Storage Account and Key Vault](e-invoicing-create-azure-storage-account-key-vault.md)

## <a name="overview"></a>ภาพรวม

ภาพประกอบต่อไปนี้แสดงขั้นตอนหลักห้าขั้นตอน ที่คุณจะกรอกข้อมูลในหัวข้อนี้ให้เสร็จสมบูรณ์

![ภาพรวมของห้าขั้นตอนในหัวข้อนี้](media/e-invoicing-services-get-started-overview-5-steps.png)

1. **การตั้งค่าทรัพยากร Azure:** ตั้งค่าคอนฟิกการจัดเก็บ Azure และการอัพโหลดใบรับรองดิจิทัลใน Azure Key Vault
2. **การตั้งค่า LCS:** ติดตั้ง add-in สำหรับ microservices
3. **การตั้งค่า RCS:** ตั้งค่าสภาพแวดล้อม การเข้าถึงของผู้ใช้ และคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์
4. **การตั้งค่าไคลเอนต์:** ตั้งค่าการเชื่อมต่อระหว่างไคลเอนต์และ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ และปิดใช้งานคุณลักษณะเดิม สำหรับการส่งและรับคำตอบสำหรับเอกสารอิเล็กทรอนิกส์
5. **ส่งใบแจ้งหนี้:** ส่งเอกสารอิเล็กทรอนิกส์ผ่านทาง add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ และรับการตอบสนอง

> [!NOTE]
> ขั้นตอนการตั้งค่าคอนฟิกบางขั้นตอนในหัวข้อนี้เป็นแบบทั่วไป และประเทศ/ภูมิภาค-การวินิจฉัย ขั้นตอนและการตั้งค่าที่เป็นเฉพาะประเทศ/ภูมิภาคจะมีการอธิบายไว้ในหัวข้อเฉพาะประเทศ/ภูมิภาค

## <a name="lcs-setup"></a>การตั้งค่า LCS

1. ลงชื่อเข้าใช้บัญชี LCS ของคุณ
2. เลือกไทล์ **การจัดการคุณลักษณะการแสดงตัวอย่าง** และในกลุ่มฟิลด์ของ **ลักษณะการทำงานการแสดงตัวอย่างสาธารณะ** ให้เลือก **BusinessDocumentSubmission**
3. ทำเครื่องหมายที่ฟิลด์ **เปิดใช้ลักษณะการทำงานการแสดงตัวอย่าง**
4. เลือกโครงการการปรับใช้ LCS ก่อนที่คุณจะสามารถเลือกโครงการได้ โครงการต้องใช้และรันอยู่
5. บนแท็บด่วน **add-in ของสภาพแวดล้อม** ให้เลือก **ติดตั้ง add-in ใหม่**
6. เลือก **การส่งเอกสารธุรกิจ**
7. ในกล่องโต้ตอบ **การตั้งค่า add-in** ในฟิลด์ **รหัสแอพลิเคชัน AAD** ป้อน **091c98b0-a1c9-4b02-b62c-7753395ccabe** ค่านี้เป็นค่าคงที่
8. ในฟิลด์ **รหัสผู้เช่า AAD** ป้อนรหัสของบัญชีการบอกรับเป็นสมาชิก Azure ของคุณ

    ![การตั้งค่ากล่องโต้ตอบ add-in ใน LCS](media/e-invoicing-services-get-started-lcs-addin-setup.png)

9. เลือกกล่องกาเครื่องหมายเพื่อยอมรับข้อกำหนดและเงื่อนไข
10. เลือก **ติดตั้ง**

## <a name="rcs-setup"></a>การตั้งค่า RCS

ในระหว่างการตั้งค่า RCS คุณจะทำงานเหล่านี้ให้เสร็จสมบูรณ์:

1. ตั้งค่า Key Vault ใน RCS
2. ตั้งค่าการรวม RCS กับเซิร์ฟเวอร์ add-on ใบแจ้งหนี้อิเล็กทรอนิกส์
3. สร้างสภาพแวดล้อมของ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์สำหรับองค์กรของคุณ

### <a name="set-up-the-key-vault-in-rcs"></a>ตั้งค่า Key Vault ใน RCS

1. ลงชื่อเข้าใช้บัญชี RCS ของคุณ
2. ในพื้นที่ทำงาน **คุณลักษณะที่ใช้ทั่วโลก** ในส่วน **สภาพแวดล้อม** ให้เลือกไทล์ **ใบแจ้งหนี้อิเล็กทรอนิกส์**
3. เลือก **สภาพแวดล้อมการบริการ**

    ![การเลือกสภาพแวดล้อมการบริการ](media/e-invoicing-services-get-started-select-service-environments.png)

> [!NOTE]
> ตัวเลือก **แอพลิเคชันที่เชื่อมโยง** อนุญาตให้เข้าถึงการตั้งค่าคอนฟิกของ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ ใน Finance หรือ Supply Management ผ่านทาง RCS อย่างไรก็ตามขณะนี้ คุณลักษณะนี้ยังคงอยู่ระหว่างการพัฒนา

4. บนบานหน้าต่างการดำเนินการ เลือก **พารามิเตอร์ Key Vault**

    ![การเลือกพารามิเตอร์ Key Vault](media/e-invoicing-services-get-started-select-key-vault-parameters.png)

5. ในบานหน้าต่างการดำเนินการ เลือก **สร้าง** เพื่อเพิ่ม Key Vault
6. ในฟิลด์ **Key Vault URI** ให้ป้อนค่าแอททริบิวต์ **ชื่อ DNS** ของทรัพยากร Key Vault ที่คุณตั้งค่าคอนฟิกไว้ใน Azure สำหรับข้อมูลเกี่ยวกับตำแหน่งที่จะค้นหาค่า **ชื่อ DNS** ให้ดู [Create Azure Storage Account and Key Vault](e-invoicing-create-azure-storage-account-key-vault.md)

    ![ฟิลด์ URI ของ Key Vault](media/e-invoicing-services-get-started-enter-key-vault-uri.png)

7. บนแท็บด่วน **ใบรับรอง** เลือก **เพิ่ม** เพื่อป้อนชื่อใบรับรองดิจิทัลทั้งหมด และความลับ Key Vault ที่จำเป็นในการสร้างการเชื่อมต่อเชื่อถือได้ ในคอลัมน์ **ชนิด** คุณสามารถระบุว่าเป็นใบรับรองหรือความลับ มีการตั้งค่าคอนฟิกชุดของค่าทั้งสองในทรัพยากร Key Vault ใน Azure

    ![การเพิ่มใบรับรอง](media/e-invoicing-services-get-started-add-digital-certificates.png)

8. ถ้าใบแจ้งหนี้เฉพาะประเทศ/ภูมิภาคของคุณ จำเป็นต้องมีกลุ่มของใบรับรองเพื่อใช้ลายเซ็นดิจิทัล ให้เลือก **กลุ่มของใบรับรอง** ในบานหน้าต่างการดำเนินการ แล้วป้อนลำดับของใบรับรองหรือความลับของ Key Vault

### <a name="set-up-the-rcs-integration-with-the-electronic-invoicing-add-on-server"></a>ตั้งค่าการรวม RCS กับเซิร์ฟเวอร์ add-on ใบแจ้งหนี้อิเล็กทรอนิกส์

1. ในพื้นที่ทำงาน **คุณลักษณะที่ใช้ทั่วโลก** ในส่วน **การตั้งค่าที่เกี่ยวข้อง** ให้เลือกลิงก์ **พารามิเตอร์การรายงานทางอิเล็กทรอนิกส์**
2. เลือก **คลิกที่นี่เพื่อเชื่อมต่อกับ Lifecycle Service** ถ้าคุณไม่ต้องการเชื่อมต่อกับ LCS ให้เลือก **ยกเลิก**
3. บนแท็บ **บริการการออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์** ในฟิลด์ **URI ปลายทางการบริการ** ให้ป้อนค่าตามเขตภูมิศาสตร์ที่พร้อมใช้งาน: `https://businessdocumentsubmission.us.operations365.dynamics.com/` หรือ `https://businessdocumentsubmission.eu.operations365.dynamics.com/`
4. ในฟิลด์ **รหัสแอพลิเคชัน** ให้ตรวจสอบว่าแสดงรหัส **0cdb527f-a8d1-4bf8-9436-b352c68682b2** หรือไม่ ค่านี้เป็นค่าคงที่
5. ในฟิลด์ **ID ของสภาพแวดล้อม LCS** ป้อนรหัสของบัญชีการบอกรับเป็นสมาชิก LCS ของคุณ

![การป้อนพารามิเตอร์ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์](media/e-invoicing-services-get-started-enter-e-invoicing-parameters.png)

### <a name="add-an-electronic-invoicing-add-on-environment"></a>เพิ่มสภาพแวดล้อม add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์

คุณสามารถสร้างสภาพแวดล้อมที่แตกต่างกันสำหรับ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ เช่น Dev, Test หรือสภาพแวดล้อมการทำงานจริง

1. ในพื้นที่ทำงาน **คุณลักษณะที่ใช้ทั่วโลก** ในส่วน **สภาพแวดล้อม** ให้เลือกไทล์ **ใบแจ้งหนี้อิเล็กทรอนิกส์**
2. เลือก **สร้าง** เพื่อสร้างสภาพแวดล้อม
3. ในฟิลด์ **บัญชีโทเค็น Storage SAS** ให้ป้อนชื่อของความลับ Key Vault ที่คุณตั้งค่าคอนฟิกไว้ใน Key Vault ใน RCS

    ![ฟิลด์บัญชีโทเค็น Storage SAS](media/e-invoicing-services-get-started-enter-sas-token-secret.png)

4. บนแท็บด่วน **ผู้ใช้** ให้เลือก **สร้าง** เพื่อให้สิทธิการเข้าถึงแก่ผู้ใช้สำหรับสภาพแวดล้อมนี้

    ![การเพิ่มผู้ใช้บริการ](media/e-invoicing-services-get-started-enter-service-users.png)

5. ในบานหน้าต่างการดำเนินการ ให้เลือก **เผยแพร่** เพื่อเผยแพร่สภาพแวดล้อมไปยังเซิร์ฟเวอร์ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์

    ![ปุ่มเผยแพร่](media/e-invoicing-services-get-started-publish-service-environment.png)

### <a name="e-invoicing-feature-setup"></a>การตั้งค่าคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์

"คุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์" คือชื่อทั่วไปสำหรับทรัพยากรที่มีการตั้งค่าคอนฟิกและเผยแพร่ เพื่อใช้เซิร์ฟเวอร์ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ การตั้งค่าของคุณลักษณะใบแจ้งหนี้อิเล็กทรอนิกส์ ซึ่งรวมถึงการใช้รูปแบบการตั้งค่าคอนฟิกการรายงานอิเล็กทรอนิกส์ (ER) เพื่อสร้างไฟล์การส่งออกและการนำเข้าที่สามารถจัดโครงแบบได้ และการใช้การดำเนินการและการดำเนินการขั้นตอน เพื่อเปิดใช้งานการสร้างกฎที่สามารถจัดโครงแบบได้ เพื่อส่งคำขอ นำเข้าการตอบสนอง และแยกเนื้อหาที่ตอบสนอง

เนื่องจากมีการเปลี่ยนแปลงในรูปแบบใบแจ้งหนี้และขั้นตอนการดำเนินการ การตั้งค่าคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์จะเป็นแบบประเทศ/ภูมิภาค-เชื่อมโยง

## <a name="set-up-electronic-invoicing-add-on-integration-in-finance-or-supply-chain-management"></a>ตั้งค่าการรวม add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ใน Finance หรือ Supply Chain Management 

ในระหว่างการตั้งค่านี้ คุณจะดำเนินงานต่อไปนี้ให้เสร็จสมบูรณ์:

1. เปิดคุณลักษณะ flighted
2. เปิดคุณลักษณะการรวม add-on ใบแจ้งหนี้อิเล็กทรอนิกส์เพื่อเปิดใช้งานการรวมกับ Finance
3. ตั้งค่า URL ของปลายทาง add-on ใบแจ้งหนี้อิเล็กทรอนิกส์
4. นำเข้าการตั้งค่าคอนฟิก ER ที่เกี่ยวข้องกับคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาค
5. เปิดคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาคที่เกี่ยวข้อง
6. นำเข้าการตั้งค่าคอนฟิก ER และตั้งค่าชนิดของการตอบสนองที่จำเป็นในการอัพเดตเอกสารใบแจ้งหนี้เฉพาะประเทศ/ภูมิภาคของคุณ อันเป็นผลมาจากกระบวนการส่ง

### <a name="open-flighted-feature"></a>เปิดคุณลักษณะการสลับ
มีการเปิดใช้งานคุณลักษณะการรวมใบแจ้งหนี้อิเล็กทรอนิกส์ผ่านทางการสลับ การสลับ เป็นแนวคิดที่ทำให้สามารถเปิดหรือปิดคุณลักษณะโดยค่าเริ่มต้นได้ ขั้นตอนต่อไปนี้จะเปิดใช้งานการสลับในสภาพแวดล้อมที่ไม่มีการทำงานจริง 

1. ดำเนินการคำสั่ง SQL ต่อไปนี้:

    INSERT INTO SYSFLIGHTING (FLIGHTNAME, ENABLED) VALUES ('BusinessDocumentSubmissionServiceEnabled', 1)
    
    INSERT INTO SYSFLIGHTING (FLIGHTNAME, ENABLED) VALUES ('ElectronicInvoicingServiceIntegrationFeature', 1)
    
2. หลังจากที่ทำการเปลี่ยนแปลงข้างต้นแล้ว ให้ดำเนินการ IISReset บน AOS ทั้งหมด

### <a name="turn-on-the-electronic-invoicing-add-on-integration-feature"></a>เปิดคุณลักษณะการรวมของ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์

1. ลงชื่อเข้าใช้ Finance หรือ Supply Chain Management
2. ในพื้นที่ทำงาน **การจัดการคุณลักษณะ** ให้ค้นหาคุณลักษณะใหม่ **การรวม add-on ของการออกใบแจ้งหนี้ทางอิเล็กทรอนิกส์ที่จัดโครงแบบ** ถ้าคุณลักษณะยังคงแสดงอยู่ในหน้าการจัดการคุณลักษณะ ให้รันฟังก์ชัน **ตรวจสอบสำหรับการอัพเดต**
3. เลือกคุณลักษณะ และจากนั้นเลือก **เปิดใช้งานตอนนี้**

### <a name="set-up-the-service-endpoint-url"></a>ตั้งค่า URL ปลายทางการบริการ

1. ไปที่ **การจัดการองค์กร \> การตั้งค่า \> พารามิเตอร์เอกสารอิเล็กทรอนิกส์**
2. บนแท็บ **บริการการส่ง** ในฟิลด์ **URL ปลายทางการบริการ** ให้ป้อน `https://businessdocumentsubmission.us.operations365.dynamics.com/`
3. ในฟิลด์ **สภาพแวดล้อม** ให้ป้อนชื่อของสภาพแวดล้อม add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ที่คุณสร้างขึ้นในระหว่างการตั้งค่า RCS

![ป้อนพารามิเตอร์การบริการ](media/e-invoicing-services-get-started-enter-service-endpoint.png)

### <a name="import-the-er-configurations"></a>นำเข้าการตั้งค่าคอนฟิกของ ER

เมื่อต้องการเปิดใช้งานข้อมูลธุรกิจที่จะรวบรวมและส่งไปยัง add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ คุณต้องนำเข้ารูปแบบข้อมูล ER และการตั้งค่าคอนฟิกรูปแบบข้อมูล ER ซึ่งเกี่ยวข้องกับคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาค ที่คุณต้องการใช้

1. ในพื้นที่ทำงาน **รายงานอิเล็กทรอนิกส์** ในส่วน **ผู้ให้บริการการตั้งค่าคอนฟิก** ให้เลือก **Microsoft** ตรวจสอบให้แน่ใจว่ามีการตั้งค่าผู้ให้บริการการตั้งค่าคอนฟิกนี้เป็น **ใช้งานอยู่** สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการตั้งค่าผู้ให้บริการเป็น **ใช้งานอยู่** ให้ดู [สร้างผู้ให้บริการการตั้งค่าคอนฟิก และทำเครื่องหมายเป็นใช้งานอยู่](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/analytics/tasks/er-configuration-provider-mark-it-active-2016-11)
3. เลือก **ที่เก็บ**
4. เลือก **ทรัพยากรส่วนกลาง** แล้วเลือก **เปิด**
5. ในกล่องโต้ตอบ **เชื่อมต่อไปยัง Lifecycle Services** ให้เลือก **คลิกที่นี่เพื่อเชื่อมต่อกับ Lifecycle Service**
6. ทั้งนี้ขึ้นอยู่กับประเทศหรือภูมิภาคที่คุณต้องการใช้คุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์ คุณต้องนำเข้า รูปแบบข้อมูล การแม็ปรูปแบบข้อมูล และรูปแบบของข้อมูลที่เกี่ยวข้อง สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการตั้งค่าคอนฟิก ER ที่คุณควรนำเข้า โปรดดูที่หัวข้อ "การเริ่มต้น add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์" ของประเทศ/ภูมิภาคเฉพาะ
7. นำเข้า **แบบจำลองบริบทของใบแจ้งหนี้ของลูกค้า** แบบจำลองนี้มีพารามิเตอร์เพิ่มเติมที่อธิบายถึง ในระหว่างสิ่งอื่นๆ สภาพแวดล้อมใน Finance ที่ใช้สำหรับ add-on ของใบแจ้งหนี้ทาอิเล็กทรอนิกส์ ในระหว่างการส่งข้อมูลธุรกิจ

### <a name="turn-on-countryregion-specific-e-invoicing-features"></a><a name="region-specific"></a>เปิดคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาคเฉพาะ

เมื่อต้องการเปิดใช้คุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาค เพื่อให้ทำงานกับ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ คุณต้องเปิดใช้งานคุณลักษณะในนิติบุคคลแต่ละแห่งที่คุณต้องการใช้ หลังจากนั้น การรวมใบแจ้งหนี้อิเล็กทรอนิกส์เก่าๆ จะไม่สามารถใช้ได้อีกต่อไป และการรวมกับ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ใหม่เปิดอยู่

1. ไปที่ **การจัดการองค์กร \> การตั้งค่า \> พารามิเตอร์เอกสารอิเล็กทรอนิกส์**
2. บนแท็บ **คุณลักษณะ** ในแถวสำหรับคุณลักษณะที่เกี่ยวข้องกับคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาคของคุณ ให้เลือกกล่องกาเครื่องหมายในคอลัมน์ **เปิดใช้งาน** สำหรับข้อมูลเพิ่มเติมเกี่ยวกับคุณลักษณะใดที่คุณควรเปิดใช้งาน โปรดดูที่หัวข้อ "การเริ่มต้น add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์" ของประเทศ/ภูมิภาคเฉพาะ

![การเปิดคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์](media/e-invoicing-services-get-started-enable-invoicing-feature.png)

> [!NOTE]
> ถ้าคุณมีนิติบุคคลหลายรายที่มีการตั้งค่าคอนฟิกสำหรับประเทศหรือภูมิภาคต่างๆ คุณสามารถเปิดใช้คุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์เฉพาะประเทศ/ภูมิภาค สำหรับแต่ละนิติบุคคล

### <a name="import-er-configurations-and-set-up-the-response-types-to-update-your-countryregion-specific-invoice-document"></a>นำเข้าการตั้งค่าคอนฟิก ER และตั้งค่าชนิดของการตอบสนองเพื่ออัพเดตเอกสารใบแจ้งหนี้เฉพาะประเทศ/ภูมิภาคของคุณ

ถ้าเอกสารใบแจ้งหนี้ที่ส่งต้องใช้การอัพเดตหลังจากการตอบสนองของการส่งไปยังบริการตรวจสอบภาครัฐ คุณต้องนำเข้าแบบจำลองข้อมูล ER พิเศษ และการตั้งค่าคอนฟิกเพื่อเปิดใช้งานสถานะของเอกสารใบแจ้งหนี้หรือฟิลด์เพิ่มเติมอื่นๆ ที่จะอัพเดต

1. ในพื้นที่ทำงาน **รายงานอิเล็กทรอนิกส์** ในส่วน **ผู้ให้บริการการตั้งค่าคอนฟิก** ให้เลือก **Microsoft**
2. เลือก **ที่เก็บ**
3. เลือก **ทรัพยากรส่วนกลาง** แล้วเลือก **เปิด**
4. นำเข้า **แบบจำลองข้อความตอบสนอง** **รูปแบบการนำเข้าข้อความตอบสนอง** **การแม็ปแบบจำลองข้อความตอบสนองไปยังปลายทาง** และ **รูปแบบการนำเข้าเนื้อหาของไฟล์**
5. ไปที่ **การจัดการองค์กร \> การตั้งค่า \> พารามิเตอร์เอกสารอิเล็กทรอนิกส์**
6. บนแท็บ **เอกสารอิเล็กทรอนิกส์** ให้เลือก **เพิ่ม** เพื่อป้อนชื่อของตารางที่เกี่ยวข้องกับเอกสารใบแจ้งหนี้เฉพาะประเทศ/ภูมิภาคของคุณ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับชื่อตารางใดที่คุณควรเลือก โปรดดูที่หัวข้อ "การเริ่มต้น add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์" ของประเทศ/ภูมิภาคเฉพาะ
7. เลือก **ชนิดของการตอบสนอง** เพื่อตั้งค่าคอนฟิกชนิดของการตอบสนอง สำหรับข้อมูลเพิ่มเติมเกี่ยวกับชื่อตารางใดที่คุณควรเลือก โปรดดูที่หัวข้อ "การเริ่มต้น add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์" ของประเทศ/ภูมิภาคเฉพาะ

ตารางต่อไปนี้จะอธิบายถึงคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์อื่นๆ ที่มีให้ดาวน์โหลดจากที่เก็บส่วนกลางของรายงานอิเล็กทรอนิกส์ เพื่อสร้างใบแจ้งหนี้อิเล็กทรอนิกส์

## <a name="electronic-invoice-processing-in-finance-and-supply-chain-management"></a>การประมวลผลใบแจ้งหนี้อิเล็กทรอนิกส์ใน Finance และ Supply Chain Management

ในระหว่างการประมวลผล คุณจะทำงานเหล่านี้ให้เสร็จสมบูรณ์:

1. ส่งเอกสารทางธุรกิจ (ใบแจ้งหนี้) ผ่านทาง add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์
2. ดูล็อกการดำเนินการส่ง

### <a name="submit-business-documents"></a>ส่งเอกสารทางธุรกิจ

ระหว่างกระบวนการส่งปกติ การสื่อสารระหว่างไคลเอนต์และ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์เป็นแบบสองทาง วัตถุประสงค์คือการทำงานหลักสองอย่างในระหว่างการส่งเอกสารอิเล็กทรอนิกส์:

1. ส่งเอกสารอิเล็กทรอนิกส์ทั้งหมดที่กำลังรอการส่งจาก Finance และมีสถานะที่ถูกต้องสำหรับการส่งและตรงตามเงื่อนไขการเลือก
2. นำเข้าไปยัง Finance การตอบสนองที่ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ส่งคืนสำหรับเอกสารอิเล็กทรอนิกส์ที่ส่งไว้ก่อนหน้านี้ หลังจากนำเข้า การตอบสนองจะถูกแยกวิเคราะห์ และสถานะของเอกสารทางธุรกิจจะถูกอัพเดตโดยสอดคล้องกัน

คุณสามารถส่งเอกสารธุรกิจด้วยตนเองหรือตามข้อกำหนดของกำหนดการของคุณได้

1. ไปที่ **การจัดการองค์กร \> ประจำงวด \> เอกสารอิเล็กทรอนิกส์ \> ส่งเอกสารอิเล็กทรอนิกส์**
2. สำหรับการส่งเอกสารฉบับแรกของเอกสารใดๆ ให้ตั้งค่าตัวเลือก **ส่งเอกสารใหม่** เป็น **ไม่** เสมอ ถ้าคุณต้องส่งเอกสารผ่านทางบริการอีกครั้ง ให้กำหนดตัวเลือกนี้เป็น **ใช่**
3. บนแท็บด่วน **เรกคอร์ดที่จะรวม** ให้เลือก **ตัวกรอง** เพื่อเปิดกล่องโต้ตอบ **การสอบถาม** ซึ่งคุณสามารถสร้างการสอบถามเพื่อเลือกเอกสารสำหรับการส่งได้

![ส่งกล่องโต้ตอบเอกสารอิเล็กทรอนิกส์](media/e-invoicing-services-get-started-submission-form.png)

### <a name="filter-query"></a>กรองการสอบถาม

1. ในกล่องโต้ตอบ **การสอบถาม** บนแท็บ **ช่วง** ป้อนเงื่อนไขการกรองข้อมูลโดยใช้ฟิลด์ **ตาราง** **ตารางที่สืบทอดมา** **ฟิลด์** และฟิลด์ **เงื่อนไข**
2. เลือก **เพิ่ม** เพื่อเพิ่มเงื่อนไขเพิ่มเติมมากเท่าที่คุณต้องการ เพื่อเลือกเอกสารธุรกิจ

    ![การตั้งค่าเงื่อนไขการกรองข้อมูลการส่ง](media/e-invoicing-services-get-started-set-up-submission-filter-criteria.png)

3. เลือก **ตกลง** เพื่อปิดกล่องโต้ตอบ **การสอบถาม**
4. เลือก **ตกลง** เพื่อส่งเอกสารทางธุรกิจที่เลือกไปยัง add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์

    > [!NOTE]
    > ในระหว่างความพยายามครั้งแรกของคุณในการส่งเอกสารผ่านทางบริการ คุณจะได้รับพร้อมท์ให้ยืนยันการเชื่อมต่อกับ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ เลือก **คลิกที่นี่เพื่อเชื่อมต่อกับ Electronic Document Submission Service**
    >
    > ![เชื่อมต่อกับกล่องข้อความ Electronic Document Submission Service](media/e-invoicing-services-get-started-dialog-form-connect-e-Invoicing-services.png)
    >
    > ถ้าการเชื่อมต่อเสร็จเรียบร้อยแล้ว คุณจะได้รับข้อความยืนยัน
    >
    > ![การยืนยันการเชื่อมต่อกับ add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์](media/e-invoicing-services-get-started-confirmation-connection-e-invoicing-services.png)

5. ปิดกล่องโต้ตอบ

> [!NOTE]
> หลังจากการส่งแต่ละครั้ง ศูนย์ปฏิบัติการจะแสดงจำนวนของเอกสารที่ส่ง
>
> ![ข้อความศูนย์ปฏิบัติการ](media/e-invoicing-services-get-started-view-action-center-messages.png)

### <a name="submission-by-batch"></a>ส่งตามชุดงาน

แทนที่จะส่งเอกสารด้วยตนเอง คุณสามารถทำให้กระบวนการส่งออกและรันในพื้นหลังได้โดยอัตโนมัติ ตามความถี่ที่ตั้งค่าคอนฟิกของการดำเนินการชุดงาน

1. ในกล่องโต้ตอบ **ส่งเอกสารอิเล็กทรอนิกส์** บนแท็บด่วน **รันในพื้นหลัง** ให้ตั้งค่าตัวเลือก **การประมวลผลชุดงาน** เป็น **ใช่**
2. บนแท็บ **การเกิดซ้ำ** ให้ตั้งค่าคอนฟิกความถี่ในการประมวลผลชุดงาน

![การตั้งค่าการส่งโดยเรียงตามชุดงาน](media/e-invoicing-services-get-started-set-up-submission-batch.png)

### <a name="view-all-submission-logs"></a>ดูล็อกการส่งทั้งหมด

1. ไปที่ **การจัดการองค์กร \> ประจำงวด \> เอกสารอิเล็กทรอนิกส์ \> ล็อกการส่งเอกสารอิเล็กทรอนิกส์**
2. ในฟิลด์ **ชนิดเอกสาร** เลือกชนิดของเอกสารที่จะกรองโดย

    ![การเลือกชนิดของเอกสารเพื่อดูล็อกการส่ง สำหรับ](media/e-invoicing-services-get-started-select-document-type-for-viewing-submission-log.png)

    > [!IMPORTANT]
    > ค่าที่แสดงในคอลัมน์ **สถานะการส่ง** แสดงสถานะที่เกี่ยวข้องกับความสมบูรณ์ของตัวกระบวนการส่งเอง ซึ่งบ่งชี้ว่าขั้นตอนการดำเนินการที่มีการตั้งค่าคอนฟิกใน RCS ได้รับการรันจนกว่าจะถึงวันที่สิ้นสุด หรือไม่โดยไม่คำนึงว่าเอกสารอิเล็กทรอนิกส์ได้รับการอนุมัติหรือปฏิเสธ ค่าที่แสดงในคอลัมน์ **สถานะการส่ง** ไม่แสดงสถานะของเอกสารที่ส่ง คุณสามารถดูสถานะของเอกสารที่ส่งแล้ว (นั่นคือ เอกสารนี้ได้รับการอนุมัติหรือปฏิเสธ) บนแท็บด่วน **ล็อกการดำเนินการประมวลผล** ในรายละเอียดล็อกการส่ง ตามที่อธิบายไว้ถัดไป

3. บนบานหน้าต่างการดำเนินการ เลือก **การสอบถาม \> รายละเอียดการส่ง**
4. ดูรายละเอียดล็อกการส่ง

    ![รายละเอียดล็อกการส่ง](media/e-invoicing-services-get-started-view-submission-log-form.png)

ผลลัพธ์ที่แสดงอยู่ในล็อกการส่งจะขึ้นอยู่กับวิธีการตั้งค่าคุณลักษณะของใบแจ้งหนี้อิเล็กทรอนิกส์ใน RCS อย่างไรก็ตาม โดยไม่คำนึงถึงการตั้งค่า ล็อกการส่งจะมีแท็บด่วนสามแท็บต่อไปนี้

- **การดำเนินการประมวลผล** – แท็บด่วนนี้แสดงล็อกการดำเนินการสำหรับการดำเนินการที่ได้รับการตั้งค่าคอนฟิกในเวอร์ชันของคุณลักษณะที่ตั้งค่าไว้ใน RCS คอลัมน์ **สถานะ** แสดงว่ามีการรันการดำเนินการเสร็จเรียบร้อยแล้วหรือไม่
- **ไฟล์การดำเนินการ** – แท็บด่วนนี้แสดงไฟล์กลางที่สร้างขึ้นในระหว่างการดำเนินการ คุณสามารถเลือก **มุมมอง** เพื่อดาวน์โหลดและดูเนื้อหาของไฟล์ได้
- **ล็อกการดำเนินการการประมวลผล** – แท็บด่วนนี้แสดงผลลัพธ์ของการสื่อสารระหว่าง add-on ของใบแจ้งหนี้อิเล็กทรอนิกส์ และบริการเว็บเป้าหมาย นอกจากนี้ยังแสดงสิ่งที่ส่งคืนโดยการประมวลผลบริการเว็บ
