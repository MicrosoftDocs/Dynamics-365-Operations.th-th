# <a name="improve-the-prediction-model-preview"></a>ปรับปรุงแบบจำลองการคาดคะเน (ตัวอย่าง)

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

หัวข้อนี้อธิบายถึงลักษณะการทำงานที่คุณสามารถใช้เพื่อปรับปรุงประสิทธิภาพการทำงานของแบบจำลองการคาดการณ์ คุณเริ่มต้นการปรับปรุงแบบจำลองของคุณในพื้นที่ทำงาน **การคาดการณ์การชำระเงินของลูกค้า** ใน Microsoft Dynamics 365 Finance ดำเนินการขั้นตอนการปรับปรุงเสร็จสมบูรณ์แล้วใน AI Builder

## <a name="select-historical-outcomes"></a>เลือกผลลัพธ์ในอดีต

คุณเลือกหนึ่งในผลลัพธ์ที่เป็นไปได้ที่เป็นไปได้สำหรับใบแจ้งหนี้อย่างน้อยหนึ่งรายการ: **ตรงเวลา** **ล่าช้า** และ **ล่าช้ามาก** ควรเลือกผลลัพธ์ทั้งสามประการ ถ้าคุณยกเลิกการเลือกผลลัพธ์ใด ๆ ใบแจ้งหนี้จะถูกกรองออกจากกระบวนการฝึกอบรมและความถูกต้องของการคาดการณ์จะถูกลดลง

[![การยืนยันผลลัพธ์](./media/confirm-3-outcomes.png)](./media/confirm-3-outcomes.png)

ถ้าองค์กรของคุณต้องการผลลัพธ์เพียงสองประการ ให้เปลี่ยนขีดจำกัด **ล่าช้า** และ **ล่าช้ามาก** เป็น 0 (ศูนย์) วัน ด้วยวิธีนี้ คุณจะยุบการคาดการณ์ไปยังสถานะไบนารีของ **ตรงเวลา** หรือ **ล่าช้า** ได้อย่างมีประสิทธิภาพ

## <a name="select-fields"></a>เลือกฟิลด์

เมื่อคุณกำลังเลือกฟิลด์ที่จะรวมไว้ในแบบจำลอง ให้ทราบว่ารายชื่อมีฟิลด์ที่พร้อมใช้งานทั้งหมดในตาราง Microsoft Dataverse ที่ถูกแม็ปกับข้อมูลในที่จัดเก็บข้อมูลดิบ Azure บางฟิลด์เหล่านี้ **ไม่** ควรเลือก ฟิลด์ที่ไม่ควรมีการเลือกอยู่เป็นหนึ่งในสามประเภทดังนี้

- ฟิลด์นี้จำเป็นสำหรับตาราง Dataverse แต่ไม่มีข้อมูลที่สำรองไว้ในที่จัดเก็บข้อมูลดิบ
- ฟิลด์เป็นรหัส และดังนั้นจึงไม่ได้ทำให้รู้สึกสำหรับลักษณะการเรียนรู้ของเครื่อง
- ฟิลด์นี้แสดงข้อมูลที่จะไม่พร้อมใช้งานในระหว่างการคาดการณ์

ส่วนต่อไปนี้จะแสดงฟิลด์ที่พร้อมใช้งานสำหรับใบแจ้งหนี้และเอนทิตีของลูกค้า และแสดงรายการฟิลด์ที่ **ไม่** ควรเลือกสำหรับการฝึกอบรม ประเภทที่ระบุไว้สำหรับแต่ละฟิลด์เหล่านั้นอ้างอิงถึงประเภทในรายการข้างหน้า
 
### <a name="invoice-dataverse-table"></a>ตาราง Dataverse ใบแจ้งหนี้

ภาพประกอบต่อไปนี้แสดงฟิลด์ที่ทีให้ใช้สำหรับตารางใบแจ้งหนี้

[![ฟิลด์ที่พร้อมใช้งานสำหรับตารางใบแจ้งหนี้](./media/available-fields.png)](./media/available-fields.png)

ไม่ควรเลือกฟิลด์ต่อไปนี้สำหรับการฝึกอบรม:

- **บัญชีใบแจ้งหนี้** (ประเภท 2)
- **ปิด** (ประเภท 3) – ฟิลด์นี้จะใช้ในการกรองข้อมูลใบแจ้งหนี้สำหรับการฝึกอบรม (ปิด) และการคาดการณ์ (ไม่ได้ปิด)
- **ชื่อ** (ประเภท 1)
- **รหัสที่มา** (ประเภท 2)
- **เรกคอร์ดที่มา** (ประเภท 2)
- **ตารางที่มา** (ประเภท 2)

### <a name="customer-dataverse-table"></a>ตาราง Dataverse ลูกค้า

ภาพประกอบต่อไปนี้แสดงฟิลด์ที่ทีให้ใช้สำหรับตารางลูกค้า

[![ฟิลด์ที่พร้อมใช้งานสำหรับตารางลูกค้า](./media/related-entities.png)](./media/related-entities.png)

ไม่ควรเลือกฟิลด์ต่อไปนี้สำหรับการฝึกอบรม:

- **คีย์ที่ไม่ซ้ำกันของแถว** (ประเภท 2)

## <a name="filters"></a>ตัวกรองข้อมูล

ตัวกรองข้อมูลไม่สนับสนุนสถานการณ์จำลองการคาดการณ์การชำระเงินของลูกค้าในขณะนี้ ดังนั้น ให้เลือก **ข้ามขั้นตอนนี้** และดำเนินการต่อไปยังหน้าสรุป
