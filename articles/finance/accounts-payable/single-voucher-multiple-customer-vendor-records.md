# <a name="single-voucher-with-multiple-customer-or-vendor-records"></a><span data-ttu-id="6e497-104">ใบสำคัญเดียวที่มีเรกคอร์ดของลูกค้าหรือผู้จัดจำหน่ายหลายรายการ</span><span class="sxs-lookup"><span data-stu-id="6e497-104">Single voucher with multiple customer or vendor records</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="6e497-105">หัวข้อนี้แสดงภาพรวมของสิ่งที่เกิดขึ้นเมื่อคุณลงรายการบัญชีใบสำคัญเดียวกับเรกคอร์ของลูกค้าหรือผู้จัดจำหน่ายหลายรายการ</span><span class="sxs-lookup"><span data-stu-id="6e497-105">This topic provides an overview of what happens when you post a single voucher with multiple customer or vendor records.</span></span> <span data-ttu-id="6e497-106">ฟังก์ชันนี้จะถูกหยุดการดำเนินการในรุ่นในอนาคต ดังนั้น เราไม่แนะนำให้ใช้วิธีของการลงรายการบัญชีนี้ เนื่องจากผลกระทบทางบัญชีต่อกระบวนการชำระเงิน</span><span class="sxs-lookup"><span data-stu-id="6e497-106">This functionality will be discontinued in future versions, as a result, we do not recommend using this method of posting because of the accounting impact to settlement processing.</span></span> 

<span data-ttu-id="6e497-107">ตัวอย่างทั่วไปที่มีการใช้ใบสำคัญหนึ่งใบสำหรับลูกค้าหรือผู้จัดจำหน่ายหลายรายการรวมถึงการโอนย้ายยอดดุลระหว่างลูกค้า และการรวมยอดดุลระหว่างลูกค้าและผู้จัดจำหน่ายในองค์กรเดียวกัน</span><span class="sxs-lookup"><span data-stu-id="6e497-107">Some common examples where one voucher is used for multiple customers or vendors include balance transfers between customers, and netting balances between customers and vendors in the same organization.</span></span> 

<span data-ttu-id="6e497-108">คุณสามารถป้อนใบสำคัญที่ประกอบด้วยลูกค้าหรือผู้จัดจำหน่ายอย่างน้อยหนึ่งรายได้โดยใช้วิธีต่อไปนี้:</span><span class="sxs-lookup"><span data-stu-id="6e497-108">A voucher containing more than one customer or vendor can be entered using one of the following methods:</span></span>

-   <span data-ttu-id="6e497-109">การใช้สมุดรายวันที่มีการเลือกตัวเลือก **หมายเลขใบสำคัญหนึ่งหมายเลขเท่านั้น** ไว้เพื่อให้รวมทุกรายการที่เพิ่มลงในสมุดรายวันรวมอยู่ในใบสำคัญเดียวกัน</span><span class="sxs-lookup"><span data-stu-id="6e497-109">Using a journal that has the **One voucher number only** option selected so that every line added to the journal is included on the same voucher.</span></span>
-   <span data-ttu-id="6e497-110">การใช้ใบสำคัญหลายรายการ ซึ่งไม่มีบัญชีแยกประเภทตรงข้าม กับลูกค้าหรือผู้จัดจำหน่ายอย่างน้อยหนึ่งราย</span><span class="sxs-lookup"><span data-stu-id="6e497-110">Using a multi-line voucher, where there is no offset ledger account, with more than one customer or vendor.</span></span>
-   <span data-ttu-id="6e497-111">การป้อนใบสำคัญที่มีบัญชีและบัญชีตรงข้ามเป็น ผู้จัดจำหน่าย/ผู้จัดจำหน่าย ลูกค้า/ลูกค้า ผู้จัดจำหน่าย/ลูกค้า หรือลูกค้า/ผู้จัดจำหน่าย</span><span class="sxs-lookup"><span data-stu-id="6e497-111">Entering a voucher with the account and offset account being vendor/vendor, customer/customer, vendor/customer, or customer/vendor.</span></span>

<span data-ttu-id="6e497-112">หัวข้อนี้แสดงวิธีการประมวลผลการชำระเงินเมื่อมีการลงรายการบัญชีใบสำคัญหนึ่งใบกับเรกคอร์ดของลูกค้าหรือผู้จัดจำหน่ายหลายรายการ</span><span class="sxs-lookup"><span data-stu-id="6e497-112">This topic shows how settlement will be processed when one voucher  with multiple customer or vendor records is posted.</span></span> <span data-ttu-id="6e497-113">นอกจากนี้ หัวข้อนี้ยังแสดงวิธีการแก้ไขปัญหาเพื่อช่วยให้คุณเข้าใจวิธีการหลีกเลี่ยงการใช้ใบสำคัญหนึ่งใบกับลูกค้าหรือผู้จัดจำหน่ายหลายรายการ</span><span class="sxs-lookup"><span data-stu-id="6e497-113">Additionally, this topic provides workarounds to help you understand how to avoid using one voucher with multiple customers or vendors.</span></span> <span data-ttu-id="6e497-114">โดยเฉพาะอย่างยิ่ง มีตัวอย่างที่แสดงสถานการณ์การชำระเงินทั่วไปสองรายการที่ได้รับผลกระทบจากการใช้ใบสำคัญหนึ่งใบกับลูกค้าหรือผู้จัดจำหน่ายหลายรายการ:</span><span class="sxs-lookup"><span data-stu-id="6e497-114">In particular, there are examples which illustrate two common settlement scenarios that are impacted by the use of one voucher with multiple customers or vendors:</span></span>

-   <span data-ttu-id="6e497-115">การบัญชีส่วนลดเงินสด</span><span class="sxs-lookup"><span data-stu-id="6e497-115">Cash discount accounting</span></span>
-   <span data-ttu-id="6e497-116">การบัญชีการประเมินค่าใหม่</span><span class="sxs-lookup"><span data-stu-id="6e497-116">Revaluation accounting</span></span>
