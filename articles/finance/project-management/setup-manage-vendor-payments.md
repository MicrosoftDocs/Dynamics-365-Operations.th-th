# <a name="set-up-and-use-pay-when-paid-vendor-payments"></a>ตั้งค่าและใช้การชำระเงินผู้จัดจำหน่ายแบบจ่ายเมื่อได้รับชำระเงิน

[!include [banner](../includes/banner.md)]

เมื่อคุณอนุมัติผู้จัดจำหน่ายให้ทำงานเป็นผู้รับเหมารายย่อย คุณอาจต้องการคงค้างการชำระเงินแก่ผู้จัดจำหน่าย จนกว่าลูกค้าของคุณจะชำระเงินสำหรับโครงการ เพื่อสนับสนุนสถานการณ์นี้ คุณสามารถตั้งค่าการจ่ายเมื่อได้รับชำระเงิน (PWP) เมื่อคุณตั้งค่าใบสั่งซื้อ (PO) กับผู้จัดจำหน่าย

ตัวอย่างเช่น เมื่อลูกค้าชำระยอดเงินในใบแจ้งหนี้โครงการ คุณสามารถนำออกใช้ยอดเงินในใบแจ้งหนี้ของผู้จัดจำหน่ายบางส่วนหรือทั้งหมด เพียงตั้งค่าเงื่อนไข PWP ที่ระบุว่าผู้จัดจำหน่ายได้รับการชำระเงิน หลังจากที่คุณได้รับเปอร์เซ็นต์ของการชำระเงินที่เกี่ยวข้องจากลูกค้า ถ้าคุณได้รับชำระเงินบางส่วนจากลูกค้า คุณสามารถนำออกใช้ใบแจ้งหนี้ของผู้จัดจำหน่ายที่เกี่ยวข้องบางส่วนด้วยตนเองสำหรับการชำระเงินได้

ตัวอย่างต่อไปนี้กำหนดโครงร่างกระบวนการ เมื่อมีการใช้เงื่อนไข PWP

## <a name="example"></a>ตัวอย่าง

องค์กรของคุณตกลงที่จะให้ลูกค้าที่มีคอมพิวเตอร์ 100 เครื่องซึ่งติดตั้งซอฟต์แวร์ไว้ สำหรับราคา 150.00 ดอลลาร์สหรัฐ (USD) ต่อคอมพิวเตอร์หนึ่งเครื่อง จากนั้น ให้คุณจ้างผู้จัดจำหน่ายเพื่อจัดเตรียมคอมพิวเตอร์ที่มีการติดตั้งซอฟต์แวร์ไว้ ตามข้อตกลง ลูกค้าต้องอนุมัติคุณภาพของคอมพิวเตอร์ ก่อนที่จะชำระเงินให้กับองค์กรของคุณ นโยบายขององค์กรของคุณคือ การระงับการชำระเงินให้แก่ผู้จัดจำหน่ายจนกว่าลูกค้าจะชำระเงินให้แก่คุณ ดังนั้น คุณจึงตั้งค่าโครงการเพื่อให้มีเปอร์เซ็นต์ PWP เป็น 100 เปอร์เซ็นต์

ผู้จัดจำหน่ายส่งคอมพิวเตอร์ 100 เครื่องที่มีการติดตั้งซอฟต์แวร์ให้คุณ พร้อมกับใบแจ้งหนี้ 10,000.00 USD เนื่องจากมีการตั้งค่าเงื่อนไข PWP สำหรับโครงการของคุณ คุณไม่ต้องชำระเงินให้แก่ผู้จัดจำหน่ายเมื่อมีการรับคอมพิวเตอร์ แต่คุณจะส่งคอมพิวเตอร์ไปยังลูกค้า พร้อมกับใบแจ้งหนี้ 15,000.00 แทน ลูกค้าตรวจสอบคอมพิวเตอร์และอนุมัติการชำระเงินเต็มในใบแจ้งหนี้โครงการ

หลังจากที่คุณได้รับการชำระเงินเต็มจำนวนจากลูกค้า คุณจะชำระเงินให้ผู้จัดจำหน่าย 10,000.00 ซึ่งเป็นการชำระเงินเต็มจำนวนใบแจ้งหนี้ของผู้จัดจำหน่าย

## <a name="set-up-pwp-terms-for-a-project"></a>ตั้งค่าเงื่อนไข PWP สำหรับโครงการ

เมื่อคุณตั้งค่าเงื่อนไข PWP สำหรับโครงการ คุณต้องระบุ เป็นเปอร์เซ็นต์ ยอดเงินต่ำสุดที่ลูกค้าต้องชำระเงินให้แก่คุณสำหรับโครงการ ก่อนที่คุณจะชำระเงินให้แก่ผู้จัดจำหน่าย จำนวนเงินขีดจำกัดจะถูกคำนวณโดยอัตโนมัติบนใบแจ้งหนี้ของลูกค้าสำหรับโครงการ ให้ทำตามขั้นตอนเหล่านี้เพื่อตั้งค่าเปอร์เซ็นต์ขีดจำกัดสำหรับเงื่อนไข PWP

1. ไปที่ **การจัดการและการบัญชีโครงการ** \> **โครงการ** \> **โครงการทั้งหมด**
2. ค้นหาและเปิดโครงการที่คุณต้องการตั้งค่าเงื่อนไข PWP
3. ใน FastTab **ข้อตกลงของผู้จัดจำหน่าย** ให้เลือก **เพิ่มรายการ**
3. ในฟิลด์ **รหัสบัญชี** ให้เลือกตัวเลือกใดตัวเลือกหนึ่งดังต่อไปนี้

    - **ตาราง** – เงื่อนไข PWP นำไปใช้กับผู้จัดจำหน่ายเดียว
    - **กลุ่ม** – เงื่อนไข PWP นำไปใช้กับผู้จัดจำหน่ายทั้งหมดในกลุ่มผู้จัดจำหน่าย
    - **ทั้งหมด** – เงื่อนไข PWP นำไปใช้กับผู้จัดจำหน่ายทั้งหมด

4. ถ้าคุณเลือก **ตาราง** หรือ **กลุ่ม** ในขั้นตอนก่อนหน้า ในฟิลด์ **ผู้จัดจำหน่าย/กลุ่มผู้จัดจำหน่าย** ให้เลือกผู้จัดจำหน่ายหรือกลุ่มผู้จัดจำหน่ายที่ต้องใช้เงื่อนไข PWP ถ้าคุณเลือก **ทั้งหมด** ในขั้นตอนก่อนหน้านี้ ฟิลด์ **ผู้จัดจำหน่าย/กลุ่มผู้จัดจำหน่าย** จะไม่สามารถแก้ไขได้
5. ถ้ามีการตั้งค่าเงื่อนไขเงินวางประกันของผู้จัดจำหน่ายสำหรับผู้จัดจำหน่ายในโครงการ ในฟิลด์ **เงื่อนไขเงินวางประกันของผู้จัดจำหน่าย** ให้เลือกรหัสกฎสำหรับเงื่อนไขเงินวางประกัน
6. ในฟิลด์ **เปอร์เซ็นต์ขีดจำกัด PWP** ให้ป้อนเปอร์เซ็นต์ขีดจำกัดสำหรับโครงการ เปอร์เซ็นต์ที่คุณป้อนสำหรับโครงการจะกำหนดยอดเงินต่ำสุดที่ลูกค้าต้องชำระให้แก่คุณ ก่อนที่คุณจะชำระเงินให้ผู้จัดจำหน่าย

## <a name="create-a-po-that-has-pwp-terms"></a>สร้าง PO ที่มีเงื่อนไข PWP

เมื่อคุณลงรายการบัญชีใบแจ้งหนี้จากผู้จัดจำหน่าย ถ้าผู้จัดจำหน่ายอยู่ภายใต้เงื่อนไข PWP เงื่อนไขเหล่านั้นจะแสดงอยู่ในรายการ PO เมื่อต้องการสร้าง PO ที่มีเงื่อนไข PWP ให้ทำตามขั้นตอนต่อไปนี้

1. ไปที่ **การจัดซื้อและการจัดหา** \> **ใบสั่งซื้อ** \> **ใบสั่งซื้อทั้งหมด**
2. บนบานหน้าต่างการดำเนินการ เลือก **สร้าง** จากนั้น ในกล่องโต้ตอบ **สร้างใบสั่งซื้อ** ให้ป้อนข้อมูลที่จำเป็น และเลือก **ตกลง**

    อีกทางหนึ่งคือ เปิด PO ที่มีอยู่บนหน้ารายการ **ใบสั่งซื้อทั้งหมด**

4. บนหน้า **ใบสั่งซื้อ** บน FastTab **รายการใบสั่งซื้อ** ให้ทบทวนรายละเอียดของรายการ PO สำหรับผู้จัดจำหน่าย ตัวเลือก **จ่ายเมื่อได้รับชำระเงิน** ถูกเลือกโดยอัตโนมัติ และค่าในฟิลด์ **เปอร์เซ็นต์ขีดจำกัด PWP** ถูกคัดลอกโดยอัตโนมัติจากฟิลด์ **เปอร์เซ็นต์ขีดจำกัด PWP** บนหน้า **โครงการ**
6. ถ้าคุณไม่ต้องการใช้เงื่อนไข PWP กับผู้จัดจำหน่ายสำหรับรายการ PO ให้ยกเลิกการเลือกตัวเลือก **จ่ายเมื่อได้รับชำระเงิน** ในกรณีนี้ ฟิลด์ **เปอร์เซ็นต์ขีดจำกัด PWP** สำหรับรายการ PO จะถูกรีเซ็ตเป็น 0 (ศูนย์)

## <a name="update-a-customer-payment-and-pay-the-vendor"></a>ปรับปรุงการชำระเงินของลูกค้า และชำระเงินให้กับผู้จัดจำหน่าย

เมื่อผู้จัดจำหน่ายทำงานในโครงการเสร็จสมบูรณ์และส่งใบแจ้งหนี้แล้ว คุณต้องตรวจทานสถานะโครงการและใบแจ้งหนี้ของลูกค้าเพื่อกำหนดว่าเงื่อนไข PWP เป็นไปตามโครงการหรือไม่ ถ้าตรงตามเงื่อนไข PWP สำหรับผู้จัดจำหน่าย คุณสามารถกำหนดรายการที่อยู่ในใบแจ้งหนี้ของผู้จัดจำหน่ายในการชำระเงิน โดยยึดตามการชำระเงินของลูกค้าสำหรับโครงการ ถ้าคุณตัดสินใจที่จะชำระเงินให้แก่ผู้จัดจำหน่าย ถึงแม้ว่าจะไม่เป็นไปตามเงื่อนไข PWP คุณสามารถแทนที่เงื่อนไข PWP บนหน้า **ใบแจ้งหนี้ของผู้จัดจำหน่ายที่มีการจ่ายเมื่อได้รับชำระเงิน**

1. ไปที่ **การจัดการและการบัญชีโครงการ** \> **การสอบถามและรายงาน** \> **การสอบถามเงินวางประกัน** \> **ใบแจ้งหนี้ของผู้จัดจำหน่ายที่มีการจ่ายเมื่อได้รับชำระเงิน**
2. บนหน้า **ใบแจ้งหนี้ของผู้จัดจำหน่ายที่มีการจ่ายเมื่อได้รับชำระเงิน** ในฟิลด์การค้นหา ให้ป้อนค่าเพื่อค้นหาใบแจ้งหนี้ของผู้จัดจำหน่ายที่คุณต้องการตรวจทาน แล้วเลือก **ค้นหา**
3. บน FastTab **รายการใบแจ้งหนี้ของผู้จัดจำหน่าย** ให้เลือกรายการที่คุณต้องการเปลี่ยนแปลง
4. ถ้าเป็นไปตามเงื่อนไข **จ่ายเมื่อได้รับชำระเงิน** สำหรับรายการใบแจ้งหนี้ ให้เลือก **นำออกใช้การชำระเงินให้แก่ผู้จัดจำหน่าย** เมื่อมีการยกเลิกการเลือกตัวเลือก **จ่ายเมื่อได้รับชำระเงิน** และค่าของฟิลด์ **พร้อมสำหรับการชำระเงิน** ถูกเปลี่ยนเป็น **ใช่**
