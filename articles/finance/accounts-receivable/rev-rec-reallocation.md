# <a name="revenue-recognition-reallocation"></a><span data-ttu-id="596a8-104">การปันส่วนใหม่ของการรับรู้รายได้</span><span class="sxs-lookup"><span data-stu-id="596a8-104">Revenue recognition reallocation</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="596a8-105">การปันส่วนใหม่ช่วยให้องค์กรสามารถคํานวณราคาสำหรับรายได้ใหม่เมื่อมีการเปลี่ยนแปลงเงื่อนไขการขายตามสัญญา</span><span class="sxs-lookup"><span data-stu-id="596a8-105">Reallocation enables organizations to recalculate revenue prices when the terms of a contractual sale are changed.</span></span> <span data-ttu-id="596a8-106">สำหรับวัตถุประสงค์การรับรู้รายได้ เอกสารใบสั่งขายจะถือเป็นสัญญา</span><span class="sxs-lookup"><span data-stu-id="596a8-106">For the purpose of revenue recognition, the sales order documents are considered the contract.</span></span>

<span data-ttu-id="596a8-107">องค์กรของคุณต้องกําหนดเองว่าจําเป็นต้องมีการปันส่วนใหม่หรือไม่</span><span class="sxs-lookup"><span data-stu-id="596a8-107">Your organization must determine for itself whether reallocation is required.</span></span> <span data-ttu-id="596a8-108">การเพิ่มรายการใหม่ลงในใบสั่งขายหรือการเพิ่มใบสั่งขายใหม่สําหรับลูกค้าอาจไม่ถือเป็นการเปลี่ยนแปลงสัญญา</span><span class="sxs-lookup"><span data-stu-id="596a8-108">The addition of a new line to a sales order, or the addition of a new sales order for a customer, might not constitute a change to the contract.</span></span> <span data-ttu-id="596a8-109">สถานการณ์ที่อาจต้องปันส่วนใหม่มีดังต่อไปนี้:</span><span class="sxs-lookup"><span data-stu-id="596a8-109">The following scenarios might require reallocation:</span></span>

- <span data-ttu-id="596a8-110">ลูกค้าเพิ่มสินค้าในใบสั่งขายหรือลบสินค้าออกจากใบสั่งขายหลังจากออกใบแจ้งหนี้สำหรับใบสั่งครบทั้งหมดหรือบางส่วนแล้ว</span><span class="sxs-lookup"><span data-stu-id="596a8-110">A customer added items to a sales order, or removed items from the sales order, after the order was fully or partially invoiced.</span></span>
- <span data-ttu-id="596a8-111">มีการป้อนใบสั่งขายหลายใบในสถานะยืนยันแล้วหรือสถานะออกใบแจ้งหนี้แล้วสําหรับสัญญาที่มีการเจรจาฉบับเดียวกัน</span><span class="sxs-lookup"><span data-stu-id="596a8-111">Multiple sales orders, in either a confirmed state or an invoiced state, were entered for the same negotiated contract.</span></span>
- <span data-ttu-id="596a8-112">ลูกค้าขอคืนหรือได้รับเครดิตสําหรับสินค้าหลังจากที่ใบสั่งขายเดิมมีการออกใบแจ้งหนี้ครบทั้งหมดหรือบางส่วนแล้ว</span><span class="sxs-lookup"><span data-stu-id="596a8-112">A customer returned or received a credit for an item after the original sales order was fully or partially invoiced.</span></span>

<span data-ttu-id="596a8-113">มีข้อจํากัดที่สําคัญบางประการเกี่ยวกับกระบวนการปันส่วนใหม่:</span><span class="sxs-lookup"><span data-stu-id="596a8-113">There are a few important limitations on the reallocation process:</span></span>

- <span data-ttu-id="596a8-114">กระบวนการนี้สามารถดำเนินการได้เพียงครั้งเดียว</span><span class="sxs-lookup"><span data-stu-id="596a8-114">The process can be run only one time.</span></span> <span data-ttu-id="596a8-115">ดังนั้นจึงเป็นเรื่องสําคัญที่คุณต้องดำเนินการหลังจากการเปลี่ยนแปลงทั้งหมดสิ้นสุดแล้วเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="596a8-115">Therefore, it's important that you run it only after all changes are finalized.</span></span>
- <span data-ttu-id="596a8-116">ไม่สามารถดำเนินกระบวนการกับใบสั่งขายโครงการได้</span><span class="sxs-lookup"><span data-stu-id="596a8-116">The process can't be run on project sales orders.</span></span>
- <span data-ttu-id="596a8-117">ถ้ามีใบสั่งขายหลายใบที่เกี่ยวข้อง ใบสั่งดังกล่าวต้องเป็นของบัญชีลูกค้ารายเดียวกัน</span><span class="sxs-lookup"><span data-stu-id="596a8-117">If multiple sales orders are involved, they must be for the same customer account.</span></span>
- <span data-ttu-id="596a8-118">ใบสั่งขายทั้งหมดที่มีการปันส่วนใหม่ต้องอยู่ในสกุลเงินธุรกรรมเดียวกัน</span><span class="sxs-lookup"><span data-stu-id="596a8-118">All sales orders that are reallocated must be in the same transaction currency.</span></span>
- <span data-ttu-id="596a8-119">กระบวนการดังกล่าวไม่สามารถย้อนกลับหรือยกเลิกได้หลังจากที่ดำเนินการแล้ว</span><span class="sxs-lookup"><span data-stu-id="596a8-119">The process can't be reversed or undone after it's run.</span></span>

## <a name="set-up-reallocation"></a><span data-ttu-id="596a8-120">ตั้งค่าการปันส่วนใหม่</span><span class="sxs-lookup"><span data-stu-id="596a8-120">Set up reallocation</span></span>

<span data-ttu-id="596a8-121">พารามิเตอร์หนึ่งจะมีผลต่อกระบวนการปันส่วนใหม่</span><span class="sxs-lookup"><span data-stu-id="596a8-121">One parameter affects the reallocation process.</span></span>

<span data-ttu-id="596a8-122">เนื่องจากการปันส่วนใหม่สามารถทําได้ในใบสั่งขายที่มีการออกใบแจ้งหนี้บางส่วนหรือครบทั้งหมดแล้ว รายการบัญชีก่อนหน้าสำหรับใบแจ้งหนี้จึงต้องแก้ไขโดยการใช้ราคาสำหรับรายได้ที่ปันส่วนใหม่แล้วของใหม่</span><span class="sxs-lookup"><span data-stu-id="596a8-122">Because reallocation can be done on a sales order that is partially or fully invoiced, any previous accounting entries for the invoice must be corrected by using the new, reallocated revenue prices.</span></span> <span data-ttu-id="596a8-123">การแก้ไขนี้จะทําได้โดยการย้อนกลับรายการบัญชีของใบแจ้งหนี้เดิมและการลงรายการบัญชีสำหรับรายการบัญชีใหม่ที่อิงอยู่กับราคาสำหรับรายได้ที่ปันส่วนใหม่</span><span class="sxs-lookup"><span data-stu-id="596a8-123">This correction is done by reversing the original invoice's accounting entry and posting a new accounting entry that is based on the reallocated revenue prices.</span></span>

<span data-ttu-id="596a8-124">ทุกองค์กรต้องตัดสินใจว่าการแก้ไขควรอัปเดตเฉพาะบัญชีแยกประเภททั่วไปเท่านั้นหรือควรอัปเดตบัญชีลูกหนี้ด้วย</span><span class="sxs-lookup"><span data-stu-id="596a8-124">Every organization must decide whether the correction should update only General ledger, or whether it should also update Accounts receivable.</span></span> <span data-ttu-id="596a8-125">เมื่อมีการตัดสินใจจะกําหนดการตั้งค่าที่เหมาะสมของตัวเลือก **ลงรายการบัญชีการแก้ไขใบแจ้งหนี้ในบัญชีลูกหนี้** บนแท็บ **การรับรู้รายได้** ของหน้า **พารามิเตอร์บัญชีแยกประเภททั่วไป** (**การรับรู้รายได้ \> การตั้งค่า \> พารามิเตอร์บัญชีแยกประเภททั่วไป**)</span><span class="sxs-lookup"><span data-stu-id="596a8-125">The decision that is reached determines the appropriate setting of the **Post invoice corrections to Accounts receivable** option on the **Revenue recognition** tab of the **General ledger parameters** page (**Revenue recognition \> Setup \> General ledger parameters**).</span></span> <span data-ttu-id="596a8-126">การตั้งค่าที่เหมาะสมจะขึ้นอยู่กับสถานการณ์เฉพาะ</span><span class="sxs-lookup"><span data-stu-id="596a8-126">The appropriate setting depends on the specific scenario.</span></span> <span data-ttu-id="596a8-127">สําหรับข้อมูลเพิ่มเติมเกี่ยวกับสถานการณ์ที่เป็นไปได้ ให้ใช้ลิงก์ในส่วน [สถานการณ์สําหรับการปันส่วนใหม่](#scenarios-for-reallocation) ที่จะกล่าวถึงต่อไปในหัวข้อนี้</span><span class="sxs-lookup"><span data-stu-id="596a8-127">For more information about possible scenarios, use the links in the [Scenarios for reallocation](#scenarios-for-reallocation) section later in this topic.</span></span>

<span data-ttu-id="596a8-128">[![แท็บการรับรู้รายได้ในหน้าพารามิเตอร์บัญชีแยกประเภททั่วไป](./media/01_RevRecScenarios.png)](./media/01_RevRecScenarios.png)</span><span class="sxs-lookup"><span data-stu-id="596a8-128">[![Revenue recognition tab on the General ledger parameters page](./media/01_RevRecScenarios.png)](./media/01_RevRecScenarios.png)</span></span>

<span data-ttu-id="596a8-129">ถ้าตัวเลือก **ลงรายการบัญชีการแก้ไขใบแจ้งหนี้ในบัญชีลูกหนี้** ตั้งค่าเป็น **ใช่** กระบวนการปันส่วนใหม่จะสร้างผลลัพธ์ต่อไปนี้:</span><span class="sxs-lookup"><span data-stu-id="596a8-129">If the **Post invoice corrections to Accounts receivable** option is set to **Yes**, the reallocation process produces the following result:</span></span>

- <span data-ttu-id="596a8-130">ระบบจะสร้างเอกสารเครดิตขึ้นในบัญชีลูกหนี้เพื่อกลับรายการใบแจ้งหนี้ที่ต้องมีการแก้ไข</span><span class="sxs-lookup"><span data-stu-id="596a8-130">A credit document is created in Accounts receivable to reverse the invoice that requires correction.</span></span>

    - <span data-ttu-id="596a8-131">เอกสารเครดิตจะนําหมายเลขใบแจ้งหนี้เดิมมาใช้ใหม่ แต่จะมีการผนวก "-1" เข้ากับหมายเลขนั้น</span><span class="sxs-lookup"><span data-stu-id="596a8-131">The credit document reuses the original invoice number, but "-1" is appended to it.</span></span>
    - <span data-ttu-id="596a8-132">เอกสารเครดิตจะมีการชำระเงินโดยอัตโนมัติกับใบแจ้งหนี้เดิม</span><span class="sxs-lookup"><span data-stu-id="596a8-132">The credit document is automatically settled against the original invoice.</span></span> <span data-ttu-id="596a8-133">ถ้ามีการชําระเงินใบแจ้งหนี้เดิมด้วยเอกสารเครดิตหรือการชําระเงินอื่นแล้ว การชําระเงินนั้นจะได้รับการกลับรายการโดยอัตโนมัติ</span><span class="sxs-lookup"><span data-stu-id="596a8-133">If the original invoice was already settled with another credit document or payment, that settlement is automatically reversed.</span></span>
    - <span data-ttu-id="596a8-134">โดยมีการลงรายการบัญชีเอกสารเครดิตในบัญชีแยกประเภททั่วไปเพื่อกลับรายการบัญชีที่ลงรายการบัญชีในใบแจ้งหนี้เดิม</span><span class="sxs-lookup"><span data-stu-id="596a8-134">The credit document is posted to General ledger to reverse the accounting entry that was posted on the original invoice.</span></span> <span data-ttu-id="596a8-135">อย่างไรก็ตาม รายการธุรกรรมสินค้าคงคลังและต้นทุนขาย (COGS) ไม่ได้กลับรายการ</span><span class="sxs-lookup"><span data-stu-id="596a8-135">However, the Inventory and Cost of goods sold (COGS) transaction entries aren't reversed.</span></span>

- <span data-ttu-id="596a8-136">ใบแจ้งหนี้ใหม่ที่อิงตามจํานวนเงินของราคาใหม่ที่มีการปันส่วนใหม่จะมีการสร้างขึ้นในบัญชีลูกหนี้</span><span class="sxs-lookup"><span data-stu-id="596a8-136">A new invoice that is based on the new, reallocated price amounts is created in Accounts receivable.</span></span>

    - <span data-ttu-id="596a8-137">ใบแจ้งหนี้ใหม่จะนําหมายเลขใบแจ้งหนี้เดิมมาใช้ใหม่ แต่จะมีการผนวก "-2" เข้ากับหมายเลขนั้น</span><span class="sxs-lookup"><span data-stu-id="596a8-137">The new invoice reuses the original invoice number, but "-2" is appended to it.</span></span>
    - <span data-ttu-id="596a8-138">ใบแจ้งหนี้ใหม่จะมีการชำระเงินโดยอัตโนมัติตามเอกสารเครดิตหรือการชำระเงินใดๆ ที่ชำระไว้ก่อนหน้านี้กับใบแจ้งหนี้เดิม</span><span class="sxs-lookup"><span data-stu-id="596a8-138">The new invoice is automatically settled against any credit document or payments that were previously settled with the original invoice.</span></span>
    - <span data-ttu-id="596a8-139">ใบแจ้งหนี้ใหม่มีการลงรายการบัญชีไปยังบัญชีแยกประเภททั่วไปโดยใช้จำนวนเงินของราคาสำหรับรายได้ใหม่ที่มีการปันส่วนใหม่</span><span class="sxs-lookup"><span data-stu-id="596a8-139">The new invoice is posted to General ledger by using the new, reallocated revenue price amounts.</span></span> <span data-ttu-id="596a8-140">การดำเนินการดังกล่าวไม่ได้ลงรายการบัญชีที่บัญชีสินค้าคงคลังและ COGS อีกครั้ง เนื่องจากรายการเหล่านั้นได้รับการคงไว้ในรายการบัญชีของใบแจ้งหนี้เดิม</span><span class="sxs-lookup"><span data-stu-id="596a8-140">It isn't posted to the Inventory and COGS accounts again, because those entries are maintained on the original invoice's accounting entry.</span></span>

<span data-ttu-id="596a8-141">ถ้าตัวเลือก **ลงรายการบัญชีการแก้ไขใบแจ้งหนี้ในบัญชีลูกหนี้** ตั้งค่าเป็น **ไม่** กระบวนการปันส่วนใหม่จะสร้างผลลัพธ์ต่อไปนี้:</span><span class="sxs-lookup"><span data-stu-id="596a8-141">If the **Post invoice corrections to Accounts receivable** option is set to **No**, the reallocation process produces the following result:</span></span>

- <span data-ttu-id="596a8-142">โดยมีการลงรายการบัญชีสำหรับรายการบัญชีที่กลับรายการไปยังบัญชีแยกประเภททั่วไปเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="596a8-142">A reversing accounting entry is posted only to General ledger.</span></span> <span data-ttu-id="596a8-143">มีการกลับรายการบัญชีทั้งหมดจากใบแจ้งหนี้เดิม ยกเว้นรายการบัญชีสินค้าคงคลังและ COGS</span><span class="sxs-lookup"><span data-stu-id="596a8-143">All the accounting from the original invoice is reversed, except the Inventory and COGS account entries.</span></span>
