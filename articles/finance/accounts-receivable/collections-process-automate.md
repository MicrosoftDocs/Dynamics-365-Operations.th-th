# <a name="collections-process-automation"></a>กระบวนการเรียกเก็บเงินระบบอัตโนมัติ

[!include [banner](../includes/banner.md)]

หัวข้อนี้จะอธิบายกระบวนการตั้งค่ากลยุทธ์กระบวนการเรียกเก็บเงินที่ระบุใบแจ้งหนี้ของลูกค้าที่จำเป็นต้องมีการแจ้งเตือนทางอีเมล กิจกรรมการเรียกเก็บเงิน (เช่น การติดต่อทางโทรศัพท์) หรือจดหมายเรียกเก็บเงินที่จะส่งให้แก่ลูกค้าโดยอัตโนมัติ 

องค์กรมักใช้เวลาอย่างมากในการค้นหารายงานยอดคงเหลือตามอายุ บัญชีลูกค้า และใบแจ้งหนี้ที่เปิดค้างไว้เพื่อเรียนรู้ว่าต้องติดต่อลูกค้ารายใดเกี่ยวกับใบแจ้งหนี้หรือยอดคงเหลือในบัญชี การศึกษานี้ใช้เวลาห่างจากเวลาของตัวแทนเรียกเก็บเงินที่ใช้ในการสื่อสารกับลูกค้าในการเรียกเก็บยอดดุลที่เลยกำหนดชำระหรือแก้ไขข้อโต้แย้งเกี่ยวกับใบแจ้งหนี้ กระบวนการเรียกเก็บเงินระบบอัตโนมัติช่วยให้คุณสามารถตั้งค่าวิธีการที่ยึดตามกลยุทธ์สำหรับกระบวนการเรียกเก็บเงินของคุณได้ การดำเนินการนี้จะช่วยให้คุณสามารถใช้กิจกรรมการเรียกเก็บเงินได้อย่างสม่ำเสมอโดยการจัดทำการแจ้งเตือนทางอีเมลที่กำหนดเอง หรือกระบวนการที่โปรแกรมสำหรับการส่งจดหมายการเรียกเก็บเงิน 

## <a name="collections-process-setup"></a>การตั้งค่ากระบวนการเรียกเก็บเงิน
คุณสามารถใช้หน้า **การตั้งค่ากระบวนการเรียกเก็บเงิน** (**สินเชื่อและการเรียกเก็บเงิน > การตั้งค่า > การตั้งค่ากระบวนการเรียกเก็บเงิน**) เพื่อสร้างกระบวนการเรียกเก็บเงินอัตโนมัติซึ่งจะจัดกำหนดการกิจกรรม ส่งข้อความอีเมล และสร้างและลงรายการบัญชีจดหมายเรียกเก็บเงินของลูกค้า ขั้นตอนของกระบวนการจะขึ้นอยู่กับใบแจ้งหนี้ที่เปิดค้างไว้ใหม่สุดหรือเก่าที่สุด แต่ละขั้นตอนจะใช้ใบแจ้งหนี้นี้เพื่อกำหนดว่าการสื่อสารหรือกิจกรรมใดควรจะเกิดขึ้นกับลูกค้าที่ระบุ  

โดยทั่วไปทีมงานเรียกเก็บเงินจะส่งประกาศล่วงหน้าที่เกี่ยวข้องกับใบแจ้งหนี้คงค้างแต่ละใบ เพื่อให้ลูกค้าได้รับการแจ้งเตือนเมื่อใบแจ้งหนี้ใกล้ครบกําหนด สามารถตั้งค่าการเลือก **การเตือนก่อนครบกำหนดชำระ** เพื่ออนุญาตให้ดำเนินการขั้นตอนหนึ่งในแต่ละลำดับชั้นของกระบวนการสำหรับใบแจ้งหนี้ทุกใบเมื่อถึงเวลาในใบแจ้งหนี้

### <a name="process-hierarchy"></a>ลำดับชั้นของกระบวนการ
กลุ่มลูกค้าแต่ละกลุ่มสามารถกำหนดให้กับลำดับชั้นกระบวนการเดียวเท่านั้น ลำดับในลำดับชั้นของขั้นตอนนี้จะระบุว่ากระบวนการใดมีความสำคัญเมื่อมีการรวมลูกค้าไว้ในกลุ่มมากกว่าหนึ่งกลุ่มที่มีการกำหนดลำดับชั้นของกระบวนการ รหัสกลุ่มจะกำหนดลูกค้าที่จะกำหนดให้กับกระบวนการ แต่ละลำดับชั้นที่ตั้งค่าสามารถถูกกำหนดไปยังหนึ่งกระบวนการระบบอัตโนมัติเท่านั้น

วันที่ไม่มีการทำงานใช้เพื่อให้มั่นใจว่าลูกค้าไม่ได้รับการติดต่อบ่อยเกินไปโดยกระบวนการอัตโนมัติ ตัวอย่างเช่น ถ้าวันที่ไม่มีการทำงานมีการตั้งค่าเป็นสอง ลูกค้าจะไม่ได้รับการติดต่อโดยกระบวนการอัตโนมัติเป็นเวลาอย่างน้อยสองวัน ถึงแม้ว่าจะมีการชำระใบแจ้งหนี้ใหม่สุดของเดิมไว้เต็มจำนวน 

เมื่อต้องการแยกลูกค้าออกจากระบบอัตโนมัติของกระบวนการหากยอดดุลอายุหนี้หรือยอดเงินในใบแจ้งหนี้ของลูกค้าน้อยกว่าค่าที่กําหนด ให้เลือก **ยอดดุลอายุหนี้ของลูกค้าที่น้อยกว่า** หรือ **ยอดเงินในใบแจ้งหนี้ที่น้อยกว่า** ในฟิลด์ **ไม่รวมไว้ระหว่างกระบวนการ** และป้อนค่ายอดเงิน

เลือก **ใช้การคาดการณ์** เพื่อสร้างกิจกรรมการเรียกเก็บเงินโดยใช้การคาดการณ์การชำระเงินของลูกค้า กิจกรรมที่สร้างขึ้นจะใช้แม่แบบกิจกรรมจาก **การคาดการณ์การชำระเงิน** ในหน้า **พารามิเตอร์บัญชีลูกหนี้** แท็บ **กระบวนการเรียกเก็บเงินระบบอัตโนมัติ** 

### <a name="process-details"></a>รายละเอียดกระบวนการ
คลิก **สร้าง** เพื่อเพิ่มรายละเอียดกระบวนการใหม่ให้กับลำดับชั้น **คำอธิบาย** ใช้เพื่อระบุวัตถุประสงค์หรือชื่อของขั้นตอนในลำดับชั้น เลือก **ชนิดของการดำเนินการ** เพื่อกำหนดว่าขั้นตอนนี้จะสร้างกิจกรรม ส่งอีเมล หรือสร้างจดหมายเรียกเก็บเงิน 

- **เอกสารธุรกิจ** กำหนดแม่แบบที่ใช้ในการสร้างชนิดของการดำเนินการ เอกสารนี้อาจเป็นแม่แบบกิจกรรม แม่แบบอีเมล หรือจดหมายเรียกเก็บเงินที่ส่งให้สำหรับลูกค้าแต่ละราย กระบวนการเรียกเก็บเงินระบบอัตโนมัติจะสร้างจดหมายเรียกเก็บเงินต่อลูกค้าเท่านั้น ไม่ว่าพารามิเตอร์การเรียกเก็บเงินอื่น ๆ จะตั้งค่าไว้อย่างไร
- **เมื่อ** กําหนดขั้นตอนกระบวนการที่จะเกิดขึ้นก่อนหรือหลังวันที่ครบกําหนดของใบแจ้งหนี้ (เก่าที่สุด) และใช้ร่วมกับตัวเลขที่แสดงอยู่ในคอลัมน์ **วันที่สัมพันธ์กับวันที่ครบกําหนดของใบแจ้งหนี้** 
- เลือกตัวเลือก **ก่อนการติดตามหนี้** เพื่อสร้างการดำเนินการให้กับทุกใบแจ้งหนี้ในขั้นตอนเดียวในลำดับชั้นของกระบวนการ การดำเนินการก่อนการติดตามหนี้คือประกาศล่วงหน้าที่เกี่ยวข้องกับใบแจ้งหนี้คงค้าง เพื่อให้ลูกค้าได้รับการแจ้งเตือนเมื่อใบแจ้งหนี้ใกล้ครบกําหนด การเตือนก่อนครบกำหนดชำระสามารถเลือกไว้ได้เพียงหนึ่งกิจกรรมต่อหนึ่งลำดับชั้นเท่านั้น เมื่อคุณเลือกชนิดการดำเนินการอีเมล ผู้รับจะใช้เพื่อกำหนดว่าข้อความอีเมลจะส่งไปยังผู้ติดต่อของลูกค้า กลุ่มการขาย หรือตัวแทนเรียกเก็บเงิน 
- ค่าในฟิลด์ **ผู้ติดต่อสำหรับวัตถุประสงค์ทางธุรกิจ** จะกำหนดผู้ติดต่อที่จากบัญชีของลูกค้าที่จะได้รับการติดต่อสื่อสาร

### <a name="business-document-details"></a>รายละเอียดเอกสารทางธุรกิจ
รายละเอียดเอกสารทางธุรกิจจะแตกต่างกันไปตามชนิดของการดำเนินการที่เลือกไว้ในรายละเอียดกระบวนการ เมื่อชนิดการดำเนินการเป็นกิจกรรม รายละเอียดเท็มเพลตกิจกรรมจะแสดงขึ้น รายละเอียดเหล่านี้รวมถึงชื่อเท็มเพลตกิจกรรม ชนิดของกิจกรรมที่จะถูกสร้าง วัตถุประสงค์ของกิจกรรม จำนวนวันที่จัดกำหนดการเพื่อดำเนินกิจกรรมให้เสร็จสมบูรณ์ และรายละเอียดของกิจกรรม กิจกรรมนี้จะเชื่อมโยงกับใบแจ้งหนี้หลักที่จะบอกผู้รับของการดำเนินการว่าจำเป็นต้องทำให้การดำเนินกิจกรรมเสร็จสมบูรณ์

ถ้าชนิดของการดำเนินการเป็นอีเมลในรายละเอียดกระบวนการ ส่วนนี้จะมีแท็บด่วนสองแท็บ แท็บแรกใช้ในการกำหนดรหัสเท็มเพลต คำอธิบายอีเมล ภาษาเริ่มต้น ชื่อผู้ใช้ที่จะกำหนดให้กับข้อความอีเมลที่ส่งโดยอัตโนมัติ และที่อยู่อีเมลของผู้ส่งที่เกี่ยวข้อง แท็บที่สองจะช่วยในการสร้างเนื้อหาของอีเมลหลังจากที่มีการบันทึกค่าในฟิลด์ **ภาษา** และ **ชื่อเรื่อง** โดยการเลือก **แก้ไข** การดำเนินการนี้จะเปิดหน้าต่างที่อนุญาตให้อัปโหลดเนื้อหา HTML 

> [!Note]
> คุณสามารถบันทึกข้อความอีเมล Outlook ที่มีข้อความเนื้อหาของการสื่อสารในรูปแบบ HTML จากนั้น คุณสามารถอัปโหลดเนื้อหาข้อความเพื่อใช้แม่แบบได้ <br> <br> ถ้ามีการเลือกชนิดการดำเนินการของจดหมายเรียกเก็บเงิน จะไม่มีส่วนรายละเอียดของเอกสารทางธุรกิจบนหน้าการตั้งค่า

ใช้ปุ่ม **ประวัติกระบวนการเรียกเก็บเงิน** เพื่อดูประวัติล่าสุดของลำดับชั้นของกระบวนการที่เลือก 

คลิกการดำเนินการ **การกำหนดกระบวนการเรียกเก็บเงิน** เพื่อดูลูกค้าที่กำหนดให้กับกระบวนการเรียกเก็บเงิน ใช้ **แสดงตัวอย่างการกำหนดลูกค้า** เพื่อดูลำดับชั้นที่กำหนดลูกค้าที่ระบุ ใช้ **แสดงตัวอย่างการกำหนดกระบวนการ** เพื่อแสดงตัวอย่างเพื่อดูตัวอย่างลูกค้าที่จะถูกกำหนดให้กับลำดับชั้นเมื่อรัน คลิด **การกำหนดด้วยตนเอง** เพื่อดูลูกค้าที่ได้รับการกำหนดให้กับกระบวนการด้วยตนเองหรือเลือกลูกค้าที่จะกำหนดให้กับกระบวนการ

คลิกการดำเนินการ **การจำลองกระบวนการ** เพื่อดูตัวอย่างการดำเนินการที่จะสร้างขึ้นหากเรียกใช้กระบวนการอัตโนมัติที่เลือกในเวลานี้
