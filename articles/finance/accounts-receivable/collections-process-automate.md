# <a name="collections-process-automation"></a>กระบวนการเรียกเก็บเงินระบบอัตโนมัติ

[!include [banner](../includes/banner.md)]

หัวข้อนี้จะอธิบายกระบวนการตั้งค่ากลยุทธ์กระบวนการเรียกเก็บเงินที่ระบุใบแจ้งหนี้ของลูกค้าที่จำเป็นต้องมีการแจ้งเตือนทางอีเมล กิจกรรมการเรียกเก็บเงิน (เช่น การติดต่อทางโทรศัพท์) หรือจดหมายเรียกเก็บเงินที่จะส่งให้แก่ลูกค้าโดยอัตโนมัติ 

องค์กรใช้เวลาอย่างมากในการค้นหารายงานยอดคงเหลือตามอายุ บัญชีลูกค้า และใบแจ้งหนี้ที่เปิดค้างไว้เพื่อเรียนรู้ว่าต้องติดต่อลูกค้ารายใดเกี่ยวกับใบแจ้งหนี้หรือยอดคงเหลือในบัญชี การศึกษานี้ใช้เวลาห่างจากเวลาของตัวแทนเรียกเก็บเงินที่ใช้ในการสื่อสารกับลูกค้าในการเรียกเก็บยอดดุลที่เลยกำหนดชำระหรือแก้ไขข้อโต้แย้งเกี่ยวกับใบแจ้งหนี้ กระบวนการเรียกเก็บเงินระบบอัตโนมัติช่วยให้คุณสามารถตั้งค่าวิธีการที่ยึดตามกลยุทธ์สำหรับกระบวนการเรียกเก็บเงินของคุณได้ การดำเนินการนี้จะช่วยให้คุณสามารถใช้กิจกรรมการเรียกเก็บเงินได้อย่างสม่ำเสมอโดยการจัดทำการแจ้งเตือนทางอีเมลที่กำหนดเอง หรือกระบวนการที่โปรแกรมสำหรับการส่งจดหมายการเรียกเก็บเงิน 

## <a name="collections-process-setup"></a>การตั้งค่ากระบวนการเรียกเก็บเงิน
คุณสามารถใช้หน้า **การตั้งค่ากระบวนการเรียกเก็บเงิน** (**สินเชื่อและการเรียกเก็บเงิน > การตั้งค่า > การตั้งค่ากระบวนการเรียกเก็บเงิน**) เพื่อสร้างกระบวนการเรียกเก็บเงินอัตโนมัติซึ่งจะจัดกำหนดการกิจกรรม ส่งข้อความอีเมล และสร้างและลงรายการบัญชีจดหมายเรียกเก็บเงินของลูกค้า ขั้นตอนของกระบวนการจะขึ้นอยู่กับใบแจ้งหนี้ที่เปิดค้างไว้ใหม่สุดหรือเก่าที่สุด แต่ละขั้นตอนจะใช้ใบแจ้งหนี้นี้เพื่อกำหนดว่าการสื่อสารหรือกิจกรรมใดควรจะเกิดขึ้นกับลูกค้าที่ระบุ  

โดยทั่วไปทีมงานเรียกเก็บเงินจะส่งประกาศล่วงหน้าที่เกี่ยวข้องกับใบแจ้งหนี้คงค้างแต่ละใบ เพื่อให้ลูกค้าได้รับการแจ้งเตือนเมื่อใบแจ้งหนี้ใกล้ครบกําหนด สามารถตั้งค่าการเลือก **การเตือนก่อนครบกำหนดชำระ** เพื่ออนุญาตให้ดำเนินการขั้นตอนหนึ่งในแต่ละลำดับชั้นของกระบวนการสำหรับใบแจ้งหนี้ทุกใบเมื่อถึงเวลาในใบแจ้งหนี้

### <a name="process-hierarchy"></a>ลำดับชั้นของกระบวนการ
กลุ่มลูกค้าแต่ละกลุ่มสามารถกำหนดให้กับลำดับชั้นกระบวนการเดียวเท่านั้น ลำดับในลำดับชั้นของขั้นตอนนี้จะระบุว่ากระบวนการใดมีความสำคัญเมื่อมีการรวมลูกค้าไว้ในกลุ่มมากกว่าหนึ่งกลุ่มที่มีการกำหนดลำดับชั้นของกระบวนการ รหัสกลุ่มจะกำหนดลูกค้าที่จะกำหนดให้กับกระบวนการ 

วันที่ไม่มีการทำงานใช้เพื่อให้มั่นใจว่าลูกค้าไม่ได้รับการติดต่อบ่อยเกินไปโดยกระบวนการอัตโนมัติ  ตัวอย่างเช่น ถ้าวันที่ไม่มีการทำงานมีการตั้งค่าเป็นสอง ลูกค้าจะไม่ได้รับการติดต่อโดยกระบวนการอัตโนมัติเป็นเวลาอย่างน้อยสองวัน ถึงแม้ว่าจะมีการชำระใบแจ้งหนี้ใหม่สุดของเดิมไว้เต็มจำนวน 

ถ้าต้องการแยกลูกค้าออกจากกระบวนการอัตโนมัติถ้ายอดดุลบัญชีหรือยอดดุลในใบแจ้งหนี้น้อยกว่าค่าที่กำหนด ให้ตั้งค่าฟิลด์ **แยกออกจากกระบวนการ** เป็น **ใช่** แล้วป้อนมูลค่าของจำนวนเงิน

## <a name="process-details"></a>รายละเอียดกระบวนการ
**คำอธิบาย** ใช้เพื่อระบุวัตถุประสงค์หรือชื่อของขั้นตอนในลำดับชั้น

**ชนิดของการดำเนินการ** กำหนดว่าขั้นตอนนี้จะสร้างกิจกรรม ส่งอีเมล หรือสร้างจดหมายเรียกเก็บเงิน

**เอกสารธุรกิจ** กำหนดเท็มเพลตที่ใช้ในการสร้างชนิดของการดำเนินการ  ซึ่งอาจเป็นเท็มเพลตกิจกรรม เท็มเพลตอีเมล หรือจดหมายเรียกเก็บเงินสำหรับลูกค้าแต่ละราย 

ชนิดของการดำเนินการสามารถสร้างขึ้นก่อนหรือหลังจากวันที่ครบกำหนดของใบแจ้งหนี้ ตามการตั้งค่าที่แสดงในคอลัมน์ **จำนวนวันที่สัมพันธ์กับวันที่ครบกำหนดของใบแจ้งหนี้**

เมื่อคุณเลือกชนิดการดำเนินการอีเมล ผู้รับจะใช้เพื่อกำหนดว่าเป็นผู้ติดต่อของลูกค้า กลุ่มการขาย หรือตัวแทนเรียกเก็บเงิน ค่าในฟิลด์ **ผู้ติดต่อสำหรับวัตถุประสงค์ทางธุรกิจ** จะกำหนดผู้ติดต่อที่จากบัญชีของลูกค้าที่จะได้รับการติดต่อสื่อสาร

## <a name="business-document-details"></a>รายละเอียดเอกสารทางธุรกิจ
รายละเอียดเอกสารทางธุรกิจจะแตกต่างกันไปตามชนิดของการดำเนินการที่เลือกไว้ในรายละเอียดกระบวนการ  เมื่อชนิดการดำเนินการเป็นกิจกรรม รายละเอียดเท็มเพลตกิจกรรมจะแสดงขึ้น  รายละเอียดเหล่านี้รวมถึงชื่อเท็มเพลตกิจกรรม ชนิดของกิจกรรมที่จะถูกสร้าง วัตถุประสงค์ของกิจกรรม จำนวนวันที่จัดกำหนดการเพื่อดำเนินกิจกรรมให้เสร็จสมบูรณ์ และรายละเอียดของกิจกรรม  กิจกรรมนี้จะเชื่อมโยงกับใบแจ้งหนี้หลักที่จะบอกผู้รับของการดำเนินการว่าจำเป็นต้องทำให้การดำเนินกิจกรรมเสร็จสมบูรณ์

ถ้าชนิดของการดำเนินการเป็นอีเมลในรายละเอียดกระบวนการ ส่วนนี้จะมีแท็บด่วนสองแท็บ  แท็บแรกใช้ในการกำหนดรหัสเท็มเพลต คำอธิบายอีเมล ภาษาเริ่มต้น ชื่อผู้ใช้ที่จะกำหนดให้กับข้อความอีเมลที่ส่งโดยอัตโนมัติ และที่อยู่อีเมลของผู้ส่งที่เกี่ยวข้อง แท็บที่สองจะช่วยในการสร้างเนื้อหาของอีเมลหลังจากที่มีการบันทึกค่าในฟิลด์ **ภาษา** และ **ชื่อเรื่อง** โดยการเลือก **แก้ไข**  การดำเนินการนี้จะเปิดหน้าต่างที่อนุญาตให้อัปโหลดเนื้อหา HTML นอกจากนี้ คุณยังสามารถพิมพ์ข้อความที่สร้างด้วยตนเองในช่องด้านซ้ายล่างของหน้าต่าง  

> [!Note]
> คุณสามารถบันทึกอีเมล Outlook พร้อมเนื้อหาที่ต้องการของการสื่อสารในรูปแบบ HTML อีเมลนี้สามารถอัปโหลดเพื่อใช้เท็มเพลตได้ <br> <br> ถ้ามีการเลือกชนิดการดำเนินการของจดหมายเรียกเก็บเงิน จะไม่มีส่วนรายละเอียดของเอกสารทางธุรกิจบนแบบฟอร์มการตั้งค่า
