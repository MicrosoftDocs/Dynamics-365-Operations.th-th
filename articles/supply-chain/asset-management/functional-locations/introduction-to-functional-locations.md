[!include [banner](../../includes/banner.md)]

 

<span data-ttu-id="6586a-104">หัวข้อนี้แสดงภาพรวมของตำแหน่งที่ทำงานในการจัดการสินทรัพย์</span><span class="sxs-lookup"><span data-stu-id="6586a-104">This topic provides an overview of functional locations in Asset Management.</span></span> <span data-ttu-id="6586a-105">ตำแหน่งที่ทำงานคือองค์ประกอบของโครงสร้างทางเทคนิค เช่น หน่วยการทำงานในระบบ</span><span class="sxs-lookup"><span data-stu-id="6586a-105">Functional locations are elements of a technical structure, such as the functional units in a system.</span></span> <span data-ttu-id="6586a-106">ตำแหน่งที่ทำงานถูกสร้างขึ้นตามลำดับชั้น และคุณติดตั้งสินทรัพย์ในนั้น</span><span class="sxs-lookup"><span data-stu-id="6586a-106">Functional locations are created hierarchically, and you install assets on them.</span></span> <span data-ttu-id="6586a-107">การตั้งค่าของตำแหน่งที่ทำงานในบริษัทของคุณจะขึ้นอยู่กับความต้องการของบริษัท</span><span class="sxs-lookup"><span data-stu-id="6586a-107">The setup of functional locations in your company depends on the company's requirements.</span></span>

- <span data-ttu-id="6586a-109">**การทำงาน** – ตำแหน่งที่ทำงานสามารถมุ่งเน้นที่ผู้ใช้ และใช้ในการจัดการทรัพย์สินที่มีพฤติกรรมที่คล้ายกัน.</span><span class="sxs-lookup"><span data-stu-id="6586a-109">**Functional** – The functional locations can be user-oriented and used to manage assets that have similar behavior.</span></span>
- <span data-ttu-id="6586a-110">**กระบวนการที่เกี่ยวข้อง** – ตำแหน่งที่ทำงานสามารถมุ่งเน้นที่เวิร์กโฟลว์</span><span class="sxs-lookup"><span data-stu-id="6586a-110">**Process-related** – The functional locations can be workflow-oriented.</span></span>
- <span data-ttu-id="6586a-111">**เชิงพื้นที่** – ตำแหน่งที่ทำงานสามารถแสดงตำแหน่งหรือไซต์ทางภูมิศาสตร์ได้</span><span class="sxs-lookup"><span data-stu-id="6586a-111">**Spatial** – The functional locations can represent geographical locations or sites.</span></span>

- <span data-ttu-id="6586a-115">ตั้งค่าความต้องการข้อมูลจำเพาะเกี่ยวกับสินทรัพย์</span><span class="sxs-lookup"><span data-stu-id="6586a-115">Set up asset specification requirements.</span></span>
- <span data-ttu-id="6586a-116">ตั้งค่าลำดับการบำรุงรักษาสำหรับการบำรุงรักษาเชิงป้องกันและเชิงรุก</span><span class="sxs-lookup"><span data-stu-id="6586a-116">Set up maintenance sequences for preventive and reactive maintenance.</span></span>

- <span data-ttu-id="6586a-118">ติดตามคำขอและใบสั่งงานที่ใช้งานอยู่ที่เกี่ยวข้องกับสินทรัพย์ที่ติดตั้ง</span><span class="sxs-lookup"><span data-stu-id="6586a-118">Track active requests and work orders that are related to installed assets.</span></span>
- <span data-ttu-id="6586a-119">ติดตามข้อบกพร่องที่ลงทะเบียนไว้ในสินทรัพย์</span><span class="sxs-lookup"><span data-stu-id="6586a-119">Track faults that are registered on assets.</span></span>
- <span data-ttu-id="6586a-120">ติดตามต้นทุนการบำรุงรักษาในสินทรัพย์ที่เกี่ยวข้องกับตำแหน่งที่ทำงาน ณ เวลาที่กำหนด</span><span class="sxs-lookup"><span data-stu-id="6586a-120">Track maintenance costs on the assets that are related to a functional location at any given time.</span></span>

<span data-ttu-id="6586a-121">ตำแหน่งที่ทำงานให้การตรวจสอบย้อนกลับของสินทรัพย์ที่เกี่ยวข้องกับคำขอ ใบสั่งงาน การลงทะเบียนข้อบกพร่อง การประเมินเงื่อนไข การลงทะเบียนหยุดการผลิต และการลงทะเบียนตัวนับสินทรัพย์</span><span class="sxs-lookup"><span data-stu-id="6586a-121">Functional locations provide traceability of assets in relation to requests, work orders, fault registrations, condition assessments, production stop registrations, and asset counter registrations.</span></span>

> <span data-ttu-id="6586a-122">แม้ว่าจะมีการติดตั้งสินทรัพย์ในตำแหน่งที่ทำงานต่างๆ ในช่วงอายุการใช้งาน แต่ต้นทุนอาจเกี่ยวข้องกับสถานที่แต่ละแห่ง</span><span class="sxs-lookup"><span data-stu-id="6586a-122">Even if an asset is installed on various functional locations during its lifetime, the costs can be related to each location.</span></span> <span data-ttu-id="6586a-123">กล่าวคือ ต้นทุนสินทรัพย์จะเกี่ยวข้องกับตำแหน่งที่ทำงานที่มีการติดตั้งสินทรัพย์ในเวลาที่ระบุเสมอ</span><span class="sxs-lookup"><span data-stu-id="6586a-123">In other words, asset costs are always related to the functional location that the asset was installed on at a given time.</span></span>

<span data-ttu-id="6586a-126">หลังจากที่คุณสร้างลำดับชั้นตำแหน่งที่ทำงาน ขั้นตอนถัดไปคือการติดตั้งสินทรัพย์บนนั้น</span><span class="sxs-lookup"><span data-stu-id="6586a-126">After you create a functional location hierarchy, the next step is to install assets on it.</span></span> <span data-ttu-id="6586a-127">สำหรับข้อมูลเพิ่มเติม ดู [ติดตั้งสินทรัพย์ในตำแหน่งที่ทำงาน](../functional-locations/install-objects-on-functional-locations.md)</span><span class="sxs-lookup"><span data-stu-id="6586a-127">For more information, see [Install assets on functional locations](../functional-locations/install-objects-on-functional-locations.md).</span></span>
