# <a name="troubleshoot-discrete-manufacturing"></a>แก้ไขปัญหาการผลิตแบบไม่ต่อเนื่อง

หัวข้อนี้จะอธิบายวิธีการแก้ไขปัญหาที่คุณอาจพบขณะทำงานกับการผลิตแบบไม่ต่อเนื่อง

## <a name="i-receive-the-following-error-message-warehouse-management-processes-are-currently-being-used-if-work-lines-are-not-yet-registered-you-can-cancel-the-created-work-and-any-load-or-shipment-lines-and-then-continue-with-the-picking-or-shipping-process"></a>ฉันได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "กระบวนการจัดการคลังสินค้ากำลังมีการใช้งานอยู่ ถ้ารายการงานยังไม่ได้รับการลงทะเบียน คุณสามารถยกเลิกงานที่สร้างขึ้นและจำนวนงานในศูนย์การผลิตหรือรายการจัดส่งใด ๆ แล้วดำเนินการต่อไปยังขั้นตอนการเบิกสินค้าหรือการจัดส่งสินค้า

### <a name="issue-description"></a>คำอธิบายปัญหา

ปัญหานี้จะเกิดขึ้นถ้าคุณพยายามที่จะจองหรือนำงานออกใช้สำหรับรายการ แต่ธุรกรรมสินค้าคงคลังมีสถานะเป็น *เบิกสินค้า* ซึ่งบ่งชี้ว่ามีการเบิกสินค้าแล้ว

### <a name="issue-resolution"></a>การแก้ไขปัญหา

เพื่อแก้ไขปัญหานี้ ให้ทำตามหนึ่งในขั้นตอนเหล่านี้

- เปลี่ยนสถานะของใบสั่งผลิตกลับเป็น *ประมาณการ* หรือ *นำออกใช้*
- เปิดหน้ารายละเอียดสำหรับใบสั่งการผลิตที่เกี่ยวข้อง บนบานหน้าต่างการดำเนินการ บนแท็บ **คลังสินค้า** ในกลุุุ่ม **หยุด** ให้เลือก **หยุดและไม่เบิกสินค้า** เพื่อไม่เบิกธุรกรรมที่เบิกสินค้าทั้งหมด เลือก **ลบ** เพื่อประมวลผลใบสั่งผลิต

ต่อไปนี้เป็นคำอธิบายเกี่ยวกับฟังก์ชัน *ไม่เบิก* และ *หยุด*:

- **การยกเลิกการเบิก** – ฟังก์ชันนี้ย้อนกลับสถานะของธุรกรรมสินค้าคงคลังสำหรับรายการสูตรการผลิต (BOM) และรายการสูตรที่มีสถานะเป็น *เบิกสินค้า* ผ่าน *ทางใบสั่ง* เมื่อทำงานสำหรับการเบิกวัตถุดิบเสร็จสมบูรณ์แล้ว สถานะของราบการถูกตั้งค่าเป็น *เบิกสินค้าแล้ว* สถานะนี้จะป้องกันไม่ให้มีการรีเซ็ตใบสั่งผลิตเป็นสถานะ *สร้างแล้ว* ในกรณีนี้ คุณสามารถใช้ฟังก์ชัน *การยกเลิกการเบิกสินค้า* เพื่อแปลงกลับธุรกรรมจากสถานะ *เบิกสินค้า* แล้วรีเซ็ตใบสั่งผลิตเป็นสถานะ *สร้างแล้ว*
- **หยุด** – ฟังก์ชันนี้จะตั้งค่าแฟล็ก **หยุด** สำหรับใบสั่งผลิตเพื่อป้องกันไม่ให้มีการอัพเดตสถานะใด ๆ ในใบสั่ง คุณสามารถค้นหาแฟล็ก **หยุด** บนแท็บด่วน **คลังสินค้า** ของหน้ารายละเอียดใบสั่งผลิต

> [!NOTE]
>
> - ปุ่มจะมองเห็นได้เฉพาะเมื่อมีการสร้างใบสั่งผลิตสำหรับสินค้าที่เปิดใช้งานสำหรับกระบวนการคลังสินค้าเท่านั้น
> - กลุ่ม **หยุด** จะมองเห็นได้เฉพาะบนแท็บ **คลังสินค้า** บนบานหน้าต่างการดำเนินการของหน้ารายละเอียดใบสั่งผลิตเท่านั้น ไม่สามารถมองเห็นได้บนแท็บด่วน **คลังสินค้า** ของหน้ารายการ **ใบสั่งผลิต**

## <a name="the-matching-resource-name-isnt-updated-after-i-change-a-worker-name-in-the-global-address-book"></a>ชื่อทรัพยากรที่ตรงกันไม่ได้รับการอัพเดตแล้วหลังจากที่เปลี่ยนชื่อผู้ปฏิบัติงานในสมุดที่อยู่สากล

### <a name="issue-description"></a>คำอธิบายปัญหา

หากคุณเปลี่ยนชื่อผู้ปฏิบัติงานในสมุดที่อยู่สากล ชื่อทรัพยากรที่ตรงกันไม่ได้รับการอัพเดตในกลุ่มทรัพยากรหลัก

### <a name="issue-resolution"></a>การแก้ไขปัญหา

สถานการณ์นี้ไม่ได้รับการสนับสนุนในขณะนี้ เมื่อต้องการแก้ไขปัญหานี้ คุณต้องอัพเดตชื่อทรัพยากรด้วยตนเอง

## <a name="when-i-create-a-new-production-order-i-dont-receive-the-following-message-insert-the-active-version-of-bill-of-material-and-route"></a>เมื่อสร้างใบสั่งผลิตใหม่ ฉันจะไม่ได้รับข้อความต่อไปนี้: "แทรกรุ่นที่ใช้งานอยู่ของสูตรการผลิตและกระบวนการผลิตหรือไม่"

### <a name="issue-description"></a>คำอธิบายปัญหา

เมื่อคุณสร้างใบสั่งผลิตใหม่หลังจากที่คุณป้อนหมายเลขสินค้า ฟิลด์ **ไซต์** และ **คลังสินค้า** จะมีการตั้งค่าให้กับไซต์และคลังสินค้าเริ่มต้นที่กำหนดไว้ในหน้า **การตั้งค่าใบสั่งเริ่มต้น** สำหรับรายการสินค้าสำเร็จรูปโดยอัตโนมัติ นอกจากนี้ จะมีการป้อนหมายเลข BOM ที่ใช้งานอยู่และหมายเลขกระบวนการผลิตโดยอัตโนมัติ คุณไม่ได้รับข้อความแสดงข้อความต่อไปนี้ที่พร้อมท์ให้คุณสำหรับค่าเหล่านี้:

> แทรกรุ่นที่ใช้งานอยู่สำหรับสูตรการผลิตและกระบวนการผลิตหรือไม่

### <a name="issue-resolution"></a>การแก้ไขปัญหา

คุณไม่ได้รับพร้อมท์ให้แทรก BOM และหมายเลขกระบวนการผลิตถ้าคุณเลือกผลิตภัณฑ์ที่มีการกำหนดไซต์และคลังสินค้าให้กับหน้า **การตั้งค่าใบสั่งเริ่มต้น** คุณได้รับข้อความแจ้งเฉพาะเมื่อมีการกำหนดค่าเหล่านั้นสำหรับผลิตภัณฑ์ที่เลือกเท่านั้น

## <a name="production-orders-arent-shown-on-the-marking-page"></a>ใบสั่งผลิตไม่แสดงอยู่บนหน้าการทำเครื่องหมาย

### <a name="issue-description"></a>คำอธิบายปัญหา

บางใบสั่งผลิตไม่แสดงอยู่บนหน้า **การทำเครื่องหมาย**

### <a name="issue-resolution"></a>การแก้ไขปัญหา

ผลิตภัณฑ์ที่มีการตั้งค่าคอนฟิกต่อไปนี้ไม่พร้อมใช้งานสำหรับการทำเครื่องหมาย ดังนั้นจึงไม่แสดงบนหน้า **การทำเครื่องหมาย**:

- ผลิตภัณฑ์ได้รับการกำหนดเป็นผลิตภัณฑ์ของชนิด *น้ำหนักจริง*
- มีการเปิดใช้งานสำหรับกระบวนการคลังสินค้าขั้นสูง
- มีการตั้งค่าคอนฟิกไว้เพื่อควบคุมโดยหลักการ *ต้นทุนมาตรฐาน*

## <a name="when-i-try-to-end-a-production-order-i-receive-the-following-error-message-calculating-bom-consumptioncost-value-must-be-negative-upon-issue-from-inventory"></a>เมื่อพยายามจบใบสั่งผลิต ฉันจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "การคำนวณค่าต้นทุนปริมาณการใช้ BOM ต้องเป็นค่าลบเมื่อมีการนำออกใช้สินค้าคงคลัง"

มีการแก้ไขปัญหานี้ในรุ่น 10.0.15

## <a name="when-the-status-of-a-production-order-is-changed-from-reported-as-finished-to-end-i-receive-the-following-error-messages-update-conflict-the-standard-cost-does-not-match-with-the-financial-inventory-value-after-the-update-and-a-critical-error-has-occurred-in-function-inventcostmovementcheckvariance"></a>เมื่อสถานะของใบสั่งผลิตมีการเปลี่ยนแปลงจาก รายงาน เมื่อเสร็จสิ้นแล้วจนจบ ฉันจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "อัพเดตความขัดแย้ง ต้นทุนมาตรฐานไม่ตรงกับมูลค่าสินค้าคงคลังทางการเงินหลังจากการอัพเดต" และ "เกิดข้อผิดพลาดร้ายแรงในฟังก์ชัน InventCostMovement.checkVariance"

ปัญหานี้เกิดขึ้นเนื่องจากมีการเปลี่ยนแปลงข้อมูลที่อยู่ภายใต้การดำเนินการอื่น กระบวนการนี้จะพยายามอัพเดตข้อมูลจนถึงห้าครั้ง ถ้าความขัดแย้งยังคงมีอยู่หลังจากความพยายามห้าครั้ง คุณจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้:

> อัพเดตความขัดแย้ง ต้นทุนมาตรฐานไม่ตรงกับมูลค่าสินค้าคงคลังทางการเงินหลังการอัพเดต

> เกิดข้อผิดพลาดร้ายแรงในฟังก์ชัน InventCostMovement.checkVariance

ลักษณะการทำงานนี้เกิดจากการออกแบบ เมื่อต้องการบรรเทาปัญหา ให้ดำเนินการชุดงานต่อไป ควรทำงานให้เสร็จสมบูรณ์

ถ้าชุดงานต่อเนื่องล้มเหลวหลังจากที่คุณดำเนินการต่อ ให้ตรวจสอบว่าความแม่นยำในการปัดเศษสำหรับสกุลเงินเริ่มต้นของบัญชีแยกประเภทเข้ากับการปัดเศษฃใช้กับค่าในตาราง InventSum ถ้าความแม่นยำในการปัดเศษถูกเปลี่ยนเป็นค่าที่ไม่สอดคล้องกัน คุณอาจต้องเปลี่ยนกลับเป็นค่าที่สอดคล้องกัน ค้นหา **ModifiedDateTime** ในกรณีนี้ ค่าโดยทั่วไปจะแสดงว่ามีการเปลี่ยนแปลงความแม่นยำในการปัดเศษเมื่อเร็ว ๆ นี้

## <a name="when-i-release-to-a-warehouse-i-receive-the-following-error-message-item-rm-could-not-be-fully-reserved-ensure-that-the-full-quantity-is-available-or-reserve-the-items-manually-if-the-reservation-field-on-the-bom-line-is-set-to-manual-or-started-could-not-release-the-order-to-warehouse-because-some-materials-could-not-be-reserved-however-the-status-is-updated-to-released"></a>เมื่อนำออกใช้ไปยังคลังสินค้าแล้ว ฉันจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "ไม่สามารถจองสินค้าได้ทั้งหมด ตรวจสอบให้แน่ใจว่าปริมาณเต็มพร้อมใช้งาน หรือจองสินค้าด้วยตนเองถ้าฟิลด์การจองบนรายการ BOM ถูกตั้งค่าเป็นด้วยตนเองหรือเริ่มต้นแล้ว ไม่สามารถนำใบสั่งออกใช้ไปยังคลังสินค้าปลายทาง เนื่องจากวัสดุบางส่วนยังไม่มีการจอง" อย่างไรก็ตาม มีการอัพเดตสถานะเป็นนำออกใช้แล้ว

### <a name="issue-description"></a>คำอธิบายปัญหา

ถ้าไม่มีสินค้าในรายการ BOM ทั้งหมดที่พร้อมใช้งานเมื่อมีการนำใบสั่งผลิตออกใช้ และนโยบาย **การนำออกใช้ไปยังคลังสินค้า** มีการตั้งค่าเป็น *ต้องจองแบบเต็ม* บนใบสั่งผลิต คุณจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้:

> ไม่สามารถจองสินค้า RM ทั้งหมดได้ ตรวจสอบให้แน่ใจว่าปริมาณเต็มพร้อมใช้งาน หรือจองสินค้าด้วยตนเองถ้าฟิลด์การจองบนรายการ BOM ถูกตั้งค่าเป็นด้วยตนเองหรือเริ่มต้นแล้ว ไม่สามารถนำใบสั่งออกใช้ไปยังคลังสินค้าปลายทาง เนื่องจากวัสดุบางส่วนยังไม่มีการจอง

### <a name="issue-resolution"></a>การแก้ไขปัญหา

ลักษณะการทำงานนี้มีโดยการออกแบบและทำงานตามที่คาดไว้

## <a name="when-i-try-to-end-a-production-order-and-report-as-finished-i-receive-the-following-error-message-total-good-quantity-reported-as-finished-for-the-production-will-be-1-feedback-for-the-last-operation-is-000-in-total"></a>เมื่อพยายามจบใบสั่งผลิตและรายงานเมื่อเสร็จสิ้นแล้ว ฉันจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้: "ปริมาณของดีทั้งหมดที่รายงานการเสร็จงานแล้วจะเป็น %1 ผลป้อนกลับสำหรับการดำเนินงานหลังสุดเป็น 0.00 ในยอดรวม"

### <a name="issue-description"></a>คำอธิบายปัญหา

เมื่อคุณพยายามลงรายการบัญชีสมุดรายวันรายงานการเสร็จงานในใบสั่งผลิต คุณจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้:

> ผลรวมปริมาณสินค้าที่ดีที่รายงานเมื่อเสร็จสมบูรณ์สำหรับการผลิตจะเป็น %1 ผลป้อนกลับสำหรับการดำเนินงานหลังสุดเป็น 0.00 ในยอดรวม

### <a name="possible-cause"></a>สาเหตุที่เป็นไปได้

ปัญหานี้จะเกิดขึ้นถ้าปริมาณที่ได้ลงรายการบัญชีไว้ในการดำเนินงานครั้งสุดท้ายไม่ถูกต้อง ตัวอย่างเช่น ถ้ามีการเริ่มต้นการผลิต แต่ปริมาณที่ต้องเริ่มต้นไม่มีการปัน ส่วนสมุดรายวันบัตรกระบวนการผลิตจะถูกลงรายการบัญชีโดยไม่มีรายการใด ๆ เมื่อต้องการยืนยันสถานการณ์ ให้เปิดใบสั่งผลิต จากนั้นในบานหน้าต่างการดำเนินการ บนแท็บ **มุมมอง** ให้เลือก **บัตรกระบวนการผลิต**

### <a name="workaround"></a>วิธีการแก้ไข

คุณสามารถแก้ไขปัญหานี้ได้ด้วยการลงรายการบัญชีสมุดรายวันเพิ่มเติมสำหรับการดำเนินงานที่มีการลงรายการบัญชีสมุดรายวันไม่ถูกต้อง เริ่มต้นใบสั่งผลิตใหม่ และเลือกเพื่อลงรายการบัญชีสมุดรายวันเพิ่มเติม การดำเนินการนี้จะไม่เริ่มต้นใบสั่งผลิต แต่จะมีการลงรายการบัญชีสมุดรายวัน บัตรกระบวนการผลิตควรแสดงปริมาณที่ลงรายการบัญชีแล้ว และคุณควรจะสามารถจบใบสั่งผลิตได้เสร็จเรียบร้อยแล้ว

## <a name="can-i-report-a-production-order-as-finished-while-i-report-the-error-quantity-but-not-while-i-report-the-time-or-material-quantity"></a>ฉันสามารถรายงานใบสั่งผลิตเมื่อเสร็จสมบูรณ์ในขณะที่ฉันรายงานปริมาณข้อผิดพลาด แต่ไม่ใช่ในขณะที่ฉันรายงานปริมาณเวลาหรือวัสดุหรือไม่

คุณไม่สามารถรายงานปริมาณของข้อผิดพลาดในใบสั่งผลิตยกเว้นว่าคุณจะรายงานปริมาณสินค้าที่ดี สถานการณ์จำลองนี้ **ไม่ได้** รับการสนับสนุน การอัพเดตการรายงานเมื่อเสร็จสมบูรณ์จะล้มเหลวจนกว่าจะถึงวันที่สิ้นสุดใบสั่งผลิต และคุณจะได้รับข้อความแสดงข้อผิดพลาดต่อไปนี้:

> ปริมาณที่รายงานการเสร็จงานขาดหายไป

## <a name="can-i-trace-the-serial-numbers-of-finished-goods-against-the-serial-numbers-of-consumed-goods"></a>ฉันสามารถติดตามหมายเลขลำดับประจำสินค้าของสินค้าสำเร็จรูปกับหมายเลขลำดับประจำสินค้าของสินค้าที่บริโภคได้หรือไม่

คุณไม่สามารถติดตามหมายเลขลำดับประจำสินค้าของสินค้าสำเร็จรูปกับหมายเลขลำดับประจำสินค้าของวัสดุที่ใบสั่งผลิตใช้ในการทำให้สินค้าสำเร็จรูป สถานการณ์นี้ไม่ได้รับการสนับสนุนในขณะนี้ วิธีการแก้ปัญหาคือการสร้างใบสั่งผลิตสำหรับปริมาณ 1 รายการ
