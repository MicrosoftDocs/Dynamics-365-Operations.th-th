<span data-ttu-id="592e3-104">หัวข้อนี้อธิบายวิธีการใช้เท็มเพลตงานและคำสั่งสถานที่ เพื่อกำหนดวิธีและตำแหน่งที่งานจะถูกดำเนินการในคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-104">This topic describes how to use work templates and location directives to determine how and where work is carried out in the warehouse.</span></span>

## <a name="work-templates"></a><span data-ttu-id="592e3-108">เท็มเพลตงาน</span><span class="sxs-lookup"><span data-stu-id="592e3-108">Work templates</span></span>

<span data-ttu-id="592e3-116">การตั้งค่าในคำอธิบายหัวข้องานสามารถใช้เพื่อกำหนดว่าเมื่อใดที่งานชิ้นใหม่ควรถูกสร้าง</span><span class="sxs-lookup"><span data-stu-id="592e3-116">The settings in the work header definition can be used to determine when a new piece of work should be created.</span></span> <span data-ttu-id="592e3-117">ตัวอย่างเช่น คุณสามารถตั้งค่าจำนวนสูงสุดของการเบิกสินค้าและเวลาที่คาดว่าจะเบิกสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-117">For example, you can set a maximum number of pick lines and a maximum expected pick time.</span></span> <span data-ttu-id="592e3-118">จากนั้น ถ้างานสำหรับกระบวนการเบิกสินค้าสำหรับใบสั่งขายนั้นเกินอย่างใดอย่างหนึ่งของค่าเหล่านี้ งานนั้นจะถูกแบ่งออกเป็นงานสองชิ้น</span><span class="sxs-lookup"><span data-stu-id="592e3-118">Then, if the work for a sales order picking process exceeds either of those values, that work is split into two pieces of work.</span></span> 

<span data-ttu-id="592e3-119">รายการงานแสดงถึงหน้าที่จริงที่จำเป็นในกระบวนการการดำเนินงาน</span><span class="sxs-lookup"><span data-stu-id="592e3-119">The work lines represent the physical tasks that are required in order to process the work.</span></span> <span data-ttu-id="592e3-120">ตัวอย่างเช่น สำหรับกระบวนการคลังสินค้าขาออกอาจมีรายการงานสำหรับการเบิกสินค้าภายในคลังสินค้าและอีกรายการหนึ่งสำหรับการย้ายสินค้าเหล่านั้นไปยังพื้นที่จัดเตรียมสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-120">For example, for an outbound warehouse process, there might be a work line for picking up the items within the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="592e3-121">จากนั้นสามารถมีรายการเพิ่มเติมสำหรับการเบิกสินค้าจากการจัดเตรียม และอีกรายการหนึ่งสำหรับการย้ายสินค้าไปยังรถบรรทุกที่เป็นส่วนหนึ่งของกระบวนการบรรทุกสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-121">There can then be an additional line for picking the items from staging, and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="592e3-122">คุณสามารถตั้งค่า *รหัสคำสั่ง* บนรายการเท็มเพลตงานได้</span><span class="sxs-lookup"><span data-stu-id="592e3-122">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="592e3-123">รหัสคำสั่งจะเชื่อมโยงกับคำสั่งสถานที่และดังนั้นจึงช่วยรับประกันว่างานคลังสินค้าจะดำเนินการในสถานที่ที่ถูกต้องในคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-123">A directive code is linked to a location directive and therefore helps guarantee that the warehouse work is processed in the correct location in the warehouse.</span></span> 

<span data-ttu-id="592e3-124">คุณสามารถตั้งค่าการสอบถามเพื่อควบคุมเมื่อเท็มเพลตงานโดยเฉพาะนั้นถูกใช้</span><span class="sxs-lookup"><span data-stu-id="592e3-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="592e3-125">ตัวอย่างเช่น คุณสามารถกำหนดข้อจำกัดเพื่อเท็มเพลตงโดยเฉพาะนั้นจะถูกใช้สำหรับงานในคลังสินค้าเฉพาะเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="592e3-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="592e3-126">อีกทางหนึ่งคือ คุณอาจมีเท็มเพลตต่างๆที่ใช้ในการสร้างงานสำหรับการประมวลผลใบสั่งขายขาออก ขึ้นอยู่กับช่องทางการขาย</span><span class="sxs-lookup"><span data-stu-id="592e3-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="592e3-127">ระบบใช้ฟิลด์ **หมายเลขลำดับ** เพื่อกำหนดลำดับที่เท็มเพลตงานมีอยู่นั้นถูกประเมิน</span><span class="sxs-lookup"><span data-stu-id="592e3-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="592e3-128">ดังนั้น ถ้าคุณมีแบบสอบถามที่ระบุเฉพาะสำหรับเท็มเพลตงานเฉพาะ คุณควรกำหนดหมายเลขลำดับต่ำให้แก่เท็มเพลตงานนั้น</span><span class="sxs-lookup"><span data-stu-id="592e3-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="592e3-129">จากนั้นแบบสอบถามนั้นจะถูกประเมินก่อนที่จะสอบถามอื่นๆ คำสั่งที่ธรรมดาขึ้น</span><span class="sxs-lookup"><span data-stu-id="592e3-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="592e3-130">เพื่อหยุดหรือหยุดกระบวนการงานชั่วคราว คุณสามารถใช้การตั้งค่า **หยุดงาน** บนรายการงานได้</span><span class="sxs-lookup"><span data-stu-id="592e3-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="592e3-131">ในกรณีดังกล่าว ผู้ปฏิบัติงานที่กำลังทำงานจะไม่ถูกขอให้ดำเนินการขั้นตอนถัดไปของรายการงาน</span><span class="sxs-lookup"><span data-stu-id="592e3-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="592e3-132">เพื่อเลื่อนไปยังขั้นตอนถัดไป ที่ผู้ปฏิบัติงานหรือผู้ปฏิบัติงานผู้อื่นต้องเลือกงานอีกครั้ง</span><span class="sxs-lookup"><span data-stu-id="592e3-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="592e3-133">คุณยังสามารถแยกหน้าภายในชิ้นงาน โดยใช้อื่นซึ่งไม่ใช่ *รหัสคลาสงาน* ในรายการเท็มเพลงานงได้</span><span class="sxs-lookup"><span data-stu-id="592e3-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="592e3-134">คำสั่งสถานที่</span><span class="sxs-lookup"><span data-stu-id="592e3-134">Location directives</span></span>
<span data-ttu-id="592e3-135">คำสั่งสถานที่มีกฎซึ่งช่วยให้ระบุการเบิกสินค้าและกำหนดสถานที่เก็บสำหรับการย้ายสินค้าคงคลัง</span><span class="sxs-lookup"><span data-stu-id="592e3-135">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="592e3-136">ตัวอย่างเช่น ในธุรกรรมใบสั่งขาย คำสั่งสถานกำหนดจุดที่จะเบิกสินค้า และจุดที่จะวางสินค้าที่เบิก</span><span class="sxs-lookup"><span data-stu-id="592e3-136">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="592e3-137">คำสั่งสถานที่ประกอบด้วยหัวข้อและรายการที่เกี่ยวข้อง และคุณสร้างหัวข้อและรายการที่เกี่ยวข้องในหน้า **คำสั่งสถานที่**.</span><span class="sxs-lookup"><span data-stu-id="592e3-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span>

### <a name="example-of-the-use-of-location-directives"></a><span data-ttu-id="592e3-256">ตัวอย่างของการใช้คำสั่งสถานที่</span><span class="sxs-lookup"><span data-stu-id="592e3-256">Example of the use of location directives</span></span>

<span data-ttu-id="592e3-257">สำหรับตัวอย่างนี้ จะพิจารณากระบวนการใบสั่งซื้อที่ซึ่งคำสั่งสถานที่ต้องค้นหากำลังการผลิตที่ว่างอยู่ภายในคลังสินค้าสำหรับสินค้าคงคลังที่เพิ่งถูกลงทะเบียน ณ ท่าสินค้ารับสินค้า</span><span class="sxs-lookup"><span data-stu-id="592e3-257">For this example, consider a purchase order process where the location directive must find free capacity within a warehouse for inventory items that have just been registered at the receiving dock.</span></span> <span data-ttu-id="592e3-258">ก่อนอื่น คุณต้องค้นหากำลังการผลิตที่ว่างอยู่ภายในคลังสินค้าด้วยการรวมเข้ากับปริมาณสินค้าคงคลังคงเหลือที่มีอยู่</span><span class="sxs-lookup"><span data-stu-id="592e3-258">First, you need to find free capacity within the warehouse by consolidating with existing on-hand inventory.</span></span> <span data-ttu-id="592e3-259">ถ้าไม่สามารถรวมได้ คุณจะต้องค้นหาสถานที่ที่ว่าง</span><span class="sxs-lookup"><span data-stu-id="592e3-259">If consolidation isn't possible, then you need to find an empty location.</span></span> 

<span data-ttu-id="592e3-260">สำหรับสถานการณ์จำลองนี้ คุณต้องกำหนดการดำเนินคำสั่งสถานที่สองรายการ</span><span class="sxs-lookup"><span data-stu-id="592e3-260">For this scenario, you must define two location directive actions.</span></span> <span data-ttu-id="592e3-261">การดำเนินการแรกในลำดับต้องใช้ยุทธ์ **รวม** และการดำเนินการที่สองควรใช้กลยุทธ์ **สถานที่ว่างโดยไม่มีงานขาเข้า**</span><span class="sxs-lookup"><span data-stu-id="592e3-261">The first action in the sequence must use the **Consolidate** strategy, and the second should use the **Empty location with no incoming work** strategy.</span></span> <span data-ttu-id="592e3-262">เว้นแต่ว่าคุณกำหนดการดำเนินการที่สามเพื่อจัดการสถานการณ์จำลองที่มากเกินไป ผลลัพธ์สองรายการอาจเป็นไปได้ เมื่อไม่มีกำลังการผลิตในคลังสินค้า: งานสามารถถูกสร้างได้ แม้ไม่มีการระบุสถานที่ หรือกระบวนการสร้างงานอาจล้มเหลวได้</span><span class="sxs-lookup"><span data-stu-id="592e3-262">Unless you define a third action to handle an overflow scenario, two outcomes are possible when there is no more capacity in the warehouse: work can be created even though no locations are defined, or the work creation process can fail.</span></span> <span data-ttu-id="592e3-263">ผลลัพธ์จะถูกกำหนดตามการตั้งค่าบนหน้า **คำสั่งสถานที่ล้มเหลว** ซึ่งคุณสามารถตัดสินใจว่าจะเลือกตัวเลิอก **หยุดงานเมื่อคำสั่งสถานที่ล้มเหลว** สำหรับแต่ละชนิดใบสั่งผลิตหรือไม่</span><span class="sxs-lookup"><span data-stu-id="592e3-263">The outcome is determined by the setup on the **Location directive failures** page, where you can decide whether to select the **Stop work on location directive failure** option for each work order type.</span></span>
