## <a name="work-templates"></a><span data-ttu-id="9db2a-108">เท็มเพลตงาน</span><span class="sxs-lookup"><span data-stu-id="9db2a-108">Work templates</span></span>

<span data-ttu-id="9db2a-116">ใช้การตั้งค่าในคำอธิบายหัวข้องานเพื่อกำหนดว่าเมื่อใดที่งานชิ้นใหม่ควรถูกสร้าง</span><span class="sxs-lookup"><span data-stu-id="9db2a-116">Use the settings in the work header definition to determine when a new piece of work should be created.</span></span> <span data-ttu-id="9db2a-117">ตัวอย่างเช่น คุณสามารถตั้งค่าจำนวนสูงสุดของการเบิกสินค้าและเวลาที่คาดว่าจะเบิกสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-117">For example, you can set a maximum number of pick lines and a maximum expected pick time.</span></span> <span data-ttu-id="9db2a-118">จากนั้น ถ้างานสำหรับกระบวนการเบิกสินค้าสำหรับใบสั่งขายนั้นเกินอย่างใดอย่างหนึ่งของค่าเหล่านี้ งานนั้นจะถูกแบ่งออกเป็นงานสองชิ้น</span><span class="sxs-lookup"><span data-stu-id="9db2a-118">Then, if the work for a sales order picking process exceeds either of those values, that work is split into two pieces of work.</span></span>

<span data-ttu-id="9db2a-123">รายการงานแสดงถึงหน้าที่จริงที่จำเป็นในกระบวนการการดำเนินงาน</span><span class="sxs-lookup"><span data-stu-id="9db2a-123">The work lines represent the physical tasks that are required to process the work.</span></span> <span data-ttu-id="9db2a-124">ตัวอย่างเช่น สำหรับกระบวนการคลังสินค้าขาออก อาจมีรายการหนึ่งสำหรับการเบิกสินค้าในคลังสินค้าและอีกรายการหนึ่งสำหรับการย้ายสินค้าเหล่านั้นไปยังพื้นที่จัดเตรียมสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-124">For example, for an outbound warehouse process, there might be one line for picking up the items in the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="9db2a-125">จากนั้นสามารถมีรายการเพิ่มเติมสำหรับการเบิกสินค้าจากการจัดเตรียม และอีกรายการหนึ่งสำหรับการย้ายสินค้าไปยังรถบรรทุกที่เป็นส่วนหนึ่งของกระบวนการบรรทุกสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-125">There can then be an additional line for picking the items from staging and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="9db2a-126">คุณสามารถตั้งค่า *รหัสคำสั่ง* บนรายการเท็มเพลตงานได้</span><span class="sxs-lookup"><span data-stu-id="9db2a-126">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="9db2a-127">รหัสคำสั่งจะเชื่อมโยงกับคำสั่งสถานที่ และดังนั้นจึงช่วยให้มั่นใจว่างานคลังสินค้าจะดำเนินการในสถานที่ที่ถูกต้องในคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-127">A directive code is linked to a location directive, and therefore helps ensure that the warehouse work is processed in the correct location in the warehouse.</span></span>

## <a name="location-directives"></a><span data-ttu-id="9db2a-139">คำสั่งสถานที่</span><span class="sxs-lookup"><span data-stu-id="9db2a-139">Location directives</span></span>

<span data-ttu-id="9db2a-140">คำสั่งสถานที่มีกฎซึ่งช่วยให้ระบุการเบิกสินค้าและกำหนดสถานที่เก็บสำหรับการย้ายสินค้าคงคลัง</span><span class="sxs-lookup"><span data-stu-id="9db2a-140">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="9db2a-141">ตัวอย่างเช่น ในธุรกรรมใบสั่งขาย คำสั่งสถานกำหนดจุดที่จะเบิกสินค้า และจุดที่จะวางสินค้าที่เบิก</span><span class="sxs-lookup"><span data-stu-id="9db2a-141">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="9db2a-142">คำสั่งสถานที่ประกอบด้วยหัวข้อและรายการที่เกี่ยวข้อง และคุณสร้างหัวข้อและรายการที่เกี่ยวข้องในหน้า **คำสั่งสถานที่**.</span><span class="sxs-lookup"><span data-stu-id="9db2a-142">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span>

<span data-ttu-id="9db2a-143">บนหัวข้อ แต่ละคำสั่งสถานต้องสัมพันธ์กับ *ชนิดใบสั่งงาน* ที่ระบุชนิดของธุรกรรมสินค้าคงคลังที่คำสั่งจะถูกใช้สำหรับ เช่นใบสั่งขาย การเติมสินค้า หรือการเบิกวัตถุดิบ</span><span class="sxs-lookup"><span data-stu-id="9db2a-143">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="9db2a-144"> *ชนิดงาน* ระบุว่าจะใช้คำสั่งสถานที่สำหรับการเบิกสินค้าหรืองานการขนย้าย หรือสำหรับกระบวนการคลังสิงค้าอื่น เช่นการนับหรือการเปลี่ยนแปลงสถานะสินค้าคงคลัง</span><span class="sxs-lookup"><span data-stu-id="9db2a-144">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="9db2a-145">คุณต้องระบุ *ไซต์* และ *คลังสินค้า*</span><span class="sxs-lookup"><span data-stu-id="9db2a-145">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="9db2a-146">*รหัสคำสั่ง* ที่คุณระบุบนหัวข้อสามารถใช้เพื่อเชื่อมโยงคำสั่งสถานที่ไปยังเท็มเพลตงานอย่างน้อยหนึ่งเท็มเพลตหรือมากกว่านั้น</span><span class="sxs-lookup"><span data-stu-id="9db2a-146">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span> 

<span data-ttu-id="9db2a-147">สำหรับเท็มเพลตงาน คุณสามารถตั้งค่าคำสั่งเพื่อกำหนดว่าเมื่อไหร่คำสั่งที่สถานที่ต้องถูกใช้งาน</span><span class="sxs-lookup"><span data-stu-id="9db2a-147">As for work templates, you can set up a query to determine when a particular location directive is used.</span></span> <span data-ttu-id="9db2a-148">ตัวอย่างเช่น คุณสามารถระบุว่าเมื่อระบบอีคอมเมิร์ซเป็นต้นกำเนิดของใบสั่งขาย สินค้าคงคลังต้องถูกเลือกจากพื้นที่ที่จัดสรรไว้ในคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-148">For example, you can specify that when e-commerce is the origin of a sales order, the inventory must be picked up from a dedicated area in the warehouse.</span></span> <span data-ttu-id="9db2a-149">ระบบใช้ฟิลด์ **หมายเลขลำดับ** เพื่อกำหนดลำดับที่คำสั่งสถานที่ที่มีอยู่ที่จะประเมิน</span><span class="sxs-lookup"><span data-stu-id="9db2a-149">The system uses the **Sequence number** field to determine the order that the available location directives are assessed in.</span></span>

<span data-ttu-id="9db2a-150">รายการคำสั่งสถานที่กำหนดข้อจำกัดการใช้กฎการค้นหาสถานที่เพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="9db2a-150">The location directive lines set additional restrictions on the application of the location finding rules.</span></span> <span data-ttu-id="9db2a-151">คุณสามารถระบุปริมาณต่ำสุดและปริมาณสูงสุดที่คำสั่งควรใช้กับ และคุณสามารถระบุว่าควรมีคำสั่งสำหรับเฉพาะหน่วยสินค้าคงคลังหรือไม่</span><span class="sxs-lookup"><span data-stu-id="9db2a-151">You can specify a minimum quantity and a maximum quantity that the directive should apply to, and you can specify that the directive should be for a specific inventory unit.</span></span> <span data-ttu-id="9db2a-152">ตัวอย่างเช่น ถ้าหน่วยวัดเป็นแท่นวางสินค้า สินค้าในแท่นวางสินค้าสามารถวางในตำแหน่งที่กำหนดไว้ได้</span><span class="sxs-lookup"><span data-stu-id="9db2a-152">For example, if the unit of measure is pallets, the items in pallets can be put in a specific location.</span></span> <span data-ttu-id="9db2a-153">คุณยังสามารถระบุว่าปริมาณที่จะสามารถถูกแบ่งสำหรับหลายสถานที่ได้หรือไม่</span><span class="sxs-lookup"><span data-stu-id="9db2a-153">You can also specify whether the quantity can be split across multiple locations.</span></span> <span data-ttu-id="9db2a-154">เช่นเดียวกับที่หัวข้อคำสั่งของสถานที่ แต่ละรายการคำสั่งสถานที่มีหมายเลขลำดับที่กำหนดลำดับที่รายการจะถูกประเมิน</span><span class="sxs-lookup"><span data-stu-id="9db2a-154">Like the location directive header, each location directive line has a sequence number that determines the order that the lines are assessed in.</span></span>

<span data-ttu-id="9db2a-155">คำสั่งสถานที่มีหนึ่งระดับของรายละเอียดเพิ่มเติม: *การดำเนินคำสั่งสถานที่*</span><span class="sxs-lookup"><span data-stu-id="9db2a-155">Location directives have one additional level of detail: *location directive actions*.</span></span> <span data-ttu-id="9db2a-156">คุณสามารถกำหนดหลายการดำเนินคำสั่งสถานที่สำหรับแต่ละรายการ</span><span class="sxs-lookup"><span data-stu-id="9db2a-156">You can define multiple location directive actions for each line.</span></span> <span data-ttu-id="9db2a-157">อีกครั้งหนึ่ง หมายเลขลำดับจะใช้เพื่อกำหนดลำดับที่การดำเนินการนั้นจะถูกประเมิน</span><span class="sxs-lookup"><span data-stu-id="9db2a-157">Once again, a sequence number is used to determine the order that the actions are assessed in.</span></span> <span data-ttu-id="9db2a-158">ในระดับนี้ คุณสามารถตั้งค่าคำสั่งเพื่อกำหนดวิธีเพื่อหาสถานที่ที่ดีที่สุดในคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-158">On this level, you can set up a query to define how to find the best location in the warehouse.</span></span> <span data-ttu-id="9db2a-159">คุณยังสามารถใช้การตั้งค่า **กลยุทธ์** ไว้ล่วงหน้าเพื่อหาสถานที่ที่เหมาะสมได้</span><span class="sxs-lookup"><span data-stu-id="9db2a-159">You can also use predefined **Strategy** settings to find an optimal location.</span></span>

<span data-ttu-id="9db2a-160">สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการสร้างและตั้งค่าคอนฟิกคำสั่งสถานที่ โปรดดูที่ [การสร้างคำสั่งสถานที่](create-location-directive.md)</span><span class="sxs-lookup"><span data-stu-id="9db2a-160">For more information about how to create and configure location directives, see [Create a location directive](create-location-directive.md).</span></span>

### <a name="how-location-directives-work"></a><span data-ttu-id="9db2a-161">คำสั่งสถานที่ทำงานอย่างไร</span><span class="sxs-lookup"><span data-stu-id="9db2a-161">How location directives work</span></span>

<span data-ttu-id="9db2a-162">คำสั่งที่ตั้งจะกำหนด *สถานที่* ที่ควรเบิกสินค้า และ *สถานที่* ที่ควรเก็บสินค้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-162">Location directives determine *where* items should be picked and *where* they should be put.</span></span> <span data-ttu-id="9db2a-163">ระบบจะประเมินคำสั่งสถานที่สำหรับแต่ละบรรทัดงานแล้วเลือกสถานที่ ตามรายละเอียดของรายการงาน</span><span class="sxs-lookup"><span data-stu-id="9db2a-163">The system evaluates a location directive against each work line and then selects a location, based on the work line details.</span></span> <span data-ttu-id="9db2a-164">ระบบจะค้นหาคำสั่งที่ตั้งทั้งหมดที่ตรงกับรายการงานหนึ่งๆ (ตัวอย่างเช่น สำหรับคลังสินค้าที่ถูกต้องและตรงกับการสอบถาม)</span><span class="sxs-lookup"><span data-stu-id="9db2a-164">The system first finds all location directives that match a particular work line (for example, they are for the correct warehouse and match the query).</span></span> <span data-ttu-id="9db2a-165">โดยเรียงตามคำสั่งที่พบ</span><span class="sxs-lookup"><span data-stu-id="9db2a-165">It then sequentially evaluates the directives that it has found.</span></span>

> [!NOTE]
> <span data-ttu-id="9db2a-166">มีกรณีพิเศษที่สถานที่เบิกสินค้าหรือสถานที่เก็บที่เลือกไว้ล่วงหน้า</span><span class="sxs-lookup"><span data-stu-id="9db2a-166">There are special cases where the pick location or put location is pre-selected.</span></span> <span data-ttu-id="9db2a-167">ตัวอย่างเช่น ในระหว่าง _การลงทะเบียนการซื้อ_ การเบิกสินค้าครั้งแรกจะมีอยู่ในที่ตั้งที่มีการลงทะเบียนอยู่เสมอ</span><span class="sxs-lookup"><span data-stu-id="9db2a-167">For example, during _purchase registration_, the first pick is always from the location where the registration occurs.</span></span> <span data-ttu-id="9db2a-168">อีกตัวอย่างหนึ่งคือ *การเคลื่อนย้ายสินค้าคงคลังโดยใช้เท็มเพลต* ซึ่งผู้ปฏิบัติงานคลังสินค้าเลือกสถานที่เบิกสินค้า และจะพบเฉพาะสถานที่ที่มีการนำส่งผ่านคำสั่งสถานที่</span><span class="sxs-lookup"><span data-stu-id="9db2a-168">Another example is *inventory movement by template*, where the warehouse worker selects the pick location, and only the put locations are found through location directives.</span></span>
