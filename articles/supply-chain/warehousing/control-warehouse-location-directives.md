# <a name="control-warehouse-work-by-using-work-templates-and-location-directives"></a>ควบคุมงานคลังสินค้าโดยเท็มเพลตงานและคำสั่งสถานที่

[!include [banner](../includes/banner.md)]

บทความนี้อธิบายวิธีการใช้เท็มเพลตงานและคำสั่งสถานที่ในการกำหนดวิธีและตำแหน่งที่งานจะถูกดำเนินการในคลังสินค้า

คำสั่งที่พนักงานคลังสินค้าได้รับบนอุปกรณ์เคลื่อนที่จะถูกกำหนดโดยเท็มเพลตงานที่คุณตั้งค่าใน Microsoft Dynamics 365 for Finance and Operations เพื่อกำหนดกระบวนการคลังสินค้าและหน้าที่ต่างๆ เท็มเพลตงานกำหนดวิธีดำเนินงานสำหรับกระบวนการคลังสินค้าแต่ละรายการ ด้วยการเชื่อมโยงคำสั่งสถานที่กับเท็มเพลตงาน คุณสามารถช่วยรับประกันว่างานเกิดขึ้นเฉพาะในพื้นที่จริงของคลังสินค้า

## <a name="work-templates"></a>เท็มเพลตงาน
หน้า **เท็มเพลตงาน** ช่วยให้คุณสามารถกำหนดการดำเนินงานที่ต้องดำเนินการในคลังสินค้า โดยปกติ การดำเนินงานคลังสินค้าประกอบด้วยสองการดำเนินการ: ผู้ปฏิบัติงานคลังสินค้าเลือกสินค้าคงคลังคงเหลือในสถานที่หนึ่งและหลังจากนั้นก็ใสนำสินค้าคงคลังที่เลือกมาไปลงไว้อีกสถานที่หนึ่ง 

เท็มเพลตงานประกอบด้วยหัวข้อและรายการที่เกี่ยวข้อง แต่ละเท็มเพลตงานมีไว้สำหรับ *ชนิดใบสั่งงาน* ที่เฉพาะเจาะจงชนิดเดียว ชนิดใบสั่งงานหลายๆชนิดเชื่อมโยงกับเอกสารต้นทาง เช่นใบสั่งซื้อหรือใบสั่งขายต่าง ๆ อย่างไรก็ตาม ชนิดลำดับงานอื่นๆที่แสดงถึงกระบวนการคลังสินค้าที่แยกต่างหาก เช่นการตรวจนับตามรอบ *รหัสกลุ่มงาน* ช่วยให้คุณสามารถจัดระเบียบงานให้เป็นกลุ่ม 

การตั้งค่าในคำอธิบายหัวข้องานสามารถใช้เพื่อกำหนดว่าเมื่อใดที่งานชิ้นใหม่ควรถูกสร้าง ตัวอย่างเช่น คุณสามารถตั้งค่าจำนวนสูงสุดของการเบิกสินค้าและเวลาที่คาดว่าจะเบิกสินค้า จากนั้น ถ้างานสำหรับกระบวนการเบิกสินค้าสำหรับใบสั่งขายนั้นเกินอย่างใดอย่างหนึ่งของค่าเหล่านี้ งานนั้นจะถูกแบ่งออกเป็นงานสองชิ้น 

รายการงานแสดงถึงหน้าที่จริงที่จำเป็นในกระบวนการการดำเนินงาน ตัวอย่างเช่น สำหรับกระบวนการคลังสินค้าขาออกอาจมีรายการงานสำหรับการเบิกสินค้าภายในคลังสินค้าและอีกรายการหนึ่งสำหรับการย้ายสินค้าเหล่านั้นไปยังพื้นที่จัดเตรียมสินค้า จากนั้นสามารถมีรายการเพิ่มเติมสำหรับการเบิกสินค้าจากการจัดเตรียม และอีกรายการหนึ่งสำหรับการย้ายสินค้าไปยังรถบรรทุกที่เป็นส่วนหนึ่งของกระบวนการบรรทุกสินค้า คุณสามารถตั้งค่า *รหัสคำสั่ง* บนรายการเท็มเพลตงานได้ รหัสคำสั่งจะเชื่อมโยงกับคำสั่งสถานที่และดังนั้นจึงช่วยรับประกันว่างานคลังสินค้าจะดำเนินการในสถานที่ที่ถูกต้องในคลังสินค้า 

คุณสามารถตั้งค่าการสอบถามเพื่อควบคุมเมื่อเท็มเพลตงานโดยเฉพาะนั้นถูกใช้ ตัวอย่างเช่น คุณสามารถกำหนดข้อจำกัดเพื่อเท็มเพลตงโดยเฉพาะนั้นจะถูกใช้สำหรับงานในคลังสินค้าเฉพาะเท่านั้น อีกทางหนึ่งคือ คุณอาจมีเท็มเพลตต่างๆที่ใช้ในการสร้างงานสำหรับการประมวลผลใบสั่งขายขาออก ขึ้นอยู่กับช่องทางการขาย ระบบใช้ฟิลด์ **หมายเลขลำดับ** เพื่อกำหนดลำดับที่เท็มเพลตงานมีอยู่นั้นถูกประเมิน ดังนั้น ถ้าคุณมีแบบสอบถามที่ระบุเฉพาะสำหรับเท็มเพลตงานเฉพาะ คุณควรกำหนดหมายเลขลำดับต่ำให้แก่เท็มเพลตงานนั้น จากนั้นแบบสอบถามนั้นจะถูกประเมินก่อนที่จะสอบถามอื่นๆ คำสั่งที่ธรรมดาขึ้น 

เพื่อหยุดหรือหยุดกระบวนการงานชั่วคราว คุณสามารถใช้การตั้งค่า **หยุดงาน** บนรายการงานได้ ในกรณีดังกล่าว ผู้ปฏิบัติงานที่กำลังทำงานจะไม่ถูกขอให้ดำเนินการขั้นตอนถัดไปของรายการงาน เพื่อเลื่อนไปยังขั้นตอนถัดไป ที่ผู้ปฏิบัติงานหรือผู้ปฏิบัติงานผู้อื่นต้องเลือกงานอีกครั้ง คุณยังสามารถแยกหน้าภายในชิ้นงาน โดยใช้อื่นซึ่งไม่ใช่ *รหัสคลาสงาน* ในรายการเท็มเพลงานงได้

## <a name="location-directives"></a>คำสั่งสถานที่
คำสั่งสถานที่มีกฎซึ่งช่วยให้ระบุการเบิกสินค้าและกำหนดสถานที่เก็บสำหรับการย้ายสินค้าคงคลัง ตัวอย่างเช่น ในธุรกรรมใบสั่งขาย คำสั่งสถานกำหนดจุดที่จะเบิกสินค้า และจุดที่จะวางสินค้าที่เบิก คำสั่งสถานที่ประกอบด้วยหัวข้อและรายการที่เกี่ยวข้อง และคุณสร้างหัวข้อและรายการที่เกี่ยวข้องในหน้า **คำสั่งสถานที่**. 

บนหัวข้อ แต่ละคำสั่งสถานต้องสัมพันธ์กับ *ชนิดใบสั่งงาน* ที่ระบุชนิดของธุรกรรมสินค้าคงคลังที่คำสั่งจะถูกใช้สำหรับ เช่นใบสั่งขาย การเติมสินค้า หรือการเบิกวัตถุดิบ 
          *ชนิดงาน* ระบุว่าจะใช้คำสั่งสถานที่สำหรับการเบิกสินค้าหรืองานการขนย้าย หรือสำหรับกระบวนการคลังสิงค้าอื่น เช่นการนับหรือการเปลี่ยนแปลงสถานะสินค้าคงคลัง คุณต้องระบุ *ไซต์* และ *คลังสินค้า* 
          *รหัสคำสั่ง* ที่คุณระบุบนหัวข้อสามารถใช้เพื่อเชื่อมโยงคำสั่งสถานที่ไปยังเท็มเพลตงานอย่างน้อยหนึ่งเท็มเพลตหรือมากกว่านั้น 

สำหรับเท็มเพลตงาน คุณสามารถตั้งค่าคำสั่งเพื่อกำหนดว่าเมื่อไหร่คำสั่งที่สถานที่ต้องถูกใช้งาน ตัวอย่างเช่น คุณสามารถระบุว่าเมื่อระบบอีคอมเมิร์ซเป็นต้นกำเนิดของใบสั่งขาย สินค้าคงคลังต้องถูกเลือกจากพื้นที่ที่จัดสรรไว้ในคลังสินค้า ระบบใช้ฟิลด์ **หมายเลขลำดับ** เพื่อกำหนดลำดับที่คำสั่งสถานที่ที่มีอยู่ที่จะประเมิน 

รายการคำสั่งสถานที่กำหนดข้อจำกัดการใช้กฎการค้นหาสถานที่เพิ่มเติม คุณสามารถระบุปริมาณต่ำสุดและปริมาณสูงสุดที่คำสั่งควรใช้กับ และคุณสามารถระบุว่าควรมีคำสั่งสำหรับเฉพาะหน่วยสินค้าคงคลังหรือไม่ ตัวอย่างเช่น ถ้าหน่วยวัดเป็นแท่นวางสินค้า สินค้าในแท่นวางสินค้าสามารถวางในตำแหน่งที่กำหนดไว้ได้ คุณยังสามารถระบุว่าปริมาณที่จะสามารถถูกแบ่งสำหรับหลายสถานที่ได้หรือไม่ เช่นเดียวกับที่หัวข้อคำสั่งของสถานที่ แต่ละรายการคำสั่งสถานที่มีหมายเลขลำดับที่กำหนดลำดับที่รายการจะถูกประเมิน 

คำสั่งสถานที่มีหนึ่งระดับของรายละเอียดเพิ่มเติม: *การดำเนินคำสั่งสถานที่* คุณสามารถกำหนดหลายการดำเนินคำสั่งสถานที่สำหรับแต่ละรายการ อีกครั้งหนึ่ง หมายเลขลำดับจะใช้เพื่อกำหนดลำดับที่การดำเนินการนั้นจะถูกประเมิน ในระดับนี้ คุณสามารถตั้งค่าคำสั่งเพื่อกำหนดวิธีเพื่อหาสถานที่ที่ดีที่สุดในคลังสินค้า คุณยังสามารถใช้การตั้งค่า **กลยุทธ์**ไว้ล่วงหน้าเพื่อหาสถานที่ที่เหมาะสมได้

### <a name="example-of-the-use-of-location-directives"></a>ตัวอย่างของการใช้คำสั่งสถานที่

สำหรับตัวอย่างนี้ เราจะพิจารณาว่ากระบวนการใบสั่งซื้อที่คำสั่งสถานที่ต้องค้นหากำลังการผลิตที่ว่างอยู่ภายในคลังสินค้าสำหรับสินค้าคงคลังที่เพิ่งถูกลงทะเบียน ณ ท่าสินค้ารับสินค้า ก่อนอื่น เราต้องการลองค้นหากำลังการผลิตที่ว่างอยู่ภายในคลังสินค้าด้วยการรวมเข้ากับปริมาณสินค้าคงคลังคงเหลือที่มีอยู่ ถ้าไม่สามารถรวมได้ เราจึงต้องการค้นหาสถานที่ที่ว่าง 

สำหรับสถานการณ์จำลองนี้ เราต้องกำหนดสองการดำเนินคำสั่งสถานที่ การดำเนินการแรกในลำดับต้องใช้ยุทธ์ **รวม** และการดำเนินการที่สองควรใช้กลยุทธ์ **สถานที่ว่างโดยไม่มีงานขาเข้า** เว้นแต่ว่าเรากำหนดการดำเนินการที่สามเพื่อจัดการสถานการณ์จำลองมากเกินไป สองผลลัพธ์อาจเป็นไปได้เมื่อไม่มีกำลังการผลิตในคลังสินค้า: งานสามารถถูกสร้างได้แม้ไม่มีการระบุสถานที่ หรือกระบวนการสร้างงานอาจล้มเหลวได้ ผลลัพธ์จะถูกกำหนดตามการตั้งค่าบนหน้า **คำสั่งสถานที่ล้มเหลว** ซึ่งคุณสามารถตัดสินใจว่าจะเลือกตัวเลิอก **หยุดงานเมื่อคำสั่งสถานที่ล้มเหลว** สำหรับแต่ละชนิดใบสั่งผลิตหรือไม่
