# <a name="replenishment"></a><span data-ttu-id="64411-103">การเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-103">Replenishment</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="64411-104">หัวข้อนี้อธิบายถึงกลยุทธ์การเติมสินค้าที่พร้อมใช้งานสำหรับคลังสินค้าที่ใช้ฟังก์ชันการทำงานที่พร้อมใช้งานในการจัดการคลังสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-104">This topic describes the replenishment strategies that are available for warehouses that use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="64411-105">ข้อมูลในหัวข้อนี้ไม่ใช้กับโซลูชันคลังสินค้าที่พร้อมใช้งานในการบริหารสินค้าคงคลัง</span><span class="sxs-lookup"><span data-stu-id="64411-105">The information in this topic doesn't apply to the warehousing solution that is available in Inventory management.</span></span>

<span data-ttu-id="64411-106">มีกลยุทธ์การเติมสินค้าดังต่อไปนี้ให้เลือก:</span><span class="sxs-lookup"><span data-stu-id="64411-106">The following replenishment strategies are available:</span></span>

- <span data-ttu-id="64411-107">**การเติมสินค้าตามความต้องการของเวฟ**– กลยุทธ์นี้สร้างงานการเพิ่มเติมสินค้าสำหรับใบสั่งหรือการบรรทุกขาออกถ้าสินค้าคงคลังไม่พร้อมใช้งานเมื่อมีเวฟสร้างงาน</span><span class="sxs-lookup"><span data-stu-id="64411-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="64411-108">ตัวอย่างเช่น งานการเติมสินค้าสามารถสร้างได้ถ้าไม่มีปริมาณที่จำเป็นสำหรับใบสั่งขายเมื่อดำเนินการเวฟ</span><span class="sxs-lookup"><span data-stu-id="64411-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
- <span data-ttu-id="64411-109">**การเพิ่มเติมสินค้าต่ำสุด/สูงสุด**– กลยุทธ์นี้ใช้ขีดจำกัดการเก็บสต็อกต่ำสุดและสูงสุดเพื่อกำหนดเวลาการเติมสินค้าในสถานที่</span><span class="sxs-lookup"><span data-stu-id="64411-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="64411-110">เกณฑ์สินค้าและสถานที่จะกำหนดสินค้าคงคลังที่ประเมินสำหรับการเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="64411-111">เท็มเพลตการเพิ่มเติมสินค้าต่ำสุด/สูงสุดคือกลไกหลักของการรักษาระดับที่ดีที่สุดในสถานที่เบิกสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="64411-112">เพื่อช่วยให้มั่นใจว่ามีสินค้าคงคลังสำหรับเบิกสินค้าที่เพียงพอเพื่อตอบสนองความต้องการของเวฟ คุณสามารถใช้การเติมสินค้าตามความต้องการเป็นส่วนเพิ่มเติมระหว่างวงจรการเพิ่มเติมสินค้าต่ำสุด/สูงสุดได้</span><span class="sxs-lookup"><span data-stu-id="64411-112">To help guarantee that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
- <span data-ttu-id="64411-113">**การเพิ่มเติมสินค้าตามความต้องการของจำนวนงานในศูนย์การผลิต**– กลยุทธ์นี้รวมความต้องการสำหรับการบรรทุกหลายรายการ และสร้างงานการเพิ่มเติมสินค้าที่จำเป็นในการจัดเก็บสถานที่เบิกสินค้าที่เกี่ยวข้อง</span><span class="sxs-lookup"><span data-stu-id="64411-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="64411-114">กลยุทธ์นี้ช่วยรับประกันว่าการบรรทุกที่สร้างขึ้นสามารถเบิกสินค้าได้ในคลังสินค้าหลังจากที่นำออกใช้แล้ว</span><span class="sxs-lookup"><span data-stu-id="64411-114">This strategy helps guarantee that the loads that are created can be picked in the warehouse after they are released.</span></span>
- <span data-ttu-id="64411-115">**การเติมสินค้าทันที** – กลยุทธ์นี้ช่วยเพิ่มสินค้าคงคลังก่อนการรันเวฟหากการปันส่วนล้มเหลวสำหรับรายการคำสั่งสถานที่ที่มีเท็มเพลตการเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-115">**Immediate replenishment** – This strategy replenishes inventory before a wave is run if allocation fails for a location directive line that has a replenishment template.</span></span> 

<span data-ttu-id="64411-116">กลยุทธ์ทั้งสี่นี้สร้างงานการเติมสินค้าตามเท็มเพลตการเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-116">All four strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="64411-117">การเพิ่มเติมสินค้าตามความต้องการของเวฟ</span><span class="sxs-lookup"><span data-stu-id="64411-117">Wave demand replenishment</span></span>
<span data-ttu-id="64411-118">การเติมสินค้าตามความต้องการของเวฟสร้างงานการเติมสินค้าตามความต้องการ ถ้าปริมาณที่จำเป็นสำหรับใบสั่งผลิต คัมบัง ใบสั่งขาออก หรือการบรรทุกขาออกไม่พร้อมใช้งานเมื่อมีเวฟสร้างงาน</span><span class="sxs-lookup"><span data-stu-id="64411-118">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders, or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="64411-119">เท็มเพลตการเพิ่มเติมสินค้าประกอบด้วยข้อมูลเกี่ยวกับเงื่อนไขสินค้า หน่วยวัด การเพิ่มความต้องการ และสถานที่</span><span class="sxs-lookup"><span data-stu-id="64411-119">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="64411-120">คำสั่งสถานที่จะถูกใช้เพื่อกำหนดว่าสถานที่ใดควรมีการเพิ่มเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-120">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="64411-121">คุณเชื่อมโยงคำสั่งสถานที่เหล่านี้กับเท็มเพลตการเพิ่มเติมสินค้าได้โดยใช้ฟิลด์ **รหัสคำสั่ง**</span><span class="sxs-lookup"><span data-stu-id="64411-121">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="64411-122">ถ้าไม่ได้ตั้งค่าฟิลด์ **รหัสคำสั่ง** แบบสอบถามจะถูกใช้เพื่อกำหนดว่าควรใช้คำสั่งสถานที่ใด</span><span class="sxs-lookup"><span data-stu-id="64411-122">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="64411-123">โปรดทราบว่าถ้าไม่ได้ระบุรหัสคำสั่งในเท็มเพลตเพิ่มเติมสินค้า และคำสั่งสถานที่มีรหัสคำสั่ง คำสั่งสถานที่จะถูกละเว้น แม้ว่าแบบสอบถามในคำสั่งสถานที่จะถูกต้อง</span><span class="sxs-lookup"><span data-stu-id="64411-123">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="64411-124">คำสั่งสถานที่เบิกสินค้าจะถูกใช้เพื่อกำหนดตำแหน่งที่จะรับสินค้าคงคลังสำหรับการเพิ่มเติมสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-124">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="64411-125">นอกจากการสร้างเท็มเพลตแล้ว คุณจะต้องระบุการเพิ่มเติมสินค้าบางอย่างในเท็มเพลตเวฟ</span><span class="sxs-lookup"><span data-stu-id="64411-125">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="64411-126">เท็มเพลตเวฟควรประกอบด้วยขั้นตอนของเวฟสำหรับการเติมสินค้าที่ใช้งานเฉพาะเมื่อการปันส่วนสินค้าไม่ประสบความสำเร็จ</span><span class="sxs-lookup"><span data-stu-id="64411-126">The wave template should contain a wave step for replenishment that is run only if an item isn't successfully allocated.</span></span> <span data-ttu-id="64411-127">ขั้นตอนของเวฟการเพิ่มเติมสินค้าใช้รหัสขั้นตอนของเวฟเพื่อกำหนดว่าควรใช้เท็มเพลตการเพิ่มเติมสินค้าใด</span><span class="sxs-lookup"><span data-stu-id="64411-127">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="64411-128">นอกเหนือจากที่มีขั้นตอนของเวฟสำหรับการเติมสินค้าแล้ว คุณจะต้องแน่ใจว่าได้เลือก **เติมสินค้า** ไว้ในส่วน **วิธี** ของเท็มเพลตเวฟ</span><span class="sxs-lookup"><span data-stu-id="64411-128">In addition to having a wave step for replenishment, you must make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span>

<span data-ttu-id="64411-133">การเพิ่มเติมสินค้าตามความต้องการได้รับการสนับสนุนสำหรับใบสั่งขาย ใบสั่งโอนย้าย ใบสั่งผลิต และคัมบัง</span><span class="sxs-lookup"><span data-stu-id="64411-133">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="64411-134">การเพิ่มเติมสินค้าต่ำสุด/สูงสุด</span><span class="sxs-lookup"><span data-stu-id="64411-134">Min/Max replenishment</span></span>
<span data-ttu-id="64411-135">ในการเพิ่มเติมสินค้าต่ำสุด/สูงสุด สินค้าคงคลังจะถูกเติมเพื่อให้มีค่าระหว่างขีดจำกัดต่ำสุดและสูงสุดที่ได้ตั้งค่าไว้</span><span class="sxs-lookup"><span data-stu-id="64411-135">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="64411-136">โดยทั่วไปแล้ว กระบวนการนี้เกิดขึ้นวันละหนึ่งครั้งเพื่อช่วยรับประกันว่าสถานที่เบิกสินค้าทั้งหมดจะถูกเติมจนถึงระดับสูงสุดก่อนที่จะเริ่มต้นการเบิกสินค้า</span><span class="sxs-lookup"><span data-stu-id="64411-136">Typically, this process occurs one time every day, to help guarantee that all picking locations are filled to the maximum level before picking starts.</span></span>
