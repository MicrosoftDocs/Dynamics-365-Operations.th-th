    <span data-ttu-id="76e49-222">ถึงแม้จะมีแนวคิดของกลุ่มสำหรับขนาด สี และลักษณะ (กลุ่มสี กลุ่มขนาด และกลุ่มลักษณะ) แต่ไม่มีแนวคิดของกลุ่มรุ่นอยู่</span><span class="sxs-lookup"><span data-stu-id="76e49-222">Although the concept of groups exists for size, color, and style (color group, size group, and style group), no version group concept exists.</span></span> <span data-ttu-id="76e49-223">กลุ่มจะช่วยให้คุณสามารถควบคุมค่าที่เกี่ยวข้องเพื่อให้ ตัวอย่างเช่น เมื่อคุณกำหนดกลุ่มสีให้กับผลิตภัณฑ์ ผลิตภัณฑ์สามารถใช้สีทั้งหมดในกลุ่มสีดังกล่าวได้</span><span class="sxs-lookup"><span data-stu-id="76e49-223">Groups let you predefine the applicable values so that when, for example, you assign a color group to a product, the product can use all the colors in that color group.</span></span> <span data-ttu-id="76e49-224">แนวคิดนี้ไม่นำมาใช้กับมิติรุ่น เนื่องจากไม่ได้กำหนดรุ่นที่ผลิตภัณฑ์จะใช้ไว้ล่วงหน้าเมื่อสร้างผลิตภัณฑ์</span><span class="sxs-lookup"><span data-stu-id="76e49-224">This concept doesn't apply to the version dimension, because the versions that a product takes aren't predefined when the product is created.</span></span> <span data-ttu-id="76e49-225">แต่รุ่นจะมีการสร้างขึ้นในระหว่างวงจรการใช้งานผลิตภัณฑ์ตามที่จำเป็น</span><span class="sxs-lookup"><span data-stu-id="76e49-225">Instead, versions are created during the lifecycle of the product, as they are required.</span></span> <span data-ttu-id="76e49-226">โดยทั่วไปแล้ว ถ้ารูปแบบ ความพอดี และฟังก์ชันของผลิตภัณฑ์ยังคงเหมือนเดิม คุณสามารถสร้างรุ่นใหม่แทนผลิตภัณฑ์ใหม่ได้</span><span class="sxs-lookup"><span data-stu-id="76e49-226">Typically, if the form, fit, and function of the product remain the same, you create a new version instead of a new product.</span></span>

- <span data-ttu-id="76e49-227">**คำแนะนำผลิตภัณฑ์ย่อยทำงานอย่างที่เป็นอยู่ในตอนนี้**</span><span class="sxs-lookup"><span data-stu-id="76e49-227">**Product variant suggestions work as they currently do.**</span></span>

    <span data-ttu-id="76e49-228">คำแนะนำผลิตภัณฑ์ย่อยจะสร้างข้อเสนอแนะสำหรับค่ามิติรุ่นทั้งหมดเช่นเดียวกับที่ทำกับมิติอื่นๆ</span><span class="sxs-lookup"><span data-stu-id="76e49-228">Product variant suggestions will create suggestions for all version dimension values, just as they do for other dimensions.</span></span> <span data-ttu-id="76e49-229">กระบวนการในการสร้างและนำผลิตภัณฑ์ที่กำหนดรุ่นออกใช้จะเหมือนกับที่ใช้สำหรับผลิตภัณฑ์ที่ใช้มิติอื่น</span><span class="sxs-lookup"><span data-stu-id="76e49-229">The process of creating and releasing versioned products is the same as it is for products that use other dimensions.</span></span> <span data-ttu-id="76e49-230">เมื่อคุณสร้างผลิตภัณฑ์ที่กำหนดรุ่น รุ่นแรก (V1) จะมีการสร้างเป็นมิติของผลิตภัณฑ์และตัวแปรจะถูกนำออกใช้</span><span class="sxs-lookup"><span data-stu-id="76e49-230">When you create a versioned product, the first version (V1) will be created as a product dimension, and the variants will be released.</span></span> <span data-ttu-id="76e49-231">เมื่อผลิตภัณฑ์เปลี่ยนแปลงและจำเป็นต้องมีรุ่นใหม่ จะมีการเพิ่มค่ารุ่นใหม่ (V2) และจะมีการนำตัวแปรที่จำเป็นออกใช้</span><span class="sxs-lookup"><span data-stu-id="76e49-231">As the product changes and a new version is needed, the new version value (V2) will be added, and the required variants will be released.</span></span> <span data-ttu-id="76e49-232">ไม่มีความคาดหวังว่าจะมีการสร้างรุ่นทั้งหมด (V1, V2 และ V3) ไว้ล่วงหน้าสำหรับผลิตภัณฑ์</span><span class="sxs-lookup"><span data-stu-id="76e49-232">There is no expectation  that all the versions (V1, V2, and V3) will be created in advance for the product.</span></span>
