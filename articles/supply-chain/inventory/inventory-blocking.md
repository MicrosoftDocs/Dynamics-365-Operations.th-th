# <a name="inventory-blocking"></a>การบล็อคสินค้าคงคลัง

[!include [banner](../includes/banner.md)]

หัวข้อนี้แสดงภาพรวมของการบล็อคสินค้าคงคลัง ซึ่งเป็นส่วนหนึ่งของกระบวนการตรวจสอบคุณภาพใน Supply Chain Management คุณสามารถใช้การบล็อคสินค้าคงคลังเพื่อป้องกันไม่ให้สินค้ามีการประมวลผลหรือใช้

คุณสามารถบล็อคสินค้าคงคลังได้ดังวิธีต่อไปนี้:

- ด้วยตนเอง
- โดยการสร้างใบสั่งตรวจสอบคุณภาพ
- โดยใช้กระบวนการที่สร้างใบสั่งตรวจสอบคุณภาพ
- โดยใช้การบล็อคสถานะสินค้าคงคลัง

## <a name="blocking-items-manually"></a>การบล็อคสินค้าด้วยตนเอง

คุณสามารถบล็อคปริมาณสินค้าด้วยการสร้างธุรกรรมในหน้า **การบล็อคสินค้าคงคลัง** ได้ เฉพาะสินค้าที่พร้อมใช้ที่เป็นสินค้าคงคลังคงเหลือทสามารถถูกบล็อคได้ด้วยตนเอง สำหรับปริมาณที่ถูกบล็อคด้วยตนเอง คุณต้องตัดสินใจว่ากิจกรรมการวางแผนจะรวมใบรับสินค้าที่คาดไว้เป็นปริมาณคงเหลือคาดไว้หรือไม่ การรับสินค้าที่คาดไว้เป็นสินค้าที่ถูกบล็อคที่คุณคาดว่าจะพร้อมใช้งาน ซึ่งอยู่ในรูปของสินค้าคงคลังคงเหลือหลังจากการตรวจสอบเสร็จสมบูรณ์ คุณสามารถเก็บรักษาวันที่คาดไว้ โดยค่าเริ่มต้น ตัวเลือก **การรับสินค้าที่คาดไว้** จะถูกเลือกสำหรับสินค้าที่ถูกบล็อคโดยใช้ใบสั่งตรวจสอบคุณภาพ คุณสามารถยกเลิกการบล็อคปริมาณด้วยตนเอง ด้วยการลบธุรกรรมในหน้า **การบล็อคสินค้าคงคลัง**

## <a name="blocking-items-by-creating-a-quality-order"></a>การบล็อคสินค้าโดยการสร้างใบสั่งตรวจสอบคุณภาพ

คุณสามารถระบุสินค้าที่ต้องถูกตรวจสอบได้โดยการสร้างใบสั่งตรวจสอบคุณภาพในหน้า **ใบสั่งตรวจสอบคุณภาพ** เมื่อคุณสร้างใบสั่งตรวจสอบคุณภาพ ปริมาณที่คุณระบุไว้สำหรับสินค้าจะถูกบล็อค แผนการสุ่มตัวอย่างที่เชื่อมโยงกับใบสั่งตรวจสอบคุณภาพควบคุมเฉพาะปริมาณของสินค้าที่ต้องถูกตรวจสอบ ไม่ใช่ปริมาณที่ถูกบล็อค ปริมาณที่ถูกป้อนบนใบสั่งตรวจสอบคุณภาพคือปริมาณที่ถูกบล็อค โดยไม่คำนึงถึงปริมาณที่การระบุแผนการสุ่มตัวอย่างควรถูกส่งไปเพื่อการตรวจสอบ

> [!NOTE]
> การใช้ทั้งวันหมดอายุของชุดงานและการปิดกั้นคุณลักษณะการทำงานสถานะของสินค้าคงคลังไม่ได้รับการสนับสนุนโดยการวางแผนหลัก การทำเช่นนี้อาจส่งผลให้มียกเว้นซ้ำซ้อนของปริมาณคงคลังคงเหลือซึ่งอาจเกิดขึ้นในระหว่างการวางแผนหลัก เราขอแนะนำให้คุณพึ่งพารหัสการโอนการครอบครองชุดงานแทนสถานะของสินค้าคงคลังสำหรับการปิดกั้นชุดงานที่หมดอายุ

## <a name="blocking-items-by-using-a-process-that-generates-a-quality-order"></a>การบล็อคสินค้าโดยใช้กระบวนการที่สร้างใบสั่งตรวจสอบคุณภาพ

ถ้ากระบวนการตรวจสอบคุณภาพระบุว่าต้องตรวจสอบสินค้า ปริมาณของสินค้าจะถูกบล็อคโดยอัตโนมัติ ดังนั้น เมื่อมีการสร้างใบสั่งตรวจสอบคุณภาพโดยอัตโนมัติ แผนการสุ่มตัวอย่างสินค้าที่เชื่อมโยงกับใบสั่งตรวจสอบคุณภาพจะควบคุมทั้งปริมาณของสินค้าที่ถูกบล็อคและปริมาณที่ต้องถูกตรวจสอบ ถ้าตัวเลือก **การบล็อคทั้งหมด** ในหน้า **การสุ่มตัวอย่างสินค้า** ถูกเลือก ปริมาณทั้งหมดของ ตัวอย่างเช่น รายการใบสั่งซื้อจะถูกบล็อคในระหว่างการตรวจสอบ โดยไม่คำนึงถึงปริมาณการสุ่มตัวอย่างสินค้า

### <a name="example"></a>ตัวอย่าง

ในตัวอย่างต่อไปนี้ ใบสั่งตรวจสอบคุณภาพจะถูกสร้างขึ้นเมื่อมีการลงรายการบัญชีบันทึกการจัดส่งใบสั่งซื้อ ในหน้า **การเชื่อมโยงคุณภาพ** คุณระบุว่าการลงรายการบัญชีบันทึกการจัดส่งใบสั่งซื้อเป็นกระบวนการที่เรียกใช้งานใบสั่งตรวจสอบคุณภาพ

|การตั้งค่า |การดำเนินการของผู้ใช้ |ผลลัพธ์ |
|----|---|---|
| การเชื่อมโยงคุณภาพระบุว่าต้องมีการสร้างใบสั่งตรวจสอบคุณภาพ เมื่อมีการลงรายการบัญชีบันทึกการจัดส่งใบสั่งซื้อ การตั้งค่าการสุ่มตัวอย่างสินค้าของใบสั่งตรวจสอบคุณภาพระบุว่าต้องตรวจสอบ 10% ของปริมาณในรายการใบสั่งซื้อ นอกจากนี้ เนื่องจากตัวเลือก **การบล็อคทั้งหมด** ได้เลือกการตั้งค่าการสุ่มตัวอย่างสินค้า ปริมาณทั้งหมดของรายการใบสั่งซื้อต้องถูกบล็อคในระหว่างการตรวจสอบอย่างละเอียด โดยไม่คำนึงถึงปริมาณที่ถูกส่งไปตรวจสอบ | ลงรายการบัญชีบันทึกการจัดส่ง | มีการสร้างใบสั่งตรวจสอบคุณภาพ 10% ของปริมาณใบสั่งซื้อสำหรับสินค้าที่ถูกส่งไปตรวจสอบอย่างละเอียด ปริมาณทั้งหมดของบรรทัดใบสั่งซื้อทั้งหมดถูกบล็อค |

## <a name="blocking-items-by-using-inventory-status-blocking"></a>การบล็อคสินค้าโดยใช้การบล็อคสถานะสินค้าคงคลัง

คุณสามารถระบุว่าสถานะสินค้าคงคลังใดเป็นสถานะการบล็อคโดยใช้พารามิเตอร์ **การบล็อคสินค้าคงคลัง** บนหน้า **สถานะสินค้าคงคลัง**  คุณไม่สามารถใช้สถานะสินค้าคงคลังเป็นสถานะการบล็อคสำหรับใบสั่งผลิต ใบสั่งขาย ใบสั่งโอนย้าย ธุรกรรมขาออก หรือการรวมโครงการได้ สำหรับงานขาออก ใช้สินค้าที่มีสถานะสินค้าคงคลังที่พร้อมใช้งาน ถ้าสินค้ามีสถานะเป็น **ไม่ทำงาน** และการวางแผนหลักมีการดำเนินงานกับสินค้าเหล่านั้น สินค้าจะถูกพิจารณาเป็นการขาด และสินค้าคงคลังจะมีการเติมโดยอัตโนมัติ

## <a name="take-care-when-blocking-items-that-use-both-inventory-status-blocking-and-quality-order-blocking"></a>ดูแลเมื่อบล็อคสินค้าที่ใช้ทั้งการบล็อคสถานะสินค้าคงคลังและการบล็อคใบสั่งตรวจสอบคุณภาพ

คุณสามารถสร้างใบสั่งตรวจสอบคุณภาพที่เชื่อมโยงกับสินค้าคงคลังที่มีสถานะสินค้าคงคลัง ซึ่งมีพารามิเตอร์ **การบล็อคสินค้าคงคลัง** เปิดใช้งานอยู่ ในกรณีนี้ ใบสั่งตรวจสอบคุณภาพจะสร้างเรกคอร์ดการบล็อคสินค้าคงคลังเพิ่มเติมจากใบสั่งที่สร้างโดยสถานะสินค้าคงคลัง เนื่องจากการบล็อคสินค้าคงคลังของใบสั่งตรวจสอบคุณภาพจะมีการเปิดใช้งานพารามิเตอร์ **การรับสินค้าที่คาดไว้** การบล็อคนี้จะสร้างธุรกรรม *สินค้าคงคลังที่สั่งซึ่งจองไว้* ซึ่งถูกบล็อคโดยสถานะสินค้าคงคลังด้วย ชุดข้อมูลนี้อาจเป็นเรื่องที่ยากที่จะเข้าใจความหมายของธุรกรรมสินค้าคงคลังที่สร้างขึ้น เนื่องจากข้อมูลจะปรากฏแม้ว่าปริมาณทั้งหมดที่ถูกบล็อคจะเกินปริมาณสินค้าคงคลังคงเหลือรวม ให้เราตรวจสอบธุรกรรมในตัวอย่างของการรับสินค้า 10 ชิ้น ของรายการ A0001 ด้วยใบสั่งตรวจสอบคุณภาพที่สร้างกับตัวอย่าง 1 ชิ้น ลักษณะการทำงานนี้ยังขึ้นอยู่กับว่าตัวเลือก **สำรองสินค้าที่สั่ง** ถูกเปิดใช้งานบนหน้า **พารามิเตอร์การจัดการสินค้าคงคลังและคลังสินค้า** หรือไม่

>[!NOTE]
>ถ้าคุณกำลังใช้การบล็อคสถานะสินค้าคงคลังและใบสั่งตรวจสอบคุณภาพร่วมกัน เราขอแนะนาเป็นอย่างยิ่งให้เปิดใช้งานตัวเลือก **สำรองสินค้าที่สั่ง**
