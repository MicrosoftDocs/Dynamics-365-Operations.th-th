# <a name="quality-management-test-groups"></a><span data-ttu-id="b6a35-103">กลุ่มการทดสอบการจัดการคุณภาพ</span><span class="sxs-lookup"><span data-stu-id="b6a35-103">Quality management test groups</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="b6a35-104">หัวข้อนี้จะอธิบายวิธีการสร้างกลุ่มทดสอบ เพื่อให้สามารถใช้การทดสอบหลายรายการกับใบสั่งตรวจสอบคุณภาพใน Microsoft Dynamics 365 Supply Chain Management</span><span class="sxs-lookup"><span data-stu-id="b6a35-104">This topic describes how to create test groups, so that multiple tests can be used with quality orders in Microsoft Dynamics 365 Supply Chain Management.</span></span>

<span data-ttu-id="b6a35-105">คุณใช้หน้า **กลุ่มทดสอบ** เพื่อตั้งค่า แก้ไข และดูกลุ่มการทดสอบ ตลอดจนการทดสอบต่างๆ ที่กำหนดให้กับกลุ่มการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-105">You use the **Test groups** page to set up, edit, and view test groups and the individual tests that are assigned to them.</span></span> <span data-ttu-id="b6a35-106">ส่วนของหน้าด้านบนจะแสดงกลุ่มการทดสอบ และส่วนด้านล่างจะแสดงการทดสอบที่กำหนดให้กับกลุ่มการทดสอบที่เลือก</span><span class="sxs-lookup"><span data-stu-id="b6a35-106">The upper part of the page shows the test groups, and the lower part shows the tests that are assigned to a selected test group.</span></span>

<span data-ttu-id="b6a35-107">คุณกำหนดนโยบายต่างๆ ให้กับกลุ่มการทดสอบ เช่น แผนการสุ่มตัวอย่าง ระดับคุณภาพที่ยอมรับได้ (AQL) และความจำเป็นสำหรับการทดสอบแบบทำลาย</span><span class="sxs-lookup"><span data-stu-id="b6a35-107">You assign several policies to a test group, such as a sampling plan, an acceptable quality level (AQL), and the requirement for destructive testing.</span></span> <span data-ttu-id="b6a35-108">จากนั้น เมื่อคุณกำหนดการทดสอบแต่ละรายการกับกลุ่มการทดสอบ คุณต้องกำหนดข้อมูลเพิ่มเติม เช่น ลำดับ เอกสาร และวันที่ที่มีผลใช้</span><span class="sxs-lookup"><span data-stu-id="b6a35-108">Then, when you assign an individual test to a test group, you define additional information, such as the sequence, documents, and validity dates.</span></span> <span data-ttu-id="b6a35-109">สำหรับการทดสอบเชิงปริมาณ ข้อมูลที่คุณกำหนดจะประกอบด้วยค่าการประเมินที่ยอมรับได้</span><span class="sxs-lookup"><span data-stu-id="b6a35-109">For a quantitative test, the information that you define also includes the acceptable measurement values.</span></span> <span data-ttu-id="b6a35-110">สำหรับการทดสอบเชิงคุณภาพ ข้อมูลนี้รวมถึงตัวแปรทดสอบและผลลัพธ์เริ่มต้น</span><span class="sxs-lookup"><span data-stu-id="b6a35-110">For a qualitative test, the information includes the test variable and default outcome.</span></span>

<span data-ttu-id="b6a35-111">กลุ่มการทดสอบที่กำหนดให้กับใบสั่งตรวจสอบคุณภาพกำหนดชุดค่าเริ่มต้นของการทดสอบที่ต้องดำเนินการกับสินค้าที่ระบุ</span><span class="sxs-lookup"><span data-stu-id="b6a35-111">The test group that is assigned to a quality order defines the default set of tests that must be performed on the specified item.</span></span> <span data-ttu-id="b6a35-112">อย่างไรก็ตาม คุณสามารถเพิ่ม ลบ หรือเปลี่ยนการทดสอบสำหรับใบสั่งตรวจสอบคุณภาพ</span><span class="sxs-lookup"><span data-stu-id="b6a35-112">However, you can add, delete, or change tests for the quality order.</span></span> <span data-ttu-id="b6a35-113">ระบบจะรายงานผลการทดสอบสำหรับการทดสอบแต่ละรายการบนใบสั่งตรวจสอบคุณภาพ</span><span class="sxs-lookup"><span data-stu-id="b6a35-113">Test results are reported for each test on a quality order.</span></span>

<span data-ttu-id="b6a35-114">เมื่อคุณกําหนดกลุ่มทดสอบ คุณสามารถเลือกที่จะระบุการสุ่มตัวอย่างสินค้าได้</span><span class="sxs-lookup"><span data-stu-id="b6a35-114">When you define a test group, you can optionally specify an item sampling.</span></span> <span data-ttu-id="b6a35-115">การสุ่มตัวอย่างสินค้าใช้ในการกําหนดจํานวนของผลิตภัณฑ์ที่ต้องทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-115">Item samplings are used to define the amount of the product that must be tested.</span></span> <span data-ttu-id="b6a35-116">สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [การสุ่มตัวอย่างสินค้าการจัดการคุณภาพ](quality-item-sampling.md)</span><span class="sxs-lookup"><span data-stu-id="b6a35-116">For more information, see [Quality management item sampling](quality-item-sampling.md).</span></span> <span data-ttu-id="b6a35-117">คุณยังสามารถระบุว่าการทดสอบในกลุ่มการทดสอบเป็นแบบทำลายหรือไม่</span><span class="sxs-lookup"><span data-stu-id="b6a35-117">You can also indicate whether the tests in a test group are destructive.</span></span> <span data-ttu-id="b6a35-118">ในการทดสอบโดยการทำลาย การสุ่มตัวอย่างสินค้าจะถูกทำลายและปริมาณจะถูกลบออกจากปริมาณคงคลังคงเหลือ</span><span class="sxs-lookup"><span data-stu-id="b6a35-118">In a destructive test, the item sampling is destroyed, and the quantity is removed from the on-hand inventory.</span></span>

## <a name="example-of-a-test-group"></a><span data-ttu-id="b6a35-119">ตัวอย่างของกลุ่มการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-119">Example of a test group</span></span>

<span data-ttu-id="b6a35-120">บริษัทผู้ผลิตแห่งหนึ่งกำหนดกลุ่มการทดสอบสำหรับความผันแปรต่างๆ ของคำแนะนำด้านคุณภาพ </span><span class="sxs-lookup"><span data-stu-id="b6a35-120">A manufacturing company defines a test group for each variation of its quality guidelines.</span></span> <span data-ttu-id="b6a35-121">กลุ่มการทดสอบต่าง ๆ สะท้อนถึงผลต่างในแผนการสุ่มตัวอย่าง ชุดของการทดสอบที่ต้องดำเนินการพร้อมกัน AQL และปัจจัยอื่น ๆ</span><span class="sxs-lookup"><span data-stu-id="b6a35-121">The various test groups reflect differences in the sampling plans, the sets of tests that must be performed together, the AQL, and other factors.</span></span> <span data-ttu-id="b6a35-122">สำหรับการทดสอบเชิงปริมาณ ยังมความแตกต่างของค่าการประเมินที่ยอมรับได้</span><span class="sxs-lookup"><span data-stu-id="b6a35-122">For quantitative tests, there are also differences in the acceptable measurement values.</span></span> <span data-ttu-id="b6a35-123">เพื่อบังคับใช้หลักเกณฑ์ด้านคุณภาพ บริษัทกําหนดกลุ่มการทดสอบให้กับกฎแต่ละกฎที่ใช้ในการสร้างใบสั่งตรวจสอบคุณภาพในหน้า **การเชื่อมโยงคุณภาพ** โดยอัตโนมัติ</span><span class="sxs-lookup"><span data-stu-id="b6a35-123">To enforce its quality guidelines, the company assigns a test group to each rule that is used to automatically generate quality orders on the **Quality associations** page.</span></span> <span data-ttu-id="b6a35-124">และกําหนดกลุ่มการทดสอบให้กับใบสั่งตรวจสอบคุณภาพที่สร้างขึ้นด้วยตนเอง</span><span class="sxs-lookup"><span data-stu-id="b6a35-124">It also assigns a test group to quality orders that are manually created.</span></span>

## <a name="create-a-test-group"></a><span data-ttu-id="b6a35-125">สร้างกลุ่มทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-125">Create a test group</span></span>

<span data-ttu-id="b6a35-126">เมื่อต้องการสร้างกลุ่มการทดสอบ ให้ทำตามขั้นตอนเหล่านี้</span><span class="sxs-lookup"><span data-stu-id="b6a35-126">To create a test group, follow these steps.</span></span>

1. <span data-ttu-id="b6a35-127">ไปที่ **การบริหารสินค้าคงคลัง \> การตั้งค่า \> การควบคุมคุณภาพ \> กลุ่มการทดสอบ**</span><span class="sxs-lookup"><span data-stu-id="b6a35-127">Go to **Inventory management \> Setup \> Quality control \> Test groups**.</span></span>
1. <span data-ttu-id="b6a35-128">ในบานหน้าต่างการดําเนินงาน ให้เลือก **สร้าง** เพื่อเพิ่มแถวลงในกริดในส่วนด้านบนของหน้า</span><span class="sxs-lookup"><span data-stu-id="b6a35-128">On the Action Pane, select **New** to add a row to the grid in the upper part of the page.</span></span> <span data-ttu-id="b6a35-129">จากนั้นให้กำหนดฟิลด์ดังต่อไปนี้สำหรับแถวใหม่:</span><span class="sxs-lookup"><span data-stu-id="b6a35-129">Then set the following fields for the new row:</span></span>

    - <span data-ttu-id="b6a35-130">**กลุ่มการทดสอบ** – ป้อนรหัสหรือชื่อเฉพาะสำหรับกลุ่มการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-130">**Test group** – Enter a unique ID or name for the test group.</span></span>
    - <span data-ttu-id="b6a35-131">**คำอธิบาย** – ป้อนคำอธิบายโดยละเอียดของกลุ่มการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-131">**Description** – Enter a detailed description of the test group.</span></span>
    - <span data-ttu-id="b6a35-132">**ระดับคุณภาพที่ยอมรับได้** – ป้อนเปอร์เซ็นต์รวมของผลการทดสอบที่ต้องผ่านเพื่อให้ถือว่ากลุ่มการทดสอบผ่าน</span><span class="sxs-lookup"><span data-stu-id="b6a35-132">**Acceptable quality level** – Enter the total percentage of test results that must pass for the group of tests to be considered passed.</span></span>
    - <span data-ttu-id="b6a35-133">**การสุ่มตัวอย่างสินค้า** – เลือกการสุ่มตัวอย่างสินค้า</span><span class="sxs-lookup"><span data-stu-id="b6a35-133">**Item sampling** – Select an item sampling.</span></span> <span data-ttu-id="b6a35-134">สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [การสุ่มตัวอย่างสินค้าการจัดการคุณภาพ](quality-item-sampling.md)</span><span class="sxs-lookup"><span data-stu-id="b6a35-134">For more information, see [Quality management item sampling](quality-item-sampling.md).</span></span>
    - <span data-ttu-id="b6a35-135">**แบบทำลาย** – เลือกกล่องกาเครื่องหมายนี้เพื่อบ่งชี้ว่ากลุ่มการทดสอบจะทำลายสินค้าที่มีการทดสอบ</span><span class="sxs-lookup"><span data-stu-id="b6a35-135">**Destructive** – Select this check box to indicate that the test group will destroy the items that are tested.</span></span>

1. <span data-ttu-id="b6a35-136">ขณะที่ยังคงเลือกแถวใหม่อยู่ ให้เลือกแท็บ **ทั่วไป** ในส่วนบนของหน้า</span><span class="sxs-lookup"><span data-stu-id="b6a35-136">While the new row is still selected, select the **General** tab in the upper part of the page.</span></span> <span data-ttu-id="b6a35-137">การตั้งค่าบางอย่างของกลุ่มทดสอบที่เลือกบนแท็บ **ภาพรวม** จะซ้ำกันที่นี่</span><span class="sxs-lookup"><span data-stu-id="b6a35-137">Some of the settings for the test group that is selected on the **Overview** tab are repeated here.</span></span> <span data-ttu-id="b6a35-138">นอกจากนี้ คุณสามารถตั้งค่าฟิลด์ต่อไปนี้สำหรับกลุ่ม:</span><span class="sxs-lookup"><span data-stu-id="b6a35-138">In addition, you can set the following fields for the group:</span></span>

    - <span data-ttu-id="b6a35-140">**อัปเดตการโอนการครอบครองชุดงาน** – เมื่อคุณตั้งค่าตัวเลือกนี้เป็น *ใช่* ถ้าสินค้าที่ทดสอบเป็นการควบคุมชุดงาน การโอนการครอบครองชุดงานจะถูกอัปเดตโดยอัตโนมัติด้วยค่าที่เลือกไว้ในฟิลด์ **การโอนการครอบครองชุดงานของใบสั่งตรวจสอบคุณภาพที่ไม่ผ่าน** หรือฟิลด์ **การโอนการครอบครองชุดงานของใบสั่งตรวจสอบคุณภาพที่ผ่านแล้ว**</span><span class="sxs-lookup"><span data-stu-id="b6a35-140">**Update batch disposition** – When you set this option to *Yes*, if the item that is being tested is batch controlled, the batch disposition will automatically be updated with the value that is selected in the **Failed quality order batch disposition** or **Passed quality order batch disposition** field.</span></span>
    - <span data-ttu-id="b6a35-141">**การโอนการครอบครองชุดงานของใบสั่งตรวจสอบคุณภาพที่ไม่ผ่าน** – เลือกรหัสการโอนการครอบครองชุดงานที่ควรมอบหมายเมื่อใบสั่งตรวจสอบคุณภาพที่มีกลุ่มทดสอบนี้ล้มเหลว เนื่องจากรหัสดังกล่าวไม่ตรงกับ AQL</span><span class="sxs-lookup"><span data-stu-id="b6a35-141">**Failed quality order batch disposition** – Select the batch disposition code that should be assigned when a quality order that includes this test group fails because it doesn't meet the AQL.</span></span>
    - <span data-ttu-id="b6a35-142">**การโอนการครอบครองชุดงานของใบสั่งตรวจสอบคุณภาพที่ผ่าน** – เลือกรหัสการโอนการครอบครองชุดงานที่ควรมอบหมายเมื่อใบสั่งตรวจสอบคุณภาพที่มีกลุ่มทดสอบนี้ผ่าน เนื่องจากรหัสดังกล่าวตรงกับ AQL</span><span class="sxs-lookup"><span data-stu-id="b6a35-142">**Passed quality order batch disposition** – Select the batch disposition code that should be assigned when a quality order that includes this test group passes because it meets the AQL.</span></span>
