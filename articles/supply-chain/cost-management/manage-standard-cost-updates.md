# <a name="manage-standard-cost-updates"></a><span data-ttu-id="db851-103">จัดการการอัพเดตต้นทุนมาตรฐาน</span><span class="sxs-lookup"><span data-stu-id="db851-103">Manage standard cost updates</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="db851-104">สามารถจัดการการอัพเดตข้อมูลต้นทุนมาตรฐานได้โดยใช้วิธีการสองวิธี - วิธีการเวอร์ชันเดียวและวิธีการสองเวอร์ชัน</span><span class="sxs-lookup"><span data-stu-id="db851-104">Updates to standard cost data can be managed by using two different approaches -  the one-version approach and the two-version approach.</span></span> 

<span data-ttu-id="db851-105">วิธีการเวอร์ชันเดียวใช้เวอร์ชันการคำนวณต้นทุนเดียวที่ประกอบด้วยเรกคอร์ดต้นทุนทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="db851-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="db851-106">เรกคอร์ดเหล่านี้รวมถึงต้นทุนเดิมและการอัพเดตต้นทุนทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="db851-106">These records include the original costs and all cost updates.</span></span>
<span data-ttu-id="db851-107">วิธีการสองเวอร์ชันใช้เวอร์ชันหนึ่งที่ประกอบด้วยเรกคอร์ดของต้นทุนเดิม และเวอร์ชันที่สองที่ประกอบด้วยเรกคอร์ดของการอัพเดตต้นทุนทั้งหมด</span><span class="sxs-lookup"><span class="sxs-lookup"><span data-stu-id="db851-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting ซึ่งแต่ละการอัพเดตที่เพิ่มขึ้นมีเวอร์ชันการคำนวณต้นทุนแยกกันที่ประกอบด้วยเรกคอร์ดต้นทุนส่วนเพิ่มขึ้น</span><span class="sxs-lookup"><span data-stu-id="db851-109">The two-version approach can be used class="sxs-lookup"><span data-stu-id="db851-112">Assume that a single costing version represents the standard costs for the current year.</span></span> <span -   <span data-ttu-id="db851-117">**วิธีการเวอร์ชันเดียวในการอัพเดตข้อมูลต้นทุน** − ในวิธีการเวอร์ชันเดียว เวอร์ชันการคำนวณต้นทุนแบบดั้งเดิม 2016-STD ประกอบด้วยเรกคอร์ดต้นทุนทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="db851-117">**One-version approach to cost data updates** − In the one-version approach, the original costing version 2016-STD contains all cost records.</span></span> <span data-ttu-id="db851-118">การอัพเดตต้นทุนจะถูกบันทึกใน 2016-STD และถูกตั้งค่าสถานะเป็น "ค้างอยู่"</span><span class="sxs-lookup"><span data-stu-id="db851-118">Cost updates are recorded in 2016-STD and are set to a status of ”Pending.”</span></span> <span data-ttu-id="db851-119">สามารถป้อนต้นทุนที่ค้างอยู่ด้วยตนเองสำหรับสินค้าที่มีการซื้อใหม่ หรือสามารถถูกคำนวณสำหรับสินค้าที่ผลิตเพื่อสะท้อนการแก้ไข เมื่อใช้วิธีการเวอร์ชันเดียว การคำนวณ BOM จะไม่จำเป็นต้องใช้แหล่งข้อมูลสำรองเนื่องจากต้นทุนที่ใช้งานอยู่ทั้งหมดจะมีอยู่ในเวอร์ชันการคำนวณต้นทุน</span><span class="sxs-lookup"><span data-stu-id="db851-119">The pending costs can be manually entered for new purchased items, or they can be calculated for a manufactured item to reflect corrections.When the one-version approach is used, the BOM calculations do not require a fallback data source because all active costs are contained in the costing version.</span></span> <span data-ttu-id="db851-120">หลังจากการเปิดใช้งานต้นทุนค้างอยู่ เวอร์ชันการคำนวณต้นทุนแบบดั้งเดิม 2016-STD จะประกอบด้วยต้นทุนปัจจุบันที่ใช้งานอยู่เช่นกัน</span><span class="sxs-lookup"><span data-stu-id="db851-120">After the pending costs become active, the original costing version 2016-STD will again contain all the current active costs.</span></span>
-   <span data-ttu-id="db851-121">**วิธีการสองเวอร์ชันในการอัพเดตข้อมูลต้นทุน** − วิธีการสองเวอร์ชันต้องมีเวอร์ชันการคำนวณต้นทุนเพิ่มเติมที่ประกอบด้วยการอัพเดตต้นทุนเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="db851-121">**Two-version approach to cost data updates** − The two-version approach requires an additional costing version that contains only the cost data-ttu-id="db851-124">ทั้งนี้เนื่องจากเวอร์ชันการคำนวณต้นทุนเพิ่มเติม 2016-STD-CHANGES ประกอบด้วยเพียงชุดย่อยของข้อมูลต้นทุนเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="db851-124">This
