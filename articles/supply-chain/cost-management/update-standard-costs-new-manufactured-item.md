# <a name="update-standard-costs-for-a-new-manufactured-item"></a><span data-ttu-id="acc27-103">อัพเดตต้นทุนมาตรฐานสำหรับสินค้าที่ผลิตขึ้นใหม่</span><span class="sxs-lookup"><span data-stu-id="acc27-103">Update standard costs for a new manufactured item</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="acc27-104">บทความนี้ให้คำแนะนำสำหรับการอัพเดตต้นทุนมาตรฐานสำหรับสินค้าที่ผลิตใหม่</span><span class="sxs-lookup"><span data-stu-id="acc27-104">This article provides guidance for updating standard costs for a new manufactured item.</span></span> 

<span data-ttu-id="acc27-105">คำแนะนำต่อไปนี้จะถือว่าคุณใช้วิธีการแบบสองเวอร์ชันเพื่ออัพเดตต้นทุนมาตรฐาน </span><span class="sxs-lookup"><span data-stu-id="acc27-105">The following guidelines assume that you use a two-version approach to update standard costs.</span></span> <span data-ttu-id="acc27-106">ในวิธีนี้ การคำนวณต้นทุนในเวอร์ชันหนึ่งประกอบด้วย ต้นทุนมาตรฐานที่กำหนดไว้ตั้งแต่เริ่มแรกสำหรับช่วงเวลาแน่นอนไม่เปลี่ยนแปลง ในขณะที่เวอร์ชันที่สองประกอบด้วย การอัพเดตข้อมูลเพิ่มเติมในส่วนของวัสดุใหม่ที่ได้จากการผลิต </span><span class="sxs-lookup"><span data-stu-id="acc27-106">In this approach, one costing version contains the standard costs that were originally defined for the frozen period, and the second costing version contains the incremental updates that pertain to the new manufactured items.</span></span> <span data-ttu-id="acc27-107">การอัพเดตเพิ่มเติมนี้จะถูกป้อนเป็นเรกคอร์ดต้นทุนในการคำนวณต้นทุนเวอร์ชันที่สอง และนำไปใช้คำนวณในขั้นตอนสุดท้าย </span><span class="sxs-lookup"><span data-stu-id="acc27-107">The incremental updates are entered as cost records in the second costing version, and eventually they are enabled.</span></span> <span data-ttu-id="acc27-108">วิธีแบบสองเวอร์ชันต้องการให้คุณกำหนดเวอร์ชันการคิดต้นทุนที่สอง </span><span class="sxs-lookup"><span data-stu-id="acc27-108">The two-version approach requires that you define a second costing version.</span></span> <span data-ttu-id="acc27-109">นี่คือคำแนะนำสำหรับการกำหนดการคิดต้นทุนนี้:</span><span class="sxs-lookup"><span data-stu-id="acc27-109">Here are the guidelines for defining this costing version:</span></span>

-   <span data-ttu-id="acc27-110">กำหนดชนิดการคำนวณต้นทุนของ **ต้นทุนมาตรฐาน**</span><span class="sxs-lookup"><span data-stu-id="acc27-110">Assign a costing type of **Standard cost**.</span></span>
-   <span data-ttu-id="acc27-111">กำหนดรหัสสำคัญที่ระบุเนื้อหาของเวอร์ชันการคำนวณต้นทุน เช่น **2016-UPDATES**</span><span class="sxs-lookup"><span data-stu-id="acc27-111">Assign a significant identifier that indicates the contents of the costing version, such as **2016-UPDATES**.</span></span>
-   <span data-ttu-id="acc27-112">ในกลุ่มฟิลด์ **อนุญาตชนิดราคา** ตรวจสอบให้แน่ใจว่า **ราคาต้นทุน** ถูกตั้งค่าเป็น **ใช่**</span><span class="sxs-lookup"><span data-stu-id="acc27-112">In the **Allow price types** field group, make sure that **Cost price** is set to **Yes**.</span></span>
-   <span data-ttu-id="acc27-113">อนุญาตให้เรกคอร์ดต้นทุนถูกป้อนสำหรับไซต์ทั้งหมด (นั่นคือ ปล่อยให้ฟิลด์ **ไซต์** ว่าง)</span><span class="sxs-lookup"><span data-stu-id="acc27-113">Allow cost records to be entered for all sites (that is, leave the **Site** field blank).</span></span> <span data-ttu-id="acc27-114">ถ้าคุณป้อนไซต์ คุณสามารถป้อนเรกคอร์ดต้นทุนสำหรับไซต์ดังกล่าวเท่านั้น</span><span class="sxs-lookup"><span data-stu-id="acc27-114">If you enter a site, cost records can be entered only for that site.</span></span>
-   <span data-ttu-id="acc27-115">ใช้หลักการถอยกลับของ **ใช้งานอยู่**</span><span class="sxs-lookup"><span data-stu-id="acc27-115">Use a fallback principle of **Active**.</span></span>

<span data-ttu-id="acc27-116">เมื่อต้องการเพิ่มรายการผลิตตลอดช่วงเวลาที่แน่นอน ให้ทำตามขั้นตอนต่อไปนี้</span><span class="sxs-lookup"><span data-stu-id="acc27-116">To add new manufacturing items throughout the frozen period, follow these steps.</span></span>

1.  <span data-ttu-id="acc27-117">ใช้หน้า **การตั้งค่าเวอร์ชันการคิดต้นทุน** เพื่อเปิดให้มีการป้อนเรกคอร์ดต้นทุนลงในการคำนวณต้นทุนเวอร์ชันที่สองที่มีข้อมูลอัพเดตเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="acc27-117">Use the **Costing version setup** page to enable cost records to be entered into the second costing version that contains the incremental updates.</span></span> <span data-ttu-id="acc27-118">ป้องกันการเรียกใช้ต้นทุนที่ค้างอยู่ ที่การเรียกใช้ได้รับอนุญาตหลังจากต้นทุนที่ค้างอยู่ได้ถูกกำหนดอย่างถูกต้องและเสร็จสมบูรณ์ทั้งหมดแล้ว </span><span class="sxs-lookup"><span data-stu-id="acc27-118">Prevent the activation of pending costs, where activation is allowed after pending costs have been completely and accurately defined.</span></span> <span data-ttu-id="acc27-119">ต่อมาให้กำหนดนโยบายของการคำนวณต้นทุนเวอร์ชันที่สองว่าไม่มีวันที่เริ่มต้น จากนั้นป้อนวันที่เริ่มต้นเมื่อคุณป้อนเรกคอร์ดต้นทุนแต่ละเรกคอร์ด </span><span class="sxs-lookup"><span data-stu-id="acc27-119">Indicate a blank from-date as a policy in the costing version, and then enter the from-date when you enter each cost record.</span></span> <span data-ttu-id="acc27-120">จากนั้นป้อนวันที่เริ่มต้นเมื่อคุณป้อนเรกคอร์ดต้นทุนแต่ละเรกคอร์ด วันที่เริ่มต้นควรเป็นวันที่ก่อนที่จะมีการสั่งซื้อหรือผลิตสินค้าใหม่</span><span class="sxs-lookup"><span data-stu-id="acc27-120">The from-date should represent a date before the new items are purchased or manufactured.</span></span>
2.  <span data-ttu-id="acc27-121">ใช้หน้า **ราคาสินค้า** เพื่อป้อนเรกคอร์ดต้นทุนสำหรับสินค้าที่สั่งซื้อใหม่ ให้ป้อนเวอร์ชันการคำนวณต้นทุนที่มีข้อมูลอัพเดตเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="acc27-121">Use the **Item price** page to enter cost records for new purchased items.</span></span> <span data-ttu-id="acc27-122">สำหรับเรกคอร์ดต้นทุนแต่ละเรกคอร์ด และใช้วันที่เริ่มต้นที่มาก่อนวันที่ผลิตที่คาดไว้ สำหรับสินค้าที่ผลิตใหม่</span><span class="sxs-lookup"><span data-stu-id="acc27-122">For each cost record, enter the costing version that contains incremental updates, and use a from-date that comes before the expected manufacturing date for new manufactured items.</span></span>
3.  <span data-ttu-id="acc27-123">คำนวณต้นทุนของสินค้าที่ผลิตใหม่ โดยใช้หน้า **การคำนวณ**</span><span class="sxs-lookup"><span data-stu-id="acc27-123">Calculate the cost of new manufactured items by using the **Calculation** page.</span></span> <span data-ttu-id="acc27-124">เปิดหน้า **การคำนวณ** จากหน้า **การบำรุงรักษาเวอร์ชันการคิดต้นทุน** และจากนั้น เลือกเวอร์ชันที่ประกอบด้วยการอัพเดตเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="acc27-124">Open the **Calculation** page from the **Costing version maintenance** page, and then select the costing version that contains the incremental updates.</span></span> <span data-ttu-id="acc27-125">ใช้เกณฑ์การเลือกเพื่อระบุสินค้าที่ผลิตใหม่ใดๆ ส่วนประกอบที่ผลิตอย่างใดอย่างหนึ่ง </span><span class="sxs-lookup"><span data-stu-id="acc27-125">Use the selection criteria to specify the new manufactured item and any one of its manufactured components.</span></span> <span data-ttu-id="acc27-126">ในโครงสร้างผลิตภัณฑ์แบบหลายระดับ คุณอาจต้องระบุสินค้าหลักใดๆที่ประกอบด้วยสินค้าที่ผลิตใหม่เป็นส่วนประกอบ </span><span class="sxs-lookup"><span data-stu-id="acc27-126">In a multilevel product structure, you might also have to specify any parent item that contains the new manufactured items as a component.</span></span> <span data-ttu-id="acc27-127">ป้อนวันที่เริ่มต้นสำหรับสูตรการผลิต (BOM) ที่สอดคล้องกับจุดเริ่มต้นของการผลิตสำหรับสินค้าที่ผลิตใหม่ </span><span class="sxs-lookup"><span data-stu-id="acc27-127">Enter a from-date for the bill of materials (BOM) calculation that corresponds to the start of manufacturing for the new manufactured items.</span></span> <span data-ttu-id="acc27-128">วันที่เริ่มต้นควรอยู่ในวันที่มีผลบังคับใช้สำหรับเวอร์ชัน BOM และเวอร์ชันกระบวนการผลิตของสินค้า </span><span class="sxs-lookup"><span data-stu-id="acc27-128">The from-date should be in the effective dates for the item’s BOM version and route version.</span></span> <span data-ttu-id="acc27-129">**หมายเหตุ:** หากสินค้าใดมีเรกคอร์ดต้นทุนไม่ครบ สินค้านั้นอาจเป็นสินค้าใหม่ที่ผลิต</span><span class="sxs-lookup"><span data-stu-id="acc27-129">**Note:** A missing cost record can indicate a new manufactured item.</span></span> <span data-ttu-id="acc27-130">การคำนวณ BOM สามารถถูกจำกัดกับสินค้าที่มีต้นทุนที่ขาดหายไป</span><span class="sxs-lookup"><span data-stu-id="acc27-130">BOM calculations can be limited to items that have missing costs.</span></span>
4.  <span data-ttu-id="acc27-131">ตรวจสอบความสมบูรณ์และความถูกต้องของการคำนวณต้นทุนสำหรับสินค้าที่ผลิตใหม่ </span><span class="sxs-lookup"><span data-stu-id="acc27-131">Verify the completeness and accuracy of the calculated costs for new manufactured items.</span></span> <span data-ttu-id="acc27-132">ใช้หน้า **เสร็จสมบูรณ์** เพื่อตรวจทานต้นทุนที่คำนวณได้ สำหรับเรกคอร์ดต้นทุนของสินค้าแต่ละเรกคอร์ด และรวมถึงตรวจทานข้อความเตือน Infolog ใดๆ</span><span class="sxs-lookup"><span data-stu-id="acc27-132">Use the **Complete** page to review the calculated costs for each item cost record, and also review any Infolog warning messages.</span></span> <span data-ttu-id="acc27-133">หรือใช้หน้า **การคำนวณ** เพื่อตรวจทานต้นทุนที่คำนวณได้</span><span class="sxs-lookup"><span data-stu-id="acc27-133">Alternatively, use the **Calculation** page to review the calculated costs.</span></span>
5.  <span data-ttu-id="acc27-134">ใช้หน้า **การตั้งค่าเวอร์ชันการคิดต้นทุน** เพื่อเปลี่ยนแฟล็กการบล็อค เพื่ออนุญาตให้เปิดใช้งานเรกคอร์ดต้นทุนที่เปิดค้างไว้ในเวอร์ชันการคำนวณต้นทุนที่สอง</span><span class="sxs-lookup"><span data-stu-id="acc27-134">Use the **Costing version setup** page to change the blocking flag to allow activation of the pending cost records in the second costing version.</span></span>
6.  <span data-ttu-id="acc27-135">ใช้หน้า **เรียกใช้งานราคา** (ซึ่งคุณสามารถเปิดได้จากหน้า **การรักษาเวอร์ชันการคิดต้นทุน** ) เพื่อเปิดใช้งานเรกคอร์ดต้นทุนสินค้าที่ค้างอยู่ทั้งหมดภายในเวอร์ชันการคำนวณต้นทุนที่สอง</span><span class="sxs-lookup"><span data-stu-id="acc27-135">Use the **Activate prices** page (which you open from the **Costing version maintenance** page) to enable all pending cost records in the second costing version.</span></span> <span data-ttu-id="acc27-136">คุณยังสามารถเปิดใช้งานเรกคอร์ดต้นทุนสินค้าที่ค้างอยู่สำหรับสินค้าแต่ละรายการ โดยการคลิกปุ่ม **เรียกใช้งาน** ในหน้า **ราคาสินค้า** ได้</span><span class="sxs-lookup"><span data-stu-id="acc27-136">You can also enable the pending cost records for individual items by clicking the **Activate** button on the **Item price** page.</span></span>
7.  <span data-ttu-id="acc27-137">ใช้หน้า **การตั้งค่าเวอร์ชันการคิดต้นทุน** เพื่อเปลี่ยนแฟล็กการบล็อคในเวอร์ชันการคำนวณต้นทุนสอง เพื่อป้องกันการบำรุงรักษาข้อมูลเพิ่มเติม</span><span class="sxs-lookup"><span data-stu-id="acc27-137">Use the **Costing version setup** page to change the blocking flags in the second costing version to prevent additional data maintenance.</span></span> <span data-ttu-id="acc27-138">นโยบายการบล็อคป้องกันรายการของต้นทุนใหม่ที่ค้างอยู่และการเรียกใช้ต้นทุนที่ค้างอยู่</span><span class="sxs-lookup"><span data-stu-id="acc27-138">The blocking policies prevent the entry of new pending costs and the activation of pending costs.</span></span>
