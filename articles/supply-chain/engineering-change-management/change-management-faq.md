## <a name="should-i-track-the-version-in-transactions"></a><span data-ttu-id="149ba-105">ฉันควรจะติดตามเวอร์ชันในธุรกรรมหรือไม่</span><span class="sxs-lookup"><span data-stu-id="149ba-105">Should I track the version in transactions?</span></span>

<span data-ttu-id="149ba-106">เมื่อคุณสร้างประเภทการเปลี่ยนแปลงทางวิศวกรรม หน้า **รายละเอียดประเภทการเปลี่ยนแปลงทางวิศวกรรม** จะมีตัวเลือกที่มีชื่อว่า **ติดตามเวอร์ชันในธุรกรรม**</span><span class="sxs-lookup"><span data-stu-id="149ba-106">When you create an engineering change category, the **Engineering change category details** page provides an option that is named **Track version in transactions**.</span></span> <span data-ttu-id="149ba-107">การตั้งค่านี้คืออะไร และทำอะไรบ้าง</span><span class="sxs-lookup"><span data-stu-id="149ba-107">What is this setting, and what does it do?</span></span>

- <span data-ttu-id="149ba-108">ถ้าคุณตั้งค่าตัวเลือก **ติดตามเวอร์ชันในธุรกรรม** เป็น *ใช่* จะมีการกรองข้อมูลฟิลด์ **กลุ่มมิติ** เพื่อให้แสดงเฉพาะกลุ่มมิติที่มีเวอร์ชันเป็นมิติที่ใช้งานอยู่</span><span class="sxs-lookup"><span data-stu-id="149ba-108">If you set the **Track version in transactions** option to *Yes*, the **Dimension group** field will be filtered so that it shows only dimension groups where version is an active dimension.</span></span>
- <span data-ttu-id="149ba-109">ถ้าคุณตั้งค่าตัวเลือก **ติดตามเวอร์ชันในธุรกรรม** เป็น *ไม่* จะมีการกรองข้อมูลฟิลด์ **กลุ่มมิติ** เพื่อให้แสดงเฉพาะกลุ่มมิติที่มีเวอร์ชันมิติ เป็นเวอร์ชันมิติที่ **ไม่ได้** ใช้งานอยู่</span><span class="sxs-lookup"><span data-stu-id="149ba-109">If you set the **Track version in transactions** option to *No*, the **Dimension group** field will be filtered so that it shows only dimension groups where the version dimension is **not** an active dimension.</span></span>

### <a name="if-you-track-the-version-in-transactions"></a><span data-ttu-id="149ba-110">หากคุณติดตามเวอร์ชันในธุรกรรม</span><span class="sxs-lookup"><span data-stu-id="149ba-110">If you track the version in transactions</span></span>

<span data-ttu-id="149ba-111">ให้กับประเภทวิศวกรรมซึ่งคุณได้เลือกกลุ่มมิติซึ่งเวอร์ชันเป็นมิติที่ใช้งานอยู่ คุณจะติดตามเวอร์ชันในธุรกรรมต่างๆ ของผลิตภัณฑ์ในประเภทนั้น</span><span class="sxs-lookup"><span data-stu-id="149ba-111">For engineering categories where you've selected a dimension group where version is an active dimension, you will track versions in transactions for products in that category.</span></span> <span data-ttu-id="149ba-112">ในกรณีนี้ ผลิตภัณฑ์ทางวิศวกรรมจะเป็นผลิตภัณฑ์หลัก และผลิตภัณฑ์แต่ละรุ่นจะเป็นผลิตภัณฑ์ย่อยที่ใช้มิติเวอร์ชัน</span><span class="sxs-lookup"><span data-stu-id="149ba-112">In this case, the engineering product will be a product master, and each version of the product will be a product variant that uses the version dimension.</span></span> <span data-ttu-id="149ba-113">สามารถใช้มิติอื่นร่วมกับมิติเวอร์ชันได้</span><span class="sxs-lookup"><span data-stu-id="149ba-113">Other dimensions can be used together with the version dimension.</span></span>

<span data-ttu-id="149ba-114">ในกรณีนี้ เวอร์ชันวิศวกรรมแต่ละเวอร์ชันจะถือว่าเป็นตัวแปรในกระบวนการทั้งหมด</span><span class="sxs-lookup"><span data-stu-id="149ba-114">In this case, each engineering version will be treated as a variant in all processes.</span></span> <span data-ttu-id="149ba-115">ดังนั้น ถ้าคุณมีตัวแปรเฉพาะในธุรกรรม (เพื่อให้คุณสามารถระบุว่ามีการขายหรือซื้อตัวแปรใด) คุณต้องจัดการสต็อกของแต่ละเวอร์ชัน การวางแผนหลักจะวางแผนการจัดหาวัสดุให้กับแต่ละเวอร์ชัน และอื่นๆ</span><span class="sxs-lookup"><span data-stu-id="149ba-115">Therefore, if you have a specific variant in a transaction (so that you can determine which variant was sold or purchased), you must manage stock for each version, master planning will plan supply for each version, and so on.</span></span> <span data-ttu-id="149ba-116">ถ้าคุณถอนเวอร์ชันจากตลาด คุณต้องปรับปรุงวันที่เริ่มมีผลบังคับและวันที่สิ้นสุดการมีผลบังคับด้วยตนเอง เพื่อให้สะท้อนถึงการเปลี่ยนแปลง</span><span class="sxs-lookup"><span data-stu-id="149ba-116">If you retire a version from the market, you must manually adjust its effective-from and effective-to dates so that they reflect the change.</span></span> <span data-ttu-id="149ba-117">คุณต้องจัดการสินค้าคงคลังด้วย เพื่อให้แน่ใจว่าคุณไม่มีสินค้าในเวอร์ชันที่ไม่ได้ใช้ในคลังสินค้าของคุณ</span><span class="sxs-lookup"><span data-stu-id="149ba-117">You must also manage your inventory to make sure that you don't have unused versions of items in your warehouses.</span></span>

<span data-ttu-id="149ba-118">ถึงแม้ว่าตัวเลือกนี้จะต้องใช้ความพยายามในการจัดการมากขึ้น แต่ขอแนะนาว่าถ้าคุณต้องมีการติดตามเวอร์ชันที่เฉพาะเจาะจงซึ่งใช้ในแต่ละธุรกรรมสูง</span><span class="sxs-lookup"><span data-stu-id="149ba-118">Although this option requires more management effort, we recommend it if you require high traceability of the specific versions that are used in each transaction.</span></span>

### <a name="if-you-dont-track-the-version-in-transactions"></a><span data-ttu-id="149ba-119">หากคุณไม่ติดตามเวอร์ชันในธุรกรรม</span><span class="sxs-lookup"><span data-stu-id="149ba-119">If you don't track the version in transactions</span></span>

<span data-ttu-id="149ba-120">ให้กับประเภทวิศวกรรมซึ่งคุณได้เลือกกลุ่มมิติซึ่งเวอร์ชันเป็นมิติที่ **ไม่ได้** ใช้งานอยู่ คุณจะ **ไม่ได้** ติดตามเวอร์ชันในธุรกรรมต่างๆ ของผลิตภัณฑ์ในประเภทนั้น</span><span class="sxs-lookup"><span data-stu-id="149ba-120">For engineering categories where you've selected a dimension group where version is **not** an active dimension, you will **not** track versions in transactions for products in that category.</span></span> <span data-ttu-id="149ba-121">ในกรณีนี้ ถ้าคุณไม่ได้ใช้มิติอื่น ผลิตภัณฑ์วิศวกรรมจะเป็นผลิตภัณฑ์เฉพาะ</span><span class="sxs-lookup"><span data-stu-id="149ba-121">In this case, if you don't use any other dimension, the engineering product will be a distinct product.</span></span> <span data-ttu-id="149ba-122">ผลิตภัณฑ์จะยังคงเป็นเวอร์ชัน และคุณสามารถจัดการข้อมูลเกี่ยวกับเวอร์ชันเฉพาะ (ตัวอย่างเช่น สูตรการผลิต \[BOM] และเส้นทาง) แต่คุณจะไม่สามารถติดตามเวอร์ชันที่ระบุที่ใช้ในแต่ละธุรกรรมได้</span><span class="sxs-lookup"><span data-stu-id="149ba-122">The product will still be versioned, and you can manage information about specific versions (for example, their bill of materials \[BOM] and route), but you won't be able to trace which specific version was used in each transaction.</span></span> <span data-ttu-id="149ba-123">วันที่เริ่มมีผลบังคับใช้และวันที่สิ้นสุดที่มีผลบังคับใช้ จะบ่งชี้การมีผลบังคับใช้ของแต่ละเวอร์ชัน</span><span class="sxs-lookup"><span data-stu-id="149ba-123">The effective-from and effective-to dates indicate the validity of each version.</span></span>

<span data-ttu-id="149ba-124">ตัวเลือกนี้จะจัดการได้ง่ายขึ้นมาก เนื่องจากถ้าคุณต้องการเปลี่ยนจากเวอร์ชันหนึ่งเป็นอีกเวอร์ชันหนึ่ง คุณเพียงต้องสร้างการเปลี่ยนแปลงที่ต้องใช้ในคำสั่งเปลี่ยนแปลง จากนั้นอัปเดตวันที่เริ่มต้นและวันที่สิ้นสุดที่มีผลบังคับใช้ในเวอร์ชันวิศวกรรม</span><span class="sxs-lookup"><span data-stu-id="149ba-124">This option is much easier to manage, because if you want to change from one version to another, you just have to make the required changes in a change order, and then update the effective-from and effective-to dates in the engineering version.</span></span> <span data-ttu-id="149ba-125">จากนั้นกระบวนการผลิตจะเบิกสินค้า BOM และกระบวนการผลิตที่ต้องใช้กับผลิตภัณฑ์ (และเวอร์ชันเฉพาะ)</span><span class="sxs-lookup"><span data-stu-id="149ba-125">The production processes will then pick up the required BOM and route for the product (and its specific version).</span></span>

<span data-ttu-id="149ba-126">องค์กรส่วนใหญ่เลือกตัวเลือกนี้ เนื่องจากให้เวอร์ชันและการจัดการการเปลี่ยนแปลง แต่ไม่เพิ่มโสหุ้ยการติดตามเวอร์ชันในแต่ละธุรกรรม ในสินค้าคงคลัง และในระหว่างการวางแผนหลัก</span><span class="sxs-lookup"><span data-stu-id="149ba-126">Most organizations choose this option because it provides version and change management, but it doesn't add the extra overhead of tracking the version in each transaction, in inventory, and during master planning.</span></span>
