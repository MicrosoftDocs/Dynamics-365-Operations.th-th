# <a name="product-readiness"></a>ความพร้อมของผลิตภัณฑ์

[!include [banner](../includes/banner.md)]

คุณสามารถใช้การตรวจสอบความพร้อมเพื่อตรวจสอบให้แน่ใจว่าข้อมูลหลักที่จำเป็นทั้งหมดมีการระบุสำหรับผลิตภัณฑ์ก่อนที่จะใช้ในธุรกรรม เมื่อมีการใช้การตรวจสอบความพร้อม ผู้ใช้หรือทีมงานจะทำหน้าที่ตรวจสอบความถูกต้องของข้อมูลที่เกี่ยวข้องกับผลิตภัณฑ์ที่กำหนดไว้ล่วงหน้าเฉพาะ ถ้ามีการตรวจสอบความพร้อมที่เปิดสำหรับผลิตภัณฑ์ ผลิตภัณฑ์ไม่สามารถนำออกใช้หรือนำมาใช้ในธุรกรรม

กล่องกาเครื่องหมาย **ที่ใช้งานอยู่** สำหรับผลิตภัณฑ์วิศวกรรม ตัวแปร หรือรุ่นพร้อมใช้งานหลังจากที่ป้อนและตรวจสอบความถูกต้องข้อมูลที่จำเป็นทั้งหมดแล้วเท่านั้น และหลังจากที่มีการประมวลผลการตรวจสอบความพร้อมทั้งหมดแล้ว ในจุดนั้น ผลิตภัณฑ์ รุ่น หรือตัวแปรสามารถนำออกใช้ไปยังบริษัทอื่นและใช้ในธุรกรรม คุณสามารถสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์ใหม่ ผลิตภัณฑ์ย่อยใหม่ และรุ่นวิศวกรรมใหม่

## <a name="types-of-readiness-checks"></a>ชนิดของการตรวจสอบความพร้อม

มีการตรวจสอบความพร้อมอยู่ 3 ชนิด:

- **การตรวจสอบระบบ** – ระบบจะตรวจสอบว่ามีเรกคอร์ดที่ถูกต้องหรือไม่ ตัวอย่างเช่น เรกคอร์ดอาจเป็นสูตรการผลิต (BOM) ที่ใช้งานอยู่
- **การตรวจสอบด้วยตนเอง** – ผู้ใช้จะตรวจสอบว่าเรกคอร์ดมีผลบังคับใช้หรือไม่ ตัวอย่างเช่น การตรวจสอบความพร้อมอาจจำเป็นต้องมีการตรวจสอบความถูกต้องของการตั้งค่าใบสั่งเริ่มต้น ในบางกรณี เช่น เมื่อผลิตภัณฑ์ยังคงถูกออกแบบ และดังนั้นจึงไม่มีการจัดเก็บอยู่ในสินค้าคงคลัง จะไม่จำเป็นต้องมีการตั้งค่าใบสั่งเริ่มต้น อย่างไรก็ตาม การตั้งค่าใบสั่งเริ่มต้นอาจจำเป็นสำหรับผลิตภัณฑ์ชนิดเดียวกัน เนื่องจากผลิตภัณฑ์สามารถจัดเก็บอยู่ในสินค้าคงคลังได้ ผู้ใช้รับผิดชอบในการทราบวิธีการตัดสินใจอย่างถูกต้องว่าจำเป็นต้องมีการตรวจสอบความพร้อมหรือไม่
- **รายการตรวจสอบ** – ผู้ใช้ตอบชุดคำถามจากรายการตรวจสอบ และระบบจะกำหนดว่าคำตอบตรงกับความคาดหวังหรือไม่ รายการตรวจสอบอาจมีชื่อเรื่องใด ๆ ตัวอย่างเช่น คุณสามารถใช้เพื่อกำหนดว่าเอกสารการตลาด หรือคู่มือผลิตภัณฑ์เสร็จสมบูรณ์หรือไม่

## <a name="how-readiness-checks-are-created-for-a-new-product-variant-or-version"></a>วิธีการสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์ ผลิตภัณฑ์ย่อย หรือรุ่นใหม่

เมื่อคุณสร้าง **ผลิตภัณฑ์** วิศวกรรมใหม่ ระบบจะกำหนดว่ามีการตั้งค่านโยบายการตรวจสอบความพร้อมสำหรับประเภทผลิตภัณฑ์วิศวกรรมหรือไม่ (สามารถใช้นโยบายการตรวจสอบความพร้อมในระดับผลิตภัณฑ์ที่นำออกใช้ ระดับผลิตภัณฑ์ย่อยที่นำออกใช้ และระดับรุ่นวิศวกรรม) ถ้ามีการตั้งค่านโยบายแล้ว เหตุการณ์ต่อไปนี้จะเกิดขึ้น:

- มีการสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์ ตามนโยบายที่เกี่ยวข้อง
- รุ่นวิศวกรรมถูกตั้งค่าเป็นไม่ได้ใช้งานอยู่เพื่อบล็อคผลิตภัณฑ์จากการใช้งาน รุ่นทั้งหมดสำหรับผลิตภัณฑ์เฉพาะที่เกี่ยวข้องถูกตั้งค่าเป็นไม่ได้ใช้งาน

ถ้ามีการสร้าง **ผลิตภัณฑ์ย่อยใหม่** สำหรับผลิตภัณฑ์ ระบบจะตรวจสอบว่ามีการตั้งค่าการตรวจสอบความพร้อมในชนิดผลิตภัณฑ์วิศวกรรมแล้วหรือไม่ (สามารถใช้การตรวจสอบความพร้อมในระดับผลิตภัณฑ์ย่อยที่นำออกใช้ และระดับรุ่นวิศวกรรม) ถ้ามีการตั้งค่าการตรวจสอบความพร้อมแล้ว เหตุการณ์ต่อไปนี้จะเกิดขึ้น:

- มีการสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์
- รุ่นวิศวกรรมถูกตั้งค่าเป็นไม่ได้ใช้งานอยู่เพื่อบล็อคผลิตภัณฑ์จากการใช้งาน

ถ้ามีการสร้าง **รุ่น** วิศวกรรมสำหรับผลิตภัณฑ์ ระบบจะตรวจสอบว่ามีการตั้งค่าการตรวจสอบความพร้อมในชนิดผลิตภัณฑ์วิศวกรรมแล้วหรือไม่ (สามารถใช้การตรวจสอบความพร้อมในระดับรุ่นวิศวกรรม) ถ้ามีการตั้งค่าการตรวจสอบความพร้อมแล้ว เหตุการณ์ต่อไปนี้จะเกิดขึ้น:

- มีการสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์
- รุ่นวิศวกรรมถูกตั้งค่าเป็นไม่ได้ใช้งานอยู่เพื่อบล็อคผลิตภัณฑ์จากการใช้งาน

## <a name="view-readiness-checks"></a>ดูการตรวจสอบความพร้อม

### <a name="view-open-readiness-checks-for-a-product-or-product-version"></a>ดูการตรวจสอบความพร้อมที่เปิดสำหรับผลิตภัณฑ์หรือรุ่นผลิตภัณฑ์

เมื่อต้องการค้นหาการตรวจสอบความพร้อมที่เปิดสำหรับผลิตภัณฑ์ ให้เปิดหน้า **รายละเอียดของผลิตภัณฑ์ที่นำออกใช้** จากนั้น บนบานหน้าต่างการดำเนินการ บนแท็บ **ผลิตภัณฑ์** ในกลุ่ม **การตรวจสอบความพร้อม** ให้เลือก **การตรวจสอบความพร้อม**

เมื่อต้องการค้นหาการตรวจสอบความพร้อมที่เปิดสำหรับรุ่นผลิตภัณฑ์ ให้เปิดหน้า **รุ่นวิศวกรรม** สำหรับผลิตภัณฑ์ที่นำออกใช้และเลือกรุ่น จากนั้น บนบานหน้าต่างการดำเนินการ บนแท็บ **ผลิตภัณฑ์** ในกลุ่ม **รายการตรวจสอบ** ให้เลือก **การตรวจสอบความพร้อม**

### <a name="view-open-readiness-checks-that-are-assigned-to-you"></a>ดูการตรวจสอบความพร้อมที่เปิดซึ่งกำหนดให้กับคุณ

เมื่อต้องการดูการตรวจสอบความพร้อมที่เปิดที่กำหนดให้กับคุณ ให้ทำตามขั้นตอนต่อไปนี้

- ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> ทั่วไป \> ความพร้อมของผลิตภัณฑ์ \> การตรวจสอบความพร้อมที่เปิดของฉัน**
- ไปที่ **การจัดการข้อมูลผลิตภัณฑ์ \> พื้นที่ทำงาน \> ความพร้อมของผลิตภัณฑ์สำหรับการผลิตที่แยกต่างหาก**

การตั้งค่าที่ระบุผู้ที่มีการกำหนดการตรวจสอบความพร้อมกระทำสำหรับประเภทผลิตภัณฑ์วิศวกรรม การตรวจสอบความพร้อมสามารถกำหนดให้กับบุคคลหรือทีมงาน ถ้ามีการกำหนดการตรวจสอบความพร้อมให้กับทีมงาน จะมีบุคคลหนึ่งคนในทีมงานที่ต้องประมวลผลการตรวจสอบความพร้อม สำหรับข้อมูลเพิ่มเติม ดูที่ [ประเภทผลิตภัณฑ์วิศวกรรมและรุ่นวิศวกรรม](engineering-versions-product-category.md)

## <a name="process-open-readiness-checks"></a>ประมวลผลการตรวจสอบความพร้อมที่เปิด

### <a name="process-system-and-manual-readiness-checks"></a>ประมวลผลการตรวจสอบความพร้อมของระบบและด้วยตนเอง

หลังจากที่คุณเปิดหน้า **การตรวจสอบความพร้อม** คุณสามารถดูชื่อเรื่องของการตรวจสอบความพร้อมของระบบและด้วยตนเองได้โดยการเลือก **ดูข้อมูลที่เกี่ยวข้อง** ในบานหน้าต่างการดำเนินการ คุณสามารถดำเนินการให้เสร็จสมบูรณ์และ/หรือตรวจสอบความถูกต้องของข้อมูลสำหรับการตรวจสอบความพร้อม การตรวจสอบความพร้อมที่เปิดมีค่า **สถานะ** เป็น *ค้างอยู่* สถานะนี้จะบ่งชี้ว่าต้องมีการประมวลผลการตรวจสอบความพร้อม เมื่อต้องการประมวลผลการตรวจสอบความพร้อม ให้ทำตามหนึ่งในขั้นตอนเหล่านี้

- ในบานหน้าต่างการดำเนินการ ให้เลือก **ตรวจสอบ/ดำเนินการให้เสร็จสมบูรณ์** เพื่อทบทวนและทำการตรวจสอบความพร้อม เมื่อเสร็จสิ้นแล้ว ฟิลด์ **สถานะ** ถูกอัพเดตเป็น *ผ่าน*
- ในบานหน้าต่างการดำเนินการ ให้เลือก **ข้าม** ถ้าคุณต้องการข้ามการตรวจสอบความพร้อมที่ไม่บังคับ ตัวอย่างเช่น คุณตั้งค่าการตรวจสอบความพร้อมสำหรับการคำนวณราคา อย่างไรก็ตาม คุณตัดสินใจที่จะข้ามการตรวจสอบนี้ในขณะที่ผลิตภัณฑ์ยังคงอยู่ในระยะของการออกแบบ ในกรณีนี้ ฟิลด์ **สถานะ** ถูกอัพเดตเป็น *ข้าม*

ทั้งนี้ขึ้นอยู่กับการตั้งค่าคอนฟิกของนโยบายความพร้อม เมื่อฟิลด์ **สถานะ** มีการอัพเดตสำหรับการตรวจสอบความพร้อมเป็น *ผ่าน* อาจต้องมีขั้นตอนพิเศษในการอนุมัติการตรวจสอบความพร้อม ในกรณีนี้ เลือก **การอนุมัติ** เพื่อให้การตรวจสอบความพร้อมเสร็จสมบูรณ์ ขั้นตอนการอนุมัตินี้จำเป็นต้องระบุเสมอเมื่อข้ามการตรวจสอบความพร้อม

เมื่อการตรวจสอบความพร้อมที่เปิดทั้งหมดสำหรับผลิตภัณฑ์ ผลิตภัณฑ์ย่อย หรือรุ่นใหม่ได้รับการประมวลผลและอนุมัติแล้วตามที่จำเป็น สินค้าจะกลายเป็นใช้งานโดยอัตโนมัติและจึงพร้อมใช้งาน

### <a name="process-checklist-readiness-checks"></a>ประมวลผลการตรวจสอบความพร้อมรายการตรวจสอบ

เมื่อต้องการเปิดรายการตรวจสอบ ให้เปิดหน้า **การตรวจสอบความพร้อม** จากนั้นบนบานหน้าต่างการดำเนินการ ให้เลือก **เริ่มต้นรายการตรวจสอบ** เมื่อคุณดำเนินการตรวจสอบเสร็จเรียบร้อยแล้ว ระบบจะตรวจสอบว่ามีการส่งผ่านการตรวจสอบความพร้อมหรือไม่ ตามการตั้งค่าในแบบสอบถาม ถ้าการตรวจสอบผ่าน ฟิลด์ **สถานะ** มีการอัพเดตฟิลด์เป็น *ผ่าน* ถ้าการตรวจสอบความพร้อมไม่บังคับ คุณสามารถข้ามการตรวจสอบความพร้อมได้ ในกรณีนี้ ฟิลด์ **สถานะ** ถูกอัพเดตเป็น *ข้าม*

ทั้งนี้ขึ้นอยู่กับการตั้งค่าคอนฟิกของนโยบายความพร้อม เมื่อฟิลด์ **สถานะ** มีการอัพเดตสำหรับการตรวจสอบความพร้อมเป็น *ผ่าน* อาจต้องมีขั้นตอนพิเศษในการอนุมัติการตรวจสอบความพร้อม ในกรณีนี้ เลือก **การอนุมัติ** เพื่อให้การตรวจสอบความพร้อมเสร็จสมบูรณ์ ขั้นตอนการอนุมัตินี้จำเป็นต้องระบุเสมอเมื่อข้ามการตรวจสอบความพร้อม

เมื่อการตรวจสอบความพร้อมที่เปิดทั้งหมดสำหรับผลิตภัณฑ์ ผลิตภัณฑ์ย่อย หรือรุ่นใหม่ได้รับการประมวลผลและอนุมัติแล้วตามที่จำเป็น สินค้าจะกลายเป็นใช้งานโดยอัตโนมัติและจึงพร้อมใช้งาน

## <a name="create-and-manage-product-readiness-policies"></a>สร้างและจัดการนโยบายความพร้อมของผลิตภัณฑ์

ใช้นโยบายความพร้อมของผลิตภัณฑ์เพื่อจัดการการตรวจสอบความพร้อมที่ใช้กับผลิตภัณฑ์ เนื่องจากมีการกำหนดนโยบายความพร้อมให้กับประเภทวิศวกรรม การตรวจสอบทั้งหมดในนโยบายความพร้อมนำไปใช้กับผลิตภัณฑ์วิศวกรรมทั้งหมดที่ยึดตามประเภทวิศวกรรม สำหรับข้อมูลเพิ่มเติม ดูที่ [ประเภทผลิตภัณฑ์วิศวกรรมและรุ่นวิศวกรรม](engineering-versions-product-category.md)

นโยบายความพร้อมแต่ละรายการจะประกอบด้วยชุดของการตรวจสอบความพร้อม เมื่อมีการกำหนดนโยบายความพร้อมให้กับประเภทผลิตภัณฑ์วิศวกรรม ผลิตภัณฑ์ทั้งหมดที่สร้างขึ้นจากประเภทผลิตภัณฑ์วิศวกรรมดังกล่าวจะมีการตรวจสอบความพร้อมที่ระบุไว้ในนโยบายความพร้อม

เมื่อต้องการทำงานกับนโยบายความพร้อมของผลิตภัณฑ์ ให้ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> การตั้งค่า \> นโยบายความพร้อมของผลิตภัณฑ์** จากนั้น ทำตามหนึ่งในขั้นตอนเหล่านี้

- เมื่อต้องการสร้างนโยบายใหม่ ให้เลือก **สร้าง** ในบานหน้าต่างการดำเนินการ แล้วกำหนดฟิลด์ตามที่อธิบายไว้ในส่วนย่อยต่อไปนี้
- ถ้าต้องการแก้ไขนโยบายที่มีอยู่ ให้เลือกนโยบายในบานหน้าต่างรายการ เลือก **แก้ไข** บนบานหน้าต่างการดำเนินการ แล้วกำหนดฟิลด์ตามที่อธิบายไว้ในส่วนย่อยต่อไปนี้
- ถ้าต้องการลบนโยบายที่มีอยู่ ให้เลือกนโยบายในบานหน้าต่างรายการ เลือก **แก้ไข** บนบานหน้าต่างการดำเนินการ จากนั้นบนแท็บด่วน **ทั่วไป** โปรดตรวจสอบให้แน่ใจว่าตัวเลือก **ที่ใช้งานอยู่** ตั้งค่าเป็น *ไม่* จากนั้น เลือก **ลบ** บนบานหน้าต่างการดำเนินการ

### <a name="header"></a>หัวข้อ

ตั้งค่าฟิลด์ต่อไปนี้บนส่วนหัวของนโยบายความพร้อมของผลิตภัณฑ์

| ฟิลด์ | คำอธิบาย |
|---|---|
| ชื่อ | ป้อนชื่อสำหรับนโยบาย |
| คำอธิบาย | ป้อนคำอธิบายเกี่ยวกับนโยบาย |

### <a name="general-fasttab"></a>FastTab ทั่วไป

ตั้งค่าฟิลด์ต่อไปนี้บนแท็บด่วน **ทั่วไป** ของนโยบายความพร้อมของผลิตภัณฑ์

| ฟิลด์ | คำอธิบาย |
|---|---|
| ชนิดผลิตภัณฑ์ | เลือกว่ามีนโยบายใช้กับผลิตภัณฑ์ของชนิด *สินค้า* หรือ *การบริการ* คุณไม่สามารถเปลี่ยนการตั้งค่านี้ได้หลังจากที่คุณบันทึกเรกคอร์ดแล้ว |
| ที่ใช้งานอยู่ | ใช้ตัวเลือกนี้เพื่อรักษานโยบายความพร้อมของคุณ ตั้งค่าเป็น *ใช่* สำหรับนโยบายความพร้อมทั้งหมดที่คุณใช้ ตั้งค่าเป็น *ไม่* เพื่อทำเครื่องหมายนโยบายความพร้อมเป็นไม่ได้ใช้งานอยู่เมื่อไม่ใช้งาน โปรดทราบว่าคุณไม่สามารถยกเลิกการเรียกใช้นโยบายความพร้อมที่กำหนดให้กับประเภทผลิตภัณฑ์วิศวกรรม และคุณสามารถลบได้เฉพาะนโยบายการนำออกใช้ที่ไม่ได้ใช้งานอยู่เท่านั้น |

### <a name="readiness-control-fasttab"></a>แท็บด่วนการควบคุมความพร้อม

สำหรับการตรวจสอบความพร้อมแต่ละชนิดที่คุณต้องการรวมไว้ในนโยบาย ให้เพิ่มแถวบนแท็บด่วน **การควบคุมความพร้อม** ใช้ปุ่มต่อไปนี้บนแถบเครื่องมือแท็บด่วนเพื่อเพิ่มและลบแถวตามที่คุณต้องการ:

- **เพิ่มการตรวจสอบ** – เพิ่มการตรวจสอบความพร้อมมาตรฐานให้กับนโยบาย เมื่อคุณเลือกปุ่มนี้ กล่องโต้ตอบ **เพิ่มการตรวจสอบ** จะปรากฏขึ้น ดั้งนั้น คุณสามารถเลือกจากรายการการตรวจสอบที่มีอยู่ได้
- **เพิ่มแบบสอบถามที่มีอยู่** – เพิ่มแถวว่างให้กับกริด จากนั้นคุณจะสามารถกำหนดแบบสอบถามที่มีอยู่ได้โดยการตั้งค่าฟิลด์ในตารางต่อไปนี้
- **คัดลอก** – เพิ่มสำเนาของแถวที่เลือกไปที่กริด
- **ลบ** – ลบแถวที่เลือกออกจากกริด

สำหรับแต่ละแถวที่คุณเพิ่ม ให้ตั้งค่าฟิลด์ต่อไปนี้

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| พื้นที่กระบวนการ | เลือกพื้นที่ที่มีการเชื่อมโยงกับการตรวจสอบ |
| ชนิดข้อมูล | เลือกว่าการตรวจสอบเป็นการตรวจสอบของระบบ การตรวจสอบด้วยตนเอง หรือรายการตรวจสอบ (แบบสอบถาม) |
| ชื่อ | ถ้าการตรวจสอบเป็นรายการตรวจสอบ ให้ป้อนชื่อ สำหรับการตรวจสอบของระบบและการตรวจสอบด้วยตนเอง ฟิลด์นี้จะถูกตั้งค่าโดยอัตโนมัติ |
| คำอธิบาย | ถ้าการตรวจสอบเป็นรายการตรวจสอบ ให้ป้อนคำอธิบาย สำหรับการตรวจสอบของระบบและการตรวจสอบด้วยตนเอง ฟิลด์นี้จะได้รับการตั้งค่าโดยอัตโนมัติ และคำอธิบายจะอธิบายถึงโฟกัสของการตรวจสอบ |
| นำการตรวจสอบไปใช้บน | เลือกว่าแถวควรสร้างการตรวจสอบความพร้อมเพื่อตอบสนองกับผลิตภัณฑ์ที่นำออกใช้ใหม่ ผลิตภัณฑ์ย่อยที่นำออกใช้ หรือรุ่นที่นำออกใช้หรือไม่ |
| ดำเนินการใน | เลือกว่าการตรวจสอบความพร้อมที่แถวสร้างนำไปใช้กับบริษัททั้งหมดหรือบริษัทเดียว |
| บริษัท | ถ้าคุณตั้งค่าฟิลด์ **ดำเนินการใน** เป็น *บริษัทเดียว* ให้เลือกบริษัท |
| ชนิดเจ้าของ | เลือกว่าการตรวจสอบความพร้อมควรมีการกำหนดแถวให้กับบุคคลหรือทีมงานหรือไม่ |
| เจ้าของ | เลือกบุคคลหรือทีมงานที่การตรวจสอบความพร้อมควรมีการกำหนดแถวให้ |
| แบบสอบถาม | เลือกแบบสอบถามที่ควรใช้สำหรับรายการตรวจสอบ รายการตรวจสอบเป็นรายการตรวจสอบเฉพาะที่ในบริษัทที่ทำการตรวจสอบความพร้อม ระบบต้องสามารถประเมินว่ารายการตรวจสอบได้รับการตอบอย่างถูกต้องหรือไม่ ดังนั้น จึงต้องตั้งค่ารายการตรวจสอบเพื่อให้การประเมินทำตามคำตอบที่ถูกต้อง สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการสร้างแบบสอบถาม ให้ดู [การใช้แบบสอบถาม](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/using-questionnaires) และหัวข้อที่เกี่ยวข้อง |
| การอนุมัติอัตโนมัติ | เรกคอร์ดการตรวจสอบความพร้อมจะรวมกล่องกาเครื่องหมาย **ที่อนุมัติ** ซึ่งแสดงสถานะการอนุมัติ เลือกกล่องกาเครื่องหมาย **การอนุมัติโดยอัตโนมัติ** สำหรับการตรวจสอบที่ควรตั้งค่าให้อนุมัติทันทีหลังจากที่ผู้ใช้ที่ได้รับมอบหมายเสร็จสมบูรณ์แล้ว ยกเลิกการเลือกกล่องกาเครื่องหมายนี้เพื่อกำหนดให้มีการอนุมัติแบบชัดแจ้งเป็นขั้นตอนพิเศษ |
| ข้อบังคับ | เลือกกล่องกาเครื่องหมายนี้สำหรับการตรวจสอบที่ต้องทำให้เสร็จสมบูรณ์โดยผู้ใช้ที่ได้รับมอบหมาย ไม่สามารถข้ามการตรวจสอบบังคับ |
