## <a name="types-of-readiness-checks"></a><span data-ttu-id="1f6e1-110">ชนิดของการตรวจสอบความพร้อม</span><span class="sxs-lookup"><span data-stu-id="1f6e1-110">Types of readiness checks</span></span>

- <span data-ttu-id="1f6e1-114">**การตรวจสอบด้วยตนเอง** – ผู้ใช้จะตรวจสอบว่าเรกคอร์ดมีผลบังคับใช้หรือไม่</span><span class="sxs-lookup"><span data-stu-id="1f6e1-114">**Manual check** – A user verifies whether the record is valid.</span></span> <span data-ttu-id="1f6e1-115">ตัวอย่างเช่น การตรวจสอบความพร้อมอาจจำเป็นต้องมีการตรวจสอบความถูกต้องของการตั้งค่าใบสั่งเริ่มต้น</span><span class="sxs-lookup"><span data-stu-id="1f6e1-115">For example, a readiness check might require validation of the default order settings.</span></span> <span data-ttu-id="1f6e1-116">ในบางกรณี เช่น เมื่อผลิตภัณฑ์ยังคงถูกออกแบบ และดังนั้นจึงไม่มีการจัดเก็บอยู่ในสินค้าคงคลัง จะไม่จำเป็นต้องมีการตั้งค่าใบสั่งเริ่มต้น</span><span class="sxs-lookup"><span data-stu-id="1f6e1-116">In some cases, such as when the product is still being designed and therefore won't be placed in stock, no default order settings are required.</span></span> <span data-ttu-id="1f6e1-117">อย่างไรก็ตาม การตั้งค่าใบสั่งเริ่มต้นอาจจำเป็นสำหรับผลิตภัณฑ์ชนิดเดียวกัน เนื่องจากผลิตภัณฑ์สามารถจัดเก็บอยู่ในสินค้าคงคลังได้</span><span class="sxs-lookup"><span data-stu-id="1f6e1-117">However, default order settings might be required for another product of the same type, because the product can be held in stock.</span></span> <span data-ttu-id="1f6e1-118">ผู้ใช้รับผิดชอบในการทราบวิธีการตัดสินใจอย่างถูกต้องว่าจำเป็นต้องมีการตรวจสอบความพร้อมหรือไม่</span><span class="sxs-lookup"><span data-stu-id="1f6e1-118">The user is responsible for knowing how to correctly decision whether a readiness check is required.</span></span>

## <a name="how-readiness-checks-are-created-for-a-new-product-variant-or-version"></a><span data-ttu-id="1f6e1-122">วิธีการสร้างการตรวจสอบความพร้อมสำหรับผลิตภัณฑ์ ผลิตภัณฑ์ย่อย หรือรุ่นใหม่</span><span class="sxs-lookup"><span data-stu-id="1f6e1-122">How readiness checks are created for a new product, variant, or version</span></span>

<span data-ttu-id="1f6e1-132">ถ้ามีการสร้าง **รุ่น** วิศวกรรมสำหรับผลิตภัณฑ์ ระบบจะตรวจสอบว่ามีการตั้งค่าการตรวจสอบความพร้อมในชนิดผลิตภัณฑ์วิศวกรรมแล้วหรือไม่</span><span class="sxs-lookup"><span data-stu-id="1f6e1-132">If a new engineering **version** is created for a product, the system checks whether readiness checks have been set up on the engineering product category.</span></span> <span data-ttu-id="1f6e1-133">(สามารถใช้การตรวจสอบความพร้อมในระดับรุ่นวิศวกรรม) ถ้ามีการตั้งค่าการตรวจสอบความพร้อมแล้ว เหตุการณ์ต่อไปนี้จะเกิดขึ้น:</span><span class="sxs-lookup"><span data-stu-id="1f6e1-133">(Readiness checks can be applied on the engineering version level.)
