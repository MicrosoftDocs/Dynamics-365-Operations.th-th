# <a name="manage-changes-to-engineering-products"></a>จัดการการเปลี่ยนแปลงที่เกิดขึ้นกับผลิตภัณฑ์วิศวกรรม

[!include [banner](../includes/banner.md)]

การจัดการการเปลี่ยนแปลงทางวิศวกรรมมีกระบวนการที่มีโครงสร้างสำหรับการจัดการการเปลี่ยนแปลงไปยังผลิตภัณฑ์วิศวกรรม คุณสามารถใช้กระบวนการ *ร้องขอการเปลี่ยนแปลงทางวิศวกรรม* เพื่อเสนอและร้องขอการเปลี่ยนแปลง แล้วจากนั้น ใช้กระบวนการ *ใบสั่งเปลี่ยนแปลงทางวิศวกรรม* เพื่อทำให้การเปลี่ยนแปลงดังกล่าวเป็นจริงได้ ผู้ใช้สามารถสร้างคำขอเปลี่ยนแปลงทางวิศวกรรมหรือใบสั่งเปลี่ยนแปลงทางวิศวกรรม และมีกระบวนการสำหรับการตรวจทานและการอนุมัติ การประเมินผลกระทบต่อธุรกรรมที่มีอยู่ และการติดตาม

## <a name="engineering-change-requests"></a>คำขอเปลี่ยนแปลงทางวิศวกรรม

กระบวนการร้องขอการเปลี่ยนแปลงทางวิศวกรรมช่วยให้คุณสามารถบันทึกการร้องขอสำหรับการเปลี่ยนแปลงจากแผนกที่เกี่ยวข้องทั้งหมดในบริษัทของคุณ ไม่ว่าคุณจะทำงานเป็นวิศวกร หรือในแผนกการผลิต การจัดหา คลังสินค้า หรือฝ่ายขาย: ทุกคนสามารถใช้คำขอเปลี่ยนแปลงทางวิศวกรรมเพื่อร้องขอการเปลี่ยนแปลง การเปลี่ยนแปลงนี้อาจเป็นความคิดสำหรับผลิตภัณฑ์ใหม่ ปัญหาที่คุณค้นพบในขณะที่คุณกำลังทำงานกับผลิตภัณฑ์ที่มีอยู่ ข้อเสนอแนะสำหรับการปรับปรุงผลิตภัณฑ์ที่มีอยู่ หรือสิ่งอื่น

หลังจากที่มีผู้ส่งการร้องขอการเปลี่ยนแปลงแล้ว กระบวนการ *รีวิวและการอนุมัติ* จะถูกจัดการโดยลำดับงานที่ระบุว่าใครจะต้องอนุมัติการเปลี่ยนแปลง (ตัวอย่างเช่น เจ้าของผลิตภัณฑ์)

เมื่อต้องการตั้งค่าลำดับงานสำหรับใบสั่งเปลี่ยนแปลงทางวิศวกรรมหรือคำขอการเปลี่ยนแปลงทางวิศวกรรม ให้ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> ลำดับงานทางวิศวกรรม** เลือก **สร้าง** เลือกว่าจะใช้ลำดับงานเพื่อรีวิวใบสั่งเปลี่ยนแปลงทางวิศวกรรมหรือคำขอการเปลี่ยนแปลงทางวิศวกรรม แล้วจากนั้นจึงตั้งค่าคอนฟิกลำดับงาน

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับลำดับงาน โปรดดูที่ [ภาพรวมของระบบลำดับงาน](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับเจ้าของผลิตภัณฑ์ ให้ดูที่ [เจ้าของผลิตภัณฑ์](product-owner.md)

### <a name="create-a-new-engineering-change-request"></a>สร้างคำขอเปลี่ยนแปลงทางวิศวกรรมใหม่

เมื่อต้องการสร้างคำขอเปลี่ยนแปลงวิศวกรรม ให้ทำตามขั้นตอนใดขั้นตอนหนึ่งต่อไปนี้

- ให้ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> ทั่วไป \> การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> คำขอเปลี่ยนแปลงทางวิศวกรรม** และจากนั้น เลือก **สร้าง** ในบานหน้าต่างการดำเนินการ
- เปิดหน้า **รายละเอียดของผลิตภัณฑ์** สำหรับผลิตภัณฑ์วิศวกรรมที่มีอยู่ จากนั้น บนบานหน้าต่างการดำเนินการ บนแท็บ **วิศวกรรม** ในกลุ่ม **การจัดการการเปลี่ยนแปลงทางวิศวกรรม** เลือก **คำขอเปลี่ยนแปลงทางวิศวกรรม \> คำขอเปลี่ยนแปลงทางวิศวกรรมใหม่**

จะมีการสร้างคำขอเปลี่ยนแปลงใหม่ ขณะนี้คุณสามารถตั้งค่าฟิลด์บน FastTab แต่ละแท็บ ตามที่อธิบายไว้ในส่วนย่อยต่อไปนี้

#### <a name="general-fasttab"></a>FastTab ทั่วไป

FastTab **ทั่วไป** ช่วยให้คุณสามารถระบุคำอธิบายพื้นฐานของคำขอเปลี่ยนแปลงได้ ตารางต่อไปนี้อธิบายถึงฟิลด์บน FastTab นี้

| ฟิลด์ | คำอธิบาย |
|---|---|
| คำขอเปลี่ยนแปลง | ป้อนชื่อสำหรับคำขอเปลี่ยนแปลงทางวิศวกรรม |
| ชื่อตำแหน่ง | ป้อนข้อความที่อธิบายโดยย่อหรือระบุการเปลี่ยนแปลงในคำขอ |
| ลำดับความสำคัญ | เลือกค่าเพื่อระบุระดับความสำคัญของการเปลี่ยนแปลง คุณสามารถเลือกกำหนดค่าที่มีอยู่สำหรับบริษัทของคุณได้ตามความจำเป็น (สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [สร้างค่าทั่วไปสำหรับการจัดการการเปลี่ยนแปลงทางวิศวกรรม](engineering-change-management-setup.md)) |
| ประเภท | เลือกค่าที่จะอธิบายถึงชนิดของการเปลี่ยนแปลงที่คุณกำลังร้องขอ คุณสามารถเลือกกำหนดค่าที่มีอยู่สำหรับบริษัทของคุณได้ตามความจำเป็น (สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [สร้างค่าทั่วไปสำหรับการจัดการการเปลี่ยนแปลงทางวิศวกรรม](engineering-change-management-setup.md)) |
| ความรุนแรง | เลือกค่าเพื่อบ่งชี้ความรุนแรงของปัญหาที่ควรได้รับการแก้ไขโดยการใช้คำขอ คุณสามารถเลือกกำหนดค่าที่มีอยู่สำหรับบริษัทของคุณได้ตามความจำเป็น (สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [สร้างค่าทั่วไปสำหรับการจัดการการเปลี่ยนแปลงทางวิศวกรรม](engineering-change-management-setup.md)) |
| ขอโดย | ชื่อของผู้ใช้ที่สร้างคำขอ |
| เมื่อ | วันที่ที่สร้างคำขอ |
| สถานะ | สถานะของคำขอ เมื่อมีการสร้างคำขอเป็นครั้งแรก สถานะจะเป็น *สร้างแล้ว* เมื่อคำขอได้รับการอนุมัติ สถานะจะเปลี่ยนเป็น *อนุมัติแล้ว* ถ้ามีการสร้างใบสั่งเปลี่ยนแปลงที่เกี่ยวข้องสำหรับคำขอ สถานะจะเปลี่ยนเป็น *ติดตามแล้ว* |
| ใบสั่งเปลี่ยนแปลง | หมายเลขใบสั่งเปลี่ยนแปลง ถ้ามีการติดตามคำขอเปลี่ยนแปลงผ่านทางใบสั่งเปลี่ยนแปลง |

#### <a name="information-fasttab"></a>FastTab ข้อมูล

FastTab **ข้อมูล** จะช่วยให้คุณสามารถเพิ่มข้อมูลเพิ่มเติมเกี่ยวกับคำขอ

เมื่อต้องการเพิ่มแถวไปยังกริด ให้เลือก **สร้าง** บนแถบเครื่องมือด้านบนของกริด แล้วจากนั้น เลือกตัวเลือกใดตัวเลือกหนึ่งต่อไปนี้:

- **ไฟล์** – อัปโหลดไฟล์
- **รูปภาพ** – อัปโหลดไฟล์รูปภาพ
- **บันทึกย่อ** – ป้อนบันทึกย่อโดยตรงในกริด
- **URL** – ป้อน URL โดยตรงในกริด

ตารางต่อไปนี้อธิบายถึงฟิลด์ในแต่ละแถว

| ฟิลด์ | คำอธิบาย |
|---|---|
| วันที่และเวลาที่สร้าง | วันที่และเวลาที่มีการสร้างแถว |
| ชนิดข้อมูล | ชนิดของข้อมูลที่มีการสร้างแถวให้ (ไฟล์ รูปภาพ บันทึกย่อ หรือ URL) |
| คำอธิบาย | ป้อนคำอธิบายเกี่ยวกับแถว |
| ข้อจำกัด | ค่าที่บ่งชี้ว่ามีการเพิ่มข้อมูลที่มาจากแหล่งที่มาภายในหรือภายนอก |
| แนบ | กล่องกาเครื่องหมายที่เลือกบ่งชี้ว่าแถวมีเอกสารแนบ (ไฟล์หรือรูปภาพ) เมื่อต้องการดาวน์โหลดเอกสารแนบ ให้เลือกแถว แล้วจากนั้น เลือก **เปิด** บนแถบเครื่องมือด้านบนของกริด |

#### <a name="products-fasttab"></a>FastTab ผลิตภัณฑ์

FastTab **ผลิตภัณฑ์** ช่วยให้คุณสามารถแสดงรายการผลิตภัณฑ์แต่ละรายการที่ได้รับผลกระทบจากคำขอเปลี่ยนแปลง คุณสามารถใช้ปุ่มบนแถบเครื่องมือเพื่อเพิ่มผลิตภัณฑ์ลงในกริด หรือเพื่อลบผลิตภัณฑ์

รายการนี้ใช้เพื่อให้ข้อมูลเท่านั้น ดังนั้น คุณจึงสามารถเพิ่มผลิตภัณฑ์ที่เกี่ยวข้องได้มากเท่าที่คุณพิจารณาว่าเกี่ยวข้อง ถ้าคุณสร้างคำขอเปลี่ยนแปลงจากหน้า **รายละเอียดผลิตภัณฑ์** สำหรับผลิตภัณฑ์ที่มีอยู่ ผลิตภัณฑ์ดังกล่าวควรแสดงรายการอยู่บน FastTab **ผลิตภัณฑ์** หลังจากที่คุณบันทึกเรกคอร์ดคำขอ

#### <a name="source-fasttab"></a>FastTab แหล่งที่มา

FastTab **แหล่งที่มา** ช่วยให้คุณสามารถติดตามจุดเริ่มต้นของคำขอเปลี่ยนแปลงได้ ซึ่งจะมีประโยชน์ ตัวอย่างเช่น ถ้าคุณต้องการดูว่ามีการสร้างคำขอเปลี่ยนแปลงจากใบสั่งขายหรือไม่ ใครเป็นผู้สร้าง และสร้างในบริษัทใด

### <a name="evaluate-the-business-impact-of-a-change-request"></a>ประเมินผลกระทบทางธุรกิจของคำขอเปลี่ยนแปลง

เมื่อคุณรีวิวคำขอสำหรับการเปลี่ยนแปลง คุณสามารถค้นหาการอ้างอิงได้ ด้วยวิธีนี้ คุณสามารถประเมินผลกระทบของการเปลี่ยนแปลงที่ร้องขอในธุรกรรมที่เปิด เช่น ใบสั่งขาย ใบสั่งผลิต และปริมาณคงคลังคงเหลือ

1. ให้ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> ทั่วไป \> การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> คำขอเปลี่ยนแปลงทางวิศวกรรม**
1. เปิดคำขอการเปลี่ยนแปลงที่มีอยู่ หรือเลือก **สร้าง** ในบานหน้าต่างการดำเนินการ เพื่อสร้างคำขอเปลี่ยนแปลงใหม่
1. บนบานหน้าต่างการดำเนินการ บนแท็บ **คำขอเปลี่ยนแปลง** ในกลุ่ม **ผลกระทบทางธุรกิจ** ให้เลือกปุ่มใดปุ่มหนึ่งต่อไปนี้:

    - **ค้นหา** – สแกนธุรกรรมที่เปิดทั้งหมด แล้วจากนั้น เปิดกล่องโต้ตอบ **ผลกระทบทางธุรกิจเมื่อต้องการเปิดธุรกรรม** ซึ่งแสดงรายการธุรกรรมทั้งหมดที่จะได้รับผลกระทบจากการเปลี่ยนแปลง
    - **ดูการค้นหาก่อนหน้านี้** – เปิดกล่องโต้ตอบ **ผลกระทบทางธุรกิจเมื่อต้องการเปิดธุรกรรม** ซึ่งแสดงรายการผลลัพธ์ของการค้นหาก่อนหน้านี้ (ไม่ได้ทำการค้นหาใหม่)

1. ถ้าพบว่าปัญหาที่ต้องการการเปลี่ยนแปลงเป็นแบบวิกฤติ คุณสามารถบล็อคธุรกรรมที่เปิดหรือแจ้งให้ผู้ใช้ที่รับผิดชอบทราบโดยใช้ปุ่มบนแถบเครื่องมือในกล่องโต้ตอบ **ผลกระทบทางธุรกิจเมื่อต้องการเปิดธุรกรรม**

### <a name="create-a-change-order-from-a-change-request"></a>สร้างใบสั่งเปลี่ยนแปลงจากคำขอเปลี่ยนแปลง

วิศวกรที่กำลังทบทวนคำขอเปลี่ยนแปลงทางวิศวกรรม สามารถสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรมโดยตรงจากหน้า **คำขอเปลี่ยนแปลงทางวิศวกรรม** บนบานหน้าต่างการดำเนินการ บนแท็บ **คำขอเปลี่ยนแปลง** ในกลุ่ม **ใบสั่งเปลี่ยนแปลงทางวิศวกรรม** ให้เลือก **คัดลอกลิงค์และผลิตภัณฑ์**

## <a name="engineering-change-orders"></a>ใบสั่งเปลี่ยนแปลงทางวิศวกรรม

ใบสั่งเปลี่ยนแปลงทางวิศวกรรมมีกระบวนการที่มีโครงสร้างสำหรับการทำการเปลี่ยนแปลงไปยังผลิตภัณฑ์วิศวกรรม คุณเสนอการเปลี่ยนแปลงโดยใช้สำเนาของข้อมูลที่เกี่ยวข้องกับวิศวกรรม ข้อมูลหลักจริงจะไม่ได้รับผลกระทบ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อมูลที่เกี่ยวข้องกับวิศวกรรม ดูที่ [ประเภทผลิตภัณฑ์วิศวกรรมและรุ่นวิศวกรรม](engineering-versions-product-category.md)

คุณสามารถสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรมซึ่งขึ้นอยู่กับคำขอเปลี่ยนแปลงทางวิศวกรรมที่ได้รับอนุมัติ นอกจากนี้ วิศวกรยังสามารถสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรมตั้งแต่เริ่มต้นได้ด้วย คุณสามารถรวมผลิตภัณฑ์หลายรายการในใบสั่งเปลี่ยนแปลงทางวิศวกรรมเดียว โดยทำตามขั้นตอนใดๆ เหล่านี้:

- เลือกผลิตภัณฑ์ด้วยตนเอง
- ใช้สูตรการผลิต (BOM) เพื่อรวมผลิตภัณฑ์ที่ต่ำกว่าในโครงสร้างผลิตภัณฑ์ (นั่นคือ รายการรอง)
- ใช้การค้นหาที่ใช้เพื่อรวมผลิตภัณฑ์ที่สูงกว่าในโครงสร้างผลิตภัณฑ์ (นั่นคือ รายการหลัก)

หลังจากข้อเสนอสำหรับการเปลี่ยนแปลงเสร็จเรียบร้อยแล้ว กระบวนการรีวิวและการอนุมัติจะถูกจัดการโดยลำดับงาน คุณสามารถตั้งค่าลำดับงานต่างๆ ได้ตามระดับความสำคัญและความรุนแรง

เมื่อต้องการตั้งค่าลำดับงานสำหรับใบสั่งเปลี่ยนแปลงทางวิศวกรรมหรือคำขอการเปลี่ยนแปลงทางวิศวกรรม ให้ไปที่ **การจัดการการเปลี่ยนแปลงทางวิศวกรรม \> ลำดับงานทางวิศวกรรม** เลือก **สร้าง** เลือกว่าจะใช้ลำดับงานเพื่อรีวิวใบสั่งเปลี่ยนแปลงทางวิศวกรรมหรือคำขอการเปลี่ยนแปลงทางวิศวกรรม แล้วจากนั้นจึงตั้งค่าคอนฟิกลำดับงาน

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับลำดับงาน โปรดดูที่ [ภาพรวมของระบบลำดับงาน](../../fin-ops-core/fin-ops/organization-administration/overview-workflow-system.md)

ต่อไปนี้เป็นส่วนหนึ่งของผู้มีส่วนได้เสียทั่วไปที่อาจต้องอนุมัติใบสั่งเปลี่ยนแปลงทางวิศวกรรม:

- **Pเจ้าของผลิตภัณฑ์** – สำหรับข้อมูลเพิ่มเติมเกี่ยวกับเจ้าของผลิตภัณฑ์ ให้ดูที่ [เจ้าของผลิตภัณฑ์](product-owner.md)
- **หัวหน้าทีมงานที่รับผิดชอบ** – ฟิลด์ **วิศวกร** ในมุมมอง **ส่วนหัว** ของใบสั่งเปลี่ยนแปลงทางวิศวกรรม แสดงให้เห็นถึงวิศวกรที่สร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรม ถ้าวิศวกรเป็นสมาชิกของทีมงานที่กำหนดไว้ในระบบ ฟิลด์ **ที่รับผิดชอบ** จะแสดงผู้นำของทีมงานนั้น
- **แผนกการเงิน** – แผนกการเงินอาจต้องรีวิวกรณีที่การเปลี่ยนแปลงเกี่ยวข้องกับต้นทุนที่สูง

คุณสามารถเลือกว่าควรประมวลผลใบสั่งเปลี่ยนแปลงทางวิศวกรรมโดยตรงหลังจากที่ได้รับอนุมัติแล้วตามเป็นส่วนหนึ่งของลำดับงาน หรือควรทำการประมวลผลในภายหลังตามขั้นตอนด้วยตนเอง ในระหว่างการประมวลผลใบสั่งเปลี่ยนแปลงทางวิศวกรรม ข้อมูลวิศวกรรมบนผลิตภัณฑ์จริงจะถูกอัปเดต

ขณะที่คุณกำลังตรวจทานคำขอสำหรับการเปลี่ยนแปลง ในบานหน้าต่างการดำเนินการ บนแท็บ **คำขอเปลี่ยนแปลง** ในกลุ่ม **ผลกระทบทางธุรกิจ** ให้เลือก **ค้นหา** เพื่อประเมินผลกระทบของการเปลี่ยนแปลงที่นำเสนอในธุรกรรมที่เปิด เช่น ใบสั่งขาย ใบสั่งผลิต และปริมาณคงคลังคงเหลือ ผลลัพธ์จะแสดงอยู่ในกล่องโต้ตอบ **ผลกระทบทางธุรกิจเมื่อต้องการเปิดธุรกรรม** ที่คุณสามารถเลือกธุรกรรมที่ได้รับผลกระทบ แล้วจากนั้น ใช้คำสั่งในแถบเครื่องมือเพื่อดูข้อมูลเพิ่มเติม โปรดแจ้งผู้ใช้ที่รับผิดชอบ หรือบล็อกธุรกรรม

### <a name="engineering-change-orders-in-engineering-or-operational-companies"></a>ใบสั่งเปลี่ยนแปลงทางวิศวกรรมในบริษัทวิศวกรรมหรือบริษัทที่ดำเนินงาน

ดังที่อธิบายไว้ใน [บริษัทวิศวกรรมและกฎความเป็นเจ้าของข้อมูล](engineering-org-data-ownership-rules.md) ข้อมูลผลิตภัณฑ์ที่คุณสามารถแก้ไขจะแตกต่างกันไป โดยขึ้นอยู่กับชนิดของนิติบุคคลที่คุณกำลังทำงานอยู่ (บริษัทวิศวกรรมเทียบกับบริษัทที่ดำเนินงาน) นอกจากนี้ กฎของความเป็นเจ้าของข้อมูลยังใช้กับใบสั่งเปลี่ยนแปลงทางวิศวกรรมด้วย ดังนั้น ทั้งนี้ขึ้นอยู่กับนิติบุคคลที่คุณสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรม สามารถทำการเปลี่ยนแปลงชนิดต่างๆ ได้ ยกตัวอย่างเช่น

- สำหรับใบสั่งเปลี่ยนแปลงทางวิศวกรรมใน **บริษัทวิศวกรรม** คุณสามารถทำการเปลี่ยนแปลงพื้นฐานของข้อมูลวิศวกรรม ตัวอย่างเช่น คุณสามารถสร้างผลิตภัณฑ์รุ่นใหม่ เปลี่ยนโครงสร้างของผลิตภัณฑ์ผ่านทาง BOM และเปลี่ยนค่าแอททริบิวต์ทางวิศวกรรม สำหรับผลิตภัณฑ์ที่ได้รับผลกระทบแต่ละรายการ ให้เลือกค่าใดค่าหนึ่งดังต่อไปนี้ในฟิลด์ **ผลกระทบ**:

    - **ไม่มี** – อัปเดตผลิตภัณฑ์รุ่นที่มีอยู่ (การอัปเดตในรุ่น)
    - **รุ่นใหม่** – สร้างรุ่นใหม่ที่ยึดตามรุ่นผลิตภัณฑ์ที่เลือก
    - **ผลิตภัณฑ์ใหม่** – สร้างผลิตภัณฑ์ใหม่ทั้งหมดหรือผลิตภัณฑ์ย่อยที่ยึดตามรุ่นผลิตภัณฑ์ที่เลือก

- สำหรับใบสั่งเปลี่ยนแปลงทางวิศวกรรมใน **บริษัทที่ดำเนินงาน** คุณสามารถเปลี่ยนข้อมูลด้านลอจิสติกส์ของผลิตภัณฑ์ได้ ตัวอย่างเช่น คุณสามารถทำให้ BOM ที่มีอยู่สมบูรณ์ด้วยการตั้งค่าสำหรับการจัดหา เพิ่มกระบวนการผลิตเฉพาะที่หรือ BOM เฉพาะที่ และแม้แต่ทำให้ BOM สมบูรณ์โดยการเพิ่มบรรทัด BOM ใหม่สำหรับวัสดุบรรจุภัณฑ์เฉพาะที่ น้ำมันหล่อลื่น หรือคำแนะนำในภาษาท้องถิ่น การตกแต่งที่ผู้ใช้ที่ทำในบริษัทที่ดำเนินงานจะถูกรักษาไว้ เมื่อมีการส่งการอัปเดตใหม่จากบริษัทวิศวกรรม สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [บริษัทวิศวกรรมและกฎความเป็นเจ้าของข้อมูล](engineering-org-data-ownership-rules.md)

    เมื่อมีการประมวลผลใบสั่งที่มีการเปลี่ยนแปลงทางวิศวกรรมในบริษัทวิศวกรรม ผลิตภัณฑ์จะถูกสร้างและ/หรืออัปเดตในบริษัทวิศวกรรมเท่านั้น ดังนั้น ถ้าข้อมูลหลักของผลิตภัณฑ์ควรมีการอัปเดตด้วย คุณยังต้องนำผลิตภัณฑ์ออกใช้กับบริษัทที่ดำเนินงานด้วย

- คุณสามารถนำผลิตภัณฑ์ออกใช้จากใบสั่งเปลี่ยนแปลงทางวิศวกรรมโดยตรง เปิดใบสั่งเปลี่ยนแปลง และจากนั้น บนบานหน้าต่างการดำเนินการ บนแท็บ **ใบสั่งเปลี่ยนแปลง** ในกลุ่ม **การนำผลิตภัณฑ์ออกใช้** ให้เลือก **นำโครงสร้างผลิตภัณฑ์ออกใช้** กระบวนการทำงานเช่นเดียวกับการทำงานเมื่อคุณนำผลิตภัณฑ์ออกใช้จากหน้า **ผลิตภัณฑ์ที่นำออกใช้** สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [นำโครงสร้างผลิตภัณฑ์ออกใช้](release-product-structure.md)
- คุณสามารถให้ผลิตภัณฑ์ถูกนำออกใช้โดยอัตโนมัติจากใบสั่งเปลี่ยนแปลงทางวิศวกรรม โดยยึดตามปัจจัยต่อไปนี้:

    - นำออกใช้อีกครั้งไปยังบริษัทที่มีการนำผลิตภัณฑ์ออกใช้ก่อนหน้านี้ เลือก **ค้นหา** เพื่อสแกนการนำออกใช้ก่อนหน้านี้ทั้งหมด แล้วจากนั้น เลือก **มุมมอง** เพื่อดูผลลัพธ์ หน้า **มุมมอง** แสดงการนำออกใช้ผลิตภัณฑ์ก่อนหน้านี้ และคุณสามารถเลือกผลิตภัณฑ์ที่คุณต้องการนำออกใช้ใหม่ได้ จากนั้น ปิดหน้า **มุมมอง** และเลือก **ประมวลผล** เพื่อนำผลิตภัณฑ์ที่เลือกออกใช้อีกครั้ง
    - การตั้งค่าการนำออกใช้โดยอัตโนมัติในการควบคุมการนำออกใช้ของประเภทผลิตภัณฑ์วิศวกรรม คุณสามารถทำการนำออกใช้นี้เป็นส่วนหนึ่งของลำดับงานได้ เมื่อมีการใช้บล็อค **รวบรวมข้อเสนอการนำออกใช้** จะมีการเติมข้อเสนอการนำออกใช้ด้วยข้อเสนอการนำออกใช้อีกครั้ง (ดูรายการสินค้าก่อนหน้านี้) และผลิตภัณฑ์จะถูกนำออกใช้ไปยังบริษัท ถ้ามีการเลือกกล่องกาเครื่องหมาย **นำออกใช้โดยอัตโนมัติ** ในการควบคุมการนำออกใช้ของประเภทผลิตภัณฑ์วิศวกรรม คุณสามารถเลือก **ดู** เพื่อดูผลลัพธ์ ตามที่อธิบายไว้ในรายการก่อนหน้านี้ นอกจากนี้ ผลิตภัณฑ์นี้จะถูกนำออกใช้ เมื่อมีการใช้บล็อค **ประมวลผลข้อเสนอการนำออกใช้** ถ้าคุณเลือกเฉพาะเพื่อรวบรวมข้อเสนอการนำออกใช้โดยเป็นส่วนหนึ่งของลำดับงาน คุณสามารถเริ่มต้นการนำออกใช้ด้วยตนเองได้โดยการเลือก **ประมวลผล** ตามที่อธิบายไว้ในรายการก่อนหน้านี้

## <a name="follow-up-on-an-engineering-change-request-via-an-engineering-change-order"></a>ติดตามคำขอเปลี่ยนแปลงทางวิศวกรรมผ่านทางใบสั่งเปลี่ยนแปลงทางวิศวกรรม

ทันทีที่คำขอเปลี่ยนแปลงทางวิศวกรรมที่ได้รับอนุมัติ คุณสามารถติดตามได้ผ่านทางใบสั่งเปลี่ยนแปลงทางวิศวกรรม คุณสามารถรวมคำขอเปลี่ยนแปลงทางวิศวกรรมหลายรายการเข้ากับใบสั่งเปลี่ยนแปลงทางวิศวกรรมรายการเดียว ใบสั่งเปลี่ยนแปลงทางวิศวกรรมรายการเดียวอาจรวมถึงผลิตภัณฑ์หลายรายการ (โดยทั่วไป คุณจะใช้วิธีการนี้ เมื่อต้องใช้การเปลี่ยนแปลงเดียวกันกับผลิตภัณฑ์หลายรายการ) อย่างไรก็ตาม คุณไม่สามารถสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรมหลายรายการจากคำขอเปลี่ยนแปลงทางวิศวกรรมรายการเดียว

เมื่อต้องการติดตามคำขอเปลี่ยนแปลงผ่านทางใบสั่งเปลี่ยนแปลง ให้เปิดคำขอเปลี่ยนแปลง และจากนั้น ในบานหน้าต่างการดำเนินการ บนแท็บ **ใบสั่งเปลี่ยนแปลง** ในกลุ่ม **ใบสั่งเปลี่ยนแปลงทางวิศวกรรม** ให้เลือก **คัดลอกลิงค์และผลิตภัณฑ์** จากนั้น คุณสามารถเลือกใบสั่งเปลี่ยนแปลงทางวิศวกรรมที่มีอยู่เพื่อเชื่อมต่อคำขอเปลี่ยนแปลง หรือคุณสามารถสร้างใบสั่งเปลี่ยนแปลงทางวิศวกรรมใหม่สำหรับคำขอที่ระบุ

## <a name="engineering-change-order-report"></a>รายงานใบสั่งเปลี่ยนแปลงทางวิศวกรรม

รายงานใบสั่งเปลี่ยนแปลงทางวิศวกรรมอธิบายถึงการเปลี่ยนแปลงที่เกิดขึ้นในใบสั่งเปลี่ยนแปลงทางวิศวกรรม ซึ่งมีประโยชน์ทั้งในระหว่างและหลังกระบวนการรีวิวและการอนุมัติ

เมื่อต้องการดูรายงานใบสั่งเปลี่ยนแปลงทางวิศวกรรม ให้เปิดใบสั่งเปลี่ยนแปลงที่เกี่ยวข้อง และจากนั้น ในบานหน้าต่างการดำเนินการ บนแท็บ **ใบสั่งเปลี่ยนแปลง** ในกลุ่ม **มุมมอง** ให้เลือก **รายงานใบสั่งเปลี่ยนแปลงทางวิศวกรรม**

## <a name="fields-on-an-engineering-change-order"></a>ฟิลด์ในใบสั่งเปลี่ยนแปลงทางวิศวกรรม

ฟิลด์ส่วนใหญ่ของใบสั่งเปลี่ยนแปลงทางวิศวกรรมจะเหมือนกับฟิลด์สำหรับผลิตภัณฑ์ที่นำออกใช้ รุ่นทางวิศวกรรม เอกสาร BOM (บรรทัด) และกระบวนการผลิต (การดำเนินงาน) อย่างไรก็ตาม ฟิลด์ในตารางต่อไปนี้จะไม่ซ้ำกันสำหรับใบสั่งเปลี่ยนแปลง

| ฟิลด์ | คำอธิบาย |
|---|---|
| เหตุผลในการเปลี่ยนแปลงทางวิศวกรรม | เลือกเหตุผลสำหรับการเปลี่ยนผลิตภัณฑ์ที่ได้รับผลกระทบ |
| คำอธิบายการเปลี่ยนแปลง | ป้อนคำอธิบายเกี่ยวกับการเปลี่ยนแปลง |
| การใช้เครื่องมือพิเศษที่จำเป็น | ระบุว่าต้องใช้เครื่องมือพิเศษเพื่อใช้การเปลี่ยนแปลงหรือไม่ |
| การโอนการครอบครองวัสดุวิศวกรรม | เลือกรหัสการโอนการครอบครองวัสดุสำหรับของเสียใดๆ ที่ผลิตเมื่อมีการใช้การเปลี่ยนแปลง |
| ต้องได้รับการอนุมัติจากลูกค้า | ระบุว่าต้องใช้การอนุมัติของลูกค้า ก่อนที่จะสามารถใช้การเปลี่ยนแปลงหรือไม่ |
| การอนุมัติของลูกค้าที่ได้รับ | ระบุสถานะของการอนุมัติของลูกค้า |
| สุขภาพและความปลอดภัยด้านสภาพแวดล้อม | ระบุว่ากฎสุขภาพและความปลอดภัยด้านสภาพแวดล้อมสามารถใช้ได้กับการเปลี่ยนแปลงหรือไม่ หากใช้ได้ คุณจะสามารถเลือกกฎที่เกี่ยวข้องได้ |

คุณสามารถใช้ปุ่ม **รักษา/สำเนาข้อมูลการเปลี่ยนแปลง** เพื่อคัดลอกข้อมูลการเปลี่ยนแปลงระหว่างผลิตภัณฑ์ที่ได้รับผลกระทบ
