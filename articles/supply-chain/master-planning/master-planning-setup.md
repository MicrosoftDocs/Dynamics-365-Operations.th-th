# <a name="set-up-master-planning"></a>ตั้งค่าการวางแผนหลัก


[!include [banner](../includes/preview-banner.md)]
[!include [banner](../includes/banner.md)]

หัวข้อนี้จะอธิบายถึงกลยุทธ์ที่สำคัญและพารามิเตอร์ต่างๆ ที่ใช้ในการตั้งค่าการวางแผนหลัก ซึ่งรวมถึงภาพรวมของชนิดของแผนที่ใช้โดยการวางแผนหลัก และอธิบายถึงกลยุทธ์แผนที่คุณควรใช้ ทั้งนี้ขึ้นอยู่กับความต้องการทางธุรกิจของคุณ นอกจากนี้ ยังอธิบายถึงพารามิเตอร์หลักที่มีผลกระทบต่อแผน และอธิบายวิธีการที่พารามิเตอร์เหล่านั้นมีผลกระทบต่อแผนการใบสั่งที่แนะนำ

## <a name="types-of-master-plans"></a>ชนิดของแผนหลัก

การวางแผนหลักมีแผนอยู่สองชนิด: แบบคงที่ และแบบไดนามิก แต่ละชนิดถูกออกแบบมาสำหรับการใช้งานที่แตกต่างกัน สำหรับประสิทธิภาพที่ดีที่สุด เราขอแนะนำให้คุณใช้แผนที่เหมาะสมสำหรับสถานการณ์จำลองของคุณ

### <a name="static-plan"></a>แผนแบบคงที่

แผนแบบคงที่ยังคงไม่เปลี่ยนแปลง โดยไม่คำนึงถึงการเปลี่ยนแปลงใดๆ ในการจัดหาวัสดุและความต้องการ จนกว่าจะมีการรันการวางแผนหลักครั้งถัดไป

แผนแบบคงที่จะใช้ในการอนุมัติและยืนยันใบสั่งที่แนะนำ ซึ่งเป็นแผนการดำเนินงานที่บุคลากรต่างๆ ของบริษัท (เช่น ผู้ซื้อ หรือผู้วางแผนการผลิต) สามารถอ้างอิงการตัดสินใจของพวกเขาและใช้ในการดำเนินการงานประจำวันและกิจกรรมของพวกเขาได้

แผนแบบคงที่ยังสนับสนุนการสร้างใหม่ด้วย แผนที่ปรับให้เหมาะสมใหม่ทั้งหมดจะถูกคำนวณทุกครั้งที่มีการรันการวางแผนหลัก และใบสั่งที่มีอยู่ที่ยังไม่ได้รับอนุมัติจะถูกลบออก

### <a name="dynamic-plan"></a>แผนแบบไดนามิก

โดยปกติแผนแบบไดนามิกจะเริ่มต้นเป็นสำเนาของแผนแบบคงที่ แต่สามารถปรับปรุงได้ทุกครั้งที่ข้อมูลหลักเปลี่ยนแปลง ตัวอย่างเช่น ถ้าข้อมูลเปลี่ยนแปลง จะมีการสร้างใบสั่งขายใหม่

แผนแบบไดนามิกใช้สำหรับการวางแผนเฉพาะกิจ ซึ่งอนุญาตให้บริษัทสามารถตรวจสอบเครือข่ายใบสั่งที่มีการเปลี่ยนแปลงและความพร้อมใช้งานของสินค้า โดยไม่รบกวนแผนแบบคงที่ที่ผู้อื่นกำลังใช้สำหรับกระบวนการปฏิบัติงานของพวกเขา โดยเฉพาะอย่างยิ่ง มีการใช้สำหรับปริมาณที่สามารถสัญญาได้ (CTP) แผนแบบไดนามิกเป็นแผนเริ่มต้นเมื่อมีการเปิดความต้องการสุทธิจากหน้าใบสั่ง (เช่น หน้าใบสั่งขาย หน้าใบสั่งซื้อ หรือหน้าใบสั่งผลิต)

แผนแบบไดนามิกใช้การเปลี่ยนแปลงสุทธิ ดังนั้น จึงช่วยรับประกันถึงเวลาในการรันที่เร็วขึ้น

## <a name="strategies-for-using-master-plans"></a>กลยุทธ์สำหรับการใช้แผนหลัก

คุณสามารถใช้หนึ่งแผน หรือสองแผน ในการวางแผนหลัก กลยุทธ์ที่คุณใช้จะขึ้นอยู่กับความต้องการทางธุรกิจของคุณ

### <a name="one-plan-strategy"></a>กลยุทธ์แบบหนึ่งแผน

สำหรับกลยุทธ์แบบหนึ่งแผน คุณใช้แผนหลักเดียวกันสำหรับแผนแบบคงที่และแผนแบบไดนามิก มีการใช้กลยุทธ์นี้สำหรับสถานการณ์จำลองของการผลิตตามสินค้าคงคลัง ที่ซึ่งโดยปกติคุณไม่จำเป็นต้องจำลองแผนที่ตอบสนองใบสั่ง

โดยทั่วไปจะใช้กลยุทธ์แบบหนึ่งแผนในอุตสาหกรรมการขายปลีกและการกระจายสินค้า หรือที่ซึ่งมีการยืนยันวันที่จัดส่งสำหรับการขายตามข้อตกลงระดับการให้บริการ (SLAs) หรือระยะเวลารอคอยสินค้า สำหรับแผน สามารถใช้ตัวควบคุมวันที่จัดส่งโดยไม่มี CTP

ถ้าคุณต้องทำการจำลอง อาจรันแผนอีกครั้งสำหรับสินค้าที่จำเป็นต้องมีการจำลอง โดยทั่วไปจะมีการยืนยันใบสั่งที่วางแผนไว้ที่การจำลองใบสั่งสร้าง

### <a name="two-plan-strategy"></a>กลยุทธ์แบบสองแผน

สำหรับกลยุทธ์แบบสองแผน คุณใช้แผนแบบคงที่และแผนแบบไดนามิกที่แตกต่างกัน โดยทั่วไปจะมีการใช้กลยุทธ์แบบสองแผนสำหรับสถานการณ์จำลองสำหรับการตั้งค่าคอนฟิกตามใบสั่งและการผลิตตามใบสั่ง ที่ซึ่งคุณต้องทำการจำลองใบสั่งขายและคำนวณวันที่จัดส่งที่แน่นอนสำหรับใบสั่งขาย แต่การคำนวณต้องไม่มีผลกระทบต่อการดำเนินงานในแต่ละวัน การจำลองเหล่านี้จะถูกดำเนินการในแผนแบบไดนามิกเสมอ กลยุทธ์แบบสองแผนมีประโยชน์ใน ตัวอย่างเช่น อุตสาหกรรมรถยนต์และอุตสาหกรรมผู้ผลิตอุปกรณ์ดั้งเดิม (OEM)

สำหรับกลยุทธ์แบบสองแผน สามารถใช้ CTP ของตัวควบคุมวันที่จัดส่ง เมื่อมีการใช้ CTP จะทริกเกอร์การรันในแผนแบบไดนามิกโดยอัตโนมัติ

### <a name="setting-up-the-plans"></a>การตั้งค่าแผน

คุณสามารถสร้างแผนบนหน้า **แผนหลัก** (**การวางแผนหลัก \> การตั้งค่า \> แผน \> แผนหลัก**)

คุณสามารถระบุว่าจะใช้แผนใดสำหรับแผนแบบคงที่และแผนแบบไดนามิกโดยการตั้งค่าฟิลด์ **แผนหลักแบบคงที่ปัจจุบัน** และฟิลด์ **แผนหลักแบบไดนามิกปัจจุบัน** บนหน้า **พารามิเตอร์การวางแผนหลัก** ( **การวางแผนหลัก \> การตั้งค่า \> พารามิเตอร์การวางแผนหลัก**) เมื่อต้องการใช้กลยุทธ์แบบหนึ่งแผน ให้เลือกแผนเดียวกันในฟิลด์ **แผนหลักแบบคงที่ปัจจุบัน** และฟิลด์ **แผนหลักแบบไดนามิกปัจจุบัน**

## <a name="types-of-planning-methods"></a>ชนิดของวิธีการวางแผน

สามารถใช้วิธีการคำนวณสามวิธีในการรันการวางแผนหลัก: การสร้างใหม่ และการเปลี่ยนแปลงสุทธิ วิธีแต่ละวิธีสร้างแผนที่แตกต่างกัน เมื่อมีการรัน

คุณระบุวิธีการวางแผนในกล่องโต้ตอบ **การรันการวางแผนหลัก** เพื่อเปิดกล่องโต้ตอบนี้ ไปยัง **การวางแผนหลัก \> การวางแผนหลัก \> การรัน \> การวางแผนหลัก** หรือเลือก **รัน** ในพื้นที่ทำงาน **การวางแผนหลัก**

### <a name="regeneration"></a>การสร้างใหม่

วิธีการวางแผนการสร้างใหม่จะลบแผนการใบสั่งที่มีอยู่ ยกเว้นว่าจะมีการยืนยัน ซึ่งสร้างแผนการใบสั่งใหม่ ตามความต้องการทั้งหมด การสร้างใหม่เป็นวิธีการวางแผนเดียวเท่านั้นที่พร้อมใช้งานสำหรับแผนแบบคงที่

- มีการพิจารณาการเปลี่ยนแปลงในการจัดหาวัสดุ การเปลี่ยนแปลงเหล่านี้รวมถึงการเปลี่ยนแปลงในการคาดการณ์
- วิธีการนี้คำนึงถึงรหัสความครอบคลุมของ **รอบระยะเวลา**
- วิธีการนี้สนับสนุนฟังก์ชันการแทนที่ผลิตภัณฑ์ (PI)

### <a name="net-change"></a>การเปลี่ยนแปลงสุทธิ

วิธีการวางแผนการเปลี่ยนแปลงสุทธิสร้างแผนการใบสั่งเพื่อครอบคลุมความต้องการที่ได้ถูกสร้างหรือเปลี่ยนแปลง ตั้งแต่การรันการวางแผนหลักครั้งล่าสุด การเปลี่ยนแปลงในการจัดหาวัสดุไม่ได้รับการพิจารณา เมื่อมีการรันวิธีการนี้ ระบบไม่พิจารณาการจัดหาวัสดุใหม่ใดๆ และแผนการใบสั่งที่สร้างขึ้นก่อนหน้านี้จะไม่ถูกลบออก ถ้าไม่จำเป็นต้องใช้อีกต่อไป วิธีการวางแผนการเปลี่ยนแปลงสุทธิรันเร็วกว่าวิธีการสร้างใหม่ ซึ่งพร้อมใช้งานเฉพาะสำหรับแผนแบบไดนามิกเท่านั้น

- จะมีการปรับปรุงวันที่และแผนล่วงหน้าของการดำเนินการสำหรับความต้องการทั้งหมด
- วิธีการนี้ไม่คำนึงถึงรหัสความครอบคลุมของ **รอบระยะเวลา**
- วิธีการนี้ไม่ได้เป็นไปตามการคาดการณ์ ถึงแม้ว่าจะมีการเลือกไว้ในแผน
- วิธีการนี้ไม่สนับสนุนฟังก์ชันการแทนที่ผลิตภัณฑ์ (PI)

### <a name="net-change-minimized"></a>การเปลี่ยนแปลงสุทธิที่ปรับให้น้อยที่สุด

วิธีการวางแผนของการเปลี่ยนแปลงสุทธิที่ปรับให้น้อยที่สุด จะสร้างแผนการใบสั่งเพื่อครอบคลุมเฉพาะความต้องการที่ได้ถูกสร้างหรือเปลี่ยนแปลง ตั้งแต่การรันการจัดกำหนดการวางแผนหลักครั้งล่าสุด สำหรับวิธีการนี้ แตกต่างจากวิธีการเปลี่ยนแปลงสุทธิ วันที่ของการดำเนินการและวันที่ในอนาคตมีการปรับปรุงสำหรับความต้องการใหม่หรือที่มีการเปลี่ยนแปลงเท่านั้น วิธีการวางแผนนี้จะพร้อมใช้งานสำหรับแผนแบบไดนามิกเท่านั้น

## <a name="types-of-scheduling-methods"></a>ชนิดของวิธีการจัดกำหนดการ

สำหรับแผนแต่ละแผน บน FastTab **ทั่วไป** ของหน้า **แผนหลัก** (**การวางแผนหลัก \> การตั้งค่า \> แผน \> แผนหลัก**) คุณต้องเลือกวิธีการจัดกำหนดการที่จะใช้สำหรับใบสั่งผลิต คุณสามารถจัดตารางการผลิตในระดับการดำเนินงานและระดับงาน

### <a name="operations-scheduling"></a>การจัดตารางการผลิตระดับการดำเนินงาน

คุณสามารถใช้การจัดตารางการผลิตระดับการดำเนินงานเพื่อให้การประเมินโดยทั่วไปของกระบวนการผลิตตามช่วงเวลา การจัดตารางการผลิตไม่กระจายการดำเนินงานสำหรับกระบวนการผลิตออกเป็นงานต่างๆ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการจัดกำหนดการของการดำเนินการ ให้ดูที่ [การจัดกำหนดการของการดำเนินการ](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/operations-scheduling)

### <a name="job-scheduling"></a>การจัดตารางการผลิตระดับงาน

การจัดตารางการผลิตระดับงานคือ การจัดกำหนดการอย่างละเอียดมากขึ้น ที่ซึ่งมีการแบ่งการดำเนินงานแต่ละรายการออกเป็นภารกิจหรืองานแต่ละอย่าง การจัดกำหนดการงานรวมข้อมูลเกี่ยวกับกำลังการผลิต โดยปกติ จะใช้ในการจัดกำหนดการแต่ละงานในการควบคุมการผลิตสำหรับกรอบเวลาแบบทันทีหรือระยะสั้น สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการจัดตารางการผลิตระดับงาน ให้ดูที่ [การจัดตารางการผลิตระดับงาน](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/production-control/job-scheduling)

## <a name="time-fences-in-days"></a>กรอบเวลาเป็นวัน

สำหรับแผนแต่ละแผน คุณสามารถเลือกระยะเวลาในอนาคตที่ต้องมีการคำนวณความต้องการต่างๆ และการพิจารณาอื่นๆ โดยการวางแผนหลัก รอบระยะเวลาเรียกอีกอย่างว่า *กรอบเวลา* สำหรับประสิทธิภาพที่ดีที่สุดในการวางแผนหลัก เราขอแนะนำให้คุณปรับกรอบเวลาที่หลากหลายให้ตอบสนองความต้องการทางธุรกิจของคุณ สำหรับแผนแต่ละแผน คุณสามารถค้นหากรอบเวลาบน FastTab **กรอบเวลาในหน่วยวัน** ของหน้า **แผนหลัก** (**การวางแผนหลัก \> การตั้งค่า \> แผน \> แผนหลัก**)

> [!NOTE]
> กรอบเวลาระบุระยะเวลาในอนาคตที่มีการคำนวณความต้องการต่างๆ และการพิจารณาอื่นๆ โดยการวางแผนหลัก กรอบเวลาที่เลือกในหน้านี้จะแทนที่กรอบเวลาที่กำหนดไว้ในกลุ่มความครอบคลุม นี่หมายความว่าการตั้งค่าตัวเลือกกรอบเวลาเป็น ใช่ และการกำหนดวัน จะแทนที่กรอบเวลาที่กำหนดไว้ในกลุ่มความครอบคลุม เมื่อตั้งค่าเป็น ไม่ กรอบเวลาจะถูกกำหนดในกลุ่มความครอบคลุม ในขั้นสุดท้าย ถ้าคุณไม่ต้องการหรือไม่จำเป็นต้องใช้ตัวเลือก (ตัวอย่างเช่น คุณไม่ต้องการใช้ข้อความการดำเนินการ) ตั้งค่าเป็น **ใช่** และจากนั้น ตั้งค่ากรอบเวลาเป็น **0** (ศูนย์) วัน

### <a name="coverage"></a>ความครอบคลุม

กรอบเวลาความครอบคลุมแสดงถึงรอบระยะเวลาการจัดกำหนดการ หรือระยะเวลาที่ควรจะรวมความต้องการ กล่าวอีกอย่างหนึ่งคือ จะบ่งชี้ถึงระดับการวางแผนของคุณ

ด้วยการตั้งค่าตัวเลือก **ความครอบคลุม** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลาความครอบคลุมที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันที่การคำนวณการจัดกำหนดการหลักควรจะครอบคลุมความต้องการ กรอบเวลาความครอบคลุมจะมีการคำนวณไปข้างหน้าจากวันที่ปัจจุบัน มีการประมวลผลความต้องการที่เกิดขึ้นก่อนวันที่ปัจจุบันเสมอ

> [!NOTE]
> สำหรับประสิทธิภาพที่ดีที่สุดในการวางแผนหลัก เราขอแนะนำให้คุณปรับกรอบเวลาความครอบคลุมเป็นระดับการวางแผนของคุณ

### <a name="freeze"></a>หยุดดำเนินการ

กรอบเวลาการหยุดดำเนินการแสดงถึงรอบระยะเวลา เมื่อแผนการใบสั่งที่มีอยู่ไม่มีการเปลี่ยนแปลง เมื่อรันกำหนดการหลักใหม่ ไม่มีหยุดดำเนินการแผนการใบสั่ง และจะไม่มีการแนะนำแผนการใบสั่งใหม่

ด้วยการตั้งค่าตัวเลือก **หยุดดำเนินการ** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลาการหยุดดำเนินการที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันซึ่งกิจกรรมการวางแผนควรจะถูกหยุดดำเนินการ จำไว้ว่า ในระหว่างช่วงเวลานี้ ไม่มีการสร้างแผนการใบสั่งใหม่ และไม่สามารถเปลี่ยนแผนการใบสั่งที่มีอยู่ได้

### <a name="firming"></a>การยืนยัน

กรอบเวลาการยืนยันจะบ่งชี้ถึงขอบเขตเวลาที่แผนการใบสั่งจะถูกแปลงเป็นใบสั่งผลิตและใบสั่งซื้อโดยอัตโนมัติ กระบวนการนี้เรียกอีกอย่างหนึ่งว่า *การยืนยันแบบอัตโนมัติของแผนการใบสั่ง*

ด้วยการตั้งค่าตัวเลือก **การยืนยัน** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลาการยืนยันที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันที่ควรมีการยืนยันแผนการใบสั่งซื้อและแผนการใบสั่งผลิตโดยอัตโนมัติ กรอบเวลาการยืนยันจะคำนวณไปข้างหน้าจากวันที่ของการจัดกำหนดการหลัก การยืนยันแบบอัตโนมัติของแผนการใบสั่งซื้อสามารถเกิดขึ้นได้ เฉพาะเมื่อสินค้าเชื่อมโยงกับผู้จัดจำหน่าย

### <a name="forecast-plan"></a>แผนการคาดการณ์

กรอบเวลาแผนการคาดการณ์บ่งชี้ว่า ระยะเวลาในอนาคตที่การวางแผนหลักจะสร้างแผนการใบสั่งสำหรับสินค้าที่มีความต้องการที่คาดการณ์ไว้

ด้วยการตั้งค่า ตัวเลือก **แผนการคาดการณ์** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลาแผนการคาดการณ์ที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันที่การคาดการณ์การขายจากแผนการคาดการณ์ควรถูกรวมอยู่ในการวางแผนหลัก

### <a name="capacity"></a>กำลังการผลิต

กรอบเวลากำลังการผลิตจะบ่งชี้ระยะห่างในอนาคตของระบบซึ่งจะพิจารณาถึงกำลังการผลิตสูงสุดของทรัพยากรของคุณ เมื่อจัดกำหนดการใบสั่ง กล่าวคือ แผนจะจัดกำหนดการใบสั่งผลิตโดยใช้กระบวนการผลิตสำหรับสินค้า และจะพิจารณาถึงทรัพยากรของกระบวนการผลิตและกำลังการผลิตสูงสุดของทรัพยากรแต่ละรายการ

ด้วยการตั้งค่าตัวเลือก **กำลังการผลิต** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลากำลังการผลิตที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันที่ควรมีการวางแผนกำลังการผลิตสำหรับแผนการใบสั่งผลิต การจัดกำหนดการหลักใช้กระบวนการผลิตที่ใช้งานอยู่สำหรับสินค้า และจัดกำหนดการย้อนหลังจากวันที่ของความต้องการ ถ้าวันที่ต้องการสำหรับแผนการใบสั่งผลิตอยู่นอกกรอบเวลากำลังการผลิต จะมีการกำหนดระยะเวลารอคอยสินค้าโดยพิจารณาจากเวลาจัดส่งของสินค้า กรอบเวลากำลังการผลิตจะมีการคำนวณไปข้างหน้าจากวันที่ปัจจุบัน

### <a name="action-message"></a>ข้อความแสดงการดำเนินการ

ข้อความการดำเนินการจะแนะนำการเปลี่ยนแปลงที่ทำกับใบสั่งการจัดหาวัสดุที่มีอยู่ เพื่อช่วยให้แผนการจัดหาวัสดุมีประสิทธิภาพสูงสุด ตัวอย่างเช่น อาจแนะนำให้คุณดำเนินการใบสั่งล่วงหน้าหรือเลื่อนใบสั่ง หรือคุณจะเพิ่มหรือลดปริมาณของใบสั่ง

ด้วยการตั้งค่า ตัวเลือก **ข้อความการดำเนินการ** เป็น **ใช่** คุณสามารถแทนที่กรอบเวลาข้อความการดำเนินการที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันที่การจัดกำหนดการหลักควรจะสร้างข้อความการดำเนินการสำหรับความต้องการ กรอบเวลาข้อความการดำเนินการจะมีการคำนวณไปข้างหน้าจากวันที่ปัจจุบัน

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับข้อความการดำเนินการ ดู [ข้อความการดำเนินการ](https://docs.microsoft.com/dynamics365/unified-operations/supply-chain/master-planning/action-messages)

> [!NOTE]
> การคำนวณของข้อความการดำเนินการจะทำให้เวลาในการรันสำหรับการวางแผนหลักยาวนานขึ้น ถ้าข้อความการดำเนินการไม่ได้ถูกวิเคราะห์และนำไปใช้อย่างสม่ำเสมอ (รายวัน รายสัปดา ห์และอื่นๆ) ให้พิจารณาการปิดการคำนวณในระหว่างการรันการวางแผนหลัก เมื่อต้องการปิดการคำนวณ บนหน้า **แผนหลัก** ให้ตั้งค่ากรอบเวลา **ข้อความการดำเนินการ** เป็น **0** (ศูนย์) สำหรับแผนหลักที่คุณกำลังรัน นอกจากนี้ โปรดตรวจสอบให้แน่ใจว่าการตั้งค่า **ข้อความการดำเนินการ** ถูกปิดสำหรับกลุ่มความครอบคลุมทั้งหมด

### <a name="calculated-delays"></a>การล่าช้าที่คำนวณ

คุณสามารถระบุความล่าช้าที่อาจเกิดขึ้นในอนาคตในแผนการใบสั่งที่มีการตรวจสอบและรายงาน ด้วยวิธีนี้ คุณสามารถจัดกำหนดการวันที่จัดส่งที่เป็นไปได้ (ล่าช้า)

ถ้าไม่สามารถเติมสินค้าตามแผนการใบสั่งได้สำหรับวันที่ที่ร้องขอ จะมีการวางแผนสำหรับวันที่ในการเติมสินค้าที่เร็วที่สุดสำหรับธุรกรรม ตามระยะเวลารอคอยสินค้า และความพร้อมใช้งานของวัสดุและกำลังการผลิต

### <a name="approved-requisitions-time-fence"></a>กรอบเวลาคำขอที่อนุมัติ

คุณสามารถตั้งค่าการวางแผนหลักเพื่อสร้างแผนการใบสั่งสำหรับความต้องการของใบขอซื้อ ตั้งค่าตัวเลือก **รวมใบขอซื้อ** เป็น **ใช่** บน FastTab **ทั่วไป** ของหน้า **แผนหลัก** จากนั้น เมื่อวัตถุประสงค์ของใบขอซื้อที่ได้รับอนุมัติคือการเติมสินค้า การวางแผนหลักจะสร้างแผนการใบสั่งที่สอดคล้องกันโดยอัตโนมัติเพื่อตอบสนองความต้องการ วิธีการเติมสินค้าจะถูกกำหนดโดยนโยบายการจัดหาวัสดุที่ได้มีการตั้งค่าไว้สำหรับสินค้าในองค์กรของคุณ หลังจากที่มีการสร้างและอนุมัติใบขอเติมสินค้า ไม่ต้องมีการดำเนินการของผู้ใช้เพิ่มเติม

ด้วยการตั้งค่า ตัวเลือก **กรอบเวลาของใบขอซื้อ** เป็น **ใช่** บน FastTab **กรอบเวลาเป็นจำนวนวัน** คุณสามารถแทนที่กรอบเวลาของใบขอซื้อที่ได้รับอนุมัติที่กำหนดไว้สำหรับสินค้าในระหว่างการจัดกำหนดการหลัก ในกรณีนี้ ให้ป้อนจำนวนของวันในอดีตที่มีการรวมความต้องการจากใบขอซื้อที่อนุมัติที่มีวัตถุประสงค์ในการเพิ่มเติมสินค้าในการจัดกำหนดการหลัก ตัวอย่างเช่น คุณสามารถระบุว่าเฉพาะความต้องการที่เลยกำหนดที่ยังไม่ได้รับการตอบสนองจากใบขอซื้อที่อนุมัติแล้วซึ่งถูกสร้างขึ้นใน 10 วันที่ผ่านมา ควรได้รับพิจารณาและวางแผน

### <a name="sequencing"></a>การจัดลำดับ

การจัดลำดับช่วยให้แผนการใบสั่งในการจัดเรียงตามแอททริบิวต์การจัดลำดับที่สัมพันธ์กับผลิตภัณฑ์สำเร็จรูป บ่อยครั้งที่ใช้ในการจัดเตรียมใบสั่งผลิตสำหรับบรรจุภัณฑ์ ตัวอย่างเช่น สามารถใช้เพื่อบรรจุกล่องในลำดับที่ระบุ ตามสีและขนาด

ด้วยการตั้งค่า ตัวเลือก **การจัดลำดับ** เป็น **ใช่** คุณสามารถระบุระยะเวลาในอนาคตที่ควรจะจัดลำดับการดำเนินงานหรืองาน พึงระลึกว่ายิ่งกรอบเวลามีขนาดใหญ่ขึ้น ระยะเวลาที่จะใช้สำหรับการวางแผนหลักที่จะรันก็จะยิ่งนานขึ้น

### <a name="calculated-delays"></a>การล่าช้าที่คำนวณ

ตัวเลือกความล่าช้าช่วยรับประกันว่าใบสั่งมีวันที่วางแผนที่เป็นไปได้ ตัวเลือกต่อไปนี้จะพร้อมใช้งานบน FastTab **ความล่าช้าที่คำนวณได้** ของหน้า **แผนหลัก**:

- **ตรวจสอบให้แน่ใจว่าแผนการใบสั่งจะไม่ถูกสร้างขึ้นก่อนวันที่รันการวางแผนหลัก** – ตั้งค่าตัวเลือกนี้เป็น **ใช่** เพื่อช่วยรับประกันว่าไม่สามารถจัดกำหนดการใบสั่งสำหรับวันที่ในอดีตได้
- **เพิ่มความล่าช้าที่คำนวณได้ในวันที่ของความต้องการ** (ภายใต้ **แผนการใบสั่ง**) – ตั้งค่าตัวเลือกนี้เป็น **ใช่** เพื่อเพิ่มความล่าช้าที่คำนวณได้ไปยังความต้องการ
- **เพิ่มความล่าช้าที่คำนวณได้ในวันที่ของความต้องการ** (ภายใต้ **แผนการใบสั่งผลิต**) – ตั้งค่าตัวเลือกนี้เป็น **ใช่** เพื่อเพิ่มความล่าช้าที่คำนวณได้ไปยังความต้องการ
- **เพิ่มความล่าช้าที่คำนวณได้ในวันที่ของความต้องการ** (ภายใต้ **แผนการโอนย้าย**) – ตั้งค่าตัวเลือกนี้เป็น **ใช่** เพื่อเพิ่มความล่าช้าที่คำนวณได้ไปยังความต้องการ
- **เพิ่มความล่าช้าที่คำนวณได้ในวันที่ของความต้องการ** (ภายใต้ **คัมบังที่วางแผนไว้**) – ตั้งค่าตัวเลือกนี้เป็น **ใช่** เพื่อเพิ่มความล่าช้าที่คำนวณได้ไปยังความต้องการ

เมื่อคุณตั้งค่าตัวเลือก **เพิ่มความล่าช้าที่คำนวณได้ในวันที่ของความต้องการ** เป็น **ใช่** เพื่อเพิ่มความล่าช้าของความต้องการ ระบบจะพิจารณากำลังการผลิตของทรัพยากรและสร้างแผนการใบสั่งที่เป็นไปได้ การคำนวณอีกครั้งของวันที่ในแผนการใบสั่งจะเพิ่มเวลาที่ใช้ในการรันสำหรับการวางแผนหลัก ดังนั้น ถ้าคุณไม่จำเป็นต้องใช้ความล่าช้า ให้ตั้งค่าตัวเลือกเป็น **ไม่**

## <a name="positive-and-negative-days"></a>วันที่เป็นค่าบวกและค่าลบ

จำนวนวันที่เป็นค่าบวกและค่าลบจะมีผลกระทบต่อวิธีที่การวางแผนหลักแนะนำแผนการใบสั่งและการดำเนินการ จำนวนวันบวกและค่าลบจะถูกกำหนดในกลุ่มความครอบคลุมสินค้าของสินค้า คุณสามารถกำหนดกลุ่มความครอบคลุมต่างๆ และตั้งค่าพารามิเตอร์ในหน้า **กลุ่มความครอบคลุม** (**การวางแผนหลัก \> การตั้งค่า \> ความครอบคลุม \> กลุ่มความครอบคลุม**)

### <a name="positive-days"></a>จำนวนวันค่าบวก

จำนวนวันที่เป็นค่าบวกบ่งชี้ระยะเวลาในอนาคตที่การวางแผนหลักจะพิจารณาถึงสินค้าคงคลังปัจจุบันหรือการรับสินค้าเพื่อตอบสนองความต้องการในอนาคต ตัวอย่างเช่น ถ้าวันที่เป็นค่าบวกถูกตั้งค่าเป็น **100** จะสามารถใช้สินค้าคงคลังปัจจุบันในการตอบสนองความต้องการในการทำงาน 100 วันถัดไป ถ้ามีใบสั่ง 150 วันนับจากวันที่ปัจจุบัน การวางแผนหลักจะสร้างแผนการใบสั่งเพื่อตอบสนองความต้องการดังกล่าว ถึงแม้ว่าปริมาณคงคลังคงเหลือสำหรับสินค้าจะสามารถตอบสนองใบสั่งได้ สำหรับสินค้าที่มีการเคลื่อนไหวรวดเร็วซึ่งมีระยะเวลารอคอยสินค้าระยะสั้น คุณอาจไม่ต้องการใช้ปริมาณคงคลังคงเหลือสำหรับใบสั่งที่อยู่ห่างไกลในอนาคต ในกรณีของการเคลื่อนไหวรวดเร็วนี้ ปริมาณคงคลังคงเหลือปัจจุบันจะหายไปอย่างรวดเร็ว และอาจมีการวางใบสั่งเพิ่มเติมในอนาคตเพื่อตอบสนองความต้องการในอนาคต ซึ่งจะเป็นไปได้เนื่องจากระยะเวลารอคอยสินค้าระยะสั้นของสินค้า

นอกจากนี้ วันที่เป็นค่าบวกจะมีผลกระทบต่อข้อความการดำเนินการด้วย ตัวอย่างเช่น ระบบอาจแนะนำให้คุณเพิ่มแผนการใบสั่งซื้อ เพื่อให้รวมความต้องการที่อยู่ภายในจำนวนของวันที่เป็นค่าบวกในอนาคต ถ้าวันที่เป็นค่าบวกถูกตั้งค่าเป็น **100** และหากมีความต้องการสินค้าใน 30 วันนับจากวันที่ปัจจุบัน ระบบจะสร้างแผนการใบสั่งเพื่อตอบสนองความต้องการดังกล่าว หากมีความต้องการสำหรับสินค้าเดียวกันใน 90 วันนับจากวันที่ปัจจุบัน ระบบจะแนะนำให้คุณเพิ่มปริมาณของใบสั่งใน 30 วันนับจากวันที่ปัจจุบัน เพื่อให้ใบสั่งครอบคลุมความต้องการใน 90 วันด้วยเช่นกัน อย่างไรก็ตาม หากมีความต้องการสำหรับสินค้าใน150 วันนับจากวันที่ปัจจุบัน ระบบจะไม่แนะนำให้คุณเพิ่มปริมาณของใบสั่งที่ถูกวางแผนไว้เรียบร้อยแล้ว แต่จะมีการสร้างแผนการใบสั่งใหม่แทน

ตามกฎ วันที่เป็นค่าบวกจะถูกตั้งค่าเป็นหมายเลขที่อยู่ระหว่างระยะเวลารอคอยสินค้าที่ยาวนานที่สุดของสินค้าและกรอบเวลาความครอบคลุม เราขอแนะนำให้คุณกำหนดสินค้าที่มีการจัดหาอย่างสม่ำเสมอ หรือมีการผลิตให้กับกลุ่มความครอบคลุม ที่ซึ่งวันที่เป็นค่าบวกเท่ากับระยะเวลารอคอยสินค้าของสินค้า

### <a name="negative-days"></a>จำนวนวันค่าลบ

วันที่เป็นค่าลบจะบ่งชี้ว่าจะอนุญาตให้มีการรับสินค้าล่าช้าได้นานเพียงใด ซึ่งแสดงถึงจำนวนของวันที่คุณเต็มใจรอ ก่อนที่คุณจะสั่งการเติมสินค้าใหม่เมื่อคุณมีสินค้าคงคลังค่าลบ หรือไม่มีสินค้าคงคลังเพียงพอ วันที่ติดลบจะตอบคำถาม เราควรสร้างใบสั่งซื้อใหม่สำหรับสินค้า หรือเราควรใช้การซื้อที่มีอยู่แม้ว่าเราจะทราบว่าสินค้าจะล่าช้า?

ตัวอย่างเช่น คุณมีใบสั่งขายสำหรับสินค้า 15 วันนับจากวันที่ปัจจุบัน นอกจากนี้ คุณยังมีใบสั่งซื้อสำหรับสินค้าเดียวกันด้วย ใบสั่งซื้อนี้จะได้รับใน 20 วันนับจากวันที่ปัจจุบัน คุณต้องการให้ระบบสร้างใบสั่งซื้อสำหรับใบสั่งขายนั้น หรือคุณต้องการใช้ใบสั่งที่มีอยู่แม้ว่าคุณจะไม่สามารถตอบสนองใบสั่งขายได้ตรงเวลา? ถ้ามีการตั้งค่าวันที่ติดลบให้น้อยกว่า **5** วัน เพื่อบ่งชี้ว่าสินค้าอาจล่าช้าได้สูงสุดห้าวัน ระบบจะสร้างแผนการใบสั่งซื้อใหม่เพื่อตอบสนองใบสั่งขาย ถ้ามีการตั้งค่าวันที่ติดลบให้มากกว่า **5** วัน ระบบจะใช้ใบสั่งที่มีอยู่สำหรับสินค้า

นอกจากนี้ วันที่เป็นค่าลบจะมีผลกระทบต่อประสิทธิภาพการทำงานของการวางแผนหลักด้วย ถ้ามีการตั้งค่าวันที่ติดลบเป็นจำนวนสูง จะมีการสร้างข้อความการดำเนินการจำนวนมาก

เราขอแนะนำให้คุณกำหนดวันที่ติดลบเป็นจำนวนที่น้อยกว่าระยะเวลารอคอยสินค้า

### <a name="dynamic-negative-days"></a>วันที่ติดลบแบบไดนามิก

วันที่ติดลบแบบไดนามิกจะพิจารณาระยะเวลารอคอยสินค้าและวันที่ติดลบที่คุณระบุ ระบบจะสร้างแผนการใบสั่งซื้อใหม่ ตามกรอบเวลาของวันที่ติดลบซึ่งคำนวณโดยใช้สูตรต่อไปนี้:

ระยะเวลารอคอยสินค้า + วันที่เป็นค่าลบ + วันที่ปัจจุบัน – วันที่ที่ต้องการ

ระบบจะใช้เฉพาะใบสั่งการจัดหาวัสดุที่วางแผนไว้ที่อยู่ภายในกรอบเวลานี้เท่านั้น และจะสร้างแผนการใบสั่งใหม่ภายนอก ข้อดีของจำนวนวันที่ติดลบแบบไดนามิกคือ จะรวมระยะเวลารอคอยสินค้าของผลิตภัณฑ์แต่ละรายการ เพื่อนำใบสั่งที่มีอยู่มาใช้ใหม่และหลีกเลี่ยงการสร้างแผนการใบสั่งใหม่ซึ่งจะสิ้นสุดด้วยวันที่ในภายหลัง เนื่องจากความล่าช้าที่เกิดจากระยะเวลารอคอยสินค้า
