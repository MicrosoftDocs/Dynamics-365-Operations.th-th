# <a name="scale-units-in-a-distributed-hybrid-topology"></a>Scale unit ในโทโพโลยีแบบไฮบริดแบบกระจาย

[!include [banner](../includes/banner.md)]

> [!IMPORTANT]
> ความสามารถด้าน scale unit ของ Microsoft Dynamics 365 Supply Chain Management มีไว้ให้คุณภายใต้เงื่อนไขที่ควบคุมการใช้งานบริการ สำหรับข้อมูลเพิ่มเติม ดูที่ [ข้อมูลทางกฎหมาย Microsoft Dynamics](https://go.microsoft.com/fwlink/?LinkID=290927)
>
> เมื่อคุณเปิดใช้งาน Cloud และ edge scale unit คุณจะได้รับการร้องขอให้ยืนยันว่าคุณเข้าใจว่าข้อมูลบางอย่างที่เกี่ยวข้องกับการตั้งค่าคอนฟิกและการประมวลผลของ cloud และ edge scale unit อาจจัดเก็บอยู่ในศูนย์ข้อมูลที่อยู่ในสหรัฐอเมริกา หากต้องการเรียนรู้เพิ่มเติมเกี่ยวกับการประมวลผลข้อมูลของหน่วย Cloud และ edge scale units ดูที่ส่วน [การประมวลผลข้อมูลในระหว่างการจัดการ scale units](#data-processing-management) ในภายหลังในหัวข้อนี้

## <a name="core-value-proposition-for-a-distributed-hybrid-topology"></a>ข้อเสนอค่าหลักเกี่ยวกับโทโพโลยีแบบไฮบริดแบบกระจาย

บริษัทที่ทำงานกับการผลิตและการกระจายสินค้าต้องสามารถเรียกใช้กระบวนการทางธุรกิจที่สำคัญได้ 24/7 โดยไม่มีการขัดจังหวะและที่ Scale โทโพโลยีแบบไฮบริดแบบกระจายเปิดใช้งานให้บริษัทเรียกใช้ภารกิจหลักของการประมวลผลการผลิตและคลังสินค้าโดยไม่มีการขัดจังหวะ แม้กระทั่งเมื่อประสบกับปัญหาการเชื่อมต่อเครือข่ายหรือเวลาแฝงเป็นครั้งคราว

โทโพโลยีแบบไฮบริดแบบกระจายจะแนะนำแนวคิดของ *scale units* ซึ่งเปิดใช้งานการกระจายปริมาณงานของการปฏิบัติการการควบคุมผลิตและคลังสินค้าระหว่างสภาพแวดล้อมต่างๆ ฟังก์ชันนี้จะช่วยปรับปรุงประสิทธิภาพการทำงาน ป้องกันการขัดจังหวะการบริการ และเพิ่มความพร้อมในการทำงาน Scale units มีให้ผ่าน Add-in ต่อไปนี้เพื่อการสั่งซื้อโดยบอกรับเป็นสมาชิก Supply Chain Management:

- Add-in ของ Cloud Scale Unit สำหรับ Dynamics 365 Supply Chain Management
- Add-in ของ Edge Scale Unit สำหรับ Dynamics 365 Supply Chain Management

ความสามารถปริมาณงานจะถูกปล่อยออกใช้อย่างต่อเนื่องโดยผ่านการปรับปรุงส่วนเพิ่ม

## <a name="scale-units-and-dedicated-workloads"></a>Scale unit และปริมาณงานที่จัดสรรไว้

Scale unit ขยายสภาพแวดล้อมฮับ Supply Chain Management ส่วนกลางของคุณ โดยการเพิ่มความสามารถการประมวลผลที่เฉพาะเจาะจง Scale unit สามารถเรียกใช้ได้ใน cloud หรือคุณสามารถเรียกใช้บน [edge](cloud-edge-edge-scale-units-lbd.md) แบบในสถานที่ในอาคารสถานที่ของคุณ

:::image type="content" source="./media/cloud_edge-HeroDiagram.png" alt-text="Dynamics 365 กับ scale unit":::

Scale units มีความน่าเชื่อถือ และสเกลของปริมาณงานที่มอบหมาย คุณสามารถยกเลิกการเชื่อมต่อ Edge scale units ชั่วคราวจากสภาพแวดล้อมฮับ Cloud และผู้ปฏิบัติงานยังคงใช้งานปริมาณงานที่ได้รับการมอบหมายบน edge

*ปริมาณงาน* เป็นชุดที่กำหนดของฟังก์ชันธุรกิจซึ่งสามารถแยกออกและมอบหมายให้กับ scale unit ถึงแม้ว่าปริมาณงานของการจัดการคลังสินค้าจะถูกปล่อยออกใช้แล้ว แต่ปริมาณงานการปฏิบัติการผลิตจะยังอยู่ในการแสดงตัวอย่าง

คุณสามารถตั้งค่าคอนฟิกสภาพแวดล้อมฮับของคุณและปริมาณงานที่เลือกบน cloud scale unit โดยใช้ [พอร์ทัล Scale Unit Manager](https://sum.dynamics.com) คุณยังสามารถกําหนดปริมาณงานหลายปริมาณต่อ scale unit ได้ หากต้องการทราบข้อมูลเพิ่มเติมเกี่ยวกับข้อจํากัดและข้อจํากัดของ cloud scale units ในผลิตภัณฑ์ปัจจุบัน โปรดดูที่ส่วน [ข้อจํากัดและข้อจํากัดของ cloud scale units](#cloud-scale-unit-prerequisites) ในหัวข้อนี้ในภายหลัง

### <a name="dedicated-warehouse-management-workload-capabilities-in-a-scale-unit"></a>ความสามารถในการจัดการของปริมาณงานการผลิตเฉพาะใน scale unit

ปริมาณงานของการจัดการคลังสินค้าช่วยให้การดําเนินงานของคลังสินค้าของคุณสามารถปรับขนาดและทำงานในสภาพแวดล้อมที่ยืดหยุ่นโดยใช้กรอบเวลาการบํารุงรักษาที่แยกต่างหาก ปริมาณงานของการจัดการคลังสินค้ารองรับกระบวนการจัดการคลังสินค้าที่เป็นฮับองค์กรส่วนใหญ่ สำหรับข้อมูลเพิ่มเติม ดูที่ [ปริมาณงานในการจัดการคลังสินค้าสำหรับสเกลยูนิตในระบบคลาวด์และ Edge](cloud-edge-workload-warehousing.md)

### <a name="dedicated-manufacturing-execution-workload-capabilities-in-a-scale-unit"></a>ความสามารถในการดำเนินงานของปริมาณงานการผลิตเฉพาะใน scale unit

ปริมาณงานการผลิตมีความสามารถดังต่อไปนี้

- ผู้ปฏิบัติงานเครื่องจักรและหัวหน้างานฝ่ายผลิตสามารถเข้าถึงแผนการผลิตที่มีการดำเนินงาน
- ผู้ปฏิบัติงานในเครื่องสามารถรักษาแผนให้ทันสมัยอยู่เสมอโดยการเรียกใช้งานการผลิตที่แยกต่างหากและมีการประมวลผล
- หัวหน้างานฝ่ายผลิตสามารถปรับปรุงแผนการดำเนินงานได้
- ผู้ปฏิบัติงานสามารถเข้าถึงเวลาและการเข้างานสำหรับการตอกบัตรเข้าและการตอกบัตรออกบน edge เพื่อให้มั่นใจว่าการคำนวณการจ่ายค่าจ้างของผู้ปฏิบัติงานถูกต้อง

สำหรับข้อมูลเพิ่มเติม ดูที่ [ปริมาณงานในการจัดการการผลิตสำหรับสเกลยูนิตในระบบคลาวด์และ Edge](cloud-edge-workload-manufacturing.md)

## <a name="considerations-before-you-enable-the-distributed-hybrid-topology-for-supply-chain-management"></a>ข้อควรพิจารณาก่อนที่คุณจะเปิดใช้งานโทโพโลยีแบบไฮบริดแบบกระจายสำหรับ Supply Chain Management

การเปิดใช้งานโทโพโลยีแบบไฮบริดแบบกระจาย คุณจึงเปลี่ยนสภาพแวดล้อมระบบคลาวด์ของ Supply Chain Management เพื่อให้บริษัทนี้เสมือนเป็นฮับ คุณยังสามารถเชื่อมโยงสภาพแวดล้อมเพิ่มเติมที่มีการตั้งค่าคอนฟิกเป็น scale units ใน Cloud หรือ Edge

### <a name="prerequisites-and-limitations-for-cloud-scale-units"></a><a name="cloud-scale-unit-prerequisites"></a>ข้อจํากัดและข้อจํากัดของ cloud scale units

ในผลิตภัณฑ์ปัจจุบันของ scale units ความสามารถบางอย่างไม่พร้อมใช้งาน แต่อาจเพิ่มในการนำออกใช้ส่วนเพิ่มในช่วงเวลา

#### <a name="you-must-be-a-licensed-customer-of-supply-chain-management"></a>คุณต้องเป็นลูกค้าที่ได้รับลิขสิทธิ์ของ Supply Chain Management

หากต้องการเตรียมพร้อมไปยังโทโพโลยีแบบกระจาย คุณต้องมีลิขสิทธิ์ Supply Chain Management สภาพแวดล้อมระบบคลาวด์ที่มีอยู่ของคุณจะกลายเป็นฮับในโทโพโลยีแบบไฮบริดบัญชีของคุณ คุณสามารถประกาศทั้งสภาพแวดล้อมภายในและสภาพแวดล้อมการผลิตเป็นสภาพแวดล้อมฮับ และคุณสามารถเพิ่ม scale units ตาม Add-in ที่คุณได้รับ

#### <a name="your-existing-project-must-be-administered-via-the-global-commercial-version-of-lcs"></a>โครงการที่มีอยู่ของคุณต้องจัดการผ่านทาง LCS รุ่นทางการค้าสากล

โครงการ Microsoft Dynamics Lifecyle Services (LCS) ที่มีอยู่ของคุณต้องเป็นไปตามข้อความต้องการของรุ่นต่อไปนี้:

- โครงการที่มีอยู่ต้องจัดการผ่านทาง LCS รุ่นทางการค้าสากลที่ [lcs.dynamics.com](https://lcs.dynamics.com)
- LCS รุ่นเฉพาะที่ (เช่น [eu.lcs.dynamics.com](https://eu.lcs.dynamics.com) และ [fr.lcs.dynamics.com](https://fr.lcs.dynamics.com)) ไม่ได้รับการรองรับ
- LCS รุ่นระบบคลาวน์ของรัฐบาลไม่ได้รับการรองรับ
- LCS รุ่น Mooncake ไม่ได้รับการรองรับ

#### <a name="your-current-production-environment-must-be-of-the-self-service-type-in-lcs"></a>สภาพแวดล้อมการผลิตปัจจุบันต้องเป็นชนิดบริการตนเองใน LCS

สภาพแวดล้อมการผลิตปัจจุบันต้องแท็กเป็นชนิด **บริการตนเอง** ใน LCS ชนิดนี้บ่งชี้ว่าผู้เช่าของโครงการ LCS ของคุณได้รับการแปลงแล้วเพื่อให้สนับสนุนแบบจำลองการโฮสต์ Azure Service Fabric

> [!IMPORTANT]
> ชนิดสภาพแวดล้อมที่เรียกใช้เป็นโครงสร้างพื้นฐานเป็นบริการ (IaaS) ไม่สนับสนุน โดยทั่วไป สภาพแวดล้อมเหล่านี้จะมีการแท็กด้วยชนิด **จัดการโดย Microsoft** ใน LCS ถ้าคุณมีสภาพแวดล้อมชนิดนี้ ทำงานกับผู้ติดต่อ Microsoft ของคุณเพื่อให้เข้าใจเส้นเวลาการย้ายของคุณไปยังชนิด **บริการตนเอง**

Microsoft อยู่ในกระบวนการเปลี่ยนสภาพแวดล้อมทั้งหมดของระบบคลาวน์ของ Supply Chain Management จากแบบจำลอง IaaS เป็นโทโพโลยีที่โฮสต์ใน Service Fabric การย้ายนี้จะปรับสเกลได้ดีขึ้นและช่วยให้การจัดการงานบริการง่ายขึ้น การดําเนินการปรับใช้และการบํารุงรักษาจึงรวดเร็วขึ้น เช่นเดียวกัน ส่วนประกอบของบริการจะถูกย้ายไปยังแนวคิดของไมโครเซอร์วิส และการบริการที่โฮสต์แบบจำลองจะ [เปลี่ยน](/virtualization/windowscontainers/about/containers-vs-vm) จากแบบจำลองเครื่องเสมือน (VM) เป็นสถาปัตยกรรมที่บรรจุลงตู้บรรจุสินค้าน้ำหนักเบา

โครงสร้างบริการที่บรรจุลงตู้บรรจุสินค้าโดยยึดตาม Service Fabric เดียวกันจะจ่ายพลังงานทั้งอินสแตนซ์ระบบคลาวน์และ edge ของบริการ ไม่ว่าอินสแตนซ์จะเป็นฮับในระบบคลาวน์ หรือ scale unit ในระบบคลาวน์หรือ edge

ก่อนที่คุณจะสามารถเริ่มติดตั้งโทโพโลยีแบบไฮบริดที่รองรับ scale units ได้ คุณต้องเปลี่ยนผู้เช่าโครงการของคุณไปยังแบบจำลองที่โฮสต์โดย Service Fabric นอกจากนี้ สภาพแวดล้อมใด ๆ ที่จะใช้เป็นฮับต้องถูกแปลง

> [!TIP]
> หากต้องการสอบถามเกี่ยวกับสถานะของผู้เช่าโครงการ LCS ของคุณ ให้ค้นหาชนิดของสภาพแวดล้อมของคุณใน [LCS](https://lcs.dynamics.com/) หรือติดต่อคู่ค้าหรือผู้ติดต่อของ Microsoft

#### <a name="local-business-data-on-premises-environments-arent-supported-as-hubs-for-scale-units"></a>สภาพแวดล้อมของข้อมูลธุรกิจเฉพาะที่ (ในองค์กร) ไม่ได้รับการสนับสนุนในฐานะฮับของ scale units

สภาพแวดล้อมในองค์กรไม่สามารถหน้าที่เป็นฮับของ scale units ได้ สภาพแวดล้อมฮับต้องเป็นโฮสต์ระบบคลาวน์เสมอ

#### <a name="scale-unit-management-capabilities-are-limited"></a>ความสามารถในการจัดการ scale units ถูกจํากัด

ความสามารถด้านการจัดการที่สามารถช่วยในการย้ายปริมาณงานจะถูกจํากัด การดําเนินงานการจัดการบางอย่างไม่ได้รับการสนับสนุนในลักษณะบริการตนเอง และคุณอาจต้องร้องขอการสนับสนุนผ่านทางคู่ค้าหรือผู้ติดต่อ Microsoft ของคุณ ตัวอย่างประกอบด้วยความเคลื่อนไหวของปริมาณงานบางอย่างระหว่าง scale units และการเคลื่อนย้ายเฉพาะกิจชั่วคราวในสถานการณ์ภัยพิบัติ

#### <a name="metrics-and-measurements-arent-yet-available"></a>เมตริกและการวัดไม่พร้อมใช้งาน

เมตริกและการวัดที่อาจช่วยคุณเลือกแอปพลิเคชันที่ดีที่สุดสำหรับ scale units ของคุณยังไม่ได้พร้อมใช้งาน ร่วมมือกับผู้ติดต่อหรือคู่ค้าของคุณจาก Microsoft เพื่อเลือกแอปพลิเคชันที่มีประโยชน์มากที่สุด

### <a name="data-processing-during-management-of-scale-units"></a><a name="data-processing-management"></a>การประมวลผลข้อมูลระหว่างการจัดการ scale units

เมื่อคุณเปิดใช้งานสภาพแวดล้อม Dynamics 365 ของคุณเพื่อสนับสนุนโทโพโลยีแบบกระจายไปยัง cloud scale units และ edge บริการการจัดการบางอย่างจะโฮสต์เฉพาะในสหรัฐอเมริกาเท่านั้น เช่นเดียวกับ LCS ลักษณะการทำงานนี้มีผลกระทบต่อการโอนย้ายและการจัดเก็บข้อมูลการจัดการและการตั้งค่าคอนฟิกบางอย่างที่ใช้โดย [พอร์ทัล Scale Unit Manager](https://sum.dynamics.com) ยกตัวอย่างเช่น

- ชื่อผู้เช่าและรหัสของคุณ
- รหัสโครงการ LCS ของคุณ
- ที่อยู่อีเมลของผู้ดูแลระบบและเจ้าของโครงการที่ใช้เพื่อลงชื่อเข้าใช้
- รหัสสภาพแวดล้อมสำหรับฮับและ scale unit
- การตั้งค่าคอนฟิกปริมาณงาน รวมถึงชื่อและที่อยู่ตามจริงของนิติบุคคลและสิ่งอำนวยความสะดวก เพื่อให้โทโพโลยีของคุณสามารถแสดงบนแผนผังทางภูมิศาสตร์ได้
- เมตริกที่รวบรวม (เช่น เวลาแฝงและปริมาณที่สามารถประมวลผลได้) ที่จะแสดงบนหน้าการวิเคราะห์แผนที่เพื่อช่วยคุณเลือกการใช้ scale units ที่มีประโยชน์มากที่สุด

ข้อมูลที่ถูกโอนย้ายไปยังและจัดเก็บในศูนย์ข้อมูลของสหรัฐจะถูกลบออกตามนโยบายการเก็บรักษาข้อมูล Microsoft ความเป็นส่วนตัวของคุณมีความสำคัญต่อ Microsoft เรียนรู้เพิ่มเติม อ่าน [คำชี้แจงสิทธิส่วนบุคคล](https://go.microsoft.com/fwlink/?LinkId=521839) ของเรา

## <a name="onboard-to-the-distributed-hybrid-topology-for-supply-chain-management"></a>เตรียมพร้อมไปยังโทโพโลยีแบบไฮบริดแบบกระจายสำหรับ Supply Chain Management

### <a name="try-out-the-distributed-hybrid-topology"></a>ลองใช้โทโพโลยีแบบไฮบริดแบบกระจาย

กระบวนการเริ่มต้นการกระจายโทโพโลยีแบบไฮบริดแบบกระจายจะมีสองขั้น ในระหว่างขั้นแรก คุณควร [ลองใช้](cloud-edge-try-out.md) โซลูชันและตรวจสอบความถูกต้องในการกำหนดเองของคุณเพื่อให้แน่ใจว่ามีงานในโทโพโลยีที่กระจายซึ่งมีสเกลยูนิต (คุณสามารถใช้สภาพแวดล้อมการพัฒนาที่มีอยู่เพื่อการตรวจสอบความถูกต้อง) จากนั้นคุณสามารถไปยังลำดับขั้นที่สองซึ่งคุณได้รับสภาพแวดล้อมการทำงานจริง

## <a name="select-your-lcs-project-tenant-and-the-detailed-onboarding-process"></a>เลือกผู้เช่าโครงการ LCS ของคุณ และกระบวนการเตรียมความพร้อมโดยละเอียด

Scale unit จะเสนอในหน่วยการเก็บสต็อกหลายหน่วย (SKU) และตัวเลือกการกําหนดราคา ดังนั้น คุณจึงสามารถเลือกตัวเลือกที่ตรงกับปริมาณธุรกรรมรายเดือนที่วางแผนไว้และความต้องการประสิทธิภาพที่คุณต้องการได้มากที่สุด

> [!TIP]
> หากต้องการระบุการปรับขนาดที่ตรงกับความต้องการของคุณมากที่สุด ให้ร่วมมือกับคู่ค้าการใช้งานและ Microsoft เพื่อความเข้าใจขนาดของธุรกรรมรายเดือนที่คุณต้องการ

SKU ระดับรายการเรียกว่า *พื้นฐาน* และ SKU ที่มีประสิทธิภาพมากกว่าจะเรียกว่า *มาตรฐาน* แต่ละ SKU จะถูกโหลดล่วงหน้าด้วยธุรกรรมรายเดือนตามจํานวนที่ระบุ อย่างไรก็ตาม คุณสามารถเพิ่มงบประมาณธุรกรรมรายเดือนโดยเพิ่ม Add-in ของค่าเกินให้กับแต่ละ SKU ได้

:::image type="content" source="media/SKUs-highlevel.png" alt-text="Add-in ของ Cloud Scale Unit":::

> [!NOTE]
> ไม่สามารถเชื่อมต่อ Add in สเกลยูนิตกับจํานวนผู้ใช้ที่จํากัด การดำเนินการนี้ใช้ได้กับผู้ใช้ในการสมัครใช้งานที่มีอยู่ (หากผู้ดูแลระบบของคุณได้กำหนดบทบาทผู้ใช้ที่ต้องการให้กับผู้ใช้เหล่านั้น)

การซื้อ Add-in ของ scale unit แต่ละหน่วยช่วยให้คุณมีปริมาตรธุรกรรมเป็นรายเดือน แต่ยังให้สิทธิ์คุณกับจํานวนช่องสภาพแวดล้อมเฉพาะใน LCS ด้วย สำหรับแต่ละ Add-in ของ Cloud Scale Unit คุณมีสิทธิ์ได้รับช่องการผลิตใหม่หนึ่งช่องและช่อง sandbox ใหม่หนึ่งช่อง ในระหว่างกระบวนการเตรียมความพร้อม โครงการ LCS ใหม่จะถูกเพิ่มเข้าไปในช่องเหล่านี้ สิทธิการใช้งานของช่องจะถูกผูกไว้เพื่อให้ช่องต้องใช้เป็น scale units ที่มีฮับระบบคลาวน์

Add-in ที่เกินไม่ได้ให้สิทธิ์คุณกับช่องสภาพแวดล้อมใหม่

หากคุณต้องการได้รับสภาพแวดล้อม sandbox ได้มากขึ้น คุณสามารถซื้อช่อง sandbox ทั่วไปเพิ่มเติมได้ จากนั้น Microsoft จะสามารถช่วยให้คุณสามารถเปิดใช้งานช่องเหล่านั้นเป็น sandbox scale units ของโทโพโลยีแบบไฮบริดได้


หลังจากที่คุณได้วางแผนวิธีการที่คุณจะเตรียมความพร้อมไปโทโพโลยีแบบไฮบริดแบบกระจายสำหรับ Supply Chain Management เสร็จสิ้นแล้ว คุณจะต้องใช้ [พอร์ทัล Scale Unit Manager](https://aka.ms/SCMSUM) เพื่อเริ่มกระบวนการเตรียมความพร้อมใหม่ ในพอร์ทัล เลือกแท็บ **ผู้เช่า Dynamics 365** แท็บนี้แสดงรายชื่อผู้เช่าซึ่งบัญชีของคุณเป็นส่วนหนึ่ง และคุณเป็นเจ้าของหรือผู้ดูแลระบบสภาพแวดล้อมสำหรับโครงการ LCS

ถ้าผู้เช่าที่คุณกำลังค้นหาไม่ได้อยู่ในรายการนี้ ให้ไปที่ [LCS](https://lcs.dynamics.com/v2) และตรวจสอบให้แน่ใจว่าคุณเป็นผู้ดูแลระบบสภาพแวดล้อมหรือเจ้าของโครงการของโครงการ LCS สำหรับผู้เช่ารายนั้น เฉพาะบัญชี Azure Active Directory (Azure AD) จากผู้เช่าที่เลือกเท่านั้นที่จะได้รับอนุญาตให้ทำการลงทะเบียนได้

> [!NOTE]
> หลังจากที่คุณใช้การเปลี่ยนแปลงกับ LCS อาจใช้เวลานานถึง 30 นาที สำหรับรายชื่อผู้เช่าเพื่อให้สะท้อนถึงการเปลี่ยนแปลง

สำหรับผู้เช่าแต่ละราย รายการจะแสดงสถานะการเตรียมความพร้อม

:::image type="content" source="media/cloud_edge-EnableHybrid1.png" alt-text="รายชื่อผู้เช่าบนแท็บ Dynamics 365 Tenants":::

เลือก **คลิกที่นี่เพื่อเริ่มต้นใช้งาน** เพื่อร้องขอการเตรียมความพร้อมผู้เช่า LCS คุณต้องยอมรับเงื่อนไข นอกจากนี้คุณต้องจัดหาที่อยู่อีเมลสำหรับธุรกิจโดย Microsoft สามารถส่งการติดต่อสื่อสารที่เกี่ยวข้องกับกระบวนการเตรียมความพร้อม

:::image type="content" source="media/cloud_edge-EnableHybrid2.png" alt-text="การส่งการลงทะเบียนสำหรับผู้เช่า":::

Microsoft จะตรวจสอบความถูกต้องของคำขอของคุณและแจ้งให้คุณทราบขั้นตอนถัดไป โดยการส่งอีเมลไปยังที่อยู่ที่คุณป้อนไว้ในแบบฟอร์มการลงทะเบียน Microsoft จะร่วมกับคุณเพื่อให้สามารถใช้ scale units ในโทโพโลยีแบบไฮบริดสำหรับสถานการณ์จำลองทางธุรกิจของคุณ

หลังจากการเตรียมความพร้อมเสร็จสมบูรณ์แล้ว คุณสามารถใช้พอร์ตเพื่อตั้งค่าคอนฟิก scale units และปริมาณงานได้

### <a name="manage-scale-units-and-workloads-by-using-the-scale-unit-manager-portal"></a><a name="scale-unit-manager-portal"></a>จัดการหน่วยปรับขนาดและปริมาณงาน โดยใช้พอร์ทัล Scale Unit Manager

ไปที่ [พอร์ทัล Scale Unit Manager](https://aka.ms/SCMSUM) แล้วล็อกอินโดยใช้บัญชีผู้เช่าของคุณ บนหน้า **ตั้งค่าคอนฟิก scale unit** คุณสามารถเพิ่มสภาพแวดล้อมฮับถ้ายังไม่ได้แสดงรายการไว้ คุณสามารถเลือกฮับที่คุณต้องการตั้งค่าคอนฟิกกับ scale unit และปริมาณงาน

:::image type="content" source="media/cloud_edge-Manage.png" alt-text="พอร์ทัล Scale Unit Manager ตั้งค่าคอนฟิกหน้าหน่วยปรับขนาด":::

เมื่อต้องการเพิ่ม scale unit หนึ่งหน่วยขึ้นไปที่มีอยู่ในการสมัครใช้งานของคุณ ให้เลือก **เพิ่ม scale unit**

บนแท็บ **ปริมาณงานที่กำหนด** ให้ใช้ปุ่ม **สร้างปริมาณงาน** เพื่อเพิ่มปริมาณงานการจัดการคลังสินค้าไปหนึ่งใน scale unit ของคุณ สำหรับแต่ละปริมาณงาน คุณต้องระบุบริบทของกระบวนการที่จะเป็นของปริมาณงาน สำหรับปริมาณงานการจัดการคลังสินค้า บริบทคือคลังสินค้าเฉพาะ ในไซต์ที่ระบุ และนิติบุคคล

:::image type="content" source="media/cloud_edge-DefineWorkload.png" alt-text="กําหนดกล่องโต้ตอบปริมาณงาน":::

#### <a name="manage-workloads"></a><a name="manage-workloads"></a>จัดการปริมาณงาน

เมื่อเปิดใช้งานปริมาณงานอย่างน้อยหนึ่งรายการ ให้ใช้ตัวเลือก **จัดการปริมาณงาน** เพื่อเริ่มต้นและจัดการกระบวนการ เช่น ที่แสดงในตารางต่อไปนี้

| การประมวลผล | คำอธิบาย |
|---|---|
| หยุดการสื่อสารหน่วยปรับขนาด | หยุดข้อความไปป์ไลน์ระหว่างฮับและหน่วยปรับขนาด กระบวนการนี้จะหยุดการสื่อสารและล้างไปป์ไลน์ข้อมูลระหว่างฮับและหน่วยปรับขนาด คุณต้องเรียกใช้กระบวนการนี้ก่อนที่คุณจะเรียกใช้การให้บริการ Supply Chain Management บนฮับหรือหน่วยปรับขนาด แต่คุณอาจใช้สิ่งนี้ในสถานการณ์อื่นได้ด้วย |
| เริ่มต้นการสื่อสารหน่วยปรับขนาดอีกครั้ง | เริ่มต้นข้อความไปป์ไลน์ระหว่างฮับและหน่วยปรับขนาดอีกครั้ง คุณอาจต้องใช้กระบวนการนี้ ตัวอย่างเช่น หลังจากที่คุณเรียกใช้การให้บริการ Supply Chain Management บนฮับหรือหน่วยปรับขนาด |
| อัพเกรดปริมาณงาน | ซิงค์ฟังก์ชันใหม่ระหว่างปริมาณงานของฮับและหน่วยปรับขนาด คุณอาจต้องใช้กระบวนการนี้ ตัวอย่างเช่น เมื่อการให้บริการเป็นเหตุให้มีการเปลี่ยนแปลงการสอบถามเกี่ยวกับการแลกเปลี่ยนข้อมูล และ/หรือได้เพิ่มตารางหรือฟิลด์ใหม่ในปริมาณงาน |
| การโอนย้ายปริมาณงานไปยังหน่วยปรับขนาด | การจัดตารางปริมาณงานที่เรียกใช้อยู่ในฮับในปัจจุบันเพื่อย้ายไปยังหน่วยปรับขนาด เมื่อกระบวนการนี้ดำเนินงาน จะเกิดการซิงโครไนส์ข้อมูล และทั้งฮับและหน่วยปรับขนาดจะถูกตั้งค่าเพื่อเปลี่ยนความเป็นเจ้าของปริมาณงาน |
| การโอนย้ายหน่วยปรับขนาดไปสู่ฮับ | การจัดตารางปริมาณงานที่เรียกใช้อยู่ในหน่วยปรับขนาดในปัจจุบันเพื่อย้ายไปยังฮับ เมื่อกระบวนการนี้ดำเนินงาน จะเกิดการซิงโครไนส์ข้อมูล และทั้งฮับและหน่วยปรับขนาดจะถูกตั้งค่าเพื่อเปลี่ยนความเป็นเจ้าของปริมาณงาน
| การเปลี่ยนไปสู่ฮับในกรณีฉุกเฉิน | <p>การโอนย้ายปริมาณงานที่มีอยู่ไปยังฮับทันที *กระบวนการนี้จะเปลี่ยนความเป็นเจ้าของเฉพาะข้อมูลที่ขณะนี้พร้อมใช้งานบนฮับ*</p><p><strong>คําเตือน:</strong> กระบวนการนี้สามารถทำให้สูญเสียข้อมูลที่ยังไม่ได้ซิงโครไนส์และการประมวลผลธุรกิจล้มเหลว ดังนั้น จึงควรใช้งานเฉพาะในกรณีฉุกเฉินเท่านั้น ที่กระบวนการทางธุรกิจต้องประมวลผลบนฮับเนื่องจากหน่วยปรับขนาดมีข้อจำกัดซึ่งไม่สามารถลดปัญหาได้ภายในเวลาที่เหมาะสม</p> |
| การยกเลิกโทโพโลยีแบบกระจาย | ลบการปรับใช้หน่วยปรับขนาดและเรียกใช้บนฮับเท่านั้น โดยไม่มีการประมวลผลปริมาณงาน |

> [!TIP]
> เมื่อเวลาผ่านไป การปรับปรุงส่วนเพิ่มจะเพิ่มลงในประสบการณ์ Scale Unit Manager เพื่อช่วยให้การดําเนินงานการจัดการรอบเวลาง่ายขึ้น ความสามารถเฉพาะของการนำออกใช้ในปัจจุบันจะบันทึกอยู่ในสมุดคู่มือการเตรียมความพร้อม ซึ่งพร้อมใช้งานกับลูกค้าที่อยู่ในกระบวนการเตรียมความพร้อมไปยังโทโพโลยีแบบไฮบริดแบบกระจายสำหรับ Supply Chain Management <!-- KFM: Add a link to the handbook when it is published -->

## <a name="feature-management-considerations-for-workloads"></a>ข้อควรพิจารณาเกี่ยวกับการจัดการลักษณะสำหรับปริมาณงาน

ส่วนนี้อธิบายแง่มุมที่สําคัญบางอย่างที่คุณควรพิจารณา เมื่อคุณติดตั้งปริมาณงาน เพิ่มคุณลักษณะ หรือเอาคุณลักษณะออกจากการปรับใช้โทโพโลยีแบบไฮบริดแบบกระจาย หลายสถานการณ์อาจกำหนดว่าคุณจะต้องเรียกใช้ [การอัปเกรดปริมาณงาน](#manage-workloads) หลังจากที่คุณทำการเปลี่ยนแปลงหรือไม่ อย่างไรก็ตาม โดยปกติแล้ว คุณจะต้องอัปเดตหรือเพิ่มการสอบถามการแลกเปลี่ยนข้อมูลใหม่และ/หรือเมื่อคุณเพิ่มตารางหรือฟิลด์ใหม่ที่ปริมาณงานที่ติดตั้งไว้ก่อนหน้านี้

### <a name="mandatory-features-for-installing-a-workload"></a>คุณลักษณะที่บังคับสำหรับการติดตั้งปริมาณงาน

เมื่อคุณติดตั้งปริมาณงาน กระบวนการติดตั้งจะสร้างข้อกำหนดปริมาณงานที่มีข้อมูลเกี่ยวกับตารางข้อมูลที่ใช้เมื่อซิงค์ข้อมูลระหว่างการปรับใช้สองชุด การสร้างข้อกำหนดปริมาณงานจะได้รับการจัดการโดยอัตโนมัติตามคุณลักษณะที่เปิดใช้งานใน [การจัดการคุณลักษณะ](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md) ตารางต่อไปนี้จะแสดงรายการคุณลักษณะที่ต้องเปิดใช้งานเพื่อสร้างข้อกำหนดปริมาณงานซึ่งต้องใช้ในการเรียกใช้คลังสินค้าหรือปริมาณงานการผลิต

| คุณลักษณะที่บังคับ | ปริมาณงาน |
|---|---|
| กำหนดแนวทางอัตโนมัติในการสร้างผู้ใช้ WHS | คลังสินค้า |
| การบล็อคงานทั่วทั้งองค์กร | คลังสินค้า |
| รายละเอียดป้ายฉลากเวฟการจัดส่ง | คลังสินค้า |
| การรองรับ Scale Unit สำหรับรายการงานของแอปคลังสินค้า | คลังสินค้า |
| การดำเนินการผลิต | การผลิต |

เมื่อคุณปรับใช้ปริมาณงานโดยใช้ [เครื่องมือการปรับใช้สเกลยูนิตสำหรับสภาพแวดล้อมการพัฒนาแบบสำเร็จรูป](https://github.com/microsoft/SCMScaleUnitDevTools) หรือ [พอร์ทัลตัวจัดการสเกลยูนิต](https://sum.dynamics.com)  คุณลักษณะที่บังคับทั้งหมดจะเปิดใช้งานโดยอัตโนมัติ อย่างไรก็ตาม ถ้าคุณทำการปรับใช้การทดสอบด้วยตนเอง ซึ่งไม่มีคุณลักษณะที่บังคับหนึ่งรายการหรือมากกว่า การติดตั้งปริมาณงานจะล้มเหลว และคุณจะได้รับข้อความที่แสดงรายการคุณลักษณะที่หายไป จากนั้น คุณต้องเปิดใช้งานคุณลักษณะเหล่านั้นด้วยตนเองและใช้งานการติดตั้งปริมาณงานอีกครั้ง

### <a name="enabling-or-disabling-features-that-have-data-synchronization-dependencies"></a>การเปิดใช้งานหรือปิดใช้งานคุณลักษณะที่อิงกับการขึ้นต่อกันของการซิงโครไนส์ข้อมูล

คุณลักษณะที่มีผลต่อการเลือกข้อมูลที่มีการซิงค์ระหว่างฮับและสเกลยูนิตยังสามารถส่งผลต่อวิธีการสร้างข้อกำหนดปริมาณงานอีกด้วย ดังนั้นคุณจึงต้องเปิดใช้งานคุณลักษณะเหล่านี้ก่อนที่คุณจะติดตั้งปริมาณงาน ถ้าคุณเปิดใช้งานคุณลักษณะชนิดนี้ขณะที่คุณเรียกใช้ปริมาณงาน คุณต้องสร้างข้อกำหนดปริมาณงานขึ้นใหม่โดยการเรียกใช้ [การอัปเกรดปริมาณงาน](#manage-workloads) หลังจากคุณเปิดใช้งานคุณลักษณะ เช่นเดียวกัน หกคุณปิดใช้งานคุณลักษณะที่มีการขึ้นต่อกันของการซิงโครไนส์ข้อมูลขณะที่คุณเรียกใช้ปริมาณงาน คุณต้องเรียกใช้ [การอัปเกรดปริมาณงาน](#manage-workloads) เพื่อเอาข้อมูลการซิงโครไนส์ข้อมูลที่เกี่ยวข้องออกจากข้อกำหนดปริมาณงาน
