# <a name="warehouse-management-workloads-for-cloud-and-edge-scale-units"></a>ปริมาณงานการจัดการคลังสินค้าสำหรับ Cloud และ edge scale unit

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

> [!WARNING]
> ฟังก์ชันธุรกิจบางอย่างไม่ได้รับการสนับสนุนอย่างเต็มที่ในตัวอย่างสำหรับสาธารณะเมื่อมีการใช้ปริมาณงาน scale unit ตรวจสอบให้แน่ใจว่าได้ใช้กระบวนการที่หัวข้อนี้อธิบายไว้อย่างชัดแจ้งว่าได้รับการสนับสนุน

## <a name="warehouse-execution-on-scale-units"></a>การดำเนินการของคลังสินค้าใน scale unit

คุณลักษณะนี้ช่วยให้ scale unit รันกระบวนการที่เลือกจากความสามารถการจัดการคลังสินค้า Cloud scale unit จะรันปริมาณงานของตนใน cloud โดยใช้กำลังการประมวลผลโดยเฉพาะในภูมิภาค Microsoft Azure ที่คุณเลือก สำหรับ edge scale unit คุณสามารถรันปริมาณงานได้อย่างอิสระในสถานที่ แม้ในขณะที่ scale units ถูกยกเลิกการเชื่อมต่อจากระบบคลาวด์ชั่วคราว

ในหัวข้อนี้ การดำเนินการจัดการคลังสินค้าในคลังสินค้าที่กำหนดเป็น scale unit เรียกว่า *ระบบปฏิบัติการของคลังสินค้า* (*WES*)

## <a name="prerequisites"></a>ข้อกำหนดเบื้องต้น

คุณต้องมีฮับ Dynamics 365 Supply Chain Management และ scale unit ที่มีการปรับใช้กับปริมาณงานการจัดการคลังสินค้า สำหรับข้อมูลเพิ่มเติมเกี่ยวกับสถาปัตยกรรมและกระบวนการปรับใช้ ให้ดูที่ [Cloud และ edge scale units สำหรับปริมาณงานการจัดการการผลิตและคลังสินค้า](cloud-edge-landing-page.md)

## <a name="how-the-wes-workload-works-on-scale-units"></a>วิธีการทำงานของปริมาณงาน WES บน scale unit

สำหรับกระบวนการในปริมาณงานการจัดการคลังสินค้า ข้อมูลจะถูกซิงค์ระหว่างฮับกับ scale unit

scale unit สามารถรักษาได้เฉพาะข้อมูลที่เป็นเจ้าของ แนวคิดเกี่ยวกับความเป็นเจ้าของของข้อมูลสำหรับ scale unit จะช่วยป้องกันความขัดแย้งแบบหลายหลัก ดังนั้น จึงเป็นเรื่องสำคัญอย่างยิ่งที่คุณจะเข้าใจว่ากระบวนการใดเป็นของฮับซึ่งเป็นของ scale unit

scale units. เป็นเจ้าของข้อมูลต่อไปนี้:

- **ข้อมูลการประมวลผลเวฟ** –วิธีการกระบวนการเวฟที่เลือกจะได้รับการจัดการเป็นส่วนหนึ่งของการประมวลผลเวฟ scale unit
- **ข้อมูลการประมวลผลงาน** –สนับสนุนการประมวลผลชนิดของใบสั่งงานต่อไปนี้:

    - ความเคลื่อนย้ายของสินค้าคงคลัง (การเคลื่อนย้ายด้วยตนเองและการเคลื่อนย้ายตามงานเท็มเพลต)
    - ใบสั่งซื้อ (putaway ทำงานโดยผ่านใบสั่งคลังสินค้า)
    - ใบสั่งขาย (งานการเบิกและการโหลดสินค้าอย่างง่าย)

- **ข้อมูลการรับสินค้าตามใบสั่งคลังสินค้า** –ข้อมูลนี้จะใช้สำหรับใบสั่งซื้อที่นำออกใช้ไปยังคลังสินค้าด้วยตนเองเท่านั้น
- **ข้อมูลป้ายทะเบียน** –ป้ายทะเบียนสามารถสร้างขึ้นบนฮับและ scale unit มีการจัดการความขัดแย้งโดยเฉพาะ โปรดทราบว่าข้อมูลนี้ไม่เป็นแบบเฉพาะคลังสินค้า

## <a name="outbound-process-flow"></a>โฟลว์กระบวนการขาออก

ฮับเป็นเจ้าของข้อมูลต่อไปนี้:

- เอกสารต้นทางทั้งหมด เช่น ใบสั่งขาย และใบสั่งโอนย้าย
- การจัดสรรใบสั่งและการประมวลผลการโหลดขาออก
- การนำออกใช้ไปยังคลังสินค้า การสร้างการจัดส่ง และกระบวนการสร้างเวฟ

scale unit เป็นเจ้าของการประมวลผลเวฟตามจริง (เช่น การปันส่วนงาน งานการเติมสินค้า และการสร้างงานที่ต้องการ) หลังจากที่เวฟนำออกใช้ ดังนั้น ผู้ปฏิบัติงานคลังสินค้าสามารถประมวลผลงานขาออกได้โดยใช้แอปคลังสินค้าที่มีการเชื่อมต่อกับ scale unit

![โฟลว์การประมวลผลเวฟ](./media/wes_wave_processing_flow.png "โฟลว์การประมวลผลเวฟ")

## <a name="inbound-process-flow"></a>โฟลว์กระบวนการขาออก

ฮับเป็นเจ้าของข้อมูลต่อไปนี้:

- เอกสารต้นทางทั้งหมด เช่น ใบสั่งซื้อ และใบสั่งส่งคืนสินค้าที่ขาย
- การประมวลผลโหลดขาเข้า

> [!NOTE]
> โฟลว์ใบสั่งซื้อขาเข้ามีแนวคิดแตกต่างจากโฟลว์ขาออก ซึ่ง scale unit ที่การประมวลผลจะขึ้นอยู่กับว่าใบสั่งถูกนำออกใช้ไปยังคลังสินค้าหรือไม่

ถ้าคุณกำลังใช้กระบวนการ *นำออกใช้ไปยังคลังสินค้า* ใบสั่งคลังสินค้าจะถูกสร้างขึ้น และเป็นเจ้าของโฟลว์การรับสินค้าที่เกี่ยวข้องจะถูกกำหนดให้กับ scale unit ฮับจะไม่สามารถลงทะเบียนการรับสินค้าขาเข้า

ผู้ปฏิบัติงานสามารถรันกระบวนการรับได้ โดยใช้แอปคลังสินค้าที่มีการเชื่อมต่อกับ scale unit ข้อมูลจะถูกบันทึกโดย scale unit และรายงานจากใบสั่งคลังสินค้าขาเข้า การสร้างและการดำเนินการ putaway ในเวลาต่อมาจะถูกจัดการโดย scale unit

ถ้าคุณไม่ได้ใช้กระบวนการ *นำออกใช้ไปยังคลังสินค้า* ดังนั้นจึงไม่ได้ใช้ *ใบสั่งคลังสินค้า* ฮับสามารถประมวลผลการรับสินค้าและการประมวลผลงานของคลังสินค้าได้โดยอิสระจาก scale unit

![โฟลว์กระบวนการขาออก](./media/wes_Inbound_flow.png "โฟลว์กระบวนการขาออก")

## <a name="supported-processes-and-roles"></a>กระบวนการและบทบาทที่ได้รับการสนับสนุน

ไม่ใช่กระบวนการจัดการคลังสินค้าทั้งหมดจะได้รับการสนับสนุนในปริมาณงาน WES บน scale unit ดังนั้น เราขอแนะนำให้คุณกำหนดบทบาทที่ตรงกับฟังก์ชันที่พร้อมใช้งานสำหรับผู้ใช้แต่ละราย

เพื่ออำนวยความสะดวกในกระบวนการนี้ บทบาทตัวอย่างที่มีชื่อว่า *ผู้จัดการคลังสินค้าในปริมาณงาน* จะรวมอยู่ในข้อมูลสาธิตที่ **การจัดการระบบ \> ความปลอดภัย \> การตั้งค่าคอนฟิกความปลอดภัย** วัตถุประสงค์ของบทบาทนี้คือ เพื่อเปิดใช้งานผู้จัดการคลังสินค้าเพื่อเข้าถึง WES บน scale unit บทบาทจะให้สิทธิ์เข้าถึงหน้าซึ่งเกี่ยวข้องกับบริบทของปริมาณงานที่โฮสต์อยู่บน scale unit

บทบาทผู้ใช้ใน scale unit จะถูกกำหนดเป็นส่วนหนึ่งของการซิงโครไนส์ข้อมูลเริ่มต้นจากฮับไปยัง scale unit

เมื่อต้องการปรับเปลี่ยนบทบาทที่กำหนดให้กับผู้ใช้ให้ ไปที่ **การจัดการระบบ \> ความปลอดภัย \> กำหนดผู้ใช้ให้กับบทบาท** บน scale unit ผู้ใช้ที่ทำหน้าที่เป็นผู้จัดการคลังสินค้าใน scale unit เท่านั้นที่ควรได้รับการกำหนดเฉพาะบทบาท *ผู้จัดการคลังสินค้าในปริมาณงาน* วิธีการนี้จะตรวจสอบให้แน่ใจว่าผู้ใช้เหล่านั้นมีสิทธิ์เข้าถึงฟังก์ชันการทำงานที่ได้รับการสนับสนุนเท่านั้น ลบบทบาทอื่นๆ ที่กำหนดให้กับผู้ใช้เหล่านั้น

ผู้ใช้ที่ทำหน้าที่เป็นผู้จัดการคลังสินค้าทั้งในฮับและ scale unit ที่ควรได้รับการกำหนดบทบาท *ผู้จัดการคลังสินค้าในปริมาณงาน* ที่มีอยู่ โปรดทราบว่าบทบาทนี้ให้สิทธิ์ผู้ปฏิบัติงานคลังสินค้าเข้าถึงคุณลักษณะต่างๆ (เช่น การประมวลผลใบสั่งโอนย้าย) ที่ปรากฏในอินเทอร์เฟสผู้ใช้ (UI) แต่ปัจจุบันไม่ได้รับการสนับสนุนบน scale unit

## <a name="supported-wes-processes"></a>กระบวนการ WES ที่ได้รับการสนับสนุน

กระบวนการดำเนินการของคลังสินค้าต่อไปนี้สามารถเปิดใช้งานสำหรับปริมาณงาน WES ใน scale unit:

- วิธีการเวฟที่เลือกสำหรับใบสั่งขายและการเติมสินค้าตามความต้องการ
- การรันใบสั่งงานจากใบสั่งขายและการเติมสินค้าตามความต้องการโดยใช้แอปคลังสินค้า
- การสอบถามปริมาณคงคลังคงเหลือโดยใช้แอปคลังสินค้า
- การสร้างและการรันการเคลื่อนย้ายสินค้าคงคลังโดยใช้แอปคลังสินค้า
- การลงทะเบียนใบสั่งซื้อและการทำงานการสำรองสินค้าโดยใช้แอปคลังสินค้า

ชนิดของใบสั่งงานต่อไปนี้ปัจจุบันได้รับการสนับสนุนสำหรับปริมาณงาน WES ในการปรับใช้ scale unit:

- ใบสั่งขาย
- การเติมสินค้า
- การเคลื่อนย้ายสินค้าคงคลัง
- ใบสั่งซื้อที่เชื่อมโยงกับใบสั่งคลังสินค้า

ไม่สนับสนุนการประมวลผลเอกสารต้นทางอื่นบน scale units ตัวอย่างเช่น สำหรับปริมาณงาน WES ใน scale unit คุณจะไม่สามารถดำเนินการต่อไปนี้ได้

- นำใบสั่งโอนย้ายออกใช้
- ประมวลผลการปฎิบัติการเบิกสินค้าและการจัดส่งคลังสินค้าขาออก

> [!IMPORTANT]
> ถ้าคุณใช้ปริมาณงานใน scale unit คุณจะไม่สามารถรันกระบวนการที่ไม่ได้รับการสนับสนุนสำหรับคลังสินค้าเฉพาะบนฮับ

ฟังก์ชันการจัดการคลังสินค้าต่อไปนี้ไม่ได้รับการสนับสนุนบน scale units:

- การประมวลผลขาเข้าและขาออกสำหรับสินค้าที่มีมิติการติดตามใดๆ ที่ใช้งานอยู่ (เช่น มิติหมายเลขชุดงานหรือหมายเลขลำดับประจำสินค้า)
- การประมวลผลการเปลี่ยนแปลงสถานะสินค้าคงคลัง
- การประมวลผลของสินค้าคงคลังที่มีค่าสถานะการบล็อค
- การรวมกับการจัดการคุณภาพ
- การรวมกับการผลิต
- การดำเนินการของรายการตามน้ำหนักจริง
- การประมวลผลของการจัดส่งมากกว่าและน้อยกว่าปริมาณที่สั่ง
- การประมวลผลของปริมาณคงคลังคงเหลือที่เป็นค่าลบ

### <a name="outbound-supported-only-for-sales-orders-and-demand-replenishment"></a>ขาออก (ที่ได้รับการสนับสนุนสำหรับใบสั่งขายและการเติมสินค้าตามความต้องการ เท่านั้น)

ตารางต่อไปนี้จะแสดงว่ามีการสนับสนุนคุณลักษณะขาออกใด และจะสนับสนุนคุณลักษณะที่ได้รับเมื่อมีการใช้ปริมาณงานการจัดการคลังสินค้าใน cloud และ edge scale unit

> [!WARNING]
> เนื่องจากการประมวลผลใบสั่งขายเท่านั้นที่ได้รับการสนับสนุน การประมวลผลการจัดการคลังสินค้าขาออกจึงไม่สามารถใช้สำหรับใบสั่งโอนย้ายได้
>
> ฟังก์ชันของคลังสินค้าบางฟังก์ชันจะไม่พร้อมใช้งานในคลังสินค้าที่กำลังรันปริมาณงานการจัดการคลังสินค้าใน scale unit

| ประมวลผล                                                      | ฮับ | ปริมาณงาน WES ใน scale unit |
|--------------------------------------------------------------|-----|------------------------------|
| การประมวลผลเอกสารต้นทาง                                   | ใช่ | หมายเลข |
| การประมวลผลการจัดการการโหลดและขนส่ง                | ใช่ | หมายเลข |
| นำออกใช้ไปยังคลังสินค้า                                         | ใช่ | หมายเลข |

## <a name="maintaining-scale-units-for-wes"></a>การรักษา scale unit สำหรับ WES

ชุดงานต่างๆ รันทั้งบนฮับและ scale unit

ในการปรับใช้ฮับ คุณสามารถรักษาชุดงานด้วยตนเองได้ คุณสามารถจัดการงานสามงานต่อไปนี้ได้เมื่อ **การจัดการคลังสินค้า \> งานประจำงวด \> การจัดการปริมาณงานของฝ่ายสนับสนุน**:

- ประมวลผลเหตุการณ์อัพเดตสถานะงาน
- ประมวลผลเหตุการณ์ถ่ายโอนการควบคุมการดำเนินการของเวฟ
- ลงทะเบียนการรับสินค้าตามใบสั่งต้นทาง

บนปริมาณงานใน scale unit คุณสามารถจัดการชุดงานสองงานต่อไปนี้ได้เมื่อ **การจัดการคลังสินค้า \> งานประจำงวด \> ปริมาณงาน**:

- ประมวลผลเรกคอร์ดตารางเวฟ
- ประมวลผลเหตุการณ์ถ่ายโอนการควบคุมการดำเนินการของเวฟ
