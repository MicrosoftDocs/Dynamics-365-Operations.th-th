# <a name="manage-voyages"></a>จัดการการเดินทาง

[!include [banner](../../includes/banner.md)]

โดยทั่วไปการเดินทางแสดงถึงเรือ อย่างไรก็ตาม ตามแนวทางปฏิบัติและขั้นตอนของคุณ อาจแสดงถึงผู้จัดผู้ขาย ใบสั่งซื้อ หรือสินค้าอื่นๆ ที่เหมาะสมขององค์กร

หน้า **การเดินทางทั้งหมด** จะมีรายละเอียดการเดินทาง ข้อมูลการจัดส่งและการคิดต้นทุน และข้อมูลเกี่ยวกับสินค้า ใบสั่งซื้อ และใบสั่งโอนย้าย เมื่อต้องการเปิดหน้า **การเดินทางทั้งหมด** ให้ไปที่ **ต้นทุนแฝง \> การเดินทาง \> การเดินทางทั้งหมด** หน้านี้แสดงรายการการเดินทางปัจจุบันทั้งหมด คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อสร้าง ลบ และทำงานกับการเดินทางได้ เลือกการเดินทางในรายการเพื่อดูรายละเอียด

> [!NOTE]
> คอนเทนเนอร์การจัดส่งสินค้าและใบแจ้งรายการถูกเชื่อมโยงกับการเดินทาง บรรทัดการซื้อจะเชื่อมโยงกับคอนเทนเนอร์การจัดส่งสินค้า หากปิดตู้คอนเทนเนอร์การจัดส่งสินค้าและใบแจ้งรายการ คุณยังสามารถเชื่อมโยงกับการเดินทางโดยตรงได้ นอกจากนี้ ต้นทุนที่ป้อนไว้ที่นี่จะถูกจัดตามส่วนกับบรรทัดการซื้อที่แนบทั้งหมด

## <a name="action-pane"></a>บานหน้าต่างการดำเนินการ

บานหน้าต่างการดำเนินการของหน้า **การเดินทาง** จะมีปุ่มที่ให้คุณสามารถใช้งานการเดินทางที่เลือกได้ แต่ละปุ่มจะปฏิบัติการเดียว บานหน้าต่างการดำเนินการยังรวมแท็บต่างๆ ซึ่งจะมีชุดปุ่มที่เกี่ยวข้อง ยกเว้นของหมายเหตุ ปุ่มและแท็บทั้งหมดจะพร้อมใช้งานทั้งในมุมมองรายการของหน้า **การเดินทางทั้งหมด** และในมุมมองโดยละเอียดของการเดินทางหนึ่งๆ ที่เลือก

### <a name="buttons-that-appear-directly-on-the-action-pane"></a>ปุ่มที่จะปรากฏโดยตรงในบานหน้าต่างการดำเนินการ

ตารางต่อไปนี้อธิบายถึงปุ่มที่พร้อมใช้งานบนบานหน้าต่างการดำเนินการโดยตรง

| ปุ่ม | คำอธิบาย |
|---|---|
| สร้าง  | สร้างการเดินทาง <!-- KFM: Link to scenario --> |
| Delete | ลบการเดินทางปัจจุบัน สามารถลบได้เฉพาะการเดินทางที่มีสถานะการเดินทางเป็น *ยืนยันแล้ว* เท่านั้น หลังจากการเดินทางออกจากท่าเรือและประมวลผลสินค้าที่อยู่ระหว่างการส่งต่อแล้ว จะไม่สามารถลบสินค้านั้นออกได้อีกต่อไป |
| ตัวแก้ไขการเดินทาง | เปิดหน้า **โปรแกรมแก้ไขการเดินทาง** ซึ่งคุณสามารถเพิ่มหรือลบรายการซื้อการเดินทาง สร้างคอนเทนเนอร์ใหม่ และแก้ไขรายละเอียดการเดินทางด้วยตัวเอง |
| ต้นทุนการเดินทาง | เปิดหน้า **ต้นทุนการเดินทาง** ที่คุณสามารถดูและเพิ่มต้นทุนการเดินทางให้กับสินค้าทั้งหมดในการเดินทาง เมื่อต้นทุนการเดินทางถูกเพิ่มไปยังการเดินทางด้วยตนเอง ต้นทุนการเดินทางจะถูกเพิ่มเข้าในหน้า **การสอบถามต้นทุน** โดยอัตโนมัติ และแบ่งตามสินค้าแต่ละรายการตามวิธีการที่ระบุในหน้า **ต้นทุนการเดินทาง** |

### <a name="buttons-on-the-voyage-tab"></a>ปุ่มบนแท็บการเดินทาง

ตารางต่อไปนี้อธิบายถึงปุ่มที่พร้อมใช้งานบนแท็บ **การเดินทาง** ของบานหน้าต่างการดำเนินการ แท็บนี้จะพร้อมใช้งานเฉพาะเมื่อคุณใช้งานในมุมมองรายการของหน้า **การเดินทางทั้งหมด** เท่านั้น

| ปุ่ม | คำอธิบาย |
|---|---|
| คอนเทนเนอร์การจัดส่ง | เปิดหน้าที่แสดงคอนเทนเนอร์การจัดส่งทั้งหมดที่เชื่อมโยงกับการเดินทางที่เลือก อาจมีหนึ่งคอนเทนเนอร์หรือหลายคอนเทนเนอร์ |
| ใบแจ้งรายการ | เปิดหน้าที่แสดงใบแจ้งรายการทั้งหมดที่เชื่อมโยงกับการเดินทางที่เลือก อาจมีหนึ่งใบแจ้งรายการหรือหลายใบแจ้งรายการ |

### <a name="buttons-on-the-manage-tab"></a>ปุ่มบนแท็บจัดการ

ตารางต่อไปนี้อธิบายถึงการดำเนินการที่พร้อมใช้งานบนแท็บ **จัดการ** ของบานหน้าต่างการดำเนินการ

| ปุ่ม | คำอธิบาย |
|---|---|
| เอกสารที่ได้รับ | อัปเดตการเดินทางเพื่อให้ตั้งค่าตัวเลือก **เอกสารที่ได้รับ** เป็น *ใช่* คุณสามารถใช้ปุ่มนี้เพื่อล็อคสินค้าและ/หรือรายการซื้อเพื่อให้ไม่สามารถอัปเดตเพิ่มเติมได้ |
| อยู่ระหว่างขนส่ง | อัปเดตฟิลด์ **สถานะการเดินทาง** เป็นสถานะในการส่งต่อที่สร้างขึ้นในหน้า **[พารามิเตอร์ต้นทุนแฝง](landed-cost-parameters.md)** ไม่มีตรรกะอื่นในกระบวนการนี้ นอกจากนี้ คุณยังสามารถอัปเดตการเดินทางโดยอัตโนมัติเป็นสถานะในการส่งต่อ ตามการตั้งค่าใน [ศูนย์ควบคุมการติดตาม](delivery-information-setup.md) ได้
| พร้อมสำหรับการคิดต้นทุน | อัปเดตฟิลด์ **สถานะการเดินทาง** เป็นสถานะพร้อมสำหรับการคิดต้นทุนที่สร้างขึ้นในหน้า **[พารามิเตอร์ต้นทุนแฝง](landed-cost-parameters.md)** คุณสามารถคิดต้นทุนการเดินทางเมื่อประมวลผลใบแจ้งหนี้ทั้งหมดแล้ว (ทั้งใบแจ้งหนี้สินค้าคงคลังและใบแจ้งหนี้ต้นทุนการเดินทาง) และสินค้าที่ได้รับแล้ว หากต้นทุนที่ประเมินซึ่งสัมพันธ์กับการเดินทางยังไม่ได้คิดต้นทุน จะเกิดข้อผิดพลาดขึ้นเมื่อคุณพยายามประมวลผลการคิดต้นทุนของการเดินทาง |
| คิดต้นทุน | ล้างข้อมูลความผิดปกติในการคิดต้นทุนใด ๆ หลังจากมีใบแจ้งหนี้อยู่เพื่อใบสั่งซื้อและต้นทุนการเดินทางทั้งหมด เมื่อคุณเลือกปุ่มนี้ กล่องโต้ตอบ **การอัปเดตการเดินทาง - คิดต้นทุน** จะปรากฏขึ้น คุณสามารถเลือกที่จะลงรายการบัญชีในวันที่ทางการเงินมาตรฐานหรือระบุวันที่ลงรายการบัญชี แล้วรันการลงบัญชี คุณสามารถดำเนินการได้บ่อยครั้งตามที่คุณต้องการ คุณยังสามารถใช้กล่องโต้ตอบที่คิดต้นทุน **การอัปเดตการเดินทาง - คิดต้นทุน** เพื่อกําหนดการเพื่อรันการดำเนินการการลงบัญชีเป็นงานช่วงเวลา (ชุดงาน) ขอแนะนำว่าคุณควรรันการดำเนินการเป็นปกติโดยการตั้งค่าเป็นชุดงาน |
| ลงรายการบัญชีรายการรับสินค้า | ลงรายการบัญชีรายการรับสินค้าจากบรรทัดใบสั่งซื้อทั้งหมดในการเดินทาง ถ้ามีการใช้การเดินทางหลายบริษัท กล่องโต้ตอบการลงรายการบัญชีรายการรับสินค้าใหม่จะเปิดขึ้นของแต่ละบริษัทและต้องมีการประมวลผลในนิติบุคคลแต่ละแห่ง |
| ลงรายการบัญชีใบรับสินค้า | ลงรายการบัญชีใบรับสินค้าจากบรรทัดใบสั่งซื้อทั้งหมดในการเดินทาง กระบวนการใบรับสินค้าในบรรทัดใบสั่งซื้อที่เชื่อมโยงกับการเดินทางจะถูกใช้ต่อเมื่อสินค้า **ไม่** ผ่านการประมวลผลสินค้าที่อยู่ระหว่างส่งต่อ ถ้าสินค้าจะผ่านกระบวนการสินค้าที่อยู่ระหว่างการส่งต่อ คุณจะได้รับข้อผิดพลาดเมื่อคุณพยายามลงรายการบัญชีใบรับสินค้าในบรรทัดใบสั่งซื้อ ถ้ามีการใช้การเดินทางหลายบริษัท กล่องโต้ตอบการลงรายการบัญชีบันทึกการจัดส่งใหม่จะถูกเปิดขึ้นของแต่ละบริษัท |
| ลงรายการบัญชีใบแจ้งหนี้ | ลงรายการบัญชีใบแจ้งหนี้จากบรรทัดใบสั่งซื้อทั้งหมดในการเดินทาง ถ้าสินค้าบนการเดินทางจะผ่านกระบวนการสินค้าที่อยู่ระหว่างการส่งต่อ รายการใบสั่งซื้อจะได้รับการออกใบแจ้งหนี้ก่อนกระบวนการรับสินค้าเสร็จสิ้น เมื่อมีการออกใบแจ้งหนี้ต้นฉบับใบสั่งซื้อ ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อที่เชื่อมโยงกับบรรทัดต้นฉบับใบสั่งซื้อจะถูกสร้างขึ้น จากนั้นใบสั่งดังกล่าวจะได้รับจากคลังสินค้า ถ้ามีการใช้การรจัดส่งหลายบริษัท กล่องโต้ตอบการลงรายการบัญชีใบแจ้งหนี้ใหม่จะถูกเปิดขึ้นของแต่ละบริษัท |
| ใบสั่งโอนย้ายการจัดส่ง | ลงรายการบัญชีการเดินทางใบสั่งโอนย้ายจากบรรทัดใบสั่งโอนย้ายทั้งหมดในการเดินทาง เมื่อเลือกปุ่มนี้ เฉพาะใบสั่งโอนย้ายเท่านั้นที่จะพร้อมใช้งานถ้าอัปเดต |
| รับใบสั่งโอนย้าย | ลงรายการบัญชีการรับสินค้าใบสั่งโอนย้ายจากบรรทัดใบสั่งโอนย้ายทั้งหมดในการเดินทาง |
| รับสินค้าที่อยู่ระหว่างส่งต่อ | รับบรรทัดใบสั่งทั้งหมดที่อยู่ระหว่างการส่งต่อการเดินทาง ปุ่มนี้เป็นหนึ่งในสามตัวเลือกที่พร้อมให้รับสินค้าที่อยู่ระหว่างการส่งต่อเพื่อการเดินทาง (ตัวเลือกอีกสองตัวเลือกคือ ปุ่ม **สร้างสมุดรายวันการมาถึง** ที่อธิบายไว้ในภายหลังในตารางนี้ และแอปการจัดการคลังสินค้าบนมือถือ) ตัวเลือกนี้เป็นตัวเลือกอย่างง่าย และจะประมวลผลสินค้าในการส่งต่อออกจากคลังสินค้าที่ส่งต่อสินค้าและไปยังคลังสินค้าปลายทางสุดท้าย ถ้าคุณต้องการควบคุมกระบวนการเพิ่มเติม ให้ใช้สมุดรายวันการมาถึงหรืออุปกรณ์เคลื่อนที่เพื่อประมวลผลการรับสินค้า |
| ค้นหาต้นทุนอัตโนมัติ | ค้นหาต้นทุนการเดินทางที่เกี่ยวข้องใด ๆ หากพบหรืออัปเดตต้นทุนเหล่านี้แล้ว คุณจะได้รับข้อความต่อไปนี้ :"มีบรรทัดต้นทุนที่ยังไม่ได้ออกใบแจ้งหนี้อยู่ คุณต้องการเขียนทับข้อมูลดังกล่าวหรือไม่ ต้นทุนใด ๆ ที่ไม่ได้เชื่อมโยงกับการเดินทางในเวลาที่สร้าง ต้นทุนการเดินทางที่แนบกับการเดินทางและออกใบแจ้งหนี้แล้วจะไม่บันทึกทับ |
| สร้างสมุดรายวันการมาถึง | <p>เปิดกล่องโต้ตอบ **สร้างสมุดรายวันการมาถึง** ซึ่งคุณสามารถสร้างสมุดรายวันการมาถึงที่ระบุสถานที่เบิกสินค้า กล่องโต้ตอบมีตัวเลือกต่อไปนี้:</p><ul><li>**สร้างจากสินค้าที่อยู่ระหว่างส่งต่อ** หรือ **สร้างจากใบสั่งโอนย้าย** – ป้ายชื่อสํารวจตัวเลือกนี้จะเปลี่ยนแปลงไปตามว่าคุณอยู่ในระหว่างกระบวนการส่งต่อหรือไม่ ตั้งค่าเป็น *ใช่* เพื่อเปิดหน้าสมุดรายวันการมาถึง ซึ่งช่วยให้คุณประมวลผลสมุดรายวันการมาถึงมาตรฐานของสินค้าที่อยู่ระหว่างการส่งต่อที่เกี่ยวข้องกับการเดินทาง ถ้าได้รับสินค้าในคลังสินค้าปลายทางสุดท้ายแล้ว จะไม่เพิ่มลงในบรรทัดสมุดรายวันการมาถึง</li><li>**เริ่มต้นปริมาณ** – ตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่อเริ่มต้นปริมาณที่จะได้รับตามปริมาณของสินค้าที่ระบุบนรายการการเดินทาง ถ้ารายการการเดินทางได้รับแล้วบางส่วน ปริมาณนี้จะเป็นปริมาณที่เหลืออยู่ เราขอแนะนำให้คุณตั้งค่าตัวเลือกนี้เป็น *ใช่*</li><li>**สร้างจากบรรทัดใบสั่ง** – ตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่อรับค่าจากบรรทัดใบสั่ง</li></ul><p>ปุ่มนี้เป็นหนึ่งในสามตัวเลือกที่พร้อมให้รับสินค้าที่อยู่ระหว่างการเดินทาง (ตัวเลือกอื่นๆ คือ ปุ่ม **รับสินค้าที่อยู่ระหว่างส่งต่อ** ที่อธิบายไว้ก่อนหน้านี้ในตารางนี้ และแอปการจัดการคลังสินค้าบนมือถือ)</p> |
| ต้นทุนค้างรับ | คุณสามารถรับรู้ต้นทุนโดยชนิดต้นทุนมีรหัสบัญชีที่ระบุไว้เป็นเดบิต โดยทั่วไป ปุ่มนี้จะใช้เมื่อสินค้าคงคลังอยู่ระหว่างการส่งต่อ หรือเมื่อได้รับและออกใบแจ้งหนี้สินค้าแล้ว |
| รวมต้นทุน | ย้ายต้นทุนจากระดับคอนเทนเนอร์การจัดส่งไปยังระดับการเดินทาง คุณอาจใช้ปุ่มนี้ในสถานการณ์บริการ/การจัดส่งที่ใช้ร่วมกัน ซึ่งหลายเอนทิตีจะใช้คอนเทนเนอร์การจัดส่งร่วมกันหรือพื้นที่ที่เป็นกล่องกระดาษ ตัวอย่างเช่น การเดินทางมีคอนเทนเนอร์ขนาด 40 ฟุตและคอนเทนเนอร์ขนาด 20 ฟุต และแบ่งตามปริมาตร ในกรณีนี้ สินค้า/เอนทิตีที่ใช้พื้นที่ร่วมกันหรือใช้พื้นที่ในคอนเทนเนอร์การจัดส่ง 20 ฟุตอาจเป็นแบบส่วนบุคคล เพื่อกระจายต้นทุนอย่างยุติธรรม บางองค์กรอาจต้องการโอนย้ายต้นทุนไปยังการเดินทางและกระจายค่าใช้จ่ายเหล่านั้นตามวิธีการจัดระดับการเดินทาง |
| เปลี่ยนเทมเพลตการเดินทาง | เปิดกล่องโต้ตอบซึ่งคุณสามารถแก้ไขแม่แบบการเดินทางได้ หลังจากที่คุณเปลี่ยนแปลงแม่แบบแล้ว ต้นทุนการเดินทางจะถูกลบออก ดังนั้น คุณจึงต้องเลือก **ค้นหาต้นทุนอัตโนมัติ** (ดูข้อความอธิบายก่อนหน้านี้ในตารางนี้) หรือเพิ่มต้นทุนด้วยตนเองอีกครั้ง |

### <a name="buttons-on-the-general-tab"></a>ปุ่มบนแท็บทั่วไป

ตารางต่อไปนี้อธิบายถึงปุ่มที่พร้อมใช้งานบนแท็บ **ทั่วไป** ของบานหน้าต่างการดำเนินการ

| ปุ่ม | คำอธิบาย |
|---|---|
| รายการรับสินค้า | เปิดรายการใบรับสินค้าจากบรรทัดใบสั่งซื้อทั้งหมดในการเดินทาง ถ้ามีการใช้การเดินทางหลายบริษัท รายการรับรายการรับใหม่จะเปิดขึ้นในแต่ละบริษัท ถ้าไม่ได้ประมวลผลรายการใบรับสินค้าไว้ ปุ่มนี้จะไม่พร้อมใช้งาน |
| ใบรับสินค้า | เปิดเรกคอร์ดใบรับสินค้าจากรายการใบสั่งซื้อที่เชื่อมโยงกับการเดินทาง หากมีการใช้เรกคอร์ดนั้น ถ้าไม่ได้ลงรายการบัญชีใบรับสินค้าไว้ ปุ่มนี้จะไม่พร้อมใช้งาน จะไม่ใช้กระบวนการใบรับสินค้า ถ้าคุณใช้การประมวลผลสินค้าที่อยู่ระหว่างส่งต่อ |
| การมาถึงของสินค้า | เปิดสมุดรายวันสินค้าที่มาถึง ถ้ามีการใช้ |
| การติดตาม | เปิดหน้า **การติดตามขาเข้า** ซึ่งคุณสามารถอัปเดตวันที่มาถึงที่คาดไว้ของสินค้าในคอนเทนเนอร์การจัดส่งและการเดินทาง และอัปเดตวันที่จัดส่งที่คาดไว้ของบรรทัดใบสั่งซื้อในเวลาต่อมา |
| การสอบถามต้นทุน | <p>เปิดหน้า **การสอบถามต้นทุน** ซึ่งแสดงต้นทุนทั้งหมดของการเดินทาง</p><p>ให้เลือก **มุมมอง** ในบานหน้าต่างการดำเนินการเพื่อปรับมุมมอง คุณสามารถดูระดับใดก็ได้ รวมทั้งรหัสชนิดสินค้าและต้นทุน</p><p>เฉพาะรหัสชนิดของต้นทุนที่เกี่ยวข้องกับรายการความเคลื่อนไหวของสินค้าคงคลังโดยตรงเท่านั้น ที่จะปรากฏในหน้า **การสอบถามต้นทุน** โดยการนำรหัสชนิดต้นทุนออก คุณสามารถปรับปรุงหน้าด้วยการจัดกลุ่มต้นทุนเข้าด้วยกันได้ ความสามารถนี้มีประโยชน์ถ้าคุณจะใช้ขนาดและสี</p><p>หน้า **การสอบถามต้นทุน** แสดงเฉพาะรหัสชนิดของต้นทุนที่มีฟิลด์ **เดบิต** ตั้งค่าเป็น *สินค้า*</p> |
| ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ | เปิดหน้า **ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ** ซึ่งแสดงเรกคอร์ดสินค้าที่อยู่ระหว่างการส่งต่อของการเดินทางที่เลือกเท่านั้น |

## <a name="lines-view"></a>มุมมองรายการ

เพื่อเปิดมุมมอง **รายการ** เปิดการเดินทาง แล้วเลือกแท็บ **รายการ** ในด้านขวาบนของหัวข้อการเดินทาง

### <a name="information-on-the-voyage-header-fasttab"></a>ข้อมูลบนแท็บด่วนหัวข้อการเดินทาง

แท็บด่วน **หัวข้อการเดินทาง** ในมุมมอง **รายการ** ประกอบด้วยข้อมูลพื้นฐานที่อธิบายการเดินทาง ฟิลด์หลายฟิลด์ที่ปรากฏบนแท็บด่วนนี้จะปรากฏในมุมมอง **ส่วนหัว** ด้วย ตามที่อธิบายไว้ในภายหลังในหัวข้อนี้

### <a name="information-on-the-voyage-lines-fasttab"></a>ข้อมูลบนแท็บด่วนรายการการเดินทาง

แท็บด่วน **รายการการเดินทาง** ในมุมมอง **รายการ** ของการเดินทางเกี่ยวข้องกับรายละเอียดการเดินทางและข้อมูลการคิดต้นทุนที่ใช้ในระดับการเดินทาง ที่นี่คุณสามารถตรวจทานคอนเทนเนอร์ ใบแจ้งรายการ และสินค้าที่แนบกับการเดินทาง ราคาและปริมาณของสินค้าการเดินทางยังแสดงด้วย คุณสามารถดูคอนเทนเนอร์การจัดส่งหรือใบแจ้งรายการใด ๆ ที่แสดงรายการโดยเลือกลิงค์ที่เกี่ยวข้อง

### <a name="information-on-the-line-details-fasttab"></a>ข้อมูลบนแท็บด่วนรายละเอียดรายการ

แท็บด่วน **รายละเอียดรายการ** ในมุมมอง **รายการ** ของการเดินทาง จะให้ข้อมูลเพิ่มเติมเกี่ยวกับบรรทัดที่เลือกในปัจจุบันบนแท็บด่วน **รายการการเดินทาง** โปรดทราบว่าแท็บด่วนนี้จะรวมรายละเอียดเกี่ยวกับตําแหน่งที่แต่ละบรรทัดมีอยู่ในคอนเทนเนอร์ และมีปริมาณที่ประกาศไว้

## <a name="header-view"></a>มุมมองหัวข้อ

เพื่อเปิดมุมมอง **หัวข้อ** เปิดการเดินทาง แล้วเลือกแท็บ **หัวข้อ** ในด้านขวาบนของหัวข้อการเดินทาง

### <a name="settings-on-the-general-fasttab"></a>การตั้งค่าบนแท็บด่วนทั่วไป

ตารางต่อไปนี้อธิบายฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **ทั่วไป** ในมุมมอง **หัวข้อ** ของการเดินทาง

| ฟิลด์ | คำอธิบาย |
|---|---|
| การเดินทาง | ป้อนหมายเลขการเดินทางหรือเที่ยวบิน |
| การอ้างอิงการจอง | ถ้าผู้จัดส่งหรือศูนย์จัดส่งของคุณมีหมายเลขอ้างอิงเพื่อระบุการเดินทาง (นั่นคือการอ้างอิงภายในของผู้จัดส่งหรือศูนย์จัดส่ง) ให้ป้อนไว้ที่นี่ |
| เรือ | ป้อนหรือเลือกเรือ ถ้าเรือไม่ได้แสดงรายการเป็นค่า คุณสามารถป้อนรหัสเรือเป็นข้อความอิสระได้ ในกรณีดังกล่าว ตารางหลักจะไม่อัปเดต เพื่อให้สามารถเลือกรหัสเรือในฟิลด์นี้ได้ในภายหลัง |
| รหัสการเดินทางภายนอก | ป้อนรหัสที่เผยแพร่ต่อการเดินทาง (เช่น หมายเลขเที่ยวบิน) |
| ใบตราส่งสินค้าทางอากาศ/ใบตราส่งหลัก | ป้อนหมายเลขใบส่งสินค้าทางอากาศหลักหรือหมายเลขใบตราส่ง คุณสามารถระบุค่านี้เมื่อคุณจัดส่งทางอากาศ |
| ใบตราส่งสินค้าทางอากาศ/ใบตราส่ง | ป้อนใบตราส่งสินค้าทางอากาศหรือใบตราส่งในคอนเทนเนอร์การจัดส่ง |
| การเช่า | เลือกกล่องกาเครื่องหมายนี้เพื่อบ่งชี้ว่าคอนเทนเนอร์เป็นคอนเทนเนอร์เช่าและต้องส่งคืน |
| คำอธิบาย | ป้อนข้อความอธิบายเกี่ยวกับการเดินทางเพื่อให้รับรู้ได้ง่ายขึ้น |
| สถานะการเดินทาง | สถานะของการเดินทาง ค่า ได้แก่ *สร้างแล้ว* *อยู่ระหว่างส่งต่อ* *ได้รับเอกสารแล้ว* และ *คิดต้นทุน* ฟิลด์นี้จะไม่คํานวณตามตรรกะ มีการอัปเดตผ่านทางการดำเนินการลงรายการบัญชีเท่านั้น มูลค่า *คิดต้นทุน* บ่งชี้ว่าได้รับใบแจ้งหนี้ของสินค้าคงคลังและต้นทุนการเดินทางทั้งหมดแล้ว นอกจากจะได้รับใบแจ้งหนี้แล้ว การเดินทางจะไม่สามารถเข้าถึงสถานะเป็น *คิดต้นทุน* |
| สถานะใบสั่งซื้อ | สถานะสูงสุดที่มาถึงแล้วในใบสั่งซื้อทั้งหมดที่เชื่อมโยงกับการเดินทาง |
| เอกสารที่ได้รับ | ค่าที่ระบุว่าบริษัทของคุณได้รับเอกสารทั้งหมดที่เกี่ยวข้องกับการเดินทางหรือไม่ เมื่อต้องการเปลี่ยนค่า ให้เลือก **เอกสารที่ได้รับ** ในกลุ่ม **การอัปเดตการเดินทาง** บนแท็บ **จัดการ** บนบานหน้าต่างการดำเนินการ |
| บริษัทจัดส่งสินค้า | เลือกบริษัทจัดส่งสินค้าสำหรับการเดินทางนี้ ฟิลด์นี้สามารถใช้เพื่อระบุต้นทุนอัตโนมัติ |
| ชื่อ | ชื่อบริษัทที่เลือกในฟิลด์ **บริษัทจัดส่งสินค้า** |
| การวัด | ฟิลด์นี้เปิดใช้งานการวัดที่จะระบุในต้นทุนอัตโนมัติ การวัดมักใช้โดยองค์กรที่ไม่ทราบปริมาตรหรือน้ำหนักของสินค้าแต่ละรายการ แต่ต้องมีการปรับตามสัดส่วนที่ถูกต้องมากกว่าตัวเลือก จำนวน และ ปริมาณที่มีให้ ผู้ขนส่งสินค้าจะระบุการวัดน้ำหนักหรือลูกบาศก์ และคุณสามารถใส่ในระดับสินค้าหรือระดับใบสั่งซื้อ คุณสามารถอัปเดตพารามิเตอร์โดยอัตโนมัติได้ถ้าเลือกพารามิเตอร์ไว้ หรือสามารถป้อนพารามิเตอร์ด้วยตนเองได้ |
| หน่วยการวัด | หน่วยวัดที่ใช้กับตัวเลขในฟิลด์ **การวัด** |
| จำนวนของแท่นวางสินค้า | ระบุจํานวนของแท่นวางสินค้าในบรรทัดการเดินทาง ฟิลด์นี้จะถูกอัปเดตโดยอัตโนมัติ ถ้าตั้งค่าตัวเลือก **จํานวนกล่องกระดาษที่มีการอัปเดตโดยอัตโนมัติ** เป็น *ใช่* บนแท็บ **ทั่วไป** บนหน้า **พารามิเตอร์ต้นทุนแฝง** |

### <a name="settings-on-the-delivery-fasttab"></a>การตั้งค่าบนแท็บด่วนการจัดส่ง

ตารางต่อไปนี้อธิบายฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **การจัดส่** ในมุมมอง **หัวข้อ** ของการเดินทาง

| ฟิลด์ | คำอธิบาย |
|---|---|
| วันที่และเวลาที่สร้าง | วันที่และเวลาที่สร้างเรกคอร์ดการเดินทาง |
| วันที่ราคาผลิตภัณฑ์หน้าโรงงาน | ฟิลด์นี้จะเลือกวันที่ราคาผลิตภัณฑ์หน้าโรงงานแรกสุดในใบสั่งซื้อที่เชื่อมโยงกับการเดินทาง วันที่ราคาผลิตภัณฑ์หน้าโรงงานพร้อมใช้งานบนส่วนหน้าของใบสั่งซื้อ และมีการอัปเดตโดยใช้ลักษณะการงานการควบคุมการติดตาม |
| วันที่จัดส่ง | วันที่ประเมินว่าเครื่องบินหรือเรือจะออกจากจุดเริ่มต้นเมื่อใด |
| วันที่ออกเดินทาง | วันที่ออกเดินทางจะเป็นวันที่ที่เครื่องบินหรือเรือออกจากท่าเรือทะเล วันที่จัดส่งและวันที่ออกเดินทางไม่ตรงกัน ฟิลด์ **วันที่ออกเดินทาง** ใช้เพื่อให้ข้อมูลเท่านั้น มีการใช้เพื่อประเมินวันที่จัดส่งที่คาดไว้ คุณลักษณะการใช้งานการควบคุมการติดตามใช้เพื่อประเมินวันที่จัดส่งที่คาดไว้ และสามารถตั้งค่าฟิลด์นี้ได้เพื่อให้มีการเติมข้อมูลโดยอัตโนมัติด้วยลักษณะการงานการควบคุมการติดตาม |
| วันที่ส่งเข้าจัดเก็บ | ฟิลด์นี้จะเลือกวันที่แรกสุด เมื่อสินค้าจากใบสั่งซื้อที่ลิงค์ไปยังการเดินทางนั้นจะพร้อมใช้งานเพื่อการขาย |
| วันที่จัดส่งโดยประมาณ | โดยปกติวันที่จัดส่งที่คาดไว้จะเป็นวันที่เมื่อสินค้าครบกําหนดมาถึงในคลังสินค้า ฟิลด์นี้ใช้เพื่อให้ข้อมูลเท่านั้น ไม่ได้ใช้สําหรับการวางแผนหลักกับสินค้า วันที่จัดส่งที่คาดไว้บนรายการใบสั่งซื้อใช้สําเร็จการวางแผนหลักกับสินค้าบนการเดินทาง และถูกอัปเดตโดยใช้คุณลักษณะการควบคุมการติดตาม สามารถตั้งค่าฟิลด์นี้เพื่อให้มีการเติมข้อมูลไว้โดยลักษณะการงานควบคุมการติดตาม |
| วันที่จัดส่งตามจริง | วันที่จัดส่งแรกสุด ตามสินค้าที่เชื่อมโยงกับการเดินทาง |
| ETA ที่ท่าเรือขนส่งสินค้า | ป้อนวันที่ที่คุณคาดว่าการเดินทางจะมาถึงที่ท่าเรือการจัดส่ง ตามข้อมูลที่ตัวแทนจัดส่งให้ |
| ได้รับเอกสารต้นฉบับแล้ว | ป้อนวันที่ที่ได้รับเอกสารต้นฉบับ |
| นายหน้าแนะนำแล้ว | ป้อนวันที่ที่นายหน้าแนะนำ |
| ส่งใบตราส่งต้นฉบับแล้ว | ป้อนวันที่ที่ส่งใบตราส่งต้นฉบับ |
| นำสินค้าออกใช้แล้ว | ป้อนวันที่ที่สินค้าถูกปล่อยออกจากศุลกากร |
| การนัดหมายลูกค้า | ป้อนวันที่นัดหมายของลูกค้า ซึ่งเป็นวันที่ที่คุณคาดว่าลูกค้าจะเป็นเจ้าของสินค้า |
| จัดส่งสินค้าที่คลังสินค้าแล้ว | ป้อนวันที่ที่จัดส่งสินค้าไปยังคลังสินค้า |
| วันที่ตรวจสอบความถูกต้อง | ป้อนวันที่ตรวจสอบความถูกต้อง |
| คำแนะนำในการจัดส่ง | ป้อนวันที่ที่ได้รับคำแนะนำในการจัดส่ง |
| วิธีการนำส่ง | ฟิลด์นี้จะแสดงข้อมูลเกี่ยวกับวิธีการที่ใช้ในการจัดส่งการเดินทาง และการเลือกต้นทุนของต้นทุนอัตโนมัติที่สัมพันธ์กับวิธีการจัดส่งนั้น |
| จากท่าเรือ | ท่าเรือการจัดส่งที่เดินทางมา |

| ผู้ขนส่งสินค้าในท้องถิ่น | ฟิลด์นี้ใช้เพื่อให้ข้อมูลเท่านั้น ควรเลือกผู้ส่งต่อเฉพาะที่จากตารางผู้จัดจำหน่าย |
| วันที่ของการขนส่งท้องถิ่น | วันที่ที่จองการขนส่งเฉพาะที่ ฟิลด์นี้สามารถช่วยคลังสินค้าได้ในการวางแผน |
| เวลาของการขนส่งท้องถิ่น | ช่วงเวลาที่จองการขนส่งเฉพาะที่ ฟิลด์นี้สามารถช่วยคลังสินค้าได้ในการวางแผน |
| เท็มเพลตการเดินทาง | แม่แบบการเดินทางที่ระบุเพื่อการเดินทาง แม่แบบการเดินทางจะใช้เพื่อป้อนท่าเรือ "สุดท้าย" และท่าเรือ "เริ่มต้น" ของคอนเทนเนอร์การจัดส่งและการเดินทาง ค่าเหล่านั้นจะช่วยระบุต้นทุนอัตโนมัติที่เชื่อมโยงกับการเดินทาง |

### <a name="settings-on-the-other-fasttab"></a>การตั้งค่าบนแท็บด่วนอื่น

ตารางต่อไปนี้อธิบายฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **อื่น** ในมุมมอง **หัวข้อ** ของการเดินทาง

| ฟิลด์ | คำอธิบาย |
|---|---|
| หมายเหตุ | ป้อนข้อมูลเพิ่มเติมที่เกี่ยวข้องกับการเดินทาง |
| วันที่ประเมินค่า | เลือกวันที่ราคาผลิตภัณฑ์หน้าโรงงานแรกสุดในใบสั่งซื้อที่เชื่อมโยงกับการเดินทาง |
| การเดินทางที่ค้างอยู่ | คุณสามารถใช้สิ่งนี้เพื่อบ่งชี้ว่าการจัดส่งสินค้าหรือการเดินทางของคุณออกเดินทางแล้วหรือไม่ ใช้เพื่อให้ข้อมูลเท่านั้น  |
| การเปลี่ยนชื่อคอนเทนเนอร์การจัดส่ง | สำหรับการเดินทางที่มีคอนเทนเนอร์มากกว่าหนึ่งคอนเทนเนอร์ จํานวนของคอนเทนเนอร์ที่ยังไม่ได้รับ |

## <a name="voyage-update-periodic-tasks"></a>งานประจำงวดการอัปเดตการเดินทาง

โมดูล **ต้นทุนแฝง** รวมงานประจำงวดต่าง ๆ ที่เกี่ยวข้องกับการเดินทาง ซึ่งสามารถอัปเดตการเดินทางได้หลายด้าน เมื่อต้องการรันหรือจัดตารางการผลิตงานช่วงเวลาเหล่านี้ ให้ไปที่ **ต้นทุนแฝง \> งานประจำงวด \> การอัปเดตการเดินทาง** และเลือกชนิดงานอย่างใดอย่างหนึ่งต่อไปนี้

- **เอกสารที่ได้รับ** – งานนี้ช่วยให้คุณสามารถตั้งค่า **เอกสารที่ได้รับ** เป็น *ใช่* เพื่อการเดินทางหลายที่ในเวลาเดียวกัน ใช้การตั้งค่า **ตัวกรอง** เพื่อกําหนดชุดการเดินทางที่คุณต้องการอัปเดต
- **อยู่ระหว่างส่งต่อ** – งานนี้ช่วยให้คุณตั้งค่าฟิลด์ **สถานะการเดินทาง** เป็นสถานะการส่งต่อที่สร้างขึ้นในหน้า **[พารามิเตอร์ต้นทุนแฝง](landed-cost-parameters.md)** ให้กับการเดินทางต่าง ๆ พร้อมกัน ใช้การตั้งค่า **ตัวกรอง** เพื่อกําหนดชุดการเดินทางที่คุณต้องการอัปเดต
- **พร้อมสำหรับคิดต้นทุน** – งานนี้ช่วยให้คุณตั้งค่าฟิลด์ **สถานะการเดินทาง** เป็นสถานะพร้อมสำหรับคิดต้นทุนที่สร้างขึ้นในหน้า **[พารามิเตอร์ต้นทุนแฝง](landed-cost-parameters.md)** ให้กับการเดินทางต่าง ๆ พร้อมกัน โดยทั่วไป คุณจะตั้งค่าให้งานนี้รันในการจัดตารางการผลิตตามปกติ
- **คิดต้นทุน** – งานนี้ช่วยให้คุณสามารถตั้งค่าฟิลด์ **สถานะการเดินทาง** เป็นสถานะการคิดต้นทุนซึ่งสร้างขึ้นในหน้า **[พารามิเตอร์ต้นทุนแฝง](landed-cost-parameters.md)** ให้กับการเดินทางต่าง ๆ พร้อมกัน ได้ หากได้คิดต้นทุนแล้วแต่ยังไม่ได้อัพเดตการเดินทาง โดยทั่วไป คุณจะตั้งค่าให้งานนี้รันในการจัดตารางการผลิตตามปกติ
