# <a name="landed-cost-parameters-setup"></a>การตั้งค่าพารามิเตอร์ต้นทุนแฝง

[!include [banner](../../includes/banner.md)]

คุณใช้หน้า **พารามิเตอร์ต้นทุนแฝง** เพื่อตั้งค่าข้อมูลทั่วไปและการตั้งค่าการตั้งค่าคอนฟิกที่ใช้ระหว่างโมดูล **ต้นทุนแฝง** สำหรับการลงรายการบัญชี การอัปเดตสถานะ ลำดับหมายเลข และลักษณะการทำงาน มีการใช้การตั้งค่าพารามิเตอร์ร่วมกันระหว่างนิติบุคคล และสามารถแก้ไขได้โดยผู้ดูแลระบบ

## <a name="open-the-landed-cost-parameters-page"></a>เปิดหน้าพารามิเตอร์ต้นทุนแฝง

เมื่อต้องการใช้งานพารามิเตอร์ ให้ไปที่ **ต้นทุนแฝง \> การตั้งค่า \> พารามิเตอร์ต้นทุนแฝง** จากนั้นตั้งค่าฟิลด์ตามที่อธิบายไว้ในส่วนย่อยต่อไปนี้

![หน้าพารามิเตอร์ต้นทุนแฝง](media/landed-cost-parameters.png "หน้าพารามิเตอร์ต้นทุนแฝง")

## <a name="general-tab"></a>แท็บทั่วไป

### <a name="general-fasttab"></a>FastTab ทั่วไป

ตารางต่อไปนี้จะอธิบายถึงฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **ทั่วไป** ของแท็บ **ทั่วไป** ของหน้า **พารามิเตอร์ต้นทุนแฝง**

| การตั้งค่า | คำอธิบาย |
|---|---|
| ใช้อัตราการจัดส่ง | อัตราค่าจัดส่งจะกําหนดไว้ให้กับรอบระยะเวลาที่กําหนด และใช้ในการประเมินต้นทุนของสินค้าที่ใช้สกุลเงินต่างๆ ตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่อใช้อัตราค่าจัดส่ง |
| ชนิดอัตราแลกเปลี่ยน | ชุดอัตราแลกเปลี่ยนเริ่มต้นที่ใช้ในการคํานวณหลายสกุลเงิน สำหรับการเดินทางและต้นทุนการเดินทาง |
| อัพเดตปริมาณใบสั่งซื้อ | เลือกสิ่งที่จะเกิดขึ้น หากผู้ใช้เปลี่ยนปริมาณในบรรทัดใบสั่งซื้อ:<ul><li>**ยอมรับ** – ปริมาณการเดินทางจะถูกปรับปรุงโดยอัตโนมัติ</li><li>**คําเตือน** – ถ้าบรรทัดแนบอยู่กับการเดินทาง จะมีคําเตือนแสดงขึ้น แต่จะมีการอัปเดตปริมาณการเดินทาง</li><li>**ข้อผิดพลาด** – ถ้าบรรทัดถูกแนบไปกับการเดินทาง จะมีข้อความแสดงข้อผิดพลาดแสดงขึ้น และไม่สามารถอัปเดตใบสั่งซื้อได้ ดังนั้น บรรทัดใบสั่งต้องถูกลบออกจากการเดินทางก่อน</li></ul> |
| อัพเดตจำนวนกล่องโดยอัตโนมัติ | เมื่อตั้งค่าตัวเลือกนี้เป็น *ใช่* กล่องกระดาษทั้งหมดจะถูกเพิ่มเข้าด้วยกัน และแสดงในระดับการเดินทางและคอนเทนเนอร์ เมื่อตั้งค่าเป็น *ไม่* จํานวนกล่องกระดาษจะถูกตั้งค่าเป็น 0 (ศูนย์) เริ่มแรก และสามารถแก้ไขได้ด้วยตนเองตามต้องการ |

### <a name="costing-fasttab"></a>แท็บด่วน การคิดต้นทุน

ตารางต่อไปนี้จะอธิบายถึงฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **การคิดต้นทุน** ของแท็บ **ทั่วไป** ของหน้า **พารามิเตอร์ต้นทุนแฝง**

| การตั้งค่า | คำอธิบาย |
|---|---|
| ข้อมูลจำเพาะเกี่ยวกับการลงรายการบัญชี | เลือกการปรับปรุงค่าในบัญชีแยกประเภท:<ul><li>**ยอดรวม** – จะมีการลงรายการบัญชีตัวเลขผลรวมในบัญชีแยกประเภท</li><li>**กลุ่มสินค้า** – การปรับปรุงถูกระบุสำหรับแต่ละกลุ่มสินค้า</li><li>**หมายเลชสินค้า** – การปรับปรุงถูกระบุสำหรับแต่ละสินค้า ค่านี้แสดงรายละเอียดมากที่สุด</li></ul> |
| อนุญาตให้มีต้นทุนเป็นศูนย์ | ตั้งค่าตัวเลือกนี้เป็น *ไม่* เพื่อแสดงข้อผิดพลาด หากผู้ใช้พยายามลงรายการบัญชีการประเมินต้นทุนในใบแจ้งหนี้การเดินทางหรือใบสั่งซื้อที่ไม่รวมค่าสำหรับต้นทุนการเดินทางที่คาดไว้ ข้อความแสดงข้อผิดพลาดระบุว่าไม่สามารถปันส่วนต้นทุน 0 (ศูนย์) ได้ และการลงรายการบัญชีใบแจ้งหนี้จะล้มเหลว ในกรณีนี้ ผู้ใช้สามารถอัแเดตการประเมินด้วยตนเอง (หรือตั้งค่าคอนฟิกต้นทุนอัตโนมัติใหม่) แล้วดึงในค่าที่อัปเดตหรือลบต้นทุนถ้าไม่มีอยู่<p>ตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่ออนุญาตให้ต้นทุนการเดินทางว่างเปล่าได้ ในกรณีนี้ ราคา 0 (ศูนย์) จะมีการปันส่วนตามพื้นที่ต้นทุน จากนั้นคุณสามารถประมวลผลใบแจ้งหนี้ต้นทุนให้แก่ผู้ขนส่ง ตามต้นทุนราคาศูนย์เมื่อได้รับการเดินทาง</p><p>ขอแนะนาว่าคุณควรตั้งค่าคอนฟิกการประเมินบนเรกคอร์ดต้นทุนอัตโนมัติ เพื่อป้องกันไม่ให้มีต้นทุนราคาเป็นศูนย์ปรากฏ แม้ว่าการประเมินนี้ไม่ถูกต้องทั้งหมด แต่การประเมินนี้ควรถูกต้องมากกว่าต้นทุนศูนย์ที่สมมติให้ถูกต้อง</p> |
| วันที่ลงรายการบัญชีการปรับปรุง | เมื่อคุณลงรายการบัญชีใบแจ้งหนี้ต้นทุนการเดินทางของบัญชีเจ้าหนี้ ตารางรายการการชำระบัญชี (การปรับปรุงสินค้าคงคลัง) จะได้รับการอัปเดตด้วย เลือกวันที่ที่จะตั้งค่าตามค่าเริ่มต้น ในหน้า **เลือกต้นทุนการเดินทาง** ขณะที่คุณอยู่ในสมุดรายวันใบแจ้งหนี้:<ul><li>**วันที่ธุรกรรม** – ใช้วันที่ของสมุดรายวัน (วันที่ลงรายการบัญชี)</li><li>**วันที่ในใบแจ้งหนี้ของใบสั่งซื้อ** – ใช้วันที่ลงรายการบัญชีทางการเงินของใบแจ้งหนี้สินค้าคงคลัง (ใบสั่งซื้อ)</li><li>**วันที่ที่เลือก** – ผู้ใช้สามารถระบุวันที่ลงรายการบัญชีได้ แม้ว่าวันที่จะสามารถเว้นว่างไว้ได้ หากวันที่ว่างเปล่าเมื่อลงรายการบัญชีใบแจ้งหนี้ต้นทุน ผู้ใช้จะได้รับข้อผิดพลาด</li></ul> |
| ใช้ใบสำคัญแสดงใบแจ้งหนี้การซื้อ | เมื่อตั้งค่าตัวเลือกนี้เป็น *ใช่* ธุรกรรมการรับรู้ต้นทุนจะใช้หมายเลขใบสำคัญเดียวกันที่ใช้กับใบแจ้งหนี้การซื้อ เมื่อตั้งค่าเป็น *ไม่* ระบบจะใช้หมายเลขที่พร้อมใช้งานถัดไป สำหรับลำดับหมายเลข **ใบสำคัยการรับรู้ต้นทุน** ที่ใช้ตั้งค่าบนแท็บ **ลำดับหมายเลข** ของหน้า **พารามิเตอร์ต้นทุนแฝง**<p>ตัวเลขนี้มีผลกระทบเมื่อมีการตั้งค่าตัวเลือก **ลงรายการบัญชีในบัญชีแยกประเภท** เป็น *ใช่* บนแท็บ **ใบแจ้งหนี้** ของหน้า **พารามิเตอร์บัญชีเจ้าหนี้**</p> |
| บล็อกการลงรายการบัญชีในบัญชีรอการโอนด้วยตนเอง | ตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่อป้องกันการลงรายการบัญชีไปยังบัญชีรอตัดบัญชี ซึ่งธุรกรรมไม่ได้เชื่อมโยงกับการเดินทาง โดยการเลือก **ฟังก์ชัน \> เลือกต้นทุนการเดินทาง** ในบานหน้าต่างการดำเนินการของสมุดรายวันใบแจ้งหนี้ผู้จัดจำหน่าย เราขอแนะนาให้คุณตั้งค่าตัวเลือกนี้เป็น *ใช่* เพื่อให้บัญชีการเดินทางและบัญชีรอการโอนได้รับการชำระเงินอย่างถูกต้อง |
| ใช้บัญชีค่าธรรมเนียมคงค้างสำหรับชนิดต้นทุน | เมื่อตั้งค่าตัวเลือกนี้เป็น *ใช่* บัญชีการรับรู้ค่าธรรมเนียมที่ตั้งค่าคอนฟิกไว้ให้กับรหัสชนิดต้นทุนที่เกี่ยวข้อง ในหน้า **รหัสชนิดต้นทุน** จะถูกใช้เพื่อรับรู้ต้นทุนเป็นค่าใช้จ่าย<p>ตัวเลขนี้มีผลกระทบเมื่อมีการตั้งค่าตัวเลือก **ลงรายการบัญชีในบัญชีแยกประเภท** เป็น *ใช่* บนแท็บ **ใบแจ้งหนี้** ของหน้า **พารามิเตอร์บัญชีเจ้าหนี้** |
| ลงรายการบัญชีการปรับปรุงเป็นผลต่าง | เมื่อตั้งค่าตัวเลือกนี้เป็น *ใช่* จะแทนที่ฟังก์ชันมาตรฐาน และบังคับให้รายการความเคลื่อนไหวของการปรับปรุงสินค้าคงคลังที่เกี่ยวข้องกับผลต่างระหว่างต้นทุนที่ประเมินและต้นทุนจริงที่จะลงรายการบัญชีที่บัญชีผลต่าง<p>เมื่อตั้งค่าตัวเลือกนี้เป็น *ไม่* รายการความเคลื่อนไหวของการปรับปรุงสินค้าคงคลังที่เกี่ยวข้องกับผลต่างจะถูกจัดการตามการตั้งค่าคอนฟิกของวิธีการ คิดต้นทุนและรหัสชนิดต้นทุน สำหรับต้นทุนมาตรฐาน ผลต่างจะยังคงมีการลงรายการบัญชีในบัญชีผลต่าง สำหรับค่าเฉลี่ยถ่วงน้ำหนัก ผลต่างจะถูกลงรายการบัญชีในบัญชีผลต่างหรือในสินค้าคงคลัง</p><p>ตัวเลขนี้มีผลกระทบเมื่อมีการตั้งค่าตัวเลือก **ลงรายการบัญชีในบัญชีแยกประเภท** เป็น *ใช่* บนแท็บ **ใบแจ้งหนี้** ของหน้า **พารามิเตอร์บัญชีเจ้าหนี้**</p> |

### <a name="validation-fasttab"></a>แท็บด่วนการตรวจสอบความถูกต้อง

ตารางต่อไปนี้จะอธิบายถึงฟิลด์ที่พร้อมใช้งานบนแท็บด่วน **การตรวจสอบความถูกต้อง** ของแท็บ **ทั่วไป** ของหน้า **พารามิเตอร์ต้นทุนแฝง**

| การตั้งค่า | คำอธิบาย |
|---|---|
| ใบแจ้งหนี้ต้นทุนหลายใบ | เลือกสิ่งที่จะเกิดขึ้น หากมีการประมวลผลใบแจ้งหนี้มากกว่าหนึ่งรายการโดยเทียบกับการเดินทาง ใบแจ้งรายการ หรือคอนเทนเนอร์ ที่มีค่าธรรมเนียมเบ็ดเตล็ดเหมือนกัน<ul><li>**ยอมรับ** – ระบบควรอนุญาตใบแจ้งหนี้ต้นทุนหลายรายการ</li><li>**คําเตือน** – แสดงคําเตือน</li><li>**ข้อผิดพลาด** - ข้อผิดพลาดจะแสดงขึ้น</li></ul> |
| ผู้จัดจำหน่ายหลายรายต่อใบแจ้งรายการ | เลือกสิ่งที่จะเกิดขึ้น หากมีการเพิ่มใบสั่งซื้อของผู้จัดจำหน่ายมากกว่าหนึ่งรายการลงในใบแจ้งรายการ<ul><li>**ยอมรับ** - ระบบควรอนุญาตให้มีการดำเนินการ</li><li>**คําเตือน** – แสดงคําเตือน แต่ยังคงสามารถดำเนินการได้</li><li>**ข้อผิดพลาด** – จะมีข้อความแสดงข้อผิดพลาดแสดงขึ้น และป้องกันการดำเนินการ</li></ul><p>นายหน้าศุลกากรหรือกฎหมายท้องถิ่นของคุณอาจมีข้อบังคับเกี่ยวกับค่าเฉพาะสำหรับฟิลด์นี้</p> |

|---|---|
| ชื่อสมุดรายวัน | เลือกสมุดรายวันเริ่มต้นที่ฟังก์ชัน *สร้างสมุดรายวันการมาถึง* ที่ควรจะใช้ |
| สถานะการเดินทาง | เลือกสถานะที่การเดินทางต้องมีก่อนที่ผู้ใช้จะสามารถตั้งค่าคอนเทนเนอร์จัดส่งการเช่าในระบบได้ โดยทั่วไป การเกิดขึ้นนี้จะเกิดขึ้นเมื่อสินค้าอยู่ระหว่างการส่งต่อหรือที่ท่าสินค้า |

| การเคลื่อนย้าย | ถ้าปริมาณที่ มากเกิน/ต่ำกว่า ปริมาณที่สั่งของการจัดส่งอยู่ภายในค่าเผื่อที่กําหนด จะมีการประมวลผลสมุดรายวันการเคลื่อนย้ายโดยอัตโนมัติ เลือกสมุดรายวันการเคลื่อนย้ายเริ่มต้นที่จะใช้ ฟิลด์ **บัญชีตรงข้าม** เกี่ยวกับชื่อสมุดรายวันการเคลื่อนย้ายที่เลือกต้องมีค่า |

## <a name="status-updates-tab"></a>แท็่บการอัปเดตสถานะ

ระบบใช้ค่าสถานะเพื่อบ่งชี้สถานะของการเดินทางแต่ละรายการ ค่าสถานะการเดินทางสามารถใช้กับการเดินทางผ่านทางการติดตามการเดินทางหรือชุดงานประจำงวดโดยอัตโนมัติได้ หรือคุณสามารถนำไปใช้ด้วยตนเองได้โดยการเปิดการเดินทาง แล้วเลือกสถานะในกลุ่ม **การอัปเดตการเดินทาง** บนแท็บ **จัดการ** ของบานหน้าต่างการดำเนินการ 

คุณสามารถสร้างค่าสถานะการเดินทางได้หลายค่าตามต้องการ อย่างไรก็ตาม ต้องกําหนดสี่อย่างเกี่ยวกับการใช้วัตถุประสงค์พิเศษบนแท็บ **การอัปเดตสถานะ** ของหน้า **พารามิเตอร์ต้นทุนแฝง** ตารางต่อไปนี้อธิบายถึงฟิลด์ที่มีอยู่ที่นั่น

| การตั้งค่า | คำอธิบาย |
|---|---|
| คิดต้นทุน | เลือกสถานะการเดินทางที่ระบุว่าการเดินทางมีการสิ้นสุดแล้ว |
| อยู่ระหว่างขนส่ง | เลือกสถานะการเดินทางที่ระบุว่าการเดินทางอยู่ระหว่างส่งต่อ |
| พร้อมสำหรับการคิดต้นทุน | เลือกสถานะการเดินทางที่ระบุว่าการเดินทางพร้อมสำหรับการคิดต้นทุน สถานะนี้จะใช้เมื่อใบแจ้งหนี้การซื้อสินค้าคงคลังและใบแจ้งหนี้ต้นทุนการเดินทางทั้งหมดที่มีการตั้งค่าฟิลด์ **เครดิตในต้นทุนการเดินทาง** เป็น *ผู้จัดจำหน่าย* ได้รับการประมวลผลสำหรับการเดินทาง การเดินทางที่กระบวนการคิดต้นทุนไม่สำเร็จ จะมีสถานะเป็น *พร้อมสำหรับการคิดต้นทุน*|
| คิดต้นทุนล่วงหน้า | เลือกสถานะการเดินทางที่ระบุว่าการเดินทางกำลังคิดต้นทุนล่วงหน้า สถานะนี้จะใช้เมื่อมีการเพิ่มธุรกรรมต้นทุนใหม่ให้กับการเดินทางหลังจากที่ได้คิดต้นทุนแล้ว ธุรกรรมต้นทุนใหม่อาจเพิ่มลงในการเดินทางที่คิดต้นทุนไว้ก่อนหน้านี้ เมื่อได้รับใบแจ้งหนี้ค่าขนส่งที่สองหรือค่าธรรมเนียมเบ็ดเตล็ดที่ไม่คาดคิด สถานะนี้จะมีการใช้โดยอัตโนมัติ เมื่อต้นทุนการเดินทางใหม่ถูกเพิ่มลงในการเดินทางที่คิดต้นทุนแล้ว |

## <a name="voyage-creator-tab"></a>แท็บผู้สร้างการเดินทาง

ตารางต่อไปนี้จะอธิบายส่วนต่างๆ บนแท็บ **ผู้สร้างการเดินทาง** ของหน้า **พารามิเตอร์ต้นทุนแฝง**

| ส่วน | คำอธิบาย |
|---|---|
| การยอมรับ | ฟิลด์ **ค่าเผื่อของปริมาตรภายนอก** และ **ค่าเผื่อน้ําหนักภายนอก** จะกําหนดขีดควบคุมซึ่งสูงกว่าปริมาณสินค้าซึ่งถือว่าปริมาณเกินและน้ําหนักเกิน เมื่อผู้ใช้เพิ่มสินค้าในหน้า **โปรแกรมแก้ไขการเดินทาง** หากปริมาตรหรือน้ําหนักเกินกว่าค่าที่คุณตั้งค่าไว้ที่นี่ ระบบจะแสดงคําเตือน ค่าของแต่ละฟิลด์จะแสดงเป็นเปอร์เซ็นต์ของปริมาตรหรือน้ําหนักสูงสุดที่ตั้งค่าไว้ให้กับชนิดคอนเทนเนอร์จัดส่งที่เกี่ยวข้อง ขอแนะนำค่าระหว่าง 5 ถึง 10 เปอร์เซ็นต์ของปริมาตรหรือน้ําหนักสูงสุด |
| การตั้งค่าการสร้างใบแจ้งรายการ | ระบบสามารถสร้างใบแจ้งรายการได้หลายใบในระหว่างกระบวนการสร้างการเดินทาง ใช้ส่วนนี้เพื่อกําหนดว่าควรจะสร้างใบแจ้งรายการใหม่เมื่อใด สำหรับแต่ละแถวในส่วนนี้ ระบบจะตรวจสอบตารางและฟิลด์ที่ระบุ และจะสร้างใบแจ้งรายการให้กับแต่ละค่าฟิลด์เฉพาะ |

## <a name="cost-estimates-tab"></a>แท็บการประเมินต้นทุน

แท็บ **การประเมินต้นทุน** ของหน้า **พารามิเตอร์ต้นทุนแฝง** จะมีเพียงฟิลด์เดียว: **เวอร์ชันการคิดต้นทุนเริ่มต้น** ฟิลด์นี้จะใช้เฉพาะเมื่อวิธีการคิดต้นทุนคือ *การคิดต้นทุนมาตรฐาน* เลือกเวอร์ชันการคิดต้นทุนเริ่มต้นที่จะใช้กับงานประจำงวด *การอัปเดตราคาต้นทุนสินค้า* คุณอาจต้องเปลี่ยนแปลงการตั้งค่านี้ทุกครั้งที่ปีทางการเงินใหม่เริ่มต้น

## <a name="inventory-dimensions-tab"></a>แท็บมิติสินค้าคงคลัง

คุณใช้แท็บ **มิติสินค้าคงคลัง** ของหน้า **พารามิเตอร์ต้นทุนแฝง** เพื่อควบคุมว่าควรจะแสดงมิติสินค้าคงคลังที่พร้อมใช้งานใดโดยค่าเริ่มต้น บนหน้าต้นทุนแฝงแต่ละหน้าที่มีการใช้มิติ

เลือกมิติ แล้วตั้งค่าตัวเลือก **บรรทัดการเดินทาง** **สินค้าที่อยู่ระหว่างส่งต่อ** และ/หรือ **การประเมินต้นทุน** เป็น *ใช่* ให้กับแต่ละหน้า ซึ่งควรจะแสดงมิติโดยค่าเริ่มต้น ทําขั้นตอนนี้ซ้ำสำหรับมิติอื่น ตามที่ต้องการ

การตั้งค่าบนแท็บนี้จะสร้างมิติเริ่มต้นให้กับแต่ละหน้า ที่กําหนดระหว่างนิติบุคคล อย่างไรก็ตาม ผู้ใช้ที่เปิดใช้งานหน้าใดหน้าหนึ่งที่ได้รับการระบุสามารถแทนที่มิติเริ่มต้น โดยการเลือกมิติ **สินค้าคงคลัง \> แสดงมิติ** ในบานหน้าต่างการดำเนินการ

## <a name="number-sequences-tab"></a>แท็บลำดับหมายเลข

แท็บ **ลำดับหมายเลข** ของหน้า **พารามิเตอร์ต้นทุนแฝง** แสดงรายการแต่ละชนิดของลำดับหมายเลขการอ้างอิงที่ต้องใช้ต้นทุนแฝง แต่ไม่มี การใช้ร่วมกันระหว่างนิติบุคคล สำหรับแต่ละข้อมูลอ้างอิงในรายการ ให้เลือกรหัสลำดับหมายเลข


## <a name="shared-number-sequences-tab"></a>แท็บลำดับหมายเลขที่ใช้ร่วมกัน

แท็บ **ลำดับหมายเลขที่ใช้ร่วมกัน** ของหน้า **พารามิเตอร์ต้นทุนแฝง** แสดงรายการแต่ละชนิดของลำดับหมายเลขการอ้างอิงที่ใช้ร่วมกันระหว่างนิติบุคคลสำหรับต้นทุนแฝง ปัจจุบัน มีลำดับหมายเลขเพียงลำดับเดียวในรายการ ใช้ลำดับหมายเลขนี้กับรหัสการเดินทาง


## <a name="feature-visibility-tab"></a>แท็บการแสดงผลคุณลักษณะ

ต้นทุนแฝงจะเพิ่มคุณลักษณะ (ฟิลด์และฟังก์ชัน) ไปยังหน้าที่ใช้บ่อยหลายหน้าใน Microsoft Dynamics 365 Supply Chain Management หน้าเหล่านี้รวมหน้าที่เกี่ยวข้องกับข้อมูลหลักของผู้จัดจำหน่าย ผลิตภัณฑ์ที่ออกใช้ ใบสั่งซื้อ ใบสั่งโอนย้าย และการตั้งค่าคลังสินค้า ถ้าคุณใช้งานต้นทุนแฝง คุณต้องทำให้คุณลักษณะเหล่านั้นแสดงผลได้ทุกแห่ง ก่อนที่คุณใช้ประโยชน์จากคุณลักษณะเหล่านั้น ถ้าคุณไม่ได้ใช้ต้นทุนแฝง คุณสามารถซ่อนคุณลักษณะนั้นเพื่อเก็บออกไปให้พ้น

บนแท็บ **การมองเห็นได้ของคุณลักษณะ** ของหน้า **พารามิเตอร์ต้นทุนแฝง** ให้ตั้งค่าตัวเลือก **เรียกใช้** เป็น *ใช่* เพื่อให้คุณลักษณะต้นทุนแฝงสามารถมองเห็นได้ในที่ที่สามารถใช้ได้ ตั้งค่าเป็น *ไม่* เพื่อซ่อนคุณลักษณะบนหน้าทั่วไปภายนอกต้นทุนแฝง อย่างไรก็ตาม แม้ว่าตัวเลือกจะถูกตั้งค่าเป็น *ไม่* ตัวโมดูลเอง รวมถึงหน้า **พารามิเตอร์ต้นทุนแฝง** จะยังคงพร้อมใช้งานแก่ผู้ใช้ที่มีสิทธิ์ที่ถูกต้องในการเข้าถึง
