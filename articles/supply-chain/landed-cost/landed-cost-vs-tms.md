# <a name="landed-cost-vs-transportation-management"></a>ต้นทุนแฝงเทียบกับการจัดการการขนส่ง

[!include [banner](../../includes/banner.md)]

Microsoft Dynamics 365 Supply Chain Management มีโมดูลที่แตกต่างกันสองโมดูลที่ใช้กับการขนส่ง: **การจัดการการขนส่ง (TMS)** และ **ต้นทุนแฝง** หัวข้อนี้สรุปฟังก์ชันที่โมดูลทั้งสองมีโดยทั่วไป และเน้นความแตกต่างระหว่างโมดูลทั้งสอง คุณสามารถใช้ข้อมูลนี้เพื่อตัดสินว่าโมดูลใดที่ควรใช้กับแนวทางที่พึงปฏิบัติในธุรกิจของคุณ คุณอาจพบว่าแนวทางทางธุรกิจบางอย่างจะพัฒนาขึ้นได้ดีขึ้นกับ TMS ขณะที่แนวทางที่พึงปฏิบัติอื่นๆ จะดีที่สุดกับต้นทุนแฝง ขึ้นอยู่กับความต้องการทางธุรกิจของคุณ คุณอาจเลือกใช้โมดูลเพียงโมดูลเดียวโดยเฉพาะ หรือคุณอาจรวมสองโมดูลเข้ากัน

หัวข้อนี้ไม่ใช่การตรวจทานคุณลักษณะทั้งหมดของโมดูลใดโมดูลหนึ่งโดยครอบคลุม แต่ฟังก์ชันนี้จะเน้นที่ฟังก์ชันที่พร้อมใช้งาน เนื่องจากการขนส่งสินค้าจากผู้จัดจำหน่ายไปยังคลังสินค้าของธุรกิจของคุณ ซึ่งสามารถใช้สินค้าได้

## <a name="terminology-reference-data-and-reporting-differences"></a>คำศัพท์ ข้อมูลอ้างอิง และความแตกต่างของการรายงาน

### <a name="terminology-comparison"></a>การเปรียบเทียบคำศัพท์

TMS และต้นทุนแฝงใช้เงื่อนไขที่แตกต่างกันในแนวคิดที่คล้ายกัน ตารางต่อไปนี้สรุปเงื่อนไขเหล่านี้และการใช้งานในสองโมดูล

| เงื่อนไข TMS | เงื่อนไขต้นทุนแฝง | บันทึกย่อ |
|----------|------------------|-------|
| **จำนวนงานในศูนย์การผลิต**<p>*โหลด* คือชุดของการจัดส่งสินค้าที่มีการขนส่งพร้อมกันในหน่วยการขนส่งเดียวกัน (เช่น รถบรรทุกหรือเรือ) การบรรทุกอาจเป็นขาเข้าหรือขาออก</p> | **การเดินทาง**<p>โดยทั่วไป *การเดินทาง* เป็นเรือใบลำเดียวการเดินทางไปพร้อมกับ *การเดินทาง* เดียว การเดินทางอาจประกอบด้วย *คอนเทนเนอร์การขนส่ง* ซึ่งรวมใบสั่งขาเข้าจากนิติบุคคลต่าง ๆ ขององค์กรของคุณ การเดินทางสามารถเป็นขาเข้าเท่านั้น</p> | TMS ยังใช้ *หมายเลขการเดินทาง* ของเงื่อนไขซึ่งอ้างอิงถึงฟิลด์ข้อมูลที่คุณสามารถป้อนตัวระบุได้ อย่างไรก็ตาม ไม่มีฟังก์ชันที่เชื่อมโยงกับฟิลด์ TMS และไม่เกี่ยวข้องกับแนวคิด *การเดินทาง* ในต้นทุนแฝง |
| **กระบวนการผลิต**<p>*กระบวนการผลิต* คือพาธจริงของการขนส่งจากจุดเริ่มต้นไปยังปลายทาง</p> | **การเดินทาง**<p>*การเดินทาง* คือพาธจริงของการขนส่งจากจุดเริ่มต้นไปยังปลายทาง การเดินทางแต่ละครั้งต้องถูกมอบหมายให้กับการเดินทางหนึ่งๆ</p> | |
| **เซกเมนต์กระบวนการผลิต**<p>กระบวนการผลิตหนึ่ง ๆ สามารถมี *เซกเมนต์กระบวนการผลิต* ได้หลายเซ็กเมนต์ ซึ่งแต่ละเซ็กเมนต์แสดงถึงขั้นตอนของการเดินทาง กระบวนการผลิตอาจประกอบด้วยผู้ขนส่งหรือบริการหลายราย ซึ่งแต่ละสายงานจะถือเป็นเซกเมนต์กระบวนการผลิต</p> | **รอบ**<p>แต่ละการเดินทางสามารถมีได้หลาย *ขา* ซึ่งแต่ละขาแสดงถึงขั้นตอนของการเดินทาง ขาอาจเป็นส่วนหนึ่งของการขนส่ง การจัดการภาษี หรือกิจกรรมอื่น</p> | |
| **ตู้บรรจุสินค้า**<p>*คอนเทนเนอร์* อาจเป็นบรรจุภัณฑ์ชนิดใด ๆ หรือบรรจุภัณฑ์ที่ใช้โดย TMS</p> | **คอนเทนเนอร์การจัดส่ง**<p>*คอนเทนเนอร์การขนส่ง* คือคอนเทนเนอร์การขนส่งทางกายภาพตัวอักษร และเรียกอีกอย่างว่าจากการจัดส่งคอนเทนเนอร์</p> | |
| **รหัสโภคภัณฑ์**<p>ใน TMS (และในที่อื่นๆ Supply Chain Management) *รหัสโภคภัณฑ์* จะระบุชนิดของชุดสินค้า ซึ่งอาจส่งผลต่อภาษี การจัดการวัตถุอันตราย และอื่นๆ</p> | **รหัสโภคภัณฑ์**<p>ในต้นทุนแฝง *รหัสโภคภัณฑ์* มีการสร้างขึ้นในระดับของนิติบุคคล ส่วนใหญ่ใช้สำหรับการรายงาน</p> | ต้นทุนแฝงสามารถใช้รหัสโภคภัณฑ์ที่ใช้กับ TMS และสถานที่อื่นๆ ใน Supply Chain Management ได้ อย่างไรก็ตาม รหัสโภคภัณฑ์ของต้นทุนแฝงจะใช้โดยต้นทุนแฝงเท่านั้น |

### <a name="some-reference-data-isnt-shared"></a>ข้อมูลอ้างอิงบางรายการไม่ได้ใช้ร่วมกัน

TMS และต้นทุนแฝงห้ามใช้ข้อมูลอ้างอิงร่วมกันระหว่างเอนทิตี เช่น การตั้งค่าต้นทุน การเดินทาง และการเดินทาง ถ้าคุณใช้ทั้งสองโมดูล คุณต้องสร้างข้อมูลอ้างอิงที่ไม่ได้ใช้ร่วมกันใหม่

### <a name="intercompany-reports-dont-work-with-goods-in-transit"></a><a name="intercompany-reports"></a>รายงานระหว่างบริษัทไม่งานกับสินค้าที่อยู่ระหว่างส่งต่อ

รายงานต่อไปนี้ไม่ได้ร่วมกับคุณลักษณะสินค้าที่อยู่ระหว่างการส่งต่อที่มีต้นทุนแฝง:

- [รายงานผลรวมสินค้าที่อยู่ระหว่างส่งต่อระหว่างบริษัท](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/intercompany-goods-in-transit-totals-report-intercompanygoodsintransittotals)
- [รายงานผลรวมสินค้าที่อยู่ระหว่างส่งต่อระหว่างบริษัท](https://docs.microsoft.com/dynamicsax-2012/appuser-itpro/intercompany-goods-in-transit-totals-report-intercompanygoodsintransittotals)

รายงานเหล่านี้จะสมมติว่ามีการส่งสินค้าไปส่งต่อทันทีที่คุณออกบันทึกการจัดส่งสินค้าขาย และเปลี่ยนสินค้าออกจากคลังสินค้าเมื่อรับสินค้า แต่สินค้าที่อยู่ระหว่างส่งต่อไม่ได้รับการประมวลผลด้วยวิธีนี้ ดังนั้น ถ้าคุณใช้สินค้าที่อยู่ระหว่างส่งต่อและลักษณะการดําเนินการระหว่างบริษัทเข้าด้วยกัน ผลลัพธ์ต่างๆ ของรายงานทั้งสองจะไม่ถูกต้อง

## <a name="using-the-two-modules-together"></a>การใช้สองโมดูลพร้อมกัน

คุณสามารถใช้ TMS ได้กับทั้งการดําเนินงานขาเข้าและขาออก แม้ว่าต้นทุนแฝงจะจัดเตรียมฟังก์ชันพื้นฐานส่วนใหญ่เหมือนกับ TMS ของการดําเนินงานขาเข้า แต่ก็จะเพิ่มฟังก์ชันบางอย่างด้วย ดังนั้น คุณจึงอาจต้องการพิจารณาการใช้ TMS ในการดําเนินงานขาออกและต้นทุนแฝงส่วนการดําเนินงานขาเข้า

โดยทั่วไป เราไม่แนะนาให้คุณใช้ทั้งสองโมดูลพร้อมกันในการดําเนินงานขาเข้า คุณควรใช้โมดูลที่ตรงกับความต้องการของคุณอย่างดีที่สุด ถ้าคุณไม่ได้ใช้ทั้งสองโมดูลด้วยกัน คุณต้องไม่แบ่งใช้เอกสารต้นทางระหว่างกัน ตัวอย่างเช่น อย่าใช้ใบสั่งซื้อเดียวกันทั้งการบรรทุกใน TMS และการเดินทางในต้นทุนแฝง โดยเฉพาะอย่างยิ่ง คุณต้องตรวจสอบให้แน่ใจว่าคุณไม่ได้ตั้งค่าระบบเพื่อสร้างสินค้าขาเข้าโดยอัตโนมัติ หากสินค้าจากใบสั่งซื้อรวมอยู่ในการเดินทาง สินค้านั้นจะได้รับการจัดการโดยเป็นส่วนหนึ่งของการบรรทุกไม่ได้

## <a name="goods-in-transit"></a>สินค้าที่อยู่ระหว่างส่งต่อ

หนึ่งในลักษณะการถูกใช้งานหลักของต้นทุนแฝง คือ ความสามารถในการประมวลผลสินค้าที่อยู่ระหว่างส่งต่อ การประมวลผลสินค้าที่อยู่ระหว่างการส่งต่อช่วยให้คุณสามารถเป็นเจ้าของทางการเงินของสินค้าที่จัดส่งก่อนที่จะได้รับทางกายภาพที่คลังสินค้าปลายทาง การประมวลผลสินค้าที่อยู่ระหว่างส่งต่อมักเป็นเอกสารที่ต้องใช้สําหรับการค้าระหว่างประเทศ

### <a name="tms-goods-in-transit-features"></a>คุณลักษณะสินค้าที่อยู่ระหว่างส่งต่อ TMS

ขณะนี้ TMS ไม่สนับสนุนการประมวลผลสินค้าในการส่งต่อ

### <a name="landed-cost-goods-in-transit-features"></a>คุณลักษณะการส่งต่อของสินค้าที่คิดต้นทุนแฝง

การประมวลผลสินค้าที่อยู่ระหว่างการส่งต่อเป็นลักษณะการใช้ฟังก์ชันหลักของต้นทุนแฝงและจัดเตรียมฟังก์ชันต่อไปนี้

- **สินค้าในคลังสินค้าส่งต่อ** – สินค้าในคลังสินค้าส่งต่อสามารถเชื่อมโยงกับคลังสินค้าแต่ละแห่งใน Supply Chain Management ได้ สินค้าจะถูกโอนย้ายไปยังคลังสินค้าส่งต่อหลังจากที่ออกใบแจ้งหนี้ใบสั่งซื้อต้นฉบับแล้ว และสินค้าเหล่านั้นจะถูกจองตามจริงและจะไม่พร้อมใช้จนกว่าจะได้รับสินค้าที่คลังสินค้าตามจริง ดังนั้น คุณจึงสามารถเป็นเจ้าของสินค้าทางการเงินโดยการออกใบแจ้งหนี้ใบสั่งซื้อ
- **บัญชีแยกประเภททั่วไปสินค้าที่อยู่ระหว่างส่งต่อ** – ต้นทุนแฝงจะเพิ่มบัญชีการลงรายการบัญชีแยกประเภททั่วไปหนึ่งๆ ในโปรไฟล์การลงรายการบัญชีใบสั่งซื้อ เพื่อจัดการการประมวลผลสินค้าที่อยู่ระหว่างส่งต่อ บัญชีแยกประเภททั่วไปสินค้าที่อยู่ระหว่างส่งต่อนี้จะหน้าที่เป็นบัญชีของชนิด "สินค้าที่ออกใบแจ้งหนี้แล้วไม่ได้รับ" เมื่อมีการออกใบแจ้งหนี้ใบสั่งซื้อต้นฉบับ และสร้างสินค้าในใบสั่งส่งต่อ ต้นทุนใบสั่งซื้อโดยตรงถูกลงรายการบัญชีในบัญชีแยกประเภททั่วไปที่ส่งต่อสินค้าจนกว่าจะได้รับสินค้าที่สถานที่ทางกายภาพสุดท้าย
- **การอัปเดตการควบคุมการติดตาม** – ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อสนับสนุนฟังก์ชันการควบคุมการติดตามในต้นทุนแฝง การควบคุมการติดตามช่วยให้คุณสามารถอัปเดตวันที่คาดว่าจะมาถึงของสินค้าขณะอยู่ในการส่งต่อ การควบคุมการติดตามจะอัปเดตการเดินทางและรายการใบสั่งซื้อที่เชื่อมโยง จากนั้นวันที่จัดส่งที่คาดไว้จะพร้อมใช้งานในการวางแผนหลักและลอจิสติกส์
- **การทริกเกอร์ธุรกรรมที่เกิน/ต่ํากว่า** – ถ้าผลต่างเกิดขึ้นระหว่างสินค้าที่คาดไว้ในรายการการเดินทางกับสินค้าจริงที่ได้รับที่คลังสินค้า ต้นทุนแฝงจะแก้ไขโดยการสร้างธุรกรรมที่เกินและ/หรือที่ต่ํากว่า จากนั้นคุณสามารถจัดการธุรกรรมเหล่านี้ ตามวิธีที่คุณต้องการประมวลผลธุรกรรม ผ่านใบสั่งซื้อหรือสมุดรายวันการเคลื่อนย้าย ธุรกรรมที่มากเกินและน้อยเกินจะให้ลิงค์กลับไปยังการเดินทาง ใบสั่งซื้อเดิม และสมุดรายวันการเคลื่อนย้ายใหม่หรือใบสั่งซื้อเพื่อความแตกต่าง
- **ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ** – ต้นทุนแฝงจะแนะนำเอกสารต้นทางใหม่ที่เรียกอีกอย่างว่า *ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ* ใบสั่งสินค้าที่อยู่ระหว่างส่งต่อช่วยให้คุณออกใบแจ้งหนี้ต้นฉบับใบสั่งซื้อเพื่อเป็นเจ้าของทางการเงินของสินค้าได้ เมื่อมีการออกใบแจ้งหนี้ใบสั่งซื้อ จะมีการสร้างเอกสารต้นทางใหม่ให้กับบรรทัดใบสั่งซื้อแต่ละบรรทัดที่มีชุดมิติสินค้าคงคลัง จากนั้นสินค้าจะถูกย้ายไปยังสินค้าในคลังสินค้าที่อยู่ระหว่างส่งต่อ ซึ่งสินค้านั้นถูกจองทางกายภาพกับสินค้าที่อยู่ระหว่างการส่งต่อ และจะใช้ไม่ได้เว้นแต่จะได้รับสินค้าในใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ

## <a name="miscellaneous-charges-and-shipment-costs"></a>ค่าธรรมเนียมเบ็ดเตล็ดและต้นทุนการจัดส่ง

แง่มุมที่สําคัญที่สุดของการจัดส่งสินค้าและการจัดการการจัดส่งสินค้าเพื่อสินค้าคือการรับรู้ต้นทุนค่าโสหุ้ยที่เกี่ยวข้องกับการจัดส่งสินค้า ต้นทุนค่าโสหุ้ยเหล่านี้ไม่ใช่ต้นทุนสินค้าคงคลังทางตรงที่เชื่อมโยงกับใบสั่งซื้อและการจัดส่งหรือการเดินทาง แต่เป็นต้นทุนเพิ่มเติมที่มาจากลักษณะการเคลื่อนย้ายสินค้าจากผู้จัดซื้อไปยังองค์กรของคุณ ต้นทุนเหล่านี้อาจรวมต้นทุนค่าขนส่งที่เชื่อมโยงกับการจัดส่งสินค้าจากสถานที่ทางกายภาพหนึ่งไปยังต้นทุนทางกายภาพอื่น หรือต้นทุนภาษีที่สัมพันธ์กับการนําเข้าสินค้าจากผู้จัดซื้อต่างประเทศ

การจัดการต้นทุนแฝงจะจัดการต้นทุนเหล่านี้โดยการสร้างโครงสร้างต้นทุนชนิดใหม่ที่เรียกอีกอย่างว่า *ต้นทุนอัตโนมัติ* TMS จะจัดการต้นทุนเหล่านี้โดยใช้ฟังก์ชันค่าธรรมเนียมเบ็ดเตล็ด

### <a name="tms-charge-and-cost-features"></a>คุณลักษณะค่าธรรมเนียมและต้นทุนของ TMS

TMS ใช้ค่าธรรมเนียมเบ็ดเตล็ดเพื่อจัดการต้นทุนเพิ่มเติมให้กับโหลด ซึ่งปันส่วนให้กับรายการเอกสารต้นทางที่เกี่ยวข้อง คุณสามารถตั้งค่าค่าธรรมเนียมเบ็ดเตล็ดเหล่านี้ในระดับรายการใบสั่งซื้อหรือส่วนหัวของใบสั่งซื้อ

ใน TMS คุณสามารถแบ่งตามส่วนค่าธรรมเนียมเบ็ดเตล็ด หรือแบ่งตามน้ําหนัก ปริมาตร หรือปริมาณของสินค้าคงคลังได้ ค่าธรรมเนียมสามารถแบ่งโดยค่าขนส่งหรือค่าธรรมเนียมอุปกรณ์เสริม การพัฒนาเพิ่มเติมจะต้องใช้วิธีการเลือกตามส่วนเพิ่มเติมใน TMS

### <a name="landed-cost-charge-and-cost-features"></a>คุณลักษณะค่าธรรมเนียมต้นทุนแฝงและต้นทุน

ต้นทุนแฝงมีชุดฟังก์ชันต้นทุนที่จัดการต้นทุนเพิ่มเติมที่เชื่อมโยงกับการจัดส่งสินค้า ต้นทุนเหล่านี้เรียกว่าต้นทุนอัตโนมัติ และจะใช้เมื่อออกใบแจ้งหนี้การจัดส่งครั้งแรกโดยใช้ยอดต้นทุนที่ประเมินไว้ ต้นทุนแต่ละชนิดมีการจัดการในการลงรายการบัญชีของตนเอง หลังจากลงรายการบัญชีใบแจ้งหนี้จริงของต้นทุนค่าโสหุ้ยแล้ว ต้นทุนที่ประเมินจะถูกอัปเดตโดยอัตโนมัติเพื่อสะท้อนต้นทุนจริง

นอกจากนี้ ต้นทุนค่าโสหุ้ยที่เชื่อมโยงกับการจัดส่งสินค้ายังสามารถออกใบแจ้งหนี้ระหว่างการเดินทางจากท่าเรือต้นทางหรือในเวลาใด ๆ หลังจากได้รับสินค้า ความสามารถนี้มีความยืดหยุ่นขึ้นต่อปริมาณการใช้สินค้า

รายการต่อไปนี้แสดงแนวคิดบางอย่างเกี่ยวกับคุณลักษณะการคิดค่าธรรมเนียมและต้นทุนในต้นทุนแฝง

- **พื้นที่ต้นทุน** – ต้นทุนและค่าธรรมเนียมสามารถถูกมอบหมายให้กับระดับหรือ *พื้นที่ต้นทุน* ต่าง ๆ บนการเดินทางได้ คุณสามารถใช้ต้นทุนที่ระดับการเดินทางและแยกย่อยให้กับสินค้าแต่ละรายการในการเดินทาง นอกจากนี้ ต้นทุนยังสามารถใช้กับระดับของคอนเทนเนอร์ ใบสั่งซื้อ สินค้า หรือใบสั่งโอนย้ายได้ ค่าธรรมเนียมต้นทุนแต่ละค่าธรรมเนียมสามารถจัดการแยกต่างหากระหว่างพื้นที่ต่าง ๆ และแบ่งย่อยเป็นต้นทุนต่อสินค้า
- **วิธีการแยกตามส่วน** – วิธีการแยกตามส่วนหลายวิธีที่พร้อมใช้งานในต้นทุนแฝง ค่าธรรมเนียมต้นทุนสามารถแบ่งตามปริมาณ จํานวนเงินดอลลาร์ มูลค่า น้ําหนัก ปริมาตร การวัด หรือการวัดเชิงปริมาตรที่ผู้ใช้กําหนด
- **การควบคุมการเคลียร์/ผลต่าง** – ค่าธรรมเนียมต้นทุนถูกตั้งค่าเป็นชนิดรหัสต้นทุนของตนเองในต้นทุนแฝง รหัสต้นทุนแต่ละชนิดช่วยให้คุณสามารถกําหนดบัญชีรอหักบัญชีต้นทุนที่ประเมิน และบัญชีผลต่างของราคาซื้อสะสมและบัญชีผลต่างของต้นทุนที่ประเมินเทียบกับต้นทุนจริงได้ เมื่อลงรายการบัญชีต้นทุนที่ประเมินไว้เริ่มแรก จะมีเครดิตที่บัญชีรอการโอน หลังจากลงรายการบัญชีใบแจ้งหนี้จริงแล้ว ค่าธรรมเนียมต้นทุนจริงจะถูกลงรายการบัญชี และต้นทุนที่ประเมินจะถูกเดบิตออกจากบัญชีระหว่างหักบัญชี

## <a name="matching-freight-bills-and-invoices"></a>การจับคู่บิลและใบแจ้งหนี้การขนส่ง

### <a name="tms-bill-and-invoice-matching-features"></a>คุณลักษณะการจับคู่บิลและใบแจ้งหนี้ TMS

TMS สามารถจับคู่บิลการขนส่งที่มีต้นทุนและใบแจ้งหนี้ที่ประเมินเข้ากับต้นทุนการขนส่งจริง ใบแจ้งหนี้อาจได้รับทั้งทางอิเล็กทรอนิกส์หรือแบบกระดาษ ผลต่างของการจับคู่ระหว่างต้นทุนที่ประเมินและต้นทุนจริงไม่ส่งผลกระทบต่อการประเมินค่าสินค้าคงคลัง

สำหรับข้อมูลเพิ่มเติม ดูที่ [กระทบยอดการขนส่งในการจัดการการขนส่ง](../transportation/reconcile-freight-transportation-management.md)

### <a name="landed-cost-bill-and-invoice-matching-features"></a>คุณลักษณะการจับคู่บิลและใบแจ้งหนี้ต้นทุนแฝง

ต้นทุนแฝงสามารถจับคู่บิลการขนส่งกับค่าธรรมเนียมต้นทุนที่ประเมิน และสามารถออกใบแจ้งหนี้ค่าธรรมเนียมต้นทุนจริงกับต้นทุนที่ประเมิน ในเวลาที่ออกใบแจ้งหนี้ ค่าธรรมเนียมการขนส่งอยู่ในสมุดรายวันใบแจ้งหนี้ และต้นทุนที่ประเมินจะถูกล้างข้อมูลออกจากบัญชีรอการโอน นอกจากนี้ ค่าธรรมเนียมต้นทุนจริงจะถูกลงรายการตามต้นทุนขายของสินค้า พร้อมกับผลต่างระหว่างค่าธรรมเนียมที่ประเมินไว้และค่าธรรมเนียมจริง ลักษณะการทำงานนี้ช่วยให้มีความยืดหยุ่นในกระบวนการออกใบแจ้งหนี้

## <a name="tracking"></a>การติดตาม

คุณลักษณะการใช้งานที่สําคัญของทั้ง TMS และต้นทุนแฝง คือ การติดตามสินค้าและระบุว่าสินค้าอยู่ในจุดใดจากจุดเริ่มต้นไปยังคลังสินค้าปลายทางสุดท้าย การติดตามช่วยให้คุณสามารถติดตามและอัปเดตสถานที่เก็บสินค้า และเมื่อใดที่คาดว่าสินค้าจะมาถึงที่คลังสินค้าสำหรับปริมาณการใช้ นอกจากสถานะของสินค้าระหว่างการเดินทางแล้ว ต้นทุนแฝงยังแสดงวันที่ที่คาดไว้และวันที่จริงของแต่ละขั้นตอนของการเดินทาง

### <a name="tms-tracking-features"></a>คุณลักษณะการติดตาม TMS

TMS มีคุณลักษณะที่จํากัดในการติดตามโหลดที่เข้ามา โดยจะแสดงวันที่และเปิดใช้งานการรวมที่สร้างเพื่อค้นหาตําแหน่งงานที่ถูกต้อง (ตัวอย่างเช่น บนหน้า **สถานะการขนส่ง**)

สำหรับแต่ละเซ็กเมนต์กระบวนการผลิต คุณสามารถป้อนตารางที่ประเมินไว้และเวลาการมาถึงได้

### <a name="landed-cost-tracking-features"></a>คุณลักษณะการติดตามต้นทุนแฝง

ต้นทุนแฝงสามารถแสดงการควบคุมการติดตามการเดินทางแต่ละอย่าง ตั้งแต่ท่าเรือต้นทางไปยังปลายทางสุดท้าย การควบคุมการติดตามจะตั้งค่าสถานะของการเดินทาง สถานะที่ระบุว่าการเดินทางจะเดินเรือ ที่ศุลกากรเพื่อตรวจสอบ หรือเพื่อจัดส่งในท้องถิ่นระหว่างทางไปยังคลังสินค้าขั้นสุดท้าย สถานะสามารถตั้งค่าในระดับของบรรทัดใบสั่งซื้อ คอนเทนเนอร์ และส่วนหัวของการเดินทาง ดังนั้น คุณจึงมีการควบคุมแบบ Granular มากขึ้น

นอกจากนี้ วันที่คาดว่าจะได้รับการยืนยันซึ่งขึ้นอยู่กับระยะเวลารอคอยสินค้าที่ระบุยังเชื่อมโยงกับแต่ละขั้นตอนของการเดินทาง วันที่จัดส่งที่ยืนยันและที่คาดไว้จะถูกเพิ่มเข้าในรายการใบสั่งซื้อและสินค้าที่อยู่ระหว่างการส่งต่อ และสามารถใช้กับการวางแผนหลักและลอจิสติกส์ได้ นอกจากวันที่ที่คาดไว้แล้ว ยังสามารถอัปเดตวันที่จริงได้ด้วย จากนั้นจะมีการอัปเดตขั้นตอนต่อมาในการเดินทาง

## <a name="multi-leg-journeys"></a>การเดินทางหลายช่วง

แนวคิดการเดินทางจะระบุสถานที่อยู่ในกระบวนการและควบคุมสถานะของสินค้าขณะที่อยู่ระหว่างการส่งต่อ สินค้าสามารถเดินทางผ่านการเดินทางได้หลายขา และคุณสามารถเชื่อมโยงต้นทุนต่าง ๆ กับขาที่ระบุได้ ตัวอย่างเช่น ค่าขนส่งในการเดินเรือ และต้นทุนการขนส่งสินค้าท้องถิ่นในการขนส่งสินค้าจากท่าเรือไปยังปลายทาง

### <a name="tms-multi-leg-journey-features"></a>คุณลักษณะการเดินทางแบบหลายขาของ TMS

ใน TMS คุณสามารถแบ่งเส้นทางออกเป็นเซกเมนต์กระบวนการผลิตเพื่อแสดงการเดินทางแบบหลายขา TMS สามารถปันส่วนราคาเฉพาะที่และค่าธรรมเนียมอุปกรณ์เสริมให้กับเซ็กเมนต์กระบวนการผลิตแต่ละเซ็กเมนต์ได้

### <a name="landed-cost-multi-leg-journey-features"></a>คุณลักษณะการเดินทางหลายขาของต้นทุนแฝง

ต้นทุนแฝงจะให้กรอบงานการเคลื่อนย้ายสินค้าจากจุดกําเนิดไปยังปลายทางผ่านหลายขา ซึ่งเป็นการหยุดหรือขั้นตอนในการเดินทาง ช่วงการเดินทางถูกรวมไว้เพื่อสร้างแม่แบบการเดินทาง ซึ่งกําหนดว่าสินค้าจะย้ายจากผู้ผู้ส่งคืนไปยังองค์กรของคุณอย่างไร

ในทุกช่วงการเดินทาง คุณสามารถกําหนดสถานะของบรรทัดใบสั่งซื้อแต่ละบรรทัดและระยะเวลารอคอยสินค้าที่เกี่ยวข้องด้วยได้ ระยะเวลารอคอยสินค้ารวมของช่วงทั้งหมดใช้เพื่อระบุวันที่จัดส่งที่ประเมินไว้ อย่างไรก็ตาม การประมวลผลสินค้าที่อยู่ระหว่างส่งต่อนั้นต้องระบุการเดินทางแบบหลายขาที่จะใช้ การเดินทางของสินค้าบนการเดินทางได้รับการจัดการในตารางเฉพาะของต้นทุนแฝง

## <a name="receiving-by-container"></a>การรับสินค้าตามคอนเทนเนอร์

อาจเป็นเรื่องสําคัญที่จะจัดการวิธีการแบ่งและจัดเก็บสินค้าบนเรือ บนเรือ สินค้าสามารถเก็บไว้ในหนึ่งคอนเทนเนอร์หรือหลายคอนเทนเนอร์ได้ เมื่อได้รับสินค้า สินค้าจะได้รับในคอนเทนเนอร์ และคอนเทนเนอร์ต่าง ๆ ในการเดินทางอาจได้รับในเวลาอื่นหรือในวันที่ต่างๆ

ทั้ง TMS และต้นทุนแฝงมีฟังก์ชันในการจัดการการรับสินค้าในคอนเทนเนอร์ TMS ใช้แนวคิดของสินค้าที่บรรทุกเพื่อจัดการสินค้า ใบสั่งซื้อ และใบสั่งโอนย้ายที่เชื่อมโยงกับคอนเทนเนอร์การจัดส่ง TMS สนับสนุนการรับตามโครงสร้างบรรจุภัณฑ์ที่ได้รับผ่านการแจ้งเตือนการจัดส่งล่วงหน้า (ASN) ต้นทุนแฝงใช้แนวคิดเกี่ยวกับคอนเทนเนอร์เพื่อประมวลผลใบสั่งซื้อและควบคุมต้นทุนค่าโสหุ้ยที่เชื่อมโยงกับคอนเทนเนอร์บนเรือ

### <a name="tms-receiving-by-container-features"></a>คุณลักษณะการรับสินค้า TMS ตามคอนเทนเนอร์

TMS สนับสนุน ASN ขาเข้า ตัวแปรทั้งหมดของการรับผ่านทางแอปการจัดการคลังสินค้าบนมือถือ และวิธีการทั้งหมดที่ได้รับผ่านทางไคลเอนต์ Supply Chain Management

### <a name="landed-cost-receiving-by-container-features"></a>คุณลักษณะการรับสินค้าต้นทุนแฝงตามคอนเทนเนอร์

เพื่อสนับสนุนการรับสินค้าตามคอนเทนเนอร์ ต้นทุนแฝงจึงสร้างเรกคอร์ดคอนเทนเนอร์การจัดส่งและเชื่อมโยงใบสั่งซื้อกับคอนเทนเนอร์เฉพาะโดยใช้รหัสคอนเทนเนอร์ จากนั้นต้นทุนค่าโสหุ้ยสามารถใช้กับคอนเทนเนอร์การจัดส่งนั้นและแบ่งย่อยเพื่อให้ต้นทุนค่าโสหุ้ยนั้นเชื่อมโยงกับใบสั่งซื้อที่เกี่ยวข้อง

คุณสามารถรับคอนเทนเนอร์ในต้นทุนแฝงผ่านชนิดใหม่ของการรับสินค้าที่เรียกอีกอย่างว่า *การรับสินค้าที่อยู่ระหว่างส่งต่อ* ผ่านสมุดรายวันการมาถึง หรือผ่านทางการรับสินค้าของอุปกรณ์เคลื่อนที่ เมื่อใช้สมุดรายวันการมาถึง สามารถเริ่มต้นปริมาณจากสินค้าในใบสั่งสินค้าที่อยู่ระหว่างส่งต่อ หรือบรรทัดใบสั่งซื้อดั้งเดิมในคอนเทนเนอร์ ต้นทุนแฝงมีงานสองชนิดเกี่ยวกับการรับผ่านแอปการจัดการคลังสินค้าบนมือถือ

ต้นทุนแฝงไม่มี ASN สำหรับการรับสินค้าทางอิเล็กทรอนิกส์ นอกจากนี้ ยังไม่ได้สนับสนุนโฟลว์แอปการจัดการคลังสินค้าบนมือถือที่ประมวลผลการรับสินค้า การรับป้ายทะเบียน หรือการรับป้ายทะเบียนแบบผสม

## <a name="rate-shopping-by-vendor"></a>จัดอันดับการซื้อสินค้าตามผู้จัดจำหน่าย

การซื้อสินค้าอัตราช่วยให้บริษัทสามารถเลือกผู้จัดจำหน่ายการขนส่งได้ตามราคาต่ำสุด เส้นทางที่เร็วที่สุด หรือข้อควรพิจารณาทั้งสองอย่าง

### <a name="tms-rate-shopping-features"></a>คุณลักษณะการซื้อสินค้าที่มีอัตรา TMS

TMS ช่วยให้คุณสามารถระบุโซลูชันผู้จัดจำหน่ายและโซลูชันสายงานการผลิตของใบสั่งขาเข้าและขาออกได้ ตัวอย่างเช่น คุณสามารถระบุเส้นทางที่เร็วที่สุดหรืออัตรามีราคาแพงน้อยที่สุดสำหรับการจัดส่ง

TMS ให้การซื้อสินค้าอัตราเต็มและประสิทธิภาพการขนส่งผ่านเวิร์กเบนช์อัตรากระบวนการผลิต ตัวเลือกการจัดอันดับแบบยืดหยุ่นผ่านกลไกการจัดอันดับ API กลไกจัดการการประเมินเพื่อรวมกับบุคคลภายนอก และการสนับสนุนน้ําหนักเชิงปริมาตร

สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [กลไกจัดการการจัดการการขนส่ง](../transportation/transportation-management-engines.md)

### <a name="landed-cost-rate-shopping-features"></a>คุณลักษณะการซื้อสินค้าที่มีอัตราต้นทุนแฝง

ต้นทุนแฝงจะให้การสนับสนุนที่จํากัดเฉพาะส่วนการซื้อสินค้าโดยผู้จัดจำหน่ายเท่านั้น แม้ว่าคุณจะสามารถป้อนค่าผู้ขนส่งสินค้าได้ แต่ต้นทุนแฝงจะไม่เปรียบเทียบระหว่างผู้จัดจำหน่ายหลายราย

## <a name="driver-check-incheck-out-with-appointment-scheduling"></a>การเช็คอิน/เช็คเอาท์โดยพนักงานควบคุมพร้อมด้วยการจัดตารางการนัดหมาย

ลักษณะการเช็คอิน/เช็คเอาท์ผู้ขับขี่ช่วยให้ระบบสามารถตรวจสอบการมาถึงและป้องกันไม่ให้เกิดข้องกับการโหลดสินค้าในท่าสินค้า

### <a name="tms-driver-check-incheck-out-features"></a>คุณลักษณะการเช็คอินและเช็คเอาท์ของพนักงานขับขี่ TMS

TMS ช่วยให้คุณสามารถสร้าง *การนัดหมาย* ได้ การนัดหมายแสดงถึงเหตุการณ์ที่เกิดขึ้นในท่าสินค้าสำหรับการรับใบสั่งซื้อ จัดส่งใบสั่งขาย หรือประมวลผลสินค้าขาเข้าหรือขาออกตามวันที่และเวลาที่ระบุ มีการสร้างการนัดหมายเพื่อลดการทำงานด้วยตนเองสำหรับเจ้าหน้าที่คลังสินค้า ซึ่งช่วยให้แน่ใจว่ามีท่าสินค้าที่พร้อมใช้งานสำหรับการบรรทุกเข้าและขนออกของสินค้า และเพื่อหลีกเลี่ยงสถานการณ์ที่ผู้ขนส่งหลายรายมาถึงสถานที่ในเวลาเดียวกัน การนัดหมายช่วยให้มั่นใจว่าท่าสินค้าจะพร้อมใช้งานโหลดและขนถ่ายสินค้า และช่วยป้องกันสถานการณ์ที่ผู้ขนส่งหลายรายมาถึงที่สถานที่หนึ่ง ๆ พร้อมกัน

ระบบสนับสนุนการอนุญาตให้พนักงานขับขี่เช็คอินที่ประตูท่าสินค้าเฉพาะได้

### <a name="landed-cost-driver-check-incheck-out-features"></a>คุณลักษณะการเช็คอินและเช็คเอาท์ของพนักงานขับขี่ต้นทุนแฝง

ต้นทุนแฝงสามารถจัดเก็บการประเมินในวันที่และเวลาที่จะจัดส่งคอนเทนเนอร์

## <a name="transfer-orders-and-additional-costs"></a>ใบสั่งโอนย้ายและต้นทุนเพิ่มเติม

บ่อยครั้งที่ธุรกิจต้องสามารถโอนย้ายสินค้าระหว่างคลังสินค้าได้ เมื่อการโอนย้ายชนิดนี้เกิดขึ้น ต้นทุนจะเชื่อมโยงด้วย และคุณอาจต้องการรับรู้ต้นทุนดังกล่าว ในต้นทุนแฝง สามารถเพิ่มใบสั่งโอนย้ายเข้าในการเดินทางหรือเรือเพื่อติดตามการเคลื่อนย้ายของสินค้าจากคลังสินค้าหรือไซต์หนึ่งไปยังคลังสินค้าอื่น ประเมินเวลาในการจัดส่งและวันที่จัดส่งที่คาดไว้ และเพิ่มต้นทุนค่าโสหุ้ยในการโอนย้ายสินค้า

### <a name="tms-transfer-order-cost-features"></a>คุณลักษณะต้นทุนใบสั่งโอนย้าย TMS

TMS สนับสนุนการสร้างค่าธรรมเนียมการขนส่งที่เชื่อมต่อกับการโอนย้าย แม้ว่าสามารถดูค่าธรรมเนียมเหล่านี้จากใบสั่งโอนย้ายได้ แต่ต้นทุนแฝงจะไม่เพิ่มลงในต้นทุนสินค้า การกระทบยอดการขนส่งได้รับการสนับสนุนผ่านการสร้างบิลการขนส่งที่ยึดตามค่าธรรมเนียมเหล่านี้ จากนั้นบิลการขนส่งนี้จะถูกจับคู่กับใบแจ้งหนี้ค่าขนส่งให้แก่ผู้จัดจำหน่าย

### <a name="landed-cost-transfer-order-cost-features"></a>คุณลักษณะต้นทุนใบสั่งโอนย้ายของต้นทุนแฝง

ต้นทุนแฝงช่วยให้คุณสามารถเพิ่มใบสั่งโอนย้ายไปยังเรือหรือการเดินทาง ด้วยวิธีนี้ คุณสามารถเพิ่มต้นทุนการจัดส่งให้กับการเดินทางเป็นการจับคู่สินค้าคงคลังในเวลาที่รับใบสั่งโอนย้ายได้ คุณสามารถออกใบแจ้งหนี้ต้นทุนค่าโสหุ้ยของต้นทุนจริงและติดตามเพื่ออัปเดตต้นทุนขาย แม้ว่าใบสั่งโอนย้ายจะไม่ใช้การประมวลผลสินค้าที่อยู่ระหว่างส่งต่อในการออกใช้มาตรฐาน แต่คุณสามารถเพิ่มใบสั่งนั้นที่การเดินทางได้ ต้นทุนแฝงจะถูกบวกเพิ่มที่ต้นทุนรวมของสินค้าแต่ละรายการ
