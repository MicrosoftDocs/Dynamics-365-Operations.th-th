# <a name="costing-parameter-values-setup"></a>การตั้งค่าค่าพารามิเตอร์การคิดต้นทุน

[!include [banner](../../includes/banner.md)]

เมื่อคุณตั้งค่าโมดูล **ต้นทุนแฝง** คุณสามารถกําหนดชุดของค่าทั่วไปและการตั้งค่าที่เกี่ยวข้องหลายชุดต่อค่า จากนั้น ค่าเหล่านี้จะพร้อมใช้งาน เมื่อคุณเลือกค่าพารามิเตอร์การคิดต้นทุนชนิดที่ระบุในส่วนอื่นๆ ของแอป หัวข้อนี้อธิบายวิธีการตั้งค่าชุดของค่าเหล่านี้

## <a name="set-up-cost-type-codes"></a>ตั้งค่ารหัสชนิดต้นทุน

รหัสชนิดต้นทุนจะระบุชนิดของต้นทุนที่เกิดขึ้น เมื่อสินค้าถูกแฝงลงในคลังสินค้า หรือเป็นต้นทุนแฝงของการเดินทาง แม้ว่าโดยปกติจะเพิ่มมูลค่าของสินค้า แต่ก็สามารถใช้เพื่อรับรู้จํานวนเงินในบัญชีแยกประเภทได้ การปรับปรุงบัญชีแยกประเภทจะถูกใช้ เมื่อต้นทุนถูกตั้งค้างรับไว้ในช่วงเวลาหรือในระหว่างชุดของการเดินทางและออฟเซ็ตในธุรกรรมเดียว

> [!NOTE]
> ถ้ามีการใช้ตารางชนิดต้นทุนร่วมกันระหว่างนิติบุคคลต่างๆ ยังต้องมีการใช้ผังบัญชีร่วมกันระหว่างนิติบุคคลต่างๆ ด้วย มิฉะนั้น ธุรกรรมการลงรายการบัญชีจะไม่ทำงานอย่างถูกต้อง

หากต้องการตั้งค่ารหัสชนิดต้นทุนของคุณ ให้ไปที่ **ต้นทุนแฝง \> การตั้งค่าการคิดต้นทุน \> รหัสชนิดต้นทุน** คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อสร้างรหัสชนิดต้นทุนใหม่ แก้ไขรหัสที่มีอยู่ หรือลบชนิดต้นทุนที่เลือก

ตารางต่อไปนี้อธิบายถึงฟิลด์ที่พร้อมใช้งานสำหรับรหัสชนิดต้นทุนแต่ละรายการ

| ฟิลด์ | คำอธิบาย |
|---|---|
| รหัสชนิดต้นทุน | ป้อนชื่อสำหรับรหัสชนิดต้นทุน |
| คำอธิบาย | ป้อนคำอธิบายของรหัสชนิดต้นทุน |
| ใช้อัตราการจัดส่ง | ตั้งค่าตัวเลือกนี้เป็น *ใช่* หากต้องใช้อัตราแลกเปลี่ยนการเดินทาง (ในบางครั้งเรียกว่า อัตราการจัดการ) เพื่อคํานวณมูลค่าของต้นทุนเหล่านี้ ในกรณีนี้ อัตราค่าจัดส่งจะถูกใช้แทนอัตราแลกเปลี่ยนเริ่มต้นมาตรฐานหรืออัตราแลกเปลี่ยนแบบพิเศษ เพื่อแลกเปลี่ยนใบแจ้งหนี้ตามสกุลเงินต่างประเทศ |
| ประเภทการรายงาน | ฟิลด์นี้จะสร้างประเภทการรายงานสำหรับชนิดต้นทุน คุณสามารถพิมพ์รายงานตามประเภทการรายงาน หรือตามชนิดต้นทุน |
| ชนิดเดบิต | เลือกว่าชนิดต้นทุนควรเดบิตสินค้า บัญชีแยกประเภท หรือผู้จัดจำหน่าย |
| การลงรายการบัญชีด้านเดบิต | ถ้ามีการตั้งค่าฟิลด์ **ชนิดเดบิต** เป็น *บัญชีแยกประเภท* ให้เลือกคำอธิบายการลงรายการบัญชี |
| บัญชีเดบิต | ถ้ามีการตั้งค่าฟิลด์ **ชนิดเดบิต** เป็น *บัญชีแยกประเภท* ให้เลือกบัญชีแยกประเภทที่จะใช้ |
| ชนิดการเครดิต | เลือกว่าชนิดต้นทุนนี้ควรเครดิตสินค้า บัญชีแยกประเภท หรือผู้จัดจำหน่าย |

| บัญชีเครดิต | ถ้ามีการตั้งค่าฟิลด์ **ชนิดเครดิต** เป็น *บัญชีแยกประเภท* ให้เลือกบัญชีแยกประเภทที่จะใช้ |
| บัญชีรอการโอน | เลือกบัญชีรอการโอนสำหรับชนิดต้นทุน เราขอแนะนำว่า คุณควรระบุบัญชีรอการโอนที่แยกต่างหากสำหรับต้นทุนแต่ละชนิด เพื่อช่วยในการกระทบยอด |
| ชนิดการลงรายการบัญชีต้นทุนมาตรฐาน | ถ้าคุณใช้การคิดต้นทุนมาตรฐาน ให้เลือกอธิบายการลงรายการบัญชี |
| บัญชีผลต่างต้นทุนมาตรฐาน | <p>ถ้าคุณใช้การคิดต้นทุนมาตรฐาน จะมีการใช้บัญชีที่ระบุไว้ที่นี่เพื่อลงรายการบัญชีผลต่างใดๆ บัญชีนี้ใช้การแบ่งต้นทุนแฝงในหน้า **ราคาสินค้า** การแบ่งนี้ถูกสร้างขึ้นโดยใช้กิจวัตรประจำงวดเพื่ออัพเดตราคา</p><p>ตัวอย่างเช่น ต้นทุนมาตรฐานของสินค้าคือ $15.00 FOB คือ $13.00 และการขนส่งคือ $2.00 เมื่อได้รับใบแจ้งหนี้สำหรับสินค้าคงคลัง จะได้รับสินค้าที่ $15.00 แต่มีความผันแปรของราคามาตรฐานเป็น $2.00 สำหรับสินค้า เนื่องจาก FOB จริงคือ $13.00 จะมีการลงรายการบัญชีผลต่างนี้ที่บัญชีผลต่างราคามาตรฐานที่ถูกตั้งค่าในโพรไฟล์การลงบัญชีสินค้า เนื่องจากค่าขนส่งคือ $2.00 โดยประมาณ จะไม่มีผลต่างเมื่อลงรายการบัญชีใบแจ้งหนี้สินค้าคงคลัง อย่างไรก็ตาม เมื่อได้รับใบแจ้งหนี้สำหรับค่าขนส่ง ค่าขนส่งคือ $2.50 ต่อหน่วย ดังนั้น ผลต่าง $0.50 จึงถูกลงรายการบัญชีในต้นทุนที่ระบุ |

| บัญชีค่าธรรมเนียมคงค้าง | เลือกบัญชีที่จะใช้เพื่อค้างรับสำหรับการประเมินต้นทุน เมื่อมีการลงรายการบัญชีใบแจ้งหนี้การซื้อ ฟิลด์นี้จะถูกใช้เฉพาะเมื่อมีการตั้งค่าตัวเลือก **ใช้บัญชีคงค้างค่าใช้จ่ายชนิดต้นทุน** เป็น *ใช่* บน FastTab **การคิดต้นทุน** บนแท็บ **ทั่วไป** ของหน้า **พารามิเตอร์ต้นทุนแฝง** |

| บัญชีผลต่าง | บัญชีที่จะใช้เพื่อออฟเซ็ตการรับรู้ค่าธรรมเนียม เมื่อมีการลงรายการบัญชีใบแจ้งหนี้การซื้อ ฟิลด์นี้จะถูกใช้เฉพาะเมื่อมีการตั้งค่าตัวเลือก **ใช้บัญชีคงค้างค่าใช้จ่ายชนิดต้นทุน** เป็น *ใช่* บน FastTab **การคิดต้นทุน** บนแท็บ **ทั่วไป** ของหน้า **พารามิเตอร์ต้นทุนแฝง** |

## <a name="vendor-cost-type-groups"></a>กลุ่มชนิดต้นทุนของผู้จัดจำหน่าย

กลุ่มชนิดต้นทุนผู้จัดจำหน่ายจะช่วยในการระบุวิธีการค้นหาและใช้ค่าธรรมเนียม *ต้นทุนอัตโนมัติ* กับการเดินทาง ผู้จัดจำหน่ายที่มีต้นทุนการนําเข้าที่คล้ายกันจะถูกเชื่อมโยงเข้าด้วยกัน ตัวอย่างเช่น ผู้จัดจําหน่ายทั้งหมดจากตลาดเกิดใหม่จะจ่ายเปอร์เซ็นต์ภาษีเดียวกันสำหรับผลิตภัณฑ์ชนิดเดียวกันกับที่ซื้อจากตลาดที่ติดตลาดแล้ว

คุณสามารถรักษากลุ่มชนิดต้นทุนผู้จัดจำหน่ายได้โดยไปที่ **ต้นทุนแฝง \> การตั้งค่าการคิดต้นทุน \> กลุ่มชนิดของต้นทุนผู้จัดจำหน่าย** หน้า **กลุ่มชนิดต้นทุนผู้จัดจำหน่าย** จะมีกริดที่แสดงรายการกลุ่มชนิดต้นทุนผู้จัดจำหน่ายทั้งหมดที่มีอยู่ คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อเพิ่ม ลบ และแก้ไขแถวในกริดได้

ตารางต่อไปนี้อธิบายถึงฟิลด์ที่พร้อมใช้งานในแถวแต่ละแถวในกริด

| ฟิลด์ | คำอธิบาย |
|---|---|
| กลุ่มชนิดต้นทุนของผู้จัดจำหน่าย | ป้อนชื่อเฉพาะสำหรับกลุ่มชนิดต้นทุนผู้จําหน่าย (เช่น *ตลาดเกิดใหม่*) |
| คำอธิบาย | ป้อนคำอธิบายของกลุ่มชนิดต้นทุนผู้จําหน่าย คำอธิบายนี้สามารถให้รายละเอียดเกี่ยวกับระดับหรือชนิดของค่าธรรมเนียมที่สัมพันธ์กับกลุ่มผู้จัดจำหน่าย |

## <a name="item-cost-type-groups"></a>กลุ่มชนิดต้นทุนของสินค้า

กลุ่มชนิดต้นทุนสินค้าจะช่วยในการระบุวิธีการค้นหาและใช้ค่าธรรมเนียม *ต้นทุนอัตโนมัติ* กับการเดินทาง สินค้าที่คล้ายกันจะถูกเชื่อมโยงเข้าด้วยกัน ตัวอย่างเช่น สินค้าทั้งหมดที่มีอัตราภาษี 5 เปอร์เซ็นต์ อาจเป็นสมาชิกของกลุ่มชนิดต้นทุนเฉพาะ

คุณสามารถรักษากลุ่มชนิดต้นทุนสินค้าได้โดยไปที่ **ต้นทุนแฝง \> การตั้งค่าการคิดต้นทุน \> กลุ่มชนิดของต้นทุนสินค้า** หน้า **กลุ่มชนิดต้นทุนสินค้า** จะมีกริดที่แสดงรายการกลุ่มชนิดต้นทุนสินค้าทั้งหมดที่มีอยู่ คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อเพิ่ม ลบ และแก้ไขแถวในกริดได้

ตารางต่อไปนี้อธิบายถึงฟิลด์ที่พร้อมใช้งานในแถวแต่ละแถวในกริด

| ฟิลด์ | คำอธิบาย |
|---|---|
| กลุ่มชนิดต้นทุนของสินค้า | ป้อนชื่อเฉพาะสำหรับกลุ่มชนิดต้นทุนสินค้า (เช่น *ภาษี 5%*) |

## <a name="cost-templates"></a>เท็มเพลตต้นทุน

คุณใช้เท็มเพลตต้นทุนเพื่อตั้งค่าค่าเริ่มต้นสำหรับการตั้งค่าที่ผู้ใช้ที่เป็นผู้ได้รับการประเมินต้นทุนอาจไม่ทราบ เท็มเพลตต้นทุนสามารถช่วยลดความซับซ้อนในกระบวนการประเมินด้วยการลดการเลือกที่ผู้ใช้ต้องสร้างให้น้อยที่สุด เพื่อให้ได้การประเมินที่ถูกต้อง

หากต้องการทำงานกับเท็มเพลตต้นทุน ให้ไปที่ **ต้นทุนแฝง \> การตั้งค่าการคิดต้นทุน \> เท็มเพลตต้นทุน** บนหน้า **เท็มเพลตต้นทุน** บานหน้าต่างรายการทางด้านซ้ายจะแสดงเท็มเพลตต้นทุนปัจจุบันทั้งหมด คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อเพิ่ม ลบ และแก้ไขเท็มเพลตได้

ตารางต่อไปนี้อธิบายถึงการตั้งค่าที่พร้อมใช้งานสำหรับเท็มเพลตแต่ละรายการ

| ฟิลด์ | คำอธิบาย |
|---|---|
| เท็มเพลตต้นทุน | ป้อนชื่อเฉพาะสำหรับเท็มเพลตต้นทุน โดยทั่วไป ชื่อจะอธิบายถึงตัวคูณหรือตัวคูณต้นทุนสำหรับเท็มเพลต |

| วิธีการนำส่ง | เลือกโหมดการจัดส่งที่เท็มเพลตต้นทุนควรใช้ เมื่อมีการคํานวณต้นทุนที่ประเมินของสินค้า ฟิลด์นี้จะช่วยระบุต้นทุนอัตโนมัติที่เชื่อมโยงกับสินค้าในการประเมินต้นทุน |

## <a name="volumetric-divisors"></a>ตัวหารเชิงปริมาตร

ตัวหารเชิงปริมาตรถูกใช้ในการคํานวณน้ำหนักเชิงปริมาตร บริษัทจัดส่งสินค้า/ขนส่งแต่ละแห่ง สร้างสูตรตัวหารเชิงปริมาตรของตนเอง นอกจากนี้ โดยทั่วไป ตัวหารของบริษัทจะแตกต่างกันไป โดยขึ้นอยู่กับโหมดการจัดส่ง ตัวอย่างเช่น ทางอากาศ และทางทะเล มักจะมีตัวหารที่แตกต่างกันมาก นอกจากนี้ บริษัทยังสามารถสร้างกฎที่ซับซ้อนมากขึ้นได้ ทั้งนี้ขึ้นอยู่กับว่าขนส่งมาจากที่ใด

ตัวอย่างเช่น บรรจุภัณฑ์ที่ส่งออกโดยทางอากาศมีปริมาตร 3 ลูกบาศก์เมตร (เมตร³) บริษัทคิดค่าธรรมเนียมตามน้ำหนักเชิงปริมาตรและใช้ตัวหารเชิงปริมาตรเป็น 6 ตัวหารนี้จะถูกคูณด้วยปริมาตรเพื่อระบุน้ำหนักเชิงปริมาตร ดังนั้น น้ําหนักเชิงปริมาตรสำหรับตัวอย่างนี้คือ 3 × 6 = 18 กิโลกรัม (กก.)

หากต้องการตั้งค่าตัวหารเชิงปริมาตร ให้ไปที่ **ต้นทุนแฝง \> การตั้งค่าการคิดต้นทุน \> ตัวหารเชิงปริมาตร** หน้า **ตัวหารเชิงปริมาตร** แสดงกริดที่แสดงรายการตัวหารเชิงปริมาตรที่มีอยู่ทั้งหมด คุณสามารถใช้ปุ่มต่างๆ ในบานหน้าต่างการดำเนินการเพื่อเพิ่ม ลบ และแก้ไขแถวในกริดได้

ตารางต่อไปนี้อธิบายถึงฟิลด์ที่พร้อมใช้งานในแถวแต่ละแถวในกริด

| ฟิลด์ | คำอธิบาย |
|---|---|
| บริษัทจัดส่งสินค้า | เลือกบัญชีผู้จัดจำหน่ายของบริษัทขนส่งที่สัมพันธ์กับตัวหารเชิงปริมาตร |
| รหัสชนิดต้นทุน | เลือกรหัสชนิดต้นทุนที่สัมพันธ์กับตัวหารเชิงปริมาตร ใช้ฟิลด์นี้เพื่อป้อนชนิดต้นทุนลงในกลุ่มการรายงาน คุณสามารถพิมพ์รายงานตามประเภทการรายงาน หรือตามชนิดต้นทุน |
| จากท่าเรือ | เลือกพอร์ต "เริ่มต้น" ที่จะใช้ตัวหารเชิงปริมาตร |
| ตัวหารเชิงปริมาตร | ป้อนค่าตัวหารเชิงปริมาตรที่จะนำไปใช้กับแถว ค่าที่คุณป้อนจะถูก *คูณ* ด้วยปริมาตรของแต่ละบรรจุภัณฑ์ เพื่อระบุน้ำหนักเชิงปริมาตรของบรรจุภัณฑ์นั้น |
