# <a name="vendor-collaboration-mobile-workspace"></a>พื้นที่ทำงานแบบเคลื่อนที่ของการทำงานร่วมกันกับผู้จัดจำหน่าย

[!include [banner](../includes/banner.md)]

หัวข้อนี้แสดงข้อมูลเกี่ยวกับพื้นที่ทำงานแบบเคลื่อนที่ของ **การทำงานร่วมกันกับผู้จัดจำหน่าย** พื้นที่ทำงานนี้ช่วยให้ผู้จัดจำหน่ายของคุณได้ติดตามสถานการณ์ปัจจุบัน เกี่ยวกับใบสั่งซื้อที่ถูกส่งไปขอการอนุมัติ นอกจากนี้ยังสามารถดูข้อมูลเกี่ยวกับใบสั่งซื้อและผู้ติดต่อใหม่และที่อัพเดตแล้ว

พื้นที่ทำงานแบบเคลื่อนที่นี้มีจุดมุ่งหมายเพื่อใช้กับแอป Finance and Operations บนมือถือ

## <a name="overview"></a>ภาพรวม 
พื้นที่ทำงานแบบเคลื่อนที่ **การทำงานร่วมกันกับผู้จัดจำหน่าย** คอยให้ข้อมูลผู้จัดจำหน่ายเกี่ยวกับใบสั่งซื้อใหม่ เพื่อให้พวกเขาสามารถดูใบสั่งซื้อและจากนั้นตอบสนองในเว็บไคลเอ็นต์ 

>[!NOTE]
> หมายเหตุ: ควรใช้พื้นที่ทำงานแบบเคลื่อนที่เป็นส่วนเพิ่มเติมของเว็บไคลเอนต์การทำงานร่วมกันของผู้จัดจำหน่าย แต่ไม่ได้แทนที่ 

ผู้จัดจำหน่ายของคุณสามารถใช้พื้นที่ทำงานแบบเคลื่อนที่ของ **การทำงานร่วมกันกับผู้จัดจำหน่าย** ได้ เพื่อดูใบสั่งซื้อใหม่ที่ถูกส่งไปเพื่อขออนุมัติ ซึ่งจะแสดงข้อมูลใบสั่งซื้อ เช่น ผลิตภัณฑ์ ปริมาณ และวันที่จัดส่งที่ร้องขอ นอกจากนี้ยังมีข้อมูลราคาด้วย โดยขึ้นอยู่กับการตั้งค่าคอนฟิกของผู้จัดจำหน่ายแต่ละราย 

ผู้ใช้ที่ลงชื่อเข้าใช้เป็นผู้จัดจำหน่าย จะเห็นว่าใบสั่งซื้อใดได้รับการตอบรับ และใบสั่งซื้อใดยังคงรอการดำเนินการของลูกค้า ตัวอย่างเช่น ใบสั่งซื้ออาจกำลังรอการดำเนินการของลูกค้า เพราะผู้จัดจำหน่ายแนะนำวันที่จัดส่งอื่น แต่ลูกค้ายังไม่ได้ยอมรับวันที่นั้น ผู้จัดจำหน่ายจะยังเห็นรายการของใบสั่งซื้อที่ได้ยืนยันแล้วแต่ยังไม่ได้ถูกจัดส่งอีกด้วย 

เมื่อต้องการตอบสนองต่อใบสั่งซื้อ ผู้จัดจำหน่ายต้องใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายที่พร้อมใช้งานในเว็บไคลเอนต์ ที่นั่น ผู้จัดจำหน่ายยังสามารถได้รับข้อมูลเพิ่มเติมเกี่ยวกับใบสั่ง เช่น สิ่งที่แนบของเอกสาร ที่อยู่จัดส่งต่อรายการ และค่าธรรมเนียมที่เกี่ยวข้องกับผู้จัดจำหน่ายอีกด้วย 

ผู้จัดจำหน่ายที่มีบทบาทความปลอดภัยพิเศษ สามารถดูได้ว่าผู้ติดต่อใดที่ลงทะเบียนสำหรับบัญชีผู้จัดจำหน่าย บทบาทความปลอดภัยเดียวกันนี้ อนุญาตให้ผู้จัดจำหน่ายสามารถดูสถานะของคำขอของผู้ใช้ใด ๆ ที่ได้ถูกส่งไป 

ต้องมีการใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายในเว็บไคลเอนต์ เพื่อสร้างผู้ติดต่อใหม่ และส่งคำขอของผู้ใช้ใหม่ 

พื้นที่ทำงานแบบเคลื่อนที่ของ **การทำงานร่วมกันกับผู้จัดจำหน่าย** ช่วยให้ผู้จัดจำหน่ายสามารถดำเนินงานเหล่านี้ได้:

-   ดูใบสั่งซื้อใหม่ที่ถูกส่งไปยังผู้จัดจำหน่าย
-   ดูใบสั่งซื้อที่ผู้จัดจำหน่ายได้ตอบสนอง และที่กำลังรอการดำเนินการของลูกค้าอยู่
-   ดูใบสั่งซื้อที่ได้รับการยืนยันแล้ว แต่ยังไม่ได้รับทั้งหมด
-   ดูข้อมูลผู้ติดต่อที่ลงทะเบียนไว้สำหรับบัญชีผู้จัดจำหน่าย (งานนี้ต้องการบทบาทความปลอดภัยเพิ่มเติม)
-   ดูข้อมูลเกี่ยวกับคำขอของผู้ใช้ที่ถูกส่งโดยผู้จัดจำหน่าย และติดตามสถานะคำขอ (งานนี้ต้องการบทบาทความปลอดภัยเพิ่มเติม)

## <a name="sign-in-to-the-mobile-app"></a>ล็อกอินเข้าสู่แอปบนมือถือ
1.  เริ่มการทำงานแอปบนอุปกรณ์เคลื่อนที่ของคุณ
2.  ใส่ URL Microsoft Dynamics 365 ของคุณ
4.  ในครั้งแรกที่คุณเข้าสู่ระบบ ระบบจะขอให้คุณป้อนชื่อผู้ใช้และรหัสผ่านของคุณ ป้อนข้อมูลประจำตัวของคุณ
5.  หลังจากที่คุณลงชื่อเข้าใช้แล้ว พื้นที่ทำงานที่พร้อมใช้งานสำหรับบริษัทของคุณจะแสดงขึ้น หมายเหตุว่าถ้าผู้ดูแลระบบของคุณเผยแพร่พื้นที่ทำงานใหม่ในภายหลัง คุณจะต้องรีเฟรชรายการของพื้นที่ทำงานแบบเคลื่อนที่

### <a name="contacts"></a>ผู้ติดต่อ
หน้า **ผู้ติดต่อ** ช่วยให้คุณสามารถดูผู้ติดต่อทั้งหมดที่ตั้งค่าไว้สำหรับบัญชีผู้จัดจำหน่าย ซึ่งแสดงชื่อของผู้ติดต่อ ที่อยู่อีเมลหลัก และนามแฝงผู้ใช้ ถ้าผู้ติดต่อมีนามแฝง หน้านี้ยังแสดงว่าบัญชีผู้ใช้ของผู้ติดต่อมีการใช้งานอยู่หรือไม่อีกด้วย เมื่อคุณเลือกผู้ติดต่อ คุณจะเห็นรายละเอียดผู้ติดต่อ เช่น บุคคลเป็นผู้ติดต่อสำหรับนิติบุคคล คุณยังจะเห็นข้อมูลการติดต่อ เช่น หมายเลขโทรศัพท์ หรือที่อยู่อีเมลอื่นอีกด้วย

### <a name="user-requests"></a>คำขอของผู้ใช้
หน้า **คำขอของผู้ใช้** ช่วยให้คุณสามารถดูคำขอของผู้ใช้ทั้งหมด ที่คุณได้ส่งผ่านเว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่าย คุณยังสามารถติดตามสถานะคำขอเหล่านั้นได้อีกด้วย เมื่อคุณเลือกคำขอของผู้ใช้ คุณสามารถดูสิ่งที่ถูกร้องขอ เพิ่ม หรือปิดการทำงานของผู้ใช้ การเปลี่ยนแปลงการรักษาความปลอดภัย และดูบทบาทความปลอดภัยที่มีการร้องขอสำหรับผู้ใช้ได้

### <a name="purchase-orders-ready-for-review"></a>ใบสั่งซื้อที่พร้อมสำหรับการตรวจทาน
หน้า **ใบสั่งซื้อที่พร้อมสำหรับการตรวจทาน** ช่วยให้คุณสามารถดูใบสั่งซื้อทั้งหมดที่ลูกค้าส่งมา และยังไม่ได้รับการตอบสนอง คุณสามารถดูข้อมูลที่เลือกเกี่ยวกับใบสั่ง เช่น ผลิตภัณฑ์ใดมีการร้องขอและจะต้องจัดส่งเมื่อใด นอกจากนี้ยังมีข้อมูลราคาด้วย โดยขึ้นอยู่กับการตั้งค่าคอนฟิกของผู้จัดจำหน่าย

คุณยังสามารถดูได้อีกว่าใบสั่งซื้อมีบันทึกย่อหรือเอกสารแนบหรือไม่ อย่างไรก็ตาม เพื่อเปิดบันทึกย่อและสิ่งที่แนบ คุณต้องใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายในเว็บไคลเอนต์ เลือก **รายการใบสั่งซื้อ** เพื่อดูรายการทั้งหมดพร้อมกับรายละเอียด สำหรับแต่ละรายการ ตัวบ่งชี้จะแสดงว่ามีบันทึกย่อหรือเอกสารแนบหรือไม่ หรือว่ามีที่อยู่จัดส่งที่แตกต่างจากที่อยู่จัดส่งที่แสดงในหัวข้อหรือไม่

เมื่อต้องการตอบสนองต่อใบสั่งซื้อ คุณต้องใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายในเว็บไคลเอนต์

### <a name="awaiting-customer-action"></a>กำลังรอการดำเนินการของลูกค้า
หน้า **กำลังรอการดำเนินการของลูกค้า** ช่วยให้คุณสามารถค้นหาใบสั่งซื้อที่คุณหรือบุคคลอื่นในบริษัทของคุณ ที่มีการเข้าถึงไปยังการทำงานร่วมกันกับผู้จัดจำหน่ายได้ตอบสนองไป ใบสั่งซื้อจะปรากฏในรายการนี้เท่านั้น ถ้าลูกค้าต้องการดำเนินการหนึ่งในการดำเนินการต่อไปนี้ในใบสั่งซื้อ:

-   ถ้าใบสั่งซื้อถูกปฏิเสธ ลูกค้าต้องอัพเดต หรือยกเลิกใบสั่งและส่งอีกครั้ง เมื่อใบสั่งซื้อถูกส่งไปอีกครั้ง ใบสั่งซื้อดังกล่าวจะหายไปจากหน้า **กำลังรอการดำเนินการของลูกค้า**
-   ถ้ามีการยอมรับใบสั่งซื้อโดยมีการเปลี่ยนแปลง ลูกค้าต้องอัพเดตใบสั่งซื้อต้นฉบับและส่งอีกครั้งเพื่อตรวจทาน หรืออัพเดตใบสั่งตามการเปลี่ยนแปลงที่ร้องขอ แล้วยืนยันโดยทันที ในทั้งสองกรณี ใบสั่งซื้อดังกล่าวจะหายไปจากหน้า **กำลังรอการดำเนินการของลูกค้า**
-   ถ้าใบสั่งซื้อได้รับการยอมรับและยังปรากฏในหน้า **กำลังรอการดำเนินการของลูกค้า** ใบสั่งซื้อไม่ได้รับการยืนยันโดยอัตโนมัติ เมื่อทำการยอมรับ ซึ่งขณะนี้กำลังรอให้ตัวแทนจัดซื้อเปลี่ยนแปลงสถานะใบสั่งเป็น **ยืนยันแล้ว** โดยทั่วไป ใบสั่งซื้อจะถูกพิจารณาเป็นข้อตกลงระหว่างลูกค้าและผู้จัดจำหน่าย ทันทีที่ผู้จัดจำหน่ายยอมรับใบสั่ง ดังนั้น การอัพเดตเป็นสถานะ **ยืนยัน** มักจะเป็นแค่แบบแผน

เมื่อคุณเลือกใบสั่งซื้อ รายละเอียดเพิ่มเติมจะปรากฏเกี่ยวกับการตอบสนอง คุณสามารถดูรายละเอียดของรายการและตอบสนองสำหรับทุกบรรทัด สถานะของรายการจะแสดงว่ามีการให้การตอบสนองใดต่อไปนี้:

-   ยอมรับ
-   ปฏิเสธแล้ว
-   ยอมรับการเปลี่ยนแปลงแล้ว
-   ใช้ทดแทน/ทดแทน
-   แบ่งเป็นบรรทัดกำหนดการ/กำหนดการ

โปรดทราบว่า ฟิลด์ **การจัดส่ง** ถูกตั้งค่าเป็น **ใช่** หรือ **ไม่ใช่** เพื่อบ่งชี้ว่ารายการจะถูกจัดส่งหรือไม่ รายการอาจไม่สามารถถูกจัดส่งได้เนื่องจากสาเหตุต่อไปนี้:

- รายการถูกปฏิเสธ
- มีการทำการแทนที่ และรายการต้นฉบับไม่ได้ถูกคาดหวังว่าจะถูกจัดส่งตามที่ร้องขอในใบสั่งที่ได้รับ
- รายการถูกแบ่งออกเป็นหลายรายการของกำหนดการ และรายการต้นฉบับไม่ได้ถูกคาดหวังว่าจะถูกจัดส่งตามที่ร้องขอในใบสั่งที่ได้รับ

การเปลี่ยนแปลงใด ๆ ที่ทำไว้กับการตอบสนองรายการใบสั่งจะแสดงขึ้น อย่างไรก็ตาม บันทึกย่อที่อัพโหลดและสิ่งที่แนบมาจะไม่แสดงขึ้น เพื่อดูบันทึกย่อและสิ่งที่แนบ คุณต้องใช้เว็บอินเทอร์เฟสการทำงานร่วมกันกับผู้จัดจำหน่ายในเว็บไคลเอนต์

### <a name="open-confirmed-orders"></a>เปิดใบสั่งที่ยืนยัน
เมื่อมีการยืนยันใบสั่งซื้อจากลูกค้า (ซึ่งหมายความว่า สถานะใบสั่งซื้อเปลี่ยนเป็น **ยืนยันแล้ว**) จะปรากฏในใบสั่งที่ยืนยันแล้วที่เปิดอยู่ ซึ่งจะยังคงอยู่ในรายการจนกว่าจะมีการลงทะเบียนว่า ได้รับแล้ว จากลูกค้า
