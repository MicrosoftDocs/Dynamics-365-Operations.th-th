<span data-ttu-id="531f4-157">หลังจากที่ผู้จัดจำหน่ายยอมรับใบสั่ง ขั้นตอนต่อไปคือการบันทึก PO เป็น กำหนดให้แล้ว</span><span class="sxs-lookup"><span data-stu-id="531f4-157">After the vendor has agreed to the order, the next step is to record the PO as committed.</span></span> <span data-ttu-id="531f4-158">คุณสามารถทำขั้นตอนนี้ให้เสร็จสิ้นได้โดยใช้การดำเนินการ **การยืนยัน** หรือการดำเนินการ **ยืนยัน**</span><span class="sxs-lookup"><span data-stu-id="531f4-158">You can complete this step by using either the **Confirmation** action or the **Confirm** action.</span></span> <span data-ttu-id="531f4-159">การดำเนินการทั้งสองจะตั้งค่าสถานะการอนุมัติของใบสั่งเป็น **ยืนยันแล้ว**</span><span class="sxs-lookup"><span data-stu-id="531f4-159">Both these actions set the approval status of the order to **Confirmed**.</span></span> <span data-ttu-id="531f4-160">การยืนยันใบสั่งเริ่มต้นจากขั้นตอนเพิ่มเติมสองรายการ:</span><span class="sxs-lookup"><span data-stu-id="531f4-160">Confirmation of an order initiates two additional processes:</span></span>

-   <span data-ttu-id="531f4-161">สมุดรายวันถูกสร้างขึ้นเพื่อเก็บสำเนาที่เหมือนกันของสิ่งได้รับการยืนยันในระบบ</span><span class="sxs-lookup"><span data-stu-id="531f4-161">A journal is created to store an exact copy of what was confirmed in the system.</span></span> <span data-ttu-id="531f4-162">บางครั้ง ใบสั่งจำเป็นต้องมีการเปลี่ยนแปลง และสมุดรายวันเพิ่มเติมจะถูกสร้างขึ้นหลังจากมีการยืนยันใบสั่งที่มีการปรับปรุงแล้ว</span><span class="sxs-lookup"><span data-stu-id="531f4-162">Sometimes, orders require changes, and additional journals are created after the updated order is confirmed.</span></span> <span data-ttu-id="531f4-163">สมุดรายวันเหล่านี้ช่วยให้คุณสามารถดูประวัติของใบสั่งรุ่นต่างๆ ที่ได้รับการยืนยัน</span><span class="sxs-lookup"><span data-stu-id="531f4-163">These journals let you view the history of the various versions of the order that were confirmed.</span></span>
-   <span data-ttu-id="531f4-164">การกระจายการลงบัญชีจะถูกสร้างขึ้น และจะเกิดการตรวจสอบคำสั่งและการตรวจสอบงบประมาณหากมีการเปิดใช้งานฟังก์ชันนี้</span><span class="sxs-lookup"><span data-stu-id="531f4-164">Accounting distributions are created, and order checks and budget checks occur if this functionality has been enabled.</span></span> <span data-ttu-id="531f4-165">ถ้าการตรวจสอบรายการใดรายการหนึ่งล้มเหลว คุณจะได้รับข้อความที่ระบุว่า PO ต้องมีการเปลี่ยนแปลงก่อนจึงจะสามารถยืนยันอีกครั้งได้</span><span class="sxs-lookup"><span data-stu-id="531f4-165">If either check fails, you receive an error message that states that changes must be made to the PO before it can be confirmed again.</span></span>

<span data-ttu-id="531f4-166">ผู้จัดจำหน่ายอาจร้องขอการรับรองบางชนิดที่จะมีการชำระเงินสำหรับการซื้อ</span><span class="sxs-lookup"><span data-stu-id="531f4-166">A vendor might request some type of assurance that payment will be provided for a purchase.</span></span> <span data-ttu-id="531f4-167">มีวิธีการต่างๆ สำหรับการให้การค้ำประกันนี้ภายในกระบวนการบัญชีเจ้าหนี้</span><span class="sxs-lookup"><span data-stu-id="531f4-167">There are various methods for providing this guarantee within accounts payable processes.</span></span> <span data-ttu-id="531f4-168">ตัวอย่างเช่น การดำเนินการ **การชำระเงินล่วงหน้า** จองเงินทุนสำหรับ PO และการชำระเงินล่วงหน้านี้จะถูกบันทึกลงใน PO</span><span class="sxs-lookup"><span data-stu-id="531f4-168">For example, the **Prepayment** action reserves funds for the PO, and this prepayment is recorded on the PO.</span></span>

## <a name="changing-purchase-orders"></a><span data-ttu-id="531f4-169">การเปลี่ยนแปลงใบสั่งซื้อ</span><span class="sxs-lookup"><span data-stu-id="531f4-169">Changing purchase orders</span></span>
<span data-ttu-id="531f4-170">ในบางสถานการณ์ คุณอาจต้องเปลี่ยนแปลง PO หลังจากมาถึงสถานะการอนุมัติเป็น **อนุมัติแล้ว** หรือ **ยืนยันแล้ว**</span><span class="sxs-lookup"><span data-stu-id="531f4-170">In some situations, you might have to change a PO after it has reached an approval status of **Approved** or **Confirmed**.</span></span>  

<span data-ttu-id="531f4-171">ถ้า PO ถูกสร้างขึ้นโดยใช้กระบวนการจัดการการเปลี่ยนแปลง คุณสามารถทำการเปลี่ยนแปลงได้โดยการระงับใบสั่ง หรือใบสั่งได้รับการอนุมัติโดยใช้การดำเนินการ **การเปลี่ยนแปลงคำขอ**</span><span class="sxs-lookup"><span data-stu-id="531f4-171">If the PO was created by using a change management process, you can make changes by recalling the order or, if the order has already been approved, by using the **Request change** action.</span></span> <span data-ttu-id="531f4-172">ในกรณีนี้ สถานะการอนุมัติจะถูกเปลี่ยนกลับไปเป็น **ร่าง** และคุณสามารถปรับเปลี่ยนใบสั่งได้หลังจากนั้น</span><span class="sxs-lookup"><span data-stu-id="531f4-172">In this case, the approval status is changed back to **Draft**, and you can then modify the order.</span></span> <span data-ttu-id="531f4-173">หลังจากที่คุณทำการเปลี่ยนแปลงเสร็จสิ้นแล้ว คุณอาจจำเป็นต้องส่ง PO เพื่อขอการอนุมัติใหม่</span><span class="sxs-lookup"><span data-stu-id="531f4-173">After you’ve finished making changes, you might have to submit the PO for re-approval.</span></span> <span data-ttu-id="531f4-174">คุณสามารถตั้งค่าคอนฟิกชนิดของการเปลี่ยนแปลงที่จำเป็นต้องมีการอนุมัติใหม่ได้โดยใช้กฎนโยบาย **กฎการอนุมัติใหม่สำหรับใบสั่งซื้อ** ในหน้า **นโยบายการจัดซื้อ**</span><span class="sxs-lookup"><span data-stu-id="531f4-174">You can configure the types of changes that require re-approval by using a **Re-approval rule for purchase orders** policy rule on the **Purchasing policies** page.</span></span>  

<span data-ttu-id="531f4-175">ถ้าปริมาณที่สั่งของรายการ PO ส่วนหนี่งถูกส่ง คุณจะไม่สามารถเปลี่ยนปริมาณที่สั่งได้</span><span class="sxs-lookup"><span data-stu-id="531f4-175">If part of the ordered quantity for a PO line has been delivered, you can’t change the ordered quantity.</span></span> <span data-ttu-id="531f4-176">อย่างไรก็ตาม คุณสามารถเปลี่ยนแปลงปริมาณ **ยอดค้างส่ง** ในรายการได้</span><span class="sxs-lookup"><span data-stu-id="531f4-176">However, you can change the **Deliver remainder** quantity on the line.</span></span> <span data-ttu-id="531f4-177">จากนั้นคุณสามารถใช้การดำเนินการ **ขั้นสุดท้าย** เพื่อยกเลิกรายการและป้องกันไม่ให้มีการประมวลผลได้</span><span class="sxs-lookup"><span data-stu-id="531f4-177">You can then use the **Finalize** action to cancel lines and prevent further processing.</span></span> 

<span data-ttu-id="531f4-178">หลังจากที่ใบสั่งได้รับการยืนยันแล้ว คุณจะไม่สามารถลบได้อีกต่อไป</span><span class="sxs-lookup"><span data-stu-id="531f4-178">After an order has been confirmed, you can no longer delete it.</span></span> <span data-ttu-id="531f4-179">อย่างไรก็ตาม คุณสามารถยกเลิกปริมาณรวมหรือปริมาณใดๆ ที่เหลืออยู่บนใบสั่งได้ถ้าไม่ได้รับปริมาณดังกล่าวหรือออกใบแจ้งหนี้</span><span class="sxs-lookup"><span data-stu-id="531f4-179">However, you can cancel the total quantity or any remaining quantity on an order, provided that the quantity hasn’t been received or invoiced.</span></span>
