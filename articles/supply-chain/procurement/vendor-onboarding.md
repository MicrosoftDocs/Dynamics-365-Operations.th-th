<span data-ttu-id="2eeca-105">สามารถเตรียมความพร้อมผู้จัดจำหน่ายใหม่และลงทะเบียนเป็นผู้จัดจำหน่ายใน Microsoft Dynamics 365 for Finance and Operations, Enterprise edition ได้ตามข้อมูลที่รวบรวมจากบุคคลที่เป็นตัวแทนของผู้จัดจำหน่าย</span><span class="sxs-lookup"><span data-stu-id="2eeca-105">New vendors can be onboarded and registered as vendors in Microsoft Dynamics 365 for Finance and Operations, Enterprise edition, based on information that is collected from a person who represents the vendor.</span></span>

<span data-ttu-id="2eeca-106">กระบวนการประกอบด้วยขั้นตอนต่อไปนี้ ซึ่งบทบาทต่างๆ ทำการดำเนินการในระบบ</span><span class="sxs-lookup"><span data-stu-id="2eeca-106">The process consists of the following steps, where various roles perform actions in the system.</span></span>

1. <span data-ttu-id="2eeca-107">**การจัดการข้อมูล OData** – การนำเข้าเอนทิตี - คำขอเริ่มต้นเป็นคำขอลงทะเบียนของผู้ที่มีแนวโน้มจะเป็นผู้จัดจำหน่าย</span><span class="sxs-lookup"><span data-stu-id="2eeca-107">**Data management OData** – Entity import - The initial request is the prospective vendor registration request.</span></span> <span data-ttu-id="2eeca-108">โดยทั่วไป คำขอนี้มาจากแหล่งที่มา เช่น เว็บไซต์ที่ลูกค้าเป็นโฮสต์ที่อนุญาตให้เข้าถึงแบบไม่ระบุชื่อ</span><span class="sxs-lookup"><span data-stu-id="2eeca-108">Typically, this request comes from a source such as a customer-hosted website that allows anonymous access.</span></span> <span data-ttu-id="2eeca-109">ผู้จัดจำหน่ายสามารถลงทะเบียนได้โดยการให้ข้อมูลพื้นฐาน เช่น ชื่อผู้จัดจำหน่าย เหตุผล หมายเลของค์กร และชื่อและที่อยู่อีเมลของผู้ติดต่อ</span><span class="sxs-lookup"><span data-stu-id="2eeca-109">Vendors can sign up by providing basic information, such as the vendor name, justification, organization number, and name and email address of the contact person.</span></span> <span data-ttu-id="2eeca-110">คำขอถูกนำเข้าผ่านอินเทอร์เฟซการจัดการข้อมูล</span><span class="sxs-lookup"><span data-stu-id="2eeca-110">The requests are imported via the Data management interface.</span></span>
2. <span data-ttu-id="2eeca-111">**หน้ารายการคำขอลงทะเบียนของผู้ที่มีแนวโน้มจะเป็นผู้จัดจำหน่าย** - ยึดตามข้อมูลที่อยู่ที่ระบุในคำขอลงทะเบียนของผู้ที่มีแนวโน้มจะเป็นผู้จัดจำหน่าย ผู้เชี่ยวชาญด้านการจัดซื้อจัดหาตัดสินใจว่า ผู้จัดจำหน่ายควรมีการเตรียมความพร้อมหรือไม่</span><span class="sxs-lookup"><span data-stu-id="2eeca-111">**Prospective vendor registration request list page** - Based on the information that is provided in the prospective vendor registration request, a procurement professional decides whether the vendor should be onboarded.</span></span> <span data-ttu-id="2eeca-112">ผู้เชี่ยวชาญด้านการจัดซื้อจัดหาดูคำขอขาเข้าในหน้ารายการ **คำขอลงทะเบียนของผู้ที่มีแนวโน้มจะเป็นผู้จัดจำหน่าย** ใน Finance and Operations</span><span class="sxs-lookup"><span data-stu-id="2eeca-112">The procurement professional views the incoming request on the **Prospective vendor registration requests** list page in Finance and Operations.</span></span>
