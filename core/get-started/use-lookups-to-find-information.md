# <a name="use-lookups-to-find-information"></a>ใช้การค้นหาเพื่อค้นหาข้อมูล

ใน Microsoft Dynamics 365 สำหรับการดำเนินงาน ฟิลด์หลายฟิลด์มีการค้นหาซึ่งช่วยให้คุณสามารถค้นหาค่าถูกต้อง หรือที่ต้องการได้อย่างง่ายดาย มีการเพิ่มการส่งเสริมหลายอย่างสำหรับการค้นหาที่ทำให้ตัวควบคุมเหล่านี้สามารถใช้งานได้มากขึ้น และทำให้ผู้ใช้มีประสิทธิผลมากขึ้น ในหัวข้อนี้ คุณจะได้เรียนรู้เกี่ยวกับการค้นหาคุณลักษณะใหม่เหล่านี้ และจะได้รับเคล็ดลับบางอย่างมีประโยชน์ในการเรียกใช้ออกจากการค้นหาที่ดีที่สุดในระบบ  

<a name="responsive-lookups"></a>ค้นหาการตอบสนอง
------------------

ในเวอร์ชันก่อนหน้าของ Dynamics 365 สำหรับการดำเนินการ เมื่อมีการโต้ตอบกับตัวควบคุมการค้นหา ผู้ใช้จะต้องดำเนินการเพื่อเปิดเมนูดรอปดาวน์ชัดเจน นี่อาจได้ โดยการพิมพ์เครื่องหมายดอกจัน (\*) ในตัวควบคุมเพื่อกรองข้อมูลการค้นหาที่ขึ้นอยู่กับค่าปัจจุบันของตัวควบคุม คลิกปุ่มดรอปดาวน์ หรือโดยใช้การ**Alt**+**ลูกศรลง**แป้นพิมพ์ลัดได้ มีการปรับเปลี่ยนตัวควบคุมการค้นหาในวิธีการต่อไปได้ดี สอดคล้องกับแนวทางปฏิบัติของเว็บปัจจุบัน:

-   เมนูแบบหล่นลงของการค้นหาจะเดี๋ยวนี้เปิดโดยอัตโนมัติหลังจากการหยุดชั่วคราวเล็กพิมพ์ข้อความ มีรายการแบบหล่นลงเมนูเนื้อหาถูกกรองโดยขึ้นอยู่กับค่าของตัวควบคุมการค้นหา
    -   โปรดสังเกตว่า ลักษณะการทำงานเดิมของการเปิดรายการแบบหล่นลงโดยอัตโนมัติหลังจากพิมพ์เครื่องหมายดอกจัน (\*) ได้รับการสนับสนุน
-   หลังจากที่มีเปิดเมนูดรอปดาวน์ค้นหา ต่อไปนี้จะเกิดขึ้น:
    -   เคอร์เซอร์จะยังคงอยู่ในตัวควบคุมการค้นหา (แทนที่จะโฟกัสย้ายเข้าไปในเมนูดรอปดาวน์) ดังนั้นคุณยังคงสามารถปรับเปลี่ยนค่าของตัวควบคุม อย่างไรก็ตาม ผู้ใช้จะยังคงสามารถใช้การ**ลูกศรขึ้น**และ**ลูกศรลง**เพื่อเปลี่ยนแถวในเมนูดรอปดาวน์ และป้อนการเลือกแถวปัจจุบันในเมนูแบบหล่นลงได้
    -   เนื้อหาของเมนูดรอปดาวน์จะปรับปรุงหลังจากการปรับเปลี่ยนใด ๆ เกิดขึ้นกับค่าของตัวควบคุมการค้นหา

ตัวอย่างเช่น พิจารณาฟิลด์การค้นหาที่เรียกว่า**เมือง** 

เมื่อโฟกัสอยู่ในตัว**เมือง**ฟิลด์ คุณสามารถเริ่มค้นหาเมืองที่คุณต้องการ ด้วยการพิมพ์ตัวอักษรไม่กี่ เช่น "คอลัมน์"  หลังจากที่คุณหยุดพิมพ์ การค้นหาจะเปิดขึ้นโดยอัตโนมัติ การกรองข้อมูลเพื่อเมืองเหล่านั้นที่ขึ้นต้น ด้วย "คอลัมน์" 

[![typeaheadLookupExample](./media/typeaheadlookupexample.png)](./media/typeaheadlookupexample.png) 

ณจุดนี้ เคอร์เซอร์จะยังคงอยู่ในฟิลด์การค้นหา ถ้าคุณพิมพ์เพื่อให้มีค่าเป็น "คอลัมน์" ต่อไป ค้นหาเนื้อหาปรับปรุงโดยอัตโนมัติเพื่อให้สะท้อนถึงค่าในตัวควบคุมล่าสุด 

![updateFilterLookupExample](./media/updatefilterlookupexample.png) 

แม้ว่าโฟกัสจะยังคงอยู่ในตัวควบคุมการค้นหา คุณสามารถใช้การ**ลูกศรขึ้น**หรือ**ลูกศรลง**คีย์เพื่อเน้นแถวที่คุณต้องการเลือกได้ ถ้าคุณกด**Enter**แถวที่เน้นจะถูกเลือกจากการค้นหา และค่าของตัวควบคุมจะถูกปรับปรุง 

![changingSelectionLookup](./media/changingselectionlookup.png)

## <a name="typing-in-more-than-ids"></a>พิมพ์รหัสมากกว่า
เมื่อป้อนข้อมูล เป็นเรื่องปกติสำหรับผู้ใช้เพื่อระบุเอนทิตี เช่นลูกค้าหรือผู้ขาย ในรูปแบบของชื่อแทนที่จะเป็นตัวระบุที่แสดงถึงเอนทิตี ในรุ่นปัจจุบันของ Dynamics 365 สำหรับการดำเนินการ ค้นหามาก (แต่ไม่ทั้งหมด) อนุญาตให้ป้อนข้อมูลตามบริบทในขณะนี้ คุณลักษณะนี้มีประสิทธิภาพช่วยให้ผู้ใช้สามารถพิมพ์รหัสหรือชื่อเกี่ยวข้องลงในตัวควบคุมการค้นหา 

พิจารณาตัวอย่าง**บัญชีลูกค้า**เมื่อสร้างใบสั่งขาย ฟิลด์นี้แสดง**รหัสบัญชี**สำหรับลูกค้า แต่ผู้ใช้โดยทั่วไปต้องการป้อนผิด**ชื่อบัญชี**แทนที่จะเป็น**รหัสบัญชี**สำหรับเขตข้อมูลนี้เมื่อสร้างใบสั่งขาย เช่น "ฟอเรสต์ Wholesales" แทน "การของสหรัฐอเมริกาการ 003 "

ถ้าผู้ใช้เริ่มต้นเมื่อต้องการป้อนตัว**รหัสบัญชี**เป็นตัวควบคุมการค้นหา เมนูดรอปดาวน์จะเปิดโดยอัตโนมัติตามที่อธิบายไว้ในส่วนก่อนหน้านี้ และผู้ใช้จะเห็นการค้นหาดังแสดงด้านล่าง

[![ค้นหาตามบริบทเมื่อป้อน ID ของบัญชีลูกค้า](./media/howtocontextuallookups-1.png)](./media/howtocontextuallookups-1.png)

อย่างไรก็ตาม ผู้ใช้สามารถป้อนจุดเริ่มต้นของนอกจากนี้ในขณะนี้มี**ชื่อบัญชี**ยังได้ ถ้านี่คือการตรวจพบ จากนั้นผู้ใช้จะเห็นการค้นหาต่อไปนี้ ประกาศวิธีการ**ชื่อ**คือย้ายคอลัมน์ไปเป็นคอลัมน์แรกในการค้นหา และวิธีการค้นหาการเรียงลำดับ และกรองข้อมูลตาม**ชื่อ**คอลัมน์

[![ค้นหาตามบริบทเมื่อป้อนชื่อของลูกค้า](./media/howtocontextuallookups-2.png)](./media/howtocontextuallookups-2.png)

## <a name="using-grid-column-headers-for-more-advanced-filtering-and-sorting"></a>ใช้เส้นตารางในส่วนหัวของคอลัมน์สำหรับการกรอง และการเรียงลำดับขั้นสูง
เพิ่มประสิทธิภาพการค้นหาที่ได้รับการอธิบายในสองหัวข้อก่อนหน้าอย่างมากที่ปรับปรุงของผู้ใช้สามารถนำทางไปยังแถวในการค้นหาที่ขึ้นอยู่กับการค้นหา "ขึ้นต้นด้วย" ของการ**ID**หรือ**ชื่อ**ในการค้นหา อย่างไรก็ตาม คือสถานการณ์ที่ขั้นสูงกรอง (หรือการเรียงลำดับ) จำเป็นต้องใช้ในการค้นหาแถวถูกต้อง ในสถานการณ์เหล่านี้ ผู้ใช้อาจต้องการใช้ตัวเลือกการเรียงลำดับ และกรองข้อมูลในส่วนหัวของคอลัมน์เส้นตารางภายในการค้นหา ตัวอย่าง พิจารณาพนักงานป้อนบรรทัดใบสั่งขายที่ต้องการค้นหา "สายเคเบิล" เป็นผลิตภัณฑ์ด้านขวา พิมพ์ "สายเคเบิล" ลงใน**หมายเลขสินค้า**ควบคุมไม่ได้ประโยชน์ เนื่องจากมีอยู่ไม่มีชื่อผลิตภัณฑ์ที่ขึ้นต้น ด้วย "สายเคเบิล" 

![emptyitemlookup](./media/emptyitemlookup.png) 

แทน ผู้ใช้จำเป็นต้องล้างค่าของตัวควบคุมการค้นหา เปิดเมนูดรอปดาวน์ค้นหา และกรองเมนูแบบหล่นลงที่ใช้ส่วนหัวของคอลัมน์กริด ดังแสดงด้านล่าง เมาส์ (หรือสัมผัส) ผู้ใช้สามารถเพียงแค่คลิก (หรือสัมผัส) ส่วนหัวของคอลัมน์ใด ๆ ในการเข้าถึงการกรองและการเรียงลำดับตัวเลือกสำหรับคอลัมน์นั้น สำหรับผู้ใช้แป้นพิมพ์ ผู้ใช้เพียงแต่จำเป็นต้องกด**Alt**+**ลง****ลูกศร**เป็นครั้งที่สองเพื่อย้ายโฟกัสไปยังเมนูแบบหล่นลง หลังจากที่ผู้ใช้สามารถแตะที่คอลัมน์ถูกต้อง และจากนั้น กด**Ctrl**+**G**เพื่อเปิดเมนูแบบหล่นลงหัวข้อคอลัมน์เส้นตาราง 

[![gridfilteritemlookup](./media/gridfilteritemlookup.png)](./media/gridfilteritemlookup.png) 

หลังจากที่ได้ถูกประยุกต์ใช้ตัวกรอง (ดูรูปภาพด้านล่าง), ผู้ใช้สามารถค้นหา และเลือกแถวตามปกติ 

![filtereditemlookup](./media/filtereditemlookup.png)
