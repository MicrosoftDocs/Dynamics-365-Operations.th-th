# <a name="dynamics-365-for-operations-glossary"></a>Dynamics 365 สำหรับอภิธานศัพท์ของการดำเนินงาน

อภิธานศัพท์ Microsoft Dynamics 365 for Operations กำหนดเงื่อนไขคีย์และแนวคิดในผลิตภัณฑ์ Dynamics 365 for Operations 

### <a name="a"></a>**A**

###### <a name="accounting-journal"></a>**สมุดรายวันการลงบัญชี**

สมุดรายวันที่ใช้เพื่อบันทึกผลกระทบทางการเงินของเหตุการณ์ทางบัญชีในระบบบัญชี

###### <a name="accounting-rule"></a>**กฎทางบัญชี**

กฎในระบบบัญชีที่ควบคุมหลักการ วิธีการ และกระบวนงานสำหรับการจัดประเภท การบันทึก และการรายงานผลกระทบทางการเงินของเหตุการณ์ทางบัญชี

###### <a name="ach-automated-clearing-house"></a>**ACH (สำนักหักบัญชีอัตโนมัติ)**

ระบบการโอนเงินทางอิเล็กทรอนิกส์พร้อมใช้งานในสหรัฐอเมริกาที่ช่วยอำนวยความสะดวกในการโอนย้ายเงินระหว่างบัญชีธนาคารของฝ่ายรับ และฝ่ายต้นทาง

###### <a name="actual-quantity"></a>**actual quantity / ปริมาณที่มีอยู่จริง**

ปริมาณที่วัดที่อินพุตหรือเอาต์พุตจากกิจกรรม

###### <a name="address-verification-service"></a>**บริการตรวจสอบความถูกต้องของที่อยู่**

บริการที่ได้รับจากตัวประมวลผลบัตรเครดิตที่ตรวจสอบว่าที่อยู่ที่เรียกเก็บเงินที่ระบุโดยผู้ถือบัตรจับคู่ข้อมูลในไฟล์กับธนาคารที่ออกบัตร

###### <a name="adjusted-unit-price"></a>**ราคาหน่วยที่ปรับปรุง**

ราคาต่อหน่วยที่คำนวณได้ขึ้นอยู่กับการกำหนดราคาตามแอททริบิวต์และความแข็งแรงจริงที่ได้รับ

###### <a name="advanced-pricing"></a>**การกำหนดราคาขั้นสูง**

ตรรกะใช้ในการคำนวณราคาซื้อของสินค้าที่มีความแข็งแรง

###### <a name="aging"></a>**aging / อายุ**

กระบวนการในการจัดประเภทรอบระยะเวลาตามอายุ

###### <a name="aging-period"></a>**รอบระยะเวลาอายุหนี้**

จำนวนวันในรอบระยะเวลาที่ใช้ในการรายงานยอดดุลการชำระเงินของลูกค้าที่พ้นกำหนดชำระ

### <a name="b"></a>**B**

###### <a name="balance-sheet"></a>**balance sheet / งบดุล**

สถานะตำแหน่งทางการเงินขององค์กรที่รายงานสถานะของสินทรัพย์ หนี้สิน และหุ้นในวันที่ระบุ

###### <a name="balance-sheet-account"></a>**บัญชีงบดุล**

บัญชีที่อธิบายการเปลี่ยนแปลงในค่าที่รายงานบนงบดุล

###### <a name="bank-reconciliation"></a>**bank reconciliation / งบกระทบยอด**

แนวทางปฏิบัติของการกระทบยอดบัญชีแยกประเภทที่แสดงถึงบัญชีธนาคาร โดยการจับคู่รายการบัญชีแยกประเภทกับรายการใบแจ้งยอดจากธนาคาร

###### <a name="batch-attribute"></a>**แอททริบิวต์ของชุดงาน**

คุณลักษณะของผลิตภัณฑ์ของชุดงานการประมวลผลหรือชุดงานโอนย้าย

###### <a name="best-before-date"></a>**วันที่ควรใช้ก่อน**

วันที่แนะนำสำหรับการได้รับคุณภาพหรือคุณลักษณะสูงสุดของผลิตภัณฑ์

###### <a name="best-before-period"></a>**รอบระยะเวลาที่ควรใช้ก่อน**

รอบระยะเวลาที่จะได้รับคุณภาพหรือคุณลักษณะสูงสุดของผลิตภัณฑ์

###### <a name="bill-of-exchange"></a>**bill of exchange / ตั๋วแลกเงิน**

เอกสารต้นทางที่แสดงคำขอแบบไม่มีเงื่อนไขสำหรับบุคคลที่สามเพื่อให้ชำระฝ่ายที่สองตามความต้องการ

###### <a name="bill-of-materials"></a>**bill of materials / รายการวัสดุและส่วนประกอบ**

รายการผลิตภัณฑ์และปริมาณที่จำเป็นในการผลิตผลิตภัณฑ์หนึ่ง

###### <a name="blocking"></a>**การบล็อค**

การดำเนินการของการระงับเอกสารหรือผลิตภัณฑ์

###### <a name="budget-control"></a>**การควบคุมงบประมาณ**

แนวทางปฏิบัติในการอนุมัติรายจ่ายเฉพาะเมื่อสามารถจองเงินงบประมาณเพื่อให้ตรงกับข้อผูกมัดในการชำระเงินในอนาคต

###### <a name="budget-control-dimension"></a>**มิติการควบคุมงบประมาณ**

ชุดของค่ามิติทางการเงินที่ใช้งานอยู่ ซึ่งใช้ในการปันส่วนเงินงบประมาณที่จะชำระสำหรับกิจกรรมที่วางแผนไว้

###### <a name="budget-control-rule"></a>**กฎการควบคุมงบประมาณ**

การเข้ารหัสการตัดสินใจทางธุรกิจเพื่อตรวจสอบรายจ่ายตามกำหนดและรายจ่ายจริงเทียบกับเงินงบประมาณที่มีซึ่งปันส่วนสำหรับกิจกรรมโดยละเอียดหรือที่รวม ที่กำหนดโดยชุดของค่ามิติการควบคุมงบประมาณที่ถูกต้อง

###### <a name="budget-cycle-time-span"></a>**ระยะเวลาวงจรงบประมาณ**

รอบระยะเวลาระบุเป็นจำนวนของรอบระยะเวลาปฏิทินทางบัญชี รอบระยะเวลางบประมาณอาจแตกต่างจากรอบระยะเวลาทางบัญชี

###### <a name="budget-group"></a>**กลุ่มงบประมาณ**

ชุดของค่ามิติทางการเงินในลำดับชั้นมิติงบประมาณที่ใช้คำนวณเงินงบประมาณรวมที่ปันส่วนให้กับค่ามิติทางการเงินย่อย โดยรวมเงินงบประมาณที่ปันส่วนให้กับค่ามิติทางการเงินย่อย

###### <a name="budget-model"></a>**budget model / รูปแบบจำลองงบประมาณ**

โครงสร้างการวางแผนที่ใช้เพื่อจัดกำหนดการปันส่วนเงินงบประมาณและค่าใช้จ่าย

###### <a name="bulk-item"></a>**สินค้าจำนวนมาก**

ป้อนข้อมูลสินค้าตามสูตรลงในกิจกรรมการจัดส่งผลิตภัณฑ์

###### <a name="bundle"></a>**bundle / กลุ่มงาน**

ชุดของหมายเลขผลิตภัณฑ์สำหรับการขายเป็นหน่วยเดียวกัน

###### <a name="business-unit"></a>**หน่วยธุรกิจ**

หน่วยปฏิบัติงานกึ่งอิสระที่สร้างขึ้นเพื่อให้ตรงตามวัตถุประสงค์เชิงกลยุทธ์ทางธุรกิจ

### <a name="c"></a>**C**

###### <a name="capacity-load"></a>**capacity load / การใช้กำลังการผลิต**

ยอดเงินสูงสุดของงานตามกำหนดการที่ศูนย์การผลิตสามารถทำได้ในระดับกำลังการผลิตที่จำเป็น

###### <a name="capacity-planning"></a>**capacity planning / การวางแผนกำลังการผลิต**

กระบวนงานสำหรับการกำหนดความต้องการกำลังการผลิตของทรัพยากรที่ตรงกับความต้องการของเอาต์พุตในอนาคตในระหว่างรอบระยะเวลาเฉพาะ

###### <a name="carry-forward-budget"></a>**งบประมาณยกไป**

งบประมาณที่โอนย้ายจากปีบัญชีหนึ่งไปยังปีบัญชีถัดไป และที่จองไว้สำหรับใบสั่งซื้อที่เปิดในปีบัญชีใหม่

###### <a name="cash-generating-unit"></a>**หน่วยสร้างเงินสด**

กลุ่มเล็กที่สุดที่จัดกลุ่มโดยสินทรัพย์ที่สร้างเงินสดอย่างเป็นอิสระจากสินทรัพย์อื่นๆภายในองค์กร กลุ่มสินทรัพย์เหล่านี้จะใช้เพื่อวัดผลขาดทุนที่จะเกิดขึ้น

###### <a name="catch-weight"></a>**น้ำหนักจริง**

น้ำหนักจริง หรือที่ตั้งไว้ของสินค้าที่ขายหรือสินค้าคงคลัง

###### <a name="chart-of-accounts"></a>**chart of accounts / ผังบัญชี**

รายการบัญชีหลัก

###### <a name="cogs-cost-of-goods-sold"></a>**COGS (ต้นทุนขาย)**

ประเภทบัญชีที่ใช้เพื่อรวมลำดับทางการเงินของการผลิตผลิตภัณฑ์และการจัดการสินค้าคงคลัง

###### <a name="commodity-pricing"></a>**การกำหนดราคาตามโภคภัณฑ์**

กลยุทธ์การกำหนดราคาสำหรับการอัพเดตราคาของสินค้าขายที่ผลิตจากสินค้าที่อยู่ภายใต้การเปลี่ยนแปลงราคาที่ใช้บ่อย

###### <a name="consolidated-order"></a>**ใบสั่งที่รวมบัญชี**

เอกสารต้นทางที่แสดงใบสั่งอ้างอิงมากกว่าหนึ่งใบ

###### <a name="contractor"></a>**contractor / ผู้รับเหมา**

บทบาทที่ถือว่าบุคคลที่มีส่วนร่วมในความสัมพันธ์แบบผู้รับเหมา-พนักงานกับนิติบุคคล

###### <a name="co-product"></a>**ผลิตภัณฑ์ร่วม**

สินค้าที่ผลิตร่วมกับสินค้าอื่น

###### <a name="cost-center"></a>**ศูนย์ต้นทุน**

หน่วยปฏิบัติงานที่ซึ่งผู้จัดการจะรับผิดชอบรายจ่ายตามงบประมาณหรือตามจริง

###### <a name="cost-of-goods-sold"></a>**ต้นทุนขาย**

ประเภทบัญชีที่ใช้เพื่อรวมลำดับทางการเงินของการผลิตผลิตภัณฑ์และการจัดการสินค้าคงคลัง

###### <a name="cost-variance"></a>**ผลต่างต้นทุน**

ผลต่างระหว่างต้นทุนจริงและต้นทุนที่คาดไว้

###### <a name="costing"></a>**การคิดต้นทุน**

กระบวนการคำนวณ กำหนด และการปันส่วนต้นทุนของทรัพยากรทางเศรษฐกิจที่ได้มา ที่ผลิต หรือส่งโดยเรียงตามองค์กร

###### <a name="ctp-capable-to-promise"></a>**CTP (ปริมาณที่สามารถสัญญาได้)**

ส่วนของผลลัพธ์ผลิตภัณฑ์จากทรัพยากรการดำเนินงานที่พร้อมใช้งานและป้อนข้อมูลผลิตภัณฑ์พร้อมใช้งานที่จำเป็นเพื่อตอบสนองความต้องการเฉพาะลูกค้า

###### <a name="currency-code"></a>**รหัสสกุลเงิน**

รหัสตัวอักษรและตัวเลขที่แสดงถึงหน่วยสกุลเงิน

###### <a name="cycle-time"></a>**เวลาวงจร**

เวลาที่ใช้ในการทำกิจกรรม

### <a name="d"></a>**D**

###### <a name="dashboard"></a>**แดชบอร์ด**

หน้าเริ่มต้นทั่วไปใน Dynamics 365 for Operations ในแดชบอร์ด ผู้ใช้เห็นส่วนสำหรับพื้นที่ทำงานแต่ละพื้นที่ที่มีการเข้าถึง แต่ละส่วนแสดงไทล์จากส่วนสรุปในพื้นที่ทำงานที่เกี่ยวข้อง แดชบอร์ดประกอบด้วยชื่อและส่วนที่มีไทล์

###### <a name="data-entity"></a>**เอนทิตี้ข้อมูล**

โครงสร้างข้อมูลที่แสดงถึงลักษณะข้อมูลของเอนทิตี้

###### <a name="delegate"></a>**การมอบหมาย**

การมอบหมายเป็นชนิดที่แสดงถึงการอ้างอิงถึงวิธีการที่มีรายการพารามิเตอร์และชนิดการส่งคืนเฉพาะ

###### <a name="delivery-note"></a>**บันทึกการจัดส่ง**

เอกสารทางธุรกิจที่แสดงการส่งมอบผลิตภัณฑ์ระหว่างสองฝ่าย

###### <a name="demand-forecast"></a>**การคาดการณ์ความต้องการ**

การคาดการณ์ความต้องการสินค้าในอนาคต

###### <a name="demand-forecasting"></a>**การคาดการณ์ความต้องการ**

กระบวนการทางธุรกิจที่ประเมินความต้องการในอนาคต และสร้างการคาดการณ์ความต้องการตามข้อมูลธุรกรรมในอดีต

###### <a name="deployable-package"></a>**แพคเกจที่สามารถปรับใช้ได้**

พาหนะที่ใช้สำหรับการปรับใช้ในการทดสอบการยอมรับของผู้ใช้ (UAT) และสภาพแวดล้อมการผลิต

###### <a name="designer"></a>**ผู้ออกแบบ**

เครื่องมือ Visual Studio ที่คุณใช้ในการสร้าง อัพเดต และตรวจสอบองค์ประกอบแบบจำลองของคุณ

###### <a name="financial-dimensionsdimension-based-configuration"></a>**.. การตั้งค่าคอนฟิก \financial-dimensions\dimension-based**

เทคโนโลยีการตั้งค่าคอนฟิกที่ใช้ในการสร้างผลิตภัณฑ์ย่อย โดยการเลือกค่าสำหรับมิติของผลิตภัณฑ์

###### <a name="distinct-product"></a>**ผลิตภัณฑ์เฉพาะ**

ผลิตภัณฑ์ที่ระบุได้ไม่ซ้ำกัน

###### <a name="document-blocking"></a>**การบล็อคเอกสาร**

วิธีการระงับกระบวนการดำเนินงานในขณะที่สินค้าที่สั่งอยู่ในกระบวนการรับประกันคุณภาพ

###### <a name="document-management-system"></a>**ระบบการจัดการเอกสาร**

บริการแอพลิเคชันสำหรับการจัดเก็บและจัดการเอกสารขององค์กร

###### <a name="duty"></a>**หน้าที่**

ในแบบจำลองความปลอดภัย ชุดสิทธิ์การเข้าถึงแอพลิเคชันที่จำเป็นสำหรับผู้ใช้เพื่อดำเนินการความรับผิดชอบของพวกเขา

### <a name="e"></a>**E**

###### <a name="eft-electronic-funds-transfer"></a>**EFT (การโอนเงินทางอิเล็กทรอนิกส์)**

ระบบเครือข่ายสำหรับการโอนย้ายเงินจากบัญชีธนาคารหนึ่งไปยังอีกบัญชีหนึ่ง

###### <a name="event"></a>**เหตุการณ์**

เหตุการณ์ที่ให้คลาสหรือวัตถุหนึ่งระบุคลาสหรือวัตถุอื่นๆเมื่อมีความสนใจบางอย่างเกิดขึ้น

###### <a name="epe-every-product-every"></a>**EPE (รอบการผลิตประจำ)**

แนวคิดแบบ Lean ที่ใช้เพื่อสร้างวงจรการผลิตซ้ำแบบปกติ

###### <a name="extension"></a>**การขยาย **

การตั้งค่าแอพลิเคชันด้วยตนเองโดยการเพิ่มฟังก์ชันไปยังรหัสที่มีอยู่

### <a name="f"></a>**ศ.**

###### <a name="financial-dimension"></a>**มิติทางการเงิน**

ตัวจำแนกข้อมูลทางการเงินที่สร้างขึ้นจากฝ่าย ที่ตั้ง ผลิตภัณฑ์ และกิจกรรมในองค์กร และใช้สำหรับการรายงานการจัดการ

###### <a name="financial-dimension-value"></a>**ค่ามิติทางการเงิน**

องค์ประกอบข้อมูลในโดเมนของมิติทางการเงิน

###### <a name="financial-statement"></a>**งบการเงิน**

รายงานที่แสดงข้อมูลทางการเงินและตำแหน่งทางการเงินขององค์กร

###### <a name="fixed-cost"></a>**fixed cost / ต้นทุนคงที่**

ต้นทุนที่ไม่แตกต่างกันด้วยการเปลี่ยนแปลงในปริมาณการจัดส่งผลิตภัณฑ์หรือเอาต์พุตการจัดส่งผลิตภัณฑ์

###### <a name="fixed-currency"></a>**สกุลเงินคงที่**

สกุลเงินที่มีอัตราแลกเปลี่ยนคงที่โดยสัมพันธ์กับสกุลเงินอื่น

###### <a name="fixed-quantity-kanban"></a>**คัมบังแบบปริมาณคงที่**

ชนิดของคัมบังที่จะใช้เมื่อกำหนดจำนวนคัมบังให้กับกฎคัมบังที่เป็นค่าคงที่

###### <a name="flexible-authentication"></a>**การรับรองความถูกต้องแบบยืดหยุ่น**

กระบวนการตรวจสอบที่อาจสามารถพิสูจน์ตัวจริงผู้ใช้ในระบบการจัดการข้อมูลประจำตัวที่หลากหลาย

###### <a name="formula"></a>**สูตร**

ความสัมพันธ์ทางตัวเลขระหว่างอินพุตกระบวนการผลิตและเอาต์พุตของกระบวนการผลิต

###### <a name="formula-item"></a>**สินค้าตามสูตร**

เอาต์พุตของกระบวนการชุดงานที่ควบคุมโดยสูตร

### <a name="g"></a>**G**

###### <a name="general-budget-reservation"></a>**การจองงบประมาณทั่วไป**

เอกสารที่มักใช้โดยภาครัฐที่กันไว้หรือทำเครื่องหมายเงินงบประมาณเพื่อให้เงินดังกล่าวไม่พร้อมใช้งานสำหรับวัตถุประสงค์อื่น

###### <a name="grace-period"></a>**ระยะปลอดหนี้**

รอบระยะเวลาเกินจากวันที่ระบุในระหว่างที่สามารถปฏิบัติตามข้อผูกมัดได้โดยไม่มีค่าปรับ

###### <a name="gst-goods-and-services-tax"></a>**GST (ภาษีสินค้าและบริการ)**

ภาษีมูลค่าเพิ่มที่เก็บในบางประเทศ/ภูมิภาค

### <a name="i"></a>**I**

###### <a name="intercompany"></a>**intercompany / ระหว่างบริษัท**

เกิดขึ้นระหว่างหรือเกี่ยวข้องกับนิติบุคคลอย่างน้อยสองรายที่เป็นส่วนหนึ่งองค์กรเดียวกันที่รวมบัญชีของนิติบุคคลทั้งหมด

###### <a name="invoice-matching"></a>**การจับคู่ใบแจ้งหนี้**

แนวทางปฏิบัติของการจับคู่ราคาใบแจ้งหนี้ของผู้จัดจำหน่ายและปริมาณผลิตภัณฑ์กับใบสั่งซื้อและใบรับสินค้า

###### <a name="item-allocation-key"></a>**คีย์การปันส่วนสินค้า**

การจัดกลุ่มตระกูลผลิตภัณฑ์ที่ใช้สำหรับการคาดการณ์และการจัดกำหนดการความต้องการ

###### <a name="item-relation"></a>**ความสัมพันธ์ของสินค้า**

การอ้างอิงถึงกลุ่มการปันส่วนสินค้า หรือสินค้า และมิติของผลิตภัณฑ์ในกฎคัมบัง

### <a name="k"></a>**K**

###### <a name="kanban"></a>**คัมบัง**

สัญญาณที่สื่อสารความต้องการสำหรับปริมาณของผลิตภัณฑ์

###### <a name="kanban-flow"></a>**ขั้นตอนคัมบัง**

กำหนดลำดับของกิจกรรมที่มีการดำเนินการสำหรับคัมบังที่ถูกสร้างขึ้นสำหรับกฎคัมบัง

###### <a name="kanban-job"></a>**งานคัมบัง**

กระบวนการหรือกิจกรรมการโอนย้ายในขั้นตอนการผลิตที่ถูกทริกเกอร์โดยคัมบัง

###### <a name="kanban-job-consumption"></a>**ปริมาณการใช้วัสดุในงานคัมบัง**

การเบิกถอนส่วนประกอบของผลิตภัณฑ์จากสินค้าคงคลังเพื่อทำให้เสร็จงานการผลิตคัมบัง

###### <a name="kanban-product-quantity"></a>**ปริมาณผลิตภัณฑ์ตามระบบคัมบัง**

จำนวนของข้อกำหนดสินค้าที่ดำเนินการในคัมบังที่ยึดตามกฎคัมบัง

###### <a name="kanban-rule"></a>**กฎคัมบัง**

กฎในระบบ lean manufacturing ที่ดำเนินการตามนโยบายการวางแผนและการเติมสินค้าวัสดุ ด้วยการควบคุมการประสานงานกิจกรรมกระบวนการและการโอนย้ายในขั้นตอนการผลิต

### <a name="l"></a>**L**

###### <a name="lean-manufacturing"></a>**การผลิตแบบ Lean**

หลักการในการดำเนินการผลิตเลื่อนระดับขั้นตอนการผลิตแบบ lean และกิจกรรมทางธุรกิจ

###### <a name="lean-schedule-group"></a>**กลุ่มกำหนดการแบบลีน**

วิธีในการรวมสินค้าสำหรับการผลิต ตัวอย่างเช่น ตามกลุ่มการตั้งค่า กลุ่มการจัดส่ง หรือกลุ่มการขนส่ง

###### <a name="ledger-account"></a>**ledger account / รหัสบัญชี**

ตัวจำแนกที่สร้างขึ้นจากชุดของบัญชีหลักและค่ามิติทางการเงินอื่นๆ ที่แสดงรายการในผังบัญชี และใช้สำหรับจัดประเภทผลกระทบทางการเงินของกิจกรรมด้านเศรษฐกิจ

###### <a name="license-code"></a>**รหัสลิขสิทธิ์**

คีย์อักษรและตัวเลขที่มอบสิทธิ์ให้แก่ฝ่ายเพื่อใช้ซอฟต์แวร์หรือส่วนประกอบของซอฟต์แวร์ตามที่อธิบายไว้ในเงื่อนไขของข้อตกลงลิขสิทธิ์ และที่เรียกใช้ และยกเลิกการเรียกใช้โมดูลซอฟต์แวร์ ความสามารถของซอฟต์แวร์ และฟังก์ชันซอฟต์แวร์

###### <a name="liquidity"></a>**liquidity / สภาพคล่อง**

ความสามารถของบุคคลที่จะใช้สินทรัพย์หมุนเวียนเพื่อจ่ายหนี้สินหมุนเวียน

### <a name="m"></a>**จ.**

###### <a name="manufacturing-overhead-based-on-material"></a>**ค่าโสหุ้ยการผลิตตามวัสดุ**

ค่าโสหุ้ยการผลิตจะถูกแสดงได้ดังสูตรการคำนวณต้นทุนทางอ้อมสำหรับเปอร์เซ็นต์การเก็บเงินเพิ่ม กลุ่มต้นทุนที่ถูกกำหนดให้กับส่วนประกอบวัสดุที่ทำหน้าที่เป็นข้อมูลพื้นฐานสำหรับสูตรการคำนวณ สามารถกำหนดสูตรการคำนวณสำหรับผลิตภัณฑ์แต่ละรายการ หรือสำหรับกลุ่มสินค้าที่ถูกกำหนดให้กับผลิตภัณฑ์ได้ เพื่อให้เพิ่มความยืดหยุ่นในการคำนวณค่าโสหุ้ยการผลิตที่ขึ้นกับวัสดุ

###### <a name="master-scheduling"></a>**master scheduling / การวางแผนหลัก**

กระบวนการสำหรับการสร้างตารางเวลาสำหรับการจับคู่การจัดหาวัสดุความต้องการ

###### <a name="model"></a>**แบบจำลอง**

แบบจำลองประกอบด้วยองค์ประกอบรหัสและข้อมูลเมตาของการอ้างอิง จะถูกสร้างและตรวจสอบใน Visual Studio

###### <a name="model-element"></a>**องค์ประกอบแบบจำลอง**

แบบจำลองคือ กลุ่มขององค์ประกอบ (ไฟล์ต้นฉบับและข้อมูลเมตา) ซึ่งเป็นส่วนหนึ่งของโซลูชันของซอฟต์แวร์แบบกระจายได้ แบบจำลองเป็นแนวคิดการออกแบบเวลา

### <a name="n"></a>**N**

###### <a name="nbv-net-book-value"></a>**NBV (มูลค่าตามบัญชีสุทธิ)**

ค่าของสินทรัพย์ถาวรที่คำนวณเป็นผลต่างระหว่างต้นทุนเดิมของสินทรัพย์ถาวรลบด้วยค่าเสื่อมราคาที่สะสม

### <a name="o"></a>**O**

###### <a name="object-permission"></a>**สิทธิ์ของออบเจ็กต์**

การดำเนินการสร้าง อ่าน อัพเดต ลบ หรือดำเนินการที่ได้รับอนุญาตบนออบเจ็กต์ที่กำหนดสิทธิ์ได้

###### <a name="operating-unit"></a>**หน่วยปฏิบัติงาน**

องค์กรที่แบ่งการควบคุมทรัพยากรทางเศรษฐกิจและกระบวนการในการดำเนินงานระหว่างบุคคลที่มีหน้าที่การใช้ทรัพยากรที่มีขาดแคลนให้เกิดประโยชน์สูงสุด เพื่อปรับปรุงกระบวนการ และลงบัญชีสำหรับประสิทธิภาพ

###### <a name="output-product"></a>**ผลิตภัณฑ์ที่ผลิตได้**

ผลิตภัณฑ์ทางกายภาพที่เป็นผลมาจากกิจกรรม

###### <a name="overlayering"></a>**การจัดทำชั้นที่มากเกินไป**

วิธีการในการกำหนดรหัสต้นทางโดยมีรหัสต้นทางใหม่ที่แทนรหัสต้นทางค่าเริ่มต้น

### <a name="p"></a>**P**

###### <a name="package"></a>**บรรจุภัณฑ์**

แพคเกจคือ หน่วยที่สามารถรวบรวมและสามารถปรับใช้ได้ของแบบจำลองหนึ่งรายการขึ้นไป

###### <a name="party"></a>**ฝ่าย**

เอนทิตี้ซึ่งรับบทบาทผูกพันทางกฎหมายเมื่อเข้าร่วมในความสัมพันธ์ตามสัญญา

###### <a name="party"></a>**ฝ่าย**

บุคคลหรือองค์กรที่เข้าร่วมในกิจกรรมทางเศรษฐกิจ

###### <a name="pegging"></a>**pegging / จำแนกประเภท**

กระบวนการในการสืบค้นกลับปริมาณความต้องการสินค้าเพื่อไปยังแหล่งที่มา

###### <a name="pegging-event"></a>**เหตุการณ์การเชื่อมโยงความต้องการกับการจัดซื้อ**

เหตุการณ์กระแสทรัพยากรที่ส่งสัญญาณถึงความต้องการสำหรับผลิตภัณฑ์

###### <a name="phantom-item"></a>**สินค้าแฝง**

สินค้าที่ผลิตได้แต่ไม่ได้เก็บในสินค้าคงคลัง

###### <a name="planned-intercompany-demand"></a>**ความต้องการระหว่างบริษัทที่วางแผนไว้**

ความต้องการที่คาดไว้สำหรับผลิตภัณฑ์โดยนิติบุคคลซึ่งสมมติบทบาทของผู้จัดจำหน่าย ความต้องการที่มีการวางแผนถูกสร้างขึ้นจากความต้องการที่มีการวางแผนสำหรับผลิตภัณฑ์จากนิติบุคคลปลายน้ำ

###### <a name="post"></a>**ลงรายการบัญชี**

การบันทึกมูลค่าเงินของเหตุการณ์ทางเศรษฐกิจในบัญชีเฉพาะ หรือเพื่อสรุป และการจัดประเภทใหม่รายการบัญชีสมุดรายวันทั่วไป และบริษัทในเครือไปยังรายการบัญชีแยกประเภททั่วไป และบริษัทในเครือ

###### <a name="preparer"></a>**ผู้จัดเตรียม**

บุคคลที่สร้างเอกสารต้นทางเมื่อต้องการเริ่มต้นคำขอสำหรับทรัพยากรทางเศรษฐกิจ

###### <a name="privilege"></a>**สิทธิ์**

สิทธิ์ที่ผู้ใช้ต้องการในการดำเนินงานอย่างน้อยหนึ่งสิทธิ์

###### <a name="procurement-catalog"></a>**แค็ตตาล็อกการจัดซื้อ**

การแสดงรายการของข้อเสนอของผลิตภัณฑ์ที่ถูกจัดกลุ่มตามประเภทการจัดซื้อ แค็ตตาล็อกการจัดซื้อใช้ในการร้องขอผลิตภัณฑ์สำหรับการใช้ภายในโดยองค์กร

###### <a name="product-component"></a>**ส่วนประกอบของผลิตภัณฑ์**

เป็นส่วนหนึ่งของผลิตภัณฑ์

###### <a name="product-dimension"></a>**มิติของผลิตภัณฑ์**

ขนาด สี หรือการกำหนดค่าคุณลักษณะของผลิตภัณฑ์ที่ใช้สำหรับ... การตั้งค่าคอนฟิก \financial-dimensions\dimension-based

###### <a name="product-family"></a>**ตระกูลผลิตภัณฑ์**

การจัดกลุ่มเฉพาะของสินค้า บริการ หรือสิทธิ์ที่อย่างใดอย่างหนึ่งเข้าร่วมในกิจกรรมการผลิตหรือการจัดส่งเดียวกันหรือที่มีเสนอเซ็กเมนต์ตลาดเดียวกัน  ใน Dynamics 365 for Operations การจัดกลุ่มจะถูกแสดงโดยใช้คีย์การปันส่วนการคาดการณ์

###### <a name="product-master"></a>**ผลิตภัณฑ์หลัก**

นำเสนอผลิตภัณฑ์มาตรฐานหรือทำงานที่เป็นข้อมูลพื้นฐานสำหรับการตั้งค่าคอนฟิกผลิตภัณฑ์ย่อย

###### <a name="product-receipt"></a>**ใบรับสินค้า**

เอกสารต้นทางที่แสดงการรับผลิตภัณฑ์ที่สั่ง การรับผลิตภัณฑ์ที่ส่งคืน หรือการรับผลิตภัณฑ์ที่ได้รับในการส่งมอบ

###### <a name="product-variant"></a>**ผลิตภัณฑ์ย่อย**

การตั้งค่าคอนฟิกผลิตภัณฑ์หลัก

###### <a name="production-flow"></a>**ขั้นตอนการผลิต**

กระบวนการผลิตที่ออกแบบโดยใช้หลักการแบบ Lean

###### <a name="production-flow-model"></a>**แบบจำลองขั้นตอนการผลิต**

แสดงกำลังการผลิตที่ได้จากกลุ่มของเซลล์ทำงานในกระบวนการขั้นตอนการผลิต

###### <a name="production-order"></a>**production order / ใบสั่งผลิต**

เอกสารต้นทางที่แสดงความต้องการสำหรับการผลิตสินค้าเพื่อให้ตรงกับความต้องการ

###### <a name="production-schedule"></a>**ตารางการผลิต**

กำหนดการในการผลิตสินค้าเฉพาะและปริมาณสินค้า ณ เวลาที่ระบุ และทรัพยากรบุคคล และการดำเนินงานที่เฉพาะเจาะจง

###### <a name="prospect"></a>**prospect / ผู้ที่มีแนวโน้มจะเป็นลูกค้า**

ผู้เข้าร่วมที่มีความสามารถที่มีอยู่หรืออาจมีในการให้บริการหรืออาจให้ผลประโยชน์ทางเศรษฐกิจในอนาคตแก่นิติบุคคล

###### <a name="purchase-agreement-policy"></a>**นโยบายข้อตกลงการซื้อ**

นโยบายที่อนุมัติให้ฝ่ายสามารถแก้ไขเงื่อนไขข้อตกลงการซื้อ

###### <a name="purchase-order-policy"></a>**นโยบายการสั่งซื้อ**

นโยบายที่อนุมัติให้บริษัท เพื่อแก้ไขเงื่อนไขการสั่งซื้อ และ เพื่อควบคุมการดำเนินการกับใบสั่ง

###### <a name="purchase-quotation"></a>**ใบเสนอราคาซื้อ**

เอกสารต้นทางที่แสดงข้อเสนอการซื้อปริมาณสินค้าสำหรับราคาที่ระบุ และ ตามวันที่ที่ระบุในการตอบสนองกับคำขอใบเสนอราคาในกระบวนการจัดซื้อ

###### <a name="purchase-requisition"></a>**ใบขอซื้อ**

เอกสารต้นทางที่แสดงคำขอผลิตภัณฑ์เพื่อให้สามารถส่งเพื่อตรวจทาน และสามารถใช้เพื่ออนุมัติการซื้อ โดยองค์กรจัดซื้อ

### <a name="q"></a>**Q**

###### <a name="quantity-variance"></a>**ผลต่างปริมาณ**

ผลต่างระหว่างปริมาณผลิตภัณฑ์ที่สั่งและปริมาณผลิตภัณฑ์ที่ได้รับเมื่อไม่ได้คาดการรับสินค้า

### <a name="r"></a>**R**

###### <a name="receipt-advice"></a>**ใบแจ้งการรับสินค้า**

เอกสารทางธุรกิจที่แสดงสรุปของผลิตภัณฑ์ที่ผู้ซื้อได้รับจากผู้จัดจำหน่าย

###### <a name="reconciliation"></a>**การกระทบยอด**

แนวทางปฏิบัติของการปรับปรุงบัญชีหรือใบแจ้งยอดอย่างน้อยสองรายการเพื่อให้ตัวเลขสอดคล้อง

###### <a name="reduction-key"></a>**เงื่อนไขสำคัญของการลด**

วิธีการที่ใช้เพื่อเพิ่มหรือลดความต้องการการคาดการณ์ในการวางแผนหลัก ตามเปอร์เซ็นต์การกำหนดโดยผู้ใช้ที่จะใช้ในระหว่างรอบระยะเวลา

###### <a name="register"></a>**ลงทะเบียน**

เรกคอร์ดที่ใช้เพื่อบันทึกผลกระทบการดำเนินการ ทางกฎหมาย และทางการเงินของเหตุการณ์กระแสทรัพยากรในระบบบัญชี

###### <a name="regulated-product"></a>**สินค้าควบคุม**

ผลิตภัณฑ์ที่ควบคุมโดยหน่วยงานราชการที่ต้องปฏิบัติตามกฎระเบียบและการรายงานการตรวจสอบบัญชี

###### <a name="rfq-request-for-quotation"></a>**RFQ (คำขอใบเสนอราคา)**

เอกสารต้นทางที่แสดงการเชิญประมูลการจัดหาวัสดุปริมาณของผลิตภัณฑ์ สำหรับราคาที่ระบุและตามวันที่ที่ระบุ

###### <a name="requester"></a>**ผู้ขอ**

บุคคลที่ร้องขอทรัพยากรทางเศรษฐกิจ

###### <a name="restricted-product"></a>**สินค้าจำกัด**

ผลิตภัณฑ์ที่การกระจายจะจำกัด หรือมีข้อจำกัดโดยหน่วยงานราชการ

### <a name="s"></a>**ส.**

###### <a name="sales-agreement"></a>**ข้อตกลงการขาย**

เอกสารต้นทางที่แสดงข้อตกลงระหว่างบริษัทอย่างน้อยสองฝ่ายตามความเข้าใจว่า จะยืนยันฝ่ายขายเพื่อขายเฉพาะปริมาณหรือมูลค่าของผลิตภัณฑ์ในรอบระยะเวลาเพื่อแลกเปลี่ยนกับราคาและส่วนลดที่พึงพอใจ

###### <a name="sales-agreement-policy"></a>**นโยบายข้อตกลงการขาย**

นโยบายที่อนุมัติให้ฝ่ายสามารถแก้ไขเงื่อนไขข้อตกลงการขาย

###### <a name="sales-order-policy"></a>**นโยบายการสั่งขาย**

นโยบายที่อนุมัติให้ฝ่ายแก้ไขเงื่อนไขใบสั่งขาย และควบคุมการดำเนินการกับใบสั่งและกระบวนการชำระเงิน

###### <a name="sales-quotation"></a>**sales quotation / ใบเสนอราคาขาย**

เอกสารต้นทางที่แสดงข้อเสนอการจัดหาปริมาณสินค้าสำหรับราคาที่ระบุ และ ตามวันที่ที่ระบุในการตอบสนองกับคำขอใบเสนอราคาในกระบวนการขาย

###### <a name="security-role"></a>**บทบาทความปลอดภัย**

ชุดที่กำหนดของสิทธิ์การเข้าถึงแอพลิเคชัน บทบาทความปลอดภัยที่กำหนดให้กับผู้ใช้กำหนดว่างานใดที่ผู้ใช้สามารถดำเนินการ และส่วนใดของอินเทอร์เฟสผู้ใช้ที่ผู้ใช้สามารถดูได้ ผู้ใช้ทั้งหมดต้องถูกกำหนดบทบาทความปลอดภัยอย่างน้อยหนึ่งรายการเพื่อให้เข้าถึงระบบ

###### <a name="segregation-of-duties"></a>**การแบ่งแยกหน้าที่**

หลักการออกแบบที่ใช้ในการลดความเสี่ยงของการฉ้อโกง ความผิดปกติ และข้อผิดพลาดที่แบ่งการจัดทำเรกคอร์ด การตรวจสอบ การอนุมัติ การดูแลสินทรัพย์ และการตรวจทานหน้าที่ของบุคคลที่เข้าร่วมใน เอกสาร หรือบันทึกผลกระทบทางการเงินของธุรกรรมทางเศรษฐกิจเป็นประจำ

###### <a name="sequencing"></a>**การจัดลำดับ**

ใบสั่งที่ดำเนินงานหรือการดำเนินงานจะทำได้ในสิ่งอำนวยความสะดวกการผลิตเพื่อให้บรรลุวัตถุประสงค์

###### <a name="shared-asset"></a>**สินทรัพย์ที่ใช้ร่วมกัน**

สินทรัพย์ที่ถูกใช้โดยหน่วยสร้างเงินสด (CGU) มากกว่าหนึ่งหน่วย ตัวอย่างคือ ศูนย์กระจายสินค้าที่ใช้ในการจัดเก็บสินค้าก่อนที่จะมีการขนส่งไปยังตลาดที่แตกต่างกันที่ใช้ศูนย์กระจายสินค้าเดียวกัน

###### <a name="single-use-kanban"></a>**คัมบังแบบใช้เฉพาะครั้ง**

ชนิดของคัมบังที่ถูกใช้กับกฎคัมบังแบบปริมาณคงที่เพื่อให้ตรงกับความต้องการอย่างสูงเป็นพิเศษ คัมบังแบบใช้เฉพาะครั้งไม่ทริกเกอร์คัมบังใหม่เมื่อถูกละทิ้ง

###### <a name="smart-rounding"></a>**การกำหนดราคาแบบมีเศษ**

แนวทางปฏิบัติการตลาดสามารถใช้ตัวเลขคี่ที่น้อยกว่าตัวเลขที่ใกล้ที่สุดอย่างมีขอบเขตเพื่อตั้งค่าราคา

###### <a name="source-requirement"></a>**แหล่งที่มาของความต้องการ**

ปริมาณผลิตภัณฑ์ที่จัดทำเอกสารบนรายการเอกสารต้นทางที่สร้างข้อกำหนดการเชื่อมโยงความต้องการกับการจัดซื้อ

###### <a name="statistical-baseline-forecast"></a>**การคาดการณ์พื้นฐานทางสถิติ**

ประเมินความต้องการในอนาคตที่สร้างขึ้นโดยใช้อัลกอริทึมการคาดการณ์กับข้อมูลธุรกรรมในอดีต

###### <a name="supermarket"></a>**ซุปเปอร์มาร์เก็ต**

สถานที่เก็บสินค้าคงคลังแบบทันเวลาที่มีกำลังการผลิตจำกัด และใช้กับวัสดุชุดงานใกล้กับจุดของการใช้งาน

###### <a name="system-defined-financial-dimension-type"></a>**ชนิดมิติทางการเงินที่ระบบกำหนด**

มิติทางการเงินที่แม็ปเข้ากับตารางหรือมุมมองในฐานข้อมูลที่แสดงถึงชนิดเอนทิตี้และค่าที่อยู่ในโดเมนของแอททริบิวต์หนึ่งของชนิดเอนทิตี้นั้น

### <a name="t"></a>**อ.**

###### <a name="takt-time"></a>**เวลาที่ใช้ในการผลิตแบบสมดุล**

เวลาที่ใช้ในการผลิตหนึ่งหน่วยของผลิตภัณฑ์

###### <a name="task-guide"></a>**คู่มืองาน**

ประสบการณ์ที่มีการควบคุมและการแนะนำผ่านขั้นตอนในการบันทึกงาน คู่มืองานนำทางผู้ใช้ผ่านประสบการณ์ที่ถูกบันทึกไว้ ความปลอดภัย ข้อมูล และลักษณะการทำงานของแอพลิเคชันทั้งหมด มีค่าเท่ากันเมื่อดำเนินการคู่มืองานเสร็จสมบูรณ์ เมื่อไม่มีคู่มืองาน คู่มืองานใช้เครื่องมือเดียวกันเป็นตัวบันทึกงาน เพื่อให้ทราบเมื่อผู้ใช้เสร็จสิ้นขั้นตอนกำหนดไว้ เพื่อที่จะสามารถแจ้งให้ผู้ใช้ดำนินขั้นตอนต่อไปในการบันทึก

###### <a name="task-recorder"></a>**ตัวบันทึกงาน**

เครื่องมือที่มีการติดตั้งไว้ล่วงหน้าใน Dynamics 365 for Operations เมื่อมีการบันทึก บันทึกเหตุการณ์ทั้งหมดที่ผู้ใช้ป้อนในอินเทอร์เฟสผู้ใช้ของ Dynamics 365 for Operations ที่ได้รับการดำเนินการคู่กับเซิร์ฟเวอร์ ซึ่งรวมถึงการเพิ่มค่า การเปลี่ยนแปลงการตั้งค่า การลบข้อมูล เป็นต้น

###### <a name="task-recording"></a>**การบันทึกงาน**

ไฟล์ที่ประกอบด้วยการดำเนินการและคำอธิบายที่ถูกรวบรวมไว้เมื่อมีการดำเนินงานตัวบันทึกงานใน Microsoft Dynamics 365 for Operations

###### <a name="three-way-matching-policy"></a>**นโยบายการจับคู่สามทาง**

นโยบายการจับคู่ที่จำเป็นต้องมีราคาใบแจ้งหนี้ของผู้จัดจำหน่ายอย่างน้อยหนึ่งรายการเพื่อจับคู่กับราคาในใบสั่งซื้ออย่างน้อยหนึ่งรายการ และที่จำเป็นต้องมีปริมาณใบแจ้งหนี้ของผู้จัดจำหน่ายอย่างน้อยหนึ่งรายการเพื่อจับคู่กับปริมาณใบรับสินค้าอย่างน้อยหนึ่งรายการ

###### <a name="transfer-batch"></a>**ชุดงานโอนย้าย**

ปริมาณของสินค้าหนึ่งรายการขึ้นไปที่โอนย้ายหรือที่สามารถโอนย้ายได้

###### <a name="two-way-matching-policy"></a>**นโยบายการจับคู่สองทาง**

นโยบายการจับคู่ที่จำเป็นต้องมีราคาใบแจ้งหนี้ของผู้จัดจำหน่ายอย่างน้อยหนึ่งรายการเพื่อจับคู่กับราคาในใบสั่งซื้ออย่างน้อยหนึ่งรายการ ขึ้นไป

### <a name="u"></a>**U**

###### <a name="user-defined-financial-dimension-type"></a>**ชนิดมิติทางการเงินที่ผู้ใช้กำหนด**

มิติทางการเงินที่แม็ปเข้ากับรายการค่าที่ผู้ใช้กำหนด

### <a name="v"></a>**V**

###### <a name="value-stream"></a>**สายธารคุณค่า**

หน่วยปฏิบัติงานที่ควบคุมขั้นตอนการผลิตหนึ่งขั้นตอนขึ้นไป

###### <a name="variant-configuration-technology"></a>**เทคโนโลยีการจัดโครงแบบผันแปร**

วิธีการในการสร้างแบบจำลองผลิตภัณฑ์หลัก และค้นหาการตั้งค่าคอนฟิกผลิตภัณฑ์ย่อย

###### <a name="vat-value-added-tax"></a>**VAT (ภาษีมูลค่าเพิ่ม)**

ภาษีสำหรับผลิตภัณฑ์ในแต่ละขั้นตอนของการผลิตโดยยึดตามมูลค่าเพิ่มในระหว่างระยะนั้น

###### <a name="vendor-catalog"></a>**แค็ตตาล็อกของผู้จัดจำหน่าย**

การแสดงรายการของข้อเสนอของผลิตภัณฑ์ที่พร้อมใช้งานสำหรับการซื้อจากผู้จัดจำหน่าย

###### <a name="vendor-invoice"></a>**ใบแจ้งหนี้ของผู้จัดจำหน่าย**

เอกสารต้นทางที่แสดงคำขอชำระเงินจากผู้จัดจำหน่าย ใบแจ้งหนี้ของผู้จัดจำหน่ายสามารถอ้างอิงถึงใบสั่งซื้อหนึ่งรายการขึ้นไป เมื่อใบแจ้งหนี้ของผู้จัดจำหน่ายได้รับอนุมัติ สามารถดำเนินการการชำระเงินกับผู้จัดจำหน่ายได้

### <a name="w"></a>**พ.**

###### <a name="work-cell"></a>**เซลล์ทำงาน**

กลุ่มทรัพยากรที่เข้าร่วมในกิจกรรมของขั้นตอนการผลิต

###### <a name="worker"></a>**ผู้ปฏิบัติงาน**

บุคคลที่รับบทบาทของพนักงานหรือผู้รับเหมา และชำระเงินเพื่อแลกเปลี่ยนกับบริการ

###### <a name="workspace"></a>**พื้นที่ทำงาน**

หน้าซึ่งแสดงภาพรวมของกิจกรรมหนึ่งที่ผู้ใช้ดำเนินงาน หน้าใช้ไทล์ รายการ และแผนภูมิเพื่อแสดงงานที่ค้างอยู่และข้อมูลที่เกี่ยวข้องจากหลายแหล่งข้อมูล และหน้าจัดการการดำเนินงานที่เกิดขึ่นบ่อยครั้งที่เกี่ยวข้องกับข้อมูลนี้ ยังมีการแสดงไทล์ที่แสดงในส่วนสรุปของพื้นที่ทำงานในส่วนที่เกี่ยวข้องในแดชบอร์ดอีกด้วย
