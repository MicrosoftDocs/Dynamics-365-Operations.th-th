# <a name="set-up-security-for-the-cost-accounting-analysis-power-bi-content"></a>ตั้งค่าความปลอดภัยสำหรับการลงบัญชีต้นทุนวิเคราะห์เนื้อหาพลังงาน BI

หัวข้อนี้อธิบายวิธีที่คุณสามารถเผยแพร่ยังระดับการเข้าถึงความปลอดภัยในการลงบัญชีต้นทุนการรักษาความปลอดภัยระดับแถวใน Microsoft พลังงาน BI ฟังก์ชันนี้ช่วยรับประกันว่า ผู้ใช้เห็นเฉพาะข้อมูล BI พลังงานที่พวกเขาได้รับสิทธิการเข้าถึง

<a name="overview"></a>ภาพรวม
--------

การ**การวิเคราะห์ต้นทุนบัญชี**เนื้อหา BI พลังงาน Microsoft ใช้ความปลอดภัยระดับแถวพลังงาน BI เพื่อจำกัดการเข้าถึงของผู้ใช้ ความปลอดภัยเป็นไปตามลำดับชั้นขององค์กรระดับการเข้าถึงที่ถูกตั้งค่าในพารามิเตอร์การบัญชีต้นทุน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการ**การวิเคราะห์การลงบัญชีต้นทุน**ดูเนื้อหา BI พลังงาน[บัญชีต้นทุนวิเคราะห์เนื้อหาพลังงาน BI](cost-accounting-analysis-content-pack.md)

## <a name="setup"></a>การตั้งค่า
เพื่อรักษาความปลอดภัยระดับการเข้าถึงการใช้พลังงาน BI เผยแพร่ เจ้าของเนื้อหา BI พลังงานต้องทำตามขั้นตอนเหล่านี้ **หมายเหตุ:**ผู้เผยแพร่**การวิเคราะห์ต้นทุนบัญชี**เนื้อหา BI พลังงานกลายเป็น เจ้าของ เฉพาะเจ้าของสามารถตั้งค่าความปลอดภัยในการใช้พลังงาน BI นอกจากนี้ จนกว่าเจ้าของเพิ่มผู้ใช้รายอื่นบน PowerBI.com ไม่มีผู้ใดยกเว้นเจ้าของสามารถดูข้อมูลใด ๆ ในการ**การวิเคราะห์ต้นทุนบัญชี**BI พลังงานเนื้อหาได้

1.  ประกาศแฟ้มข้อกำหนดในการใช้พลังงาน BI
2.  ล็อกอินเพื่อ PowerBI.com
3.  ค้นหาชุดข้อมูลสำหรับการ**การวิเคราะห์ต้นทุนบัญชี**BI พลังงานเนื้อหา
4.  เปิดหน้าการรักษาความปลอดภัย 

    [![เปิดหน้าการรักษาความปลอดภัย](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-1.png)](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-1.png)

5.  การ**ควบคุมวัตถุต้นทุน**บทบาทถูกสร้างไว้แล้ว เพิ่มสมาชิกอื่น ๆ ที่เป็นส่วนหนึ่งของการลงบัญชีต้นทุนระดับการเข้าถึงลำดับชั้นองค์กร 

    [![การเพิ่มสมาชิก](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-2.png)](https://msdynamics.blob.core.windows.net/media/2017/02/CA-picture-2.png)

ผู้ใช้ที่ถูกเพิ่มเข้าไป**ควบคุมวัตถุต้นทุน**บทบาทจะเห็นเฉพาะข้อมูลที่พวกเขาได้รับอนุญาตให้ดู ตามข้อกำหนดในการลงบัญชีต้นทุนระดับการเข้าถึงลำดับชั้นองค์กร **หมายเหตุ:**รักษาความปลอดภัยระดับแถวใช้ไพ่ และรายงานใน 365 Dynamics Microsoft สำหรับการดำเนินงานที่ฝังอยู่จาก BI พลังงาน

## <a name="updating-security"></a>ปรับปรุงการรักษาความปลอดภัย
ถ้าจะมีปรับปรุงระดับการเข้าถึงความปลอดภัยในการลงบัญชีต้นทุน และคุณต้องใช้พลังงาน BI เพื่อสะท้อนถึงการปรับปรุงเหล่านั้น คุณต้องปรับปรุงร้านค้าสำหรับเอนทิตี**การวิเคราะห์ต้นทุนบัญชี**BI พลังงานเนื้อหา หลังจากทำการปรับปรุงที่เก็บเอนทิตีจาก 365 Dynamics สำหรับการดำเนินงาน คุณต้องปรับปรุงสิ่งประดิษฐ์บน PowerBI.com สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการทำการปรับปรุงที่เก็บเอนทิตี้ ดู[ปรับปรุงเอนทิตีเก็บ](power-bi-integration-entity-store.md#update-entity-store) เจ้าของ**การวิเคราะห์ต้นทุนบัญชี**BI พลังงานเนื้อหายังต้องปรับปรุงเอนทิตีร้านค้าถ้าผู้ใช้ใหม่ได้รับสิทธิการเข้าถึงลำดับชั้นขององค์กร นอกจากนี้ เจ้าของต้องเพิ่มผู้ใช้ใหม่ไป**ควบคุมวัตถุต้นทุน**บทบาทบน PowerBI.com ดังนั้นการรักษาความปลอดภัยระดับแถวนั้นถูกนำไปใช้สำหรับอุปกรณ์เหล่านี้

## <a name="disabling-security"></a>การปิดใช้งานความปลอดภัย
เราสมมติว่า องค์กรของคุณต้องการจำกัดการเข้าถึงข้อมูล ถ้า สำหรับเหตุผลบาง พารามิเตอร์การรักษาความปลอดภัยจะปิดใช้งานเมื่อคุณเรียกใช้การลงบัญชีต้นทุน เจ้าของต้องเพิ่มผู้ใช้ไปยัง**นักบัญชีต้นทุน**บทบาทในพลังงาน BI แทน ถ้าคุณเปลี่ยนความปลอดภัยจากสถานะเปิดใช้งานไปยังสถานะที่ถูกปิดใช้งาน เป็นความคิดที่ดีเมื่อต้องการเอาออกผู้ใช้จากการ**ควบคุมวัตถุต้นทุน**บทบาท และในทางกลับกัน ถ้าคุณเปิดใช้งานความปลอดภัยอีกครั้ง ผู้ใช้สามารถเป็นสมาชิกของบทบาททั้งสองนั้น เข้าร่วมกันคือ ทั้งสองบทบาทของสหภาพ ในกรณีของการ**การวิเคราะห์ต้นทุนบัญชี**BI พลังงานเนื้อหา ผู้ใช้ที่มีสิทธิ์เข้าร่วมได้ไม่จำกัดการเข้าถึงข้อมูล ถ้าเป้าหมายของคุณเป็นการ จำกัดการเข้าใช้ ต้องมีกำหนดผู้ใช้เท่านั้น**ควบคุมวัตถุต้นทุน**บทบาท ปรับปรุงการรักษาความปลอดภัยระดับแถวเหล่านี้จะมีผลทันที ผู้ใช้ที่ได้รับผลกระทบควรฟื้นฟูเบราว์เซอร์ของพวกเขา

## <a name="additional-resources"></a>ทรัพยากรเพิ่มเติม
เมื่อต้องการเรียนรู้เพิ่มเติมเกี่ยวกับการรักษาความปลอดภัยระดับแถวพลังงาน BI ดู[จัดการความปลอดภัยบนแบบจำลองของคุณในการใช้พลังงาน BI](https://powerbi.microsoft.com/en-us/documentation/powerbi-admin-rls/#manage-security-on-your-model)
