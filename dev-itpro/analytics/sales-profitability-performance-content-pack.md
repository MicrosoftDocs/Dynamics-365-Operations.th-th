# <a name="sales-and-profitability-performance-power-bi-content"></a>การขายและกำไรจากประสิทธิภาพเนื้อหาพลังงาน BI

หัวข้อนี้อธิบายถึงสิ่งที่จะรวมใน 365 Dynamics สำหรับการดำเนินงาน - การขายและกำไรจากประสิทธิภาพเนื้อหา pack สำหรับ Microsoft พลังงาน BI อธิบายถึงวิธีการเข้าถึงรายงานรวมอยู่ในแพเนื้อหา และแสดงข้อมูลเกี่ยวกับรูปแบบจำลองข้อมูลและเอนทิตีที่ใช้ในการสร้างแพเนื้อหา

<a name="overview"></a>ภาพรวม
--------

ชุดเนื้อหานี้ถูกสร้างขึ้นสำหรับผู้จัดการฝ่ายขายเพื่อตรวจสอบหลักขายของรายได้ กำไรขั้นต้น และกำไร จะใช้ข้อมูลเกี่ยวกับธุรกรรมการขายจาก Dynamics 365 สำหรับการดำเนินงาน และแสดงทั้งมุมมองรวมของตัวเลขยอดขายของทั้งบริษัทและการแบ่งรายละเอียดของประสิทธิภาพการขายสำหรับลูกค้าและผลิตภัณฑ์ โดยเน้นการเปลี่ยนแปลงในขนาดของรายได้และกำไรในช่วงเวลาหนึ่ง รายงานสามารถใช้เพื่อจัดการการแจ้งเตือนเกี่ยวกับแนวโน้มของค่าบวก และค่าลบสำหรับลูกค้าแต่ละรายและผลิตภัณฑ์ ประเภทและผู้จัดการภูมิภาคจะค้นพบประโยชน์เพื่อให้แผนภูมิที่เปรียบเทียบรายได้และผลกำไรของประเภทผลิตภัณฑ์ที่แตกต่างกันและลูกค้ากลุ่มกันเดียวหา laggards และหัวหน้า งาน รายงานโดยละเอียดที่ลงจุดของลูกค้าแต่ละรายได้และกำไรเบื้องต้นผู้จัดการฝ่ายบัญชีของข้อเสนอสำรองข้อมูลพื้นฐานเพื่อ attune ความพยายามด้านการขาย และการตลาดของลูกค้าแต่ละส่วนกำหนดค่าที่เกี่ยวข้อง การขายและกำไรจากประสิทธิภาพ pack ช่วยให้ผู้จัดการฝ่ายขายเพื่อวิเคราะห์ประสิทธิภาพการขายตามเนื้อหา:

-   รายได้ ปีจนถึงปัจจุบัน (ตามกลุ่มลูกค้า และลูกค้าแต่ละราย ประเภทการขาย และแต่ละผลิตภัณฑ์ และ geographies)
-   การเปลี่ยนแปลงรายได้ over-ปี (ตามประเภทลูกค้าของภูมิภาคและการขาย)

กำไรจากการที่สามารถนำมาวิเคราะห์โดย:

-   กำไรขั้นต้นและกำไรเบื้องต้น (ตามกลุ่มลูกค้าและผลิตภัณฑ์ประเภทการขาย)
-   กำไรขั้นต้นในการเปลี่ยนแปลง over-ปี
-   กำไรจากลูกค้า (ตามรายได้และกำไรขั้นต้น)

## <a name="accessing-the-content-pack"></a>เข้าถึงแพเนื้อหา
การขายและกำไรจากประสิทธิภาพพลังงาน BI เนื้อหาการบรรจุถูกประกาศเป็นสินทรัพย์นำไปใช้ในวัฏจักรบริการ (LCS) และสามารถเข้าถึงได้จาก 365 Dynamics สำหรับการดำเนินงาน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเข้าถึง และการเปิดใช้รายงานพลังงาน BI ดู[เนื้อหาพลังงาน BI ใน LCS จาก Microsoft และคู่ค้าของคุณ](power-bi-content-microsoft-partners.md)

## <a name="understanding-the-data-model-and-entities"></a>การทำความเข้าใจเกี่ยวกับแบบจำลองข้อมูลและเอนทิตี้
Dynamics 365 สำหรับข้อมูลการดำเนินงานถูกใช้เพื่อรวบรวมรายงานในการขายและกำไรจากชุดเนื้อหาของประสิทธิภาพการทำงาน ซึ่งจะแสดงเป็นหน่วยวัดรวมที่มีการแบ่งระยะในเอนทิตีเก็บ ซึ่งเป็นฐานข้อมูล Microsoft SQL ที่ปรับให้เหมาะสมสำหรับการวิเคราะห์ อ่านเพิ่มเติมเกี่ยวกับในบล็อก[BI พลังงานรวมกับร้านค้าเอนทิตีใน Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/) การประเมินรวมในชุดเนื้อหานี้คือ ชุดย่อยของการวัดรวมที่พร้อมใช้งานใน Cube ขายใน Dynamics AX 2012 และ AX 2012 R3 ระยะของ cube ประเมินรวมในร้านค้าของเอนทิตีคุณต้องการให้ deployable สำหรับข้อมูลเพิ่มเติม ให้ดูขั้นตอนเกี่ยวกับวิธีการประเมินรวมในการจัดเก็บเอนทิตีในบล็อกระยะ[BI พลังงานรวมกับร้านค้าเอนทิตีใน Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/) การต่อไปนี้คีย์รวมของเอนทิตีในบรรทัดใบแจ้งหนี้ที่จะใช้เป็นข้อมูลพื้นฐานของชุดเนื้อหา
