# <a name="manage-the-electronic-reporting-configuration-lifecycle"></a>จัดการวงจรการตั้งค่าคอนฟิกการรายงานทางอิเล็กทรอนิกส์

หัวข้อนี้อธิบายวิธีการจัดการวัฏจักรของอิเล็กทรอนิกส์ (ER) การตั้งค่าคอนฟิกสำหรับ 365 Dynamics ของ Microsoft สำหรับโซลูชันที่มีการดำเนินการรายงาน

<a name="overview"></a>ภาพรวม
--------

การรายงานทางอิเล็กทรอนิกส์ (ER) คือกลไกจัดการที่สนับสนุนเอกสารอิเล็กทรอนิกส์เฉพาะประเทศที่จำเป็นตามกฎหมายใน Microsoft Dynamics 365 for Operations โดยทั่วไป ER ถือว่ามีความสามารถในการทำงานต่อไปนี้ สำหรับเอกสารอิเล็กทรอนิกส์เดี่ยว  สำหรับรายละเอียดเพิ่มเติม ดู[ภาพรวมรายงานอิเล็กทรอนิกส์](general-electronic-reporting.md)

-   ออกแบบเท็มเพลตสำหรับเอกสารอิเล็กทรอนิกส์:
    -   ระบุแหล่งข้อมูลที่จำเป็นที่สามารถแสดงในเอกสารได้:
        -   365 Dynamics ต้นแบบสำหรับข้อมูลการดำเนินงาน เช่นตารางข้อมูล ข้อมูล และเอนทิตี ระดับชั้น
        -   คุณสมบัติเฉพาะของกระบวนการ เช่นวันดำเนินการ และ เวลา และโซนเวลา
        -   ผู้ใช้พารามิเตอร์ค่านำเข้า ระบุ โดยผู้ใช้ในขณะทำงาน
    -   กำหนดองค์ประกอบของเอกสารที่จำเป็น และโทโพโลยีเพื่อระบุรูปแบบของเอกสารขั้นสุดท้าย
    -   ตั้งค่าคอนฟิกการไหลของข้อมูลที่ต้องการ จากแหล่งข้อมูลที่ถูกเลือกไปยังองค์ประกอบของเอกสารที่กำหนด (ผ่านทางแหล่งข้อมูลที่รวมกับส่วนประกอบของรูปแบบของเอกสาร) และระบุตรรกะการควบคุมกระบวนการ
-   ทำให้เท็มเพลตพร้อมใช้งาน เพื่อให้สามารถใช้ในอินสแตนซ์อื่นของ Dynamics 365 for Operations ได้:
    -   แปลงเท็มเพลตเอกสารที่ถูกสร้างขึ้นใน Dynamics 365 for Operations ลงในการตั้งค่าคอนฟิก ER และส่งออกการตั้งค่าคอนฟิกจากอินสแตนซ์ของ Dynamics 365 for Operations ปัจจุบัน เป็นแพคเกจ XML ที่สามารถถูกจัดเก็บเฉพาะที่ หรือใน LCS ได้
    -   แปลงการตั้งค่าคอนฟิก ER ลงในเท็มเพลตเอกสาร Dynamics 365 for Operations
    -   นำเข้าแพคเกจ XML ที่ถูกจัดเก็บเฉพาะที่ หรือใน LCS ลงในอินสแตนซ์ Dynamics 365 for Operations ปัจจุบัน
-   เลือกกำหนดเท็มเพลตของเอกสารอิเล็กทรอนิกส์:
    -   นำเท็มเพลตจาก LCS ไปยังอินสแตนซ์ Dynamics 365 for Operations ปัจจุบัน เป็นการตั้งค่าคอนฟิก ER
    -   ออกแบบเวอร์ชันที่กำหนดเองของการตั้งค่าคอนฟิก ER และเก็บการอ้างอิงไปยังเวอร์ชันพื้นฐาน
-   รวมเท็มเพลตกับกระบวนการทางธุรกิจเฉพาะ เพื่อให้พร้อมใช้งานใน Dynamics 365 for Operations:
    -   ตั้งค่าคอนฟิกการตั้งค่าเพื่อให้ Dynamics 365 for Operations เริ่มใช้การตั้งค่าคอนฟิก ER โดยอ้างอิงถึงการตั้งค่าคอนฟิกในพารามิเตอร์ที่เกี่ยวข้องกับกระบวนการ ตัวอย่างเช่น อ้างถึงการตั้งค่าคอนฟิก ER ในเฉพาะบัญชีเจ้าหนี้วิธีการชำระเงินเพื่อสร้างข้อความการชำระเงินทางอิเล็กทรอนิกส์สำหรับการประมวลผลใบแจ้งหนี้
-   ใช้เท็มเพลตในกระบวนการทางธุรกิจเฉพาะ:
    -   เรียกใช้การตั้งค่าคอนฟิก ER ในกระบวนการทางธุรกิจ ตัวอย่างเช่น เพื่อสร้างข้อความการชำระเงินทางอิเล็กทรอนิกส์สำหรับการประมวลผลใบแจ้งหนี้เมื่อวิธีการชำระเงินที่อ้างอิงถึงการตั้งค่าคอนฟิก ER ที่ถูกเลือก

## <a name="concepts"></a>แนวคิด
บทบาทต่อไปนี้และกิจกรรมที่เกี่ยวข้องจะเกี่ยวข้องกับการตั้งค่าคอนฟิก ER วัฏจักร

| บทบาท                                       | กิจกรรม                                                      | คำอธิบาย                                                                                                                                                                                                                  |
|--------------------------------------------|-----------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ที่ปรึกษาด้านการทำงานของการรายงานทางอิเล็กทรอนิกส์ | สร้างและจัดการส่วนประกอบ ER (แบบจำลองและรูปแบบ)           | ธุรกิจผู้ออกแบบจำลองข้อมูลโดเมน – เฉพาะ ER ออกแบบแม่แบบจำเป็นสำหรับเอกสารอิเล็กทรอนิกส์ และผูกให้สอดคล้องกัน                                                                           |
| นักพัฒนาการรายงานทางอิเล็กทรอนิกส์             | สร้างและจัดการการแม็ปแบบจำลองข้อมูล                          | มี 365 Dynamics สำหรับผู้เชี่ยวชาญด้านการดำเนินงานที่เลือก 365 Dynamics จำเป็นสำหรับแหล่งข้อมูลของการดำเนินงาน และผูกข้อมูลเหล่านั้นกับแบบจำลองข้อมูลโดเมน – เฉพาะ ER                                                                 |
| หัวหน้างานฝ่ายบัญชี                      | ตั้งค่าคอนฟิกการตั้งค่าที่เกี่ยวข้องกับกระบวนการ ซึ่งอ้างอิงวัตถุ ER | ตัวอย่างเช่น การ**หัวหน้างานฝ่ายบัญชี**บทบาทที่ช่วยให้การตั้งค่าของการตั้งค่าคอนฟิก ER ที่จะใช้ในวิธีการชำระเงินเจ้าหนี้ของบัญชีหนึ่ง ๆ เพื่อสร้างข้อความการชำระเงินทางอิเล็กทรอนิกส์สำหรับการประมวลผลใบแจ้งหนี้ |
| เจ้าหน้าที่ฝ่ายชำระเงินบัญชีเจ้าหนี้            | ใช้วัตถุ ER ในกระบวนการทางธุรกิจเฉพาะ:                | ตัวอย่างเช่น การ**เจ้าหน้าที่ฝ่ายชำระเงินเจ้าหนี้ของบัญชี**บทบาทที่อนุญาตให้ชำระเงินทางอิเล็กทรอนิกส์จะถูกสร้างขึ้นสำหรับการประมวลผลใบแจ้งหนี้ ขึ้นอยู่กับรูปแบบ ER ที่ถูกกำหนดค่าสำหรับวิธีการชำระเงินเฉพาะ           |

## <a name="er-configuration-development-lifecycle"></a>วงจรชีวิตพัฒนาการตั้งค่าคอนฟิก ER
สำหรับเหตุผลที่เกี่ยวข้องกับ ER ต่อไปนี้ ขอแนะนำให้คุณออกแบบการตั้งค่าคอนฟิก ER ใน สิ่งแวดล้อมการพัฒนา เป็นอินสแตนซ์ที่แยกต่างหากของ Dynamics 365 for Operations:

-   ผู้ใช้ในบทบาท **นักพัฒนารายงานทางอิเล็กทรอนิกส์** หรือบทบาท **ที่ปรึกษาฟังก์ชันรายงานอิเล็กทรอนิกส์** อย่างใดอย่างหนึ่งสามารถแก้ไขการตั้งค่าคอนฟิก และรันเพื่อทดสอบวัตถุประสงค์ได้ สถานการณ์จำลองนี้สามารถทำให้เกิดการเรียกของวิธีการของคลาสและตารางที่อาจเป็นอันตรายสำหรับข้อมูลธุรกิจและประสิทธิภาพการทำงานของการใช้อินสแตนซ์ Dynamics 365 for Operations
-   การเรียกวิธีการของคลาสและตารางที่เป็นแหล่งข้อมูล ER ของการตั้งค่าคอนฟิก ER ไม่ได้ถูกจำกัดโดยจุดเข้าใช้งาน Dynamics 365 for Operations และล็อกเนื้อหาของบริษัท ดังนั้น ข้อมูลในบทบาท **นักพัฒนารายงานทางอิเล็กทรอนิกส์** หรือบทบาท **ที่ปรึกษาฟังก์ชันรายงานอิเล็กทรอนิกส์** อย่างใดอย่างหนึ่งสามารถเข้าถึงข้อมูลที่อ่อนไหวทางธุรกิจได้

สามารถอัพโหลดโครงแบบ ER ที่ถูกออกแบบมาในสภาพแวดล้อมการพัฒนาไปยังสภาพแวดล้อมการทดสอบสำหรับการประเมินการตั้งค่าคอนฟิก (รวมกระบวนการที่เหมาะสม ความถูกต้องของผลลัพธ์ และประสิทธิภาพการทำงาน) และการรับรองคุณภาพ เช่นความถูกต้องของสิทธิการเข้าถึงควบคุมการเข้าถึงข้อมูลและการแบ่งแยกหน้าที่หรือไม่ ลักษณะการทำงานที่เปิดใช้งานการแลกเปลี่ยนการตั้งค่าคอนฟิก ER สามารถใช้สำหรับวัตถุประสงค์นี้ ในตอนท้าย โครงแบบ ER พิสูจน์สามารถอัพโหลด จะ LCS ซึ่งพวกเขาสามารถใช้ร่วมกับสมาชิกการบริการ หรือสภาพแวดล้อมการผลิตสำหรับใช้ภายใน แสดงในภาพประกอบต่อไปนี้เช่นกัน ![ER วัฏจักรการตั้งค่าคอนฟิก](./media/ger-configuration-lifecycle.png)
