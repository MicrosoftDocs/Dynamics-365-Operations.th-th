# <a name="electronic-reporting-overview"></a>ภาพรวมการรายงานทางอิเล็กทรอนิกส์

บทความนี้แสดงภาพรวมของเครื่องมือการรายงานทางอิเล็กทรอนิกส์ (ER) ซึ่งรวมถึงข้อมูลเกี่ยวกับแนวคิดหลัก สถานการณ์จำลองที่ ER สนับสนุน และรายการรูปแบบที่ได้รับการออกแบบและนำออกใช้เป็นส่วนหนึ่งของโซลูชัน

การรายงานทางอิเล็กทรอนิกส์ (ER) คือเครื่องมือที่คุณสามารถใช้เพื่อตั้งค่าคอนฟิกรูปแบบสำหรับเอกสารอิเล็กทรอนิกส์ที่สอดคล้องกับข้อกำหนดตามกฎหมายของประเทศ/ภูมิภาคต่างๆ ER ช่วยให้คุณสามารถจัดการรูปแบบเหล่านี้ในระหว่างรอบการใช้งาน ตัวอย่างเช่น คุณสามารถนำข้อบังคับใหม่มาใช้ และสามารถสร้างเอกสารทางธุรกิจในรูปแบบที่กำหนดเพื่อแลกเปลี่ยนข้อมูลกับหน่วยงานของรัฐบาล ธนาคาร และฝ่ายอื่นๆ ทางอิเล็กทรอนิกส์ กลไกจัดการ ER มีเป้าหมายที่ผู้ใช้ทางธุรกิจแทนนักพัฒนา เนื่องจากคุณตั้งค่าคอนฟิกรูปแบบ ไม่ใช่รหัส กระบวนการของการสร้างและการปรับรูปแบบสำหรับเอกสารอิเล็กทรอนิกส์จึงรวดเร็วและง่ายขึ้น ขณะนี้ ER สนับสนุนรูปแบบแผ่นงาน TEXT, XML และ OPENXML อย่างไรก็ตาม อินเทอร์เฟซส่วนขยายให้การสนับสนุนสำหรับรูปแบบเพิ่มเติม

## <a name="capabilities"></a>ความสามารถ
กลไกจัดการ ER มีความสามารถต่อไปนี้:

-   นั้นแสดงถึงเครื่องมือทั่วไปเดียวสำหรับการรายงานทางอิเล็กทรอนิกส์ในโดเมนที่แตกต่างกัน และแทนโปรแกรมเท่านั้นที่แตกต่างกันมากกว่า 20 ที่ทำบางชนิดของการรายงานทางอิเล็กทรอนิกส์สำหรับ 365 Dynamics Microsoft สำหรับการดำเนินงาน ใหม่
-   จะทำให้รูปแบบของรายงานที่ insulated จาก 365 Dynamics ปัจจุบันสำหรับการใช้งานการดำเนินงาน (อีกนัยหนึ่ง รูปแบบจะใช้ได้สำหรับเวอร์ชันต่าง ๆ ของ Dynamics 365 สำหรับการดำเนินงาน)
-   สนับสนุนการสร้างรูปแบบที่กำหนดเองที่เป็นไปตามรูปแบบเดิม ประกอบด้วยความสามารถในการอัพเกรดรูปแบบที่กำหนดเองโดยอัตโนมัติเมื่อเกิดการเปลี่ยนแปลงไปเป็นรูปแบบเดิมเนื่องจากมีการแนะนำความต้องการในการแปล / การกำหนดค่าด้วยตนเอง
-   จะกลายเป็นเครื่องมือมาตรฐานหลักในการสนับสนุนความต้องการในการแปลในการรายงานทางอิเล็กทรอนิกส์ทั้งสำหรับ Microsoft และคู่ค้า Microsoft
-   จะสามารถสนับสนุนความสามารถในการจัดรูปแบบที่มีการกระจายสำหรับคู่ค้าและลูกค้าโดยใช้ Microsoft Dynamics Lifecycle Services (LCS)

## <a name="concepts"></a>แนวคิด
### <a name="components"></a>ส่วนประกอบ

ER สนับสนุนชนิดของส่วนประกอบสองชนิด **แบบจำลองข้อมูล **และ **รูปแบบ**

#### <a name="data-model-components"></a>ส่วนประกอบแบบจำลองข้อมูล

ส่วนประกอบของแบบจำลองข้อมูลเป็นการนำเสนอทางนามธรรมของโครงสร้างข้อมูลที่จะใช้เพื่ออธิบายพื้นที่โดเมนทางธุรกิจบางพื้นที่ ซึ่งมีการแสดงรายละเอียดเพียงพอที่จะตอบสนองความต้องการรายงานในโดเมนนี้ ส่วนประกอบของแบบจำลองข้อมูลประกอบด้วยส่วนต่างๆ ดังต่อไปนี้:

-   แบบจำลองข้อมูลที่เป็นชุดของเอนทิตี้ทางธุรกิจเฉพาะโดเมนรวมทั้งคำนิยามทางโครงสร้างตามลำดับชั้นของความสัมพันธ์ระหว่างกัน
-   กระแสข้อมูลแบบจำลองที่การแม็ปการเชื่อมโยงที่เลือก Dynamics 365 สำหรับแหล่งข้อมูลของการดำเนินการกับแต่ละองค์ประกอบของแบบจำลองข้อมูลที่ระบุ ในขณะที่ใช้งาน และกฎของประชากรข้อมูลทางธุรกิจไปยังข้อมูลแบบจำลองส่วนประกอบ

เอนทิตี้้ทางธุรกิจของรูปแบบจำลองข้อมูลถูกแสดงเป็นตู้บรรจุสินค้า (เรกคอร์ด) คุณสมบัติของเอนทิตี้ทางธุรกิจจะถูกแสดงเป็นรายการข้อมูล (ฟิลด์) รายการข้อมูลแต่ละรายการมีชื่อเฉพาะ ป้ายชื่อ คำอธิบาย และค่า ค่าของรายการข้อมูลแต่ละรายการสามารถถูกออกแบบเพื่อให้มีการรับรู้เป็นสตริง จำนวนเต็ม จำนวนจริง วัน ชนิดการแจง บูลีน และอื่นๆ ได้ นอกจากนี้ยังสามารถเป็นเรกคอร์ดหรือรายการเรกคอร์ดอื่นได้ ส่วนประกอบของแบบจำลองข้อมูลเดี่ยวสามารถประกอบด้วยลำดับชั้นหลายลำดับของเอนทิตี้ทางธุรกิจเฉพาะโดเมนรวมทั้งการแม็ปแบบจำลองเพื่อสนับสนุนลำดับข้อมูลเฉพาะรายงานเฉพาะที่เวลาการดำเนินงาน ลำดับชั้นจะถูกทำให้แตกต่างกันโดยเรกคอร์ดเดี่ยวที่ได้ถูกเลือกให้เป็นรากของการแม็ปแบบจำลอง ตัวอย่างเช่น แบบจำลองข้อมูลของพื้นที่โดเมนการชำระเงินอาจสนับสนุนการแม็ปต่อไปนี้:

-   บริษัท -&gt;ของผู้จัดจำหน่าย -&gt;ธุรกรรมการชำระเงินของจุดการเข้าถึงโดเมน
-   ลูกค้า -&gt;บริษัท -&gt;ธุรกรรมการชำระเงินของโดเมน AR

โปรดสังเกตว่าเอนทิตี้ธุรกิจ (เช่น ธุรกรรมของบริษัทและการชำระเงิน) ได้รับการออกแบบเพียงครั้งเดียวเท่านั้น การแม็ปที่ต่างกันจึงนำมาใช้ใหม่ การแม็ปแบบจำลองมีความสามารถต่อไปนี้:

-   ก็สามารถใช้ 365 Dynamics ที่แตกต่างกันสำหรับชนิดของข้อมูลการดำเนินงานเป็นแหล่งข้อมูลสำหรับแบบจำลองข้อมูล ตัวอย่างเช่น สามารถใช้ตาราง เอนทิตีข้อมูล วิธีการ หรือ enums
-   สนับสนุนพารามิเตอร์ป้อนเข้าของผู้ใช้ที่สามารถกำหนดให้เป็นแหล่งข้อมูลของแบบจำลองข้อมูลเมื่อต้องระบุข้อมูลบางอย่างขณะทำงาน
-   สนับสนุนการแปลงของ Dynamics 365 สำหรับข้อมูลการดำเนินงานเป็นกลุ่มที่จำเป็น การกรอง การเรียงลำดับ และรวมข้อมูล และยัง ผนวกกับตรรกะคำนวณเขตข้อมูลที่ถูกออกแบบมา โดย Microsoft Excel – เหมือนกับสูตร (ดูรายละเอียดเพิ่มเติม[โปรแกรมออกแบบสูตรในการรายงานทางอิเล็กทรอนิกส์](general-electronic-reporting-formula-designer.md))

[![ตัวแก้ไขสูตร Excel เหมือน](./media/pic-formula-1024x615.png)](./media/pic-formula.png)เป็นส่วนประกอบของแบบจำลองข้อมูลถูกออกแบบมาสำหรับแต่ละธุรกิจโดเมนที่ควรใช้เป็นแหล่งข้อมูลประกอบการสำหรับการรายงาน ที่สามารถแยกรายงานจากการปฏิบัติการทางกายภาพของ Dynamics 365 สำหรับแหล่งข้อมูลของการดำเนินงาน และแสดงถึงแนวความคิดทางธุรกิจเฉพาะของโดเมนและฟังก์ชันในแบบฟอร์มที่ทำให้การออกแบบเริ่มต้นของรูปแบบรายงานและการบำรุงรักษามีประสิทธิภาพมากขึ้น

#### <a name="format-components"></a>ส่วนประกอบรูปแบบ

ส่วนประกอบรูปแบบมีโครงร่างของการรายงานผลลัพธ์ที่จะถูกสร้างขณะเวลาดำเนินงาน โครงร่างประกอบด้วยองค์ประกอบต่อไปนี้:

-   รูปแบบที่กำหนดโครงสร้างและเนื้อหาของเอกสารการรายงานทางอิเล็กทรอนิกส์ที่ถูกสร้างขึ้นขณะเวลาดำเนินงาน
-   แหล่งข้อมูลที่เป็นชุดของพารามิเตอร์ข้อมูลป้อนเข้าของผู้ใช้และแบบจำลองข้อมูลที่เฉพาะเจาะจงโดเมนที่ใช้การแม็ปแบบจำลองที่เลือก
-   การแม็ปรูปแบบให้เป็นชุดของการผูกแหล่งข้อมูลรูปแบบกับองค์ประกอบแต่ละรายการของรูปแบบที่ระบุขณะเวลาดำเนินงานลำดับข้อมูล และกฎของการสร้างผลลัพธ์รูปแบบ
-   การตรวจสอบรูปแบบที่เป็นชุดของกฎที่ตั้งค่าคอนฟิกได้ ซึ่งควบคุมการสร้างรายงานที่ขณะดำเนินงานซึ่งขึ้นอยู่กับการดำเนินบริบท (ตัวอย่างเช่น กฎที่หยุดการสร้างผลลัพธ์ของการชำระเงินของผู้จัดจำหน่าย และแสดงข้อยกเว้นเมื่อแอททริบิวต์เฉพาะของผู้จัดจำหน่ายที่เลือกหายไป เช่น หมายเลขบัญชีธนาคาร)

ส่วนประกอบของรูปแบบสนับสนุนฟังก์ชันต่อไปนี้:

-   การสร้างผลลัพธ์การรายงานเป็นไฟล์แต่ละรายการในรูปแบบต่างๆ: ข้อความ XML หรือแผ่นงาน
-   การสร้างหไฟล์หลายรายการที่แยกต่างหาก และยังประกอบด้วยไฟล์เหล่านั้นลงในไฟล์ zip

ส่วนประกอบรูปแบบช่วยให้สามารถแนบไฟล์เฉพาะที่สามารถใช้ในการรายงานผลลัพธ์ได้:

-   สมุดงาน Excel ซึ่งประกอบด้วยแผ่นงานที่สามารถนำมาใช้เป็นเท็มเพลตสำหรับผลลัพธ์ในรูปแบบของแผ่นงาน OPENXML
-   ไฟล์อื่นๆที่สามารถรวมเป็นผลลัพธ์ของรูปแบบดังที่ได้กำหนดไว้ล่วงหน้า

#### <a name="component-versioning"></a>การกำหนดเวอร์ชันส่วนประกอบ

การกำหนดเวอร์ชันได้รับการสนับสนุนสำหรับส่วนประกอบ ER มีการระบุลำดับงานต่อไปนี้สำหรับการจัดการการเปลี่ยนแปลงในส่วนประกอบ ER:

-   รุ่นที่ถูกสร้างขึ้นแต่เดิมถูกทำเครื่องหมายเป็นรุ่น **ร่าง** รุ่นนี้สามารถแก้ไขได้และจะพร้อมใช้งานสำหรับการรันการทดสอบ
-   รุ่น** ร่าง** สามารถแปลงเป็นรุ่น **เสร็จสมบูรณ์** ได้ รุ่นนี้สามารถใช้ในกระบวนการรายงานภายใน
-   การ**เสร็จสมบูรณ์**สามารถแปลงรุ่น**ร่วม**รุ่นได้ รุ่นนี้มีการเผยแพร่บน LCS และสามารถใช้ในกระบวนการรายงานส่วนกลาง
-   รุ่น **ใช้ร่วมกัน **สามารถแปลงเป็นรุ่น **ที่ไม่ต่อเนื่อง** ได้ จากนั้นสามารถลบรุ่นนี้ออกได้

รุ่นที่มีทั้ง ** ** เสร็จสมบูรณ์ หรือ**ร่วม**สถานะพร้อมใช้งานสำหรับการแลกเปลี่ยนข้อมูลอื่น ๆ ส่วนประกอบที่มีสถานะเหล่านี้สามารถกระทำการดำเนินการเหล่านี้ได้:

-   พวกเขาสามารถทำให้เป็นอนุกรมในรูปแบบ XML และจาก 365 Dynamics สำหรับการดำเนินงานที่ส่งออกเป็นไฟล์ในรูปแบบ XML
-   พวกเขาสามารถ reserialized จากแฟ้ม XML และนำเข้ามายัง 365 Dynamics สำหรับการดำเนินงานเป็นคอมโพเนนต์ ER รุ่นใหม่

#### <a name="component-date-effectivity"></a>วันที่มีผลบังคับใช้ส่วนประกอบ

รุ่นส่วนประกอบ ER คือวันที่มีผลบังคับใช้ วันที่** มีผลบังคับใช้ตั้งแต่ **สามารถถูกกำหนดสำหรับส่วนประกอบ ER เพื่อระบุวันที่ที่ส่วนประกอบนี้จะมีผลบังคับใช้สำหรับกระบวนการการรายงาน 365 Dynamics สำหรับวันที่รอบเวลาของการดำเนินงานถูกใช้เพื่อกำหนดว่า ส่วนประกอบถูกต้องสำหรับการดำเนินการ ถ้ารุ่นมากกว่าหนึ่งรุ่นใช้ได้กับวันที่เฉพาะ รุ่นล่าสุดจะใช้สำหรับกระบวนการการรายงาน

#### <a name="component-access"></a>การเข้าถึงส่วนประกอบ

การเข้าถึงส่วนประกอบรูปแบบ ER ขึ้นอยู่กับการตั้งค่ารหัสประเทศ/ภูมิภาค ISO เมื่อการตั้งค่านี้เป็นค่าว่างสำหรับเวอร์ชันที่เลือกของการตั้งค่าคอนฟิกรูปแบบ คอมโพเนนต์ของรูปแบบที่สามารถเข้าถึงจาก 365 Dynamics ใด ๆ สำหรับบริษัทที่ดำเนินงานในขณะใช้งาน เมื่อตั้งค่านี้ประกอบด้วยรหัสของประเทศ/ภูมิภาค ISO คอมโพเนนต์รูปแบบจะพร้อมใช้งานจาก 365 Dynamics สำหรับบริษัทการดำเนินงานที่มีที่อยู่หลักที่กำหนดไว้สำหรับหนึ่งรหัสประเทศ/ภูมิภาค ISO ของคอมโพเนนต์รูปแบบ เท่านั้น รุ่นต่างๆของส่วนประกอบของรูปแบบข้อมูลอาจมีการตั้งค่าที่แตกต่างกันตามรหัสประเทศ/ภูมิภาค ISO

#### <a name="configuration"></a>การจัดโครงแบบ

การตั้งค่าคอนฟิก ER เป็น wrapper ของส่วนประกอบ ER เฉพาะ **แบบจำลองข้อมูล **หรือ **รูปแบบ** อย่างใดอย่างหนึ่ง การตั้งค่าคอนฟิกอาจประกอบด้วยเวอร์ชันต่างๆของส่วนประกอบ ER เฉพาะ การตั้งค่าคอนฟิกแต่ละรายการถูกทำเครื่องหมายว่าเป็นเจ้าของโดยผู้ให้บริการการตั้งค่าคอนฟิกเฉพาะ การ**ร่าง**รุ่นของคอมโพเนนต์ตัวจัดโครงแบบที่สามารถแก้ไขได้เมื่อมีการเลือกเจ้าของการตั้งค่าคอนฟิกเป็นตัวให้บริการที่ใช้งานอยู่ในการตั้งค่า ER ใน 365 Dynamics สำหรับการดำเนินงานได้ การตั้งค่าคอนฟิกแบบจำลองแต่ละรายการประกอบด้วยส่วนประกอบ **แบบจำลองข้อมูล** การตั้งค่าคอนฟิกรูปแบบใหม่สามารถถูกสร้าง (ได้รับ) จากการตั้งค่าคอนฟิกแบบจำลองข้อมูลที่เฉพาะเจาะจง การตั้งค่าคอนฟิกรูปแบบที่สร้างขึ้นจะแสดงในแผนภูมิการตั้งค่าคอนฟิกเป็นข้อมูลรองของการตั้งค่าคอนฟิกแบบจำลองข้อมูลเดิม การตั้งค่าคอนฟิกรูปแบบที่สร้างขึ้นประกอบด้วยส่วนประกอบ **รูปแบบ ** ส่วนประกอบ **แบบจำลองข้อมูล** ของการตั้งค่าคอนฟิกแบบจำลองเดิมจะถูกแทรกโดยอัตโนมัติไปยังส่วนประกอบ **รูปแบบ **ของการตั้งค่าคอนฟิกรูปแบบรองที่สร้างขึ้นเป็นแหล่งข้อมูลเริ่มต้น การตั้งค่าคอนฟิก ER ถูกใช้ร่วมกันสำหรับ Dynamics 365 สำหรับบริษัทที่ดำเนินงาน

#### <a name="provider"></a>ผู้จัดหา

ผู้ให้บริการ ER คือ ตัวระบุฝ่ายที่ใช้เพื่อบ่งชี้ผู้เขียน (เจ้าของ) ของการตั้งค่าคอนฟิก ER แต่ละรายการ ER ช่วยให้คุณสามารถจัดการรายการของผู้ให้บริการการตั้งค่าคอนฟิก การตั้งค่าคอนฟิกที่นำออกใช้สำหรับเอกสารอิเล็กทรอนิกส์เป็นส่วนหนึ่งของ 365 Dynamics สำหรับโซลูชันที่มีการดำเนินการจะถูกทำเครื่องหมายเป็นเจ้าของ การจัดรูปแบบแบบ**Microsoft**ตัวให้บริการการตั้งค่าคอนฟิก

#### <a name="repository"></a>ที่เก็บ

ที่เก็บ ER เก็บการตั้งค่าคอนฟิก ER ประเภทของการเก็บข้อมูล ER อยู่ในขณะนี้ได้รับการสนับสนุน:**ทรัพยากรการดำเนินงาน**และ**โครงการ LCS** ผิด ** ให้คุณเข้าถึงรายการของการตั้งค่าคอนฟิกที่นำออกใช้เป็นส่วนหนึ่งของ 365 Dynamics สำหรับโซลูชันที่มีการดำเนินการ โดย Microsoft เป็นตัวให้บริการการตั้งค่าคอนฟิก ER ** ทรัพยากรการดำเนินงานที่เก็บไว้ การตั้งค่าคอนฟิกเหล่านั้นสามารถนำเข้ามายัง 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงาน และใช้สำหรับการรายงานทางอิเล็กทรอนิกส์ สามารถใช้สำหรับการแปลเป็นภาษาท้องถิ่น / การเลือกกำหนดเพิ่มเติม ที่เก็บ **โครงการ LCS **มอบการเข้าถึงรายการของการตั้งค่าคอนฟิกโครงการ LCS ที่เฉพาะเจาะจง (ไลบรารีสินทรัพย์โครงการ LCS) ที่ได้เลือกไว้ในขั้นตอนการลงทะเบียนที่เก็บ ER ช่วยให้คุณสามารถอัปโหลดการกำหนดค่าที่ใช้ร่วมกันจาก 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงานเฉพาะ**โครงการ LCS**ที่เก็บไว้ คุณยังสามารถนำเข้าการกำหนดค่าจากเฉพาะ**โครงการ LCS**ที่เก็บเป็นแบบ 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงานได้ จำเป็นต้องใช้**โครงการ LCS**เก็บข้อมูลคุณสามารถลงทะเบียนแต่ละรายการสำหรับตัวให้บริการการตั้งค่าคอนฟิกแต่ละตัว 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงานได้ ที่เก็บแต่ละที่สามารถใช้ได้เฉพาะกับผู้ให้บริการการตั้งค่าคอนฟิกเฉพาะ

## <a name="supported-scenarios"></a>สถานการณ์จำลองที่ได้รับการสนับสนุน
### <a name="building-a-data-model"></a>การสร้างแบบจำลองของข้อมูล

ER มอบผู้ออกแบบแบบจำลองที่คุณสามารถใช้ในการสร้างแบบจำลองข้อมูลสำหรับโดเมนธุรกิจหนึ่งๆ เอนทิตีทางธุรกิจเฉพาะโดเมนทั้งหมดและความสัมพันธ์ระหว่างกัน สามารถถูกนำเสนอในแบบจำลองข้อมูลให้เป็นโครงสร้างตามลำดับชั้นได้ ภาพประกอบต่อไปนี้แสดงตัวอย่างของชนิดของแบบจำลองข้อมูล (รูปแบบจำลองข้อมูลโดเมนการชำระเงิน) [![ตัวอย่างของแบบจำลองข้อมูล](./media/pic-data-model-1024x550.png)](./media/pic-data-model.png)จะคุ้นเคยกับรายละเอียดของสถานการณ์นี้ เล่น**โมเดลข้อมูลเฉพาะของโดเมนออกแบบ ER**แนะนำงาน (เป็นส่วนหนึ่งของการ**7.5.4.3 IT รับ/จัดทำคอมโพเนนต์ โซลูชัน/service (10677)**กระบวนการทางธุรกิจ)

### <a name="translating-data-model-content"></a>การแปลเนื้อหาแบบจำลองข้อมูล

เนื้อหาแบบจำลองข้อมูล (ป้ายชื่อและคำอธิบาย) สามารถถูกแปลเป็นภาษาอื่น ๆ ที่สนับสนุน 365 Dynamics สำหรับการดำเนินงาน คุณอาจต้องการแปลเนื้อหาแบบจำลองข้อมูลสำหรับเหตุผลต่อไปนี้:

-   ในขณะออกแบบ เพื่อทำให้เนื้อหาเข้าใจได้ง่ายมากขึ้นสำหรับผู้ออกแบบรูปแบบที่พูดภาษาอื่น และผู้ที่จะใช้แบบจำลองข้อมูลสำหรับการแม็ปข้อมูลของส่วนประกอบรูปแบบ
-   เมื่อรันไทม์ เพื่อทำให้เนื้อหาผู้ใช้เพิ่มเติมจำง่าย โดยการแสดงกล่องโต้ตอบการนำเสนอ และวิธีใช้สำหรับพารามิเตอร์ขณะใช้งาน และยัง กำหนดค่าข้อความการตรวจสอบ (ข้อผิดพลาดและคำเตือน), ในภาษาที่ต้องการอยู่ในขณะนี้เข้าสู่ระบบ Dynamics 365 สำหรับผู้ใช้ในการดำเนินงาน

ภาพประกอบต่อไปนี้แสดงตัวอย่างของวิธีการแปลเนื้อหาแบบจำลองข้อมูลจากภาษาอังกฤษเป็นภาษาญี่ปุ่น [![ข้อมูลแบบจำลองเนื้อหาภาษาอังกฤษ](./media/pic-translate-en-1024x495.png)](./media/pic-translate-en.png)[![ข้อมูลแบบจำลองเนื้อหาที่แปลเป็นภาษาญี่ปุ่น](./media/pic-translate-ja-1024x495.png)](./media/pic-translate-ja.png)

### <a name="configuring-data-model-mappings"></a>การตั้งค่าคอนฟิกการแม็ปแบบจำลองข้อมูล

ER แสดงตัวออกแบบการแม็ปแบบจำลองที่ช่วยให้ผู้ใช้แมปข้อมูลแบบจำลองที่มีการออกแบบกับ 365 Dynamics เฉพาะสำหรับแหล่งข้อมูลของการดำเนินงาน ภาพประกอบต่อไปนี้แสดงตัวอย่างของการแม็ปแบบจำลองข้อมูลชนิดนี้ (การแม็ปแบบจำลอง **การโอนย้ายเครดิต SEPA** ของแบบจำลองข้อมูลโดเมนการชำระเงิน) [![ตัวอย่างของการแม็ปแบบจำลองข้อมูล](./media/pic-model-mapping-1024x551.png)](./media/pic-model-mapping.png)จะคุ้นเคยกับรายละเอียดของสถานการณ์นี้ เล่น**ER กำหนดแบบจำลองการแม็ป และเลือกแหล่งข้อมูล**และ**โมเดลข้อมูล ER แมปไปยังแหล่งข้อมูลที่เลือก**งานแนะนำ (เป็นส่วนหนึ่งของการ**7.5.4.3 IT รับ/จัดทำคอมโพเนนต์ โซลูชัน/service (10677)**กระบวนการทางธุรกิจ)

### <a name="storing-a-designed-model-component-as-a-model-configuration"></a>การจัดเก็บส่วนประกอบแบบจำลองที่ได้ออกแบบไว้ให้เป็นการตั้งค่าคอนฟิกแบบจำลอง

ER สามารถจัดเก็บแบบจำลองข้อมูลที่ถูกออกแบบมาพร้อมกับการแมปข้อมูลที่เกี่ยวข้องเป็นโครงแบบจำลองของการ 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงาน ภาพประกอบต่อไปนี้แสดงตัวอย่างของชนิดของการตั้งค่าคอนฟิกแบบจำลองข้อมูล (การตั้งค่าคอนฟิกแบบจำลองการชำระเงิน) [![ตัวอย่างของการกำหนดค่าแบบจำลองข้อมูล](./media/pic-model-configuration-1024x585.png)](./media/pic-model-configuration.png)จะคุ้นเคยกับรายละเอียดของสถานการณ์นี้ เล่น**โมเดลข้อมูล ER แมปไปยังแหล่งข้อมูลที่เลือก**คำแนะนำของงาน (เป็นส่วนหนึ่งของการ**7.5.4.3 IT รับ/จัดทำคอมโพเนนต์ โซลูชัน/service (10677)**กระบวนการทางธุรกิจ)

### <a name="building-a-format-that-uses-a-data-model-as-a-base"></a>การสร้างรูปแบบที่ใช้แบบจำลองข้อมูลเป็นพื้นฐาน

ER สนับสนุนผู้ออกแบบรูปแบบที่คุณสามารถใช้ในการสร้างรูปแบบของเอกสารอิเล็กทรอนิกส์เฉพาะ สำหรับโดเมนทางธุรกิจที่เลือกซึ่งเลือกส่วนประกอบแบบจำลองให้เป็นพื้นฐาน ผู้ออกแบบรูปแบบ ER เดียวกันช่วยให้คุณสามารถแม็ปรูปแบบที่คุณสร้างไปยังการแม็ปแบบจำลองข้อมูลของโดเมนที่เลือกให้เป็นแหล่งข้อมูล ภาพประกอบต่อไปนี้แสดงตัวอย่างของรูปแบบชนิดนี้ (การตั้งค่าคอนฟิกรูปแบบที่สนับสนุนรูปแบบการชำระเงิน **BACS** สำหรับสหราชอาณาจักร) [![ตัวอย่างของรูปแบบที่มีแบบจำลองข้อมูลเป็นฐาน](./media/pic-format-1024x690.png)](./media/pic-format.png)จะคุ้นเคยกับรายละเอียดของสถานการณ์นี้ เล่น**รูปแบบเฉพาะของโดเมนออกแบบ ER**คำแนะนำของงาน (เป็นส่วนหนึ่งของการ**7.5.4.3 IT รับ/จัดทำคอมโพเนนต์ โซลูชัน/service (10677)**กระบวนการทางธุรกิจ)

ตัวออกแบบรูปแบบ ER สามารถใช้ในการสร้างเอกสารอิเล็กทรอนิกส์เฉพาะในรูปแบบของแผ่นงาน OPENXML ภาพประกอบต่อไปนี้แสดงตัวอย่างของรูปแบบ (รูปแบบโครงแบบที่จะสร้างแผ่นงาน OPENXML ที่ มีรายละเอียดของสมุดรายวันการชำระเงินที่เลือก) ชนิดนี้:[![Pic-ER-รูปแบบ Excel](./media/pic-er-format-excel.jpg)](./media/pic-er-format-excel.jpg)จะคุ้นเคยกับรายละเอียดของสถานการณ์นี้ เล่น**ER สร้างการกำหนดค่าสำหรับการรายงานในรูปแบบ OPENXML**คำแนะนำของงาน (เป็นส่วนหนึ่งของการ**7.5.4.3 IT รับ/จัดทำคอมโพเนนต์ โซลูชัน/service (10677)**กระบวนการทางธุรกิจ) ใช้การอ้างอิงด้านล่าง Excel แฟ้มเป็นแม่แบบของรูปแบบการออกแบบ ER เพื่อทำตามขั้นตอนของการนำเข้าแม่แบบรูปแบบของคำแนะนำงานนี้:[แม่แบบของรายงานการชำระเงิน (SampleVendPaymWsReport.xlsx)](https://go.microsoft.com/fwlink/?linkid=845202)

### <a name="storing-a-designed-format-component-in-a-format-configuration"></a>การจัดเก็บส่วนประกอบรูปแบบที่ได้ออกแบบไว้ในการตั้งค่าคอนฟิกรูปแบบ

ER สามารถเก็บรูปแบบที่ถูกออกแบบมาพร้อมกับการแม็ปข้อมูลการกำหนดค่าเป็นการตั้งค่าคอนฟิกรูปแบบของการ 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงาน ภาพประกอบก่อนหน้านี้แสดงตัวอย่างของการตั้งค่าคอนฟิกรูปแบบชนิดนี้ (**BACS (สหราชอาณาจักร)**ซึ่งเป็นข้อมูลรองของการตั้งค่าคอนฟิก **รูปแบบการชำระเงิน **) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **รูปแบบเฉพาะโดเมนการออกแบบ ER** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="configuring-dynamics-365-for-operations-to-start-to-use-a-created-format-internally"></a>การตั้งค่าคอนฟิก 365 Dynamics สำหรับการดำเนินงานเพื่อเริ่มต้นการใช้รูปแบบที่สร้างขึ้นภายใน

Dynamics 365 สำหรับการดำเนินงานสามารถถูกกำหนดค่าเพื่อเริ่มต้นการใช้รูปแบบที่สร้างขึ้นในการสร้างรายงานทางอิเล็กทรอนิกส์ การอ้างอิงถึงการตั้งค่าคอนฟิกรูปแบบที่สร้างไว้ควรจะถูกกำหนดในการตั้งค่าของโดเมนเฉพาะ ตัวอย่าง การใช้การตั้งค่าคอนฟิกรูปแบบ ER สำหรับการชำระเงินของผู้จัดจำหน่ายทางอิเล็กทรอนิกส์ในรูปแบบ BACS เริ่มต้น การตั้งค่าคอนฟิกรูปแบบควรใช้อ้างอิงในระบุวิธีการชำระเงิน ดังที่แสดงในภาพประกอบต่อไปนี้: 

[![การตั้งค่าคอนฟิกรูปแบบ BACS (สหราชอาณาจักร)](media/ger-bacs-uk-format-configuration.png) 

[![การอ้างอิงรูปแบบ BACS (สหราชอาณาจักร) ในวิธีการชำระเงิน](media/ger-bacs-uk-format-method.png) 

เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER ใช้รูปแบบในการสร้างเอกสารทางอิเล็กทรอนิกส์สำหรับการชำระเงิน** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )

## <a name="handling-er-components"></a>การจัดการส่วนประกอบ ER
### <a name="publishing-an-er-component-in-lcs-to-offer-it-externally-localization"></a>การเผยแพร่องค์ประกอบ ER ใน LCS เพื่อนำเสนอภายนอก (การแปล)

เจ้าของส่วนประกอบ (แบบจำลองหรือรูปแบบ) ที่สร้างขึ้นสามารถใช้ ER สำหรับการเผยแพร่เวอร์ชันที่เสร็จสมบูรณ์ของส่วนประกอบไปยัง LCS ต้องมีการระบุที่เก็บของชนิด **โครงการ LCS **สำหรับผู้ให้บริการการตั้งค่าคอนฟิก ER ปัจจุบัน เมื่อสถานะของเวอร์ชันที่เสร็จสมบูรณ์ของส่วนประกอบถูกเปลี่ยนแปลงจาก **เสร็จสมบูรณ์แล้ว** ไปเป็น **ใช้ร่วมกัน** เวอร์ชันนั้นได้รับการเผยแพร่ใน LCS เมื่อส่วนประกอบได้รับการเผยแพร่ไปยัง LCS เจ้าของส่วนประกอบนั้นจะกลายเป็นผู้ให้บริการเพื่อสนับสนุนองค์ประกอบนั้น ตัวอย่างเช่น ถ้าส่วนประกอบรูปแบบนี้มีวัตถุประสงค์เพื่อสร้างเอกสารทางอิเล็กทรอนิกส์ที่จำเป็นตามกฎหมาย (ตัวอย่างเช่น ให้สอดคล้องกับสถานการณ์จำลองการแปล) บริการนี้สันนิษฐานให้รักษารูปแบบนี้ที่สอดคล้องกับการเปลี่ยนแปลงทางกฎหมาย และผู้ให้บริการจะออกใช้ส่วนประกอบเวอร์ชันใหม่เมื่อใดก็ตามที่มีความต้องการทางกฎหมายใหม่ เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพโหลดการตั้งค่าคอนฟิกลงใน Lifecycle Services **(ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="importing-an-er-component-from-lcs-to-use-it-internally"></a>การนำเข้าส่วนประกอบ ER จาก LCS เพื่อใช้เป็นการภายใน

ER ช่วยให้คุณสามารถนำเข้าคอมโพเนนต์ ER จาก LCS ไป 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงาน ต้องมีการระบุที่เก็บของชนิด **โครงการ LCS ** เมื่อคอมโพเนนต์ ER ถูกนำเข้าจาก LCS ไป 365 Dynamics ปัจจุบันสำหรับอินสแตนซ์ของการดำเนินงาน เจ้าของอินสแตนซ์ที่กลายเป็น ผู้บริโภคของบริการที่ให้ไว้ โดยเจ้าของ (ผู้เขียน) ของส่วนประกอบนำเข้า ตัวอย่างเช่น ถ้าส่วนประกอบของรูปแบบมีวัตถุประสงค์เพื่อสร้างเอกสารอิเล็กทรอนิกส์เฉพาะจาก 365 Dynamics สำหรับการดำเนินงานในรูปแบบเฉพาะประเทศ/ภูมิภาค (แปลสถานการณ์), ก็จะถือว่าว่า ผู้ที่ใช้บริการจะสามารถขอรับการปรับปรุงใด ๆ ที่เกิดขึ้นกับรูปแบบนั้น เพื่อให้เข้ากันได้กับความต้องการ legislative เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER นำเข้าการตั้งค่าคอนฟิกจาก Lifecycle Services** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="building-a-format-selecting-another-format-as-a-base-customization"></a>การสร้างรูปแบบที่เลือกอีกรูปแบบหนึ่งให้เป็นพื้นฐาน (การเลือกกำหนด)

ER ทำให้คุณสามารถสร้าง (สืบทอดมา) ส่วนประกอบใหม่จากส่วนประกอบ (พื้นฐาน) รุ่นปัจจุบันที่ถูกนำเข้าจาก LCS ตัวอย่างเช่น ผู้ใช้ต้องการสืบทอดรูปแบบใหม่เพื่อดำเนินการข้อกำหนดพิเศษบางอย่างสำหรับเอกสารอิเล็กทรอนิกส์ (เช่น ฟิลด์เพิ่มเติมหรือคำอธิบายที่ครอบคลุม) เพื่อสนับสนุนสถานการณ์จำลองที่กำหนดเอง เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพเกรดรูปแบบโดยการใช้เวอร์ชันพื้นฐานใหม่** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )

### <a name="upgrading-a-format-selecting-a-new-version-of-base-format-rebase"></a>การอัพเกรดรูปแบบที่เลือกเวอร์ชันใหม่ของรูปแบบพื้นฐาน (ปรับใช้ซ้ำ)

ER ทำให้คุณสามารถนำการเปลี่ยนแปลงมาใช้โดยอัตโนมัติ ของเวอร์ชันล่าสุดของส่วนประกอบพื้นฐานในเวอร์ชันแบบร่างปัจจุบันของส่วนประกอบที่ได้รับมา กระบวนการนี้เรียกว่า *การปรับใช้ซ้ำ* ตัวอย่างเช่น การเปลี่ยนแปลงข้อบังคับใหม่ทีได้นำไปใช้ในเวอร์ชันล่าสุดของรูปแบบที่มีการนำเข้าจาก LCS สามารถรวมโดยอัตโนมัติในเวอร์ชันที่กำหนดเองของเอกสารทางอิเล็กทรอนิกส์รูปแบบนี้ การเปลี่ยนแปลงใดๆ ที่ไม่สามารถผสานโดยอัตโนมัติจะถือว่าเป็นความขัดแย้ง ความขัดแย้งเหล่านี้จะแสดงสำหรับการแก้ปัญหาด้วยตนเองในเครื่องมือผู้ออกแบบสำหรับส่วนประกอบที่เหมาะสม เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพเกรดรูปแบบโดยการใช้เวอร์ชันพื้นฐานใหม่** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )
