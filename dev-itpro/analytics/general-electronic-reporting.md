# <a name="electronic-reporting-overview"></a>ภาพรวมการรายงานทางอิเล็กทรอนิกส์

[!include[banner](../includes/banner.md)]


บทความนี้แสดงภาพรวมของเครื่องมือการรายงานทางอิเล็กทรอนิกส์ (ER) ซึ่งรวมถึงข้อมูลเกี่ยวกับแนวคิดหลัก สถานการณ์จำลองที่ ER สนับสนุน และรายการรูปแบบที่ได้รับการออกแบบและนำออกใช้เป็นส่วนหนึ่งของโซลูชัน

การรายงานทางอิเล็กทรอนิกส์ (ER) คือเครื่องมือที่คุณสามารถใช้เพื่อตั้งค่าคอนฟิกรูปแบบสำหรับเอกสารอิเล็กทรอนิกส์ที่สอดคล้องกับข้อกำหนดตามกฎหมายของประเทศ/ภูมิภาคต่างๆ ER ช่วยให้คุณสามารถจัดการรูปแบบเหล่านี้ในระหว่างรอบการใช้งาน ตัวอย่างเช่น คุณสามารถนำข้อบังคับใหม่มาใช้ และสามารถสร้างเอกสารทางธุรกิจในรูปแบบที่กำหนดเพื่อแลกเปลี่ยนข้อมูลกับหน่วยงานของรัฐบาล ธนาคาร และฝ่ายอื่นๆ ทางอิเล็กทรอนิกส์ กลไกจัดการ ER มีเป้าหมายที่ผู้ใช้ทางธุรกิจแทนนักพัฒนา เนื่องจากคุณตั้งค่าคอนฟิกรูปแบบ ไม่ใช่รหัส กระบวนการของการสร้างและการปรับรูปแบบสำหรับเอกสารอิเล็กทรอนิกส์จึงรวดเร็วและง่ายขึ้น ขณะนี้ ER สนับสนุนรูปแบบแผ่นงาน TEXT, XML และ OPENXML อย่างไรก็ตาม อินเทอร์เฟซส่วนขยายให้การสนับสนุนสำหรับรูปแบบเพิ่มเติม

## <a name="capabilities"></a>ความสามารถ
กลไกจัดการ ER มีความสามารถต่อไปนี้:

-   แสดงถึงเครื่องมือเดี่ยวทั่วไปสำหรับการรายงานทางอิเล็กทรอนิกส์ในโดเมนอื่นๆ และมีการแทนที่กลไกจัดการมากกว่า 20 แบบกลไกที่แตกต่างกันที่ทำการรายงานทางอิเล็กทรอนิกส์ของ Microsoft Dynamics 365 for Operations
-   ซึ่งทำให้รูปแบบของรายงานมีป้องกันจากการใช้งาน Dynamics 365 for Operations (นั่นคือรูปแบบดังกล่าวสามารถใช้ได้กับ Dynamics 365 for Operations เวอร์ชันต่างๆ)
-   สนับสนุนการสร้างรูปแบบที่กำหนดเองที่เป็นไปตามรูปแบบเดิม ประกอบด้วยความสามารถในการอัพเกรดรูปแบบที่กำหนดเองโดยอัตโนมัติเมื่อเกิดการเปลี่ยนแปลงไปเป็นรูปแบบเดิมเนื่องจากมีการแนะนำความต้องการในการแปล / การกำหนดค่าด้วยตนเอง
-   จะกลายเป็นเครื่องมือมาตรฐานหลักในการสนับสนุนความต้องการในการแปลในการรายงานทางอิเล็กทรอนิกส์ทั้งสำหรับ Microsoft และคู่ค้า Microsoft
-   จะสามารถสนับสนุนความสามารถในการจัดรูปแบบที่มีการกระจายสำหรับคู่ค้าและลูกค้าโดยใช้ Microsoft Dynamics Lifecycle Services (LCS)

## <a name="concepts"></a>แนวคิด
### <a name="components"></a>ส่วนประกอบ

ER สนับสนุนชนิดของส่วนประกอบสองชนิด **แบบจำลองข้อมูล**และ **รูปแบบ**

#### <a name="data-model-components"></a>ส่วนประกอบแบบจำลองข้อมูล

ส่วนประกอบของแบบจำลองข้อมูลเป็นการนำเสนอทางนามธรรมของโครงสร้างข้อมูลที่จะใช้เพื่ออธิบายพื้นที่โดเมนทางธุรกิจบางพื้นที่ ซึ่งมีการแสดงรายละเอียดเพียงพอที่จะตอบสนองความต้องการรายงานในโดเมนนี้ ส่วนประกอบของแบบจำลองข้อมูลประกอบด้วยส่วนต่างๆ ดังต่อไปนี้:

-   แบบจำลองข้อมูลที่เป็นชุดของเอนทิตี้ทางธุรกิจเฉพาะโดเมนรวมทั้งคำนิยามทางโครงสร้างตามลำดับชั้นของความสัมพันธ์ระหว่างกัน
-   การแม็ปแบบจำลองที่เชื่อมโยงแหล่งข้อมูล Dynamics 365 for Operations ที่เลือก กับแบบจำลองข้อมูลนี้ที่ระบุ ณ เวลาในการดำเนินการ ลำดับข้อมูลและกฎของการเติมข้อมูลทางธุรกิจไปยังส่วนประกอบของแบบจำลองข้อมูล

เอนทิตี้้ทางธุรกิจของรูปแบบจำลองข้อมูลถูกแสดงเป็นตู้บรรจุสินค้า (เรกคอร์ด) คุณสมบัติของเอนทิตี้ทางธุรกิจจะถูกแสดงเป็นรายการข้อมูล (ฟิลด์) รายการข้อมูลแต่ละรายการมีชื่อเฉพาะ ป้ายชื่อ คำอธิบาย และค่า ค่าของรายการข้อมูลแต่ละรายการสามารถถูกออกแบบเพื่อให้มีการรับรู้เป็นสตริง จำนวนเต็ม จำนวนจริง วัน ชนิดการแจง บูลีน และอื่นๆ ได้ นอกจากนี้ยังสามารถเป็นเรกคอร์ดหรือรายการเรกคอร์ดอื่นได้ ส่วนประกอบของแบบจำลองข้อมูลเดี่ยวสามารถประกอบด้วยลำดับชั้นหลายลำดับของเอนทิตี้ทางธุรกิจเฉพาะโดเมนรวมทั้งการแม็ปแบบจำลองเพื่อสนับสนุนลำดับข้อมูลเฉพาะรายงานเฉพาะที่เวลาการดำเนินงาน ลำดับชั้นจะถูกทำให้แตกต่างกันโดยเรกคอร์ดเดี่ยวที่ได้ถูกเลือกให้เป็นรากของการแม็ปแบบจำลอง ตัวอย่างเช่น แบบจำลองข้อมูลของพื้นที่โดเมนการชำระเงินอาจสนับสนุนการแม็ปต่อไปนี้:

-   บริษัท -&gt; ผู้จัดจำหน่าย -&gt; ธุรกรรมการชำระเงินของโดเมน AP
-   ลูกค้า -&gt; บริษัท -&gt; ธุรกรรมการชำระเงินของโดเมน AR

โปรดสังเกตว่าเอนทิตี้ธุรกิจ (เช่น ธุรกรรมของบริษัทและการชำระเงิน) ได้รับการออกแบบเพียงครั้งเดียวเท่านั้น การแม็ปที่ต่างกันจึงนำมาใช้ใหม่ การแม็ปแบบจำลองมีความสามารถต่อไปนี้:

-   สามารถใช้ชนิดข้อมูลที่แตกต่างกันของ Dynamics 365 for Operations เป็นแหล่งข้อมูลสำหรับแบบจำลองข้อมูล ตัวอย่างเช่น สามารถใช้ตาราง เอนทิตีข้อมูล วิธีการ หรือ enums
-   สนับสนุนพารามิเตอร์ป้อนเข้าของผู้ใช้ที่สามารถกำหนดให้เป็นแหล่งข้อมูลของแบบจำลองข้อมูลเมื่อต้องระบุข้อมูลบางอย่างขณะทำงาน
-   สนับสนุนการแปลงข้อมูล Dynamics 365 for Operations เป็นกลุ่ม การกรอง การเรียงลำดับ และข้อมูลการรวมที่กำหนด และยังผนวกกับฟิลด์ที่คำนวณตามตรรกะที่ได้รับการออกแบบผ่านสูตรที่เหมือนกับ Microsoft Excel (ดูรายละเอียดเพิ่มเติมที่ [โปรแกรมออกแบบสูตรในการรายงานทางอิเล็กทรอนิกส์](general-electronic-reporting-formula-designer.md))

[![โปรแกรมแก้ไขสูตรเหมือน Excel](./media/pic-formula-1024x615.png)](./media/pic-formula.png) ส่วนประกอบแบบจำลองข้อมูลถูกออกแบบมาสำหรับโดเมนธุรกิจแต่ละโดเมนที่ควรใช้เป็นแหล่งข้อมูลรวมสำหรับการรายงานที่แยกการรายงานออกจากการใช้งานทางกายภาพของแหล่งข้อมูลของ Dynamics 365 for Operations และแสดงถึงแนวคิดทางธุรกิจเฉพาะโดเมนและฟังก์ชันในแบบฟอร์มที่ทำให้การออกแบบเริ่มต้นของรูปแบบการรายงานและการบำรุงรักษาในอนาคตมีประสิทธิภาพมากขึ้น

#### <a name="format-components"></a>ส่วนประกอบรูปแบบ

ส่วนประกอบรูปแบบมีโครงร่างของการรายงานผลลัพธ์ที่จะถูกสร้างขณะเวลาดำเนินงาน โครงร่างประกอบด้วยองค์ประกอบต่อไปนี้:

-   รูปแบบที่กำหนดโครงสร้างและเนื้อหาของเอกสารการรายงานทางอิเล็กทรอนิกส์ที่ถูกสร้างขึ้นขณะเวลาดำเนินงาน
-   แหล่งข้อมูลที่เป็นชุดของพารามิเตอร์ข้อมูลป้อนเข้าของผู้ใช้และแบบจำลองข้อมูลที่เฉพาะเจาะจงโดเมนที่ใช้การแม็ปแบบจำลองที่เลือก
-   การแม็ปรูปแบบให้เป็นชุดของการผูกแหล่งข้อมูลรูปแบบกับองค์ประกอบแต่ละรายการของรูปแบบที่ระบุขณะเวลาดำเนินงานลำดับข้อมูล และกฎของการสร้างผลลัพธ์รูปแบบ
-   การตรวจสอบรูปแบบที่เป็นชุดของกฎที่ตั้งค่าคอนฟิกได้ ซึ่งควบคุมการสร้างรายงานที่ขณะดำเนินงานซึ่งขึ้นอยู่กับการดำเนินบริบท (ตัวอย่างเช่น กฎที่หยุดการสร้างผลลัพธ์ของการชำระเงินของผู้จัดจำหน่าย และแสดงข้อยกเว้นเมื่อแอททริบิวต์เฉพาะของผู้จัดจำหน่ายที่เลือกหายไป เช่น หมายเลขบัญชีธนาคาร)

ส่วนประกอบของรูปแบบสนับสนุนฟังก์ชันต่อไปนี้:

-   การสร้างผลลัพธ์การรายงานเป็นไฟล์แต่ละรายการในรูปแบบต่างๆ: ข้อความ XML หรือแผ่นงาน
-   การสร้างหไฟล์หลายรายการที่แยกต่างหาก และยังประกอบด้วยไฟล์เหล่านั้นลงในไฟล์ zip

ส่วนประกอบรูปแบบช่วยให้สามารถแนบไฟล์เฉพาะที่สามารถใช้ในการรายงานผลลัพธ์ได้:

-   สมุดงาน Excel ซึ่งประกอบด้วยแผ่นงานที่สามารถนำมาใช้เป็นเท็มเพลตสำหรับผลลัพธ์ในรูปแบบของแผ่นงาน OPENXML
-   ไฟล์อื่นๆที่สามารถรวมเป็นผลลัพธ์ของรูปแบบดังที่ได้กำหนดไว้ล่วงหน้า

#### <a name="component-versioning"></a>การกำหนดเวอร์ชันส่วนประกอบ

การกำหนดเวอร์ชันได้รับการสนับสนุนสำหรับส่วนประกอบ ER มีการระบุลำดับงานต่อไปนี้สำหรับการจัดการการเปลี่ยนแปลงในส่วนประกอบ ER:

-   รุ่นที่ถูกสร้างขึ้นแต่เดิมถูกทำเครื่องหมายเป็นรุ่น **ร่าง** รุ่นนี้สามารถแก้ไขได้และจะพร้อมใช้งานสำหรับการรันการทดสอบ
-   รุ่น**ร่าง** สามารถแปลงเป็นรุ่น **เสร็จสมบูรณ์** ได้ รุ่นนี้สามารถใช้ในกระบวนการรายงานภายใน
-   รุ่น **เสร็จสมบูรณ์** สามารถแปลงเป็นรุ่น **ใช้ร่วมกัน** ได้ รุ่นนี้ถูกเผยแพร่บน LCS และสามารถใช้ในกระบวนการรายงานส่วนกลาง
-   รุ่น **ใช้ร่วมกัน**สามารถแปลงเป็นรุ่น **ที่ไม่ต่อเนื่อง** ได้ จากนั้นสามารถลบรุ่นนี้ออกได้

รุ่นที่มีสถานะ **เสร็จสมบูรณ์** หรือ **ใช้ร่วมกัน** อย่างใดอย่างหนึ่งจะพร้อมใช้งานสำหรับการแลกเปลี่ยนข้อมูลอื่นๆ ส่วนประกอบที่มีสถานะเหล่านี้สามารถกระทำการดำเนินการเหล่านี้ได้:

-   สามารถทำให้เป็นแบบอนุกรมในรูปแบบ XML และส่งออกจาก Dynamics 365 for Operations เป็นไฟล์ในรูปแบบ XML
-   สามารถทำให้เป็นแบบอนุกรมอีกครั้งจากไฟล์ XML และนำเข้ามาใน Dynamics 365 for Operations เป็นส่วนประกอบ ER รุ่นใหม่

#### <a name="component-date-effectivity"></a>วันที่มีผลบังคับใช้ส่วนประกอบ

รุ่นส่วนประกอบ ER คือวันที่มีผลบังคับใช้ วันที่**มีผลบังคับใช้ตั้งแต่**สามารถถูกกำหนดสำหรับส่วนประกอบ ER เพื่อระบุวันที่ที่ส่วนประกอบนี้จะมีผลบังคับใช้สำหรับกระบวนการการรายงาน วันที่รอบเวลา Dynamics 365 for Operations จะใช้เพื่อกำหนดว่าส่วนประกอบถูกต้องสำหรับการดำเนินการหรือไม่ ถ้ารุ่นมากกว่าหนึ่งรุ่นใช้ได้กับวันที่เฉพาะ รุ่นล่าสุดจะใช้สำหรับกระบวนการการรายงาน

#### <a name="component-access"></a>การเข้าถึงส่วนประกอบ

การเข้าถึงส่วนประกอบรูปแบบ ER ขึ้นอยู่กับการตั้งค่ารหัสประเทศ/ภูมิภาค ISO เมื่อการตั้งค่านี้เป็นค่าว่างสำหรับรุ่นที่เลือกของการตั้งค่าคอนฟิกรูปแบบ ส่วนประกอบรูปแบบสามารถเข้าถึงจากบริษัท Dynamics 365 for Operations ใดๆ ในขณะใช้งาน เมื่อการตั้งค่านี้ประกอบด้วยรหัสประเทศ/ภูมิภาค ISO ส่วนประกอบรูปแบบที่ใช้งานได้จากเฉพาะบริษัท Dynamics 365 for Operations เท่านั้น ซึ่งมีการกำหนดที่อยู่หลักสำหรับหนึ่งในรหัสประเทศ/ภูมิภาค ISO ของส่วนประกอบรูปแบบ รุ่นต่างๆของส่วนประกอบของรูปแบบข้อมูลอาจมีการตั้งค่าที่แตกต่างกันตามรหัสประเทศ/ภูมิภาค ISO

#### <a name="configuration"></a>การจัดโครงแบบ

การตั้งค่าคอนฟิก ER เป็น wrapper ของส่วนประกอบ ER เฉพาะ **แบบจำลองข้อมูล**หรือ **รูปแบบ** อย่างใดอย่างหนึ่ง การตั้งค่าคอนฟิกอาจประกอบด้วยเวอร์ชันต่างๆของส่วนประกอบ ER เฉพาะ การตั้งค่าคอนฟิกแต่ละรายการถูกทำเครื่องหมายว่าเป็นเจ้าของโดยผู้ให้บริการการตั้งค่าคอนฟิกเฉพาะ เวอร์ชัน **ร่าง** ของส่วนประกอบการตั้งค่าคอนฟิกสามารถแก้ไขได้ เมื่อมีการเลือกเจ้าของการตั้งค่าคอนฟิกให้เป็นผู้ให้บริการที่ใช้งานในการตั้งค่า ER ใน Dynamics 365 for Operations การตั้งค่าคอนฟิกแบบจำลองแต่ละรายการประกอบด้วยส่วนประกอบ **แบบจำลองข้อมูล** การตั้งค่าคอนฟิกรูปแบบใหม่สามารถถูกสร้าง (ได้รับ) จากการตั้งค่าคอนฟิกแบบจำลองข้อมูลที่เฉพาะเจาะจง การตั้งค่าคอนฟิกรูปแบบที่สร้างขึ้นจะแสดงในแผนภูมิการตั้งค่าคอนฟิกเป็นข้อมูลรองของการตั้งค่าคอนฟิกแบบจำลองข้อมูลเดิม การตั้งค่าคอนฟิกรูปแบบที่สร้างขึ้นประกอบด้วยส่วนประกอบ **รูปแบบ** ส่วนประกอบ **แบบจำลองข้อมูล** ของการตั้งค่าคอนฟิกแบบจำลองเดิมจะถูกแทรกโดยอัตโนมัติไปยังส่วนประกอบ **รูปแบบ**ของการตั้งค่าคอนฟิกรูปแบบรองที่สร้างขึ้นเป็นแหล่งข้อมูลเริ่มต้น การตั้งค่าคอนฟิก ER ถูกใช้ร่วมกันสำหรับบริษัท Dynamics 365 for Operations

#### <a name="provider"></a>ผู้จัดหา

ผู้ให้บริการ ER คือ ตัวระบุฝ่ายที่ใช้เพื่อบ่งชี้ผู้เขียน (เจ้าของ) ของการตั้งค่าคอนฟิก ER แต่ละรายการ ER ช่วยให้คุณสามารถจัดการรายการของผู้ให้บริการการตั้งค่าคอนฟิก การตั้งค่าคอนฟิกรูปแบบที่ถูกนำออกใช้สำหรับเอกสารทางอิเล็กทรอนิกส์โดยเป็นส่วนหนึ่งของโซลูชัน Dynamics 365 for Operations ถูกทำเครื่องหมายว่าเป็นเจ้าของโดยผู้ให้บริการการตั้งค่าคอนฟิก **Microsoft**

#### <a name="repository"></a>ที่เก็บ

ที่เก็บ ER เก็บการตั้งค่าคอนฟิก ER ประเภทของที่เก็บ ER ต่อไปนี้ได้รับการสนับสนุนอยู่ในขณะนี้: **ทรัพยากรการดำเนินการ** และ **โครงการ LCS** ที่เก็บ **ทรัพยากรการดำเนินงาน** มอบการเข้าถึงรายการของการตั้งค่าคอนฟิกที่ถูกนำออกใช้โดยเป็นส่วนหนึ่งของโซลูชัน Dynamics 365 for Operationsโดย Microsoft เป็นผู้ให้บริการการตั้งค่าคอนฟิก ER การตั้งค่าคอนฟิกเหล่านั้นสามารถนำเข้าไปยังอินสแตนซ์ของ Dynamics 365 for Operations ปัจจุบัน และใช้สำหรับการรายงานทางอิเล็กทรอนิกส์ สามารถใช้สำหรับการแปลเป็นภาษาท้องถิ่น / การเลือกกำหนดเพิ่มเติม ที่เก็บ **โครงการ LCS**มอบการเข้าถึงรายการของการตั้งค่าคอนฟิกโครงการ LCS ที่เฉพาะเจาะจง (ไลบรารีสินทรัพย์โครงการ LCS) ที่ได้เลือกไว้ในขั้นตอนการลงทะเบียนที่เก็บ ER ทำให้คุณสามารถอัพโหลดการตั้งค่าคอนฟิกที่ใช้ร่วมกันจากอินสแตนซ์ Dynamics 365 for Operations ปัจจุบันไปยังที่เก็บ **โครงการ LCS** เฉพาะ นอกจากนี้คุณยังสามารถนำเข้าการตั้งค่าคอนฟิกจากที่เก็บ **โครงการ LCS** เฉพาะลงในอินสแตนซ์ Dynamics 365 for Operations ปัจจุบัน ที่เก็บข้อมูล **โครงการ LCS** ที่จำเป็นสามารถลงทะเบียนแยกกันสำหรับผู้ให้บริการการตั้งค่าคอนฟิกอินสแตน Dynamics 365 for Operations ปัจจุบัน ที่เก็บแต่ละที่สามารถใช้ได้เฉพาะกับผู้ให้บริการการตั้งค่าคอนฟิกเฉพาะ

## <a name="supported-scenarios"></a>สถานการณ์จำลองที่ได้รับการสนับสนุน
### <a name="building-a-data-model"></a>การสร้างแบบจำลองของข้อมูล

ER มอบผู้ออกแบบแบบจำลองที่คุณสามารถใช้ในการสร้างแบบจำลองข้อมูลสำหรับโดเมนธุรกิจหนึ่งๆ เอนทิตีทางธุรกิจเฉพาะโดเมนทั้งหมดและความสัมพันธ์ระหว่างกัน สามารถถูกนำเสนอในแบบจำลองข้อมูลให้เป็นโครงสร้างตามลำดับชั้นได้ ภาพประกอบต่อไปนี้แสดงตัวอย่างของชนิดของแบบจำลองข้อมูล (รูปแบบจำลองข้อมูลโดเมนการชำระเงิน) [![ตัวอย่างของรูปแบบข้อมูล](./media/pic-data-model-1024x550.png)](./media/pic-data-model.png) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **แบบจำลองข้อมูลเฉพาะโดเมนการออกแบบ ER** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="translating-data-model-content"></a>การแปลเนื้อหาแบบจำลองข้อมูล

เนื้อหาแบบจำลองข้อมูล (ป้ายชื่อและคำอธิบาย) สามารถแปลเป็นภาษาอื่นๆ ที่ Dynamics 365 for Operations สนับสนุน คุณอาจต้องการแปลเนื้อหาแบบจำลองข้อมูลสำหรับเหตุผลต่อไปนี้:

-   ในขณะออกแบบ เพื่อทำให้เนื้อหาเข้าใจได้ง่ายมากขึ้นสำหรับผู้ออกแบบรูปแบบที่พูดภาษาอื่น และผู้ที่จะใช้แบบจำลองข้อมูลสำหรับการแม็ปข้อมูลของส่วนประกอบรูปแบบ
-   ในขณะเวลาดำเนินงาน เพื่อทำให้เนื้อหาง่ายต่อผู้ใช้งานมากขึ้นโดยการนำเสนอพร้อมต์และความช่วยเหลือของพารามิเตอร์เวลาดำเนินงาน เช่นเดียวกับข้อความการตรวจสอบที่มีการตั้งค่าคอนฟิก (ข้อผิดพลาด คำเตือน) ในภาษาที่ผู้ใช้ Dynamics 365 for Operations ที่ล็อกออนในขณะนี้ต้องการ

ภาพประกอบต่อไปนี้แสดงตัวอย่างของวิธีการแปลเนื้อหาแบบจำลองข้อมูลจากภาษาอังกฤษเป็นภาษาญี่ปุ่น [![เนื้อหาแบบจำลองข้อมูลเป็นภาษาอังกฤษ](./media/pic-translate-en-1024x495.png)](./media/pic-translate-en.png) [![เนื้อหาแบบจำลองข้อมูลที่แปลเป็นภาษาญี่ปุ่น](./media/pic-translate-ja-1024x495.png)](./media/pic-translate-ja.png)

### <a name="configuring-data-model-mappings"></a>การตั้งค่าคอนฟิกการแม็ปแบบจำลองข้อมูล

ER แสดงโปรแกรมออกแบบการแม็ปแบบจำลองที่ช่วยให้ผู้ใช้แม็ปแบบจำลองข้อมูลที่ออกแบบไปยังแหล่งข้อมูล Dynamics 365 for Operations เฉพาะ ภาพประกอบต่อไปนี้แสดงตัวอย่างของการแม็ปแบบจำลองข้อมูลชนิดนี้ (การแม็ปแบบจำลอง **การโอนย้ายเครดิต SEPA** ของแบบจำลองข้อมูลโดเมนการชำระเงิน) [![ตัวอย่างของการแม็ปแบบจำลองข้อมูล ](./media/pic-model-mapping-1024x551.png)](./media/pic-model-mapping.png) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่น **ER กำหนดการแม็ปแบบจำลอง และเลือกแหล่งข้อมูล** และคู่มืองาน **ER แม็ปแบบจำลองข้อมูลไปยังแหล่งข้อมูลที่เลือก** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)**

### <a name="storing-a-designed-model-component-as-a-model-configuration"></a>การจัดเก็บส่วนประกอบแบบจำลองที่ได้ออกแบบไว้ให้เป็นการตั้งค่าคอนฟิกแบบจำลอง

ER สามารถจัดเก็บแบบจำลองข้อมูลที่ออกแบบไว้รวมกับการแม็ปข้อมูลที่เชื่อมโยงเป็นการตั้งค่าคอนฟิกแบบจำลองของอินสแตนซ์ปัจจุบันของ Dynamics 365 for Operations ภาพประกอบต่อไปนี้แสดงตัวอย่างของชนิดของการตั้งค่าคอนฟิกแบบจำลองข้อมูล (การตั้งค่าคอนฟิกแบบจำลองการชำระเงิน) [![ตัวอย่างของการตั้งค่าคอนฟิกแบบจำลองข้อมูล](./media/pic-model-configuration-1024x585.png)](./media/pic-model-configuration.png) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **แม็ปแบบจำลองข้อมูลไปยังแหล่งข้อมูลที่เลือกของ ER** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="building-a-format-that-uses-a-data-model-as-a-base"></a>การสร้างรูปแบบที่ใช้แบบจำลองข้อมูลเป็นพื้นฐาน

ER สนับสนุนผู้ออกแบบรูปแบบที่คุณสามารถใช้ในการสร้างรูปแบบของเอกสารอิเล็กทรอนิกส์เฉพาะ สำหรับโดเมนทางธุรกิจที่เลือกซึ่งเลือกส่วนประกอบแบบจำลองให้เป็นพื้นฐาน ผู้ออกแบบรูปแบบ ER เดียวกันช่วยให้คุณสามารถแม็ปรูปแบบที่คุณสร้างไปยังการแม็ปแบบจำลองข้อมูลของโดเมนที่เลือกให้เป็นแหล่งข้อมูล ภาพประกอบต่อไปนี้แสดงตัวอย่างของรูปแบบชนิดนี้ (การตั้งค่าคอนฟิกรูปแบบที่สนับสนุนรูปแบบการชำระเงิน **BACS** สำหรับสหราชอาณาจักร) [![ตัวอย่างของรูปแบบที่มีแบบจำลองข้อมูลเป็นพื้นฐาน](./media/pic-format-1024x690.png)](./media/pic-format.png) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ออกแบบโดเมนของรูปแบบที่เฉพาะเจาะจงของ ER** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="building-a-configuration-to-generate-electronic-documents-in-openxml-worksheet-format"></a>การสร้างการตั้งค่าคอนฟิกเพื่อสร้างเอกสารอิเล็กทรอนิกส์ในรูปแบบของแผ่นงาน OPENXML

### <a name="storing-a-designed-format-component-in-a-format-configuration"></a>การจัดเก็บส่วนประกอบรูปแบบที่ได้ออกแบบไว้ในการตั้งค่าคอนฟิกรูปแบบ

ER สามารถจัดเก็บรูปแบบที่ออกแบบไว้รวมกับการแม็ปข้อมูลที่มีการตั้งค่าคอนฟิกให้เป็นการตั้งค่าคอนฟิกรูปแบบของอินสแตนซ์ปัจจุบันของ Dynamics 365 for Operations ภาพประกอบก่อนหน้านี้แสดงตัวอย่างของการตั้งค่าคอนฟิกรูปแบบชนิดนี้ (**BACS (สหราชอาณาจักร)**ซึ่งเป็นข้อมูลรองของการตั้งค่าคอนฟิก **รูปแบบการชำระเงิน**) เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **รูปแบบเฉพาะโดเมนการออกแบบ ER** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="configuring-dynamics-365-for-operations-to-start-to-use-a-created-format-internally"></a>การตั้งค่าคอนฟิก Dynamics 365 for Operations เพื่อเริ่มต้นการใช้รูปแบบที่มีการสร้างไว้ภายใน

สามารถตั้งค่าคอนฟิก Dynamics 365 for Operations ได้เพื่อเริ่มต้นการใช้รูปแบบที่สร้างไว้เพื่อสร้างรายงานทางอิเล็กทรอนิกส์ การอ้างอิงถึงการตั้งค่าคอนฟิกรูปแบบที่สร้างไว้ควรจะถูกกำหนดในการตั้งค่าของโดเมนเฉพาะ ตัวอย่างเช่น เมื่อต้องการเริ่มต้นการใช้การตั้งค่าคอนฟิกรูปแบบ ER สำหรับการชำระเงินของผู้จัดจำหน่ายทางอิเล็กทรอนิกส์ในรูปแบบ BACS การตั้งค่าคอนฟิกรูปแบบควรถูกอ้างอิงในวิธีการชำระเงินที่ระบุ ดังที่แสดงในภาพประกอบต่อไปนี้:

เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER ใช้รูปแบบในการสร้างเอกสารทางอิเล็กทรอนิกส์สำหรับการชำระเงิน** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )

## <a name="handling-er-components"></a>การจัดการส่วนประกอบ ER
### <a name="publishing-an-er-component-in-lcs-to-offer-it-externally-localization"></a>การเผยแพร่องค์ประกอบ ER ใน LCS เพื่อนำเสนอภายนอก (การแปล)

เจ้าของส่วนประกอบ (แบบจำลองหรือรูปแบบ) ที่สร้างขึ้นสามารถใช้ ER สำหรับการเผยแพร่เวอร์ชันที่เสร็จสมบูรณ์ของส่วนประกอบไปยัง LCS ต้องมีการระบุที่เก็บของชนิด **โครงการ LCS**สำหรับผู้ให้บริการการตั้งค่าคอนฟิก ER ปัจจุบัน เมื่อสถานะของเวอร์ชันที่เสร็จสมบูรณ์ของส่วนประกอบถูกเปลี่ยนแปลงจาก **เสร็จสมบูรณ์แล้ว** ไปเป็น **ใช้ร่วมกัน** เวอร์ชันนั้นได้รับการเผยแพร่ใน LCS เมื่อส่วนประกอบได้รับการเผยแพร่ไปยัง LCS เจ้าของส่วนประกอบนั้นจะกลายเป็นผู้ให้บริการเพื่อสนับสนุนองค์ประกอบนั้น ตัวอย่างเช่น ถ้าส่วนประกอบรูปแบบนี้มีวัตถุประสงค์เพื่อสร้างเอกสารทางอิเล็กทรอนิกส์ที่จำเป็นตามกฎหมาย (ตัวอย่างเช่น ให้สอดคล้องกับสถานการณ์จำลองการแปล) บริการนี้สันนิษฐานให้รักษารูปแบบนี้ที่สอดคล้องกับการเปลี่ยนแปลงทางกฎหมาย และผู้ให้บริการจะออกใช้ส่วนประกอบเวอร์ชันใหม่เมื่อใดก็ตามที่มีความต้องการทางกฎหมายใหม่ เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพโหลดการตั้งค่าคอนฟิกลงใน Lifecycle Services**(ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="importing-an-er-component-from-lcs-to-use-it-internally"></a>การนำเข้าส่วนประกอบ ER จาก LCS เพื่อใช้เป็นการภายใน

ER ช่วยให้คุณสามารถนำเข้าส่วนประกอบ ER จาก LCS ไปยังอินสแตนซ์ Dynamics 365 for Operations ปัจจุบัน ต้องมีการระบุที่เก็บของชนิด **โครงการ LCS** เมื่อส่วนประกอบ ER ได้ถูกนำเข้าจาก LCS ไปยังอินสแตนซ์ปัจจุบันของ Dynamics 365 for Operations เจ้าของอินสแตนซ์ Dynamics 365 for Operations นี้จะกลายเป็นผู้ใช้ของบริการที่ได้รับจากเจ้าของของส่วนประกอบที่มีการนำเข้า (ผู้เขียน) ตัวอย่างเช่น ถ้าส่วนประกอบรูปแบบนี้มีวัตถุประสงค์เพื่อสร้างเอกสารอิเล็กทรอนิกส์เฉพาะจาก Dynamics 365 for Operations ในรูปแบบเฉพาะบางประเทศ/ภูมิภาค (สถานการณ์จำลองการแปล) จะถือว่าผู้ใช้บริการจะสามารถได้รับการปรับปรุงใดๆ ที่กระทำกับรูปแบบนั้น เพื่อให้สอดคล้องกับความต้องการทางกฎหมาย เมื่อต้องการทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER นำเข้าการตั้งค่าคอนฟิกจาก Lifecycle Services** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ส่วนประกอบกระบวนได้รับ/พัฒนาการบริการ/โซลูชันด้านไอที (10677)** )

### <a name="building-a-format-selecting-another-format-as-a-base-customization"></a>การสร้างรูปแบบที่เลือกอีกรูปแบบหนึ่งให้เป็นพื้นฐาน (การเลือกกำหนด)

ER ทำให้คุณสามารถสร้าง (สืบทอดมา) ส่วนประกอบใหม่จากส่วนประกอบ (พื้นฐาน) รุ่นปัจจุบันที่ถูกนำเข้าจาก LCS ตัวอย่างเช่น ผู้ใช้ต้องการสืบทอดรูปแบบใหม่เพื่อดำเนินการข้อกำหนดพิเศษบางอย่างสำหรับเอกสารอิเล็กทรอนิกส์ (เช่น ฟิลด์เพิ่มเติมหรือคำอธิบายที่ครอบคลุม) เพื่อสนับสนุนสถานการณ์จำลองที่กำหนดเอง เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพเกรดรูปแบบโดยการใช้เวอร์ชันพื้นฐานใหม่** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )

### <a name="upgrading-a-format-selecting-a-new-version-of-base-format-rebase"></a>การอัพเกรดรูปแบบที่เลือกเวอร์ชันใหม่ของรูปแบบพื้นฐาน (ปรับใช้ซ้ำ)

ER ทำให้คุณสามารถนำการเปลี่ยนแปลงมาใช้โดยอัตโนมัติ ของเวอร์ชันล่าสุดของส่วนประกอบพื้นฐานในเวอร์ชันแบบร่างปัจจุบันของส่วนประกอบที่ได้รับมา กระบวนการนี้เรียกว่า *การปรับใช้ซ้ำ* ตัวอย่างเช่น การเปลี่ยนแปลงข้อบังคับใหม่ทีได้นำไปใช้ในเวอร์ชันล่าสุดของรูปแบบที่มีการนำเข้าจาก LCS สามารถรวมโดยอัตโนมัติในเวอร์ชันที่กำหนดเองของเอกสารทางอิเล็กทรอนิกส์รูปแบบนี้ การเปลี่ยนแปลงใดๆ ที่ไม่สามารถผสานโดยอัตโนมัติจะถือว่าเป็นความขัดแย้ง ความขัดแย้งเหล่านี้จะแสดงสำหรับการแก้ปัญหาด้วยตนเองในเครื่องมือผู้ออกแบบสำหรับส่วนประกอบที่เหมาะสม เพื่อทำความคุ้นเคยกับสถานการณ์นี้ในรายละเอียด เล่นคู่มืองาน **ER อัพเกรดรูปแบบโดยการใช้เวอร์ชันพื้นฐานใหม่** (ส่วนหนึ่งของกระบวนการทางธุรกิจ **7.5.4.3 ได้รับ/พัฒนาส่วนประกอบบริการ/โซลูชันทางด้าน IT (10677)** )
