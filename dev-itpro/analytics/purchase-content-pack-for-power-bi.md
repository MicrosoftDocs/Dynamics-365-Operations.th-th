# <a name="purchase-spend-analysis-power-bi-content"></a>ซื้อใช้จ่ายวิเคราะห์เนื้อหาพลังงาน BI

หัวข้อนี้อธิบายถึงสิ่งที่จะรวมในการซื้อการใช้จ่ายวิเคราะห์ชุดเนื้อหาสำหรับ BI พลังงานของ Microsoft จะอธิบายถึงวิธีการเข้าถึงรายงานที่รวมอยู่ในแพเนื้อหา และให้ข้อมูลเกี่ยวกับรูปแบบจำลองข้อมูลและเอนทิตีที่ใช้ในการสร้างแพเนื้อหา

<a name="overview"></a>ภาพรวม
--------

การซื้อการใช้จ่ายวิเคราะห์เนื้อหา pack สำหรับ Microsoft พลังงาน BI ที่สร้างสำหรับผู้จัดการและผู้จัดการที่รับผิดชอบสำหรับงบประมาณการจัดซื้อ ได้ออกแบบเพื่อช่วยในการเฝ้าระวังในการซื้อการใช้จ่าย จะใช้ข้อมูลเกี่ยวกับธุรกรรมการซื้อจาก Microsoft Dynamics 365 สำหรับการดำเนินงาน และช่วยให้ทั้งมุมมองรวมของตัวเลขซื้อทั้งบริษัทและการแบ่งรายละเอียดของการซื้อที่มีการใช้จ่าย โดยผู้จัดจำหน่ายและผลิตภัณฑ์ รายงานเน้นการเปลี่ยนแปลงในการซื้อการใช้จ่ายเมื่อเวลาผ่านไป ดังนั้น จึง จะสามารถใช้เพื่อจัดการการแจ้งเตือนเกี่ยวกับแนวโน้มการใช้จ่ายค่าบวก และค่าลบสำหรับแต่ละผู้ขายและผลิตภัณฑ์ แผนภูมิแสดงการซื้อวงเงินการใช้จ่ายสำหรับประเภทการจัดซื้อที่แตกต่างกันและกลุ่มผู้จัดจำหน่าย ประเภทและผู้จัดการภูมิภาคอาจพบว่ามีประโยชน์เมื่อต้องการใช้แผนภูมิเหล่านี้เพื่อช่วยในการระบุการเปลี่ยนแปลงในลักษณะการทำงานที่ใช้จ่าย ชุดเนื้อหาเราผู้จัดการฝ่ายจัดซื้อและผู้จัดการที่รับผิดชอบงบประมาณวิเคราะห์ซื้อวงเงินการใช้จ่ายโดยใช้วิธีต่อไปนี้:

-   ซื้อปีจนถึงปัจจุบัน (ตามกลุ่มผู้จัดจำหน่าย และผู้จัดจำหน่ายแต่ละ ประเภทการจัดซื้อ และแต่ละผลิตภัณฑ์ และที่ตั้งของผู้จัดจำหน่าย)
-   เปลี่ยนแปลงการซื้อเกินปี (ตามประเภทของผู้จัดจำหน่ายของกลุ่มและการจัดซื้อ)

## <a name="accessing-the-content-pack"></a>เข้าถึงแพเนื้อหา
การซื้อการใช้จ่ายวิเคราะห์เนื้อหาการบรรจุถูกประกาศเป็นสินทรัพย์นำไปใช้ใน Microsoft Dynamics วัฏจักรบริการ (LCS) และสามารถเข้าถึงได้จาก 365 Dynamics Microsoft สำหรับการดำเนินงาน สำหรับข้อมูลเพิ่มเติมเกี่ยวกับวิธีการเข้าถึงและใช้พลังงาน BI เปิดรายงาน ดู[เนื้อหาพลังงาน BI ใน LCS จาก Microsoft และคู่ค้าของคุณ](power-bi-content-microsoft-partners.md)

## <a name="data-model-and-entities"></a>รูปแบบข้อมูลและเอนทิตี
Dynamics 365 สำหรับข้อมูลที่จะใช้สำหรับรายงานในการซื้อการผลิตใช้จ่าย pack เนื้อหาการวิเคราะห์ ข้อมูลนี้จะแสดงเป็นหน่วยวัดรวมที่มีการแบ่งระยะในเอนทิตีเก็บ ซึ่งเป็นฐานข้อมูล Microsoft SQL ที่ดีที่สุดสำหรับการวิเคราะห์ สำหรับข้อมูลเพิ่มเติมเกี่ยวกับเอนทิตีร้านค้า ให้ดู[BI พลังงานรวมกับร้านค้าเอนทิตีใน Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/)ติดประกาศในบล็อก การประเมินรวมในชุดเนื้อหานี้คือ ชุดย่อยของการวัดรวมที่พร้อมใช้งานใน Cube ใน Microsoft Dynamics AX 2012 และ 365 Dynamics Microsoft สำหรับ R3 2012 การดำเนินการซื้อ ระยะของ cube ประเมินรวมในเอนทิตีร้านค้า คุณต้องการให้ deployable สำหรับข้อมูลเพิ่มเติม ให้ดูขั้นตอนในการจัดเตรียมการประเมินรวมในร้านค้าในเอนทิตี[BI พลังงานรวมกับร้านค้าเอนทิตีใน Dynamics](https://blogs.msdn.microsoft.com/dynamicsaxbi/2016/06/09/power-bi-integration-with-entity-store-in-dynamics-ax-7-may-update/)ติดประกาศในบล็อก การประเมินรวมคีย์ต่อไปนี้จะพร้อมใช้งานได้โดยตรงจากเอนทิตีในบรรทัดใบแจ้งหนี้ และใช้เป็นข้อมูลพื้นฐานของชุดเนื้อหา

| เอนทิตี้        | หลักการประเมินรวม | แหล่งข้อมูลสำหรับ 365 Dynamics สำหรับการดำเนินงาน | ฟิลด์              | คำอธิบาย                           |
|---------------|----------------------------|---------------------------------------------|--------------------|---------------------------------------|
| รายการใบแจ้งหนี้ | การซื้อ                   | VendInvoiceTrans                            | SUM(LineAmountMST) | ยอดเงินในสกุลเงินทางบัญชี |

ตารางต่อไปนี้แสดงคีย์การวัดที่คำนวณในแพเนื้อหาจากเอนทิตีในบรรทัดใบแจ้งหนี้

| หน่วยวัด               | การคำนวณ                                                                                         |
|-----------------------|-----------------------------------------------------------------------------------------------------|
| ซื้อปีปัจจุบัน | ซื้อปีปัจจุบัน = SUM ('บรรทัดใบแจ้งหนี้'\[ซื้อ\])                                            |
| ซื้อปีที่แล้ว    | ซื้อปี =คำนวณ (SUM ('บรรทัดใบแจ้งหนี้'\[ซื้อ\]), SAMEPERIODLASTYEAR (วัน\[วัน\])) |
| การเติบโตการซื้อ YOY   | การเติบโตซื้อ YOY =\[ซื้อปีปัจจุบัน\]–\[ซื้อปีที่แล้ว\]                            |

มิติในเนื้อหา pack คีย์ต่อไปนี้จะใช้เป็นตัวกรองเพื่อแบ่งส่วนการประเมินรวม เพื่อให้คุณสามารถใช้ส่วนประกอบและความเข้าใจวิเคราะห์ทางลึกเพิ่มเติม

| เอนทิตี้                 | ตัวอย่างของแอตทริบิวต์                                |
|------------------------|-------------------------------------------------------|
| ผู้จัดจำหน่าย                | กลุ่มผู้จัดจำหน่าย ผู้จัดจำหน่ายประเทศ หรือ ภูมิภาค ชื่อผู้จัดจำหน่าย |
| ผลิตภัณฑ์               | หมายเลขผลิตภัณฑ์ ชื่อผลิตภัณฑ์ ชื่อกลุ่มสินค้า        |
| ประเภทการจัดซื้อ | ประเภทการจัดซื้อ ชื่อประเภทการจัดซื้อ      |
| นิติบุคคล         | ชื่อนิติบุคคล                                     |
| วันที่                  | วัน ปีออฟเซ็ต                                    |

โดยค่าเริ่มต้น ชุดเนื้อหาแสดงข้อมูลสำหรับปีปฏิทินปัจจุบัน อย่างไรก็ตาม คุณสามารถเปลี่ยนตัวกรองวันที่ในส่วนตัวกรองรายงาน คุณยังสามารถเปลี่ยนตัวกรองของบริษัท
