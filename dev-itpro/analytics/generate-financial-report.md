# <a name="generate-a-financial-report"></a>สร้างรายงานทางการเงิน

[!include[banner](../includes/banner.md)]


หัวข้อนี้แสดงข้อมูลเกี่ยวกับการสร้างรายงานทางการเงิน 

เมื่อต้องการสร้างรายงาน เปิดข้อกำหนดของรายงาน และคลิกปุ่ม สร้าง ในแถบเครื่องมือ หน้าต่าง สถานะคิวรายงาน จะเปิด และระบุที่ตั้งของรายงานของคุณในคิวนั้น โดยค่าเริ่มต้น รายงานที่สร้างขึ้นจะเปิดในวิวเวอร์เว็บ
| ![หมายเหตุ](https://i-technet.sec.s-msft.com/areas/global/content/clear.gif "หมายเหตุ")**หมายเหตุ**        |
|------------------------------------------------------------------------------------------------|
| คุณสามารถสร้างรายงานได้เฉพาะกับโฟลเดอร์และตำแหน่งที่ตั้งที่คุณมีสิทธิ์ในการเข้าถึง |

ตารางต่อไปนี้อธิบายตัวเลือกต่างๆที่พร้อมใช้งานสำหรับการสร้างรายงาน

| ตัวเลือก                                                                                | สำหรับข้อมูลเพิ่มเติม |
|---------------------------------------------------------------------------------------|----------------------|
| ตั้งค่ากำหนดการเพื่อสร้างรายงานหรือกลุ่มของรายงานโดยอัตโนมัติ              |                      |
| ตรวจสอบบัญชีที่สูญหาย หรือข้อมูลในรายงาน และตรวจสอบความถูกต้องของรายงาน |                      |

เมื่อคุณสร้างรายงาน ตัวเลือกที่คุณได้ระบุในแท็บ รายงานคำนิยาม จะถูกใช้ แท็บ ผลผลิตและการกระจาย ช่วยให้คุณสามารถระบุที่ตั้งไลบรารีรายงาน ซึ่งเป็นวิธีที่ง่ายในแบ่งปันข้อมูลรายงานนั้น

## <a name="schedule-report-generation"></a> สร้างรายงานกำหนดการ
หลายบริษัทมีชุดหลักของรายงานต่างๆที่จะดำเนินในช่วงเวลาที่จัดกำหนดการไว้เพื่อให้สอดคล้องกับกระบวนการทางธุรกิจ คุณสามารถจัดกำหนดการรายงานที่จะสร้างเป็นประจำ เช่นรายวัน รายสัปดาห์ รายเดือน หรือรายปี นี่อาจเป็นรายงานเดียวหรือกลุ่มรายงานที่ประกอบด้วยหลายบริษัท คุณต้องป้อนข้อมูลประจำตัวของคุณสำหรับแต่ละบริษัทที่ระบุ เช่น ในคำนิยามแผนภูมิการรายงาน ถ้าข้อมูลประจำตัวไม่ถูกต้อง รายงานจะแสดงเฉพาะข้อมูลที่คุณมีสิทธิ์เข้าถึง เช่น บริษัทที่คุณล็อกออนเข้าในเวลานั้น ข้อมูลขาออกจะถูกอ่านเป็นลำดับแรกจากกลุ่มรายงาน แล้วจากแต่ละรายงาน

เมื่อกำหนดการรายงานถูกสร้าง และบันทึก กำหนดการรายงานนั้นจะแสดงอยู่ในบานหน้าต่างนำทางภายใต้ กำหนดการรายงาน คุณสามารถสร้างโฟลเดอร์เพื่อจัดระเบียบรายงาน แม้รายงานหนึ่งในกำหนดการไม่ดำเนินการ รายงานอื่น ๆ ทั้งหมดจะดำเนินการต่อไป
| ![สำคัญ](https://i-technet.sec.s-msft.com/areas/global/content/clear.gif "สำคัญ")**สำคัญ**                                                                                                           |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| เมื่อต้องการสร้าง ปรับเปลี่ยน และลบกำหนดการรายงาน คุณต้องมีบทบาทเป็นผู้ออกแบบหรือผู้ดูแลระบบ เมื่อรันรายงานหนึ่ง ข้อมูลประจำตัวของผู้ใช้ที่สร้างการจัดกำหนดการถูกใช้ในการสร้างรายงาน |

### <a name="create-a-report-schedule"></a>สร้างกำหนดการรายงาน

1.  ในโปรแกรมออกแบบรายงาน ใน เมนู ไฟล์ คลิก ใหม่แล้ว เลือก กำหนดการรายงาน กล่องโต้ตอบ กำหนดการรายงานใหม่ จะเปิดขึ้น
2.  ภายใต้ การตั้งค่า เลือกรายงานเดียวหรือกลุ่มรายงานเพื่อจัดกำหนดการ รายงานหรือกลุ่มการรายงานสำหรับการเลือกบริษัทหรือบล็อคส่วนประกอบที่คุณล็อกออนอยู่เท่านั้นที่พร้อมใช้งาน
3.  เลือกกล่องกาเครื่องหมายที่ ใช้งานนี้ เพื่อกำหนดการรายงานนั้น เฉพาะผู้สร้างรายงานหรือผู้ดูแลระบบสามารถเรียกใช้ หรือยกเลิกใช้กำหนดการรายงาน
4.  คลิกปุ่ม สิทธิ์ เพื่อป้อนข้อมูลประจำตัวของบริษัท โดยค่าเริ่มต้น ข้อมูลการเข้าสู่ระบบของคุณใช้สำหรับบริษัทที่คุณล็อกออน ถ้าบริษัทอื่น ๆ รวมมาด้วย เช่น ในการรายงานคำนิยามแผนภูมิ เลือก ใช้ข้อมูลประจำตัวแยกต่างหากแล้ว ป้อนข้อมูลประจำตัวสำหรับบริษัทอื่น ๆ ที่รวมอยู่ในกำหนดการจัดทำรายงาน คุณสามารถเลือก การพิสูจน์ตัวจริงของ Windows หรือพิมพ์ชื่อผู้ใช้และรหัสผ่านสำหรับแต่ละบริษัทได้ เลือกกล่องกาเครื่องหมาย บันทึกข้อมูลประจำตัว เพื่อบันทึกข้อมูลประจำตัวสำหรับบริษัทเหล่านี้ และคลิก ตกลง เพื่อปิดกล่องโต้ตอบ
5.  ภายใต้ ความถี่ในฟิลด์ เริ่มการเกิดซ้ำ เลือกวันที่จัดกำหนดการเพื่อเริ่มต้น โดยค่าเริ่มต้น มีการวันที่ของระบบปัจจุบันของคอมพิวเตอร์ไคลเอนต์
6.  ในฟิลด์ รันรายงาน เลือกเวลาควรรันรายงาน ถ้าคุณป้อนเวลาที่อยู่ก่อนหน้าเวลาปัจจุบันของระบบ รายงานจะรันในวันที่จัดกำหนดการถัดไป
7.  ในพื้นที่ รูปแบบการเกิดซ้ำ ระบุความถี่ที่รันรายงาน โดยค่าเริ่มต้น ประจำวัน ถูกเลือกด้วยช่วงเวลา (วัน) ค่า 1 ตัวเลือกอื่น ๆ ได้แก่ รายสัปดาห์ รายเดือน และรายปี
8.  ในพื้นที่ ช่วงของการเกิดซ้ำ เลือก รายงานควรหยุดขณะจะถูกสร้าง
    -   ไม่มีวันสิ้นสุด– รันกำหนดการรายงานโดยไม่จำกัดเวลา
    -   กำหนดจำนวนของการเกิดเหตุการณ์– กำหนดการรายงานเรียกใช้สำหรับจำนวนครั้งที่ระบุ และจะปิดใช้งาน
    -   สิ้นสุดเมื่อ– กำหนดการรายงานสิ้นสุดในวันที่ระบุ

9.  คลิก บันทึก ในแถบเครื่องมือ ในโต้ตอบกล่อง บันทึกเป็น ป้อนเฉพาะชื่อและคำอธิบายสำหรับกำหนดการรายงานนั้น

เมื่อต้องการคัดลอกกำหนดการรายงาน คุณต้องมีบทบาทเป็นผู้ออกแบบหรือผู้ดูแลระบบ แม้ว่าผู้ดูแลระบบได้ปรับเปลี่ยนกำหนดการรายงาน รายงานจะยังรักษาข้อมูลประจำตัวของผู้ใช้ที่สร้างรายงานนั้น
### <a name="copy-a-report-schedule"></a>คัดลอกกำหนดการรายงาน

1.  ในโปรแกรมออกแบบรายงาน คลิก กำหนดการรายงาน ในบาน หน้าต่างนำทาง และเปิดรายงานการจัดกำหนดการเพื่อคัดลอก
2.  ในเมนู ไฟล์ คลิก บันทึกเป็นแล้ว ป้อนชื่อใหม่กับคำอธิบายสำหรับกำหนดการในกล่องโต้ตอบ บันทึกเป็น คลิก ตกลงและกำหนดการใหม่จะแสดงขึ้นในบานหน้าต่างนำทาง
3.  ในการจัดกำหนดการใหม่ ปรับเปลี่ยนฟิลด์และข้อมูลที่จำเป็น และคลิก บันทึก บนแถบเครื่องมือ หรือคลิก บันทึก ในเมนู ไฟล์

เมื่อต้องการลบกำหนดการรายงาน คุณต้องเป็นเจ้าของกำหนดการรายงานนั้น หรือมีบทบาทเป็นผู้ดูแลระบบ
### <a name="delete-a-report-schedule"></a>ลบกำหนดการรายงาน

1.  ใน Report Designer คลิก กำหนดการรายงานต่างๆ ในบานหน้าต่างนำทาง
2.  เลือกกำหนดการรายงานเพื่อลบ แล้วคลิก ลบ หรือกดคีย์ ลบ
3.  ในกล่องโต้ตอบการตรวจสอบการลบ คลิก ใช่ เพื่อลบกำหนดการรายงานอย่างถาวร ถ้าคุณไม่มีสิทธิ์ในการลบกำหนดการนั้น ข้อความจะแสดงขึ้น และรายงานนั้นจะลบไม่ได้

### <a name="credentials-and-report-schedules"></a>ข้อมูลประจำตัวและกำหนดการรายงาน

ถ้าคุณไม่ได้ป้อนข้อมูลประจำตัวที่จำเป็นสำหรับบริษัททั้งหมดที่รวมอยู่ในรายงาน คุณได้รับข้อความต่อไปนี้เมื่อคุณบันทึกกำหนดการรายงาน: "คุณต้องป้อนข้อมูลประจำตัวของคุณสำหรับบริษัทที่มีอยู่ในกำหนดการรายงานนี้ เลือกปุ่ม สิทธิ์ เพื่อป้อนข้อมูลประจำตัวของคุณ

ตัวอย่างเช่น สาธินีล็อกออนยังบริษัทโดยใช้การเข้าสู่ระบบและรหัสผ่านของตนเอง เธอสร้างกำหนดการสำหรับการรายงานที่ใช้คำนิยามแผนภูมิการรายงานเพื่อรวบรวมข้อมูลจากหลายบริษัท เมื่อกำหนดการรายงานนี้ถูกบันทึก สาธินีถูกเตือนให้ป้อนข้อมูลประจำตัวสำหรับบริษัทอื่น ๆ ที่ระบุไว้ในคำนิยามของแผนภูมิที่รายงาน เมื่อข้อมูลประจำตัวของคุณหมดอายุ รายงานที่ได้รับผลกระทบในกำหนดการรายงานจะไม่ถูกสร้างจนกว่าจะได้รับการปรับปรุงข้อมูลประจำตัว ข้อความจะแสดงขึ้นในคิวรายงานเพื่อชี้ว่า ต้องอัพเดตสิทธิ กำหนดการรายงานจะล้มเหลวถ้ามีสถานการณ์สมมติต่อไปนี้เกิดขึ้น (เนื่องจากใช้ข้อมูลส่วนบุคคล):
-   มีการเพิ่มบริษัทใหม่ไปยังรายงานแผนภูมิสำหรับแต่ละรายงาน
-   รายงานหนึ่งที่อยู่ในกลุ่มการรายงานได้ถูกปรับเปลี่ยนแล้ว
-   รายงานใหม่ที่มาจากการเพิ่มรายชื่อบริษัทได้เพิ่มไปยังกลุ่มรายงานแล้ว

เมื่อต้องการดำเนินต่อ คลิกปุ่ม สิทธิ์ ในกล่องโต้ตอบ การจัดกำหนดรายงาน และจากนั้น ป้อนข้อมูลประจำตัวที่เหมาะสม

## <a name="missing-account-analysis-feature"></a> ลักษณะการวิเคราะห์บัญชีที่สูญหาย
คุณสามารถค้นหาบัญชีและมิติทางการเงินต่างๆที่อาจหายไปในคำนิยามแถว คำนิยามแผนภูมิการรายงาน และข้อกำหนดของรายงานในบล็อคส่วนประกอบทั้งหมด ซึ่งมีประโยชน์เมื่อคุณสร้าง หรืออัพเดตบัญชีหรือบล็อกส่วนประกอบต่าง ๆ ในระหว่างรอบระยะเวลาอันสั้น และคุณต้องการตรวจสอบว่า ข้อมูลใหม่ทั้งหมดจะรวมอยู่ในรายงานของคุณ

บัญชีที่ขาดหายไปถูกกำหนดโดยใช้ค่าต่ำสุด และสูงสุดจากคำนิยามแถว หรือคำนิยามแผนภูมิการรายงาน และจากนั้น แสดงรายการของบัญชีที่ไม่ได้อยู่ในคำนิยามแถวหรือคำนิยามแผนภูมิรายงาน แต่อยู่ในข้อมูลทางการเงิน ถ้าบัญชีที่สูญหายมีค่ามากกว่า หรือน้อยกว่าค่าในคำนิยามแถว บัญชีนั้นจะไม่รวมอยู่ในรายการบัญชีที่ขาดหายไป
| ![คำแนะนำ](https://i-technet.sec.s-msft.com/areas/global/content/clear.gif "คำแนะนำ")**คำแนะนำ**                                             |
|----------------------------------------------------------------------------------------------------------------------------------|
| สำหรับวัตถุประสงค์ในการตรวจสอบ กระบวนการนี้ควรดำเนินการก่อนที่คุณสร้างรายงานประจำเดือน และ เมื่อคุณสร้างบล็อกส่วนประกอบใหม่ |

รายงานที่มีช่วงของค่าน้อยจะไม่พบในบัญชีสูญหาย เมื่อเป็นไปได้ ใช้ช่วงในบล็อกส่วนประกอบซึ่งจะถูกรวมเมื่อสร้างบัญชีใหม่ ถ้ามีการตั้งค่าข้อกำหนดของรายงานใด ๆ เป็นบริษัท @ANY จากนั้นคุณสามารถเข้าสู่ระบบไปยังบริษัทเฉพาะ และดำเนินการวิเคราะห์บัญชีที่ขาดหายไปสำหรับบริษัทนั้น
| ![หมายเหตุ](https://i-technet.sec.s-msft.com/areas/global/content/clear.gif "หมายเหตุ")**หมายเหตุ**                                                                                           |
|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ถ้ามีการเพิ่มบริษัทใหม่ คุณต้องเพิ่มบริษัทใหม่เป็นแผนภูมิรายงานในรายงานใด ๆ ที่มีอยู่ หรือบริษัทจะไม่ถูกรวมในการวิเคราะห์บัญชีที่ขาดหายไป |

### <a name="run-missing-account-analysis"></a>ดำเนินการวิเคราะห์บัญชีที่สูญหาย

1.  ใน Report Designer คลิก เครื่องมือ แล้ว คลิก วิเคราะห์บัญชีที่สูญหาย
2.  ในฟิลด์ ตัวกรองบริษัท เลือกบริษัทที่จะกรองผลลัพธ์ หรือเลือก ทั้งหมด (ไม่มีตัวกรอง) เพื่อแสดงผลลัพธ์จากบริษัทที่พร้อมใช้งานทั้งหมด
3.  ในฟิลด์ ตัวกรองมิติ เลือกมิติเพื่อกรองข้อมูลผลลัพธ์ หรือเลือก ทั้งหมด (ไม่มีตัวกรอง) เพื่อดูรายละเอียดข้อมูลมิติทั้งหมดที่มีอยู่สำหรับมิติที่พร้อมใช้งานทั้งหมด
4.  ใน กลุ่มที่จัดตาม ฟิลด์ เลือกตัวเลือกสำหรับการเรียงลำดับผลลัพธ์ คุณสามารถเรียงลำดับผลลัพธ์ตามบล็อคส่วนประกอบที่ได้รับผลกระทบ หรือคุณสามารถเรียงลำดับผลลัพธ์ตามชุดมิติและมูลค่า
5.  ตรวจทานผลลัพธ์ที่แสดง เมื่อคุณเลือกสินค้าในบานหน้าต่างด้านบน บานหน้าต่างด้านล่างจะแสดงข้อมูลเพิ่มเติมเกี่ยวกับข้อยกเว้น ซึ่งรวมถึงมิติ ค่า และการรายงานต่างๆที่เกี่ยวข้อง
6.  เพื่อเปิดรายการที่ได้รับผลกระทบ คลิกไอคอนการเชื่อมโยงที่จะแสดงในบานหน้าต่างรายการ หรือคลิกขวาที่รายการ และเลือก เปิด เพื่อเลือกหลายรายการ กด Ctrl ค้างไว้ในขณะที่คุณเลือกสินค้าในบานหน้าต่างด้านล่าง
7.  หากมีค่า บล็อกส่วนประกอบ หรือสินค้าใดถูกส่งกลับโดยที่ไม่ควรรวมไว้ในการวิเคราะห์ คลิกขวารายการนั้น แล้วเลือก ไม่รวมหรือเลือก กล่องกาเครื่องหมายไม่รวม ที่อยู่ถัดจากรายการเพื่อลบสินค้าออกจากรายการ สินค้าที่ไม่รวมจะไม่รวมเมื่อรายการถูกรีเฟรช เพื่อเลือกหลายรายการ กด Ctrl ค้างไว้ในขณะที่คุณเลือกสินค้าในบานหน้าต่างด้านล่าง การดูสินค้าทั้งหมด รวมผลลัพธ์ใด ๆ ที่คุณเลือกไว้ก่อนหน้านี้จะไม่รวมการวิเคราะห์ เลือกการ แสดงแยกบล็อกส่วนประกอบและกล่องกาเครื่องหมายมูลค่า และจากนั้น คลิก รีเฟรช
8.  คลิก รีเฟรช เพื่อรีเฟรชข้อยกเว้นที่คุณระบุไว้ คลิก ใช่ เพื่อทำการรีเฟรชแบบเต็มของผลลัพธ์ทั้งหมด หรือคลิก ไม่ใช่ เพื่อทำการรีเฟรชจ่าหน้าสินค้าบางส่วน
    | ![หมายเหตุ](https://i-technet.sec.s-msft.com/areas/global/content/clear.gif "หมายเหตุ")**หมายเหตุ**                    |
    |------------------------------------------------------------------------------------------------------------|
    | แบบฟอร์มจะรีเฟรชอัตโนมัติเมื่อถูกเปิด นอกเสียจากแบบฟอร์มถูกเปิดใน 15 นาทีสุดท้าย |

9.  เมื่อปัญหาได้รับการแก้ไข คลิก ตกลง เพื่อปิดกล่องโต้ตอบ

## <a name="keyboard-shortcuts-for-missing-account-analysis"></a>แป้นพิมพ์ลัดสำหรับการวิเคราะห์บัญชีที่สูญหาย
