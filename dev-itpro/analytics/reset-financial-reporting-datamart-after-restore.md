# <a name="reset-the-financial-reporting-data-mart-after-restoring-a-database"></a>รีเซ็ตต้ารายงานทางการเงินหลังจากการคืนค่าฐานข้อมูล

หัวข้อนี้อธิบายวิธีการรีเซ็ตต้ารายงานทางการเงินหลังจากการคืนค่าเป็น 365 Dynamics Microsoft สำหรับฐานข้อมูลการดำเนินงาน 

มีหลายสถานการณ์ที่คุณอาจจำเป็นต้อง 365 Dynamics ของคุณสำหรับฐานข้อมูลการดำเนินการคืนค่าจากสำเนาสำรอง หรือคัดลอกฐานข้อมูลจากสภาพแวดล้อมอื่น เมื่อเหตุการณ์นี้เกิดขึ้น คุณต้องทำตามขั้นตอนต่าง ๆ ที่เหมาะสมเพื่อให้แน่ใจว่า ต้ารายงานทางการเงินได้อย่างถูกต้องใช้ 365 Dynamics การกู้คืนข้อมูลสำหรับฐานข้อมูลการดำเนินงานด้วย ถ้าคุณมีคำถามเกี่ยวกับการตั้งค่าต้ารายงานทางการเงินสำหรับเหตุผลภายนอกคืน 365 แบบ Dynamics สำหรับฐานข้อมูลการดำเนินงาน การอ้างอิงไปยัง[ต้าโปรแกรมรายงานการจัดการการตั้งค่าใหม่](https://blogs.msdn.microsoft.com/dynamics_financial_reporting/2016/06/28/resetting-the-management-reporter-data-mart/)สำหรับข้อมูลเพิ่มเติม โปรดสังเกตว่า ขั้นตอนในกระบวนการนี้ได้รับการสนับสนุนสำหรับ Dynamics 365 สำหรับการดำเนินการย่อย 2016 พฤษภาคม (App build 7.0.1265.23014 และ 7.0.10000.4 การสร้างรายงานทางการเงิน) และรุ่นที่ใหม่กว่า ถ้าคุณมีรุ่น 365 Dynamics สำหรับการดำเนินงานก่อนหน้า โปรดติดต่อทีมสนับสนุนของเราสำหรับความช่วยเหลือ

## <a name="export-report-definitions"></a>ส่งออกนิยามรายงาน
ก่อนอื่น ส่งออกรูปแบบรายงานที่อยู่ในตัวออกแบบรายงาน ทำตามขั้นตอนต่อไปนี้:

1.  ในตัวออกแบบรายงาน ไป**บริษัท**&gt;**กลุ่มแบบเอกสารสำเร็จรูป**
2.  เลือกกลุ่มแบบเอกสารสำเร็จรูปเพื่อส่งออก และคลิ**ส่งออก** **หมายเหตุ:**สำหรับ 365 Dynamics สำหรับการดำเนินงาน กลุ่มแบบเอกสารสำเร็จรูปเดียวเท่านั้นได้รับการสนับสนุน**เริ่มต้น**
3.  เลือกคำนิยามรายงานการส่งออก:
    -   เมื่อต้องการส่งออกข้อกำหนดของรายงานและบล็อคส่วนประกอบที่เกี่ยวข้องทั้งหมดของคุณ คลิก **เลือกทั้งหมด**
    -   เมื่อต้องการส่งออกรายงาน แถว คอลัมน์ แผนภูมิ หรือเซ็ตมิติที่เฉพาะ คลิกแท็บที่เหมาะสม และจากนั้นเลือกรายการที่จะส่งออก กดค้างปุ่ม Ctrl เพื่อเลือกหลายรายการในแท็บ เมื่อคุณเลือก รายงานการส่งออกแถวที่เกี่ยวข้อง คอลัมน์ ทรี และเซ็ตมิติจะถูกเลือก

4.  คลิ**ส่งออก**
5.  ใส่ชื่อแฟ้ม และตำแหน่งที่ปลอดภัยที่คุณต้องการบันทึกข้อกำหนดของรายงานที่ส่งออกที่เลือก
6.  Click **Save**.

แฟ้มสามารถคัดลอก หรืออัปโหลดไปยังตำแหน่งที่ปลอดภัย ยินยอมที่จะนำเข้าลงในสภาพแวดล้อมที่แตกต่างกันในเวลาอื่น ข้อมูลเกี่ยวกับการใช้บัญชีผู้ใช้จัดเก็บ Azure ของ Microsoft สามารถพบได้ใน[การถ่ายโอนข้อมูล โดยใช้ยูทิลิตี้การบรรทัดคำสั่ง AzCopy](https://docs.microsoft.com/en-gb/azure/storage/storage-use-azcopy)ได้ **หมายเหตุ:** Microsoft ไม่ให้บัญชีจัดเก็บเป็นส่วนหนึ่งของ 365 Dynamics ของคุณสำหรับข้อตกลงการดำเนินงาน คุณต้องเก็บข้อมูลบัญชีการซื้อ หรือใช้บัญชีเก็บข้อมูลจากการสมัครใช้งาน Azure ที่แยกต่างหาก **สิ่งสำคัญ:**ระวังของลักษณะการทำงานของไดรฟ์ D บนเครื่องเสมือน Azure ไม่ได้เก็บกลุ่มส่งออกแบบเอกสารสำเร็จรูปของคุณที่นี่อย่างถาวร สำหรับข้อมูลเพิ่มเติมเกี่ยวกับไดรฟ์ชั่วคราว ดู[การทำความเข้าใจเกี่ยวกับไดรฟ์ชั่วคราวบนเครื่องเสมือน Windows Azure](https://blogs.msdn.microsoft.com/mast/2013/12/06/understanding-the-temporary-drive-on-windows-azure-virtual-machines/)

## <a name="stop-services"></a>หยุดบริการ
ใช้เดสก์ท็อประยะไกลเพื่อเชื่อมต่อกับคอมพิวเตอร์ทั้งหมดในสภาพแวดล้อม และหยุดบริการ Windows ต่อไปนี้ โดยใช้ services.msc:

-   เวิลด์ไวด์เว็บเผยแพร่บริการ (บนคอมพิวเตอร์ AOS ทั้งหมด)
-   365 Dynamics Microsoft สำหรับบริการจัดการชุดของการดำเนินงาน (คอมพิวเตอร์ที่ไม่ใช่ส่วนตัว AOS เท่านั้น)
-   บริการกระบวนการจัดการโปรแกรมรายงาน 2012 (บน BI คอมพิวเตอร์เท่านั้น)

บริการเหล่านี้จะมีการเปิดการเชื่อมต่อกับ 365 Dynamics สำหรับฐานข้อมูลการดำเนินงาน

## <a name="reset"></a>รีเซ็ต
#### <a name="locate-the-latest-dataupgradezip-package"></a>ค้นหาแพคเกจ DataUpgrade.zip ล่าสุด

ค้นหาโดยใช้คำแนะนำที่พบในแพคเกจ DataUpgrade.zip ล่าสุด[ดาวน์โหลดสคริปต์ DataUpgrade.zip](..\migration-upgrade\upgrade-data-to-latest-update.md) คำแนะนำที่อธิบายวิธีการค้นหารุ่นถูกต้องของแพคเกจการอัพเกรดข้อมูลสำหรับสภาพแวดล้อมของคุณ

#### <a name="execute-scripts-against-dynamics-365-for-operations-database"></a>เรียกใช้สคริปต์กับ Dynamics 365 สำหรับฐานข้อมูลการดำเนินงาน

เรียกใช้สคริปต์ต่อไปนี้กับ 365 Dynamics สำหรับฐานข้อมูลการดำเนินการ (ไม่เทียบกับฐานข้อมูลเพื่อการรายงานทางการเงิน)

-   DataUpgrade.zip\\AosService\\สคริปต์\\ConfigureAxReportingIntegration.sql
-   DataUpgrade.zip\\AosService\\สคริปต์\\GrantAzViewChangeTracking.sql

สคริปต์เหล่านี้ให้แน่ใจว่า ผู้ใช้ บทบาท และตั้งค่าการติดตามการเปลี่ยนแปลงถูกต้อง

#### <a name="execute-powershell-command-to-reset-database"></a>ดำเนินการคำสั่ง PowerShell เพื่อตั้งค่าฐานข้อมูล

ดำเนินการคำสั่งต่อไปนี้ โดยตรงบนคอมพิวเตอร์ AOS การตั้งค่าการรวมระหว่าง 365 Dynamics สำหรับการดำเนินงานและการรายงานทางการเงิน:

1.  เปิด Windows PowerShell เป็นผู้ดูแล
2.  ดำเนินการ: f:
3.  ดำเนินการ: ซีดี f:\\MRApplicationService\\MRInstallDirectory
4.  ดำเนินการ: นำเข้าโมดูล \\เซิร์ฟเวอร์\\MRDeploy\\MRDeploy.psd1
5.  ดำเนินการ: รีเซ็ต-DatamartIntegration-ReasonDetail - เหตุผลอื่น "&lt;เหตุผลของฉันสำหรับการตั้งค่าใหม่&gt;"
    -   คุณจะถูกขอให้ใส่ "Y" เพื่อยืนยัน

คำอธิบายของพารามิเตอร์:

-   ค่าถูกต้องสำหรับ - เหตุผลคือ: การบริการ BADDATA, OTHER
-   พารามิเตอร์ - ReasonDetail มีข้อความอิสระ
-   เหตุผลและ reasonDetail จะถูกบันทึกในการตรวจสอบ telemetry/สภาพ แวดล้อม

## <a name="start-services"></a>เริ่มต้นบริการ
ใช้ services.msc เพื่อเริ่มการทำงานของบริการที่คุณหยุดการทำงานก่อนหน้านี้:

-   เวิลด์ไวด์เว็บเผยแพร่บริการ (บนคอมพิวเตอร์ AOS ทั้งหมด)
-   365 Dynamics Microsoft สำหรับบริการจัดการชุดของการดำเนินงาน (คอมพิวเตอร์ที่ไม่ใช่ส่วนตัว AOS เท่านั้น)
-   บริการกระบวนการจัดการโปรแกรมรายงาน 2012 (บน BI คอมพิวเตอร์เท่านั้น)

## <a name="import-report-definitions"></a>นำเข้าข้อกำหนดของรายงาน
การนำเข้าของคุณออกแบบรายงานจาก Report Designer โดยใช้ไฟล์สร้างขึ้นในระหว่างการส่งออก:

1.  ในตัวออกแบบรายงาน ไป**บริษัท**&gt;**กลุ่มแบบเอกสารสำเร็จรูป**
2.  เลือกกลุ่มแบบเอกสารสำเร็จรูปเพื่อส่งออก และคลิ**ส่งออก** **หมายเหตุ:**สำหรับ 365 Dynamics สำหรับการดำเนินงาน กลุ่มแบบเอกสารสำเร็จรูปเดียวเท่านั้นได้รับการสนับสนุน**เริ่มต้น**
3.  เลือก**เริ่มต้น**สร้างบล็อกและคลิ**นำเข้า**
4.  เลือกแฟ้มประกอบด้วยข้อกำหนดของรายงานที่ส่งออก แล้วคลิก**เปิด**
5.  ในกล่องโต้ตอบ นำเข้า เลือกข้อกำหนดของรายงานที่จะนำเข้า:
    -   เมื่อต้องการนำเข้าข้อกำหนดของรายงานและบล็อคส่วนประกอบสนับสนุนทั้งหมด คลิก **เลือกทั้งหมด**
    -   การนำเข้ารายงาน แถว คอลัมน์ แผนภูมิ หรือ เซ็ตมิติที่เฉพาะ ให้เลือกรายงาน แถว คอลัมน์ แผนภูมิ หรือเซ็ตมิติที่จะนำเข้า

6.  Click **Import**.
