# <a name="help-overview"></a>ภาพรวมของวิธีใช้

[!include[banner](../includes/banner.md)]


บทความนี้แสดงภาพรวมของคอมโพเนนต์ของระบบวิธีใช้ Microsoft Dynamics 365 for Operations นอกจากนี้ยังอธิบายวิธีระบุเอกสารแบบกำหนดเองและการฝึกอบรมองค์กรของคุณ 

Dynamics 365 for Operations ประกอบด้วยระบบวิธีใช้ที่ขึ้นอยู่กับสองส่วนประกอบหลัก:

-   ไซต์เอกสารประกอบ
-   คู่มืองาน

คุณสามารถเข้าถึงทั้งบทความและคู่มืองานงานจากบานหน้าต่าง วิธีใช้ใน Dynamics 365 for Operations ดังที่แสดงในภาพหน้าจอต่อไปนี้ [![บานหน้าต่างวิธีใช้](./media/help-pane-ops-task-guides-1024x741.png)](./media/help-pane-ops-task-guides.png) บทความนี้จะอธิบายระบบวิธีใช้ และอธิบายวิธีที่คุณสามารถสร้างเอกสารที่กำหนดเองและการฝึกอบรมสำหรับองค์กรของคุณ

## <a name="help-on-docsmicrosoftcom"></a>วิธีใช้เกี่ยวกับ docs.microsoft.com
ไซต์ docs.microsoft.com ([docs.microsoft.com/dynamics365/operations](/dynamics365/#pivot=solutions&panel=solutions_operations) เป็นแหล่งข้อมูลหลักของเอกสารประกอบผลิตภัณฑ์สำหรับ Dynamics 365 for Operations ไซต์มีลักษณะการทำงานดังต่อไปนี้:

-   **เข้าถึงเนื้อหาล่าสุด** – ไซต์ทำให้เราสร้าง จัดส่ง และอัพเดตเอกสารผลิตภัณฑ์รวดเร็วและมีความยืดหยุ่นมากขึ้น ดังนั้น จึงจะช่วยรับประกันว่า คุณมีสิทธิ์เข้าถึงข้อมูลทางเทคนิคล่าสุด
-   **เนื้อหาที่เขียนโดยผู้เชี่ยวชาญ** – ไซต์แสดงชุดของเอกสารผลิตภัณฑ์ที่สมบูรณ์กว่า ซึ่งสามารถเพิ่มเติมโดยสมาชิกของชุมชนทั้งภายใน และภาย นอก Microsoft
-   **การเข้าถึงเนื้อหาชนิดต่างๆ** – ไซต์ช่วยให้คุณได้เข้าถึงเนื้อหาชนิดต่างๆ เกี่ยวกับ Dynamics 365 for Operations เช่น Microsoft Office ได้อย่างรวดเร็ว ตัวอย่างเช่น รวมงานนำเสนอของ Microsoft Office คู่มืองาน วิดีโอ และหัวข้อ
-   **เนื้อหาที่สนับสนุนกระบวนการทางธุรกิจของคุณ** – ไซต์ครอบคลุมถึงกระบวนการทางธุรกิจ – เน้นเนื้อหาที่โฟกัส ซึ่งใช้ประโยชน์จากแบบจำลองกระบวนการทางธุรกิจ (BPM) ใน Microsoft Dynamics Lifecycle Services (LCS)

เราได้ย้ายเนื้อหาทั้งหมดจากWiki วิธีใช้ก่อนหน้าของเราไปยังเอกสารแล้ว เราตื่นเต้นมากเกี่ยวกับไซต์ใหม่ของเรา และหวังว่าคุณจะรู้สึกเช่นกัน

### <a name="when-can-we-use-it"></a>เมื่อไรที่เราจะสามารถใช้ได้

คุณสามารถอ่านเนื้อหาบนเอกสารได้ทันที-- เป็นข้อมูลสาธารณะทั้งหมด และค้นหาได้โดยไม่ต้องมีการลงชื่อ คุณสามารถใช้โปรแกรมค้นหาใดก็ตามที่คุณชอบเพื่อค้นหาเนื้อหา คุณสามารถแสดงข้อคิดเห็นเกี่ยวกับบทความบนไซต์ได้ถ้าคุณเลือกโดยการลงชื่อเข้าใช้


## <a name="task-guides"></a>คู่มืองาน
คู่มืองานคือคู่มือในการควบคุม ประสบการณ์การโต้ตอบ ที่นำคุณผ่านขั้นตอนของงานหรือกระบวนการทางธุรกิจ คุณสามารถเปิด (เล่น) คู่มืองานจากบานหน้าต่าง วิธีใช้ ได้ เมื่อคุณคลิกคู่มืองานเป็นครั้งแรก บานหน้าต่าง วิธีใช้ จะแสดงคำแนะนำทีละขั้นตอนสำหรับงานนั้น คู่มืองานที่เป็นภาษาท้องถิ่นพร้อมใช้งานแล้วในขณะนี้ [![มุมมองการอ่านคู่มืองาน](./media/task-guide-ops-1024x742.png)](./media/task-guide-ops.png) เพื่อเริ่มการแนะนำ ประสบการณ์โต้ตอบ คลิก **เริ่มต้นคู่มืองาน** ที่ด้านล่างของบานหน้าต่างวิธีใช้ ตัวชี้สีดำเปิดขึ้น และบ่งชี้ถึงการดำเนินการที่คุณจำเป็นต้องทำ ทำตามคำแนะนำที่ปรากฏใน UI และป้อนข้อมูลตามที่กำหนด [![คำแนะนำตามขั้นตอนของคู่มืองาน](./media/task-guide-step-1-ops.png)](./media/task-guide-step-1-ops.png) **สำคัญ:** ข้อมูลที่คุณป้อนเมื่อคุณเล่นคำแนะนำในการปฏิบัติงานต้องเป็นความจริง ถ้าคุณอยู่ในสภาพแวดล้อมการผลิต ข้อมูลจะถูกป้อนในบริษัทที่คุณกำลังใช้อยู่

### <a name="it-all-begins-with-task-recorder"></a>ทั้งหมดเริ่มต้นด้วยตัวบันทึกงาน

คำแนะนำของงานจะสร้างขึ้นโดยใช้ตัวบันทึกงาน เมื่อคุณใช้ตัวบันทึกงาน การดำเนินการทั้งหมดของคุณใน Dynamics 365 for Operations UI (เช่น การคลิกที่เมนู การเปลี่ยนแปลงการตั้งค่า และการป้อนข้อมูล) จะถูกบันทึกไว้ ขั้นตอนที่คุณบันทึกจะถูกเรียกรวมกันว่า บันทึกงาน ดังที่เราได้อธิบายไว้ในหัวข้อก่อนหน้า บันทึกงานที่สามารถแสดงในบานหน้าต่าง วิธีใช้ และเล่นเป็นคู่มืองาน อย่างไรก็ตาม มีวิธีอื่น ๆ ที่คุณสามารถใช้การบันทึกงานได้:

-   **บันทึกการบันทึกงานไปยัง BPM** – คุณสามารถบันทึกการบันทึกงานของลำดับชั้นในไลบรารี BPM ใน LCS ได้ เมื่อคุณบันทึกการบันทึกงานไปยัง BPM ไดอะแกรมแผนผังลำดับงานจะถูกสร้างและแสดงร่วมกับขั้นตอนของการบันทึก **หมายเหตุ:** เพื่อแสดงการบันทึกงานในบานหน้าต่าง วิธีใช้ ของ Dynamics 365 for Operations และเล่นเป็นคู่มืองาน คุณต้องบันทึกการบันทึกไปยังไลบรารี BPM
-   **บันทึกเป็นเอกสาร Word** – โดยการบันทึกการบันทึกงานเป็นเอกสาร Microsoft Word คุณสามารถจัดทำคำแนะนำในการฝึกอบรมที่สามารถพิมพ์ได้สำหรับองค์กรของคุณได้อย่างง่ายดาย

ดูข้อมูลเพิ่มเติมเกี่ยวกับตัวบันทึกงานที่ [ตัวบันทึกงานใน Dynamics 365 for Operations](../user-interface/task-recorder.md)

### <a name="creating-customized-task-recordings"></a>สร้างบันทึกภารกิจที่กำหนดเอง

คุณสามารถสร้างบันทึกงานของคุณเอง หรือคุณสามารถดาวน์โหลด และบันทึกงานที่ Microsoft จัดเตรียมให้ ดังนั้น คุณสามารถสร้างวิธีใช้ที่กำหนดเองสำหรับองค์กรของคุณ ซึ้งจะแสดงใช้งาน Dynamics 365 for Operations แบบเฉพาะ เพื่อแสดงการบันทึกงานในบานหน้าต่าง วิธีใช้ ของ Dynamics 365 for Operations และเล่นเป็นคู่มืองาน คุณต้องบันทึกการบันทึกไปยังไลบรารี BPM ใน LCS ถ้าคุณเป็นคู่ค้า และคุณเลื่อนลำดับเป็นไลบรารีของบริษัท และรวมอยู่ในโซลูชัน ไลบรารีนั้นจะพร้อมใช้งานสำหรับลูกค้าของคุณ สำหรับคำแนะนำโดยละเอียด ดู [การใช้การบันทึกงานเพื่อสร้างเอกสารหรือการฝึกอบรม](../user-interface/task-recorder.md)

## <a name="in-product-help"></a>วิธีใช้ภายในผลิตภัณฑ์
ในการเข้าถึงเนื้อหาวิธีใช้ภายใน Dynamics 365 for Operations คลิกไอคอน **วิธีใช้** (**?**) และจากนั้นเลือก วิธีใช้ หรือกด Ctrl+Shift+? อย่างใดอย่างหนึ่งก็ได้ ในทั้งสองกรณี บานหน้าต่าง วิธีใช้ จะเปิดขึ้น จากบานหน้าต่าง วิธีใช้ คุณสามารถเข้าถึงบทความหรือคู่มืองานได้ [![บานหน้าต่างวิธีใช้](./media/help-pane-wiki-1024x684.png)](./media/help-pane-wiki.png)

### <a name="accessing-articles-from-the-help-pane"></a>เข้าถึงบทความจากบานหน้าต่างวิธีใช้

จากบานหน้าต่าง วิธีใช้ คุณสามารถเข้าถึงบทความที่ใช้ไปยังไคลเอนต์ Dynamics 365 for Operations ได้ เมื่อคุณเปิดบานหน้าต่างวิธีใช้ และคลิกแท็บ **Wiki** คุณจะเห็นบทความที่ใช้กับเพจที่คุณกำลังอยู่ใน Dynamics 365 for Operations ถ้าไม่พบบทความ คุณสามารถป้อนคำสำคัญเพื่อจำกัดการค้นหา เมื่อคุณคลิกบทความในบานหน้าต่าง วิธีใช้ เบราเซอร์ของคุณแท็บจะเปิดแท็บใหม่ และแสดงบทความนั้น 

### <a name="accessing-task-guides-from-the-help-pane"></a>เข้าถึงคู่มืองานจากบานหน้าต่างวิธีใช้

ก่อนที่คุณจะสามารถเข้าถึงคู่มืองานจากบานหน้าต่าง **วิธีใช้** ผู้ดูแลระบบต้องไปยังหน้า พารามิเตอร์ระบบ ใน Dynamics 365 for Operations และตั้งค่าคอนฟิกบางอย่าง **หมายเหตุ:**

-   เพื่อกำหนดค่าวิธีใช้ คุณต้องเข้าสู่ระบบด้วยบัญชีผู้ใช้ในผู้เช่าเดียวกันกับผู้เช่าที่ใช้ใน Dynamics 365 for Operations
-   จะไม่สามารถเชื่อมต่อไปยังไลบรารี LCS จากอินสแตนซ์ของ Dynamics 365 for Operations ที่ติดตั้งในฮาร์ดไดรฟ์เสมือนเฉพาะที่ (VHD)

[![แบบฟอร์มพารามิเตอร์ระบบพร้อมการตั้งค่าวิธีใช้](./media/system-parameters_ops-1024x437.png)](./media/system-parameters_ops.png) บนหน้า **พารามิเตอร์ระบบ** ให้ปฏิบัติตามขั้นตอนต่อไปนี้:

1.  **สิ่งสำคัญ:**ในครั้งแรกที่คุณเปิดแท็บ วิธีใช้ คุณต้องเชื่อมต่อกับ Lifecycle Services ตรวจสอบให้แน่ใจว่าคลิกลิงค์ในกึ่งกลางของแบบฟอร์ม รอสำหรับการเชื่อมต่อ ปิดกล่องโต้ตอบ และคลิก ตกลง เพื่อเข้าถึงแบบฟอร์มพารามิเตอร์[![เชื่อมต่อกับ LCS](./media/connect-to-lcs-crop-1024x365.png)](./media/connect-to-lcs-crop.png)
2.  เลือกโครงการ Lifecycle Services เพื่อเชื่อมต่อ
3.  เลือกไลบรารี BPM (ภายในโครงการที่เลือก) เพื่อดึงข้อมูลการบันทึกงาน
4.  ตั้งค่าลำดับการแสดงผลของไลบรารี BPM ซึ่งจะกำหนดลำดับว่าบันทึกงานใดจะปรากฏขึ้นในบานหน้าต่าง วิธีใช้

หลังจากผู้ดูแลระบบเสร็จสิ้นขั้นตอนเหล่านี้ คุณสามารถเปิดบานหน้าต่าง วิธีใช้ และคลิกแท็บ **คู่มืองาน** ขณะนี้คุณจะเห็นคู่มืองานที่นำไปใช้กับเพจที่คุณกำลังอยู่ใน Dynamics 365 for Operations ถ้าไม่พบคู่มืองาน คุณสามารถป้อนคำสำคัญเพื่อจำกัดการค้นหา หลังจากที่คุณคลิกคู่มืองานในบานหน้าต่าง วิธีใช้ บานหน้าต่าง วิธีใช้ จะแสดงคำแนะนำทีละขั้นตอน และคุณสามารถเล่นคู่มืองานได้ [![มุมมองการอ่านคู่มืองาน](./media/task-guide-ops-1024x742.png)](./media/task-guide-ops.png)

### <a name="where-are-the-translated-task-guides"></a>คู่มืองานที่แปลแล้วอยู่ที่ใด

คู่มืองานที่แปลแล้วจะถูกนำออกใช้ในไลบรารี "ภาษาทั้งหมด" ในชื่อเรื่อง ใน Dynamics 365 for Operations เมื่อต้องการดูวิธีใช้ของคู่มืองานที่เป็นภาษาท้องถิ่น ให้ตรวจสอบให้แน่ใจว่าคุณได้เชื่อมต่อไปยังไลบรารีที่เหมาะสมแล้ว ภาษาที่ปรากฏในคู่มืองานมีการควบคุมสำหรับผู้ใช้แต่ละรายโดยการตั้งค่าภาษาภายใต้ **ตัวเลือก** &gt; **การกำหนดลักษณะ** 
-   ถ้าคู่มืองานได้รับการแปลแล้ว เมื่อคุณเปิดคู่มืองานนั้น ข้อความทั้งหมดของคู่มืองานจะปรากฏในภาษาที่คุณเลือก
-   ถ้าคู่มืองานยังไม่ได้รับการแปล เมื่อคุณเปิดคู่มืองานนั้น จะปรากฏข้อความเพียงบางส่วน (ข้อความของการควบคุม) ในภาษาที่คุณเลือก

## <a name="additional-resources"></a>ทรัพยากรเพิ่มเติม
