# <a name="single-voucher-and-currency-revaluation-upgrade-for-microsoft-dynamics-365-for-operations-version-1611"></a>ใบสำคัญเดียวและการประเมินค่าใหม่ตามสกุลเงินปรับรุ่นสำหรับ Microsoft Dynamics 365 สำหรับการดำเนินงานรุ่น 1611

บางองค์กรป้อนสมุดรายวันที่ประกอบด้วยใบสำคัญเดียวที่มีลูกค้าหรือผู้ขายมากกว่าหนึ่ง และพวกเขายังเรียกใช้บัญชีลูกหนี้หรือบัญชีกระบวนการประเมินค่าใหม่ตามสกุลเงินต่างประเทศเจ้าหนี้ หัวข้อนี้อธิบายถึงขั้นตอนที่องค์กรเหล่านี้ควรปฏิบัติตามเมื่อจะปรับรุ่นเป็น Microsoft Dynamics 365 สำหรับการดำเนินงานรุ่น 1611

ทำตามขั้นตอนเหล่านี้เมื่อคุณปรับรุ่นเป็น Microsoft Dynamics 365 สำหรับการดำเนินงานรุ่น 1611

1.  ก่อนที่คุณปรับรุ่น 365 Dynamics สำหรับการดำเนินการ เรียกใช้กระบวนการประเมินค่าใหม่ตามสกุลเงินต่างประเทศสำหรับบัญชีลูกหนี้และบัญชีเจ้าหนี้ ตั้งค่าการ**วิธี**เขต**วันในใบแจ้งหนี้** ธุรกรรมการประเมินค่าใหม่ถูกสร้างขึ้นโดยย้อนกลับการประเมินค่าใหม่ตามสกุลเงินต่างประเทศล่าสุด ดังนั้น ธุรกรรมที่เปิดมีการประเมินค่าสกุลเงินทางบัญชีของต้นฉบับ
2.  อัพเกรด Dynamics 365 สำหรับการดำเนินงานรุ่น 1611
3.  เรียกใช้บัญชีลูกหนี้และบัญชีกระบวนการประเมินค่าใหม่ตามสกุลเงินต่างประเทศเจ้าหนี้อีกครั้ง ขณะนี้ ที่ตั้ง**วิธี**เขต**มาตรฐาน** ธุรกรรมการประเมินค่าใหม่ถูกสร้างขึ้นโดยขึ้นอยู่กับอัตราแลกเปลี่ยนปัจจุบัน ธุรกรรมนี้จะบันทึกกำไร/ขาดทุนที่เกิดขึ้นจริงและบัญชีแยกประเภทสรุปที่ถูกต้อง
