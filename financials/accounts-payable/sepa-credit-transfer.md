# <a name="sepa-credit-transfer-overview"></a>ภาพรวมของการโอนย้ายเครดิต SEPA

บทความนี้ให้ข้อมูลทั่วไปเกี่ยวกับ ISO 20022 เครดิตโอนย้าย ซึ่งรวมถึงการโอนย้ายเครดิตเดียวยูโรชำระพื้นที่ (SEPA) และใด ๆ อื่น ๆ ชำระเงินทางอิเล็กทรอนิกส์สำหรับผู้จัดจำหน่าย การโอนย้ายเครดิต SEPA คือ การชำระเงินในสกุลเงินยูโร จากบริษัทหนึ่ง หรือแต่ละบริษัทอีกแห่งหนึ่ง หรือแต่ละชนิดระบุ หัวข้อนี้ยังอธิบายวิธีการตั้งค่า และส่งไฟล์การชำระเงินการโอนย้ายเครดิต

## <a name="what-is-a-credit-transfer-message"></a>ข้อความโอนย้ายเครดิตคืออะไร
ข้อความโอนย้ายเครดิตเป็นการร้องขอที่ส่งฝ่ายที่มี initiating (บริษัทของคุณ) เพื่อย้ายเงินจากบัญชีของตนเองเป็นเจ้าหนี้ มีการปฏิบัติการ เฉพาะประเทศ/ภูมิภาค และธนาคารเฉพาะที่มากของการโอนย้ายเครดิตข้อความ บางแฟ้มจะถูกใช้ภายในประเทศ/ภูมิภาคหนึ่ง และบางส่วนจะกลายเป็นมาตรฐาน มาตรฐานสากลที่สร้างไว้แล้วอย่างดีหนึ่งคือ ISO 20022 และข้อความเริ่มต้นของ เช่นการโอนย้ายเครดิต ภาพประกอบต่อไปนี้แสดงความสัมพันธ์และความครอบคลุมสำหรับข้อความการโอนย้ายเครดิตที่เลือก 
![การถ่ายโอนเครดิต](./media/credit-transfer.jpg)ข้อความที่โอนย้ายเครดิต\[/ป้ายคำอธิบาย\] 

## <a name="what-are-iso-20022-and-sepa-payments"></a>มีอะไรบ้าง ISO 20022 และการชำระเงิน SEPA
Single Euro Payments Area (SEPA) ตั้งค่าตามคณะกรรมาธิการยุโรปล และกำหนดว่าการชำระเงินทางอิเล็กทรอนิกส์ทั้งหมดจะถือว่าเป็นในประเทศ โดยไม่คำนึงถึงประเทศ/ภูมิภาคซึ่งบุคคล ธุรกิจ หรือองค์กร และธนาคารตั้งอยู่ ไม่มีความแตกต่างระหว่างการชำระเงินของประเทศและการชำระเงินข้ามแดนอยู่ SEPA รวมสมาชิก 28 รัฐของสหภาพยุโรป (EU), และยังไอซ์แลนด์ ลิกเตนสไตน์ นอร์เวย์ สวิตเซอร์แลนด์ โมนาโก และซานมารีโน SEPA ช่วยจัดรูปแบบตลาดเดียวสำหรับธุรกรรมการชำระเงินภายในเขตเศรษฐกิจยุโรป (EEA) ท้ายที่สุด SEPA คาดว่าจะลดจำนวนของรูปแบบการชำระเงินที่ธนาคาร ธุรกิจ และบุคคลต้องใช้ คณะกรรมาธิการยุโรปกำหนดพื้นฐานทางกฎหมายสำหรับการชำระเงิน SEPA โดยใช้คำสั่งบริการชำระเงิน (PSD) สภาการชำระเงินของยุโรป (EPC) ให้การสนับสนุน SEPA ผ่านกิจกรรมต่อไปนี้:

-   ตั้งค่ามาตรฐานสำหรับการชำระเงินอิเล็กทรอนิกส์ SEPA โดยใช้ ISO 20022 Universal รูปแบบ XML โครงร่างข้อความอุตสาหกรรมเงิน
-   สร้างกฎและคำแนะนำเกี่ยวกับการจัดการการชำระเงินสกุลเงินยูโร

EPC ซึ่งประกอบด้วยธนาคารในยุโรป พัฒนากรอบงานเชิงพาณิชย์ และทางเทคนิคสำหรับเครื่องมือการชำระเงิน SEPA การชำระเงิน SEPA มีสามชนิด:

-   การโอนย้ายเครดิต
-   การหักบัญชีเงินฝากอัตโนมัติ
-   บัตร

## <a name="what-is-a-sepa-credit-transfer"></a>การโอนย้ายเครดิต SEPA คืออะไร
การโอนย้ายเครดิต SEPA คือการชำระเงินจากบริษัทหนึ่งหรือจากบุคคลหนึ่ง ให้แก่บริษัทอีกแห่งหนึ่งหรือบุคคลหนึ่ง การชำระเงินต้องเป็นสกุลเงินยูโร และทั้งสองฝ่ายต้องมีหมายเลขบัญชีธนาคารระหว่างประเทศ (IBAN) และรหัสระบุธนาคาร (BIC) (หรือที่เรียกอีกอย่างว่า BIC มีรหัสสมาคมเพื่อการสื่อสารทางการเงินระหว่างธนาคารทั่วโลก\[SWIFT\]รหัส) นอกจากนี้ ต้นทุนของธุรกรรมต้องถูกแบ่งกันระหว่างทั้งสองฝ่าย การโอนย้ายเครดิตที่เกิดขึ้นระหว่างฝ่ายควรใช้ไฟล์ XML ที่สอดคล้องกับมาตรฐานการประมวลผลการชำระเงิน 20022 ISO และรูปแบบ XML ตามที่ระบุไว้ใน EPC

## <a name="how-is-a-credit-transfer-implemented"></a>วิธีการโอนย้ายเครดิตใช้ได้อย่างไร
รูปแบบการชำระเงินโอนย้ายเครดิตสำหรับประเทศต่าง ๆ ในยุโรปถูกนำมาใช้ โดยใช้รายงานทางอิเล็กทรอนิกส์ (ER) และเมธอดของฟังก์ชันการชำระเงินใน Dynamics 365 สำหรับการดำเนินงาน รูปแบบโอนย้ายเครดิตบางที่ใช้ในภูมิภาคอื่นยังคงใช้กรอบงานการชำระเงินแบบดั้งเดิม ระหว่างหลายรูปแบบอื่น ๆ มี 12 ISO 20022 เครดิตโอนย้ายรูปแบบไฟล์ที่พร้อมใช้งาน รูปแบบการส่งออกเหล่านี้สอดคล้องกับมาตรฐาน SEPA ISO 20022 XML พวกเขาจะใช้ในการสร้างการโอนย้ายการชำระเงินที่ไม่ใช่ยูโรสำหรับประเทศ/ภูมิภาคที่นำมาใช้และชำระเงินยูโรตามที่ระบุในรุ่น 8.2 SEPA เครดิตโอนย้ายโครงร่าง Rulebook ที่ EPC ออกจำหน่าย ก่อนที่คุณสามารถดำเนินการโอนย้ายเครดิต คุณต้องติดต่อธนาคารของคุณเพื่อขอรับซอฟต์แวร์ที่จำเป็นเพื่อให้การอัปโหลดแฟ้มระบบธนาคารอิเล็กทรอนิกส์ คุณจะใช้ซอฟต์แวร์นั้นเพื่อถ่ายโอนแฟ้ม XML ที่ประกอบด้วยใบสั่งการชำระเงินไปยังธนาคารของคุณ

## <a name="what-credit-transfer-formats-are-currently-supported-in-dynamics-365-for-operations"></a>ในขณะนี้สนับสนุนรูปแบบการโอนย้ายสินเชื่อใดใน 365 Dynamics สำหรับการดำเนินงาน
คุณควรเสมอไปยังไลบรารีสินทรัพย์ใช้ร่วมกันบนบริการ Microsoft Dynamics วัฏจักร (LCS) และดูรายการล่าสุดของแฟ้มที่พร้อมใช้งานที่มีชนิดเป็นสินทรัพย์**ตั้งค่าคอนฟิกการจัดการ** อยู่ถัดจาก นั้น "อะไรบ้างที่ฉันมีการตั้งค่า" มีการเชื่อมโยงไปยังหัวข้อที่อธิบายถึงวิธีการสร้างที่เก็บตัว LCS เพื่อตรวจทานการตั้งค่าคอนฟิกที่มีอยู่ และนำเข้าการตั้งค่าคอนฟิกที่เลือก ไว้

## <a name="what-do-i-have-to-set-up"></a>ฉันจำเป็นต้องตั้งค่าอะไรบ้าง
-   ก่อนที่คุณสามารถสร้างการโอนย้ายเครดิตไฟล์ ตั้งค่าคอนฟิกการโอนย้ายเครดิตที่ใช้งานอยู่อย่างน้อยหนึ่งต้องนำเข้าไว้ในการตั้งค่าคอนฟิกของคุณ ER สำหรับคำแนะนำ[อิเล็กทรอนิกส์ดาวน์โหลดที่รายงานการตั้งค่าคอนฟิกจากบริการวัฏจักร](https://docs.microsoft.com/en-us/dynamics365/operations/dev-itpro/analytics/download-electronic-reporting-configuration-lcs)
-   เมื่อคุณกำหนดค่าวิธีการชำระเงินบัญชีเจ้าหนี้ เลือก**การรายงานทางอิเล็กทรอนิกส์ทั่วไป**กล่องกาเครื่องหมาย และเลือกรูปแบบการโอนย้ายเครดิตที่เหมาะสม (ตัวอย่างเช่น**ISO 20022 เครดิตโอนย้าย (AT)**) เป็นการตั้งค่าคอนฟิกรูปแบบการส่งออก
-   นอกจากนี้คุณต้องตั้งค่ารายละเอียดทางกฎหมายของเอนทิตีและบัญชีธนาคารใน 365 Dynamics สำหรับการดำเนินงาน
-   หมายเลขบัญชีธนาคาร IBANs และบางครั้ง SWIFT รหัส (BICs) หรือรหัสอื่น ๆ จำเป็นในการสร้างการชำระเงินการโอนย้ายเครดิตที่ถูกต้อง ดังนั้น คุณต้องตั้งค่าเหล่านี้สำหรับบัญชีธนาคารของผู้ขายและบัญชีธนาคารสำหรับองค์กรที่กำลังร้องขอการโอนย้าย
-   ข้อมูลเพิ่มเติมอาจต้องใช้ เช่นหมายเลขภาษีมูลค่าเพิ่ม (VAT) สำหรับฝ่ายที่ถูกอ้างถึงในข้อความโอนย้ายเครดิต ข้อมูลนี้ต้องตั้งค่า สำหรับผู้จัดจำหน่าย และนิติบุคคลเมื่อมีการร้องขอ
-   บางวิธีการชำระบัญชีการชำระเงิน ส่วนใหญ่แล้วตาม ISO 20022 – วิธีการชำระเงิน คุณอาจต้องการตั้งค่าเพิ่มเติมสำหรับ**ชุดรูปแบบการชำระเงิน**เช่น**บริการชนิด** = **SLEV**ได้ รหัสดังกล่าวจะใช้เป็นการเพิ่มเติมแท็กสำหรับธุรกรรมการชำระเงินในระหว่างการประมวลผลการชำระเงิน ค่าของรหัสการชำระเงิน เริ่มต้นต้อง**วัตถุประสงค์ประเภท**,**แบเรอร์ค่าธรรมเนียม**,**ท้องถิ่นตราสาร**และ**บริการระดับ**สามารถตั้งสองแห่งได้ สถานแรกคือ**หัวข้อสมุดรายวันการชำระเงินเจ้าหนี้บัญชี**และที่สองคือ**บัญชีวิธีเจ้าหนี้สำหรับการชำระเงิน** เมื่อ มีการสร้างบรรทัดสมุดรายวันการชำระเงิน ค่ารหัสการชำระเงินที่ตั้งในหัวข้อสมุดรายวันการชำระเงินจะถูกโอนย้ายไปยังบรรทัดสมุดรายวัน ถ้า ไม่มีตั้ง ใช้ค่าจากวิธีการชำระเงิน

## <a name="what-parameters-are-available-for-generating-credit-transfer-payments"></a>พารามิเตอร์ใดจะพร้อมใช้งานสำหรับการสร้างการชำระเงินการโอนย้ายเครดิตหรือไม่

## <a name="what-are-ibans-and-bics"></a>IBANs และ BICs คืออะไร
หมายเลขบัญชีธนาคารระหว่างประเทศ (IBAN) และรหัสระบุธนาคาร (BIC) จะใช้เพื่อระบุบัญชีใด ๆ ในหลายประเทศ/ภูมิภาคทั่วโลก ได้แก่ 34 SEPA ประเทศ/ภูมิภาค ป้อน BIC ในการ**รหัส SWIFT**ฟิลด์และ IBAN ในการ**IBAN**ฟิลด์ สำหรับทั้งเจ้าหนี้ของบัญชีธนาคารและบัญชีธนาคารของลูกค้า เขตข้อมูลเหล่านี้จะอยู่บน**รหัสเพิ่มเติม**ด่วนบน**บัญชีธนาคาร**แท็บของการ**บัญชีธนาคาร**หน้า ไม่มีบังคับใช้สำหรับการชำระเงิน SEPA BIC

## <a name="how-do-i-transmit-a-payment-file-to-the-bank"></a>ฉันจะส่งไฟล์การชำระเงินไปธนาคารได้โดยวิธีใด
เมื่อคุณสร้างการชำระเงิน มีสร้างไฟล์การชำระเงิน และโปรแกรมจะถามเพื่อบันทึกจากเว็บเบราว์เซอร์ของคุณไปยังตำแหน่งใด ๆ พร้อมใช้งาน ขั้นตอนถัดไปคือการ ส่งไฟล์ XML ไปยังธนาคารของคุณ กระบวนการนี้แตกต่างกันไปในแต่ละธนาคาร ทำตามคำแนะนำจากธนาคารของคุณเพื่อส่งไฟล์ไปยังธนาคารสำหรับการประมวลผล
